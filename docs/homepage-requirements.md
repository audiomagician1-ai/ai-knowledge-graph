# 知识领域选择页 — 功能需求文档

> 参考交互：Apple Watch Home Screen（透镜 + 无限画布模型）
> 更新日期：2026-03-21

---

## 1. 设计理念

当前实现将所有 37 个领域一次性全部展示在屏幕内，导致：
- 信息密度过高，缺乏焦点
- 无法扩展——新增领域会撑破布局
- 拖拽交互无意义（所有内容已经可见）

**目标模型**：参考 Apple Watch Home Screen，改为 **「透镜浏览 + 无限画布」**：
- 画布远大于屏幕，只展示局部
- 屏幕中心是"焦点透镜"，放大显示详情
- 四周气泡逐渐缩小，营造球面深度感
- 拖拽让不同领域滑入焦点区，探索感十足

## 2. 参考图关键特征提取

| # | 特征 | Apple Watch 表现 | 需要实现的效果 |
|---|------|-----------------|--------------|
| 1 | 局部可见 | 屏幕只显示约 15-20 个 icon，大量在屏幕外 | 画布 hex 网格远大于视口，初始可见 ~15-25 个气泡 |
| 2 | 屏幕中心 Fisheye | 放大效果锚定在屏幕物理中心，不随拖拽移动 | fisheye 基于气泡到**屏幕中心**的像素距离 |
| 3 | 极端尺寸差 | 中心 icon 约为边缘的 3-4 倍大 | 中心 r≈48-56px，边缘 r≈12-16px |
| 4 | 边缘裁切感 | 圆形表盘裁切，icon 被切掉一半消失 | 用椭圆 vignette 渐变遮罩模拟柔和裁切 |
| 5 | 信息递进 | 大 icon 显示图标+名字，小的只是彩色点 | 大气泡：名字+统计；中气泡：仅名字；小气泡：纯色球 |
| 6 | 惯性+回弹 | 快速划动有惯性漂移，边界弹性回弹 | 惯性滑动 + 画布边界 rubber band 回弹 |

## 3. 核心功能需求

### F1：无限画布 + 局部渲染

- **画布模型**：Hex 网格自动根据领域数量生成足够多的环（如 50 个领域 = 5 环，100 个 = 6 环…）
- **初始视口**：只有靠近屏幕中心的 ~15-25 个气泡可见（取决于缩放）
- **Off-screen 裁切**：完全离开视口的气泡跳过渲染（性能保障）
- **扩展性**：支持 50-200+ 个领域，无需手动调整布局参数

### F2：屏幕中心固定 Fisheye

- **锚点**：Fisheye 计算基于气泡**在屏幕上的像素坐标**到屏幕中心的距离，不是世界坐标
- **缩放曲线**：平滑连续函数，建议 `scale = minScale + (maxScale - minScale) * (1 - d/R)^power`
  - `maxScale` ≈ 1.6-1.8（中心气泡放大到 ~52px）
  - `minScale` ≈ 0.35-0.45（边缘缩到 ~14px）
  - `R` = 屏幕对角线的 ~60%
  - `power` ≈ 2（二次缓出，中心区域平坦，边缘急剧缩小）
- **位置挤压**：fisheye 同时微调气泡位置，中心区域间距略大，边缘更紧密（Apple Watch 特征）

### F3：信息递进显示（Progressive Disclosure）

根据气泡当前半径决定显示内容：

| 半径范围 | 显示内容 | 示例 |
|---------|---------|------|
| r ≥ 40px | 名称 + 统计信息（概念数/子领域数） | "AI编程" + "400 知识点 · 15 子领域" |
| 28px ≤ r < 40px | 仅名称 | "AI编程" |
| 16px ≤ r < 28px | 仅彩色球体（无文字） | ⚫ |
| r < 16px | 极小彩色点或不渲染 | · |

### F4：拖拽平移 + 惯性 + 回弹

- **拖拽平移**：鼠标/触摸拖拽移动画布，让不同领域滑入中心焦点
- **惯性漂移**：松手后根据速度惯性滑动，摩擦衰减
- **边界回弹**：滑到网格边缘时有弹性回弹效果（rubber band），不允许无限拖出空白区
- **速度上限**：防止飞太远

### F5：椭圆 Vignette 遮罩

- 在视口四周绘制柔和的椭圆渐变遮罩（背景色 → 透明）
- 效果：气泡不是"硬边切断"，而是逐渐融入背景消失
- 遮罩形状：水平椭圆（适配宽屏），中心完全透明，边缘 ~80px 渐变到背景色
- 营造"球面透镜"的深度感

### F6：Hover 交互

- 鼠标悬停在气泡上：
  - 该气泡额外放大 ~15%
  - 显示浮动 tooltip（领域名称 + 简要统计）
  - 光标变为 pointer
- 悬停状态不影响其他气泡的 fisheye 计算

### F7：点击进入

- 点击气泡进入该领域的 3D 知识图谱页
- 点击时播放放大过渡动画（气泡扩展填满屏幕）
- 区分点击 vs 拖拽（移动距离 > 6px 视为拖拽）

## 4. 视觉设计规范

### 4.1 配色

- 背景色：`#f0f0ec`（暖灰）
- 气泡配色：统一的柔和粉彩色系（HSL 饱和度 40-55%，明度 60-72%）
- 禁止使用：纯红、纯黄、荧光绿、棕色、橄榄色等高对比/浊色
- 气泡渲染：带径向渐变的 3D 球体效果（高光 + 阴影）

### 4.2 气泡尺寸参数

| 参数 | 值 | 说明 |
|------|-----|------|
| BASE_R | 32px | 基准半径（fisheye scale=1.0 时） |
| HEX_SPACING | 88px | hex 网格中心间距 |
| FISHEYE_MAX | 1.7 | 中心最大缩放 |
| FISHEYE_MIN | 0.4 | 边缘最小缩放 |
| FISHEYE_RADIUS | 屏幕对角线 60% | fisheye 影响半径 |

### 4.3 文字

- 领域名称：白色，12-14px，居中
- 统计信息：白色半透明，9-10px，名称下方
- 仅在气泡足够大时显示（见 F3 信息递进）

---

## 5. 技术约束

- 渲染方式：Canvas 2D（当前方案，性能足够）
- 帧率目标：60fps（requestAnimationFrame）
- DPR 适配：支持 Retina（devicePixelRatio capped at 2）
- 无外部依赖：纯 React + Canvas，不引入 Three.js/PixiJS

---

## 6. 与当前实现的主要差异

| 维度 | 当前实现 | 目标实现 |
|------|---------|---------|
| 可见数量 | 全部 37 个 | 初始 ~15-25 个，其余在视口外 |
| Fisheye 锚点 | 跟随 pan 偏移（世界坐标） | 固定在屏幕中心（屏幕坐标） |
| 缩放幅度 | 1.0x - 1.3x（几乎没差） | 0.4x - 1.7x（4 倍差异） |
| 文字显示 | 所有气泡都显示名字 | 按大小递进：大=全信息，中=名字，小=无 |
| 边缘处理 | alpha fade 淡出 | 椭圆 vignette 遮罩 + 尺寸缩小 |
| 拖拽意义 | 无（已全部可见） | 核心交互：探索不同领域 |
| 扩展性 | 固定 37 个 | 支持 N 个（50-200+） |

## 7. 实施优先级

1. **P0**：修改 Fisheye 为屏幕坐标锚定 + 加大缩放幅度 → 立刻产生"透镜"效果
2. **P0**：信息递进显示 → 解决信息过载
3. **P0**：椭圆 vignette 遮罩 → 营造深度感
4. **P1**：边界回弹 → 完善拖拽手感
5. **P2**：增加更多 demo 领域（50-60 个）→ 验证扩展性
