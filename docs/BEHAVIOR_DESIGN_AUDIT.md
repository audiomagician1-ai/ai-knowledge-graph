# 行为设计审计报告：AI 知识图谱（AI Knowledge Graph）

> **审计日期**：2026-04-06
> **审计方法**：基于福格行为模型（B=MAP）+ 自我决定理论（SDT）+ Kahneman 双系统 + 行为经济学偏差 + JTBD 多透镜诊断
> **审计工具**：behavior-design-audit skill v2
> **审计范围**：全链路（首页 → 选域 → 图谱 → 学习对话 → 评估 → 回访留存）

---

## 审计范围

- **产品**：AI 知识图谱 — 通过"教 AI"来学会知识的平台，结合费曼学习法、苏格拉底式对话、3D 知识图谱技能树点亮
- **目标用户**：自学者、大学生/研究生、终身学习者、需要快速掌握新领域的专业人士
- **核心目标行为**：
  - **首要**：完成一次学习对话并获得评估（GreenDot → GreenPath）
  - **次要**：每天回来做间隔复习（GreenPath）
  - **远期**：持续探索新知识域，点亮技能树（PurplePath）
- **行为类型**：GreenPath — 需要养成长期学习习惯
- **当前状态**：V1.0 feature-complete，无线上用户数据（分析基于产品结构和代码）

## 关键发现（按严重度）

### 首页体验诊断（步骤①②③）

**MAP 诊断：**
- **M 动机 🟡**：用户带着学习意图来，但首页**没有强化动机的设计**。标题"选择你的知识领域"是指令，不是价值承诺
- **A 能力 🔴**：脑力瓶颈严重——60+域 = 决策瘫痪。用户需要同时理解产品概念、域的含义、做出选择。非常规性瓶颈——鱼眼拖拽交互是非标准模式
- **P 提示 🟡错配**：彩色气泡是 Spark 形式，但用户状态是"动机中等+能力低"，需要 Facilitator

**认知架构诊断：** 强迫 System 2 过早激活。产品概念不明、选择过多、交互模式陌生、缺乏预期设定。

**决策设计诊断：** 锚定效应❌、社会证明❌、默认效应❌、禀赋效应❌——几乎所有决策杠杆都缺失。

**JTBD 诊断：** 用户 Job 是"我想学会某个知识"，产品回应是"这里有60个领域自己选"——没有帮缩短路径。

### 学习对话体验（步骤④⑤⑥）— 产品的护城河

**MAP 全🟢**：选项按钮降低门槛、AI 主动讲解体验好、"4轮后可评估"是清晰 Signal。

**关键隐患**：Aha Moment 延迟 — 产品真正价值需要 4+轮对话(5-10分钟)才显现。

**SDT**：自主性🟢、胜任感🟡（评估卡片好但两次评估间缺进步信号）、关联性🔴（完全孤岛体验）。

### 回访与留存（步骤⑦⑧）

**MAP-P 🔴 缺失**：没有任何提示引导用户回来。无推送、无邮件、无 PWA 徽章。FSRS 复习系统入口深藏。

**决策设计缺失**：损失厌恶❌、沉没成本❌（积累不可见）、峰终定律🟡（评估后终点冰冷——只有两个按钮）。

## 优化建议

### P0 — 止血

#### 建议 1：增加首屏价值主张与引导层
- **诊断来源**：认知架构 — 产品概念不明 + JTBD 功能性任务错配
- **行为科学原理**：S1/S2 理论 — 先用 S1 传达价值再让用户决策；框架效应 — 用户语言而非产品语言
- **方案**：首次访问时叠加引导卡片：① 一句话价值主张 ② 30秒体验预览 ③ 明确 CTA
- **实施难度**：低（纯前端，1-2天）
- **验证**：首页→域页面转化率；首页停留时间

#### 建议 2：首页添加推荐锚点 + 减少认知负荷
- **诊断来源**：MAP-A + 锚定效应 + 选择过载
- **方案**：鱼眼中心默认展示 3-5 个精选域标注"✨推荐"；增加热度标签；增加搜索入口
- **实施难度**：低
- **验证**：选域转化率；选择集中度

#### 建议 3：构建回访提示体系
- **诊断来源**：MAP-P 提示缺失
- **行为科学原理**：Fogg Prompt — 无提示就无行为；损失厌恶 — 损失框架比获得框架有效 2x
- **方案**：
  - 短期：学习完成页增加"明日复习提醒"；首页顶部增加"今日待复习 X 个"横幅
  - 中期：Web Push Notification；登录用户邮件摘要
- **实施难度**：短期=低（2-3天），中期=中（1-2周）
- **验证**：次日回访率；FSRS 复习完成率

### P1 — 高杠杆

#### 建议 4：缩短 Aha Moment 路径——"快速检测"模式
- **诊断来源**：MAP 行为链 — Aha Moment 延迟 + 峰终定律
- **方案**：新增"快速检测"模式，2-3 个选择题 1 分钟内完成迷你评估，然后引导进入完整对话
- **实施难度**：中
- **验证**：首次完成评估比例；新用户首次 session 时长

#### 建议 5：让进度和积累可见——"我的知识宇宙"
- **诊断来源**：SDT 胜任感 + 沉没成本 + 宜家效应
- **方案**：首页增加个人进度摘要条；域气泡增加完成度环；评估后增加"版图扩大"动效
- **实施难度**：中
- **验证**：周留存率；人均掌握概念数

#### 建议 6：评估后体验终点重设计
- **诊断来源**：峰终定律 — 终点冰冷
- **方案**：达标时全屏庆祝动效；未达标时温暖鼓励+一键深入；所有情况增加"推荐下一个"+"设置明日提醒"+"连续学习天数"
- **实施难度**：低-中
- **验证**：评估后→下一个概念转化率

### P2 — 体系优化

#### 建议 7：轻量社交层——"同伴在学"
- **诊断来源**：SDT 关联性 + 社会证明
- **方案**：Level1 标注"正在学·X人"；Level2 "今天X人和你学了同一概念"；Level3 排行榜
- **实施难度**：Level1=低，Level2-3=中

#### 建议 8：FSRS 复习前置化
- **诊断来源**：MAP-P + 功能公墓
- **方案**：首页醒目横幅"📖 今日复习·X个概念到期"；域图谱优先弹出复习提示
- **实施难度**：低

## 核心结论

AI 知识图谱的**核心学习体验（对话引擎+评估系统）质量很高**，是产品真正的护城河。问题集中在核心体验的**两头**：

1. **入口端**（首页→开始学习）：用户看不懂、选不了、走不进来 —"好产品被坏门面耽误"
2. **出口端**（完成学习→明天回来）：没有提示、没有种子、没有钩子 — 用户走了找不回来

**最高杠杆的 3 件事**（如果只能做 3 件）：
1. **首页加引导层+推荐锚点**（P0，2-3天，预计首页转化率 +50-80%）
2. **构建回访提示体系**（P0，3-5天，预计次日留存 +200-400%）
3. **评估后体验终点重设计**（P1，2天，让每次学习都种下"明天回来"的种子）

这三件事共同特点：**实施成本低、行为科学依据明确、对核心指标影响最大。**
