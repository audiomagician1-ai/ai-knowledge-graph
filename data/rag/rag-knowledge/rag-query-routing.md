# RAG查询路由

## 核心概念

RAG查询路由是根据用户查询的意图和特征，将其导向最合适的数据源或检索策略的机制。不同类型的查询可能需要不同的处理方式——简单事实查询走向量检索，复杂分析查询走数据库，最新信息查询走实时搜索。

## 路由策略

### 1. 基于LLM的路由
使用LLM分析查询意图并选择路由：
```
[用户查询] → [LLM分类] → 选择: 向量库 / SQL数据库 / 搜索API / 直接回答
```
优点：灵活，能理解复杂意图
缺点：延迟增加，成本上升

### 2. 基于分类器的路由
训练专用分类器进行路由决策：
- 使用嵌入模型计算查询向量
- 通过分类器（SVM、KNN等）判断路由
- 延迟低，适合高吞吐场景

### 3. 基于规则的路由
预定义匹配规则：
- 关键词匹配（如"最新"→实时搜索）
- 正则表达式模式
- 实体类型识别

## 多数据源路由

大型RAG系统通常有多个数据源：
- **向量数据库**: 语义相似性搜索
- **关系数据库**: 结构化数据查询
- **全文搜索引擎**: 关键词精确匹配
- **知识图谱**: 实体关系查询
- **实时API**: 最新数据获取

路由器根据查询特征选择一个或多个数据源。

## 多步路由

复杂查询可能需要分步处理：
1. 分解原始查询为子查询
2. 每个子查询独立路由
3. 聚合各数据源返回的结果
4. 合成最终回答

## 路由评估

衡量路由质量的指标：
- **准确率**: 查询被路由到正确数据源的比例
- **延迟**: 路由决策的耗时
- **回答质量**: 最终回答的准确性和完整性

## 与RAG Pipeline的关系

查询路由是RAG管道（Pipeline）的前置组件。在标准RAG流程中，查询路由决定了检索阶段使用哪个策略和数据源。
