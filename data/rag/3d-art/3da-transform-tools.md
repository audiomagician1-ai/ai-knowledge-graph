# 变换工具

## 概述

变换工具（Transform Tools）是3D软件中用于改变物体位置、朝向和大小的一组基础操作集合，具体包含移动（Grab/Move）、旋转（Rotate）和缩放（Scale）三种核心变换类型。这三种操作在Blender中对应的快捷键分别是 **G**、**R**、**S**，在Maya中对应 **W**、**E**、**R**，是所有3D建模工作的最基本操作层级。

变换工具的数学本质是线性代数中的仿射变换（Affine Transformation）。每个3D物体在软件内部维护着一个4×4的变换矩阵，记录其相对于世界坐标系的位置、旋转和缩放数据。3D软件在1980年代商业化普及时，就已将这三类变换封装为独立工具栏，成为行业通用操作范式。

理解变换工具对于3D建模的重要性在于：哪怕是一个简单的布景摆放任务，错误理解轴向约束或轴心点位置，都会导致物体出现在完全错误的坐标位置，或以意外角度旋转。新手最常见的"模型消失"问题，有约60%是因为误操作变换工具、将物体移出视图范围导致的。

## 核心原理

### 移动（Move/Grab）

移动操作将物体从当前坐标位置平移到新坐标位置，变化量体现在变换矩阵的位移向量 **T = (tx, ty, tz)** 上。在Blender中按 **G** 激活移动后，可立即输入轴向字母限制移动方向，例如 **G → Z** 代表只沿Z轴移动，**G → Shift+Z** 则代表在X、Y平面内移动（排除Z轴）。移动量支持直接键入数值，如 **G → X → 2** 表示沿X轴移动2个单位。

位移的参考系有**世界坐标系（Global）**和**本地坐标系（Local）**之分。当一个物体已经被旋转过45°后，其本地Y轴方向已不再与世界Y轴对齐，此时选择不同坐标系执行移动操作，结果会截然不同。Blender中切换方式是在激活G后，按两次轴向字母（如 **G → Y → Y**）即切换为本地坐标系约束。

### 旋转（Rotate）

旋转操作围绕某一轴线转动物体，旋转角度以度（°）为单位存储。在Blender中按 **R** 激活后，默认围绕视图垂直方向旋转；若按 **R → X → 45**，则精确地将物体绕X轴旋转45°。3D软件中旋转的顺序对结果有直接影响：先绕X轴转90°再绕Z轴转90°，与先绕Z轴转90°再绕X轴转90°，最终朝向**不相同**，这是四元数（Quaternion）与欧拉角（Euler Angles）旋转表示方式产生差异的根源。

Blender默认使用欧拉角存储旋转数据，用户在属性面板中看到的 **X/Y/Z Rotation** 三个数值即为欧拉角的三个分量。当出现"万向节死锁"（Gimbal Lock）时，软件才会建议切换为四元数模式。

### 缩放（Scale）

缩放操作改变物体的大小，通过将变换矩阵中缩放向量 **S = (sx, sy, sz)** 乘以目标比例系数实现。在Blender中按 **S → 2** 代表均匀放大2倍，**S → X → 0.5** 代表仅在X轴方向压缩至原来的一半。

**非均匀缩放（Non-Uniform Scale）是建模中的常见陷阱**：当一个物体在X、Y、Z三轴上的缩放值不相等时（如 Scale = 2, 1, 0.5），该物体若进行细分或布尔运算，往往出现扭曲变形。这是因为子级物体或修改器会继承未应用的缩放数据。正确做法是在完成造型后，执行"应用变换"（Apply Scale，Blender中快捷键为 **Ctrl+A → Scale**）将缩放值归一化为 (1, 1, 1)。

### 轴心点（Pivot Point）

轴心点决定了旋转和缩放操作围绕哪个中心点执行，是变换工具中最容易被忽视的参数。Blender提供5种轴心点模式：

- **原点（Origin Point）**：围绕物体自身的原点操作
- **3D游标（3D Cursor）**：围绕游标当前位置操作
- **各自的原点（Individual Origins）**：多选物体时，每个物体各自围绕自己的原点独立变换
- **边界框中心（Bounding Box Center）**：围绕选中内容的包围盒中心
- **质心（Median Point）**：围绕选中内容的几何中心

以缩放操作为例：同样是将两个相距10个单位的球体缩放为0.5倍，使用"各自的原点"模式时，两球体体积各自减半但位置不变；使用"边界框中心"模式时，两球体体积减半且互相靠近，间距变为5个单位。

## 实际应用

**场景摆放**：在制作室内场景时，将桌子模型移动到精确坐标位置需要使用移动工具配合数值输入。例如将桌子放置在距离墙壁0.3米处，直接按 **G → Y → 0.3** 比用鼠标拖拽更精确。

**角色道具对齐**：为角色手部绑定武器时，常需要将武器的轴心点移动到握把中心，再以该点为轴心旋转，确保挥动动画的旋转中心正确。此操作直接依赖轴心点的精确设置。

**建筑模型构建**：在建造对称建筑时，将3D游标放置在建筑中轴线，切换轴心点为"3D游标"后执行旋转复制，可将左侧楼梯以中轴为圆心精确镜像到右侧，误差为零。

**应用变换的时机**：导出模型到游戏引擎（如Unity或Unreal Engine）之前，必须对所有物体执行"应用全部变换"（Apply All Transforms），否则引擎读取到的缩放值可能不为1，导致物理碰撞体积错误或动画异常。

## 常见误区

**误区一：认为缩放值不为1只是"外观问题"**
许多初学者认为只要物体看起来大小正确，Scale值是否为(1,1,1)无关紧要。实际上，当为该物体添加"厚度"修改器（Solidify Modifier）时，如果原始Scale = (2, 2, 2)，修改器计算出的厚度数值会以原始单位为基准，视觉上的厚度将是预期值的2倍。同理，导入游戏引擎后碰撞盒体积计算也会出现2³ = 8倍的误差。

**误区二：混淆物体的"原点"与"几何中心"**
物体的橙色圆点（原点/Origin）是变换操作的参考点，但它不一定位于物体的几何中心。在Blender中新建一个默认立方体时，原点默认在几何中心；但如果对该立方体进行过移动顶点操作后未更新原点，原点位置将与几何中心偏离。此时围绕"原点"旋转，视觉效果会像物体在绕一个悬空的点转动，这是大量新手困惑的来源。

**误区三：认为旋转角度可以随意叠加**
有学生认为"先旋转30°再旋转30°等同于直接旋转60°"。在只有单轴旋转的情况下，这是正确的。但若在X轴旋转30°后再在Y轴旋转30°，结果并不等于某单一旋转轴上的60°——三维旋转是非交换运算，旋转顺序改变会导致不同的最终朝向，属性面板中的欧拉角数值仅是当前旋转状态的一种表示方式，不能直接相加。

## 知识关联

**前置概念**：学习变换工具需要先理解3D建模的坐标系体系，即世界坐标系中X/Y/Z三轴的方向定义（在Blender中，Z轴默认为竖直向上方向，而在Maya中，Y轴默认为竖直向上方向），否则在执行轴向约束操作时无法预判结果。

**直接后续——编辑模式**：掌握物体级别（Object Mode）的变换工具后，进入编辑模式（Edit Mode）同样使用G/R/S快捷键，但此时操作对象变为顶点、边、面，变换基准从物体原点改为选中元素的质心，变换工具的底层逻辑完全延续，仅作用域发生变化。

**直接后续——轴心与原点**：变换工具中轴心点的5种模式仅是入门级使用，深入理解原点坐标的设置方法（如"将原点移动到游标"、"将原点移动到几何中心"）是制作复杂模型和绑定骨骼时的必要技能，直接建立在对轴心点概念的基础理解之上。
