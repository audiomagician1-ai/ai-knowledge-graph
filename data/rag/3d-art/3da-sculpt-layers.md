# 雕刻层

## 概述

雕刻层（Sculpt Layer）是数字雕刻软件中用于管理细节叠加的非破坏性工作机制。与图像编辑软件中的图层概念类似，雕刻层允许艺术家将不同的表面细节（如皮肤褶皱、划痕纹理、肌肉形变）分别存储在独立的层中，而不直接永久修改基础网格数据。每个雕刻层本质上储存的是顶点位移偏移量（vertex displacement offset），记录该层相对于下方层的法线方向位移差值。

雕刻层功能最早在 ZBrush 3.0（2007年发布）中以"图层（Layer）"工具形式得到系统化实现，Blender 则在 2.81 版本（2019年）通过多分辨率修改器正式支持雕刻层工作流。这一功能的出现解决了传统雕刻流程中"一旦落笔即永久改变网格"的问题，使角色细化、版本对比和客户修改反馈流程变得可逆且高效。

在影视和游戏角色制作流程中，雕刻层的价值体现在动态形变（blend shape）的准备阶段——艺术家可以在同一套多分辨率网格上，用不同的层分别制作"休息姿势""微笑表情""受伤状态"下的面部细节形变，最终导出时分别提取各层的位移数据。

## 核心原理

### 顶点位移的存储方式

每个雕刻层存储的不是完整的顶点坐标，而是**增量位移向量**（delta displacement vector）。设基础网格某顶点坐标为 $P_0$，第一层位移为 $\Delta_1$，第二层位移为 $\Delta_2$，则最终顶点位置为：

$$P_{final} = P_0 + w_1 \cdot \Delta_1 + w_2 \cdot \Delta_2$$

其中 $w_1$、$w_2$ 是各层的强度权重值，范围为 0.0 到 1.0。这意味着只要调整权重，细节的"浓淡程度"可以连续变化，而不需要重新雕刻。

### 层强度（Layer Intensity）与叠加顺序

层强度权重直接影响位移量的缩放比例。将某层权重从 1.0 降至 0.5，等同于将该层所有顶点的位移偏移量减半，而不改变位移方向。叠加顺序（stack order）在大多数软件中遵循"从下到上累加"的规则：位于堆叠底部的层先应用，上方的层在此基础上叠加偏移量。当两层对同一区域施加方向相反的位移时，结果是两个 $\Delta$ 向量的矢量相加，可能产生相互抵消或增强的效果，这一特性被用于制作"对抗性肌肉"细节。

### 层的混合与烘焙（Bake）

当多个层的效果确定后，艺术家可以将所选层"烘焙"（merge/bake）到基础网格或指定的目标层中。烘焙操作将所有参与层的 $\Delta$ 向量求和后，直接写入目标层或基础坐标，之后原始独立层数据被销毁。在 ZBrush 中，此操作通过 Layer 面板的 "Bake All" 按钮执行，执行后层数据不可恢复，因此通常在确认细节前保留原始文件备份。Blender 的多分辨率雕刻层则通过"应用修改器"步骤完成同等操作。

### 迭代修改与版本管理

雕刻层的最大优势是支持**非线性迭代**：可以随时返回单独一层（关闭其他层的显示）、修改该层内容，再重新组合。这使得应对"将眼袋加深 30% 但保持颧骨细节不变"此类修改需求时，无需重雕整个面部——只需定位到存储眼袋细节的层，调整权重或用雕刻笔刷局部修改该层的位移数据即可。

## 实际应用

**角色面部表情辅助雕刻**：在电影角色制作中，艺术家通常在"基础面部形态层"之上建立"疲惫皱纹层"和"微表情细纹层"，通过调整这两层的权重（例如分别设为 0.8 和 0.4），快速预览不同疲惫程度的角色状态，而无需重复雕刻。

**皮肤纹理的多级细节叠加**：在 ZBrush 的典型工作流中，艺术家会建立至少三个独立层：第一层存储大型皮肤面包状凸起（pores macro structure），第二层存储毛孔细节，第三层存储划痕或特定伤疤。导出时，三层的位移数据可以分别烘焙为三张 16 位 EXR 格式的位移贴图，供着色器分层驱动。

**布料形变细节**：服装雕刻中，"静止布料层"存储基础褶皱，"运动拉扯层"存储模拟布料受力后的额外皱褶。通过将运动层权重从 0 动画化到 1，可以直接在雕刻阶段预览布料形变过渡效果。

## 常见误区

**误区一：雕刻层等同于多分辨率细分层级**。多分辨率层级（subdivision level）控制网格的细分密度，决定可以雕刻的最大细节频率；雕刻层则在某一固定细分级别上存储位移偏移量。两者是正交的概念——同一细分级别可以有多个雕刻层，同一雕刻层也可以跨越多个细分级别进行编辑（取决于软件实现）。混淆二者会导致艺术家误以为"新建雕刻层"等于"提升细分精度"。

**误区二：层权重为 0 等于该层被删除**。将某层权重设为 0.0 仅是将该层的 $\Delta$ 向量乘以零，顶点位移数据依然完整保存在文件中，不占用额外渲染计算资源，但文件体积不变。只有执行显式的"删除层（Delete Layer）"操作才会真正清除该层的顶点偏移数据。

**误区三：烘焙多层后可以恢复单独层**。烘焙是不可逆的求和操作，一旦将五个层合并为一个基础层，原来各层的独立 $\Delta$ 数据无法从合并结果中分离还原。这是雕刻层工作流中最常见的导致"后悔操作"的误解，正确做法是烘焙前保存独立的 .ZPR 或 .blend 备份文件。

## 知识关联

雕刻层的有效使用以**多分辨率雕刻**为基础——只有在多分辨率修改器激活状态下，网格才具有足够的顶点密度来存储精细的 $\Delta$ 位移，雕刻层才能记录毛孔级别的细节。若在低于 Subdivision Level 3 的网格上新建雕刻层，每个层能存储的细节频率会受顶点数量严格限制，皮肤纹理等高频细节将出现锯齿化失真。

雕刻层产出的各层位移数据，是后续**位移贴图烘焙（Displacement Map Baking）**工作流的直接输入来源。每个独立雕刻层的 $\Delta$ 数据可以被单独烘焙为一张浮点位移贴图，在游戏引擎或离线渲染器中通过着色器节点重新组合叠加，实现与雕刻阶段完全一致的视觉效果——这正是雕刻层工作流连接雕刻制作阶段与技术美术管线的关键数据接口。
