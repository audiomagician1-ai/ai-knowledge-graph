# 手脚建模

## 概述

手部和脚部是人体建模中拓扑密度最高、解剖细节最复杂的区域。手部包含27块骨骼（8块腕骨、5块掌骨、14块指骨），脚部包含26块骨骼（7块跗骨、5块跖骨、14块趾骨），这种高密度骨骼结构直接决定了建模时需要为每个关节分配独立的环形循环（edge loop）来支持蒙皮变形，而非像躯干或大腿那样以大块面处理。

手脚建模的技术规范在2000年代初期随影视特效需求而系统化。Weta Digital在制作《指环王》（2001-2003）期间，为甘道夫、萨茹曼等角色手部特写镜头建立了一套内部拓扑规范，要求每根手指的近指间关节（PIP关节）区域保留至少两条平行环线以控制弯曲时皮肤的堆叠与压缩效果。这一规范后来成为影视级角色建模的行业参考标准，并被收录于Scott Spencer所著《ZBrush Digital Sculpting Human Anatomy》（Wiley, 2010）中。

手脚的建模质量对观看者的感知影响极为直接：人类视觉系统对手指比例失调、拇指弯曲方向错误或足弓消失的感知阈值极低，任何解剖误差都会触发"恐怖谷效应"（uncanny valley），使角色丧失可信度。正因如此，正确的手脚建模必须从骨骼层级出发规划几何体分布，而非从外轮廓雕刻逼近。

## 核心原理

### 手部骨骼解剖与拓扑对应关系

手部建模的几何起点是掌骨的扇形排列。五根掌骨并非平行——第一掌骨（拇指掌骨）与其余四根掌骨之间存在约40°~60°的张角，这个角度是拇指完成对掌运动（opposition）的解剖基础。建模时，拇指的腕掌关节（CMC关节）必须作为独立的旋转轴心处理；若错误地将其与食指掌骨基底放在同一水平线，拇指将无法完成抓握姿势的蒙皮变形。

指节比例遵循近似黄金比例的递减规律。以中指为例：近节指骨（PP）约45mm，中节指骨（MP）约28mm，远节指骨（DP）约21mm，三段之比接近 $1 : 0.62 : 0.47$，与黄金比例 $\phi \approx 1.618$ 的倒数序列高度吻合。在拓扑规划中，多边形边线应集中分配于关节处（每关节至少2~3条横向环），指节中段可适当减少环数。指甲区域需要独立的支撑环（support loop）来维持甲面平整与甲沟深度，游戏模型中通常以1~2条距甲缘1mm处的额外边线实现此效果。

四根手指（食指至小指）的掌指关节（MCP关节）还具有侧向展开能力，张开幅度约20°~25°。这意味着手掌背侧的拓扑必须允许手指基底区域有一定横向拉伸空间，在蒙皮权重设置中，MCP关节的权重影响区域需要延伸至掌骨中段，而非仅覆盖关节正上方的面片。

### 脚部骨骼解剖与足弓处理

脚部最容易被建模者忽略的解剖特征是三点支撑结构：跟骨（calcaneus）后缘、第一跖骨头和第五跖骨头共同构成三角形负重平面。正常直立时，内侧纵弓（medial longitudinal arch）离地高度约15~18mm，横弓（transverse arch）在第二至第四跖骨头下方形成轻微曲面。建模脚底时若将整个脚掌处理成完全平面，足弓特征消失，角色在行走动画落地帧会产生脚掌穿透地面的问题。

脚趾节数存在不对称性：大脚趾（第一趾）只有两节指骨（近节+远节），而第二至第五趾各有三节指骨，大脚趾宽度约为小趾的1.5~2倍。这意味着大脚趾的拓扑仅需一个IP关节环线组，而其余脚趾需要PIP和DIP两组关节环线。

脚踝区域存在一个关键的高度不对称：腓骨外踝（lateral malleolus）的最低点低于胫骨内踝（medial malleolus）约10~12mm。游戏角色的鞋模接缝线必须依据这一高度差分别定位于踝骨两侧，否则换鞋系统（swap shoe system）中鞋模与脚模的接缝边将在踝部产生明显错位。这一问题在《Assassin's Creed: Odyssey》（2018）的角色技术文档中被专门提及，是开放世界换装系统建模规范的经典参考案例。

### 拓扑流规划与关节蒙皮优化

手指每节横截面的多边形数量存在明确的行业分级标准：游戏实时渲染用途约8边截面（每手总面数控制在1500~2500个三角面），过场动画用途约12~16边截面，影视特效雕刻基础体约24~32边截面。

拓扑布线的核心规则是：**所有弯曲方向的边必须与弯曲轴垂直**。手指IP关节的弯曲轴为水平轴，因此沿指节走向分布的环线必须是横向闭合环，若边线呈斜向菱形分布，弯曲时将产生螺旋扭曲变形（twist artifact）。脚趾的DIP关节弯曲幅度较小（约30°~40°），但趾根MTP关节（跖趾关节）的背屈幅度可达70°~90°（行走蹬地阶段），因此MTP关节处的拓扑密度不低于手指MCP关节。

手掌的"一化五"分叉区域是整个手部拓扑中最复杂的部分。从手腕单一圆柱截面过渡到五根手指，需要在手掌中段插入极点（pole point），通常以4个五边形聚集区域作为过渡节点——这种布局被称为"掌面星形拓扑（palm star topology）"，在Autodesk Maya的Human IK模板与Blender的Rigify骨架模板中均有内置实现。

## 关键公式与拓扑计算

在规划游戏角色手部面数预算时，可使用以下简化公式估算手部三角面总数：

$$
F_{hand} = F_{palm} + 5 \times (n_{seg} \times r \times 2) + F_{nail} \times 5
$$

其中：
- $F_{palm}$：手掌+腕部面数（通常为300~500三角面）
- $n_{seg}$：单根手指的指节段数（拇指=2，其余四指=3）
- $r$：单段截面多边形边数（游戏常用 $r=8$）
- $F_{nail}$：单个指甲的面数（约20~40三角面）

以食指为例，$n_{seg}=3, r=8, F_{nail}=30$，则食指贡献约 $3 \times 8 \times 2 + 30 = 78$ 个三角面。五指合计（含手掌500面）约为 $500 + (78 + 78 + 78 + 78 + 58) + 150 \approx 1020$ 三角面，符合移动端游戏的手部面数预算要求。

以下是在Blender Python中批量检测手指环线数量是否满足最低要求（每关节2条）的辅助脚本示例：

```python
import bpy

def check_finger_loops(obj_name, min_loops_per_joint=2):
    """
    检测指定网格对象中，手指关节区域的边循环密度。
    要求每个关节顶点组至少有 min_loops_per_joint 条横向环线。
    """
    obj = bpy.data.objects.get(obj_name)
    if obj is None or obj.type != 'MESH':
        print(f"对象 {obj_name} 不存在或非网格类型")
        return

    # 获取所有以 "finger" 或 "toe" 命名的顶点组
    joint_groups = [vg for vg in obj.vertex_groups
                    if "finger" in vg.name.lower() or "toe" in vg.name.lower()]

    print(f"共检测到 {len(joint_groups)} 个手指/脚趾顶点组")
    for vg in joint_groups:
        # 统计该顶点组包含的顶点数量作为环线密度的近似指标
        verts_in_group = [
            v for v in obj.data.vertices
            if any(g.group == vg.index for g in v.groups)
        ]
        loop_count_approx = len(verts_in_group) // 8  # 以8边截面为基准
        status = "✓ 合格" if loop_count_approx >= min_loops_per_joint else "✗ 不足"
        print(f"  {vg.name}: 约 {loop_count_approx} 条环线 {status}")

check_finger_loops("Character_Hand", min_loops_per_joint=2)
```

## 实际应用

### 游戏角色手部的LOD分级策略

在实时渲染管线中，手部模型通常需要三级LOD（Level of Detail）：
- **LOD0（近景，距摄像机 < 3m）**：完整指甲面片、指节褶皱支撑环、8边截面，约1500~2500三角面
- **LOD1（中景，3~8m）**：移除指甲独立面片、合并指节中段环线，约600~900三角面
- **LOD2（远景，> 8m）**：手指退化为4边截面圆柱，约200~350三角面

脚部因通常被鞋子遮挡，LOD0标准可略低于手部，约800~1200三角面；但在赤脚角色（如《Ghost of Tsushima》中部分场景）中，脚部LOD0标准应与手部持平。

### 影视级手部雕刻的分层工作流

影视角色（如VFX合成或预渲染过场动画）的手部建模通常分四个阶段：
1. **基础体（Base mesh）**：以16~24边截面建立手指圆柱基础形，整体约5000~8000面
2. **中级雕刻（Mid-level sculpt）**：在ZBrush中推至约10万面，添加指节褶皱、掌纹分布区域
3. **高级雕刻（High-level sculpt）**：约50~100万面，添加皮肤毛孔（直径约0.2~0.5mm）、指纹纹路
4. **法线贴图烘焙（Normal map bake）**：将高模细节烘焙至低模，贴图分辨率通常为手部专用 2K 或与手臂共用 4K

## 常见误区

**误区1：将拇指与四指使用相同截面数建模**
拇指的CMC关节具有鞍形关节（saddle joint）结构，可完成内收、外展、屈曲和对掌四个自由度，其蒙皮权重的影响范围延伸至大鱼际肌（thenar eminence）区域，面积约为小鱼际肌的1.5倍。若使用与其他四指相同的简单圆柱截面处理拇指根部，大鱼际区域将在对掌动作中产生明显的几何塌陷。正确做法是在拇指掌骨基底单独分配8~12个独立顶点以容纳鱼际肌的体积变化。

**误区2：脚踝建模时外踝与内踝高度相同**
如前述，腓骨外踝比胫骨内踝低约10~12mm，且外踝位置偏后（约在脚踝横截面的后1/3处）。若将两侧踝骨建模在同一高度，穿鞋角色在踝关节内翻（inversion）动作时将产生外踝穿帮。

**误区3：手指IP关节布线使用斜向菱形网格**
部分建模者为追求面数均匀分布，在指节区域使用斜45°菱形网格。这种布线在静止姿势下外观正常，但在手指弯曲90°时会产生螺旋扭曲，在蒙皮权重涂抹阶段极难通过权重调整消除。必须在建模阶段保证关节处的环线与弯曲轴严格平行。

**误区4：
