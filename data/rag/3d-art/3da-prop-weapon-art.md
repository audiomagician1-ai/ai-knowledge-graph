# 武器美术管线

## 概述

武器美术管线（Weapon Art Pipeline）是指一件游戏武器道具从概念设计图纸到最终在引擎中实时渲染的完整制作流程，涵盖概念草图、高模雕刻、低模拓扑、UV展开、PBR纹理绘制、引擎导入与材质设置六个连续阶段。武器与普通场景道具不同，因为玩家视角中它长期处于屏幕中心区域（第一人称射击游戏的握枪位置），纹理分辨率通常分配到 **2048×2048** 甚至 **4096×4096**，而同等体积的场景摆件可能只需 **512×512**。

该管线的规范化起源于早期主机游戏（2000年代初 PlayStation 2 时代）对多边形预算的严格控制。当时一把主武器的低模面数上限仅约 **1500 tri**，迫使美术师在极低面数下保留剪影特征，由此形成了"高模烘焙→低模承载"的核心工作逻辑。进入 PBR 工作流的主流化（约2013年前后，Substance Designer 1.0 发布）后，武器管线增加了金属度/粗糙度贴图制作环节，使流程复杂度显著上升。

武器资产属于 **Hero资产**（高规格焦点道具），其管线严格程度高于普通道具，因为武器的金属高光、磨损边缘、机械接缝需要通过多张贴图协同表达，任何环节的疏漏都会在动态光照下暴露为视觉瑕疵。

## 核心原理

### 1. 概念阶段：三视图与材质注释

武器美术管线的起点是由概念设计师提交的 **正面/侧面/顶面三视图**，以及一份材质分区注释——例如标注哪部分是哑光喷漆钢铁、哪部分是橡胶握把、哪部分是镀铬金属。三视图的正交比例必须与最终模型的真实尺寸匹配；若一把步枪在世界坐标中应为 **90 cm × 8 cm**，建模师需在 Maya/Blender 中将参考图缩放至对应真实单位。缺少准确三视图会导致建模师反复对比截图，大幅拉长阶段耗时。

### 2. 高低模工作流：烘焙的本质

高模阶段使用 **ZBrush** 雕刻细节（螺丝凹槽、铸造纹理、划痕），多边形数量可达 **200万～500万 tri**；低模阶段在 Maya 或 Blender 中将同一形体压缩至 **5000～15000 tri**（现代主机标准）。两者之间通过 **法线贴图烘焙**（Normal Map Bake）建立视觉等价关系：用 Marmoset Toolbag 或 Substance Painter 的 Baker，将高模的曲面法线方向投影至低模的 UV 空间，生成一张 RGB 贴图，其中 R/G/B 三通道分别存储切线空间 X/Y/Z 法线偏移量。烘焙的 **Cage 偏移值**（Cage Offset）设置不当是该阶段最常见的错误来源，导致法线贴图出现黑色接缝。

### 3. UV 展开策略：武器的特殊分配原则

武器 UV 的核心挑战是在单张或双张 UV 集中最大化枪管、机匣等高曝光区域的 **纹素密度**（Texel Density）。行业通行标准是：一把 FPS 主武器的纹素密度目标值约为 **512 texels/m**（在 2048 分辨率贴图下），意味着机匣侧面分配的 UV 面积应远大于握把底部或枪托内侧。武器 UV 通常需要 **硬边切割**（Hard Edge Split）与 UV 缝对齐——凡是表面法线角度差超过 **90°** 的边，必须同时设为硬边和 UV 缝，以避免法线贴图在边缘产生梯度误差。

### 4. PBR 纹理制作：贴图通道的分工

现代武器纹理遵循 **金属度/粗糙度（Metallic/Roughness）PBR 工作流**，核心贴图包括：

| 贴图类型 | 存储内容 | 武器典型值 |
|----------|----------|------------|
| Base Color | 固有色，金属区域趋近于黑色 | 枪身钢铁区域 sRGB (40,40,40) |
| Metallic | 0=非金属，1=金属 | 钢铁部分=1，橡胶=0 |
| Roughness | 0=镜面，1=完全漫反射 | 磨损边缘≈0.2，喷漆表面≈0.6 |
| Normal | 法线偏移（由烘焙生成） | — |
| AO | 缝隙遮蔽信息 | 螺丝根部≈0.3 |

在 **Substance Painter** 中，武器材质通常从 Smart Material 的钢铁基底出发，通过 **Mask Editor** 的曲率（Curvature）通道自动在凸起棱边生成磨损高光，这是做旧与风化流程与纹理阶段的直接交接点。

### 5. 引擎导入与材质绑定

武器低模以 **.FBX** 格式导出，在 Unreal Engine 或 Unity 中导入时需确认：坐标轴方向（UE5 使用 Z-up 转换）、平滑组（Smoothing Groups）是否写入 FBX、LOD 层级设置。武器通常设置 **3-4 级 LOD**：LOD0 为原始面数，LOD1 削减约 40%，LOD2 削减约 70%，LOD3 仅保留外轮廓（用于极远距离或阴影投射）。引擎材质实例（Material Instance）应只暴露颜色叠加、磨损强度等少数参数，禁止在实例层修改着色器逻辑。

## 实际应用

**第一人称射击游戏主武器（以突击步枪为例）**：制作周期通常为 **3～5 周**，其中概念确认 3 天、高模雕刻 5 天、低模+UV 4 天、纹理绘制 7 天、引擎集成与 QA 修改 3 天。纹理集规格为 2048×2048，法线贴图与 Roughness/Metallic/AO 打包为 ORM 合并贴图（O=AO 存 R 通道，R=Roughness 存 G 通道，M=Metallic 存 B 通道），以节省 GPU 采样次数。

**换装/皮肤系统的武器管线变体**：当武器需要支持皮肤系统时，Base Color 贴图会拆分为"结构信息层"与"颜色叠加层"，颜色叠加通过引擎材质参数驱动，避免为每套皮肤重新制作完整纹理集，节省约 60% 的纹理显存占用。

## 常见误区

**误区一：认为高模越精细，最终效果越好**。高模细节能被法线贴图捕捉的前提是低模 UV 有足够分辨率容纳该细节。若螺丝凹槽直径对应的低模 UV 区域只有 **4×4 像素**，高模上再精细的螺纹也会在烘焙后模糊消失。正确做法是先确认低模 UV 的纹素密度，再决定高模细节的最小尺寸下限。

**误区二：PBR 工作流中 Base Color 可以包含光影信息**。这是从旧式 Diffuse 工作流迁移的美术师最常犯的错误。在 PBR 中，Base Color 贴图应只存储固有色，**不得包含任何烘焙光影或 AO 信息**，否则在引擎实时光照下会出现"双重阴影"现象——贴图中的预烘焙暗部与引擎计算的实时阴影叠加，导致武器表面异常发黑。

**误区三：认为 LOD 降面可以全自动完成**。引擎的自动 LOD 生成算法（如 UE5 的 Auto LOD）对武器的机械结构处理效果差，常在 LOD1 阶段就破坏枪口、准星等薄壁结构的形态。武器 LOD1 和 LOD2 通常需要美术师手动检查并修复这些关键剪影部位。

## 知识关联

武器美术管线直接建立在 **Hero资产** 规范之上：Hero资产要求高纹素密度、完整 PBR 贴图集和多级 LOD，武器管线正是 Hero资产规格在武器品类上的具体实施方案，包括面数预算、贴图分辨率和制作周期的具体数值均从 Hero资产标准中派生。

**做旧与风化**技术是武器纹理阶段的核心内容输入：边缘磨损（Edge Wear）、划痕叠加（Scratch Layer）、油污积聚（Grime Accumulation）在 Substance Painter 中通过程序化遮罩生成，这些技术若未在纹理阶段之前掌握，会导致武器表面呈现"全新出厂"状态，缺乏视觉重量感。武器管线将做旧技术整合进纹理制作的第四至第
