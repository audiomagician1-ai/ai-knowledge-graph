# 脊柱绑定

## 概述

脊柱绑定（Spine Rigging）是专门针对角色躯干脊椎骨骼结构所设计的绑定方案，核心目标是用少量控制器驱动多节脊椎骨骼，同时保持脊柱弯曲时的体积感和流畅曲线。人体脊柱通常由24节椎骨组成，而在3D角色绑定中一般只放置5到9根脊椎骨骼，但控制器数量会进一步精简至2到4个，以提高动画师操作效率。

脊柱绑定在3D动画产业早期主要依赖简单的FK链，动画师需要逐节旋转每根骨骼。2000年代以后，Maya等软件引入了Spline IK（样条IK）功能，允许用一条NURBS曲线的CV点来控制整条骨骼链的弯曲，大幅降低了脊柱动画的工作量。此后，Ribbon（丝带）方案逐渐流行，成为当前游戏与影视绑定中两种并行使用的主流脊柱方案。

脊柱绑定的质量直接影响角色弯腰、扭腰、侧倾等核心动作的表现力。一套设计不当的脊柱绑定会产生椎骨之间的体积塌陷、扭转角度丢失，或控制器层级混乱导致动画师无法分离腰部与胸腔的独立旋转，这些问题在格斗、舞蹈类动画中尤为致命。

## 核心原理

### Spline IK 样条曲线方案

Spline IK 使用一条 NURBS 曲线作为路径，骨骼链上的每根骨骼沿曲线自动排列和旋转。在 Maya 中执行 `Skeleton > Create Spline IK Handle` 时，软件会自动生成一条默认带有4个CV点的 NURBS 曲线，骨骼的位置由曲线形状决定，而骨骼的朝向则由 `Advanced Twist Controls` 参数中的 `World Up Type` 控制。

Spline IK 的关键参数是 **Twist** 和 **Roll**。Roll 控制整条链的整体滚动，Twist 通过起始点和末端点的角度差来分配每根骨骼的扭转量。若不启用高级扭转控制，脊柱旋转时会出现经典的"翻转（Flip）"问题——当骨骼方向接近万向锁临界角度时朝向突然翻转。解决方法是将 `World Up Type` 设置为 `Object Rotation Up (Start/End)`，分别绑定到胸腔控制器和骨盆控制器上，让扭转量沿骨骼链线性插值分配。

曲线 CV 点通常绑定到集群（Cluster）变形器上，再将集群约束到层级控制器。典型的三控制器脊柱会将CV点分别分配给：骨盆控制器（下方2个CV）、中腰控制器（中间1个CV）、胸腔控制器（上方2个CV），共5个CV驱动5节脊椎骨骼。

### Ribbon 丝带方案

Ribbon 方案不使用 IK 句柄，而是将一张细长的 NURBS 平面（宽度极窄，例如宽度为0.01、长度等于脊柱总长）作为驱动曲面，每根脊椎骨骼通过 `follicle`（毛囊节点）附着在曲面上，follicle 的 U 值均匀分布（例如5根骨骼的 U 值分别为0.1、0.3、0.5、0.7、0.9）。驱动骨骼蒙皮到 Ribbon 曲面上，再由控制器驱动驱动骨骼，实现对 Ribbon 的形变，从而带动 follicle 上的脊椎骨骼运动。

Ribbon 的核心优势在于**自动分段扭转**——曲面本身的法线方向会随着弯曲自然插值，不需要手动配置 Twist 参数，因此扭转结果更稳定。此外，follicle 还能提取曲面的法线方向信息，让每节椎骨始终垂直于脊柱曲线，而不是像 Spline IK 那样需要单独处理 Up Vector。

Ribbon 的缺点是计算节点较多，当骨骼数量超过10根时，场景中的 follicle 节点数量会显著增加，影响场景刷新速度。在实时游戏引擎（如 Unreal Engine）中，Ribbon 方案需要在导出前将其烘焙为标准骨骼动画，不能将 follicle 节点直接导出。

### 拉伸与体积保持

脊柱拉伸（Stretch）是脊柱绑定的重要扩展功能，允许角色在极度弯曲时脊柱总长度保持不变，避免椎骨之间出现间隙。Spline IK 方案实现拉伸需要测量 NURBS 曲线的实际弧长（`arclen` 节点），用原始长度除以当前弧长得到缩放比例，再将该比例驱动每根骨骼的 `Scale Y`（沿骨骼延伸轴）。

体积保持（Volume Preservation）公式为：若骨骼沿 Y 轴缩放比例为 `S`，则 X 和 Z 轴缩放比例应设为 `1/√S`，以保持骨骼体积恒定。这一公式常通过 Maya 的 `setDrivenKey` 或表达式节点实现驱动。

## 实际应用

**游戏角色脊柱绑定**：在面向实时引擎的角色中，脊柱通常只有3根骨骼（下脊、中脊、上脊），采用简化的 Spline IK 或纯 FK 方案，以便导出到 Unreal/Unity 时骨骼层级清晰。Ribbon 方案若用于游戏角色，需在发布版本中将曲面驱动烘焙掉，保留标准骨骼层级。

**影视角色脊柱绑定**：影视级别的角色脊柱骨骼数量常达到7至9节，控制器系统会同时保留 Spline IK 和 FK 两套控制通道，并通过 `blendColors` 节点实现 IK/FK 切换，动画师可以在关键帧精确控制时切换到 FK，在流畅连续动作时切换到 Spline IK。

**卡通角色夸张变形**：卡通绑定中脊柱常需要极大的拉伸（伸长比例可达200%以上），此时需为 Spline IK 曲线额外添加中间 CV 点（增加到7个CV），确保极端姿态下曲线依然平滑，不出现折角。

## 常见误区

**误区一：Spline IK 可以不处理扭转直接使用**
很多初学者创建 Spline IK 后发现脊柱弯曲正常，便认为扭转也会自动处理。实际上，默认状态下 Spline IK 的骨骼扭转是固定绑定在世界坐标的，当骨盆旋转时胸腔不会跟随扭转，导致腰部扭转动画完全失效。必须手动配置 `Advanced Twist Controls` 并指定起始和末端对象，才能让扭转沿脊柱正确传递。

**误区二：Ribbon 比 Spline IK 更适合所有脊柱场景**
Ribbon 在扭转稳定性上优于 Spline IK，但 Ribbon 曲面本身不具备全局 Roll 参数，整体躯干的滚动旋转必须通过额外的父级控制器叠加，而 Spline IK 的 Roll 属性可以直接实现全身滚动。在需要大量躯干翻滚动作的角色（如杂技、格斗类型）中，Spline IK 搭配正确的高级扭转配置反而更易控制。

**误区三：脊柱控制器越多操控越精细**
新手常为脊柱添加5个以上控制器，认为粒度越细越好。实际上，控制器过多会让动画师难以维护脊柱曲线的整体流畅感，因为每个控制器的独立操作可能破坏相邻段的曲率连续性（C1连续性）。专业绑定通常将主控制器限制在3至4个，再额外提供次级微调控制器（Sub-Control）供精修使用。

## 知识关联

脊柱绑定直接建立在 **IK 与 FK** 的基础知识之上——Spline IK 是 IK 系统的一种特殊变体，它将标准 IK 的末端效应器（End Effector）替换为 NURBS 曲线，使得整条骨骼链都受到约束而非仅末端。理解 FK 链的骨骼旋转层级是分析 Spline IK 扭转传递问题的前提，而 Ribbon 方案中驱动骨骼蒙皮曲面的部分也涉及 FK 骨骼对曲面的直接影响。
