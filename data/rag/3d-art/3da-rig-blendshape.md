# BlendShape / 变形目标

## 概述

BlendShape（在 Maya 中的名称）与 Morph Target（在 3ds Max、Blender 等软件中的通用称呼）是同一技术的不同命名，其核心原理是：在保持网格拓扑结构（顶点数量与连接关系）完全不变的前提下，通过对顶点位置进行加权插值，使一个基础网格平滑过渡到一个或多个预先雕刻好的形态目标。每个变形目标本质上存储的是一组顶点偏移向量 Δ**P**ᵢ，最终顶点位置由公式 **P** = **P_base** + Σ(wᵢ × Δ**P**ᵢ) 计算得出，其中 wᵢ 为每个目标的权重值，通常范围为 0.0（完全不激活）到 1.0（完全激活）。

该技术最早在 1990 年代的高端影视制作流程中普及，皮克斯等工作室在制作《玩具总动员》（1995 年）时已大量使用 Morph Target 驱动角色面部表情。与骨骼驱动面部相比，BlendShape 能够表达更自然的肌肉堆积、皮肤褶皱和脂肪挤压效果，因为这些形变是由艺术家直接雕刻完成的，而非依赖关节旋转的近似计算。

在当代游戏与影视制作中，BlendShape 是面部绑定的标配手段。虚幻引擎 5 的 MetaHuman 角色使用超过 100 个独立的 Morph Target 来构成完整的 FACS（面部动作编码系统）表情集，每个 Target 对应一块特定肌肉的运动形态，由此可见该技术在高精度面部表现中不可替代的地位。

## 核心原理

### 变形目标的创建流程

创建 BlendShape 的第一步是准备一个拓扑干净的基础网格（Base Mesh），这是所有变形目标的共同前提——每一个目标形态必须与基础网格拥有完全相同的顶点数量和顺序。通常做法是将基础网格复制多份，分别在雕刻软件（如 ZBrush）中进行形态调整，然后将修改后的网格作为变形目标导回绑定软件。

在 Maya 中，创建流程为：先选中所有目标形态网格，最后加选基础网格，执行 **变形 > BlendShape（Deform > BlendShape）**，软件便会自动计算每个目标相对于基础网格的顶点偏移量并将其存储在 BlendShape 节点中。建议每个目标只表达单一且独立的肌肉运动，例如"左眼眉上挑"或"上唇撇起"，而非同时包含多种动作，以便后续自由组合和精细调整。

### 权重驱动与组合方式

BlendShape 的权重驱动是绑定师的核心工作。在实践中，权重值允许超过 1.0 或低于 0.0，前者称为"过驱动"（Over-drive），会产生夸张变形效果，后者则会使顶点向偏移方向的反方向运动，常用于制作校正形态（Corrective Shape）。

多个 BlendShape 权重同时叠加时，由于各目标的偏移量是线性相加的，可能出现形态穿插或体积塌陷的问题。例如，嘴角同时向左和向右拉伸时，如果直接叠加可能导致嘴唇扭曲变形。解决方案是制作专门的**组合校正目标（Combination Shape）**：当且仅当两个特定目标同时激活时，通过额外的校正形态修补叠加误差。在 Maya 中可通过设置驱动关键帧（Set Driven Key）或使用乘法节点将两个权重相乘，作为校正目标的驱动条件。

### In-Between 中间形态

部分高级变形需要在权重从 0 到 1 的过程中经过一个特定的中间形态，而不是简单的线性插值。Maya 的 BlendShape 节点支持 **In-Between Target**，允许在权重值为 0.5 时强制经过一个预设的中间网格形态。例如，嘴唇闭合的动作在 50% 权重时需要经过唇部轻轻接触但仍有弧度的状态，而不是直接线性插值到完全闭合。In-Between 的权重激活值可以在 Maya 的 BlendShape 编辑器中精确设置为任意自定义数值，如 0.3 或 0.7。

## 实际应用

**游戏面部系统**：在 Unity 和虚幻引擎中，Morph Target 数据以 FBX 格式导入后可在运行时通过代码实时控制权重，常见于角色对话口型同步（Lipsync）和情绪系统驱动。虚幻引擎提供 `SetMorphTarget` 蓝图节点，接受目标名称字符串和 0.0-1.0 的浮点值参数，让程序员可以将语音分析数据直接映射到对应口型的 BlendShape 权重上。

**影视捕捉工作流**：在使用光学动作捕捉（如 Faceware）获取面部数据后，绑定师会将捕捉到的 FACS 动作单元数值（AU 编码，如 AU4 代表眉毛下压）映射到对应的 BlendShape 权重通道，从而将演员的真实表情重定向到 CG 角色脸上。这一工作流要求 BlendShape 库的命名规范严格对应 FACS 编码，否则自动化映射脚本无法正确识别。

**次世代角色优化**：出于性能考虑，移动平台游戏通常将单角色 Morph Target 数量控制在 20 个以内，而主机/PC 平台角色可使用 50-100 个。对于不可见区域（如头发遮盖下的头皮），绑定师会将这部分顶点权重归零或直接从 BlendShape 运算中剔除，以减少每帧的变形计算量。

## 常见误区

**误区一：认为 BlendShape 可以替代所有骨骼驱动**
BlendShape 只能处理预先雕刻好的固定形态，无法应对程序化或物理模拟驱动的动态变形。例如，随风飘动的头发、受重力影响的面部脂肪垂坠，或者被外部物体碰触后产生的实时凹陷，均无法用预录制的 Morph Target 表达。实际制作中，面部绑定通常是 BlendShape 与骨骼关节（用于大范围运动如下颌开合）协同工作，而非单独依赖其中一种。

**误区二：变形目标越多越好**
初学者常以为堆砌大量 BlendShape 能获得更细腻的表情。但每增加一个激活的 Morph Target，GPU 或 CPU 都需要额外遍历并累加所有受影响顶点的偏移向量。对于一个拥有 15,000 个顶点、激活 80 个目标的角色，每帧需要处理最多 1,200,000 次向量加法运算。因此，合理的策略是将高度相关的形态合并，并在不激活时从运算中跳过（权重恰好为 0 时许多引擎会自动跳过该 Target 的计算）。

**误区三：拓扑不同可以强行映射**
有时艺术家会对基础网格进行重新拓扑（Retopology）后，试图将旧的 BlendShape 目标沿用到新网格上。由于顶点数量或顺序发生变化，直接复用会导致顶点错位和网格崩坏。正确做法是使用 Maya 的 **Wrap 变形器** 或 ZBrush 的 **Project 功能** 将旧目标的形态投射到新拓扑网格上，重新生成一套对齐的变形目标库。

## 知识关联

BlendShape 技术的上游是**面部绑定**的整体规划：绑定师需要先完成面部骨骼关节的布局、确定哪些动作由关节负责（如下颌旋转、眼球转动），然后再确定哪些细节形变需要交由 BlendShape 补充。通常的分工原则是：大范围、可参数化的运动用骨骼，细微的肌肉褶皱、皮肤堆积和表情微调用 BlendShape。

在实际绑定层级中，BlendShape 节点通常位于变形栈（Deformer Stack）的最前端，在蒙皮（Skinning）之前被计算，这意味着 BlendShape 作用于局部空间中的基础网格，之后才由蒙皮骨骼将其变换到世界空间。理解这一计算顺序对于排查变形穿插 bug 至关重要。掌握 BlendShape 之后，进阶方向是学习基于机器学习的神经网络变形器（如 Autodesk Bifrost 中的 Neural Morph），以及用于高效压缩大规模 Morph Target 数据的 PCA（主成分分析）技术，两者均以本节所讲的顶点偏移原理为基础。
