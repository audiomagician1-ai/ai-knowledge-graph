# 褶皱权重（Crease Weight）

## 概述

褶皱权重（Crease Weight）是细分曲面建模中的一项边属性技术，允许艺术家在不增加额外几何布线的情况下，控制细分算法对某条边的"锐利程度"处理。具体而言，每条边可以被赋予一个介于 0.0 到 1.0 之间的浮点数值，当值为 0.0 时该边与普通边无异，完全受细分平滑影响；当值为 1.0 时该边被视为完全锐利边（Sharp Edge），细分后依然保持折角形态，不发生任何平滑过渡。

褶皱权重概念源自 Pixar 在 1998 年发表的论文《Subdivision Surfaces in Character Animation》，该论文正式引入了边褶皱（Edge Crease）的数学定义，将其并入了后来广为人知的 OpenSubdiv 规范。Blender 从 2.50 版本起原生支持 Crease Weight，快捷键为 **Shift+E**，随后在 Maya 的 Crease Tool 与 3ds Max 的 OpenSubdiv 修改器中也得到了广泛实现。

褶皱权重对硬表面建模的实际意义在于：机械零件、车辆车身、武器装备等对象通常同时拥有大量圆滑面与明确折角，如果全部依赖倒角边（Bevel）保持硬边，网格的面数会以几何级数增长，导致后续 UV 展开、法线烘焙和游戏引擎导入的复杂度大幅上升。使用 Crease Weight，艺术家可以在极低面数的 Cage Mesh 上精确控制细分后的折痕位置。

## 核心原理

### Catmull-Clark 细分与褶皱的数学关系

标准 Catmull-Clark 细分会将每条边的顶点位置按照周围面和顶点的加权平均来移动，使网格趋于平滑。褶皱权重本质上是对这一加权公式的"插值系数"施加限制：当 Crease Weight 为 *c*（0≤c≤1）时，该边在细分时的平滑贡献被乘以 (1−c) 因子，即：

> **平滑后的边点位置 = (1−c) × Catmull-Clark边点 + c × 原始中点**

当 c=1 时，边点完全锁定在原始中点，与倒角添加两条支撑边的视觉效果高度近似，但不会改变拓扑结构。

### 整数级别褶皱（Integer Crease）

部分软件（如 OpenSubdiv 底层规范）支持整数级别的褶皱值，表示"在前 N 次细分迭代中保持锐利，之后逐渐平滑"。例如，设置整数级别为 2 的褶皱意味着该边在第 1、2 次细分时保持硬边，第 3 次细分开始参与平滑。Blender 的界面将 0~1 的小数值暴露给用户，底层会将其换算为 OpenSubdiv 可识别的整数级别。

### 褶皱权重与支撑边的等效关系

一条完全褶皱（Crease Weight=1.0）的边在视觉效果上等效于在该边两侧分别添加两条间距极小的支撑边（Loop Cut），但两者的网格密度截然不同：支撑边方案在一个矩形面上需要额外增加至少 4 条边和 4 个顶点，而褶皱权重方案的顶点和面数量保持不变。在 Blender 中，一个边数为 E 的模型使用 Crease Weight 后细分至第 2 级，面数为原始的 4²=16 倍；如果改用支撑边方案保持相同锐利度，支撑边增加的额外面可能额外再翻倍，差距十分显著。

## 实际应用

**机械装甲板建模**：制作机器人装甲或车辆外壳时，面板与面板之间的接缝需要清晰的折痕，而面板本身需要保持光滑曲面。艺术家通常在 Cage Mesh 上为所有接缝边赋予 Crease Weight=1.0，为面板中部微弱弧度边赋予 0.3~0.5 的中间值，从而在单一低面数网格上同时表达刚硬结构与曲面过渡。

**游戏引擎导出工作流**：当模型需要从 Blender 导出至 Unreal Engine 或 Unity 时，Crease Weight 数据本身无法随 FBX/OBJ 格式传递。正确的流程是：在导出前对 Cage Mesh 应用 Subdivision Surface 修改器，将褶皱权重已"烘焙"进高面数网格，再用该高模进行法线贴图烘焙，最终游戏中使用低模+法线贴图组合还原硬边视觉效果。

**与 Bevel 修改器混合使用**：复杂硬表面上某些边需要带有一定宽度的倒角过渡（如导弹尾翼与机身的结合处），而非完全锐利折角。此时可设置 Crease Weight=0.7~0.8，而非 1.0，使细分后该边区域产生约为原始边长 5%~10% 宽度的柔和过渡，同时避免手动添加精确间距支撑边带来的布线复杂度。

## 常见误区

**误区一：认为 Crease Weight=1.0 与倒角布线在所有情形下完全等效**

两者在平滑预览中外观相似，但在法线烘焙时会产生差异。Crease Weight 方案的低模 Cage 在折角处只有一个顶点，烘焙工具在计算该点的法线偏差时采样范围不足，可能导致烘焙法线在转角处出现宽度约 1~2 个像素的黑边伪影。倒角方案因存在物理的支撑边，转角处的几何过渡更加平滑，烘焙法线质量通常更优。在需要高质量法线烘焙的游戏资产制作中，工业界仍普遍采用倒角布线而非纯褶皱权重作为最终高模方案。

**误区二：将 Crease Weight 与平滑组（Smoothing Group）混淆**

平滑组（或 Blender 中的"标记锐边 Mark Sharp + 边线细分修改器"）控制的是面法线的渲染插值，属于着色层面的操作，不影响网格几何形状。Crease Weight 直接修改细分算法的几何计算，影响的是实际顶点坐标。一条边可以同时拥有 Crease Weight 和 Sharp 标记，两者分别在细分阶段和着色阶段各自独立发挥作用，混淆两者会导致在未开启细分时模型外观正常、开启后出现意外圆角的调试难题。

**误区三：认为中间值（如 0.5）会产生"一半锐利"的可预期效果**

Crease Weight 的中间值在不同细分级别下表现差异显著。同一条 Crease Weight=0.5 的边，在细分级别 1 时折角角度可能保留约 60%，但在细分级别 3 时可能仅保留约 20%，呈非线性衰减。使用中间值时，艺术家必须结合目标细分级别实时观察效果，而不能凭直觉预判最终形态。

## 知识关联

褶皱权重与倒角边工作流（Bevel Edge Workflow）解决同一个问题——细分后保持硬边——但路径相反：倒角边通过增加几何密度为细分提供支撑，而褶皱权重通过修改细分算法本身的计算权重来绕过对额外几何的需求。掌握倒角边工作流之后，学习褶皱权重能让艺术家建立起"低多边形 Cage + 权重控制"的替代思维，在原型阶段和概念验证阶段快速迭代造型而无需频繁重新布线。

在生产管线中，两种技术常被同一模型混合使用：主要轮廓折角采用褶皱权重保持灵活可调，细节倒角（如螺孔、卡扣槽）则依赖实际布线以确保烘焙质量。理解褶皱权重在 OpenSubdiv 规范中的数学定义，也为理解更高级的细分曲面拓扑优化（如极点处理、N-Gon 的细分行为）奠定了计算几何层面的认知基础。
