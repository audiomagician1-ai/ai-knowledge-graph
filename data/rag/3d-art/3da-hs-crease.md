# 褶皱权重

## 概述

褶皱权重（Crease Weight）是一种在细分曲面（Subdivision Surface）修改器中用于控制边缘锐利程度的数值参数，取值范围为 0.0 到 1.0。当数值为 0 时，该边在细分后完全平滑；当数值为 1 时，该边在细分后保持完全锐利，无论细分级别多高，边缘形态都不会圆润化。这项功能最初由皮克斯公司在其 OpenSubdiv 规范中正式提出并标准化，Blender 从 2.63 版本起将其整合进编辑器工作流，Maya 则在其 Smooth Mesh Preview 系统中以同样的逻辑实现此功能。

褶皱权重解决了硬表面建模中一个长期存在的矛盾：为了让细分曲面保持锐利边缘，传统方式需要在边缘附近添加两条甚至三条"支撑循环线"（Support Loop），而这些额外的布线会大幅增加面数，并在后续的 UV 展开、法线烘焙和动画变形阶段带来不必要的复杂度。褶皱权重通过在边本身存储一个浮点数值，让修改器在计算细分时对该边施加额外的张力，从而在不改变拓扑结构的前提下实现硬边效果，彻底绕开了额外布线的需求。

在工业管线中，褶皱权重属于非破坏性工作流（Non-Destructive Workflow）的重要组成机制。模型师可以在低多边形代理网格（Low-Poly Proxy Mesh）阶段就完成所有硬边决策，之后随时通过调整数值来修改造型，而无需重新布线。这对于需要频繁迭代的产品可视化和影视资产制作尤为关键。

## 核心原理

### 数学层面：褶皱权重如何影响细分算法

Catmull-Clark 细分算法在处理每条边时，会根据相邻面的顶点坐标计算出平滑后的新顶点位置。褶皱权重的本质是向该算法注入一个权重因子 *w*，修改插值公式：

**新顶点位置 = (1 - w) × 平滑位置 + w × 原始边中点位置**

当 *w* = 0，完全采用 Catmull-Clark 平滑结果；当 *w* = 1，新顶点强制落在原始边的中点上，不受邻近面影响，从而保持锐利。0.5 之类的中间值会产生介于平滑与锐利之间的"半硬边"效果，常用于工业零件的轻微倒角表现，以及需要微妙过渡的金属折弯区域。

### 存储方式：边数据层（Edge Crease Layer）

在 Blender 中，褶皱权重以"自定义数据层"（Custom Data Layer）的形式存储在网格的边（Edge）数据块上，每条边独立持有一个 float32 值。这意味着同一个网格可以同时让若干边保持锐利（值为 1），另一些边处于半锐利状态（值为 0.3～0.7），其余边完全平滑（值为 0），所有这些状态共存于同一个低多边形网格中，不需要任何额外几何体。在 Blender 编辑模式下，选中边后按 **Shift + E** 即可进入褶皱权重调节，也可在 N 面板的"Item"选项卡中直接输入精确数值。

### 褶皱权重与顶点褶皱的区别

除了边级别的 Crease Weight，部分软件（包括 Blender 4.0 版本后）还支持顶点级别的褶皱（Vertex Crease），用于控制顶点在细分后的位置张力。顶点褶皱主要处理三边相交的角点，例如一个立方体的八个角。如果仅对三条棱边设置 Crease Weight = 1 而不对角顶点设置 Vertex Crease，细分后角点处仍会出现轻微的内缩现象；将角顶点的 Vertex Crease 同样设为 1，才能还原出完全锐利的 90° 直角。这一细节在制作电子产品外壳和建筑构件时尤为关键。

## 实际应用

### 硬件产品可视化中的机箱建模

以一个标准 ATX 机箱为例：机箱顶部与侧板之间的折边需要保持接近 90° 的锐利转角，而面板中间的大片区域需要保持光滑。传统倒角边工作流需要在折边两侧各加一条间距约 0.02 个单位的支撑线，这会使该区域的边循环密度翻倍。使用褶皱权重后，只需选中折边并设置 Crease = 1.0，整个侧面可以维持极简的矩形拓扑，细分后的效果与六条支撑线的传统方案完全一致，但网格面数减少约 40%。

### 汽车车身硬边与软边的混合表现

汽车的腰线（Character Line）通常需要一种"半锐利"的效果：既不像折纸般的硬折边，也不像气泡般的完全圆滑。将腰线所在的边设置 Crease Weight = 0.65～0.75，可以在细分后得到一条明显的高光分割线，同时边缘仍有约 2～3mm 视觉宽度的微圆角过渡，完全匹配实际钣金冲压工艺的视觉特征，而这种细腻的表现若用支撑线实现则需要三条以上的精确间距布线。

### 与法线烘焙工作流的配合

在影视或游戏资产制作中，高模通常需要将细节烘焙到法线贴图上供低模使用。使用褶皱权重的高模网格在烘焙时，锐利边处的法线方向变化等同于同位置倒角边的效果，烘焙结果与传统布线高模几乎没有视觉差异，但高模的面数因省去大量支撑线而降低 30%～50%，直接缩短了烘焙渲染时间。

## 常见误区

### 误区一：褶皱权重可以完全替代倒角边

褶皱权重在细分曲面渲染和高模烘焙场景中效果出色，但它并不创建真实的几何倒角。当模型需要被导出为游戏引擎的实时网格（如 FBX 格式用于 Unreal Engine）时，细分修改器通常需要被应用（Apply），此时褶皱权重信息不会被大多数引擎识别或传递，边缘会失去锐利效果。因此，面向实时渲染的最终交付资产仍然需要倒角边（Bevel）或手动支撑线来维持硬边外观；褶皱权重主要在制作阶段和离线渲染管线中发挥作用。

### 误区二：将褶皱权重与"锐利边"（Sharp Edge / Smooth by Angle）混淆

"锐利边"（Blender 中的 Mark Sharp）是法线显示层面的标记，控制的是表面法线在渲染时是否在该边处断开，与几何形态无关。褶皱权重控制的是细分算法对几何顶点位置的计算方式，是对几何形态本身的干预。一条边完全可以同时持有 Crease Weight = 1 和 Mark Sharp 标记，或者只持有其中一项，两者相互独立，不能互相替代。初学者经常设置了 Mark Sharp 之后以为已经"锁住了硬边"，然后困惑于为什么细分后边缘仍然圆润——原因正是混淆了这两个完全不同的参数。

### 误区三：对所有边都设置 Crease = 1 可以让网格整体变硬

如果对网格中所有边都设置 Crease Weight = 1，细分后的网格确实不会有任何平滑，但也失去了细分曲面的意义——此时与直接使用低多边形网格没有区别。更严重的问题是，非流形（Non-Manifold）拓扑区域在全边 Crease = 1 的情况下，细分后可能出现几何撕裂。褶皱权重应当被选择性地施加在真正需要保持锐利的折边上，其余区域保持 Crease = 0 才能发挥细分曲面的造型能力。

## 知识关联

褶皱权重的直接前置知识是**倒角边工作流**（Bevel Edge Workflow）。理解倒角边工作流能让学习者清楚地认识到支撑线的几何本质——它们的作用是通过缩短邻近边的长度来"欺骗" Catmull-Clark 算法，使其在该区域产生更小的圆角半径。有了这个认识，褶皱权重就可以被理解为对同一目标（锁定边缘锐利度）的数学捷径：直接向算法注入张力权重，而非通过操纵拓扑间接影响算法输出。在实际项目中，两种方法常常混合使用：用褶皱权重处理制作阶段的主要折边，用倒角边处理需要交付实时网格的关键区域，从而在制作灵活性与最终交付兼容性之间取得平衡。
