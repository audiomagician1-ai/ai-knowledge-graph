# 轴心与原点

## 概述

在三维软件中，每一个物体都有一个特殊点称为"原点"（Origin），也叫做"轴心点"（Pivot Point）。这个点是物体自身坐标系的中心，所有的位移、旋转、缩放操作都以此为参考基准执行。例如在Blender中，新建一个默认立方体时，其原点自动放置于立方体的几何中心；而在3ds Max中，该点被称为Pivot，默认位于物体的底部中心。

原点概念最早随三维建模软件的发展在1980年代逐步确立。当时SGI工作站上的早期三维软件需要一种固定参考点来描述物体的空间变换，原点由此成为三维变换矩阵的核心锚点。每个物体的变换矩阵（Transform Matrix）包含旋转、缩放和位移信息，而这些信息都相对于原点计算——移动原点而不移动网格，等同于修改了物体的变换矩阵偏移量。

原点的位置直接影响艺术家对物体执行变换时的视觉结果与技术正确性。一扇门如果原点放在几何中心而非合页侧边，旋转时会围绕自身中央转动而不是绕合页开合，导致动画完全错误。对于游戏引擎导入而言，原点的世界坐标位置决定了物体在引擎场景中的初始落点，原点不在世界坐标(0,0,0)的模型导入Unity或Unreal Engine时往往出现位置偏移的问题。

## 核心原理

### 原点与变换矩阵的关系

物体的完整变换由一个4×4矩阵表示，其中旋转和缩放操作均以原点为中心执行。以旋转为例，若物体原点位于(5, 0, 0)而非(0, 0, 0)，绕Y轴旋转90度后，物体会围绕世界Y轴偏移5个单位处旋转，而非原地自转。这个原理决定了"原点在哪里，旋转就围绕哪里转，缩放就从哪里扩张或收缩"。在Blender中可通过快捷键`Ctrl+Shift+Alt+C`（旧版）或右键菜单"Set Origin"来独立移动原点而不影响网格的世界位置。

### 原点设置的标准规范

不同类型的资产有不同的原点放置规范：
- **建筑与道具类**：原点通常放置于物体底部中心，使其与地面的Y=0平面对齐，方便在引擎中直接将Y坐标设为0即可落地。
- **角色模型**：原点放置于角色双脚之间的地面位置，即世界坐标(0, 0, 0)处。
- **旋转部件**（如轮子、风扇、门）：原点必须放置于真实的旋转轴心处，例如车轮原点须位于轮轴中心，否则动画师在制作旋转动画时会得到错误的轨迹。
- **粒子发射源**：原点即粒子的发射位置，因此需要精确对齐到喷口或发射点。

### 原点与游戏引擎导入的关联

FBX格式导出时，物体的原点坐标会被记录为该节点在场景层级中的本地位置偏移（Local Transform）。当一个模型的原点位于世界空间(3, 0, -2)时，导入Unity后该物体的Transform组件中Position会显示为(3, 0, -2)而非(0, 0, 0)，这常常让引擎工程师感到困惑。在Unreal Engine的蓝图中，若模型原点偏移，碰撞体的自动生成位置也会随之偏移，导致物理碰撞检测出现误差。因此行业通用规范要求：**所有静态网格物体在导出前，其原点必须位于世界坐标(0, 0, 0)**，并且网格本身摆放于希望在引擎中呈现的相对位置。

## 实际应用

**案例一：制作旋转门动画**
在3ds Max中制作一扇单开门时，首先建立门板几何体，然后将Pivot点精确移动至门板左侧边缘（即合页所在边）。具体操作：进入Hierarchy面板 → 勾选"Affect Pivot Only" → 使用移动工具将Pivot拖至左边缘并对齐。之后对该门设置旋转关键帧时，门就会绕正确的合页轴心开合，旋转角度一般为-90°到0°的范围。若跳过此步骤直接制作旋转动画，门将绕其几何中心旋转，视觉效果完全不可用。

**案例二：游戏道具的原点规范化**
假设制作一个木箱资产，其几何体底面正好贴合世界Z=0平面，而建模师在建模时将原点保留在了几何中心（即Z=0.5m处）。导入Unity后，将此木箱拖入场景，其Transform.Position.y会显示为0.5，而非0。关卡设计师每次摆放都需要手动调整Y值为-0.5才能使木箱落地，严重降低工作效率。正确做法：在Blender中选中木箱，执行 Object → Set Origin → Origin to Geometry（若底面在Z=0则用"Origin to 3D Cursor"配合游标对齐底部中心）。

**案例三：镜像操作依赖原点位置**
在制作对称模型时，Blender的镜像修改器（Mirror Modifier）默认以物体原点为镜像平面的中心。若角色身体模型的原点偏离躯干中轴线，镜像后左右两侧的连接处会出现错位缝隙或重叠。将原点精确设置在角色身体的对称轴上（通常是世界X=0处），才能保证镜像修改器产生无缝对称的结果。

## 常见误区

**误区一：认为移动原点就是移动物体**
许多初学者在Blender中执行"Set Origin to Geometry"后发现物体在视口中"跳位"，误以为是物体被意外移动了。实际上是原点位置改变了，但物体的网格在世界空间中并未移动——改变的是原点与网格之间的相对偏移关系。反之，在3ds Max中勾选"Affect Pivot Only"后移动Pivot，网格同样不会移动。这两种情况下，物体的世界位置（网格顶点的绝对坐标）保持不变，只有原点坐标发生了变化。

**误区二：认为原点必须在物体内部**
原点不必位于物体的网格范围之内。对于一组需要绕某固定参考点公转的卫星模型，其原点可以设置在距离网格较远的公转中心处，这样旋转该物体时就会形成公转轨迹。游戏中的门、机械臂关节、炮台旋转都利用了原点置于网格外部的特性来实现正确的轴心旋转。

**误区三：应用变换（Apply Transform）与重置原点混淆**
在Blender中，"Apply All Transforms"（Ctrl+A）会将物体当前的旋转和缩放值写入顶点坐标，使旋转归零、缩放归一，但原点位置不一定回到(0,0,0)。而"Set Origin → Origin to 3D Cursor（游标在世界原点）"才是将原点归位至世界(0,0,0)的正确操作。混淆这两个操作会导致导入引擎后模型缩放正常但位置依然偏移。

## 知识关联

学习轴心与原点之前，需要掌握**变换工具**的基本使用——位移（G）、旋转（R）、缩放（S）的操作逻辑，因为只有理解了"变换是相对于参考点执行的"，才能明白原点位置为何如此关键。轴心与原点的概念构成了后续学习骨骼绑定（Rigging）的直接前置认知：骨骼系统中每块骨骼的头尾关节本质上就是旋转轴心点，其设置逻辑与物体原点完全相通。此外，UV展开时"UV的原点"与物体原点是两个独立的坐标系，不可混淆——物体原点影响3D变换，而UV原点(0,0)至(1,1)的范围仅影响贴图采样坐标。
