# 细分曲面修改器

## 概述

细分曲面修改器（Subdivision Surface Modifier）是3D建模中将低面数网格转化为高密度光滑曲面的工具，其数学核心是由Edwin Catmull与Jim Clark于1978年在论文《Recursively Generated B-Spline Surfaces on Arbitrary Topological Meshes》中共同提出的Catmull-Clark细分算法（Catmull & Clark, 1978, *Computer-Aided Design*, Vol.10, No.6, pp.350-355）。该算法的基本规则是：每执行一次细分，网格中的每个四边面被分割成4个更小的四边面，同时对所有顶点位置按照加权平均公式重新计算，使棱角分明的硬表面趋向光滑的B样条极限曲面。这一数学极限曲面具有G2曲率连续性（在非极点处），这正是它能生成视觉上"有机感"圆滑曲面的几何学原因。

在Blender 3.x版本中，细分曲面修改器位于修改器面板（扳手图标）的"生成"类别下，提供Catmull-Clark与Simple两种模式：前者重新定位顶点实现光滑，后者仅分割面而不移动顶点，常用于需要保留轮廓但增加几何密度的场景（例如置换贴图的基底网格）。

## 核心原理

### Catmull-Clark顶点重定位公式

Catmull-Clark算法中，每次细分后各类新顶点的坐标由以下三类加权规则决定：

**面点（Face Point）$F$**：取该面所有顶点坐标的算术平均值：

$$F = \frac{1}{n}\sum_{i=1}^{n} V_i$$

**边点（Edge Point）$E$**：取该边两个端点与相邻两个面点的平均：

$$E = \frac{V_1 + V_2 + F_1 + F_2}{4}$$

**原始顶点更新位置 $V'$**：设顶点 $P$ 的相邻面数（价数）为 $n$，$Q$ 为相邻面点均值，$R$ 为相邻边中点均值，则：

$$V' = \frac{Q}{n} + \frac{2R}{n} + \frac{(n-3)P}{n}$$

当 $n=4$（标准四边面环境）时，公式简化为 $V' = \frac{Q + 2R + P}{4}$，这正是最常见的"四价顶点"情况，也是Catmull-Clark在全四边面拓扑下表现最稳定的原因。当 $n \neq 4$ 时（三角面汇聚点 $n=3$，五星点 $n=5$），原始顶点的权重 $(n-3)/n$ 发生偏移，导致极限曲面在该点附近的曲率连续性降为G1甚至C0，在高光反射中表现为肉眼可见的皱褶。

### 细分级别与面数的指数增长

细分级别（Subdivision Level）每增加1级，网格面数乘以4。下表展示从100个初始四边面出发的增长趋势：

| 细分级别 | 四边面数量 | 顶点数量（近似） |
|---------|-----------|----------------|
| 0（原始）| 100       | ~102           |
| 1       | 400       | ~404           |
| 2       | 1,600     | ~1,604         |
| 3       | 6,400     | ~6,404         |
| 4       | 25,600    | ~25,604        |

Blender细分修改器将"视口细分级别"与"渲染细分级别"分开设置，推荐工作流为：视口保持1~2级确保交互帧率高于24fps，渲染级别设为2~3级。超过4级在当前主流GPU（如RTX 3070级别）上会导致单帧视口刷新时间超过200ms，交互流畅度急剧下降，而视觉细节相较3级并无实质改善，因为3级细分已超过4K屏幕单像素对应的几何分辨率。

### 控制边（Support Edge）的作用机制与间距控制

Catmull-Clark细分会无差别软化所有边，导致建模时的直角转折在细分后退化为大幅度圆弧。解决方法是在需要保留锐利度的边两侧各添加一条紧贴的平行循环边（Loop Cut，Blender快捷键 `Ctrl+R`），称为"控制边"或"支撑边"。

控制边与目标边的间距直接决定细分后圆角的曲率半径：

- 间距为边长的 **5%以内**：细分后视觉近似硬角（半径 < 0.1个单位）
- 间距约为边长的 **15%~20%**：产生工业产品常见的小圆角过渡
- 间距约为边长的 **30%以上**：产生有机感的大圆弧过渡，适合角色皮肤褶皱

例如，对一个边长为2m（建模单位）的正方体应用2级细分，若在顶部边缘各侧添加间距0.1m（5%）的控制边，细分后圆角半径约为0.08m，视觉上接近冲压钢材的典型工艺圆角。

Blender还提供**折痕权重（Crease Weight）**系统（快捷键 `Shift+E`，范围0.0~1.0）。折痕值1.0时，该边在Catmull-Clark细分中被强制视为"固定边界"，细分后完全不软化，等效于在两侧各贴一条控制边，但不增加任何实际几何面数。折痕权重适合在概念设计阶段快速测试硬边效果，但最终交付生产前建议转换为实际控制边，因为折痕权重在FBX导出至其他DCC软件时不被普遍识别（Maya、3ds Max均不支持读取Blender的Crease数据）。

## 关键公式与算法实现

以下Python伪代码展示Catmull-Clark单次细分中面点计算的核心逻辑，帮助理解Blender内部对每个面执行的操作：

```python
def compute_face_points(mesh):
    """
    对网格中每个面计算Catmull-Clark面点（Face Point）。
    face_point = 面内所有顶点坐标的算术平均值
    """
    face_points = {}
    for face in mesh.faces:
        verts = [mesh.vertices[v_idx].co for v_idx in face.vertex_indices]
        n = len(verts)
        face_point = sum(verts, Vector((0, 0, 0))) / n  # Vector逐分量求和后除以n
        face_points[face.index] = face_point
    return face_points

def compute_edge_points(mesh, face_points):
    """
    对每条内部边计算边点（Edge Point）。
    edge_point = (v1 + v2 + adjacent_face_point_1 + adjacent_face_point_2) / 4
    """
    edge_points = {}
    for edge in mesh.edges:
        v1 = mesh.vertices[edge.v1].co
        v2 = mesh.vertices[edge.v2].co
        adj_faces = edge.adjacent_faces  # 假设每条内部边恰好有2个相邻面
        f1 = face_points[adj_faces[0]]
        f2 = face_points[adj_faces[1]]
        edge_points[edge.index] = (v1 + v2 + f1 + f2) / 4
    return edge_points
```

上述代码仅为教学示意，实际Blender的细分实现以C++编写于 `source/blender/blenkernel/intern/subdiv_mesh.cc` 中，并通过OpenSubdiv库（Pixar开源，2012年发布）加速计算，支持自适应细分（Adaptive Subdivision），可根据摄像机距离动态调整各区域的细分密度，避免全局高细分带来的资源浪费。

## 实际应用

### 工业硬表面产品建模

对一个保温杯模型（初始网格约320个四边面）应用2级细分修改器时，若不添加控制边，杯口圆弧会退化为半径约5mm的大圆角，失去金属冲压工艺的锐利感。正确工作流：在杯口内环与外环各添加两条控制边，间距约为壁厚（建模值2mm）的15%（即0.3mm间距），2级细分后圆角半径约0.25mm，接近真实钢材冲压工艺的0.2~0.5mm标准范围。

对于杯底凹陷的内凹圆角（内角），需在凹陷边缘同样添加控制边，否则Catmull-Clark的加权平均会将凹陷区域向外拉伸，导致底部平面出现不自然的隆起。这一现象在凹入深度与宽度比超过1:3时尤为明显。

### 角色脸部有机建模

人脸眼眶区域集中了高密度布线，标准做法是在眼眶外圈布置6~8条放射状循环边并最终汇聚到耳前的"汇流区"（Edge Flow Sink）。若眼眶处存在三角面（常见于初学者为"修复多边形"随意连线产生），2级细分后三角面汇聚极点附近会出现直径约2~5mm（建模单位）的碗形凹陷，在强侧光打光时高光失真明显。

角色皮肤通常需要2级细分才能让基础网格的分面感消失：以人脸为例，一个典型的低模脸部网格约含500~800个四边面，2级细分后达到8,000~12,800面，单个多边形的屏幕投影面积在1080p分辨率下低于2个像素，基本消除棱角感。3级细分（32,000~51,200面）在写实渲染中会配合法线贴图使用，而非作为纯几何细节手段。

### 细分修改器在修改器栈中的位置

细分修改器在修改器栈中的顺序至关重要。标准顺序为：**镜像修改器 → 细分曲面修改器 → 表面形变修改器（Surface Deform）**。若将细分修改器置于镜像修改器之前，镜像拼接处在细分后会出现缝隙（因细分算法将接缝边视为边界边，权重计算不同于内部边）。具体而言，边界边的边点公式退化为 $E = (V_1 + V_2) / 2$，不纳入面点影响，导致接缝处曲率不连续。

## 常见误区

**误区1：直接提高细分级别替代布线优化**
许多初学者在硬表面模型圆角不理想时直接将细分级别从2提升到4，面数暴增16倍，但根本问题是控制边缺失。正确做法是保持2级细分，在目标位置补充控制边，面数不增加而效果完全等效。

**误区2：折痕权重可以完全替代控制边**
折痕权重值为1.0时虽然视觉上等同于硬边，但其在Catmull-Clark极限曲面上的数学行为实际上是将该边退化为分段线性边（Piecewise Linear），而非平滑过渡的圆角。在靠近折痕边的区域曲率会出现急剧变化（即G1而非G2连续），在斜向光照的高光反射中会暴露出细微的"折痕光带"。控制边则通过实际几何位置引导算法，在极限曲面上保持C2连续性直至控制边内侧区域。

**误区3：三角面在细分后问题"不大"**
在平坦区域（法线变化小于5°/单位长度）的三角面细分后瑕疵不明显，但在高曲率区域（如关节弯曲、皱纹褶皱），三角面极点引发的曲率不连续会被曲面弯曲放大，产生肉眼明显的"鼓包"。游戏角色建模规范（如Epic Games《Fortnite》角色制作规范文档）明确要求：高曲率区域严禁三角面，平坦区域每1000面允许不超过3个三角面。

**误区4：应用（Apply）细分修改器后再继续造型**
将细分修改器应用到网格后，面数急剧增加（2级细分后为原始的16倍），此后所有造型操作都在高密度网格上进行，极大降低效率。正确工作流是始终在低面数基础网格（Base Mesh）上建模，保留修改器不应用，仅在最终烘焙法线贴图或导出时选择性应用。
