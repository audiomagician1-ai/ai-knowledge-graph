# 材质混合

## 概述

材质混合（Material Blending）是3D环境美术中将两种或多种材质在同一表面上进行无缝过渡融合的技术。与直接在单一材质中绘制纹理细节不同，材质混合通过控制权重蒙版（weight mask）决定每个像素点上各材质的显示比例，从而在草地与泥土、岩石与沙地等自然边界处产生真实可信的过渡效果。

材质混合技术在游戏行业的系统化应用可追溯至2000年代中期，随着可编程着色器（programmable shader）的普及而成熟。早期引擎依赖预先烘焙好的混合贴图，而现代引擎如Unreal Engine 4/5则允许实时计算基于高度和世界坐标的动态混合，大幅减少了美术资产的重复制作量。

材质混合的价值在于它将可平铺纹理资产（tileable texture）的重复利用率最大化——同一套岩石纹理既可单独使用，也可通过混合系统与草地纹理自然衔接，从而使地形表面兼顾细节密度与制作效率。

## 核心原理

### 顶点颜色混合（Vertex Color Blending）

顶点颜色混合利用模型顶点上存储的RGBA四个通道作为混合权重，最多可在一个网格上同时承载4种材质的混合信息。每个通道（R/G/B/A）对应一种材质层，顶点颜色值从0到1代表该材质在该顶点附近的显示权重。在着色器中，混合公式为：

```
最终颜色 = 材质A × (1 - VertexColor.R) + 材质B × VertexColor.R
```

顶点颜色的精度受网格拓扑密度限制——低多边形地形网格上，顶点之间的插值过渡往往显得模糊，边界宽度无法精确控制。因此顶点颜色混合更适用于大范围地形上的宏观区域划分，而非石块缝隙级别的精细过渡。在Unreal Engine中，可以使用顶点绘制工具（Mesh Paint Tool）在编辑器视口中直接刷入顶点颜色权重。

### 高度混合（Height-Based Blending）

高度混合通过读取各材质自身的高度贴图（heightmap）来锐化混合边界，使材质过渡沿着表面凹凸轮廓分布。其核心公式为：

```
BlendFactor = saturate((HeightA + MaskA - HeightB - MaskB) / TransitionSharpness)
```

其中 `HeightA`/`HeightB` 是各材质高度图采样值，`MaskA`/`MaskB` 是来自顶点颜色或控制贴图的基础权重，`TransitionSharpness` 控制边界锐利程度（典型值范围0.05–0.3）。

高度混合的视觉效果远优于线性混合：泥土会从岩石表面凸起处向凹陷处自然渗入，雪会在物体高处积聚而在侧面消退。这种效果在《巫师3》《地平线：零之曙光》等写实风格游戏地形中被大量采用。

### 世界坐标混合（World-Space Position Blending）

世界坐标混合利用片元（fragment）在世界空间中的坐标值驱动混合权重，最常见的应用是基于Y轴（高度轴）实现积雪效果——物体海拔越高，雪材质权重越大：

```
SnowBlend = saturate((WorldPosition.Z - SnowStartHeight) / SnowFalloff)
```

除高度外，世界坐标混合还可以利用 `WorldPosition.XZ` 结合噪波函数生成宏观地面纹理变化，打破可平铺贴图的重复感。法线方向混合（Normal-Based Blending）是世界坐标混合的衍生形式，通过计算表面法线与世界向上向量的点积（dot product）决定积雪、青苔或灰尘的沉积位置，点积值接近1.0（水平朝上面）时材质权重最大。

## 实际应用

**地形地表过渡**：在开放世界游戏地形制作中，设计师通常将基础草地设为默认材质（权重100%），然后在道路两侧、悬崖脚下等位置叠加泥土/碎石材质。使用高度混合时，草地高度图的凸起草叶纹理会从泥土平面中"穿出"，形成草边蔓延到泥路边缘的自然视觉。

**建筑与地面衔接**：建筑物底部墙角处的地面脏污效果可以用世界坐标高度混合实现——在墙基以下约15–30cm范围内，将灰尘/青苔材质权重从0渐变至1，无需美术手工绘制每栋建筑的独立贴图。

**程序化岩石覆盖**：在Unreal Engine的材质编辑器中，将顶点颜色R通道与高度混合结合，可以在同一岩石静态网格（Static Mesh）上实现干燥面/湿润面/苔藓覆盖面三种状态，通过运行时修改顶点颜色实现季节切换或天气变化响应。

## 常见误区

**误区1：认为高度混合与高度贴图用途相同**

高度贴图（heightmap）在材质中通常用于视差遮蔽映射（Parallax Occlusion Mapping）来模拟表面凹凸，而高度混合中读取的高度贴图仅作为混合权重的调制信号，并不直接影响法线或位移计算。两者读取的是同一张贴图，但采样结果进入了完全不同的计算路径。

**误区2：过度依赖顶点颜色混合导致边界模糊**

顶点颜色精度由网格顶点密度决定，在顶点间距为1米的低密度地形网格上，两种材质的过渡区域可能宽达2–3米，无法呈现锐利边界。解决方案是在顶点颜色混合基础上叠加一层高度混合或噪波遮罩，将过渡宽度压缩至0.1–0.3米视觉级别。

**误区3：世界坐标混合会随物体移动而产生"游泳"现象**

这一问题确实存在于动态移动物体上：当物体在世界中移动时，基于WorldPosition计算的纹理坐标会在表面上"流动"。对于静态场景物体（地形、建筑、岩石群）此问题不存在，而对于需要移动的道具应改用UV坐标混合或顶点颜色混合代替世界坐标方案。

## 知识关联

材质混合建立在**可平铺资产**的制作规范之上——混合的每一层材质本身必须是无缝可平铺的，否则在大面积铺展时重复接缝会使混合失去意义。可平铺贴图的分辨率通常为512×512至2048×2048（2的幂次），这一规格直接决定了高度混合中高度贴图的细节频率，进而影响混合边界的精细程度。

在更高级的地形着色工作流中，材质混合会与地形系统的图层权重（Layer Weight）、Shader中的MipMap级别控制以及LOD系统配合使用，确保远景地形的混合计算成本被有效压缩。掌握本文介绍的三种混合模式后，美术师可以进一步学习基于运行时虚拟纹理（Runtime Virtual Texture，RVT）的地形材质混合方案——这是Unreal Engine 5中处理大规模开放世界地形混合的主流技术路线，它将顶点颜色、高度混合与世界坐标混合的结果预先缓存至虚拟纹理，从而将混合计算的GPU开销降低约40%–60%。
