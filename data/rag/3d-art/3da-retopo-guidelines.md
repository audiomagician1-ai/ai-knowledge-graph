# 拓扑规范

## 概述

拓扑规范（Topology Standard）是工业级3D项目中对模型网格结构制定的具体技术要求与验收标准，涵盖面数上限、边线流向、极点密度、UV兼容性等可量化指标。与面数规划关注"用多少面"不同，拓扑规范的核心在于规定"面该怎么连"——即网格拓扑结构的质量评判维度与合规判定流程。

拓扑规范作为独立文档体系，最早在2000年代初的游戏工业（以EA、Ubisoft为代表）中系统化成文，彼时PS2/Xbox世代的严格硬件限制迫使团队必须将口耳相传的经验转化为可执行的书面标准。此后随着次时代（PS3/Xbox 360）引入法线烘焙流程，拓扑规范的内容扩展至低模与高模的拓扑对应关系，形成今日常见的双层规范体系。

在多人协作项目中，拓扑规范的价值在于将"好看不好看"的主观判断转化为"合规不合规"的客观结论。一份完整的规范文档可以让美术主管在不打开Maya/Blender的情况下，仅凭截图与统计数据完成初步验收，将迭代周期从数天压缩至数小时。

## 核心原理

### 1. 多边形类型限制规则

工业规范对多边形类型的管控分三级：**三角面（Tri）**、**四边面（Quad）**、**N-gon（5边以上多边形）**。

- **实时渲染模型**：要求100%为三角面或四边面，严禁N-gon，因为实时引擎（UE5、Unity）在导入时会对N-gon进行自动三角化，切割结果不可控，可能产生非平面三角面导致法线错误。
- **影视/动画高模**：允许N-gon存在于平整表面（如建筑墙面），但凡参与细分平滑（Subdivision Surface）的区域必须保持四边面结构。
- **验收判定**：可在Blender中执行"Select All by Trait → Face Sides → Greater Than 4"快速定位违规N-gon；3ds Max的"STL Check"修改器可一键报告。

极点（Pole）规范同样属于此类别。三角极点（3条边汇聚）与五星极点（5条边汇聚）是可接受的拓扑收束方案；6条以上边汇聚的高价极点在曲面上会造成明显褶皱，须标记为不合规，允许存在于平整非形变区域但不得超过整体面数的**0.5%**。

### 2. 边线流向（Edge Flow）合规判定

边线流向规范是拓扑规范中最难量化但最关键的子项，通常以"结构边是否跟随几何体曲率变化"作为判定依据。

**正向判定标准**：
- 循环边（Edge Loop）必须沿肌肉走向或结构轮廓闭合，不得在中途断裂或产生T型接缝（T-junction）；
- 硬表面转角处的支撑边（Support Edge / Bevel Edge）间距须均匀，工业标准通常为主边两侧各一条支撑边，两支撑边间距离约为转角半径的**1/3到1/4**；
- 关节区域（角色肘部、膝部）必须具备至少**3条连续环线**以确保蒙皮变形品质。

**负向判定标准**（直接不合规）：
- 无法到达任何边界的"孤岛"边线（Stray Edge）；
- 穿越曲率突变区域的直通边（导致细分结果出现脊线）；
- 在动态区域使用三角扇形（Fan Triangulation）替代四边面网格。

### 3. 面数密度均匀性规范

拓扑规范要求模型不同区域的面密度与该区域在屏幕空间或动态重要性上相匹配，这一指标称为**面密度一致性（Poly Density Uniformity）**。

具体量化方法：取模型包围盒对角线长度 $L$，将模型表面划分为若干 $0.1L \times 0.1L$ 的取样窗口，计算各窗口内面数方差。工业标准通常要求非LOD切换区域的面密度方差不超过均值的**±40%**。若某局部区域面密度超过平均值3倍以上，需提出重新布线申请。

UV接缝位置是面密度突变的高发区——美术人员常在UV切割处堆积额外面数来隐藏接缝，但这会违反面密度规范。正确做法是将UV接缝置于视觉盲区（内侧、遮挡处），不增加额外面数。

### 4. 几何体清洁度标准

清洁度规范列出以下**零容忍**错误类型（任意一项出现即整体不通过验收）：

| 错误类型 | 检测命令（Maya） | 检测命令（Blender） |
|---|---|---|
| 重叠面（Overlapping Face） | Mesh Cleanup → Faces with zero map area | 3D Print Toolbox → Intersecting Faces |
| 非流形边（Non-manifold Edge） | Mesh → Cleanup → Non-manifold | Select → Select All by Trait → Non Manifold |
| 内部面（Interior Face） | Mesh Cleanup → Lamina Faces | 3D Print Toolbox → Zero Faces |
| 孤立顶点（Isolated Vertex） | EditMesh → Delete Vertex | Mesh → Merge by Distance |

清洁度检查应作为提交前的**最终自检流程**，在大型项目中通常集成至资产管道的自动化检查脚本（Python/MEL）中，未通过清洁度检查的文件拒绝上传至版本库。

## 实际应用

### 游戏角色资产的拓扑验收流程

以一款主机游戏（目标平台PS5）的次要NPC角色为例：面数规划文档已批准预算为**8,000三角面**。拓扑规范在此基础上进一步规定：

1. 头部区域占用不超过总预算的**30%（约2,400 Tris）**，眼眶、嘴角须保留变形用环线；
2. 手部单只不超过**400 Tris**，五指各由独立边线组定义；
3. 服装与皮肤接合处（领口、袖口）须有**0.02个单位**的间隙面避免Z-fighting；
4. 模型提交格式为FBX 2020，Y轴向上，世界坐标原点位于角色脚底中心，允许误差**±0.001单位**。

美术主管验收时使用Marmoset Toolbag的Wireframe模式进行截图对比，重点审查膝部和肩部的环线结构是否符合绑定师预留的权重影响区域。

## 常见误区

**误区一：认为全四边面（All-Quad）等同于高质量拓扑**

四边面是工具，而非目标。在模型平整区域强行维持四边面有时反而浪费面数，例如一块方形地板贴图区域用两个三角面（一个矩形Quad拆分为Tris）完全合规，没有任何质量损失。拓扑规范允许在平整非形变表面使用三角面，强制追求100% Quad会导致不必要的拓扑复杂性。

**误区二：拓扑规范仅在最终提交前检查**

这是效率最低的工作方式。拓扑规范应在**基础形态（Blockout）阶段结束后**就开始介入，重点检查边线流向与关节区域环线数量是否符合规范。若在精细细节阶段才发现关节环线不足，需要对整个拓扑进行重构，代价远高于早期介入。

**误区三：将规范文档视为一成不变的固定标准**

工业拓扑规范应随项目技术需求迭代。当引擎升级导致法线烘焙算法变更（例如从Mikkt切线空间迁移至其他算法），支撑边的间距标准需相应调整。规范文档必须注明**版本号与生效日期**，历史版本需存档但明确标注为已废弃，避免美术人员使用过期标准生产资产。

## 知识关联

**上游依赖——面数规划**：面数规划确定了各部件的多边形预算上限，拓扑规范则将这个"总量"分解为各局部区域的密度分配规则和结构约束。没有面数规划的数据支撑，拓扑规范中的百分比分配无从依据。

**下游衔接——游戏拓扑管线**：本文的拓扑规范属于单个资产的验收标准，而游戏拓扑管线将多个资产的规范整合为批量处理流程，包括LOD自动生成（如Unreal的Nanite代理网格配置）、骨骼绑定的拓扑前置检查、以及实时碰撞体的拓扑简化规则。理解拓扑规范的单资产逻辑是掌握整条管线自动化节点配置的前提条件。
