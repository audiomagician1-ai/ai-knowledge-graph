# 头部建模

## 概述

头部建模是指在三维软件中，从基础几何体出发，通过精确控制边循环（Edge Loop）走向和面部解剖比例定位，最终构建出具备正确颅骨结构与动画友好拓扑的完整人头模型的全流程技术。该流程覆盖颅顶椭球形态、眼眶凹陷深度、颧弓突起位置、下颌轮廓转折等多个结构层次，要求建模师在几何精度与面部表情变形兼容性之间维持精确平衡。

头部建模的规范化体系形成于1990年代末的影视特效行业。Pixar在制作《怪兽电力公司》（Monsters, Inc., 2001年）期间系统确立了以边循环驱动面部肌肉走向的建模标准，将眼轮匝肌和口轮匝肌的收缩方向直接映射到拓扑布线中。此前的早期三维头部模型（如1995年以前的游戏角色）普遍采用无规律静态网格，仅能支持极为有限的顶点动画变形。这一技术转变在 《The VES Handbook of Visual Effects》（Jeffrey A. Okun & Susan Zwerman, 2010, Focal Press）中有系统性记录，书中明确指出边循环驱动方法将面部动画质量提升了约40%。

由于人类大脑对同类面孔的感知精度极高，头部模型的任何比例偏差都极易触发"恐怖谷效应"（Uncanny Valley），这一现象由日本机器人学家森政弘（Masahiro Mori）于1970年在《Energy》杂志首次量化描述。因此，眼眶与嘴唇周围的环形拓扑结构、颅顶与下颌的面数分配策略，乃至单条边循环的起止位置，都需要以外科手术般的精度加以规划和执行。

## 核心原理

### 黄金比例与面部测量网格

人体面部遵循严格可量化的解剖比例关系。建模时通常以"三庭五眼"规则作为初始硬性约束：面部垂直方向被发际线、眉弓上缘、鼻底三条水平线等分为三段（每段约占面部总高度的33.3%）；水平方向以五个眼宽等分全脸宽度，其中两侧耳缘各占一个眼宽，两眼之间的距离亦等于一个眼宽。

以下是更具体的解剖数值参考，可直接输入三维软件的标尺工具进行核验：

- **眼眶宽高比**：成人眼裂平均宽度约为30–34mm，高度约为10–12mm，宽高比约为3:1
- **颧弓-颅宽比**：颧弓最宽处与头颅最宽处之比约为0.72–0.76
- **鼻尖-下颌距比**：眼眶中心到鼻尖的距离，与鼻尖到下颌底部距离之比约为1.1:1.0
- **耳朵高度对齐**：耳廓顶端与眉弓平齐，耳垂底端与鼻底平齐

在Blender或Maya中导入正面与侧面参考图后，应立即在这些关键位置设置辅助线（Guide Line），并在多视图中同时校验，避免仅在单一视角下调整造成的三维形态失真。

### 边循环驱动的拓扑构建

头部拓扑必须沿面部表情肌的收缩方向布线，而非按均匀网格铺排。主要肌肉组对应的拓扑规则如下：

**眼眶区域（眼轮匝肌，Orbicularis Oculi）**：需形成封闭的环形循环（Orbital Loop），通常包含8至12条边。该循环从上眼睑外眦出发，绕过泪阜内侧，经下眼睑返回，形成完整的椭圆形闭合路径。眼眶环形循环会同时向太阳穴方向和鼻梁方向传播拓扑线，因此它是决定整张脸网格走向的"主干循环"，应在建模流程的第二阶段最优先建立。

**嘴唇区域（口轮匝肌，Orbicularis Oris）**：需独立的环形循环，一般包含6至10条边。上唇丘比特弓（Cupid's Bow）处需要额外的一条短循环来表现双峰形态，嘴唇内侧需要内缩循环（Inset Loop）确保张嘴时嘴角不发生撕裂变形。

**鼻唇沟区域**：鼻翼至嘴角的法令纹路径需要一条独立的斜向循环，其走向角度约为45°–55°（从鼻翼基底到嘴角外侧），这条循环的密度直接影响年龄感的表现能力。

### 面数分配的行业标准

不同用途的头部模型存在明确的面数参考范围：

| 用途 | 拓扑类型 | 面数范围 |
|------|---------|---------|
| AAA游戏 LOD0 | 三角面（渲染用） | 5,000–15,000 tri |
| 手机游戏 | 三角面 | 1,500–4,000 tri |
| 影视低模控制网格 | 四边面（细分前） | 800–2,000 quad |
| 影视细分2级后 | 四边面 | 12,800–32,000 quad |
| 虚拟数字人实时 | 三角面 | 10,000–25,000 tri |

影视低模控制网格依靠Catmull-Clark细分算法（Ron Catmull & Jim Clark, 1978年提出）在渲染阶段达到所需精度，每细分一级面数扩展为原来的4倍，即：

$$F_n = F_0 \times 4^n$$

其中 $F_0$ 为控制网格原始面数，$n$ 为细分级别，$F_n$ 为细分后的面数。例如，一个 $F_0 = 1200$ 的控制网格在 $n=2$ 时得到 $F_2 = 1200 \times 16 = 19200$ 个四边面，已足够支持影视级特写镜头渲染。

## 关键建模流程

### 从基础形状到完整结构的四阶段推进

**第一阶段——颅骨大形**：从一个经过6次Loop Cut（循环切割）的球体或立方体开始，使用仅16至24个四边面勾勒颅顶椭球形态与后枕骨的下压弧度。此阶段绝对不涉及任何面部细节，目标是确立颅高:颅宽:颅深 ≈ 1.0:0.85:1.1 的正确三维比例关系。在Maya中，此阶段对应"粗模检视（Rough Block-In）"步骤，通常耗时30–60分钟。

**第二阶段——面部功能区划分**：在眼眶位置插入两个凹陷的环形面循环，在鼻翼两侧定义鼻孔-上唇边界，在下颌底部插入下颌体线。三处插入的优先顺序为：眼眶 → 嘴唇 → 鼻翼，因为眼眶循环建立后会自动约束鼻梁与太阳穴的网格流向，先处理眼眶可减少后期大规模重拓扑（Retopology）的概率。

**第三阶段——五官形体雕刻**：使用Extrude（挤出）和Inset（内插面）操作推进眼睑的上下弧度（上眼睑峰点位于眼宽约60%处，而非正中央）、鼻梁侧面收窄（鼻根宽约为鼻翼宽的55–60%）、上下唇的体积翻转关系（下唇体积通常略大于上唇，比例约为1.2:1.0）。

**第四阶段——下颌与颈部过渡**：下颌角处的面数需从面部较密拓扑（约每厘米3–4条边）平滑过渡到颈部较稀拓扑（约每厘米1–2条边），通常通过2至3次"Dissolve Edge（融合边）"操作完成减面，并在胸锁乳突肌（Sternocleidomastoid）的前缘位置插入一条纵向循环，确保头部转动动画时颈部侧面不塌陷。

## 实际应用

### 案例：游戏角色头部的完整制作流程

以一个AAA规格的写实男性角色头部为例，说明具体的软件操作参数与时间分配：

1. **参考图设置（Blender）**：在前视图导入正面参考图（分辨率建议2048×2048），在右视图导入侧面参考图，使用`Image as Plane`导入后锁定其位置不参与变换操作。正侧两张参考图的眼睛高度必须对齐至同一Y坐标，误差不超过±1个像素。

2. **低模建模（Maya/Blender）**：控制网格四边面数量控制在1,200–1,500个，耗时约3–5小时（有参考图情况下）。

3. **高模雕刻（ZBrush）**：在ZBrush中对导入的低模执行4级细分（得到约19,200–24,000个面），使用ClayBuildup笔刷添加颧骨突起与眉弓细节，使用Standard笔刷雕刻法令纹和眼角鱼尾纹（深度约0.3–0.8mm的表面置换）。

4. **法线贴图烘焙（Marmoset Toolbag / Substance Painter）**：将高模细节烘焙至4096×4096像素的法线贴图（Normal Map），最大化存储皮肤毛孔（直径0.05–0.2mm）、细纹等无法在游戏低模中直接表现的微观几何信息。

```python
# Blender Python API：批量检测头部网格中的非四边面（用于拓扑质量控制）
import bpy
import bmesh

obj = bpy.context.active_object
bm = bmesh.from_edit_mesh(obj.data)

non_quad_faces = [f for f in bm.faces if len(f.verts) != 4]
tri_faces = [f for f in bm.faces if len(f.verts) == 3]
ngon_faces = [f for f in bm.faces if len(f.verts) > 4]

print(f"总面数: {len(bm.faces)}")
print(f"三角面数: {len(tri_faces)}（应控制在0，集中于耳后和颅顶）")
print(f"多边面(N-gon)数: {len(ngon_faces)}（头部建模中应为0）")
print(f"四边面占比: {(len(bm.faces)-len(non_quad_faces))/len(bm.faces)*100:.1f}%")

bmesh.update_edit_mesh(obj.data)
```

例如，在上述脚本检测后，若发现三角面集中出现在太阳穴区域，通常意味着眼眶循环与耳前循环的汇合处拓扑处理不当，需要通过插入一条从太阳穴到耳廓上方的过渡循环来消解三角面。

## 常见误区

### 误区一：眼眶环形循环方向接反

初学者常将眼眶循环的边方向设置为"以眼睛为圆心向外辐射"而非"绕眼眶圆周闭合"。前者会导致眼睑闭合时（Blink Blend Shape）出现明显的放射状拉伸变形，而后者才能支持平滑的上下眼睑滑动。**判断方法**：在Maya中选中眼眶周围的所有面，执行"Select Edge Loop"，若无法形成一个完整的闭合环而是出现断裂，则说明方向设置有误。

### 误区二：过早追加面数

在颅骨大形未确定之前就追加面数是最常见的时间浪费行为。正确做法是在颅骨大形阶段（第一阶段）将整个头部压缩到20–30个四边面，从正面、侧面、顶面、底面四个视角确认比例完全正确后，再进入第二阶段添加循环。否则比例问题会随着面数增加而越来越难以修正，因为移动少量顶点对整体形态的影响范围会大幅收窄。
