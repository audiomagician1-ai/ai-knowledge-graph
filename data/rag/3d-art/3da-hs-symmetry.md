# 对称与阵列

## 概述

对称（Mirror）与阵列（Array）是硬表面建模中两类专门用于减少重复劳动的修改器工具。对称修改器通过在指定轴向上镜像复制网格数据，使模型的两半或多侧保持几何完全一致；阵列修改器则按照固定偏移、缩放或旋转规律批量生成相同的网格实例，两者共同构成硬表面流程中"只建一半、自动补全"的核心加速手段。

这两类工具在三维软件发展早期便已存在，Blender 2.80 版本重构工作流后将它们统一纳入非破坏性修改器堆栈（Non-Destructive Modifier Stack），允许建模者在最终应用（Apply）前随时调整参数、撤销或重新排序。3ds Max 将对应功能命名为 Symmetry Modifier 和 Array Tool，Maya 则通过 Mirror Geometry 和 Duplicate Special 实现类似效果，但 Blender 的修改器堆栈方式因保留原始数据而更受硬表面艺术家青睐。

在硬表面建模场景中，机甲、枪械、汽车、头盔等对象几乎100%含有左右或旋转对称结构。若不使用这两类修改器，建模者必须手动同步两侧网格的每一次顶点移动，稍有疏漏便会导致焊缝不对称或法线断裂，严重拖慢迭代速度。掌握这两个修改器可将具有对称特征的硬表面模型制作效率提升40%–70%。

## 核心原理

### Mirror 修改器的镜像轴与融合阈值

Mirror 修改器在 Blender 中提供 X、Y、Z 三个轴向的独立开关，可以同时勾选多轴以实现四分之一建模甚至八分之一建模。其工作原理是：以物体的原点（Origin）为对称中心，将现有网格按选定轴翻转坐标后生成镜像副本，副本本身不占用独立的对象槽位，只在修改器层级计算。

**融合阈值（Merge Threshold）**是 Mirror 修改器最容易被忽视的关键参数。当原始网格边缘顶点与镜像轴的距离小于该阈值（默认值 0.001 米）时，两侧顶点会被自动焊接为同一顶点，消除中线处的接缝。若阈值设置过大，远离中轴的顶点也会被错误焊接；若阈值过小，中线顶点保持分离，应用修改器后将出现可见的黑色断缝。在实际操作中，建模前应将物体缩放清空（Ctrl+A > Scale），确保世界空间单位与融合阈值匹配。

**裁剪（Clipping）**选项与融合阈值配合使用：开启后，位于镜像平面附近的顶点在编辑模式中无法被拖拽穿越对称轴，强制维持模型的连续性，防止两半网格在编辑过程中错位分离。

### Array 修改器的三种偏移模式

Array 修改器提供三种决定阵列间距的偏移类型，理解其差异是正确使用的前提：

| 模式 | 计算方式 | 典型用途 |
|------|----------|----------|
| 固定偏移（Constant Offset） | 每个副本在X/Y/Z方向上增加固定的绝对距离（单位：米） | 等间距螺栓排列、散热孔列 |
| 相对偏移（Relative Offset） | 偏移量 = 物体包围盒尺寸 × 系数 | 自适应于物体大小的重复结构 |
| 物体偏移（Object Offset） | 以另一个物体的变换差值作为每步偏移 | 沿曲线路径排列、旋转阵列 |

对于需要环形排列的结构（如涡轮叶片、枪管散热孔环形阵列），需将"物体偏移"模式与一个空对象（Empty）配合：将 Empty 放置在旋转中心，赋予其每步旋转角度（例如叶片数量为9片时，每步旋转 360°÷9 = 40°），再将 Array 修改器的 Object Offset 指向该 Empty，即可获得精确的环形排布。

### 修改器堆栈顺序对结果的影响

在 Blender 中，修改器按从上到下的顺序依次执行，顺序颠倒会导致截然不同的结果。以"先 Array 后 Mirror"与"先 Mirror 后 Array"为例：若先对一个螺栓应用 Array 沿 X 轴生成6颗，再 Mirror 沿 Y 轴镜像，最终得到12颗关于 Y 轴对称的螺栓；反之若先 Mirror 再 Array，则先生成关于 Y 轴对称的2颗，再沿 X 轴阵列为12颗，两者排布方式完全不同。此外，Subdivision Surface（细分曲面）修改器通常放在 Mirror 之后、Array 之后，确保焊接完成后才进行平滑细分，否则中线接缝处会出现错误的高光。

## 实际应用

**机甲头盔制作**是 Mirror 修改器最典型的应用场景。建模者只需雕刻头盔左半侧的装甲板、通风格栅和传感器凸起，Mirror 修改器实时生成右侧对应结构。当左侧调整完毕后，执行 Apply 将修改器烘焙进网格，再手工添加左右不对称的细节（如单侧天线、不同徽章），形成"对称基础 + 非对称细节"的专业硬表面工作流。

**散热格栅建模**是 Array 修改器的经典案例。首先建一条宽3mm、厚1mm的单片格栅条，使用 Constant Offset 模式设置Y轴偏移4mm（含1mm间隙），Count设为20，即可瞬间生成20片等间距格栅。若格栅需要装配在弯曲表面上，可叠加 Curve 修改器使阵列沿路径弯曲，无需手动逐片调整角度。

**涡扇发动机叶片**是 Object Offset 阵列的代表案例：单片叶片围绕引擎中轴线按 360°/18 = 20° 间隔排列18片，仅需制作1片基础叶片即可。调整单片叶片的扭转角度时，所有副本同步更新，大幅简化了叶片角度迭代过程。

## 常见误区

**误区一：应用 Mirror 修改器前忘记清除缩放**
当物体在物体模式下被非均匀缩放（如 X 轴缩放为 2、Y 轴缩放为 1）且未执行 Ctrl+A 清除缩放时，Mirror 修改器所参考的本地轴方向与视觉方向不一致，导致镜像结果偏斜或沿错误轴翻转。解决方法是在添加 Mirror 修改器之前始终执行 Apply Scale，确保所有轴向缩放值归1。

**误区二：将 Array 的 Count 数量与实际可见数量混淆**
Array 修改器的 Count 参数表示总副本数量，包含原始物体本身。Count = 1 时只显示原物体，没有任何阵列；Count = 6 时显示原物体加5个副本，共6个。初学者常误以为 Count = 6 会在原物体之外额外生成6个，造成数量比预期多出1个的布局错误，在制作等分孔洞时尤为明显。

**误区三：在 Mirror 修改器已应用后才发现中线顶点未焊接**
中线未焊接的表现是：应用 Mirror 后，在边缘选择模式下可以看到中线处存在两条重叠的边，渲染时该区域出现黑色条纹。预防方法是在应用前在编辑模式下临时关闭修改器可见性，检查原始网格中线顶点是否已被拉到距对称轴 < 0.001 米的位置，或适当提高 Merge Threshold 数值。

## 知识关联

学习对称与阵列修改器需要建立在**硬表面建模概述**的基础上，尤其是对"物体原点（Origin）"与"世界原点"的区分理解——因为 Mirror 修改器的对称轴永远穿过物体自身原点，若原点位置错误，镜像结果将完全偏离预期位置。同时，"非破坏性工作流"的意识是正确使用修改器堆栈的前提，避免在完成布线优化之前就过早 Apply 修改器。

在进阶方向上，对称与阵列修改器是**曲面细分建模（Subdivision Modeling）**和**硬表面高模烘焙**流程的直接上游工序：高模制作阶段通常保留 Mirror 和 Array 修改器处于未应用状态，只在导出前统一 Apply，以便随时对称调整支撑循环（Support Loop）的位置。此外，掌握 Object Offset 阵列后，艺术家可以自然衔接 Blender 的**曲线路径（Curve Path）**工具，实现沿任意路径排列硬表面零件的高级技巧。
