# 法线烘焙

## 概述

法线烘焙（Normal Map Baking）是将高模（High-Poly Mesh）表面的凹凸几何细节转换为切线空间法线贴图（Tangent-Space Normal Map）的技术流程。其核心操作是：将高模每个采样点的表面法向量，从世界空间转换至低模（Low-Poly Mesh）对应UV点的切线空间坐标系中，并将三个分量（X、Y、Z）分别编码至贴图的R、G、B通道，最终存储为一张RGB图像文件。

这项技术最早随着第七世代主机（Xbox 360、PS3，约2005年前后）的普及而大规模应用于游戏行业，用于在多边形数量受限的实时渲染模型上还原高精度雕刻细节。切线空间法线贴图相较于世界空间法线贴图的显著优势在于：贴图可随模型旋转、对称复用，一张法线贴图可以同时应用于左右对称的模型两侧而不产生光照错误。

法线烘焙之所以区别于其他烘焙类型（如AO烘焙、曲率烘焙），在于它直接影响实时渲染的光照计算结果——渲染器使用法线贴图中存储的向量替代几何法线，参与镜面反射（Specular）和漫反射（Diffuse）的逐像素计算，因此法线烘焙的精度直接决定模型在各角度光照下的表面质感。

## 核心原理

### 切线空间与TBN矩阵

法线烘焙的数学基础是TBN矩阵（Tangent-Bitangent-Normal Matrix）。每个低模顶点都拥有一个局部切线空间坐标系，由切线向量T（沿UV的U方向）、副切线向量B（沿UV的V方向）、顶点法线N三个正交向量构成。烘焙时，高模采样点的世界空间法线向量 **n_world** 通过TBN矩阵的逆变换投影至切线空间，得到切线空间法线 **n_tangent**，再将其分量从 [-1, 1] 映射至 [0, 1] 存入贴图：

```
存储值 = (n_tangent + 1.0) / 2.0
```

因此，切线空间法线贴图的"平坦"区域（法线垂直于表面）对应RGB值为 (128, 128, 255)，呈现为标志性的蓝紫色。

### 投影方式与射线发射

烘焙器从低模表面沿法线方向向外发射射线（Ray），与高模相交并采样法线信息。射线的发射策略直接影响烘焙质量。常见设置包括：

- **内/外偏移距离（Ray Distance / Cage Offset）**：控制射线起点距低模表面的偏移量，过小会导致低模自遮挡（Self-Intersection），过大会采样到错误的高模区域。在 Marmoset Toolbag 中此参数称为"Front / Rear Distance"，典型取值依模型尺寸而定，常见为模型最大尺寸的 0.01 至 0.05 倍。
- **平均法线 vs 顶点法线**：烘焙器可使用低模的平滑法线（Averaged Normals）作为射线方向，使烘焙结果在曲面过渡处更平滑，避免因低模硬边导致的接缝。

### 法线贴图的方向约定（OpenGL vs DirectX）

切线空间法线贴图存在两种行业标准，区别仅在于G通道（副切线方向Y轴）的正负翻转：

- **OpenGL 约定（+Y朝上）**：Unity、Blender 默认使用此标准，G通道高值对应表面向上凸起。
- **DirectX 约定（-Y朝上）**：Unreal Engine、早期 xNormal 默认使用此标准，G通道高值对应表面向下凹入。

两种格式在视觉上的表现为：相同的凸起细节，G通道的明暗关系完全相反。若引擎与贴图约定不匹配，模型在侧光下会呈现凹凸倒置的视觉错误。

## 实际应用

**游戏资产制作流程示例**：以角色装甲为例，典型流程如下：

1. ZBrush中雕刻高精度装甲纹理（多边形数可达 2000万+），导出 OBJ 或 FBX 高模。
2. 在 Maya 或 Blender 中制作游戏用低模（目标面数约 5000 三角面），展开无重叠UV（UV Padding 建议留出 texel 大小的 2 倍间距，防止边缘溢色）。
3. 在 Marmoset Toolbag 4 或 Substance Painter 中导入高低模，配对同名 High/Low 网格，设置切线空间为 MikkTSpace（与 Unreal Engine 5 使用相同切线空间标准，保证引擎内结果一致）。
4. 输出分辨率通常为 2048×2048 或 4096×4096，16位EXR格式可保留更高精度，但游戏引擎导入时通常压缩为 BC5 格式（仅存储R、G通道，B通道由着色器实时重建）。

**拼合模型的硬边处理**：对于由多块硬表面拼合的模型（如枪械），需在接缝处于低模UV边界对应位置设置硬边（Hard Edge），并确保UV岛屿（UV Island）边界与硬边一一对应，否则法线贴图在接缝处会出现明显的明暗断裂。

## 常见误区

**误区一：烘焙结果中的蓝色区域代表质量问题**

初学者常误认为法线贴图中大面积的纯蓝色（RGB 128, 128, 255）是烘焙失败的标志。实际上，纯蓝色代表该区域法线方向与低模几何法线完全一致，即高模在此处没有额外细节需要编码，是完全正常的结果。真正的烘焙问题表现为黑色斑块、锯齿状硬边、或颜色突变的接缝线。

**误区二：高低模必须完全重叠才能烘焙正确**

部分教程强调高低模需要"完美对齐"，导致初学者花费大量时间调整模型位置。实际上，烘焙器的射线偏移（Cage/Ray Distance）正是为了处理高低模体积差异而设计的。高模体积可以略大于低模，只要偏移值设置合理，烘焙器仍能正确采样。真正的刚性要求是：高低模必须使用**完全相同的切线空间算法**（如统一使用 MikkTSpace），否则即便视觉看起来正常，在引擎中受光照时仍会出现接缝。

**误区三：法线贴图分辨率越高越好**

将输出分辨率从 2K 提升至 8K 并不线性提升质量，瓶颈通常在于低模的UV展开精度（texel density）。若低模某块区域UV空间占比极小，即使使用 8K 贴图，该区域实际可用的像素数量仍然极少，倒不如优化UV布局后使用 2K 烘焙。此外，BC5/BC7 等引擎内压缩格式会对高频法线细节造成损失，超高分辨率在压缩后的实际收益需权衡。

## 知识关联

**前置知识**：烘焙概述确立了高低模配对、UV展开等基础概念，法线烘焙在此基础上引入了切线空间变换这一特有数学层。理解顶点法线的插值方式（Gouraud Shading）有助于理解为何硬边设置会影响法线烘焙的接缝表现。

**Cage设置**是法线烘焙参数调整的直接延伸：当默认射线偏移无法解决复杂几何体的自遮挡问题时，需要手动编辑 Cage Mesh 来精确控制每个区域的射线起始包络，是法线烘焙质量优化的进阶手段。

**烘焙瑕疵**专门讨论法线烘焙过程中的常见错误（如接缝、暗斑、斜面渐变错误），是对本文"常见误区"的系统化扩展。**Substance烘焙**、**Marmoset烘焙**和**xNormal烘焙**则分别是在三个具体软件环境下执行法线烘焙流程的操作细节，各软件的切线空间选项和Cage实现方式存在差异，需结合本文原理进行对比学习。
