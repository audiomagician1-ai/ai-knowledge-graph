# 投影展开

## 概述

投影展开（Projection Unwrap）是UV展开中最基础的自动化方式，其核心思路是将三维网格从特定方向"投影"到假想的几何体表面，再将该几何体"展开"成平面，从而获得UV坐标。与手动缝合展开不同，投影展开不需要艺术家逐条标记缝合线，软件按预设的几何形状自动计算每个顶点的UV位置，速度极快，适合对拉伸要求不高的资产。

投影展开的概念直接来源于传统制图学中的地图投影技术。地图制作者将球形地球"压平"成纸质地图时，早在15世纪就系统研究了平面投影、圆柱投影（如墨卡托投影）和球形投影各自的形变特征。三维软件将同一套数学原理迁移到游戏和影视资产的UV生成流程中，Autodesk Maya、3ds Max、Blender等主流工具均将平面/圆柱/球形/盒形四种投影作为标准内置操作。

投影展开的价值在于它的"快速近似"特性——对于地形、建筑墙面、环境道具等大量不会被玩家近距离审视的资产，投影展开能以几秒钟的操作获得可用的UV布局，节省大量手动展开时间。但一旦模型形状与所选投影类型差异过大，UV就会产生严重拉伸，因此正确选择投影类型是使用这一工具的关键技能。

## 核心原理

### 平面投影（Planar Projection）

平面投影从单一方向将整个网格的顶点垂直投射到一个平面上，所有顶点的UV坐标由其在投影方向上的X、Y轴偏移量决定，公式为：

```
u = (x - x_min) / (x_max - x_min)
v = (y - y_min) / (y_max - y_min)
```

其中x、y是顶点在投影平面坐标系中的位置，x_min/x_max、y_min/y_max是网格的包围框边界。平面投影对于接近平面的几何体效果极佳，例如地面、墙壁单面、旗帜布料。最严重的问题发生在与投影方向接近平行的面（即侧面和背面），这些面的UV会被极度压缩，形成"拉伸条带"（Stretching Streaks）。3ds Max中该操作对应 **UVW Map Modifier → Planar**，Blender中为 **Project from View**。

### 圆柱投影（Cylindrical Projection）

圆柱投影将网格顶点从轴心向外映射到一个假想圆柱的侧面，再将圆柱沿一条竖线剪开展平。每个顶点的U坐标由其绕轴旋转角度θ决定（u = θ / 2π），V坐标由沿轴高度决定（v = h / H）。圆柱投影对于瓶子、树干、柱子、角色手臂、腿部等旋转体状网格最为合适，通常能将拉伸控制在可接受范围内。圆柱的两个端盖（顶面和底面）不参与侧面映射，会产生强烈收缩，需要额外用平面投影或手动处理。在Maya中，圆柱投影有专属的"Sweep Angle"（默认360°）和"Cap"选项控制端盖处理方式。

### 球形投影（Spherical Projection）

球形投影将顶点映射到一个假想球体表面，使用经纬坐标（longitude/latitude）计算UV，与地球经纬度系统数学等价：u = (arctan2(z, x) + π) / 2π，v = arccos(y / r) / π。该投影最适合接近球形的物体，例如眼球、行星、圆球形果实。球形投影在两极（即UV的v=0和v=1边界）存在极点收缩问题，所有顶点汇聚为一个点，形成明显的"钉点"（Pinching），使用时需将极点置于视觉上不显眼的位置，或在建模阶段减少极点附近的面数。

### 盒形投影（Box/Cubic Projection）

盒形投影等价于对六个方向同时应用平面投影，软件根据每个多边形法线方向将其分配给最接近的那个面（+X、-X、+Y、-Y、+Z、-Z中的一个），再各自执行平面投影。六个投影共享一张贴图，UV岛被统一打包进0到1的UV空间。盒形投影对建筑外墙、方形机械零件、箱子、岩石等具有六面体特征的资产效果突出，与Tri-planar Shader配合使用时可以完全消除接缝。Blender的 **Smart UV Project** 实质上是盒形投影的扩展版本，支持角度阈值（Angle Limit，默认66°）控制分组精度。

## 实际应用

**游戏地形贴图**：在UE5的地形系统中，高度图生成的Landscape网格默认使用平面投影（从+Z方向），一块256×256分辨率的地形贴图直接以世界坐标映射，1个纹素对应100cm，完全依赖平面投影的均匀分布特性。

**角色手臂绑定前UV**：为人形角色手臂创建UV时，圆柱投影通常作为第一步。以Maya为例，选中手臂网格，设定圆柱轴向为X轴（沿手臂方向），Sweep Angle保持360°，可在5秒内生成基础UV，再针对手腕弯曲部分手动修正拉伸，整体节省约60%的展开时间。

**道具快速迭代阶段**：在概念美术阶段，道具建模完成后需要快速上色验证，此时对一个水壶使用盒形投影，配合无缝材质球，可以在10分钟内看到完整上色效果，而不需要等待正式UV展开完成。

**天空球（Skydome）贴图**：全景HDRI贴图应用于天空球时，球形投影是唯一正确选项。天空球模型通常是一个内翻法线的UV球体（Blender默认UV球经纬线为32段×16段），球形投影的经纬坐标与HDRI的等距柱状投影（Equirectangular）格式完全对齐，接缝位于玩家视线通常无法达到的正后方。

## 常见误区

**误区一：以为盒形投影没有接缝**。盒形投影的六个面之间存在接缝，只是接缝位于面与面的夹角转折处，视觉上被遮挡或模糊。若模型某一边有45°斜面，该斜面会在两个相邻投影方向的边界处产生明显的UV断裂拉伸，不经过手动调整无法消除。Tri-planar Shader是在着色器层面混合三个方向的采样来消除接缝，而非投影展开本身解决了接缝问题。

**误区二：圆柱投影适合所有有"弧度"的物体**。弯曲的香蕉、L形管道这类轴向本身弯曲的物体不适合单一圆柱投影，因为圆柱投影假设中心轴是一条直线。对弯曲管道强行使用圆柱投影，弯曲外侧UV会被拉伸约30-50%，内侧压缩同等幅度。正确做法是沿路径分段使用多个圆柱投影，或切换为手动缝合展开。

**误区三：投影展开后UV像素密度自动均匀**。投影展开只保证被投影面（法线接近投影方向的面）像素密度较均匀，与投影方向平行的面的UV被极度压缩，像素密度接近0。一个底面朝向摄像机的圆柱体侧面用圆柱投影后密度正常，但顶盖和底盖的UV（被映射到圆形）像素密度仅为侧面的约30%，若需要顶盖清晰纹理，必须单独处理。

## 知识关联

**前置概念——UV展开概述**：理解UV空间是0到1的正方形坐标系、UV岛的概念以及拉伸的含义，是判断投影展开质量的必要基础。不了解UV坐标系就无法解读投影公式中u、v值的物理含义。

**后续概念——展开工具**：Maya的Unfold3D、Blender的Unwrap（基于LSCM算法）等展开工具通常在投影展开之后使用，以投影结果作为初始UV状态，进一步用弹簧模拟或保角映射算法减少拉伸——投影展开的合理初始化质量直接决定了展开工具的优化效果和迭代次数。

**横向关联——接缝标记**：盒形投影和圆柱投影产生的接缝位置是固定的，而手动缝合展开通过Mark Seam（Blender）或Cut（Maya）让艺术家自定义接缝位置。两种方式在同一个软件工程中可以混合使用，例如对建筑主体用盒形投影，对装饰圆柱体用圆柱投影，对复杂有机部件手动标记缝合线，分区域完成UV布局。
