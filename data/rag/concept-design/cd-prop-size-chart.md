# 尺寸对比图

## 概述

尺寸对比图（Size Comparison Sheet，也称 Scale Reference Sheet）是道具设计流程中专门用于标注道具与参照物之间相对大小关系的工程性图纸。它以视觉化方式将道具放置在角色身旁或与其他道具并排，通过同一画面内的相对高度、宽度和体积差异，向后续制作环节（建模师、道具制作师、动画师）传达准确的尺度信息。

尺寸对比图作为独立图纸类型，在20世纪中期的好莱坞电影道具部门开始被系统性使用。迪士尼动画工作室在1950年代的《仙履奇缘》（Cinderella，1950）制作期间，便已形成一套将道具与标准角色高度对照的规范流程，要求每件关键道具必须注明其相对于6英尺（约183cm）参考人体的比例关系。游戏行业则在PlayStation时代（1994年后）因多边形预算限制，将尺寸对比图纳入美术技术文件的强制交付物。

尺寸对比图的价值在于消除制作团队内部因"大小感"主观理解不同而产生的错误。如果仅凭概念草图和文字说明（如"一把中等大小的剑"），不同建模师制作出的剑可能从60厘米到120厘米不等，与角色组合后比例失调，导致返工成本剧增。根据游戏美术制作管线文献《The Game Concept Art Workflow》（Solarski, 2012）的记录，缺少尺寸对比图的项目中，道具类资产的一次性验收通过率比有完整比例文件的项目低约40%。

## 核心原理

### 参照基准的选择

尺寸对比图必须选定一个固定参照基准，所有道具的相对大小都以此为锚点。最常用的基准是**项目标准角色（Turnaround Reference Character）**，通常取项目中身形居中的主角，将其正面站姿设为100%高度单位。部分工作室（如育碧蒙特利尔工作室在《刺客信条：奥德赛》制作中）采用固定的"1.80米成年男性剪影"作为中性参照，以蓝色填充区分于道具本身，避免特定角色体型变更时引发全套图纸连锁更新。

若道具体量过小（如戒指、钥匙等小件），仅与全身角色对比会使道具在图中缩至辨认困难，此时需增加一个**局部放大对比窗（Detail Scale Insert）**：在主对比图旁附加一个以手掌为参照的近景对比。人体手掌宽度约为身高的1/9，例如身高180cm的角色手掌宽约20cm，可作为小道具的过渡基准单位。这一惯例在皮克斯动画工作室的美术文件规范手册（Pixar's Art Production Bible，内部文件，2008年修订版）中有明确描述。

### 标注规范与数值体系

专业尺寸对比图须同时包含**相对比例**和**绝对尺寸**两套数值。相对比例标注格式为道具高度与角色身高的倍数关系，绝对尺寸则以厘米（或英寸）标注每个部件的实测设计值。仅有相对比例而无绝对尺寸，会导致实物道具制作师无从下手；仅有绝对尺寸而无对比图，则无法直观判断道具与角色的视觉协调性。

下面的公式描述了道具在图纸上的绘制高度与实际设计高度之间的换算关系：

$$H_{draw} = \frac{H_{prop}}{H_{char}} \times H_{char\_draw}$$

其中：
- $H_{draw}$：道具在图纸上的绘图高度（单位：像素或毫米）
- $H_{prop}$：道具的实际设计高度（单位：cm）
- $H_{char}$：角色的实际设计身高（单位：cm）
- $H_{char\_draw}$：角色在该图纸上的绘图高度（单位：像素或毫米）

例如，角色身高设定为180cm，在图纸上绘制为360像素高；一把长剑设计长度为105cm，则其在图纸上应绘制为 $\frac{105}{180} \times 360 = 210$ 像素，即角色绘图高度的58.3%。

绘制时，参照角色和道具必须**处于同一地平线（Ground Line）**，即所有对象的脚底或底部对齐于同一条基准线，而非顶部对齐。顶部对齐会虚假压缩高个角色与矮道具之间的相对差异，使读图者误判道具尺度。

### 多道具并排的排列逻辑

当一张图需要展示系列道具（如一整套武器库或一个角色所有随身物品）时，排列顺序遵循**从高到低、从左到右**的视觉习惯，形成阶梯状轮廓线，方便读图者快速扫描各件道具之间的层级关系。同一功能类别的道具（如三把不同尺寸的刀）应彼此紧邻排列，通过负空间（间距）与其他类别隔开。角色剪影通常放置在最左侧或中间位置，充当视觉锚点。

当系列道具数量超过8件时，建议将尺寸对比图拆分为多张，每张聚焦一个功能类别（近战武器、远程武器、日用道具等），避免图面过于拥挤导致标注线交叉难以阅读。

## 关键公式与绘制规范

除上文的绘图换算公式外，道具设计师还常用**头身比换算规范**来快速判断道具与角色的视觉协调性。动画行业通常以"头身比（Head-to-Body Ratio）"描述角色体型，标准写实人体为7~7.5头身，美式英雄漫画风格为8~8.5头身，日式动画少年角色常见5~6头身。

头部高度 $H_{head}$ 与角色总身高 $H_{char}$ 的关系为：

$$H_{head} = \frac{H_{char}}{n}$$

其中 $n$ 为头身比数值。以6头身、身高150cm的动画少女角色为例，其头高为25cm，手掌宽约为 $150 \div 9 \approx 16.7$cm。若为该角色设计一把魔法棒，棒长设定为90cm（即3.6个头高），在视觉上会呈现出远超角色身高一半的夸张感，符合魔法少女类型的道具夸张化设计惯例。

以下是一段用于自动计算并输出尺寸对比图标注数据的 Python 伪代码示例，体现数字化管线中尺寸对比图的应用：

```python
# 尺寸对比图标注数据生成工具（示意）
def scale_comparison(char_height_cm, char_draw_px, props: dict):
    """
    char_height_cm: 角色实际设计身高（cm）
    char_draw_px:   角色在图纸上的绘图高度（px）
    props:          道具字典 {道具名称: 实际高度(cm)}
    返回每件道具的绘图高度及占角色身高百分比
    """
    results = {}
    for name, height_cm in props.items():
        draw_px = (height_cm / char_height_cm) * char_draw_px
        ratio_pct = round((height_cm / char_height_cm) * 100, 1)
        results[name] = {
            "绘图高度(px)": round(draw_px, 1),
            "占角色身高比例": f"{ratio_pct}%"
        }
    return results

# 示例调用
props = {"短剑": 60, "长剑": 105, "双手巨剑": 155}
output = scale_comparison(char_height_cm=180, char_draw_px=360, props=props)
# 输出: 短剑→120px(33.3%), 长剑→210px(58.3%), 巨剑→310px(86.1%)
```

## 实际应用

### 武器道具案例

设计一套奇幻RPG游戏中的剑类武器时，设计师以角色（身高180cm设定，头身比7.5）为基准，绘制三件道具的并排对比图：短剑全长60cm（剑柄14cm＋护手4cm＋剑刃42cm，占角色身高33.3%）、长剑全长105cm（剑柄22cm＋护手8cm＋剑刃75cm，占58.3%）、双手巨剑全长155cm（剑柄32cm＋护手10cm＋剑刃113cm，占86.1%）。

图中清晰可见巨剑在站立持握时剑尖超过角色头顶17cm，这一视觉信息立即提示动画师该武器需要特殊的持握和攻击动作，无法像长剑那样单手平持或在室内狭窄空间内正常挥击。此类信息若仅靠文字描述"巨剑较长"，难以传达给动画绑定师和关卡设计师。

### 日用道具案例

在动画片道具设计中，角色的书包高度若设计为角色身高的35%（以身高130cm的小学生角色为例，书包高约45.5cm），在实际成品中会呈现出书包几乎覆盖角色整个背部的视觉效果，符合儿童动画中书包"压垮小孩"的夸张喜剧感；若将书包压缩至身高20%（约26cm），则更贴近现实写实风格。两个方案在单独的草稿阶段难以直观判断，但通过尺寸对比图并排展示，创作总监可以在5秒内做出风格取舍决策。

### 影视道具制作案例

影视实物道具部门同样依赖尺寸对比图。《指环王》（2001-2003）道具总监 Richard Taylor（维塔工作室）在制作"安都利尔"（Andúril）宝剑时，以演员Viggo Mortensen身高180cm为基准，将剑全长定为114.3cm（约等于演员身高的63.5%），使剑尖在单手持握、手臂自然下垂时距地面约15cm，满足拍摄时不拖地但视觉上显得威严的双重需求。这一决策正是通过1:1比例的尺寸对比图在前期确认的，而非在道具制作完成后才与演员比对。

## 常见误区

**误区一：顶部对齐而非底部对齐**
将道具与角色的顶部（头顶与道具最高点）对齐，而非底部基线对齐。这会让一件实际高度只有角色60%的道具，看起来与角色"等高"，严重误导建模师的比例判断。正确做法是所有对象底部置于同一Ground Line上。

**误区二：忽略道具的厚度和宽度标注**
尺寸对比图仅标注高度，忽略道具的最大宽度和厚度（特别是三维体量感强的道具如盾牌、箱子、陶罐）。一面盾牌的正面宽度与厚度同样影响角色持握时的遮挡面积和运动空间，仅凭高度数据建模会导致盾牌过厚或过薄。解决方案是在高度对比图旁附加侧视图宽度标注。

**误区三：使用非项目标准角色作为参照**
以一个只在某一章节登场的NPC或特殊体型角色作为比例基准，导致当该参照角色设定调整时，所有依赖它的道具尺寸对比图全部失效，需要重绘。专业流程中，比例参照角色必须是全程冻结设定的"基准人（Scale Master）"，任何对其身高的修改都需要走正式变更审批流程。

**误区四：数字资产与纸质图纸比例不同步**
在数字化制作管线中，设计师在Photoshop/Procreate中绘制的尺寸对比图，其画布坐标中的像素高度比例与3D软件（Maya/Blender）中的场景单位（Unit）不统一，导致建模师导入参考图时比例偏差。最佳实践是在3D软件中直接创建一个与角色等高的参考平面，将尺寸对比图贴图映射上去后再进行建模，确保视口参考与建模坐标系一致。
