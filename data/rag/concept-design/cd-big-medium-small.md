# 大中小形状

## 概述

大中小形状（Large-Medium-Small Shapes，简称 LMS 原则）是构图设计中用于组织画面块面的尺度分配原则：将画面中所有可辨识的形状按照大（Large）、中（Medium）、小（Small）三个层级进行有意识的比例分配，使剪影轮廓产生节奏感与视觉趣味。该原则的核心要求是三个尺度之间必须存在**肉眼可见的显著落差**，而非均匀分布——若大形与中形面积比值低于 2:1，视觉层次将发生坍塌，画面趋于平淡。

该原则的理论根源可追溯至两条独立的历史线索。其一是日本传统花道（池坊流，成立于1462年）中的"天地人"三段构成——天枝最高代表主导形，地枝居中，人枝最短，三者高度比约为 7:5:3，这一比例本身便是不均等尺度分配的早期文字化表述。其二是19世纪英国批评家约翰·罗斯金（John Ruskin）在1843年出版的《现代画家》第一卷（*Modern Painters, Vol.1*）中对"画面质量"（Quantity）的论述：罗斯金明确批评画面中尺寸相近的形状会产生"令人疲倦的重复"（fatiguing repetition），并以威尼斯画派为正面案例——提香习惯在构图中安排一个支配性大块面，配合一到两个中等形状和若干点缀性小形，形成层次鲜明的视觉秩序。

在当代概念设计教学体系中，LMS 原则通常在"缩略图阶段"（thumbnail stage）就被强制应用。标准检验方法是将草图缩减至约 **50×50 像素**的灰度剪影——若此分辨率下三个尺度层级仍可区分，则认为构图通过了形状完整性测试；若三层模糊成均质灰块，则需返回修改块面比例，而非继续深入细节刻画。

## 核心原理

### 三个尺度的比例关系

大中小形状并非均等分配，也没有绝对固定的黄金比例，但业界有若干经过验证的参考区间。被概念设计师广泛引用的经验比例为 **60% : 30% : 10%**（面积占比），其来源可追溯至美国概念艺术家 Nathan Fowkes 在 Gnomon Workshop 系列教程（2009年）中的系统化总结。这一比例使大形成为画面的视觉锚点，中形形成过渡节奏，小形作为细节点缀和最终吸引眼球的"惊喜元素"。

以量化方式表述三层级之间的约束关系：

$$
A_L \geq 2 \times A_M \geq 4 \times A_S
$$

其中 $A_L$、$A_M$、$A_S$ 分别表示大形、中形、小形所占的画面面积比例。该不等式要求大形与中形之间至少存在 2 倍的面积差距，中形与小形之间同样如此——若将三者面积调整为 40% : 35% : 25%，则两个不等式均不成立，视觉上会立刻呈现出"拥挤而缺乏主次"的感受，因为人眼难以在三个过于接近均等的区域中自动建立层级判断。

更直觉化的记忆方式可以表达为：

> **A（大）>> B（中）> C（小）**

其中"$>>$"表示大形与中形之间的差异须达到质的跳跃，"$>$"表示中形与小形之间保持适度差异。两处差距不必对称，但两处都不可省略。

### 块面感与剪影清晰度

大中小形状操作的对象是**块面（shape/mass）**，而非线条轮廓或表面细节。理解这一点需要区分两类在视觉上容易混淆的小形：

- **功能性小形**：在剪影层面真正作为独立视觉节奏点存在，与大形、中形共同构成三层结构。例如骑士盔甲上的羽饰，其轮廓足够突出，在缩略图中作为独立形状可辨认。
- **纹理小形**：构成表面质感的重复单元，单个尺寸极微小，大量聚合后在视觉上融合为中等大小的纹理区域，不属于 LMS 分级中的"小形"层级。

若设计师将盔甲上数十颗铆钉逐一视为独立小形，将导致"小形碎片化"——这些铆钉在缩略图中只会显示为一片均质纹理，真正的小形层级反而因此缺失。正确的处理是将铆钉视为纹理块面的组成部分，再另行安排一个真正具有形状跳跃性的功能性小形（如盾牌上的纹章标志）。

### 形状层级与焦点的协同机制

LMS 原则必须与画面焦点（focal point）协调运作，两者之间存在一条明确的视觉规律：**大形与小形的边界交汇处天然形成对比度峰值区域，是视线最容易停留的焦点候选位置**。这是因为该区域在单位距离内集中了最大的尺度变化量，符合人眼对"变化率"敏感度高的神经生理特性（参见 Marr, 1982, *Vision*, MIT Press 中对边缘检测的零交叉理论）。

在实际操作中，设计师可以通过刻意将小形"贴靠"大形轮廓边缘来主动放置焦点，而非依赖后期明暗对比来补救构图缺陷。中形在这一机制中扮演视觉缓冲角色：它阻止观者目光从大形直接跳至小形细节，迫使视线在画面中完成一段有韵律的"大→中→小"行进路径，从而延长观看时间并提升画面的信息层次感。

## 关键公式与检验方法

在实际绘图流程中，LMS 原则可以通过以下伪代码逻辑进行系统性自查：

```python
def check_LMS(shapes):
    """
    输入：画面中所有可辨识块面的面积列表（归一化至总面积=1.0）
    输出：LMS 层级是否通过检验
    """
    sorted_shapes = sorted(shapes, reverse=True)  # 从大到小排列

    # 合并相邻尺度过近的形状为同一层级
    large = sorted_shapes[0]
    medium = sorted_shapes[1] if len(sorted_shapes) > 1 else 0
    small  = sorted_shapes[2] if len(sorted_shapes) > 2 else 0

    # 核心约束：大形须为中形的至少2倍，中形须为小形的至少2倍
    if large >= 2 * medium and medium >= 2 * small and small > 0:
        return "通过：三层级比例清晰"
    else:
        return f"未通过：当前比例 L={large:.0%} M={medium:.0%} S={small:.0%}，需调整"

# 案例测试
print(check_LMS([0.60, 0.30, 0.10]))  # 通过
print(check_LMS([0.40, 0.35, 0.25]))  # 未通过
```

该逻辑的核心思想来自对 Loomis（1947）在《创意插画》（*Creative Illustration*, Viking Press）中"主导形状"（Dominant Shape）理论的数值化翻译——Loomis 虽未给出精确比例，但反复强调"一个画面只能有一个占支配地位的形状单元"，2:1 的面积倍数关系正是对这一定性描述的最小化量化边界。

## 实际应用

### 角色剪影设计

在概念角色设计中，LMS 原则直接决定角色在游戏 UI 图标（通常为 64×64 像素）或远景镜头中的辨识度。以暴雪娱乐《守望先锋》（2016年）的角色设计为例：法老之鹰（Pharah）的大形是躯干与大型肩甲的整体轮廓（占剪影面积约 60%），中形是背部飞行背包的喷射器组件（约 25%），小形是头盔顶部的鸟形饰纹（约 10%）——三个尺度在 50 像素缩略图中仍可辨识，使她在六名角色同屏显示的战场 HUD 中保持唯一性。对比同游戏中早期被玩家反馈"辨识度低"的某些皮肤版本，其问题正是改动破坏了原有 LMS 结构，将中形护肩缩小至与小形饰件尺寸接近，导致剪影层次坍塌为"大形+一堆相似小块"的两层结构。

### 场景构图切分

在场景概念设计中，LMS 原则的应用往往通过主动操控地平线位置来实现。以典型的荒野日落构图为例：将地平线定于画面 70% 高度处，天空占据约 30% 面积（大形），中景山体群落轮廓约占 45%（中形），前景枯树剪影约占 15%（小形）——三者形成倒置的 LMS 结构，虽然打破常规天空为大形的习惯，但因比例落差仍然清晰，视觉层次依然成立。

若一张背景图中出现三栋高度比例为 1:0.9:0.8 的相近建筑，视觉上会产生"等差数列式"的单调重复感。解决方案是将其中一栋的高度拉升至其余两栋之和，使整体构图回归到具有一个支配性大形的 LMS 结构。

### 武器与道具设计

武器设计是 LMS 原则的高密度应用场景，因为武器形状的复杂度往往在有限的轮廓中被高度压缩。以一把幻想类双手巨剑为例：**剑刃**是大形（占整体轮廓面积约 65%，修长且轮廓简洁），**护手（Guard）**是中形（占约 25%，横向突出形成与剑刃明显的形状对比），**剑柄尾端的宝石镶嵌**是小形（占约 10%，圆形块面与两端的直线形成最终的尺度跳跃）。若设计师在剑刃中部额外添加与护手等宽的"二段护手"，则原有中形被分裂为两个面积相似的中等形状，LMS 结构退化为 LMMS，视觉节奏随即变得迟滞。

## 常见误区

### 误区一：将"三种尺度"理解为"恰好三个形状"

LMS 原则规定的是**尺度层级的数量**，而非画面中形状数量的上限。一个画面中可以有一个大形、三个中形和十五个小形——关键在于所有中形加总面积约为大形的 30%~50%，所有小形加总面积约为大形的 10%~20%。将"三层级"误读为"三个形状"的设计师往往会过度简化画面，导致中形和小形层级"空洞"，画面反而失去节奏感。

### 误区二：大形必须居于画面中央

大形是面积最大的块面，但其位置并不限于中央。偏心放置的大形（例如靠近画面左侧三分之一）反而是构图中制造动态张力的常见手法，此时小形和中形会被安排在大形留出的"负空间"一侧，形成视觉上的平衡补偿（参见 Betty Edwards, 1979, *Drawing on the Right Side of the Brain*, Tarcher/Penguin，其中对正负形空间关系的分析）。将大形固定在中央只是初学者降低构图风险的保守策略，并非 LMS 原则本身的要求。

### 误区三：小形越精细越好

小形的功能是作为视觉旅程的终点站，提供最终的"细节奖励"，但这并不意味着小形需要内部填充大量复杂细节。小形的**轮廓辨识度**比内部复杂度更重要——一个轮廓独特的简洁小形（如某个标志性符文）比一个轮廓模糊的精细小形（如密集的划痕纹理）更能有效完成 LMS 结构中小形层级的视觉任务。
