# 供应商筛选

## 概述

供应商筛选是游戏外包流程中将潜在合作候选方缩减至最终合作伙伴的结构化决策过程，具体包括对外包供应商进行量化评估、组织试做测试（Art Test / Technical Test），以及依据综合评分选定合作方。这一过程直接决定了外包资产的质量上限——一旦选错供应商，后续的合同条款和验收标准再严格也难以弥补。

供应商筛选的系统化方法在游戏行业约于2000年代初期随着大型主机游戏的外包规模扩大而成熟，彼时北美和欧洲工作室开始大量将环境美术和角色建模外包至东欧（如波兰、乌克兰）及亚洲（如中国、越南）工作室。这种跨国外包催生了标准化试做测试流程的需求，因为文化沟通差异和远程协作风险使得直接签约的成本极高。

在实际项目管理中，供应商筛选阶段通常发生在外包策略确定之后、合同谈判之前，是整个外包链条中信息密度最高的环节。一个典型3A项目的供应商筛选周期为4至8周，涉及初筛、试做、评审、商务四个阶段，每个阶段会淘汰50%至70%的候选供应商。

## 核心原理

### 评估标准体系（Vendor Scorecard）

供应商筛选通常使用加权评分卡（Weighted Scorecard）对候选方进行量化评估，常见的评估维度及参考权重如下：

- **作品集质量（Portfolio Quality）**：30%权重，重点审查与项目风格匹配度，例如候选供应商是否做过写实风格PBR材质或卡通渲染（Toon Shading）的项目
- **团队规模与稳定性**：20%权重，需确认核心骨干成员的留存率及在职年限，避免签约后核心人员流失
- **工具链兼容性**：15%权重，包括是否熟悉项目使用的DCC软件（如Maya、Blender、Substance Painter）及版本
- **沟通响应速度**：15%权重，通常以邮件回复时效（理想值≤24小时）和时区覆盖情况衡量
- **报价与性价比**：10%权重，与同类供应商横向对比，而非单纯追求最低报价
- **过往合作口碑**：10%权重，向供应商提供的参考客户（Reference Client）进行背景调查

### 试做测试（Art Test / Tech Test）设计

试做测试是供应商筛选中最具区分度的环节。一个设计良好的试做测试应包含以下要素：

1. **参考资料包（Reference Package）**：提供项目风格指南、Color Palette、Poly Count限制（如角色LOD0限制12,000三角面）和贴图尺寸规格（如2048×2048 Albedo/Normal/ORM三图）
2. **明确交付清单**：规定文件格式（FBX + TGA/PNG）、命名规范（如`SM_ENV_Rock_01`）、Pivot Point位置等
3. **时间窗口**：通常给予5至10个工作日，过短会筛掉有真实项目在身的优质供应商，过长会延误项目排期
4. **评分维度**：形态准确度、拓扑布线合理性（Topology / Edge Flow）、UV拆分效率、材质表现力，以及是否遵守技术规范

付费试做（Paid Art Test）在行业内逐渐成为惯例，金额通常为正式外包报价的20%至40%，这既能筛掉不认真的供应商，也能维护行业伦理。

### 多轮筛选漏斗（Screening Funnel）

标准筛选流程分为四层：

- **第一层——资料初筛**：收集Portfolio和公司简介，按评分卡过滤，目标从20至50家候选方缩减至5至10家
- **第二层——视频/电话访谈**：验证团队构成和工作流程，重点问题包括"你们如何处理反馈迭代循环（Feedback Loop）"和"你们的最大并行项目数是多少"
- **第三层——试做测试**：从5至10家缩减至2至3家
- **第四层——商务谈判与背景调查**：最终决策，输出合同草案

## 实际应用

**案例一：角色外包供应商筛选**
某国内中型工作室在开发一款ARPG时需要外包60个NPC角色，计划周期18个月。初筛阶段从行业数据库（如CGTrader Outsource、Artstation Jobs列表）和展会（如GDC、Unite）人脉中收集了32家候选供应商。评分卡过滤后保留7家进入试做阶段，测试内容为制作一个写实风格守卫角色（含盔甲，LOD0 16,000三角面，4套贴图集）。最终2家供应商进入商务阶段，主要差异体现在Topology质量和修改响应速度上，而非价格。

**案例二：环境美术外包的工具链筛选**
一个使用Unreal Engine 5的项目要求供应商具备Nanite-ready资产制作能力（高模烘焙至低模，支持Virtual Shadow Maps工作流）。这一技术要求在试做测试阶段作为硬门槛，直接淘汰了两家报价最低但无UE5经验的供应商，避免了后期因资产重做产生的额外成本。

## 常见误区

**误区一：仅凭作品集决策，跳过试做测试**
作品集中的资产可能来自多年前的项目，或由已离职的核心人员制作。没有经过针对性试做测试，无法验证当前团队在你的技术规范和风格要求下的真实产出能力。许多外包事故正是因为跳过试做直接签约导致的。

**误区二：将最低报价等同于最优选择**
外包美术的报价通常反映了供应商的真实质量定位。若某供应商的单角色报价低于市场均价40%以上，则需要重点审查其人员流动率和历史项目质量，因为超低价格往往意味着承接过量项目、使用初级人员或压榨员工等问题，最终转化为频繁的修改成本（Revision Cost），总成本反而更高。

**误区三：忽视时区与沟通成本**
供应商与甲方的时差超过8小时时，有效的工作日重叠时段不足2小时，这会将单轮反馈周期从1天延长至2至3天。在总周期固定的项目中，这相当于实质性缩短了可用工作时间。在评分卡中量化沟通成本，并将异步协作工具（如Ftrack、ShotGrid）的使用能力纳入评估维度，是规避此风险的具体方法。

## 知识关联

**前置知识——外包策略**：供应商筛选的评估标准必须由外包策略阶段确定的外包范围（Scope）和质量目标驱动。如果外包策略中已明确"角色外包目标：写实风格，主机品质，优先选择有AAA经验的供应商"，则评分卡的Portfolio Quality维度就有具体的对标标准。没有明确的外包策略，筛选标准容易流于主观。

**后续知识——合同管理**：供应商筛选阶段产生的试做测试结果、评分卡记录和背景调查报告，是合同管理阶段制定验收标准（Acceptance Criteria）和里程碑付款节点（Milestone Payment）的直接依据。例如，试做阶段发现某供应商需要平均3轮修改才能达标，这一数据应直接影响合同中的修改次数条款（Revision Rounds Clause）的约定方式。
