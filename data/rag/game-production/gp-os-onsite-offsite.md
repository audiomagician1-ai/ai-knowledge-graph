# 驻场 vs 远程：游戏外包合作模式的选择与管理

## 概述

在游戏外包管理中，**驻场外包**（On-site Outsourcing）指外包团队或个人在委托方公司的办公地点长期工作，而**远程外包**（Remote Outsourcing）指外包人员在其自有场所（通常是异地城市甚至境外）独立完成交付任务。这两种模式在沟通效率、资产安全、成本结构和用工合规方面呈现出截然不同的特征，直接影响游戏项目的推进节奏与美术/程序质量管控。

驻场模式在中国游戏行业约于2008年前后随页游规模化生产开始普及，彼时上海、北京的中型厂商会将角色美术供应商整组引入内部工位，以便美术总监每日直接审核资产输出。远程模式则随着协作工具（如飞书、TAPD、Perforce远程部署）的成熟在2015年后快速增长，并在2020年疫情期间成为许多项目的唯一选项，促使行业建立了更完整的远程外包SOP流程。

选择驻场还是远程，不仅是一个沟通偏好问题，更涉及知识产权保护级别、用工合规风险（驻场人员的劳动关系认定）以及单位资产产出成本的综合核算。参考《游戏项目制作管理实务》（葛颖，2021，人民邮电出版社）的分类框架，实际项目中通常将外包任务按"IP敏感度 × 交互频次"两个维度矩阵化，再据此选择驻场或远程方案。

## 核心原理

### 沟通效率与迭代速度

驻场外包的核心优势是**零延迟反馈**。美术或程序外包人员坐在项目组旁边，制作人或主美可以当场在Photoshop / Maya文件上直接标注问题，一次10分钟的面对面沟通可替代3～5封来回的异步消息。行业经验数据显示，同等复杂度的角色概念稿，驻场团队的单次修改确认周期通常在2～4小时以内，而远程团队平均为1～2个工作日；若涉及海外供应商（如越南、乌克兰工作室），时区差异可将确认周期拉长至48小时以上。

远程模式要弥补这一差距，必须建立严格的**每日同步机制**：固定时段的视频站会（通常15分钟，建议设在北京时间10:00以兼顾东南亚时区）、标准化的FTP/SVN提交规范，以及带有图层截图批注的反馈模板。缺乏这套流程的远程外包项目，往往在进入第3个里程碑后出现"风格漂移"——即外包产出与游戏整体美术方向逐渐偏离，到第5个里程碑时返工量可占总工时的25%以上。

**迭代速度量化对比：**

$$T_{\text{远程迭代}} = T_{\text{制作}} + T_{\text{传输}} + T_{\text{时区等待}} + T_{\text{审核}}$$

$$T_{\text{驻场迭代}} = T_{\text{制作}} + T_{\text{审核}}$$

对于一个标准角色立绘任务，$T_{\text{传输}} + T_{\text{时区等待}}$ 在远程场景下通常为4～24小时，占总迭代周期的30%～60%。这也是为什么高返修率任务（如概念探索阶段的原画）更适合驻场，而低返修率任务（如已有参考的批量场景道具）更适合远程。

### 知识产权与资产安全

驻场模式在知识产权保护方面具有结构性优势：外包人员使用甲方设备，素材不离开内网，源文件直接提交至甲方Perforce服务器，物理上几乎不存在资产外泄路径。这对于含有核心IP角色设计（如未公开的主角立绘、世界观设定文档）或技术demo的项目至关重要。

远程外包则必须依赖合同条款与技术手段的双重保障。常见技术措施包括：

1. **云桌面工作站隔离**：向外包方提供无法本地导出的云桌面（如腾讯云HAG游戏工作站、阿里云无影方案），所有文件存留在甲方云端，外包人员仅拥有操作权限而无下载权限。
2. **分层资产下发**：将高精模型的UV拆分与贴图分批下发，外包人员在任一时间点持有的资产均不构成完整成品。
3. **水印溯源**：对下发的参考图和分辨率预览图嵌入隐形水印（如频域水印），一旦泄露可追溯至具体外包个人。
4. **NDA分级管理**：针对核心IP资产（A级）、普通美术资产（B级）、技术文档（A级）分别签订不同赔偿条款的保密协议，A级泄露赔偿上限不低于合同总额的5倍。

即便采取上述措施，远程外包的IP泄露概率仍高于驻场。因此业内通行规则是：**核心角色原画、世界观概念图必须驻场完成，批量场景道具、UI切图可远程外包**。

### 用工合规与劳动关系风险

这是驻场模式最容易被忽视的隐性成本。根据《劳动合同法》第10条及相关司法解释，若外包人员在甲方地点持续工作超过**6个月**，且接受甲方的日常管理指令（如打卡、排班、接受主美直接管理），法院在劳动争议案件中可能认定存在事实劳动关系，要求甲方补缴社保、支付经济补偿金。

2019年北京市朝阳区劳动仲裁委员会曾有多起类似案例，涉案游戏公司最终被裁定为事实雇主，需补缴最长24个月的社保，并支付相当于月薪2倍的经济补偿。为规避此类风险，正规项目通常采取以下措施：
- 驻场外包合同明确约定"委托服务"性质，甲方仅对交付物提出要求，不介入外包方的日常工作安排；
- 驻场周期原则上不超过**4个月**，到期可续签但需间隔30天以上；
- 外包人员使用其自有设备或明确标注为"外包商设备"的机器，不参与甲方内部行政管理。

## 关键决策公式与选择框架

在实际项目中，可用以下**外包模式选择评分模型**对每个外包任务包进行量化评估：

```python
def outsourcing_mode_score(ip_sensitivity, interaction_freq, cost_budget, duration_months):
    """
    ip_sensitivity: 1~5, 5=最高敏感度 (核心IP角色)
    interaction_freq: 1~5, 5=每日高频修改 (概念探索阶段)
    cost_budget: 1~5, 5=预算极紧
    duration_months: 驻场预计时长（月）

    返回值 > 0 推荐驻场，< 0 推荐远程
    """
    legal_risk_penalty = max(0, (duration_months - 4) * 0.5)  # 超4个月加重法律风险
    onsite_score = (ip_sensitivity * 0.4 + interaction_freq * 0.35) - \
                   (cost_budget * 0.15 + legal_risk_penalty * 0.1)
    remote_score = (cost_budget * 0.3) - \
                   (ip_sensitivity * 0.35 + interaction_freq * 0.35)
    return onsite_score - remote_score

# 案例：主角SSR立绘包 (ip=5, 交互=4, 预算=3, 时长=2个月)
score = outsourcing_mode_score(5, 4, 3, 2)
# score ≈ +2.1 → 推荐驻场

# 案例：批量场景道具 (ip=2, 交互=2, 预算=4, 时长=6个月)
score = outsourcing_mode_score(2, 2, 4, 6)
# score ≈ -1.3 → 推荐远程
```

该模型并非精确预测工具，而是帮助制作人在多任务包并发时快速排优先级、将有限的驻场工位分配给最需要的任务。

## 实际应用场景

### 场景一：手游主角SSR立绘包（高IP敏感度）

某二次元手游公司在制作新角色SSR立绘时，要求原画外包方整组（3名原画师 + 1名主管）驻场6周。公司为其提供4个工位，使用内网隔离的绘图电脑，素材直接提交至内网SVN。美术总监每日上午10:30进行15分钟立绘进度确认，当日修改意见可在下午收班前完成。最终6周内完成12套SSR立绘的草稿到精稿全流程，单张平均耗时约2.5天，远低于同类远程项目的4天均值。

### 场景二：开放世界场景道具批量生产（低IP敏感度）

某端游项目需要生产约800件场景小物件（木箱、石墩、灯具等），IP敏感度低，参考图已完备，修改频次预期极低。该项目选择与越南胡志明市一家外包工作室远程合作，对方有32名3D美工，报价约为国内同级驻场的60%。项目方通过以下流程保障质量：每周一视频站会（北京时间9:00，越南时间8:00）、每批次提交使用标准Checklist（多边形数量上限、UV利用率≥85%、贴图命名规范等17项检查项），以及每提交50件触发一次甲方QA抽检。最终800件道具在14周内完成，返工率约9%，低于同类驻场批量项目的12%均值。

### 场景三：混合模式——核心驻场 + 批量远程

某MMORPG项目采用混合策略：将5名核心外包原画师（负责主线角色设计）驻场于北京研发中心，同时将约200套NPC服装的3D制作任务发包给成都一家远程工作室。两条线并行，驻场原画师的输出作为远程3D工作室的参考基准，确保整体美术风格统一。该项目项目总监王磊（2022年GDC China分享）表示，混合模式将整体外包成本控制在纯驻场方案的72%，同时将核心IP资产泄露风险降至与纯驻场相当的水平。

## 常见误区

### 误区一：远程外包"节省了工位成本就等于节省了总成本"

许多制作人只计算了工位费用（北京/上海人均800～2000元/月）的节省，却忽视了远程模式带来的**隐性管理成本**：专属项目协调人工时（通常需要额外0.3～0.5个FTE）、ShotGrid/Hansoft等协作工具许可费（约$30～$50/用户/月）、以及因质量问题发现滞后导致的返工成本。综合核算后，远程外包的实际单位资产成本通常仅比驻场低15%～20%，而非表面上的40%～50%。

### 误区二：驻场就能解决所有沟通问题

驻场解决的是**物理距离**问题，但若甲方内部缺乏明确的审核流程，驻场外包人员同样会面临"一稿被五个人分别提意见、意见相互矛盾"的困境。驻场的前提是甲方有清晰的美术总监负责制：所有修改意见须经美术总监汇总后统一下达，否则驻场外包人员的时间损耗甚至高于远程。

### 误区三：驻场超过6个月不会有法律风险，"签好合同就行"

NDA和外包服务合同无法改变实质性劳动关系的认定。法院在劳动争议中采用"实质认定"原则，若外包人员实际上接受甲方的工作时间管理、使用甲方设备、报酬按月固定支付，则无论合同如何约定，均可能被认定为事实劳动关系。正确做法是严格控制驻场时长，或通过有资质的人力资源公司进行劳务派遣，由派遣公司作为用工主体承担相应义务。
