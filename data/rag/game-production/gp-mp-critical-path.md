# 关键路径法

## 概述

关键路径法（Critical Path Method，简称CPM）是一种基于有向无环图（DAG）的项目进度网络分析技术，由杜邦公司与雷明顿兰德公司的工程师于1957年联合开发，最初用于化工厂的维护调度。其核心数学基础是对任务网络中所有路径进行前向遍历（计算最早开始/完成时间）和后向遍历（计算最晚开始/完成时间），从而识别出决定项目最短完工时间的任务序列。

在游戏制作的里程碑规划中，关键路径法解决的核心问题是：在数十甚至数百个相互依赖的任务中，**哪些任务的延误会直接推迟里程碑交付日期**。例如，一款手游的Alpha里程碑可能包含核心玩法编程、美术资源集成、QA验证三条并行路径，而只有最长路径上的任务才是真正不可拖延的关键任务。

## 核心原理

### 网络图构建与路径枚举

CPM的第一步是将项目分解为带有**完成到开始（Finish-to-Start）依赖关系**的任务节点，构成活动节点网络（AON图）。每个节点记录三个属性：任务名称、**工时估算值（duration）**、以及前置任务列表。对于游戏制作场景，典型的依赖链如：概念原画 → 3D建模 → 绑定蒙皮 → 动画制作，四个任务必须按序完成，总工期为各工时之和。

### 前向遍历与最早时间计算

前向遍历从起始节点出发，计算每个任务的**最早开始时间（ES）**和**最早完成时间（EF）**：

> **EF = ES + Duration**
> **ES（后续任务）= max（所有前置任务的 EF）**

以某游戏Beta里程碑为例：若"关卡编辑器开发"耗时10天，"关卡白盒搭建"依赖前者且耗时5天，"关卡美化"依赖白盒且耗时8天，则这条路径的EF = 0+10+5+8 = 23天。若另一条"音效制作"路径总工时仅14天，则23天的路径为当前关键路径。

### 后向遍历与浮动时间计算

后向遍历从终点反向推算每个任务的**最晚开始时间（LS）**和**最晚完成时间（LF）**：

> **LS = LF - Duration**
> **LF（前置任务）= min（所有后续任务的 LS）**

浮动时间（Float，又称松弛时间）定义为：

> **Total Float = LS - ES = LF - EF**

浮动时间为0的任务连成的序列即为**关键路径**。浮动时间为正值（例如某配音录制任务Float = 6天）意味着该任务可以在6天范围内推迟或延长，而不影响里程碑日期。游戏制作中，美术与程序双线并行时，通常程序端路径为关键路径，美术端存在数天至数周的浮动时间。

### 进度压缩技术

当项目网络计算结果超出里程碑截止日期时，CPM提供两种压缩关键路径的方法：

- **赶工（Crashing）**：向关键路径上的任务投入额外资源（如增加程序员），以增加成本换取时间缩短。赶工只对关键路径任务有效，向浮动时间充足的任务加人无法缩短总工期。
- **快速跟进（Fast Tracking）**：将原本串行的关键任务改为部分并行执行，例如在3D模型未完全交付时让绑定师先处理已完成部分。快速跟进会引入返工风险，适用于后期设计变更可能性较低的资产管线。

## 实际应用

在游戏制作里程碑规划中，CPM最常用于**主线内容管线的进度预警**。以一款30人规模的AA游戏为例，制作人在Sprint规划阶段识别出关键路径为：引擎升级（15天）→ 核心战斗系统重构（20天）→ 关卡逻辑集成（10天）→ 整合测试（7天），总计52天。若该项目距Alpha仅剩48天，制作人可对"核心战斗系统重构"实施赶工，增派1名程序员将其压缩至16天，使总路径压缩至48天，恰好符合节点要求。

CPM还被用于**里程碑风险评估**：任务工时估算误差率若超过20%，关键路径可能发生"路径转移"——原本浮动时间为3天的次关键路径变为新的关键路径。因此专业的制作人会同时监控浮动时间低于5天的"近关键路径"任务。

## 常见误区

**误区一：所有任务都应该消除浮动时间。** 浮动时间并非资源浪费，而是进度缓冲。强行压缩非关键路径上的浮动时间不会缩短里程碑日期，反而可能因过度密集的排期导致团队疲劳、增加返工率。

**误区二：关键路径在项目执行过程中固定不变。** 随着任务实际完成时间与估算值出现偏差，关键路径会动态迁移。例如原本关键路径上的任务提前3天完成，而原来浮动2天的旁路出现延误，两条路径的Float差值归零，路径转移随即发生。游戏制作中建议每个Sprint结束后重新运行CPM计算。

**误区三：CPM工时等同于日历工期。** CPM计算的Duration是纯工作时间，未考虑节假日、团队成员假期、审批等待等日历因素。将52个工作日的关键路径直接换算为自然日时，必须加入团队实际可工作天数的折算系数（通常为1.3~1.5倍）。

## 知识关联

CPM的计算质量直接依赖于**工时估算**的准确性：若前序的工时估算环节采用了类比估算或三点估算（PERT），这些估算值将作为CPM各节点的Duration输入，估算误差会在网络图中被放大至关键路径层面。特别是PERT的三点估算公式（`te = (O + 4M + P) / 6`）可以与CPM结合使用，计算关键路径的统计期望工期及标准差，为里程碑置信度分析提供依据。

在掌握CPM之后，**甘特图与时间线**是其自然的可视化延伸：甘特图本质上是将CPM网络图中的任务节点按时间轴展开，关键路径上的任务通常以不同颜色高亮显示（如红色），浮动时间则以灰色延伸条表示。现代游戏项目管理工具（如Shotgun/ShotGrid、Hansoft）均支持从CPM网络图自动生成甘特图视图，并在任务实际进度更新后动态重绘关键路径。
