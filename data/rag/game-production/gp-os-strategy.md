# 外包策略

## 概述

外包策略（Outsourcing Strategy）是游戏制作中用于决定哪些工作内容应由内部团队完成、哪些应委托外部供应商执行的系统性决策框架。这一策略的核心任务是在成本效益、质量控制与核心竞争力保护之间找到最优平衡点。不同于单纯的"省钱手段"，外包策略本质上是对制作资源的战略性配置决策。

外包策略在游戏行业的系统性应用可追溯至1990年代中后期，当时日本主机游戏厂商（如Konami、Capcom）开始将部分美术资产委托给东南亚工作室制作，以应对3D时代带来的资产量爆炸问题。进入2010年后，随着手游市场扩张，大型游戏公司平均将30%至60%的非核心美术资产外包制作，外包策略的精细化管理需求随之急剧上升。

掌握外包策略对制作人和美术总监尤为关键：错误的外包决策不仅导致质量风险，还可能使团队对核心技术产生路径依赖，最终丧失自主迭代能力。准确运用外包决策框架，可以在不稀释核心竞争力的前提下，将制作周期缩短20%至40%。

## 核心原理

### 自研 vs 外包决策矩阵

外包决策的基本分析工具是"战略价值 × 内部能力"四象限矩阵。横轴代表某项工作对游戏差异化竞争力的战略价值（高/低），纵轴代表团队当前执行该工作的内部能力（强/弱）：

- **高战略价值 + 内部能力强** → 坚决自研，属于核心资产，外包将稀释竞争优势
- **高战略价值 + 内部能力弱** → 优先自研并培育能力，必要时选择战略性合作而非纯外包
- **低战略价值 + 内部能力强** → 可选择外包以释放内部人力，聚焦更高价值任务
- **低战略价值 + 内部能力弱** → 强烈建议外包，最典型的外包候选区

以一款动作RPG为例，角色动作设计（战斗手感）通常属于第一象限，应100%自研；而场景中的通用道具模型（木桶、箱子等）则属于第四象限，是最理想的外包对象。

### 核心能力保护原则

核心能力保护原则要求团队在外包任何内容之前，先明确界定"不可外包边界"（Non-Outsourceable Boundary）。判断某项能力是否属于核心的三项检验标准为：

1. **替换成本检验**：若该岗位人员离职，是否会造成超过3个月的不可恢复性知识损失？
2. **差异化检验**：该能力是否直接体现在玩家的产品体验差异中？
3. **迭代依赖检验**：后续版本迭代是否必须依赖该能力的持续积累？

满足上述任意两条，该能力应判定为核心能力，原则上不得外包。例如，某射击游戏的射击手感调参经验、某开放世界游戏的关卡叙事设计逻辑，均属于需保护的核心能力。

### 外包比例与成本计算

外包比例（Outsource Ratio, OR）的合理区间因内容类型而异。经验数据显示：

- **2D原画概念图**：OR建议不超过20%，因风格统一性高度依赖内部标准
- **3D场景环境资产**：OR可达50%至70%，标准化程度高，易于规格化输出
- **特效（VFX）**：OR建议不超过30%，与引擎参数耦合度高，外包返工率较高
- **本地化文本翻译**：OR可达90%以上，属于高度标准化可量化任务

成本对比时应使用**全周期成本公式**：
> **外包全周期成本 = 外包直接费用 + 内部管理人时成本 + 返工率 × 平均返工费用 + 质量风险溢价**

若仅比较外包报价与内部人工成本，会系统性低估外包的实际成本，导致错误的外包扩张决策。

## 实际应用

**案例：某国产手游项目的外包策略调整**

某手游项目初期将角色立绘全部委托给4家外包供应商，外包比例达100%。项目进行至第8个月时，美术总监发现不同供应商输出的角色面部结构比例存在系统性差异，导致玩家评测中出现"人物风格不统一"的高频投诉。溯源分析表明，由于内部没有保留核心原画师，团队对外包商的风格校正只能依赖文字描述，缺乏实际执行能力，修改周期从平均5天延长至14天。

该项目随后将角色主体立绘调整为内部自研，仅将服装配件、背景场景和UI图标委托外包，外包比例从100%降至约45%。调整后的版本上线后，美术风格统一性投诉下降了约62%，同时将内部美术团队从低价值重复性工作中解放，专注于核心角色设计迭代。

## 常见误区

**误区1：以单价便宜作为外包决策的主要依据**

许多制作团队在选择外包时，将单件资产报价（如"单张原画300元 vs 内部成本800元"）视为核心决策依据，忽略了管理耗时、返工率和知识流失的隐性成本。实际项目数据显示，当外包返工率超过25%时，外包的实际综合成本往往高于内部制作。正确做法是使用上文所述的全周期成本公式进行评估。

**误区2：将外包等同于"核心能力不需要培养"**

部分中小团队长期依赖外包完成美术资产，认为只要有钱就能持续购买能力，无需建立内部积累。这一思路在项目早期看似合理，但会导致团队在需要风格大幅迭代或遭遇供应商断供时完全丧失自主调整能力。典型案例是某手游运营至第3年，因主力外包供应商工作室关闭，而内部美术团队对原有风格已无法还原，被迫进行整体美术重制，损失超过6个月的有效开发时间。

**误区3：外包策略是一次性决策**

外包策略并非在项目立项时确定后就固定不变。随着团队能力成长、项目阶段变化（Alpha/Beta/运营期）及供应商市场波动，外包范围应定期重新评估，建议每季度或每个里程碑节点进行一次OR复盘，动态调整自研与外包的边界。

## 知识关联

学习外包策略需要具备**外包管理基础**知识，即了解外包流程中的合同结构、资产验收标准和保密协议基本要素，才能在决策时准确评估管理成本项。

外包策略的决策结论直接驱动后续两个方向的工作：其一是**供应商筛选**，当策略层面确定了外包内容类型和质量要求后，供应商筛选需围绕这些具体参数设计评估标准，例如对高返工风险内容应优先选择具有丰富该类型经验的供应商；其二是**产能规划**，外包比例的确定直接影响内部团队人力配置计划，外包策略文档是产能规划中计算内部需求人头数的核心输入依据。三者共同构成游戏制作中外包管理体系的完整决策链条。
