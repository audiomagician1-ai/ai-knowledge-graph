# 序列帧概述

## 概述

序列帧（Sprite Sheet Animation / Frame Sequence）是一种将动画拆解为若干静止图像帧、按照固定时间间隔连续播放以产生运动错觉的技术。每一帧都是完整的单张图像，通常以PNG或JPEG格式存储，帧与帧之间记录了角色、粒子或特效在不同时刻的状态。当播放速率达到每秒24帧（24fps）或以上时，人眼的视觉暂留效应使这些静止图像呈现出流畅的动态效果。

序列帧技术的历史可追溯至1878年埃德沃德·迈布里奇（Eadweard Muybridge）拍摄奔马连续照片的实验，他用24台相机在0.5秒内拍下马匹奔跑的连续姿态，首次科学验证了视觉暂留原理。在游戏与实时特效领域，序列帧从2D精灵动画时代延续至今，因其在GPU上仅需采样贴图坐标偏移便能完成动画播放，计算开销极低，成为移动端和网页端特效的主流方案之一。

序列帧特效区别于骨骼动画或程序化动画的核心优势在于：它可以预先烘焙任意复杂度的视觉效果——包括流体模拟、布料解算、粒子爆炸——并以"静态贴图播放"的极低运行时代价还原这些效果。一套1024×1024像素的8×8序列帧图集可存储64帧画面，在不增加Drawcall的前提下播放约2.7秒（以24fps计算）的特效动画。

## 核心原理

### 帧序列的构成与播放逻辑

序列帧动画由N张尺寸相同的图像帧按顺序排列而成。每帧图像对应动画时间轴上的一个时间切片，帧与帧之间的时间间隔称为帧间隔（Frame Interval），其倒数即为帧率（FPS）。播放时，系统以当前经过的时间除以帧间隔取整，得到当前应显示的帧索引：

```
帧索引 = floor(经过时间 × 帧率) % 总帧数
```

此公式中`floor`取整确保切换是离散跳变而非线性插值，`% 总帧数`实现循环播放。若要播放一次后停止，则在帧索引达到总帧数减一时锁定即可。

### 图集排布方式与UV偏移计算

序列帧通常以"图集（Atlas）"形式存储，即将所有帧按行列排布在一张大贴图中，而非存储为独立文件。设图集共有`rows`行、`cols`列，第`i`帧的UV偏移计算公式为：

```
U偏移 = (i % cols) / cols
V偏移 = floor(i / cols) / rows
UV缩放 = (1/cols, 1/rows)
```

在Shader中，最终采样坐标为`原始UV × UV缩放 + (U偏移, V偏移)`。这种方式使整个动画只占用一个贴图槽位，显著减少材质切换次数。常见图集规格有4×4（16帧）、8×8（64帧）、4×8（32帧）等，总像素数必须是2的幂次以兼容GPU的MIP贴图生成。

### 播放速率与时间控制

序列帧播放速率并非越高越好。帧率过低（如8fps）会产生明显的"翻页感"，帧率过高则要求图集存储更多帧，贴图内存占用呈线性增长。爆炸、烟雾等一次性特效常用16~24fps，循环燃烧火焰等周期性特效常用12~16fps以节省贴图资源。特效引擎（如Unity的Particle System）支持通过"随机初始帧（Random Start Frame）"参数打破多个粒子同步播放的规律感，使效果更自然。

## 实际应用

**移动端角色技能特效**：在《王者荣耀》等移动MOBA游戏中，技能释放的光效、爆炸环等复杂特效大量采用序列帧实现。开发团队预先在离线渲染器（如Houdini）中完成高精度流体或粒子模拟，导出为512×512像素、8×4共32帧的图集，在手机GPU上以Additive叠加混合模式播放，既保留了高质量视觉效果，又将单个特效的渲染开销控制在1次Drawcall以内。

**网页端Loading动画**：CSS的`animation`与`steps()`时间函数专为序列帧设计，`steps(8)`表示将一次动画周期强制分为8个离散跳变步骤，正好对应8列图集的帧切换逻辑，无需JavaScript即可驱动序列帧播放，是网页轻量动效的标准做法。

**粒子系统贴图动画**：在Unity粒子系统的"Texture Sheet Animation"模块中，可以为每个粒子独立指定起始帧和播放速度，使大量粒子播放同一图集时呈现出随机差异化的动态，常用于火花、泡泡、叶片飘落等场景。

## 常见误区

**误区一：帧数越多，动画质量越好**。序列帧动画的视觉质量受帧率与单帧分辨率共同决定。将一张1024×1024的图集分成16×16共256帧后，每帧实际分辨率仅有64×64像素，在放大播放时会明显模糊。实际项目中需要在帧数与单帧清晰度之间权衡，通常16~64帧已能满足大多数特效需求。

**误区二：序列帧与GIF是同一回事**。GIF是一种文件格式，其动画原理也是序列帧，但GIF仅支持256色调色板且无透明度渐变（Alpha通道只有完全透明或完全不透明），而游戏中的序列帧图集使用完整32位RGBA色彩空间，并通过GPU Shader实现半透明混合，两者的显示能力和使用场景有本质区别。

**误区三：序列帧只适用于2D游戏**。3D游戏中的序列帧特效同样广泛使用，3D粒子系统中的每个粒子本质上是朝向摄像机的四边形面片（Billboard），在其上播放序列帧图集可以以极低代价模拟体积感较强的烟雾、闪光等三维视觉效果。

## 知识关联

学习序列帧概述需要先理解**渲染器类型**的基础知识：不同渲染器（如Unity的URP与HDRP）对贴图采样、混合模式的支持存在差异，决定了序列帧图集的格式要求和Shader写法是否需要适配。例如URP下序列帧特效需使用URP兼容的Particles/Unlit Shader，而不能直接套用Built-in管线的材质。

掌握序列帧概述后，下一步需要学习**图集制作**：了解如何使用TexturePacker、Houdini Karma或After Effects等工具将多帧动画导出并排布为符合2的幂次规格的图集文件，以及如何设置合适的压缩格式（移动端常用ETC2或ASTC）以在视觉质量与内存占用之间取得平衡。序列帧概述建立的帧索引与UV偏移计算逻辑，将直接指导图集制作阶段对行列数的规划决策。
