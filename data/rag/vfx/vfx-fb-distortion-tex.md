# 扭曲序列帧

## 概述

扭曲序列帧是一种通过播放 Distortion（扭曲）纹理的帧序列来模拟光线折射效果的特效技术，常用于实现热浪（Heat Haze）、冲击波（Shockwave）、隐形斗篷扰动等视觉现象。与普通序列帧直接将颜色叠加到画面不同，扭曲序列帧的每一帧存储的不是 RGB 颜色值，而是 UV 偏移量——即告诉渲染器"把屏幕上这个像素的采样位置向某个方向移动多少"。

这项技术的理论基础源于屏幕空间后处理（Screen Space Post-Processing）管线。早期游戏（如2004年的《半条命2》）已通过折射贴图实现热气扰动效果，而将该原理封装为序列帧动画，则是移动游戏与实时特效领域在2010年代广泛普及的工程化做法。通过预烘焙的序列帧动画驱动扭曲，可以在不依赖复杂物理模拟的情况下，获得高度可控、性能友好的折射动画效果。

扭曲序列帧的核心价值在于它的"非破坏性"叠加方式：这类特效粒子本身几乎不渲染任何可见颜色，而是通过偏移背景的采样坐标来扭曲场景，因此能够干净地叠加在任何背景之上，无需美术单独适配每种场景颜色。

## 核心原理

### UV 偏移量的存储与解码

扭曲序列帧的纹理中，每个像素的 RG 通道分别存储水平（U）和垂直（V）方向的偏移量。由于纹理通道值域为 [0, 1]，需要将其重映射到有符号偏移范围：

```
offset = (texel.rg - 0.5) × 2 × distortionStrength
```

其中 `texel.rg` 为采样到的原始通道值，减去 0.5 后变为 [-0.5, 0.5]，再乘以 2 得到 [-1, 1] 的归一化方向，最后乘以 `distortionStrength`（扭曲强度，通常为 0.01 ～ 0.1 的小数）控制偏移幅度。最终将这个 offset 叠加到屏幕空间 UV 坐标上，再用新坐标采样 GrabPass 或 Camera Render Texture，即可获得扭曲后的背景颜色。

### 序列帧动画的驱动方式

扭曲序列帧的帧动画驱动逻辑与自发光序列帧相同——将多帧扭曲图排列在一张图集（Atlas）中，在 Shader 中通过当前帧索引计算 UV 偏移量来逐帧切换。一张典型的热浪扭曲图集通常为 8×8 = 64 帧，分辨率 512×512，单帧有效分辨率约 64×64 像素。帧率一般设置在 12～24 fps 之间，过高的帧率会让扭曲显得过于规律而失去自然感。

关键区别在于：这里切换的每一帧，其 RG 通道描述的是**形状不同的扭曲场**（例如冲击波从中心向外扩散的形变梯度），而非颜色渐变。美术在 Substance Designer 或 Houdini 中预先烘焙这套扭曲场序列，确保动画在时间上具有物理合理的扩张或收缩节奏。

### GrabPass 与 Render Texture 的渲染流程

实现扭曲效果需要访问当前帧已渲染的背景颜色，主要有两种方案：

- **GrabPass（Unity 内置）**：在该 Pass 执行前将屏幕内容复制到 `_GrabTexture`，随后在同一 DrawCall 中用扭曲后的 UV 重新采样，开销约为一次全屏 Blit。在移动端（OpenGL ES）上性能较差，适合 PC/主机平台。
- **Camera Render Texture**：提前用单独相机将场景渲染到 RenderTexture，再传入材质参数。移动端推荐此方案，可配合 Opaque Texture（URP 内置的 `_CameraOpaqueTexture`）以约 0.3ms 的额外开销实现同等效果。

## 实际应用

**热浪效果**：在沙漠、岩浆或爆炸后的高温区域，将低强度（`distortionStrength ≈ 0.02`）的扭曲序列帧粒子铺在地面或热源上方，配合缓慢的 8～12 fps 播放速度，即可还原空气因温差产生的连续扰动感。热浪的扭曲场通常为竖向波浪形，RG 通道中 G 方向（垂直）幅度明显大于 R 方向（水平）。

**冲击波**：爆炸瞬间触发的冲击波需要扭曲强度高（`distortionStrength ≈ 0.08～0.15`）且持续时间短（总帧数 16～32 帧，24 fps 播放，约 0.7～1.3 秒播完）。扭曲场从中心向外辐射扩张，美术需要在第 1 帧设置最大环形偏移，随后逐帧衰减至 0，配合粒子透明度曲线同步淡出。

**技能命中扭曲**：在 MMORPG 或动作游戏中，角色受击时播放一帧持续约 0.1 秒的强扭曲帧（可退化为单帧静态图而非序列），再快速衰减，给玩家"打击感"的视觉反馈。此场景下往往不需要完整序列帧，2～4 帧即可。

## 常见误区

**误区一：扭曲强度越大越真实**
初学者倾向于将 `distortionStrength` 设为 0.2 以上，结果背景产生大幅撕裂，看起来像贴图错误而非物理折射。真实的热浪折射率极低，空气折射率约为 1.0003，对应到屏幕空间的位移通常不超过屏幕宽度的 2%～3%，即 0.02～0.03 的归一化偏移。

**误区二：把扭曲序列帧放在透明渲染队列前面**
扭曲粒子必须在被扭曲的背景对象**之后**渲染，否则 GrabPass 或 `_CameraOpaqueTexture` 中并不包含应该被扭曲的内容，效果会完全错误。具体做法是将扭曲材质的 Render Queue 设置为 `Transparent` 的起始值 3000 之后，或明确设为 3001 以确保渲染顺序正确。

**误区三：用法线贴图替代专用扭曲图集**
部分教程直接将法线贴图的 XY 通道当作扭曲向量使用，虽然技术上可行，但法线贴图为切线空间方向，需要额外的空间变换，且其动态范围（16位法线）与为扭曲专门设计的8位 RG 贴图在精度和视觉一致性上存在差异，容易在偏移幅度较大时产生噪点状伪影。

## 知识关联

学习扭曲序列帧需要掌握**自发光序列帧**中的图集采样逻辑（帧索引计算、UV Tiling/Offset 的设置方法），扭曲序列帧在此基础上将"采样颜色并输出"替换为"计算偏移量并采样背景"，两者的帧驱动代码几乎相同，差异仅在 Shader 的最终输出阶段。

在后续学习**序列帧 Shader** 时，扭曲序列帧是理解"多 Pass 特效"和"GrabPass 机制"的直接切入点：序列帧 Shader 会将自发光、扭曲、溶解等多种效果合并到单一 Shader 中，届时需要同时管理多个纹理通道和多套 UV 动画参数，而这些设计决策都可以追溯到扭曲序列帧中对 RG 通道的分离使用逻辑。此外，理解屏幕空间 UV（`ComputeGrabScreenPos` 或 `_CameraOpaqueTexture` 的采样坐标）是后续学习全屏后处理特效（如径向模糊、色散）的直接前置知识。
