# 弹簧系统

## 概述

弹簧系统（Spring System）是粒子物理特效中一种通过弹簧约束（Spring Constraint）将多个粒子连接起来，模拟弹性形变、布料摆动、绳索悬挂等柔体运动的技术框架。其核心思想来源于胡克定律（Hooke's Law）：`F = -k * x`，其中 `k` 为弹性系数（stiffness），`x` 为当前弹簧长度与自然长度（rest length）之差，负号表示弹力方向始终指向平衡位置。当粒子群依据此约束互相牵引时，整个系统呈现出类似真实弹性材料的宏观行为。

弹簧系统的理论基础可追溯至1678年罗伯特·胡克提出的弹性定律。在计算机图形学领域，1987年Jerry Weil在SIGGRAPH上发表的布料仿真论文首次系统性地将弹簧网格（spring mesh）引入粒子动力学，标志着弹簧系统正式成为实时特效的工程工具。现代游戏引擎如Unreal Engine的Niagara和Unity的VFX Graph均内置了弹簧约束节点，允许美术人员在不编写物理求解器的情况下快速搭建布料简化效果。

在特效制作中，弹簧系统提供了一条性价比极高的柔体模拟路径。相比全有限元方法（FEM）每帧数千次矩阵运算，弹簧系统的每粒子计算量仅为O(n·c)，其中n为粒子数、c为平均邻接弹簧数（布料四边形网格中通常c≈4~8），使其能以每帧毫秒级耗时处理数千粒子，非常适合游戏实时特效和影视粒子预可视化阶段。

## 核心原理

### 弹簧约束的力学模型

每根弹簧由两端粒子 `P_i` 和 `P_j`、自然长度 `L_0`、弹性系数 `k` 以及阻尼系数 `d` 共同定义。每个时间步内，系统首先计算当前实际距离 `L = |P_j - P_i|`，再按胡克定律求得弹力幅值 `F_spring = k * (L - L_0)`，方向沿弹簧轴线。为防止纯弹力引发持续振荡，系统额外施加阻尼力 `F_damp = -d * (v_ij · n_ij)`，其中 `v_ij` 为两端粒子相对速度，`n_ij` 为弹簧单位方向向量。合力 `F = F_spring + F_damp` 分别以等量反向形式作用于两端粒子，满足牛顿第三定律。

### 弹簧拓扑类型与布料简化

实际布料简化通常在粒子网格上同时布置三类弹簧：**结构弹簧**（structural spring）连接水平和垂直直接相邻粒子，维持基本网格形状；**剪切弹簧**（shear spring）沿对角线连接，抵抗菱形变形；**弯曲弹簧**（bend spring）跨越一个粒子间隔连接，模拟布料抗折叠刚性。三类弹簧的典型刚度比例约为 `k_struct : k_shear : k_bend = 1.0 : 0.5 : 0.1`，此比例直接影响布料在受力时是偏向"橡皮"还是"棉布"的视觉质感。纯粒子链状连接则可模拟绳索或头发缕，此时仅需结构弹簧即可。

### 数值积分与稳定性

弹簧系统的动力学求解常用显式欧拉（Explicit Euler）或Verlet积分两种方法。显式欧拉公式为：
```
v(t+Δt) = v(t) + (F/m) * Δt
x(t+Δt) = x(t) + v(t+Δt) * Δt
```
Verlet积分则通过 `x(t+Δt) = 2x(t) - x(t-Δt) + a(t)*Δt²` 天然保持能量守恒，无需显式存储速度。显式欧拉在时间步 `Δt > 2/√(k/m)` 时会发散（爆炸），因此在 `k=500 N/m`、`m=0.01 kg` 的典型特效参数下，安全时间步上限约为 `Δt_max ≈ 0.009 s`，即最少需要约111 Hz的子步采样频率。Niagara中常通过将物理子步（Physics Sub-step）设为2~4次来维持稳定性。

### 约束松弛与迭代求解

Position Based Dynamics（PBD）是弹簧系统的另一种实现路线，将弹簧视为距离约束而非力，通过迭代投影直接修正粒子位置：每次迭代按 `Δx = (L - L_0) / (2 * L) * n` 修正两端粒子，迭代3~10次即可收敛。PBD方案天然无条件稳定，但收敛速度受迭代次数限制，弹性效果偏"柔软"，适合布料实时模拟；而基于力的胡克定律方案在参数调整后可模拟更高频振动，适合弹性绳索与弹跳球特效。

## 实际应用

**旗帜与布料特效**：将64×64粒子网格的顶边两角设为固定粒子（质量无限大），其余粒子依三类弹簧连接，配合风力场（Wind Field）施加周期性侧向力，即可实现飘扬旗帜效果。Unreal Engine Niagara中专门提供"Grid Location"模块辅助此类网格初始化，旗帜特效通常将 `k_struct` 设置在200~800之间调节布料硬度。

**角色头发与绳索**：单链弹簧系统（每粒子仅连接前后两个邻居）可模拟角色发丝或武器挂绳。链末端粒子设为自由粒子，链头端固定在角色骨骼挂点。由于绳索不抵抗压缩，需额外添加最小长度约束（`L >= 0`），防止绳索"压缩穿插"。

**爆炸碎片与蛛网**：蛛网结构由多条径向弹簧链和环形弹簧链交叉组合而成，中心节点设为锚点。当子发射器（Sub-emitter）产生的碎片粒子撞击蛛网节点时，可通过修改该节点速度触发弹簧系统响应，模拟蛛网受击形变和震荡传播，是弹簧系统与子发射器联动的典型范例。

## 常见误区

**误区一：弹性系数越大，布料越"真实"**
提高 `k` 值会使布料更硬更接近真实棉布，但同时会缩小数值稳定的安全时间步上限（`Δt_max ∝ 1/√k`）。若不相应增加子步数，系统会在高 `k` 值时产生粒子位置爆炸（爆炸发散），表现为粒子瞬间飞出屏幕。正确做法是：调高 `k` 的同时，同比例增加物理子步数或改用PBD求解器。

**误区二：自然长度等于初始粒子间距**
自然长度 `L_0` 是弹簧无应力状态下的长度，它可以人为设置为小于初始间距，此时弹簧处于预拉伸状态，系统静止时粒子会自发向内收缩，可用于模拟预应力网格或收缩性生物组织。若将 `L_0` 直接硬编码为初始间距，则无法实现此类效果，也会在粒子被子发射器动态添加时引发错误的"弹出"行为。

**误区三：增加阻尼系数可以无限制抑制振荡**
阻尼系数 `d` 过高会导致"过阻尼"（over-damping）：布料失去弹性感，形变后缓慢蠕动回原位，看起来像在蜂蜜中运动。布料弹性特效的临界阻尼条件为 `d_critical = 2 * √(k * m)`，在 `k=300, m=0.01` 时约为 `d_critical ≈ 3.46`，实际审美上通常取 `d = 0.3 * d_critical` 左右以保留适量振荡。
