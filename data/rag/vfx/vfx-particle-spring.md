# 弹簧系统

## 概述

弹簧系统（Spring System）是粒子物理特效中通过弹簧约束（Spring Constraint）将多个粒子相互连接，使粒子群体产生弹性形变、振荡和阻尼运动的模拟技术。每根弹簧约束定义了一对粒子之间的静止长度（Rest Length）与刚度系数（Stiffness），当两粒子间距偏离静止长度时，弹簧力会驱使其恢复，从而模拟布料、绳索、果冻等弹性物体的物理行为。

弹簧系统在数字特效领域的系统性应用可追溯至1986年Pixar短片《小台灯》（Luxo Jr.）制作期间对柔性电线的简化模拟。1995年Provot发表的论文《Deformation Constraints in a Mass-Spring Model to Describe Rigid Cloth Behaviour》奠定了质点-弹簧布料模型的理论基础，提出通过限制单帧内弹簧伸长量不超过静止长度的10%来防止布料过度拉伸，这一"10%约束"至今仍被广泛引用。

弹簧系统之所以在游戏特效与影视粒子模拟中备受重视，是因为它用极低的计算代价实现了视觉上令人信服的弹性效果：相比完整的有限元布料解算，质点-弹簧网格的计算复杂度为O(n)（其中n为粒子数量），可在实时环境中驱动数百个粒子构成的弹性网格。

## 核心原理

### 胡克定律与弹簧力计算

弹簧系统的物理基础是胡克定律。对于连接粒子i和粒子j的弹簧，其弹簧力公式为：

**F = -k · (|d| - L₀) · (d / |d|)**

其中：
- **k** 为弹簧刚度系数（Stiffness），单位N/m，值越大弹簧越硬
- **d** 为从粒子i指向粒子j的位移向量，即 d = xⱼ - xᵢ
- **|d|** 为两粒子当前距离
- **L₀** 为弹簧的静止长度（Rest Length）

该力作用于粒子i，粒子j受到大小相等、方向相反的反作用力。刚度k通常在实时特效中取值范围为50至500 N/m；影视级布料解算中k可达5000 N/m以上。

### 阻尼力与能量耗散

纯弹簧系统（无阻尼）会产生永久振荡，不符合真实物理现象。因此每根弹簧还附加阻尼力（Damping Force）：

**F_damp = -c · (vⱼ - vᵢ) · (d / |d|)**

其中 **c** 为阻尼系数（Damping Coefficient），它沿弹簧方向对相对速度施加制动力。当c取临界阻尼值 **c_critical = 2√(k·m)**（m为粒子质量）时，系统不振荡地最快回归平衡；欠阻尼（c < c_critical）则产生衰减振荡，常用于模拟果冻、橡皮等材质。在Unreal Engine的Cloth模拟器中，阻尼系数默认值为0.01，正是欠阻尼状态以保留布料飘动感。

### 弹簧拓扑类型

质点-弹簧网格通常在粒子之间定义三类弹簧，共同决定布料的宏观行为：

- **结构弹簧（Structural Springs）**：连接网格相邻的水平与垂直粒子（间距1个单元），抵抗拉伸与压缩
- **剪切弹簧（Shear Springs）**：连接对角线方向粒子（间距√2个单元），抵抗平行四边形形变，防止布料过度剪切
- **弯曲弹簧（Bend Springs）**：跨越一个粒子连接间距为2个单元的粒子，模拟布料的弯曲阻力（如硬挺面料的弯折抵抗）

三类弹簧的刚度比例直接决定布料材质感：丝绸的弯曲弹簧刚度远低于结构弹簧；皮革则三类弹簧刚度均较高。

## 实际应用

**布料简化特效**：游戏中角色的披风、旗帜常用8×8到16×16的粒子网格配合弹簧系统模拟。Epic Games在《堡垒之夜》中对旗帜使用了64个粒子（8×8网格）、192根结构弹簧和128根剪切弹簧，以保证在移动平台上60帧运行。顶部一行粒子固定（质量设为无穷大或约束位置），下方粒子自由运动，形成悬挂布料效果。

**绳索与链条特效**：将粒子串联为一维链，相邻粒子间仅设置结构弹簧，可快速构建绳索。此场景中弹簧静止长度L₀等于绳段自然长度，刚度k设置较高（通常>1000 N/m）以防止绳索异常拉伸。Unity的Line Renderer配合粒子弹簧链是实现蜘蛛丝、电线特效的常见方案。

**粒子碰撞后的弹性响应**：在爆炸特效中，粒子碰撞地面后，通过弹簧约束连接的粒子群（如碎片群）会相互牵引产生二次弹跳，增强物理真实感。Houdini的Wire Solver节点正是以质点-弹簧模型为基础实现金属丝、头发等弹性线条的特效解算。

## 常见误区

**误区一：刚度k越高越接近真实布料**

许多初学者认为将刚度系数设得极高（如k=10000）能让布料更"真实"，实际上过高的k会导致数值积分不稳定（刚性方程问题），出现粒子爆炸或穿透现象。使用显式欧拉积分时，时间步长Δt必须满足稳定条件 **Δt < 2√(m/k)**；若k=10000、m=0.1kg，则Δt必须小于约0.006秒。正确做法是配合隐式积分器（如隐式欧拉或Verlet积分）使用高刚度值，而非盲目调高k。

**误区二：阻尼系数c设为0可以模拟"理想弹性材料"**

零阻尼在理论上对应完全弹性振荡，但在实时模拟中会因浮点误差积累导致粒子能量持续增大，最终系统发散。正确模拟弹性橡皮或弹力球时，应将c设为较小的正值（如0.001-0.01），而不是零。

**误区三：弹簧系统可以完整替代布料解算器**

质点-弹簧系统在压缩状态下弹簧会"推开"粒子而非产生真实褶皱屈曲，因此在近距离观察的布料（如特写镜头中的衣物）中明显失真。这种情况需要使用基于位置的动力学（Position Based Dynamics, PBD）或有限元方法（FEM）。弹簧系统适用于远景、实时游戏特效以及绳索类一维弹性结构，而非主角服装的影视级特写模拟。

## 知识关联

**与子发射器的衔接**：子发射器负责在特定触发条件下生成新粒子群（如碰撞后碎裂出的二级粒子）。弹簧系统可在子发射器产生粒子的瞬间自动建立粒子间的弹簧约束，使新生粒子群立即具备弹性连接关系。例如，主发射器的粒子碰撞地面后，子发射器喷出的碎片粒子可通过预设弹簧拓扑形成弹性碎片团，无需对每个粒子单独施加反弹力。

**通向群集行为的桥梁**：群集行为（Flocking/Swarming）要求粒子在感知邻居的基础上协调运动。弹簧系统为此提供了一种基于物理约束的"邻居感知"原型：弹簧连接的粒子天然知道其绑定邻居的位置和速度，这与群集算法中的分离（Separation）和内聚（Cohesion）规则在数学结构上高度相似——都是根据粒子间距离计算力或速度修正量。从弹簧系统的距离恢复力过渡到群集行为的距离感知规则，是粒子物理学习中由约束驱动转向行为驱动的关键一步。
