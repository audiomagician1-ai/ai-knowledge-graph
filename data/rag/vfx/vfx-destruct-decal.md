# 破坏痕迹

## 概述

破坏痕迹（Destruction Decals）是指通过贴花（Decal）技术在物体表面叠加弹痕、裂缝、焦痕、划痕等视觉信息的特效子系统。与变形破坏不同，破坏痕迹不改变物体的几何形态，而是通过投影纹理的方式在表面"印刷"损伤外观，以极低的性能代价营造出真实的破坏感。

贴花系统在游戏引擎中的标准实现可追溯到 2001 年前后的第一人称射击游戏时代，当时 id Software 在《Quake III》中使用 Mark Surfaces 技术将弹孔贴片附着到几何表面，奠定了破坏贴花的基本范式。此后随着延迟渲染管线（Deferred Rendering）的普及，Deferred Decal 成为主流方案，允许贴花在 G-Buffer 阶段修改法线、粗糙度等 PBR 材质通道，而不仅仅是修改漫反射颜色。

破坏痕迹的核心价值在于它是一种**无需重新拓扑的损伤表达**。一颗子弹击中墙体时，如果要通过真实几何变形来表现弹孔，需要实时细分网格并处理物理碰撞体的同步更新；而弹痕贴花只需在命中点投射一个 512×512 或更小的纹理图集片段，渲染耗时通常在 0.1 毫秒以内。

## 核心原理

### 贴花投影与 UV 空间映射

破坏痕迹贴花本质上是一个定向包围盒（OBB），其局部空间的 XY 平面定义贴花朝向，Z 轴定义投影深度。引擎将包围盒内的所有片段重新计算 UV 坐标：`UV = (localPos.xy / boxSize.xy) * 0.5 + 0.5`。这一公式将三维局部坐标线性映射到 [0,1] 的纹理空间，使弹痕纹理可以顺着任意角度的表面法线方向"压印"在几何体上，无论是砖墙还是斜面地板均适用。

延迟贴花（Deferred Decal）在 G-Buffer 写入完毕后、光照计算之前执行。它读取深度缓冲区重建世界坐标，确认片段是否落在包围盒内，再将弹痕的法线扰动和粗糙度变化写回对应的 G-Buffer 通道。这意味着破坏痕迹可以同时改变表面的**漫反射、法线、金属度、粗糙度**四个 PBR 通道——例如焦痕贴花在焦黑区域将粗糙度设为 0.95、金属度降为 0，而中心炭化区甚至可以写入自发光信息来模拟余烬。

### 破坏痕迹纹理图集的结构

专业破坏痕迹资产通常采用纹理图集（Texture Atlas）格式，将多种损伤类型打包进同一张 2048×2048 或 4096×4096 的纹理。一个典型的弹痕图集包含：

- **弹孔本体**：带有放射状裂纹，外圈半透明羽化
- **冲击粉尘环**：以弹孔为中心的浅色粉尘扩散区
- **法线图层**：中心凹陷、边缘隆起的法线信息，使弹孔在灯光下具有立体感
- **粗糙度/金属度变化图层**：金属被击中后的变形金属光泽区

裂缝贴花则与弹痕不同，通常需要**方向性控制参数**，让美术可以沿特定轴线拉伸裂缝图案。在 Unreal Engine 5 中，Material Parameter Collection 允许将裂缝的方向角度和宽度作为运行时参数传入 Decal 材质蓝图，实现不同碰撞力度下裂缝长度的动态调节（常见范围 0.1 到 3.0 的拉伸系数）。

### 破坏痕迹的生命周期管理

场景中大量累积的破坏贴花会显著增加 G-Buffer 的 Overdraw 压力。主流引擎采用**环形缓冲（Ring Buffer）池化策略**：预先分配固定数量（例如 64 个）的贴花对象，最新的破坏痕迹覆盖最旧的，避免无限累积。Unity HDRP 的 Decal Projector 组件支持设置 `fadeFactor` 参数，使旧贴花在被替换前逐渐淡出，而不是突然消失，视觉上更平滑。

焦痕的特殊处理涉及**颜色温度渐变**：爆炸中心贴花使用从纯黑（#1A1A1A）到焦褐（#3D2B1A）再到外圈烟灰色（#6B6B6B）的三段径向渐变，配合法线图表现燃烧后材料的起泡和剥落形态。

## 实际应用

**枪战游戏的弹痕系统**：在《使命召唤：现代战争》（2019）中，不同武器口径对应不同的弹痕贴花尺寸——手枪弹痕半径约 2–4 cm（游戏单位），狙击步枪弹痕半径 6–10 cm，且狙击弹痕附带更长的放射状裂纹。系统在每次命中时查询材质类型（混凝土、金属、木材、玻璃），从对应的贴花图集中随机选取 1/8 的图集分区，保证同一位置不会重复出现完全相同的弹痕。

**环境叙事中的预烘焙破坏痕迹**：破坏贴花不仅用于实时交互，也是关卡美术的叙事工具。废墟场景中美术师会手动在墙壁上放置弹孔、焦痕、划痕贴花，配合几何变形（凹陷、缺口）和粒子烟尘共同构建战争氛围，这类静态贴花通常会被烘焙进光照贴图，不占用运行时 Deferred Decal 的 Pass 预算。

**赛车游戏的车身划痕**：车辆碰撞时，系统沿碰撞切线方向生成方向性划痕贴花，贴花的长度与碰撞相对速度线性相关（例如每 10 km/h 增加 5 cm 划痕长度）。划痕贴花修改车漆的金属度和粗糙度通道，露出底层"裸金属"的材质参数，无需切换整个车体材质。

## 常见误区

**误区一：破坏痕迹可以无限叠加**
许多新手美术认为不断在同一点射击会叠加更多贴花细节，但实际上过度叠加的贴花 Overdraw 会在 G-Buffer Pass 造成显著的性能瓶颈。正确做法是对同一区域半径内的贴花进行**合并或替换**——当新弹痕落点距离已有弹痕中心不足 5 cm（游戏单位）时，用新贴花替代旧贴花，而不是追加一层。

**误区二：贴花法线方向不需要与表面法线对齐**
破坏贴花在曲面（如圆柱体）上投影时，若不根据表面法线修正贴花的局部坐标系，贴花会在曲率变化处产生明显的"拉伸"或"裁剪"失真。需要在贴花的投影方向与实际表面法线夹角超过 60° 时，对贴花边缘执行 `smoothstep` 淡出，避免侧面投影的变形噪点被玩家察觉。

**误区三：焦痕贴花可以用弹痕贴花调色代替**
焦痕和弹痕的法线通道完全不同：弹痕法线表现的是**穿透凹陷**（中心凹、边缘微凸起），焦痕法线表现的是**材料受热膨胀后的起泡剥落**（多处不规则隆起、无中心对称性）。直接对弹痕纹理叠加黄色/黑色调色不会产生可信的焦痕质感，必须使用专门制作的焦痕法线图。

## 知识关联

破坏痕迹系统建立在**变形破坏**的概念之后学习，是对变形破坏的视觉补充：变形破坏处理几何形状的改变，破坏痕迹则以贴花技术填充变形破坏无法覆盖到的细节层次，二者在实际项目中通常配合使用——炮击产生几何缺口（变形破坏），缺口边缘由焦痕贴花进行视觉细化。

学习破坏痕迹之后，自然衔接**破碎音效**的学习。破碎音效的触发时机与破坏贴花的生成事件高度绑定：弹痕贴花生成时同步播放子弹击中材质的音效样本，裂缝扩展时触发结构开裂的音频序列，二者共同构成完整的损伤反馈闭环。理解破坏贴花的材质类型分类体系（混凝土、金属、玻璃等），将直接对应破碎音效资产的分类标签设计
