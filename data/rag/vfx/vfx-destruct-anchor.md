# 锚点与支撑

## 概述

锚点与支撑（Anchor & Support System）是破碎模拟中用于固定刚体碎块、阻止结构在重力或外力作用下自由坍塌的约束机制。在Houdini、Phoenix FD等特效软件的Voronoi破碎流程中，如果不设置锚点，所有碎块在第一帧就会因重力直接散落，无法实现"受力后局部破碎"的视觉效果。锚点本质上是将特定碎块的速度和位移强制归零，使其表现得如同嵌入地面或墙体的固定基座。

该系统的核心思想来源于结构工程中的"约束自由度"概念，在Houdini 16.5版本引入Bullet Solver的簇（Cluster）工作流后被系统化整合进特效管线。一栋建筑破碎时，底部基础碎块通常被标记为锚点，上方楼层碎块通过约束网络（Constraint Network）挂载其上；当炸药或撞击力超过约束的断裂阈值时，约束断开，仅受影响区域的碎块才会脱离锚点运动。

锚点设置的准确性直接决定破碎动画是否符合物理直觉：设置过多会导致结构"铁板一块"，施加大量力量也纹丝不动；设置过少则整栋建筑在0.02秒（即第一个物理步长）内全部坍塌。正确的锚点分布是控制破碎范围与顺序的前提条件。

## 核心原理

### 锚点的技术实现：`active`属性标记

在Houdini的RBD（刚体动力学）系统中，锚点通过在碎块几何体上写入整型点属性 `i@active = 0` 来实现。当 `active = 0` 时，Bullet Solver将该碎块视为**静态刚体**（Static Body），其质量被视为无限大，外力无法改变其位置。对应地，`active = 1` 的碎块为动态刚体，参与完整的物理计算。典型的锚点设置代码如下：

```vex
// 在 Wrangle 节点中，选取Y轴坐标低于阈值的碎块设为锚点
if (@P.y < chf("anchor_height")) {
    i@active = 0;
}
```

这个条件式允许美术师通过一个滑杆参数精确控制"哪个高度以下的碎块保持固定"，而非手动逐块指定。

### 约束网络与支撑传递

锚点单独存在时仅能固定自身，真正支撑整体结构的是**约束网络（Constraint Network）**——一套将相邻碎块两两连接的弹簧-破坏（Spring-Break）关系图。约束的物理参数包含：
- **线性刚度（Linear Stiffness）**：通常设置在 1×10⁴ 到 1×10⁶ 之间，数值越高碎块间越难相对位移
- **断裂阈值（Break Threshold）**：当约束受到的力超过此值时，连接断开，对应的两块碎块分离

支撑力通过约束链从锚点向上传递：锚点→第一层碎块→第二层碎块→…→顶层碎块，形成树状支撑图。若某一层的约束全部断裂，其上方所有碎块（即使自身约束完好）也会因失去支撑路径而整体坍落，这正是爆破拆楼中"层层塌落"效果的物理依据。

### 动态解锁：从静态到动态的切换

高级破碎场景中，锚点不是永久固定的，而是通过**SOP Solver或POP Wrangle**在运行时将 `active` 从0切换为1。典型触发条件包括：
- 与锚点相连的所有约束均已断开（孤立锚点自动激活）
- 外部冲击力超过锚点自身的"根部强度"（Base Strength）阈值
- 时间线到达指定帧数（用于预设的定时爆破效果）

切换时机若偏差1帧（约0.0167秒，按60fps计），就可能产生碎块"穿地"或"悬空跳变"的穿插错误，因此动态解锁通常配合子帧插值（Substep）使用，Bullet Solver的子步长建议设为2~4步。

## 实际应用

**混凝土柱破碎**：在制作桥墩被船只撞击的特效时，桥墩底部埋入地面的20%碎块设为永久锚点（`active=0`），中部碎块通过断裂阈值为5000N的约束连接锚点区，顶部桥面碎块挂载在中部约束网络上。船只碰撞体在中部产生冲击力，触发局部约束断裂，桥面碎块随之下塌，底部锚点区始终保持静止，还原真实桥墩断裂形态。

**定向爆破拆楼**：楼体底层特定位置的支柱碎块设置带有时间条件的动态锚点，按照`$F == 24`、`$F == 30`、`$F == 36`的帧序列依次解锁，模拟炸药延迟引爆。楼体随着支撑点依次失效，从预定方向向下倒塌，整个过程不超过90帧（约3秒），与真实定向爆破的时序控制逻辑完全吻合。

**城墙被投石车击中**：城墙底部全段设为锚点，被弹丸命中的局部区域约束阈值调低至正常值的10%，使撞击区域率先破碎崩散，两侧未被命中的墙体依然屹立，精确还原石质结构的局部破坏特征。

## 常见误区

**误区一：认为锚点越多越稳定**
增加锚点数量确实能防止坍塌，但过度锚定会导致约束网络中出现"超静定"状态——即使某区域的约束已全部断裂，碎块仍因被多余锚点间接固定而无法运动，破碎动画失去真实感。正确做法是只锚定**结构力学意义上的基础支撑点**，通常占总碎块数的5%~15%。

**误区二：把锚点与"粘附约束"混淆**
锚点（`active=0`）是将碎块固定在世界坐标系中的绝对约束，不随任何其他物体运动。粘附约束（Glue Constraint）则是将两个碎块**相互绑定**，两者可以共同运动但不能相对分离。用粘附约束代替锚点时，整组碎块仍会在重力下整体坠落，无法起到固定支撑的作用。

**误区三：忽视锚点碎块的碰撞体积**
即使碎块被设为 `active=0`，它仍然参与碰撞检测。如果锚点碎块的碰撞几何体（Collision Geo）过于简化，动态碎块会穿透锚点区域造成视觉穿插。建议对锚点碎块使用**凸包碰撞体（Convex Hull）**而非简化包围盒，以保证接触面精度。

## 知识关联

**前置概念——簇组管理**：锚点通常不作用于单个碎块，而是作用于**簇（Cluster）**——若干碎块合并成的刚体组。簇组管理决定了哪些碎块被合并为同一簇，锚点标记则决定哪些簇整体固定。如果簇组划分不合理（如将底部支撑碎块和中部碎块混入同一簇），锚定该簇会意外固定不该固定的区域。

**后续概念——变形破坏**：锚点与支撑处理的是刚体的**位移约束**，而变形破坏进一步引入碎块本身的**形变**——碎块在断裂前会发生弯曲、压缩等非刚性形变。变形破坏系统在锚点框架基础上增加了有限元（FEM）或软体约束层，使混凝土弯折、木材劈裂等效果成为可能。掌握锚点的约束传递逻辑，是理解变形破坏中"形变从固定端向自由端传播"规律的直接基础。
