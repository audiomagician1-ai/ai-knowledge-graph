# 血液与液体

## 概述

血液与液体特效是游戏特效领域中专门针对高粘度、非均匀流体（如血液、油漆、泥浆）进行模拟与渲染的技术体系。与雨滴模拟中处理的低粘度薄层水膜不同，血液的粘度约为水的3至4倍（动力粘度约为3–4 mPa·s），这一物理属性决定了血液在表面的扩散速率更慢、边缘更不规则，需要专门的流体衰减模型。

该技术在20世纪90年代末随着第一人称射击游戏的兴起而快速发展。2004年发布的《半条命2》首次在商业游戏中大规模使用程序化血迹贴花系统（Procedural Decal System），其血液溅射能够根据命中角度、表面材质实时生成不同形态的贴花，标志着该技术从预烘焙贴图迈向运行时生成的重要转折点。

在实际游戏开发中，血液与液体的模拟必须在视觉真实度与性能预算之间精确权衡。一个标准的AAA游戏场景中，血迹贴花数量若不加限制可达数千个，因此大多数引擎会设定贴花池上限（通常为256至512个），超出后按照LRU（最近最少使用）策略回收最旧的贴花，这一限制直接影响了美术对血液蔓延持久度的设计决策。

## 核心原理

### 贴花投影与表面附着

血液特效的基础渲染单元是贴花（Decal），其本质是一个朝向法线方向投影的正交体积盒（Orthographic Box Volume）。当血液粒子命中表面时，系统计算命中点的世界法线方向，将贴花纹理沿法线反向投影到表面几何体上，从而实现血迹"粘附"在墙壁、地面甚至角色皮肤上的效果。

为了模拟血液在粗糙与光滑表面上的不同扩散行为，贴花系统通常采样目标表面的粗糙度贴图（Roughness Map）：在粗糙度值高于0.7的表面（如混凝土），血迹边缘被噪波纹理扰动，呈现不规则渗透状；在光滑表面（粗糙度低于0.3，如金属、瓷砖），边缘则更为清晰，并添加高光反射以模拟液体光泽。

### 扩散动画与粘度模型

血液流淌的视觉核心是时间驱动的UV展开动画。引擎通过在着色器中随时间`t`推进一个蔓延遮罩（Spread Mask）的阈值来模拟液体向外扩散：

```
spreadRadius(t) = maxRadius × (1 - e^(-t / τ))
```

其中`τ`为粘度时间常数，血液的`τ`通常设定为0.8至1.5秒，而水的`τ`接近0.1秒。当`t`远大于`τ`时，蔓延半径趋近于`maxRadius`，模拟血液静止凝固后的最终形态。此公式基于指数衰减函数，确保扩散速度初始快、后期趋缓，符合高粘度流体的物理行为。

对于垂直表面上的血液流淌（如墙壁上的血迹向下滴落），还需在UV动画中叠加沿重力方向的偏移量，偏移速度受粘度参数控制，并在流淌末端生成拉丝（Drip Tail）形态，通常通过预制的Drip Sprite粒子与贴花系统联动实现。

### 颜色随时间的变化（氧化模拟）

真实血液因血红素氧化，会在约10至30分钟内从鲜红色（RGB近似值：#8B0000 至 #CC0000）逐渐变为深棕色（#3B0000 至 #5C1010）。游戏中通过在材质的Shader中引入一个`oxidationTimer`浮点参数，以线性插值或曲线插值在两个颜色锚点之间过渡：

```
finalColor = lerp(freshBloodColor, driedBloodColor, saturate(oxidationTimer / 1800.0))
```

这一机制在《最后生还者》系列中被大量使用，使玩家能通过血迹颜色判断战斗发生的时间，形成叙事上的环境信息传递功能。

## 实际应用

**射击游戏命中反馈**：在《战地》系列中，血液溅射特效同时服务于视觉反馈和打击感设计。子弹命中角色时，系统根据命中角度将溅射粒子分为两组：沿子弹方向前向溅射的细小血雾粒子（粒子数约30至50个），以及垂直于命中面的大颗粒血滴（约10至15个），后者在命中表面生成贴花。两组粒子的速度范围、生命周期均不同，确保远近观察时视觉效果都具有说服力。

**油漆与涂鸦模拟**：《斯普拉遁》将油漆液体作为核心玩法元素，其墨水覆盖系统使用一张全局Render Texture记录场景地面的颜色归属，每次油漆命中时以GPU绘制操作（DrawCall）在该纹理上叠加一个形状不规则的墨水印记。这种方案将整个地图的油漆状态压缩为单张512×512的纹理，极大降低了内存占用。

**医疗/恐怖游戏中的创伤视觉**：在《寂静岭》等恐怖游戏中，血液贴花常与程序噪波（Perlin Noise）结合生成异常形态，使血迹呈现出违反物理规律的蔓延路径，从而放大恐怖氛围。这是血液特效技术被刻意反向使用的典型案例——物理上的"错误"在叙事上是"正确"的。

## 常见误区

**误区一：血液模拟应追求完全物理正确**
许多初学者认为血液特效应该严格遵循纳维-斯托克斯方程进行流体模拟，但实际上游戏中的血液几乎从不使用真实流体求解器。原因是血液的粘性流动在视觉上细节极其丰富，完整模拟一帧的计算量远超实时预算。贴花+UV动画+粒子的组合方案能以不到真实流体计算1%的开销，达到80%以上的视觉可信度，这种"视觉近似优于物理精确"的思路是该领域的核心工程哲学。

**误区二：贴花数量越多，画面越真实**
无限制叠加血迹贴花会带来两个具体问题：一是深度测试与混合操作（Alpha Blending）的透支（Overdraw）急剧上升，在移动平台上会造成明显帧率下滑；二是过多贴花叠加会产生颜色饱和过度的视觉问题，使画面显得极不自然。专业的做法是在同一位置叠加超过3至4层贴花时，将底层贴花烘焙合并为一张静态贴图，以此控制Overdraw层数。

**误区三：所有液体用同一套参数**
血液（粘度高、氧化变色）、油漆（粘度中等、颜色固定）、水（粘度低、透明）三种液体在`τ`值、边缘噪波强度、反射率上差异显著。将血液的粘度时间常数`τ = 1.2s`直接套用在水的模拟上，会得到"流动极其迟缓的水"，视觉上明显失真。每种液体材质都需要独立调校其物理参数集。

## 知识关联

从前置知识**雨滴模拟**迁移到血液与液体时，最关键的扩展点在于粘度参数的引入。雨滴模拟中液体几乎被视为无粘度的薄膜，而血液的高粘度特性要求在蔓延公式中显式建模时间常数`τ`，这是两套系统在数学模型上最核心的分叉点。此外，雨滴模拟通常只处理下表面（地面、水平玻璃），而血液贴花必须处理任意角度的表面投影，需要掌握贴花的法线空间投影矩阵计算。

向后续课题**GPU流体计算**延伸时，血液与液体中介绍的UV蔓延遮罩技术是理解GPU流体求解器的直观铺垫。GPU流体计算将把本章中用动画关键帧近似的扩散过程，升级为在计算着色器（Compute Shader）中逐像素迭代求解的真实偏微分方程，处理的流体自由度（Degree of Freedom）从数十个贴花上升到百万级别的网格单元，计算范式发生了质的跃迁。
