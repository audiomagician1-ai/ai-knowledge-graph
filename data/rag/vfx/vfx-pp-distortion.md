# 画面扭曲

## 概述

画面扭曲（Screen Distortion）是一种通过偏移屏幕像素采样坐标来模拟光线折射、冲击波传播或介质形变的后处理特效技术。其核心数学原理是在Fragment Shader中对屏幕纹理（Screen Texture）进行UV扰动采样：将原始UV坐标加上一个由法线图或噪声图驱动的偏移量，使最终渲染像素从错误的屏幕位置读取颜色，从而产生视觉上的扭曲感。

该技术最早在游戏中大规模应用于2000年代初期的爆炸冲击波效果，《半条命2》（2004年）的AR2冲击波枪是经典案例之一。在热浪场景中，《孤岛危机》系列对热力形变的模拟将该技术推广到环境特效领域。不同于粒子系统或几何形变，画面扭曲完全在屏幕空间（Screen Space）内完成，不修改任何3D几何体，因此性能开销主要集中在额外一次全屏Pass的带宽消耗上。

该特效在热浪、水下视角、爆炸冲击、传送门边缘、幽灵显现等场景中不可替代，原因在于它能在几乎零几何成本的前提下，向玩家传递"空气密度异常"或"空间介质扭曲"的物理直觉。

## 核心原理

### UV偏移与扰动贴图

画面扭曲的基础公式为：

```
distortedUV = screenUV + sampleNormal(distortionTex, uv * tiling + time * speed).rg * intensity
```

其中 `distortionTex` 是一张法线图或噪声图，`.rg` 通道分别提供X和Y方向的偏移分量，`intensity` 控制最大扭曲像素偏移量（通常取值范围为0.005～0.05，超过0.1时会出现明显穿帮）。`time * speed` 使扰动贴图随时间滚动，模拟热气流的持续流动。

最终效果的扰动采样公式为：
```
finalColor = tex2D(screenTexture, distortedUV)
```

### 冲击波扭曲的环形扩散算法

冲击波型扭曲与热浪扭曲的核心区别在于：冲击波是**径向扩散**的，需要计算当前像素到冲击波圆心的距离，并用该距离生成一个环形遮罩。典型实现中，扭曲强度 = `sin((dist - waveRadius) * frequency) * envelope`，其中 `waveRadius` 随时间线性增大（速度约80～200单位/秒），`envelope` 是一个随扩散衰减的包络函数，使冲击波圆环宽度固定而整体淡出。当 `dist > waveRadius + ringWidth` 时强度归零，保证波前之外的区域不受干扰。

### 折射系数与物理参数映射

水下扭曲需要模拟斯涅尔定律（Snell's Law）的折射效果：水的折射率约为1.333，这意味着从水下看水面上方的物体时，视线偏折角约为48.6°（全内反射临界角）。在实现中，通过水面法线图的扰动方向与观察向量的点积来动态调整UV偏移量，使摄像机朝向水面正上方时扭曲最小，侧视时扭曲最大，从而近似物理折射行为。扰动频率通常设为0.3～0.8Hz模拟水波周期。

### 深度感知遮罩

在冲击波或热浪特效中，若不加深度遮罩，扭曲会错误地作用于HUD（血条、准星等UI元素）以及近处物体。解决方案是在Shader中采样深度缓冲（Depth Buffer），当像素深度低于特定阈值（通常为0.9，即距摄像机极近处）时将扭曲强度乘以0，使近景和UI免疫扭曲。这一步骤也是画面扭曲区别于简单全屏滤镜的重要特征。

## 实际应用

**热浪效果**：在沙漠或熔岩场景中，热浪扭曲贴图采用垂直方向为主的噪声图，Speed设为0.02～0.05（垂直方向），Intensity约0.008～0.015，模拟热气向上流动的视觉特征。Unity URP中可通过将扭曲Pass插入`AfterRenderingTransparents`节点实现，避免透明物体被扭曲穿帮。

**爆炸冲击波**：在爆炸发生后0.05秒触发冲击波扭曲Shader，将 `waveRadius` 初始化为爆炸世界坐标投影到屏幕的半径（约5～15像素），以每帧+30像素的速度扩散，在半径达到屏幕对角线长度的60%时开始淡出，总持续时间0.3～0.6秒。

**传送门/魔法边缘**：将扭曲范围限制为传送门几何体的Stencil遮罩区域，配合色差（Chromatic Aberration）偏移R/G/B通道各±2～4像素，产生超自然的时空折叠感。

## 常见误区

**误区一：Intensity设置过大导致"背面穿帮"**。许多初学者将Intensity调至0.05以上追求夸张效果，但这会导致distortedUV超出屏幕边界或采样到完全无关的区域（如地面采样到天空），产生明显的错误色块。正确做法是用 `clamp(distortedUV, 0.001, 0.999)` 限制采样边界，并保持Intensity低于0.04。

**误区二：将画面扭曲与顶点形变（Vertex Displacement）混淆**。画面扭曲仅改变屏幕像素的采样来源，不移动任何顶点，因此物体的轮廓边缘在3D空间中不变形。若需要几何体本身扭曲（如软体物理），必须在顶点着色器中进行位移，两者技术路径完全不同，不可互相替代。

**误区三：在前向渲染中对半透明物体做扭曲**。若扭曲Pass早于透明物体渲染，透明物体将不受扭曲影响；若晚于透明物体，则透明粒子本身也会被扭曲，产生错误的双重折射感。正确的插入时机是在所有不透明物体之后、UI之前，并通过深度检测跳过已标记为HUD层的像素。

## 知识关联

画面扭曲建立在**描边效果**的屏幕空间Pass概念之上——描边效果教会我们如何对全屏纹理进行二次采样处理，而画面扭曲将这个采样步骤从固定偏移扩展为由动态贴图驱动的可变偏移。来自**碎片与残骸**阶段对爆炸生命周期的理解（触发时机、衰减曲线）直接指导了冲击波扭曲的时序参数设计：碎片散开速度与冲击波扩散速度应保持协调，通常冲击波速度是碎片平均速度的1.5～2倍，才能产生可信的物理感。

在掌握画面扭曲之后，**镜头光晕**（Lens Flare）是顺序学习的下一个后处理主题。两者的联系在于：强烈爆炸特效往往同时触发冲击波扭曲和镜头炫光，需要在同一后处理管线中协调两个Pass的渲染顺序——通常扭曲Pass先执行以变形背景，光晕Pass后执行以在扭曲后的画面上叠加光学伪影，从而保证视觉层次的正确性。
