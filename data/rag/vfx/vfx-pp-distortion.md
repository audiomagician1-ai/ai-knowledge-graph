# 画面扭曲

## 概述

画面扭曲（Screen Distortion）是一种后处理特效技术，通过对渲染完成的屏幕图像进行UV坐标偏移，模拟热浪、爆炸冲击波、水下折射、传送门边缘等视觉现象。与描边效果或碎片效果在世界空间中操作不同，画面扭曲完全在屏幕空间（Screen Space）中运算，因此它影响的是最终呈现给玩家的整张画面，而非场景中的单个物体。

该技术的核心思路最早来自光学物理中的折射定律（斯涅尔定律：n₁sinθ₁ = n₂sinθ₂），但在实时渲染中并不真正计算折射角，而是用一张法线贴图或噪声贴图来驱动像素偏移量，以低廉的计算成本近似模拟折射视觉效果。Unity引擎在URP/HDRP管线中通过"Distortion"Pass实现该效果，其内置的偏移强度参数单位为屏幕UV的百分比（0.0 ~ 1.0）。

热浪扭曲、冲击波扩散和水下涟漪虽然视觉差异显著，但它们共用同一套UV偏移框架，区别仅在于驱动贴图的类型、动画速度和偏移幅度。掌握这套框架可以用极少的Shader代码覆盖游戏中90%以上的折射类屏幕特效需求。

## 核心原理

### UV偏移公式

画面扭曲的本质是将屏幕UV坐标从原始位置`uv`偏移到新位置`uv + offset`，然后用新UV采样屏幕颜色缓冲：

```
finalColor = tex2D(_ScreenTex, screenUV + distortionOffset * _Intensity)
```

其中`distortionOffset`通常来自一张RG双通道的法线贴图，将其数值从[0,1]重映射到[-1,1]区间后乘以强度系数`_Intensity`。`_Intensity`的合理取值范围为0.005 ~ 0.05（相对于屏幕宽度），超过0.1时画面通常会出现明显撕裂感，破坏真实性。

### 驱动贴图的三种类型

**噪声型贴图**：Perlin噪声或Simplex噪声生成的RG贴图，配合时间变量`_Time.y * _Speed`做UV滚动，天然适合热浪和火焰边缘的随机抖动效果。典型滚动速度设置为0.2 ~ 0.8 UV单位/秒。

**径向法线贴图**：以圆心向外辐射的渐变法线贴图，静止时无扭曲，随冲击波半径扩张动态缩放UV采样范围，专门用于爆炸冲击波的环形扩散效果。其扭曲强度随半径增大而快速衰减，衰减曲线常用`1 - smoothstep(0.3, 1.0, radius)`来控制外边缘的硬截断。

**折射法线贴图（水面型）**：基于正弦波叠加生成的法线贴图（通常为两层不同频率、不同速度的噪声叠加），模拟水面涟漪的规律性折射，方向性更强。Unity的Water系统中默认使用两张法线贴图以0.05和0.07的不同速度分别沿X轴和Y轴移动并做加权混合。

### 深度遮罩与物体边界处理

裸UV偏移会导致一个经典BUG：扭曲区域边界处的前景不透明物体出现"撕裂"或"透视穿帮"，即扭曲采样到了不该显示的背景像素。解决方案是引入深度差值遮罩：采样当前像素深度`depthScene`与扭曲源物体深度`depthDistort`，当`depthScene < depthDistort`时将偏移量归零，公式为：

```
offset *= step(depthDistort, depthScene);
```

这一深度测试步骤使扭曲效果只作用于扭曲源"后方"的像素，从而正确保留前景遮挡关系。

## 实际应用

**热浪效果（Heat Haze）**：在荒漠关卡或火焰场景中，将Perlin噪声贴图绑定到粒子系统的Billboard面片上，面片设置为加性透明混合，启用Distortion Material并将`_Intensity`设为0.015，`_Speed`设为0.4。由于面片跟随摄像机朝向，玩家移动时扭曲区域会自然随视角变化，无需额外计算。

**爆炸冲击波**：在爆炸粒子特效触发后0.05秒延迟，生成一个径向扭曲Mesh（外半径从0扩张至8米，持续0.3秒后消失）。强度曲线从峰值0.04在0.3秒内线性衰减至0，结合冲击波粒子的运动感知效果显著增强爆炸打击感。《战地》系列的近身爆炸扭曲正是采用类似的时序参数设计。

**水下场景**：在摄像机进入水体触发器时，全屏后处理Pass激活，用两层法线贴图以不同速度混合驱动整张屏幕的UV偏移，同时叠加蓝绿色色调（Color Grading）和景深模糊，使玩家立刻感知到介质切换，无需更换场景。偏移强度通常低至0.008以维持可读性。

## 常见误区

**误区一：强度越大效果越震撼**。许多初学者将`_Intensity`设到0.1以上，结果画面出现严重的像素块状撕裂和颜色采样错误。实际上0.02左右的强度在大多数情况下已经足够写实，高强度只适用于极端近场冲击波的单帧峰值，且峰值持续时间不应超过3帧。

**误区二：画面扭曲可以替代折射Shader**。真正的玻璃球、水晶等透明物体折射应在物体渲染阶段用GrabPass或Scene Color Node实现，而非后处理扭曲。后处理扭曲无法处理物体自身的折射高光，也无法正确响应物体的曲率变化，强行使用会导致折射方向与物体法线完全对不上。

**误区三：扭曲贴图分辨率越高越好**。驱动扭曲的噪声或法线贴图分辨率超过256×256通常没有视觉提升，因为偏移量本身是模糊的连续梯度，高频细节在偏移运算中会被均匀化。使用128×128并开启双线性过滤是性能与质量的最优平衡点。

## 知识关联

从**描边效果**过渡到画面扭曲，意味着Shader的操作对象从世界空间法线转向了屏幕空间UV，理解描边中使用的屏幕法线外扩逻辑有助于快速建立"屏幕空间操作"的直觉。**碎片与残骸**特效中产生的爆炸时序信息（起爆时间、冲击半径）直接为冲击波扭曲提供驱动参数，两套系统在时序上协同工作。

在完成画面扭曲的学习之后，下一个后处理特效**镜头光晕（Lens Flare）**同样在屏幕空间中运算，但其UV操作以光源屏幕坐标为原点做辐射状分布，与扭曲的随机噪声驱动形成互补。掌握了UV偏移强度的控制手感后，镜头光晕的遮挡检测与亮度衰减算法会显得更直观。
