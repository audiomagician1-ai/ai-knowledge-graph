# 极坐标变换

## 概述

极坐标变换是Shader中将UV坐标系从笛卡尔直角坐标（x, y）转换为极坐标（r, θ）的数学操作，其核心公式为：`r = sqrt(x² + y²)`，`θ = atan2(y, x)`。通过这一转换，原本沿水平或垂直方向的纹理图案会被"卷绕"成围绕中心点旋转的环形结构，直接赋予特效漩涡、传送门、雷达扫描等视觉形态。

这一技术在游戏引擎Shader中被广泛使用，Unity的ShaderGraph内置了Polar Coordinates节点，其底层实现正是上述两个公式的组合。从历史角度看，极坐标在数学领域由雅各布·伯努利于1691年系统化描述，但将其引入实时图形着色器成为标准技法，是随着可编程管线（约2002年DirectX 9引入HLSL）普及后才真正流行的。

在游戏特效设计中，极坐标变换的价值在于：只需对一张普通的横向渐变纹理进行UV重映射，无需手工绘制旋转贴图，就能生成完整的旋涡动画。配合时间变量对θ进行偏移，仅需一行额外代码 `θ += _Time.y * speed` 即可实现持续旋转动效。

## 核心原理

### 坐标中心化与归一化

进行极坐标变换前，必须先将UV原点从左下角（0,0）移到画面中心（0.5, 0.5）。操作方式是令 `uv_centered = uv - float2(0.5, 0.5)`。若跳过这一步，atan2计算的角度将以画面角落为圆心，产生扭曲而非正圆形旋涡。在非正方形纹理（如宽高比为16:9的渲染目标）中，还需用 `uv_centered.x *= aspectRatio` 修正长宽比，否则圆形会被压扁成椭圆。

### r与θ的实际含义与范围

- **r（半径）**：表示当前像素距离中心点的距离，范围在正方形UV下为 `[0, ~0.707]`（中心到角点的距离是 sqrt(0.5² + 0.5²) ≈ 0.707）。将r归一化到[0, 1]时，通常除以0.5而非0.707，因为这样圆形边界恰好贴合画面四边，圆外区域r > 1可以直接clip掉。
- **θ（角度）**：由HLSL/GLSL的 `atan2(y, x)` 返回，范围是 `[-π, π]`（即约 -3.1416 到 3.1416）。使用前通常归一化为 `[0, 1]`：`theta_n = (atan2(y, x) + PI) / (2 * PI)`，这样θ可以直接作为横向UV采样纹理，形成完整的360°环形贴图。

### 旋涡效果的实现机制

漩涡效果的关键是在θ分量上叠加一个与r相关的偏移量。公式为：`θ_swirl = θ + r * swirlStrength`。当swirlStrength为6.28（即2π）时，外圈纹理恰好比内圈多旋转整整一圈，形成经典漩涡形态。若改为 `θ_swirl = θ + (1 - r) * strength`，则内圈旋转更剧烈、外圈平缓，产生类似传送门拉伸的吸入感。最终用 `(r, θ_swirl)` 重新组合成直角坐标进行纹理采样，公式为：`x = r * cos(θ_swirl)`，`y = r * sin(θ_swirl)`。

## 实际应用

**传送门特效**：将一张云雾噪声贴图（后续噪声生成章节会介绍其制作方式）赋予极坐标UV，θ随时间线性递增（乘以 `_Time.y`），r通道叠加 `smoothstep(0.4, 0.5, r)` 控制边缘透明度，即可实现带旋转内核和清晰圆形边框的传送门。Unity内置的Portal Shader案例中正是采用这一三层结构。

**雷达扫描线**：将极坐标的θ分量映射到一个细窄的高光条纹（如 `frac(θ_n * 1.0 - _Time.y * 0.3) < 0.05` 则为高亮），并叠加r方向的衰减，即可生成顺时针旋转的雷达扫掠光束，常见于策略游戏的战场探索UI。

**漩涡扭曲采样**：不将极坐标用于直接上色，而是用其计算出的偏移量去扭曲另一张背景纹理的采样坐标，可制作"黑洞吸引周围环境"的视觉形变效果。此时swirlStrength建议控制在1.0到3.0之间，超过5.0会使纹理出现过度重复的走样现象。

## 常见误区

**误区一：忘记处理atan2的不连续性**
`atan2` 在θ = ±π处（即UV左侧水平轴）存在数值跳变，从+3.1416 突然变为-3.1416。若纹理在这一角度处本身有渐变，则会出现一条明显的缝合线。修复方法是使用有符号角度差或确保纹理在θ方向是无缝循环贴图（横向wrap模式设为Repeat）。

**误区二：认为r归一化到0.707比到0.5更"正确"**
将r除以0.707确实能让正方形画面的四角r值恰好等于1.0，但实际项目中几乎没有效果需要把能量分布到画面角落。除以0.5使圆形边界与画面四边相切，便于clip和蒙版操作，是更实用的标准做法。

**误区三：直接对极坐标结果进行线性插值**
θ是角度量，对两个θ值做lerp可能走"短弧"或"长弧"，取决于具体数值的正负。例如从θ=0.1插到θ=0.9在数值上是线性的，但从θ=0.05插到θ=0.95如果写成 `lerp(-0.05*2π, 0.05*2π)` 则方向完全相反。旋转动画应始终通过对 `_Time.y` 乘以角速度后直接累加到θ实现，而非在两个固定θ值之间插值。

## 知识关联

极坐标变换以**深度渐变**所建立的UV重映射思维为前提——理解如何将[0,1]区间的UV值用来索引颜色渐变，才能进一步理解把(r, θ)重新组合成UV坐标去采样纹理的逻辑。深度渐变处理的是线性UV偏移，而极坐标变换处理的是角度与半径的非线性重映射，两者共享"UV作为间接索引"的核心思路但操作空间从线段扩展到了平面。

后续的**噪声生成**与极坐标变换的配合是游戏特效中最常见的组合之一：将FBM（分形布朗运动）噪声采样的UV替换为极坐标UV，可以直接生成圆形分布的有机纹路，免去手动绘制贴图的工作。具体来说，把极坐标的θ作为噪声采样的横坐标、r作为纵坐标，可以得到天然围绕中心点对称分布的旋涡状噪声，这是传送门、魔法阵等特效的基础构建块。
