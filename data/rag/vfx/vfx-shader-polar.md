# 极坐标变换

## 概述

极坐标变换（Polar Coordinate Transform）是一种将笛卡尔坐标系（x, y）中的UV坐标转换为极坐标系（r, θ）的数学映射技术。在Shader编程中，这种转换将矩形纹理的横轴映射为旋转角度、纵轴映射为径向距离，从而使原本线性的图案自然弯曲成环形或螺旋状。

该技术的数学基础来自极坐标系的经典定义，转换公式为：`r = sqrt(x² + y²)`，`θ = atan2(y, x)`，其中r代表像素到中心点的距离，θ代表该像素相对于中心的方位角（范围为 -π 到 π，约 -3.14159 到 3.14159）。在游戏引擎和实时图形应用中，极坐标变换能以极低的计算成本实现漩涡、传送门、法阵光环等视觉效果，是Unity Shader Graph和Unreal Material Editor中的常用节点功能。

极坐标变换之所以在特效制作中广泛使用，是因为它将"旋转运动"从复杂的矩阵运算简化为UV偏移运算——只需对θ分量加上一个随时间变化的值，就能驱动整个纹理产生旋转效果，计算量远低于传统旋转矩阵变换。

## 核心原理

### 笛卡尔到极坐标的转换流程

在Shader中执行极坐标变换，通常需要先将UV坐标重新居中。标准UV的范围是[0, 1]，需先减去0.5使坐标原点移至中心，变为[-0.5, 0.5]的范围。随后应用转换公式：

```glsl
vec2 centered = uv - vec2(0.5, 0.5);
float r = length(centered);           // 等价于 sqrt(x²+y²)
float theta = atan(centered.y, centered.x); // atan2函数
vec2 polarUV = vec2(theta / (2.0 * PI), r);
```

最终，`theta / (2π)` 将角度归一化至[0, 1]区间作为新UV的U分量，r作为V分量。此时对新UV进行纹理采样，矩形贴图的内容将被"卷曲"成一个以中心为圆心的圆环状排列。

### 旋转动画的实现机制

极坐标变换的旋转动画本质上是对θ分量进行时间偏移。将`_Time.y`（Unity中的全局时间变量）乘以速度系数后加到归一化角度上：

```glsl
float animatedTheta = (theta / (2.0 * PI)) + _Time.y * _RotationSpeed;
vec2 animatedPolarUV = vec2(frac(animatedTheta), r);
```

`frac()`函数确保U值维持在[0, 1]范围内形成平滑循环。通过这种方式，一张静态的辐射纹理可以被驱动产生持续旋转效果，而无需每帧重新生成几何体或修改顶点数据。

### 漩涡效果的螺旋偏移

漩涡（Swirl）效果在纯旋转基础上增加了径向耦合：让旋转角度随距离r的变化而叠加额外偏移。公式为 `theta_swirl = theta + r * _SwirlStrength`，其中`_SwirlStrength`控制螺旋的紧密程度。当该值设置为6.28（即2π）时，图案会产生恰好一圈完整的螺旋弯曲。这种r与θ的线性耦合正是漩涡区别于简单旋转的核心特征，也是传送门类特效实现"中心加速外圈减速"视觉感受的数学依据。

## 实际应用

**传送门光环特效**：将一张从左到右颜色渐变的横向纹理（例如蓝→白→蓝的渐变条）通过极坐标变换展开，即可得到一个颜色在圆周方向渐变的光环。叠加时间驱动的θ偏移后，该光环会持续旋转，配合r分量的Alpha裁切（r < 0.4时透明，r > 0.5时透明）可精确控制光环宽度。

**法阵符文旋转**：将一张包含符文图案的方形贴图应用极坐标变换后，符文会以中心点向外辐射排列。设置两个不同旋转速度的层（如外圈速度0.3、内圈速度-0.5），可实现反向旋转的双层法阵效果，这种效果在《魔兽世界》法师技能视觉设计中有大量参考案例。

**流体漩涡与水面效果**：将噪声纹理先经过极坐标变换再配合`_SwirlStrength = 4.0`的螺旋偏移，可模拟排水口漩涡或魔法吸引效果。相比直接用顶点动画模拟流体，这种纯纹理方式的移动端性能开销减少约60%。

## 常见误区

**误区一：忘记居中处理导致旋转中心偏移**
许多初学者直接对[0, 1]范围的UV应用`atan2`函数，此时坐标原点在图像左下角，旋转中心会错位至角落而非图像中心。正确做法是先减去`vec2(0.5)`将原点移至中心，转换完成后视需求决定是否将r的范围从[0, 0.707]（对角线距离）重新归一化。

**误区二：混淆atan与atan2的适用场景**
单参数的`atan(y/x)`在x=0时会产生除零错误，且无法区分第二、三象限（均返回相同符号），导致半边图像错误翻转。双参数的`atan(y, x)`（GLSL）或`atan2(y, x)`（HLSL）能正确处理所有四个象限，θ值覆盖完整的[-π, π]范围，在极坐标变换中必须使用后者。

**误区三：误以为极坐标变换可以直接替代旋转矩阵**
极坐标变换改变的是纹理的采样方式（UV重映射），而非几何体本身的旋转。对一张各向异性的贴图（如箭头图标）应用极坐标变换后，图案会发生拉伸畸变——距中心越近（r趋近于0），纹理压缩越严重，圆心处会出现明显的"汇聚点"撕裂问题。此类情况应在纹理设计阶段预留中心空白区域（通常留出半径0.15以内的圆形空白）。

## 知识关联

**前置概念——深度渐变**：深度渐变训练了对UV坐标进行数学运算和重映射的基本思维。极坐标变换在此基础上引入了`length()`和`atan2()`两个新的坐标计算函数，r分量本质上就是以中心点为参考的"径向深度"，与深度渐变中的线性距离计算逻辑一脉相承。

**后续概念——噪声生成**：掌握极坐标变换后，可将生成的极坐标UV直接输入噪声采样函数，使原本方向随机的噪声图案自动呈现出圆形分布特征。更重要的是，在极坐标空间中对噪声的θ分量进行时间偏移，能以极低成本制作出旋转扭曲的有机噪声动画，这是火焰旋涡、魔法球等高级特效的标准技术组合。
