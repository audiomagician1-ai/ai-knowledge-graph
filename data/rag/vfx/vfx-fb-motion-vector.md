# 运动矢量

## 概述

运动矢量（Motion Vector）是一种编码像素或像素块在相邻帧之间位移信息的技术手段，通常以二维向量场（UV格式）存储在纹理贴图中。在序列帧特效系统中，运动矢量贴图的每个像素的R、G通道分别记录水平（X轴）和垂直（Y轴）方向的位移量，使得渲染引擎能够在两帧图像之间精确地"追踪"每个区域的运动轨迹，从而实现帧间运动补偿（Motion Compensation）。

运动矢量的概念最早源于视频压缩领域，H.264/AVC等编码标准从1990年代起就大量依赖运动矢量估计来减少冗余数据。在游戏特效领域，这一概念在2010年代随着高质量烟雾、火焰、流体等序列帧特效的普及而被引入，成为提升低帧率序列帧平滑度的关键技术。Unreal Engine 4.20及之后的版本内置了对运动矢量贴图的原生支持，Unity的Shader Graph同样提供了相关节点。

运动矢量之所以重要，在于它解决了序列帧特效固有的"跳帧"问题。一套30帧的爆炸序列帧若以24fps播放，每帧停留约41.7毫秒，肉眼可见明显的帧切换抖动。通过运动矢量引导的插值，渲染引擎可以在任意子帧位置合成出"虚拟帧"，使视觉连续性大幅提升，而无需增加原始素材的帧数。

## 核心原理

### 运动矢量贴图的编码格式

运动矢量贴图采用RG16F（16位浮点双通道）或RG8（8位无符号双通道）格式存储。以RG8为例，数值0对应偏移量-1，数值127对应偏移量0（无运动），数值255对应偏移量+1，编解码公式为：

**实际偏移量 = (texel值 / 255.0) × 2.0 - 1.0**

对于UV空间中的偏移，+1.0代表向右/向下偏移一个完整纹理宽度。因此在实际使用时，运动矢量的幅度通常都很小（如0.01~0.05范围内），代表帧间局部像素仅移动了纹理尺寸的1%~5%。使用RG16F精度更高，适合大幅度运动或需要精确追踪细节的特效（如液体表面涟漪）。

### 帧间运动补偿的插值算法

在着色器中，设当前时间在第A帧和第B帧之间的混合因子为 `t`（0~1），运动矢量贴图为 `motionTex`，则像素颜色计算如下：

```
vec2 motion = texture(motionTex, uv).rg * 2.0 - 1.0;
vec4 colorA = texture(seqTex, uv - motion * t);
vec4 colorB = texture(seqTex, uv + motion * (1.0 - t));
vec4 result = mix(colorA, colorB, t);
```

关键在于A帧采样时UV沿运动方向**反向**偏移 `t` 倍，B帧采样时UV沿运动方向**正向**偏移 `(1-t)` 倍。这样两帧的像素在子帧位置"相向而行"地汇聚到正确位置，消除了简单线性混合导致的重影（Ghost Artifact）。

### 运动矢量的生成方式

运动矢量贴图通常通过两种途径生成：**离线流体模拟导出**和**光流算法计算**。

离线导出方式常见于Houdini制作的流体特效：在模拟完成后，使用Houdini的`moVec`或专用ROP节点将每帧的速度场（Velocity Field）以纹理形式输出，速度向量经过归一化后写入RG通道。这种方式精度最高，矢量与实际物理运动完全对应。

光流算法（Optical Flow）则作用于已有的图像序列，通过分析相邻帧的像素亮度梯度变化估算运动方向。Flowframes、DaVinci Resolve的光流插帧、以及Adobe After Effects的"像素运动"效果均基于这一原理。对于非物理模拟的手绘序列帧（如2D动画），光流是唯一可用的运动矢量生成方式，但在遮挡边界处容易产生误差。

## 实际应用

**火焰与烟雾特效**：一套8×8排列（共64帧）的火焰Flipbook通常配套一张同等分辨率的运动矢量Flipbook。在UE5的Niagara中，`FlipbookBlend`模块直接接收运动矢量贴图输入，实现60fps下的平滑火焰播放，即便源素材只有24fps。

**水面涟漪序列帧**：涟漪的向外扩散运动是典型的径向位移场，其运动矢量贴图呈现明显的放射状渐变图案（从中心红绿=0.5,0.5向外辐射至不同颜色）。结合法线贴图的折射扭曲效果，运动矢量确保涟漪扩散的法线在帧间平滑过渡，不会产生法线跳变导致的高光闪烁。

**爆炸冲击波**：冲击波序列帧的运动矢量幅度通常在0.03~0.08之间（RG值约为147~168），相比烟雾特效（0.005~0.02）大得多。若运动矢量幅度设置过小，插值帧的像素偏移量不足以覆盖实际运动，仍会出现重影；过大则会产生UV拉伸（UV Stretching），在冲击波边缘出现明显的纹理撕裂。

## 常见误区

**误区一：运动矢量贴图可以直接用于任意帧率的无损插值**

运动矢量只编码了相邻两帧之间的"一步"位移，当序列帧本身的相邻帧运动幅度过大（如爆炸核心区域每帧位移超过10%纹理尺寸）时，运动矢量引导的插值仍会失效。此类情况下正确做法是提高源素材帧率（如从24fps增至48fps），缩短每步运动的幅度，而不是单纯依赖运动矢量插值解决所有跳帧问题。

**误区二：运动矢量的RG值为(0,0)表示无运动**

这是一个极具迷惑性的错误。对于采用RG8编码的贴图，(0,0)代表最大负方向偏移（-1,-1），而**无运动**对应的正确值是(127,127)或归一化后的(0.498, 0.498)。在Photoshop或Substance中制作或检查运动矢量贴图时，"灰色"区域（RGB约127,127,x）才是静止区域，这与法线贴图中Z通道的约定类似，但初学者经常将其与全黑混淆导致特效整体产生漂移。

**误区三：运动矢量可以替代帧间混合（Alpha混合）直接使用**

单独使用运动矢量偏移而不进行帧间混合，在子帧位置只能看到偏移后的单帧图像，在帧切换点会出现瞬间跳变。运动矢量必须与帧间混合（`mix(colorA, colorB, t)`）配合才能发挥作用，两者是互补关系：帧间混合负责透明度过渡，运动矢量负责确保两帧采样的像素来自"空间上正确"的位置。

## 知识关联

**与帧间混合的关系**：帧间混合是运动矢量插值的必要前提——若没有帧间混合提供的子帧时间系数 `t`，运动矢量无从确定当前子帧的偏移权重。学习帧间混合时建立的"双采样+线性插值"心智模型，在运动矢量中被直接扩展为"双采样+运动引导偏移+线性插值"。

**与序列帧捕获的关系**：序列帧捕获阶段决定了运动矢量能否被正确生成。在Houdini中捕获序列帧时，需要同步输出速度场AOV（通常命名为`Cd`或`v`通道），否则后期只能依赖精度较低的光流算法反推运动矢量。捕获参数中的快门角度（Shutter Angle）也会影响运动矢量的准确性。

**与折射扭曲的关系**：运动矢量掌握后，折射扭曲特效在着色器层面使用了几乎相同的UV偏移机制——区别在于折射扭曲的UV偏移量来自法线贴图的XY分量而非运动矢量贴图，且不涉及时间插值。两者都属于"用贴图驱动UV采样坐标偏移"的技术范畴，共享相同的数学结构。
