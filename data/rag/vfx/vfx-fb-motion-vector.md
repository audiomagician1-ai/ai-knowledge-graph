# 运动矢量

## 概述

运动矢量（Motion Vector）是一种二维向量场贴图，用于描述图像帧中每个像素从当前帧到下一帧（或上一帧）的位移方向与大小。在序列帧特效系统中，运动矢量贴图以 RGB 或 RG 双通道的形式存储这种位移信息：红色通道存储水平方向（X轴）位移，绿色通道存储垂直方向（Y轴）位移，每个像素值通常以归一化的浮点偏移量表示，中间值（0.5, 0.5）代表零位移。

运动矢量的概念起源于视频压缩领域，最早在 MPEG-1 标准（1993年）中被标准化为宏块运动补偿技术，用于减少相邻帧之间的冗余数据。游戏与实时渲染领域在 2010 年代初期借鉴了这一思路，将其引入序列帧特效中，用来解决低帧率 Flipbook 动画在帧间切换时产生的"跳帧"硬切现象。Houdini、EmberGen 等离线模拟软件在导出 Flipbook 序列时，可同步输出运动矢量贴图，供实时引擎采样使用。

运动矢量之所以在特效制作中不可或缺，根本原因在于 Flipbook 动画帧率通常仅有 8~24 fps，远低于游戏渲染帧率（60~120 fps）。如果不做帧间插值，每秒将出现明显的离散跳变。运动矢量贴图提供了像素级别的运动信息，使着色器能够在两帧之间构造出平滑的中间态，而无需存储额外的中间帧，从而在内存与质量之间取得最优平衡。

## 核心原理

### 矢量编码格式与解码公式

运动矢量贴图通常使用 8-bit 或 16-bit 无符号整型 PNG/EXR 存储。由于位移方向包含负值（像素向左或向上移动），8-bit 格式需要将值域 [-1, 1] 映射到 [0, 255]，具体编码公式为：

```
存储值 = (位移量 × 0.5 + 0.5) × 255
```

在着色器中解码时，对应公式为：

```
位移量 = texture.rg × 2.0 - 1.0
```

其中 `texture.rg` 为采样到的归一化 [0,1] 值。16-bit EXR 格式则可直接存储有符号浮点值，精度更高，适用于位移幅度较大的爆炸、流体特效。使用 8-bit 格式时，精度限制在 1/255 ≈ 0.0039，在高分辨率（2048×2048）贴图下可能产生细微的走样条纹，此时应改用 16-bit 半浮点格式。

### 帧间运动补偿的采样机制

运动矢量驱动帧间补偿的核心是**对两帧分别进行扭曲采样后加权混合**。设当前帧序号为 `A`，下一帧为 `B`，当前帧间混合权重为 `t`（范围 0~1），则：

1. 以当前 UV 坐标采样运动矢量贴图，得到位移向量 `MV`；
2. 对帧 A 的纹理，偏移量为 `UV + MV × t`；
3. 对帧 B 的纹理，偏移量为 `UV - MV × (1.0 - t)`；
4. 最终颜色 = `lerp(sampleA, sampleB, t)`。

这种双向扭曲方案（Bidirectional Warping）确保了在 t=0 时完全显示帧 A 原始内容，t=1 时完全显示帧 B 原始内容，中间时刻则对两帧做带运动补偿的插值，视觉上消除了硬切跳变。

### 运动矢量的生成方式

**光流法（Optical Flow）** 是离线生成运动矢量的主流算法，通过分析相邻帧亮度梯度求解像素位移场，Houdini 的 `VEX` 节点或 After Effects 的光流插件均采用此原理。实时渲染管线中，TAA（时域抗锯齿）所需的几何运动矢量（Geometry Motion Vector）由 GPU 在 G-Buffer 阶段逐像素计算顶点的屏幕空间位移，与序列帧运动矢量在生成方式上有本质不同——后者是离线预计算并烘焙到贴图的静态数据。

## 实际应用

**烟雾与火焰 Flipbook**：在 UE5 的 Niagara 系统中，导入由 Embergen 生成的 8帧/秒 烟雾序列时，配合同步导出的运动矢量贴图，可在 120fps 渲染下将视觉帧率提升至接近原始模拟的流畅度。材质蓝图中通过 `FlipbookMotionVectors` 材质函数直接接入 RG 通道数据，参数 `MotionVectorScale` 用于缩放位移强度，典型值设置在 0.1~0.3 之间，超出此范围会导致采样出 UV 边界外的黑色区域。

**水面涟漪循环**：循环型序列帧（Loop Flipbook）要求首尾帧运动矢量无缝衔接。制作时需在 Houdini 中将模拟首尾帧融合，并确保最后一帧的运动矢量方向与第一帧对应，避免循环点出现闪烁。这一场景下，运动矢量的正确性直接决定了循环边界是否可见。

**子弹时间（Bullet Time）特效**：将序列帧播放速度动态降至接近 0 时，`t` 值长时间停留在 0~1 区间内，运动矢量的扭曲幅度被持续放大，暴露出光流法在遮挡边界处的估算误差，表现为边缘拉伸鬼影（Ghosting）。常见修复方案是在运动矢量幅度超过阈值（如 0.5）的区域 mask 掉补偿，退回到简单的帧混合。

## 常见误区

**误区一：将运动矢量直接用于折射扰动**。运动矢量贴图的 RG 通道编码的是帧间时序位移，其方向和幅度随帧号不断变化。若将其常量地接入折射偏移节点，会导致画面静止时仍出现持续单向拉伸，因为运动矢量描述的是"从A帧到B帧的路径"而非"该纹素在空间中的永久方向"。折射扭曲应使用独立的法线或噪声贴图驱动。

**误区二：认为运动矢量只需单通道（灰度图）**。灰度图只能表达单轴位移大小，无法区分水平与垂直方向，也无法表达斜向运动。流体和烟雾的运动轨迹绝大多数包含二维甚至三维分量，单通道运动矢量只适用于纯竖向流动（如简单瀑布）等极特殊情况，强行降维会造成横向运动估算全部失效。

**误区三：运动矢量贴图分辨率越高越好**。运动矢量描述的是宏观位移趋势，本质上是低频信息。实践中将运动矢量贴图分辨率设为 Flipbook 原始分辨率的 1/2 乃至 1/4，几乎不会损失补偿质量，却能将内存占用减少 75%。Unity HDRP 官方示例资产中的烟雾 Flipbook 即采用 256×256 运动矢量配合 512×512 颜色帧的配置。

## 知识关联

运动矢量建立在**帧间混合**的基础之上：帧间混合提供了 `t` 权重参数和两帧颜色的线性插值框架，而运动矢量则将这种插值从"像素位置固定"升级为"沿真实运动路径插值"，解决了帧间混合在高速运动区域产生的重影模糊问题。没有帧间混合作为载体，运动矢量的扭曲采样结果将无处合并。

掌握运动矢量后，**序列帧捕获**是下一个自然延伸——要生成高质量的运动矢量，必须理解在离线模拟软件中如何正确设置光流算法参数、帧率与快门角度，以及如何验证导出贴图的编码正确性。**折射扭曲**则在采样机制上与运动矢量共享 UV 偏移的数学形式（`finalUV = baseUV + offset`），但其偏移量来自法线贴图的切线空间分量，与运动矢量的时序帧间含义截然不同，对比学习这两者能有效防止在材质节点连接中产生通道混用错误。
