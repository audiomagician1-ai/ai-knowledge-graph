# 色散

## 概述

色散（Chromatic Aberration）是一种模拟摄影镜头光学缺陷的后处理特效，其原理来自真实光学现象：当白光通过玻璃镜片时，不同波长的光（红、绿、蓝）折射率不同，导致它们无法聚焦在同一点上，从而在图像边缘产生彩色条纹或色彩偏移的视觉现象。在后处理特效中，色散通过将RGB三个颜色通道分别进行微小偏移来模拟这一缺陷。

这一光学现象由艾萨克·牛顿在1666年通过棱镜实验首次系统描述。镜头制造商长期将消除色差作为镜头设计目标，发展出"消色差镜头"（Achromatic Lens）和"复消色差镜头"（Apochromatic Lens）两类校正方案。然而在影视与游戏领域，色散反而被刻意引入后处理流程，用来增加画面的真实感、电影感和镜头质感，传递"这是通过真实相机拍摄"的视觉信号。

色散在游戏后处理管线中的难度较低（难度2/9），通常在暗角效果之后执行，是构建"摄影机感"（Camera Feel）视觉风格的重要组成部分，尤其在恐怖游戏、赛博朋克风格作品和电影化叙事游戏中被大量使用。

## 核心原理

### RGB通道分离偏移

色散效果最基础的实现方式是将图像的红（R）、绿（G）、蓝（B）三个颜色通道在UV坐标空间内分别朝不同方向偏移采样。标准做法是保持绿色通道G不动，将红色通道R向图像中心外侧偏移一定量，蓝色通道B向内侧偏移等量，最终重新合并为RGB输出。偏移量通常以像素或UV比例表示，典型值在0.002到0.008 UV单位之间即可产生可见但不夸张的效果。

偏移方向的计算公式如下：

```
offset = (uv - 0.5) * chromaticStrength
R_sample = texture(image, uv + offset * redScale)
G_sample = texture(image, uv)
B_sample = texture(image, uv - offset * blueScale)
result = vec3(R_sample.r, G_sample.g, B_sample.b)
```

其中 `uv - 0.5` 将采样坐标转换为以图像中心为原点的向量，`chromaticStrength` 控制整体偏移强度，`redScale` 与 `blueScale` 分别控制红蓝通道的独立偏移倍率。

### 径向分布特性

真实镜头色差在图像中心几乎不可见，越靠近画面边角越明显——这是因为边缘入射光线与镜头光轴的夹角越大，折射差异越显著。因此，正确的色散实现必须引入**径向强度衰减**：偏移量应与当前像素到图像中心的距离成正比，甚至与距离的平方成正比（quadratic falloff）。若忽略这一特性，将偏移量均匀应用于全图，则画面中央也会出现色散，与真实镜头表现完全相悖。

径向距离通常计算为 `r = length(uv - 0.5)`，最大值约为0.707（画面角落到中心距离），可据此将强度归一化。

### 轴向色差与横向色差

色散在光学上分为两种类型，均可在后处理中模拟。**横向色差（Lateral CA）** 表现为颜色通道沿垂直于光轴方向的错位，是最常见的后处理模拟对象，实现即为上述UV偏移方式。**轴向色差（Longitudinal CA）** 则表现为焦点前后不同颜色通道的模糊程度差异，通常通过对不同通道施加不同半径的高斯模糊来近似——例如红色通道模糊半径1.5px，绿色通道1.0px，蓝色通道0.5px。轴向色差的计算成本更高，多见于追求精度的影视级渲染器中。

## 实际应用

**游戏引擎中的实现**：Unity的Post-Processing Stack v2和v3均内置了Chromatic Aberration模块，参数 `intensity` 范围为0到1，对应偏移强度从0到约10像素（基于1080p分辨率）。Unreal Engine的后处理体积中提供 `Chromatic Aberration` 滑块，推荐值通常不超过0.3以保持可读性。

**风格化应用**：《赛博朋克2077》在过场对话和界面受到黑客入侵时，动态提升色散强度至夸张程度（偏移量可达20px以上），配合画面撕裂效果，传递数字故障（Glitch）美学。《异形：隔离》则使用轻微、持续的色散增强CRT监视器画面的复古感。

**动态触发**：色散常与游戏事件绑定，在角色受到冲击、镜头晃动或快速运动瞬间临时增强强度，随后以0.1至0.3秒的时间快速衰减至基础值，产生"镜头受到冲击"的物理反馈感。

## 常见误区

**误区一：全图均匀施加偏移**
许多初学者将相同的偏移量均匀应用于图像每一个像素，导致画面中央区域也出现彩色边缘。真实镜头色差遵循径向分布规律，中央区域几乎为零，边缘最强。正确实现必须将偏移量乘以像素到中心的距离权重。

**误区二：三个通道等量反向偏移**
将红通道和蓝通道以完全对称的方式等量偏移会产生过于规律的"彩虹条纹"感，缺乏真实感。真实色差中各通道的偏移量略有不等，且偏移方向并非严格关于绿通道对称。适当引入红蓝通道独立的偏移倍率系数（如红色1.0×，蓝色0.8×）可使效果更自然。

**误区三：混淆色散与色调分离**
色散（Chromatic Aberration）是空间位置上的颜色通道错位，发生在UV坐标层面；色调分离（Color Grading/Split Toning）是对亮度区间施加不同色相偏转，发生在颜色值层面。二者目视结果有时相似（边缘出现偏色），但原理、实现方式和适用场景完全不同，不可相互替代。

## 知识关联

色散在后处理管线中紧接在**暗角效果**之后执行，二者均属于"镜头缺陷模拟"（Lens Artifact Simulation）特效类别。暗角处理的是图像四角亮度衰减，解决的是采样UV到图像中心距离的亮度映射问题；色散解决的是同一径向距离下颜色通道的空间分离问题——两种效果共享"以图像中心为参考点的径向计算"这一核心数学模式，在代码实现上可以复用相同的径向距离计算逻辑。

后续学习的**胶片颗粒**效果进一步叠加噪声纹理来模拟胶片感光颗粒，三种效果（暗角→色散→胶片颗粒）共同构成经典"模拟胶片相机"后处理组合。胶片颗粒通常在色散之后叠加，因为噪声需要施加在最终合并的RGB图像上，而不是在通道分离状态下分别添加。掌握色散的通道分离思路，有助于理解后续更复杂的多通道后处理技术，例如光晕（Lens Flare）的波长依赖性散射模拟。
