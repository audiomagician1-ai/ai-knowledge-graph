# 色散（色差效果）

## 概述

色散（Chromatic Aberration），又称色差效果，是一种模拟真实镜头光学缺陷的后处理特效技术。在物理世界中，透镜无法将不同波长的光线聚焦到同一点，导致红、绿、蓝三个颜色通道在成像平面上产生轻微位移，形成彩色边缘或色晕。这种光学现象在廉价镜头、广角镜头或大光圈拍摄时尤为明显，因此在游戏和影视后处理中模拟它能强化画面的真实感与镜头感。

色差效果最早作为一种"缺陷美学"被引入数字媒体创作。2010年代初，随着逼真渲染（Physically Based Rendering）在游戏引擎中普及，开发者开始系统性地将镜头缺陷模拟纳入后处理管线，色散与暗角、胶片颗粒共同构成了"镜头真实感三件套"。《使命召唤：现代战争》系列和虚幻引擎4的后处理模块是将色散效果标准化的早期代表案例。

在创作层面，色散效果并非单纯的技术噱头。适度的色散能暗示画面处于压力、失焦或受损状态，常用于爆炸瞬间、受击反馈、梦境过渡等叙事节点，帮助玩家感知特定的情绪信号。

## 核心原理

### 三通道分离位移

色散效果的最基本实现方式是对屏幕纹理的红（R）、绿（G）、蓝（B）三个颜色通道分别进行UV偏移采样。核心公式如下：

```
R_channel = tex2D(screenTex, uv + offset * strength)
G_channel = tex2D(screenTex, uv)
B_channel = tex2D(screenTex, uv - offset * strength)
```

其中 `offset` 是偏移方向向量，通常从屏幕中心向外辐射；`strength` 控制色散强度，通常取值在 0.002 至 0.01（归一化UV空间）之间。绿色通道保持不动是因为人眼对绿色最敏感，以绿色为基准可最大程度保留画面清晰感。

### 径向色差与切向色差

真实镜头存在两种主要色差类型：**纵向色差（Longitudinal CA）**表现为整体画面不同颜色聚焦距离不同，产生全局色晕；**横向色差（Lateral/Transverse CA）**表现为色散强度随距图像中心距离增加而增大，边缘区域最为明显。

在实时渲染中，横向色差更常被模拟，其偏移量计算依赖于像素到屏幕中心的距离 `r`：

```
distortion = r^2 * aberrationScale
```

当 `aberrationScale` 设为 1.5 至 3.0 之间时，画面边缘会出现明显色晕而中心保持清晰，贴合真实广角镜头的视觉特征。

### 采样数量与性能取舍

简单的3次纹理采样（每通道一次）即可实现基础色散效果，性能开销极低。然而高质量实现会对每个通道进行6至8次线性插值采样，沿色散方向进行模糊，使色散边缘更加柔和自然而非生硬的位移条纹。在移动平台上，通常退回到3次采样并限制 `strength` 最大值以控制带宽消耗。虚幻引擎5的后处理色散实现默认使用4次采样并支持通过LOD偏置动态缩减至2次。

## 实际应用

**游戏中的受击反馈**：在FPS游戏中，玩家受到伤害的瞬间，色散强度从0在约0.1秒内急速升高至峰值（strength ≈ 0.008），再在0.3至0.5秒内缓慢衰减归零，配合屏幕边缘红晕，形成强烈的生理不适感模拟。《死亡空间》系列将此技法运用得极为成熟。

**过渡与闪回效果**：影视后期和游戏过场中，色散常与镜头扭曲、运动模糊组合使用，标志画面进入回忆或幻觉状态。此时三通道偏移不仅做径向扩散，还会加入旋转分量，制造出迷幻的旋涡色晕。

**赛博朋克视觉风格**：《赛博朋克2077》大量使用持续性轻度色散（strength ≈ 0.003），配合扫描线和噪点，模拟劣质全息显示屏的成像风格，成为该游戏视觉语言的组成标识之一。

**摄影风格化滤镜**：在Photoshop和DaVinci Resolve中，色散通过"通道分离"功能手动实现，将红色通道向右移动2至4像素、蓝色通道向左移动相同距离，是洛菲（Lo-fi）和胶片模拟风格的标准处理步骤之一。

## 常见误区

**误区一：色散强度越大越真实**

许多初学者倾向于将 `strength` 设置过高（>0.015），认为更夸张的色差能增强镜头感。实际上，真实摄影中明显可见的色差通常意味着低质量镜头或严重的对焦失误，过强的色散反而会破坏画面可读性，使角色和UI文字出现彩色重影。专业电影中有意呈现的色散通常控制在肉眼勉强可辨的范围内（横向位移约为画面宽度的0.2%至0.5%）。

**误区二：色散效果应始终开启**

部分开发者将色散作为全局持续特效叠加在所有场景中。然而色散的叙事意义来自其"突发性或局部性"——如果玩家始终处于色散状态，该效果便失去了传递信号的能力。正确做法是将色散与游戏状态绑定，仅在特定触发条件（受伤、过载、场景切换）下激活，并确保日常游玩时 `strength` 归零或维持在视觉感知阈值以下（<0.001）。

**误区三：色散与镜头扭曲是同一种效果**

色散（Chromatic Aberration）仅对颜色通道做偏移，不改变画面几何结构；镜头扭曲（Lens Distortion）则对整个画面UV做桶形或枕形变形，两者成因和视觉结果均不同。在实现层面，色散作用于颜色采样阶段，镜头扭曲作用于UV变换阶段，正确的处理顺序是先应用镜头扭曲，再在扭曲后的结果上叠加色散采样。

## 知识关联

**与暗角效果的关系**：暗角和色散都属于镜头缺陷模拟类后处理效果，通常在同一后处理Pass中先后执行。暗角通过距中心距离计算亮度衰减，色散通过同样的距中心距离驱动通道偏移量，两者共享同一个"屏幕中心到像素距离"的计算结果，在Shader中可复用该值避免重复计算，降低约15%的指令数。

**与胶片颗粒的衔接**：色散处理完成后，其输出结果会作为胶片颗粒效果的输入。胶片颗粒在色散已分离的颜色通道上叠加随机噪点，使不同通道的噪点分布产生轻微差异，进一步强化模拟胶片的有机感。如果顺序颠倒——先加颗粒再做色散——颗粒本身也会产生色差，造成视觉上的过度混乱，通常不是预期效果。
