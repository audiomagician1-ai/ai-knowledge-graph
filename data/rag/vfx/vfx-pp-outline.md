# 描边效果

## 概述

描边效果（Outline Effect）是后处理管线中一种将角色或物体边缘以可见线条勾勒出来的屏幕空间技术。与传统卡通渲染中在模型几何体上额外添加法线外扩面不同，后处理描边在屏幕空间对深度缓冲（Depth Buffer）和法线缓冲（Normal Buffer）进行像素级采样，通过对比相邻像素的差异来识别并绘制边缘线条，无需修改原始网格数据。

描边效果的后处理实现方式最早在2000年代的日系游戏风格渲染研究中得到广泛讨论，随着延迟渲染（Deferred Rendering）管线的普及，G-Buffer中现成的深度与法线数据使得屏幕空间描边变得极为高效。《塞尔达传说：风之杖》（2002）使用的是几何法，而现代游戏和引擎（如Unity URP/HDRP后处理栈）更倾向于后处理的Sobel或Roberts Cross算子方案，因为它对任意复杂度的模型都统一有效。

后处理描边的核心价值在于它能以极低的几何体开销为场景增添手绘感或卡通感，并且描边颜色、宽度、阈值均可在运行时动态调整。对于有大量动态角色的游戏（如多人竞技场景），统一通过后处理一次性处理所有物体的描边，比逐物体几何扩展在Draw Call层面更具优势。

## 核心原理

### 深度缓冲检测

深度缓冲检测是描边效果最常见的基础手段。在渲染完成后，后处理Pass读取深度缓冲中以[0,1]存储的NDC深度值，对当前像素P的上下左右四个相邻像素（偏移量通常为1个纹素宽度）分别采样深度值，计算差分：

```
DepthEdge = |d(P) - d(P+offset_x)| + |d(P) - d(P-offset_x)|
           + |d(P) - d(P+offset_y)| + |d(P) - d(P-offset_y)|
```

当 `DepthEdge` 超过设定阈值（如 0.01）时，该像素被标记为边缘并叠加描边颜色。深度差分法对物体的轮廓外边缘（silhouette edge）响应良好，但对同一物体表面上相邻面片之间的折痕识别能力有限，例如正方体同一平面的内部区域不会产生深度突变。

### 法线缓冲检测

法线缓冲检测弥补了纯深度检测在曲面折痕上的不足。后处理Pass从G-Buffer读取视图空间（View Space）法线，将其编码为RGB颜色（通常将法线从[-1,1]重映射到[0,1]）。对相邻像素做法线向量的点积差值：

```
NormalEdge = dot(n(P), n(P+offset)) 
```

当两相邻像素法线夹角超过约15°~30°时，点积结果低于阈值（如0.9），判定为折痕边缘。组合使用深度与法线检测，公式常写为：

```
Edge = step(depthThreshold, DepthEdge) + step(normalThreshold, NormalEdge)
```

最终只要深度或法线任一检测触发，就输出边缘，使轮廓线和折痕线都能被捕捉到。

### Sobel算子边缘检测

Sobel算子是描边效果中精度最高的经典图像处理方案。它对目标缓冲（深度或法线通道）应用两个3×3卷积核，分别检测水平和垂直方向的梯度：

```
Gx = [[-1, 0, +1],       Gy = [[-1, -2, -1],
      [-2, 0, +2],              [ 0,  0,  0],
      [-1, 0, +1]]              [+1, +2, +1]]

EdgeStrength = sqrt(Gx² + Gy²)
```

Sobel算子需要对每个像素采样周围3×3共9个纹素，相比简单差分的4次采样增加了开销，但输出边缘方向更连续、噪点更少。在描边宽度为2像素以上时，Sobel方案的视觉质量明显优于简单差分。为降低开销，也可用Roberts Cross算子（仅2×2采样）作为折中方案。

### 描边宽度与距离衰减

描边宽度由采样偏移量控制，偏移量 = `outlineWidth / screenResolution`。若希望描边在近处宽、远处窄以模仿手绘透视感，可将深度值线性映射到偏移量上，实现基于距离的描边粗细衰减，公式为：

```
adjustedOffset = baseOffset * (1.0 - linearDepth * depthFadeScale)
```

## 实际应用

**卡通风格游戏渲染**：在Unity URP中配置Renderer Feature，添加自定义描边Pass，采样`_CameraDepthTexture`与`_CameraNormalsTexture`，使用Sobel算子输出边缘遮罩，再以黑色或特定颜色合成到最终画面，能以单一全屏Pass为场景所有物体生成统一描边。

**角色高亮选中效果**：在角色渲染时将其Stencil Buffer标记为特定值（如值1），描边后处理Pass只对Stencil值为1的像素执行深度法线检测，从而仅对选中角色绘制彩色描边（如黄色）而不影响场景背景。

**UI与3D混合场景**：在移动端游戏的3D棋盘或模型展示界面，后处理描边可让模型具备类似插图的视觉风格，无需美术为每个模型制作双Pass轮廓Shader，将各类模型描边维护成本大幅降低。

## 常见误区

**误区一：描边只需检测深度就够了**。许多初学者以为深度缓冲差分能处理所有边缘情况。实际上，一个立方体的棱角会被深度法检测到，但一个弯折超过30°的平滑曲面（如角色手臂弯折处）深度变化微乎其微，只有加入法线缓冲检测才能显示折痕描边。单独使用深度检测会导致复杂角色的内部折痕完全消失。

**误区二：Sobel算子描边性能开销可以忽略不计**。Sobel需要9次纹素采样，在4K分辨率下每帧全屏执行意味着数亿次纹理访问。未做分辨率缩放（如先在1/2分辨率Render Target上执行Sobel，再上采样）的实现会对中低端GPU产生显著帧率压力，不应在移动端不加优化地直接使用原始Sobel描边。

**误区三：后处理描边与几何扩展法等效**。几何扩展法（Inverted Hull）能精确控制单个模型的描边厚度并在前向渲染管线无需G-Buffer，在不透明物体上效果稳定；后处理描边则在半透明物体边缘会因无深度写入而产生边缘缺失，两者各有适用场景，不能互相替代。

## 知识关联

**前置概念：体积雾**。体积雾同样依赖深度缓冲在屏幕空间重建像素世界坐标，理解深度缓冲从NDC值线性化为视图空间深度的过程，是正确设定描边深度阈值的基础。体积雾中学到的深度采样与重建方法可直接复用于描边深度检测Pass。

**后续概念：画面扭曲（Image Distortion）**。画面扭曲同属全屏后处理特效，使用UV偏移对颜色缓冲进行采样变形。掌握描边效果的后处理Pass结构——包括如何读取多个缓冲、如何将结果叠加到颜色缓冲——为理解画面扭曲Pass的扭曲向量图采样与颜色缓冲重采样流程提供了直接的结构参照。在同一后处理栈中，描边Pass通常排在扭曲Pass之前，以避免扭曲破坏边缘线条的连续性。
