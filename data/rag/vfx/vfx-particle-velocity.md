# 速度与加速度

## 概述

在粒子特效系统中，速度（velocity）与加速度（acceleration）是控制每个粒子运动轨迹的两个基本物理量。速度决定粒子在某一瞬间的移动方向和快慢，以三维向量 **v** = (vx, vy, vz) 表示，单位通常为"像素/帧"或"单位/秒"。加速度则描述速度随时间的变化率，同样是三维向量 **a** = (ax, ay, az)，单位为"单位/秒²"。粒子系统每帧更新时，会按照 **v(t+Δt) = v(t) + a·Δt**、**p(t+Δt) = p(t) + v(t)·Δt** 的递推公式逐步计算每个粒子的新位置。

这两个概念在20世纪90年代的粒子系统软件（如ILM开发的早期特效工具及后来的Houdini）中就已成为标准参数。发展至今，Unity的Shuriken粒子系统和Unreal Engine的Niagara系统均将初速度范围、加速度场列为最基础的粒子属性模块。对特效师而言，掌握速度与加速度的配置，意味着能够不依赖关键帧动画，仅靠物理参数就驱动出爆炸、火焰、烟雾等千变万化的运动形态。

## 核心原理

### 粒子初速度分布

粒子从发射器诞生时被赋予的速度称为**初速度（Initial Velocity）**。特效系统很少给所有粒子同一个固定值，而是使用**随机范围**或**概率分布**来产生自然感。常见做法是设定一个最小值与最大值区间——例如在Unity Shuriken中将 Speed 设为"Random Between Two Constants"，配置为 [2.0, 8.0]，则每个粒子出生时会在此区间内均匀随机取一个标量速度，再乘以发射方向向量得到初速度向量。

初速度的**方向分量**同样受发射锥角控制：锥角为0°时所有粒子沿同一方向射出，锥角为90°时粒子散布在半球面内。具体地，初速度方向可用极坐标表示为 **v₀ = v·(sinθ·cosφ, sinθ·sinφ, cosθ)**，其中θ在 [0, α/2]（α为锥角）内随机，φ在 [0, 2π) 内随机。

### 加速度场

加速度场（Acceleration Field）是作用在粒子上的持续力的抽象，分为**全局加速度**与**局部加速度场**两类。

**全局加速度**对所有粒子施加相同的常量加速度向量，最典型的应用是重力：通常设置为 (0, −9.8, 0) m/s²（或按场景比例缩放）。在Houdini的POP Gravity节点中，这个值默认为 [0, −9.80665, 0]，对应标准重力加速度。

**局部加速度场**则是空间位置相关的加速度，常见类型包括：
- **湍流场（Turbulence/Noise Field）**：基于柏林噪声（Perlin Noise）生成空间变化的加速度，让烟雾粒子产生不规则飘动；
- **漩涡场（Vortex Field）**：产生围绕某轴的切向加速度，驱动粒子做螺旋运动；
- **吸引/排斥场（Attractor/Repulsor）**：加速度方向指向或背离某一点，强度通常与距离平方成反比，公式为 **a = k·(p_target − p_particle) / |p_target − p_particle|³**。

### 速度衰减（Drag / Damping）

真实空气阻力会使粒子速度随时间降低，这在特效系统中通过**速度衰减系数（Drag Coefficient / Damping）**实现。最常用的线性衰减模型是每帧将速度乘以一个衰减因子 d（0 < d < 1）：

> **v(t+Δt) = (v(t) + a·Δt) × d**

例如 d = 0.98 时，粒子速度每帧降至原来的98%；经过60帧（约1秒）后，速度衰减至初始值的 0.98⁶⁰ ≈ 0.298，即约30%。这一机制使烟雾、羽毛类粒子表现出飘散后逐渐静止的效果。若使用二次阻力模型（模拟流体阻力），衰减力与速度平方成正比：**F_drag = −c·|v|·v**，适合模拟水中气泡或高速运动的碎片。

## 实际应用

**爆炸特效**：爆炸粒子通常配置较大的初速度范围（如 [5.0, 30.0] 单位/秒），方向分布覆盖完整球面（锥角360°），同时叠加向下的重力加速度（0, −9.8, 0）和较强的速度衰减（d ≈ 0.92），让碎片飞出后迅速减速落地，形成爆炸由快到慢的视觉节奏。

**火焰特效**：火焰粒子初速度较小且方向朝上（vz ∈ [0.5, 2.0]），叠加弱湍流加速度场使火舌左右摇摆，同时速度衰减设置偏高（d ≈ 0.995），让粒子在上升过程中缓慢减速，模拟热气流减弱的物理过程。

**粒子轨迹调试**：当特效出现粒子"穿地"或"飞出场景"等问题时，第一步应检查加速度量级是否与场景单位匹配——若场景单位为厘米而重力设置为 −9.8（以米为单位），实际效果仅为正确重力的1/100，粒子会表现为几乎漂浮不落。

## 常见误区

**误区一：混淆速度方向与发射方向**。发射方向由发射模型（Emitter Shape）决定，初速度大小可以独立设置，但最终粒子的速度向量 = 速度标量 × 发射方向单位向量。部分初学者误以为加大速度值可以改变粒子的运动方向，实际上方向由发射器的朝向决定，速度值只影响快慢。

**误区二：加速度设置后忘记考虑生命周期与帧率的关系**。加速度是"每秒²"的变化量，若粒子生命周期只有0.5秒，设置 −9.8 m/s² 的重力影响很微弱；但若粒子存活5秒，最终下坠速度可达 9.8×5 = 49 m/s，粒子会急速坠落出画面。加速度参数必须结合粒子生命周期共同评估。

**误区三：速度衰减等同于粒子消失**。衰减（Damping）只降低速度，粒子仍然存在并继续运动，直到其生命周期结束或透明度归零才消失。将衰减系数设为0会使粒子完全静止悬浮，而非消失，这是两套完全独立的参数体系。

## 知识关联

学习速度与加速度需要先理解**发射模型**——发射器的形状和朝向决定了初速度的方向基准，没有发射模型的概念，初速度向量的方向分量无从界定。发射模型提供"粒子从哪里以什么方向诞生"，而速度与加速度则接管"粒子诞生后如何运动"。

掌握速度与加速度之后，下一步将深入**重力模拟**。重力是加速度场中最基础的一类，但真实的重力模拟还涉及地面碰撞响应（弹性系数、摩擦力）、粒子质量差异对加速度的影响（轻重粒子在同一重力场下加速相同，但在空气阻力场中因质量不同而产生不同轨迹），这些内容都建立在对速度与加速度向量运算的熟练理解之上。
