# 颜色演变

## 概述

颜色演变（Color Evolution）是粒子系统中控制单个粒子在其生命周期内色彩变化的核心机制。具体而言，每个粒子从被发射的瞬间（生命值 = 0%）到消亡时刻（生命值 = 100%），其 RGBA 四通道值可以按照设计者预设的曲线或梯度表连续插值变化。与静态着色不同，颜色演变赋予了每颗粒子"生老病死"的视觉表现力。

这一概念最早在 1980 年代的离线渲染软件中以"Color Over Life"节点形式出现，后随 Unreal Engine 和 Unity 的普及，成为实时粒子系统的标准功能。Unreal Engine 的 Cascade 和 Niagara 系统分别将其实现为 `ColorOverLife` 模块，Unity 的 Shuriken 粒子系统则提供了 `Color over Lifetime` 渐变条（Gradient Strip）编辑器。

颜色演变的重要性在于它以极低的性能开销（仅需每帧对浮点数做线性插值）实现了视觉上极为丰富的效果。火焰从白芯到橙体再到暗红尾迹的颜色变化、烟雾从黑色到透明灰的消散感，以及魔法粒子随速度加快而变蓝的能量感，都依赖颜色演变而非贴图切换来完成。

## 核心原理

### 生命周期驱动的 RGBA 插值

颜色演变最基本的驱动变量是粒子归一化生命值 `t ∈ [0, 1]`，其中 `t = CurrentAge / Lifetime`。颜色由一条渐变曲线（Gradient）定义，渐变曲线上至少包含两个关键帧（Keyframe），每个关键帧存储一个 RGBA 颜色值和该帧在 `[0,1]` 轴上的位置。插值公式为：

```
C(t) = lerp(C_n, C_{n+1}, (t - t_n) / (t_{n+1} - t_n))
```

其中 `C_n` 与 `C_{n+1}` 是相邻关键帧颜色，`t_n` 与 `t_{n+1}` 是其对应生命值位置。Alpha 通道（透明度）同样受此公式控制，这意味着粒子淡入淡出效果本质上是颜色演变的 A 通道子集。典型火焰设置为：`t=0` 时白色（1,1,1,1），`t=0.3` 时橙色（1,0.5,0.1,1），`t=0.8` 时深红（0.4,0.05,0,0.8），`t=1` 时全透明（0,0,0,0）。

### 速度驱动的颜色映射

除生命周期外，粒子的瞬时速度大小（Speed）也可作为颜色演变的驱动变量。在 Niagara 系统中，这通过将速度值归一化到 `[MinSpeed, MaxSpeed]` 区间后输入同一渐变曲线实现。例如，将 `MinSpeed = 0 cm/s` 映射为冷色蓝（0.2, 0.4, 1.0），将 `MaxSpeed = 500 cm/s` 映射为热色白（1, 1, 0.9），可直观表现粒子的动能状态。这种用法在模拟等离子体、闪电残影或喷射流时尤为常见。速度驱动与生命周期驱动可同时叠加，最终颜色 = 两个渐变曲线输出值相乘（Multiply 混合模式）。

### 初始颜色随机化与演变的叠加

颜色演变并非孤立工作，它通常叠加在粒子的**初始颜色**之上。粒子系统允许在发射时为每颗粒子随机指定一个初始色调偏移（Hue Offset），范围通常为 ±15° 色相旋转。颜色演变曲线的输出会与该初始颜色通过 Modulate（调制）方式相乘，即：

```
FinalColor(t) = InitialColor * GradientColor(t)
```

由于初始颜色为白色（1,1,1,1）时调制结果等于渐变曲线本身，因此白色初始色是调试颜色演变曲线的标准起点。随机初始色的引入可使同一批粒子呈现出既统一又多样的视觉效果，例如落叶粒子中混入黄绿交替的色调。

## 实际应用

**火焰特效**：标准火焰粒子的颜色演变设置中，Alpha 通道在 `t=0.0` 到 `t=0.1` 区间从 0 快速升至 1（淡入防止粒子突然闪现），在 `t=0.7` 到 `t=1.0` 区间从 1 降至 0（烟雾消散）。RGB 通道则模拟普朗克黑体辐射的冷却过程：高温白→中温橙→低温红。

**魔法传送门特效**：粒子速度越快越靠近门的中心，颜色演变速度驱动将高速粒子映射为发光青白色（HDR 值 R=2.0, G=2.0, B=2.5，超过 1.0 以触发泛光效果），慢速粒子保持深紫色，形成动态能量汇聚感。

**爆炸烟雾**：爆炸粒子的颜色演变曲线在 `t=0` 到 `t=0.05` 设置橙色爆闪，`t=0.05` 至 `t=0.3` 快速切换至黑色浓烟，`t=0.3` 之后 Alpha 通道线性衰减至 0，模拟烟雾扩散后的能见度降低。

## 常见误区

**误区一：认为颜色演变只控制 RGB，忽略 Alpha 通道的演变作用**。许多初学者单独使用"Alpha over Lifetime"模块控制透明度，同时又在颜色演变曲线中将 Alpha 固定为 1，导致两者冲突或行为不一致。正确做法是将淡入淡出的 Alpha 曲线统一写入颜色演变模块的 A 通道，避免多个模块争夺 Alpha 控制权。

**误区二：混淆颜色演变与材质自发光强度控制**。颜色演变曲线的 RGB 值范围默认为 `[0, 1]`，无法直接触发 HDR 泛光；若需要粒子在生命前段发出强烈光晕，必须将粒子材质的 `EmissiveMultiplier` 参数同步通过动态材质实例（Dynamic Material Instance）随生命值更新，或将粒子系统的颜色混合模式设为 `Additive` 并允许 RGB 超过 1.0 的 HDR 输入。

**误区三：在高密度粒子系统中使用过多颜色演变关键帧**。渐变曲线每增加一个关键帧，GPU 在每帧需要额外进行一次区间判断和插值运算。当单屏粒子数超过 10 万时，将关键帧数量从 8 个压缩至 4 个可节省约 3%～5% 的 GPU 粒子计算时间（数据来源：Epic Games 2019 GDC 性能优化报告）。

## 知识关联

**前置概念——尺寸变化**：尺寸变化（Size over Lifetime）与颜色演变共享相同的 `t = CurrentAge / Lifetime` 归一化时间轴和关键帧插值机制。掌握尺寸变化曲线的编辑方式后，颜色演变曲线的操作逻辑完全相同，区别仅在于关键帧存储的数据类型由单个浮点数变为 RGBA 四元组。两者在粒子系统中通常并行工作：典型烟雾粒子在尺寸增大的同时，颜色演变曲线将 Alpha 减小，形成"越扩散越透明"的同步视觉效果。

**后续概念——旋转控制**：旋转控制（Rotation over Lifetime）在驱动变量和关键帧结构上与颜色演变高度一致，但其输出值为角速度（度/秒），且存在符号（正转/反转）。颜色演变中学到的"将效果与生命周期百分比绑定"的设计思维，将直接应用于旋转控制中处理粒子在消亡前加速旋转或停止旋转的时序设计。
