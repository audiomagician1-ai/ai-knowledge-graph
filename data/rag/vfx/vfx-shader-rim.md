# 边缘光（Rim Light Shader）

## 概述

边缘光（Rim Light）是一种基于视角方向与表面法线夹角来计算发光强度的Shader技术，其核心数学依据是：当观察视线与物体表面几乎平行时（法线与视线接近垂直），该表面边缘会显示出明亮的发光效果。这种效果因在现实摄影中常由背后打光产生的轮廓光（Rim Lighting）而得名，在游戏特效中被广泛用于角色受击高光、被选中提示、角色描边等交互反馈场景。

边缘光的概念源于物理光照中的菲涅耳效应（Fresnel Effect）——由法国物理学家奥古斯丁·菲涅耳于19世纪初提出，描述光在介质交界面的反射率随入射角增大而增强的现象。游戏Shader中的边缘光借鉴了这一物理规律，通过Fresnel公式的简化版本实现了轻量、高效的视觉边缘发光，无需额外几何描边Pass，成为角色UI反馈特效的首选方案之一。

在游戏项目中，边缘光解决了两个实际问题：一是以极低的性能开销（单Pass、无额外Draw Call）为角色提供清晰的轮廓提示；二是通过颜色和强度的动画化，传达受击、选中、冰冻、燃烧等不同状态，在2D UI层面无法实现这种与角色形体完全契合的边缘特效时，边缘光Shader是不可替代的方案。

## 核心原理

### 菲涅耳计算：法线与视线的点积

边缘光的强度由以下公式计算：

```
rimIntensity = pow(1.0 - dot(N, V), rimPower)
```

其中：
- `N` = 顶点或片元的世界空间法线（归一化）
- `V` = 从片元指向摄像机的观察向量（归一化）
- `dot(N, V)` 的值域为 `[0, 1]`，值越接近0表示越靠近边缘
- `rimPower` 控制边缘光的宽窄，典型取值范围为 `1.0 ~ 8.0`；`rimPower = 1.0` 时边缘光漫散至整个表面，`rimPower = 5.0` 时呈现细且锐利的边缘线

`pow()` 函数在这里至关重要：通过指数运算将 `[0,1]` 的线性渐变压缩到靠近0的区间，使只有真正的"边缘像素"才有高强度输出，而非整个表面均匀发光。

### 颜色混合与强度控制

计算出 `rimIntensity` 后，将其与设定的边缘光颜色相乘并叠加到原始漫反射颜色之上：

```hlsl
float3 rimColor = _RimColor.rgb * rimIntensity * _RimStrength;
finalColor = baseColor + rimColor;
```

`_RimStrength` 参数通常暴露给策划或美术，取值 `0.5 ~ 3.0` 用于普通选中，受击特效可在极短时间内将其拉升至 `5.0 ~ 10.0` 配合闪白使用。加法混合（Additive）确保边缘光不会遮盖原始贴图细节，这也是与直接替换颜色方案的本质区别。

### 动态边缘光：时间驱动的脉冲与颜色变化

静态边缘光只适合"选中"状态，对于受击、冰冻、中毒等状态，需要对 `rimIntensity` 或 `rimColor` 进行时间函数调制：

```hlsl
// 受击闪光：利用指数衰减模拟瞬时高光后快速消退
float hitFlash = exp(-_HitTimer * 4.0);
rimIntensity *= hitFlash;

// 选中脉冲：利用sin函数产生0~1循环呼吸感
float pulse = (sin(_Time.y * _PulseSpeed) * 0.5 + 0.5);
rimIntensity *= pulse;
```

`_HitTimer` 由CPU端在受击事件触发时重置为0，随帧累加，边缘光会在约0.25秒内衰减至不可见，精确模拟受击打光感。`_PulseSpeed` 典型值为 `2.0~4.0`，对应每秒1~2次呼吸周期。

## 实际应用

**角色受击反馈**：在受击瞬间将 `_RimColor` 设置为纯白或敌方阵营色（如红色 `#FF3030`），`_RimStrength` 跳升至 `8.0` 并配合 `exp(-t*4)` 衰减曲线，整个闪光持续约0.2~0.3秒。玩家在大规模战斗中即便无法听到音效也能通过边缘光闪烁感知伤害来源方向。

**单位选中高光**：策略游戏（如《星际争霸2》类）中，被框选的单位需要持续显示所属阵营颜色的边缘光。将 `rimPower = 3.0`、`_PulseSpeed = 2.5`，配合阵营颜色（蓝方 `#4488FF`，红方 `#FF4444`），即可实现清晰且不喧宾夺主的选中状态提示。

**技能状态标记**：冰冻状态使用浅蓝色 `#AADDFF`、`rimPower = 6.0` 产生细冷边缘；燃烧状态使用橙红渐变色配合噪声贴图对 `rimIntensity` 进行扰动（此处即与噪声生成相连接），使边缘光产生火焰般不规则抖动，比均匀边缘光更具表现力。

**UI面板中的3D角色预览**：在角色立绘或装备预览界面，固定方向的边缘光（取 `V` 为固定向量而非摄像机方向）可模拟摄影棚打光效果，无需额外光源即可让角色模型在黑色背景下轮廓清晰。

## 常见误区

**误区1：在切线空间计算点积**
部分初学者直接使用切线空间法线贴图采样结果与视线进行 `dot()` 计算，导致边缘光随贴图细节抖动而非跟随几何轮廓分布。正确做法是将法线贴图解码后转换回世界空间（`mul(unity_ObjectToWorld, normalTangent)`），再与世界空间视线方向计算点积，边缘光才会稳定地分布在角色几何轮廓上。

**误区2：使用 `saturate(dot(N,V))` 代替 `1.0 - dot(N,V)`**
有些实现忘记将点积结果取反，直接对 `dot(N,V)` 做 `pow()`，会得到正好相反的效果——正对摄像机的表面中心区域最亮，边缘最暗，变成聚光灯效果而非边缘光。`1.0 - dot(N,V)` 这一步取反是边缘光公式最关键也最易遗漏的步骤。

**误区3：认为 `rimPower` 越大效果越好**
`rimPower` 超过 `8.0` 后，边缘光宽度会压缩到不足1~2个像素，在低分辨率模型或移动端低分辨率渲染下几乎不可见，并且边缘会因锯齿产生断断续续的闪烁感。实际项目中移动端通常将 `rimPower` 限制在 `2.0~5.0` 范围内，并在Shader变体中提供高低质量两档预设。

## 知识关联

**前置概念——噪声生成**：在边缘光的进阶实现中，可以对 `rimIntensity` 乘以一张流动噪声贴图的采样值，使边缘光产生非均匀的溶解或燃烧纹理感。噪声贴图（如Perlin Noise或Worley Noise）为边缘光提供了空间上的随机扰动依据，是从静态边缘光进化为动态状态特效的必要步骤。

**后续概念——全息效果**：全息效果（Hologram Shader）是在边缘光基础上增加扫描线纹理、透明度抖动和颜色偏移等多层叠加的复合特效。其中全息边缘的蓝色发光轮廓本质上仍是一个 `rimPower = 2.0~3.0` 的边缘光层，只是与扫描线纹理和Alpha Clip共同构成完整的全息视觉系统。掌握边缘光的法线-视线点积计算和动态强度控制后，全息效果只是在其输出结果上继续叠加层次的延伸工作。
