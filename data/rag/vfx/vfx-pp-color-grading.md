# 颜色分级

## 概述

颜色分级（Color Grading）是指在渲染完成后，对图像的色彩、亮度、对比度和色调进行系统性的调整，以实现特定的视觉风格或情感氛围。与颜色校正（Color Correction）不同，颜色分级不仅仅是修复问题，更是主动塑造画面的"调色"艺术——它可以让一个普通的日间场景呈现出寒冷的蓝青色调（如科幻风格），或者赋予画面温暖的橙棕色（如西部片风格）。

颜色分级起源于胶片电影时代的光化学冲洗工艺，技师通过调整显影液浓度和曝光时间来改变胶片的色彩特性。进入数字时代后，DaVinci Resolve等专业软件将这套工作流移植到了实时渲染领域。在游戏引擎的后处理管线中，颜色分级通常是色调映射之后的最后一道处理步骤，在线性空间完成色调映射后，颜色分级的操作发生在伽马空间（sRGB）中。

颜色分级对于游戏和影视渲染的重要性在于：它允许美术在不修改任何场景资产的情况下，统一整个画面的视觉风格，区分不同关卡或时间段的情感基调。例如，《使命召唤》系列通过将绿色通道压低、蓝色抬高，塑造了其标志性的战场灰绿色调。

## 核心原理

### LUT 查找表（Look-Up Table）

LUT 是颜色分级中最高效的工具，本质上是一个预计算的颜色映射表。**1D LUT** 仅对每个通道的亮度值做独立映射，适合做简单的亮度曲线调整；**3D LUT** 则将 RGB 三个通道作为三维坐标系的轴，在一个三维色彩立方体中存储每个输入颜色对应的输出颜色，可以实现任意复杂的颜色变换。

游戏引擎中常用的 3D LUT 尺寸为 **16×16×16**（共 4096 个颜色节点）或 **32×32×32**（共 32768 个节点）。实际渲染时，对于落在两个节点之间的颜色，GPU 使用三线性插值（Trilinear Interpolation）计算精确输出值。Unreal Engine 的后处理设置中，LUT 以一张 **256×16 像素**的 2D 纹理存储，将 16×16×16 的三维立方体展开排列成一行。使用 LUT 的性能开销极低，每个像素仅需一次纹理采样操作。

### Lift / Gamma / Gain 模型

Lift/Gamma/Gain 是来自专业调色软件的三段式颜色控制模型，分别作用于图像的暗部、中间调和高光区域：

- **Lift（提升）**：对应图像的暗部（阴影），调整黑点位置。将 Lift 值向上推会使阴影整体变亮，让黑色"浮起来"，常用于模拟胶片的暗部柔化特性。公式表达为：`Output = Input × (1 - Lift) + Lift`
- **Gamma（伽马）**：对应中间调，通过幂次函数改变中灰的亮度。Gamma > 1 使中间调变亮，Gamma < 1 使中间调变暗。公式：`Output = Input ^ (1 / Gamma)`
- **Gain（增益）**：对应高光区域，调整白点位置，以乘法方式缩放整个亮度范围。公式：`Output = Input × Gain`

三者组合可以精细控制画面的整体色调分布：例如将 Lift 偏向蓝色、Gain 偏向橙色，可以模拟电影《黑客帝国》中绿色调加强、暗部偏蓝的效果。

### HSV / 色相-饱和度调整

除了亮度控制外，颜色分级还包含对色相（Hue）、饱和度（Saturation）和明度（Value）的独立调整。饱和度为 0 时图像完全去色变为灰阶，这在引擎的后处理中是一个单独的浮点参数（Saturation = 1.0 为原始值）。色相偏移（Hue Shift）以 0–360 度为单位旋转整个色环，例如将所有颜色的色相旋转 180 度会产生反色效果，旋转 30 度可以将红色系偏向橙色系。

## 实际应用

**游戏场景氛围统一**：在开放世界游戏中，白天场景通过高饱和度、暖色 Gain 和轻微正向 Gamma 传达活力感；同一关卡切换到夜晚时，仅替换一张 3D LUT 即可将整个画面压低亮度、增加蓝青色相偏移，而无需修改任何灯光资源。

**电影感 LUT 制作流程**：美术人员先在 DaVinci Resolve 或 Photoshop 中对一批截图进行调色，确认满意后将该颜色变换导出为 `.cube` 格式的 3D LUT 文件（标准格式，每行记录一个 RGB 输出值），再导入 Unreal Engine 或 Unity 的后处理组件中应用到整个渲染管线。

**跨平台亮度适配**：HDR 显示器支持 1000 nit 以上的亮度，而 SDR 显示器上限约为 100 nit。颜色分级配合色调映射，可为两个平台维护不同的 LUT：HDR LUT 保留更大的高光细节范围，SDR LUT 则通过 Gain 压缩高光防止过曝。

## 常见误区

**误区一：颜色分级可以在线性空间随意进行**。颜色分级应在色调映射完成、图像转换回伽马空间（sRGB，Gamma ≈ 2.2）之后进行。如果直接在线性光照数据上应用 LUT，会导致颜色映射失真——因为 LUT 是针对感知均匀的 sRGB 空间设计的，线性空间的亮度分布完全不同（线性空间中 0.5 的数值对应人眼感知约 73% 的亮度）。

**误区二：Lift 只影响纯黑色**。实际上 Lift 对整个图像都有作用，只是对暗部影响最明显。根据公式 `Output = Input × (1 - Lift) + Lift`，当 Input = 0 时输出等于 Lift 值（直接改变黑点），而当 Input = 1.0（纯白）时输出仍为 1.0，说明 Lift 对纯白没有影响，但中间值会受到线性偏移。许多初学者以为 Lift 是局部操作，实则是全局的加法偏移。

**误区三：3D LUT 精度越高越好**。将 LUT 从 16³ 提升到 64³ 并不总是值得的。16³ 的 LUT 配合三线性插值，对于平滑的颜色渐变（如天空、皮肤）误差极小，肉眼难以察觉；但在处理极端的颜色弯曲变换（如将红色映射到绿色）时，较低分辨率 LUT 的插值才会出现明显带状伪影。因此选择 LUT 精度时应根据实际颜色变换的剧烈程度决定，而非一味追求大尺寸。

## 知识关联

**前置概念——色调映射**：色调映射将 HDR 线性光照数据压缩到 [0, 1] 范围，这是颜色分级的输入数据来源。颜色分级必须在色调映射之后进行，否则 LUT 中预设的颜色节点与实际输入的亮度范围不匹配，导致整个颜色映射偏移。ACES 等色调映射算法自带一定的风格化倾向，颜色分级则在此基础上做进一步的创意调整。

**后续概念——暗角效果**：暗角（Vignette）通常紧接在颜色分级之后应用，它根据像素到画面中心的距离衰减亮度，将观众视线引导向中心主体。暗角的颜色偏移（如深蓝色暗角 vs 黑色暗角）需要与颜色分级的整体色调保持风格一致，两者共同构成后处理管线的最终视觉输出。掌握了颜色分级后，可以进一步利用 Lift 的颜色控制为暗角区域定制专属的色调偏移，使暗角与整体调色风格无缝融合。
