# 颜色分级

## 概述

颜色分级（Color Grading）是后处理流程中对最终渲染画面进行风格化色彩调整的技术，其目标不仅是修正颜色偏差，更是主动塑造画面的情绪基调——例如将冷色调的蓝绿色用于科幻赛博朋克风格，或将橙色与蓝色的互补对比用于好莱坞商业大片的标志性外观。与前一阶段的色调映射（Tone Mapping）不同，色调映射解决的是HDR信号压缩到显示范围的技术问题，而颜色分级是在线性或Gamma空间之上施加创作意图。

颜色分级最早源于20世纪50年代电影胶片的化学冲洗工艺（称为"Bleach Bypass"漂白旁路等技法），数字时代则由达芬奇（DaVinci Resolve）等调色软件在2000年代初将其带入标准化流程。实时渲染领域在2010年前后将颜色分级引入游戏引擎的后处理管线，Unity的Post Processing Stack和Unreal Engine 4的Post Process Volume均内置了完整的颜色分级工具链。

颜色分级对游戏视觉一致性至关重要，因为它是唯一能在不修改场景资产的情况下统一整个画面色调的手段。一个错误配置的颜色分级参数会让高质量的PBR材质看起来平淡无味，而正确的分级能让简单的场景产生专业级的视觉冲击力。

## 核心原理

### LUT查找表（Look-Up Table）

LUT是颜色分级最高效的实现方式，其本质是一张预计算的颜色映射表。3D LUT通常以17³或33³的网格存储RGB颜色空间中每个采样点的输出颜色，文件格式常见`.cube`（33×33×33共35,937个颜色节点）。运行时GPU将输入像素的RGB值作为三维坐标在LUT纹理中进行三线性插值，从而以极低的性能代价（约0.1ms）应用复杂的颜色变换。

使用LUT的关键工作流是：先在DaVinci Resolve或Photoshop中截取游戏的中性画面，调色完成后导出`.cube`文件，再将其导入引擎作为后处理LUT纹理。Unreal Engine的LUT强度参数（LUT Blend Weight）取值范围0到1，可在运行时通过Blueprint动态混合多张LUT以实现昼夜切换或情绪转变效果。

### Lift / Gamma / Gain 三段式调整

这是颜色分级的最基础数学框架，对应阴影、中间调、高光三个亮度区间的独立控制：

- **Lift（抬升）**：作用于暗部，公式为 `Output = Lift + (1 - Lift) × Input`，调整黑色基底，正值会让阴影偏灰（扁平化对比度）。
- **Gamma（伽马）**：作用于中间调，公式为 `Output = Input^(1/Gamma)`，Gamma值大于1时中间调变暗，小于1时变亮。
- **Gain（增益）**：作用于高光，公式为 `Output = Input × Gain`，是乘法操作，不影响纯黑区域。

三者组合使用时，专业调色师通常先用Lift和Gain设定对比度范围，再用Gamma调整整体曝光感，最后对各通道的R/G/B分量分别施加偏移来引入色彩倾向。

### HSL（色相/饱和度/亮度）分量调整

HSL调整允许对特定颜色通道单独操控，例如将人物皮肤的橙色色相精确旋转4度以修正摄像机白平衡差异，或将植被的绿色饱和度降低20%以获得干旱荒芜的场景氛围。Unreal Engine的颜色分级面板提供了0到2的饱和度倍增器（1为原始值），以及-1到1的色相偏移滑块（单位为归一化圆周，1.0等于360度旋转）。

### 色温与色调（Temperature / Tint）

色温参数模拟白平衡调整，以开尔文（K）为参考单位：降低色温值使画面偏蓝冷（模拟6500K日光变为4000K钨丝灯色），升高色温使画面偏橙暖。色调（Tint）则沿洋红-绿轴做补偿，专门用于纠正荧光灯的绿色偏色。两者配合使用能精确还原或主动创造特定光照环境的视觉感受。

## 实际应用

**末世/废土风格**：Lift的R/G/B分量分别偏向橙色（R略高），Gamma整体压低至0.85增加压抑感，Gain的蓝色通道降低约15%，最终导出LUT后叠加轻微去饱和（Saturation约0.8），实现《辐射》系列标志性的褪色黄绿色调。

**水下场景切换**：通过Blueprint在玩家进入水体触发器时，用0.5秒的时间将LUT从中性LUT线性混合至蓝绿色调LUT（色温参数降低到−0.3），同时将整体饱和度从1.0降低到0.7，模拟光线在水中散射后颜色信息丢失的现象。

**恐怖游戏的情绪强化**：将阴影区域的Lift绿色通道推至轻微负值（约−0.02），使暗部偏向洋红/红色，这种接近尸斑颜色的微妙暗示会在潜意识层面增加玩家不适感，是很多恐怖游戏刻意使用的技巧。

## 常见误区

**误区一：颜色分级应该在线性空间之前进行**。正确的管线顺序是：PBR渲染 → 色调映射（将HDR压缩至[0,1]） → 颜色分级 → 输出至显示设备。若在色调映射之前进行颜色分级，高光区域的颜色偏移会被随后的色调映射压缩，导致调色结果完全不可预测。许多初学者因错误顺序而发现调色效果"在明亮场景中失灵"。

**误区二：LUT可以随意截图制作**。LUT的源图像必须是"中性LUT Identity图"（一张包含所有颜色渐变的标准测试图），不能用游戏任意截图作为LUT基础。用普通截图导出的调色参数只对该特定场景的颜色分布有效，应用到其他场景会产生颜色断层和偏色。

**误区三：饱和度越高画面越好看**。过高的饱和度（超过1.3倍）会导致颜色超出sRGB显示色域，在标准显示器上产生颜色剪裁（Clipping），使渐变区域出现硬边色带（Banding）。专业颜色分级通常在应用LUT后将饱和度控制在0.9到1.15之间，通过色相对比而非饱和度堆砌来增强视觉冲击力。

## 知识关联

颜色分级建立在**色调映射**的输出结果之上：色调映射的ACES或Reinhard算法决定了高光和阴影的压缩形状，这个形状直接影响颜色分级中Lift和Gain能有效作用的亮度范围。若使用ACES色调映射，其内置的胶片感S曲线已经强化了对比度，此时颜色分级的Contrast参数应相应降低，避免过度对比导致阴影丢失细节。

完成颜色分级后，后处理管线的下一个环节是**暗角效果（Vignette）**。暗角在视觉上依赖于颜色分级建立的整体色调基础——暗角的颜色应从分级后的画面暗部颜色中取样，而非使用固定黑色，否则暗角边缘会与经过颜色分级的阴影产生色调不匹配的割裂感。
