# 水花序列帧

## 概述

水花序列帧是将水花飞溅、液体碰撞、水滴扩散等流体动态过程拆分成若干静态帧图像，以PNG或TGA格式按时序排列并在游戏引擎中逐帧播放的特效技术。与火焰序列帧的扩张发光特性不同，水花序列帧的核心视觉特征是透明度的快速衰减与高光反射的精准捕捉——每帧画面需要保留水珠边缘的菲涅耳高光和半透明水膜，才能还原真实水花的折射感。

这一技术在格斗游戏和动作游戏中大规模普及的节点可追溯至1996年：《超级马里奥64》使用了简化的8帧水花序列来表现水面扰动，而《波斯王子：时之沙》（2003年）则率先将32帧水花序列帧与实时法线贴图结合，在PS2硬件限制下实现了当时令人信服的流体溅射效果。现代AAA游戏通常使用32帧乃至64帧的高精度序列，配合法线贴图通道实现立体感。在移动端游戏中，8×8排列的精灵表（即64格，实际循环帧数通常为16至24帧）是兼顾视觉质量与GPU带宽的行业平衡方案，而PC/主机平台则常见16×8排列共128格的规格。

水花序列帧的难度高于火焰序列帧，根本原因在于水的透明性与折射性要求Alpha通道具备更高的精度层次。水珠飞溅产生的细丝状拉伸形变，要求每帧之间的关键形变量不超过前一帧像素位移的15%，超出此阈值则播放时会出现肉眼可见的跳跃感，业内称之为"水花抖动"（Splash Jitter）。

## 核心原理

### 流体形态的帧率设计

水花飞溅的完整生命周期通常分为三个阶段：**碰撞冲击阶段**（第1至6帧）、**飞溅展开阶段**（第7至18帧）、**落水消散阶段**（第19至24帧）。冲击阶段形变最剧烈，需要最密集的关键帧采样；消散阶段水花逐渐透明，Alpha值从约180递减至0，此段曲线应采用缓出（Ease Out）插值而非线性插值，否则消失过程视觉上会显得生硬截断。

在Houdini的FLIP Solver中模拟水花时，业界标准做法是将模拟帧率锁定在120FPS执行，再通过"每5帧抽取一帧"的下采样方式输出24帧最终序列。120FPS模拟保证了物理计算的精确性，而下采样至24FPS则将单次导出的EXR序列文件从约2.4GB压缩至约480MB，大幅降低后续烘焙的时间成本（参考《Houdini特效技术解析》，人民邮电出版社，2021年）。

### Alpha通道的双层结构

水花序列帧与火焰序列帧最显著的技术差异体现在Alpha通道的处理方式上。火焰通常使用单一亮度蒙版作为Alpha来源，而水花需要"形状Alpha"与"折射Alpha"两层叠加：

- **形状Alpha**：控制水珠的轮廓边界，由流体模拟中的粒子SDF（Signed Distance Field）烘焙而来，边缘羽化半径建议设为2至4像素。
- **折射Alpha**：控制水膜内部的半透明程度，基于菲涅耳公式计算：

$$F(\theta) = F_0 + (1 - F_0)(1 - \cos\theta)^5$$

其中 $F_0$ 为水的基础反射率，约为 $0.02$（即空气-水界面在垂直入射时反射率为2%），$\theta$ 为视线向量与水面法线的夹角。当 $\theta$ 趋近90°（掠射角）时，$F(\theta)$ 趋近1.0，表现为水边缘的强烈高光；当 $\theta$ 为0°（正视）时，$F(\theta) = 0.02$，表现为水膜中心的高透明度。

将形状Alpha与折射Alpha相乘后烘焙进贴图的Alpha通道，能在不增加额外Draw Call的情况下实现更自然的水珠透明过渡。

### UV布局与精灵表排列

水花序列帧的精灵表（Sprite Sheet）标准布局为每行8帧，按从左到右、从上到下的Z形顺序排列。贴图分辨率通常选择2048×2048，此时单帧有效区域为256×256像素，纹素密度足以表现直径约30厘米的游戏内水花效果。需要特别注意的是，水花的飞溅具有方向不对称性——主飞溅方向的帧画面宽度往往大于辅飞溅方向，因此在UV裁剪时不能对所有帧使用完全相同的采样矩形，而应在着色器中引入`per-frame UV偏移`参数。

在Unity的Particle System中，"Texture Sheet Animation"模块支持通过曲线控制逐帧的UV采样区域起始点（Start Frame）与循环次数（Cycles），正是为应对水花不对称飞溅而设计的灵活方案。在Unreal Engine 5中，Niagara的`SpriteRenderer`模块则通过`Sub UV Animation`属性提供类似的逐帧UV控制，可直接绑定曲线资产驱动非均匀采样。

## 关键公式与着色器实现

水花序列帧在实时渲染中的着色器核心逻辑涉及两个计算：UV帧序列采样和折射扰动叠加。以下是一段HLSL伪代码示意（适用于Unity URP自定义粒子着色器）：

```hlsl
// 水花序列帧 UV 采样（8列×8行精灵表）
float2 SpriteSheetUV(float2 baseUV, float frameIndex, float totalCols, float totalRows)
{
    float col = fmod(frameIndex, totalCols);
    float row = floor(frameIndex / totalCols);
    float2 cellSize = float2(1.0 / totalCols, 1.0 / totalRows);
    return (baseUV + float2(col, row)) * cellSize;
}

// 菲涅耳折射Alpha混合
float FresnelAlpha(float3 viewDir, float3 normal, float F0)
{
    float cosTheta = saturate(dot(viewDir, normal));
    float fresnel = F0 + (1.0 - F0) * pow(1.0 - cosTheta, 5.0);
    return fresnel;
}

// 最终Alpha = 形状Alpha × 折射Alpha
float shapeAlpha   = tex2D(_MainTex, spriteUV).a;
float refrAlpha    = FresnelAlpha(viewDir, normalWS, 0.02);
float finalAlpha   = shapeAlpha * lerp(0.4, 1.0, refrAlpha);
```

上述代码中，`lerp(0.4, 1.0, refrAlpha)` 将折射Alpha的作用范围限制在0.4至1.0之间，防止水膜中心区域完全透明而导致视觉上的"空洞感"——这一0.4的下限值是根据真实水珠照片测量得到的经验数值，在《Real-Time Rendering, 4th Edition》（Akenine-Möller et al., 2018）第9章中有对应的透明液体渲染论述支撑。

## 实际应用

**角色踩水动效**：在俯视角RPG中，角色每步踩在水面时触发一次水花序列帧播放。通常将序列帧粒子的初始Scale设为0.3至0.5，在第3帧时快速放大至1.0再逐渐缩小，模拟水花向外扩散的视觉冲击。脚底水圈涟漪使用单独的UV动画贴图，与水花序列帧叠加显示，两套特效需要通过深度偏移（Depth Bias约0.001）防止Z-Fighting。

**武器打水面技能**：法师类角色的水系技能通常在命中点同时触发3至5个水花序列帧实例，各实例在Y轴上随机偏移±0.05单位，在初始旋转角上分别相差72°（即360°/5），形成放射状飞溅。为避免多个实例同帧导致粒子填充率超限，各实例的播放起始帧应错开2至3帧，使GPU在同一渲染帧内不必处理完全相同的Alpha混合层叠。

**例如**：《原神》水元素命中地面效果中，主水花序列帧使用24帧循环、2048分辨率精灵表，同时搭配一张256×256的泡沫消散序列帧（16帧），两层序列帧的总Batch数量通过GPU Instancing压缩至2个Draw Call，确保移动端60FPS不掉帧。

**下雨积水场景**：地面积水的持续水花效果需要将序列帧设置为循环播放（Loop），但直接循环第1帧到第24帧会产生明显的衔接跳跃。解决方案是在精灵表末尾额外追加4帧过渡帧，使消散末期的水面状态与冲击初期进行形态混溶（Cross-dissolve），过渡帧的Alpha差值应控制在每帧不超过20/255，肉眼才无法察觉循环点。

## 常见误区

**误区1：将水花序列帧的Blend Mode设为"Additive（叠加）"**。叠加模式适用于火焰、闪光等自发光效果，因为它能使亮部叠亮。但水花本质上是透明折射体，使用叠加模式会导致水珠高光在深色背景下过曝（亮度被强行叠加），而在浅色背景下几乎不可见。正确的做法是使用"Alpha Blend（Alpha混合）"模式，同时开启预乘Alpha（Premultiplied Alpha），避免水珠边缘出现黑边伪影。

**误区2：Alpha通道直接使用RGB图像的亮度值**。水花的飞溅丝线在摄影参考图中往往呈半透明白色，亮度值高但不代表不透明度高。若直接将RGB亮度转为Alpha，会把水花中央较暗的水膜区域误判为透明，而实际上水膜中心的不透明度应高于边缘细丝。正确流程是在Houdini中单独输出`surface_id`蒙版作为形状Alpha，再叠加由法线驱动的折射Alpha。

**误区3：用16帧序列替代24帧序列以节省内存**。从24帧减少至16帧，精灵表面积从2048×2048（约16MB，RGBA8格式）缩减至1024×1024（约4MB），内存节省明显。但水花飞溅展开阶段（原第7至18帧，共12帧）将被压缩至约8帧，帧间形变量从平均12%增大至约18%，超过15%的"水花抖动"安全阈值，导致展开过程出现明显跳帧感。若内存紧张，建议保留24帧但将贴图格式从RGBA8改为BC7（DX11+），可在几乎无损画质的情况下将内存降至约4MB。

**误区4：忽略法线贴图通道对水花立体感的贡献**。仅有RGB颜色与Alpha通道的水花序列帧在平视角下看起来是平面贴片，缺乏水珠的立体隆起感。在序列帧精灵表中额外添加一张同步的法线序列帧（存储在独立贴图的RG通道，B通道重建Z值），并在着色器中将法线与屏幕空间光照叠加，可以使水珠在动态光源下呈现真实的高光滚动效果。这一技术被《战地2042》（2021年）的水花特效系统所采用，其法线序列帧与颜色序列帧共享同一UV动画曲线，保证两者帧数同步。

## 知识关联

**前置概念——火焰序列帧**：掌握火焰序列帧后，水花序列帧的最大新增难点集中在双层Alpha的合成逻辑上。火焰的Alpha处理是单层亮度蒙版，而水花的折射Alpha引入了菲涅耳角度依赖计算，需要在序列帧制作阶段（Houdini）而非引擎阶段预先烘焙视角相关信息，或在引擎着色器中实时计算。
