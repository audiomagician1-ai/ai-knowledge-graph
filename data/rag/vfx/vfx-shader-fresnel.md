# 菲涅尔效果

## 概述

菲涅尔效果（Fresnel Effect）得名于19世纪法国物理学家奥古斯丁-让·菲涅尔（Augustin-Jean Fresnel），描述的是光线在介质表面发生反射的强度随观察角度变化的物理现象。当视线方向与物体表面垂直时（即正视表面），反射率最低；当视线方向与物体表面趋于平行时（即掠射角），反射率急剧上升至接近100%。这一原理解释了为什么平静湖面正中心看起来透明，而远处湖面边缘却显得高亮闪烁。

在Shader特效领域，菲涅尔效果被简化为一个依赖"视线向量与法线向量夹角"的边缘增亮公式，广泛用于模拟护盾能量层、全息投影外壳、角色轮廓描边等视觉效果。它不需要复杂的光线追踪计算，仅凭顶点或像素级的向量点积运算即可实现高度可信的物理感光晕，是游戏和影视特效中性价比极高的渲染技术。

菲涅尔效果的重要性在于它能以极低的性能开销赋予物体"体积感"和"能量感"。一个没有菲涅尔的护盾特效只是一层均匀透明的几何体，而加入菲涅尔后，盾牌边缘自然亮起，中心保持通透，立刻传达出能量聚集于轮廓的视觉叙事。

## 核心原理

### 菲涅尔公式与Schlick近似

精确的菲涅尔方程涉及复数折射率计算，对实时渲染过于昂贵。1994年Christophe Schlick提出的近似公式是游戏Shader中的标准实现：

```
F(θ) = F₀ + (1 - F₀) × (1 - cosθ)⁵
```

其中：
- **F₀** 是材质在垂直入射（cosθ = 1）时的基础反射率，金属约为0.8~1.0，水面约为0.02，皮肤约为0.028
- **cosθ** = dot(ViewDir, Normal)，即视线方向向量与表面法线向量的点积
- **幂次5**是Schlick近似的关键参数，控制边缘过渡的锐利程度

在特效Shader中，F₀通常被替换为可调的`FresnelColor`颜色参数，幂次也常由艺术家自定义为`FresnelPower`（典型值范围1~8），以便在护盾偏柔和与护盾偏硬朗之间灵活调节。

### 视线-法线点积的空间计算

实现菲涅尔效果的核心步骤是在世界空间或切线空间中正确计算 dot(N, V)：

1. **获取法线向量 N**：来自顶点法线或法线贴图解码后的法线，需归一化（normalize）
2. **获取视线向量 V**：摄像机世界坐标减去顶点/像素世界坐标后归一化
3. **计算点积并取反**：`fresnel = 1.0 - saturate(dot(N, V))`，得到边缘为1、正面为0的遮罩值
4. **应用幂次**：`fresnel = pow(fresnel, FresnelPower)`，幂次越高边缘越窄越锐利

需特别注意：若法线朝向有翻转（如双面渲染的护盾球体内侧），dot(N, V)会为负数，必须使用`abs()`或`saturate()`处理，否则内侧表面的菲涅尔会完全消失。

### 菲涅尔遮罩的混合与着色

原始菲涅尔值是一个0~1的灰度遮罩，特效制作中通常有三种使用方式：

- **加法叠加（Additive）**：`Output = BaseColor + FresnelColor × FresnelMask`，用于护盾发光、能量轮廓，与Bloom泛光配合可产生HDR过曝的亮边效果
- **插值混合（Lerp）**：`Output = lerp(BaseColor, RimColor, FresnelMask)`，用于全息材质的边缘色调渐变
- **透明度控制**：将FresnelMask写入Alpha通道，使护盾正面接近透明（Alpha≈0），边缘趋于不透明（Alpha→1），实现物理上准确的能量护盾透视感

## 实际应用

### 能量护盾特效

《光晕》系列游戏中标志性的蓝色护盾效果是菲涅尔Shader的经典案例。实现方式是：在一个包裹角色的略大球形网格上，将菲涅尔遮罩以加法模式叠加到半透明的蓝色基础色上，FresnelPower设为约3.0，FresnelColor为亮蓝白色（HDR值超过1.0），配合Bloom泛光产生向外溢出的光晕。当护盾受击时，还可以将受击点的UV坐标传入，叠加一个从受击点向外扩散的涟漪扭曲（此处与折射扭曲效果联动），菲涅尔决定涟漪在边缘处更亮、中心处更暗。

### 全息投影外壳

全息角色的常见Shader做法是：基础透明度固定为0.3左右，再将菲涅尔遮罩加到透明度上，使轮廓处不透明度提升至0.7~0.9。同时FresnelColor指定为青色（Cyan，#00FFFF），与扫描线贴图叠加，实现"内部可见身体但外壳有明显能量轮廓"的科幻质感。FresnelPower通常设为2.5，避免边缘过于锐利而显得廉价。

### 卡通轮廓描边

区别于传统的Pass外扩描边法，菲涅尔描边无需额外的背面几何体，在单Pass中即可实现。将FresnelPower调高至5~8，FresnelColor设为纯黑，以乘法模式压暗边缘颜色，可模拟卡通渲染中的"厚涂风格边缘阴影"，且描边粗细会随模型曲率自动变化，在弯曲处更宽、在平面处更窄，比固定宽度的几何描边更具表现力。

## 常见误区

### 误区一：菲涅尔幂次越高效果越好

许多初学者会将FresnelPower设为10甚至20，认为越高越"锐利越酷炫"。但幂次超过6后，菲涅尔边缘宽度会压缩到不足几个像素，在低面数模型上会呈现明显的分段锯齿状，完全失去圆润的能量感。护盾类特效的FresnelPower通常在2~4之间，轮廓描边在4~6之间，需根据模型面数和目标分辨率实测调整，而不是一味追高。

### 误区二：菲涅尔等同于边缘描边，不适用于平面物体

菲涅尔依赖法线与视线的夹角，平面物体（如一张卡片）的法线处处相同，正对摄像机时dot(N,V)≈1，全面片几乎没有菲涅尔效果。但这并不意味着平面物体不能用菲涅尔——只需在Shader中引入顶点颜色或自定义法线（将边缘顶点的法线手动向外弯折），即可在平面几何体上伪造出边缘增亮，这在UI能量条、平面护盾闸门特效中很常见。

### 误区三：菲涅尔颜色必须是单色

实际上FresnelColor可以是纹理采样结果，可以随时间动画，也可以是来自光源颜色的动态值。将菲涅尔遮罩乘以一张流动的噪声纹理，边缘就会出现闪烁的能量纹路；将FresnelColor与场景中主光源颜色相乘，护盾就会在日落时变成橙红色边缘，在蓝天下变成冷色边缘，大幅提升场景融合感。

## 知识关联

菲涅尔效果与**折射扭曲**之间存在天然的协作关系：折射扭曲Shader负责护盾内部的背景畸变，而菲涅尔决定哪些区域的扭曲强度更高——将菲涅尔遮罩作为折射扭曲强度的乘数，可以使护盾边缘扭曲更剧烈、中心扭曲微弱，符合物理上边缘折射角更大的直觉。两者组合是能量护盾特效的完整基础套件。

菲涅尔Shader输出的遮罩值也是**Bloom泛光**系统的理想输入素材：将HDR值（大于1.0）的菲涅尔颜色写入HDR颜色缓冲，Bloom后处理会自动对这些高亮边缘进行辉光扩散，形成向外渗出的光芒。两者在渲染管线中处于不同阶段（Bloom是屏幕后处理），但视觉上相互依存，缺少Bloom的菲涅尔边缘会显得生硬，而Bloom配合适当的菲涅尔强度控制才能避免全屏溢出。
