# 菲涅尔效果

## 概述

菲涅尔效果（Fresnel Effect）得名于19世纪法国物理学家奥古斯丁-让·菲涅尔，描述的是光线在与表面夹角不同时，反射率随观察角度发生变化的光学现象。当视线方向与表面法线平行时（正对着表面），反射率最低；当视线方向接近垂直于法线时（掠射角，即观察边缘部分），反射率趋近于1，表现为强烈的边缘高光。这一物理规律被游戏和影视特效领域广泛用于模拟护盾、全息投影、能量轮廓等视觉效果。

在Shader实现中，菲涅尔效果的核心计算公式为 `F = pow(1 - dot(N, V), power)`，其中 N 是顶点法线向量，V 是摄像机到顶点的视线方向向量，`power` 参数控制边缘光的衰减锐利度（通常取2到5之间）。`dot(N, V)` 的结果越接近0，说明当前片元越靠近模型边缘，菲涅尔值 F 越大，边缘越亮。这个公式是简化版的 Schlick 近似公式，相比完整菲涅尔方程计算量更小，适合实时渲染。

菲涅尔效果之所以在护盾类特效中不可替代，是因为它能在不增加额外几何体的前提下，仅凭观察角度差异就制造出立体的发光轮廓感。与单纯的边缘检测后处理不同，菲涅尔效果是在物体自身的表面着色阶段完成的，因此在VR等需要双目渲染的场景中，左右眼视角各自计算得到的边缘光位置天然符合立体视觉，不会产生视差错误。

## 核心原理

### 法线与视线夹角的计算

菲涅尔计算的输入必须是**世界空间或切线空间中归一化的法线向量 N 和视线向量 V**。在Unity Shader中，视线向量通过 `normalize(_WorldSpaceCameraPos - worldPos)` 获得，法线通过 `UnityObjectToWorldNormal(v.normal)` 转换到世界空间。`dot(N, V)` 的计算结果范围是 -1 到 1，但对于正面朝向摄像机的片元，结果应在 0 到 1 之间；若出现负值需要用 `saturate()` 函数截断到0，避免 `pow()` 计算产生非预期结果。

### power参数对视觉风格的控制

`power` 参数直接决定菲涅尔边缘光的宽窄和硬度：
- **power = 1**：边缘光非常宽泛，整个球面都会有明显渐变，适合大气散射或次表面散射模拟。
- **power = 3~4**：边缘光较窄且清晰，是护盾效果最常用的取值范围，边缘明显但内部仍保持透明。
- **power = 8~10**：边缘光极其锐利，几乎只有最边缘的几个像素才会发光，适合科幻风格的能量刀刃轮廓。

在美术调试阶段，通常将 `power` 暴露为材质参数，配合曲线调整，而不是使用固定数值。

### 与Bloom泛光的配合方式

单独的菲涅尔 Shader 输出的是模型表面的边缘颜色，颜色值通常在0到1的SDR范围内，视觉上缺少发光感。要制造真正的"自发光"视觉效果，需要将菲涅尔计算结果乘以一个大于1的HDR亮度值（例如乘以3.5），使输出颜色超过1.0进入HDR范围，再交由后处理管线中的Bloom泛光模块进行光晕扩散处理。这就是为什么菲涅尔效果的前置知识包含Bloom泛光——仅有边缘高光而没有Bloom，护盾效果会显得"塑料感"而非"能量感"。

## 实际应用

### 能量护盾特效

在护盾Shader中，菲涅尔值 F 同时控制两件事：边缘发光强度和透明度。透明度公式通常写为 `alpha = F * shieldOpacity + baseAlpha`，使护盾正面几乎完全透明而边缘不透明，配合六边形扫描线纹理贴图叠加，形成经典的科幻护盾视觉。当护盾受到攻击时，通过在 `baseAlpha` 上叠加一个随时间衰减的冲击波纹理，可以实现被击打区域短暂变白变实的反馈效果，而菲涅尔边缘光始终保持稳定。

### 全息投影人物轮廓

全息角色通常使用两个Pass渲染：第一个Pass只写菲涅尔边缘光（关闭深度写入），第二个Pass渲染半透明的扫描线主体。菲涅尔的 `power` 在全息效果中通常设为5到6，并在边缘颜色上叠加随时间变化的噪声扰动，模拟全息信号不稳定的闪烁感。将菲涅尔值与 `sin(time * 8.0 + worldPos.y * 20.0)` 相乘，可以产生随高度和时间变化的扫描线噪波，无需额外纹理采样。

### 水面与玻璃的真实感菲涅尔

在需要物理准确的折射扭曲材质（如水面、玻璃）中，菲涅尔效果决定了何时显示反射、何时显示折射内容的混合比例。正对水面俯视时折射比例高（能看到水底），以小角度斜视水面时反射比例趋近于100%（镜面效果）。这与护盾特效的用法完全不同：前者是物理混合，后者是纯粹的视觉风格化，但底层的 `dot(N, V)` 计算完全一致。

## 常见误区

**误区一：忘记法线向量归一化导致边缘光不均匀**

在顶点着色器中插值传递到片元着色器的法线向量，由于GPU的线性插值机制，到达片元阶段时模长不再是1。如果在片元着色器中直接使用未归一化的法线进行 `dot(N, V)` 计算，边缘光在模型弯曲较大的区域（如球体两侧）会出现明显的亮度不均匀条带。正确做法是在片元着色器入口处对法线重新执行 `normalize()`，虽然增加了少量计算量，但在移动端以外的平台上性能代价几乎可以忽略。

**误区二：将菲涅尔颜色直接叠加到漫反射而不使用加法混合**

初学者常将菲涅尔发光颜色与漫反射颜色做乘法混合（Multiply），结果是越暗的表面边缘光越弱，失去了边缘强调的效果。护盾和全息特效中，菲涅尔颜色必须使用**加法叠加**（`final = diffuse + fresnelColor * F`），这样边缘光效果才能独立于底色亮度，在黑色背景上也能清晰显示。

**误区三：在正交摄像机下使用不兼容的视线向量计算**

在透视摄像机下，每个片元的视线方向不同，需要逐片元计算 `normalize(cameraPos - worldPos)`。但在正交摄像机下，所有片元的视线方向相同，直接等于摄像机朝向向量。如果在正交摄像机场景（如2D游戏的UI护盾效果）中仍使用透视摄像机的视线计算方式，会得到正确值；但若美术切换摄像机类型却发现效果消失，需要检查是否使用了渲染管线内置的视线方向宏，因为部分宏在正交模式下返回零向量。

## 知识关联

**折射扭曲与菲涅尔的协同关系**：在水面、护盾玻璃类材质中，折射扭曲负责对背景进行UV偏移采样制造变形感，而菲涅尔效果负责控制这种变形在边缘位置减弱（边缘更多显示反射而非折射）。两者共用同一套法线数据，折射扭曲使用切线空间法线贴图，菲涅尔使用世界空间顶点法线，分工明确。

**迈向顶点动画的过渡**：菲涅尔效果完全在片元着色器中计算，不修改顶点位置。学习顶点动画时，会接触到在顶点着色器中直接修改 `worldPos`，此时需要注意：顶点位移后的顶点法线若不同步更新，菲涅尔边缘光的位置会与实际视觉边缘产生错位。因此在带有顶点动画的护盾Shader中，通常在顶点着色器中同时对法线进行与位置一致的变换处理。
