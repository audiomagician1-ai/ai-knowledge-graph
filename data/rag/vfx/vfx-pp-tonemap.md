# 色调映射

## 概述

色调映射（Tone Mapping）是将高动态范围（HDR）图像压缩至显示设备可呈现的低动态范围（LDR）的数学变换过程。真实世界的亮度范围可跨越10个数量级（从0.001 cd/m² 的夜空到100,000 cd/m² 的直射阳光），而普通显示器的亮度范围仅约为0.1到400 cd/m²，色调映射正是为了弥合这一巨大差距而设计的。

这一技术的研究起源可追溯至1993年，Gregory Ward在全局光照渲染领域首次将摄影测光概念引入计算机图形学，提出了基于人眼适应的自动曝光模型。2002年，Erik Reinhard等人在SIGGRAPH发表了后来被广泛引用的Reinhard色调映射算法，将照相术中的"区域系统"理论移植到实时渲染中。此后，电影工业级别的ACES（Academy Color Encoding System）于2014年由美国电影艺术与科学学院正式发布，成为现代游戏与电影渲染管线的行业标准。

色调映射在后处理链路中处于颜色分级之前的核心步骤，其选择直接决定最终画面的整体感知亮度曲线、高光细节保留程度与阴影对比度。不同的色调映射曲线会赋予同一帧画面截然不同的视觉风格——从Reinhard的自然柔和到ACES的胶片感高饱和，选择失误会导致整体色彩方向偏离美术预期。

## 核心原理

### Reinhard算法：简单但有效的全局映射

Reinhard算法是最经典的全局色调映射公式，其核心表达式为：

$$L_{out} = \frac{L_{in}}{1 + L_{in}}$$

其中 $L_{in}$ 为输入的HDR亮度值，$L_{out}$ 为输出的LDR亮度值（范围0到1）。该函数是一条以原点为起点、无限趋近于1的渐近曲线，保证任意正值输入都不会超出\[0,1\]范围。扩展版本引入白点参数 $W$：

$$L_{out} = \frac{L_{in}(1 + L_{in}/W^2)}{1 + L_{in}}$$

当 $W = 11.2$ 时，可与Uncharted 2的Filmic曲线在感知上对齐。Reinhard算法最大的缺点是高光区域对比度损失严重，天空与强光源在接近白色时趋于扁平，缺乏层次感。

### Filmic曲线：Uncharted 2的工业实践

2010年，Naughty Dog程序员John Hable（又名Filmic Games）为《神秘海域2》设计了一条分段多项式曲线，俗称"Uncharted Filmic Tone Mapping"：

$$f(x) = \frac{x(Ax + CB) + DE}{x(Ax + B) + DF} - \frac{E}{F}$$

其中默认参数为 A=0.22（肩部强度），B=0.30（线性强度），C=0.10（线性角），D=0.20（趾部强度），E=0.01（趾部数字部分），F=0.30（趾部分母部分）。这组参数经过精心调校，使中间灰度区域呈近线性响应，而高光和暗部均有柔和的S形过渡，视觉上接近摄影胶片的曝光曲线。

### ACES：基于物理的色彩空间映射

ACES并非单一曲线，而是一套完整的色彩管理框架，包含输入变换（IDT）、参考渲染变换（RRT）与输出设备变换（ODT）三层结构。在实时渲染中常用的ACES近似公式（由Stephen Hill简化）为：

$$\text{ACES}(x) = \frac{x(2.51x + 0.03)}{x(2.43x + 0.59) + 0.14}$$

该公式将输入值先乘以约0.6的曝光预偏移，再经过有理多项式映射至LDR。与Reinhard相比，ACES在中高亮度区域对比度更高，颜色饱和度在压缩过程中有轻微提升，带来接近胶片的色彩表现。Unreal Engine 4从4.15版本起将ACES设为默认色调映射模式。

### 局部色调映射与自动曝光

局部色调映射在不同空间区域使用不同的映射强度，通常需要构建图像亮度的高斯金字塔。实时渲染中更常见的是自动曝光（Eye Adaptation）：通过计算场景亮度直方图的对数平均值（Log-Average Luminance），每帧动态调整曝光参数 $EV$：

$$\bar{L}_w = \exp\left(\frac{1}{N}\sum_{i}\ln(L_w(x_i) + \epsilon)\right)$$

自动曝光的适应速度通常使用0.1到2秒范围内的插值控制，模拟人眼从明处进入暗室时约需20秒适应的生理特性（虽有简化，但足以服务游戏体验）。

## 实际应用

**游戏引擎集成**：Unity HDRP中，色调映射作为Volume组件附加于后处理栈，提供Neutral、ACES、Custom三种模式。开发者可通过Toe Strength（趾部强度）、Shoulder Strength（肩部强度）等参数手动调节Custom曲线，在不改变光照的前提下快速改变整体风格。

**HDR显示器支持**：随着HDR10与Dolby Vision显示器的普及，现代管线区分了"HDR输出路径"与"SDR输出路径"。HDR10路径的输出范围为0到10,000 nit，色调映射曲线须相应调整或完全跳过全局压缩，仅保留轻微的高光限幅，以充分发挥显示器的高峰值亮度。

**VFX合成**：在影视后期合成中，Nuke等软件的ACES工作流要求所有素材（CG渲染、实拍镜头、特效元素）在ACES2065-1线性空间中合成，最后在输出阶段一次性应用色调映射，避免多次映射造成的色彩偏移。

## 常见误区

**误区一：色调映射等同于伽马校正**
色调映射是对亮度范围的非线性压缩，处理的是HDR到LDR的范围映射问题；伽马校正（通常为2.2）是对线性光照空间到感知均匀空间的编码变换，二者作用于不同的问题域，且在后处理管线中伽马校正固定发生在色调映射之后。将两者混淆会导致在错误的空间中叠加双重非线性，产生严重的色彩失真。

**误区二：ACES总是比Reinhard"更好"**
ACES的高对比度和颜色饱和度提升在某些场景下反而不理想——例如需要呈现柔和、梦幻感的卡通渲染，或低亮度环境艺术（如月夜场景），ACES可能导致本应低调的区域过度饱和，反而破坏美术意图。选择色调映射曲线须以项目的视觉风格目标为准，而非盲目追求"电影感"。

**误区三：色调映射参数可以在SDR与HDR显示路径间通用**
同一组Filmic曲线参数在SDR显示器（最大亮度300 nit）与HDR显示器（最大亮度1000 nit）上的视觉结果差异极大。未针对HDR路径单独校准的项目往往在HDR设备上出现画面整体偏暗或高光过曝，这是因为曲线设计时预设的"白点"在两条路径中对应的物理亮度根本不同。

## 知识关联

色调映射的前置知识点**运动模糊**同属后处理链路，运动模糊在线性HDR缓冲区中完成混合后，才将结果传入色调映射阶段，因此模糊核采样的像素亮度保持HDR精度是保证运动模糊与色调映射良好交互的前提。若运动模糊在LDR空间（色调映射之后）执行，会因亮度被截断而在高亮运动物体边缘产生错误的暗色拖影（称为"黑边伪影"）。

色调映射的后续概念**颜色分级**（Color Grading）在色调映射完成、图像进入LDR线性或对数空间之后进行。颜色分级通过3D LUT（查找表，通常为33×33×33或65×65×65采样点）对输出色彩做风格化调整，其依赖于色调映射已将亮度范围控制在可预测区间内，否则LUT采样会超出设计范围，产生色彩断裂或梯度跳变。
