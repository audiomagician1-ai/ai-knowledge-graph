# 法线序列帧

## 概述

法线序列帧是将多帧法线贴图按时间顺序排列成图集（Atlas），通过逐帧切换来模拟表面法向量随时间变化的技术。与普通颜色序列帧只改变像素颜色不同，法线序列帧改变的是每个像素点的光照响应方向，使动态特效在不同光源方向下都能产生正确的明暗变化。

该技术最早在2010年代游戏特效管线中普及，源于对粒子特效"平面感"问题的解决需求——火焰、烟雾等使用颜色序列帧的粒子在侧面光照下依然均匀发亮，缺乏体积感。通过引入法线序列帧，同一张粒子面片可以在不同光照角度下产生符合三维形态的高光和阴影滚动效果。

在移动端特效中，法线序列帧以极低的额外性能代价（相对于实时体积光照）实现了动态光照响应，其价值在于：法线贴图本身不增加顶点数，额外的渲染开销仅为一次额外纹理采样，通常不超过原绘制调用开销的15%。

## 核心原理

### 法线信息的帧间排布规则

法线序列帧图集中，每一帧的RGB通道存储的是切线空间法向量，其中R通道对应X轴（切线方向），G通道对应Y轴（副切线方向），B通道对应Z轴（法线方向）。每帧的编码方式与静态法线贴图完全相同：向量值从 [-1, 1] 线性映射到 [0, 255]，因此纯蓝色（RGB: 128, 128, 255）表示完全朝向屏幕的平坦表面。

图集排布时必须确保帧与帧之间没有颜色渗漏（Bleeding），因为相邻帧边缘的法线值若混合，会导致错误的中间法向量出现在帧切换瞬间。标准做法是在每帧周围保留至少2像素的填充边（Padding），填充内容为该帧边缘法线值的复制而非黑色，以避免双线性过滤引入错误的 (0,0,0) 向量。

### UV偏移采样机制

Shader中通过 `floor(_Time.y * _FPS) % _FrameCount` 计算当前帧索引，再将帧索引转换为图集中的UV偏移量。设图集为4×4排布（共16帧），则每帧UV尺寸为 (0.25, 0.25)，第N帧的UV起始坐标为：

```
col = N % 4;
row = floor(N / 4);
uvOffset = float2(col * 0.25, (3 - row) * 0.25);
```

注意行方向取反（3 - row）是因为UV坐标原点在左下角而图集通常从左上角开始排列。采样后得到的法线值需要经过 `normal = tex * 2.0 - 1.0` 解码还原为 [-1, 1] 范围的向量，再参与光照计算。

### 与颜色序列帧的通道打包集成

实际工程中，法线序列帧很少单独存储为独立贴图，通常利用通道打包将法线的RG两通道与颜色序列帧的Mask或其他数据合并。最常见的做法是：将颜色序列帧作为RGB贴图，将法线序列帧压缩为仅保存RG通道（B通道可由 `sqrt(1 - R² - G²)` 在Shader中实时重建），然后将这两个RG通道打包进一张RGBA贴图的不同通道中，使总纹理采样次数从2次减为1次。重建公式中要求输入法线已归一化，若原始法线在切线空间Z分量始终为正（面向观察者），该重建始终有效。

## 实际应用

**火焰特效的体积感增强**：在面向光源方向会变亮、背光方向变暗的场景光照下，使用法线序列帧的火焰粒子可以呈现出火焰"侧面受光、内部阴暗"的层次感。制作法线序列帧时，通常先渲染高多边形的3D流体模拟序列，再用法线烘焙工具逐帧提取法线贴图，最后拼合为Atlas。典型分辨率为每帧128×128，共16帧，图集总尺寸512×512。

**水面涟漪的实时光照**：水面扩散涟漪的法线序列帧可以驱动水面的高光位置随涟漪波形移动，每帧记录不同阶段的波形法线，配合Phong或Blinn-Phong模型，Specular高光会在涟漪峰值处聚集并随动画向外扩散，这是仅靠颜色序列帧完全无法实现的效果。

**角色技能命中闪光**：命中闪光特效在不同场景光照（室内暖光/室外冷光）环境中，借助法线序列帧使闪光面片产生与场景光源方向匹配的高光滚动，避免"自带光源"的突兀感。

## 常见误区

**误区一：认为法线序列帧可以像颜色序列帧一样做线性插值过渡**。颜色值的线性插值在视觉上是平滑过渡，但法线向量的线性插值会导致中间态向量的模长小于1（例如两个正交法线的插值结果模长为 √0.5 ≈ 0.707），产生错误的光照强度衰减。法线插值必须使用球面线性插值（Slerp），然而在实时Shader中Slerp计算成本较高，因此法线序列帧通常采用硬切换而非插值，或专门设计帧间过渡帧来替代插值。

**误区二：法线序列帧图集可以使用有损压缩格式（如ETC1/DXT1）**。颜色贴图可以容忍有损压缩引入的颜色误差，但法线贴图压缩误差会导致法向量偏离单位球，产生错误的光照条纹。移动端应使用ETC2或ASTC 6×6以上质量级别，PC端应使用BC5（专为法线RG双通道设计的有损压缩格式，压缩误差最小）而非BC1/DXT1。

**误区三：将法线序列帧的帧率设置得与颜色序列帧不同**。同一特效中若颜色序列帧以24fps播放而法线序列帧以12fps播放，会在光照变化与颜色变化之间产生明显的错位感，尤其在有明显高光的帧上表现突出。两者必须使用相同帧率和相同帧数，且推荐共用同一套UV偏移计算逻辑以确保绝对同步。

## 知识关联

**前置依赖——通道打包**：法线序列帧的制作流程高度依赖通道打包技术。将法线序列帧的RG通道与颜色Mask打包进同一RGBA贴图，是减少Drawcall中纹理切换次数的标准操作。若不理解通道打包的位深度限制（每通道8bit意味着法线精度约为0.0078，即1/128），就无法正确判断是否需要使用16bit法线图集。

**后续延伸——自发光序列帧**：掌握法线序列帧后，自发光序列帧是其自然延伸。自发光序列帧同样需要解决逐帧UV偏移问题，但其Shader逻辑中不参与光照计算，而是将帧数据直接叠加到自发光通道（Emission）。两者结合使用时，同一Shader同时驱动法线Atlas和自发光Atlas，可实现既对外部光源产生正确响应又有自身发光的复合特效，例如能量护盾受击效果。
