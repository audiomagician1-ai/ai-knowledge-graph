# 法线序列帧

## 概述

法线序列帧是将多帧法线贴图按时间顺序排列成图集（Atlas），通过逐帧切换采样UV坐标，使物体表面的光照法线方向随时间动态变化的技术。与普通颜色序列帧不同，法线序列帧存储的是RGB通道编码的切线空间法线向量（R=X轴偏移，G=Y轴偏移，B=Z轴法线分量），其播放结果不是颜色变化，而是表面凹凸对光源的响应动态变化。

该技术最早在游戏特效领域出现于2008年前后，当时用于模拟流动熔岩、涌动水面等需要"凹凸感会动"的效果。传统方法需要实时法线计算或高精度几何模型，而法线序列帧将预烘焙的动态法线信息压缩进一张Sprite Sheet，单张贴图可容纳4×4=16帧乃至8×8=64帧法线数据，大幅降低了运行时计算开销。

法线序列帧的核心价值在于：它赋予了2D序列帧面片"对光源方向敏感"的能力。一团火焰特效如果只有颜色序列帧，灯光从侧面打来时表面不会产生任何明暗变化；加入法线序列帧后，同一帧动画的火焰涌动部分在不同光照角度下会呈现截然不同的高光与阴影分布，使特效在动态光照场景中具备体积感。

## 核心原理

### 切线空间法线的帧数据编码规则

法线序列帧中每一帧存储的是切线空间（Tangent Space）法线向量，将范围 [-1, 1] 的XYZ分量映射为 [0, 1] 的RGB值，公式为：

```
R = (Nx + 1) / 2
G = (Ny + 1) / 2  
B = (Nz + 1) / 2
```

因此纯朝向正面的法线向量 (0, 0, 1) 对应颜色 (0.5, 0.5, 1.0)，即法线贴图典型的偏蓝色外观。采样后必须执行反映射 `N = RGB * 2.0 - 1.0` 才能还原真实法线方向，随后参与光照计算（如Lambert漫反射 `diffuse = max(dot(N, L), 0)`）。序列帧的每一帧都是这样一张法线贴图，切换帧时等同于切换表面的凹凸朝向。

### UV偏移的帧切换机制

设法线序列帧图集为 N×M 布局（N列M行），当前帧索引为 `frame`，则采样UV偏移计算为：

```
col  = frame % N
row  = floor(frame / N)
uvOffset.x = col / N
uvOffset.y = row / M
uvScale    = float2(1.0/N, 1.0/M)
sampleUV   = inputUV * uvScale + uvOffset
```

帧率由材质中的时间参数控制：`frame = floor(Time * FPS) % (N * M)`。以常见的4×4布局16帧、24FPS为例，完整循环周期为 16/24 ≈ 0.667秒。每次UV跳变时，采样到的法线数据整体刷新，产生凹凸动态感。

### 与通道打包的结合方式

法线序列帧贴图本身已占用RGB三个通道存储法线XYZ，其Alpha通道可通过通道打包技术存储额外信息。常见方案是将法线序列帧的A通道打包粗糙度序列帧或遮罩动画——例如UE5的材质中，从同一张贴图的Alpha提取随时间变化的镜面反射强度，使高光焦散斑点与法线凹凸动画同步播放，无需额外贴图采样。注意法线XY分量重建时可从RG双通道推算Z分量（`Nz = sqrt(1 - Nx² - Ny²)`），此时可释放B通道另作他用，进一步提升通道利用率。

## 实际应用

**流体类特效（熔岩、水面波纹）**  
熔岩流效果通常使用32帧法线序列帧（8×4图集），以12FPS播放，表现熔岩表面气泡涌动的动态凹凸。点光源放置在熔岩正上方时，帧切换会使光斑随涌动节奏跳动，这种效果用静态法线贴图加UV流动无法复现，因为UV流动会导致法线方向连续插值而失去气泡"弹出"的离散感。

**爆炸冲击波特效**  
爆炸扩散时的冲击波面片常使用16帧法线序列帧配合透明度序列帧，法线数据呈现从中心向外辐射的扰动纹理。动态点光源照射时，冲击波外缘产生高光环，内部出现投影阴影，增强爆炸的立体质感。冲击波的法线序列帧制作通常由Houdini流体模拟烘焙输出，再由Substance Designer将高度图转换为切线空间法线序列帧图集。

**角色技能特殊光效（如魔法护盾）**  
护盾特效的法线序列帧模拟能量流涌动，配合场景主光源方向，护盾受攻击面产生高光冲击波纹。此类应用中法线序列帧通常不使用Alpha混合，而是叠加在角色基础法线上（法线混合公式：将两个切线空间法线Whiteout混合：`N = normalize(float3(N1.xy + N2.xy, N1.z * N2.z))`），保留角色本体表面细节。

## 常见误区

**误区一：法线序列帧等同于颜色序列帧叠加偏蓝色调**  
部分初学者将法线贴图视为"特殊颜色贴图"，直接把法线序列帧图集连接到颜色输出节点，结果渲染出蓝绿色图案，且不随光源移动而变化。根本原因是法线数据必须经过 `N = RGB * 2.0 - 1.0` 反映射和归一化处理，并输入到光照计算节点（如UE5的Normal输入槽），才能参与光照运算。

**误区二：法线序列帧播放速度可以随意提高**  
由于法线序列帧是离散帧跳变（无插值），过高帧率（如60FPS以上）会使相邻帧间法线方向剧变，高光产生闪烁噪点。实践中法线序列帧的FPS通常控制在8–24之间，依赖人眼对快速法线变化不敏感的特性形成平滑错觉；而颜色序列帧可以运行在30FPS甚至更高。若需要高帧率法线动画，应改用实时法线计算或法线流动（UV扰动法线贴图）方案。

**误区三：法线序列帧图集可与颜色序列帧共用同一图集布局**  
实践中法线图集与颜色图集的帧数和分辨率可以不同。法线贴图的精度要求高于颜色贴图（错误的法线方向直接导致光照异常），而颜色对低分辨率容忍度更高。强制将二者压入同一张图集会导致法线帧分辨率不足，出现光照块状失真。通常做法是法线序列帧使用独立贴图并开启BC5压缩格式（专为双通道法线贴图优化），颜色序列帧使用BC3/BC7格式。

## 知识关联

**前置依赖：通道打包**  
法线序列帧的制作必须掌握通道打包，原因在于切线空间法线的Z分量（B通道）在绝大多数情况下可由XY推算，空余的B通道和A通道通过打包技术承载粗糙度、遮罩等附加数据，将四张序列帧贴图的信息压缩进一张RGBA图集，是移动端特效优化的标准做法。

**后续概念：自发光序列帧**  
掌握法线序列帧后，自然延伸到自发光序列帧的学习。自发光序列帧存储的是HDR亮度信息而非法线方向，其UV采样和帧切换机制与法线序列帧完全相同，但输出目标是材质的Emissive节点而非Normal节点，且无需光照计算——这使自发光序列帧的调试和理解比法线序列帧更直观。两者在同一特效资产中常常组合使用：法线序列帧提供受光面的动态凹凸，自发光序列帧叠加不依赖外部光源的固有发光动画。
