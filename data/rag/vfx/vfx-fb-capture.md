# 序列帧捕获

## 概述

序列帧捕获（Flipbook Capture / Sequence Frame Capture）是指将三维模拟软件或合成软件中的动态效果，以固定时间间隔逐帧渲染并导出为有序编号图像文件集合的工作流程。区别于实时渲染，序列帧捕获的本质是将高计算成本的模拟结果"预烘焙"为静态图像序列，供游戏引擎或实时渲染管线以贴图形式回放。典型的输出格式包括 PNG 序列、EXR 序列，以及将多帧排列在单张纹理上的 Sprite Sheet（图集）。

该技术在游戏特效流程中的大规模普及始于 2010 年前后，随着 Houdini 的普及和 UE4/Unity 的粒子系统对 Flipbook 贴图原生支持，序列帧捕获成为流体、烟雾、爆炸类特效的标准制作手段。Epic Games 在 2018 年的 GDC 演讲中专门介绍了基于 Houdini Flipbook 导出的 Niagara 特效工作流，使该流程获得了行业级规范。

序列帧捕获之所以重要，是因为它将物理精确的 Pyro 烟雾或 FLIP 流体模拟"锁定"为可在移动端或主机平台实时播放的纹理资产，解决了复杂模拟与实时性能之间的根本矛盾。一张 8×8 帧、分辨率 2048×2048 的图集可以存储 64 帧动画，在 GPU 上的采样成本仅相当于一次普通贴图采样。

## 核心原理

### 相机设置与正交投影规范

序列帧捕获通常使用**正交相机（Orthographic Camera）**而非透视相机，以确保所有帧的像素比例一致，避免近大远小导致的动画贴图在粒子上播放时产生形变。在 Houdini 的 Flipbook 设置中，正交相机的 `orthowidth` 参数需要覆盖效果的最大横向尺寸，建议设置为模拟包围盒宽度的 1.2 倍以留出边距。渲染分辨率通常选择 **512×512、1024×1024 或 2048×2048** 的正方形，以适配后续图集拼合的均匀网格布局。

相机朝向通常锁定为 **Y 轴向上、-Z 方向朝场景**，即从正前方拍摄效果。对于需要多角度播放的效果（如体积烟雾），则需要捕获 8 个或 16 个均匀分布的水平角度，每个角度独立输出一套序列，再合并至同一图集的不同行区域，这种方案被称为 Impostor 多角度图集，与单方向的 Flipbook 图集在捕获流程上存在本质差异。

### 帧数、帧率与时长的三角关系

序列帧捕获需要明确三个参数：**总帧数（Frame Count）、帧率（FPS）、图集网格尺寸（Grid Size）**。三者满足以下关系：

```
总帧数 = 列数 × 行数
动画时长（秒）= 总帧数 ÷ 播放帧率
```

例如一个 8×8 图集共 64 帧，若在引擎中以 24 FPS 播放，则动画总时长约为 **2.67 秒**。模拟的实际时长必须与此对齐，即 Houdini 中的时间线需覆盖恰好 64 帧的有效动画内容。若模拟帧率为 24 FPS 而捕获帧率为 12 FPS，需要在 Houdini 的 Flipbook 设置中将 `Frame Range` 步进设为 2，以实现等间隔抽帧，而不能直接截取前 64 帧。

常用图集规格对照：
- **4×4 = 16 帧**：适合短促爆炸、击中火花
- **8×8 = 64 帧**：适合烟雾上升、火焰燃烧
- **8×16 = 128 帧**：适合需要更长循环的环境特效

### 通道内容与 EXR 多通道导出

序列帧捕获不仅捕获 RGB 颜色，还需同步捕获**运动矢量（Motion Vector）**通道用于帧间插值，以及 **Alpha 遮罩**通道控制透明度。在 Houdini 的 Karma 或 Mantra 渲染器中，可通过配置 `Extra Image Planes` 将 `Cf`（颜色）、`Af`（Alpha）、`motionvector`（运动矢量）同时输出至单个 **16-bit EXR 文件**的独立通道中。

运动矢量通道的值域为 [-1, 1]，在导出时需重映射至 [0, 1] 以适配 PNG 的 8-bit 存储，具体公式为：

```
存储值 = (运动矢量 + 1) × 0.5
```

若省略运动矢量通道，图集帧率较低时（如 12 FPS 以下）会出现明显的"跳帧"卡顿感，而有了运动矢量，引擎材质可在相邻两帧之间进行像素级插值，使 12 帧的图集达到接近 30 帧的视觉流畅度。

## 实际应用

**Houdini Flipbook 工具**是序列帧捕获最常用的路径。在 Houdini 的 Scene View 视口中，执行 `Render > Flipbook` 即可调出捕获面板，设置分辨率、帧范围与输出路径。文件命名须遵循 `filename.$F4.png` 的四位帧号格式（如 `smoke_0001.png`），确保操作系统按正确顺序读取文件。捕获完成后，使用 Houdini 的 **Labs Flipbook Textures SOP** 节点可自动完成序列图像到图集的拼合，并输出引擎可识别的 JSON 元数据文件记录行列数量。

**After Effects** 的序列帧捕获常用于 2D 手绘风格特效或基于 CC Particle World 的粒子效果。导出时选择 `文件 > 导出 > 添加到渲染队列`，输出模块设置为 `PNG 序列`，帧率锁定为目标帧率。AE 导出的序列命名默认包含五位数字（如 `comp_00001.png`），在 Photoshop 的 **Image Processor** 或 TexturePacker 中拼合图集时需注意排序一致性。

在 UE5 的 Niagara 系统中，捕获好的图集贴图需要设置 **Sub UVAnimation** 模块，填入与图集完全一致的行数和列数。若此处填写错误（例如将 8×8 图集误填为 4×16），粒子播放时每帧采样的 UV 坐标将错位，出现画面撕裂或帧序混乱的问题。

## 常见误区

**误区一：用透视相机捕获导致边缘形变**
部分初学者直接使用场景主相机（透视相机）进行 Flipbook 捕获，导致效果靠近相机时像素放大、远离时缩小。在粒子系统中播放时，这种透视畸变无法被相机补偿，最终呈现出贴图"呼吸"的视觉错误。正确做法是专门创建一个正交相机节点，绑定至效果中心并调整 `orthowidth` 覆盖完整包围盒。

**误区二：图集帧数超过总帧数导致空白帧**
若模拟只有 50 帧有效内容，却强行使用 8×8=64 帧的图集，最后 14 个格子将输出全黑帧。在引擎中循环播放时，这些黑帧会导致每次循环末尾出现短暂的"闪黑"现象。解决方案是要么将模拟延长至恰好 64 帧，要么改用 **7×8=56 帧**（但非正方形图集需引擎材质额外支持），或对最后几帧进行静帧填充（Hold Last Frame）。

**误区三：将运动矢量通道混入颜色通道**
在 Houdini 的多通道 EXR 工作流中，若错误地将运动矢量数据输出至 RGB 通道而非独立的自定义 AOV，会导致颜色通道被矢量数据污染，最终图集的颜色呈现为紫色或青色偏移。检查方法是在 Houdini 的 MPlay 中单独查看各通道，确认 `motionvector` 通道的值域分布在 0.5 附近（对应静止区域的零矢量重映射值）。

## 知识关联

序列帧捕获对**运动矢量**的依赖体现在：捕获阶段输出的运动矢量 AOV 是后续帧间插值的数据基础，若运动矢量通道精度不足（如压缩为 8-bit 而非 16-bit），插值结果会出现条带状噪点。理解运动矢量的编码规范（将方向向量重映射至 0-1 区间）是正确配置多通道 EXR 导出的前提。
