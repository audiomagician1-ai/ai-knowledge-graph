# LLM幻觉与事实性

## 概述
大语言模型(LLM)的幻觉(Hallucination)指模型生成的内容看起来流畅合理但实际上不正确或无法验证。这是LLM最大的可靠性挑战之一。

## 核心概念

### 幻觉的类型
1. **事实性幻觉(Factual Hallucination)**: 模型编造不存在的事实、数据或引用
2. **逻辑性幻觉(Logical Hallucination)**: 推理链中出现逻辑跳跃或矛盾
3. **忠实性幻觉(Faithfulness Hallucination)**: 模型输出与给定上下文不一致

### 幻觉产生的原因
- **训练数据限制**: 知识截止日期后的信息、训练集中的错误信息
- **概率建模本质**: LLM优化的是下一个token的概率分布而非事实准确性
- **过度自信(Overconfidence)**: 模型对不确定的内容仍生成确定性的回答
- **上下文窗口限制**: 长文本处理中遗漏关键信息

### 检测方法
1. **基于参考的检测**: 与知识库/搜索引擎事实核查
2. **自我一致性检测**: 多次生成比较一致性(SelfCheckGPT)
3. **不确定性估计**: 通过token概率分布评估置信度
4. **NLI-based方法**: 自然语言推理模型检查文本自洽性

### 缓解策略
- **检索增强生成(RAG)**: 提供外部知识源减少知识边界幻觉
- **思维链(CoT)推理**: 逐步推理提高逻辑一致性
- **RLHF对齐训练**: 训练模型在不确定时说"我不知道"
- **置信度校准**: 输出不确定性分数
- **事实性微调**: 使用事实对齐的数据集进行专门训练

## 评估指标
- **FActScore**: 细粒度原子事实验证得分
- **HaluEval**: 综合幻觉评估基准
- **TruthfulQA**: 评估模型如实回答的能力

## 实践建议
- 在关键应用中组合RAG+事实核查
- 提示模型标注不确定性("我不确定, 但...")
- 使用结构化输出约束模型行为
