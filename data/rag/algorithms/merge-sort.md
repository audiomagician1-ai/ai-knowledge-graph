# 归并排序

## 概述

归并排序（Merge Sort）是一种基于**分治策略**的比较排序算法，由约翰·冯·诺依曼（John von Neumann）于1945年发明。其核心思想是将一个待排序数组反复二分，直到每个子数组只剩1个元素（单元素数组天然有序），再逐层将相邻的有序子数组**合并**成更大的有序数组，最终完成整体排序。

归并排序的时间复杂度在最坏、最好、平均情况下均为 **O(n log n)**，这一性质使它优于冒泡排序和插入排序的 O(n²) 上界，且不受输入数据分布的影响。与快速排序不同，归并排序是**稳定排序**，即相等元素在排序前后的相对顺序保持不变，这在需要多字段排序（如先按姓氏、再按名字）的场景中至关重要。

在AI工程领域，归并排序常用于外部排序（External Sort）场景，例如对无法完整载入内存的大型训练数据集进行预处理排序。其可预测的 O(n log n) 性能和稳定性，使它成为需要确定性行为的数据管道中的首选算法。

## 核心原理

### 分解阶段：递归二分

归并排序将长度为 n 的数组从中间位置 `mid = (left + right) / 2` 切分为两半，对左子数组 `[left, mid]` 和右子数组 `[mid+1, right]` 分别递归调用自身。递归的**终止条件**是子数组长度为1（即 `left >= right`）。整个分解过程形成一棵深度为 ⌈log₂n⌉ 的二叉递归树。当 n = 8 时，递归树深度为3，共需经历3层分解。

### 合并阶段：双指针归并

合并（Merge）是归并排序的关键操作。给定两个已排序的子数组 L 和 R，使用两个指针 `i`、`j` 分别指向 L 和 R 的当前元素，每次将较小的元素写入辅助数组，指针前移，直到某一子数组耗尽，再将另一子数组的剩余元素全部拷贝。该操作的时间复杂度为 **O(k)**，其中 k 为两子数组的元素总数。

合并函数伪代码要点：
```
merge(arr, left, mid, right):
    创建辅助数组 temp[left..right]
    i = left, j = mid+1, k = left
    while i <= mid and j <= right:
        if arr[i] <= arr[j]: temp[k++] = arr[i++]
        else: temp[k++] = arr[j++]
    拷贝剩余元素
    将 temp 写回 arr[left..right]
```

### 空间复杂度：O(n) 辅助空间

归并排序需要与原数组等长的辅助空间来完成合并操作，因此空间复杂度为 **O(n)**。这是归并排序相对于原地排序（如堆排序）的主要缺点。在嵌入式设备或内存受限环境中，这一额外开销可能不可接受。存在一种原地归并排序变体，但其实现复杂度极高，实际使用极少。

### 递推关系与主定理

归并排序满足递推式：**T(n) = 2T(n/2) + O(n)**。应用主定理（Master Theorem），其中 a=2、b=2、f(n)=O(n)，满足第二种情形（n^(log_b a) = n^1 = n，与 f(n) 同阶），因此 T(n) = **O(n log n)**。

## 实际应用

**大规模训练数据预处理**：在机器学习数据管道中，若需对包含亿级样本的 CSV 文件按时间戳排序，单机内存不足以加载全部数据。此时使用**外部归并排序**：先将文件分块读入内存、块内排序后写入磁盘（产生多个有序的"归并段"），再多路归并这些段，最终输出全局有序文件。这一流程直接利用归并排序天然的分块合并结构。

**Python 标准库 Timsort**：Python 的 `list.sort()` 和 `sorted()` 函数底层使用 **Timsort** 算法（由 Tim Peters 于2002年实现），其核心是对归并排序的优化——识别数组中已存在的自然有序子序列（称为"run"），然后用归并排序将这些 run 合并。Java 的 `Arrays.sort()` 对对象数组也采用同样策略。

**多路归并（K-way Merge）**：在分布式排序场景中，K 台机器各自产出一段有序数据，协调节点使用**最小堆辅助的 K 路归并**，每次从堆顶取出最小元素，时间复杂度为 O(n log K)。这是 MapReduce Shuffle 阶段合并排序文件的基础机制。

## 常见误区

**误区一：归并排序比快速排序更快**
许多初学者看到归并排序的 O(n log n) 最坏情况优于快速排序的 O(n²)，便误以为归并排序实际更快。事实上，快速排序的**常数因子更小**，缓存局部性更好，在随机输入下平均比归并排序快约30%～40%。归并排序的优势在于**最坏情况保证**和**稳定性**，而非绝对速度。

**误区二：辅助数组必须在每次 merge 调用中重新分配**
低效实现常在每次 `merge()` 调用时动态创建临时数组，导致大量内存分配开销。正确做法是在排序开始时**一次性分配与原数组等长的辅助数组**，整个递归过程复用它，将内存分配次数从 O(n) 次减少到1次。

**误区三：归并排序对链表无优势**
实际上，归并排序特别适合**单链表排序**。链表的随机访问代价高（O(n)），快速排序需要随机访问pivot，效率很低。归并排序只需顺序遍历即可完成合并，且对链表无需辅助空间（直接重新链接节点），空间复杂度可降为 O(log n)（递归栈），是链表排序的首选算法。

## 知识关联

**前置概念**：归并排序的实现直接依赖**递归**机制——每次 `mergeSort(left, mid)` 和 `mergeSort(mid+1, right)` 均是对同一函数的递归调用，理解递归的调用栈与终止条件是读懂归并排序代码的前提。**分治法**是其设计范式：分解（Divide）对应二分，解决（Conquer）对应递归排序子数组，合并（Combine）对应 merge 操作，三步缺一不可。

**后续概念**：学习**快速排序**时，可与归并排序做直接对比。快速排序同样是分治算法，但其"工作量"集中在**分解阶段**（Partition），合并阶段是空操作；归并排序则相反，分解是简单二分，工作量集中在**合并阶段**。这一对比揭示了分治算法设计中"分解难度"与"合并难度"之间的权衡关系，也解释了为何快速排序原地运行而归并排序需要辅助空间。
