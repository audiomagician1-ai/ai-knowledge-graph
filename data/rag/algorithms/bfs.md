# 广度优先搜索（BFS）

## 概述

广度优先搜索（Breadth-First Search，BFS）是一种以**层级扩展**为特征的图遍历算法：从起始节点出发，先访问所有距离为1的邻居节点，再访问所有距离为2的节点，依此类推，形成同心圆式的扩散模式。BFS由Konrad Zuse在1945年首次描述于其博士论文中，后经Edward F. Moore于1959年在迷宫最短路径问题中正式引入，并由C. Y. Lee于1961年独立重新发现并应用于电路布线问题。

BFS的核心数据结构是**FIFO队列**（先进先出队列），而非深度优先搜索使用的栈或递归调用栈。这一选择直接决定了它的遍历顺序——先入队的节点先被处理，从而保证了"按层遍历"的行为。BFS对无权图具有一个极为重要的性质：当所有边的权重相等时，BFS找到的从起点到任意可达节点的路径，**保证是边数最少的路径**，即最短路径。

在AI工程领域，BFS是无信息搜索（Uninformed Search）策略的典型代表，常作为基准算法与启发式搜索方法（如A*）进行对比。理解BFS的时间复杂度、空间复杂度以及完备性保证，是选择搜索算法的基础判断依据。

## 核心原理

### 算法执行流程

BFS的标准实现步骤如下：

1. 将起始节点 `s` 加入队列，并标记为"已访问"
2. 从队列头部取出节点 `u`，访问 `u` 的所有未访问邻居节点
3. 将每个未访问邻居标记为"已访问"后加入队列尾部
4. 重复步骤2-3，直到队列为空

"已访问"标记必须在**入队时**而非出队时设置，否则同一节点可能被重复入队，导致指数级的冗余计算。以下是Python风格的伪代码框架：

```
def BFS(graph, start):
    visited = {start}
    queue = deque([start])
    while queue:
        node = queue.popleft()
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
```

### 时间与空间复杂度

设图中顶点数为 `V`，边数为 `E`，BFS的时间复杂度为 **O(V + E)**——每个顶点入队出队各一次（O(V)），每条边最多被检查两次（O(E)）。

BFS的空间复杂度是 **O(W)**，其中 `W` 是图的最大宽度（即层级中节点数量的最大值）。在最坏情况下（如完全图或宽度极大的树），W可达 O(V)，这意味着BFS在宽而浅的图中内存消耗远高于DFS。对于分支因子为 `b`、目标深度为 `d` 的树形搜索空间，BFS需要存储 **O(b^d)** 个节点，当 `b=10, d=6` 时即需存储约100万个节点。

### BFS的完备性与最优性

BFS具备两个重要的理论保证：

- **完备性（Completeness）**：只要目标节点存在于有限图中，BFS一定能找到它。
- **最优性（Optimality）**：在所有边权重相同（或均为单位权重）的图上，BFS找到的路径边数最少，即为最优解。

当边权重不等时，BFS的最优性**不再成立**。例如对于边权为 `(A→B: 1, B→C: 1, A→C: 10)` 的图，BFS会认为 `A→C` 的直接路径（1条边）优于 `A→B→C`（2条边），但实际总权重恰好相反。这正是Dijkstra算法诞生的动机。

### BFS生成树与层级编号

BFS执行过程中隐式构建了一棵**BFS树（BFS Tree）**，树中每个节点到根节点的路径即为原图中的最短路径。记录每个节点的**层级编号 `d[v]`**（即到起点的最短边数），可以用公式表达：

> `d[s] = 0`；若 `u` 是 `v` 在BFS树中的父节点，则 `d[v] = d[u] + 1`

## 实际应用

**社交网络的六度分隔验证**：Facebook曾使用BFS对其社交图进行分析，2016年的研究数据显示平均最短路径约为3.57跳。BFS可直接计算任意两用户间的最短连接度数。

**网络爬虫的层级爬取策略**：搜索引擎爬虫使用BFS从种子URL出发，按层级抓取网页链接。第1层是种子页面的直接链接，第2层是第1层页面的链接，依此扩展，可控制爬取深度以限制规模。

**棋盘与迷宫的最优步数求解**：在"骑士巡游"（Knight's Tour）或"华容道"等问题中，将每种棋盘状态作为图节点，BFS可保证找到从初始状态到目标状态的最少步数路径。

**Android/iOS UI层级分析**：移动端UI测试框架使用BFS遍历View树，按层级顺序检查所有界面元素，确保父节点在子节点之前被检查到。

**多源BFS（Multi-Source BFS）**：将多个起点同时加入初始队列，可在 O(V + E) 时间内计算图中所有节点到最近源节点的距离，常用于"最近出口""最近加油站"等问题。

## 常见误区

**误区一：认为BFS总能找到最短路径**

BFS的最短路径保证仅适用于**无权图或等权图**。对于加权图，即使BFS找到了边数最少的路径，该路径的总权重也可能不是最小的。将BFS用于带权最短路径问题会得到错误答案，必须改用Dijkstra算法（基于优先队列而非普通队列）。

**误区二：已访问标记在出队时设置**

许多初学者在节点出队时才标记"已访问"，而非入队时。这会导致同一节点被多次加入队列——在稠密图中，某个节点可能被其多个邻居反复入队，将时间复杂度从 O(V + E) 退化至 O(V × E)，在大图上产生严重的性能问题。

**误区三：BFS适用于所有无信息搜索场景**

BFS的 O(b^d) 空间复杂度使其在深度较大时极不实用。当搜索深度 `d` 很大而解路径较深时，**迭代加深深度优先搜索（IDDFS）**在时间复杂度相同（O(b^d)）的同时，空间复杂度仅为 O(d)，远优于BFS。盲目使用BFS而不考虑搜索空间的宽深比，会导致内存溢出。

## 知识关联

**前置知识衔接**：BFS的运行依赖两个数据结构。**图（邻接表或邻接矩阵）**定义了节点与边的存储方式，决定了遍历邻居的时间开销（邻接表为 O(degree)，邻接矩阵为 O(V)）。**队列的FIFO特性**是BFS按层扩展的直接机制——将队列替换为栈即得到迭代版DFS，这一对比直观展示了两种数据结构如何产生截然不同的遍历顺序。

**向后延伸**：**Dijkstra最短路径算法**可理解为"使用优先队列（最小堆）替代普通队列的BFS"——将固定步长的层级扩展改为按累积权重排序的扩展，使最优性从等权图推广到任意非负权图。**深度优先搜索（DFS）**与BFS互为对照：DFS使用栈、空间效率高（O(V)）但不保证最短路径，适合拓扑排序和环检测等场景。**A\*搜索算法**在BFS的层级扩展思想上引入启发函数 `h(n)`，通过 `f(n) = g(n) + h(n)` 优先扩展"最有希望"的节点，是BFS从无信息搜索到有信息搜索的核心演进路径。
