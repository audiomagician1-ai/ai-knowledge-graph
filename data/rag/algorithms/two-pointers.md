# 双指针

## 概述

双指针（Two Pointers）是一种使用两个索引变量同时遍历数组或链表的算法技巧，通过控制两个指针的移动方向和速度，将原本需要 O(n²) 的暴力枚举降低至 O(n) 或 O(n log n) 的时间复杂度。两个指针可以从同一端出发（快慢指针），也可以从两端向中间收拢（对撞指针），具体选择取决于问题的约束条件。

双指针技巧最早在算法竞赛领域被系统总结，与荷兰国旗问题（Dijkstra 1976年提出）密切相关——该问题的三路划分解法正是双指针思想的经典体现。LeetCode 平台上标注为"Two Pointers"的题目超过 150 道，是面试高频考点之一，也是滑动窗口、快速排序分区等高级技巧的直接前身。

双指针的核心价值在于利用数据的有序性或特定约束消除冗余计算。以在有序数组中寻找两数之和为例：暴力枚举需要检查所有 C(n,2) 个对，而对撞指针每一步都能根据当前和与目标值的大小关系确定性地排除至少一个元素，使总操作次数严格不超过 n 次。

## 核心原理

### 对撞指针（左右指针）

对撞指针将左指针 `left` 初始化为 0，右指针 `right` 初始化为 `n-1`，两者向中间移动直到相遇。其正确性依赖于**单调性**：当 `arr[left] + arr[right] > target` 时，右移左指针只会让和变大，因此必须左移右指针；反之右移左指针。每次操作必然使搜索空间缩小至少 1，故循环至多执行 n 次。

典型场景：**两数之和（有序数组）**。给定升序数组 `nums` 和目标值 `target`，对撞指针的判断逻辑为：
- `nums[left] + nums[right] == target`：返回 `[left, right]`
- `nums[left] + nums[right] < target`：`left++`
- `nums[left] + nums[right] > target`：`right--`

时间复杂度 O(n)，空间复杂度 O(1)，相比哈希表解法节省了 O(n) 的额外空间。

### 快慢指针（同向指针）

快慢指针中，快指针 `fast` 每步前进更快或负责"探路"，慢指针 `slow` 负责"记录"合法位置。两者始终同向移动，不会出现交叉。

**原地去重**是最典型的快慢指针应用：`slow` 指向已处理区间的末尾，`fast` 扫描新元素，当 `nums[fast] != nums[slow]` 时执行 `nums[++slow] = nums[fast]`。对于长度为 n 的数组，fast 遍历一次即完成，时间复杂度严格为 O(n)。

**链表找环**（Floyd 判圈算法）也属于快慢指针：令 fast 每次走 2 步，slow 每次走 1 步。若链表有环，fast 和 slow 必然在环内相遇，相遇时 fast 走过的路程是 slow 的 2 倍，利用此等式可以精确计算环的入口节点位置。

### 指针移动的合法性证明

双指针正确性的核心在于每次指针移动都是"无损"的——被跳过的候选组合可以被证明不满足条件。以三数之和（LeetCode 15）为例：外层固定一个数 `nums[i]`，内层用对撞指针在 `[i+1, n-1]` 区间寻找剩余两数。当遇到重复元素时需要跳过，因此循环中包含 `while (left < right && nums[left] == nums[left+1]) left++` 的去重逻辑，这一步骤不会导致遗漏，因为相同值的左指针产生的所有组合已经在第一次匹配时被记录。

## 实际应用

**容器盛水问题（LeetCode 11）**：给定 n 个高度值，对撞指针从两端向内收缩，每次移动**较矮**一侧的指针。理由是当前容积受限于较矮一侧，移动较高一侧只会减少宽度且不增加高度，必然不优。此贪心论证使对撞指针在 O(n) 内得出全局最优解。

**AI工程中的特征相似度计算**：在向量检索预处理阶段，对已按 L2 范数排序的候选向量列表，可用对撞指针快速过滤范数之和不满足阈值条件的向量对，将两两比较的 O(n²) 降为 O(n)，显著加速批量余弦相似度的候选集剪枝。

**数据管道的滑动比对**：在 ETL 流程中比对两个时间戳有序的日志序列时，双指针分别扫描两个序列，根据时间戳大小决定哪个指针前进，实现 O(m+n) 的归并式对齐，而非 O(m×n) 的嵌套循环匹配。

## 常见误区

**误区一：对无序数组直接套用对撞指针。** 对撞指针的移动决策依赖数组的单调性，若数组未排序，`arr[left] + arr[right] < target` 时右移左指针并不能保证和变大，算法会给出错误结果。正确做法是先排序（O(n log n)）再使用对撞指针，或改用哈希表方案。

**误区二：快慢指针速度比固定为 2:1。** Floyd 判圈算法中 fast 走 2 步是为了保证数学性质（相遇点到环入口的距离等于头节点到环入口的距离），但在"找链表中点"场景中同样是 2:1；而在某些需要间距恰好为 k 的题目（如找倒数第 k 个节点）中，应先让 fast 领先 k 步再同速移动，速度比是 1:1。混淆这两种模式会导致错误的指针定位。

**误区三：认为双指针只适用于数组。** 双指针同样适用于链表（Floyd 判圈）、字符串（回文判断）以及两个独立序列的归并比对。约束不在于数据结构，而在于是否存在可利用的单调性或顺序约束，缺少这一前提时双指针无法给出比暴力更优的复杂度。

## 知识关联

**前置知识——数组/列表**：双指针操作的合法性建立在对数组下标随机访问为 O(1) 的基础上；若底层结构是链表，则需要额外的"提前 k 步"技巧来弥补链表不支持随机访问的缺陷。理解数组的连续内存布局有助于分析双指针的缓存友好性。

**后续方向——滑动窗口**：滑动窗口可视为双指针的特化形式，其中右指针负责扩张窗口、左指针负责收缩窗口，两者始终保持 `left ≤ right` 且窗口内维护某个动态统计量（如字符频率、元素之和）。从双指针到滑动窗口的跨越在于引入了窗口内的状态维护逻辑，时间复杂度同样保持 O(n)。

**后续方向——单调栈**：当问题不再是两端收缩或同向扫描，而是需要在扫描过程中动态查询"前方最近的更大元素"时，双指针无法直接处理，需要升级为单调栈。两者共同的出发点都是通过某种结构化遍历避免暴力枚举，区别在于单调栈维护的是一个动态的候选集合，而非简单的两个索引。
