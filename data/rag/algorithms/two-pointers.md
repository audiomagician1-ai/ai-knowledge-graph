# 双指针

## 概述

双指针（Two Pointers）是一种通过维护两个索引位置同时遍历数组或链表的算法技巧，本质是将某些暴力O(n²)搜索问题压缩到O(n)时间复杂度。两个指针可以从同一端出发向同一方向移动（快慢指针），也可以从两端向中间靠拢（对撞指针），具体选择取决于问题结构。

双指针作为独立技术在1970年代的排序与搜索研究中逐渐被形式化。"Two Sum II"问题是教科书级经典案例：给定一个**已排序**数组，找出两数之和等于目标值的下标对。暴力枚举需要O(n²)次比较，而双指针仅用O(n)次即可完成，核心利用的是排序后数组的单调性。

双指针在AI工程的数据预处理、特征工程、滑动批量采样等场景中高频出现。例如在NLP的序列标注任务中，用双指针从原始文本中提取固定长度上下文窗口；在时序数据清洗中，用快慢指针原地去除重复样本点。

## 核心原理

### 对撞指针（两端向中间）

对撞指针将 `left` 初始化为 `0`，`right` 初始化为 `n-1`，每次根据当前两指针元素与目标值的大小关系决定移动哪一侧：

```
while left < right:
    current_sum = arr[left] + arr[right]
    if current_sum == target:   # 找到答案
        return (left, right)
    elif current_sum < target:
        left += 1               # 需要更大的值
    else:
        right -= 1              # 需要更小的值
```

**正确性依赖前提**：数组必须有序。如果数组无序直接使用对撞指针，跳过的元素对可能包含正确答案，导致漏解。

### 快慢指针（同向移动）

快慢指针中 `fast` 每次移动步数大于或等于 `slow`，常见比例是 `fast` 每轮走2步、`slow` 走1步。Floyd判环算法（1967年）是最著名的快慢指针应用：若链表中存在环，快指针必定在环内追上慢指针。数学上，设环长为 `c`，非环部分长为 `F`，相遇时慢指针走了 `F + a` 步，快指针走了 `F + a + n·c` 步（n为整数），由 `2(F+a) = F+a+n·c` 得 `F = n·c - a`，这一等式可进一步用于定位环入口。

### 同向双指针（去重/压缩）

对有序数组原地去重时，`slow` 指向已处理区域末尾，`fast` 向前扫描：

```
slow = 0
for fast in range(1, len(arr)):
    if arr[fast] != arr[slow]:
        slow += 1
        arr[slow] = arr[fast]
return slow + 1   # 去重后数组长度
```

整个操作空间复杂度为O(1)，因为没有分配额外数组，只是原地交换写入位置。

### 时间复杂度分析

无论哪种双指针变体，每个元素最多被 `left` 或 `right` 访问**各一次**，因此总操作次数上界为 `2n`，时间复杂度为 **O(n)**。前提是每次指针移动方向不可逆——一旦 `left` 右移，就不会再左移回来，这是双指针能将O(n²)降为O(n)的本质约束。

## 实际应用

**1. 三数之和（3Sum）**
先排序（O(n log n)），固定第一个数 `nums[i]`，再对 `[i+1, n-1]` 区间使用对撞指针寻找两数之和等于 `-nums[i]`。外层遍历O(n)，内层指针O(n)，总复杂度O(n²)，远优于三重循环的O(n³)。

**2. 链表中点查找**
快指针每次走2步，慢指针每次走1步，当快指针到达末尾时，慢指针恰好在链表中点。在图神经网络（GNN）的链表形邻接表处理中，此技巧用于将链表均等切分为两段以便并行处理。

**3. 容器盛水问题（LeetCode 11）**
给定高度数组 `height`，两端设指针，每次移动**较矮一侧**的指针。移动较高一侧不可能使面积增大，因为宽度减小而高度受限于较矮侧。该贪心决策严格依赖对撞指针的单调收缩性质。

**4. 数据预处理中的滑动采样**
在机器学习训练数据生成中，用两个指针维护一个合法样本窗口：`right` 向右扩展直到窗口不合法，然后 `left` 右移收缩。这是滑动窗口的直接前身，所有样本恰好被添加和删除各一次，总时间O(n)。

## 常见误区

**误区1：对无序数组直接使用对撞指针**
Two Sum原题（LeetCode 1）的输入是无序数组，不能直接套对撞指针。必须先用哈希表O(n)或先排序再用指针O(n log n)。很多初学者见到"找两数之和"就上对撞指针，忽略了"有序"这一不可缺少的前提条件。

**误区2：认为快慢指针的步长比必须是2:1**
Floyd判环算法使用2:1是因为它保证在O(n)步内相遇，但判断链表是否有环**只需快指针比慢指针快即可**，步长比3:1或4:1同样成立，只是相遇位置的数学关系会改变，定位环入口的计算需要重新推导。

**误区3：双指针可替代所有两层循环**
双指针能优化的两层循环必须具备**单调决策性**：移动一个指针之后，另一指针的搜索范围必须单调收缩或扩展。若内外层没有这种单调关联（如矩阵按行列独立搜索），双指针无法适用，强行使用会产生错误答案。

## 知识关联

**前置基础**：双指针操作的对象是数组或链表，必须熟悉下标访问（`arr[i]`）、链表节点的 `next` 引用，以及数组排序的基本语义。若不理解有序数组的单调性，无法理解对撞指针移动方向的贪心依据。

**通向滑动窗口**：滑动窗口是同向双指针的直接扩展——`right` 持续扩张窗口，`left` 在窗口违反约束时收缩。双指针要求指针移动不可逆，滑动窗口继承并强化了这一性质，额外引入了窗口内部的聚合状态维护（如窗口元素之和或字符频次哈希表）。

**通向单调栈**：某些双指针能解的问题（如接雨水）在变形后需要处理"每个元素左右两侧最近的更大值"，此时双指针的线性扫描不足以捕捉非对称局部极值，必须借助单调栈在O(n)内维护一个始终保持单调递增或递减的辅助结构。
