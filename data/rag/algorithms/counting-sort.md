# 计数排序

## 概述

计数排序（Counting Sort）是一种非比较型整数排序算法，其核心思想是统计每个元素出现的次数，而不是通过元素间的大小比较来确定顺序。由于完全绕开了比较操作，它突破了基于比较的排序算法 O(n log n) 的下界，在特定条件下可以实现线性时间 O(n + k) 的排序，其中 k 是数据的值域范围。

该算法由Harold H. Seward于1954年提出，最初应用于整数序列的高效处理。其基本假设是：输入数据为非负整数，且数值范围 k 相对于数据量 n 不过分巨大。例如，对一组范围在 0 到 100 之间的10万个成绩数据排序，计数排序远比快速排序更高效。

计数排序在AI工程中的价值体现在特征预处理与数据分桶环节：当需要对离散类别编码（如用户ID、标签索引）进行大规模排序或频率统计时，计数排序提供了一种无须构建比较树的高效路径，同时它还是基数排序（Radix Sort）的底层子过程。

## 核心原理

### 三阶段执行流程

计数排序的标准实现分为三个明确阶段：

**第一阶段——计数统计：** 创建大小为 k+1 的计数数组 C，遍历输入数组 A，对每个元素 A[i] 执行 `C[A[i]] += 1`，时间复杂度 O(n)。

**第二阶段——累加前缀和：** 将计数数组转换为前缀累加数组，执行 `C[i] = C[i] + C[i-1]`，使 C[j] 表示"值 ≤ j 的元素共有多少个"，时间复杂度 O(k)。

**第三阶段——反向填充：** 创建输出数组 B，从右向左遍历输入数组 A，对每个 A[i]，将其放入 B[C[A[i]] - 1] 的位置，并执行 `C[A[i]] -= 1`，时间复杂度 O(n)。

反向遍历这一细节至关重要：它保证了算法的**稳定性**，即相同元素在输出中保持原有相对顺序，这是计数排序能作为基数排序子例程的前提。

### 时间与空间复杂度分析

| 场景 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 最优/平均/最差 | O(n + k) | O(n + k) |

这里 k = max(A) - min(A) + 1，即值域宽度。当 k = O(n) 时，总复杂度退化为 O(n)，实现真正的线性排序。但当 k >> n 时，例如对5个元素排序但值域横跨0到10亿，k 值将完全主导复杂度，此时计数排序不再适用。

### 稳定性的数学保证

稳定性来源于第三阶段的逆序扫描。设输入数组中有两个相同值 v，分别在位置 i < j，逆序扫描时 j 先被处理并放置在 B[C[v]-1]，然后 C[v] 自减，i 被处理时放入 B[C[v]-1] 即更靠前的位置，最终 i 对应元素排在 j 之前，与原数组顺序一致。

## 实际应用

### 应用场景一：考试成绩排序

100名学生成绩范围0-100，n=100，k=101，总操作约201次，远低于快速排序的约664次（n log₂ n ≈ 664）。计数数组直接建立频率直方图，额外获得每个分数段的人数分布，是成绩管理系统的常见选择。

### 应用场景二：AI推荐系统的物品ID排序

电商推荐模型中，用户行为日志包含数十万条记录，物品ID为整数且范围有限（如1到50000）。对用户交互序列按物品ID排序后进行批量嵌入查找时，计数排序可将 IO 访问模式从随机变为顺序，显著提升缓存命中率。

### 应用场景三：基数排序的子过程

对32位整数使用基数排序时，通常拆分为4轮计数排序，每轮处理8位（k=256）。每轮计数排序的稳定性保证了低位排序结果不会被高位排序破坏，最终实现 O(4n) = O(n) 的完整整数排序。

### 应用场景四：NLP词频统计后的索引重建

词表大小为30000的分词器在对 token ID 序列排序时，若直接使用计数排序并记录频次，可在一次 O(n) 扫描中同时完成排序与频率统计，无需额外的哈希表。

## 常见误区

### 误区一：认为计数排序"总是"比快速排序快

计数排序的性能严重依赖 k 与 n 的比值。当对1000个范围在0到10⁹之间的整数排序时，需要分配约4GB的计数数组（每个整数4字节），这在内存上完全不可行，实际运行速度远慢于快速排序。判断是否使用计数排序的经验阈值是：当 k ≤ 2n 时，计数排序在时间和空间上通常具有优势。

### 误区二：认为计数排序只能处理非负整数

计数排序本质上需要将元素值映射为数组下标。对于含负数的数据，只需平移处理：令 offset = -min(A)，将每个元素映射为 A[i] + offset，计数数组大小为 max(A) - min(A) + 1。例如数据范围 -50 到 50，offset=50，计数数组大小101，完全可以处理。

### 误区三：忽略第三阶段逆序遍历的必要性

部分初学者为简化实现，在第三阶段改用正序遍历，或直接根据计数数组逐个填充输出（即简化版计数排序）。简化版虽然也能排序，但**不具有稳定性**，无法保留相同元素的原始相对顺序。在基数排序等依赖稳定子排序的场合，简化版计数排序会导致错误结果。实际上，只有包含前缀累加与逆序填充的完整三阶段实现才是正式的计数排序。

## 知识关联

**前置概念——数组/列表：** 计数排序的整个执行流程建立在三个数组的下标操作上：输入数组 A（读取）、计数/前缀数组 C（随机写入与读取）、输出数组 B（按计算下标写入）。熟悉数组的随机访问特性（O(1)时间）是理解为何计数排序能在 O(n+k) 内完成的直接基础——若使用链表实现，随机下标写入将失去 O(1) 保证。

**延伸方向——基数排序：** 计数排序直接作为基数排序的子例程，每一位数字的稳定排序均调用一次完整的计数排序流程（k=10或k=256）。掌握计数排序的稳定性原理，是理解基数排序正确性证明的前提。

**延伸方向——桶排序：** 桶排序是计数排序的泛化，将值域划分为若干区间（桶），每个桶内部再使用其他排序算法。当桶数量等于值域大小且每桶容量为1时，桶排序退化为计数排序。

**算法设计视角：** 计数排序是"用空间换时间"思想的典型案例，也是非比较排序能突破 Ω(n log n) 下界的具体证据——该下界的证明基于信息论的决策树模型，仅适用于基于元素间比较的排序算法，计数排序通过利用整数值的绝对大小（而非相对顺序）绕过了这一限制。
