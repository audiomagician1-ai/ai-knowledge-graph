# 位运算技巧

## 概述

位运算（Bitwise Operation）直接操作整数的二进制表示中的每一位（bit），CPU在硬件层面原生支持这些操作，执行速度通常比加减乘除快数倍。在算法竞赛与AI工程的底层实现中，位运算常用于替代昂贵的乘除法、加速集合操作、压缩内存布局。例如，`x * 2`可以用左移`x << 1`替代，`x / 4`可以用右移`x >> 2`替代，减少一次乘除指令开销。

位运算最早在C语言（1972年）中被系统化地引入高级语言语法，但其硬件根源可追溯到早期汇编指令集（如x86的`AND`、`OR`、`XOR`、`SHL`、`SHR`指令）。Python、Java、C++均继承了这套操作符体系：`&`（按位与）、`|`（按位或）、`^`（按位异或）、`~`（按位取反）、`<<`（左移）、`>>`（右移）。

在AI工程场景中，位运算出现在神经网络权重量化（如1-bit BWN网络中用位打包存储二值权重）、哈希索引（SimHash、LSH）、特征掩码压缩等关键路径上。掌握位运算技巧能够将部分热点代码的运行时间压缩至原来的1/3到1/10。

## 核心原理

### 基本运算规则与真值表

六种基本位运算的精确行为如下：

| 运算 | 符号 | 规则 |
|------|------|------|
| 按位与 | `&` | 两位均为1时结果为1，否则为0 |
| 按位或 | `\|` | 至少一位为1时结果为1 |
| 按位异或 | `^` | 两位不同时结果为1，相同时为0 |
| 按位取反 | `~` | 0变1，1变0（在32位整数中`~x = -x - 1`） |
| 左移 | `<<` | 各位向左移n位，低位补0，等价于乘以`2^n` |
| 右移 | `>>` | 各位向右移n位，对于有符号整数高位补符号位 |

重要公式：对于32位有符号整数，`~x = -(x+1)`，因为计算机用补码表示负数。

### 常用位运算公式与技巧

**判断奇偶**：`x & 1 == 1`表示x为奇数，比`x % 2`快，因为避免了取模运算。

**提取最低位1**（Lowest Set Bit，LSB）：`lowbit(x) = x & (-x)`。这一公式利用了补码的性质：`-x`等于`~x + 1`，使得最低位1以下的所有位全部取反后再加1恰好还原，仅保留最低位1。Fenwick树（树状数组）的核心索引跳转正是依赖`i += lowbit(i)`实现O(log n)更新。

**清除最低位1**：`x & (x - 1)`。例如`0b1010 & 0b1001 = 0b1000`，每次操作消除一个1。统计整数中1的个数（popcount）的Brian Kernighan算法即循环执行此操作，复杂度为O(k)，k为1的个数，优于O(32)的逐位扫描。

**交换两数（无临时变量）**：
```
a = a ^ b
b = a ^ b   // b = (a^b)^b = a
a = a ^ b   // a = (a^b)^a = b
```
注意：当`a`和`b`指向同一地址时此技巧会出错，实际工程中慎用。

**快速判断2的幂次**：`x > 0 && (x & (x-1)) == 0`。2的幂次在二进制中仅有一个1，减1后所有低位变为1，两者按位与为0。

**位掩码（Bitmask）构造**：`mask = (1 << n) - 1`生成低n位全为1的掩码，用于提取整数的低n位：`x & mask`。AI推理引擎中量化权重的位字段提取大量使用此模式。

### 位运算在集合操作中的应用

将一个不超过64个元素的集合映射到一个64位整数（`uint64_t`或Python的任意精度整数），每个bit代表对应元素是否在集合中：

- **集合并**：`A | B`
- **集合交**：`A & B`
- **集合差**（A中去掉B中的元素）：`A & ~B`
- **判断x属于集合A**：`(A >> x) & 1`
- **向集合A加入元素x**：`A | (1 << x)`
- **从集合A删除元素x**：`A & ~(1 << x)`

状态压缩动态规划（State Compression DP）依赖这套操作，TSP问题的Held-Karp算法时间复杂度为O(2^n × n²)，其状态就是n位整数表示的访问集合。

## 实际应用

**神经网络二值化（BNN）**：BNN（Binary Neural Network）将权重和激活值压缩为1-bit，一个32位整数可打包32个权重。矩阵乘法退化为`XNOR + popcount`操作：两个二值向量的点积等于`popcount(~(a ^ b))`（统计相同位数）减去`popcount(a ^ b)`（统计不同位数），硬件`POPCNT`指令（如x86的`_mm_popcnt_u64`）单条指令完成64位popcount，显著加速前向传播。

**Transformer中的Attention Mask**：在批量推理时，不同长度序列的padding mask用整数位向量表示，避免存储bool矩阵。一个32个序列的batch的mask可以用32个int32表示，而不是32×seq_len的bool数组。

**哈希去重与MinHash**：SimHash算法将文档转化为64位指纹，用`bin(a ^ b).count('1')`（即Hamming距离，等价于`popcount(a ^ b)`）衡量相似度，Hamming距离小于3则认为重复。实际工程中Python调用`gmpy2.popcount()`或C扩展的`__builtin_popcountll()`。

## 常见误区

**误区1：右移负数等于除以2的幂次**

对于有符号负数，算术右移（`>>`）向负无穷舍入，而整数除法向零舍入。例如在C/Java中，`-7 >> 1 = -4`（舍入到-4），但`-7 / 2 = -3`（舍入到-3）。用右移替代有符号整数的除法会引入细微bug，只有确认操作数为非负时才能安全替换。

**误区2：`~x`等于`-x`**

按位取反`~x`的结果是`-x - 1`（补码定义），而不是`-x`。例如`~5 = -6`，`~0 = -1`，`~(-1) = 0`。要取算术负值应使用一元减号`-x`，不要混淆。

**误区3：Python中整数位运算与C完全一致**

Python的`int`是任意精度整数，不存在32位或64位溢出。`~x`在Python中等于`-(x+1)`且可以是任意大负数，没有C语言中无符号整数的位截断行为。在实现需要固定宽度语义的算法（如CRC32、MurmurHash）时，必须手动用`& 0xFFFFFFFF`截断到32位，否则结果完全不同。

## 知识关联

位运算技巧建立在**二进制与数制**的基础上：必须熟悉补码（Two's Complement）表示才能正确理解`~x = -x-1`以及`x & (-x)`的LSB提取原理。补码规则直接决定了有符号整数右移的符号位填充行为，也解释了为何`-x = ~x + 1`。

位运算向上支撑多个AI工程与算法方向：**Fenwick树（树状数组）**的区间查询与点更新完全依赖`lowbit`操作；**状态压缩DP**用整数bitmask编码子集状态；**二值神经网络（BNN/XNOR-Net）**的前向传播将浮点乘加替换为popcount；**布隆过滤器**的多哈希位标记与查询用位或和位与实现。此外，密码学中的AES、SHA系列算法大量依赖字级别的位旋转（Rotate Left/Right），这些都是位运算技巧在工程中的直接延伸。
