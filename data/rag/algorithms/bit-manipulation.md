# 位运算技巧

## 概述

位运算（Bitwise Operation）是直接对整数在内存中的二进制位进行操作的运算方式，CPU 原生支持这类指令，执行速度通常比乘除法快 1 个数量级以上。在现代 64 位处理器上，一条位运算指令的延迟仅为 1 个时钟周期，而整数除法则需要 20-90 个时钟周期。这种性能优势使得位运算在 AI 推理加速、嵌入式部署和高频算法竞赛中具有独特价值。

位运算的基础操作包括六种：按位与（AND，`&`）、按位或（OR，`|`）、按位异或（XOR，`^`）、按位取反（NOT，`~`）、左移（`<<`）和右移（`>>`）。这六种操作的组合可以实现几乎所有数学运算的高效替代版本。例如，判断一个整数 `n` 是否为 2 的幂次，用算术方法需要循环或对数运算，而用位运算只需一行：`n > 0 && (n & (n - 1)) == 0`。

在 AI 工程领域，位运算的应用已从底层算法延伸至深度学习模型压缩。1-bit 量化（Binary Neural Networks）的核心推理加速原理，正是用 XNOR 操作替代浮点乘法、用 popcount 指令替代求和，使得矩阵乘法的计算量在理论上降低至原来的 1/64。

## 核心原理

### 基础位运算的数学本质

六种基础位运算均可用真值表精确定义。以 XOR 为例：`a ^ b = 1` 当且仅当 `a ≠ b`，这使 XOR 成为"差异检测器"。其代数性质包括：交换律 `a^b = b^a`、结合律 `(a^b)^c = a^(b^c)`、自反性 `a^a = 0`、零元 `a^0 = a`。这四条性质共同决定了 XOR 可用于不借助临时变量交换两个整数值：`a ^= b; b ^= a; a ^= b`，三步后 a、b 互换。

移位运算具有明确的数学等价关系：`n << k` 等价于 `n × 2^k`，`n >> k`（算术右移）等价于 `⌊n / 2^k⌋`。因此，将一个数乘以 6 可以写成 `(n << 2) + (n << 1)`，完全避免乘法指令。注意 C/C++ 中对有符号数右移的行为是实现定义的（implementation-defined），Python 中 `>>` 始终为算术右移。

### 常用位运算公式集

以下是经过验证的高频技巧，每条均有其数学依据：

- **取最低位1**：`n & (-n)`。因为 `-n` 在补码中等于 `~n + 1`，相加会在最低位1处产生对齐。例如 `n=12 (1100)`，`-n = 0100`（截取低4位），`12 & -12 = 4`。
- **消除最低位1**：`n & (n - 1)`。`n-1` 会将 n 最低位的1变为0，并将其后所有0变为1，AND 后恰好消除。
- **获取指定位**：`(n >> k) & 1`，将第 k 位（从0计）移至最低位后取出。
- **设置指定位**：`n | (1 << k)`，将第 k 位强制置1。
- **清除指定位**：`n & ~(1 << k)`，将第 k 位强制置0。
- **翻转指定位**：`n ^ (1 << k)`，利用 XOR 的自反性翻转单个位。

### 位掩码（Bitmask）与状态压缩

位掩码技术将多个布尔状态压缩进单个整数，一个 64 位整型可同时表示 64 个独立开关状态。在 AI 工程的特征工程中，稀疏二值特征向量通常使用 `uint64_t` 数组存储，内积运算转化为逐元素 AND 后统计 1 的个数（popcount），这比逐一比较 bool 数组快约 64 倍。

状态压缩动态规划（Bitmask DP）是算法竞赛和组合优化中的经典应用。旅行商问题（TSP）的 Held-Karp 算法用 `dp[mask][i]` 表示"访问了 mask 所代表的城市集合且最后停在城市 i 的最短路径"，将时间复杂度从 O(n!) 降至 O(2^n × n²)，其中 mask 从 0 枚举到 `(1 << n) - 1`。

## 实际应用

**神经网络二值化推理加速**：在 Binary Neural Network（BNN）中，权重和激活值被量化为 +1/-1，映射到比特位后，矩阵乘法中的一次浮点乘法等价于一次 XNOR 操作，累加等价于 popcount。NVIDIA 在 2017 年的 XNOR-Net 论文中验证，基于 popcount 的 BNN 推理速度在 CPU 上可达 FP32 的 58 倍，内存占用降低至 1/32。

**哈希表的快速取模**：当哈希表的桶数量 `m` 为 2 的幂时，`hash % m` 可替换为 `hash & (m - 1)`，消除代价昂贵的 `%` 指令。Python 的 `dict` 实现和 Java 的 `HashMap` 均采用此设计，这也是 HashMap 容量始终为 2 的幂（如 16, 32, 64）的根本原因。

**特征集合运算**：在推荐系统的用户画像中，将用户兴趣标签编码为 bitmask，两个用户的共同兴趣数量可用 `popcount(user_a & user_b)` 一步得出，Jaccard 相似度计算也可用 `popcount(a&b) / popcount(a|b)` 实现。

**随机数生成与初始化**：Xorshift 算法是一种基于位运算的伪随机数生成器，其核心操作仅三行：`x ^= x << 13; x ^= x >> 7; x ^= x << 17`，周期为 `2^32 - 1`，速度远快于 Mersenne Twister。

## 常见误区

**误区一：认为 `n >> 1` 与 `n / 2` 完全等价**。对于非负整数两者结果相同，但对于负奇数则不同。在 C 语言中，`-7 / 2 = -3`（向零截断），而 `-7 >> 1` 在大多数架构上得 `-4`（算术右移，向负无穷截断）。在 AI 框架的量化代码中混用这两种写法处理有符号激活值，会引入系统性偏差。

**误区二：XOR 交换技巧可无条件替代临时变量**。当 `a` 和 `b` 指向同一内存地址时，`a ^= b; b ^= a; a ^= b` 会将该位置清零而非交换。在 C++ 中若写出 `swap(arr[i], arr[i])`，且内部实现用 XOR 技巧，则结果错误。标准库 `std::swap` 使用临时变量正是为了规避这一问题。

**误区三：位运算在所有语言中行为一致**。Python 的整数是任意精度的（big integer），`~5` 的结果是 `-6`（因为 Python 整数视为无限精度补码），而 C 语言中 `~5u`（无符号）为 `4294967290`。此外 Python 没有无符号右移运算符，需要用 `(n & 0xFFFFFFFF) >> k` 模拟 Java/C++ 的 `>>>` 运算符。

## 知识关联

位运算技巧以**二进制与数制**为直接前置知识：必须理解补码表示法才能解释 `n & (-n)` 的工作原理，必须理解二进制位权才能推导移位与乘除法的等价关系。不了解补码的学习者会在处理有符号数的位运算时产生难以调试的错误。

在 AI 工程路径上，位运算技巧向上支撑**模型量化与压缩**（INT8/INT4/BNN 推理）和**高效数据结构**（Bloom Filter 的多哈希位图实现、Roaring Bitmap 的压缩区间表示）。在算法竞赛路径上，它是**状态压缩 DP**和**图论中的子集枚举**（枚举掩码 `mask` 的所有子集：`for s = mask; s > 0; s = (s-1) & mask`）的必备工具。
