# 单调栈

## 概述

单调栈是一种在普通栈的基础上增加了**元素单调性约束**的数据结构，栈内元素从栈底到栈顶始终保持单调递增或单调递减的顺序。与普通栈不同，单调栈在每次入栈前会主动弹出违反单调性的元素，正是这个"弹出"动作携带了算法所需的关键信息——被弹出的元素找到了它的"第一个更大/更小邻居"。

单调栈的思想最早在解决"下一个更大元素"（Next Greater Element）问题时被系统化提出，后来在 LeetCode 第 84 题"柱状图中最大的矩形"上成为最优解的核心工具，使该问题的时间复杂度从暴力的 O(n²) 降至 O(n)。理解这一结构，意味着掌握了一类"以空间换时间、以弹出换答案"的线性扫描范式。

单调栈之所以重要，是因为它将大量看似需要双重循环的问题压缩到单次线性扫描。栈中每个元素**最多入栈一次、出栈一次**，因此总操作次数为 2n，整体时间复杂度稳定为 O(n)。在 AI 工程中，滑动窗口最大值、时间序列的峰谷检测等场景均可借助单调栈在线性时间内完成。

## 核心原理

### 单调递减栈的入栈机制

以单调递减栈（从栈底到栈顶递减）为例，处理数组 `[2, 1, 5, 6, 2, 3]` 时：

1. 遍历每个元素 `x`；
2. 若栈不为空且**栈顶元素 < x**，则不断弹出栈顶元素，弹出的那个元素的"下一个更大元素"即为 `x`；
3. 将 `x` 入栈。

关键公式：对于被弹出的元素 `stack.top()`，其右侧第一个更大元素的下标为当前遍历下标 `i`，左侧第一个更大元素的下标为弹出后新的栈顶元素下标 `stack.top_after_pop`。这两个信息在同一次弹出操作中**同时获得**，这是单调栈效率的本质。

### 柱状图最大矩形的面积计算

LeetCode 84 题是单调栈最经典的应用。对于高度数组 `heights`，以每根柱子的高度作为矩形的最小高度，需要找到以该柱子为最低点时，向左右能延伸的最远边界。

设柱子 `i` 的高度为 `h[i]`，其左边第一个比它矮的柱子下标为 `left[i]`，右边第一个比它矮的柱子下标为 `right[i]`，则以 `i` 为最低柱子的矩形面积为：

$$S_i = h[i] \times (right[i] - left[i] - 1)$$

用**单调递增栈**一次遍历即可同时求出所有 `left[i]` 和 `right[i]`：当 `h[i] < h[stack.top()]` 时，栈顶元素的 `right` 就是 `i`，其 `left` 就是弹出后的新栈顶。答案为所有 `S_i` 的最大值。

### 哨兵元素技巧

在实际编码中，常在数组首尾各插入高度为 0 的"哨兵"元素（sentinel），例如将 `heights` 变为 `[0] + heights + [0]`。这样做的效果：

- 右侧哨兵 `0` 确保遍历结束时栈内剩余元素全部被强制弹出，无需在循环外额外处理；
- 左侧哨兵 `0` 确保弹出元素后栈永远不会为空，避免空栈判断的边界错误。

哨兵技巧将"末尾残余元素"这一特殊情况统一进主循环，使代码从约 20 行缩减至约 12 行，在竞赛和面试中极为实用。

### 单调栈与双指针的本质区别

双指针通过两个指针的相向/同向移动在 O(n) 内求解问题，适合连续区间的收缩/扩展；单调栈通过**弹出事件**隐式维护"左右最近边界"，适合离散的"前驱/后继极值"查询。两者都是 O(n)，但单调栈能够在弹出瞬间**同时获取左右边界信息**，双指针无法做到这一点。

## 实际应用

### 下一个更大元素（循环数组变体）

LeetCode 503 题要求在循环数组中找下一个更大元素。处理方式：将数组逻辑复制一遍，遍历下标 `i % n`（i 从 0 到 2n-1），使用单调递减栈，只在 `i < n` 时记录答案。这避免了真实复制数组带来的额外 O(n) 空间消耗。

### 接雨水问题

LeetCode 42 题"接雨水"中，位置 `i` 能积的水量取决于其左右两侧的最大高度：

$$water[i] = \max(0,\ \min(maxLeft[i],\ maxRight[i]) - height[i])$$

使用单调递减栈可按"层"计算积水：每当弹出栈顶元素 `mid` 时，以 `min(height[i], height[stack.top()]) - height[mid]` 为水柱高度，以当前下标与新栈顶下标之差减 1 为宽度，累加面积。

### 滑动窗口最大值（单调队列变体）

LeetCode 239 题中，单调递减的**双端队列**（deque）是单调栈的队列化变体——队头维护窗口内最大值，队尾弹出小于新元素的旧值，同时从队头移除已离开窗口的元素。这是单调栈思想在滑动窗口场景下的直接延伸。

## 常见误区

### 误区一：认为单调栈只能处理"右侧"信息

很多学习者初学时只会用单调栈找"右侧第一个更大元素"，不知道弹出操作**同时**给出了左侧信息（弹出后的新栈顶就是左侧第一个更大/更小元素的下标）。柱状图最大矩形问题正是利用了这一点，在一次遍历中同时确定每根柱子的左右边界，而不是分两次遍历。

### 误区二：混淆单调递增栈与单调递减栈的使用场景

单调**递增**栈（栈底到栈顶递增）在元素入栈时弹出比它**大**的元素，用于寻找"下一个更小元素"；单调**递减**栈弹出比它**小**的元素，用于寻找"下一个更大元素"。选错方向会导致找到的边界完全相反。柱状图最大矩形需要找"下一个更矮柱子"，因此应使用单调**递增**栈（栈内高度从底到顶递增）。

### 误区三：认为 O(n) 时间复杂度来自于"每次只弹出一个元素"

实际上，单次遍历中某一步可能连续弹出多个元素（例如遇到极大值时），但均摊分析表明：**n 个元素的总入栈次数为 n，总出栈次数也不超过 n**，因此 while 循环内的总弹出操作数上界为 n，整体是 O(n) 而非 O(n²)。不理解均摊复杂度会错误地以为单调栈有退化风险。

## 知识关联

**与栈（前置概念）的关系**：单调栈复用了栈的 push/pop/top 操作，但在 push 前增加了 while 循环清理不满足单调性的元素。若对普通栈的 LIFO 特性和 O(1) 操作不熟悉，单调栈的弹出时机判断会产生混乱。建议先能手写基于数组的栈实现，再学习单调栈。

**与双指针（前置概念）的关系**：双指针是单调栈的"思维对照组"——两者都能在 O(n) 内解决接雨水等问题，但路径不同。先理解双指针的两端收缩逻辑，再对比单调栈的弹出机制，能更清晰地感受到单调栈"以栈模拟左右边界"的独特视角。

**延伸方向**：掌握单调栈后，可自然延伸到**单调双端队列**（解决滑动窗口最大值）、**笛卡尔树**（将单调栈的弹出关系显式建树）以及**稀疏表/ST 表**（静态区间最值查询的 O(1) 方案）。这三个方向分别代表单调栈在"窗口"、"树形结构"和"预处理"场景下的进一步抽象。
