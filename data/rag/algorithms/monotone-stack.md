# 单调栈

## 概述

单调栈（Monotonic Stack）是一种在普通栈的基础上附加了"栈内元素始终保持单调有序"约束的数据结构。具体而言，栈内元素从栈底到栈顶要么严格单调递增（单调递增栈），要么严格单调递减（单调递减栈）。每当新元素入栈前，需要将所有破坏单调性的元素弹出，这一弹出过程正是单调栈的核心计算价值所在——被弹出元素的"下一个更大/更小元素"恰好就是触发弹出的那个新元素。

单调栈的思想可追溯至1970年代对"次序统计"（Order Statistics）问题的研究。Donald Knuth 在1973年出版的《计算机程序设计艺术》（The Art of Computer Programming, Vol. 1）中已系统讨论了栈的变种结构与序列问题的关联；而"柱状图最大矩形"问题的线性算法则由 Knuth 的工作启发，在1980年代的算法教材中逐渐成型。作为算法竞赛中的独立技巧，单调栈在2000年代随着 ACM-ICPC 题库与后来 LeetCode 平台的普及才被系统整理归纳。其时间复杂度为 $O(n)$——尽管外层循环 $n$ 次、内层似乎会多次弹出，但每个元素最多入栈和出栈各一次，总操作次数恰好是 $2n$，这种均摊分析（Amortized Analysis）是理解单调栈效率的关键。

单调栈在 AI 工程的算法面试中高频出现，并在时序特征提取、滑动窗口极值计算等工程场景中有实际应用。掌握它能将一类需要双重循环 $O(n^2)$ 的"前后关系查询"问题压缩到线性时间 $O(n)$，效率提升在 $n = 10^6$ 量级时尤为显著——朴素双重循环需要约 $10^{12}$ 次操作，而单调栈仅需约 $2 \times 10^6$ 次，差距达六个数量级。

## 核心原理

### 单调递增栈与单调递减栈的维护规则

**单调递增栈**（栈底→栈顶元素递增）：遍历数组时，若新元素 `x` **小于或等于**栈顶元素，则持续弹出栈顶，直到栈为空或栈顶元素小于 `x`，再将 `x` 压栈。此结构用于寻找每个元素**左侧第一个更小值**或**右侧第一个更小值**。

**单调递减栈**（栈底→栈顶元素递减）：若新元素 `x` **大于或等于**栈顶元素，则持续弹出，直到栈为空或栈顶大于 `x`，再压栈。此结构用于寻找每个元素的**下一个更大元素（Next Greater Element，NGE）**。

弹出时机决定了答案的绑定关系：当元素 `top` 被 `x` 弹出时，`x` 就是 `top` 的"右侧第一个触发弹出的元素"，而此刻栈中 `top` 下方的新栈顶就是 `top` 的"左侧第一个未被弹出的元素"——这两个信息同时在一次弹出中完整获得，是单调栈能够"一石二鸟"的根本原因。

以数组 `[3, 1, 4, 1, 5]` 为例演示单调递减栈的维护过程：压入3；遇到1，1 < 3 不弹出，压入1；遇到4，4 > 1弹出1（answer[1]=4），4 > 3弹出3（answer[0]=4），压入4；遇到1，1 < 4不弹出，压入1；遇到5，5 > 1弹出1（answer[3]=5），5 > 4弹出4（answer[2]=5），压入5。全程仅遍历一次，共5次压栈、4次弹栈，总操作9次，远小于朴素 $O(n^2)$ 的25次比较。

### 均摊复杂度的精确分析

单调栈的均摊时间复杂度证明可用**势能法（Potential Method）**形式化描述。定义势能函数 $\Phi$ 为当前栈的深度，则：

$$T_{\text{total}} = \sum_{i=1}^{n} \hat{c}_i = \sum_{i=1}^{n} (c_i + \Delta\Phi_i) \leq 2n$$

其中 $c_i$ 为第 $i$ 次操作的实际代价，$\hat{c}_i$ 为均摊代价，$\Delta\Phi_i = \Phi_i - \Phi_{i-1}$ 为势能变化量。每次压栈使势能加 1，每次弹栈使势能减 1，而总压栈次数恰好为 $n$，故总弹栈次数不超过 $n$，总代价上界为 $2n$，均摊每次操作为 $O(1)$。这一结论在 Cormen 等人的经典教材中有系统阐述（Cormen et al., 2009），势能法的完整框架见第17章"均摊分析"。

空间复杂度方面，单调栈在最坏情况下（输入数组严格单调递增时）所有元素均留在栈中，空间复杂度为 $O(n)$；最好情况（输入严格单调递减时，每个新元素都立即触发清空）空间复杂度为 $O(1)$（栈始终只有一个元素）。

### 下一个更大元素（Next Greater Element）

经典题目：给定数组 `nums = [2, 1, 2, 4, 3]`，求每个位置右侧第一个比它大的元素（LeetCode #496）。

使用单调递减栈，栈中存储**元素下标**（存下标而非值，方便记录位置对应关系）：

```
遍历到 index=0, val=2：栈空，压入 0。栈：[0]
遍历到 index=1, val=1：1<2，不弹出，压入 1。栈：[0,1]
遍历到 index=2, val=2：2>1，弹出1，answer[1]=2；2==栈顶值2，严格大于规则不触发弹出，压入2。栈：[0,2]
遍历到 index=3, val=4：4>2，弹出2，answer[2]=4；4>2，弹出0，answer[0]=4。栈：[3]
遍历到 index=4, val=3：3<4，压入4。栈：[3,4]
遍历结束，栈中剩余元素（下标3和4）无右侧更大值，answer=-1。
```

结果：`answer = [4, 2, 4, -1, -1]`，整个过程仅用一次线性遍历即可完成。

### 柱状图中的最大矩形（Largest Rectangle in Histogram）

这是单调栈最经典的压轴应用（LeetCode #84），输入为柱子高度数组 `heights`，要求找出能勾勒出的最大矩形面积。该问题的暴力解法枚举所有 $\binom{n}{2}$ 对左右边界，复杂度为 $O(n^2)$；单调栈将其压缩至 $O(n)$。

算法使用**单调递增栈**，栈内存储下标。当 `heights[i] < heights[栈顶]` 时，弹出栈顶 `top`，以 `heights[top]` 为矩形高度，宽度计算公式为：

$$\text{width} = \begin{cases} i - \text{stack.peek()} - 1 & \text{栈非空} \\ i & \text{栈为空（top左侧均} \geq \text{heights[top]）} \end{cases}$$

$$\text{area} = \text{heights[top]} \times \text{width}$$

为避免遍历结束后栈非空的边界处理，通常在 `heights` 末尾追加一个高度为 0 的哨兵（sentinel），强制将所有剩余元素弹出结算。哨兵技巧使代码从"循环后额外处理"变为"统一在循环内处理"，是工程实践中减少边界错误的常用手段。

**例如**，以 `heights = [2, 1, 5, 6, 2, 3]` 为例，遍历过程中当 `i=4, val=2` 时触发弹出：先弹出 index=3（高度6），面积 $6 \times 1 = 6$；再弹出 index=2（高度5），面积 $5 \times 2 = 10$；继续遍历后最终最大面积为 **10**（由高度5和6形成宽度为2的矩形），这也是该测试用例在 LeetCode 平台的标准答案。

### 接雨水（Trapping Rain Water）

LeetCode #42，给定地形高度数组，计算可积水总量。这道题于2011年前后开始频繁出现在 Google、Facebook 等公司的面试中，是单调栈"横向计算水量"思路的代表题。

单调递减栈的解法：当新高度 `h[i]` 大于栈顶时，弹出栈顶 `bottom`，以 `min(h[i], h[stack.peek()]) - h[bottom]` 为水层高度，`i - stack.peek() - 1` 为宽度，逐层累加积水。每次弹出对应一层横向水坑的体积计算：

$$\text{water} += \left(\min(h[i],\; h[\text{stack.peek()}]) - h[\text{bottom}]\right) \times (i - \text{stack.peek()} - 1)$$

此方法按"水坑层"横向计算，与双指针法的纵向逐列计算形成互补，两种方法在时间复杂度上均为 $O(n)$，但思维模型截然不同。以 `heights = [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]` 为例，单调栈解法的积水结果为6个单位，与双指针法结果一致，但代码逻辑中对"坑"的理解角度是水平层而非垂直柱。

## 关键公式与模型

单调栈相关问题的核心公式汇总如下，便于快速检索：

**① 下一个更大/更小元素**（Next Greater/Smaller Element）：

$$\text{answer}[i] = \text{nums}[j], \quad j = \min\{k > i \mid \text{nums}[k] > \text{nums}[i]\}$$

若不存在满足条件的 $j$，则 $\text{answer}[i] = -1$。类似地，"下一个更小元素"只需将不等号方向反转，并改用单调递增栈即可。

**② 柱状图最大矩形面积**：

$$S_{\max} = \max_{i} \left( \text{heights}[i] \times (\text{right}[i] - \text{left}[i] - 1) \right)$$

其中 $\text{left}[i]$ 为 $i$ 左侧第一个严格小于 `heights[i]` 的下标，$\text{right}[i]$ 为右侧第一个严格小于的下标。单调栈可在 $O(n)$ 内同时求出全部 $\text{left}$ 和 $\text{right}$ 数组。若改为两次分向遍历（一次从左求 `left`，一次从右求 `right`），代码更易理解，但常数因子增大为原来的两倍，实际运行时间仍为 $O(n)$。

**③ 股票价格跨度**（Stock Span Problem）：

$$\text{span}[i] = i - \text{prev\_greater}[i]$$

其中 $\text{prev\_greater}[i]$ 为 $i$ 左侧最近一个价格严格大于 $\text{price}[i]$ 的下标（若不存在则为 $-1$）。对股价序列 `[100, 80, 60, 70, 60, 75, 85]`，各天跨度分别为 `[1, 1, 1, 2, 1, 4, 6]`，最后一天85的跨度6表示连续6天价格均不超过85。

**④ 均摊复杂度上界**（势能法）：

$$\sum_{i=1}^{n} c_i \leq 2n \implies T(n) = O(n)$$
