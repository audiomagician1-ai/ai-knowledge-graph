# 单调栈

## 概述

单调栈（Monotonic Stack）是一种在普通栈的基础上附加了"栈内元素始终保持单调有序"约束的数据结构。具体而言，栈内元素从栈底到栈顶要么严格单调递增（单调递增栈），要么严格单调递减（单调递减栈）。每当新元素入栈前，需要将所有破坏单调性的元素弹出，这一弹出过程正是单调栈的核心计算价值所在——被弹出元素的"下一个更大/更小元素"恰好就是触发弹出的那个新元素。

单调栈的思想可追溯至1970年代对"次序统计"问题的研究，但作为算法竞赛中的独立技巧，它在2000年代随着 LeetCode 等题库的普及才被系统整理成型。其时间复杂度为 $O(n)$——尽管外层循环 $n$ 次、内层似乎会多次弹出，但每个元素最多入栈和出栈各一次，总操作次数恰好是 $2n$，这种均摊分析（Amortized Analysis）是理解单调栈效率的关键。

单调栈在 AI 工程的算法面试中高频出现，并在时序特征提取、滑动窗口极值计算等工程场景中有实际应用。掌握它能将一类需要双重循环 $O(n^2)$ 的"前后关系查询"问题压缩到线性时间 $O(n)$，效率提升在 $n = 10^6$ 量级时尤为显著。

## 核心原理

### 单调递增栈与单调递减栈的维护规则

**单调递增栈**（栈底→栈顶元素递增）：遍历数组时，若新元素 `x` **小于或等于**栈顶元素，则持续弹出栈顶，直到栈为空或栈顶元素小于 `x`，再将 `x` 压栈。此结构用于寻找每个元素**左侧第一个更小值**或**右侧第一个更小值**。

**单调递减栈**（栈底→栈顶元素递减）：若新元素 `x` **大于或等于**栈顶元素，则持续弹出，直到栈为空或栈顶大于 `x`，再压栈。此结构用于寻找每个元素的**下一个更大元素（Next Greater Element）**。

弹出时机决定了答案的绑定关系：当元素 `top` 被 `x` 弹出时，`x` 就是 `top` 的"右侧第一个触发弹出的元素"，而此刻栈中 `top` 下方的新栈顶就是 `top` 的"左侧第一个未被弹出的元素"——这两个信息同时在一次弹出中完整获得，是单调栈能够"一石二鸟"的根本原因。

### 均摊复杂度的精确分析

单调栈的均摊时间复杂度证明可用**势能法（Potential Method）**形式化描述。定义势能函数 $\Phi$ 为当前栈的深度，则：

$$T_{\text{total}} = \sum_{i=1}^{n} \hat{c}_i = \sum_{i=1}^{n} (c_i + \Delta\Phi_i) \leq 2n$$

其中 $c_i$ 为第 $i$ 次操作的实际代价，$\hat{c}_i$ 为均摊代价，$\Delta\Phi_i = \Phi_i - \Phi_{i-1}$ 为势能变化量。每次压栈使势能加 1，每次弹栈使势能减 1，而总压栈次数恰好为 $n$，故总弹栈次数不超过 $n$，总代价上界为 $2n$，均摊每次操作为 $O(1)$。这一结论在 Cormen 等人的经典教材中有系统阐述（Cormen et al., 2009）。

### 下一个更大元素（Next Greater Element）

经典题目：给定数组 `nums = [2, 1, 2, 4, 3]`，求每个位置右侧第一个比它大的元素。

使用单调递减栈，栈中存储**元素下标**（存下标而非值，方便记录位置对应关系）：

```
遍历到 index=0, val=2：栈空，压入 0。栈：[0]
遍历到 index=1, val=1：1<2，不弹出，压入 1。栈：[0,1]
遍历到 index=2, val=2：2==栈顶值1时，弹出1，answer[1]=2；2==栈顶值2时，根据严格大于的规则不弹出，压入2。栈：[0,2]
遍历到 index=3, val=4：弹出2，answer[2]=4；弹出0，answer[0]=4。栈：[3]
遍历到 index=4, val=3：3<4，压入4。栈：[3,4]
遍历结束，栈中剩余元素无右侧更大值，answer=-1。
```

结果：`answer = [4, 2, 4, -1, -1]`，整个过程仅用一次线性遍历即可完成，对应 LeetCode #496。

### 柱状图中的最大矩形（Largest Rectangle in Histogram）

这是单调栈最经典的压轴应用（LeetCode #84），输入为柱子高度数组 `heights`，要求找出能勾勒出的最大矩形面积。

算法使用**单调递增栈**，栈内存储下标。当 `heights[i] < heights[栈顶]` 时，弹出栈顶 `top`，以 `heights[top]` 为矩形高度，宽度计算公式为：

$$\text{width} = \begin{cases} i - \text{stack.peek()} - 1 & \text{栈非空} \\ i & \text{栈为空（top左侧均} \geq \text{heights[top]）} \end{cases}$$

$$\text{area} = \text{heights[top]} \times \text{width}$$

为避免遍历结束后栈非空的边界处理，通常在 `heights` 末尾追加一个高度为 0 的哨兵（sentinel），强制将所有剩余元素弹出结算。

**例如**，以 `heights = [2, 1, 5, 6, 2, 3]` 为例，遍历过程中当 `i=4, val=2` 时触发弹出：先弹出 index=3（高度6），面积 $6 \times 1 = 6$；再弹出 index=2（高度5），面积 $5 \times 2 = 10$；最终最大面积为 **10**（由高度5和6形成宽度为2的矩形），这也是该测试用例在 LeetCode 平台的标准答案。

### 接雨水（Trapping Rain Water）

LeetCode #42，给定地形高度数组，计算可积水总量。单调递减栈的解法：当新高度 `h[i]` 大于栈顶时，弹出栈顶 `bottom`，以 `min(h[i], h[stack.peek()]) - h[bottom]` 为水层高度，`i - stack.peek() - 1` 为宽度，逐层累加积水。每次弹出对应一层横向水坑的体积计算：

$$\text{water} += \left(\min(h[i],\; h[\text{stack.peek()}]) - h[\text{bottom}]\right) \times (i - \text{stack.peek()} - 1)$$

此方法按"水坑层"横向计算，与双指针法的纵向逐列计算形成互补，两种方法在时间复杂度上均为 $O(n)$，但思维模型截然不同。

## 关键公式与模型

单调栈相关问题的核心公式汇总如下，便于快速检索：

**① 下一个更大/更小元素**（Next Greater/Smaller Element）：
$$\text{answer}[i] = \text{nums}[j], \quad j = \min\{k > i \mid \text{nums}[k] > \text{nums}[i]\}$$

若不存在满足条件的 $j$，则 $\text{answer}[i] = -1$。

**② 柱状图最大矩形面积**：
$$S_{\max} = \max_{i} \left( \text{heights}[i] \times (\text{right}[i] - \text{left}[i] - 1) \right)$$

其中 $\text{left}[i]$ 为 $i$ 左侧第一个严格小于 `heights[i]` 的下标，$\text{right}[i]$ 为右侧第一个严格小于的下标。单调栈可在 $O(n)$ 内同时求出全部 $\text{left}$ 和 $\text{right}$ 数组。

**③ 股票价格跨度**（Stock Span Problem）：
$$\text{span}[i] = i - \text{prev\_greater}[i]$$

其中 $\text{prev\_greater}[i]$ 为 $i$ 左侧最近一个价格严格大于 $\text{price}[i]$ 的下标（若不存在则为 $-1$）。

**④ 均摊复杂度上界**（势能法）：
$$\sum_{i=1}^{n} c_i \leq 2n \implies T(n) = O(n)$$

## 实际应用

**滑动窗口最大值（Sliding Window Maximum，LeetCode #239）**：使用**单调递减双端队列**（本质是单调栈思想的扩展），在 $O(n)$ 时间内求每个大小为 $k$ 的窗口的最大值，比朴素 $O(nk)$ 方法在 $k=1000,\, n=10^6$ 时快约1000倍。在时序信号处理和在线学习特征提取中，这种效率提升具有实用价值，例如 PyTorch 的某些时间序列数据加载器内部即采用类似策略做窗口统计。

**股票价格跨度（Stock Span Problem）**：对每天股价，求连续多少天的价格都不超过今天。单调递减栈直接在弹出过程中累计跨度天数，常用于金融数据预处理特征工程。例如股价序列 `[100, 80, 60, 70, 60, 75, 85]`，最后一天85的跨度为6，对应 LeetCode #901。

**每日温度（Daily Temperatures，LeetCode #739）**：求每天需等待多少天才能遇到更高温度。答案数组通过 $\text{answer}[i] = j - i$（$j$ 是触发 $i$ 弹出的下标）直接在单次遍历中完成填充，无需额外空间记录中间结果。

**AI 特征工程中的时序极值**：在多变量时间序列模型（如 LSTM 输入预处理）中，往往需要为每个时间步计算"距上次局部最大值的时间差"特征。单调递减栈可在 $O(n)$ 内批量生成此类特征，相比逐窗口扫描节省大量预处理时间，在序列长度 $n > 10^5$ 的场景下效果尤为明显。

## 常见误区

**误区一：混淆"单调递增栈"用于找更大值还是更小值。**
栈内单调递增意味着新元素压栈时会弹出比它大的旧元素，被弹出元素找到的是"右侧第一个更小值"，而**不是**更大值。找"下一个更大元素"需要用单调递减栈——新元素比栈顶大时才触发弹出，被弹出元素的答案是这个更大的新元素。方向判断混乱是初学者最高频的错误，可用口诀记忆：**"递减栈找更大，递增栈找更小"**。

**误区二：栈中存值而非下标。**
对"下一个更大元素"问题，若栈中只存值，则弹出时无法定位原始位置，导致无法填写答案数组对应位置。几乎所有单调栈题目都应存储**下标**，通过 `nums[index]` 取值。唯一例外是股票跨度问题，因为答案直接是计数而非下标差，可以在栈中存储 `(price, span)` 二元组。

**误区三：忘记处理栈中剩余元素。**
遍历结束后，栈中仍留有元素意味着这些元素右侧（或左侧）无满足条件的更大/更小值，应将其答案统一赋值为 -1 或 0。柱状图最大矩形问题通过追加高度为 0 的哨兵来优雅地解决这一问题，避免额外的尾部处理逻辑。

**误区四：在循环数组中忘记"翻倍遍历"技巧。**
