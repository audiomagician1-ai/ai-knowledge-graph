# 插入排序

## 概述

插入排序（Insertion Sort）是一种通过逐步构建有序序列来完成排序的算法。其基本思想是：将数组分为"已排序"和"未排序"两部分，每次从未排序部分取出第一个元素，将其插入到已排序部分的正确位置。算法从第二个元素（索引1）开始，因为单个元素本身视为已排好序的。

插入排序的思想来源于人类整理扑克牌的直觉行为——摸到一张新牌时，将其从右向左与手中的牌逐一比较，找到合适位置后插入。这种算法由Donald Knuth在1968年出版的《计算机程序设计艺术》中进行了系统性分析，被归类为"原地比较排序"算法。

插入排序在元素数量较少（通常 n ≤ 20）或数据已经"接近有序"时表现优越，实际工程中许多高级排序算法（如 Timsort）在小规模子数组上会回退到插入排序来提升整体效率。

## 核心原理

### 算法步骤与伪代码

插入排序的具体执行流程如下：

1. 从索引 `i = 1` 开始，取出当前元素 `key = arr[i]`
2. 令指针 `j = i - 1`，向左扫描已排序区间
3. 当 `j >= 0` 且 `arr[j] > key` 时，将 `arr[j]` 向右移动一位（`arr[j+1] = arr[j]`），并令 `j--`
4. 将 `key` 填入空出的位置 `arr[j+1] = key`
5. `i` 递增，重复直到遍历完整个数组

以数组 `[5, 3, 8, 1, 2]` 为例，第一轮取出 `key=3`，将 `5` 右移，得到 `[3, 5, 8, 1, 2]`；第三轮取出 `key=1`，依次右移 `8`、`5`、`3`，得到 `[1, 3, 5, 8, 2]`。

### 时间复杂度与空间复杂度

插入排序的时间复杂度取决于输入的有序程度：

- **最好情况**（数组已完全有序）：每个元素只需与前一个元素比较一次即停止移动，共进行 **n-1** 次比较，时间复杂度为 **O(n)**
- **最坏情况**（数组完全逆序）：第 `i` 个元素需要比较 `i` 次，总比较次数为 `1+2+3+...+(n-1) = n(n-1)/2`，时间复杂度为 **O(n²)**
- **平均情况**：时间复杂度为 **O(n²)**

空间复杂度为 **O(1)**，因为插入排序是原地（in-place）算法，仅需要一个 `key` 变量作为临时存储，不需要额外的辅助数组。

### 稳定性

插入排序是一种**稳定排序**算法。在内层循环的比较条件为严格大于（`arr[j] > key`）时，当遇到与 `key` 相等的元素，移动会立即停止，`key` 插入在该相等元素的**右侧**，从而保持了相等元素原有的相对顺序。这一特性使插入排序在需要保持记录原始顺序的场合（如按姓名排序后再按成绩稳定排序）具有实际价值。

### 与移动操作 vs 交换操作的区别

插入排序的内层循环使用**移动（shift）**而非**交换（swap）**。交换操作需要3次赋值（借助临时变量），而移动只需1次赋值。这意味着对于 `n` 个逆序元素，移动方式的赋值次数比频繁交换的冒泡排序少约三分之二，因此在实际常数因子上插入排序优于冒泡排序。

## 实际应用

### Python内置排序中的应用

CPython 的内置 `sorted()` 函数和 `list.sort()` 使用的 **Timsort** 算法，当待排序的子数组长度小于 **64** 时（实际阈值可调，默认为32或64），会切换到插入排序处理该子数组。这是因为 O(n²) 在 n 很小时的实际运行时间比 O(n log n) 算法的常数开销更小。

### 在线排序场景

插入排序天然支持"**在线排序**"（Online Sorting）：数据不需要一次性全部到位，每来一个新元素就可以立即将其插入已有的有序序列中。这在处理实时数据流（如股票价格实时排名、传感器数据实时维护有序列表）时非常有用，因为无需等待所有数据到来后重新排序。

### 链表排序

插入排序特别适合对**单向链表**排序。链表不支持随机访问，归并排序等算法在链表上的实现复杂度高，而插入排序只需要顺序扫描和修改指针，不需要索引操作，因此在链表数据结构上是常用选择。

## 常见误区

### 误区一：认为插入排序的比较次数与移动次数相同

实际上，每一轮内层循环的**比较次数**等于移动次数加1（最后一次比较确定停止位置但不发生移动）。对于已排序数组，每轮只比较1次、移动0次；对于逆序数组，第 `i` 轮比较 `i` 次、移动 `i` 次。混淆两者会导致对算法常数因子的错误估计。

### 误区二：认为插入排序在任何情况下都劣于归并排序

这一看法忽视了常数因子和缓存局部性。对于 n ≤ 10 到 20 的小数组，插入排序的实测速度往往快于 O(n log n) 的归并排序，因为归并排序的递归调用、额外内存分配和缓存不命中带来的开销不可忽视。这正是 Timsort 混合使用两种算法的工程动机。

### 误区三：将"插入位置查找"优化为二分查找后认为时间复杂度降为 O(n log n)

使用**二分插入排序**（Binary Insertion Sort）确实可以将每轮的**比较次数**降至 O(log n)，但**移动次数**仍为 O(n)（因为插入后需要移动后续所有元素腾出位置）。因此整体时间复杂度仍为 **O(n²)**，二分查找只减少了比较的常数因子，不改变渐近复杂度。

## 知识关联

**前置知识**：插入排序直接操作**数组/列表**的索引和赋值。理解数组的连续内存结构有助于解释为何"移位"操作成本高（需要依次搬移多个元素），以及为何插入排序在链表上无需移位只需改指针。

**横向对比**：与同为 O(n²) 的**冒泡排序**相比，插入排序在最好情况下是 O(n)，而标准冒泡排序（无优化版本）始终是 O(n²)；与**选择排序**相比，选择排序不稳定且始终需要 O(n²) 次比较，插入排序在近似有序数据上更高效。

**进阶拓展**：**希尔排序**（Shell Sort）是插入排序的直接改进版，通过先对间隔为 `h` 的元素做插入排序（h-排序），逐步缩小间隔至1，从而将时间复杂度优化至 O(n^1.3) 到 O(n^1.5)，是学习分治优化思路的重要过渡案例。
