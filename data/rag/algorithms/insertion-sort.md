# 插入排序

## 概述

插入排序（Insertion Sort）是一种通过逐步构建有序序列来完成排序的算法。其工作方式类似于人手动整理扑克牌：每次从未排序的牌中取一张，将其插入已有的有序手牌中的正确位置。这种直觉上的自然性使插入排序成为理解排序算法的入门首选。

插入排序最早在计算机科学教育中作为基础排序算法被系统化描述，在1950年代随着早期计算机程序设计的发展而广泛记录。与冒泡排序不同，插入排序在处理"接近有序"的数组时表现出色——当数组已接近有序状态时，其时间复杂度可以趋近于 O(n)，而非最坏情况下的 O(n²)。

插入排序的实际价值在于：当输入数组规模较小（通常 n < 50）或数组已经部分有序时，它往往比归并排序、快速排序等复杂算法更快。因此，许多标准库的混合排序算法（如 Python 的 Timsort）在子数组规模较小时会自动切换为插入排序执行。

## 核心原理

### 算法步骤与循环不变式

插入排序维护一个"已排序前缀"，初始时该前缀仅包含第 0 个元素。算法从下标 i = 1 开始，每轮迭代执行以下操作：

1. 取出 `arr[i]` 作为"待插入元素"（key）
2. 从 i-1 向左扫描，将所有大于 key 的元素依次向右移动一位
3. 将 key 放入空出的位置

循环不变式保证：每轮迭代开始时，`arr[0..i-1]` 是已排序的子数组。当 i 增长至 n 时，整个数组排序完成。

### 时间复杂度分析

插入排序的时间复杂度取决于输入数据的初始有序程度：

- **最优情况**：数组完全有序，内层循环每次立即终止，共执行 n-1 次比较，时间复杂度为 **O(n)**
- **最坏情况**：数组完全逆序，第 i 轮需移动 i 次，总比较次数为 1+2+…+(n-1) = n(n-1)/2，时间复杂度为 **O(n²)**
- **平均情况**：期望比较次数约为 n²/4，时间复杂度为 **O(n²)**

空间复杂度为 **O(1)**，因为排序在原数组上就地完成，只需一个额外变量 key 存储待插入值。

### 稳定性与原地性

插入排序是**稳定排序**：当待插入元素 key 与已排序序列中某元素相等时，算法将 key 插入该元素的右侧（即内层循环条件为严格大于 `>`，而非大于等于 `>=`），因此相等元素的相对顺序不会被改变。这一稳定性在按多字段排序时至关重要，例如先按姓名排序后再按年龄插入排序，同龄人之间的姓名顺序将被保留。

### 伪代码实现

```
INSERTION-SORT(arr, n):
  for i from 1 to n-1:
      key = arr[i]
      j = i - 1
      while j >= 0 and arr[j] > key:
          arr[j+1] = arr[j]
          j = j - 1
      arr[j+1] = key
```

注意内层 while 循环采用"移位"而非"交换"策略：移位只需 1 次赋值，而交换需要 3 次，因此插入排序在常数因子上优于等量交换次数的冒泡排序。

## 实际应用

### Python Timsort 中的角色

Python 标准库的内置排序 `sorted()` 和 `list.sort()` 使用 Timsort 算法。Timsort 将数组分割为若干"run"（子段），当某个 run 的长度小于 **minrun**（通常为 32 或 64）时，Timsort 使用插入排序对该 run 进行排序，然后再通过归并将各 run 合并。这种设计利用了插入排序在小规模和近有序数据上的高效性。

### 在线算法场景

插入排序具备天然的"在线"特性：它可以在数据流逐一到来时实时维护有序序列，无需预先知道总数据量。例如，实时排行榜系统每收到一个新分数，可以将其插入已排好序的榜单中，而无需对整个榜单重新排序。

### 小型数组的实际性能

在实验测量中，对长度为 10 的随机整数数组，插入排序的实际运行时间常快于快速排序，原因是快速排序的递归调用和分区操作带来的函数调用开销在小规模时超过了插入排序 O(n²) 的计算代价。这就是为什么 C++ STL 的 `std::sort` 实现（introsort）在子数组规模降至约 16 以下时切换为插入排序。

## 常见误区

### 误区一：认为插入排序总是低效

许多初学者看到 O(n²) 的平均时间复杂度便认为插入排序没有实用价值。实际上，对于已经有序或近似有序的数据（逆序对数量为 O(n) 级别），插入排序的实际运行时间是线性的。例如，对一个 10000 个元素的数组插入 1 个新元素并保持有序，插入排序只需约 1 次比较（最优位置在末尾时），而重新调用快速排序则需要约 O(n log n) 的时间。

### 误区二：混淆"移位"与"交换"

有些学生将插入排序的内层操作描述为"交换相邻元素"，从而将其与冒泡排序混同。插入排序的内层循环执行的是**单向移位**：仅将 `arr[j]` 复制到 `arr[j+1]`，被插入元素 key 在找到最终位置之前一直保存在临时变量中，只写入一次。冒泡排序则在每次比较时执行完整的三步交换。这一区别导致插入排序的赋值次数约为冒泡排序的 1/3。

### 误区三：内层循环条件写错导致不稳定

若将内层循环条件写为 `arr[j] >= key`（大于等于），则遇到相等元素时会继续移动，将 key 插入相等元素的左侧，破坏稳定性。正确实现应使用严格大于 `arr[j] > key`，确保相等元素不被越过。

## 知识关联

**前置知识：数组/列表**
插入排序的核心操作——"向右移位"——本质上是对数组的连续下标赋值 `arr[j+1] = arr[j]`，依赖对数组随机访问（O(1) 读写）和下标操作的掌握。若使用链表存储，移位操作可省去，但查找插入位置仍需 O(n) 遍历，整体复杂度不变。

**横向对比：与选择排序、冒泡排序**
三种 O(n²) 排序算法中，插入排序是唯一在最优情况下达到 O(n) 的算法，也是唯一稳定且对近有序数据高效的算法。选择排序无论何种输入始终执行 n(n-1)/2 次比较；冒泡排序虽可通过标志位提前终止，但移动次数远多于插入排序。

**向上延伸：Shell 排序**
Shell 排序（1959年由 Donald Shell 提出）是插入排序的直接改进：它先对间隔为 h 的元素子序列分别执行插入排序，逐步缩小 h 至 1，最终完成全局排序。Shell 排序将插入排序对近有序数据高效的特性推广到更大规模，突破了 O(n²) 的平均时间限制，是学习更复杂排序算法的重要中间步骤。
