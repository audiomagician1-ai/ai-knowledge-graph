# 插入排序

## 概述

插入排序（Insertion Sort）是一种通过逐步构建有序序列来完成排序的算法。其工作方式与人类整理扑克牌完全一致：每次从未排序部分取出一张牌，将其插入到已排序部分的正确位置，直到所有牌都已归位。算法将数组逻辑划分为"已排序区间"（初始只含第0个元素）和"未排序区间"两部分，每轮迭代将未排序区间的第一个元素插入已排序区间的合适位置。

插入排序的概念可追溯至计算机科学的早期发展阶段，在1945年前后随着早期编程实践的出现而被系统化描述。尽管它的时间复杂度并非最优，但由于其极低的实现复杂度和在特定数据分布下的优异表现，至今仍被广泛嵌入 Python 的 Timsort、Java 的 Arrays.sort 等混合排序算法的内部逻辑中——当子数组长度小于约16个元素时，这些工业级算法会自动切换为插入排序。

插入排序对于小规模数据集或近乎有序（nearly-sorted）的数据具有独特优势。在最优情况下（数组已完全有序），插入排序只需执行 n−1 次比较，不发生任何元素移动，时间复杂度降至 O(n)，这是冒泡排序和选择排序均无法达到的特性。

## 核心原理

### 算法步骤与循环不变量

插入排序的精确执行流程如下：从下标 `i = 1` 开始，取出 `arr[i]` 存入临时变量 `key`；然后令指针 `j = i - 1`，将所有大于 `key` 的 `arr[j]` 向右移动一位（即 `arr[j+1] = arr[j]`，`j` 递减），直到找到 `arr[j] <= key` 或 `j < 0` 为止；最后将 `key` 放入 `arr[j+1]`。

算法维护一个关键的**循环不变量**：在每次外层循环开始前，子数组 `arr[0..i-1]` 始终是有序的。这一不变量可用于严格证明算法的正确性：初始时 `i=1`，单元素数组自然有序（初始成立）；每次插入操作保持该区间有序（维持成立）；循环结束时 `i=n`，整个数组有序（终止正确）。

### 时间与空间复杂度

插入排序的复杂度因输入数据的初始有序程度而存在显著差异：

| 情况 | 时间复杂度 | 触发条件 |
|------|-----------|---------|
| 最优 | O(n) | 数组已完全正序排列 |
| 平均 | O(n²) | 随机排列的数组 |
| 最坏 | O(n²) | 数组完全逆序排列 |

最坏情况下，第 `i` 轮需要移动 `i` 次，总移动次数为 `1+2+3+…+(n-1) = n(n-1)/2`，即 O(n²)。空间复杂度始终为 **O(1)**，因为除了存储 `key` 的单个临时变量外，排序完全在原数组上进行（in-place）。

### 稳定性

插入排序是**稳定排序（stable sort）**。当算法判断"当前元素是否需要移动"时，条件为严格大于（`arr[j] > key`），相等的元素不会触发移动。因此，若原数组中有两个值相同的元素，排序后它们的相对先后顺序与原始数组一致，不会发生交换。这一特性使插入排序适用于多关键字排序场景（例如先按姓名排序后再按年龄稳定排序）。

## 实际应用

**场景一：近乎有序数据的实时插入**  
在在线算法（Online Algorithm）场景中，数据以流式方式逐条到达，每到达一条新数据就需要维护整体有序性。例如，股票行情系统中维护一个已排序的价格列表，每新增一条交易记录只需将其插入正确位置，插入操作平均只需比较 log n 次（若结合二分查找优化，称为**二分插入排序**），移动次数仍为 O(n)。

**场景二：混合排序算法的底层组件**  
Python 标准库使用的 Timsort 算法在处理长度小于 **64** 的子数组时，直接调用插入排序而非递归地执行归并操作，原因是插入排序的常数因子极小，在小数组上的实际运行速度快于任何 O(n log n) 算法。

**场景三：AI工程中的候选集重排**  
推荐系统中，召回阶段输出的候选列表通常数量不超过200个，且已经按粗排分数大致排好序。对此类"小规模+近乎有序"的候选集进行精排时，插入排序可以在实测中比快速排序快30%以上，同时保持稳定性以避免同分候选项顺序随机化。

## 常见误区

**误区一：认为插入排序每次操作的是"交换"而非"移动"**  
许多初学者将插入排序与冒泡排序混淆，认为其核心操作是相邻元素的两两交换。实际上，插入排序使用的是"右移覆盖"操作：`arr[j+1] = arr[j]`，只写入目标位置，不交换。整个循环中 `key` 只被赋值一次（最终放入 `arr[j+1]`）。这使得插入排序的赋值次数约为冒泡排序的 **1/3**，常数因子更小。

**误区二：认为二分插入排序在时间复杂度上优于普通插入排序**  
使用二分查找确定插入位置后，比较次数从 O(n²) 降至 O(n log n)，但元素移动次数仍然是 O(n²)——因为找到位置后仍然需要将目标位置之后的所有元素向右移动一格。由于在随机存取内存（RAM）模型中移动元素的代价与比较相当，二分插入排序的实际时间复杂度**仍为 O(n²)**，并不是 O(n log n)。

**误区三：认为插入排序对所有小数组都优于快速排序**  
"小数组切换插入排序"这一策略有其阈值范围。实验表明该阈值通常在 **8 到 32** 之间，具体值与硬件缓存行大小和编译器优化有关。若数组元素为大型对象（每次移动涉及大量内存拷贝），这一阈值会显著降低，甚至可能完全不适合使用插入排序。

## 知识关联

**前置知识：数组/列表的索引访问**  
插入排序的"右移覆盖"操作依赖对数组连续内存地址的随机访问（`arr[j]`、`arr[j+1]`），需要掌握数组的下标操作和原地修改语义。链表虽然支持 O(1) 插入，但查找插入位置仍需 O(n) 遍历，且缓存局部性差，实际性能通常不如数组实现。

**横向对比：与选择排序和冒泡排序的区别**  
三种 O(n²) 排序算法各有特点：选择排序每次找最小值，交换次数最少（恰好 n-1 次），但不稳定；冒泡排序通过相邻交换，稳定但赋值次数最多；插入排序通过右移覆盖，稳定且在近乎有序数据上退化为 O(n)，是三者中适应性（Adaptive）最强的算法。这种适应性使插入排序成为工程实践中最具实用价值的初等排序算法。
