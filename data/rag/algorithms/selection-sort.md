# 选择排序

## 概述

选择排序（Selection Sort）是一种基于"逐次选取最小值"思想的原地比较排序算法。其基本逻辑是：将数组分为已排序区间和未排序区间两部分，每轮从未排序区间中找出最小元素，将其与未排序区间的第一个元素交换，从而将已排序区间向右扩展一位。重复执行 n-1 轮后，整个数组完成排序。

选择排序的思想最早可追溯到计算机科学早期的排序研究（1950年代），是教科书中与冒泡排序、插入排序并列的三大基础排序算法之一。尽管其时间复杂度不占优势，但由于逻辑极为直观——"每次选最小的放到最前面"——它长期作为讲授排序概念的首选示例。

选择排序最显著的特点是**交换次数极少**。对一个长度为 n 的数组，选择排序最多执行 n-1 次元素交换，而冒泡排序在最坏情况下需要执行 n(n-1)/2 次交换。在"交换代价远高于比较代价"的场景（例如数组元素为体积庞大的对象）中，这一特性使选择排序具有实际价值。

## 核心原理

### 算法步骤与伪代码

选择排序的执行过程如下：

1. 设当前轮次起始位置为 `i`（从 0 到 n-2）
2. 在子数组 `arr[i..n-1]` 中扫描，记录最小值的下标 `min_idx`
3. 若 `min_idx ≠ i`，则交换 `arr[i]` 与 `arr[min_idx]`
4. 将 `i` 加 1，进入下一轮

Python 实现如下：

```python
def selection_sort(arr):
    n = len(arr)
    for i in range(n - 1):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr
```

外层循环执行 n-1 次，内层循环在第 i 轮执行 n-1-i 次比较，总比较次数为固定的 **n(n-1)/2** 次。

### 时间与空间复杂度

选择排序的时间复杂度在**最好、最坏和平均情况下均为 O(n²)**，这一点与插入排序不同——插入排序在数组接近有序时可达 O(n)。这是因为选择排序每轮都必须完整扫描剩余未排序区间，即便数组已经完全有序，也无法提前终止内层循环。

空间复杂度为 **O(1)**，选择排序是原地排序（in-place sort），仅需一个临时变量用于交换，不需要额外分配与数组大小相关的存储空间。

### 稳定性分析

选择排序是**不稳定的排序算法**。考虑数组 `[5a, 5b, 1]`（两个值为5的元素），第一轮选取最小值1后，将 `1` 与 `5a` 交换，得到 `[1, 5b, 5a]`，原本在前的 `5a` 被移到了 `5b` 的后面，相对顺序发生改变。这一不稳定性源于选择排序使用"跳跃式交换"而非相邻元素逐步移位的方式。

## 实际应用

### 嵌入式系统中的少量数据排序

在内存极度受限的单片机（如 Arduino，通常仅有 2KB SRAM）中，选择排序因其 O(1) 额外空间和极少的写操作次数而被采用。对传感器采集的少量数据点（通常 n < 20）进行排序时，O(n²) 的时间代价完全可接受，而最少的内存写入次数有助于延长 Flash 存储器的使用寿命。

### 手工排序的直观模拟

在扑克牌、竞赛排名等人工整理场景中，人们自然地使用选择排序逻辑：从一堆牌中找出最小的拿出来，再从剩余的牌中找次小的。这使选择排序成为算法教学中解释"算法设计模拟人的直觉行为"这一思想的经典案例。

### 与其他算法的性能对比验证

在算法工程实验中，选择排序常被用作基准（baseline）对照组。例如，测试归并排序在 n=10,000 的数组上耗时约 1.3ms，而选择排序在同等规模下耗时约 130ms（Python 环境，约100倍差距），直观验证了 O(n log n) 与 O(n²) 的量级差异。

## 常见误区

### 误区一：认为已排序数组可以让选择排序更快

很多初学者认为，如果输入数组已经有序，选择排序应该能检测到并提前结束。实际上，选择排序**没有任何针对有序输入的优化机制**。对于已排序的数组 `[1, 2, 3, 4, 5]`，选择排序仍然会老老实实执行全部 n(n-1)/2 = 10 次比较（n=5时），只是每轮发现 `min_idx == i` 时不执行交换而已。相比之下，冒泡排序和插入排序在有序数组上均可优化至 O(n)。

### 误区二：认为交换次数少意味着选择排序总是比冒泡排序快

"最多 n-1 次交换"让人误以为选择排序在实践中总优于冒泡排序。但在现代 CPU 上，两者的实际运行速度接近，甚至在某些测试中冒泡排序更快——因为缓存友好的相邻元素访问模式在硬件层面得到了优化。选择排序的交换次数优势只在交换操作具有显著额外成本（如涉及 I/O 或复杂对象复制）时才体现出来。

### 误区三：混淆选择排序与堆排序的关系

堆排序（Heap Sort）本质上是"用堆数据结构加速选取最小值"的优化版选择排序——同样是每轮找最小值放到正确位置，但利用堆将"找最小值"的操作从 O(n) 降至 O(log n)，从而将总复杂度从 O(n²) 降至 O(n log n)。理解这一关系有助于掌握算法优化的思路：**选择排序的瓶颈在于线性扫描查找最小值，而不在于交换操作本身**。

## 知识关联

**前置知识——数组/列表**：选择排序的两个关键操作"下标随机访问"和"元素交换"均直接依赖数组的索引特性。链表结构虽然也可以实现选择排序，但交换操作需要修改指针，实现更复杂，且无法享受数组 O(1) 随机访问的便利。

**通往堆排序的桥梁**：掌握选择排序后，学习者可以提出"能否不用线性扫描而更快地找到最小值？"这个问题，自然引出优先队列（Priority Queue）和堆（Heap）的概念，进而理解堆排序为何能在保持选择排序"每次取最优"思路的同时实现 O(n log n) 的时间复杂度。

**与插入排序的对比学习**：选择排序每轮确定"把哪个元素放到当前位置"（找最小值），插入排序每轮确定"把当前元素插入到哪个位置"（找插入点）。两者均维护已排序前缀，但遍历方向和操作策略相反，对比学习能加深对两类排序策略的本质理解。
