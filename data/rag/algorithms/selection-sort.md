# 选择排序

## 概述

选择排序（Selection Sort）是一种基于"逐轮找最小值"思路的原地比较排序算法。其基本过程是：将数组分为已排序区间和未排序区间，每轮从未排序区间中找出最小（或最大）的元素，将其与未排序区间的第一个元素交换，从而使已排序区间逐步扩大，直到整个数组排序完成。

选择排序的思想最早可追溯到计算机科学发展初期，是教学中最常被引用的"朴素排序"之一。与冒泡排序不同，选择排序每轮只进行**一次交换操作**，而不是多次相邻交换，这使得它在交换代价高昂的场景（如链式存储结构或磁盘写入）中略具优势。然而，其时间复杂度为 O(n²)，在数据量大时效率较低，实际工程中较少直接使用，但它是理解排序算法思想的重要入门模型。

## 核心原理

### 算法步骤与伪代码

设数组长度为 n，选择排序执行 n-1 轮迭代（第 n 个元素无需比较）。每轮 i（从 0 开始）的操作如下：

1. 假设当前最小值下标为 `min_idx = i`
2. 遍历下标 j 从 i+1 到 n-1，若 `arr[j] < arr[min_idx]`，则更新 `min_idx = j`
3. 将 `arr[min_idx]` 与 `arr[i]` 交换

以 Python 表示：

```python
def selection_sort(arr):
    n = len(arr)
    for i in range(n - 1):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr
```

### 时间与空间复杂度分析

选择排序的**比较次数**与输入数据的排列顺序无关，始终为：

$$C = \frac{n(n-1)}{2}$$

对于 n=10，比较次数固定为 45 次；n=100 时为 4950 次。这意味着无论数组是已排序、逆序还是随机排列，选择排序的比较次数都相同，因此最好、平均、最坏时间复杂度均为 **O(n²)**。

交换次数最多为 n-1 次（每轮至多一次），最少为 0 次（数组已排序）。空间复杂度为 **O(1)**，因为只使用了 `min_idx` 等常数个额外变量，是原地排序算法。

### 稳定性问题

选择排序是**不稳定**的排序算法。具体原因在于：交换操作可能将一个元素跳过另一个与其相等的元素。例如，对数组 `[5a, 5b, 2]`（5a 和 5b 值相同但顺序需保留），第一轮选出最小值 2 与 5a 交换，得到 `[2, 5b, 5a]`，原本 5a 在 5b 之前的相对顺序被破坏。这是选择排序与归并排序、插入排序在稳定性上的关键区别。

## 实际应用

### 嵌入式系统中的小数组排序

在内存极度受限的嵌入式设备（如单片机，RAM 可能仅有几 KB）中，选择排序的 O(1) 空间复杂度使其成为对少量传感器数据（例如 8~16 个采样值）进行排序时的实用选择。此类场景下数组规模 n 很小，O(n²) 的时间代价可以忽略不计。

### 数据可视化与算法教学演示

选择排序因其行为规律性强（每轮恰好确定一个最终位置的元素），被广泛用于算法可视化工具中。例如在 Visualgo 和 Sorting.at 等平台上，选择排序的每一步"扫描+单次交换"过程比冒泡排序更易于逐步演示，帮助学生直观理解"已排序区间逐步扩大"的概念。

### 找 Top-K 元素的部分排序

若只需从 n 个元素中找出最小的 k 个元素（k << n），可以只执行选择排序的前 k 轮，时间复杂度降为 O(k·n)。例如从 1000 个数据中找最小的 3 个，只需进行 3 轮扫描（共约 3000 次比较），而无需完整排序。

## 常见误区

### 误区一：认为已排序数组上选择排序更快

很多初学者认为"输入已是有序数组时，选择排序会提前结束"。实际上并非如此——选择排序没有任何提前终止机制，即使数组已完全有序，它仍会执行完整的 n(n-1)/2 次比较。这与插入排序形成鲜明对比：插入排序在已排序数组上的时间复杂度为 O(n)，而选择排序始终是 O(n²)。

### 误区二：混淆"交换少"与"整体更快"

选择排序每轮只做一次交换，最多 n-1 次交换，初学者容易认为这使它"总体上比冒泡排序快很多"。但决定时间复杂度的主要操作是**比较**，而非交换。两者比较次数同为 O(n²)，冒泡排序经过优化（添加有序标志位）可以在最优情况下达到 O(n)，反而优于选择排序。选择排序的"交换少"优势仅在交换操作本身代价极高时才有实际意义。

### 误区三：认为选择排序可以通过简单修改变为稳定排序

有观点认为将"交换"改为"移位插入"即可使选择排序稳定。虽然这在技术上可行（每次找到最小值后，将其插入到正确位置而非直接交换，时间复杂度变为 O(n²) 但常数更大），但修改后的算法本质上已接近插入排序的变体，失去了选择排序"单次交换"的核心特征，工程上不具备实用意义。

## 知识关联

**前置概念：数组/列表**
选择排序的两个核心操作——按下标访问元素（`arr[j]`）和原地交换（`arr[i], arr[min_idx] = arr[min_idx], arr[i]`）——直接依赖数组的随机访问特性。链表虽然可以实现选择排序的逻辑，但因缺乏 O(1) 随机访问，需要额外的指针操作，实现复杂度更高。

**横向对比：与插入排序和冒泡排序的关系**
三者同属 O(n²) 的简单排序算法。选择排序的交换次数最少（≤ n-1 次），但最优情况下的比较次数不可优化；插入排序在近似有序数据上表现最佳（最优 O(n)）；冒泡排序最直观但交换次数最多。理解选择排序与这两者的差异，能帮助在不同约束条件下（内存、写入次数、数据分布）做出合理的算法选型判断。
