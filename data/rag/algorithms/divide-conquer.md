# 分治法

## 概述

分治法（Divide and Conquer）是一种将大规模问题拆分为若干个结构相同的小规模子问题、分别求解后再合并结果的算法设计策略。其核心逻辑可表达为递归式：**T(n) = aT(n/b) + f(n)**，其中 a 为子问题个数，b 为规模缩小比例，f(n) 为分拆与合并的额外代价。这个递推关系是分析分治算法时间复杂度的出发点。

分治法的思想可追溯至 1945 年冯·诺依曼（John von Neumann）设计归并排序算法时的系统化表述，此后在 1959 年 Anatolii Karatsuba 提出大整数乘法优化时得到进一步彰显——Karatsuba 算法将两个 n 位整数相乘从 O(n²) 降至 O(n^1.585)，正是分治策略突破朴素算法效率上限的经典案例。

在 AI 工程场景中，分治法直接影响着数据预处理流水线的设计：大批量训练数据的并行切分、K-D 树的空间划分、决策树的递归分裂规则，均以分治为底层逻辑。掌握分治法能帮助工程师将看似无从下手的 O(n²) 问题改写为可并行化的 O(n log n) 方案。

## 核心原理

### 三步骤：分解、解决、合并

分治法的执行过程严格遵循三个阶段：

1. **分解（Divide）**：将规模为 n 的原问题拆成 a 个规模为 n/b 的子问题。关键要求是子问题必须与原问题**同质**（结构相同），否则无法递归调用同一求解函数。
2. **解决（Conquer）**：对每个子问题递归调用算法本身；当规模缩小至基础情形（base case）时直接求解，通常设定为 n ≤ 1 或 n ≤ 某常数阈值。
3. **合并（Combine）**：将子问题的解整合为原问题的解。合并步骤的复杂度直接决定整体效率——归并排序的合并是 O(n)，而二分查找的"合并"仅需 O(1)。

### 主定理（Master Theorem）

主定理给出了求解 T(n) = aT(n/b) + Θ(n^d) 形式递推式的三条规则：

- 若 d < log_b(a)，则 T(n) = Θ(n^(log_b(a)))——子问题工作量主导
- 若 d = log_b(a)，则 T(n) = Θ(n^d · log n)——两者均衡
- 若 d > log_b(a)，则 T(n) = Θ(n^d)——合并工作量主导

以归并排序为例：a=2，b=2，d=1，因 log₂2=1=d，所以复杂度为 Θ(n log n)。主定理是工程师评估分治算法性能时**最直接**的量化工具，无需逐层展开递归树。

### 递归树可视化

当主定理不适用（例如子问题规模不均等，如快速排序最坏情形）时，需要手动展开递归树。递归树的每一层代表一次分解，树高为 log_b(n)，每层的合并代价累加即为总复杂度。快速排序在随机输入下期望树高为 O(log n)，但最坏情形（已排序数组选首元素为枢轴）退化为 O(n) 层深的链状树，导致 O(n²) 时间。

## 实际应用

**最近点对问题（Closest Pair of Points）**：给定平面上 n 个点，朴素枚举需 O(n²) 次距离计算。分治策略将点集按 x 坐标中位数切分为左右两半，递归求出各半的最近距离 δ，再在宽度为 2δ 的中间条带内检验跨区域点对。条带内每个点最多与 7 个点比较（可由鸽巢原理证明），合并步骤为 O(n)，总体降至 **O(n log n)**。

**大整数乘法（Karatsuba 算法）**：将两个 n 位数 x、y 各拆为高低两段：x = x₁·10^(n/2) + x₀，y = y₁·10^(n/2) + y₀。朴素分治需要 4 次乘法，而 Karatsuba 通过计算 (x₁+x₀)(y₁+y₀) 将递归调用减至 **3 次**，对应 T(n) = 3T(n/2) + O(n)，由主定理得 O(n^log₂3) ≈ O(n^1.585)。

**K-D 树构建**：在机器学习的 KNN 算法中，K-D 树通过分治在每一维度的中位数处切割空间，将 n 个 k 维数据点组织为二叉树结构。构建过程为 O(kn log n)，查询期望复杂度降至 O(k·n^(1-1/k))，相比暴力搜索在低维特征空间（k ≤ 20）中效果显著。

**矩阵快速幂**：在 AI 工程中计算状态转移矩阵的 n 次幂时，利用分治可将矩阵乘法次数从 O(n) 降至 O(log n)——每次将指数折半，奇数情形多乘一次原矩阵，偶数情形直接平方。

## 常见误区

**误区一：认为所有分治都能达到 O(n log n)**
分治的效率取决于子问题数量 a 和合并代价 f(n) 的具体值。若 a 过大（如 a=4，b=2），主定理给出 Θ(n²)，与朴素算法相当甚至更差（因为额外的递归调用开销）。Strassen 矩阵乘法将 a 从 8 降至 7，才能使复杂度从 O(n³) 降至 O(n^2.807)。因此**减少子问题数量**才是分治优化的关键杠杆。

**误区二：忽略 base case 导致无限递归**
分治的递归终止条件不能仅凭直觉设定。若 base case 设为 n=0 而实际输入会传入 n=1 的子问题，或者在浮点数场景下因舍入使子问题规模不严格缩小，均会导致栈溢出。工程实践中常见的处理方式是：当 n 小于某个阈值（如 n < 32）时，切换为插入排序等非递归方法，这也是 Timsort 算法的设计思路之一。

**误区三：混淆分治与动态规划的适用场景**
分治要求子问题**相互独立**——归并排序的左右两半互不依赖。一旦子问题之间存在重叠计算（如斐波那契数列 F(n) = F(n-1)+F(n-2)），单纯分治会引入指数级重复计算，此时应改用动态规划缓存子问题结果。判断标准：画出递归树后若出现相同参数的节点被多次访问，则需要记忆化或 DP。

## 知识关联

**前置概念——递归**：分治法的"解决"阶段必须依赖递归调用，理解递归的调用栈、终止条件和栈帧开销是读懂分治代码的前提。分治与纯递归的区别在于：分治强调将问题**规模缩小至 1/b**，而不是仅减少 1，这正是分治能达到对数深度的根本原因。

**后续概念——归并排序与快速排序**：归并排序是分治"均匀分解"策略的教科书实现（始终二分），快速排序则是"分解不均匀但合并代价极低"的变体，两者共同展示了主定理中平衡与非平衡分治的性能差异。

**后续概念——线段树**：线段树的构建、区间查询和区间更新全部基于分治思想——每次将查询区间拆分为左右子区间递归处理。掌握分治的"分解-合并"框架后，线段树的懒标记（lazy propagation）机制可理解为"延迟合并"的优化变体，其操作复杂度 O(log n) 亦由分治的递归深度直接决定。
