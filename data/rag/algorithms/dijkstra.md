# Dijkstra最短路径

## 概述

Dijkstra最短路径算法由荷兰计算机科学家Edsger W. Dijkstra于1956年构思，并于1959年发表在论文《A note on two problems in connexion with graphs》中。该算法解决的是带权有向图（或无向图）中**单源最短路径**问题——即从一个固定起点出发，计算到图中所有其他节点的最短距离。据Dijkstra本人回忆，他在阿姆斯特丹的一家咖啡馆里仅用20分钟就完成了这个算法的构思，且最初设计时甚至没有使用纸和笔。

Dijkstra算法的关键约束是：**所有边的权重必须为非负数**。这一限制源于算法的贪心策略——它假定一旦某节点被确定为"已访问"，其最短距离就不可能再被更新。如果存在负权边，这一假设将不成立，算法会产生错误结果。这正是Dijkstra与BFS的本质区别：BFS处理无权图（或等权图），依赖层级扩展；Dijkstra处理带权图，依赖累积距离的贪心选择。

在AI工程领域，Dijkstra算法是路径规划、网络路由（如OSPF协议直接基于Dijkstra）以及地图导航系统的基础模块。理解其时间复杂度在不同实现下从O(V²)到O((V+E)logV)的演变，是选择高效路径算法的前提。

## 核心原理

### 贪心松弛机制

Dijkstra算法维护一个距离数组`dist[]`，初始时`dist[source] = 0`，其余所有节点的距离设为正无穷（∞）。每次迭代从**未访问节点中选取dist值最小的节点u**，然后对u的所有邻居v执行**松弛操作**：

```
if dist[u] + weight(u, v) < dist[v]:
    dist[v] = dist[u] + weight(u, v)
```

这个松弛公式是算法的核心驱动力。每次松弛成功意味着找到了一条更短的路径前缀。节点u一旦被选出并标记为已访问，其`dist[u]`值就是从源点到u的真实最短距离，这一性质由归纳法可以严格证明。

### 时间复杂度与数据结构选择

算法的时间复杂度直接取决于如何寻找"未访问节点中的最小dist值"：

- **朴素实现（线性扫描）**：每次扫描全部V个节点找最小值，总复杂度为 **O(V²)**，适合稠密图（边数接近V²）。
- **二叉堆（优先队列）实现**：使用最小堆维护待处理节点，每次取最小值耗时O(logV)，总复杂度为 **O((V+E)logV)**，适合稀疏图。
- **斐波那契堆实现**：将复杂度进一步降至 **O(E + VlogV)**，是理论最优，但实现复杂，常数因子大，工程中较少使用。

以Python的`heapq`实现为例，每次将`(dist[v], v)`推入堆中，同一节点可能被多次入堆，需要通过检查当前距离是否等于`dist[v]`来跳过过时条目。

### 算法执行步骤（以具体图为例）

考虑节点{A, B, C, D}，边权：A→B=4, A→C=2, C→B=1, B→D=5, C→D=8。

1. 初始：`dist = {A:0, B:∞, C:∞, D:∞}`，选A
2. 松弛A的邻居：`dist = {A:0, B:4, C:2, D:∞}`，选C（dist最小=2）
3. 松弛C的邻居：`dist = {A:0, B:3, C:2, D:10}`，B从4更新为2+1=3，选B（dist=3）
4. 松弛B的邻居：`dist = {A:0, B:3, C:2, D:8}`，D从10更新为3+5=8，选D
5. 最终结果：A到D的最短路径为A→C→B→D，距离为8

## 实际应用

**地图导航系统**：Google Maps和高德地图在处理道路网络时，将交叉口建模为节点，道路建模为带权边（权重可为距离、时间或费用）。由于真实道路权重非负，Dijkstra是计算两点间最短行驶路线的基础算法。实际产品中会结合双向Dijkstra（从起点和终点同时搜索，速度提升约2倍）来加速计算。

**网络路由协议（OSPF）**：开放最短路径优先协议（OSPF，RFC 2328）在每台路由器上运行Dijkstra算法，基于链路状态数据库计算到各目标网络的最短路径树。路由器将链路代价（如带宽的倒数）作为边权，Dijkstra的非负权假设天然满足。

**AI游戏路径规划**：在早期电子游戏（如《星际争霸》地图网格）中，Dijkstra用于计算单位移动的最低代价路径。网格中不同地形（平地、山地、水域）被赋予不同移动代价，算法输出最优移动序列。不过当目标明确且有地理启发信息时，现代游戏引擎更倾向使用A*算法替代Dijkstra。

## 常见误区

**误区一：认为Dijkstra能处理负权边**

许多初学者误以为只要最终路径权重和为正，算法就能正确运行。实际上，负权边会破坏算法的"已访问节点距离不再更新"这一核心假设。例如，若存在边B→A=-10，则当A已被标记为已访问后，通过B再绕回A可能得到更短的路径，但Dijkstra不会重新处理A。处理负权边需改用Bellman-Ford算法（允许负权但无负权环）或SPFA算法。

**误区二：混淆Dijkstra与BFS的适用场景**

BFS在无权图（或所有边权相等时）能以O(V+E)找到最短路径，此时用Dijkstra反而增加了log因子的开销。反之，对带不同权重的图直接套用BFS，会错误地以"跳数"而非"权重和"作为最短路径标准。两者的正确选择依据是：**边权是否全部相等**。

**误区三：认为优先队列实现中节点只会入堆一次**

使用懒删除（lazy deletion）的堆实现中，同一节点v可能因多次被松弛而多次加入堆。若不判断取出节点时其dist值是否已经是最优值，将导致重复处理，虽不影响正确性但会降低效率。正确做法是在取出堆顶`(d, v)`时，若`d > dist[v]`则跳过，确保每个节点的最终最短路径只被处理一次。

## 知识关联

**与BFS的关系**：BFS是Dijkstra在边权全为1时的特例。BFS用队列（FIFO）按层展开，等价于Dijkstra在所有边权相等时优先队列的退化形式。掌握BFS的层级扩展逻辑，有助于理解为什么Dijkstra把"层"替换成"累积权重"后就能处理带权图。

**通向Bellman-Ford**：Bellman-Ford算法通过对所有边进行V-1轮松弛，放弃了Dijkstra的贪心选择，因此能处理负权边，代价是时间复杂度从O((V+E)logV)上升至O(VE)。两者都使用相同的松弛公式，区别在于松弛的顺序和终止条件。

**通向A\*算法**：A\*是Dijkstra加入启发函数h(v)（估计v到目标的剩余距离）后的变体，将优先队列的排序键从`dist[v]`改为`dist[v] + h(v)`。当h(v)=0时，A\*退化为Dijkstra；当h(v)是可采纳的（不高估实际距离），A\*保证最优性且通常比Dijkstra扩展更少节点。

**通向Floyd-Warshall**：Dijkstra解决单源最短路径（一个起点到所有终点），而Floyd-Warshall解决全源最短路径（所有节点对之间的最短距离），使用动态规划而非贪心，时间复杂度为O(V³)，可处理负权边（但不能有负权环）。
