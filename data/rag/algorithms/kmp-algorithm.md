# KMP算法

## 概述

KMP算法（Knuth-Morris-Pratt算法）是一种线性时间复杂度的字符串匹配算法，由Donald Knuth、Vaughan Pratt和James Morris于1977年联合发表在《Journal of the ACM》上。其核心突破在于：当模式串（pattern）与文本串（text）发生失配时，算法利用已匹配部分的内部结构信息，避免文本串指针回退，从而将朴素匹配算法O(m×n)的最坏时间复杂度降低到O(m+n)，其中m为模式串长度，n为文本串长度。

KMP算法的革命性思想来自对"已匹配前缀"的重利用。朴素算法每次失配后将模式串右移一位、文本指针回退，而KMP通过预先计算模式串的"部分匹配表"（即next数组），直接跳转到下一个可能匹配的位置。这一思想在AI工程中的文本检索、日志分析、生物信息学序列比对等场景中均有直接应用，是理解更复杂字符串算法（如Aho-Corasick多模式匹配）的必要前置。

## 核心原理

### next数组的定义与构建

next数组是KMP算法的灵魂，其定义为：`next[i]`表示模式串`P[0..i]`中**最长公共前后缀**的长度（不含自身）。例如，模式串`"ABABC"`的next数组为`[-1, 0, 0, 1, 2]`（使用-1标记边界的版本）。

以`"ABAAB"`为例逐步推导：
- `next[0] = -1`（边界初始化）
- `next[1] = 0`：`"A"`无公共前后缀
- `next[2] = 0`：`"AB"`无公共前后缀
- `next[3] = 1`：`"ABA"`最长公共前后缀为`"A"`，长度1
- `next[4] = 1`：`"ABAA"`最长公共前后缀为`"A"`，长度1
- `next[5] = 2`：`"ABAAB"`最长公共前后缀为`"AB"`，长度2

构建next数组的代码逻辑使用双指针：指针`k`追踪当前最长前后缀长度，指针`i`遍历模式串。当`P[i] == P[k]`时，`next[i+1] = k+1`；失配时令`k = next[k]`递归回跳，整个构建过程时间复杂度为**O(m)**。

### 匹配过程详解

匹配阶段使用两个指针：`i`指向文本串当前位置，`j`指向模式串当前位置。匹配规则如下：

- **匹配成功**：`T[i] == P[j]`，同时推进`i++, j++`
- **匹配失败且`j > 0`**：令`j = next[j]`，**`i`不回退**，这是KMP相比朴素算法的核心优势
- **匹配失败且`j == 0`**：`i++`，从头开始尝试

当`j == m`（模式串长度）时，找到一处匹配，位置为`i - m`，随后令`j = next[j]`继续寻找下一个匹配位置。

以文本串`"AABABAC"`匹配模式串`"ABABC"`为例，在位置`i=4, j=4`处失配（`T[4]='B'`与`P[4]='C'`不符），此时`j = next[4] = 2`，直接跳到模式串第3个字符继续比较，跳过了朴素算法中大量冗余比较。

### 时间与空间复杂度分析

KMP算法的总时间复杂度为**O(m+n)**，其中预处理next数组消耗O(m)，匹配过程消耗O(n)。空间复杂度为**O(m)**，仅需存储next数组。

可以用摊还分析证明匹配阶段是线性的：文本串指针`i`只增不减，而`j`的总减少量不能超过其总增加量（最多增加n次），因此整个匹配过程比较次数上界为**2n**次。

## 实际应用

**日志异常检测**：在AI系统的实时日志流中，用KMP匹配特定错误模式串（如`"OutOfMemoryError"`），对长达数GB的日志文件单次扫描即可完成，比正则引擎在固定模式下效率更高。

**生物信息学序列比对**：DNA序列中寻找特定基因片段（如`"ATCGATCG"`在人类基因组约30亿碱基对中的位置），KMP的O(n)保证使其成为基础匹配工具。BLAST等高级工具的局部搜索步骤也包含类似思想。

**中文分词预处理**：在基于词典的中文分词系统中，KMP可用于在文本中快速定位词典词条，配合Aho-Corasick算法扩展为多模式同时匹配，支撑每秒处理百万级字符的分词需求。

**编译器词法分析**：源代码中的关键字识别（如在C++源文件中定位`"template"`、`"typename"`等关键字串）可用KMP做快速扫描，作为词法分析器的预筛阶段。

## 常见误区

**误区一：next数组下标从0还是从1**。KMP算法有多种实现版本，`next[0] = -1`（0-indexed）与`next[1] = 0`（1-indexed）在初始化和失配回跳逻辑上写法不同，混用会导致死循环或漏匹配。统一原则是：`next[j]`表示失配时`j`应跳转到的位置，要与具体实现的下标约定严格一致。

**误区二：将next数组理解为"跳过字符数"而非"跳转位置"**。`next[j]`的含义是模式串`P[0..j-1]`的最长公共真前后缀长度，也直接等于失配后`j`的新值（在0-indexed的k值法中）。它不是"跳过了多少字符"，而是"下一次从模式串第几位开始比"。初学者常将其与Boyer-Moore算法的坏字符跳转距离混淆。

**误区三：KMP在所有场景下都优于朴素算法**。实测表明，对于随机英文文本（字符集大、失配发生早），朴素算法平均比较次数接近O(n)，实际速度因缓存友好性甚至快于KMP。KMP的优势在**高重复度模式串**（如`"AAAAAAB"`）和**二进制/DNA小字符集**场景中最为显著，此时朴素算法退化而KMP保持线性。

## 知识关联

**前置概念**：学习KMP前需掌握字符串匹配的朴素算法（暴力双指针），理解其O(m×n)最坏情况来自文本指针的反复回退。KMP的next数组本质上是对模式串做了一次"自我匹配"，理解这一点需要熟悉前缀函数（Prefix Function）的概念。

**后续概念**：Rabin-Karp算法采用完全不同的哈希滚动思路解决多模式匹配，其哈希碰撞处理策略与KMP的确定性跳转形成方法论对比。掌握KMP后，可自然延伸到Aho-Corasick算法——后者将KMP的next数组推广为基于Trie树的失败指针（failure link），实现同时匹配数千个模式串的O(n+m₁+m₂+...+mₖ)效率，是工业级多关键词过滤系统的标准实现基础。
