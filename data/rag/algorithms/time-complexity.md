# 时间复杂度（Big-O 记号）

## 概述

时间复杂度是衡量算法运行时间随输入规模 n 增长而变化的数学描述方式，用大 O 记号（Big-O notation）表示。它由德国数学家保罗·巴赫曼（Paul Bachmann）于 1894 年在《解析数论》中首次引入，后经埃德蒙·朗道（Edmund Landau）系统推广，因此有时也称为"朗道符号"。Big-O 描述的是**最坏情况**下算法运行步骤数量的上界，而非实际的秒数或毫秒数，这使得分析结果独立于硬件性能和编程语言。

在 AI 工程中，时间复杂度直接决定一个算法是否能用于生产环境。例如，暴力搜索所有可能的神经网络权重组合是 O(2ⁿ) 的操作，对于 n=100 个参数已经在宇宙寿命内无法完成；而反向传播算法将同一问题压缩到 O(n)，才使深度学习成为可能。理解 Big-O 是判断"这个算法能跑多大的数据集"的基础量化工具。

## 核心原理

### Big-O 的数学定义

若存在正常数 c 和 n₀，使得对所有 n ≥ n₀ 均有 f(n) ≤ c·g(n)，则称 f(n) = O(g(n))。这里 f(n) 是算法实际执行的操作数，g(n) 是我们用来描述上界的函数（如 n²、n log n 等）。关键在于**忽略常数系数和低阶项**：一个执行 3n² + 7n + 100 次操作的算法，其时间复杂度记为 O(n²)，因为当 n 足够大时，7n 和 100 的影响可以忽略不计。

### 常见复杂度类别与增长速度

从快到慢排列，AI 工程中最常见的复杂度类别为：

- **O(1)**：哈希表查找，无论输入多大，单次操作时间恒定。
- **O(log n)**：二分查找，每次将问题规模减半，n = 10⁶ 时仅需约 20 步。
- **O(n)**：遍历一次数组，如计算前缀和的线性扫描。
- **O(n log n)**：归并排序和堆排序的时间复杂度，是基于比较的排序算法的理论下界。
- **O(n²)**：双重 for 循环遍历二维矩阵的每对元素，如朴素的矩阵乘法中间步骤。
- **O(2ⁿ)**：穷举子集，n = 30 时已有约 10⁹ 种情况，在实践中通常不可接受。

将 n = 10,000 代入各复杂度：O(log n) ≈ 13 步，O(n) = 10,000 步，O(n²) = 100,000,000 步，差距高达千万倍。

### 如何从代码结构推导复杂度

**单层 for 循环**遍历长度为 n 的数组，执行 n 次操作，复杂度为 O(n)。**嵌套双层 for 循环**（外层 n 次 × 内层 n 次）复杂度为 O(n²)。若内层循环每次将索引翻倍（`i *= 2`），则内层执行 log₂n 次，总复杂度降为 O(n log n)。**递归**的复杂度由递推关系式决定：二分查找的递推式为 T(n) = T(n/2) + O(1)，解得 T(n) = O(log n)；斐波那契数列朴素递归的递推式为 T(n) = T(n-1) + T(n-2) + O(1)，解得 T(n) = O(2ⁿ)，这正是为什么必须用动态规划将其优化至 O(n)。

### 主定理（Master Theorem）简介

对于形如 T(n) = aT(n/b) + O(nᵈ) 的递推关系，主定理给出三种情形下的封闭解：
- 若 d > log_b(a)，则 T(n) = O(nᵈ)
- 若 d = log_b(a)，则 T(n) = O(nᵈ log n)
- 若 d < log_b(a)，则 T(n) = O(n^(log_b a))

例如归并排序满足 T(n) = 2T(n/2) + O(n)，此时 a=2, b=2, d=1，log₂2=1=d，代入第二条得 T(n) = O(n log n)。

## 实际应用

**KNN（K近邻）分类器**在预测阶段需要计算查询点与所有 n 个训练样本的距离，若特征维度为 d，则单次预测的时间复杂度为 O(n·d)。当 n = 1,000,000、d = 512（如图像特征向量）时，每次推理需要约 5 亿次浮点运算，这正是工程上必须引入近似最近邻（ANN）算法（如 HNSW，复杂度降为 O(log n)）的直接原因。

**Transformer 注意力机制**的时间复杂度为 O(n²·d)，其中 n 是序列长度，d 是维度。这意味着将输入文本从 512 个 token 扩展到 4096 个 token 时，计算量增加 64 倍，这一瓶颈直接催生了 Sparse Attention、FlashAttention 等优化方法，将复杂度压缩至 O(n·√n·d) 或在硬件层面减少内存访问次数。

**排序的工程选择**：Python 内置的 `sorted()` 函数使用 Timsort 算法，最坏情况 O(n log n)，最好情况（已排序数据）O(n)。在 AI 数据预处理管道中，对 10⁸ 条记录排序时，选择 O(n log n) 而非 O(n²) 的冒泡排序，可将运行时间从数天缩短至数分钟。

## 常见误区

**误区一：O(n²) 总是比 O(n log n) 慢。** Big-O 忽略了常数系数。若某 O(n²) 算法的实际操作数为 0.001·n²，而某 O(n log n) 算法的实际操作数为 1000·n·log n，则在 n < 约 10,000 时前者反而更快。因此，对于小规模数据（如 n < 50），插入排序（O(n²)）由于常数极小，实践中常比快速排序（O(n log n)）更快，这也是 Timsort 对小数组回退使用插入排序的原因。

**误区二：时间复杂度等于运行时间。** Big-O 度量的是操作次数的增长趋势，而非实际运行的毫秒数。同样是 O(n) 的算法，一个涉及内存缓存命中率高的顺序访问，和一个频繁触发缓存缺失的随机访问，实际运行时间可能差 10 倍以上。GPU 并行化也可以让 O(n²) 的矩阵乘法在实际中比某些理论上更优的算法运行更快。

**误区三：只需考虑最坏情况。** Big-O 描述最坏情况，但快速排序（Quicksort）的平均情况是 O(n log n)，最坏情况（每次选到最大/最小值作为 pivot）才是 O(n²)。对于随机数据，快速排序的实际表现优于堆排序，尽管两者最坏情况不同。此外还存在 Big-Ω（下界/最好情况）和 Big-Θ（上下界紧确）记号，三者共同构成完整的渐进分析体系。

## 知识关联

**前置概念**：分析 for 循环的时间复杂度要求能识别循环的迭代次数，这依赖于对**循环（for/while）**控制流的理解；分析数组操作（如遍历、随机访问）的复杂度需要知道**数组/列表**的内存结构，其中随机访问为 O(1)、顺序遍历为 O(n)。

**后续概念**：**空间复杂度**是与时间复杂度对称的概念，同样使用 Big-O 记号，度量算法所需内存随输入规模的增长，两者往往存在时空权衡（time-space tradeoff）。**前缀和**是将数组区间求和从 O(n) 预处理换来 O(1) 查询的经典技术，是 Big-O 分析指导算法优化的典型案例。**贪心算法**的设计目标往往是将暴力枚举的指数复杂度降至多项式复杂度，需要用 Big-O 验证优化是否有效。**NP 完全性**则将 Big-O 的分析推广至计算复杂性理论，讨论是否存在多项式时间算法解决某类问题。
