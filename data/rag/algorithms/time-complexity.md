# 时间复杂度（Big-O 记号）

## 概述

时间复杂度是衡量算法随输入规模增长时，执行步骤数增长趋势的数学工具。它不关心算法在特定机器上跑了多少毫秒，而是描述"当输入从 n 变成 2n 时，计算量如何变化"。这种机器无关的视角由德国数学家 Paul Bachmann 于 1894 年在著作《解析数论》中首次引入，后经 Edmund Landau 推广，因此 Big-O 也称为 Bachmann–Landau 记号。

Big-O 记号的形式定义为：若存在正常数 C 和 n₀，使得对所有 n ≥ n₀ 都有 f(n) ≤ C·g(n)，则称 f(n) = O(g(n))。换言之，g(n) 是 f(n) 增长速度的一个上界（渐近上界）。在 AI 工程中，模型推理、特征检索、图搜索等操作的性能瓶颈几乎都需要用 Big-O 来预判——O(n²) 的算法在 n=10,000 时已需执行亿级操作，实时系统根本无法接受。

## 核心原理

### 常见复杂度等级及含义

以下七个等级按增长速度从慢到快排列，是 AI 工程师必须记住的基准：

| 记号 | 名称 | 典型操作 |
|------|------|----------|
| O(1) | 常数 | 哈希表单次查询 |
| O(log n) | 对数 | 二分搜索 |
| O(n) | 线性 | 遍历数组 |
| O(n log n) | 线性对数 | 归并排序 |
| O(n²) | 平方 | 冒泡排序、朴素矩阵乘法 |
| O(2ⁿ) | 指数 | 穷举子集搜索 |
| O(n!) | 阶乘 | 旅行商问题暴力解法 |

当 n=1,000 时，O(n log n) 约需 10,000 步，而 O(n²) 需要 1,000,000 步，差距已达 100 倍。

### 如何分析代码的时间复杂度

**循环嵌套决定乘法规则**：一个 for 循环执行 n 次，内嵌另一个 for 循环也执行 n 次，总步骤为 n×n = n²，即 O(n²)。两个顺序执行的独立 for 循环则为 n+n = 2n，因为 Big-O 舍弃常数系数，化简为 O(n)。

**取最高阶项**：若某算法完整复杂度为 f(n) = 3n³ + 100n² + 500，则 Big-O 只保留最高阶 n³，系数 3 也去掉，结果为 O(n³)。这是因为当 n 趋于无穷时，低阶项的贡献可忽略不计。

**递归的复杂度用递推式分析**：以归并排序为例，其递推式为 T(n) = 2T(n/2) + O(n)，用主定理（Master Theorem）求解得 T(n) = O(n log n)。主定理适用于形如 T(n) = aT(n/b) + O(nᵈ) 的递推，其中 a≥1，b>1，根据 log_b(a) 与 d 的大小关系决定最终复杂度。

### 最优、最坏与平均情形

Big-O 默认描述最坏情形（worst case），但三种情形各有专用记号：最坏用 O，最优用 Ω（Big-Omega），平均用 Θ（Big-Theta）。以快速排序为例：随机输入时平均复杂度为 Θ(n log n)，但若每次选到最小元素作为 pivot（如输入已有序），最坏情形退化为 O(n²)。实际工程中，分析最坏情形是保障系统稳定性的基本要求。

## 实际应用

**K 近邻（KNN）推理的复杂度优化**：朴素 KNN 在 n 个训练样本、d 维特征下，单次预测需计算 n 次欧氏距离，每次距离计算耗时 O(d)，总复杂度 O(nd)。当 n=1,000,000，d=128 时，每次推理需 1.28 亿次浮点运算，无法实时响应。引入 KD 树后，平均查询复杂度降至 O(d·log n)，但高维（d>20）时 KD 树退化，需改用近似最近邻（ANN）算法如 HNSW，将复杂度控制在 O(log n) 级别。

**矩阵乘法与神经网络前向传播**：全连接层的矩阵乘法 A(m×k) × B(k×n) 朴素实现为 O(mkn)。深度学习框架实际使用 Strassen 算法将 2×2 矩阵乘法从 8 次乘法降至 7 次，理论渐近复杂度约为 O(n^2.807)，而目前已知最优的 Coppersmith–Winograd 系列算法已将其压缩至约 O(n^2.373)，这直接影响大模型训练的计算成本。

**前缀和的 O(1) 区间查询**：预处理数组得到前缀和数组需 O(n)，此后任意区间 [l, r] 的求和查询只需 prefix[r] - prefix[l-1]，即 O(1)。若直接用原始数组每次遍历求和则为 O(n)，当需要回答 q 次查询时，总复杂度从 O(nq) 降至 O(n+q)，这是 AI 特征工程中滑动窗口统计的基础技巧。

## 常见误区

**误区一：Big-O 越小的算法一定更快**。O(n log n) 算法的常数系数可能很大，实际上当 n 很小（如 n<50）时，O(n²) 算法因常数极小反而更快。例如 Python 的 `list.sort()` 在小数组阶段会切换到插入排序（O(n²)），正是利用了这一特性。判断实际性能必须结合输入规模和硬件常数，不能只看 Big-O 等级。

**误区二：O(1) 意味着"一步完成"**。O(1) 只代表执行步骤数不随输入规模变化，实际可能包含固定的大量操作。Python 字典查询均摊为 O(1)，但触发哈希冲突链条很长时单次查询可能相当耗时。此外，哈希表在最坏情况下查询复杂度为 O(n)（所有键碰撞），因此安全场景下不能依赖均摊分析。

**误区三：时间复杂度分析中忽略输入的多维度**。图算法的复杂度通常同时依赖节点数 V 和边数 E，如 BFS 为 O(V+E)，不能简写为 O(n)。在自然语言处理中，Transformer 的自注意力机制复杂度为 O(n²·d)，其中 n 是序列长度，d 是特征维度，混淆这两个参数会导致对模型扩展性的错误预判。

## 知识关联

**依赖前置概念**：分析 for/while 循环的嵌套层数是推导 O(n²)、O(n³) 等多项式复杂度的直接方法；数组/列表的索引操作是 O(1) 而顺序遍历是 O(n) 这一区别，是理解"数据结构影响复杂度"的最基础案例。

**通向后续主题**：空间复杂度与时间复杂度使用相同的 Big-O 框架，但度量对象从"操作步骤数"换成"额外内存占用量"，两者存在经典的时间-空间权衡（如用哈希表换时间）。贪心算法的正确性证明通常需要验证其时间复杂度确实优于暴力枚举。NP 完全性理论本质上是在讨论一类问题是否存在多项式时间（即 O(nᵏ) 级别）的算法——理解了 Big-O 的增长等级，才能理解为何指数级算法被视为"不可行"的分水岭。前缀和技巧的价值正是将 O(n) 的重复查询优化为 O(1)，是时间复杂度优化思想的直接工程实践。
