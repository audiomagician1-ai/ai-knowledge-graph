# 动态规划

## 概述

动态规划（Dynamic Programming，简称DP）由数学家理查德·贝尔曼（Richard Bellman）于1950年代正式提出，其名称中的"programming"实为"规划"而非"编程"，源自军事运筹学中的"数学规划"概念。贝尔曼最初将其应用于多阶段决策过程（Multi-stage Decision Process），核心思想是：将一个复杂问题分解为相互关联的子问题，通过存储子问题的解来避免重复计算，从而将指数级别的时间复杂度降低至多项式级别。

动态规划与普通分治法的本质区别在于**子问题重叠性**（Overlapping Subproblems）。以斐波那契数列为例，朴素递归计算 `fib(40)` 需要约 2^40 次操作，而DP通过记录已计算结果将其压缩到 O(n) 次。这一"以空间换时间"的策略在图最短路径、序列比对、资源分配等AI工程场景中具有不可替代的价值——例如自然语言处理中的Viterbi算法本质上就是在隐马尔可夫模型上运行的动态规划。

动态规划之所以在算法设计中占据独特地位，是因为它能够从数学上**保证全局最优解**。只要问题满足其两个核心条件，DP给出的解一定是整个解空间中的最优值，而非贪心算法那样的局部最优。

## 核心原理

### 两个必要条件

动态规划适用的问题必须同时满足以下两个条件，缺一不可：

**1. 最优子结构（Optimal Substructure）**
问题的最优解可以由其子问题的最优解构造而来。形式化表达为：若问题 P 的最优解包含子问题 S 的解，则该子问题的解也必须是 S 自身的最优解。以最短路径问题为例，从城市 A 到城市 C 的最短路径，经过城市 B，则 A→B 段和 B→C 段分别也必须是各自的最短路径。

**2. 子问题重叠（Overlapping Subproblems）**
递归求解过程中，相同的子问题会被反复计算多次。若所有子问题互不相同（如归并排序），则分治法更合适；若子问题大量重复，则DP通过将结果存入表格（通常是数组或哈希表）将冗余计算从指数级降至线性或多项式级。

### 贝尔曼方程

动态规划的数学核心是**贝尔曼最优性方程**：

```
V*(s) = max_a [ R(s, a) + γ · Σ P(s'|s,a) · V*(s') ]
```

其中 `V*(s)` 是状态 s 的最优价值，`R(s,a)` 是即时奖励，`γ ∈ [0,1)` 是折扣因子，`P(s'|s,a)` 是状态转移概率。这一方程直接催生了强化学习中的Q-learning和值迭代算法，是现代AI工程中DP最重要的理论延伸。

### 两种实现策略：自顶向下 vs 自底向上

**自顶向下（Top-Down，记忆化递归）**：保留递归的直觉写法，用一个缓存表记录已计算的子问题结果。代码可读性高，但存在函数调用栈开销，Python中递归深度默认限制为1000层。

**自底向上（Bottom-Up，DP表格法）**：从最小子问题出发，按拓扑顺序填充DP表格，无递归栈风险，通常空间效率更优。例如经典的0/1背包问题，用二维数组 `dp[i][w]` 表示"前i个物品、容量为w时的最大价值"，状态转移方程为：

```
dp[i][w] = max(dp[i-1][w], dp[i-1][w-weight[i]] + value[i])
```

其中 `w >= weight[i]` 时才执行第二个选项，否则 `dp[i][w] = dp[i-1][w]`。

### 状态设计与转移方程

DP解题的关键步骤是**定义状态**——即用什么维度的数组索引来唯一描述子问题。状态维度越少，空间复杂度越低；但若状态定义不完整，转移方程将无法正确推导。以最长公共子序列（LCS）为例，状态 `dp[i][j]` 定义为"字符串A的前i个字符与字符串B的前j个字符的LCS长度"，转移方程为：

```
若 A[i] == B[j]：dp[i][j] = dp[i-1][j-1] + 1
否则：           dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

最终答案存储在 `dp[m][n]` 中，时间与空间复杂度均为 O(m×n)。

## 实际应用

**基因序列比对**：生物信息学中的Smith-Waterman算法和Needleman-Wunsch算法均是DP的直接应用，通过填充评分矩阵求解两条DNA/蛋白质序列的最优局部或全局对齐，时间复杂度为 O(m×n)。

**语音识别与NLP**：Viterbi算法利用DP在隐马尔可夫模型（HMM）中寻找最可能的状态序列，其时间复杂度为 O(T × N²)，其中T为观测序列长度，N为状态数量。若用朴素穷举法，复杂度为 O(N^T)，在实际语音识别任务中完全不可行。

**强化学习中的值迭代**：OpenAI Gym的网格世界环境中，值迭代算法反复应用贝尔曼方程更新每个状态的价值估计，直到收敛（通常定义为连续两次迭代的最大差值 < 10⁻⁶），这是强化学习入门中DP的标准实践场景。

**编辑距离（Levenshtein Distance）**：计算两个字符串之间的最小编辑操作数（插入、删除、替换各计1步），DP解法时间复杂度 O(m×n)，被广泛用于拼写检查、DNA变异分析和模糊字符串匹配。

## 常见误区

**误区1：认为DP总是需要二维数组**
许多初学者看到DP就联想到二维表格，但大量问题只需一维数组。0/1背包问题若按逆序遍历容量维度，可将空间从 O(n×W) 压缩到 O(W)。更极端的情况下，斐波那契数列的DP只需两个变量（O(1) 空间）。状态空间的维度取决于问题本身，而非DP方法的限制。

**误区2：将DP与贪心算法混淆**
贪心算法在每步做出局部最优选择，不回溯，速度快但不保证全局最优。典型反例是硬币找零问题：面值为 {1, 3, 4} 的硬币凑出6分，贪心给出 4+1+1=3枚，而DP找到 3+3=2枚的最优解。判断标准是问题是否满足贪心选择性质（Greedy Choice Property）——若不满足，必须用DP。

**误区3：认为DP一定比递归快**
DP通过消除重复计算加速，但若一个问题的子问题**不重叠**（每个子问题仅被求解一次），DP的额外存储开销反而是浪费。例如在一棵无重复结构的二叉树上做后序遍历，朴素递归与DP的效率相同，没有引入DP的必要。

## 知识关联

**前置概念承接**：动态规划的自顶向下实现直接依赖**递归**的写法框架——DP只是在递归的基础上加入了缓存机制（Memoization）。理解**迭代vs递归**的权衡，有助于判断某个DP问题应采用自顶向下的递归记忆化还是自底向上的迭代填表，前者代码更自然，后者通常更省内存且无栈溢出风险。

**后续概念延伸**：**记忆化搜索**是自顶向下DP的系统化实现技术，专门解决状态转移顺序不明确时的DP问题；**DP表格法**则深入探讨自底向上的填表顺序、边界条件和空间压缩技巧。**稀疏表（Sparse Table）**利用DP的预处理思想，在 O(n log n) 时间内构建结构，支持 O(1) 的区间最值查询。**Floyd-Warshall算法**是DP在全源最短路径问题上的经典应用，其三重循环本质是对中间节点集合的状态转移，状态方程为 `dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])`，是DP思想在图论中最优雅的体现之一。
