# 字符串匹配（KMP）

## 概述

KMP算法（Knuth-Morris-Pratt算法）由Donald Knuth、Vaughan Pratt和James H. Morris于1977年联合发表，是解决字符串匹配问题的经典线性时间算法。其核心问题是：给定文本串T（长度为n）和模式串P（长度为m），找出P在T中所有出现的位置。朴素算法在最坏情况下时间复杂度为O(n×m)，而KMP通过预处理模式串将总复杂度压缩至O(n+m)。

KMP的革命性突破在于引入了"部分匹配表"（又称失配函数或next数组）。当匹配过程中出现字符不一致时，朴素算法会将模式串回退到头部重新匹配，浪费了已经匹配成功的信息。KMP利用模式串自身的前缀后缀重叠结构，使得文本串指针**永不回退**，仅调整模式串指针的位置，从而避免重复比较。

在AI工程领域，KMP广泛应用于日志解析、特征提取管道中的模式定位、以及自然语言处理中的关键词精确匹配场景。相比正则引擎，KMP在固定模式的高频匹配中具有可预测的线性时间保证，对实时流式数据处理尤为关键。

## 核心原理

### 失配函数（next数组）的构建

next数组的定义：`next[i]`表示模式串`P[0..i]`中，**最长真前缀与真后缀相等**的长度。"真"意味着该前/后缀不等于字符串本身。

以模式串 `P = "ABABC"` 为例：

| 索引 i | 子串 | next[i] |
|--------|------|---------|
| 0 | A | 0 |
| 1 | AB | 0 |
| 2 | ABA | 1（"A"既是前缀也是后缀）|
| 3 | ABAB | 2（"AB"既是前缀也是后缀）|
| 4 | ABABC | 0 |

构建next数组的代码逻辑使用双指针：慢指针`i`遍历模式串，快指针`j`追踪当前最长匹配前缀长度。当`P[i] == P[j]`时，`next[i] = ++j`；失配时，`j = next[j-1]`（回退而非归零），整个构建过程的时间复杂度为O(m)。

### 匹配过程中的指针移动策略

KMP匹配时维护两个指针：`i`指向文本串当前位置，`j`指向模式串当前比较位置。匹配成功则同步推进（`i++, j++`）；失配时执行关键操作：

- 若`j > 0`：令`j = next[j-1]`，**i不移动**
- 若`j == 0`：令`i++`，模式串从头开始

这一策略的本质是：已知`P[0..j-1]`与文本某段匹配，且`P[0..next[j-1]-1]`是其最长相等前后缀，因此可以安全地将模式串"滑动"到下一个可能匹配的位置，跳过所有不可能产生匹配的对齐方式。

### 时间复杂度的严格证明要点

KMP的O(n+m)复杂度可通过势能分析（amortized analysis）证明。定义势能函数为当前`j`的值（模式串指针位置）。每次`i`递增时，`j`最多增加1；每次`j`回退时，`j`严格减小。由于`j`的总增加量不超过n次（受i的推进次数限制），`j`的总减小量也不超过n次，故匹配阶段总操作数为O(n)，加上预处理的O(m)，总复杂度为O(n+m)。

## 实际应用

### AI日志流中的异常模式检测

在分布式AI训练系统中，需要从TB级日志流中实时定位特定错误模式（如`"CUDA out of memory at layer"`）。使用KMP预先构建该模式串的next数组（仅需一次，O(m)），随后对连续传入的日志字节流进行线性扫描。相比每次调用Python的`str.find()`（底层为BMH算法，最坏情况O(nm)），KMP在模式串含大量重复字符时（如`"AAAAAAB"`）性能优势显著。

### NLP管道中的实体边界标记

在基于规则的命名实体识别（NER）预处理阶段，KMP用于在分词后的字符序列中定位多字实体词典项。例如定位实体`"人工智能"`（长度4）在文本中的所有出现位置，next数组为`[0, 0, 0, 0]`（无重复前后缀），此时KMP退化为线性扫描，但对含重复结构的实体如`"中中中国`"仍能保证线性时间。

### 基因序列特征定位

在生物信息学AI模型的数据预处理中，DNA序列可视为字母表为`{A, T, G, C}`的字符串。KMP用于在长度约3×10⁹个碱基对的人类基因组中定位特定启动子序列（典型长度6-20bp），其O(n+m)复杂度使得单次全基因组扫描在线性时间内完成。

## 常见误区

### 误区一：next数组的索引定义混淆

KMP存在两种常见的next数组定义，初学者常将其混用导致错误。**定义A**（本文采用）：`next[i]`为`P[0..i]`的最长相等真前后缀长度，初始值从0开始；**定义B**（部分教材）：`next[i]`存储失配时模式串应跳转到的**索引位置**，通常在定义A基础上整体偏移−1，且`next[0] = -1`。两种定义在代码实现中完全等价，但混用必然导致数组越界或漏匹配。

### 误区二：认为KMP总是最优的字符串匹配算法

KMP保证O(n+m)的最坏情况，但其**平均性能**不如Boyer-Moore算法。Boyer-Moore利用坏字符规则和好后缀规则，在英文文本等字母表较大的场景下，平均复杂度可达O(n/m)（即每m个字符才需一次完整比较）。KMP的优势在于字母表极小（如二进制流、DNA序列）且模式串含大量重复结构的场景，以及需要**严格线性时间保证**的实时流处理系统。

### 误区三：next数组构建中j回退时直接赋0

当`P[i] != P[j]`且`j > 0`时，错误做法是令`j = 0`重新比较。正确做法是`j = next[j-1]`，利用已知的前缀信息继续尝试更短的匹配。以`P = "AABAABC"`为例，若在`i=5, j=2`处失配（即`P[5]='A'`与`P[2]='B'`不等），应令`j = next[1] = 1`（因为"A"是"AA"的最长相等真前后缀），继续比较`P[5]`与`P[1]`，而非从`P[0]`重新开始——后者会导致遗漏位置`i=3`处的匹配。

## 知识关联

**前置知识衔接**：字符串操作的基础（索引访问、切片、子串比较）是理解KMP双指针遍历逻辑的直接前提。具体而言，理解`P[0..k]`这种子串表示法，以及字符串前缀/后缀的概念，是正确构建next数组的必要条件。

**后续算法延伸**：KMP引入的"模式串自身结构预处理"思想直接催生了两个方向：（1）**KMP算法的深化**——理解其在多模式匹配中的扩展（Aho-Corasick算法将KMP的单模式next数组推广为Trie上的失配指针）；（2）**Rabin-Karp算法**——采用完全不同的哈希滚动思路，将模式串映射为数值，通过哈希碰撞检测候选匹配位置，在多模式匹配场景下（同时搜索k个模式）具有O(n+km)的期望性能优势，与KMP的确定性线性时间形成方法论上的对比。
