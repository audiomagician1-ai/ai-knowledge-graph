# 字符串匹配（KMP）

## 概述

KMP算法（Knuth-Morris-Pratt算法）是1977年由Donald Knuth、James Morris和Vaughan Pratt联合发表的线性时间字符串匹配算法。它解决了朴素匹配算法（暴力搜索）在最坏情况下时间复杂度退化为O(m×n)的问题——其中m为模式串长度，n为文本串长度。KMP算法通过预处理模式串，将整体时间复杂度压缩至O(m+n)，即使面对"aaaaaab"这类高度重复字符串也不会退化。

KMP的核心创新在于：当匹配失败时，不将指针退回到文本串的下一个起始位置重新比较，而是利用已经匹配的信息，将模式串向前"跳跃"一段距离继续比较。这个"跳跃距离"由一张称为**部分匹配表**（Partial Match Table，也叫`next`数组或`failure function`）的预计算数据结构决定。正是这张表使得文本串指针永远不需要回退，保证了线性时间复杂度。

在AI工程场景中，KMP被广泛用于日志分析、模式过滤、NLP预处理管道中的关键词检测，以及生物信息学中的DNA序列匹配（例如在人类基因组约30亿碱基对中搜索特定基因片段）。

## 核心原理

### 部分匹配表（next数组）的定义与构建

`next[i]`的值定义为：模式串`P[0..i]`的**最长真前缀**与**真后缀**相等的长度。"真"表示不等于字符串本身。例如，对于模式串`"ABCABD"`：

| 索引 | 子串 | 最长相等真前后缀 | next值 |
|------|------|-----------------|--------|
| 0 | A | 无 | 0 |
| 1 | AB | 无 | 0 |
| 2 | ABC | 无 | 0 |
| 3 | ABCA | "A" | 1 |
| 4 | ABCAB | "AB" | 2 |
| 5 | ABCABD | 无 | 0 |

构建`next`数组本身也使用双指针法，时间复杂度为O(m)，代码逻辑与KMP主匹配过程几乎相同——用模式串对自身进行匹配。

### 匹配失败时的跳跃公式

设文本串为`T`，模式串为`P`，当`T[i] ≠ P[j]`时，不令`i`回退，而是令：

```
j = next[j - 1]   （当j > 0时）
j = 0，i++         （当j = 0时）
```

这意味着模式串向右移动了`j - next[j-1]`个位置。移动量之所以安全，是因为`next[j-1]`保证了模式串中已匹配的前`next[j-1]`个字符与文本串当前位置之前的字符必然对齐，无需重复验证。这是KMP相对于暴力算法节省时间的本质原因。

### 算法完整流程

1. **预处理阶段**：对长度为m的模式串P计算`next`数组，耗时O(m)。
2. **匹配阶段**：维护文本指针`i`（从不后退）和模式指针`j`，逐字符比较：
   - 若`T[i] == P[j]`，则`i++, j++`；
   - 若`j == m`，匹配成功，记录位置`i - m`，令`j = next[j-1]`继续搜索；
   - 若`T[i] != P[j]`，按上述公式调整`j`。
3. 整个过程中`i`最多递增n次，`j`的减少总量不超过增加总量，因此主循环执行次数不超过2n次，耗时O(n)。

## 实际应用

### 文本编辑器中的"查找"功能

主流文本编辑器（如Vim的`/`命令）在搜索长文档时，若模式串包含重复字符（如搜索`"ababac"`），KMP的`next`数组使得匹配失败后不必从头比较。对于一个100MB的日志文件，KMP相比暴力搜索可节省约40%的比较次数（当模式串含重复前缀时）。

### AI日志流中的实时关键词检测

在流式日志处理管道中，KMP天然支持**增量匹配**：文本指针`i`永不回退，这意味着可以将网络数据包或日志分块逐段传入，只需保存当前`j`值即可跨块延续匹配状态。这一特性在Kafka消费者处理实时日志告警时尤为关键。

### DNA序列分析

在生物信息学中，搜索长度为20的引物序列（primer）在基因组中的位置时，KMP的O(n+m)复杂度远优于暴力搜索的O(n×m)。对于n=3×10⁹（人类基因组）、m=20的场景，理论比较次数从6×10¹⁰次降至约3×10⁹次。

## 常见误区

### 误区1：next数组的索引定义混乱

不同教材对`next`数组的初始化存在两种主流约定：一种`next[0] = -1`（以-1作哨兵，代码中使用`j = next[j]`跳转），另一种`next[0] = 0`（代码中使用`j = next[j-1]`跳转）。混用两种约定会导致死循环或越界错误。面试或竞赛中，必须在动手写代码前明确自己采用哪种下标约定，并全程保持一致。

### 误区2：认为KMP在所有场景都优于暴力算法

对于短模式串（m < 10）或文本串中字符分布极度随机（如英文自然语言），暴力匹配平均每次比较约1-2个字符即失败，实际性能接近O(n)。在这类场景下，KMP的`next`数组预计算带来的常数开销反而使其不如暴力算法快。KMP的优势集中在**高重复性**模式串（如`"aaaaaab"`）和**大字母表外的小字母表**（如DNA的ACGT四字符）场景。

### 误区3：混淆"部分匹配值"与"移动步数"

很多初学者直接用`next[j]`作为移动步数，这是错误的。实际移动步数为`j - next[j-1]`，而不是`next[j-1]`本身。`next[j-1]`是新的`j`指针位置（即继续比较时模式串指针应处于的位置），两者含义截然不同。

## 知识关联

**前置基础**：字符串操作中的双指针技术和字符串切片操作是理解KMP两个指针`i, j`协同推进的必要基础；前缀（prefix）与后缀（suffix）的定义直接决定了`next`数组的语义。

**延伸方向**：KMP可视为后续**KMP自动机**（将`next`数组扩展为完整的确定性有限自动机DFA，预处理时间O(m×|Σ|)）的前身。**Rabin-Karp算法**则走了另一条路——用滚动哈希代替逐字符比较，在多模式匹配场景（同时搜索k个模式串）中以O(n+km)的预处理换取更简洁的实现，两者代表了字符串匹配优化的两种根本思路：利用结构信息跳跃 vs. 利用哈希加速比较。
