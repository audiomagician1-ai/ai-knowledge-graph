# DP表格法

## 概述

DP表格法（Tabulation）是动态规划的一种自底向上（Bottom-Up）的实现方式，通过预先分配一张多维数组（DP Table），按照子问题的依赖顺序逐格填写，最终在表格的某个固定位置读取答案。与记忆化搜索（Memoization）的按需递归不同，表格法从最小子问题出发，主动推进到目标问题，不存在递归调用栈的开销。

该方法的系统化使用可追溯到1950年代Richard Bellman对动态规划的奠基性工作。Bellman在求解多阶段决策过程时，明确地将状态空间表示为矩阵，并以行或列为单位逐步填充，这一思路直接催生了现代DP表格法的标准形式。在经典教材《算法导论》（CLRS）中，Floyd-Warshall全源最短路算法和矩阵链乘法均以三维或二维表格的填充顺序作为算法描述的核心。

表格法在工程中的价值在于内存访问模式高度规律，可利用CPU缓存的空间局部性。当DP表大小固定且所有子问题均需计算时，表格法通常比记忆化搜索快20%~40%（视缓存命中率而定），且允许进一步做"滚动数组"空间压缩。

## 核心原理

### 状态定义与表格维度设计

表格法的第一步是将问题的状态映射为表格的下标维度。以经典的0-1背包问题为例：定义 `dp[i][w]` 表示"考虑前 i 件物品，在容量不超过 w 的背包中能获得的最大价值"。这里 i 对应物品编号维度（0 到 n），w 对应容量维度（0 到 W），表格规模为 (n+1) × (W+1)。每个维度的上下界必须覆盖所有合法的子问题，否则填表时会越界或遗漏。

### 状态转移方程与填表顺序

填表的核心是状态转移方程，它规定了每个格子如何由已填好的格子计算得出。0-1背包的转移方程为：

```
dp[i][w] = max(dp[i-1][w],  dp[i-1][w - weight[i]] + value[i])   (当 w >= weight[i])
dp[i][w] = dp[i-1][w]                                              (当 w < weight[i])
```

此方程表明 `dp[i][w]` 依赖于第 i-1 行，因此外层循环必须从 i=1 递增到 n，内层循环 w 的方向任意（0 到 W 均可）。填表顺序必须保证"当前格子所依赖的格子已经被计算"，这是表格法区别于递归的核心约束。Floyd-Warshall算法中，三重循环顺序 k→i→j 也是由同样的依赖关系决定的，若将 k 循环放在内层则结果错误。

### 边界条件初始化

表格法要求在填表前显式初始化所有边界格子（base case）。对于背包问题，`dp[0][w] = 0`（无物品时价值为0）和 `dp[i][0] = 0`（容量为0时价值为0）必须在主循环开始前设置。最长公共子序列（LCS）中，`dp[0][j] = 0` 和 `dp[i][0] = 0` 同理。边界条件初始化错误是表格法中最常见的bug来源，漏写一个边界可能导致整张表的计算结果全部偏移。

### 空间优化：滚动数组技术

当状态转移方程中 `dp[i][...]` 只依赖 `dp[i-1][...]` 时，可将二维表压缩为一维数组，每次迭代时用当前行覆盖上一行，称为"滚动数组"。0-1背包的一维写法为：

```python
for i in range(1, n+1):
    for w in range(W, weight[i]-1, -1):   # 必须从大到小遍历
        dp[w] = max(dp[w], dp[w - weight[i]] + value[i])
```

注意此处 w 必须**从大到小**遍历，因为若从小到大，`dp[w - weight[i]]` 在本轮已被修改，等效于完全背包而非0-1背包。这一方向差异是滚动数组使用中最易出错的细节。

## 实际应用

**编辑距离（Levenshtein Distance）**：`dp[i][j]` 定义为字符串 s1 的前 i 个字符变换为 s2 的前 j 个字符所需的最少操作数（插入、删除、替换各计1次）。表格大小为 (m+1)×(n+1)，按行从左到右填充。对于两个长度均为1000的字符串，完整表格需 1001×1001 ≈ 100万个整数单元，空间复杂度 O(mn)，但可用滚动数组压缩至 O(min(m,n))。

**矩阵链乘法**：`dp[i][j]` 表示将第 i 到第 j 个矩阵连乘所需的最少标量乘法次数。此问题的填表顺序不是按行，而是按子链**长度 l** 从2到n递增，这是因为 `dp[i][j]` 依赖所有满足 i ≤ k < j 的 `dp[i][k]` 和 `dp[k+1][j]`，需要更短子链的结果已就绪。

**股票买卖含冷冻期（LeetCode 309）**：通过引入三个状态 `hold[i]`、`sold[i]`、`rest[i]` 构建一维状态表格，每天的状态仅依赖前两天的值，可用6个变量代替完整数组，是表格法极致压缩的实例。

## 常见误区

**误区一：认为表格法和记忆化搜索总是等价的**。两者在正确性上等价，但表格法会计算所有子问题（包括实际上用不到的），而记忆化搜索只计算被调用的子问题。对于稀疏依赖图（如某些区间DP只访问少数状态），记忆化搜索可能更快，表格法反而浪费计算。

**误区二：滚动数组时遍历方向随意可换**。许多初学者在0-1背包一维写法中将内层循环改为从小到大，导致同一物品被重复选取，实际变成了完全背包。两种方向在代码上只差一行，但语义完全不同，必须根据转移方程是否允许"在当前行内自我依赖"来判断。

**误区三：表格维度越少越好**。将二维状态强行压缩为一维有时需要额外的临时变量来保存被覆盖的值，引入新的复杂度。LCS在滚动数组时若不保存 `dp[i-1][j-1]` 的值（即左上角值），直接原地更新会读到错误数据。正确做法是用变量 `prev` 在内层循环开始前保存。

## 知识关联

**前置依赖**：使用表格法前必须掌握动态规划的最优子结构（Optimal Substructure）和重叠子问题（Overlapping Subproblems）两个性质，这两个性质决定了哪些问题适合构建DP表格。若子问题之间不存在重叠，表格法退化为普通递推，无需记录历史状态。

**与贪心算法的边界**：活动选择、Huffman编码等问题初看也需要表格，但它们满足贪心选择性质（Greedy Choice Property），每步局部最优即全局最优，无需填满整张表。DP表格法的适用判断标准是：最优解是否依赖多个子问题的最优解的组合（转移方程有多个候选项取max/min）。

**进阶方向**：在表格法基础上，可以引入矩阵快速幂加速线性递推型DP（如斐波那契的 O(log n) 解法），以及单调队列/单调栈优化使某些 O(n²) 的DP降至 O(n)（如滑动窗口最大值在DP中的应用）。这些优化都以正确构建DP表格为前提。
