# 冒泡排序

## 概述

冒泡排序（Bubble Sort）是一种基于相邻元素比较与交换的排序算法。其名称来源于较大的元素会像气泡一样逐步"浮"到数组末尾的过程。每一轮遍历中，算法对数组中相邻的两个元素进行大小比较，若顺序错误则交换它们，直到整个数组有序为止。

冒泡排序的历史可追溯至1956年，由计算机科学家在早期排序算法研究中正式提出并描述。它是最早被系统研究的排序算法之一，因逻辑简单直观而常作为算法教学的入门案例。尽管其实际性能不如快速排序或归并排序，但它所体现的"逐步比较、局部交换"思想是理解更复杂排序逻辑的基础。

冒泡排序的时间复杂度为 O(n²)，空间复杂度为 O(1)（原地排序），是一种稳定排序算法。在n较小（通常 n ≤ 20）或数据接近有序的场景下，其实际运行效率并不差。理解它的最坏情况、最好情况以及优化手段，能帮助开发者在实际工程中做出正确的算法选型判断。

## 核心原理

### 基本比较与交换过程

冒泡排序对一个长度为 n 的数组，执行最多 n-1 轮外层循环。每轮外层循环 i（从0到n-2），内层循环从索引0遍历到 n-1-i，对每对相邻元素 `arr[j]` 与 `arr[j+1]` 进行比较。若 `arr[j] > arr[j+1]`，则交换两者。每完成一轮内层循环，当前未排序部分中最大的元素就被"冒泡"到它应在的位置——即数组右端的第 i+1 个位置。

以数组 `[5, 3, 8, 1]` 为例：
- 第1轮：5与3比较→交换→`[3,5,8,1]`；5与8不换；8与1交换→`[3,5,1,8]`，8沉底。
- 第2轮：3与5不换；5与1交换→`[3,1,5,8]`，5归位。
- 第3轮：3与1交换→`[1,3,5,8]`，排序完成。

### 时间复杂度的具体推导

内层循环的比较次数在第1轮为 n-1 次，第2轮为 n-2 次，依此类推，总比较次数为：

$$\sum_{i=1}^{n-1} i = \frac{n(n-1)}{2}$$

当 n=10 时，比较次数为 45；当 n=1000 时，比较次数约为 499,500。这就是 O(n²) 的直接含义——比较次数随数据规模平方增长，这也是冒泡排序不适用于大规模数据的根本原因。

### 稳定性与原地排序特性

冒泡排序是**稳定排序**：当两个元素值相等时（即 `arr[j] == arr[j+1]`），算法不执行交换，因此相等元素的相对顺序在排序前后保持不变。这一特性使冒泡排序适用于多关键字排序场景（如先按姓名排序再按分数排序时需保留上一轮顺序）。

由于排序过程中只需一个临时变量用于交换，额外空间占用为 O(1)，是原地排序算法。

### 提前终止优化（Early Termination）

标准冒泡排序无论数据是否已有序，始终执行 n-1 轮。引入布尔标志位 `swapped` 可实现提前终止：若某一轮内层循环结束后未发生任何交换，说明数组已有序，立即退出外层循环。

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n - 1):
        swapped = False
        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:
            break
    return arr
```

加入此优化后，对于已完全有序的数组，时间复杂度降至 O(n)（仅需1轮、n-1次比较），这是冒泡排序的最好情况。

## 实际应用

**嵌入式系统中的小数组排序**：在内存极度受限的微控制器（如Arduino，RAM仅2KB）中，对传感器采集的5～10个数值进行排序时，冒泡排序的 O(1) 空间特性使其具有实际价值，无需额外内存开销。

**教学与可视化演示**：冒泡排序是算法动画可视化最常用的案例。每次相邻比较和交换的步骤清晰独立，便于通过逐帧展示帮助学习者直观理解排序过程，这是快速排序等递归算法难以替代的教学优势。

**近乎有序数据的排序**：在日志系统中，时间戳已基本有序但偶有乱序条目时，加入 `swapped` 优化的冒泡排序能以接近 O(n) 的效率完成整理，此时比插入排序的实现更为直观简洁。

## 常见误区

**误区1：认为冒泡排序每轮只移动一个元素**  
事实上，每轮内层循环中可能发生多次交换，多个元素位置都会改变。只是每轮结束后，最大的未排序元素**保证**到达其最终位置，其他元素的移动则是不确定次数的中间步骤。

**误区2：认为加入 `swapped` 优化后冒泡排序与插入排序性能相当**  
两者最好情况同为 O(n)，但平均情况不同。冒泡排序的平均比较次数约为 n²/4，而插入排序的平均比较次数约为 n²/4 但交换次数更少（插入排序每次插入最多移动一趟，冒泡每轮交换次数无上界）。在随机数据下，插入排序通常比冒泡排序快约2倍。

**误区3：认为冒泡排序是不稳定排序**  
由于冒泡排序仅在 `arr[j] > arr[j+1]` 时才交换（严格大于，不包含等于），相同值的元素不会被交换，因此冒泡排序是稳定排序。若错误地将条件写成 `>=`，才会破坏稳定性。

## 知识关联

**前置知识：数组/列表**  
冒泡排序直接操作数组的索引访问（`arr[j]` 和 `arr[j+1]`）与元素原地交换，必须掌握数组的随机访问特性和 Python 列表的下标操作才能正确实现。链表结构由于不支持 O(1) 随机访问，不适合直接应用冒泡排序。

**横向对比：选择排序与插入排序**  
三者时间复杂度同为 O(n²)，但机制不同：选择排序每轮找最小值放到左端，交换次数最少（恰好 n-1 次）；插入排序将元素插入已排序部分，对近乎有序数据性能最优；冒泡排序通过相邻交换实现，交换次数最多，但具有稳定性优势且逻辑最易验证正确性。

**延伸方向：算法复杂度分析**  
冒泡排序是学习大O表示法的理想载体——通过计算 n(n-1)/2 的确切比较次数，可以直观推导出 O(n²) 的含义，进而引出快速排序 O(n log n) 为何能大幅提升性能的对比讨论。
