# 拓扑排序

## 概述

拓扑排序（Topological Sort）是对**有向无环图（DAG，Directed Acyclic Graph）**中所有节点进行线性排序的算法，排序结果满足：若图中存在有向边 u → v，则在结果序列中 u 必须出现在 v 之前。拓扑排序仅适用于有向无环图——如果图中存在环，则无法给出合法的拓扑序列。

拓扑排序最早由 Arthur Kahn 于 1962 年在论文《Topological sorting of large networks》中以算法形式正式描述，即后来广为人知的 **Kahn 算法**。另一种基于深度优先搜索的实现方式在同一年代也被系统化，两种方法各有优势，共同构成了现代拓扑排序的标准实现路径。

在 AI 工程中，拓扑排序是**计算图调度**的底层算法。TensorFlow、PyTorch 等框架在执行神经网络前向传播时，需要确定各算子（Op）的执行顺序——这正是一个典型的 DAG 拓扑排序问题。此外，工作流引擎、CI/CD 流水线任务依赖解析、数据 Pipeline 的节点调度，都依赖拓扑排序来保证"依赖先于被依赖者执行"。

## 核心原理

### Kahn 算法（BFS 方式）

Kahn 算法基于**入度（In-degree）**概念实现，步骤如下：

1. 计算图中每个节点的入度，将所有入度为 0 的节点加入队列 Q。
2. 从 Q 中取出一个节点 u，将其加入结果序列。
3. 遍历 u 的所有出边 u → v，将 v 的入度减 1；若 v 的入度变为 0，将 v 加入 Q。
4. 重复步骤 2-3，直到 Q 为空。
5. 若结果序列长度等于节点总数 n，则排序成功；否则图中存在环，拓扑排序不可行。

**时间复杂度**：O(V + E)，其中 V 为节点数，E 为边数。**空间复杂度**：O(V)，用于存储入度数组和队列。

### DFS 后序反转方式

基于 DFS 的拓扑排序利用深度优先搜索的**完成时间戳**特性：在 DFS 中，当一个节点的所有后继节点都被访问完毕后，将该节点压入栈。最终将栈中元素从顶到底读出，即为合法的拓扑序列。

具体实现中，每个节点有三种状态：**未访问（white）、访问中（gray）、已完成（black）**。若 DFS 过程中遇到 gray 节点，说明存在后向边，即图中有环。这一三色标记法使得环检测与拓扑排序可在同一次 DFS 遍历中完成，无需额外开销。

### 拓扑序列的唯一性

一个 DAG 的拓扑序列**不一定唯一**。当图中存在多个入度同时为 0 的节点时，这些节点可以以任意顺序排列，每种顺序都是合法的拓扑序列。拓扑序列唯一的充要条件是：在 Kahn 算法执行过程中，**每一步队列中恰好只有 1 个节点**，即图中存在唯一的哈密顿路径（Hamiltonian Path）。在实际的 AI 编译器优化中，多种合法拓扑序列为算子重排序提供了优化空间，调度器会在合法拓扑序中选取使内存占用或延迟最小的方案。

## 实际应用

**AI 计算图执行调度**：在 ONNX 计算图中，每个算子节点（如 Conv、ReLU、MatMul）通过张量依赖关系连接成 DAG。推理引擎（如 ONNX Runtime）使用拓扑排序确定算子执行顺序，保证所有输入张量在算子执行前已被计算完毕。

**机器学习 Pipeline 编排**：MLflow、Kubeflow Pipelines 等平台将数据预处理、特征工程、模型训练、评估等步骤建模为 DAG。每次运行前，平台对 DAG 执行拓扑排序，自动确定并行与串行执行边界——入度为 0 的多个步骤可并发执行，从而缩短总体运行时间。

**课程选修依赖解析**：给定 n 门课程和 m 条先修关系，判断是否可以完成所有课程，本质是判断课程依赖图是否为 DAG，并输出合法修课顺序。这是 LeetCode 第 207 题（Course Schedule）和第 210 题（Course Schedule II）的原型，也是面试中考查拓扑排序的经典题型。

**编译器符号依赖解析**：C/C++ 编译器在链接阶段需要确定各目标文件的链接顺序，避免符号未定义错误，其核心同样是拓扑排序。

## 常见误区

**误区一：认为拓扑排序可以处理有环图**
部分初学者尝试对含环的有向图进行拓扑排序。实际上，环的存在使得"A 依赖 B，B 依赖 A"这类循环依赖无法被线性化。Kahn 算法的自检机制（最终序列长度 < n）可以检测出环的存在，但不能"绕过"它。若业务场景中确实存在循环依赖，需要首先通过 Tarjan 算法找到强连通分量（SCC）并进行图的缩点处理，而不是直接套用拓扑排序。

**误区二：混淆 DFS 拓扑排序的压栈时机**
使用 DFS 实现时，常见错误是在**首次访问**节点时就将其加入结果，而非在**DFS 递归返回**（即节点完成时）压栈。首次访问时节点的后继尚未处理完毕，此时的顺序不满足拓扑性质。正确做法是：仅在节点所有邻居均处理完毕、DFS 回溯时，才将节点压入栈。

**误区三：认为拓扑排序等价于层次遍历（BFS）**
拓扑排序的 Kahn 实现虽然使用了队列，但其语义与 BFS 的"按层扩展"不同。BFS 按节点距离源点的跳数分层，而 Kahn 算法按入度动态更新——同一"层"中的节点出队顺序因队列实现（FIFO/优先级队列）不同而改变，但所有输出都是合法的拓扑序，而非唯一的 BFS 层次序。

## 知识关联

**前置依赖——深度优先搜索（DFS）**：DFS 的递归回溯机制是理解"后序压栈"式拓扑排序的直接基础。DFS 中节点的完成顺序（finish time）与拓扑序列的逆序完全对应：在无环图中，finish time 越晚的节点，在拓扑序中越靠前。理解 DFS 的灰/黑节点标记，可以直接复用到拓扑排序的环检测逻辑中。

**延伸方向——关键路径（Critical Path Method）**：在带权 DAG 中，对拓扑序列中每个节点计算最早开始时间（Earliest Start Time）和最晚开始时间（Latest Start Time），差值为 0 的节点连成关键路径，决定整个工程的最短完工时间。关键路径算法以拓扑排序为前置步骤，广泛应用于项目管理与 AI 训练任务的时序优化中。

**延伸方向——强连通分量（SCC）**：当输入图可能含环时，需先用 Tarjan 或 Kosaraju 算法找出所有 SCC，将每个 SCC 缩为单一节点，得到 DAG 后再执行拓扑排序。这种"SCC 缩点 + 拓扑排序"的组合模式是处理一般有向图依赖问题的通用范式。
