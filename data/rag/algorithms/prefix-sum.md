# 前缀和

## 概述

前缀和（Prefix Sum）是一种将数组的**累加信息预计算并存储**的技术，使得任意区间 `[l, r]` 的元素之和可以在 O(1) 时间内查询，而非每次重新遍历 O(n) 的时间。其核心思想是：用额外 O(n) 的空间换取后续每次区间查询从线性时间降至常数时间。

前缀和的概念在计算机科学教育中由来已久，在竞赛编程领域（如 ACM-ICPC、LeetCode）是频率最高的基础算法之一。在 AI 工程场景中，卷积神经网络中的**积分图（Integral Image）**本质上就是二维前缀和——Viola 和 Jones 在 2001 年发表的人脸检测论文中将其命名为 Integral Image，并用它将 Haar 特征的计算从 O(k²) 降至 O(1)，使实时人脸检测成为可能。

在大规模数据处理、滑动窗口特征提取、注意力机制的局部求和加速等 AI 工程任务中，前缀和是避免重复计算的标准工具。理解它的构造方式和查询公式，是学习树状数组（Binary Indexed Tree）等进阶数据结构的必要前提。

## 核心原理

### 一维前缀和的构造与查询

设原始数组为 `a[1..n]`，定义前缀和数组 `P[0..n]`，其中：

$$P[i] = \sum_{k=1}^{i} a[k], \quad P[0] = 0$$

构造过程只需一次线性遍历：`P[i] = P[i-1] + a[i]`，时间复杂度 O(n)，空间复杂度 O(n)。

查询区间 `[l, r]` 的元素和时，利用公式：

$$\text{sum}(l, r) = P[r] - P[l-1]$$

此公式成立的原因是 P[r] 包含了下标 1 到 r 的所有元素，P[l-1] 包含了 1 到 l-1 的元素，两者相减恰好得到 l 到 r 的部分。注意下标从 1 开始并设置哨兵 `P[0]=0` 的习惯，可以避免 `l=1` 时的边界特判。

### 二维前缀和的构造与查询

设二维数组 `a[i][j]`（1-indexed），定义二维前缀和：

$$P[i][j] = \sum_{r=1}^{i}\sum_{c=1}^{j} a[r][c]$$

构造递推公式为（利用容斥原理，避免重复计数）：

$$P[i][j] = P[i-1][j] + P[i][j-1] - P[i-1][j-1] + a[i][j]$$

查询矩形区域左上角 `(r1, c1)` 到右下角 `(r2, c2)` 的元素之和：

$$\text{sum} = P[r2][c2] - P[r1-1][c2] - P[r2][c1-1] + P[r1-1][c1-1]$$

容斥中的 `+P[r1-1][c1-1]` 是因为减去两个 L 形区域时，左上角矩形被重复减去两次，需加回一次。这个公式在 OpenCV 的 `cv::integral()` 函数中被直接实现，用于快速计算任意矩形窗口内的像素统计值。

### 前缀和的差分逆操作

前缀和的"逆运算"是**差分数组**（Difference Array）。对数组 `a` 构造差分数组 `d[i] = a[i] - a[i-1]`，则对差分数组做前缀和可还原 `a`。这个关系使得"对区间 [l,r] 整体加上常数 c"的操作变为 `d[l] += c; d[r+1] -= c`，时间复杂度 O(1)，最终一次 O(n) 的前缀和还原结果。两者组合解决的是**批量区间更新 + 最终读取**的场景，与前缀和解决**预处理 + 频繁查询**的场景形成互补。

## 实际应用

**子数组和等于 K（LeetCode 560）**：在统计有多少个连续子数组的和等于目标值 k 时，暴力做法是 O(n²)。利用前缀和 `P[i]`，问题转化为：对于每个 i，寻找之前有多少个 j 满足 `P[i] - P[j] = k`，即 `P[j] = P[i] - k`。配合哈希表记录每个前缀和出现次数，整体降至 O(n)。

**CNN 的积分图加速**：Viola-Jones 人脸检测器中，一张 384×288 的灰度图包含约 11 万个像素，计算所有 Haar 特征（每个特征需要 2~3 个矩形区域的像素和）若逐一遍历需巨大计算量。通过预先计算二维前缀和（即积分图），每个矩形区域的像素和只需 4 次加减法。论文报告其检测速度在 2001 年的硬件上达到了 15 帧/秒，前缀和是核心加速手段。

**滑动窗口特征统计**：在 NLP 的 n-gram 频率统计或时序数据的窗口均值计算中，对长度固定为 w 的窗口求和，利用前缀和只需 `P[i] - P[i-w]`，将每次窗口计算从 O(w) 降至 O(1)，在 w 较大时（如语音处理中 w = 400 个采样点）效果显著。

## 常见误区

**误区一：下标从 0 开始导致边界错误**

很多实现中数组下标从 0 开始，直接套用 `sum(l, r) = P[r] - P[l-1]` 会在 `l=0` 时访问 `P[-1]`。正确做法是将前缀和数组定义为"长度比原数组多 1"，令 `P[0] = 0`，原始数组占据 `a[1..n]`，从而无论 l 取何值，`P[l-1]` 最小也是合法的 `P[0]`。将数组整体右移一位是消除这类 off-by-one 错误的标准处理方式。

**误区二：将前缀和用于频繁更新的场景**

前缀和是**静态数据结构**：一旦原始数组中某个元素被修改，整个前缀和数组中从该位置开始的所有 P[i] 都需要重新计算，最坏情况 O(n)。若需要同时支持"单点更新"和"区间查询"，前缀和不适用，应使用树状数组（BIT），其单点更新和区间查询均为 O(log n)。在 AI 工程中，若特征数组会被在线实时修改，误用前缀和会导致严重的性能退化。

**误区三：二维前缀和查询时遗漏容斥中的加回项**

许多初学者能记住减去两个边界项，但忘记加回左上角重叠项 `P[r1-1][c1-1]`，导致结果偏小。一个辅助记忆方法是：在白纸上画出 4 个矩形的覆盖关系，用集合论的容斥公式 `|A∪B| = |A| + |B| - |A∩B|` 逆推，交叉区域被减去两次，所以必须加回一次。漏掉这一项是二维前缀和实现中出现频率最高的 bug。

## 知识关联

**依赖的前置知识**：前缀和操作的对象是连续内存的数组结构，理解数组的随机访问特性（O(1) 按下标寻址）是理解为何 `P[r] - P[l-1]` 能在 O(1) 内完成的基础。此外，前缀和将区间查询从 O(n) 降至 O(1) 的分析需要用到时间复杂度的 Big-O 概念，特别是"预处理 O(n) 换取后续每次查询 O(1)"的经典时空权衡思路。

**通向树状数组（BIT）**：前缀和解决了静态数组的区间查询问题，但无法高效支持元素修改。树状数组（Fenwick Tree，由 Peter Fenwick 于 1994 年提出）本质上是前缀和思想的动态化扩展：它通过二进制分解将数组分成若干前缀和块，使更新和查询均摊到 O(log n)。学习树状数组时，`lowbit(x) = x & (-x)` 的操作决定了每个节点管理的区间范围，理解这一机制需要以一维前缀和的区间分解思想为基础。
