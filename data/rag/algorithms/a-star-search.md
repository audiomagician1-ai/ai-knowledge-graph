# A*搜索算法

## 概述

A*（读作"A-star"）是一种在加权图上寻找从起点到终点最短路径的启发式搜索算法，由Peter Hart、Nils Nilsson和Bertram Raphael于1968年在斯坦福研究院发表的论文《A Formal Basis for the Heuristic Determination of Minimum Cost Paths》中首次提出。A*的本质是将Dijkstra算法的保证最优性与贪心最佳优先搜索的高效性结合在一起，通过引入启发函数来指导搜索方向，避免盲目扩展所有节点。

A*的核心评估公式为 **f(n) = g(n) + h(n)**，其中 g(n) 是从起点到当前节点 n 的已知实际代价，h(n) 是从节点 n 到终点的启发式估计代价，f(n) 是对经过节点 n 的完整路径总代价的估计。这个公式决定了A*优先展开哪些节点：f(n)值最小的节点优先被处理，这正是A*使用优先队列（最小堆）而非普通队列的原因。

A*在游戏AI、机器人导航、地图路径规划等领域被广泛采用。例如，早期的《星际争霸》和《魔兽争霸》引擎均使用A*变体处理单位寻路；现代自动驾驶系统的局部路径规划模块也大量依赖A*或其衍生算法。理解A*的关键在于掌握启发函数的可容性条件，这直接决定算法是否能保证找到最优解。

## 核心原理

### Open表与Closed表机制

A*维护两个集合：**Open表**（待扩展节点的优先队列）和**Closed表**（已扩展完毕的节点集合）。算法启动时将起点加入Open表，每次从Open表中取出f(n)最小的节点作为当前节点，将其移入Closed表，然后遍历其所有邻居节点。对于每个邻居：若在Closed表中则跳过；若不在Open表中则加入并记录其父节点；若已在Open表中且新路径的g值更小，则更新该节点的g值和父节点。当终点节点被从Open表中取出时，算法结束，通过逐级回溯父节点指针即可还原最优路径。

### 启发函数的可容性（Admissibility）

启发函数 h(n) 必须满足**可容性条件**：对所有节点 n，h(n) ≤ h*(n)，其中 h*(n) 是节点 n 到终点的真实最短距离。换言之，h(n) 绝不能高估实际代价。若该条件成立，A*保证找到最优解。常见可容性启发函数包括：
- **曼哈顿距离**：h(n) = |Δx| + |Δy|，适用于只允许上下左右四方向移动的网格地图。
- **欧几里得距离**：h(n) = √(Δx² + Δy²)，适用于允许任意方向移动的场景。
- **切比雪夫距离**：h(n) = max(|Δx|, |Δy|)，适用于允许八方向移动的网格。

若允许对角线移动却使用曼哈顿距离，h(n) 依然满足可容性（因为曼哈顿距离 ≥ 欧几里得距离在此情况下不成立……实际上曼哈顿距离 ≥ 实际对角线距离，仍为可容），而若使用欧几里得距离则更精确但不会超估。

### 一致性（Consistency）与效率权衡

比可容性更强的条件是**一致性（Consistency）**，也称单调性：对于节点 n 及其任意后继节点 n'，需满足 h(n) ≤ c(n, n') + h(n')，其中 c(n, n') 是 n 到 n' 的边代价。满足一致性的启发函数必然满足可容性，且能保证每个节点只被扩展一次（Closed表中的节点无需重新处理），从而显著提升效率。上述三种距离公式在标准网格地图上均满足一致性。

当工程上可以牺牲最优性换取速度时，可使用**加权A***：将评估函数改为 f(n) = g(n) + ε·h(n)，其中 ε > 1。此时算法找到的路径长度不超过最优解的 ε 倍，但搜索速度显著加快。ε = 1 即退化为标准A*；ε → ∞ 则退化为贪心最佳优先搜索。

### 时间与空间复杂度

A*的时间复杂度为 **O(b^d)**，其中 b 是分支因子，d 是最优解深度，但实际性能高度依赖启发函数质量。当 h(n) = 0 时，A*退化为Dijkstra算法，需要扩展更多节点；当 h(n) 精确等于 h*(n) 时，A*几乎只沿最优路径扩展节点，效率最高。A*的空间复杂度同为 O(b^d)，因为需要在内存中维护Open表和Closed表中的所有节点，这是其在内存受限环境下的主要瓶颈。针对此问题，IDA*（迭代加深A*）用深度限制替代优先队列，将空间复杂度降至 O(d)，但代价是重复计算部分路径。

## 实际应用

**游戏地图寻路**：在二维网格地图中，设定每格移动代价为1（直线）或1.414（对角线），以曼哈顿或切比雪夫距离为启发函数，A*可在毫秒内为数百个游戏单位实时计算路径。为进一步优化，《龙与地下城OL》等游戏引入了**HPA*（Hierarchical Pathfinding A*）**，将地图分为粗粒度区块和细粒度局部，先在区块层运行A*，再在局部层细化，将搜索节点数减少约90%。

**机器人运动规划**：ROS（机器人操作系统）的 `navfn` 和 `global_planner` 插件均以A*或Dijkstra为基础，在占用栅格地图（Occupancy Grid）上规划全局路径。实际使用时，地图分辨率通常设为0.05米/格，启发函数采用欧几里得距离，并对靠近障碍物的格子附加惩罚代价，以使机器人保持安全距离。

**GPS导航**：高德、百度地图等导航系统在路网图上使用A*的变体（如双向A*）。双向A*同时从起点和终点向中间搜索，当两个搜索前沿相遇时停止，理论上可将搜索节点数降至单向A*的平方根量级，在城市级路网（百万级节点）中提速尤为显著。

## 常见误区

**误区一：h(n) 越大越好，搜索越快**。部分初学者认为将 h(n) 设置得尽量大可以最快达到终点。实际上，一旦 h(n) > h*(n)（违反可容性），A*可能跳过最优路径的关键节点，找到的解不再是最短路径。例如在10×10网格中，若真实最短路径代价为15但 h(n) 估计为20，算法会优先扩展看似更近但实则绕远的路径，最终返回次优解。

**误区二：A*总是比Dijkstra快**。A*的速度优势完全依赖启发函数的精准度。在边权差异极大的非网格图（如稀疏路网）或启发函数估计误差较大的场景中，A*展开的节点数可能接近甚至超过Dijkstra。当启发函数精度不足以"过滤"大量低价值节点时，维护优先队列的额外开销反而使A*略慢于针对该图结构优化过的Dijkstra实现。

**误区三：Closed表中的节点绝对不需要重新处理**。此结论仅在启发函数满足**一致性**时成立。若 h(n) 仅满足可容性但不满足一致性，则理论上一个节点可能在首次扩展后通过另一条路径发现更小的 g 值，需要重新加入Open表。在工程实现中忽略此情况会导致算法找到次优解，因此在使用自定义启发函数时必须验证其一致性。

## 知识关联

**与BFS的关系**：BFS是A*在所有边权为1且h(n)=0时的特殊情况，使用普通队列（FIFO）而非优先队列。BFS保证在无权图中找到跳数最少的路径，但无法处理加权图，也不利用任何启发信息。学习A*需要先理解BFS的逐层扩展逻辑，因为A*的Open表扩展顺序本质上是BFS逐层展开的加权推广。
