# A*搜索算法

## 概述

A*（读作"A-star"）是一种在加权图上寻找最短路径的启发式搜索算法，由Peter Hart、Nils Nilsson和Bertram Raphael于1968年在斯坦福研究院发表的论文《A Formal Basis for the Heuristic Determination of Minimum Cost Paths》中首次提出。与Dijkstra算法不同，A*通过引入启发式函数来估算当前节点到目标的剩余代价，从而优先展开更有希望的路径，大幅减少不必要的节点扩展。

A*的核心评估公式为 **f(n) = g(n) + h(n)**，其中 g(n) 表示从起点到节点 n 的已知实际代价，h(n) 表示从节点 n 到目标的启发式估算代价，f(n) 是对该路径总代价的估计值。算法始终优先扩展 f(n) 最小的节点，这使它在大多数情况下比纯代价驱动的Dijkstra更高效。

A*算法的重要性体现在其被广泛应用于游戏AI路径规划（如《魔兽争霸》系列的单位寻路）、机器人运动规划、地图导航系统（Google Maps早期版本使用A*的变体）以及自动驾驶路径优化中，是工业级路径搜索的事实标准算法。

## 核心原理

### 启发式函数 h(n) 的设计与可采纳性

启发式函数 h(n) 必须满足**可采纳条件（Admissibility）**：对于任意节点 n，h(n) 必须永远不超过从 n 到目标的真实最小代价 h*(n)，即 h(n) ≤ h*(n)。若违反此条件，A* 可能绕过真正的最优路径，返回次优解。

在二维网格地图中，常用的可采纳启发式函数包括：
- **曼哈顿距离**（仅允许上下左右移动时）：h(n) = |x_n - x_goal| + |y_n - y_goal|
- **欧几里得距离**（允许任意方向移动时）：h(n) = √((x_n - x_goal)² + (y_n - y_goal)²)
- **切比雪夫距离**（允许8方向移动时）：h(n) = max(|Δx|, |Δy|)

若 h(n) 恒等于0，A* 退化为 Dijkstra 算法；若 h(n) 完全等于真实代价，A* 将沿最优路径零浪费地直达目标。

### 一致性（Consistency）条件

除可采纳性外，更强的条件是**一致性（Consistency）**，也称单调性：对于任意节点 n 及其后继节点 n'，需满足 h(n) ≤ c(n, n') + h(n')，其中 c(n, n') 是从 n 到 n' 的实际边代价。一致性保证每个节点只需被扩展一次，使算法的时间复杂度稳定在 O(E log V)（E为边数，V为节点数）。曼哈顿距离和欧几里得距离在标准网格上都满足一致性，但某些自定义启发式函数可能仅满足可采纳性而不满足一致性。

### 开放列表与关闭列表的维护机制

A* 维护两个集合：**开放列表（Open List）** 存储已发现但未扩展的节点，**关闭列表（Closed List）** 存储已扩展的节点。实现时，开放列表通常使用**二叉堆（Binary Heap）** 或**斐波那契堆**存储，以 f(n) 值为优先级，确保每次取出最小代价节点的时间复杂度为 O(log n)。

算法流程如下：
1. 将起点加入开放列表，令 g(start) = 0
2. 取出开放列表中 f(n) 最小的节点作为当前节点
3. 若当前节点为目标，通过回溯父节点指针重建路径，算法结束
4. 将当前节点移入关闭列表
5. 遍历所有邻居节点：若邻居在关闭列表中则跳过；若不在开放列表中或发现了更小的 g 值，则更新其 g(n)、h(n)、f(n) 并记录父节点

## 实际应用

### 游戏地图寻路

在《星际争霸II》等即时战略游戏中，地图被离散化为多边形网格（Navigation Mesh），A* 在该网格上实时计算单位的行进路径。为降低计算量，游戏引擎常采用**分层A***（Hierarchical A*）：先在粗粒度的区域层找到宏观路径，再在细粒度的局部层精化路线。单次寻路时间通常需控制在 1 毫秒以内以维持 60 帧流畅度。

### 机器人运动规划

在 ROS（Robot Operating System）的导航栈中，全局规划器默认使用 A* 算法在占用栅格地图（Occupancy Grid）上计算路径，栅格分辨率通常为 0.05 米/格。障碍物单元格代价被设为无穷大，自由空间代价为1，靠近障碍物的缓冲区通过代价膨胀（Cost Inflation）被赋予中间代价值，确保机器人路径与障碍物保持安全距离。

### 15数码问题

A* 是求解15数码（15-Puzzle）滑块问题的标准方法。启发式函数常用**曼哈顿距离之和**（所有方块到目标位置的曼哈顿距离总和），这是可采纳的，因为每次移动只能让一个方块向目标靠近一步。使用此启发式的A*相比BFS可将搜索节点数从数十亿减少至数万量级。

## 常见误区

### 误区一：h(n) 越大越好，搜索越快

部分初学者认为启发式估值越激进（越接近甚至超过真实距离）搜索越快。实际上，若 h(n) > h*(n)（称为**不可采纳启发式**），A* 会失去最优性保证，可能返回一条比最短路径代价更高的路径。这种以最优性换速度的变体有专属名称——**加权A*（Weighted A*）**，写作 f(n) = g(n) + ε·h(n)（ε > 1），是一个有意识的设计权衡，而非A*本身的行为。

### 误区二：A* 一定优于 Dijkstra

A* 在启发式函数精准时效率远超 Dijkstra，但当图结构复杂（如高维空间、均匀代价随机图）时，h(n) 提供的指导信息极为有限，A* 的实际扩展节点数接近 Dijkstra，但额外计算 h(n) 反而带来了开销。在城市路网中，A* 通常比 Dijkstra 快2到5倍；但在完全随机权重图中优势几乎消失。选择算法时需根据领域特性判断启发式的质量。

### 误区三：开放列表中的节点不需要更新

当通过不同路径二次到达同一节点且发现更小 g 值时，必须更新该节点在开放列表中的 f 值（称为**decrease-key操作**）。若省略此步骤，在非一致性启发式或存在负权边的图中，算法可能输出非最优路径。使用 Python 的 `heapq` 实现时，常见做法是将旧条目标记为失效（lazy deletion），插入新条目，并在弹出时检查是否已处理。

## 知识关联

A* 是 **BFS** 和 **Dijkstra** 两种算法的自然扩展：BFS 相当于所有边代价为1且 h(n)=0 的特例，Dijkstra 相当于 h(n)=0 的加权图搜索，而 A* 在 Dijkstra 的基础上加入了领域知识编码的 h(n) 以引导搜索方向。理解 Dijkstra 的优先队列机制是正确实现 A* 开放列表的直接前提，而 BFS 的层次扩展直觉有助于理解为什么可采纳的 h(n) 能保证找到最短路径。

在更高级的路径规划领域，A* 衍生出多个专用变体：**IDA***（迭代加深A*）用内存换时间，适合内存极度受限的嵌入式环境；**D* Lite** 支持动态环境中的增量重规划，被 NASA 的火星探测器采用；**Jump Point Search（JPS）** 在均匀代价网格上通过跳点加速将 A* 速度提升一个数量级。掌握标准 A* 的 f(n) = g(n) + h(n) 框架和可采纳性证明，是理解这些高级变体的必要基础。
