# 欧拉路与哈密顿路

## 概述

欧拉路（Eulerian Path）是指图中经过**每条边恰好一次**的路径，若此路径为回路（起点终点相同），则称为欧拉回路（Eulerian Circuit）。哈密顿路（Hamiltonian Path）则是经过图中**每个顶点恰好一次**的路径，对应的回路称为哈密顿回路（Hamiltonian Circuit）。二者看似相似，实则在判定复杂度上存在根本性差异。

欧拉路的研究起源于1736年列昂哈德·欧拉（Leonhard Euler）对柯尼斯堡七桥问题的分析。欧拉将陆地抽象为顶点、桥梁抽象为边，证明了无法一次性不重复地走过所有七座桥，由此奠定了图论的基础。哈密顿路则由威廉·罗恩·哈密顿（William Rowan Hamilton）于1859年在其发明的"周游世界游戏"（Icosian Game）中提出，要求在正十二面体的二十个顶点上找出经过每个顶点恰好一次的回路。

二者的重要性在于判定复杂度的鲜明对比：欧拉路存在性可在线性时间 O(V+E) 内判定；而哈密顿路判定是NP完全问题，目前不存在已知的多项式时间算法。这一对比是计算复杂性理论的经典例证。

## 核心原理

### 欧拉路与欧拉回路的判定定理

**无向图**中：
- **欧拉回路存在**当且仅当：图连通，且每个顶点的度数均为**偶数**。
- **欧拉路（非回路）存在**当且仅当：图连通，且恰好有**两个奇度顶点**（这两个顶点分别是路径的起点和终点）。

**有向图**中：
- **欧拉回路存在**当且仅当：图弱连通，且每个顶点的入度等于出度（in-degree = out-degree）。
- **欧拉路（非回路）存在**当且仅当：图弱连通，恰好有一个顶点满足出度 = 入度 + 1（起点），一个顶点满足入度 = 出度 + 1（终点），其余顶点入度等于出度。

柯尼斯堡七桥问题中，四块陆地的度数分别为3、3、3、5，全部为奇数（4个奇度顶点），既不满足欧拉回路条件，也不满足欧拉路条件，因此无解。

### 欧拉路的构造算法：Hierholzer算法

Hierholzer算法（1873年发表）是构造欧拉回路/欧拉路的经典算法，时间复杂度为 **O(E)**，步骤如下：

1. 从合法起点出发，沿任意未使用边行走，直到无路可走，形成一条初始回路。
2. 若初始回路尚未包含所有边，找出回路中仍有未使用出边的顶点。
3. 从该顶点出发再走一圈子回路，将子回路插入初始回路中对应位置。
4. 重复步骤2-3直至所有边均被使用。

该算法的关键点是维护当前路径的"插入位置"，通常用栈实现，避免重新遍历整条路径。

### 哈密顿路的判定困难性

哈密顿路判定属于**NP完全问题**，这意味着：目前没有任何已知算法能在多项式时间内判定任意图是否存在哈密顿路。最直接的暴力方法是枚举全排列，复杂度为 O(n!)，即使使用**Held-Karp动态规划算法**（1962年），复杂度也仅能降至 O(2ⁿ · n²)，空间复杂度 O(2ⁿ · n)，仍为指数级。

Held-Karp算法的状态定义：设 `dp[S][v]` 表示从起点出发，经过顶点集合 S 中所有顶点且当前停在顶点 v 处是否可行，状态转移方程为：

> dp[S][v] = OR { dp[S \ {v}][u] | (u, v) ∈ E, u ∈ S }

尽管缺乏通用高效算法，有几个充分条件可用于判断哈密顿回路存在：
- **Dirac定理（1952年）**：若无向简单图有 n ≥ 3 个顶点，且每个顶点的度数 δ(G) ≥ n/2，则哈密顿回路必然存在。
- **Ore定理（1960年）**：若对图中任意不相邻的两顶点 u、v，均有 deg(u) + deg(v) ≥ n，则哈密顿回路必然存在。

注意：这两个定理均只是充分条件，不满足并不代表哈密顿回路不存在。

## 实际应用

**欧拉路的应用——中国邮递员问题（Chinese Postman Problem）**：邮递员需走遍所有街道并返回出发点，若街道图存在欧拉回路，则最优路线恰好等于所有边权之和；否则需要通过在奇度顶点间添加"重复边"使图满足欧拉回路条件，这一问题由中国数学家管梅谷于1962年提出。

**欧拉路的应用——DNA序列拼接**：在基因组学中，将DNA片段抽象为有向图的边，每个k-mer序列片段对应一条边，利用Hierholzer算法寻找欧拉路来重构完整基因序列，这是de Bruijn图拼接算法的核心思想，被广泛用于现代基因组测序软件（如Velvet、SPAdes）。

**哈密顿路的应用——旅行商问题（TSP）**：在带权完全图中寻找权重最小的哈密顿回路，是运筹学中的经典NP难问题，实际中通过近似算法（如Christofides算法，近似比1.5）或启发式算法（遗传算法、模拟退火）求解，应用于物流配送路径规划、芯片布线等领域。

**哈密顿路的应用——哈密顿图的棋盘问题**：国际象棋中的"骑士巡游"问题（Knight's Tour），要求马从某一格出发，按马的走法恰好经过8×8棋盘上所有64个格子，本质即在图中寻找哈密顿路，已知在标准棋盘上存在大量解（开放式巡游和封闭式巡游均存在）。

## 常见误区

**误区一：度数全为偶数就保证存在欧拉回路**

这一误区忽略了**图的连通性**条件。一个由两个分离的偶度顶点三角形组成的图，所有顶点度数均为2（偶数），但由于图不连通，并不存在欧拉回路。正确判定必须同时满足"连通"和"全偶度"两个条件。注意，孤立顶点（度数为0）不影响欧拉回路的存在性，但实际应用中通常要求去除所有孤立顶点后的子图连通。

**误区二：哈密顿路和欧拉路判定都很难**

恰恰相反。欧拉路可以用 O(V+E) 时间判定，而哈密顿路是NP完全问题，二者在计算复杂度上相差指数级。这一差异的本质在于：欧拉路对边的约束（每边一次）具有局部可验证性，只需检查顶点度数；而哈密顿路对顶点的约束（每点一次）在组合结构上极难在局部特征中反映出来。

**误区三：Dirac/Ore定理是哈密顿回路存在的充要条件**

Dirac定理（δ ≥ n/2）和Ore定理只是**充分条件**。例如，Petersen图（10个顶点，每个顶点度数为3，不满足Dirac条件）实际上存在哈密顿回路；而一个度数全为1的完美匹配图（n个顶点，度数均为1）不满足Ore条件，也确实不存在哈密顿回路。因此不能通过这些定理的否命题否定哈密顿回路的存在性。

## 知识关联

**与图论基础的联系**：欧拉路的判定直接依赖顶点度数（degree）和图的连通性，这两个概念是图论基础的核心内容；有向图欧拉路还需要理解入度/出度的差值关系。没有对度数序列和连通分量的清晰认识，无法正确应用欧拉路定理。

**与树的联系**：Hierholzer算法在实现时通常借助深度优先搜索（DFS）遍历，而DFS在树结构上的行为是理解其在一般图上工作的基础；此外，哈密顿路问题的某些特例在树结构上可以高效求解——树上的哈密顿路等价于求树的直径路径，可用两次BFS/DFS在 O(V) 时间内完成，是一般哈密顿路问题在特殊结构上的多项式可解情形。
