# 欧几里得算法

## 概述

欧几里得算法（Euclidean Algorithm），又称辗转相除法，是一种通过反复取余来求两个正整数最大公因数（GCD）的方法。其核心等式为：**gcd(a, b) = gcd(b, a mod b)**，算法在 `b = 0` 时终止，此时 `a` 即为所求的最大公因数。

该算法出自古希腊数学家欧几里得约于公元前 300 年所著的《几何原本》（Elements）第七卷，是人类历史上有文字记载的最古老算法之一，比中国《九章算术》中的"更相减损术"早约两百年。《几何原本》中欧几里得用几何语言描述了对线段长度反复相减的过程，而现代版本将其转化为取余操作，效率大幅提升。

欧几里得算法的重要性在于：它不仅是数论中分解因子、化简分数的基础工具，更是现代密码学（如 RSA 算法求模逆元）、计算机科学（如多项式 GCD、格密码）中不可替代的组件。其时间复杂度为 **O(log min(a, b))**，远优于逐一枚举公因数的暴力方法。

## 核心原理

### 算法的数学基础

欧几里得算法成立的关键定理是：若 `a = bq + r`（其中 `0 ≤ r < b`），则 **gcd(a, b) = gcd(b, r)**。

**证明**：设 d = gcd(a, b)，则 d | a 且 d | b，从而 d | (a − bq) = r，即 d 也是 gcd(b, r) 的因子。反之，设 d' = gcd(b, r)，则 d' | b 且 d' | r，从而 d' | (bq + r) = a，即 d' 也整除 a，故 d' 是 gcd(a, b) 的因子。因此两个 GCD 互为因子，相等。

### 算法执行过程（以具体数为例）

以求 gcd(252, 105) 为例：

| 步骤 | a   | b   | a mod b |
|------|-----|-----|---------|
| 1    | 252 | 105 | 42      |
| 2    | 105 | 42  | 21      |
| 3    | 42  | 21  | 0       |

当 `b = 0` 时，`a = 21` 即为所求，共执行 3 次取余。递推形式为：gcd(252,105) → gcd(105,42) → gcd(42,21) → gcd(21,0) = **21**。

### 时间复杂度与 Lamé 定理

1844 年，Gabriel Lamé 证明：欧几里得算法的步骤数不超过输入中**较小数十进制位数的 5 倍**。更精确地说，若 a > b，算法所需的除法次数不超过 **5 × ⌊log₁₀ b⌋ + 1**。

这一结论与斐波那契数列密切相关——使欧几里得算法步骤数最多的输入恰好是相邻的斐波那契数。例如，gcd(F_{n+1}, F_n) 恰好需要 n 步才能终止，这是该算法最坏情况的典型构造。

### 递归与迭代两种实现

**递归实现**（伪代码）：
```
function gcd(a, b):
    if b == 0: return a
    return gcd(b, a mod b)
```

**迭代实现**（避免调用栈溢出）：
```
function gcd(a, b):
    while b != 0:
        (a, b) = (b, a mod b)
    return a
```

迭代版本在处理极大整数（如密码学中的 2048 位数）时更为安全，不会因递归深度过大而溢出栈空间。

## 实际应用

**分数化简**：将分数 `336/504` 化为最简分数，先求 gcd(336, 504)。执行算法：504 = 1×336 + 168，336 = 2×168 + 0，故 gcd = 168，分数化简为 2/3。

**RSA 密码学中的扩展应用**：RSA 算法在选择公钥 e 时，要求 gcd(e, φ(n)) = 1（即 e 与欧拉函数互质）。验证这一条件直接使用欧几里得算法。例如，取 φ(n) = 3120，e = 17，gcd(3120, 17) = gcd(17, 3120 mod 17) = gcd(17, 1) = **1**，验证通过。

**多项式 GCD**：欧几里得算法可推广到多项式环。对于多项式 f(x) = x²−1 和 g(x) = x−1，用多项式带余除法：x²−1 = (x+1)(x−1) + 0，故 gcd = x−1。这是计算机代数系统（如 Mathematica）化简有理函数的基础。

**日常节拍问题**：两台机器分别每 48 秒和 36 秒触发一次，gcd(48, 36) = 12，即每 12 秒两台同时触发，此为工程调度中的同步周期计算。

## 常见误区

**误区一：认为 gcd(a, 0) 需要特殊处理，结果为 0**。实际上，数学定义规定 gcd(a, 0) = a（对任意正整数 a），因为 a 整除 0，而 a 本身是 a 的最大因子。算法终止条件 `b = 0` 时返回 `a` 正是基于此定义，而非边界特例。

**误区二：混淆辗转相除法与更相减损术**。更相减损术的操作是反复将较大数减去较小数：gcd(252, 105) 需做 252−105=147，147−105=42，105−42=63，63−42=21，42−21=21，共 5 次减法才到达重复，远比取余低效。当两数比值很大时（如 gcd(1000001, 1)），更相减损术需 1000000 步，而欧几里得算法仅需 1 步。

**误区三：认为算法只适用于整数**。欧几里得算法本质上依赖带余除法的存在，因此可推广到任何**欧几里得整环**（Euclidean Domain），包括高斯整数 ℤ[i]（复数 a+bi，其中 a,b∈ℤ）和多项式环 F[x]（F 为域）。在高斯整数中，gcd(5, 2+i) 的计算遵循相同的递推结构，结果为 2+i。

## 知识关联

**前驱概念**：学习欧几里得算法需要熟悉**最大公因数**的定义（两数公有因子中最大者）以及整数带余除法（a = bq + r，0 ≤ r < b）。若对 GCD 的概念不清晰，将无法理解算法终止时返回值的意义。

**直接后继——裴蜀等式（Bézout's Identity）**：欧几里得算法的"扩展版本"（Extended Euclidean Algorithm）在求 gcd(a, b) 的同时，反向追踪每步的商，能同时求出整数 x, y 满足 **ax + by = gcd(a, b)**。例如 gcd(252, 105) = 21，扩展算法可给出 21 = 2×252 − 4×105（即 x=2, y=−4），这是裴蜀等式的具体实现，也是求模逆元的核心步骤。

**后续拓展——连分数**：欧几里得算法每步的商 q₁, q₂, q₃, … 恰好构成两数之比的**连分数展开**。以 252/105 为例，商序列为 [2; 2, 2]，对应连分数 2 + 1/(2 + 1/2) = 12/5，正是 252/105 化简后的比值 12/5。这一联系使欧几里得算法成为研究有理数逼近和连分数理论的核心工具。
