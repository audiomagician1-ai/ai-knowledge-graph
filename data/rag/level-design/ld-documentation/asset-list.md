# 资产清单

## 概述

资产清单（Asset List）是关卡设计文档体系中的一份专项清单，专门记录一个关卡从开始制作到最终交付所需的全部美术资源，包括3D模型、贴图、特效、动画、UI元素和音效素材等具体条目。它的核心价值在于让美术团队能够提前规划工作量，避免在关卡集成阶段才发现缺少关键资产而导致的返工延误。

资产清单的使用习惯在商业游戏开发中大约从1990年代末、3D游戏制作流水线成熟之后逐渐被系统化。早期的2D游戏由于资产数量有限，设计师往往以非正式的便签或口头沟通替代正式清单；但随着《古墓丽影》《神秘海域》等3A项目的资产数量突破数千乃至数万条，没有结构化清单便几乎不可能追踪制作进度。

资产清单不仅是一份"购物清单"，它还承载着优先级信息——告诉美术部门哪些资产是本次里程碑（Milestone）必须完成的P0级别资产，哪些是Nice-to-Have的P2级别资产。这种优先级分层使得项目经理可以在资源紧张时做出有依据的取舍决策。

## 核心原理

### 资产条目的四要素结构

一条标准的资产清单条目通常包含四个字段：**资产名称**（如`ENV_Cave_StalactiteLarge_01`）、**资产类型**（模型/贴图/特效/音效等）、**优先级等级**（P0/P1/P2或高/中/低）、**负责人与截止时间**。资产名称必须遵循项目统一的命名规范，例如`[类别前缀]_[关卡缩写]_[描述]_[序号]`，否则在引擎中引用时将产生命名冲突或查找困难。一个完整的关卡资产清单条目示例：

```
资产名称: ENV_L03_RuinWall_Moss_02
类型: 静态模型（含贴图集）
多边形预算: ≤2500 tris
优先级: P0（关卡阻塞性资产）
负责人: 张三
截止日期: Alpha里程碑前两周
状态: 制作中
```

### 优先级分层方法

关卡设计师在填写优先级时需要回答一个具体问题："如果这个资产缺席，关卡能否运行或被评审？" **P0（阻塞级）**：缺少该资产则关卡无法完整游玩，例如玩家必须穿越的大门模型、触发剧情的关键道具；**P1（重要级）**：缺少后关卡功能完整但视觉残缺，例如背景建筑装饰、地面细节贴图；**P2（增强级）**：缺少后不影响评审结论，例如灯柱上的锈迹贴图变体、次要NPC的服装细节。一般而言，P0资产数量应控制在整体清单的20%～30%以内，若P0比例超过50%，说明关卡范围（Scope）本身需要重新谈判。

### 资产复用率统计

资产清单的一个高价值用途是计算**复用率**（Reuse Rate）。设计师在填写清单时应标注每条资产是"新建"还是"复用自关卡X"或"复用自公共资产库"。若一个关卡的新建资产比例超过60%，通常意味着美术工作量将超出正常预算。Naughty Dog等工作室的内部标准要求单个关卡的新建环境模型不超过整体模型数量的40%，其余来自已有资产库，以此控制制作成本。复用率的公式为：

> **复用率 = 复用资产数量 ÷ 总资产数量 × 100%**

## 实际应用

**示例：一个室内废墟关卡的资产清单片段**

某第一人称射击游戏的第三关为一段30秒步行的废弃工厂场景，其资产清单包含约120条条目。其中P0资产共28条，涵盖：走廊墙体模块（8种变体）、破损地板模块（4种）、工厂大门（含开关动画）、爆炸桶道具、主光源吊灯模型。P1资产约55条，包括管道装饰件、窗户玻璃破碎特效、墙面污渍贴图叠加层。P2资产37条，主要为背景海报贴图、地面积水反射材质变体等。

在实际制作流程中，关卡设计师在**白盒（Greybox）阶段**完成后即应提交初版资产清单，让美术主管能在白盒评审会上同步确认资产范围是否合理。随着关卡迭代，清单会经历2～3次更新：第一次在白盒通过后（范围确认），第二次在Alpha里程碑前（冻结P0列表），第三次在Beta阶段（确认P2的取舍）。

## 常见误区

**误区一：把资产清单当成单纯的"愿望清单"，不设优先级**
很多初级关卡设计师提交的清单仅罗列资产名称，不标注优先级。这会让美术师在面对时间压力时无法自主决策，每件事都需要重新询问设计师，造成大量沟通成本。正确做法是在清单初稿时就为每条记录填写P0/P1/P2标签，而非等到截止日临近才补充。

**误区二：将音效资产排除在资产清单之外**
关卡设计文档的资产清单往往只覆盖视觉资产，而遗漏音效（如场景环境音、触发音效、NPC语音触发条目）。然而音效资产同样有制作周期，例如一条定制化的环境循环音效（Ambience Loop）从录音到混音通常需要1～2周。若到Beta才提出音效需求，很可能来不及完成。

**误区三：混淆"资产清单"与"关卡道具摆放列表"**
资产清单记录的是"这个模型文件是否已存在/需要被制作"，而道具摆放列表（Object Placement List）记录的是"这个已有模型在关卡中被摆放在哪些坐标"。两者常被混写在同一张表中，导致美术师误以为列出的坐标数量代表需要制作同等数量的独立资产，实际上同一个模型可以被实例化（Instance）数百次。

## 知识关联

资产清单是在完成**LD文档概述**之后自然衍生出的执行性文档。LD文档概述确定了关卡的叙事目标、玩法机制和大致空间规模，资产清单则将这些抽象描述转化为可被美术团队执行的具体工作包。例如，概述中写"玩家穿越一段被破坏的城市街道"，资产清单就需要将其分解为：建筑立面模块（几种）、地面破碎道路（几种多边形规格）、倒塌电线杆（含碰撞体）等可量化条目。

在白盒阶段，关卡设计师使用引擎内置的基础几何体（Blockout Mesh）代替正式资产占位，资产清单此时作为"待替换清单"驱动美术生产。当关卡进入最终美化阶段，资产清单还可以演变为**资产验收检查表**，美术主管逐条确认每项资产的技术指标（多边形数、贴图分辨率、LOD级别数量）是否符合项目规格书的要求，确保关卡在目标平台上能够达到性能预算。
