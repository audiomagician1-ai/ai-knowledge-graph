# 道路与河流

## 概述

道路与河流是关卡地形设计中两类最基本的线性地理元素，指通过地形网格的修改与纹理绘制，将道路网络和河流路径自然融合进游戏世界地形的设计技术。两者都属于"线性地形特征"（Linear Terrain Feature），但融合方式截然相反：道路需要**填平**地形起伏以实现通行，河流则需要**下切**地形以形成凹槽容纳水流。

道路与河流的设计思路最早在1990年代的RTS（即时战略）游戏中系统化，《命令与征服》（Command & Conquer，1995年，Westwood Studios）的关卡编辑器已经提供了基础的道路刷工具。随着高度图（Heightmap）技术的普及，现代引擎如 Unreal Engine 5 的 Landscape Spline 系统和 Unity 6 的 Terrain Spline 工具，可以沿路径自动压平或下挖地形，使单条道路的融合操作从手动需要约2小时压缩至约15分钟。

道路和河流不只是视觉装饰，它们直接影响玩家的移动路径与战术决策。一条穿越山地的道路会引导玩家行进方向，形成天然的视觉引导线（Visual Lead Line）；一条河流则因无法直接渡过，在地图上制造分隔区域的天然屏障，迫使玩家寻找渡口或桥梁。《地形设计：游戏世界构建手册》（Totten, 2019, CRC Press）将这两类元素列为"地图骨架"的核心组件，认为一张设计良好的关卡地图，其道路与河流系统应在俯视图上即可读出清晰的节奏与引导意图。

## 核心原理

### 道路的地形压平技术

道路在地形上的核心操作是"廊道压平"（Corridor Flattening）：沿路径中心线两侧各取一定宽度（通常为2～8个地形单位，以Unreal Engine的Landscape为例，1个地形单位默认对应100cm），将该区域的高度值统一插值到路径基准高度，再向外以1∶3到1∶6的坡比（Slope Ratio）过渡回原始地形，避免道路边缘出现突兀的断层。

地形压平的核心混合公式为：

$$h_{\text{final}} = \text{lerp}(h_{\text{original}},\ h_{\text{road}},\ \text{smoothstep}(0,\ w,\ w - d))$$

其中：
- $h_{\text{original}}$ 为当前顶点的原始地形高度；
- $h_{\text{road}}$ 为道路样条线在该点的目标高度；
- $d$ 为当前顶点到路径中心线的水平距离；
- $w$ 为压平过渡带的半宽（通常取道路宽度的1.5～2倍）；
- $\text{smoothstep}(0, w, w-d)$ 在 $d=0$（路面中心）时输出1，在 $d=w$（过渡带边缘）时输出0，保证边缘平滑衔接。

山地道路还需要考虑**坡度限制**：现实公路最大坡度约为6%～8%，游戏设计可放宽至15%～20%，但超过此值会让玩家产生明显的"爬坡感"，适合用在需要刻意减慢玩家节奏的战术卡点段落。《黑暗之魂3》（FromSoftware, 2016）高墙路段的主干道坡度经测量约为18%，刻意制造了压迫性的向上推进节奏。

以下是在 Unity 中通过脚本对地形顶点执行廊道压平的核心逻辑示例：

```csharp
// 沿样条线执行廊道压平（简化示意）
float[,] heights = terrain.terrainData.GetHeights(0, 0, resolution, resolution);
for (int x = 0; x < resolution; x++) {
    for (int z = 0; z < resolution; z++) {
        Vector3 worldPos = HeightmapToWorld(x, z, terrain);
        float distToSpline = GetMinDistToSpline(worldPos, roadSpline);
        float roadH = GetSplineHeightAt(worldPos, roadSpline);
        float t = 1f - Mathf.SmoothStep(0f, blendWidth, distToSpline);
        heights[x, z] = Mathf.Lerp(heights[x, z], roadH, t);
    }
}
terrain.terrainData.SetHeights(0, 0, heights);
```

### 河流的下切与流向逻辑

河流设计的第一原则是**高程单调递减**：河流路径在高度图上必须保持从上游到下游的严格递减关系，任何一个局部"上坡"节点都会制造视觉上的倒流悖逻辑。设计师在规划河流走向时，应先在地形高度图上标注等高线，再沿低洼谷地布置河床中心线。实践中推荐每隔10个地形单位至少下降0.5个高度单位，以保证水流方向始终可读。

河床的横截面形态有两类常用选择：

- **V形截面**：适合山区急流，两岸坡度陡峭（约45°～70°），河道窄（宽度约3～6个地形单位）而深（下切约2～4个高度单位），视觉冲击力强，且天然阻断玩家横渡；
- **U形截面**：适合平原缓流，两岸坡度平缓（约10°～20°），河道宽（约8～15个地形单位）而浅（下切约0.5～1个高度单位），便于在特定位置设置浅滩过河点（Ford），引导玩家选择路线。

河流与地形的纹理融合同样关键。以 Unreal Engine 的 Landscape Layer Blend 节点为例，河床底部权重100%绘制湿润沙砾（Wet Gravel）纹理，距河岸0～1.5个地形单位内混入湿土（Wet Soil）纹理，1.5～3个地形单位内再过渡至苔藓草地（Mossy Grass），形成三层渗水带，彻底消除水面与草地直接相接的硬切感。

### 道路与河流的交叉处理

当道路与河流相交时，交叉点天然成为地图上高战略价值的"咽喉节点"（Chokepoint Node）。最常见的处理方式是**桥梁节点**：在河流横断处将道路高度抬升至水位以上1.5～3个地形单位，两侧做半径约4～8个单位的弧形坡道过渡。桥面宽度通常等于道路宽度，若要制造激烈的战术对抗，可将桥面宽度压缩至仅允许2～3个角色并排通过。

另一种交叉处理是**浅滩渡口**（Ford）：在河流特定段落将河床高度抬升至水位以上约0.1～0.3个高度单位（保持视觉上仍为浅水），允许步行单位缓速通过，但对载具或重装单位设置移速惩罚。浅滩位置应放置在玩家需要绕路才能到达的地点，从而使"是否绕道走浅滩"成为一个有意义的路线决策。

例如，《魔兽争霸3》（Blizzard, 2002）经典对战地图"Lost Temple"中，地图四角的主基地区域被河流与断崖共同隔开，两条桥梁节点是全图仅有的地面入侵通道，所有战术决策几乎都围绕这两处桥头展开——这正是交叉节点设计的经典范例。

## 关键公式与参数速查

| 参数 | 道路推荐值 | 河流推荐值 |
|------|-----------|-----------|
| 路/河床宽度 | 3～10 地形单位 | 3～15 地形单位 |
| 边缘过渡带半宽 $w$ | 道路宽度的1.5～2倍 | 河道宽度的1～1.5倍 |
| 坡度上限 | 15%～20% | 无明确上限，但每10单位需下降≥0.5高度 |
| 下切深度（河流） | — | V形：2～4单位；U形：0.5～1单位 |
| 桥梁抬升高度 | 水位以上1.5～3单位 | — |

河流的水力学坡降比（Hydraulic Gradient）可用以下简化公式验证流向合理性：

$$S = \frac{\Delta h}{L}$$

其中 $\Delta h$ 为上下游两端的高度差（单位：地形高度单位），$L$ 为河流路径长度。$S > 0$ 为必要条件；游戏中 $S$ 值通常控制在 $0.005$（平原缓坡）至 $0.05$（山区急流）之间，超出此范围则视觉上会显得不自然。

## 实际应用案例

**案例一：《塞尔达传说：旷野之息》的道路消解策略**

《旷野之息》（Nintendo EPD, 2017）的海拉鲁大地几乎没有明确的硬质道路，而是通过**地形微压平**和**植被稀疏化**暗示历史上曾经存在的道路网络。具体手法是：沿历史道路走廊将地形坡度控制在5%以内，两侧10个单位内的植被密度降低约40%，地面纹理混入约30%的碎石路面（Gravel Path）权重。这使得玩家在探索时能够"感知"到道路的存在，但地表并无任何明显硬化，维持了开放世界的自然感。

**案例二：《星际争霸2》地图编辑器中的河流屏障设计**

《星际争霸2》（Blizzard, 2010）地图编辑器（Galaxy Editor）使用离散地形格（Tile）而非连续高度图，河流通过"水面层"叠加在低洼地形格上实现。标准战术地图中，河流屏障宽度通常为2～4格（约4～8游戏单位），恰好使近战单位无法直接跨越但远程单位可以攻击对岸，制造非对称的交战压力。这一宽度标准在《星际争霸2》的平衡补丁（2015年及2018年）中经过多次调整，最终定型为3格作为"标准屏障宽度"。

## 常见误区

### 误区一：河流任意折弯导致视觉倒流

新手设计师在使用样条线绘制河流时，常为了美观而添加急剧的S形弯曲，却未检查每个控制点的高度值是否严格递减。结果是水面材质的流动UV动画方向与地形走势相反，在弯道处产生明显的"水往坡上流"视觉错误。**解决方法**：绘制完样条线后，立即在编辑器中启用高度折线图视图（Height Profile View），逐节点确认 $\Delta h < 0$（上游高于下游）。

### 误区二：道路压平过宽破坏地形辨识度

将过渡带半宽 $w$ 设置得过大（例如超过道路宽度的3倍），会导致大面积地形被压平，山地的高低起伏特征消失，关卡在俯视图上变成一条宽阔的"平地走廊"，失去地形设计的战术层次。**正确做法**：$w$ 不超过道路宽度的2倍，并在压平区域边缘添加人工凸起（如路缘石地形装饰物）以强调道路边界。

### 误区三：桥梁节点坡道比例失调

桥梁两侧引道的坡度若超过25%，玩家角色会在上桥时产生明显卡顿或滑落感（尤其在物理驱动的载具游戏中）。此外，引道长度过短（小于桥梁抬升高度的3倍）会导致坡面过陡。**参考标准**：引道水平长度应为抬升高度的5～8倍，坡度控制在12%～18%之间。
