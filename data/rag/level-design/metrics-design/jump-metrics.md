# 跳跃度量

## 概述

跳跃度量（Jump Metrics）是关卡设计中用于量化角色跳跃能力的一套数值标准体系，具体包括最大跳跃高度、最大水平跳跃距离、最小跳跃距离、滞空时间、起跳初速度、游戏内重力加速度以及落地缓冲区间等可独立测量的参数。这些数值直接决定了设计师在摆放平台、陷阱、道具和秘密区域时所能使用的最小与最大空间间隔。

跳跃度量的系统化源于1980至1990年代平台游戏的工业化发展。任天堂在设计《超级马里奥兄弟》（1985）时，将马里奥的标准跳跃高度定为约4个砖块单位（每砖块16×16像素），水平跳跃距离在满速助跑状态下可达约5个砖块，这一高宽比约为4:5的跳跃比例成为后续2D平台游戏长达三十年的参考基准。Naughty Dog的关卡设计师Mark Cerny在1990年代的GDC演讲中明确指出，不事先固化跳跃度量就开始摆放平台，等价于"在没有量尺的情况下裁剪布料"。

核心价值在于：跳跃度量将"玩家感觉跳不过去"这一主观反馈转化为可用公式验证的数字约束，使关卡原型阶段的白盒测试效率提升数倍。跳跃度量必须在角色度量（Character Metrics）确定之后才能冻结，因为角色的碰撞箱尺寸、质心高度直接影响抛物线起点与落点的计算位置。

## 核心原理

### 跳跃抛物线的物理基础

游戏内角色跳跃本质上是一段在匀加速虚拟重力作用下的抛物线运动。最大跳跃高度 $H$ 与起跳初速度 $v_0$、游戏内重力加速度 $g$ 的关系为：

$$H = \frac{v_0^2}{2g}$$

滞空时间 $t_{air}$（从起跳到落回同一高度）为：

$$t_{air} = \frac{2v_0}{g}$$

水平最大跳跃距离 $D$ 为：

$$D = v_x \cdot t_{air} = v_x \cdot \frac{2v_0}{g}$$

其中 $v_x$ 为水平移动速度（在满速助跑条件下取最大值）。

以《超级马里奥奥德赛》（2017，Nintendo Switch）的公开数据为例：马里奥地面普通跳跃的 $v_0 \approx 8.8$ 游戏单位/秒，$g \approx 25.7$ 游戏单位/秒²，代入公式得 $H \approx 1.51$ 游戏单位（约合3个标准台阶高度）；满速助跑时 $v_x \approx 7.0$ 游戏单位/秒，$t_{air} \approx 0.685$ 秒，计算得 $D \approx 4.8$ 游戏单位。任天堂关卡团队在白盒阶段用此组数值划定平台间距上限，确保任何必经路径的水平间距不超过 $D \times 0.90 = 4.32$ 游戏单位。

注意：游戏重力通常远大于现实重力（9.8 m/s²），因为设计师刻意缩短滞空时间以保持操控手感的紧绷感。《蔚蓝》（Celeste, 2018）的重力约为现实的9倍（约900像素/秒²），这使得玩家在落地前有极短但精确可控的修正窗口，强化了精密平台游戏的技巧表达空间。

### 跳跃度量的五个关键参数

**① 最大跳跃高度（Max Jump Height, MJH）**：角色从地面起跳至轨迹最高点的垂直距离，通常为角色碰撞箱高度的2.5至4倍。该值决定障碍物"封顶高度"的最低限制——例如《空洞骑士》（2017）中，骑士碰撞箱高度约为0.9单位，最大跳跃高度约为4.5单位（即碰撞箱的5倍），因此天花板距离地面至少需要4.6单位才不会遮断全跳轨迹。

**② 最大水平跳跃距离（Max Horizontal Jump Distance, MHJD）**：满速助跑条件下从起跳点碰撞箱前缘到落点碰撞箱前缘的水平间距。设计师将其划分为三个实用区间：
- 舒适跳跃距离 = $MHJD \times 0.70$（大多数玩家无需精确操作即可达成）
- 挑战跳跃距离 = $MHJD \times 0.85 \sim 0.95$（需要满速助跑，但容错率尚可）
- 精英技巧跳跃 = $MHJD \times 0.96 \sim 1.00$（像素级精度要求，不应用于必经路径）

**③ 最小跳跃距离（Min Jump Distance, MinJD）**：原地静止点按跳跃键时，角色碰撞箱前缘的最小水平前进量。该值用于密集平台段的设计，防止平台间距过小导致"跳过头"——若平台间距小于 $MinJD$，玩家将无法精准停在目标平台上。

**④ 滞空时间（Air Time, AT）**：从起跳帧到落地帧的总帧数，通常以秒或帧（frame）为单位记录。滞空时间与游戏帧率直接相关：以60fps为标准，《蔚蓝》的标准跳跃滞空时间约为28帧（约0.47秒），每帧对应约18像素的垂直位移（在上升段顶峰前后剧烈递减）。

**⑤ 落地缓冲区间（Coyote Time / Landing Buffer）**：角色走出平台边缘后仍可起跳的额外帧数窗口（通常6至12帧，即0.1至0.2秒），以及按下跳跃键但尚未落地时系统自动缓存跳跃指令的"预输入缓冲"帧数（通常8至15帧）。这两个参数均属于跳跃度量的容错设计层，对确保玩家感受到"操控响应灵敏"至关重要。

### 可变跳跃高度与度量区间化

现代平台游戏普遍采用**可变跳跃高度（Variable Jump Height）**机制：跳跃键的按住时长决定最终跳跃高度。技术实现方式为：在角色上升阶段，若玩家提前松开跳跃键，则立即将垂直速度乘以一个衰减系数（通常为0.45至0.60），使上升轨迹提前截断。

《空洞骑士》主角骑士拥有3档可控跳跃高度：
- 点按（<5帧）：跳跃高度约1.5单位
- 中按（5至15帧）：跳跃高度约3.0单位
- 长按（>15帧）：跳跃高度约4.5单位

Team Cherry的关卡设计师针对三档高度分别制定平台高度标准表，确保短跳路径（点按可过）与全跳路径（必须长按）在视觉上有明确的高度差异（至少1.8单位），让玩家在不死亡的情况下通过视觉预判选择正确的跳跃力度。

这意味着跳跃度量不是单一数值，而是一个包含最小值与最大值的**参数区间集合**，关卡设计师必须为区间的下限（最小跳跃）和上限（最大跳跃）分别规划安全路径，而非仅针对最大值设计。

## 关键公式与计算工具

在关卡原型阶段，设计师常用以下Python片段快速计算跳跃参数区间：

```python
# 跳跃度量计算器 Jump Metrics Calculator
# 输入：起跳初速度 v0（单位/秒）、水平速度 vx（单位/秒）、游戏重力 g（单位/秒²）

def jump_metrics(v0, vx, g):
    H_max = (v0 ** 2) / (2 * g)           # 最大跳跃高度
    t_air = (2 * v0) / g                   # 滞空时间（秒）
    D_max = vx * t_air                     # 最大水平跳跃距离

    D_comfortable = D_max * 0.70           # 舒适跳跃距离
    D_challenge   = D_max * 0.90           # 挑战跳跃距离
    D_elite       = D_max * 0.98           # 精英技巧跳跃距离

    return {
        "最大高度 H":         round(H_max, 2),
        "滞空时间 t_air(s)":  round(t_air, 3),
        "最大水平距离 D":     round(D_max, 2),
        "舒适距离 (×0.70)":   round(D_comfortable, 2),
        "挑战距离 (×0.90)":   round(D_challenge, 2),
        "精英距离 (×0.98)":   round(D_elite, 2),
    }

# 示例：以《蔚蓝》近似参数运行
result = jump_metrics(v0=11.0, vx=9.0, g=32.0)
for k, v in result.items():
    print(f"{k}: {v}")
```

**示例输出**：
```
最大高度 H: 1.89
滞空时间 t_air(s): 0.688
最大水平距离 D: 6.19
舒适距离 (×0.70): 4.33
挑战距离 (×0.90): 5.57
精英距离 (×0.98): 6.07
```

设计师将此脚本嵌入Unity或Unreal的关卡工具脚本层，在每次修改角色物理参数后自动重新生成跳跃度量表，使物理调整与关卡间距验证保持同步。

## 实际应用

**平台高度分级（以《蔚蓝》为例）**：Maddy Thorson与Noel Berry在《蔚蓝》的关卡设计文档（Game Developers Conference 2019, "Designing Celeste"演讲）中公开了平台高度的三级分类体系：A级平台高度（短跳可达，约2格）、B级平台高度（中跳可达，约5格）、C级平台高度（全跳方可抵达，约8格），三级均附有具体像素数值，设计师在铺设关卡时严格按级别选择，避免出现"视觉像2格但需要全跳才能上去"的歧义情况。

**陷阱间距校准**：在侧滚动关卡中，地面尖刺群的单个刺尖间距若小于角色碰撞箱宽度的1.2倍，就会出现"无论以何种轨迹落下都必中刺"的无解局面。跳跃度量中记录的最小落点水平误差范围（通常为碰撞箱宽度×0.5至0.8）直接用于排查此类设计缺陷，确保每段陷阱区至少存在一个满足 $宽度 \geq 碰撞箱宽度 \times 1.5$ 的安全落点。

**3D平台游戏的跑酷序列验证**：在3D平台游戏（如《超级马里奥奥德赛》）中，设计师会将连续跳跃序列中每段平台间距除以最大跳跃距离，得出该段的"间距难度系数"（Gap Difficulty Ratio, GDR）。GDR低于0.70为热身段，0.70至0.85为正常玩法段，0.85至0.93为挑战段，超过0.93的段落在必经路径上需经过至少5名不同水平测试员的盲测验证。

**双角色/多角色关卡兼容性**：在拥有多个可切换角色的游戏中（如《星之卡比》系列），每个角色有独立的跳跃度量表。设计师在同一关卡中使用"最保守交集"原则：平台间距不超过所有角色MHJD中的最小值，平台高度不超过所有角色MJH中的最小值，确保任意角色均可通关。
