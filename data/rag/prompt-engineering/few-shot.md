# 少样本提示

## 概述

少样本提示（Few-Shot Prompting）是一种在提示词中嵌入若干输入-输出示例对，从而引导大语言模型（LLM）理解任务格式和期望输出风格的技术。与零样本提示（Zero-Shot）不同，少样本提示通过2至8个具体示例向模型传递"任务说明"，使模型在推理时执行in-context learning，无需更新任何权重参数。

该概念由 Brown 等人在 2020 年 GPT-3 的论文《Language Models are Few-Shot Learners》中系统化提出。论文展示了 GPT-3 在仅提供少量示例的情况下，翻译、算术、问答等任务的准确率大幅超越同规模的零样本模型。这标志着"通过示例教模型"取代"通过微调教模型"成为工程实践中的可行路径。

在 Prompt 工程中，少样本提示之所以重要，是因为它以极低成本（无需GPU训练）换取显著的任务适配效果。对于分类、信息抽取、格式转换等结构化任务，使用3至5个精心挑选的示例通常可将模型准确率提升10%至30%，而添加这些示例的边际成本仅是多消耗的token数量。

## 核心原理

### 示例的结构：输入-输出对

少样本提示的基本单元是**示例对（Exemplar）**，格式为：

```
输入: <样本输入>
输出: <期望输出>
```

模型通过对这些示例对的模式归纳，推断出任务的隐含规则。例如在情感分析任务中：

```
文本: "这家餐厅的菜太咸了"  → 情感: 负面
文本: "服务员非常热情周到"  → 情感: 正面
文本: "价格和分量都很合理"  → 情感: [模型填充]
```

关键点在于最后一行不提供答案，模型依靠前两个示例学到的"映射规则"完成推理。示例之间的格式必须严格一致，否则模型会将格式差异误认为是语义信息的一部分。

### 示例数量与模型规模的关系

示例数量并非越多越好。GPT-3 论文的实验数据显示，对于大多数任务，从0个示例增加到1个示例是准确率提升最陡峭的区间；从1个增加到4个仍有稳定提升；超过8个示例后，提升收益趋于平稳，且会占用宝贵的上下文窗口。此外，模型规模越大，从少量示例中学习的能力越强：参数量为 6.7B 的模型需要约8个示例才能达到 175B 模型使用2个示例的效果。

对于受 token 限制约束的工程场景（如 GPT-4 Turbo 的 128K 窗口被长文档占用时），通常选择3至5个示例作为工程平衡点。

### 示例选择策略：质量优先于数量

示例的选择直接决定少样本提示的效果，有三种核心策略：

**随机采样**：从标注集中随机抽取示例，实现简单但性能不稳定，方差较大。

**多样性采样**：确保示例覆盖不同的输入模式。例如，分类任务中每个类别至少出现一次，避免模型产生预测偏置。实验表明，对于4类分类任务，若4个示例全属于同一类别，模型对该类别的召回率会虚高约15%至25%。

**基于相似度的动态检索（Dynamic Few-Shot）**：使用 embedding 模型（如 text-embedding-3-small）计算当前输入与示例库的余弦相似度，实时检索最相关的示例。此方法在实际生产系统中表现最优，但需要额外的向量检索基础设施（如 FAISS 或 Pinecone）。

## 实际应用

**实体抽取任务**：在从合同文本中抽取"付款日期"和"违约金额"的场景中，提供3个已标注合同片段作为示例，可使 GPT-4 的抽取 F1 分数从零样本的约72%提升至约88%。示例应覆盖日期格式多样（如"2024年3月1日"与"2024/03/01"）的情况。

**代码风格转换**：将 Python 2 代码转换为 Python 3 时，提供`print "x"` → `print("x")`、`unicode` → `str` 等2至3个典型示例对，模型便能准确处理`xrange`、`dict.iteritems()`等长尾语法。

**客服意图分类**：某电商系统将用户消息分为"退款"、"物流查询"、"商品咨询"等7个类别，使用每类1个示例（共7个）的少样本提示，在内部测试集上准确率达到91.3%，而零样本仅为78.6%。

## 常见误区

**误区一：示例标签错误反而效果更好**

部分工程师认为示例内容不重要，只要格式对即可。Min 等人 2022 年的研究《Rethinking the Role of Demonstrations》确实发现输入分布比标签准确性更重要，但这仅适用于标签与任务本身语义关联弱的情况。对于逻辑严密的任务（如数学推理、代码生成），错误标签会导致模型准确率下降超过20%。不能将该结论泛化应用。

**误区二：示例越长、越详细越好**

冗长的示例会消耗大量 token，且超出模型有效注意力范围。实验表明，示例长度超过整体提示词长度的60%时，模型对任务指令本身的遵从度反而下降。示例应聚焦于展示输入-输出映射，而非提供背景知识。

**误区三：少样本提示可以替代微调（Fine-tuning）**

少样本提示适合快速原型和低成本部署，但在大规模推理时每次调用都需携带示例（成本随调用次数线性增长），而微调模型只需一次训练成本。当每日调用量超过约10万次且任务高度专一时，微调在总成本上通常优于少样本提示。

## 知识关联

**前置概念**：掌握"提示词基础"中的角色设定、指令清晰度原则是使用少样本提示的前提——示例对本质上是对指令的具体化补充，模糊的任务指令配合清晰示例仍会产生偏差。

**延伸至思维链（CoT）**：少样本提示可与思维链技术直接组合，形成**少样本CoT提示**。具体做法是在每个示例的输出部分加入逐步推理过程（而非直接给出答案），Wei 等人2022年的论文证明这一组合在GSM8K数学基准上将 PaLM 540B 的准确率从17.9%提升至56.9%，是目前已知效果提升最显著的提示组合方式之一。

**延伸至Prompt模板设计模式**：当少样本示例需要在生产系统中动态替换时，需要设计支持示例插槽（Example Slot）的模板结构。Prompt模板设计模式中的"示例池管理"和"动态组装"策略，正是对少样本提示在工程化落地中所面临的示例维护问题的系统性解答。
