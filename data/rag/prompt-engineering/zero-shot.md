# 零样本提示

## 概述

零样本提示（Zero-Shot Prompting）是一种无需在提示词中提供任何示例，直接让大语言模型完成任务的提示策略。这种方法依赖于模型在预训练阶段所学到的大量知识和语言模式，通过清晰的任务描述让模型"凭空"执行从未在提示中见过的指令。GPT-3 在2020年的论文《Language Models are Few-Shot Learners》中正式将"零样本"与"少样本"概念区分开来，为该方法的系统化研究奠定了基础。

零样本提示的本质是一种指令泛化能力的测试——它验证模型是否真正"理解"了任务意图，而非仅仅模仿示例中的格式和模式。像 ChatGPT、Claude 这类经过指令微调（Instruction Fine-tuning）和人类反馈强化学习（RLHF）训练的模型，零样本能力显著强于仅预训练的基础模型。这也是为什么同样一句"将这段话翻译成英文"的指令，在 GPT-3.5 上执行效果远优于原始 GPT-3 的原因。

在实际工程场景中，零样本提示是调用 API 时成本最低的提示方式——无需在 prompt 中附加多个示例，直接减少了输入 token 数量，从而降低了调用费用和延迟。对于任务类型明确、语言模型已充分覆盖的场景（如摘要、分类、翻译），零样本往往是首选起点。

## 核心原理

### 指令微调赋予模型零样本能力

零样本提示之所以有效，根本原因在于**指令微调**（Instruction Fine-tuning）。OpenAI 的 InstructGPT 论文（2022）显示，仅用1.3B参数经过指令微调的模型，在零样本任务上的人类评分可超过175B参数的原始 GPT-3。这说明零样本能力的强弱并非纯粹由模型规模决定，而是由训练方式决定。提示词工程师选择模型时，应优先选择有明确"chat"或"instruct"标记的变体，而非原始基础模型。

### 有效零样本提示的结构要素

一个高质量的零样本提示通常包含三个要素：**角色定义**、**任务描述**、**输出约束**。例如：

```
你是一位资深法律文书编辑。
请将以下合同条款改写为普通用户能理解的白话文。
输出格式：每条款对应一段，不超过50字。
[合同内容]
```

与空泛的指令"简化这段合同"相比，明确角色（法律编辑）、任务动词（改写）、约束条件（50字、白话文）三要素的零样本提示，在 GPT-4 上的准确率实验中平均提升约15-30个百分点。任务描述中的动词选择尤为关键："列举"、"分析"、"分类"、"总结"这类强动词比"处理"或"看一下"更能激活模型的正确行为模式。

### 零样本提示的失效边界

零样本提示并非万能。当任务涉及**特定输出格式**（如严格的 JSON Schema）、**领域专有风格**（如某公司的内部报告模板）或**复杂推理链**时，零样本往往不可靠。一个典型案例：要求模型在零样本下输出包含嵌套字段的 JSON 时，字段名称错误率显著上升——此时应使用 Few-Shot 补充格式示例，或改用结构化输出 API（如 OpenAI 的 `response_format: json_object`）。此外，任务的模糊性与零样本失败率正相关：提示词越模糊，模型越容易输出不符合预期的结果。

## 实际应用

**文本分类场景**：电商平台评论情感分析，零样本提示可直接写作："判断以下评论的情感倾向，从'正面'、'负面'、'中性'中选择一个词回答：[评论内容]"。这种封闭选项约束（Constrained Output）是零样本分类任务成功率的关键设计，避免模型自由发挥。

**内容转换场景**：将 Markdown 格式的技术文档转换为适合微信公众号的阅读风格，零样本提示可以是："你是一位科技媒体编辑，将以下 Markdown 技术文档改写为适合普通读者的微信文章，保留核心信息，语气轻松，段落不超过4句话：[文档内容]"。此类转换任务因为格式模式清晰，零样本成功率接近生产可用标准。

**快速原型验证**：在开发 AI 应用的早期阶段，工程师通常先用零样本评估模型的基础能力边界。若零样本结果已满足需求，则直接进入生产；若结果差距明显，再升级到少样本或微调方案。这种"零样本优先"的原则可以节省大量标注数据准备和工程成本。

## 常见误区

**误区一：认为零样本等同于"随意写几个字"**。许多初学者将零样本理解为"不需要认真设计提示词"，实际上零样本要求提示词设计更精确。没有示例兜底，任何模糊的描述都会直接导致输出偏离。"帮我写一篇文章"是糟糕的零样本提示，"以说明文体裁，用600字介绍光合作用的三个阶段"才是有效的零样本提示。

**误区二：将零样本失败归因于模型能力不足，而非提示词质量**。当零样本输出结果不理想时，第一步应该检查提示词中是否缺乏角色、约束或明确的任务动词，而不是立刻换用更大的模型。研究发现，同一任务经过提示词优化后，GPT-3.5 的零样本结果可超过未经优化的 GPT-4。

**误区三：混淆零样本与无提示（Raw Prompting）**。零样本提示依然是经过设计的完整提示，包含明确指令；而"无提示"或"裸查询"指不加任何任务说明直接提问。两者的输出质量差距显著。例如，直接输入一段合同文本（无提示）与加上"请提取以下合同中的违约条款"（零样本提示）相比，后者的准确性和一致性更高。

## 知识关联

零样本提示建立在**提示词基础**知识之上——理解提示词的 System/User/Assistant 角色分工、token 概念以及上下文窗口长度限制，是写出有效零样本提示的前提。没有对这些基础结构的掌握，无法判断提示词为何失效。

从零样本出发，自然延伸到**少样本提示（Few-Shot Prompting）**的比较学习：当明确知道零样本在哪类任务（特殊格式、风格一致性）上失效时，才能有针对性地设计少样本示例。零样本与少样本之间的选择，本质上是一个"成本-质量权衡"问题：零样本 token 消耗少但可控性弱，少样本 token 消耗多但格式可预测性强。这一对比认知是 Prompt 工程师系统化选择提示策略的基础判断框架。
