# 提示词基础

## 概述

提示词（Prompt）是用户或系统传递给大语言模型的输入文本，它直接决定模型从其参数空间中检索并生成何种输出。与传统软件通过函数调用控制程序行为不同，提示词是以自然语言为介质的"软性指令"，模型根据提示词在概率分布上采样生成响应，而非执行确定性逻辑。

提示词工程（Prompt Engineering）作为独立领域的兴起可追溯至2020年GPT-3发布后的研究实践。OpenAI在其技术报告中首次系统展示了通过改变输入文本格式即可在零样本或少样本场景下激活模型能力，这打破了"必须微调才能适配任务"的传统认知。2022年前后，随着ChatGPT的普及，提示词工程从学术研究迅速渗透到工业应用层面。

理解提示词基础对于后续所有Prompt技术的学习至关重要。提示词的质量直接影响模型输出的准确性、格式一致性和安全性——同一个GPT-4模型，面对"翻译这段文字"与"你是一位专业翻译，请将以下文字从英文译为正式中文，保留专业术语"这两种提示词，输出质量可能存在显著差距。

## 核心原理

### 提示词的四种基本组成元素

一个完整的提示词通常由以下四类元素构成，并非每个提示词都需要包含全部四类，但了解各类作用有助于按需构建：

- **指令（Instruction）**：明确告诉模型要执行的任务，例如"总结"、"分类"、"翻译"。指令越具体，模型的输出越可预测。
- **上下文（Context）**：为模型提供背景信息，帮助其缩小推理空间。例如"以下文本来自医学文献"会引导模型采用更专业的词汇体系。
- **输入数据（Input Data）**：需要模型处理的具体内容，通常用特殊符号（如三重反引号 ` ``` ` 或`<tag>`标签）与其他部分区隔，以防模型混淆指令与数据。
- **输出指示（Output Indicator）**：规定输出格式或风格，例如"用三点以内的要点回答"或"以JSON格式返回结果"。

### 模型如何"读取"提示词：Tokenization视角

大语言模型并非以字符或单词为单位处理提示词，而是将其切分为**Token**（词元）序列。以GPT-4使用的cl100k_base分词器为例，汉字通常每个字对应2-3个Token，而英文单词平均约1.3个Token/词。这意味着相同语义内容用中文表达会消耗更多Token，从而影响**上下文窗口（Context Window）**的使用效率。GPT-4 Turbo的上下文窗口为128,000 Token，但实际应用中提示词过长会导致模型对早期内容的"注意力稀释"。

提示词在被处理时，模型依据其预训练分布对每个后续Token赋予概率。**温度参数（Temperature）**控制采样时的随机程度：Temperature=0时模型接近贪心解码，始终选择概率最高的Token，输出高度确定；Temperature=1时保持预训练分布的原始随机性；超过1则增大低概率Token被选中的机会，输出更具创意但也更不稳定。

### 提示词格式对输出的影响

提示词的排版和分隔符选择会实质性影响输出质量。研究（Wei et al., 2022）表明，在提示词末尾加入"Let's think step by step"（逐步思考）可将GPT-3在MultiArith数学数据集上的准确率从17.7%提升至78.7%。这说明**触发特定推理模式**是提示词工程的核心技术之一。

常见格式规范包括：使用`###`或`---`分隔指令与输入数据；使用XML风格标签如`<document></document>`包裹长文本；在要求结构化输出时提供输出样例（示范格式）而非仅描述格式。避免使用礼貌性填充语（如"请你帮我……"中的多余语气词），因为这些词会消耗Token且不增加语义约束力。

## 实际应用

**文本分类任务中的提示词构建**：假设需要对用户评论进行情感分类，弱提示词为"判断这条评论的情感"，而强提示词为：

```
将以下用户评论分类为【正面】、【负面】或【中性】三类之一。
只输出分类标签，不需要解释。

评论：```{user_review}```
分类：
```

后者通过限定输出标签集合、禁止解释、明确分隔符三个手段，将模型输出的格式稳定性大幅提升，适合作为API调用中下游程序直接解析的输入。

**代码生成中的上下文注入**：在要求模型生成Python函数时，仅写"写一个排序函数"效果远不如提供类型注解要求、已有代码风格示例以及具体约束（如"不使用内置sort方法"）。上下文越能缩小模型的解空间，生成结果越贴合需求。

## 常见误区

**误区一：提示词越长越好**。许多初学者认为加入大量说明能让模型理解得更准确，但冗长的提示词会引入噪声，导致模型注意力分散。实验表明，在GPT-3.5上，超过3000 Token的提示词对于简单分类任务往往不优于精炼后的300 Token版本。有效的提示词应追求信息密度，每句话都应承担具体的约束或说明功能。

**误区二：将提示词视为固定模板，忽略模型版本差异**。针对GPT-3.5优化的提示词在GPT-4上可能表现不同，因为两者的预训练数据、RLHF对齐方式和指令遵循能力存在本质差异。GPT-4对隐式指令（如通过示例暗示格式）的敏感度更高，而GPT-3.5有时需要更显式的格式说明才能稳定输出。提示词应针对目标模型版本进行专项测试和调优。

**误区三：认为提示词只是"输入框里的文字"**。实际工程系统中，提示词包含System Prompt（系统提示）、Human Turn（用户轮次）和Assistant Turn（助手轮次）等多个角色位置，不同位置的内容对模型行为的影响机制不同。System Prompt中的指令通常比用户消息中的指令具有更高的"遵循优先级"，这是因为模型在RLHF训练阶段被强化了对系统角色的服从行为。

## 知识关联

学习提示词基础需要依赖对GPT与解码器模型的理解：模型的自回归生成机制（每次预测下一个Token）直接解释了为何提示词的措辞顺序和措辞方式会影响模型的生成路径。理解Transformer的注意力机制有助于解释为何提示词中的关键信息最好放在开头或结尾——这与模型对序列位置的注意力分布有关。

在提示词基础之上，**零样本提示**探讨不提供示例时如何通过纯指令激活模型能力；**少样本提示**系统研究示例数量与质量对模型输出的影响规律；**结构化输出（JSON Mode）**解决如何强制模型输出机器可解析格式的工程问题；**System Prompt设计**则聚焦于对话系统中系统角色提示词的最佳实践。**提示注入攻防**研究恶意用户如何通过构造特殊输入覆盖原始提示词指令，以及如何防御——这一安全威胁的根源正在于本文所描述的提示词对模型行为的直接控制能力。
