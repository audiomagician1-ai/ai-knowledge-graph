# 每日/周常任务（Daily/Weekly Recurring Quests）

## 概述

每日/周常任务（Daily/Weekly Recurring Quests）是网络多人游戏中以固定时间周期重置的任务系统，玩家在每个周期内完成指定目标即可获得奖励，周期结束后任务自动刷新。这类任务的核心特征在于**时间门控（Time Gating）**——奖励的总量被时间窗口严格限制，而非仅由玩家投入的时长决定。

以《魔兽世界》为例，其每日任务系统自2.1版本（燃烧的远征，2007年5月）起正式在"风暴之眼"日常任务区引入，并在2.4版本（太阳之井高地，2008年3月）将每日任务上限明确设定为每账号25个/天。这一上限直接调节了玩家的声望获取速度，将最高声望等级（崇拜）的刷取时长强制拉长至至少14天，防止玩家在首周就榨干全部内容。

每日任务与周任务的区别不仅在于重置间隔，更体现在奖励体量与驱动逻辑上。日任务设计为**习惯形成驱动器（Habit Loop Driver）**——小额但稳定的奖励利用行为心理学中的"固定间隔强化"（Fixed-Interval Reinforcement）原理，促使玩家每天登录；周任务则设计为**里程碑事件（Milestone Event）**，单次奖励体量大、稀有度高，给玩家提供每周一次的强烈满足感。在排行榜系统中，每日/周任务是控制积分增长曲线的关键阀门，确保付费玩家与免费玩家之间的积分差距不会因无限刷任务而无限扩大。

本文的理论基础参考了游戏设计学者 Juho Hamari 等人在《Why do people play games? A meta-analysis》（Hamari & Keronen, 2017, *Computers in Human Behavior*）中对任务驱动留存率的研究，以及 Raph Koster 的经典著作《游戏设计快乐之道》（《A Theory of Fun for Game Design》，2004）中关于奖励间隔与学习曲线的论述。

## 核心原理

### 时间周期设计与重置机制

日任务通常在服务器时间的固定时刻重置，最常见的是 UTC+0 凌晨 0 点或地区服务器的当地午夜。《原神》（miHoYo，2020年9月上线）采用每日任务在 UTC+8 时间 04:00 重置的设计，而非自然午夜。这一选择并非随意——04:00 是中国大陆玩家活跃度曲线的最低谷，将重置时刻设于此处可显著减少重置瞬间的服务器并发压力，同时避免"跨天任务"引发的玩家投诉（即23:50开始做任务却无法在重置前完成的情况）。

周任务的重置时间选择同样有深刻历史渊源。北美服《魔兽世界》自2004年起将团队副本周重置定在周二服务器维护后，中国区代理商网易将周重置调整为周四，这一差异导致中美两服的玩家行为节奏出现明显分叉：美服玩家习惯周二/周三冲击排行榜，中服玩家则在周四至周六形成活跃高峰。

重置时刻的选择直接影响排行榜的积分分布曲线。若重置发生在玩家活跃时段，排行榜排名在重置后 2–4 小时内会出现"积分地震"——大量玩家同步刷新积分，头部玩家之间的名次在数十分钟内反复易主。《命运2》（Bungie，2017年）为此采用"滚动式每日挑战"（Rolling Daily Challenges）机制：将每日挑战细分为"公共区域挑战"（每天 17:00 UTC 重置）、"罢工挑战"（每天 17:00 UTC 重置）和"熔炉/战争模式挑战"（每周四 17:00 UTC 重置）三类，不同子类在不同时刻分批生效，使排行榜积分的增长曲线从"阶跃函数"变得更接近"分段线性函数"，有效平摊了服务器压力和排行榜波动。

### 奖励调控公式与积分上限

每日任务的积分设计遵循以下基本约束，可形式化表达为：

$$
\sum_{d=1}^{7} S_{\text{daily}}(d) \leq \alpha \cdot S_{\text{weekly}}
$$

其中 $S_{\text{daily}}(d)$ 表示第 $d$ 天日任务的积分上限，$S_{\text{weekly}}$ 为周任务的积分上限，系数 $\alpha \in [0.85, 1.20]$。该不等式的含义是：**7天累计日任务积分不应超过周任务积分的120%**，以防止日任务成为碾压周任务的刷分捷径，也不应低于85%，以防止玩家完全放弃日任务转而只打周任务。

以《英雄联盟》赛季积分体系（Riot Games，2023赛季）为例，每日任务提供的蓝色精粹（Blue Essence）上限约为 1500–2000 点/天，而每周胜场挑战任务单次可提供 6000–10000 点，满足 $7 \times 2000 = 14000 \approx 1.4 \times 10000$，略高于上限系数 1.20，因此 Riot 在2023赛季微调了每日任务的积分权重，将上限压缩至 1200 点/天，使比值回落至约 $1.05 \times S_{\text{weekly}}$。

奖励梯度（Reward Gradient）设计同样关键。以《炉石传说》（暴雪，2014年上线）的日常任务为例，奖励从 40 金币（"击败 2 位对手"）到 80 金币（"在某特定职业中赢得 3 场对局"）不等。玩家每天可手动"重新滚动"一次低价值任务，但系统保证重新滚取的任务奖励**不低于**被替换任务的价值，且每日只允许一次替换——这个设计防止玩家无限重抽以获得最优任务组合，同时保留了一定的玩家能动性，提升任务参与感。

### 任务池多样性与防刷设计

任务池的多样性直接决定玩家是否愿意长期参与竞争。若任务池只有5种固定类型，老玩家在第2周便会产生明显疲劳感（根据 Hamari & Keronen 的研究，任务多样性不足是导致第30天留存率骤降的第二大因素，仅次于社交功能缺失）。《命运2》的每日挑战池（截至"巫师王"资料片，2022年）包含超过 60 种子任务类型，涵盖战役击杀、公共活动、秘境副本和PvP模式，确保连续 30 天的日任务体验中单一任务类型的重复率低于 15%。

防刷机制（Anti-Farming）是每日任务设计的核心约束，常见手段包括：

- **硬上限（Hard Cap）**：每日任务最多完成 N 个，超出部分不计入积分或奖励（如《魔兽世界》经典服的 25 个/天上限）。
- **衰减机制（Diminishing Returns）**：同类任务连续完成3次后，第4次起奖励递减 50%，强制玩家切换任务类型。《战争框架》（Warframe，Digital Extremes）的声望积分即采用此机制。
- **账号绑定奖励（Account-Bound Rewards）**：任务奖励标记为"灵魂绑定"，不可交易或邮寄给小号，从根源上杜绝工作室批量刷任务后在拍卖行倾销。
- **IP/设备指纹检测**：检测同一IP下短时间内多账号完成相同任务的异常行为，触发后对该账号的当日任务积分归零。

## 关键公式与算法

以下是一段用于计算每日任务奖励衰减的伪代码，展示了防刷机制的实现逻辑：

```python
def calculate_quest_reward(player, quest_type, base_reward):
    """
    计算玩家本次任务实际获得的奖励积分。
    
    Args:
        player: 玩家对象，包含 today_quest_log（当日任务日志）
        quest_type: 任务类型标识符，如 "pvp_win" / "dungeon_clear"
        base_reward: 该任务的基础奖励积分
    
    Returns:
        actual_reward: 衰减后的实际奖励积分
    """
    # 统计今日同类任务完成次数
    same_type_count = player.today_quest_log.count(quest_type)
    
    # 每日总任务数硬上限（参照《魔兽世界》25个/天）
    DAILY_HARD_CAP = 25
    total_today = len(player.today_quest_log)
    if total_today >= DAILY_HARD_CAP:
        return 0  # 超出上限，本次任务不计入奖励
    
    # 衰减规则：同类任务第4次起奖励乘以0.5
    DIMINISHING_THRESHOLD = 3
    DIMINISH_FACTOR = 0.5
    
    if same_type_count >= DIMINISHING_THRESHOLD:
        actual_reward = int(base_reward * DIMINISH_FACTOR)
    else:
        actual_reward = base_reward
    
    # 记录本次任务到当日日志
    player.today_quest_log.append(quest_type)
    return actual_reward
```

该算法中，`DAILY_HARD_CAP = 25` 直接对应《魔兽世界》2.4版本的设计参数，`DIMINISH_FACTOR = 0.5` 则参照《战争框架》的声望衰减系数。在实际生产环境中，这两个参数通常存储于服务器端的远程配置（Remote Config）表中，支持运营团队在不停服的情况下实时调整。

## 实际应用

### 排行榜积分供给控制

在竞技类游戏中，每日任务积分构成排行榜的"基础票仓"。《王者荣耀》（腾讯天美，2015年上线）的每日任务系统为每位玩家提供固定的赛季积分增量，设计目标是使稳定完成每日任务的玩家，在180天赛季末的积分优势约为从不完成者的 **1.3–1.8 倍**。该倍数被刻意控制在此区间：低于 1.3 倍则日任务激励不足，玩家懒得每日登录；高于 1.8 倍则排行榜被"签到党"主导，纯竞技能力强的玩家会因积分差距放弃追榜。

### 留存率与任务设计参数的关联

根据移动游戏数据分析平台 GameAnalytics 发布的《2022 Mobile Gaming Benchmarks》报告，配置了每日任务系统的手游，其第7日留存率（D7 Retention）平均比无日任务系统的同类游戏高出 **12–18 个百分点**。具体设计参数影响如下：

- 每日任务完成时间若控制在 **8–15 分钟**，D30 留存率最优；低于 5 分钟则给玩家"太简单没挑战"的感受，高于 20 分钟则产生"今天没时间做"的放弃心理。
- 周任务的最优完成时间窗口为 **45–90 分钟**（可分多次完成），设计为必须一次性连续完成的周任务会导致该任务完成率下降约 35%。

### 案例：《原神》每日委托与树脂系统的联动

《原神》将每日委托（4个/天，每个奖励 60 原粹树脂等值的材料）与体力系统（原粹树脂，上限 160 点，每 8 分钟恢复 1 点）深度绑定。这一设计使玩家每天的有效游戏时间被硬性限制在约 **30–50 分钟**（4个委托约15分钟 + 消耗160树脂刷本约20–35分钟），精准对应了其目标用户群体（18–28岁上班族）的工作日可用游戏时间。树脂系统本质上是对每日任务奖励的二次门控——即使玩家完成了每日委托，若不消耗树脂，材料掉落效率也会大幅下降，迫使玩家每天在固定时间段登录，将日活跃
