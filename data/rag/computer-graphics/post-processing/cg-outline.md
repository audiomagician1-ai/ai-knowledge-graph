# 描边效果

## 概述

描边效果（Outline / Edge Detection）是一种在渲染后处理阶段通过检测相邻像素之间的深度或法线不连续性来自动绘制轮廓线的技术。与直接在几何体边缘放置额外面片的"物体空间描边"不同，这种方法完全在屏幕空间（Screen Space）中工作，无需修改原始网格，只需对已渲染的G-Buffer进行采样分析。

该技术最早在1990年代的卡通渲染研究中被系统化。日本游戏《噬神者》（God Eater）系列和《苍蓝誓约》等采用的赛璐璐风格渲染器都将这种后处理描边作为管线的核心环节。学术上，Saito和Takahashi于1990年在SIGGRAPH发表的论文《Comprehensible rendering of 3-D shapes》首次提出用法线和深度缓冲区进行边缘检测，为该领域奠定了理论基础。

后处理描边的价值在于它能以极低的网格修改成本，在任意复杂度的模型上生成稳定的描边线条。对于包含数千个独立对象的场景，逐物体的几何描边代价极高，而后处理描边只需额外一至两个全屏Pass即可覆盖整个场景。

## 核心原理

### 深度不连续性检测

深度检测的基本思路是对当前像素周围的邻近像素进行深度值采样，若差值超过阈值则判定为边缘。常用的算子是**Roberts交叉算子**，它使用2×2的对角像素对进行差分：

```
Edge = |D(x,y) - D(x+1,y+1)| + |D(x+1,y) - D(x,y+1)|
```

其中 `D(x,y)` 表示像素 `(x,y)` 处的线性深度值。当 `Edge` 超过用户设定的阈值 `threshold`（通常为0.01到0.1之间，取决于场景尺度）时，该像素被标记为边缘像素并染为描边颜色。

深度检测的局限性在于它只能捕捉**轮廓边**（Silhouette Edge），即物体与背景或前后物体之间的分界线。物体表面内部的折痕、细节结构等"内部边"对深度值影响极小，单纯依赖深度无法检出。

### 法线不连续性检测

法线检测通过对比相邻像素的世界空间或视图空间法线向量来发现曲率突变区域。最常用的是**Sobel算子**，它在水平和垂直两个方向各使用一个3×3卷积核：

```
Gx = [-1,0,+1; -2,0,+2; -1,0,+1] · N
Gy = [-1,-2,-1;  0,0, 0; +1,+2,+1] · N
Edge_Normal = sqrt(Gx² + Gy²)
```

其中 `N` 是法线向量的某个分量（对RGB三个分量分别计算后取最大值）。法线检测能够发现深度检测盲区内的折痕和硬边，两者结合使用可覆盖绝大多数需要描边的几何特征。法线需要存储在G-Buffer中，通常编码为RGB8格式，每个分量映射到 `[0,1]` 区间：`encoded = normal * 0.5 + 0.5`。

### 深度与法线的融合策略

实际渲染管线中，深度描边和法线描边的检测结果需要通过逻辑OR合并：任意一项检测到边缘即输出描边颜色。为避免描边宽度在远处物体上变得过细、近处物体上过粗，许多实现会引入**深度自适应阈值**：将检测阈值乘以线性深度值，使远处物体的阈值更宽松，维持视觉上一致的描边宽度。Unity HDRP的卡通描边示例中，这一系数通常设为 `threshold = base_threshold * depth * 0.1`。

## 实际应用

**《原神》风格描边**：该游戏同时采用后处理描边和顶点外扩描边两种方案。后处理阶段的深度+法线描边负责处理远景角色和场景物体的轮廓，而角色近景则叠加顶点外扩方案以获得更可控的线条粗细。

**Unity Shader Graph实现**：在Unity的后处理栈（URP/HDRP）中，可通过自定义Renderer Feature采样`_CameraDepthTexture`和`_CameraNormalsTexture`实现描边。Roberts算子只需4次纹理采样（对角线上的两对像素），计算开销约为每帧0.1至0.3ms（在1080p分辨率下，依GPU型号而异）。

**游戏《卧龙：苍天陨落》**：该作使用了仅基于深度的描边配合色彩分级，描边颜色并非纯黑色，而是当前位置环境色的深色变体，使描边融入光照环境，减少突兀感。

## 常见误区

**误区一：认为增大采样半径就能获得更粗的描边**。简单增大Roberts或Sobel算子的采样间距（stride）确实可以加宽描边，但会导致边缘检测精度下降，在曲面相交处出现"虚影"或断裂。正确做法是先检测出单像素宽的精确边缘，再通过**膨胀（Dilation）操作**用卷积核扩展边缘宽度，通常使用3×3或5×5的矩形最大值滤波即可。

**误区二：认为后处理描边可以完全替代几何描边**。后处理描边对透明物体（Transparent Objects）无效，因为透明物体不写入深度缓冲区，其法线也通常不进入G-Buffer。角色头发、玻璃等透明部件的轮廓必须依赖几何外扩或单独的Stencil描边Pass处理。

**误区三：在视图空间法线中直接比较点积**。有开发者用 `1 - dot(N1, N2) > threshold` 来判断法线差异，但这对接近90°的法线变化（如建筑直角拐角）不敏感，因为点积在接近0时变化缓慢。应改用对各分量分别做差分的Sobel/Roberts算子，对全角度范围的法线变化保持均匀的检测灵敏度。

## 知识关联

学习后处理描边需要对**后处理概述**中介绍的全屏Pass机制有明确认识，包括如何将渲染结果绑定为纹理并在后处理着色器中采样。描边效果依赖G-Buffer中的深度纹理和法线纹理，因此理解这两个缓冲区的编码方式（线性深度 vs. 非线性NDC深度、八面体法线编码等）是实现正确结果的前提。

掌握后处理描边后，自然过渡到**赛璐璐着色**的学习。赛璐璐渲染将描边效果与色阶量化光照、高光和阴影结合，构成完整的卡通渲染管线。描边是赛璐璐风格视觉表达的重要组成部分，控制线条的颜色、粗细对最终风格化效果影响显著。此外，屏幕空间环境光遮蔽（SSAO）同样使用深度和法线缓冲区，其采样策略与法线描边的差异——SSAO关注半球内的遮蔽积分而非局部梯度突变——有助于加深对屏幕空间算法族的整体理解。
