# 后处理体积

## 概述

后处理体积（Post-Process Volume）是虚幻引擎（Unreal Engine）中用于在三维空间中定义后处理效果覆盖区域的场景对象。它的核心功能是让开发者将景深、色调映射、曝光、泛光等后处理参数与特定的空间区域绑定，从而实现玩家进入不同区域时后处理效果自动切换或混合。

后处理体积的概念随着游戏引擎的场景化编辑工具的发展而出现。在早期的渲染管线中，全局后处理参数由代码统一控制，切换效果需要手动触发逻辑。虚幻引擎 3 时代引入了后处理体积的雏形，到 UE4 和 UE5 中已发展为功能完整的体积Actor，支持无限范围模式与有界形状两种工作形态。

对于需要表达不同光照环境的场景来说，例如室外阳光地带与室内阴暗地窖之间的视觉切换，后处理体积提供了一套无需额外代码的参数化空间方案。美术和关卡设计师可以直接在编辑器中拖放体积并调整参数，而无需程序员介入，这显著降低了迭代成本。

## 核心原理

### 体积的空间判定与混合权重

后处理体积在场景中以碰撞体积（通常是 Box Component 或 Sphere Component）定义其边界。摄像机（而非玩家角色）的位置决定当前处于哪个体积的覆盖范围内。当摄像机进入体积边界后，系统将根据 **Blend Radius** 参数（以厘米为单位）和摄像机距边界的距离计算混合权重（Blend Weight），公式为：

**BlendWeight = clamp((BlendRadius - distanceToBorder) / BlendRadius, 0, 1)**

其中 `distanceToBorder` 为摄像机到体积边界表面的最近距离。当 BlendRadius 设为 0 时，体积参数在边界处硬切换；设为较大值（如 500 cm）时，过渡将在摄像机接近边界时提前开始软混合。

### 优先级系统（Priority）

当多个后处理体积的范围互相重叠时，引擎通过 **Priority** 字段决定混合顺序。Priority 是一个浮点数，值越高的体积越晚被混合，意味着其参数对最终结果的影响权重更大。具体地说，引擎按照 Priority 从低到高对所有处于激活状态的体积进行排序，逐一将参数叠加混合，最高优先级的体积参数将最后覆盖。

这意味着优先级为 10 的局部小体积（如角落房间）可以完全覆盖优先级为 0 的全局体积，但若两者都激活且优先级相同，混合结果取决于各自的 Blend Weight 比例，不存在确定性的"谁覆盖谁"。

### 无界模式（Unbound）

将体积的 **Infinite Extent (Unbound)** 选项设为 true 后，该体积不再需要摄像机在其物理边界内，而是始终对全场景生效，Blend Radius 也随之失去意义。这种全局体积通常设置最低 Priority（如 0），作为场景的基础后处理层，再由各局部有界体积以更高优先级在上层叠加特定区域效果。一个典型做法是在关卡中放置一个 Unbound 体积承载全局色调映射与自动曝光参数，而洞穴入口处的局部体积负责额外增强阴影部分的色彩饱和度。

### 参数的"覆盖"开关

后处理体积内的每个参数旁边都有一个勾选框，未勾选的参数不参与混合计算，直接被跳过。这意味着一个体积可以只覆盖"景深焦距"单一参数，而将色调映射和曝光完全留给其他体积处理。这种细粒度的参数激活机制避免了各体积之间不必要的参数冲突。

## 实际应用

**恐怖游戏的走廊过渡**：在一条从明亮大厅通向阴暗密室的走廊中，放置一个 Blend Radius 为 300 cm 的后处理体积覆盖密室区域。当玩家走进走廊时，景深模糊量（Focal Distance）从无限远逐渐降低至 200 cm，同时色调映射中的 Shadow Tint 向蓝绿色偏移，整个过程在 3 米内完成，无任何代码触发。

**室外到水下切换**：水面以下放置一个高优先级（Priority = 5）的无界体积，仅勾选色彩校正（Color Grading）和散射雾参数。当摄像机低于水面碰撞体时，该体积的 Blend Weight 直接设为 1（硬切），叠加蓝绿色滤镜与能见度缩减效果。

**动态体积控制**：在 UE5 的蓝图中，可以通过 `SetBlendWeight` 节点在运行时修改体积的混合权重，结合时间轴实现随剧情触发的后处理渐变。例如角色服药后，程序将特定体积的 BlendWeight 在 2 秒内从 0 插值到 1，触发色差（Chromatic Aberration）和暗角（Vignette）强化效果。

## 常见误区

**误区一：认为是玩家位置而非摄像机位置触发体积**。后处理体积的空间判定始终以摄像机的世界坐标为准，而非角色胶囊体的位置。在第三人称视角中，摄像机可能在角色身后数米，当角色跨入体积时摄像机尚未进入，效果不会立即激活。开发者若以角色位置为参考设计体积边界，往往会发现过渡时机比预期晚。

**误区二：Blend Radius 越大越平滑**。Blend Radius 决定的是混合发生的空间距离，但如果摄像机移动速度极快（如载具高速行驶），同样的 Blend Radius 实际产生的时间过渡可能仅有几帧，视觉上仍显突兀。平滑程度同时受到移动速度的影响，单纯增大 Blend Radius 无法解决快速穿越时的闪烁问题，此时需要配合蓝图插值或材质参数过渡。

**误区三：高优先级体积会完全覆盖低优先级体积的所有参数**。实际上，高优先级体积只影响其已勾选激活的参数。若高优先级体积没有勾选"泛光强度"，则低优先级体积的泛光参数仍然有效。混合是逐参数进行的，而非体积整体的替换。

## 知识关联

学习后处理体积需要先理解后处理概述中的各类效果参数（如色调映射、景深、曝光），因为体积本身只是这些参数的空间容器，对参数含义不熟悉则无法有效配置体积内容。

在掌握后处理体积的优先级与混合机制之后，下一步可以学习**后处理栈设计**，它研究如何在复杂场景中规划多个后处理体积的层次结构，包括全局基础层、区域覆盖层与特效触发层的分工，以及如何避免大量体积重叠导致的参数冲突与性能问题。后处理体积的 Priority 字段是后处理栈设计中分层架构思想的直接实现基础。
