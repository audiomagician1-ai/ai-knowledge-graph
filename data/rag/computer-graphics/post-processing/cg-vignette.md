# 暗角

## 概述

暗角（Vignette）是指图像边缘和角落相对于中心区域亮度降低的视觉效果。这一现象最初源于真实摄影中的光学物理缺陷：当光线穿过镜头时，斜入射的边缘光束强度弱于轴线中心光束，导致传感器或胶片四周曝光不足。这种由镜头口径和光圈共同决定的自然衰减，在摄影史早期被视为需要消除的像差，但随着艺术摄影和电影摄影的发展，它逐渐演变为一种被主动运用的视觉语言。

在图形学后处理管线中，暗角作为一种参数化效果被有意引入渲染结果。游戏引擎（如 Unity 和 Unreal Engine）均在其后处理堆栈中提供独立的暗角模块，允许美术人员精确控制衰减范围、强度和形状，使最终画面具有电影质感或营造特定的氛围压迫感。区别于光学暗角的被动产生，后处理暗角完全由数学函数驱动，具备可逆性和实时调节能力。

## 核心原理

### 光学暗角的物理成因

真实镜头产生的光学暗角遵循 **余弦四次方定律**（Cos⁴ Law）：当光线以角度 θ 斜入射时，像平面上的照度 E 满足：

> **E(θ) = E₀ · cos⁴(θ)**

其中 E₀ 是轴线中心的照度，θ 是入射光与光轴的夹角。这意味着当 θ = 30° 时，边缘照度仅为中心的约 56%，而 θ = 45° 时则降至约 25%。后处理中的光学暗角模拟，正是以这一物理规律为参考，用 UV 坐标到画面中心的距离替代角度 θ，重现这种非线性衰减特征。

### 艺术暗角的参数化数学模型

艺术暗角不受物理约束，通常使用以下参数化模型：

1. **Intensity（强度）**：控制暗角最深处的遮蔽程度，取值范围一般为 \[-1, 1\]，负值会产生边缘增亮的"反向暗角"效果。
2. **Smoothness / Feather（羽化）**：决定从亮区到暗区的过渡带宽度，数学上对应 smoothstep 函数的插值区间宽度。
3. **Roundness（圆度）**：控制暗角形状从椭圆向圆形的渐变，取值 -1 时为方形贴合屏幕边缘，取值 +1 时为完美圆形。

其基础计算片段可表示为：

```
float2 uv = screenUV - float2(0.5, 0.5);
float vignette = smoothstep(outerRadius, innerRadius, length(uv));
color.rgb *= lerp(1.0, vignette, intensity);
```

`outerRadius` 和 `innerRadius` 分别定义衰减的外圆半径与内圆半径，二者之差即为羽化带宽。

### 与屏幕宽高比的关系

当屏幕宽高比不为 1:1 时（如 16:9），直接使用 UV 坐标计算的 `length(uv)` 会产生椭圆形暗角，而非视觉上的圆形。为校正这一问题，需要将水平方向的 UV 偏移量乘以宽高比系数：

```
uv.x *= screenWidth / screenHeight;
```

这一校正在引擎内部通常自动处理，但在自定义 Shader 中需要手动实现，否则在超宽屏（如 21:9）显示器上会出现明显的椭圆畸变。

## 实际应用

**游戏氛围营造**：在 RPG 或恐怖游戏中，高强度暗角（Intensity ≈ 0.6 以上）配合暗部场景可以有效压缩玩家视野，增强心理上的局促感和紧张情绪。《黑魂》系列即大量运用强烈暗角烘托绝望气氛。

**玩家受伤反馈**：许多第一人称射击游戏在玩家受到伤害时，会动态将暗角强度从 0 快速插值到 0.8 左右，并染上红色色调，以不遮挡核心视野的方式传达生命值告急的信息。这是暗角参数化控制的典型实时动态应用。

**电影感后处理**：在追求电影质感的游戏（如《荒野大镖客：救赎2》）中，暗角与色彩分级（Color Grading）、胶片颗粒（Film Grain）组合使用，模拟胶片摄影的整体视觉特征，而非单独作为孤立效果使用。

**UI 与全屏过渡**：在菜单界面或截图模式中，柔和的暗角（Smoothness 接近 1.0）可以自然引导视线聚焦于画面中心的角色或主体，无需额外的对焦框或遮罩层。

## 常见误区

**误区一：认为暗角越强越有"电影感"**
电影摄影中的暗角通常非常微弱，Intensity 约在 0.1 至 0.3 之间，仅作为一种隐形的视线引导。过度强烈的暗角（Intensity > 0.7）反而会破坏画面均衡感，遮挡重要游戏信息。真正的电影感来自暗角与整体色调的协调，而非单一参数的最大化。

**误区二：混淆暗角与景深（Depth of Field）的视觉功能**
暗角作用于整个屏幕平面的边缘，是一个二维屏幕空间的效果，完全不感知场景的三维深度信息。景深则根据物体距摄像机的 Z 距离进行差异化模糊处理。二者均能引导视线，但暗角仅能处理画面的"位置边缘"，无法区分远近景物。将二者混为同一类效果会导致参数调整方向错误。

**误区三：认为暗角无需校正宽高比**
在将暗角 Shader 从 1080p（16:9）移植到其他分辨率（如 4:3 的平板或 21:9 的超宽屏）时，若不对 UV 坐标进行宽高比补偿，暗角会呈现明显的椭圆变形。这一问题在引擎内置后处理中通常已被处理，但在自定义全屏 Shader Pass 中极易被忽略。

## 知识关联

学习暗角之前，需要理解**后处理概述**中介绍的屏幕空间 UV 坐标系统：后处理效果均在一个覆盖全屏的四边形（Fullscreen Quad）上以片段着色器运行，暗角的核心计算 `length(uv - 0.5)` 正是基于这一坐标体系。没有屏幕空间 UV 的概念，就无法理解为何暗角从"中心距离"出发而非从三维世界坐标计算。

暗角作为难度最低的后处理效果，其参数化思路（强度、范围、形状的分离控制）是理解更复杂后处理效果的入门模板。色彩分级中的"区域蒙版"、景深中的"焦点半径"，以及泛光（Bloom）中的"阈值控制"，都沿用了"用数学函数定义影响范围，再用强度参数混合"的相同设计逻辑。暗角是学习这一设计模式最直观的起点。
