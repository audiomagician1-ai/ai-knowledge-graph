# 色散

## 概述

色散（Chromatic Aberration，也称色差）是指光线通过光学系统时，不同波长的光因折射率差异而产生分离，导致同一物体的红、绿、蓝三个颜色通道在成像平面上出现位置偏移的现象。在真实相机镜头中，这种偏移由镜头的阿贝数（Abbe number）决定，阿贝数越低，色散越严重。图形学后处理中模拟的正是这一物理现象。

色散作为后处理效果的应用，源于20世纪胶片摄影时代廉价镜头的光学缺陷。当游戏和电影制作者希望赋予画面"真实感"或"电影感"时，开始将这种原本属于缺陷的效应作为艺术手段引入渲染管线。在现代游戏中，《控制》（Control，2019）和《赛博朋克2077》都将色散作为核心视觉风格的组成元素，前者用它强化超自然失真感，后者将其与噪点结合模拟劣质全息显示。

在后处理实现层面，色散之所以重要，是因为它仅需对已渲染的帧缓冲进行通道分离采样，计算成本极低，却能显著增强画面的摄影质感。不同于景深或运动模糊需要大量采样，色散效果通常只需3次纹理采样（分别对应R、G、B通道），在移动端等性能受限场景下也能以极低代价提升画面感知品质。

## 核心原理

### 通道偏移采样

色散效果的基础实现是对屏幕空间UV坐标进行通道差异化采样。以画面中心为原点，对每个像素计算其与中心的偏移向量，然后按不同缩放系数对三个颜色通道分别采样：

```
vec2 direction = uv - vec2(0.5, 0.5);
float r = texture(screenTex, uv + direction * offsetR).r;
float g = texture(screenTex, uv + direction * offsetG).g;
float b = texture(screenTex, uv + direction * offsetB).b;
```

其中 `offsetR`、`offsetG`、`offsetB` 通常满足 `offsetR > offsetG > offsetB`（对应红光折射率最小、蓝光折射率最大的真实物理规律）。一个典型的数值配置为 `offsetR = 0.009`、`offsetG = 0.006`、`offsetB = 0.003`，以0到1归一化UV空间计算。

### 径向强度衰减

真实镜头的色散并非均匀分布，而是在画面边缘最强、中心最弱（因为轴上光线色差最小）。通过引入径向强度函数，可以精确控制这一衰减特性：

```
float distFromCenter = length(uv - vec2(0.5));
float aberrationStrength = pow(distFromCenter, 2.0) * maxStrength;
```

指数参数直接控制边缘过渡的锐利程度：指数为1.0时色散线性扩散，指数为3.0时则产生仅在极端边角才出现的剧烈色散，模拟廉价广角镜头效果。部分实现使用 `smoothstep(0.2, 0.8, distFromCenter)` 替代幂函数，以便美术人员用两个断点参数更直观地控制过渡区间。

### 色散方向与变体类型

物理上存在两种主要色差类型，后处理需分别对待：
- **轴向色差（Axial CA）**：颜色通道在焦平面方向分离，表现为物体边缘出现彩色光晕，沿UV径向方向偏移
- **横向色差（Lateral CA）**：颜色通道在垂直于光轴方向分离，表现为画面四角的彩色条纹，偏移方向与径向方向一致

纯艺术驱动的实现有时会故意混合两种类型，或加入基于时间的正弦扰动 `sin(time * 0.5) * 0.002` 使色散产生轻微呼吸感，模拟摄像机自动对焦时的动态色差变化。

## 实际应用

**界面边缘强化**：在UI设计中，将色散偏移量设置为仅在屏幕边缘3%区域内激活，可以模拟CRT显示器的老式电视美感，这在复古风格游戏（如《死亡细胞》的TV模式）中被广泛使用。

**受击视觉反馈**：将色散强度参数绑定到伤害事件，在受击瞬间将 `maxStrength` 从0插值到0.02再衰减归零，衰减时间约0.3秒，可以在不依赖抖动屏幕的前提下传递受击冲击感，尤其适用于第一人称射击游戏。

**虚拟现实优化变体**：VR场景中，由于眼镜镜片本身存在真实色散，过度叠加后处理色差反而会产生双重失真。Unity XR工具链建议在VR模式下将色散强度降低至桌面版本的30%，或完全关闭，仅在非VR模式下激活完整效果。

**与镜头光晕联动**：在包含镜头光晕的帧中，光晕源的高亮像素经过色散处理后，红、蓝通道的偏移会使光晕尾迹产生彩虹状分裂，自然形成棱镜散射的视觉效果，无需额外的彩虹渲染通道。

## 常见误区

**误区一：色散偏移量越大效果越好**
许多初学者将RGB偏移量设置在0.02以上，认为更明显的色散意味着更强的视觉冲击力。实际上超过0.015的偏移在1080p分辨率下会导致颜色通道之间出现明显的黑边（因为边缘外侧采样到空白区域），且强色散会严重影响文字可读性。专业项目中，常见的运行时上限值为 `maxStrength = 0.008`，仅在特殊事件（如爆炸）期间临时提升至0.015。

**误区二：色散与颜色分级顺序无关**
色散必须在Tonemapping和颜色分级之前执行还是之后，会产生完全不同的结果。若在线性空间颜色分级前执行色散，三个通道的偏移发生在HDR范围内，高亮区域的色散会携带过曝信息产生鲜艳的色散光晕；若在Tonemapping后的LDR空间执行，色散仅在0到1的范围内操作，效果更柔和。两种顺序均有应用，但同一项目中必须保持一致，否则会导致不同场景视觉风格不统一。

**误区三：色散只适用于写实风格**
色散效果因常出现在写实类游戏中而被误认为是写实专属工具。实际上，将偏移方向从径向改为水平方向，并将偏移量设置为固定值而非径向衰减，可以模拟液晶显示屏的水平色散，适合赛博朋克风格；将三个通道的偏移改为任意方向的随机抖动，结合高强度参数（0.03以上），则产生数据损坏或信号干扰的故障艺术（Glitch Art）效果。

## 知识关联

色散与**镜头光晕**共享屏幕空间后处理的基础框架，两者均以帧缓冲纹理为输入、以全屏四边形着色器为载体。镜头光晕处理的是高亮度区域的光能扩散，而色散处理的是全局所有区域的UV偏移，两者的着色器可以合并为同一个后处理Pass，通过条件编译宏（`#pragma multi_compile`）控制开关，减少Pass数量。

从技术依赖角度，色散需要能够对屏幕缓冲进行随机UV采样的渲染目标（Render Target），因此依赖后处理栈提供的屏幕纹理访问接口。在Unity的URP管线中，色散通过`ScriptableRenderPass`实现，可在`RenderPassEvent.AfterRenderingPostProcessing`阶段插入，也可选择在`BeforeRenderingPostProcessing`阶段以便与其他效果正确合成。熟练掌握色散效果后，可自然过渡至像素化、扫描线等同样基于UV操作的故障视觉效果族群。
