# 色散

## 概述

色散（Chromatic Aberration，简称 CA）是指光学系统中不同波长的光因折射率不同而产生的分离现象。在图形学后处理管线中，色散效果模拟真实相机镜头的光学缺陷：红、绿、蓝三个颜色通道在成像平面上发生轻微的空间偏移，导致画面边缘出现彩色条纹或"幻影"效果。这种效果在物理世界中是镜头像差的体现，在屏幕角落和高对比度边缘处尤为明显。

色散作为后处理效果在游戏和电影中的广泛应用始于2010年代初，随着基于物理的渲染（PBR）管线普及而逐渐标准化。经典电影《疾速追杀》（John Wick，2014）和大量 AAA 游戏如《赛博朋克2077》均使用了强烈的色散效果，来强调赛博朋克视觉风格或摄像机受到冲击的瞬间。从技术角度看，色散属于屏幕空间（Screen Space）效果，不依赖场景几何信息，因此计算成本相对较低。

色散在艺术上的价值远超"真实感"本身：适度的色散能让画面具有电影质感，而夸张的色散则可以用于传达眩晕、梦境、数字故障等叙事状态。掌握色散实现与参数控制，能让美术和技术美术人员在不重新渲染场景的前提下，用极少的性能预算获得强烈的视觉风格变化。

## 核心原理

### 三通道独立采样偏移

色散的最基础实现方式是对渲染结果纹理的 R、G、B 三个颜色通道分别以不同的 UV 偏移量进行采样：

```
vec2 offset = uv - vec2(0.5); // 以屏幕中心为原点
float strength = length(offset); // 距中心越远，色散越强

vec2 rOffset = offset * (1.0 + aberrationStrength);
vec2 gOffset = offset;
vec2 bOffset = offset * (1.0 - aberrationStrength);

float r = texture(screenTex, rOffset + vec2(0.5)).r;
float g = texture(screenTex, gOffset + vec2(0.5)).g;
float b = texture(screenTex, bOffset + vec2(0.5)).b;
```

其中 `aberrationStrength` 通常取值范围为 0.001 到 0.02 之间。偏移量以中心为圆点向外辐射，模拟径向色散（Radial Aberration），这与真实镜头中枕形/桶形畸变伴生的色差行为一致。红通道向外扩张，蓝通道向内收缩（或反之），绿通道保持不变，因为人眼对绿色中间频段最敏感，绿通道偏移会破坏亮度感知。

### 非线性强度映射

真实镜头的色散并非线性分布——边缘区域比中心强度高出数倍。常用的非线性映射公式为：

```
float edgeFactor = pow(length(uv - 0.5) * 2.0, 2.2);
float dynamicStrength = aberrationStrength * edgeFactor;
```

指数 2.2 并非随意选取，它与 sRGB 的 gamma 曲线一致，使得色散强度在感知上呈均匀过渡。屏幕角落（UV 距中心约 0.707 个单位）的色散强度约为边缘中点的两倍，这符合实拍镜头的测量数据。部分实现会将 `edgeFactor` 替换为 `pow(d, 3.0)` 来获得更夸张的边角色散，常用于故障（Glitch）艺术风格。

### 动态色散与事件驱动

静态色散强度会让玩家习惯而失去效果。更高级的做法是将 `aberrationStrength` 与游戏事件绑定，实现动态色散：

- **爆炸/冲击**：在碰撞帧将强度从 0.02 瞬间跳升至 0.08，随后在 0.3 秒内通过指数衰减（`strength *= 0.85 per frame`）回归基准值。
- **慢动作**：在子弹时间期间持续施加 0.012 的恒定色散，配合动态模糊共同营造"时间扭曲"感。
- **屏幕受伤/污染**：仅在画面特定区域（如屏幕四个角落用遮罩图限定）叠加额外 0.015 的色散，模拟镜头沾染血迹或雨水。

## 实际应用

### Unity URP 中的自定义色散 Pass

在 Unity URP 的后处理框架下，色散通常以 `ScriptableRenderPass` 实现，在 `BlitWithMaterial` 步骤中将着色器应用到 `_CameraColorAttachmentA`。关键参数通过 `VolumeComponent` 暴露给美术：`intensity`（基础强度）、`radialExponent`（非线性指数）和 `chromaAngle`（可旋转色散方向，实现斜向色差）。

### 故障艺术（Glitch Art）中的极端色散

《控制》（Control，2019）等游戏在角色能力激活时将色散强度推高至 0.05，并同时将 R/G/B 三通道偏移角度设为非对称值（如 R 向右上方偏移，B 向左下方偏移），制造出明显的数字故障感。与普通径向色散不同，这种方向性色散不需要以屏幕中心为轴，可以自定义偏移向量，在视觉上更具冲击力。

### 2D 像素游戏中的色散模拟

在像素风格游戏中，色散需要特殊处理：由于目标分辨率极低（如 320×180），亚像素级的 UV 偏移无法在采样后获得可见效果。解决方案是将场景先渲染到高分辨率缓冲区（2×或4×目标分辨率），在此分辨率下应用色散，再降采样到目标像素分辨率，从而在保留像素风格的同时获得可见的色差效果。

## 常见误区

### 误区一：色散强度越高越好

许多初学者为了"有电影感"将 `aberrationStrength` 设到 0.05 以上并全程保持，结果画面文字和 UI 元素变得难以辨读。实际上，人眼对文字边缘的色差极度敏感——超过 0.015 的全屏恒定色散会在 30 分钟游玩后导致视觉疲劳。正确做法是将色散限制在非 UI 层的游戏画面上，并将基准强度保持在 0.005 以内，仅在特定事件时短暂提升。

### 误区二：色散与径向模糊是同一类效果

色散和径向模糊都以屏幕中心为原点产生向外的效果，但机制完全不同。径向模糊在单通道内进行方向性模糊采样，结果是亮度信息的扩散；而色散是三通道各自进行独立采样，保留每个通道的清晰度，结果是颜色信息的分离。两者叠加使用时需要注意顺序：色散应在径向模糊**之前**应用，否则色散条纹本身会被模糊掉，丢失色彩分离的视觉特征。

### 误区三：色散偏移应始终沿 X 和 Y 轴方向分离

部分教程将 R 通道向右偏移、B 通道向左偏移，形成纯水平方向的色散，这实际上是摄影中"横向色差"的特例，并不符合大多数镜头的真实表现。径向色散要求偏移方向始终指向/远离屏幕中心，这意味着屏幕上方的色散是垂直分离，屏幕角落的色散是对角线方向分离。使用固定轴偏移会让色散看起来像是前期合成错误，而非有机的光学现象。

## 知识关联

色散的先修概念是**镜头光晕（Lens Flare）**：两者都属于相机光学缺陷的模拟，镜头光晕对应镜片内部的光线多次反射，而色散对应折射率的波长依赖性。在技术实现层面，两者都在屏幕空间操作，且都与屏幕边缘距离相关——理解镜头光晕的屏幕空间遮罩计算方式有助于直接套用到色散的非线性强度映射中。

在后处理管线的执行顺序中，色散通常在 Bloom 和 Tone Mapping **之前**执行，因为色散扩散的应该是线性光照数据；若在 Tone Mapping 之后执行，色散条纹的亮色区域会因已经映射到 [0,1] 范围而失去高光分离效果。将色散放在 TAA（时间抗锯齿）之后执行则会导致闪烁（ghosting），因为TAA依赖帧间一致的像素位置，而色散改变了颜色通道的有效采样位置。
