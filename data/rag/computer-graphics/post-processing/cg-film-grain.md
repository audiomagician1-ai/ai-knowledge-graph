# 胶片颗粒

## 概述

胶片颗粒（Film Grain）是一种模拟传统银盐胶片化学感光颗粒特性的后处理效果。在物理胶片上，银卤化物晶体（卤化银颗粒）在曝光与显影过程中形成不均匀的随机纹理，这种纹理在暗部和中间调区域最为明显，在高光区域则相对稀薄。数字图形学中的胶片颗粒后处理，正是对这一物理现象的程序化模拟。

该技术在电影工业中的数字化转型期（约2000年代初）开始被广泛研究。当数字摄影机取代胶片机后，制作方发现数字画面过于"干净"，缺乏胶片固有的质感温度感，因此对颗粒效果的程序化重现产生了强烈需求。游戏引擎如Unreal Engine 4和Unity均在后处理栈中内置了胶片颗粒模块，用于赋予渲染画面更具电影感的视觉风格。

胶片颗粒在游戏和电影的渲染管线中不只是装饰性效果，它还能在视觉上打破时间帧之间的规律性，缓解因TAA（时间抗锯齿）不足而残留的闪烁，并在低动态范围输出中营造更丰富的亮度层次感。

## 核心原理

### 程序化噪声生成

数字胶片颗粒的核心是在屏幕空间每像素叠加一个随机亮度偏移量。最基础的实现使用均匀分布随机数，但更接近真实胶片特性的实现采用**高斯分布**噪声，因为真实卤化银颗粒的密度偏差近似服从正态分布。

叠加公式通常为：

```
Output = Color + Grain * Intensity * LumaWeight(Color)
```

其中 `Grain` 是 [-1, 1] 范围内的高斯随机值，`Intensity` 控制颗粒强度（典型范围0.1–0.5），`LumaWeight` 是一个基于像素亮度的权重函数，使颗粒在中灰区（亮度约0.4–0.6）最强、在纯黑与纯白处减弱，从而匹配胶片的物理曲线特性。

### 时间抖动（Temporal Dithering）

静态噪声在屏幕上会形成肉眼可见的固定斑点图案，与真实胶片颗粒每帧随机更换的特性相悖。因此，正确实现胶片颗粒必须引入**每帧独立的随机种子**，使噪声纹理在时间轴上逐帧变化。

常见做法是将帧计数器（frame index）混入噪声的种子值，例如：

```glsl
float seed = dot(fragCoord, vec2(12.9898, 78.233)) + float(frameIndex) * 1.1234;
float grain = fract(sin(seed) * 43758.5453) * 2.0 - 1.0;
```

这种 `sin + fract` 的伪随机函数计算成本极低，在片元着色器中每帧可在GPU上全屏高效运行。时间抖动使颗粒在视觉上呈现"活的"运动感，与胶片放映时颗粒随帧跳动的外观高度一致。

### 颗粒尺寸与空间频率

真实胶片颗粒的物理尺寸在约1–20微米之间，换算到标准35mm底片的画面坐标后，呈现为亚像素到数像素级别的随机斑点。数字模拟时，若在原始分辨率下逐像素添加白噪声，所得频率偏高，视觉感偏硬。更真实的做法是在**降采样分辨率**（如原始分辨率的1/2或1/3）下生成颗粒纹理，再双线性上采样叠加到原图。这一操作使颗粒的空间频率降低，每粒噪点在屏幕上占据2–4像素，视觉上更接近ISO 400–1600感光度下的中粒胶片特性。

## 实际应用

**游戏后处理栈中的实现**：在《最后生还者》和《赛博朋克2077》等AAA游戏中，胶片颗粒后处理作为最终色调映射之后的一道Pass叠加。颗粒强度通常提供给玩家一个0–100的滑动条调节，内部映射到Intensity参数约0.05–0.3的范围。由于该Pass在LDR（低动态范围）输出空间执行，颗粒值被直接加减在[0,1]归一化的颜色分量上，不会破坏HDR数据。

**电影数字中间片（DI）流程**：在DaVinci Resolve的Film Grain生成器中，颗粒被分为R/G/B三个独立通道分别施加，因为真实彩色负片的三层乳剂（黄、品红、青染料层）颗粒大小和分布各不相同，绿色通道的颗粒感通常最强，蓝色通道次之。

**噪声类型对比**：使用Value Noise替代纯白噪声可以产生更大、更团块状的颗粒，适合模拟高ISO胶片（如Kodak 5219 500T）；而使用高频白噪声则模拟低ISO精细颗粒胶片（如Fuji Velvia 50）。

## 常见误区

**误区一：颗粒应该在HDR颜色空间中叠加**  
许多初学者将颗粒Pass放在色调映射之前，导致高亮区域的颗粒因Tonemapper曲线的非线性压缩而在最终画面中几乎消失不见，或在亮部产生明显的色彩偏移。正确做法是将胶片颗粒置于**Tonemapper之后、Gamma校正之前**（如果在线性空间输出）或直接在最终sRGB输出上叠加。

**误区二：颗粒强度应对所有颜色通道一致**  
将相同的灰度噪声值同时施加到R、G、B三个通道，只会产生亮度颗粒（luma grain），而真实胶片中还存在明显的色度颗粒（chroma grain）——即颗粒使色相发生随机偏移。若要模拟彩色胶片，应对每个通道独立生成噪声，或在色度通道（Cb/Cr）中单独叠加幅度较小的颗粒。

**误区三：静止画面可以不用时间抖动**  
即使摄像机和场景完全静止，若颗粒不逐帧更新种子，在屏幕上会呈现固定的摩尔纹状斑点，与胶片停格（freeze frame）时颗粒仍在跳动的实际行为不符，且容易与屏幕上的其他噪声效果（如SSAO的Dithering）发生规律性叠加干涉。

## 知识关联

学习胶片颗粒需要了解**后处理概述**中的渲染Pass链结构，因为胶片颗粒必须被正确地插入到色调映射和伽马校正的特定位置，插入顺序不同会导致截然不同的视觉结果。

胶片颗粒使用的逐像素随机数生成，与**抖动（Dithering）**技术共享基础数学工具——两者都依赖低差异序列或伪随机函数在空间或时间上分散量化误差。若进一步研究，可以将本文中的时间抖动概念与**时间抗锯齿（TAA）**中的历史帧采样机制对照理解，TAA有时甚至直接复用胶片颗粒的帧序列噪声，作为其子像素抖动的随机源。
