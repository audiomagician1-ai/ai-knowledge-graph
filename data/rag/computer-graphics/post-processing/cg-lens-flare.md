# 镜头光晕

## 概述

镜头光晕（Lens Flare）是光线通过相机镜头时，因镜片内部多次反射与折射而产生的光学伪影现象。当强光源（如太阳、车灯）进入镜头视野时，光线在多片镜片之间来回弹射，形成一串沿光源到画面中心连线排列的光斑、光圈和光柱。现代3D渲染中的镜头光晕特效正是对这一物理现象的屏幕空间模拟（Screen-Space Lens Flare）。

该效果最早在胶片摄影中被视为缺陷，摄影师会使用遮光罩来抑制它。然而1977年乔治·卢卡斯在《星球大战》中有意保留了镜头光晕，使其从"失误"转变为"电影感"的标志性元素。游戏领域中，《光晕：士官长合集》（Halo）和《镜之边缘》（Mirror's Edge）均大量使用后处理镜头光晕来强化画面的真实感与沉浸感。

在实时渲染后处理管线中，镜头光晕的重要性在于它能以极低的计算开销（通常整帧消耗不超过0.3ms）显著提升画面的光学拟真度，让玩家产生"透过摄像机看世界"的心理感受。

## 核心原理

### 光晕元素的组成

一个完整的镜头光晕由若干标准元素构成：

- **光晕核心（Flare Core）**：位于光源处的中心亮斑，通常以高斯模糊的圆形或星形呈现；
- **鬼影光斑（Ghost）**：沿光源到屏幕中心的对称轴上均匀分布的圆形或多边形光斑，每个Ghost对应一次镜片内反射；
- **光晕光晕环（Halo）**：围绕光源的大圆环，由镜头前组镜片的衍射产生；
- **光芒（Streak / Starburst）**：由镜头光圈叶片的形状决定，6叶片光圈产生6条光芒，8叶片产生8条。

Ghost光斑在屏幕空间中的位置公式为：

```
Ghost_Position = Screen_Center + t × (Light_Screen_Pos - Screen_Center)
```

其中参数 `t` 的典型取值范围为 −2.0 到 3.0，不同的 `t` 值对应不同镜片层产生的鬼影，`t` 为负数时Ghost出现在光源的另一侧。

### 屏幕空间的实现流程

现代实时渲染中镜头光晕的典型实现步骤如下：

1. **亮度阈值提取**：对当前帧的HDR颜色缓冲执行阈值过滤（通常亮度阈值 > 1.0，即超过标准曝光值），提取高亮区域；
2. **下采样**：将提取结果缩放至1/4分辨率，减少后续处理开销；
3. **Ghost生成**：在降采样图上，按上述位置公式多次采样并叠加，每次采样赋予不同的色调偏移（模拟色散）和缩放比例；
4. **光晕环生成**：对图像执行径向模糊或特定半径的环形采样；
5. **Starburst卷积**：使用预先生成的星形卷积核（通过快速傅里叶变换FFT）对光源区域进行卷积；
6. **合成**：将所有元素叠加后通过Additive混合写回主帧缓冲。

### 遮挡处理

屏幕空间实现的一个关键问题是光晕不能正确感知场景遮挡。若光源被山体遮挡，光晕不应出现。常见解决方案是在深度缓冲中对光源位置进行遮挡查询（Occlusion Query），或使用GPU的遮挡查询（`glBeginQuery(GL_SAMPLES_PASSED, ...)`）异步检测，将查询结果用于控制光晕的整体强度，实现平滑的淡入淡出（过渡时间通常为0.1–0.3秒）。

## 实际应用

**游戏引擎内置实现**：Unity的HDRP（High Definition Render Pipeline）将镜头光晕拆分为"Lens Flare SRP"组件与"Screen Space Lens Flare"后处理效果两个独立系统，前者允许美术手动在光源对象上挂载并精细调整每个Ghost的颜色、尺寸，后者则自动检测帧缓冲中的高亮区域全局生成光晕。Unreal Engine 5的后处理体积（Post Process Volume）中的"Lens Flares"参数BokehSize控制光斑的散焦模糊半径，默认值为3.0。

**HDR显示适配**：在支持HDR输出的平台上，光晕的亮度上限需要随显示器的峰值亮度（Peak Nit）动态调整。若目标显示器峰值为1000 nit，光晕核心亮度可设定为200–400 nit以保持视觉自然；若为SDR显示器（约100 nit），则需在色调映射后叠加，避免过曝。

**影视级离线渲染**：在ACES（Academy Color Encoding System）工作流中，镜头光晕通常在场景线性空间（Scene-Linear）中叠加，而非在显示空间操作，以保证光晕颜色在色调映射后仍能正确还原。

## 常见误区

**误区一：光晕颜色应该是白色或黄色**。真实镜头光晕的颜色由镜片镀膜的折射特性决定，不同镜片对不同波长光线的反射率差异造成每个Ghost呈现不同颜色——Canon镜头的Ghost多偏绿色，Zeiss镜头则偏洋红。简单地使用统一白色光晕会显得假。同时，各Ghost之间应有轻微的色散偏移（波长差约15–30nm），这也是下一步学习色散效果的基础。

**误区二：光晕越亮越好看**。过强的镜头光晕会遮蔽场景信息，降低可读性。在游戏中，常见做法是将光晕最大不透明度限制在0.6以下，并根据摄像机与光源夹角（Angle of Incidence）进行衰减：夹角小于15°时光晕最强，超过60°时基本消失，符合物理现实中掠射角度下光晕减弱的规律。

**误区三：屏幕空间光晕完全等同于物理光晕**。屏幕空间方法依赖帧缓冲中已有的像素信息，因此当光源位于屏幕边缘时，Ghost可能因采样超出边界而产生截断（Clipping Artifact）。解决方法是在降采样阶段对图像边缘进行Vignette遮罩处理，或将采样坐标Clamp到边缘以外一段距离后强制衰减至零。

## 知识关联

学习镜头光晕需要先理解后处理管线的基本工作方式，特别是HDR缓冲、亮度阈值提取和Additive混合操作，这些操作在后处理概述阶段已有介绍，镜头光晕的Ghost生成与Starburst卷积都直接建立在这些操作之上。

掌握镜头光晕之后，自然进入的下一个主题是**色散**（Chromatic Aberration）。色散描述不同波长光线因折射率差异而产生的横向偏移，镜头光晕中Ghost的彩色边缘正是色散的具体表现。理解了Ghost颜色来自何处，就能更自然地理解色散后处理如何对整个画面的边缘施加RGB通道分离效果，两者共用同一套物理成因——镜片对不同波长光线的差异化响应。
