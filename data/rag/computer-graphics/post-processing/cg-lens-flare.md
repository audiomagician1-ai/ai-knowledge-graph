# 镜头光晕

## 概述

镜头光晕（Lens Flare）是由强光源（如太阳、车灯、爆炸光效）经过相机或眼睛的多片光学镜片时，发生多次内部反射与折射而产生的一系列可见光学伪影。其典型表现包括：沿光源到画面中心连线排列的多边形光斑、彩虹状晕圈（Halo）、以及辐射状光芒（Starburst）。这些图案并非场景中真实存在的光，而是镜头本身的物理特性所引发的成像副产品。

在物理摄影领域，镜头光晕曾长期被视为需要消除的缺陷，各大镜头厂商通过镀增透膜（AR Coating）来抑制它。然而1970年代起，电影摄影师开始有意制造光晕来营造真实感，J.J.艾布拉姆斯在《星际迷航2009》中大量使用真实光晕成为业界里程碑。在实时图形学中，镜头光晕作为一种后处理效果被独立模拟，其目的是让渲染画面拥有"像是通过真实镜头拍摄"的质感，与运动模糊、景深效果共同构成相机模拟后处理管线。

在游戏和影视渲染中，屏幕空间（Screen-Space）镜头光晕的意义在于：它不需要光线追踪即可在GPU上实时运行，完全在后处理阶段的全屏四边形（Fullscreen Quad）上完成计算，性能开销可控。虚幻引擎4默认启用了镜头光晕后处理，Unity的HDRP管线也内置了独立的Lens Flare组件，说明该效果已是现代实时渲染管线的标配模块。

## 核心原理

### 1. 屏幕空间光晕的生成流程

屏幕空间镜头光晕的起点是从HDR亮度缓冲中提取超过阈值的亮斑——这一步通常复用Bloom效果的亮度提取Pass（Brightness Threshold，典型值为1.0以上的线性光照强度）。提取到亮斑像素后，核心操作是对这些像素执行**反射翻转（Flip & Scale）**：将采样坐标沿屏幕中心（UV = 0.5, 0.5）取镜像，公式为：

```
flareUV = (0.5 - sampleUV) * scale + 0.5
```

其中 `scale` 控制光斑离光源的距离比例，不同 `scale` 值（如 0.5、0.8、1.2、1.5）生成的多个光斑叠加，就形成了沿"光源—画面中心"轴线分布的光斑序列，这与真实镜头中光线沿光轴多次反射的路径相吻合。

### 2. 鬼像（Ghost）的形状与颜色

每个单独的光斑被称为"Ghost"。Ghost的形状由镜头光圈的叶片数量决定：6叶片光圈产生六边形Ghost，8叶片产生八边形Ghost，圆形光圈产生圆形Ghost。在着色器中通过SDF（有向距离场）或预制纹理Atlas来绘制不同形状的Ghost。颜色上，由于色散效应，不同Ghost会被赋予不同的色调（从紫红到蓝绿的渐变），模拟真实镜头中短波长和长波长光线折射角不同导致的分色现象。

### 3. 光芒（Starburst）与晕圈（Halo）

星形光芒由光圈衍射产生，其射线数量等于光圈叶片数的2倍（6叶片产生12条光芒）。在后处理中，Starburst通常通过对亮点进行多方向的带状卷积滤波（Directional Blur）来模拟，每个方向对应一条光芒射线，角度间隔为360°/射线总数。晕圈（Halo/Lens Ring）是围绕光源的圆形光圈，半径与相机焦距相关，用高斯模糊后的径向遮罩叠加实现。

### 4. 遮挡处理与可见性

真实镜头光晕在光源被物体遮挡时应逐渐消失。屏幕空间实现中，通常对光源的屏幕坐标位置进行**遮挡查询（Occlusion Query）**或采样深度缓冲，计算光源可见性比例（0.0到1.0之间），再将该比例乘以整个光晕的最终亮度，实现平滑的淡入淡出。虚幻引擎将此参数暴露为 `LensFlareOcclusionMaskDarkness`，允许美术调节遮挡过渡的深度。

## 实际应用

**游戏引擎中的参数配置**：在Unity HDRP的Lens Flare（SRP）组件中，一个典型的太阳光晕资产包含3到6个Ghost元素、1个Starburst元素和1个Halo元素。Ghost的 `Position`（即上文的scale参数）分别设为 -0.4、0.6、1.2等，`Intensity` 依次递减，模拟越远的反射光斑越暗的物理规律。

**影视VFX中的用途**：在Nuke等合成软件中，光晕常在镜头跟踪完成后手动绘制在画面上，其位置通过2D追踪数据绑定到光源，Ghost的颜色偏移量参考了实际镜头测量数据（例如Cooke S4镜头的特定紫色Ghost签名）。

**性能预算**：在移动平台上，完整的镜头光晕后处理（含Ghost、Starburst、Halo三个Pass）通常需要0.5到1.5毫秒的GPU时间（1080p分辨率，Mali-G77测试基准），这要求美术将Ghost数量控制在4个以内，并禁用高分辨率Starburst卷积。

## 常见误区

**误区一：镜头光晕等同于Bloom效果**。Bloom是画面中所有亮区向外扩散的柔和发光，它模拟的是传感器/胶片的曝光溢出。镜头光晕的Ghost是离散的、沿特定轴线排列的几何形状，两者使用不同的着色器Pass，即便关闭Bloom，镜头光晕依然可以独立渲染。将两者混为一谈会导致错误地用Bloom参数去调整Ghost的位置或形状，结果无效。

**误区二：光晕的光斑数量越多越逼真**。真实镜头的Ghost数量取决于镜片组的数量，一个6片5组的镜头理论上只会产生有限数量的主要Ghost。使用十几个Ghost堆叠出"炫目"效果，往往超出了物理合理范围，反而使画面显得廉价。专业影视后期通常将主要Ghost控制在3到5个，辅以精确的颜色偏移来追求真实感而非数量。

**误区三：屏幕空间光晕可以正确处理画面边缘的光源**。当光源位于屏幕外时，屏幕空间方法无法采样到它，Ghost序列会突然消失，出现明显的切断感（Pop-in）。解决方案是引入世界空间光源位置的投影计算，将屏幕坐标范围扩展到[-0.2, 1.2]之类的边界外区域，这是纯屏幕空间算法的固有局限，必须通过混合世界空间信息来弥补。

## 知识关联

**前置概念——后处理概述**：镜头光晕依赖后处理框架中的HDR颜色缓冲作为输入。理解后处理Pass的执行顺序（镜头光晕通常在Bloom之后、色调映射之前执行）是正确调参的前提，否则在LDR空间中提取亮斑会因亮度已被截断而失去对超亮区域的区分能力。

**后续概念——色散（Chromatic Aberration）**：镜头光晕中Ghost的彩色化正是色散原理的局部体现——不同波长光线折射率不同（折射率与波长的关系由Cauchy公式 $n(\lambda) = A + B/\lambda^2$ 描述）。色散作为独立后处理效果，会在整个画面边缘产生RGB通道的空间偏移，而光晕中的色散仅作用于Ghost本身的颜色分布，两者在实现粒度上有所不同，但共享同一物理根源。
