# GPU内存层级

## 概述

GPU内存层级是指从寄存器到VRAM（显存）之间按访问速度和容量分层排列的存储体系，共分为五个主要层级：寄存器（Register）、共享内存（Shared Memory）、L1缓存、L2缓存和VRAM。每一层级的访问延迟相差数十倍甚至数百倍，决定了GPU能否充分发挥其数千个并行计算核心的性能。

这一层级体系随GPU架构演进而逐渐明确。NVIDIA在2006年发布的Tesla架构中首次引入了程序员可控的共享内存，使开发者能够手动管理片上存储；2012年的Kepler架构进一步将L1缓存和共享内存统一为可配置的片上存储池（共48KB或16KB两档分配方案）。理解这一层级体系对于优化CUDA或OpenCL程序的显存带宽利用率至关重要，错误的内存访问模式会导致实际带宽利用率不足理论峰值的10%。

## 核心原理

### 寄存器：最快但数量有限

寄存器是GPU内存层级中访问延迟最低的存储单元，通常仅需1个时钟周期即可完成读写。每个CUDA线程独占自己的寄存器空间，在NVIDIA Ampere架构（A100）中，每个流式多处理器（SM）最多拥有65536个32位寄存器。若一个线程使用过多寄存器（通常超过255个），编译器会将溢出数据存入本地内存（Local Memory），而本地内存实际上映射在VRAM中，延迟骤升至数百个周期——这一现象称为"寄存器溢出"（Register Spilling）。

### 共享内存：线程块内的协作缓存

共享内存位于SM片上，一个线程块（Thread Block）内的所有线程共享同一块共享内存，访问延迟约为32个时钟周期。在Ampere架构中，每个SM的L1缓存与共享内存合并为最大192KB的统一存储，其中共享内存最多可配置为164KB。共享内存被划分为32个"bank"，每个bank宽度为4字节，若多个线程同时访问同一bank的不同地址，将发生"bank冲突"，导致访问被串行化，带宽下降为原来的1/32。矩阵转置操作中添加1列填充（padding）是消除bank冲突的经典技巧。

### L2缓存：全局共享的中间层

L2缓存由GPU上所有SM共享，容量远大于L1。A100 GPU的L2缓存容量为40MB，而H100更扩展至50MB。L2的访问延迟约为200个时钟周期，带宽在A100上约为每秒5TB（片上）。L2缓存对程序员透明，由硬件自动管理，主要用于缓解全局内存（VRAM）的访问压力，尤其在随机访问模式下效果显著。

### VRAM：容量最大但延迟最高

VRAM（Video RAM）即显存，是GPU的主存储器，采用GDDR6或HBM2e等高带宽显存技术。A100 SXM4版本配备40GB HBM2e显存，理论带宽为1555 GB/s，远超CPU的DDR4内存带宽（约50 GB/s）。然而其访问延迟高达600-800个时钟周期，因此GPU的大量线程并发（隐藏延迟）是弥补这一缺陷的核心机制——当一组线程等待VRAM数据时，调度器立即切换到其他已就绪的线程执行。

### 各层级带宽对比

| 层级 | 典型延迟 | 典型容量（每SM） |
|------|----------|-----------------|
| 寄存器 | ~1周期 | 256KB |
| 共享内存 | ~32周期 | 最大164KB |
| L1缓存 | ~28周期 | 共享至多192KB |
| L2缓存 | ~200周期 | 全局40-50MB |
| VRAM | ~600周期 | 数十GB |

## 实际应用

**矩阵乘法优化（Tiling技术）**：在计算 C = A × B 时，若直接从VRAM读取数据，每次乘加运算需要访问显存，效率极低。通过将矩阵分块（Tile），先将16×16或32×32的子块加载到共享内存，再在线程块内完成局部计算，可将VRAM访问次数减少至原来的1/N（N为分块尺寸），显著提升计算访存比（Arithmetic Intensity）。cuBLAS库的GEMM实现正是依赖这一技术将性能逼近硬件峰值。

**纹理采样中的L1缓存**：GPU的纹理单元（Texture Unit）有专用的L1纹理缓存（Texture Cache），在图形渲染管线中，fragment shader对纹理的双线性或三线性采样具有空间局部性，相邻像素访问相近的纹理坐标，L1纹理缓存命中率通常可达80%以上，大幅降低VRAM带宽消耗。

**常量内存（Constant Memory）**：CUDA提供64KB的常量内存，映射于VRAM但有专用L1缓存。当一个Warp（32个线程）的所有线程访问同一常量内存地址时，仅需一次读取即可广播给全部线程，延迟与共享内存相当，适合存储不变的变换矩阵等数据。

## 常见误区

**误区一：共享内存越大越好**。部分开发者误以为尽量多地使用共享内存总是有益的。实际上，每个SM的共享内存是全体驻留线程块共同竞争的资源。若单个线程块占用48KB共享内存，而SM共有96KB，则同时只能驻留2个线程块；减少每块共享内存用量至24KB，则可驻留4个线程块，提高SM占用率（Occupancy），反而更有利于延迟隐藏。

**误区二：L1缓存命中等同于共享内存速度**。两者虽在物理上共享同一SRAM，但访问机制不同：共享内存由程序员显式控制，地址访问模式可预测；L1缓存由硬件自动管理，存在缓存行替换开销，且L1不保证数据在线程块内的一致性视图，不可用于线程间通信。

**误区三：VRAM带宽等于实际可用带宽**。厂商标注的1555 GB/s是理论峰值，仅在完全合并访问（Coalesced Access，即一个Warp的连续线程访问连续内存地址）时才能接近。若访问步幅为2（stride-2 access），实际带宽约降至峰值的50%；完全随机访问则可能低至峰值的5%以下。

## 知识关联

GPU内存层级建立在GPU硬件管线的基础之上：理解SM（流式多处理器）的结构是理解寄存器和共享内存归属关系的前提，Warp调度机制解释了为何高VRAM延迟可以通过多线程并发被隐藏。掌握各层级的容量与带宽数据，是后续学习CUDA内存优化（如合并访存、共享内存分块、寄存器分配策略）以及分析性能瓶颈（使用Nsight Compute等性能分析工具时解读"L2 Hit Rate"和"Global Memory Throughput"指标）的直接基础。在图形渲染领域，纹理缓存和帧缓冲区的内存管理策略同样依赖对这一层级体系的深入认知。
