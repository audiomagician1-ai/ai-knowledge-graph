# GPU内存层级

## 概述

GPU内存层级是指GPU芯片上从最快最小到最慢最大的多层存储结构，具体包括寄存器（Register）、共享内存/L1缓存（Shared Memory / L1 Cache）、L2缓存和显存（VRAM）四个主要层次。每一层的访问延迟相差数十倍甚至数百倍，因此理解这一层级对于编写高性能着色器和CUDA/OpenCL计算程序至关重要。

这一分层概念随GPU从固定功能管线向可编程架构演进而逐渐成型。NVIDIA在2006年推出的G80架构（GeForce 8800系列）首次引入了统一着色器模型，并明确区分了每个SM（流多处理器）内的共享内存与全局显存，奠定了现代GPU内存层级的基本格局。此后每代架构不断调整各层容量与带宽比例，例如Ampere架构将每个SM的L1/共享内存总容量扩展至128 KB。

这一层级结构之所以存在，根本原因是物理定律的制约：离计算核心越近的存储单元，面积越小、速度越快，但制造成本也越高。GPU拥有数千个并行执行单元，若每个核心都直接访问片外VRAM，总线宽度根本无法支撑如此高并发的带宽需求。通过寄存器→L1→L2→VRAM的四级缓冲，GPU才能将数千核心的计算效率最大化。

## 核心原理

### 寄存器（Register）

寄存器是GPU内存层级中速度最快、延迟最低的存储单元，访问延迟通常为1个时钟周期，无需任何缓存查找。在NVIDIA Ampere架构中，每个SM拥有65,536个32位寄存器，每个线程在编译时静态分配寄存器数量。当一个着色器或CUDA核函数使用过多寄存器（例如超过每线程32个）时，会触发"寄存器溢出"（Register Spilling），多余数据被推送到本地内存（Local Memory），而本地内存实际上映射在VRAM上，延迟骤升至数百周期。GLSL/HLSL编译器会自动管理寄存器分配，但过度复杂的着色器逻辑会导致溢出。

### 共享内存与L1缓存

共享内存（Shared Memory）和L1缓存在物理上共享同一块SRAM，但用途不同：共享内存由程序员显式控制（在CUDA中使用`__shared__`关键字声明），而L1缓存由硬件自动管理。在Ampere架构中，这两部分合计128 KB，可以通过`cudaFuncSetAttribute`配置其分配比例，例如将96 KB分配给共享内存、32 KB分配给L1。共享内存的访问延迟约为20–30个时钟周期，带宽约为128 bytes/cycle（每个SM）。共享内存按"Bank"组织，Ampere有32个Bank，每个Bank宽度为32位；若多个线程同时访问同一Bank的不同地址，会产生Bank冲突，导致串行化访问，性能大幅下降。

### L2缓存

L2缓存是所有SM共享的片上缓存，是VRAM与SM之间的全局缓冲层。RTX 3090（Ampere GA102）的L2缓存达到6 MB，而RTX 4090（Ada Lovelace AD102）进一步扩展至96 MB——这一巨大提升使许多中等规模的纹理数据集可以完全驻留在L2中，显著减少对VRAM的访问次数。L2缓存的访问延迟约为200个时钟周期，带宽在GA102上约为5.6 TB/s（内部总线）。

### 显存（VRAM）

VRAM是GPU的片外主存，使用高带宽内存技术（如GDDR6X或HBM2e）来弥补其相对高的延迟（约500–700个时钟周期）。RTX 3090搭载的GDDR6X显存带宽为936 GB/s，内存总线宽度为384位。VRAM存储纹理、帧缓冲、顶点缓冲、全局变量等所有大型数据。GPU通过内存控制器以128字节为单位的"内存事务"访问VRAM，因此在着色器中实现连续内存访问（Coalesced Access）可将多个线程的请求合并为单次事务，效率提升可达16倍。

## 实际应用

**纹理采样**：GPU在采样纹理时，会先查找L1纹理缓存（约16–32 KB/SM），命中则仅需约20周期；未命中则查L2，最终可能触发VRAM读取。Mipmap的意义之一正是提升L1命中率——远处物体使用低分辨率Mip层级，占用更少缓存空间。

**波函数折叠/过程生成**：在实时体素场景生成中，大量随机访问全局内存会导致L2命中率极低。将查找表（LUT）复制到CUDA共享内存中，可将延迟从~500周期压缩至~25周期，吞吐量提升约10倍。

**顶点着色器中的寄存器压力**：编写复杂骨骼动画着色器时，若矩阵运算展开过多，每线程寄存器占用超过64个，则GPU的活跃Warp数（Occupancy）会从理论最大值下降，隐藏延迟的能力减弱，导致实际吞吐量低于硬件峰值的50%。

## 常见误区

**误区一：共享内存等同于缓存**。共享内存需要程序员手动分配和管理数据搬运（在CUDA中使用`__syncthreads()`同步），而L1/L2缓存完全由硬件透明管理。两者虽物理相邻，但使用方式根本不同。依赖硬件缓存自动优化共享数据，往往不如手动搬入共享内存的效果好，因为缓存替换策略并不知道哪些数据会被多次重用。

**误区二：更大的VRAM容量直接等于更高性能**。VRAM容量决定能加载的资源总量，但渲染性能的瓶颈通常是VRAM带宽和L2命中率，而非容量本身。一张12 GB GDDR6X显卡（带宽936 GB/s）在带宽敏感场景下会快于一张24 GB GDDR6显卡（带宽672 GB/s）。

**误区三：减少VRAM访问只需提高L1/L2缓存命中率**。内存访问的合并（Coalescing）与缓存命中率是两个独立的优化维度。即使数据已在L2中，若线程束（Warp）内32个线程访问的地址分散在不同的128字节块，仍会产生32次独立事务而非1次合并事务，带宽利用率仅为1/32。

## 知识关联

理解GPU内存层级需要以GPU硬件管线为前提，特别是SM（流多处理器）内部的执行单元与Warp调度机制——因为寄存器分配数量直接影响每个SM能同时激活的Warp数，而这一参数（Occupancy）是评估内存层级配置是否合理的核心指标。

GPU内存层级的知识直接支撑着色器性能优化实践：诊断一个着色器是"计算瓶颈"还是"内存带宽瓶颈"，需要对照各层级的理论带宽数字（如L1的~128 bytes/cycle/SM vs VRAM的~936 GB/s全局）与GPU性能计数器（如NSight Graphics中的`l1tex__t_bytes`和`dram__bytes`指标）进行分析。纹理压缩格式（BC7、ASTC）的价值也直接体现在减少VRAM带宽消耗和提升L2缓存容纳的有效纹素数量上。
