# 光栅化概述

## 概述

光栅化（Rasterization）是将几何图元——即点、线段、三角形——转换为屏幕上离散像素集合的过程。这一过程的名称来源于拉丁语"rastrum"（耙子），形象地描述了将连续图形"耙成"离散像素网格的操作。在实时图形渲染领域，光栅化是目前主流GPU硬件所采用的核心渲染范式，与光线追踪相对立。

光栅化渲染管线的工业化实现可追溯至1989年SGI（Silicon Graphics Inc.）推出的IRIS GL图形库，以及随后1992年基于此演化而来的OpenGL 1.0标准。在此之前，Gouraud着色（1971年由Henri Gouraud提出）和Phong着色（1973年由Bui Tuong Phong提出）已经为光栅化的逐顶点与逐像素计算奠定了理论基础。实时光栅化之所以在过去30年统治游戏和交互图形领域，根本原因在于其时间复杂度与场景中三角形数量呈线性关系O(n)，而非与场景整体空间复杂度相关，使GPU能以每秒数十亿三角形的速度处理画面。

## 核心原理

### 几何图元到像素的映射流程

光栅化渲染管线按顺序经历以下固定阶段：顶点处理 → 图元组装 → 光栅化 → 片段处理 → 逐片段操作。其中"光栅化"阶段本身负责两项具体任务：一是**三角形遍历**（Triangle Traversal），确定哪些像素中心落在三角形内部；二是**插值计算**，将顶点属性（如颜色、纹理坐标、深度值）用重心坐标在像素间插值。整个管线的设计使得每个三角形的处理互相独立，这正是GPU大规模并行架构能够加速它的根本原因。

### 重心坐标插值

判断像素点P是否在三角形ABC内部，以及计算其插值权重，依赖重心坐标（Barycentric Coordinates）。对于像素中心点P，其重心坐标（α, β, γ）满足：

**P = αA + βB + γC**，且 **α + β + γ = 1**

其中 α = Area(PBC)/Area(ABC)，β = Area(APC)/Area(ABC)，γ = Area(ABP)/Area(ABC)。当且仅当 α ≥ 0、β ≥ 0、γ ≥ 0 时，P在三角形内部。任意顶点属性值（如纹理坐标u）在像素处的插值结果为 u_P = αu_A + βu_B + γu_C。需注意，在透视投影下，必须先对顶点属性进行透视除法后再插值，即**透视正确插值**，否则纹理会出现扭曲。

### 深度缓冲（Z-Buffer）算法

光栅化管线通过Z-Buffer解决可见性问题，这一算法由Edwin Catmull于1974年提出。每个像素维护一个深度值，初始化为最大值（通常为1.0，对应远裁剪平面）。当一个三角形的片段到达某像素时，将其深度值与缓冲区中的现有值比较：若新片段更近（深度值更小），则更新颜色缓冲和深度缓冲；否则丢弃该片段。Z-Buffer的空间复杂度为O(w×h)，即与屏幕分辨率成正比，与场景中物体数量无关，这使它成为比画家算法更适合实时渲染的可见性方案。

### 三角形作为基本图元的原因

光栅化管线以三角形而非四边形或其他多边形为基本单位，原因有三：其一，三角形是唯一保证共面的多边形（3点确定一个平面）；其二，任意凸多边形可分解为三角形扇；其三，三角形的重心坐标系统在数学上具有唯一性，简化了硬件实现。现代GPU的光栅化单元（Rasterizer）在硬件层面专门针对三角形操作进行了优化，例如NVIDIA的Fermi架构每个SM配备专用的三角形设置单元（Triangle Setup Engine）。

## 实际应用

**实时游戏渲染**：2022年发布的《赛博朋克2077》使用光栅化管线渲染大部分画面，仅对反射和阴影部分可选地引入光线追踪，这种混合方案说明即使在光追硬件普及后，光栅化仍是主体渲染方式。在典型的1080p游戏帧（约60fps）中，GPU每帧需要光栅化数以百万计的三角形，这一数量级是光线追踪单独无法在同等硬件上实时完成的。

**离线渲染的预计算阶段**：即使在以光线追踪为主的离线渲染器中，光栅化也常用于生成Shadow Map（阴影贴图）——从光源视角对场景执行一次光栅化pass，将深度信息写入纹理，供后续光线追踪阶段查询遮挡关系，这将阴影计算成本降低约60%～80%。

**移动端图形**：智能手机GPU（如Apple的A系列芯片和ARM Mali系列）普遍采用**基于瓦片的延迟光栅化**（Tile-Based Deferred Rendering, TBDR）架构，将屏幕分割为16×16像素的瓦片分块处理，以减少对主存储器的带宽压力，这是桌面GPU即时模式光栅化（Immediate Mode Rendering）在移动场景下的重要变体。

## 常见误区

**误区一：光栅化只是"画三角形"**。光栅化是一套完整的渲染管线范式，包含从3D空间变换到最终像素写入的全部逻辑——MVP矩阵变换、裁剪、透视除法、视口变换、插值、深度测试、模板测试、混合等环节缺一不可。仅将"三角形填充"等同于光栅化，会错误地认为深度冲突（Z-fighting）和透视纹理扭曲是独立问题，而实际上它们都是光栅化管线特定阶段的直接产物。

**误区二：光栅化比光线追踪"精度更低"**。光栅化与光线追踪是两种不同的**可见性求解策略**，而非精度高低之分。光栅化对每个三角形询问"它覆盖了哪些像素"，光线追踪对每个像素询问"哪条光线首先与哪个三角形相交"——这两种遍历顺序的切换导致了全局光照效果（如多次反射、折射）在光栅化中难以高效实现，但并不代表光栅化本身的颜色计算"不准确"。

**误区三：现代渲染已经放弃光栅化**。尽管NVIDIA于2018年推出了支持硬件光线追踪的RTX 20系列GPU，但截至2024年，全球绝大多数实时3D应用（游戏、CAD、AR/VR）仍以光栅化为主干渲染路径，光线追踪仅作为特定效果的补充。光栅化管线在GPU硬件成本、驱动成熟度和能耗效率上的优势，决定了它不会在短期内被完全取代。

## 知识关联

学习光栅化概述后，下一步应深入**图形管线阶段**——即对顶点着色器、几何着色器、片段着色器等可编程阶段的具体分工进行学习，理解哪些阶段是可编程的（顶点/片段着色器），哪些是固定功能的（三角形遍历、深度测试）。在掌握了完整光栅化管线之后，可进一步研究**混合渲染管线**（Hybrid Rendering Pipeline）——即光栅化与光线追踪协同工作的架构，这正是当前AAA游戏引擎（如Unreal Engine 5的Lumen全局光照）所采用的技术路线。光栅化的Z-Buffer思想也是理解**延迟着色**（Deferred Shading）的前提，后者将光照计算与几何信息的收集分离到两个独立pass中，是现代游戏引擎处理大量动态光源的标准方案。
