# 扫描线转换

## 概述

扫描线转换（Scanline Conversion）是光栅化阶段的核心算法，其任务是将几何图元（通常是三角形）转化为屏幕上的像素集合。算法的基本思路是：对多边形所覆盖的每一条水平扫描线（y = 常数），计算出该扫描线与多边形左右边界的交点，然后填充两交点之间所有整数x坐标对应的像素。这一"逐行扫描、左右填充"的策略使得算法具有良好的内存访问局部性，与显示设备的行扫描刷新方式天然契合。

扫描线填充算法的理论基础可追溯至1967年Ivan Sutherland等人的早期光栅图形研究，1970年代随着帧缓冲显示器的普及而获得广泛应用。相比逐像素的点测试（判断像素中心是否在三角形内），扫描线算法在CPU端的传统实现中效率更高，因为它避免了对三角形包围盒内大量非覆盖像素的冗余计算。现代GPU虽采用并行的边函数（edge function）测试替代串行扫描，但理解扫描线算法仍是掌握光栅化工作原理的关键路径。

## 核心原理

### 活性边表（AET）与边表（ET）数据结构

扫描线算法的高效实现依赖两个核心数据结构。**边表（Edge Table，ET）**按多边形各边的最小y值分桶存储，每条边的条目包含四个字段：`y_max`（边的最大y坐标）、`x_min`（该边在y_min处的x值）、`Δx`（增量值，等于边的水平跨度除以垂直跨度，即 `Δx = (x2 - x1) / (y2 - y1)`）。**活性边表（Active Edge Table，AET）**则在当前扫描线处维护所有与该扫描线相交的边，按x坐标升序排列。

当扫描线从 `y_min` 向 `y_max` 逐行推进时，算法执行三步操作：①将ET中y值等于当前扫描线的边插入AET；②从AET中移除 `y_max` 等于当前y的边（边已离开视野）；③对AET中的每对相邻边之间的像素区间执行填充，然后将所有活性边的x值更新为 `x += Δx`。这一增量更新策略使得每行扫描的计算量仅为 O(活性边数)，而非重新计算每条交点。

### 交点的奇偶规则与非零绕数规则

对于自相交多边形，填充哪些区间需要规则约定。**奇偶规则（Even-Odd Rule）**规定：从任意点向外引射线，若与多边形边界的交叉次数为奇数则填充，偶数则不填充。**非零绕数规则（Non-Zero Winding Rule）**则额外考虑边的方向：向左穿越边计+1，向右穿越边计-1，结果非零则填充。SVG和PostScript均使用非零绕数规则作为默认填充规则，而CSS的 `fill-rule` 属性允许在两者之间切换。

### 斜率增量与水平边的特殊处理

`Δx = 1/slope` 是扫描线算法的核心递推公式，其中 `slope = (y2 - y1)/(x2 - x1)`。水平边（`y1 == y2`，斜率为零，Δx无穷大）不进入边表，因为水平边不会与水平扫描线产生有意义的单点交叉，其贡献已由相邻两条非水平边的端点隐含处理。对于顶点恰好落在扫描线上的情况，惯例规定：若顶点是两条边的局部最低点（valley顶点），则计入两次交点；若是局部最高点（peak顶点），则不计入，从而保证填充的正确性和无重叠性。

## 实际应用

**软件光栅化渲染器**：早期游戏引擎如Quake（1996年）在CPU上实现扫描线光栅化，按多边形的 `y_min` 排序后逐行扫描，同时在x方向上插值纹理坐标（u/v）和深度值（z），实现透视校正纹理映射。

**字体渲染**：FreeType库在栅格化TrueType/OpenType轮廓时使用扫描线填充，处理二次或三次贝塞尔曲线构成的字形边界。FreeType的扫描线转换器（raster模块）先将曲线细分为短直线段，再用AET填充，最终产生8位灰度的抗锯齿覆盖值。

**多边形裁剪与填充工具**：PostScript解释器和PDF渲染引擎在绘制填充路径时直接使用扫描线算法，对复杂路径（含镂空、子路径）应用非零绕数规则决定哪些像素点亮。

## 常见误区

**误区一：认为每次扫描都需要重新计算边与扫描线的交点**。事实上，扫描线算法的精髓在于增量更新：当y增加1时，新的x交点仅为 `x_old + Δx`，不需要重新做直线方程求解。忽略这一点会将算法复杂度从 O(n + 总像素数) 提升至 O(n × 行数)，在处理高分辨率图像时产生数倍性能差距。

**误区二：水平边需要像其他边一样插入边表**。水平边的斜率倒数 Δx 为无穷大，若强行插入AET会导致x坐标更新溢出或逻辑错误。正确做法是在构建ET阶段直接跳过所有水平边，其覆盖范围已由上下相邻扫描线的非水平边正确涵盖。

**误区三：混淆扫描线转换与Z-buffer的职责**。扫描线转换解决的是"哪些像素在多边形内"的问题，而Z-buffer解决的是"多个重叠多边形谁在前面"的问题。两者通常结合使用：扫描线填充生成候选像素后，再与Z-buffer中存储的深度值比较，决定是否更新颜色缓冲。

## 知识关联

**前置概念——三角形装配**：扫描线转换的输入是三角形装配阶段输出的屏幕空间三角形顶点（已完成视口变换，坐标为屏幕像素单位）。装配阶段确定了顶点的 `(x, y)` 屏幕坐标和深度 `z`，扫描线算法在此基础上沿x方向插值这些属性，生成每个片段（fragment）的插值数据。

**延伸方向——抗锯齿与MSAA**：标准扫描线转换产生硬边缘锯齿（aliasing）。理解扫描线填充后，可进一步学习多重采样抗锯齿（MSAA）如何在每个像素内部设置多个子采样点，以及超采样（SSAA）如何以更高分辨率运行扫描线再降采样，从而平滑多边形边缘。

**横向关联——边函数光栅化**：现代GPU采用边函数（edge equation）方法并行测试像素，其公式 `E(x,y) = (y2-y1)x - (x2-x1)y + (x2y1-x1y2)` 判断像素是否在三角形三条边的正侧。两种方法在数学上等价，但扫描线适合串行CPU执行，边函数适合GPU的SIMD并行架构。
