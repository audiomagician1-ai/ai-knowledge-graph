# 属性插值

## 概述

属性插值是光栅化阶段的关键计算步骤，指将三角形顶点处已知的颜色、纹理坐标、法线等属性值，通过数学方法推算出三角形内部每一个像素（片元）对应的属性值。没有属性插值，光栅化器只能绘制出纯色三角形，无法实现任何平滑着色或纹理映射。

属性插值在计算机图形学的发展中经历了两个重要阶段。早期的 Gouraud 着色（1971年由 Henri Gouraud 提出）正是依赖顶点颜色的线性插值实现平滑效果。然而，这种简单线性插值在透视投影下会产生明显的扭曲错误，直到透视校正插值被引入（在 SIGGRAPH 1994 前后成为 GPU 标准流程），才从根本上解决了这一问题。

属性插值之所以不可或缺，是因为现代实时渲染的所有视觉效果——贴图采样点的准确位置、Phong 光照模型中每像素法线的计算、屏幕空间的深度值传递——都依赖于它的正确性。插值误差会直接导致纹理游泳（texture swimming）、深度冲突（z-fighting）等视觉瑕疵。

## 核心原理

### 重心坐标与线性插值

三角形内任意一点 $P$ 可以用重心坐标 $(\lambda_1, \lambda_2, \lambda_3)$ 表示，满足：

$$P = \lambda_1 V_1 + \lambda_2 V_2 + \lambda_3 V_3, \quad \lambda_1 + \lambda_2 + \lambda_3 = 1, \quad \lambda_i \geq 0$$

其中 $V_1, V_2, V_3$ 为三角形的三个顶点。重心坐标 $\lambda_i$ 的几何意义是：$\lambda_i$ 等于点 $P$ 与对边围成的子三角形面积除以总三角形面积。对于顶点处的属性值 $A_1, A_2, A_3$（如 RGB 颜色分量），屏幕空间线性插值直接给出：

$$A(P) = \lambda_1 A_1 + \lambda_2 A_2 + \lambda_3 A_3$$

这一公式在 2D 屏幕空间中是正确的，但当属性实际定义于 3D 世界空间或相机空间中时，透视投影会破坏线性关系，必须引入校正。

### 透视校正插值

透视投影会将 3D 中等距的属性值在 2D 屏幕上压缩成不等距分布。具体原因在于投影变换是非线性的——深度 $z$ 越大，同等世界空间间距在屏幕上越小。设三个顶点的裁剪空间齐次 $w$ 分量（即相机空间深度）为 $w_1, w_2, w_3$，正确的透视校正插值公式为：

$$A(P) = \frac{\lambda_1 \frac{A_1}{w_1} + \lambda_2 \frac{A_2}{w_2} + \lambda_3 \frac{A_3}{w_3}}{\frac{\lambda_1}{w_1} + \frac{\lambda_2}{w_2} + \frac{\lambda_3}{w_3}}$$

分子是各属性除以深度后的加权和，分母是各深度倒数的加权和，两者相除等价于在相机空间中执行了线性插值再投影回屏幕。现代 GPU（如 OpenGL 和 Direct3D 的固定功能流水线）默认对所有 varying 变量执行透视校正插值，GLSL 中可通过 `noperspective` 限定符关闭此校正，改为屏幕空间线性插值。

### 重心坐标的增量式计算

实际光栅化器不会对每个像素独立计算面积，而是利用重心坐标相对于像素坐标是线性函数这一性质，采用增量式（incremental）计算。沿扫描线从左到右移动一个像素时，$\lambda_1$ 的变化量 $\Delta\lambda_1$ 是常数，可在扫描行开始时预计算。这使得每个像素的插值计算降低为 3 次加法和 3 次乘加运算，避免了逐像素的面积除法，是光栅化高效运行的重要原因。深度值 $z$ 同样采用此方式增量插值，并直接写入深度缓冲。

## 实际应用

**纹理坐标插值**：每个顶点携带 UV 坐标 $(u, v)$，通过透视校正插值后得到片元的 UV，再用此 UV 对纹理贴图采样。若错误地使用屏幕空间线性插值，在斜视角下棋盘格纹理的格线会明显弯曲——这是经典的透视校正缺失症状，在早期 PlayStation 1 游戏中（其 GPU 不支持透视校正）频繁出现，被称为"affine texture warping"。

**法线插值与 Phong 着色**：Phong 着色模型将每个顶点的法线向量插值到每个片元，再逐片元执行光照计算。插值法线在使用前必须重新归一化（`normalize`），因为两个单位向量的线性插值结果长度通常小于 1，不再是单位向量。

**深度插值**：光栅化阶段对每个片元插值得到深度值 $z$，并与深度缓冲中的存储值比较。由于透视变换后 $z/w$ 相对于屏幕坐标是线性的，GPU 实际插值的是 $z/w$（NDC 深度），而非相机空间的线性深度，这也是深度缓冲精度在远处非线性下降的根源。

## 常见误区

**误区一：忽略透视校正导致的纹理扭曲**
很多初学者在软件光栅化器中直接用屏幕空间重心坐标插值 UV，得到"差不多正确"的效果，但一旦相机倾斜观察平面，纹理格线立刻弯曲。正确做法必须使用 $A_i / w_i$ 除以 $1/w_i$ 的公式，不能省略 $w$ 的参与。

**误区二：认为插值法线无需归一化**
插值后的法线 $\vec{n}_{interp} = \lambda_1\vec{n}_1 + \lambda_2\vec{n}_2 + \lambda_3\vec{n}_3$ 的模长严格小于等于 1（等号仅在三顶点法线完全相同时成立）。若直接用于 `dot(n, lightDir)` 计算漫反射，会导致高曲率区域亮度偏暗，必须在片元着色器中调用 `normalize` 后再使用。

**误区三：混淆屏幕空间深度与线性深度**
GPU 插值并写入深度缓冲的是非线性的 NDC 深度 $z_{ndc} \in [-1, 1]$，而非相机空间中真实的线性距离。若需要在着色器中还原线性深度（例如用于 SSAO 或景深效果），必须通过投影矩阵的 $m_{33}$ 和 $m_{34}$ 分量反推，公式为 $z_{linear} = \frac{2 \cdot near \cdot far}{far + near - z_{ndc}(far - near)}$，直接使用深度缓冲值做距离计算是错误的。

## 知识关联

**前置概念——三角形装配**：属性插值的输入来自三角形装配阶段输出的顶点属性数组（position、normal、texcoord 等）。装配阶段确定了哪三个顶点构成待光栅化的三角形，并提供了透视校正插值所需的 $w$ 分量；若装配阶段的 $w$ 值计算有误，属性插值的透视校正必然出错。

**后续概念——深度缓冲**：属性插值产生的逐片元深度值 $z$ 是深度缓冲算法的直接输入。深度测试比较插值深度与缓冲中存储值，决定片元是否存活；深度写入的精度直接受属性插值中 $z/w$ 线性化的影响，因此理解非线性深度分布是调试 z-fighting 问题的前提。

**后续概念——Alpha 混合**：Alpha 值作为顶点属性之一，同样经过透视校正插值传递到片元。逐片元的 Alpha 值决定混合方程 $C_{out} = \alpha C_{src} + (1-\alpha)C_{dst}$ 中的系数，若 Alpha 插值不准确（如错误使用 `noperspective`），半透明物体的过渡边缘在透视变形下会出现错误的不连续跳变。
