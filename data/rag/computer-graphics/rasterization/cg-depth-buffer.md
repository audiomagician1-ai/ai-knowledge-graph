# 深度缓冲

## 概述

深度缓冲（Depth Buffer），又称 Z-Buffer，是光栅化管线中用于解决**可见性问题**的算法。它通过为每个像素维护一个深度值，在片元着色之前或之后判断当前片元是否应该覆盖已有结果，从而正确渲染遮挡关系。该算法由 Edwin Catmull 于 1974 年在其博士论文中提出，并于 1970 年代末被硬件厂商逐步集成到图形加速卡中，成为实时渲染的标准配置。

在 Z-Buffer 出现之前，画家算法（Painter's Algorithm）是解决可见性的主要手段——即将三角形从远到近排序后依次绘制。但画家算法无法处理三角形相互穿插的场景，且排序本身需要 O(n log n) 的开销。Z-Buffer 以牺牲一块与屏幕分辨率等大的内存为代价，将每像素的可见性判断降低到 O(1)，彻底解决了上述问题。

现代 GPU 上的深度缓冲精度通常为 32 位浮点或 24 位定点，直接影响场景中近裁面与远裁面之间的深度分辨能力。理解深度缓冲的精度分布特性，是避免 Z-Fighting 等伪影的关键。

## 核心原理

### Z-Buffer 算法流程

Z-Buffer 算法维护两块缓冲：一块颜色缓冲（Color Buffer）和一块深度缓冲。深度缓冲在帧开始时被清为最大深度值（默认为 1.0，代表最远处）。对每个光栅化产生的片元，执行如下逻辑：

```
若 fragment.depth < depthBuffer[x][y]:
    colorBuffer[x][y]  = fragment.color
    depthBuffer[x][y]  = fragment.depth
```

深度值由顶点着色器输出的裁剪坐标经透视除法后得到 NDC 深度，再通过视口变换映射到 [0, 1] 区间写入缓冲。默认比较函数为 `GL_LESS`（OpenGL）或 `D3D11_COMPARISON_LESS`（DirectX），但可更改为 `LESS_EQUAL`、`ALWAYS` 等，用于实现特殊效果（如天空盒使用 `LESS_EQUAL` 避免被剔除）。

### 深度精度与非线性分布

透视投影矩阵将视锥体映射到 NDC 空间时，深度值并非线性分布。标准 OpenGL 透视矩阵将 NDC 深度 $z_{ndc}$ 表示为：

$$z_{ndc} = \frac{f + n}{f - n} - \frac{2fn}{(f - n) \cdot z_{view}}$$

其中 $n$ 为近裁面距离，$f$ 为远裁面距离，$z_{view}$ 为视空间深度（负值）。该函数是关于 $z_{view}$ 的双曲线，导致深度精度在靠近近裁面处极度密集，而在远处极度稀疏。

以典型参数 $n = 0.1$，$f = 1000$ 为例，NDC 空间约 **90%** 的深度精度集中在前 1% 的视空间距离内（即 0.1 到 1.1 米之间）。这意味着远处两个相差数米的物体，其 NDC 深度值可能仅相差 1 个 ULP（Unit in the Last Place），直接导致 Z-Fighting 闪烁问题。

### 反向 Z（Reversed-Z）

反向 Z 是解决深度精度问题的现代方案，由 NVIDIA 工程师在 2015 年前后在行业中广泛推广。其核心思路是将深度缓冲的清除值改为 **0.0**，比较函数改为 `GREATER`，并修改投影矩阵使近裁面映射到 NDC 深度 1.0，远裁面映射到 0.0。

当配合 **32 位浮点深度缓冲**（`DEPTH_COMPONENT32F`）使用时，反向 Z 利用了浮点数在接近 0.0 时精度更高的特性——IEEE 754 单精度浮点在 [0, 1] 区间内接近 0 的一侧拥有更密集的表示。修改后的投影矩阵最后一行将 $w$ 分量的符号翻转，使原来映射到 1.0 的近裁面改为映射到 1.0（仍然），但远裁面从 1.0 改为映射到 0.0——实际上等效于将无穷远映射到恰好 0.0，从而消除了对 $f$ 的依赖，理论上支持无限远裁面（Infinite Far Plane）。

## 实际应用

**避免 Z-Fighting**：在渲染贴花（Decal）或道路标线叠加在地面时，两个几何体共面会导致 Z-Fighting。常见做法是使用 `glPolygonOffset(factor, units)` 施加一个基于斜率和深度单位的偏移量，或者改用深度值不写入缓冲的透明混合方案。

**天空盒渲染**：天空盒顶点在顶点着色器中将 NDC 深度强制设为 1.0（令 `gl_Position.z = gl_Position.w`），并将深度比较函数设为 `LESS_EQUAL`，确保天空盒始终在其他不透明物体之后绘制，同时不依赖实际几何位置。

**阴影映射（Shadow Map）**：Shadow Map 本质上是一张从光源视角渲染的深度缓冲。在采样阶段，将当前像素在光源空间的深度值与 Shadow Map 中存储的最小深度做比较，判断是否处于阴影中。深度偏移（Shadow Bias）的设置与深度缓冲的精度分布直接相关。

**粒子系统软粒子**：粒子面片与场景几何体相交时会产生硬切边。软粒子技术读取当前像素的深度缓冲值，与粒子片元的深度做差，将差值映射为透明度，使粒子在接近几何体时自然淡出。此操作需要深度缓冲被绑定为着色器只读输入（`sampler2D depthTex`）。

## 常见误区

**误区一：深度值是线性的**。很多初学者认为深度缓冲中存储的是视空间中的线性距离。实际上，标准透视投影下深度缓冲存储的是经过透视除法的 NDC 深度，它关于视空间距离是非线性的（双曲线关系）。在 GLSL 中若需要线性深度，需手动用 `(2.0 * near) / (far + near - depth * (far - near))` 还原线性深度，或在顶点着色器直接输出视空间 Z 值并用重心坐标插值（注意此时需要透视正确插值）。

**误区二：深度精度只与位宽有关**。深度精度同时受位宽与近/远裁面比值（$f/n$）影响。24 位定点深度缓冲配合 $n=0.1$、$f=10000$ 时，远处精度极差；而改用反向 Z + 32 位浮点，同样参数下精度分布则近乎均匀。单纯增大位宽而不调整裁面比，改善效果有限。

**误区三：关闭深度写入等同于关闭深度测试**。`glDepthMask(GL_FALSE)` 仅禁止将新深度值写入缓冲，但深度测试仍然进行——片元仍会与缓冲中的现有深度比较，不通过的片元会被丢弃。这在渲染透明物体时非常有用：透明物体按从后到前排序渲染，开启深度测试但关闭深度写入，使其不破坏已有的不透明物体深度信息。

## 知识关联

**前置知识——属性插值**：顶点的深度值（NDC z 坐标）在三角形光栅化过程中需要经过**透视正确插值**而非简单线性插值。这与颜色、纹理坐标等属性插值的处理方式完全相同，均需用 $1/w$ 进行加权（即先对 $z/w$ 做线性插值，再除以插值得到的 $1/w$）。理解属性插值的透视修正原理，才能解释为什么深度值在屏幕空间内表现为非线性分布。

**后续概念——模板缓冲**：模板缓冲（Stencil Buffer）与深度缓冲共享同一块物理内存，通常打包为 `DEPTH24_STENCIL8` 格式（24 位深度 + 8 位模板）。模板测试在深度测试之前执行，两者共同构成片元的逐像素可见性判断链，用于实现镜像反射、轮廓描边、阴影体等效果。

**后续概念——Early-Z 优化**：GPU 硬件在执行片元着色器之前，可以提前进行深度测试，丢弃不可见片元以节省着色开销，这称为 Early-Z。然而，当片元着色器中包含 `discard` 指令或对 `gl_FragDepth` 的写入时，Early-Z 会被自动禁用，因为此时 GPU 无法在着色前得知最终深度值。合理设计着色器以保留 Early-Z 是性能优化的重要方向。
