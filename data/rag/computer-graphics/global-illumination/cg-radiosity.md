# 辐射度方法

## 概述

辐射度方法（Radiosity Method）是一种基于有限元分析的全局光照算法，专门用于计算完全漫反射场景中的光能分布。该方法将场景中每个表面的辐射能量（以瓦特每平方米表示的辐射度值 B）视为未知量，通过建立并求解一个线性方程组来同时确定所有表面的光照强度。与光线追踪不同，辐射度方法是视点无关的，一次计算结果可从任意角度渲染，非常适合建筑室内光照模拟。

辐射度方法起源于1984年由Cornell大学的Goral等人发表的论文《Modeling the Interaction of Light Between Diffuse Surfaces》，该论文首次将热工程中的辐射换热理论引入计算机图形学。热工程领域早已使用辐射度分析炉膛内的热辐射传递，Goral团队将这套形状因子（Form Factor）计算框架移植到可见光波段，奠定了基于物理的全局光照算法的基础。

辐射度方法的核心价值在于它能天然地模拟色溢（Color Bleeding）效应——红色墙壁会将粉红色光反射到相邻白色地板上。这种漫射到漫射的光能传递在局部光照模型中完全无法体现，而辐射度方法通过全局联立求解，使每个表面的亮度都受到场景中所有其他表面的影响。

## 核心原理

### 辐射度方程

辐射度方法的核心是对每个表面 patch $i$ 建立能量平衡方程：

$$B_i = E_i + \rho_i \sum_{j=1}^{N} F_{ij} B_j$$

其中：
- $B_i$：patch $i$ 的辐射度（W/m²），即单位面积发出的总辐射功率
- $E_i$：patch $i$ 自身的自发辐射（光源项），普通表面为0
- $\rho_i$：patch $i$ 的漫反射率（0到1之间）
- $F_{ij}$：从 patch $i$ 到 patch $j$ 的形状因子
- $N$：场景中 patch 的总数

将 $N$ 个表面的方程整理成矩阵形式 $(I - \rho F)B = E$，其中 $I$ 是单位矩阵，$F$ 是形状因子矩阵，$B$ 和 $E$ 是列向量。求解这个 $N \times N$ 的线性方程组即得到所有表面的辐射度值。

### 形状因子计算

形状因子 $F_{ij}$ 表示从表面 $i$ 离开的辐射能量中有多少比例到达表面 $j$，其数学定义为：

$$F_{ij} = \frac{1}{A_i} \int_{A_i} \int_{A_j} \frac{\cos\theta_i \cos\theta_j}{\pi r^2} V(x,y) \, dA_j \, dA_i$$

其中 $\theta_i$ 和 $\theta_j$ 分别是两点连线与各自法线的夹角，$r$ 是两点间距离，$V(x,y)$ 是可见性函数（互相可见时为1，被遮挡时为0）。

形状因子满足两个重要性质：**互易性** $A_i F_{ij} = A_j F_{ji}$，以及**完整性** $\sum_{j=1}^{N} F_{ij} = 1$（封闭场景）。互易性可将计算量减半，完整性可用于验证计算正确性。实际计算形状因子的常用方法包括半立方体（Hemicube）法——将 patch $i$ 的半球投影到一个边长为1的半立方体五个面上，通过光栅化技术近似计算积分。

### 有限元离散化与方程求解

辐射度方法将连续的表面离散成 $N$ 个小平面 patch，patch 越细结果越精确但计算量以 $O(N^2)$ 增长（形状因子矩阵有 $N^2$ 个元素）。典型中等规模场景的 patch 数量在 1000 到 10000 之间，形状因子矩阵存储本身就需要大量内存。

求解线性方程组通常不使用直接的高斯消去法（代价 $O(N^3)$），而采用**迭代求解**，其中渐进辐射度（Progressive Radiosity）是最实用的变体：每轮迭代选取当前未分配辐射度最大的 patch 作为"发射者"，将其能量分配给场景中所有其他 patch，循环直到收敛。这种方式允许在中途停止并得到一个近似但视觉上有意义的结果。

## 实际应用

**建筑光照预可视化**：在建筑设计阶段，辐射度方法被用于预测室内日光分布。由于计算结果与视点无关，设计师可以在同一次计算结束后在三维模型中自由漫游，直接评估自然采光效果，无需为每个视角重复计算。AutoCAD的早期光照插件和专业照明分析软件（如RADIANCE的前身）均使用了这一原理。

**游戏引擎中的光照贴图预烘焙**：虚幻引擎（Unreal Engine）和Unity的Lightmass/Enlighten系统采用基于辐射度思想的光照贴图烘焙技术。将辐射度值存储为纹理贴图后，运行时仅需纹理采样即可获得全局光照效果，规避了实时求解线性方程组的高计算成本。这正是辐射度方法"一次计算、多次使用"特性的直接体现。

**科学仪器热分析**：航天器热控设计中，同样的辐射度方程（使用红外波段的发射率替换漫反射率）用于计算太空舱各表面之间的热辐射交换，预测各组件的平衡温度。

## 常见误区

**误区一：辐射度方法能处理所有材质**。辐射度方法严格只对完全漫反射表面（朗伯表面）有效。镜面反射、光泽材质（Glossy）均不满足辐射能量各向均匀分布的假设。对包含镜面的场景直接使用辐射度方法，镜子会被错误地处理成亮度均匀的灰色发光面，完全丢失镜像信息。解决方案是使用"辐射度+光线追踪"混合方法，或更通用的路径追踪。

**误区二：形状因子只与几何形状有关，与材质无关**。这是正确的，形状因子 $F_{ij}$ 仅由两个 patch 的几何位置、朝向和遮挡关系决定，与表面颜色、反射率完全无关。然而有些初学者误以为浅色表面和深色表面之间的形状因子不同，实际上材质信息仅通过 $\rho_i$ 进入辐射度方程，而非形状因子矩阵。

**误区三：patch 划分越细效果越好**。实际上，过细的 patch 划分会引入"暗角"（Dark Corner）伪像——两个相邻 patch 的公共边附近由于形状因子计算的数值误差，常出现不真实的暗带。工程实践中通常需要在 patch 边界处进行双线性插值（Gouraud 辐射度插值）来平滑这一伪像。

## 知识关联

辐射度方法直接建立在**渲染方程**的漫反射简化形式上。将渲染方程限定为仅处理漫反射项（BRDF为常数 $\rho/\pi$）并对整个表面积分，就得到了辐射度方程。理解渲染方程中 $L_o = L_e + \int f_r L_i \cos\theta \, d\omega$ 的结构，能帮助清晰认识辐射度方法的适用边界——它本质上是渲染方程在漫反射假设下的有限元离散化版本。

在全局光照的发展脉络中，辐射度方法与光线追踪（Ray Tracing）形成互补：前者擅长漫射间接光照，后者擅长镜面反射和焦散。1986年James Kajiya提出的路径追踪（Path Tracing）方法统一了两者，通过蒙特卡洛积分求解完整的渲染方程，最终在理论上超越了辐射度方法的局限性。现代实时渲染中的**光照探针**（Light Probe）和**球谐光照**（Spherical Harmonics Lighting）技术，都可以看作是辐射度方法"预计算全局光照、运行时查表"这一思路的延续与演化。
