# 云渲染

## 概述

云渲染是实时图形学中对体积云进行视觉模拟的技术，核心挑战在于如何用有限的计算资源表达云的半透明体积结构、光线在云内部的多次散射以及与天空大气的光学耦合。云不是一个表面，而是由数亿个微小水滴或冰晶构成的参与介质（Participating Medium），光线在其中既被吸收又被散射，导致云的外观极其复杂。

这一领域的现代突破来自2015年 Guerrilla Games 在《地平线：零之黎明》开发过程中发表的 SIGGRAPH 技术演讲，他们将 Ray Marching 与三维 Perlin-Worley 噪声相结合，实现了实时体积云渲染，将原本需要离线渲染数小时的效果压缩到了单帧毫秒级。此后这套技术框架成为游戏引擎体积云的标准范式，Unreal Engine 的 Volumetric Cloud 组件即基于类似思路构建。

云渲染的重要性不仅在于视觉效果，更在于它涉及大气光照的物理正确性——云的亮度分布直接影响场景整体光照感知，错误的云渲染会让整个天空显得假。实时云渲染还必须兼顾帧率，因此几乎所有工业方案都采用时间累积（Temporal Accumulation）来分摊多帧计算代价。

## 核心原理

### Ray Marching 体积采样

Ray Marching 是云渲染的基础求解方法。对每条从摄像机发出的射线，沿射线方向以固定步长 `ds` 逐步推进，在每个采样点读取云的密度值 `σ(x)`，累积透射率（Transmittance）和散射贡献。

透射率由 Beer-Lambert 定律给出：

$$T(a, b) = \exp\left(-\int_a^b \sigma_t(x)\,dx\right)$$

其中 `σ_t` 为消光系数（吸收系数加散射系数之和）。在离散 Ray Marching 中近似为：

$$T \approx \exp\left(-\sum_i \sigma_t(x_i) \cdot ds\right)$$

实际实现中，Guerrilla 方案采用两级采样策略：外层使用较大步长（约 128 步覆盖云层）做粗粒度密度检测，当检测到密度大于阈值时切换到小步长（约 8~16 步）精细采样，这样可以在空气区域快速跳过，节省约60%的采样数量。

### 三维噪声建模云密度

云的形状通过三维程序噪声生成，不依赖任何静态贴图。具体做法是用低频 Perlin 噪声（频率约 4~8 个单元格覆盖天空域）控制云的宏观轮廓，再用高频 Worley 噪声（也称 Cellular 噪声）叠加云的卷曲细节和侵蚀效果。

密度函数形如：

$$\rho(x) = \text{remap}(\text{Perlin}(x) - \text{Worley}(x) \cdot 0.625, 0, 1, 0, 1)$$

高度梯度（Height Gradient）在此基础上乘以一个基于高度的权重函数，使云的底部较平整（积云底面特征），顶部呈花椰菜状。卷积云、积雨云等不同云类型通过调整 Worley 噪声的权重和高度范围来区分。

### 光照散射：Henyey-Greenstein 相位函数

光在云中的散射方向分布由相位函数描述。云渲染中最常用的是 Henyey-Greenstein 相位函数（1941年提出）：

$$p(\theta) = \frac{1}{4\pi} \cdot \frac{1 - g^2}{(1 + g^2 - 2g\cos\theta)^{3/2}}$$

其中 `g` 为不对称因子，取值范围 `[-1, 1]`。云的 `g` 值通常设为 `0.7~0.8`，表示强烈的前向散射——这解释了为何逆光观察云边缘时会出现银色光晕（Silver Lining）效果。

实际渲染中，每个采样点都需要向太阳方向发射一条"遮挡光线"，通过少量采样（通常6步）估算该点到太阳的透射率，再乘以相位函数得到单次散射贡献。多次散射通常用 Powder Effect 近似：在透射率基础上叠加一个额外吸收项，模拟光在云内深处的多次弹射后变暗的效果。

## 实际应用

**游戏引擎中的实时云层**：《地平线：零之黎明》将云层高度范围设定在海拔 1500m~4000m 之间，渲染分辨率为屏幕分辨率的 1/4，配合时间累积恢复全分辨率，单帧 GPU 开销控制在 2ms 以内（PS4 基准）。Unreal Engine 5 的 Volumetric Cloud 支持光线追踪阴影，云可以在地面投射体积阴影，但这会将开销上升至 8~12ms。

**天气系统联动**：云渲染与降水系统的集成需要将云密度场与粒子系统共享。当某区域云密度积分超过阈值（通常为 `σ_t × 厚度 > 2.0`），触发降雨粒子生成，这要求密度场以 3D Texture（通常 128×128×64 分辨率）存储在 GPU 显存中以便双方采样。

**大气散射耦合**：云渲染的光照颜色不能直接使用白色或固定太阳色，必须从大气散射系统（如 Bruneton 模型）中采样天光颜色，否则黎明黄昏时云的粉橙色调无法正确出现。具体做法是预计算大气 LUT（Look-Up Table），云渲染着色器在每个采样点用高度和天顶角查询该 LUT 获取环境光。

## 常见误区

**误区一：认为步长均匀分布越密越好**。实际上均匀密集采样会在空气区域浪费大量采样，而在云边缘处精度也未必更高。正确做法是使用上文提到的粗-细两级采样，配合抖动（Jitter）避免固定步长导致的条纹状锯齿。均匀128步的方案在视觉质量上往往不如粗16步+精细32步的自适应方案。

**误区二：把云的透明度直接当作 Alpha 混合**。云的正确合成依赖 Beer-Lambert 透射率的连乘，而非简单 Alpha 加法。若用 `out.alpha += density * ds` 替代透射率连乘，会导致云在低密度区域颜色过饱和，且无法正确表达云内部的自遮挡加暗效果。

**误区三：忽略云对地面光照的影响**。云层对阳光的遮挡会使地面获得的直射光减少，这要求地面着色器读取云的投影遮挡值。若跳过此步骤，晴朗天空下的云下方地面仍会显示全强度太阳光，造成明显的光照不一致。正确方案是将云的透射率投影到地面深度图，生成一张低分辨率（如512×512）的体积阴影图。

## 知识关联

云渲染在技术上直接依赖**天空与大气**的渲染基础：大气散射确定了到达云层的太阳光颜色和强度，这是云亮度计算的物理前提。Bruneton 大气模型所生成的 Transmittance LUT 和 Irradiance LUT 会被云渲染着色器复用，两者共享同一套大气参数（Rayleigh 散射系数、Mie 散射系数、臭氧吸收率等）。理解了大气中 Mie 散射的前向散射特性，就能直接推导出云渲染中 Henyey-Greenstein 函数 `g` 值选取的物理依据——两者描述的都是大颗粒介质对可见光的散射行为，只是云粒子（直径约10μm）比气体分子大得多，散射各向异性更强。此外，云渲染中的 Participating Medium 积分方程是体积光渲染（Volumetric Fog、光柱）的通用数学框架，掌握云渲染后可直接迁移到任何需要处理散射介质的场景。
