# 几何抗锯齿

## 概述

几何抗锯齿（Geometric Anti-Aliasing，简称 GAA）是一种通过修改图元的几何形状本身来消除锯齿的抗锯齿技术，主要针对线框渲染、细线条、多边形边缘等场景。与 MSAA（多重采样抗锯齿）在光栅化阶段对子像素进行多次采样不同，几何抗锯齿不增加采样数量，而是在顶点着色器或几何着色器阶段，将线条或三角形边缘扩展出一个半透明的"羽化区域"，通过 Alpha 混合模拟边缘的平滑过渡。

这一思路最早在 2006 年前后的游戏引擎实践中出现，尤其在移动图形和早期 WebGL 场景中得到推广，因为这些平台往往无法承受 MSAA 的带宽和显存开销。几何抗锯齿的核心思想可追溯到 Jim Blinn 关于"宽线近似"的研究——将一条零宽度的线几何扩展为一个四边形（quad），并在边缘插值 Alpha 值，从而得到视觉上平滑的细线。

几何抗锯齿的重要性在于：它完全在 CPU 或 GPU 的顶点处理阶段完成，与后处理管线无关，因此不会受到深度缓冲或延迟渲染的限制，对 HDR 渲染管线也友好。对于调试可视化工具、工程 CAD 渲染、游戏中的 UI 线条等需要大量绘制细线的场景，几何抗锯齿往往是性能最优的解决方案。

## 核心原理

### 线段扩展为四边形

几何抗锯齿处理线段时，将一条从点 $P_0$ 到 $P_1$ 的逻辑线段，在屏幕空间中沿法线方向各扩展半个像素宽度（通常为 0.5～1.0 像素），生成一个矩形四边形（由两个三角形组成）。扩展后，四边形沿宽度方向的两侧边缘顶点被赋予 $\alpha = 0$，中心轴线上的顶点被赋予 $\alpha = 1$。光栅化插值后，边缘像素自然产生从 1 到 0 的 Alpha 渐变，视觉上即呈现平滑的抗锯齿效果。

扩展向量的计算公式为：

$$\vec{n} = \text{normalize}(\vec{P_1} - \vec{P_0})^{\perp} \cdot \frac{w}{2}$$

其中 $w$ 为线条的目标渲染宽度（像素），$\perp$ 表示取垂直方向。最终每条线段从 2 个顶点扩展为 4 个顶点，三角形数量从 0 增加到 2。

### Alpha 覆盖函数

边缘像素的 Alpha 覆盖值通常使用线性插值或平滑阶跃函数（smoothstep）计算：

$$\alpha(d) = \text{smoothstep}(0, 1, 1 - |d|)$$

其中 $d$ 是当前像素到线条中轴线的归一化距离，$d \in [-1, 1]$。当 $|d| = 1$ 时 $\alpha = 0$，当 $d = 0$ 时 $\alpha = 1$。使用 `smoothstep` 相较于线性插值能更好地模拟高斯模糊的视觉效果，减少"软糊"感，使线条看起来更锐利但无锯齿。

### 端点处理（Line Caps）

线段两端的帽盖（line caps）处理是几何抗锯齿中容易被忽略的细节。若不特殊处理，扩展后的矩形四边形在端点处会出现硬截断。常见解决方案是在每条线段的两端各附加一个半圆或半正方形的三角扇（由 4～8 个额外三角形构成），端点处同样赋予 $\alpha$ 渐变。这会使单条线段的顶点数从 4 个增加到约 12～20 个，但视觉完整性显著提升。对于性能敏感场景，可用片段着色器中的 SDF（有符号距离场）计算代替几何扩展来处理端点，避免额外的三角形提交。

## 实际应用

**WebGL 中的图表线条渲染**：浏览器 WebGL 环境中，`gl.LINES` 原语的线宽在非 Chrome 浏览器上最大仅支持 1 像素（Khronos WebGL 规范明确允许实现忽略 `lineWidth` 调用），因此几乎所有 WebGL 图表库（如 deck.gl、three.js 的 Line2 扩展）都采用几何抗锯齿方案渲染线条，将每条逻辑线段转换为两个三角形，并在片段着色器中计算边缘 Alpha。

**游戏中的调试可视化**：Unreal Engine 的 `DrawDebugLine` 实现在某些渲染路径下使用几何扩展而非原生 GL 线条，以保证在 DX12 / Vulkan 管线中线条宽度的一致性。

**地图渲染中的道路线**：Mapbox GL 的矢量地图渲染器将道路、国界线等线要素全部转换为几何扩展的四边形 mesh，在片段着色器中通过 `v_width` 插值变量控制边缘软化程度，支持动态调整线宽而无需重新提交顶点数据。

**移动端 UI 线框**：在 iOS Metal 和 Android Vulkan 上，原生 API 均不支持线宽超过 1 像素，游戏 UI 框架如 Unity UGUI 的自定义线条组件通常采用几何抗锯齿方案，在顶点着色器中完成扩展，整个过程无需额外的 render pass 或 framebuffer 读取。

## 常见误区

**误区一：几何抗锯齿等同于加粗线条**

几何抗锯齿扩展的额外像素 Alpha 值从 0 到 1 渐变，这些边缘像素实际上是半透明的，视觉感知宽度与原始逻辑宽度相同（或仅宽出约 1 个像素）。而单纯加粗线条（如将线宽从 1px 改为 3px）不产生任何渐变，边缘仍然是硬切割的。很多初学者看到"扩展四边形"就认为线条变粗了，但实际上 $\alpha = 0$ 的区域对不透明背景视觉上不可见，感知宽度由 $\alpha > 0.5$ 的像素范围决定。

**误区二：几何抗锯齿可以替代所有场景的 MSAA**

几何抗锯齿仅对线条和多边形的显式边缘有效，对三角形内部的纹理走样、高频几何细节（如密集网格的远景锯齿）完全无能为力。若场景中存在大量普通三角形面的锯齿问题，几何抗锯齿无法解决，仍需 MSAA 或 TAA。几何抗锯齿与 MSAA 的适用场景互补而非替代关系。

**误区三：扩展宽度越大抗锯齿效果越好**

若将扩展宽度设置为 2～3 像素而非 0.5～1 像素，Alpha 渐变斜率会变得非常平缓，导致线条出现明显的"发光"或"模糊"视觉效果，反而比未抗锯齿的硬边更难看。实践中，扩展宽度通常控制在 **0.5 个像素**（单侧），总扩展为 1 像素，这一数值与像素的奈奎斯特极限相对应。

## 知识关联

几何抗锯齿以抗锯齿概述中介绍的 Alpha 混合和覆盖率近似概念为基础：抗锯齿概述中的覆盖率模型（一个像素被图元覆盖的比例决定其颜色贡献）是理解几何抗锯齿 Alpha 渐变合理性的直接依据——边缘扩展区域的 Alpha 值模拟的正是该像素被"无限细线"覆盖的面积比例。

从学习路径来看，掌握几何抗锯齿后，可以进一步研究 **SDF 字体渲染**（同样利用距离场编码边缘覆盖信息，本质原理与几何抗锯齿的 Alpha 渐变一脉相承）以及 **MSAA 的几何阶段优化**（如 Conservative Rasterization），这些技术都是对"如何精确描述像素与图元边界关系"这一问题的不同维度解答。
