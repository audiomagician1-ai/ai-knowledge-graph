# 几何抗锯齿

## 概述

几何抗锯齿（Geometric Anti-Aliasing）是一种专门针对线段、线框和细边缘进行抗锯齿处理的技术方案，其核心思路是在几何层面对图元本身进行扩展，通过修改顶点数据和 Alpha 混合来模拟亚像素级别的平滑效果，而非依赖多重采样或后处理滤波。该技术特别适合游戏中的线框渲染、HUD 细线条、技术可视化中的细边线等场景，在这些场合中传统的 MSAA 对线条的改善效果非常有限。

该方案最早在 2009 年前后的移动图形社区中被广泛讨论，并在 GPUGems 系列文章中出现了多个工程实践版本。由于 OpenGL ES 2.0 时代移动 GPU 普遍不支持 MSAA，几何抗锯齿成为移动端渲染细线条的主流替代方案。其核心优势在于完全运行于顶点着色器阶段，GPU 采样负担极低，与透明度混合管线天然契合。

几何抗锯齿之所以在当今仍有价值，是因为它解决的是一个特定且高频的视觉问题：当一条线段宽度小于 1 像素时，光栅化器的覆盖测试会使线条产生剧烈闪烁（temporal aliasing）。几何抗锯齿通过将线条宽度强制扩展至至少 1 像素并以 Alpha 值编码"溢出"的亚像素信息，使得视觉上细线始终平滑可见。

## 核心原理

### 线段几何扩展与 Alpha 编码

几何抗锯齿的基本操作是将原始线段（GL_LINES 或等效图元）在切线方向上向两侧各扩展半个像素，将其转化为一条具有一定宽度的四边形 Quad。设原始线段宽度为 `w`（可以小于 1 像素），扩展后的 Quad 宽度为 `W = max(w, 1.0) + 1.0` 像素（额外的 1 像素用于两侧的抗锯齿过渡带）。

对于 Quad 上的每个顶点，沿法线方向赋予归一化坐标 `d`，范围从 −1 到 +1，中心轴为 0。在片段着色器中，最终的 Alpha 值由以下公式计算：

```
float coverage = clamp(0.5 + (w / 2.0 - abs(d_pixel)) / feather, 0.0, 1.0);
```

其中 `d_pixel` 是该片段距线条中轴的实际像素距离，`feather` 是过渡带宽度（通常取 0.5～1.0 像素），`w` 是目标视觉宽度。当 `w < 1` 时，线条中心的 Alpha 值也会随之降低，从而在保持几何宽度不变的前提下通过透明度来表达"更细"的视觉感知。

### 法线方向计算

在顶点着色器中，计算每条线段的屏幕空间法线是几何抗锯齿的关键步骤。给定线段的两个端点经过 MVP 变换后的裁剪坐标 `p0` 和 `p1`，需要先透视除法得到 NDC 坐标，再乘以视口尺寸 `(viewport_width, viewport_height)` 得到屏幕空间像素坐标 `s0` 和 `s1`。切线方向为 `tangent = normalize(s1 - s0)`，法线方向为 `normal = vec2(-tangent.y, tangent.x)`。Quad 的四个顶点在屏幕空间中沿 `±normal` 方向各偏移 `W/2` 像素，再经逆视口映射转换回 NDC 空间写入 `gl_Position`。这一过程必须在屏幕空间完成而非局部空间，否则透视投影会导致法线方向扭曲。

### 与 Alpha 混合管线的配合

几何抗锯齿强依赖正确配置的 Alpha 混合状态，通常使用 `SRC_ALPHA + ONE_MINUS_SRC_ALPHA` 的标准混合模式。如果渲染目标开启了预乘 Alpha（Premultiplied Alpha），则 coverage 值需要同时乘以颜色通道：`output = vec4(color.rgb * coverage, coverage)`，并将混合模式改为 `ONE + ONE_MINUS_SRC_ALPHA`。在透明物体排序场景中，几何抗锯齿产生的半透明过渡带与其他半透明几何体共同参与深度排序，必须注意渲染顺序避免过渡带被不透明物体错误遮挡。

## 实际应用

**线框调试渲染**：在游戏引擎的调试可视化模块中，线框叠加通常在不开启 MSAA 的调试摄像机下渲染，几何抗锯齿可在不增加采样数的情况下使网格边缘保持清晰。Unity 的 Gizmos 系统和 Unreal Engine 的调试绘制工具均在内部使用类似的 Quad 扩展方案。

**地图与 UI 矢量线条**：在地图应用或战略游戏的 HUD 层中，道路、边界线等矢量线条宽度往往在 1～3 像素之间。几何抗锯齿可以使这类细线在不同 DPI 屏幕和缩放级别下始终保持平滑，Google Maps SDK 的早期 OpenGL ES 渲染后端就使用该技术处理道路线段。

**技术可视化**：在科学计算可视化软件（如医学影像三维重建工具）中，神经纤维束或血管的管状结构截面轮廓线通常使用几何抗锯齿处理，原因是这类工具运行在不支持 MSAA 的离线渲染上下文中，且需要以固定的 1 像素视觉宽度呈现成千上万条细线而不能承受高昂的多重采样开销。

## 常见误区

**误区一：认为几何抗锯齿可以替代 MSAA 处理三角形边缘**
几何抗锯齿只对线条和细线框有效，它处理的是线段两侧法线方向的 Alpha 过渡，对三角形内部填充区域的斜边锯齿完全无效。如果将其用于处理多边形边缘，需要额外生成边缘 Quad 并处理顶点的角落连接问题（称为 miter joint 或 bevel joint），实现复杂度显著增加，此时通常不如直接使用 MSAA 或 FXAA。

**误区二：在深度测试开启时直接使用该技术**
几何抗锯齿生成的半透明过渡带 Fragment 的深度值与线段中心相同，当线条叠加在近处物体上时，深度测试会正确通过，但如果线条后方有更近的几何体，过渡带可能因深度测试失败而被裁剪，导致线条边缘出现硬切现象。标准解决方案是在渲染线条时启用多边形偏移（`glPolygonOffset`）或在着色器中手动将深度值向相机轻微偏移（bias 通常在 `1e-4` 量级）。

**误区三：认为 feather 参数越大越平滑**
过大的 feather 值会使过渡带超出 Quad 边界导致不必要的几何溢出，但更重要的问题是过渡带宽度过大时，线条边缘会显现出模糊感，在低分辨率屏幕上尤为明显。实验数据表明，feather 取 0.5 像素在 1080p 以上分辨率下视觉质量最优，取 1.0 像素适合 720p 及以下分辨率补偿物理像素密度不足的问题。

## 知识关联

**前置概念**：抗锯齿概述中介绍的屏幕空间采样理论和 Alpha 混合基础是理解几何抗锯齿 coverage 计算的前提。需要明确光栅化中的覆盖率（Coverage）概念：MSAA 通过多个采样点投票计算覆盖率，而几何抗锯齿通过解析公式直接计算距离来替代这一投票过程，本质上是将采样问题转化为距离函数求值问题。

**并列技术**：几何抗锯齿与有符号距离场抗锯齿（SDF Anti-Aliasing）同属"解析计算覆盖率"的思路，但 SDF 方案需要预计算距离贴图，适合字体和图标，而几何抗锯齿在运行时动态生成 Quad，适合实时变化的线条几何。两者在原理层面共享 `coverage = smoothstep(edge - feather, edge + feather, distance)` 的核心模式。

**工程扩展**：在掌握单条线段的几何抗锯齿后，实际工程中还需处理线段连接处（Line Join）和端点（Line Cap）的几何拼接，这涉及顶点着色器中的 miter limit 计算（当两线段夹角接近 180° 时 miter 向量趋向无穷大，需 clamp 到 bevel 方案）。这些扩展方案属于矢量图形 GPU 渲染的进阶主题，在 NanoVG 和 Skia 的 GPU 后端中有完整实现可供参考。
