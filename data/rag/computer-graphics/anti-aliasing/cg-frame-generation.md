# 帧生成

## 概述

帧生成（Frame Generation）是DLSS 3（2022年9月随RTX 40系列显卡发布）和FSR 3（AMD于2023年发布）中引入的核心技术，通过分析相邻两帧之间的运动信息，使用光流算法（Optical Flow）插值合成一张全新的中间帧，从而将实际渲染帧率倍增。与传统的超分辨率重建不同，帧生成不依赖于提升渲染分辨率，而是在时间维度上创造新内容。

DLSS 3的帧生成模块被称为DLSS Frame Generation，它与DLSS超分辨率（SR）和NVIDIA Reflex并列运行，并且强制要求硬件支持——仅限搭载第四代Tensor Core和新增的光流加速器（Optical Flow Accelerator）的RTX 40系列GPU使用。AMD FSR 3的帧生成实现则不绑定特定硬件，可在更广泛的GPU上运行，但整体效果和延迟特性与DLSS 3存在差异。帧生成的意义在于，它打破了CPU瓶颈限制渲染帧率的场景——即使CPU端提交命令的速度无法提升，GPU依然可以通过插帧输出更高频率的画面。

## 核心原理

### 光流估计与运动向量生成

帧生成的第一步是计算相邻两帧之间每个像素的运动方向和速度，即光流场（Optical Flow Field）。DLSS 3在RTX 40系列的专用光流加速器上完成此计算，该加速器与渲染管线并行工作，不占用主要的CUDA核心资源。光流估计输出一张与分辨率相对应的运动向量贴图，记录从帧N到帧N+1中每个像素块的位移矢量（Δx, Δy）。FSR 3同样使用光流技术，但其光流估计在标准着色器上运行，计算开销会消耗部分渲染算力。

### 中间帧的插值合成

获得光流场后，算法将运动向量乘以插值系数0.5，分别从帧N和帧N+1向中间时刻进行双向warp（前向投影和反向投影）。投影结果会产生空洞区域（Occlusion Holes），这是物体运动后露出背景、但背景在任一参考帧中均被遮挡的区域。DLSS 3使用专门的神经网络对空洞进行填充修复，该网络同时利用帧N和帧N+1的颜色信息、以及深度缓冲区和运动向量缓冲区进行判断。最终生成的插帧会被插入两个真实渲染帧之间，使输出序列变为：真实帧N → 插值帧 → 真实帧N+1。

### 延迟代价与Reflex的配合

帧生成并非没有代价。插帧在时间上滞后于最新渲染帧，导致感知输入延迟（Perceived Input Latency）增加，通常约增加约半帧的显示延迟。NVIDIA为此强制捆绑NVIDIA Reflex技术：Reflex通过压缩渲染队列、减少CPU到GPU之间的命令排队延迟，将总体系统延迟降低约25%～35%，从而在启用帧生成后仍维持与原始帧率相近的实际延迟体验。FSR 3同样支持AMD Anti-Lag+作为对等方案。

## 实际应用

在《赛博朋克2077》中，DLSS 3帧生成被作为最早的完整商业案例展示：在RTX 4090上，启用DLSS质量模式+帧生成后，游戏帧率从约85 FPS跳升至约160 FPS以上，实现近乎2倍的帧率增益。由于《赛博朋克2077》属于严重GPU瓶颈的场景，帧生成的效果尤为显著。

FSR 3在2024年的《星空》（Starfield）等游戏中落地，其开放实现允许开发者通过FSR 3的公开SDK集成，而不必依赖NVIDIA专有驱动。开发者需要向管线提供颜色缓冲、深度缓冲和运动向量三组输入，FSR 3框架内部完成光流分析和帧合成。

值得注意的是，帧生成对于渲染帧率本身很低（低于30 FPS）的场景效果较差：插帧所依赖的运动估计误差会随帧间间隔增大而累积，导致鬼影（Ghosting）和空洞修复失败的概率明显上升。NVIDIA官方建议原始帧率至少达到60 FPS以上再启用帧生成，以保证插帧质量。

## 常见误区

**误区一：帧生成等同于传统视频插帧（如SVP/DAIN）**
游戏帧生成与离线视频插帧有本质区别。传统视频插帧只有颜色信息，必须纯粹从像素亮度变化推断运动。游戏帧生成可以直接访问渲染管线输出的精确运动向量缓冲区、深度缓冲区和模板缓冲区，使运动估计的精度远高于视频插帧，尤其在处理UI元素时会将其单独剔除（DLSS 3通过深度缓冲识别UI层并直接从最新真实帧中复制，而非插值）。

**误区二：帧生成会降低渲染画质或减少GPU渲染工作量**
帧生成不减少任何实际渲染的几何体或光照计算。真实帧N和帧N+1依然是完整渲染的帧，帧生成仅在这两帧之间额外插入一张合成帧。因此其性能开销是加法而非乘法——RTX 40系列上DLSS帧生成自身大约消耗0.5～1.5毫秒的额外GPU时间，这部分时间来自光流加速器和Tensor Core的并行工作。

**误区三：帧生成适用于所有游戏类型**
对于输入延迟极度敏感的竞技类FPS游戏，即使配合Reflex，帧生成带来的额外半帧延迟也可能影响高水平玩家的操作反馈。此外，在快速旋转镜头或高速粒子效果密集的场景下，光流估计容易出现大面积误差，产生明显的运动模糊瑕疵或画面撕裂感，这是帧生成技术当前阶段固有的局限性。

## 知识关联

帧生成建立在DLSS超分辨率的架构经验之上，但两者解决的是不同维度的问题：DLSS SR在空间维度提升分辨率，帧生成在时间维度补充帧率。理解DLSS的Tensor Core推理流程和运动向量的使用方式，是掌握帧生成插值逻辑的必要前提——尤其是运动向量的坐标系定义（NDC空间中的亚像素偏移量）和时间抗锯齿（TAA）中的历史帧复用策略，与帧生成的双向warp机制高度相关。帧生成目前是DLSS/FSR技术栈中时序性最强、硬件依赖性差异最大的一环，未来随着光流估计精度的提升，其在低帧率场景和竞技类游戏中的适用边界有望进一步扩展。
