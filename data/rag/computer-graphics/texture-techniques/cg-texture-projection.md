# 纹理投影

## 概述

纹理投影（Projective Texture Mapping）是一种将纹理像投影仪一样"投射"到几何体表面的技术，由Paul Heckbert和Henry Moreton于1991年在SIGGRAPH上系统性阐述。与标准UV纹理映射不同，纹理投影不依赖顶点预设的UV坐标，而是通过一个独立的投影矩阵在运行时计算每个片元的纹理坐标，使同一纹理可以动态投射到任意几何形状上。

纹理投影的价值在于其"与模型无关"的特性。场景中的物体无需事先展UV，投影来源（虚拟投影仪）可以在运行时自由移动和旋转，纹理会实时重新分布在所有接受投影的表面上。这使它成为实现幻灯片投影、阴影贴图、激光图案、车灯照明纹理、以及贴花系统的基础技术。

## 核心原理

### 投影纹理坐标的计算

纹理投影的核心是用一个"投影相机"的变换矩阵来生成纹理坐标，该坐标系被称为投影纹理空间（Projector Space）。计算流程如下：

1. 将顶点位置从世界空间变换到投影相机的裁剪空间，使用矩阵 `M_proj = P_light × V_light`，其中 `P_light` 是投影相机的投影矩阵，`V_light` 是其视图矩阵。
2. 得到裁剪坐标 `q = (q_x, q_y, q_z, q_w)` 后，执行透视除法并将NDC坐标从 `[-1, 1]` 映射到 `[0, 1]`：

$$
s = \frac{q_x / q_w + 1}{2}, \quad t = \frac{q_y / q_w + 1}{2}
$$

3. 最终使用 `(s, t)` 在纹理中采样。

在实际GPU实现中，这步坐标变换通常写成一个偏移矩阵 `M_bias`（将NDC的 `[-1,1]` 范围缩放偏移到 `[0,1]`）与投影矩阵的乘积，在顶点着色器中一次性完成。

### 透视校正与齐次坐标传递

关键细节在于：透视插值必须在**片元着色器中用齐次坐标形式**完成，而非在顶点着色器中提前做透视除法。若在顶点阶段就将 `q_x/q_w` 传给片元，光栅化插值将产生线性错误，导致纹理在倾斜表面出现"拉伸扭曲"。

正确做法是将 `q_x, q_y, q_w` 作为三个独立的 varying 变量传入片元着色器，在片元阶段执行：

```glsl
vec2 projCoord = vec2(q_x / q_w, q_y / q_w);
vec4 color = texture2D(projTex, projCoord * 0.5 + 0.5);
```

GLSL内置的 `textureProj()` 函数正是封装了这一透视除法步骤，接受 `vec3(s, t, q)` 或 `vec4(s, t, r, q)` 作为参数，自动执行 `s/q, t/q` 的除法操作。

### 背面投影与裁剪处理

当几何体处于投影相机**后方**时，`q_w < 0`，透视除法后坐标符号翻转，会导致纹理被"镜像投射"到物体背面。解决方案有两种：

- **在片元着色器中丢弃背面片元**：检查 `if (q_w < 0.0) discard;`
- **使用投影相机的近远平面裁剪**：将 `q_z / q_w` 限制在 `[0, 1]` 范围内，超出则不采样。

此外，当投影坐标超出 `[0, 1]` 范围时，纹理的 Wrap 模式会决定边缘行为。贴花类应用通常将纹理边缘设为透明并使用 `Clamp to Border`（边框颜色 RGBA = 0），防止纹理在目标区域外重复出现。

## 实际应用

**动态投影灯光纹理**：游戏中模拟带图案的聚光灯（如蝙蝠侠的蝙蝠侠信号灯），将一张蝙蝠图案纹理通过投影矩阵投射到场景几何体上。投影相机跟随灯光位置旋转，纹理随实时光源移动自然扭曲到建筑表面。

**Shadow Mapping（阴影贴图）**：这是纹理投影最广泛的应用。深度图从光源视角渲染后，作为投影纹理投射回场景，每个片元的投影坐标 `(s, t)` 用于在深度图中查询遮挡深度，并与该点自身深度 `q_z/q_w` 比较，决定是否处于阴影中。OpenGL的 `sampler2DShadow` 类型专门为此设计，支持PCF（百分比渐近过滤）的硬件加速。

**屏幕空间贴花（Screen-space Decal）的前身**：早期游戏（如Quake时代）使用纹理投影实现弹痕、血迹等贴花，将贴花纹理从法线方向投射到接受几何体。现代贴花系统仍保留这一投影思路，只是将接受体从任意网格改为轴对齐包围盒内的延迟渲染缓冲区。

## 常见误区

**误区一：认为纹理投影等同于"把UV坐标投影到物体上"**。纹理投影的坐标是由独立投影相机的变换矩阵在渲染时计算的，与模型自身的UV完全无关。修改模型UV不会影响投影结果；移动"投影仪"位置才会改变纹理分布。两者的区别类似于"给物体涂色"与"用投影仪打光"的区别。

**误区二：在顶点着色器中提前完成透视除法**。将 `projCoord = clipPos.xy / clipPos.w` 在顶点阶段计算后传递给片元，会使光栅化插值在非垂直视角的斜面上产生线性误差，导致纹理扭曲。必须传递未除法的 `vec4` 齐次坐标，或使用 `textureProj()` 在片元阶段进行透视正确的插值。

**误区三：忽视背面投影问题**。遗漏对 `q_w < 0` 的检查会导致投影出现在物体背面并镜像翻转，初学者往往误以为是纹理坐标计算错误，反复检查矩阵运算，实际问题只需一行裁剪判断即可解决。

## 知识关联

**前置概念**：纹理投影建立在纹理映射概述中采样、过滤、Wrap模式的基础上——理解 `Clamp to Border` 和 `textureProj()` 的行为，需要先熟悉纹理坐标的基本寻址方式。投影矩阵的构造原理（将平截头体映射到NDC）也直接复用了渲染管线的透视变换知识。

**后续概念**：贴花系统（Decal System）直接以纹理投影为核心机制，将投影方向固定为贴花法线方向，并增加对深度缓冲区的采样来限制贴花只附着在目标表面而非穿透几何体。掌握纹理投影中背面裁剪和 `Clamp to Border` 的处理方式，是理解贴花系统如何控制投影范围和混合边界的直接前提。
