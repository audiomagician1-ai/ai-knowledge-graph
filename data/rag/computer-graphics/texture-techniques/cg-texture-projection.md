# 纹理投影

## 概述

纹理投影（Projective Texture Mapping）是一种将纹理像投影仪打光一样投射到三维场景几何体表面的技术。与标准纹理映射直接使用顶点绑定的UV坐标不同，纹理投影使用一个独立的"投影矩阵"来动态计算每个片元的纹理坐标，使纹理坐标不再依附于模型自身的展开，而是由投影方向和位置决定。这一技术由Lance Williams在1978年的阴影贴图论文中隐含提出，后被Segal等人于1992年在SIGGRAPH论文《Fast Shadows and Lighting Effects Using Texture Mapping》中系统化应用。

纹理投影的价值在于它允许同一张纹理以动态方式投射到任意几何体上，无需为每个接收物体预先展开UV。这使它成为聚光灯纹理、投影仪效果、阴影贴图（Shadow Map）接收阶段以及贴花系统的基础机制。理解纹理投影是掌握实时渲染中众多视觉效果的前提。

## 核心原理

### 投影纹理坐标的计算

纹理投影的核心是将世界空间或视图空间的顶点坐标，通过一个"投影矩阵"变换为纹理坐标。设顶点世界坐标为 **P**，投影纹理坐标 **(s, t, r, q)** 的计算公式为：

```
[s, t, r, q]ᵀ = M_proj × P
```

其中 **M_proj** = **M_bias × M_perspective × M_view_light**，分别是偏置矩阵、投影矩阵（透视或正交）和投影灯的视图矩阵。最终采样纹理时，GPU执行透视除法 `s/q` 和 `t/q` 得到归一化的2D纹理坐标。偏置矩阵 **M_bias** 的作用是将裁剪空间的 [-1, 1] 范围映射到纹理空间的 [0, 1] 范围，其标准形式为：

```
M_bias = | 0.5  0    0    0.5 |
         | 0    0.5  0    0.5 |
         | 0    0    0.5  0.5 |
         | 0    0    0    1   |
```

### 透视投影与正交投影的区别

纹理投影可以使用透视投影或正交投影两种方式。透视投影模拟真实投影仪或聚光灯，越远离投影中心的区域纹理越大，坐标计算中 `q` 值随深度变化，必须做透视除法。正交投影则用于平行光源或贴花系统中沿法线方向的投射，此时 `q` 固定为1，纹理在整个深度范围内保持相同尺寸。选择哪种投影方式直接影响最终视觉效果：聚光灯投影必须使用透视投影，而建筑墙面弹孔贴花通常使用正交投影。

### GLSL中的纹理投影采样

在OpenGL GLSL着色器中，专门提供了 `textureProj()` 函数来处理透视除法，它等价于手动执行 `texture(sampler, coord.xy / coord.w)`。具体写法为：

```glsl
uniform sampler2D projTexture;
uniform mat4 projMatrix;

void main() {
    vec4 projCoord = projMatrix * vec4(worldPos, 1.0);
    vec4 color = textureProj(projTexture, projCoord);
}
```

`textureProj()` 内部自动完成 `projCoord.xy / projCoord.w` 的计算，避免手动除法引入的精度误差。此外，还需要配置纹理的 `GL_CLAMP_TO_BORDER` 包裹模式并将边界颜色设为黑色或透明，防止超出投影范围的区域出现纹理重复。

### 投影范围控制

裁剪坐标系中，只有 `q > 0` 的片元才位于投影体的正前方。当 `q ≤ 0` 时，片元位于投影源背面，若不过滤会产生"背投影"伪影——纹理被镜像投射到物体背面。常见解决方法是在着色器中添加 `if (projCoord.w <= 0.0) discard;` 判断，或使用阴影贴图的比较采样模式自动剔除背面。

## 实际应用

**聚光灯投影**：在游戏中模拟手电筒或舞台追光灯时，纹理投影可将一张带有渐变光晕的纹理按透视方式投射到场景几何体上，光锥之外区域自动变暗。《Half-Life 2》中的手电筒效果正是使用纹理投影而非实时光源计算实现的，大幅降低了GPU开销。

**阴影贴图接收**：阴影贴图的第二阶段（接收阴影）本质上是纹理投影——将光源视角渲染的深度贴图以光源投影矩阵重新投射到场景，比较每个片元的深度值与贴图中存储的深度，判断是否处于阴影中。这是纹理投影在实时渲染中应用最广泛的场景。

**贴花系统基础**：贴花（Decal）如弹孔、血迹、脚印，需要将图案附着在任意朝向的表面。使用正交纹理投影，可以沿碰撞法线方向投射一个小矩形区域的纹理，不需要修改原始模型的UV数据，是动态贴花系统的核心实现方式。

## 常见误区

**误区一：认为纹理投影坐标就是普通UV坐标**。标准UV坐标是二维的 `(u, v)`，每个顶点固定绑定，而纹理投影坐标是四维齐次坐标 `(s, t, r, q)`，需要在每帧根据投影矩阵动态计算，并在片元着色器中执行透视除法。混淆两者会导致错误地在顶点着色器插值时忽略透视矫正，产生明显的纹理扭曲。

**误区二：忽略背投影问题**。初次实现纹理投影时，许多开发者会发现纹理出现在物体背面并呈镜像状态。这是因为没有过滤 `q ≤ 0` 的片元。解决方法不是"反转坐标"，而是必须在采样前判断 `projCoord.w` 的符号并丢弃背面片元。

**误区三：将纹理投影等同于屏幕空间投影**。纹理投影的投影源是一个独立的"投影灯"，有自己的位置、方向和视锥体，与渲染摄像机完全无关。屏幕空间技术（如SSAO）使用的是摄像机的NDC坐标，两者在矩阵来源、坐标空间和应用场景上均不同。

## 知识关联

纹理投影建立在**纹理映射概述**的基础上——理解标准UV采样和纹理坐标系是计算投影坐标的前提。纹理投影引入的"以矩阵动态计算纹理坐标"思想，直接延伸至下一个主题**贴花系统**：贴花系统在纹理投影的正交模式之上，增加了深度排序、法线检测和混合模式管理，构建出完整的动态表面细节系统。此外，纹理投影与**阴影贴图**技术高度重合——阴影贴图的光源矩阵构建方式与纹理投影的 `M_proj` 构建方式完全一致，学习两者时可以共用同一套矩阵推导流程。
