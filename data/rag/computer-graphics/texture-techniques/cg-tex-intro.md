# 纹理映射概述

## 概述

纹理映射（Texture Mapping）是一种将二维图像"贴"到三维几何体表面的技术，使模型在不增加顶点数量的前提下获得丰富的视觉细节。其本质是建立三维表面上每个点与二维图像像素之间的对应关系，GPU 据此在光栅化阶段为每个片元（Fragment）查找并混合颜色值。

纹理映射的概念由 Edwin Catmull 于 1974 年在其博士论文中首次提出，最初用于将参数化曲面与图像数据绑定。1976 年 Jim Blinn 进一步将该技术应用于环境反射模拟，奠定了现代纹理管线的基础。今天，实时渲染中一个角色模型往往同时使用 4 到 16 张不同用途的纹理，纹理采样操作占据 GPU 渲染带宽的相当大比例。

纹理映射之所以关键，在于它实现了"几何复杂度"与"视觉复杂度"的解耦——一个仅有 1000 个三角形的低多边形石墙模型，通过一张 2048×2048 的纹理图像，可以呈现出数百万像素级别的砖缝细节，而无需消耗额外的顶点变换算力。

## 核心原理

### UV 坐标系统

三维模型的每个顶点除了携带 XYZ 空间坐标外，还存储一对 UV 坐标（也称纹理坐标），其中 U 对应图像的水平轴，V 对应垂直轴。UV 坐标的标准范围是 [0.0, 1.0]，(0, 0) 对应纹理图像的左下角，(1, 1) 对应右上角（OpenGL 约定；DirectX 中 V 轴方向相反，(0, 0) 在左上角）。

顶点的 UV 坐标由建模师或展UV工具（如 Blender 的 Smart UV Project）预先计算好并存储在网格数据中。GPU 在光栅化三角形时，对三个顶点的 UV 值进行**透视正确插值**，为三角形内部每个片元计算出精确的 UV 坐标。这一插值过程需要除以顶点的齐次坐标 w，而非简单线性插值，否则在透视投影下会出现明显的纹理扭曲（这一错误曾是早期 PlayStation 1 游戏的标志性视觉缺陷）。

### 纹理对象与采样器

在现代图形 API（如 OpenGL 4.x 或 Vulkan）中，纹理映射涉及两个独立的对象：**纹理对象**（Texture Object）存储像素数据（Texel），**采样器对象**（Sampler Object）定义如何从纹理中读取数据。

采样器的核心参数包括：

- **过滤模式（Filter Mode）**：当片元的 UV 坐标落在两个 Texel 之间时，`GL_NEAREST` 取最近邻像素（产生马赛克感），`GL_LINEAR` 取双线性插值（产生模糊过渡）。
- **环绕模式（Wrap Mode）**：当 UV 坐标超出 [0,1] 范围时，`GL_REPEAT` 使纹理无缝平铺，`GL_CLAMP_TO_EDGE` 将边界像素延伸，`GL_MIRRORED_REPEAT` 使纹理镜像重复。

在 GLSL 着色器中，纹理采样的基本语句为：

```glsl
vec4 color = texture(uTexture, vTexCoord);
```

其中 `uTexture` 是 `sampler2D` 类型的 uniform 变量，`vTexCoord` 是从顶点着色器插值传入的 `vec2` 类型 UV 坐标，`texture()` 函数根据当前绑定的采样器设置执行查找并返回 RGBA 四通道颜色值。

### 纹素（Texel）与分辨率映射

纹理图像中的单个像素称为**纹素（Texel，Texture Element 的缩写）**，以区别于屏幕上的像素（Pixel）。纹素与屏幕像素的比例关系直接影响渲染质量：

- **放大（Magnification）**：屏幕上一个纹素覆盖多个像素，纹理显得模糊或有锯齿，此时双线性插值可改善效果。
- **缩小（Minification）**：多个纹素压缩到一个像素内，若不处理会产生严重的摩尔纹走样（Aliasing），这正是 Mipmap 技术要解决的问题。

一张 RGB 格式的 512×512 纹理在未压缩情况下占用 512 × 512 × 3 = 786,432 字节（约 768 KB）显存；而同等尺寸的 RGBA8 纹理则占用约 1 MB。

## 实际应用

**漫反射贴图（Diffuse/Albedo Map）** 是纹理映射最基础的应用，将物体的基础颜色信息存入纹理。例如一个木箱模型只需要 24 个顶点构成 6 个面，通过绑定一张木纹漫反射贴图，片元着色器采样后即可渲染出完整的木质外观。

**UV 平铺（UV Tiling）** 是地形和建筑渲染的常用技巧：将砖块纹理的 UV 缩放为 (10, 10)，让它在一面大墙上重复贴 100 次（10×10），从而以单张低分辨率纹理覆盖大面积表面，同时保持纹素密度均匀。

在 Unity 的材质面板中，Tiling 参数（X, Y）直接控制 UV 的缩放系数，Offset 参数控制 UV 的平移量，底层对应的变换公式为：`UV_final = UV_vertex * Tiling + Offset`。

## 常见误区

**误区一：UV 坐标是三维坐标的一部分。** 实际上 UV 是完全独立的属性，不受模型位移、旋转或缩放的影响（除非着色器中显式修改）。同一个顶点在不同 UV 通道（UV0、UV1）中可以有不同的坐标值，例如 UV1 常用于存储光照贴图（Lightmap）的坐标。

**误区二：纹理坐标越出 [0,1] 就会报错或被裁剪。** 超出范围的 UV 坐标行为完全由采样器的环绕模式决定——`GL_REPEAT` 下坐标 (2.3, 0.7) 等同于 (0.3, 0.7)，不会产生任何错误，这是平铺贴图的工作原理。

**误区三：更高分辨率的纹理总是更好。** 当模型在屏幕上仅占 50×50 像素时，使用 4096×4096 的纹理不仅浪费显存（未压缩约 64 MB），还因缺少 Mipmap 而在缩小时产生走样。纹理分辨率应与模型的最大屏幕占用面积相匹配。

## 知识关联

纹理映射是后续所有纹理技术的起点。**Mipmap** 直接建立在缩小滤波问题上——它通过预生成一系列分辨率递减（每级宽高各减半）的纹理层级来解决纹素与像素比例失调的走样问题。**法线贴图**复用了同样的 UV 坐标和采样机制，只是将纹理数据从颜色值改为存储表面法线的 RGB 向量（R→X, G→Y, B→Z）。**立方体贴图**则放弃了传统的 UV 坐标，改用三维方向向量索引六个面的纹理，适用于天空盒和环境反射。**纹理压缩**格式（如 BC1/DXT1、ASTC）针对 GPU 硬件解码特性，将纹理的显存占用降低为原始大小的 1/4 到 1/8。**纹理图集**（Texture Atlas）将多张小纹理打包进一张大纹理，通过调整每个网格的 UV 范围来引用各自的区域，从而减少 Draw Call 数量。
