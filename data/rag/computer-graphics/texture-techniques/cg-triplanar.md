# 三平面映射

## 概述

三平面映射（Triplanar Mapping）是一种不依赖UV展开的纹理投影技术，通过沿世界空间（或切线空间）三个正交轴方向——X轴、Y轴、Z轴——分别投影纹理，再将三张投影结果按法线权重混合，最终得到完整的表面纹理颜色。由于完全绕过了UV坐标系统，它特别适合那些几何体复杂、难以手工展UV的模型，例如程序化生成的地形或岩石体积。

该技术最早在游戏地形渲染领域得到广泛应用，2009年前后随着GPU可编程着色器的普及进入实时渲染管线。Ryan Geiss等图形程序员在公开的着色器代码中对其混合权重算法进行了系统阐述。现代游戏引擎如Unreal Engine 4/5和Unity均内置了三平面映射节点，使其成为地形材质系统中的标准工具。

三平面映射的核心价值在于彻底消除"UV缝隙"（UV Seam）和"极点拉伸"（Pole Stretching）问题。对于曲面复杂的有机物体，传统UV展开需要耗费数小时艺术家工作量，而三平面映射仅需在着色器中设置纹理缩放参数即可实现无缝覆盖，代价是额外增加约两倍的纹理采样次数。

## 核心原理

### 三轴投影坐标的计算

三平面映射首先将每个片元（Fragment）的世界空间位置坐标分解为三对UV坐标：

- **YZ平面投影**：使用 `(worldPos.y, worldPos.z)` 作为UV，对应X轴方向法线朝向的表面
- **XZ平面投影**：使用 `(worldPos.x, worldPos.z)` 作为UV，对应Y轴方向（顶面/底面）的表面
- **XY平面投影**：使用 `(worldPos.x, worldPos.y)` 作为UV，对应Z轴方向法线朝向的表面

这三套坐标均以世界空间单位为基础，因此纹理重复频率由世界坐标的物理尺度决定。例如，若地形网格单位为1米，纹理缩放因子设为0.5，则每2米世界距离重复一次贴图。

### 混合权重的计算公式

三轴投影的混合权重 `w` 由世界空间法线 `N = (Nx, Ny, Nz)` 的绝对值决定，核心公式为：

```
w = pow(abs(N), k)
w = w / (w.x + w.y + w.z)
```

其中 `k` 为锐化指数（Sharpening Exponent），典型取值范围为 **1 到 8**。当 `k=1` 时混合过渡宽泛柔和，适合沙土地面；当 `k=8` 时过渡带极窄，投影边界清晰，适合具有明确岩层分界的石壁。最终对三张采样结果按 `w.x`、`w.y`、`w.z` 加权求和，即得到混合颜色值。

### 法线贴图的特殊处理

当三平面映射应用于法线贴图时，不能直接对三轴采样出的切线空间法线进行混合，因为三个投影面各自拥有不同的切线坐标系。正确做法是将每个面采样到的切线空间法线分别转换回世界空间，再按权重求和后归一化。常见做法称为"whiteout混合"，其公式为：

```
n_world_x = float3(n.xy + N.zy, abs(N.x)) * sign(N.x)
n_world_y = float3(n.xy + N.xz, abs(N.y)) * sign(N.y)
n_world_z = float3(n.yx + N.xy, abs(N.z)) * sign(N.z)
result = normalize(n_world_x * w.x + n_world_y * w.y + n_world_z * w.z)
```

若跳过此步直接混合切线空间法线值，斜面45°方向将出现明显的法线方向错误，导致光照计算异常。

## 实际应用

### 游戏地形材质

在大型开放世界游戏中，地形网格通常由运行时生成，顶点数可达百万级，手工UV展开根本不可行。三平面映射可直接基于地形顶点的世界坐标驱动泥土、岩石、草地等多层纹理，并利用Y轴权重（即顶面权重 `w.y`）自动在水平地面显示草地纹理，在垂直崖壁显示岩石纹理，无需额外的顶点绘制数据。

### 程序化体积几何体

SDF（有向距离场）生成的岩石、洞穴等体积网格没有UV坐标。以Houdini或GPU Marching Cubes生成的网格直接在片元着色器中应用三平面映射，可在不改变几何体生成管线的前提下即时获得具有细节的表面纹理，是程序化内容管线中的标准做法。

### UE5材质节点示例

Unreal Engine 5的材质编辑器提供名为"World Aligned Texture"的内置节点，其内部即实现了三平面映射逻辑。开发者只需连接贴图资产并设置"Texture Size（World Space）"参数（单位为厘米），即可无缝应用到任意静态网格，该参数直接对应上述世界坐标缩放因子。

## 常见误区

### 误区一：认为三平面映射完全没有视觉接缝

三平面映射虽然消除了UV接缝，但当锐化指数 `k` 较低时，三个投影平面的过渡区（即45°斜面附近）会出现两张不同纹理内容的模糊叠加，形成视觉上的"混合带"。这并非接缝，但在高对比度纹理（如砖块、木纹）上同样会造成明显的视觉不自然感。解决办法通常是提高 `k` 值或在混合区专门使用过渡纹理变体。

### 误区二：三平面映射与对象空间坐标等价

部分初学者将世界空间三平面映射与对象局部空间三平面映射混淆。世界空间版本中，多个相同模型在场景中摆放时，它们的纹理相位会因世界坐标不同而自动错开，显得自然；而对象空间版本中，所有实例纹理相位相同，密集摆放时会产生"复制感"。两者在着色器中仅有坐标空间变换一行之差，但视觉效果截然不同。

### 误区三：三次采样性能开销可忽略不计

三平面映射最少需要对同一纹理进行3次采样（若同时采样颜色图、法线图、粗糙度图，则为9次），相比标准UV采样增加两倍采样指令。在移动端GPU上，纹理采样是主要性能瓶颈之一，直接套用桌面端三平面映射逻辑可能导致帧率下降30%以上。优化策略包括：仅对主导权重最大的1~2个轴进行采样（Dominant Axis Triplanar），将 `w` 值最小的轴投影丢弃。

## 知识关联

**前置知识**：纹理映射概述中介绍的UV坐标系统是理解三平面映射"绕过UV"这一设计动机的必要背景——只有清楚标准UV管线的接缝成因，才能体会三平面映射以世界坐标替代UV坐标的优越性。法线向量的含义以及点积运算也是理解权重计算公式 `pow(abs(N), k)` 的基础。

**延伸方向**：掌握三平面映射后，可进一步研究基于高度图的纹理混合（Height-Based Blending），该技术在三平面混合权重基础上叠加纹理自身的高度信息，使岩石凸起处优先显示岩石纹理、凹陷处优先显示泥土，混合效果更接近真实物理侵蚀规律。此外，随机化三平面映射（Stochastic Triplanar）通过在每个投影方向引入哈希偏移，可打破纹理重复的周期性规律，是大尺度地形渲染的进阶技术。
