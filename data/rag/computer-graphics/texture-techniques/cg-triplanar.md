# 三平面映射

## 概述

三平面映射（Triplanar Mapping）是一种无需UV展开坐标的纹理投影技术，通过沿X、Y、Z三个世界空间轴向分别投影同一张纹理，再根据表面法线方向对三张投影结果进行加权混合，最终得到覆盖整个几何体的连续纹理效果。这项技术的核心价值在于彻底绕过了传统UV展开的工作流程，使得复杂地形、程序化生成的网格或任意形状的几何体都能自动获得纹理覆盖。

三平面映射最早在游戏地形渲染领域得到广泛应用，约在2000年代中期随着开放世界游戏的兴起而普及。Epic Games在虚幻引擎的地形系统中将其内置，使开发者无需对高度图生成的地形网格手动制作UV。与此同时，该技术也在程序化内容生成（PCG）工作流中扮演重要角色——算法生成的岩石、洞穴和建筑废墟往往形状不规则，传统UV展开既耗时又容易产生接缝，三平面映射完全规避了这一瓶颈。

该技术对于关卡美术师和技术美术师的意义在于速度与质量的平衡：一个标准地形场景如果改用UV展开，展开工时可能占据整个纹理流程的40%以上，而三平面映射将这部分成本降至接近零，代价是额外的GPU采样消耗（理论上为普通纹理采样的3倍）。

## 核心原理

### 三轴投影与坐标采样

三平面映射使用顶点或片段的世界空间坐标（World Position）作为纹理UV坐标：
- **XY平面投影**：用 `(worldPos.x, worldPos.y)` 采样纹理，对应从Z轴方向观察的投影
- **XZ平面投影**：用 `(worldPos.x, worldPos.z)` 采样纹理，对应从Y轴（顶部/底部）方向的投影
- **YZ平面投影**：用 `(worldPos.y, worldPos.z)` 采样纹理，对应从X轴方向的投影

由于坐标直接来自世界空间，当物体移动时纹理会随之"滑动"（称为Texture Swimming问题），因此通常将坐标乘以一个缩放因子 `k`（如 `worldPos.x * 0.1`）来控制纹理密度。

### 法线权重计算

三个投影结果的混合权重由表面法线的分量绝对值决定。标准权重公式为：

```
weights = abs(normal)
weights = pow(weights, sharpness)   // sharpness 通常取 4.0 ~ 8.0
weights = weights / (weights.x + weights.y + weights.z)
```

其中 `sharpness`（锐度）参数控制投影切换的过渡区域宽度：值越大，不同轴向投影之间的过渡带越窄，混合越生硬；值为1时呈线性混合，接缝区域最宽。最终三个采样结果按权重线性插值：

```
color = colorXY * weights.z + colorXZ * weights.y + colorYZ * weights.x
```

注意权重分量与平面对应关系：法线Z分量最大的表面（接近水平的顶面）应主要显示XZ平面投影结果，因此 `weights.y` 权重对应 `colorXZ` 采样。

### 法线贴图在三平面映射中的处理

当使用法线贴图（Normal Map）时，三平面映射面临额外挑战：每个轴向投影的法线贴图采样结果处于不同的切线空间，需要分别转换到世界空间后才能混合。标准做法是使用"Whiteout混合"法，针对每个轴向分别进行切线空间到世界空间的旋转，三轴的切线空间矩阵由世界法线隐式确定。如果直接在混合后对法线贴图采样，接缝处会出现明显的法线方向错误，这是初学者最常犯的错误之一。

## 实际应用

### 游戏地形系统

虚幻引擎的Landscape材质系统和Unity的地形着色器均内置了三平面映射支持。以Unity HDRP为例，其Terrain Lit着色器在坡度超过设定阈值（默认约60度）时自动从标准UV采样切换到三平面映射，避免山坡上的纹理因UV拉伸而失真。开发者可在材质的"Terrain"选项卡中直接启用`Enable Triplanar`选项，无需手动编写Shader代码。

### 程序化岩石与洞穴

在Houdini等DCC软件中，程序化生成的岩石几何体常与三平面映射结合使用。美术师在Houdini内直接编写VOP节点完成三平面采样，导出材质时将世界坐标烘焙为顶点色或额外UV通道，使后续引擎渲染时无需重新计算世界空间位置，提升运行时性能。

### 细节叠加层（Detail Overlay）

即使原始网格已有UV，三平面映射也常用于叠加全局细节纹理。例如在角色身上叠加一层全局污垢纹理（Dirt Layer）时，使用三平面映射可以保证污垢纹理不受角色UV分布的影响，在躯干、四肢、面部之间无接缝衔接。

## 常见误区

### 误区一：三个轴向必须使用同一张纹理

三平面映射的标准实现确实对三个轴使用相同纹理，但这并非硬性要求。在某些地形场景中，美术师会对Y轴（顶面）投影使用草地纹理，对X/Z轴（侧面）投影使用岩石纹理，通过混合权重在垂直壁面和水平地面之间实现自然过渡，这种变体称为"方向感知三平面映射"（Angle-Based Triplanar），是地形渲染的常见扩展手法。

### 误区二：三平面映射性能开销可以忽略不计

三平面映射在着色器中进行3次完整的纹理采样，若材质包含Albedo、Normal、Roughness、Metallic共4张纹理，则实际采样次数为 `4 × 3 = 12` 次，而普通UV材质仅需4次。在移动平台或纹理带宽受限的场景下，12次采样与4次采样的性能差距可能导致帧率从稳定60fps下降至45fps以下。对此常用的优化方案是仅对法线贴图应用三平面映射，Albedo等颜色贴图仍使用单次投影，将采样次数控制在6次以内。

### 误区三：混合过渡带接缝是不可避免的

若接缝处出现明显色差，通常原因是法线权重的`sharpness`值设置过高，或者没有对法线贴图进行正确的切线空间转换。通过将`sharpness`调整到4.0~6.0的合理区间，并采用前述Whiteout法线混合，接缝可以完全消除，在视觉上与无缝纹理无异。

## 知识关联

三平面映射建立在**纹理映射概述**中UV坐标概念的基础上——理解"UV坐标是纹理空间的二维索引"之后，三平面映射的本质就是用世界坐标的三对分量（XY、XZ、YZ）分别充当三套UV坐标。学习三平面映射后，可进一步探索**凹凸贴图与法线贴图**，因为法线贴图在三平面映射中的切线空间处理是该技术的最复杂环节，也是深入掌握三平面映射着色器实现的关键步骤。此外，**LOD与Mipmap**技术在三平面映射中同样适用：三次独立采样均受益于各自的Mip级别计算，需要注意在视角掠射时三个轴向Mip级别不一致可能引起的视觉闪烁问题（Mip Seam），这在地形远景渲染中需要专门处理。
