# 程序纹理

## 概述

程序纹理（Procedural Texture）是通过数学函数或算法在运行时动态计算像素颜色值的纹理生成技术，与传统位图纹理不同，它不依赖预先存储的图像文件，而是在着色器代码或CPU程序中实时求解函数 `f(x, y, z) → color`，输入通常是三维空间坐标或纹理坐标，输出为颜色、法线或其他材质属性。

程序纹理的技术根源可追溯至1985年，Ken Perlin在为电影《TRON》（电子世界争霸战）开发视觉效果时提出了Perlin噪声算法，该成果于1997年获得奥斯卡科学技术奖。同年，Ken Musgrave系统化地将分形几何与噪声函数结合，建立了程序纹理的理论框架，使得云层、山脉、大理石等自然材质的计算机模拟成为可能。

程序纹理在现代图形管线中的核心价值体现在三个可量化的优势上：存储空间接近零（一个复杂大理石纹理的着色器代码不超过2KB，而对应的4K位图需要约48MB）；支持任意分辨率无损缩放；以及天然的三维连续性——切割一个程序纹理木材球体，截面的纹理与外表面完全物理一致。

## 核心原理

### Perlin噪声的数学构造

Perlin噪声是绝大多数程序纹理的基础模块，其核心思想是在规则整数网格的每个格点上预先分配一个随机梯度向量，对空间中任意点 `P = (x, y, z)`，找到其所在单元的8个顶角格点，计算每个顶角梯度向量与该顶角到P的偏移向量的点积，再利用**三次平滑插值函数** `f(t) = 6t⁵ - 15t⁴ + 10t³` 对8个点积值进行三线性插值，最终得到 [-1, 1] 范围内的连续浮点值。

Ken Perlin最初使用的是 `f(t) = 3t² - 2t³`（即平滑步进函数smoothstep），但在2002年的改进版本中换用五次多项式，原因是五次多项式在 t=0 和 t=1 处的一阶和二阶导数均为零，消除了格点位置处法线场的不连续伪影，对需要法线贴图应用的场景尤为关键。

### 分形布朗运动（fBm）

单层Perlin噪声生成的纹理过于平滑，缺乏自然材质的多尺度细节。分形布朗运动（Fractional Brownian Motion，fBm）通过叠加多个不同频率和振幅的噪声层来模拟这种多尺度特征，其公式为：

```
fBm(P) = Σ(i=0 to N-1) amplitude^i × noise(frequency^i × P)
```

其中 `frequency` 通常取2.0（每层频率翻倍），`amplitude` 通常取0.5（每层振幅减半），这个参数在物理学上称为**赫斯特指数H=0.5**对应的分形维数约为2.5。叠加层数N一般取4至8层，层数越多细节越丰富但计算量线性增长。

### 域扭曲（Domain Warping）

域扭曲是程序纹理中产生湍流流体、岩石纹理等有机形态的关键技术，由Inigo Quilez系统整理并推广。其核心思路是先用一个fBm函数计算坐标的偏移量，再用偏移后的坐标采样另一个fBm，形成自引用结构：

```
q = fBm(P)
r = fBm(P + q)
result = fBm(P + r)
```

每级扭曲都会将原始规则网格的格点结构进一步打散，产生类似云朵卷积边缘或岩浆流动的视觉效果。一次域扭曲（只算q层）生成的是较柔和的变形；二次扭曲（加入r层）则产生明显的漩涡卷曲结构。

### Worley噪声（细胞噪声）

1996年Steven Worley在SIGGRAPH提出的Worley噪声（又称细胞噪声/Voronoi噪声）与Perlin噪声的梯度机制完全不同。它在空间中随机散布特征点，对每个着色点计算其到最近特征点的距离 `F1` 和第二近特征点的距离 `F2`，通过 `F1`、`F2-F1`、`F2/F1` 等组合生成皮肤细胞、泡沫、鳞片等生物纹理特征。

## 实际应用

**实时游戏着色器**：Unreal Engine和Unity的shader中常用fBm叠加3到5层噪声生成动态云层，通过对时间参数做偏移 `noise(uv + time * 0.1)` 实现云朵流动，整个着色器代码约50行GLSL/HLSL，无需任何纹理采样带宽。

**大理石纹理合成**：经典大理石程序纹理使用公式 `color = sin(P.x * 5.0 + fBm(P) * 8.0)`，将fBm结果作为正弦函数的相位扰动，产生不规则弯曲条纹；再通过颜色渐变映射（Color Ramp）将 [-1, 1] 值域映射到白色-灰色-黑色的大理石色带。

**地形高度图生成**：程序地形生成工具（如World Machine、Houdini）将多层fBm直接用作高度场 `h(x,z) = fBm(x,z)`，赫斯特指数H控制地形的粗糙程度：H=0.9生成平滑丘陵，H=0.3生成嶙峋山脊。Minecraft的地形生成算法基础也是三维Perlin噪声的密度场采样。

## 常见误区

**误区一：程序纹理等于"随机纹理"**
Perlin噪声是**确定性的伪随机函数**，相同输入坐标永远返回相同值。之所以视觉上显得随机，是因为它在不同空间位置返回平滑变化但视觉上无规律的值。真正的随机函数（每次调用返回不同值）无法产生空间连续的纹理，直接使用GPU的 `rand()` 只会得到白噪声（电视雪花状）而非有机纹理。

**误区二：噪声函数频率越高纹理越好**
采样频率超过着色分辨率的奈奎斯特频率（即每像素采样2次以上的最高空间频率）会引发**纹理走样（Aliasing）**，在程序纹理中表现为摩尔纹或闪烁。正确做法是用`fwidth(P)`计算当前像素的微分宽度，动态剔除超出可分辨频率的fBm高频层，这一技术称为**带限程序纹理（Band-Limited Procedural Texture）**。

**误区三：三维程序纹理开销必然高于二维贴图采样**
在GPU上，一次纹理采样涉及纹理缓存读取，在高分辨率或大LOD跳变时会产生大量缓存未命中。对于简单的4层fBm（约20次乘加运算），其ALU计算成本在现代GPU上通常低于一次L2缓存未命中的纹理采样延迟（约200-400个时钟周期）。

## 知识关联

学习程序纹理需要具备**纹理映射**的基础知识，特别是UV坐标的概念——程序纹理同样以UV或三维世界空间坐标作为 `f` 的输入，理解坐标系变换对于调试纹理的拉伸和对齐问题至关重要。

程序纹理是**纹理合成**技术的前驱知识：纹理合成的目标是从样本位图出发自动生成无缝平铺纹理，而程序纹理则从数学函数出发直接生成。两者在频谱特性分析、多尺度结构构造等理论工具上高度共通——理解fBm的频率叠加原理后，再学习基于统计频谱匹配的纹理合成算法（如Portilla-Simoncelli模型）会容易得多。

Perlin噪声同样是**着色器编程**和**PCG（程序化内容生成）**的核心基础构件，在地形生成、粒子效果、材质系统等方向均有直接应用，掌握其数学推导而非仅会调用API函数，是深度应用的前提。
