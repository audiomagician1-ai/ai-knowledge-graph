# 程序纹理

## 概述

程序纹理（Procedural Texture）是指通过数学函数而非预存图像数据来动态生成纹理颜色的技术。与传统位图纹理不同，程序纹理在运行时由CPU或GPU执行算法计算每个纹素的值，因此不受分辨率限制，可以在任意缩放级别下保持清晰细节，且不占用纹理内存。

程序纹理的现代基础由Ken Perlin在1983年参与电影《创：战纪》（Tron）视觉效果制作时奠定。他于1985年在SIGGRAPH发表了经典噪声函数算法，后来被称为"Perlin噪声"，并因此获得奥斯卡技术成就奖。2001年，他又发布了改进版本"Simplex噪声"，解决了原版在高维空间计算量过大的问题：Perlin噪声在3D中需要12个梯度向量插值，而Simplex噪声在3D中仅需4个顶点，计算复杂度从O(2^n)降至O(n^2)。

程序纹理在游戏与影视渲染中的价值在于无限细节与运行时参数化：同一套噪声代码通过调整种子（seed）值即可生成风格迥异的大理石、火焰、云层纹理，而整个逻辑仅需数百字节着色器代码，而非数十MB的纹理贴图。

## 核心原理

### Perlin噪声的数学构成

Perlin噪声的输出值由三步计算得到：**梯度向量随机化 → 距离向量点积 → 平滑插值**。

给定输入坐标 `(x, y, z)`，首先将其映射到整数网格，在每个网格顶点处查表得到一个伪随机梯度向量 `g`，再计算从顶点到输入点的距离向量 `d`，做点积 `dot(g, d)` 得到影响值。最终通过缓动函数（ease curve）进行插值：

```
f(t) = 6t^5 - 15t^4 + 10t^3
```

这个五次多项式（由Perlin在2002年替换原版三次曲线而来）保证了一阶和二阶导数在节点处均为零，从而消除了明显的网格状伪影。

### 分形布朗运动（fBm）

单层Perlin噪声的视觉效果过于平滑，实际应用中通常叠加多个不同频率与振幅的噪声层，称为**分形布朗运动（Fractional Brownian Motion，fBm）**：

```
fBm(x) = Σ(i=0 to n-1) amplitude_i × noise(x × frequency_i)
```

其中每层的频率乘以一个称为**lacunarity**（间隙度，典型值2.0）的系数，振幅乘以一个称为**gain**（增益，典型值0.5）的系数。叠加6至8层后，生成的噪声在视觉上呈现出云层或山脉的自然分形质感。Hurst指数 H 控制分形维度，H=0.5 时接近布朗运动特性。

### Worley噪声（细胞噪声）

1996年Steven Worley提出了另一类程序纹理基础：**Worley噪声**，又称细胞噪声（Cellular Noise）。算法在空间中随机撒布特征点，对于任意查询点，计算它到最近特征点的距离 `F1` 和次近距离 `F2`。不同的距离组合产生不同视觉效果：
- `F1`：产生泡状细胞图案，常用于皮肤、鹅卵石纹理
- `F2 - F1`：产生蜂窝状脉络，常用于龟裂地面、生物组织

为保证实时性能，算法将空间划分为均匀网格，每格预置固定数量的特征点，查询时只需检查相邻27个（3D情况）网格单元，时间复杂度为O(1)。

## 实际应用

**游戏地形生成**：Minecraft的地形高度图由多层fBm噪声叠加后经阈值化生成，山地生物群系使用较高lacunarity（约2.5）使地形起伏更陡峭。虚幻引擎5的材质编辑器内置了名为"Noise"的材质节点，支持直接在GPU着色器中调用Perlin、Simplex和Voronoi（Worley）三种噪声类型。

**实时火焰与流体效果**：火焰着色器中，程序纹理的UV坐标随时间偏移，通过 `uv.y -= time * speed` 驱动噪声场向上流动，叠加多层不同速度的fBm实现湍流感，整个效果不需要任何预渲染动画帧。

**影视VFX中的体积云**：Disney在《超能陆战队》制作中使用Curl噪声（由Perlin噪声的无散度旋度场推导而来）驱动云体粒子运动，保证了烟雾流场的无散度物理正确性。

**着色器代码的移植**：GLSL中实现2D Perlin噪声只需约40行代码，Shadertoy平台上数以千计的程序纹理shader证明了其在WebGL实时环境下的可行性，单帧计算时间通常在0.1ms至2ms之间。

## 常见误区

**误区一：认为程序纹理完全替代位图纹理**。程序纹理在生成几何自然纹样（大理石、木纹、云雾）时有显著优势，但对于需要精确艺术控制的角色面部纹理、手绘风格图案，位图纹理依然不可替代。程序纹理的输出难以与真实照片扫描材质匹配，这是它在PBR工作流中通常作为辅助层而非主材质使用的原因。

**误区二：将Perlin噪声的值域误认为 [-1, 1] 均匀分布**。实际上，Perlin噪声输出集中在0附近，极值处概率极低，整体呈近似正态分布。直接用原始噪声值映射颜色会导致对比度低、画面发灰。正确做法是将输出值重映射为 `(noise + 1) / 2` 后再调整对比度，或使用幂函数曲线拉伸分布。

**误区三：混淆Perlin噪声与白噪声（White Noise）**。白噪声是完全随机、无空间相关性的，相邻像素之间没有连续性，表现为电视雪花状。Perlin噪声的核心特征是**空间连续性**——相邻坐标的输出值平滑过渡，且具有可控的频率特征，这也是它能模拟自然现象的根本原因。

## 知识关联

程序纹理建立在**纹理映射概述**的UV坐标体系之上：噪声函数的输入坐标通常直接使用模型的UV坐标或世界空间坐标，理解UV平铺与变换对于控制程序纹理的尺度和方向至关重要。若UV坐标存在扭曲，程序纹理会同步变形，这与位图纹理的表现完全一致。

掌握程序纹理后，自然过渡到**纹理合成**（Texture Synthesis）领域。纹理合成关注的是如何从一小块样本图像出发，生成任意大小的同风格纹理，与程序纹理"从零用数学生成"的思路互补——前者依赖样例统计，后者依赖显式算法，两者结合（如使用程序噪声驱动纹理合成中的随机扰动）在现代渲染管线中十分普遍。此外，学习程序纹理为理解**体积渲染**中的密度场建模和**地形程序化生成**奠定了直接的技术基础。
