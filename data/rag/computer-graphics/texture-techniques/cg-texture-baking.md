# 纹理烘焙

## 概述

纹理烘焙（Texture Baking）是一种将高多边形模型（高模）的表面细节信息预计算并存储到二维纹理贴图中，再映射到低多边形模型（低模）上的技术。其核心目的是让低模在渲染时呈现出接近高模的视觉效果，同时大幅减少实时渲染的几何计算负担。例如，一个拥有 800 万面的雕刻高模可以将其表面凹凸细节烘焙到一张 4096×4096 像素的法线贴图中，再由仅有 8000 面的低模使用该贴图进行渲染，节省约 99.9% 的几何数据量。

纹理烘焙技术在 1990 年代末随着游戏引擎对实时渲染性能的要求日益严苛而逐渐成熟。早期工具如 3ds Max 的 Render to Texture 功能于 2002 年前后开始普及，随后 xNormal（2006 年发布）成为独立烘焙工具的标杆，Marmoset Toolbag 和 Substance Painter 则将烘焙工作流进一步整合到现代 PBR 管线中。

烘焙流程直接决定了最终资产的视觉质量上限。法线贴图烘焙中产生的接缝、光照遮挡（AO）贴图的精度、以及曲率（Curvature）贴图的准确性，均会影响后续材质制作阶段的每一个细节效果。因此，烘焙是高模到低模制作流程中不可省略的关键工序。

## 核心原理

### 光线投射与法线烘焙

法线烘焙的基本原理是从低模表面沿法线方向向外发射射线，与高模表面相交后，读取高模在该交点处的切线空间法线方向，并将其编码为 RGB 颜色值存储到贴图中。具体编码规则为：法线向量的 X 分量映射到 R 通道，Y 分量映射到 G 通道，Z 分量映射到 B 通道，每个分量从 -1~1 线性压缩到 0~255 的 8 位整数范围。这意味着切线空间法线贴图中大面积的紫蓝色（RGB 约为 128, 128, 255）代表该区域法线方向垂直于表面，没有偏转。

投射笼（Cage）是法线烘焙中控制射线起始位置的关键配置。Cage 是低模的略微膨胀版本，将射线起点从低模表面向外偏移，避免射线与低模自身相交产生自遮挡错误。Cage 偏移量过大会导致高模细节采样错位，偏移量过小则仍会出现黑色噪点伪影，通常建议将 Cage 偏移值设置为模型包围盒最短边长度的 0.5%~2% 之间。

### 环境光遮蔽（AO）烘焙

AO 贴图记录的是每个表面点被周围几何体遮挡后能接收到的环境光比例，值域为 0（完全遮蔽，黑色）到 1（无遮蔽，白色）。其计算公式为：

$$AO(p) = \frac{1}{\pi} \int_\Omega V(p, \omega) (\hat{n} \cdot \omega) \, d\omega$$

其中 $V(p, \omega)$ 为可见性函数（射线未被遮挡时为 1，被遮挡时为 0），$\hat{n}$ 为表面法线，$\omega$ 为半球积分方向。实际烘焙时通过蒙特卡洛采样近似该积分，采样射线数量通常设置在 64~512 之间，采样数越高结果越平滑但耗时越长。AO 烘焙还有一个重要参数是**最大遮蔽距离（Max Distance）**，只有距离小于该阈值的遮挡几何体才会产生遮蔽效果，避免远处无关结构污染烘焙结果。

### 曲率（Curvature）烘焙

曲率贴图记录表面的凸起和凹陷信息，凸起区域编码为亮色（接近白色），凹陷区域编码为暗色（接近黑色），平坦区域为中灰（128/255）。曲率值通过计算每个顶点相邻法线之间的夹角差异得到，数学上对应二阶微分（拉普拉斯算子）的近似。曲率贴图在 Substance Painter 中被广泛用于驱动边缘磨损和缝隙污渍等程序化材质效果，是 PBR 材质制作中使用频率最高的辅助贴图之一。

### UV 展开与纹理集匹配

烘焙精度直接受低模 UV 展开质量约束。每个 UV 孤岛（Island）之间必须保留足够的边距（Padding），在 4096×4096 分辨率的贴图中建议 Padding 不低于 8 像素（即 8/4096 ≈ 0.2%），以防止 GPU 进行 mipmap 降采样时相邻 UV 孤岛产生颜色渗透（Bleeding）。此外，烘焙时低模的 UV 坐标必须与最终材质纹理集所使用的 UV 通道完全一致，否则烘焙贴图与材质将出现错位。

## 实际应用

**游戏角色资产制作**：在制作一个写实人类角色时，艺术家先在 ZBrush 中雕刻出包含皮肤毛孔、衣物褶皱和铠甲铆钉的 500 万面高模，随后在 Maya 中建立 15000 面左右的低模并完成 UV 展开，最后在 Marmoset Toolbag 4 中将法线、AO 和曲率同时烘焙到同一套 4096 纹理集内，整个烘焙过程通常在 GPU 加速下耗时 30 秒到 3 分钟不等。

**建筑可视化静态光照烘焙**：Unreal Engine 的 Lightmass 系统将全局光照（GI）、阴影信息烘焙到专用 Lightmap UV 通道中，该通道与材质 UV 通道相互独立，Lightmap 分辨率设置为 64 到 512 之间的 2 的幂次，直接控制烘焙阴影的清晰程度。

**硬表面道具烘焙**：金属枪械等硬表面道具的曲率贴图烘焙尤为重要。枪托的棱角曲率数据在 Substance Painter 中直接驱动边缘磨损发生器（Edge Wear Generator），使金属边角自动产生磨光效果而不需要手工绘制，极大提升材质制作效率。

## 常见误区

**误区一：法线贴图可以完全替代几何细节**。法线贴图只能模拟光照层面的凹凸，无法改变模型的实际轮廓形状。当从侧视角或掠射角（Grazing Angle，通常小于 15°）观察模型时，法线贴图无法模拟出凸起或凹陷对轮廓的影响，此时高模与低模的轮廓差异会暴露出来。因此，低模的几何形状必须在大轮廓层面与高模保持匹配，细节才能由烘焙法线补偿。

**误区二：高模与低模不需要精确对齐**。烘焙时高模和低模必须在世界空间中完全重叠，任何位置偏移都会导致射线采样到错误位置，产生大面积错误法线（表现为贴图上异常的彩色色块）。许多初学者在导入烘焙软件时忘记检查高低模的轴心点（Pivot Point）是否归零，导致模型错位烘焙失败。

**误区三：烘焙分辨率越高越好**。在一张 4096 贴图上，单个像素覆盖约为 0.024% 的 UV 空间。如果模型 UV 孤岛面积极小，则即使提升到 8192 分辨率也无法获得更多细节，反而大幅增加贴图内存占用（8192 贴图的 VRAM 占用约为 4096 的 4 倍）。应优先优化 UV 展开使孤岛面积分布均匀，再选择合适的烘焙分辨率。

## 知识关联

纹理烘焙直接依赖**材质纹理集**的 UV 划分结果。纹理集决定了烘焙目标的 UV 空间布局，烘焙所产生的法线、AO、曲率贴图均以纹理集的 UV 坐标为基准进行像素写入。若纹理集 UV 出现重叠或比例失调，烘焙结果将直接继承这些错误，且无法在后期纠正。

烘焙产出的各类贴图（法线、AO、曲率）在后续 PBR 材质制作中分别承担不同职责：法线贴图进入 PBR Shader 的 Normal 输入槽，AO 贴图叠乘于漫反射颜色通道，曲率贴图则作为程序化生成器的遮罩输入。三张贴图的质量共同决定了最终实时渲染中材质的真实感上限，因此高质量烘焙是高效材质制作工作流的前提条件。
