# 纹理合成

## 概述

纹理合成（Texture Synthesis）是指从一张小的纹理样本图像出发，通过算法自动生成视觉上相似、尺寸更大的纹理图像的技术。与程序纹理通过数学公式直接生成纹理不同，纹理合成的输入是真实的像素样本——例如一块砖墙照片或一段布料扫描图——输出则是可任意扩展的同类纹理。这使得艺术家无需手工绘制大型纹理，只需提供一个小样本即可。

纹理合成的研究起步于1990年代末。Efros 和 Leung 于1999年发表的论文《Texture Synthesis by Non-parametric Sampling》奠定了基于像素的合成框架，提出通过在样本中搜索相似邻域来逐像素填充输出图像。2001年，Efros 和 Freeman 进一步提出了 Image Quilting 算法，将合成单元从单像素升级为小图块（patch），并引入最小误差边界缝合（minimum error boundary cut），显著提升了合成速度和质量。这两篇论文构成了现代基于样本纹理合成方法的基础。

在实时图形渲染中，纹理合成解决了一个核心视觉问题：小纹理贴图直接平铺（tiling）会产生肉眼可辨的重复图案，破坏真实感。纹理合成通过将样本中的统计结构自然延伸，避免了规律性重复，是生成游戏地形、建筑材质和自然景观纹理的关键手段。

## 核心原理

### 基于邻域匹配的像素级合成

Efros-Leung 方法将纹理合成视为一个条件概率问题：给定已合成像素形成的邻域窗口 L（通常取 5×5 或 9×9 的正方形），从样本图像中搜索所有具有相似邻域的像素点，然后从中随机采样一个像素值填入当前位置。相似度通过加权高斯距离衡量：

$$d(L, L') = \sum_{i} w_i (L_i - L'_i)^2$$

其中 $w_i$ 是距离中心像素越近权重越大的高斯核系数，$L_i$ 和 $L'_i$ 分别是目标邻域和候选邻域中第 $i$ 个已知像素的颜色值。算法按光栅扫描顺序逐像素合成，每步只使用已合成区域的像素参与邻域计算。该方法的主要缺点是时间复杂度极高，合成一张 200×200 的图像可能需要数分钟。

### Image Quilting 与最小误差缝合

Image Quilting 将合成过程从逐像素改为逐图块（patch）。算法将输出图像划分为等大小的方块区域，每次从样本中选取一个与已有相邻图块重叠区域误差最小的候选块：

$$\text{patch}^* = \arg\min_{\text{patch}} \|E(\text{overlap})\|^2$$

其中重叠区域（overlap）宽度通常设置为图块边长的 1/6 左右。为消除图块边界的可见拼接痕迹，算法在重叠区域内计算最小误差边界缝合路径（minimum error boundary cut）——使用动态规划找出一条从上到下（或从左到右）的路径，该路径上两个图块的差值平方和最小，然后沿此路径混合两张图块。这使得图块边界从视觉上"消失"，合成图像的自然感显著优于简单平铺。

### 无缝平铺纹理生成

无缝平铺（seamless tiling）是纹理合成的一个特殊目标：生成的纹理图像在水平和垂直方向首尾相接时不产生任何可见接缝。实现无缝平铺的经典方法之一是**环形纹理合成**：在合成过程中，将输出图像的边界像素的邻域同时包含图像对侧的像素，即把输出图像视为拓扑环面（torus）进行邻域查找。另一种常见方法是频域混合法：将纹理样本进行傅里叶变换，强制使频谱满足周期性边界条件后再变换回空间域，可以快速生成适合直接平铺的无缝纹理，但对非规则随机纹理效果有限。

## 实际应用

**游戏地形纹理**：大型开放世界游戏的地形系统需要覆盖数平方公里的草地、泥土和岩石表面。使用 Image Quilting 或其改进算法（如 Wei-Levoy 2000年提出的多分辨率版本），美术师只需提供一张约 256×256 像素的地面照片样本，即可合成覆盖整个场景的非重复纹理贴图。

**建筑与材质扩展**：在建筑可视化中，扫描真实砖墙或混凝土表面得到的高分辨率样本往往受拍摄条件限制尺寸有限。纹理合成可将一张 512×512 的砖墙样本扩展为 4096×4096 的无缝贴图，同时保持砖缝宽度、砖块颜色分布等统计特征的一致性，而不像简单平铺那样暴露 4×4 的重复单元。

**图像补全与纹理修复**：纹理合成原理被直接应用于图像修复（inpainting）。Criminisi 等人2004年的算法利用 patch 匹配从图像已知区域合成内容来填充被遮挡或损坏的区域，这本质上是在已有图像上局部执行纹理合成，要求填充区域的局部统计特征与周围区域一致。

## 常见误区

**误区一：纹理合成等同于简单平铺**。简单平铺（tiling）将原始样本直接重复排列，任何有规律的纹理都会暴露出整齐的矩形网格。纹理合成则通过随机选取 patch 或像素，破坏了这种周期性规律，使大型纹理看起来更像真实表面的自然变化，而非复制粘贴的结果。两者在算法复杂度和视觉输出上有根本差异。

**误区二：样本越大，合成质量越高**。实际上，Efros-Leung 方法的邻域窗口大小（5×5 到 25×25 像素）而非样本总尺寸才是控制合成局部结构准确性的关键参数。过大的邻域会找不到足够相似的候选邻域，导致合成失败或引入不相关的纹理结构。对于随机纹理（如沙地），较小的邻域窗口（5×5）即可获得良好结果；对于结构性纹理（如砖墙），则需要较大的窗口（15×15 或更大）才能保持砖块边缘的连续性。

**误区三：纹理合成只能处理二维颜色图像**。现代纹理合成技术同样适用于多通道纹理数据，包括法线贴图（normal map）、粗糙度贴图（roughness map）和位移贴图（displacement map）。对这些通道执行联合合成（joint synthesis）时，需要保持各通道之间的统计相关性，例如砖块凸起区域在法线贴图中的朝向向量应与颜色贴图中的砖块位置保持一致。

## 知识关联

程序纹理（如 Perlin 噪声）通过数学公式生成纹理，无需样本输入但难以精确模拟真实材质的复杂细节；纹理合成则以真实样本为驱动，在复现真实感上具有优势，但对于完全随机的噪声类纹理，程序纹理的效率更高。两种技术在实际管线中经常结合使用：先用程序纹理生成宏观变化（如地形高度分布），再用纹理合成在局部区域填充细节样本。

纹理合成中的 patch 匹配技术与计算机视觉中的光流估计（optical flow）和图像检索方法共享相同的数学基础——高维特征空间中的最近邻搜索。近年来，基于深度学习的纹理合成方法（如 2015年 Gatys 等提出的 Neural Style 使用 VGG 网络特征 Gram 矩阵）将卷积神经网络激活值的统计分布作为纹理的描述符，是经典邻域匹配思想在深度特征空间中的延伸。
