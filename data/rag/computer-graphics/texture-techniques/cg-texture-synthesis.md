# 纹理合成

## 概述

纹理合成（Texture Synthesis）是指从一块小的样本纹理出发，通过算法自动生成尺寸更大、视觉上连贯的新纹理图像的技术。与程序纹理不同，纹理合成的输入是真实照片或手绘的像素样本，输出结果保留了样本的视觉风格与统计特性，同时避免了简单平铺所产生的明显重复接缝。

该领域的奠基性工作发表于2000年：Efros 和 Leung 在论文 *Texture Synthesis by Non-Parametric Sampling* 中提出了逐像素的Markov随机场模型，将纹理合成问题形式化为：根据已生成像素的邻域在样本图中寻找统计相似区域，逐步填充新像素。这一工作将纹理合成从参数化统计模型（如Heeger-Bergen的多尺度直方图匹配）推向了数据驱动的非参数方法。

纹理合成在游戏地形贴图、影视特效背景、建筑可视化等场景中不可或缺。当需要覆盖数百平方米的石墙或草地时，手工制作大尺寸贴图成本极高，而简单重复铺贴会暴露明显的tile边界。纹理合成可以从一张256×256的样本自动生成2048×2048的无缝纹理，且视觉质量接近手工绘制。

## 核心原理

### 基于邻域的像素级合成（Efros-Leung算法）

Efros-Leung方法为每个待合成像素维护一个正方形邻域窗口（典型尺寸为5×5至25×25像素）。合成时从图像左上角按光栅顺序推进，对于每个新像素 p，算法在样本图中搜索所有与 p 的已知邻域**加权欧氏距离**最小的候选像素，再从距离低于阈值 `(1+ε)·d_min`（ε通常取0.1）的候选集合中随机抽取一个，从而引入多样性防止重复。该方法的时间复杂度为 O(N·M)，其中 N 为输出像素数，M 为样本像素数，对大图像非常耗时。

### 基于图像块的合成（Efros-Freeman Image Quilting）

2001年Efros和Freeman提出 Image Quilting，将合成单位从单像素升级为矩形图像块（patch），大幅提升速度。算法将输出分割为重叠的块网格，相邻块在**重叠区域**内通过动态规划寻找最优拼接缝（Minimum Error Boundary Cut）。拼接缝的代价函数为：

```
E(缝) = Σ (B1[i,j] - B2[i,j])²
```

其中 B1、B2 是两个相邻块在重叠区的像素值。最优缝沿重叠区从上到下（或从左到右）以最小误差路径切割，使块边界在视觉上不可察觉。这一方法将合成速度提升了约100倍，成为后续算法的标准基线。

### 基于优化的方法（PatchMatch与神经网络）

Barnes等人2009年提出的PatchMatch算法将块匹配加速至接近实时：通过随机初始化匹配场再利用"传播+随机搜索"迭代，在每像素约O(1)次操作内收敛到高质量的近似最近邻场（Approximate Nearest Neighbor Field，ANNF）。现代深度学习方法（如2015年后基于卷积神经网络的纹理合成）则使用预训练VGG网络提取特征的Gram矩阵作为纹理损失，通过梯度下降直接优化输出图像像素，能够合成具有长程结构的复杂纹理（如砖墙的规律排列），这是基于块的局部方法难以处理的场景。

## 实际应用

**游戏地形无缝贴图**：在Unreal Engine等引擎的地形系统中，美术师提供一张512×512的砂岩样本，使用Image Quilting生成4096×4096的地形贴图。合成时选取块尺寸约为样本宽度的1/4（即128像素），重叠区为块尺寸的1/6（约20像素），可在数秒内完成高质量合成。

**影视数字双重效果**：在VFX流程中，扫描演员皮肤得到约2cm×2cm的高分辨率皮肤样本，通过纹理合成扩展到覆盖全身的UV贴图，保留毛孔、皱纹等微细结构的统计特性，同时消除接缝。

**建筑材质扩展**：室内设计软件（如Substance 3D）允许用户上传手机拍摄的木纹照片，后台调用基于PatchMatch的合成算法实时生成任意比例的无缝木纹贴图，供家具渲染使用。

## 常见误区

**误区一：认为纹理合成等同于简单的图像拼接平铺**

直接将样本图像重复平铺（tile）不是纹理合成。平铺在样本边界处会产生可见接缝，因为左右、上下边缘的像素颜色往往不连续。真正的纹理合成通过分析样本的局部统计结构来生成新内容，Image Quilting的最优缝算法正是专门为消除这种不连续性而设计的。"无缝平铺"本身只是纹理合成的目标之一，而非方法。

**误区二：认为更大的邻域窗口总会产生更好的结果**

在Efros-Leung算法中，增大邻域窗口会提高对已生成区域的约束，减少随机性，但同时会导致生成像素时在样本中找不到足够相似的匹配（样本覆盖率不足），产生"垃圾"像素或合成失败。Efros的原始论文指出，对于大多数随机纹理，11×11的窗口在质量与鲁棒性之间取得最佳平衡；对于结构性较强的纹理则需要更大窗口（21×21甚至更大），但同时需要更大的样本图像。

**误区三：神经网络方法可以完全替代基于块的方法**

基于VGG Gram矩阵的神经网络纹理合成（Gatys等，2015）在捕捉纹理"风格"方面表现优异，但它优化的是全局统计量而非局部空间结构，因此对于具有明确重复单元的结构性纹理（如砖墙、格子布）容易产生模糊或结构错乱的结果。Image Quilting这类基于块的方法在保留锐利结构细节方面仍具优势，实际生产流程中两类方法针对不同类型纹理各有适用场景。

## 知识关联

纹理合成直接依赖**程序纹理**中建立的纹理坐标系与UV映射概念：合成生成的纹理图最终仍需通过UV坐标映射到三维模型表面，因此理解纹理坐标的参数化方式是正确使用合成结果的前提。同时，程序纹理中噪声函数（如Perlin噪声）产生的随机性概念与Efros-Leung中的随机采样策略在思路上形成对比——前者通过数学函数生成，后者从真实样本的统计分布采样。

在图形学更广泛的技术栈中，纹理合成生成的高分辨率贴图常与MIP映射、法线贴图烘焙等技术配合使用：合成结束后通常需要对输出图像进行频率分析，检查高频细节是否与原样本的功率谱一致，以确保在不同观察距离下的视觉正确性。纹理合成技术还与图像补全（Image Inpainting）高度相关，后者可视为纹理合成的特殊情形：在已有图像内部的缺失区域进行填充，PatchMatch算法同样是当今图像补全（如Photoshop的"内容感知填充"功能）的核心引擎。
