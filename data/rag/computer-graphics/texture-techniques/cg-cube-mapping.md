# 立方体贴图

## 概述

立方体贴图（Cubemap）是一种由六张正方形纹理组成的特殊纹理类型，这六张图像分别对应一个立方体的六个面：正X（+X）、负X（−X）、正Y（+Y）、负Y（−Y）、正Z（+Z）、负Z（−Z）。与普通的2D纹理使用`(u, v)`两个坐标采样不同，立方体贴图使用一个三维方向向量`(x, y, z)`来进行采样——GPU将该向量射出后与虚拟立方体的六个面相交，取交点处的纹素颜色。

立方体贴图最早随OpenGL 1.3（2001年发布）进入图形标准，此前已在专业图形工作站的私有扩展中使用多年。其核心价值在于能够用极低的渲染开销模拟出场景中来自四面八方的环境光照信息，使实时渲染中的天空、反射和折射效果成为可能，而无需在运行时重新渲染整个场景。

立方体贴图对于实时3D应用至关重要，因为替代方案——逐帧重绘周围环境——在复杂场景中代价极高。一张分辨率为512×512的立方体贴图包含六张512×512图像，总计约150万纹素，存储量约等于一张1024×1536的2D纹理，却能编码完整的360°全景环境信息。

## 核心原理

### 方向向量采样机制

立方体贴图采样的关键在于方向向量的分量比较。给定采样向量 `(x, y, z)`，GPU首先找到绝对值最大的分量——称为**主轴分量（Major Axis）**——来确定使用哪一个面。例如，若 `|x|` 最大且 `x > 0`，则采样正X面。随后，另外两个分量被除以主轴分量的绝对值并映射到`[0, 1]`范围，得到该面上的2D纹理坐标。以正X面为例，UV坐标计算为：

```
u = (−z / |x| + 1) / 2
v = (−y / |x| + 1) / 2
```

这种设计使六张图像在立方体面交界处能够完美无缝衔接，前提是原始捕获时摄像机精确对准六个轴向，且视角恰好为90°。

### 天空盒（Skybox）的实现方式

天空盒是立方体贴图最常见的应用，其渲染技巧是绘制一个包围整个场景的立方体，但**始终将其置于摄像机位置**——即每帧绘制前将模型矩阵的位移分量清零，只保留旋转部分。这样无论玩家如何移动，天空永远看起来"无限远"。另一个关键细节是天空盒必须**最后绘制**（或配合深度测试写入技巧），将深度值强制设为1.0（NDC空间中的最远处），避免遮挡场景中的真实物体。现代实现通常用全屏三角形加逆矩阵变换替代实际几何体，进一步减少顶点处理开销。

### 环境反射与菲涅尔效应结合

利用立方体贴图实现镜面反射时，核心公式是：

```
R = 2(N · V) * N − V
```

其中 `N` 是顶点或片段的法线向量，`V` 是从片段指向摄像机的视线向量，`R` 是反射向量。将 `R` 作为方向向量传入立方体贴图采样函数（GLSL中为 `texture(cubemap, R)`），即可得到该方向对应的环境颜色。实际应用中常与**菲涅尔近似**混合——掠射角度时反射更强，正视角度时反射减弱——公式为 `F = F0 + (1 − F0)(1 − dot(N, V))^5`，其中 `F0` 是材质的基础反射率。

### 动态立方体贴图的生成

预烘焙的静态立方体贴图无法反映场景中移动的物体。动态立方体贴图需要在每帧从反射物体位置出发，朝六个轴向各渲染一次场景（共6次Draw Call），将结果写入帧缓冲对象（FBO）绑定的立方体纹理。这一过程使渲染成本乘以6，因此通常只对主要反射物启用，或以较低分辨率（如128×128）执行。

## 实际应用

**天空盒**是游戏引擎中最普遍的用途，Unity和Unreal Engine均原生支持HDR格式的立方体贴图天空盒，存储格式通常为`.hdr`或`.exr`，以16位浮点数保留高动态范围的光照信息，支持基于图像的光照（IBL）计算。

**汽车漆面反射**是立方体贴图在写实渲染中的标志性场景。《极品飞车》《GT赛车》等驾驶类游戏大量使用局部校正立方体贴图（Parallax-Corrected Cubemap），通过为反射探针设置一个有限的包围盒，修正反射向量与实际场景几何的偏差，使车身反射出更符合周围建筑位置的图像，而非完全来自无限远处的方向。

**折射效果**同样依赖立方体贴图，采样向量改为折射向量，使用GLSL内置函数 `refract(I, N, eta)` 计算，其中 `eta` 为折射率之比（如空气到玻璃约为 `1.0/1.52 ≈ 0.658`）。水面、玻璃球等透明物体效果由此实现。

## 常见误区

**误区一：认为六张图像可以是任意视角的照片**
立方体贴图的六张图像必须由同一位置、90°水平与垂直视角、分别朝向六个正交方向拍摄或渲染。若使用不满足这些条件的图像（如全景照片直接裁切），则在面的接缝处会出现明显的错位或颜色不连续。现实中常用等距柱状投影（Equirectangular）全景图通过数学变换转换为立方体贴图六面格式，工具有Blender、`cmft`等。

**误区二：立方体贴图反射可以正确反映邻近物体**
基础立方体贴图采样仅使用方向信息，完全忽略距离。这意味着一面近在咫尺的墙和无限远处的山在反射中看起来一致。这一误区导致初学者对反射效果感到困惑。正确解决方案是使用前述的局部校正立方体贴图，或采用屏幕空间反射（SSR）技术处理近处物体。

**误区三：立方体贴图的纹理坐标可以直接手动指定UV**
由于立方体贴图的采样机制是基于三维方向向量而非二维UV坐标，在GLSL中其Sampler类型为 `samplerCube` 而非 `sampler2D`，使用 `texture(samplerCube, vec3)` 调用。初学者若按2D纹理思路手动指定坐标将导致无法编译或采样错误，因为两者的接口根本不同。

## 知识关联

立方体贴图建立在**纹理映射概述**的基础之上：理解纹素寻址、过滤模式（双线性、三线性）、Mipmap生成等概念，对于立方体贴图同样适用——事实上立方体贴图同样支持Mipmap，且在基于图像的光照中，不同Mip级别被用于编码不同粗糙度下的预卷积环境光。

掌握立方体贴图后，可直接进入**基于图像的光照（IBL）**领域，其核心是将立方体贴图分别进行漫反射预积分（生成辐照度贴图，Irradiance Map）和镜面反射预积分（生成预滤波环境贴图，结合BRDF查找表），从而实现物理正确的全局光照效果，是现代PBR渲染管线不可缺少的组成部分。
