# 立方体贴图

## 概述

立方体贴图（Cubemap）是一种由6张正方形纹理图像组成的特殊纹理类型，这6张图像分别对应一个虚拟立方体的六个面：正X（+X）、负X（−X）、正Y（+Y）、负Y（−Y）、正Z（+Z）、负Z（−Z）。采样时不使用传统的二维UV坐标，而是使用一个从立方体中心出发的三维方向向量，GPU根据该向量的最大分量确定采样哪个面，再将剩余两个分量换算为该面上的二维坐标。这种采样机制使立方体贴图天然适合表达"从某一点向四面八方观察"的全向信息。

立方体贴图最早在1986年由Ned Greene在论文《Environment Mapping and Other Applications of World Projections》中系统提出，作为球面映射的替代方案。与球面贴图相比，立方体贴图在极点处不存在严重的像素拉伸问题，且可以用标准双线性过滤进行平滑采样。OpenGL在1.3版本（2001年）将立方体贴图纳入核心规范（`GL_TEXTURE_CUBE_MAP`），此后它成为实时渲染中最基础的全向纹理形式。

立方体贴图之所以重要，在于它用6张共计N×N×6个像素的图像，以较低的存储代价近似编码了场景中某点的完整球面视野，直接支撑了天空盒渲染、实时环境反射和动态漫反射辐照度等多项核心技术。

## 核心原理

### 方向向量到面坐标的映射

给定一个单位化的方向向量 **r** = (rx, ry, rz)，GPU首先找到绝对值最大的分量，设其为 m。若 |rx| 最大且 rx > 0，则采样+X面；以此类推。随后，另两个分量被归一化到 [0, 1]：

```
u = 0.5 * (sc / |m| + 1.0)
v = 0.5 * (tc / |m| + 1.0)
```

其中 sc、tc 是与该面对应的另两个分量（各面的 sc/tc 映射方向在OpenGL规范附录中有精确定义）。这一步骤完全在硬件中完成，开发者只需向着色器传入方向向量即可。

### 天空盒渲染原理

天空盒利用立方体贴图模拟无限远的天空背景。实现时，以摄像机位置为中心绘制一个单位立方体，顶点着色器中**去除观察矩阵的平移分量**（只保留旋转），使天空盒始终跟随摄像机旋转而不随移动产生视差。片元着色器将顶点的本地坐标直接作为方向向量采样立方体贴图。深度缓冲写入需设为`gl_FragDepth = 1.0`或将天空盒最后渲染，确保所有实体几何体覆盖天空盒像素，避免过度绘制消耗。

### 镜面环境反射

对一个反射型表面，片元着色器用入射视线向量 **v** 和表面法线 **n** 计算反射向量：

```
r = reflect(v, n) = v − 2·dot(v, n)·n
```

用 **r** 采样立方体贴图即得该像素"看到"的环境颜色。该技术称为**环境映射（Environment Mapping）**，单次纹理采样即可近似镜面反射，计算代价远低于光线追踪。代价在于立方体贴图是预烘焙的静态快照，无法反映动态物体，且假设光源无限远（反射向量不随位置偏移）。

### 动态立方体贴图与漫反射辐照度

动态场景中可将摄像机放置于物体中心，朝6个坐标轴方向各用90°视场角渲染一帧，结果写入实时立方体贴图，逐帧或按需更新。此操作每帧需额外6次完整绘制调用，代价高昂，通常仅对场景中最重要的一两个反射对象执行。另一个进阶用途是将高频镜面立方体贴图通过球谐函数（Spherical Harmonics，常用L2阶，9个系数）或卷积积分压缩为低频漫反射辐照度贴图，用于PBR材质的环境漫反射项计算。

## 实际应用

**天空盒**是立方体贴图最直接的应用，几乎所有3D游戏和实时渲染应用都使用它。Unity中将6张图片设置为`Cubemap`资源类型后，直接赋予天空盒材质即可；Unreal Engine的`SkyAtmosphere`系统也可与立方体贴图天空盒结合使用。

**汽车漆面和镜子**等高光材质通过环境映射立方体贴图实现。游戏《极品飞车》系列和大量赛车游戏使用预烘焙立方体贴图为车漆提供廉价但高质量的反射效果。在WebGL应用中，Three.js的`CubeCamera`类封装了动态立方体贴图的6面渲染流程，开发者可直接调用。

**IBL（基于图像的照明）**中，HDR格式的立方体贴图（如`.hdr`全景图转换而来）作为环境光源，经过不同粗糙度的预过滤，存储在立方体贴图的不同Mipmap层级（称为Pre-filtered Environment Map）。UE4的Lumen出现前，这是实时PBR的主流环境光方案。

## 常见误区

**误区一：立方体贴图6张图的接缝会导致可见裂缝**  
早期实现中，相邻面边缘像素采样确实存在接缝问题，但OpenGL 3.2引入了`GL_TEXTURE_CUBE_MAP_SEAMLESS`标志，启用后GPU会在面边界处执行跨面双线性过滤，彻底消除接缝。开发者需要主动调用`glEnable(GL_TEXTURE_CUBE_MAP_SEAMLESS)`，忘记启用是出现接缝的常见原因。

**误区二：采样立方体贴图的向量必须先手动归一化**  
许多教程要求手动`normalize(r)`，但这并非必要——GPU在执行立方体贴图采样时会自动使用向量方向而非长度（确定最大分量后除以|m|），未归一化的向量只要方向正确，采样结果完全相同。归一化不会造成错误，但属于多余操作。

**误区三：立方体贴图比等距柱状投影（Equirectangular）全景图存储更大**  
6面立方体贴图总像素数为 6×N²，而覆盖相同分辨率的等距柱状图分辨率通常为 2N×N，总像素 2N²。两者比值为3:1，立方体贴图反而**存储更多像素**。但等距柱状图在极点处存在大量冗余像素（有效角分辨率不均匀），实际有效信息量与立方体贴图相当，这是两种格式各有使用场景的原因。

## 知识关联

**依赖基础**：立方体贴图建立在二维纹理映射的概念之上——双线性过滤、Mipmap生成、纹理环绕模式等机制在立方体贴图的每个面上独立工作，方式与普通2D纹理完全相同。理解UV坐标如何索引纹理像素，有助于理解立方体贴图的面坐标计算方式。

**横向关联**：立方体贴图与球面谐波（SH）是互补的全向表达工具：立方体贴图保留高频细节（适合镜面反射），SH只能表示低频信息（适合漫反射）。在PBR渲染管线中两者常同时使用，分别处理镜面和漫反射环境光贡献。阴影立方体贴图（Shadow Cubemap）将同样的6面渲染思路用于点光源阴影，是立方体贴图在光照以外的重要扩展应用。
