# 网格压缩

## 概述

网格压缩（Mesh Compression）是将三维多边形网格数据通过编码手段缩减存储体积的技术，核心目标是在尽可能保持几何保真度的前提下降低文件大小与传输带宽。一个典型的未压缩三角形网格需要为每个顶点存储3个浮点坐标（每个12字节），外加每个三角面的3个顶点索引（每个约12字节），一个含10万三角面的模型原始数据量可达数MB级别。网格压缩通过量化、预测编码和熵编码三个层级将这一数据量压缩至原始大小的5%–20%。

该领域的奠基性工作是Deering于1995年在SIGGRAPH发表的"Geometry Compression"论文，首次系统性地将连通性编码与顶点数据编码分离处理。此后Touma和Gotsman于1998年提出了基于顶点度数预测的方案，将连通性编码压缩到接近信息论下界（约1.5–2比特/三角面）。Google在2013年开源的Draco库则将上述理论工程化，成为WebGL与glTF生态的标准压缩方案。

网格压缩的工程意义体现在Web3D渲染、移动端游戏资产传输、医学影像的三维重建等场景中。glTF 2.0规范通过KHR_draco_mesh_compression扩展将Draco压缩纳入标准，一个40MB的未压缩骨骼人体模型经Draco压缩后可降至约3MB，加载延迟显著缩短。

## 核心原理

### 量化（Quantization）

量化是网格压缩的第一步，将浮点坐标映射为有限精度的整数。标准做法是将模型包围盒内的坐标用N位整数均匀量化：

$$v_{int} = \text{round}\left(\frac{v_{float} - v_{min}}{v_{max} - v_{min}} \times (2^N - 1)\right)$$

其中 $v_{min}$、$v_{max}$ 是坐标轴方向的包围盒边界，$N$ 是量化位数。Draco默认使用11位量化（2048级），在保持视觉质量的同时将每坐标分量从32位浮点缩减至11位整数，节省约65%的坐标存储。法线向量可用8位量化（256级），UV坐标通常用12位。位数选择需权衡：8位量化在大型地形网格上会产生可见的阶梯状伪影，而16位量化已能覆盖大多数工程精度需求。

### 连通性编码（Connectivity Encoding）

连通性描述网格的拓扑结构，即哪些顶点构成哪些三角面。朴素存储每个三角面的3个顶点索引在32位索引下需要约12字节/面。

**Edgebreaker算法**（Rossignac，1999）是最广泛使用的连通性压缩方案。它沿生成树遍历三角形，用五种操作符（C、L、E、R、S，每种对应不同的边折叠情形）编码每个三角面的拓扑关系，每个操作符仅需约2–3比特表示。Edgebreaker可将连通性编码压至约1.5比特/三角面，相比朴素方案节省约95%。

Draco的连通性编码基于Edgebreaker的变种，通过预测顶点度数并对预测误差编码，进一步将常见的度数值（如6）的编码开销趋近于0比特。

### 顶点属性预测编码（Attribute Prediction）

量化后的整数坐标仍需进一步压缩。预测编码利用空间相邻顶点坐标的高度相关性，仅存储预测值与真实值的残差（通常远小于绝对值）。

**平行四边形预测**是最经典的几何预测方法：对于已知顶点 $A$、$B$、$C$ 构成的三角形，下一个共享边 $BC$ 的顶点 $D$ 被预测为：

$$\hat{D} = A + B + C - \text{（某参考点）}$$

实际上Draco使用 $\hat{D} = B + C - A$，即平行四边形的第四个顶点。残差 $\Delta D = D - \hat{D}$ 的数值通常集中在零附近，显著降低了后续熵编码的比特数。

### 熵编码（Entropy Coding）

残差序列经过符号化后，使用熵编码进一步消除统计冗余。Draco采用**ANS（Asymmetric Numeral Systems）**编码，相比传统Huffman编码具有更高的编码效率（接近香农熵下界）且解码速度更快（单指令延迟约为Huffman的1/3）。

熵编码阶段的压缩收益取决于残差分布的集中程度：当预测精度高时，残差分布尖锐（大量零值），熵编码可将残差比特率压缩至接近每符号1–2比特；若模型几何不规则（如噪声扫描数据），预测误差分散，熵编码收益下降。

## 实际应用

**Web3D资产传输**：Three.js与Babylon.js均内置Draco解码器支持。一个游戏角色网格含5万三角面，未压缩glTF约8MB，启用Draco后降至约700KB，首屏加载时间从4秒降至0.4秒（在4G网络下）。

**医学三维重建**：CT扫描生成的等值面网格往往含数百万三角面。使用Open3D的网格压缩管线（内部调用Draco），一个脑部分割网格从120MB压缩至9MB，同时以12位量化保留亚毫米级精度，满足临床测量要求。

**点云与扫描网格**：LiDAR扫描生成的网格因噪声较大，平行四边形预测精度下降。此时可调整Draco的`quantization_bits`至14位、关闭法线压缩，以牺牲部分压缩比换取几何保真度。实测中，Matterport室内扫描网格在该配置下压缩比约为4:1。

## 常见误区

**误区一：量化位数越高越安全**。部分开发者习惯性地将所有属性量化位数设为16位，认为"精度足够就不会出问题"。实际上过高的量化位数会降低后续熵编码的效率，因为残差分布变宽导致比特率上升。对于大多数游戏资产，顶点坐标11位、法线8位、UV 12位的组合已是压缩效率与精度的最优平衡点。

**误区二：网格压缩等同于网格简化（LOD）**。网格简化（如Quadric Error Metrics算法）通过减少三角面数量来降低数据量，是有损的几何操作，会改变模型拓扑；网格压缩保留原有的所有顶点与连通性，仅在编码层面减少存储比特数。两者常配合使用，但目的和实现机制完全不同。

**误区三：Draco压缩是无损的**。Draco提供有损模式（默认）和无损模式（`--point_cloud_method=0`并关闭量化），默认有损量化会引入最大约 $\frac{1}{2} \times \frac{v_{max}-v_{min}}{2^N-1}$ 的坐标误差。对于精密工程模型（如CAD零件配合面），必须显式启用无损模式或提高量化位数至16位以上。

## 知识关联

网格压缩以**网格表示**为直接前提：只有理解半边结构、索引三角形列表等存储格式，才能明确量化操作的作用对象（顶点坐标数组与面索引数组的结构差异），以及Edgebreaker为何需要先构建对偶图遍历顺序。顶点属性的预测编码需要访问相邻顶点，这依赖于网格表示中邻接关系的高效查询能力（如半边结构的O(1)相邻面遍历）。

在几何处理的技术路径中，网格压缩通常与**网格简化**配合部署：先通过QEM等简化算法降低面数，再对简化后的网格进行压缩编码，形成完整的资产优化管线。理解网格压缩后，进一步学习**渐进式网格（Progressive Mesh）**可以看到另一条技术路线——Hoppe于1996年提出的渐进式网格将简化序列本身作为压缩表示，支持流式传输时逐步恢复细节，是网格压缩与LOD系统的融合方向。
