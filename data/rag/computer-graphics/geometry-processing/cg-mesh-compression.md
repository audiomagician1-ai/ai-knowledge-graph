# 网格压缩

## 概述

网格压缩（Mesh Compression）是针对三角网格数据的编码技术，通过减少存储顶点坐标、拓扑连接关系所需的比特数来降低文件体积，同时保持几何形状的视觉保真度。一个未压缩的三角网格通常需要为每个顶点存储3个32位浮点坐标（共96位），为每个三角形存储3个32位整数索引（共96位）；而现代压缩算法能将这一开销降低至每个顶点约8–20位、每个三角形约1–2位。

网格压缩领域的奠基性工作出现在1990年代中期。Deering在1995年SIGGRAPH论文中提出了"Geometry Compression"方案，首次系统描述了将量化、预测编码和熵编码组合应用于三角网格的完整流程。此后，Touma和Gotsman在1998年提出了基于顶点遍历的面法表（valence-driven）压缩算法，将连接信息的编码降至约每三角形1.5位，成为后续众多方法的基准。MPEG-4标准中的3DMC（3D Mesh Coding）也在2001年正式将网格压缩纳入国际规范。

网格压缩在游戏资产传输、Web 3D（如glTF的Draco扩展）、医学影像三维模型和地理信息系统（GIS）的地形网格传输中具有直接工程价值。未压缩的高精度扫描模型动辄数百MB，经Draco等工具压缩后可缩减至原体积的5%–15%，大幅减少网络带宽消耗和GPU显存占用。

## 核心原理

### 量化（Quantization）

量化将顶点的浮点坐标映射为有限精度的整数。具体做法是计算网格的包围盒，将每个轴的范围等分为 $2^b$ 个区间，$b$ 为量化位数（通常取10–16位）。编码后的坐标值 $q_i$ 由下式给出：

$$q_i = \left\lfloor \frac{v_i - v_{\min}}{v_{\max} - v_{\min}} \cdot (2^b - 1) + 0.5 \right\rfloor$$

其中 $v_i$ 为原始浮点坐标，$v_{\min}$、$v_{\max}$ 为该轴的包围盒边界。量化引入的最大绝对误差约为包围盒边长的 $\frac{1}{2^{b+1}}$。以10位量化为例，若模型边长为1米，最大误差约为0.5毫米，对多数视觉应用已足够。

### 几何预测编码（Geometry Prediction）

量化后的整数坐标仍存在相邻顶点间的空间相关性。预测编码利用已解码的邻域顶点估计当前顶点位置，只存储**预测残差**（预测值与实际值之差），从而显著压缩残差的动态范围。

常见的预测器包括：
- **Delta预测**：用前一个顶点坐标作为预测，残差 $r_i = q_i - q_{i-1}$，适用于顶点按空间顺序遍历的情形。
- **平行四边形预测**（Parallelogram Predictor）：利用已遍历的相邻三角形的三个已知顶点 $A$、$B$、$C$ 预测第四顶点 $P' = A + C - B$，这是Touma-Gotsman算法中的核心预测器，残差通常只有1–3个量化单位。

### 连接信息压缩（Connectivity Compression）

连接信息描述哪些顶点构成哪些三角形，其编码独立于几何压缩进行。主流方法基于**面法（valence）编码**：以广度优先或深度优先顺序遍历网格的双连通性，用每个顶点的价（valence，即相邻边数）序列来重构拓扑。对于封闭流形三角网格，顶点平均价约为6，利用算术编码或Huffman编码压缩价序列，连接信息可达约每三角形1.0–1.5位。

Google Draco库中的EdgeBreaker算法将每条边的操作类型（C、L、E、R、S五种）编码为2–5位符号，配合上下文自适应算术编码，连接信息压缩至约每三角形1.2位。

### 熵编码（Entropy Coding）

残差序列经算术编码（Arithmetic Coding）或ANS（Asymmetric Numeral Systems）编码进一步压缩至接近信息熵下界。算术编码为残差值 $r$ 分配约 $-\log_2 P(r)$ 位，其中 $P(r)$ 是该残差值在当前上下文中的概率模型（通常为拉普拉斯分布或高斯分布）。Draco中使用的概率模型会在解码过程中动态更新，以适应不同局部几何复杂度的区域。

## 实际应用

**glTF + Draco扩展**：Web端3D内容标准glTF 2.0支持通过Draco扩展压缩网格。一个包含约100,000个三角形的游戏角色模型，原始glTF二进制文件约12MB，启用Draco压缩（量化位数12位）后降至约1.2MB，传输时间在普通4G网络下从约2秒缩短至0.2秒。Three.js、Babylon.js等主流引擎均内置Draco解码器支持。

**医学STL压缩**：CT扫描生成的颅骨三角网格通常含数百万面，STL格式每个三角形占50字节。使用Open3D的网格压缩接口对500万面的STL文件（约240MB）进行16位量化＋EdgeBreaker压缩，可压缩至约18MB，且几何误差在CT分辨率（约0.5mm体素）以内，不影响医学分析。

**游戏引擎LOD与流式加载**：Unreal Engine 5的Nanite虚拟几何系统在底层对簇（cluster）级别的子网格使用量化压缩，以减少显存带宽需求。每个顶点坐标以相对于簇包围盒的局部坐标存储，量化至8–16位，相比全局浮点坐标节省约50%的顶点缓存带宽。

## 常见误区

**误区一：量化位数越高，压缩效果越好**
量化位数增加会直接增大残差的动态范围，使残差序列的熵上升，导致最终压缩后文件反而更大。12位量化的残差熵往往低于16位量化，因为更高精度的坐标分布更分散。实际选择量化位数时，需要在几何精度与压缩率之间根据应用场景做出权衡，而非一味提高精度。

**误区二：连接信息比几何信息更难压缩**
直觉上认为连接信息是纯整数、熵高，实则相反。利用面法编码后，连接信息仅需约1–2位/三角形；而几何坐标即使经过量化和预测，通常仍需5–15位/顶点。对于包含精细细节的高曲率模型，几何编码开销远超连接编码。

**误区三：网格压缩与网格简化等价**
网格压缩是**无损或近无损**的编码过程——在目标量化精度内，解码后的网格拓扑结构与顶点位置完全恢复；而网格简化（如QEM算法）是**有损的几何简化**，会主动减少三角形数量以降低模型复杂度。两者目标、手段和输出均不同，实际管线中常先简化再压缩。

## 知识关联

学习网格压缩需要先掌握**网格表示**中的半边结构（Half-Edge）或索引面列表（Indexed Face Set），因为连接信息压缩的遍历顺序直接依赖对网格邻接关系的遍历能力——EdgeBreaker算法需要在半边结构上进行相邻三角形的广度优先搜索。不了解顶点价（valence）和流形网格的定义，将无法理解面法编码的可行性前提。

量化压缩的误差分析与**采样理论**和**数值精度**有直接关联；熵编码部分则连接到信息论（Shannon熵、算术编码），这些是理解压缩率理论下界的必要背景知识。在几何处理管线中，网格压缩通常是将处理完毕的网格存储或传输的最后一步，其前序操作包括网格简化、参数化和纹理烘焙。
