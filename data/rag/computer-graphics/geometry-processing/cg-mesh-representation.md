# 网格表示

## 概述

网格表示（Mesh Representation）是指用离散的顶点、边和面来近似表达三维几何形体的数据结构方法。最常用的形式是**三角网格**（Triangle Mesh），即将曲面剖分为若干三角形面片的集合。三角形之所以成为首选单元，是因为它是唯一能保证共面性的多边形——任意三个不共线的点必然确定一个唯一平面，这使得三角形面片的法向量计算和光栅化处理都极为稳定。

网格表示的历史可追溯到1960年代计算机图形学的萌芽时期。Ivan Sutherland在1963年的Sketchpad系统中已涉及多边形表示的雏形，而三角网格作为标准格式在1970至1980年代随有限元分析和计算机动画的普及而逐渐确立。1987年Stanford Graphics Lab推广的PLY格式（Polygon File Format）成为早期三角网格文件的重要标准，OBJ、OFF等格式也相继出现并沿用至今。

网格表示在几何处理管线中处于基础地位，因为后续的网格简化、细分曲面、布尔运算、变形等操作全部依赖于网格拓扑和几何信息的组织方式。选择合适的网格数据结构，直接决定了邻接查询的时间复杂度——例如"求顶点的一环邻域"在朴素数组中需要O(F)遍历，而在半边结构中只需O(valence)时间。

## 核心原理

### 三角网格的基本要素

三角网格由三类基本元素构成：
- **顶点**（Vertex）：携带三维坐标 $(x, y, z)$，通常还附带法向量、UV纹理坐标、颜色等属性；
- **边**（Edge）：连接两个顶点的线段，每条内部边恰好被两个三角形面片共享；
- **面**（Face）：三个顶点的有序三元组 $(v_i, v_j, v_k)$，顶点的排列顺序（逆时针或顺时针）决定面法向量的朝向。

对于一个封闭的流形三角网格，顶点数 $V$、边数 $E$、面数 $F$ 满足**欧拉公式**：
$$V - E + F = 2(1 - g)$$
其中 $g$ 为亏格（Genus），即曲面的"洞"的数量。球面 $g=0$，环面 $g=1$。根据三角网格的几何关系还有约束 $E = \frac{3F}{2} + \frac{b}{2}$（$b$ 为边界边数量），这给出了典型三角网格中 $F \approx 2V$、$E \approx 3V$ 的近似关系。

### 半边结构

**半边结构**（Half-Edge Data Structure，也称 DCEL，Doubly Connected Edge List）是几何处理中最常用的三角网格拓扑表示。其核心思想是将每条边拆分为方向相反的两条**半边**（Half-Edge），每条半边记录以下指针：

| 字段 | 含义 |
|------|------|
| `vertex` | 该半边的终点顶点 |
| `twin` | 与之方向相反的配对半边 |
| `next` | 同一面内的下一条半边（逆时针） |
| `prev` | 同一面内的上一条半边 |
| `face` | 所属的三角面片 |

通过以上5个指针，半边结构可以在 $O(1)$ 时间内完成单步拓扑遍历。例如，遍历顶点 $v$ 的所有一环邻居（1-ring neighbor），只需沿 `twin→next` 循环即可，无需全局搜索。这是实现网格简化中的**边折叠**（Edge Collapse）操作的关键基础。

半边结构的内存开销约为每个三角形存储 3 条半边，总半边数为 $2E \approx 6V$，每条半边存 5 个整数指针，实践中一个百万面的网格半边结构约占 50–80 MB。

### 其他邻接表示方式

除半边结构外，还有两类常见的简化表示：

1. **索引面表（Indexed Face Set，IFS）**：最简单的表示，仅存储顶点坐标数组和面索引数组。WebGL/OpenGL的VBO（Vertex Buffer Object）本质上就是IFS，查询效率低但存储紧凑，适合渲染管线而非几何处理。

2. **顶点-面邻接表**：为每个顶点存储其关联的面列表。构建代价为 $O(V + F)$，可在 $O(\text{valence})$ 内求一环面，但无法直接求边-边邻接关系。Libigl等几何处理库使用稀疏矩阵 `VF`（Vertex-Face adjacency）和 `VFi`（局部索引）配合来加速查询。

## 实际应用

**三维扫描后处理**：使用结构光或激光扫描仪获得的点云，经泊松重建或Marching Cubes（1987年由Lorensen和Cline提出）算法转换为三角网格后，需要依靠半边结构提供的拓扑信息来检测并填补扫描孔洞（hole-filling）。

**游戏资产制作**：游戏引擎中角色模型通常使用1万至10万个三角形，以IFS格式存储并上传到GPU。关节绑定（Skinning）时需要快速查询每个顶点所在的面法向量，这依赖顶点-面邻接表而非半边结构，因为蒙皮变形不需要边级别的拓扑操作。

**有限元分析（FEA）**：工程仿真中，网格质量直接影响数值解的收敛性。三角网格中衡量面片质量的常用指标是**纵横比**（Aspect Ratio），定义为最长边与最短高之比，理想等边三角形的纵横比为 $\frac{2}{\sqrt{3}} \approx 1.155$，超过10的面片会导致刚度矩阵病态。

## 常见误区

**误区一：认为四边形网格比三角网格"更精确"**。四边形网格（Quad Mesh）在动画行业和细分曲面中更受欢迎，是因为它对边流（Edge Loop）的控制更直觉，而非精度更高。事实上，四边形面片的四个顶点未必共面，在渲染时仍需被细分为两个三角形。三角网格不存在非平面面片的问题，精度上反而更稳定。

**误区二：半边结构适用于所有网格类型**。半边结构要求网格是**流形**（Manifold），即每条边最多被两个面共享、每个顶点的邻域同胚于圆盘（或半圆盘，若在边界上）。如果输入网格存在非流形边（三个面共享一条边）或非流形顶点（蝴蝶型连接），半边结构的 `twin` 指针将无法唯一定义，导致结构崩溃。此时应先使用网格修复工具处理，或改用更通用的有向图表示。

**误区三：顶点法向量是面片法向量的简单平均**。顶点法向量由周围面法向量加权平均得到，但权重方案有多种，最常见的是以**面片面积**或**对应顶点的内角**作为权重。纯粹按面片数量平均（不加权）在网格密度不均匀时会产生明显的法向量偏差，影响光照效果和后续几何分析。

## 知识关联

学习网格表示需要先理解**几何处理概述**中三维空间坐标系、法向量方向约定以及拓扑与几何的区别——拓扑描述顶点-边-面之间的连接关系（整数索引），几何描述顶点的空间位置（浮点坐标），两者分离存储是所有网格表示的共同原则。

掌握半边结构中的边折叠操作后，可以直接延伸到**网格简化**算法（如QEM, Quadric Error Metrics，1997年Garland和Heckbert提出），该算法在每次迭代中执行一次边折叠，依赖半边结构的 $O(1)$ 局部更新。细分曲面（如Loop细分和Catmull-Clark细分）同样需要在半边结构上遍历一环邻域来插入新顶点和更新权重。**网格布尔运算**和**网格修复**则对流形性检测有严格要求，需要运用欧拉公式和半边结构的合法性检查作为前提条件。**网格变形**（如ARAP变形）需要构建顶点-顶点权重矩阵（cotangent权重 $w_{ij} = \frac{1}{2}(\cot\alpha_{ij} + \cot\beta_{ij})$），其中 $\alpha_{ij}$ 和 $\beta_{ij}$ 是共享边 $e_{ij}$ 对面的两个对角，这需要通过半边结构快速定位对角顶点。
