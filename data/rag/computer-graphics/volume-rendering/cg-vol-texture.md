# 3D纹理

## 概述

3D纹理（Three-Dimensional Texture）是一种将颜色、密度或其他属性数据存储在三维体素网格中的纹理类型，其本质是一个由宽度（W）×高度（H）×深度（D）构成的三维数组，每个体素（Voxel）存储一个或多个标量值。与2D纹理只能映射到表面不同，3D纹理可以在空间中任意位置进行采样，从而支持体积渲染中对云层、烟雾、火焰、医学CT数据等三维场景的真实模拟。

3D纹理的概念随着体积渲染技术的发展而成熟。1988年，Drebin等人在SIGGRAPH上发表的体积渲染论文推动了体积数据在图形学中的应用，而GPU对3D纹理的硬件原生支持最早出现在OpenGL 1.2（1998年）中，通过`GL_TEXTURE_3D`目标实现。DirectX 9引入了`IDirect3DVolumeTexture9`接口，使得实时体积渲染成为游戏引擎的可选功能。

3D纹理在体积渲染中的重要性体现在：它将离散的体积数据预存储在GPU显存中，使得光线步进（Ray Marching）过程中每一步的密度查询可以在硬件三线性插值（Trilinear Interpolation）的支持下高效完成，而不必每帧重新计算或从CPU传输数据。

## 核心原理

### 3D纹理的存储结构

3D纹理在内存中以分层切片的方式线性排列。一个分辨率为128×128×128的3D纹理，每个体素存储一个8位灰度值，总大小为128³ = 2,097,152字节（约2MB）。若存储RGBA四通道32位浮点数，则大小增至128³×16 = 33,554,432字节（约32MB）。GPU对3D纹理的最大尺寸通常有限制，OpenGL规范要求至少支持2048×2048×2048，但实际应用中受显存约束，常用分辨率为64³、128³或256³。

3D纹理的坐标系使用归一化的(s, t, r)三元组，每个分量范围为[0, 1]，分别对应X、Y、Z三个轴向。坐标(s, t, r)对应体素索引的计算公式为：

```
i = floor(s × W),  j = floor(t × H),  k = floor(r × D)
```

其中W、H、D为三个维度的体素分辨率。

### 采样与插值

3D纹理支持三种采样模式：最近邻（Nearest）、三线性（Trilinear）和三线性+Mipmap（Trilinear Mipmap）。三线性插值是体积渲染中最常用的方式，它本质上是在x、y、z三个方向上连续执行线性插值，需要访问8个相邻体素。设目标点的小数坐标分量为(fx, fy, fz)，8个顶点体素值为c₀₀₀到c₁₁₁，则三线性插值结果为：

```
result = c₀₀₀·(1-fx)(1-fy)(1-fz)
       + c₁₀₀·fx(1-fy)(1-fz)
       + c₀₁₀·(1-fx)fy(1-fz)
       + c₁₁₀·fxfy(1-fz)
       + c₀₀₁·(1-fx)(1-fy)fz
       + c₁₀₁·fx(1-fy)fz
       + c₀₁₁·(1-fx)fyfz
       + c₁₁₁·fxfyfz
```

GPU硬件采样单元（TMU）可以在单个纹理指令周期内完成上述8次访问和插值计算，这是3D纹理相比CPU端体积查找的核心性能优势。

3D纹理同样支持Mipmap，但其存储开销远大于2D纹理。完整Mipmap链中，层级L的体素总数为原始的(1/8)^L，所有层级总体素数之和约为原始体素数的1/7×8/7 ≈ 1.143倍（几何级数求和），因此完整Mipmap链仅增加约14.3%的额外存储，与2D纹理的33%相比更为节省。

### 3D纹理的压缩

由于3D纹理的尺寸通常较大，压缩是实际工程中必须考虑的问题。传统2D纹理常用的BC（Block Compression）格式（如BC1/DXT1、BC4/BC5）在大多数现代GPU上不原生支持对3D纹理进行硬件加速解压，这是一个重要限制。常见的应对策略包括：

1. **逐切片BC压缩**：将3D纹理的每个Z切片视为独立2D纹理进行BC4压缩，再重组为3D格式，但会丢失Z方向上的相关性压缩增益。
2. **稀疏纹理（Sparse Textures）**：OpenGL 4.3和DirectX 11.2引入了稀疏纹理（Sparse/Tiled Resources），允许3D纹理仅将实际有数据的体素块（通常为64KB的Tile）驻留在显存，空白区域无需占用显存，适合医学CT数据等稀疏体积场景。
3. **量化降位存储**：将浮点密度值从32位量化为8位（`GL_R8`格式），精度损失通常在体积渲染的可接受范围内，存储减少75%。

## 实际应用

**云与大气渲染**：游戏引擎如Unreal Engine 5使用128×128×128的3D噪声纹理（基于Worley噪声和Perlin噪声的组合）存储云层密度场，在实时天空渲染中以Ray Marching方式步进采样，每步查询一次3D纹理以获取当前位置的云密度值。

**医学影像**：CT扫描数据天然具有3D纹理的形式。一次全身CT扫描的原始数据分辨率可达512×512×1500体素，每个体素存储12位Hounsfield单位值，总大小约为786MB。实际渲染时会降采样至256³以适配显存，使用传递函数将HU值映射为RGBA颜色后上传为`GL_R16`格式的3D纹理。

**流体模拟**：实时流体（如烟雾、火焰）模拟通常在128³的3D纹理中存储速度场（RGB通道）和密度场（A通道），每帧通过计算着色器（Compute Shader）更新体素值，然后在渲染阶段直接采样同一3D纹理进行显示，避免了CPU-GPU数据回传。

## 常见误区

**误区一：3D纹理就是多层2D纹理叠加（Texture Array）**
3D纹理（`GL_TEXTURE_3D`）与2D纹理数组（`GL_TEXTURE_2D_ARRAY`）在采样行为上有根本区别。3D纹理的第三坐标r参与三线性插值，相邻切片之间会自动进行插值混合；而2D纹理数组的层索引是整数，层与层之间不插值，仅用于选择特定切片。在体积渲染中若误用2D纹理数组，光线步进过程中会在切片边界处产生明显的离散跳变伪影。

**误区二：3D纹理分辨率越高越好**
受GPU显存带宽限制，3D纹理的采样速度与体素总数成正比下降。一个256³的单通道3D纹理在光线步进128步时，单帧在1080p分辨率下的纹理采样次数约为1920×1080×128 ≈ 2.65亿次，对显存带宽压力极大。实践中需要平衡分辨率与采样步数，128³配合更精细的步长往往比256³配合粗步长获得更好的性能-质量比。

**误区三：3D纹理可以存储任意尺寸而无需考虑对齐**
GPU要求3D纹理的每个维度为2的幂次（在旧API或特定硬件上），即使现代API允许非2次幂（NPOT）尺寸，非2次幂的3D纹理在某些移动GPU上仍无法生成Mipmap或使用重复（Repeat）寻址模式，且内存对齐不当会导致体素行之间存在Padding字节，读取时出现错位。

## 知识关联

3D纹理建立在体积渲染概述所介绍的体素数据模型之上：体积渲染概述中定义的光学模型（吸收、发射、散射）决定了3D纹理中每个体素需要存储哪些物理量（密度、颜色、散射系数等），而3D纹理的三线性插值采样则是将离散体素数据转化为连续密度场的具体实现机制。理解3D纹理的Mipmap层级结构有助于后续学习光线步进中的自适应步长策略，即在密度变化剧烈的区域（对应高频Mipmap层）使用短步长，在稀疏区域使用长步长，从而在保证渲染质量的同时降低采样次数。
