# 体积云实现

## 概述

体积云实现是通过三维噪声函数构建云层密度场，再结合光线步进（Ray Marching）和大气散射物理模型，在实时或离线渲染管线中生成真实感云朵的技术方案。与基于纹理贴图的传统云层不同，体积云在任意视角和光照条件下均能呈现正确的透射、散射和自遮蔽效果，云朵内部结构也具备真实的三维深度感。

该技术在2015年前后随着游戏《地平线：黎明时分》和育碧《刺客信条》系列的技术博客公开后进入主流实时渲染领域。其核心突破在于将Perlin Noise与Worley Noise（又称细胞噪声）组合叠加，以模拟积云的宏观轮廓与内部翻涌细节，从而将原先需要离线渲染数小时的云层效果压缩至单帧数毫秒级别的GPU计算。

在游戏、影视特效和气象模拟三个领域，体积云实现的质量直接决定了天空场景的可信度。相较于粒子云或公告牌云，体积云能够正确响应昼夜光照方向变化，并产生云层内部的光晕（Silver Lining）和体积阴影，是当前AAA级实时渲染的标准解决方案。

## 核心原理

### 密度场构建：噪声函数的分层叠加

云层密度场 `D(x,y,z)` 通常由低频形状噪声和高频细节噪声两层叠加构成，公式为：

```
D(p) = remap(fbm_low(p), 1.0 - cloud_coverage, 1.0, 0.0, 1.0)
     × remap(fbm_high(p), 0.0, 1.0, D_base, 1.0)
```

其中 `remap(x, in_min, in_max, out_min, out_max)` 是将值域线性重映射的工具函数；`fbm_low` 使用频率约为4~8倍频的Perlin-Worley噪声定义云团轮廓，`fbm_high` 使用更高频的Worley噪声（3~4倍频）侵蚀云团边缘，制造卷积云边缘的细碎感。`cloud_coverage` 参数在0到1之间控制全局云量，通过remap将噪声底部截断以产生明显的云-天空边界。

### 光照：单次散射与Henyey-Greenstein相位函数

云层的光照分为透射衰减和散射增益两部分。从着色点 `p` 到太阳方向发射一条光线，沿途累积密度得到透射率 `T`：

```
T = exp(-σ_ext × ∫ D(s) ds)
```

其中消光系数 `σ_ext` 对于液态水云粒约为 `0.04 ~ 0.1 m⁻¹`（标准化后在归一化坐标中通常取0.05）。

散射量使用Henyey-Greenstein（HG）相位函数描述光在云粒子上的角度分布：

```
p_HG(θ, g) = (1 - g²) / (4π × (1 + g² - 2g·cosθ)^(3/2))
```

其中不对称因子 `g` 取值范围 `-1 ~ 1`，对于云层通常取 `g = 0.6 ~ 0.85`，使前向散射明显强于后向散射，从而重现云边缘强烈的"光晕"现象。实际实现常采用两个HG函数的线性混合（双瓣HG）以同时模拟前向和后向散射。

### 性能优化：分辨率降采样与天气纹理遮罩

全分辨率的体积云Ray Marching代价极高，实际工程实现通常以1/4甚至1/8分辨率进行云层积分，然后用时域重投影（Temporal Reprojection）将4帧或8帧样本累积至满分辨率。此策略需要为每个采样点存储历史帧的深度和颜色信息，并用运动向量进行偏移矫正以防止残影。

步进间距采用自适应策略：在密度接近0的空区（`D < 0.001`）使用大步长（约8~16步/大气层厚度），一旦检测到非零密度则切换至小步长（约64~128步）进行精细积分。这种空区跳过（Empty Space Skipping）可减少约60%的无效采样。

## 实际应用

**《地平线：黎明时分》的层化云系统**：Guerrilla Games在GDC 2015分享的方案将云层划分为低云层（距地1.5~5 km）和卷云层（8~12 km）两层，低云使用完整的密度场+光照积分，卷云则用单张二维扰动纹理近似，以节省GPU预算。最终在PS4上单帧云层渲染预算控制在2 ms以内。

**Unity HDRP的体积云模块**：在Shader中以世界坐标对一张128³分辨率的3D噪声纹理（预烘焙的fbm_low结果）进行采样，再在运行时叠加从CPU传入的实时风场偏移量`wind_offset = wind_speed × time`，实现云层随风飘移的动态效果，避免了每帧重新计算噪声的开销。

**气象可视化场景**：将WRF（Weather Research and Forecasting）数值模拟输出的含水量数据直接映射为密度场，替换程序化噪声，可生成与真实天气系统一一对应的体积云，用于飞行模拟器和气象教学软件。此时 `D(x,y,z)` 直接来自气象数据插值而非噪声函数。

## 常见误区

**误区一：认为只需一种噪声即可生成真实云层**
仅使用Perlin Noise生成的密度场会产生大量均匀鼓包，缺乏真实积云的"花椰菜"质感。这是因为真实积云边缘的锯齿形态由对流上升气流的湍流侵蚀形成，在噪声模拟中必须用Worley Noise的细胞边界特性进行额外的边缘侵蚀（erosion）。跳过高频侵蚀步骤是新手实现中最常见的视觉缺陷来源。

**误区二：光照步进与视线步进使用相同步长**
云层光照计算中，从着色点向太阳方向做的次级光线（shadow ray）只需采样5~6个等距样本即可获得足够精度的透射率估算，因为该步骤对精度的敏感性远低于视线积分。若将次级光线步长设置为与主光线相同的细粒度，GPU负载会成倍增加，却不会带来可见的画质提升。

**误区三：忽略云层底部的环境光遮蔽效果**
真实积云底部呈现暗灰色，并非因为底部密度更高，而是由于云层内部的多次散射使光子向下传播时大量损耗。简单地只计算单次散射会导致云底与顶部一样明亮。实用近似方案是：在光照积分中以当前点在云层内的归一化高度 `h ∈ [0,1]` 乘以一个底部衰减因子 `pow(h, 0.3)`，模拟多次散射导致的底部变暗效果，无需实际追踪多次散射路径。

## 知识关联

**前置知识衔接**：体积云实现直接建立在Ray Marching的基础框架之上——云层渲染正是Ray Marching在半透明介质中的具体应用，视线积分公式 `C = ∫ σ_s · L_in · T · p(θ) dt` 中的每一项在Ray Marching中均已有明确对应的数值积分实现。掌握Ray Marching中的步进策略（固定步长、自适应步长）是理解云层性能优化的直接前提。

**横向关联技术**：体积云使用的HG相位函数同样适用于体积雾、皮肤次表面散射的近似以及海水散射建模，这些场景的着色差异仅在于密度场的来源和 `g` 参数的取值。体积云密度场构建中的噪声分层思路（低频形状+高频细节）与地形高度图生成、程序化纹理合成使用的技术框架完全一致，学习体积云实现后这些相关技术可快速迁移理解。
