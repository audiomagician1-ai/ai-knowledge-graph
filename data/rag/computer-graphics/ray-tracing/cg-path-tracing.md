# 路径追踪

## 概述

路径追踪（Path Tracing）是一种基于蒙特卡洛积分的全局光照算法，由James Kajiya于1986年在论文《The Rendering Equation》中首次提出，同年与渲染方程一同发表。其核心思想是从摄像机向场景发射光线，在光线与表面交点处随机采样反射/折射方向，持续递归追踪直至击中光源或超过最大弹跳次数，通过大量路径样本的平均值来估算到达像素的辐射亮度。

与早期的Whitted风格光线追踪不同，路径追踪能够自然处理漫反射间接照明、色彩渗透（Color Bleeding）、焦散等复杂光传输现象，无需对每种效果单独编码。代价是收敛速度慢——在单个像素只有少量样本时，图像呈现明显的白色噪点，这是蒙特卡洛方法方差的直接体现。现代离线渲染器如Pixar的RenderMan、Autodesk Arnold以及Blender Cycles均以路径追踪为基础。

## 核心原理

### 渲染方程与蒙特卡洛估算

路径追踪所求解的对象是Kajiya渲染方程：

$$L_o(\mathbf{x}, \omega_o) = L_e(\mathbf{x}, \omega_o) + \int_{\Omega} f_r(\mathbf{x}, \omega_i, \omega_o)\, L_i(\mathbf{x}, \omega_i)\, (\omega_i \cdot \mathbf{n})\, d\omega_i$$

其中 $L_o$ 为出射辐射亮度，$L_e$ 为自发光项，$f_r$ 为双向反射分布函数（BRDF），$L_i$ 为入射辐射亮度，$\omega_i \cdot \mathbf{n}$ 为余弦衰减项，积分域 $\Omega$ 为法线半球。由于 $L_i$ 本身依赖另一个表面的出射亮度，该方程是递归的，无法解析求解，因此使用蒙特卡洛估算：

$$\langle L_o \rangle = \frac{1}{N} \sum_{k=1}^{N} \frac{f_r(\omega_{i,k}, \omega_o)\, L_i(\omega_{i,k})\, \cos\theta_{i,k}}{p(\omega_{i,k})}$$

其中 $p(\omega_{i,k})$ 是采样方向的概率密度函数，$N$ 是每像素样本数（SPP）。估算值的标准误差以 $1/\sqrt{N}$ 的速率下降，因此要将噪声减半需要将样本数翻四倍。

### 俄罗斯轮盘赌终止策略

路径追踪的递归不能无限继续。简单截断（如最大弹跳5次）会系统性地丢失多次反射的能量，造成能量不守恒。标准解法是**俄罗斯轮盘赌（Russian Roulette）**：在每次弹跳时，以概率 $q$（通常取表面反射率的最大分量）继续追踪，并将路径贡献除以 $q$ 进行无偏补偿：

$$\text{如果 } \xi > q: \text{终止路径，贡献为0}$$
$$\text{如果 } \xi \leq q: \text{继续，贡献 } \times \frac{1}{q}$$

其中 $\xi \in [0,1]$ 是均匀随机数。这一策略保证了估算在数学期望意义下是无偏的，同时避免了无限递归。

### 直接光照与下一事件估计

朴素路径追踪（Naive Path Tracing）极少有路径能自然击中光源，导致高方差。**下一事件估计（Next Event Estimation, NEE）** 是标准优化手段：在每个着色点直接对光源面积采样，显式计算直接光照贡献，同时继续追踪间接光照路径。这使得即便是面积很小的光源也能被有效采样。使用NEE时需注意避免双重计数：若随机追踪的路径恰好也击中了光源，应将该光源的直接发光项设为零（或对两者做多重重要性采样加权）。

## 实际应用

**电影视觉特效渲染**：Pixar《寻梦环游记》（2017）中大量彩色灯笼产生的色彩渗透效果，正是路径追踪多次漫反射弹跳的结果。传统光栅化方法需要艺术家手动添加假光源来模拟此类间接光，而路径追踪自动生成这些效果。

**实时路径追踪**：NVIDIA于2018年发布的RTX 20系列显卡引入硬件光线追踪单元（RT Core），配合降噪算法（如DLSS），使每帧仅需1-4 SPP的路径追踪结果经神经网络重建后达到可接受的实时画质。游戏《控制》（Control）中的全局光照即采用这一架构。

**离线渲染基准**：Arnold渲染器中，常规生产渲染通常使用4到16 SPP配合降噪，而高质量参考图像需要4096 SPP以上。渲染方程中余弦项对极浅角度的压制意味着路径追踪在处理掠射角高光时需要更多样本。

## 常见误区

**误区一：更多弹跳次数总是更准确**。增加最大弹跳次数（如从5增至20）在大多数室内场景中收益极小，因为经过5次漫反射后路径贡献已衰减至近乎忽略不计。不加俄罗斯轮盘赌地大幅提升弹跳限制只会浪费计算资源，而非提高精度。真正需要多弹跳的场景是高反射率密闭空间（如镜面包围盒内部）。

**误区二：路径追踪等价于光线追踪**。光线追踪（Ray Tracing）是一种通用的几何求交技术，可用于阴影测试、反射、折射等，但不一定求解渲染方程。路径追踪是在光线追踪求交能力之上，通过蒙特卡洛积分**完整求解渲染方程**的特定算法。Whitted-style光线追踪只追踪镜面反射和折射，无法处理漫反射间接光，不是路径追踪。

**误区三：噪点是算法错误的表现**。路径追踪的噪点是蒙特卡洛方差的正常体现，不是Bug。对于无偏路径追踪器，在样本数趋于无穷时图像必然收敛到正确解。噪点分布均匀（白噪声）本身就是估算无偏性的证明；若出现系统性亮斑或暗斑，才说明实现存在偏差（Bias）。

## 知识关联

**前置概念**：路径追踪建立在光线-场景求交（BVH加速结构、Möller–Trumbore算法）和BRDF模型之上，需要理解辐射度量学中辐射亮度（Radiance）与辐照度（Irradiance）的区别才能正确解读渲染方程各项含义。

**重要性采样**：朴素路径追踪对 $f_r \cdot \cos\theta$ 乘积分布均匀采样，方差高。重要性采样根据被积函数的形状设计非均匀采样分布，使 $p(\omega_i)$ 接近 $f_r \cos\theta$，从而大幅降低方差——这是路径追踪最关键的加速手段，多重重要性采样（MIS）更结合了BRDF采样与光源采样两种策略。

**双向路径追踪（BDPT）**：针对路径追踪在焦散和光源被遮挡场景中收敛缓慢的问题，BDPT同时从摄像机和光源出发构建子路径并在中间连接，极大降低了光传输难以采样情形下的方差。光子映射则采用两步离线算法从另一角度解决同类问题，在处理折射焦散时效率优于路径追踪。
