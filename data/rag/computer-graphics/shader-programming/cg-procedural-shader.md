# 程序化着色器

## 概述

程序化着色器（Procedural Shader）是指不依赖预先存储的纹理图像，而是通过数学函数在GPU上实时计算颜色、法线、粗糙度等表面属性的着色器程序。与传统纹理映射不同，程序化着色器的输出完全由输入坐标和数学算法决定，因此可以在任意分辨率下无损渲染，也不需要占用纹理内存带宽。

程序化纹理的概念可追溯至1985年Ken Perlin在电影《TRON》特效制作中发明的Perlin噪声算法。他因此获得了1997年的奥斯卡技术成就奖。这一发明使得石头、云雾、木纹等自然表面可以完全通过数学描述，此后几十年间，程序化着色器逐渐成为电影特效与游戏渲染领域不可或缺的技术。

程序化着色器最显著的优势在于其无限细节特性：当摄像机无限接近表面时，传统纹理会出现像素化模糊，而程序化着色器可以持续提供任意细节层次。Blender的Cycles渲染器、Unreal Engine的材质编辑器以及Substance Designer工具链均以程序化节点为基础构建复杂材质。

## 核心原理

### Perlin噪声：梯度噪声基础

Perlin噪声（1985）的核心思想是在整数格点上预先存储随机梯度向量，对于任意查询点 **P**，先找到其周围的格点，计算 **P** 到每个格点的偏移向量与该格点梯度向量的点积，再用平滑插值函数混合结果。

平滑插值使用五次多项式 **fade(t) = 6t⁵ − 15t⁴ + 10t³**，相比线性插值，此函数在端点处一阶和二阶导数均为零，消除了格点边界上的视觉接缝。Perlin噪声的频率可以堆叠：将不同频率和振幅的噪声叠加称为**分形布朗运动（fBm）**，公式为：

> fBm(P) = Σ aⁿ · noise(fⁿ · P)，n从0到层数N

其中 **a** 为振幅衰减系数（通常取0.5），**f** 为频率倍增系数（通常取2.0）。叠加6到8层即可产生逼真的云层或山地地形纹理。

### Simplex噪声：降低计算复杂度

Ken Perlin于2001年提出Simplex噪声以改进原始算法。在2D空间中，Perlin噪声对4个格点进行插值，而Simplex噪声将空间划分为三角形（单纯形），每次只需计算3个顶点，减少了约40%的乘法运算量。在3D情况下，这一优势更为明显：传统格点噪声需要插值8个顶点，Simplex噪声只需4个。Simplex噪声在各方向上的统计特性也更为各向同性，不会出现Perlin噪声中沿坐标轴方向的轻微条纹伪影。

### Worley噪声：细胞纹理

Worley噪声（又称细胞噪声或Voronoi噪声）由Steven Worley于1996年提出。算法核心是：在空间中随机分布若干特征点，对于查询点 **P**，计算其到最近特征点的距离 **F1**、第二近距离 **F2**，以及二者的组合 **F2 − F1**。

- 使用 **F1**：生成类似生物细胞壁、鹅卵石地面的图案
- 使用 **F2 − F1**：生成清晰的细胞边界线，可模拟皮肤裂纹、干涸泥地
- 使用 **F1 / F2**：产生金属腐蚀斑纹效果

为避免逐点暴力搜索，实现时将空间划分为固定大小的格子，每格内随机放置一个特征点，查询时只需检查周围3×3（2D）或3×3×3（3D）的邻格，将时间复杂度从O(N)降低至O(1)。

### 程序化着色器的坐标系选择

程序化着色器通常在三种坐标空间中计算噪声：
1. **对象空间（Object Space）**：纹理随物体变形，适合木纹、大理石等"内部"材质
2. **世界空间（World Space）**：物体移动时纹理保持固定，适合地面苔藓等"外部附着"效果
3. **UV空间**：依赖传统UV展开，丧失程序化的无缝优势，通常不推荐单独使用

## 实际应用

**大理石纹理**是Perlin噪声最经典的应用案例。核心思路是计算 `sin(P.x * frequency + fBm(P) * turbulence)`，用正弦函数的条纹与噪声扰动叠加产生自然弯曲的矿脉纹路。调整`turbulence`参数（通常范围2.0到8.0）可从细腻到粗犷的大理石质感之间切换。

**云层与体积雾**利用3D fBm噪声配合密度阈值实现：对世界坐标查询噪声值，超过某阈值则判定为云，低于阈值则透明。Unreal Engine的体积云系统底层正是使用多层Perlin-Worley混合噪声（Perlin-Worley Noise），兼顾大尺度形状（Worley）与小尺度蓬松细节（Perlin）。

**程序化地形着色**可以根据高度和坡度混合不同材质：低平处用Worley噪声生成草地细节，陡坡用fBm生成岩石裂缝，雪线以上用白色加轻微Perlin扰动。整个材质不依赖任何贴图文件即可在GPU上完成。

在GLSL中，一个最简单的2D Perlin噪声着色器通常只需在片元着色器中调用 `noise(fragCoord * scale)`，并将输出映射到 `[0,1]` 范围后赋给 `fragColor`，代码量不超过20行。

## 常见误区

**误区一：程序化噪声完全随机，每次结果不同**
Perlin噪声和Simplex噪声是**确定性函数**：相同的输入坐标永远返回相同的输出值。"随机"体现在空间分布的不规则外观上，而非计算过程的随机性。只有使用时变参数（如加入时间 `t` 作为第三维坐标）才会产生动态变化效果。

**误区二：程序化着色器一定比纹理采样快**
实际上，多层fBm（如8层叠加）在GPU上的计算开销可能高于一次标准纹理采样指令。纹理采样有专用硬件单元（TMU）加速，而复杂的程序化噪声消耗大量ALU（算术逻辑单元）周期。在移动平台上，超过4层的fBm可能导致明显的性能下降。

**误区三：Worley噪声就是Voronoi图**
Worley噪声与Voronoi图虽然共享特征点概念，但Voronoi图是离散几何结构（描述哪个区域归属哪个特征点），而Worley噪声是连续标量场（输出的是浮点距离值）。在着色器中使用的是距离值本身，而非区域归属关系，这使得Worley噪声可以产生平滑渐变而非硬边界分区。

## 知识关联

程序化着色器直接建立在**片元着色器**的基础上：噪声计算发生在片元着色阶段，以片元的世界坐标或UV坐标作为噪声函数的输入。理解片元着色器如何接收插值后的顶点属性（如 `fragCoord`、`vUV`）是实现程序化纹理的前提。

Perlin噪声的梯度向量哈希表与GLSL中的 `hash()` 函数实现密切相关，理解浮点精度限制有助于避免在大坐标值（如世界坐标超过1000单位）时出现的周期性重复伪影——这是程序化着色器在开放世界游戏中最常见的实践陷阱之一。

Worley噪声的细胞结构可以与**法线贴图生成**结合：对Worley距离场求梯度即可得到法线方向，实现完全程序化的凹凸表面，无需任何外部资产。这一技术在Substance Designer的"Cell"节点中有标准实现。
