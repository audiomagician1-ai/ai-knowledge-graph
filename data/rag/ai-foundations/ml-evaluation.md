# 模型评估指标

## 概述

模型评估指标（Model Evaluation Metrics）是用于量化机器学习模型预测能力的数学标准，不同的指标从不同角度衡量模型输出与真实标签之间的差距或吻合程度。选择错误的评估指标会导致模型在实际部署后表现远低于预期，例如在信用卡欺诈检测任务中若仅使用准确率（Accuracy），由于欺诈样本仅占总交易量的0.1%左右，一个永远预测"非欺诈"的模型也能获得99.9%的准确率，却对业务毫无价值。

评估指标的系统化研究随着统计学习理论的发展而完善。ROC曲线（Receiver Operating Characteristic Curve）最初由二战期间美国军方雷达工程师于1941年用于分析雷达信号检测质量，后被引入机器学习领域。BLEU分数（Bilingual Evaluation Understudy）由IBM研究院的Papineni等人于2002年提出，专门用于机器翻译质量评估。这些指标的出现时间节点揭示了评估体系与具体应用场景密不可分的演化路径。

## 核心原理

### 分类任务中的混淆矩阵与派生指标

混淆矩阵（Confusion Matrix）是分类评估的基础数据结构，包含四个关键数值：真正例（TP）、假正例（FP）、真负例（TN）、假负例（FN）。从这四个数值可以推导出以下关键指标：

- **精确率（Precision）**：`P = TP / (TP + FP)`，衡量预测为正的样本中真正为正的比例
- **召回率（Recall）**：`R = TP / (TP + FN)`，衡量所有真正为正的样本中被正确识别的比例
- **F1分数**：`F1 = 2 × P × R / (P + R)`，精确率与召回率的调和平均，当两者严重失衡时F1会被较低的一方主导

更广义的Fβ分数公式为：`Fβ = (1 + β²) × P × R / (β² × P + R)`，当β > 1时更重视召回率，当β < 1时更重视精确率。在癌症诊断场景中通常令β = 2以降低漏诊率。

### ROC曲线与AUC值

ROC曲线以假正例率（FPR = FP / (FP + TN)）为横轴，真正例率（TPR，即召回率）为纵轴，通过改变分类阈值从0到1绘制出一条曲线。AUC（Area Under Curve）是该曲线下的面积，取值范围为[0, 1]。AUC = 0.5表示模型等同于随机猜测，AUC = 1.0表示模型完美分类，实践中AUC > 0.85通常被认为具备较强的判别能力。

AUC有一个重要的概率解释：**随机抽取一个正样本和一个负样本，模型对正样本评分高于负样本的概率等于AUC值**。这一性质使AUC在类别不平衡场景下比准确率更具参考价值。PR曲线（Precision-Recall曲线）在正负样本高度不平衡时（正样本比例低于5%）比ROC曲线更能敏感地反映模型性能变化。

### 回归任务的评估指标

回归任务的指标衡量预测值 $\hat{y}$ 与真实值 $y$ 的数值偏差：

- **均方误差（MSE）**：`MSE = (1/n) × Σ(yᵢ - ŷᵢ)²`，对大误差惩罚更重，适合当大偏差代价高时使用
- **平均绝对误差（MAE）**：`MAE = (1/n) × Σ|yᵢ - ŷᵢ|`，对异常值更鲁棒，误差单位与原始数据相同
- **均方根误差（RMSE）**：`RMSE = √MSE`，与原始数据同单位，常用于报告误差大小
- **决定系数（R²）**：`R² = 1 - SS_res / SS_tot`，其中SS_res为残差平方和，SS_tot为总平方和；R² = 1表示完美拟合，R² = 0表示模型等效于预测均值，R²可能为负值（模型比均值预测更差）

### 排序与生成任务的专用指标

信息检索任务使用**平均精确率均值（MAP，Mean Average Precision）**：首先对单个查询计算AP（Average Precision），即在每个召回节点上精确率的平均值，再对所有查询取均值。NDCG（Normalized Discounted Cumulative Gain）还引入了位置折扣，排名第k位的增益折扣系数为`1/log₂(k+1)`，更好地反映搜索结果排序质量。

生成任务（如文本摘要、机器翻译）使用BLEU分数，其核心是n-gram精确率的几何平均乘以一个长度惩罚因子（Brevity Penalty），防止模型通过输出极短文本来刷高n-gram精确率。

## 实际应用

**医疗诊断系统**：某COVID-19快速检测模型的评估报告显示Accuracy = 95%，但若阳性样本仅占测试集的8%，Recall仅为60%意味着40%的感染者被漏检。此时应优先优化F2分数而非准确率，将决策阈值从默认的0.5下调至0.3可使Recall提升至82%，而Precision从78%降至61%，此权衡在公共卫生场景下是可接受的。

**推荐系统A/B测试**：电商平台在评估商品推荐模型时，通常同时报告HR@10（Hit Rate at 10，即真实购买商品出现在推荐列表前10位的比例）和NDCG@10，因为前者忽略了推荐位置的重要性，后者对排在第1位的命中奖励是排在第10位的`log₂(11)/log₂(2) ≈ 3.46`倍。

**回归模型选择**：在房价预测任务中，若训练数据包含少量豪宅异常值，MSE优化会导致模型对高端市场过拟合。改用MAE作为训练目标和评估指标，可以使普通住宅价格预测的中位数绝对误差降低约15%，同时模型对2%-3%的异常价格数据不再敏感。

## 常见误区

**误区一：用准确率评估不平衡数据集**

当某类别样本占比超过90%时，准确率失去判别价值。正确做法是至少同时报告每个类别的Precision、Recall以及宏平均F1（Macro-F1）和加权平均F1（Weighted-F1）。宏平均对每个类别一视同仁，加权平均按样本数量加权，两者之间的差距反映了模型在少数类上的性能短板。

**误区二：混淆AUC-ROC与AUC-PR的适用场景**

AUC-ROC在正负样本极不平衡时（如欺诈检测，欺诈率0.5%）会显示虚高的数值（可能达到0.95），因为TN基数庞大使FPR始终极小。在这种场景下AUC-PR更为严格，一个AUC-ROC = 0.95的模型AUC-PR可能仅为0.30，两者都应报告以呈现完整图景。

**误区三：将测试集指标等同于泛化性能**

若在模型调参过程中反复使用同一测试集来选择超参数，会导致测试集指标虚高，这一现象称为"测试集污染"（Test Set Leakage）。标准实践是严格区分验证集（用于调参）和测试集（仅最终报告一次），或使用嵌套交叉验证（Nested Cross-Validation）来同时完成模型选择和泛化误差估计。

## 知识关联

理解模型评估指标需要熟悉**机器学习基础**中的监督学习框架——只有明确区分了分类任务、回归任务和排序任务，才能针对性地选择Precision/Recall族指标、MSE/MAE指标或NDCG族指标。混淆矩阵的正确解读依赖对数据集划分（训练集/验证集/测试集）和标签定义的理解。

评估指标与**过拟合与正则化**密切联动：过拟合现象的定义本身就是"训练集指标远优于验证集/测试集指标"。训练曲线（Training Curve）通过对比训练损失（如交叉熵）和验证集F1/AUC随轮次变化来诊断过拟合，L1/L2正则化参数的调整效果也必须通过验证集上的评估指标变化来判断是否有效。选择合适的评估指标是构建学习曲线诊断框架的前提，也是后续进行超参数搜索和模型对比的量化依据。
