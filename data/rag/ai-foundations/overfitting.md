# 过拟合与正则化

## 概述

过拟合（Overfitting）是指机器学习模型在训练集上表现出色，但在未见过的测试集或真实数据上性能显著下降的现象。其本质原因是模型学习了训练数据中的噪声和随机波动，而非数据背后的真实规律。一个经典的判断标准是：训练集误差与验证集误差之间的差距超过可接受阈值，即偏差-方差分解中方差项过大。

这一问题在统计学习领域被研究已久。Vapnik 和 Chervonenkis 于1971年提出 VC 维理论，为量化模型复杂度与泛化误差之间的关系奠定了理论基础。正则化（Regularization）技术的系统化应用可追溯至 Tikhonov 于1943年提出的 Tikhonov 正则化（即 Ridge 回归的数学前身），其核心思想是在损失函数中添加惩罚项，主动压制模型复杂度。

过拟合与正则化之所以关键，是因为所有有监督学习任务的最终目标是泛化能力，而非训练精度。一个在 CIFAR-10 数据集上训练准确率达99%但测试准确率仅75%的模型毫无实用价值。正则化是在不增加训练数据的前提下，以数学约束方式提升泛化的核心手段。

---

## 核心原理

### 偏差-方差权衡（Bias-Variance Tradeoff）

泛化误差可以分解为三个部分：

$$\text{E}[(y - \hat{f}(x))^2] = \text{Bias}^2[\hat{f}(x)] + \text{Var}[\hat{f}(x)] + \sigma^2$$

其中 $\text{Bias}^2$ 表示模型假设空间与真实函数的系统性偏差，$\text{Var}$ 表示模型对训练数据微小变化的敏感程度，$\sigma^2$ 是不可消除的噪声。过拟合对应高方差低偏差：模型对每份不同的训练数据都会拟合出截然不同的参数，但平均预测值接近真实值。正则化的数学本质就是主动引入少量偏差，以换取方差的大幅降低。

### L1 正则化与 L2 正则化

**L2 正则化（Ridge）** 在损失函数中添加参数的平方和惩罚：

$$L_{\text{Ridge}} = L_{\text{原始}} + \lambda \sum_{j=1}^{p} w_j^2$$

L2 正则化使权重向零收缩但不会恰好变为零，适合参数贡献较均匀的场景。**L1 正则化（Lasso）** 添加参数的绝对值之和：

$$L_{\text{Lasso}} = L_{\text{原始}} + \lambda \sum_{j=1}^{p} |w_j|$$

L1 的几何特性（其可行域为菱形，顶点在坐标轴上）使得最优解倾向于落在坐标轴顶点处，从而产生稀疏解——大量权重恰好为0，这等价于自动特征选择。超参数 $\lambda$ 控制正则化强度：$\lambda=0$ 时退化为无正则化，$\lambda \to \infty$ 时所有权重趋于零。ElasticNet 将两者线性组合，适用于特征间存在强相关性的情形。

### Dropout 正则化

Dropout 是专为神经网络设计的正则化技术，由 Srivastava 等人于2014年在论文《Dropout: A Simple Way to Prevent Neural Networks from Overfitting》中提出。训练时，每个神经元以概率 $p$（通常设为0.5）被随机置零，迫使网络不能依赖任何单一神经元。

Dropout 可被理解为在指数量级（$2^n$，$n$ 为神经元数）的子网络上进行集成学习。推理时不进行随机丢弃，但需要将权重乘以 $(1-p)$ 以保持输出期望值一致，或采用等价的 Inverted Dropout（训练时将激活除以 $(1-p)$）。实验表明，在 MNIST 上使用 Dropout 可将测试错误率从1.6%降至1.25%。

### 早停法（Early Stopping）

早停法通过监控验证集损失来决定何时终止训练。当验证集损失连续若干个 epoch（patience 参数，通常设为5至20）不再下降时，停止训练并回滚到验证损失最低时的参数。其原理是神经网络在训练初期以学习真实模式为主，训练后期开始记忆噪声，验证损失的上升拐点正是过拟合开始的标志。早停法等效于对参数范数施加软约束。

## 实际应用

**图像分类中的 L2 正则化**：ResNet-50 在 ImageNet 上训练时标准配置使用 $\lambda = 10^{-4}$ 的 L2 正则化（weight decay）。若不使用，模型在 1.28M 训练图像上仍会出现过拟合，Top-1 准确率可能下降2至3个百分点。

**NLP 中的 Dropout**：BERT 模型在每个 Transformer 层的注意力权重和前馈层输出后均施加 $p=0.1$ 的 Dropout。若去除 Dropout，在下游任务（如 SQuAD 问答）的 F1 分数通常下降约1至2分。

**决策树的正则化**：决策树通过限制最大深度（max_depth）、最小叶节点样本数（min_samples_leaf）以及最大叶节点数来实现正则化，本质是约束 VC 维。未剪枝的决策树在小数据集上可轻易达到100%训练准确率但测试准确率极低。

**超参数选择**：$\lambda$ 应通过 k 折交叉验证（通常 k=5 或 k=10）在对数尺度上搜索（如 $10^{-5}, 10^{-4}, ..., 10^{-1}$），而非线性搜索，因为正则化效果对量级变化敏感。

## 常见误区

**误区一：验证集损失上升就一定是过拟合**。验证集损失在训练早期波动是正常现象，需要至少连续数个 epoch 持续上升才能判断过拟合。此外，若训练集和验证集的数据分布不一致（分布偏移），损失差异并非来自过拟合而是来自数据质量问题，此时增大正则化强度并无帮助。

**误区二：正则化强度越大越好**。$\lambda$ 过大会导致欠拟合（Underfitting）——模型偏差过大，无法拟合训练数据中的真实规律。例如，将 $\lambda$ 从 $10^{-4}$ 调整到 $10^{-1}$ 可能使 ResNet 的 ImageNet 验证准确率下降超过10%。正则化是寻找偏差-方差最优权衡点，而非无限压制参数。

**误区三：L1 和 L2 正则化效果等价**。L1 生成稀疏权重（适合高维、真实特征稀疏的场景，如文本分类），L2 生成小而非零的权重（适合特征整体有用的场景，如图像识别）。在相同 $\lambda$ 下，两者产生的参数分布截然不同：L1 对应拉普拉斯先验，L2 对应高斯先验（贝叶斯视角下的 MAP 估计）。

## 知识关联

**与模型评估指标的关联**：过拟合的检测依赖于对训练误差与验证误差的对比分析，这需要准确理解准确率、AUC、MSE 等评估指标在不同数据集上的含义。学习曲线（训练集和验证集误差随训练样本量或训练步数的变化）是诊断过拟合与欠拟合的标准可视化工具，其纵轴的具体指标选择取决于任务类型。

**与数据增强的关联**：正则化从参数约束角度减少过拟合，而数据增强从扩充有效训练样本的角度解决同一问题。两者作用机制完全不同，通常联合使用效果更佳。在 ImageNet 训练中，随机裁剪、水平翻转等数据增强操作配合 $\lambda=10^{-4}$ 的 weight decay，相比单独使用任一方法可额外提升约1至2%的验证准确率。数据增强实质上是一种隐式正则化，通过增加训练分布的多样性降低模型对特定样本的记忆。
