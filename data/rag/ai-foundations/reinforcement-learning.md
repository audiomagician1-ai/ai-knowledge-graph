# 强化学习基础

## 概述

强化学习（Reinforcement Learning, RL）是一种通过**智能体与环境的交互**来学习最优行为策略的机器学习范式。与监督学习不同，强化学习不依赖带标签的训练样本，而是依靠环境返回的**奖励信号（reward）**来指导学习。智能体在每个时间步选择动作，环境返回新状态和标量奖励，智能体的目标是最大化长期累积奖励。

强化学习的理论基础可追溯至1950年代Richard Bellman提出的动态规划理论，以及Markov决策过程（MDP）的数学框架。现代强化学习的突破性进展包括：1992年TD-Gammon程序通过时序差分学习掌握西洋双陆棋，2013年DeepMind将深度神经网络与Q-learning结合提出DQN，在Atari游戏上超越人类水平，以及2016年AlphaGo结合策略梯度与蒙特卡洛树搜索击败世界围棋冠军。

强化学习在AI工程中的独特价值在于：它能解决**序列决策问题**，即当前动作影响未来状态和奖励的场景——这是监督学习无法直接处理的问题类型。理解强化学习是掌握RLHF（人类反馈强化学习）的先决条件，后者是当前大语言模型对齐技术的核心机制。

## 核心原理

### 马尔可夫决策过程（MDP）

强化学习问题的标准数学描述是**马尔可夫决策过程**，由五元组 `(S, A, P, R, γ)` 定义：

- **S**：状态空间（State space）
- **A**：动作空间（Action space）
- **P(s'|s,a)**：状态转移概率，给定状态s和动作a，转移到s'的概率
- **R(s,a,s')**：奖励函数，执行动作后获得的即时标量奖励
- **γ ∈ [0,1)**：折扣因子（discount factor），控制未来奖励的当前价值

MDP的"马尔可夫性"要求：下一状态 s' 仅取决于当前状态 s 和动作 a，与历史轨迹无关，即 `P(s_{t+1}|s_t, a_t) = P(s_{t+1}|s_0,a_0,...,s_t,a_t)`。**累积折扣奖励**定义为 `G_t = Σ_{k=0}^{∞} γ^k R_{t+k+1}`，γ=0时智能体只关注即时奖励，γ→1时等权重考虑所有未来奖励。

### 策略与值函数

**策略（Policy）** π(a|s) 是从状态到动作的映射，可以是确定性的（π(s) = a）或随机性的（π(a|s) 为概率分布）。

**状态值函数** `V^π(s) = E_π[G_t | s_t = s]` 表示从状态s出发，遵循策略π的期望累积折扣奖励。**动作值函数（Q函数）** `Q^π(s,a) = E_π[G_t | s_t=s, a_t=a]` 表示在状态s执行动作a后再遵循策略π的期望回报。两者通过**Bellman方程**联系：

```
V^π(s) = Σ_a π(a|s) Σ_{s'} P(s'|s,a)[R(s,a,s') + γ V^π(s')]
```

**最优Q函数** Q*(s,a) 满足Bellman最优方程：`Q*(s,a) = Σ_{s'} P(s'|s,a)[R + γ max_{a'} Q*(s',a')]`，最优策略直接取 π*(s) = argmax_a Q*(s,a)。

### Q-Learning：时序差分方法

Q-Learning是一种**无模型（model-free）**的离策略（off-policy）算法，直接从与环境的交互中学习Q*(s,a)，无需知道转移概率P。其更新规则为：

```
Q(s,a) ← Q(s,a) + α[r + γ max_{a'} Q(s',a') - Q(s,a)]
```

其中 α 是学习率，方括号内的量称为**TD误差（Temporal Difference error）**。DQN的关键创新是用神经网络 Q(s,a;θ) 逼近Q函数，引入**经验回放（Experience Replay）**——将 (s,a,r,s') 四元组存入缓冲区随机采样——以及**目标网络（Target Network）**每隔固定步数同步参数，解决训练不稳定问题。

### 策略梯度方法

策略梯度方法**直接参数化策略** π(a|s;θ)，通过梯度上升最大化期望回报 J(θ) = E_π[G_0]。**策略梯度定理**给出：

```
∇_θ J(θ) = E_π[∇_θ log π(a|s;θ) · Q^π(s,a)]
```

REINFORCE算法用采样回报 G_t 替代 Q^π(s,a)，但方差极高。**Actor-Critic**架构解决这一问题：Actor（策略网络）负责选择动作，Critic（值网络）负责评估动作，用 `A(s,a) = Q(s,a) - V(s)`（优势函数）替代Q值，大幅降低梯度方差。**PPO（Proximal Policy Optimization）**通过裁剪目标函数 `L^{CLIP} = E[min(r_t(θ)Â_t, clip(r_t(θ), 1-ε, 1+ε)Â_t)]`（ε通常取0.1或0.2）限制每次策略更新幅度，是目前最广泛使用的策略梯度算法，也是RLHF的核心优化算法。

## 实际应用

**游戏AI**：DQN在49个Atari游戏中平均超过人类水平229%（2015年Nature论文），输入为原始像素帧（84×84灰度图），动作空间为18个离散按键。AlphaGo Zero完全不使用人类棋谱，仅通过自博弈和策略梯度在40天训练后超越所有人类选手。

**机器人控制**：连续动作空间的机器人操控使用**SAC（Soft Actor-Critic）**算法，其在最大化累积奖励的同时最大化策略熵，公式为 `J(π) = Σ_t E[R(s_t,a_t) + α H(π(·|s_t))]`，α为温度参数。这使机器人在抓取任务中对未见物体有更好的泛化能力。

**大语言模型对齐（RLHF预备）**：在InstructGPT和ChatGPT的训练流程中，PPO被用于根据人类偏好奖励模型微调语言模型。此时"状态"是对话上下文，"动作"是生成的下一个token，"奖励"来自奖励模型对完整回复的打分——这是强化学习框架直接应用于NLP的典型范式。

## 常见误区

**误区1：折扣因子γ越大越好**
许多初学者认为γ→1能让智能体考虑更长期的收益，但这会导致**值函数不收敛**（无穷级数发散）且训练极不稳定。实践中γ通常设为0.99而非1.0；对于回合制任务（episodic task）可以设γ=1，但对连续任务（continuing task）必须γ<1。此外，过高的γ会使梯度信号在时间上弥散，导致智能体难以区分哪个具体动作对长期结果有贡献（**信用分配问题**加剧）。

**误区2：Q-Learning与策略梯度可以任意选择**
Q-Learning（及DQN）只适用于**离散、低维动作空间**——argmax操作在连续动作空间中计算上不可行。策略梯度方法（DDPG、SAC、PPO）才能直接处理连续动作空间（如机器人关节角度）。反之，策略梯度在稀疏奖励环境中收敛极慢，而DQN结合优先经验回放（Prioritized Experience Replay）在此类场景下表现更优。混淆两者的适用场景会导致算法选型错误。

**误区3：强化学习的"奖励函数"等同于监督学习的"损失函数"**
损失函数直接衡量预测与真实标签的差距，梯度可直接回传。而RL中奖励函数是**环境定义的外部信号**，通常稀疏（如围棋只有终局胜负才有奖励），且智能体无法对奖励函数求梯度——策略梯度定理绕过这一问题，通过对**对数策略**求梯度并以回报加权来间接优化。**奖励塑造（Reward Shaping）**是工程上提供密集中间奖励的技术，但如果设计不当会产生**奖励欺骗（reward hacking）**，智能体可能找到与设计意图完全不同的高分捷径。
