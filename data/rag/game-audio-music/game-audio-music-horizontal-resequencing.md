# 水平重组

## 概述

水平重组（Horizontal Re-sequencing）是自适应音乐系统中的一种核心编排技术，其原理是将一首音乐拆分为若干个独立的片段（Segment），然后根据游戏实时状态的变化，动态调整这些片段的播放顺序，从而让音乐在情感上始终与当前游戏场景保持匹配。与通淡入淡出或音量调节不同，水平重组操作的是音乐的"横向时间轴"——它改变的是"先播放哪段、后播放哪段"这一叙事逻辑。

这项技术在1990年代随着 LucasArts 的 iMUSE（Interactive Music Streaming Engine）系统的诞生而进入业界视野。iMUSE 由 Michael Land 和 Peter McConnell 于1991年开发，并首次应用于《猴岛小英雄2》（Monkey Island 2: LeChuck's Revenge，1991）。iMUSE 能够在 MIDI 播放过程中，在一个小节边界处无缝跳转到另一个片段，这正是水平重组最早的工程实现。

水平重组的价值在于它能赋予线性录制的音乐片段一种"非线性讲述能力"。游戏作曲家不需要为每一种游戏状态都创作一首独立的音乐，只需设计足够灵活的模块化片段，并编写切换逻辑，就能让几十秒的素材在实际游戏中支撑数小时的情感变化。

## 核心原理

### 片段划分与过渡点设计

水平重组的第一步是将音乐作品切割为若干"片段"（Segment 或 Stinger）。每个片段通常在音乐上是自洽的——拥有完整的和声进行、可循环的小节结构，或以自然的半终止/全终止结尾，方便后续拼接。常见的切割粒度有三种：**小节级（Bar-level）**、**短语级（Phrase-level）**和**段落级（Section-level）**。粒度越细，系统响应游戏状态变化的速度越快，但对作曲精度的要求也越高。

### 过渡触发与同步窗口

当游戏状态变化（例如玩家从探索状态进入战斗状态）触发切换指令时，系统不会立刻截断当前片段，而是等待下一个合法的"同步点"（Sync Point）。这个等待窗口可以是下一个小节线、下一个四分音符拍点，或下一个预定义的"出口节点"（Exit Point）。等待时长 T 满足：

> **T = (N - t_current mod N)**

其中 N 为一个片段的总时长（以 tick 或毫秒计），t_current 为当前播放位置。通过将过渡点精确对齐到音乐拍格，玩家听到的切换几乎不会产生节奏断层感。

### 状态机驱动的片段路由

实践中，水平重组系统往往通过一个**有限状态机（FSM）**来管理片段路由。游戏定义若干音乐状态（如 Explore、Combat_Low、Combat_High、Victory），每个状态对应一个或多个可播放片段，状态之间的转换边上注明允许过渡的出口节点编号。Wwise 等中间件将这套逻辑封装为"Music Segment"和"Music Switch Container"，开发者可以在不编写底层代码的情况下，通过可视化界面配置完整的水平重组逻辑。

## 实际应用

**《塞尔达传说：荒野之息》（2017）** 的广野音乐采用了极度碎片化的水平重组策略：作曲家近藤浩治和牛尾憲輔将昼间探索音乐拆分为大量短小的钢琴即兴片段，系统根据玩家位置和行为随机或按规则从片段库中选取下一段播放，营造出一种"音乐日记"式的流动感，整体几乎不会完整重复任何一首曲子。

**《光环》系列（Halo）** 的战斗音乐则展示了更结构化的水平重组：Marty O'Donnell 将战斗音乐设计为"Intro → Combat Loop A / Combat Loop B → Outro"的分支路由。当战斗强度提升时，系统在最近的小节边界跳转到更激烈的 Loop B；当战斗结束时，系统路由到 Outro 片段，再衔接环境音乐。这一机制保证了战斗情绪的升降与玩家实际战况精确同步，而非仅依赖简单的音量变化。

在独立游戏开发中，使用 Unity + FMOD 实现水平重组的典型做法是：在 FMOD Studio 中创建 Multi Instrument，将各片段拖入并在 Timeline 上标注 Loop Points，随后通过 Game Parameter（如`combat_intensity`值从0到1的变化）驱动片段跳转，每次触发均等待到最近的一个小节边界再执行切换。

## 常见误区

**误区一：水平重组等同于随机播放列表**

许多初学者认为，只要把几段音乐随机排列就实现了水平重组。实际上，真正的水平重组要求片段之间在调性、节拍、速度（BPM）上高度兼容，且切换必须发生在经过精心设计的同步点上。随机播放不考虑过渡点的对齐，会产生明显的节奏撕裂感，这与水平重组刻意维护的"无缝性"背道而驰。《猴岛小英雄2》的 iMUSE 系统专门花费大量逻辑处理 MIDI tick 级别的对齐，正是为了克服这一问题。

**误区二：水平重组与垂直分层在功能上完全重叠，可以互相替代**

水平重组改变的是**播放哪个片段**（时间轴上的顺序），而垂直分层改变的是**同时播放哪些音轨**（音层的叠加）。前者擅长处理音乐情节的宏观转折（如从探索切入战斗），后者擅长处理同一场景内情绪强度的细腻渐变（如战斗中随敌人数量增加弦乐层）。在实际项目中，这两项技术通常联合使用，用水平重组切换"场景"，用垂直分层调节"强度"。

**误区三：片段越短，游戏响应越灵敏，所以片段应尽可能短**

片段长度存在下限。如果片段短于4小节，音乐往往缺乏足够的和声解决和情绪弧度，玩家会感知到明显的音乐碎片感。水平重组的同步等待窗口也意味着，对于BPM=120的4/4拍音乐，一个2小节的同步单元对应的最长等待时间约为4秒——这在快速战斗中已经显得滞后。实践中，主流AAA游戏的水平重组片段长度通常在8到16小节之间，以平衡音乐完整性与响应速度。

## 知识关联

学习水平重组之前，需要掌握**自适应音乐概述**中介绍的"游戏状态驱动音乐变化"这一基本范式，以及对 MIDI 时间码（PPQ/Tick）或音频帧的基本认知，因为水平重组的无缝切换机制在底层完全依赖精确的时间同步。

掌握水平重组之后，下一个值得学习的技术是**垂直分层（Vertical Layering）**。垂直分层在音乐的"纵轴"（音层叠加）上做文章，与水平重组的"横轴"（片段顺序）形成正交关系。许多工业级自适应音乐系统——如《战神》系列或《最后生还者》的音乐引擎——正是将这两种技术混合使用，才能在复杂的游戏场景中实现既有宏观情节转折、又有微观情绪渐变的丰富音乐表现。
