# 战斗AI行为

## 概述

战斗AI行为是指游戏中敌人角色通过算法驱动，自主决策攻击、防御、移动等战斗动作的技术设计体系。它的核心目标不是让AI"更聪明"，而是让AI的行为**可预测但不显得愚蠢**，从而给玩家提供可读懂且富有挑战性的战斗对手。

战斗AI行为设计的主流技术方案在1990年代随着实时策略游戏兴起而发展，早期《星际争霸》（1998）采用简单的有限状态机控制单位行为，而2005年前后《光晕2》的开发团队率先将行为树（Behavior Tree）引入商业游戏AI，此后行为树逐渐成为3A动作游戏敌人AI设计的标准工具。

战斗AI行为的设计质量直接影响玩家的挫败感与成就感的平衡。一个设计不当的AI要么会被玩家轻易利用固定漏洞（如无限卡位），要么因行为过于随机而失去"对抗感"，导致战斗体验崩塌。

## 核心原理

### 有限状态机（FSM）：战斗AI的基础骨架

有限状态机（Finite State Machine）是战斗AI最基础的行为组织方式。它将敌人的行为拆解为若干**离散状态**，每个状态对应一段具体行为逻辑，状态之间通过**转换条件**触发切换。

一个典型的近战敌人FSM包含以下状态：

- **巡逻（Patrol）**：按预设路径移动，检测玩家距离
- **警觉（Alert）**：停止移动，播放发现动画，等待确认
- **追击（Chase）**：向玩家位置持续移动
- **攻击（Attack）**：进入攻击范围后执行攻击动作
- **撤退（Retreat）**：血量低于30%时拉开距离

FSM的状态转换公式可以用五元组 `M = (S, Σ, δ, s₀, F)` 表示，其中 S 是状态集合，δ 是转换函数 `δ(状态, 条件) → 新状态`。FSM的致命缺陷是"状态爆炸"：当敌人需要同时考虑血量、玩家距离、掩体位置、队友状态等多个变量时，状态数量会以指数级增长，维护成本极高。

### 行为树（Behavior Tree）：战斗AI的模块化决策

行为树用**树形结构**组织AI决策逻辑，解决了FSM状态爆炸的问题。树中每个节点返回三种状态之一：**Success（成功）、Failure（失败）、Running（执行中）**。

行为树的核心节点类型：

- **选择节点（Selector）**：从左至右依次执行子节点，遇到第一个Success即停止，等同于逻辑"OR"
- **序列节点（Sequence）**：从左至右依次执行子节点，遇到第一个Failure即停止，等同于逻辑"AND"
- **装饰节点（Decorator）**：修改子节点的返回值，例如"反转"或"重复执行N次"

以《黑暗之魂》类型Boss为例，其攻击决策行为树可设计为：选择节点下挂载"近距离攻击序列"（条件：玩家距离<2米 → 执行重击动画）和"远距离投射序列"（条件：玩家距离>5米 → 发射火球），每帧从根节点重新遍历整棵树。

行为树相比FSM的核心优势在于**可组合性**：设计师可以将"低血量撤退"模块接入任何敌人的树中，而无需修改其他状态逻辑。

### 威胁感知与攻击令牌系统

多敌人同时进攻时会产生"猪突"问题——所有敌人同时攻击玩家，导致游戏无法进行。现代战斗AI通过**攻击令牌（Attack Token）系统**协调多个敌人的行为。

系统原理：全局只允许同时持有攻击令牌的敌人数量为 N（通常N=1~3，根据游戏难度调整）。敌人进入攻击范围后向管理器申请令牌，获取令牌者执行攻击，未获取者保持原地等待或绕行骚扰。《只狼：影逝二度》中杂兵的"围攻但不群殴"效果正是通过类似机制实现的。

此外，**感知系统**决定AI何时从巡逻状态切入战斗状态。视觉感知通常用**视锥体（Vision Cone）**计算：半径R、张角θ（常见值为120°），玩家进入视锥且无遮挡时触发发现。听觉感知则以玩家动作产生的噪声值与距离的乘积是否超过阈值来判断。

## 实际应用

**《最后生还者》的残障AI协同**：顽皮狗团队在敌人之间设计了语言通信AI——当一名敌人失去视野时，会呼叫队友"他跑了！"，其他AI随即切换为"搜索"状态而非保持"攻击"状态。这个设计用台词播放触发状态转换，同时向玩家明确传达了AI的状态变化，是行为树与叙事融合的经典案例。

**《忍者龙剑传》的方向性攻击AI**：敌人AI在攻击前会读取玩家最近3帧的移动方向，预测玩家下一帧位置，在预测位置处发动攻击。这使得玩家无法通过机械性地向固定方向闪避来规避所有攻击，提升了战斗深度。

**《怪物猎人：世界》的生态行为叠加**：怪兽的战斗AI在行为树之外叠加了一层"生态需求状态机"，包含饥饿、疲劳、领地巡逻等状态。疲劳状态下，怪兽攻击行为树中的"蓄力重击"节点会被装饰节点设为"失败"，从而降低蓄力重击的出现频率，对应怪兽体力不足的设定。

## 常见误区

**误区一：难度提升等于AI攻击频率提升**

许多初学者将"困难难度"实现为缩短AI攻击间隔或提高命中率。这种方式只提升了数值压力而非战略压力，会让玩家感到"被数值碾压"而非"被聪明的对手击败"。更合理的困难度区分方式是：普通难度AI攻击令牌数N=1，困难难度N=2，同时增加AI感知范围而非攻击伤害。

**误区二：行为树越复杂越好**

行为树节点数量过多时，设计师自己也难以预测AI的行为结果，导致出现"AI站着不动"或"AI疯狂自转"等奇异Bug。《战神》（2018）的开发团队公开分享过，德雷奇的行为树控制在约40个叶节点以内，并通过黑板（Blackboard）系统集中管理所有共享变量，是节点规模与可维护性平衡的参考标准。

**误区三：AI应该尽全力击败玩家**

战斗AI的最终目标是服务于游戏体验而非"赢得战斗"。一个能够通过计算完美预判玩家输入并100%命中的AI毫无乐趣。设计师有时会刻意在行为树中加入"等待节点"（等待0.3~0.8秒后再攻击）或引入随机装饰节点（以20%概率跳过攻击），制造可被玩家利用的窗口期，这是有意为之的设计而非缺陷。

## 知识关联

**前置概念：敌人设计**  
敌人的视觉造型、攻击动作集合（Attack Set）和生命值设定是战斗AI行为的输入参数。AI的攻击决策树中每个叶节点对应一个具体攻击动画，若敌人设计阶段没有规划足够丰富的攻击动作类型（近战/远程/AOE），行为树的决策分支将无素材可用。此外，敌人的移动速度属性直接影响"追击"状态下AI能否追上玩家，过快的移速会使玩家失去逃脱机会，破坏令牌系统的设计意图。

**横向关联：关卡设计中的AI战场**  
战斗AI行为的设计必须与战场地形配合。视锥体感知的参数需要根据关卡走廊宽度调整——开阔地图上120°视锥的敌人几乎无法被绕后，应缩窄至70°~90°。攻击令牌系统在开放地形中效果良好，但在狭窄走廊中会退化为排队攻击，需要配合空间布局进行针对性调整。
