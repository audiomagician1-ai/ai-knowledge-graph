# 防具系统

## 概述

防具系统是游戏战斗设计中专门处理"如何减少或抵消伤害"的机制集合，涵盖护甲数值计算、元素/物理抗性分层以及套装被动效果三大支柱。与攻击系统专注于输出伤害不同，防具系统的设计目标是给玩家提供有意义的"生存决策"——选择什么护甲、叠多少抗性、是否凑成套装，每一步都影响角色在特定战斗情境下的存活率。

防具系统最早在桌游《龙与地下城》（1974年）中以"护甲等级（Armor Class, AC）"的形式出现：AC值越高，敌人命中需要骰到的数字越大，本质上是将护甲建模为命中概率修正而非伤害减免。电子游戏继承并发展了这一思路，在1980至1990年代的RPG（如《最终幻想》系列）中逐渐引入直接数值减伤的概念，将护甲从命中修正演变为伤害吸收量或百分比减免，形成了现代防具系统的基本框架。

理解防具系统对游戏设计师至关重要，因为错误的护甲公式会导致两种极端失衡：护甲收益线性增长时高护甲玩家变得无法击杀；护甲收益指数衰减过快时，防御型装备失去投资价值。设计一套护甲公式，本质上是在"可攻克性"与"生存感"之间寻找平衡点。

## 核心原理

### 护甲减伤公式

最常见的护甲减伤模型是**百分比减伤公式**，《英雄联盟》和《暗黑破坏神3》均采用类似结构：

```
实际伤害 = 原始伤害 × 100 / (100 + 护甲值)
```

以护甲值为100为例：实际受到的伤害为原始伤害的 100/200 = 50%，即护甲100提供50%减伤。护甲值200时减伤约66.7%，护甲值300时约75%。这一公式的关键特性是**边际收益递减**——每增加100点护甲，减伤提升量依次递减，有效防止护甲堆叠变得无限强大。

另一种模型是《魔兽世界》早期使用的**护甲上限机制**，将减伤硬性限定在75%或85%，配合复杂的护甲穿透系数，使高级内容中的护甲收益曲线更可控。还有更简单的**固定数值减免**模型（如《火焰纹章》系列的"守备"属性），原始伤害直接减去守备值，剩余部分造成伤害，实现直觉性强但容易产生"免疫小怪"问题。

### 抗性系统的分层设计

抗性系统通常与护甲并列，专门处理元素伤害（火、冰、雷、毒等）的减免。抗性的设计哲学与护甲不同：护甲往往是普适减伤，而抗性是**针对特定伤害类型的专项防御**，鼓励玩家根据敌人属性调整装备。

《暗黑破坏神2》将抗性分为火焰、冰冷、闪电、毒素四项，每项满值75%（噩梦难度上限50%，地狱难度上限75%），通过"抗性减少诅咒"可将其压至负值。这种设计迫使玩家主动管理4条独立数值，制造了大量装备选择深度。《艾尔登法环》则将抗性拆分为物理（标准/斩击/刺穿/打击）和元素（魔法/火焰/雷电/神圣）共8种，使不同形态的盾牌和护甲呈现出差异化的专项价值。

### 套装效果机制

套装效果（Set Bonus）是防具系统中产生"1+1>2"协同感的核心机制。设计逻辑是：单件防具提供基础属性，当玩家集齐同名套装的N件时，激活额外的被动或主动效果。

《暗黑破坏神2》的"全套武器大师"套装凑齐6件后角色获得完整套装光效与大幅属性提升，这种视觉+数值的双重奖励使收集完整套装成为玩家的长期目标。《精灵与森林》类游戏中的"2件套/4件套"结构则是现代MMO的主流模板——2件套提供基础增益（如+15%攻击速度），4件套提供质变效果（如某技能冷却时间减半），用分阶段奖励维持玩家在获取套装过程中的持续动力。

套装效果的数值设计需注意"套装锁定成本"：凑套装意味着放弃部分非套装单件的高数值，设计师需确保套装激活收益足以补偿这一机会成本，通常套装4件激活效果的等效价值应高于同等级最优4件散件组合的10%至20%。

## 实际应用

**《黑暗之魂3》的护甲减重设计**：游戏将"负重比"（装备重量/最大负重）与翻滚帧数直接挂钩，负重比低于30%时可进行快速翻滚（iFrame最多），超过70%变为蜗牛滚。这使防具选择不仅是防御数值的比较，还涉及机动性的取舍，让穿全重甲与轻甲流派产生根本性的操作风格差异。

**《英雄联盟》的护甲穿透与减甲分层**：游戏区分"护甲穿透（百分比）"和"护甲削减（固定值）"，两者计算顺序不同。固定值削减先结算，百分比穿透后结算，且固定削减可将护甲压至0以下（意味着敌人受到额外伤害）。这一机制使物理输出英雄的出装路线拥有真实的对局针对性，对抗堆甲敌人时出"最后的轻语"（45%穿透）与出"黑切"（削减10点护甲）的适用场景截然不同。

**手游《原神》的元素抗性阈值**：敌人元素抗性存在三段区间：抗性为负值时伤害倍率>1（负抗性怪受到额外伤害）；0%至75%时线性减伤；超过75%后采用不同公式（抗性/(1+抗性×0.5)）计算，使高抗性怪仍然可以被打死。这一设计避免了抗性堆至100%后完全免疫特定伤害的极端情况。

## 常见误区

**误区一：护甲越高越好，线性追求护甲数值**

许多玩家认为护甲是纯粹的"越多越好"属性，忽视边际收益递减的存在。以上文百分比公式为例，护甲从0提升到100节省了50%的伤害，而从1000提升到1100只额外节省约0.8%。在资源有限的情况下，将同样的强化资源投入生命值上限往往比继续堆叠已经很高的护甲更有效，因为生命值提升是线性的。

**误区二：抗性满值就是"完全应对"了某类伤害**

不少玩家将抗性75%目标视为"解决了该伤害类型"，但实际上75%减伤意味着仍然承受25%的原始伤害。当敌人单次技能造成8000点雷电伤害时，75%抗性后仍受到2000点，并不等同于免疫。更重要的是，许多高难度内容的BOSS会使用"抗性降低debuff"将玩家抗性强制压低20至40点，使满值抗性玩家实际抗性变为35%至55%，届时事先预留一定缓冲空间的玩家生存能力反而更稳定。

**误区三：凑套装总是最优选择**

套装效果固然诱人，但当某一槽位的最优散件属性远超套装对应部件时，强行凑套装会造成数值亏损。例如在《暗黑破坏神3》的赛季中，若玩家当前持有一件远超套装对应护手的传奇散件，应先计算套装2件激活收益是否能补偿散件属性差值，而非默认凑套装一定划算。

## 知识关联

防具系统建立在**战斗系统概述**所确立的伤害流转模型之上——只有先理解"攻击方产生伤害→防御方承受伤害"的基础流程，才能明确护甲和抗性在何处介入这一流程进行修正。护甲公式（如`100/(100+护甲值)`）的设计选择直接决定了整个战斗数值体系的"天花板感"与"成长曲线陡峭度"，因此防具系统的数值边界往往是战斗设计师最早需要锁定的参数之一。

防具系统还与**装备掉落设计**密切相关：套装件的稀缺度设定、抗性词条的权重分配，直接影响玩家的装备获取节奏。此外，若游戏包含**状态异常系统**（如中毒、燃烧、冰冻），防具系统需要额外定义"抗性是否同时影响状态异常的持续时间或触发概率"，这是防具设计在更复杂战斗
