# 锁定系统

## 概述

锁定系统（Lock-On System）是第三人称动作游戏战斗设计中的目标管理与摄像机控制机制，允许玩家以单键操作将角色朝向、攻击轴线与镜头视角同时对准特定敌人，在移动、闪避、出招全程维持与目标的空间关系，而无需手动拨动右摇杆调整视角。

这一机制的历史节点可精确追溯至1998年11月发售的《塞尔达传说：时之笛》（Nintendo 64）。宫本茂与青沼英二领导的开发团队在3D空间中复现了2D塞尔达的"面向目标攻击"逻辑，将其实现为"Z注目"（Z-Targeting）系统——按住Z键后，林克自动锁定最近的敌人或NPC，摄像机以目标为锚点绕玩家旋转，侧步、盾反的方向计算均以锁定轴为基准。这是业界第一个将锁定逻辑标准化并面向普通玩家公开设计意图的3D动作游戏系统，直接影响了此后《鬼泣》《黑暗之魂》《只狼》《艾尔登法环》等数十款里程碑作品的战斗框架。

锁定系统的核心设计价值在于将"对准"这一感知-运动负担从玩家剥离并交由程序处理。在无锁定系统的游戏中，玩家需要同时维护三条独立的控制线：摄像机轴（右摇杆）、角色朝向（左摇杆+转向速度）与攻击输入（按键时机）；锁定系统将前两条强制合并，使玩家的认知资源得以集中在闪避窗口、招式选择与资源消耗等更高阶的战斗决策上。这正是该系统适合难度等级2/9教学场景的根本原因——它降低的是操作复杂度，而非战斗深度。

参考文献：《Game Feel: A Game Designer's Guide to Virtual Sensation》（Steve Swink, 2008, Morgan Kaufmann）第11章对摄像机-玩家-目标三角关系有系统性论述，是理解锁定系统设计动机的重要一手资料。

## 核心原理

### 目标选择算法

当玩家按下锁定键时，系统需在同一帧（0帧延迟）内完成候选敌人的评分与选定。业界通用做法是对屏幕视锥体内所有处于激活状态的敌人进行**加权综合评分**，典型公式如下：

$$
Score = W_1 \cdot \left(1 - \frac{D}{D_{max}}\right) + W_2 \cdot \left(1 - \frac{A}{A_{max}}\right) + W_3 \cdot T
$$

各变量含义：
- $D$：敌人与玩家角色的世界空间距离（单位：游戏内米）
- $D_{max}$：距离裁剪上限，超过此值的敌人不参与评分（《黑暗之魂3》约设定为20米）
- $A$：敌人投影坐标与屏幕中心的角度偏差（单位：度）
- $A_{max}$：角度裁剪上限，通常为60°–90°
- $W_1, W_2, W_3$：权重系数，典型值为 $W_1 = 0.4,\ W_2 = 0.6,\ W_3 = 0.3$（$W_2$ 大于 $W_1$ 意味着优先锁定"玩家已在看"的目标）
- $T$：威胁加权项，若敌人当前处于攻击前摇动画则取1，否则取0

得分最高者被选为锁定目标。注意 $W_2 > W_1$ 这一设定并非任意，其背后的用户研究逻辑是：玩家在决定按下锁定键之前，通常已经用视线"预选"了目标，屏幕中心偏差正是这一视线意图的代理指标。《鬼泣5》的设计文档（Capcom GDC 2019分享）证实，其锁定算法的屏幕偏差权重是距离权重的1.5倍。

以下为一个简化的伪代码实现，展示锁定目标选择的基本流程：

```python
def select_lock_target(player, enemies, camera):
    best_score = -1
    best_target = None
    D_max = 20.0   # 最大锁定距离（米）
    A_max = 90.0   # 最大屏幕角度偏差（度）
    W1, W2, W3 = 0.4, 0.6, 0.3

    for enemy in enemies:
        if not enemy.is_visible_in_frustum(camera):
            continue
        D = distance(player.position, enemy.position)
        if D > D_max:
            continue
        A = screen_center_angle(enemy, camera)
        if A > A_max:
            continue
        T = 1 if enemy.is_in_attack_animation() else 0
        score = W1*(1 - D/D_max) + W2*(1 - A/A_max) + W3*T
        if score > best_score:
            best_score = score
            best_target = enemy

    return best_target
```

### 摄像机跟随与约束

锁定激活后，摄像机不再单纯追随玩家背后，而是将**玩家与目标连线的中点**作为构图参考锚点，动态调整Yaw（水平旋转角）与Pitch（俯仰角），确保双方同时出现在画面内。具体实现步骤：

1. 每帧计算玩家坐标 $P$ 与目标坐标 $T$ 的中点 $M = (P + T) / 2$
2. 以 $M$ 为目标构图中心，反算摄像机的理想位置与朝向
3. 使用**指数衰减插值**（Exponential Decay Lerp）平滑过渡：

$$
Cam_{t} = Cam_{t-1} + (Cam_{ideal} - Cam_{t-1}) \times (1 - e^{-\lambda \Delta t})
$$

其中 $\lambda$ 为衰减系数，典型值为8–12（数值越大摄像机越"紧"，越小越"软"）。之所以使用指数衰减而非线性插值，是因为线性插值在近距离快速移动时会产生明显的"摄像机滞后抖动"，而指数衰减能在运动初期快速响应、末尾平滑收敛，符合玩家对摄像机控制感的直觉预期。

当玩家与目标的距离超过 $D_{max}$（如《只狼》设定为约18米），系统触发**自动解锁**，UI显示锁定断开图标，防止摄像机产生极端仰角导致玩家失去方向感。

### 目标切换机制

多敌场景中，玩家需在保持锁定状态下切换目标。以下是三款代表作的不同实现策略：

- **《鬼泣5》**：右摇杆拨动超过50%量程即触发切换检测，在当前目标左右各±60°扇形范围内选取评分最高的候选敌人。切换摄像机过渡时间约为100毫秒。
- **《只狼：影逝二度》**：因战斗设计侧重单挑，切换触发阈值提高至70%摇杆量，且仅在当前目标死亡或超距时主动推送切换建议，避免玩家在与BOSS周旋时误触。
- **《艾尔登法环》**：引入"软切换"逻辑——在骑马状态下，切换响应阈值降低至40%，因为骑马战斗中快速调整目标的需求远高于步战场景。

目标切换的摄像机过渡时间设计区间为80–150毫秒：低于80毫秒会引发视觉抖动（画面跳切感），高于150毫秒则让玩家在切换期间感到方向失控，尤其在高速战斗节奏下体验最为明显。

## 关键公式与参数

锁定系统涉及三个核心可调参数组，设计师在调参时需理解其相互制约关系：

| 参数 | 典型值范围 | 调大效果 | 调小效果 |
|------|-----------|---------|---------|
| $D_{max}$（锁定距离上限） | 15–25米 | 允许远程锁定，适合弓箭手角色 | 强制近战思维，增加压迫感 |
| $\lambda$（摄像机衰减系数） | 6–14 | 摄像机响应更紧，追踪感强 | 摄像机更平滑，快速移动时不晕 |
| 切换触发阈值（摇杆量程） | 40%–70% | 更难误触切换 | 切换更灵敏，适合多敌混战 |

**例如**：《黑暗之魂3》的BOSS战设计者将 $D_{max}$ 设为20米，但对特定大型BOSS（如苦艾树的老王）单独设置了28米的超距补偿值，原因是该BOSS的攻击范围本身已超过标准20米，若不扩展锁定距离，玩家在后退闪避时会频繁触发自动解锁，严重破坏战斗流畅性。这一细节体现了锁定参数并非全局统一，而是可以按战斗场景单独配置的工程实践。

## 实际应用

**《塞尔达传说：时之笛》的Z注目（1998年）**：林克按住Z键后锁定最近目标，盾反动作的判定方向强制绑定锁定轴，使防御具有明确的空间意义而非单纯的时机判断。Z注目还支持锁定NPC触发对话，将战斗机制与交互机制统一在同一套输入框架下，是跨系统复用设计的早期典范。

**《黑暗之魂》系列的软锁/硬锁分层设计**：FromSoftware同时维护两套锁定逻辑——"软锁定"（Soft Lock）仅调整角色朝向与普通攻击的追踪角度，不接管摄像机；"硬锁定"（Hard Lock，按R3触发）完全接管摄像机，并限制角色移动为围绕目标的圆弧步伐。这种分层让同一套战斗规则在杂兵清场（软锁快速扫射）与BOSS单挑（硬锁精准管理距离）中均能良好运作，避免了"为BOSS战优化的锁定在杂兵群中令人窒息"的常见设计陷阱。

**手机平台的自动锁定**：触屏设备缺乏物理摇杆，《原神》采用"自动软锁定"方案——角色攻击时系统自动选取攻击方向锥角内评分最高的敌人作为隐式目标，技能动画优先朝该目标释放，但摄像机不被强制锁定，保留玩家的视角自由度。这一设计牺牲了精准度换取了单手操作的流畅性，与主机版锁定设计取向截然不同，说明锁定系统的实现形态必须与输入设备特性共同设计。

## 常见误区

**误区一：锁定范围越大越好**
部分设计新手认为扩大 $D_{max}$ 能提升便利性。但当锁定距离超过摄像机自然视野范围时（通常约15米），画面中目标的可读性已极低，锁定标志虽然存在，玩家却无法清楚看到敌人的攻击前摇，导致"有锁定但无法有效应对攻击"的矛盾体验。《黑暗之魂3》将20米作为标准上限，正是在"覆盖大多数战斗距离"与"保证目标可读性"之间取得平衡的经验值。

**误区二：锁定期间完全禁止玩家控制摄像机**
完全剥夺摄像机控制权会在特定场景下造成视角死角，例如敌人移动到玩家正后方时，锁定摄像机会急剧旋转180°，导致严重的方向感丧失。正确做法是允许玩家在锁定状态下通过右摇杆施加"摄像机偏移量"（Camera Offset），系统以锁定逻辑为基础叠加玩家输入，而非完全压制玩家输入。《怪物猎人：世界》的锁定辅助即采用此"基础锁定+玩家偏移"的叠加架构。

**误区三：目标切换动画过渡可以无限缩短**
测试数据表明，当切换过渡时间
