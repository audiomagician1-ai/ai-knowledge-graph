# 无障碍UI

## 概述

无障碍UI（Accessible User Interface）是指在游戏界面设计中，依据WCAG（Web Content Accessibility Guidelines，网页内容无障碍指南）等标准，让有视觉、听觉、运动或认知障碍的玩家都能有效操作和理解游戏界面的设计实践。WCAG 2.1标准由W3C于2018年正式发布，其核心框架围绕"可感知、可操作、可理解、健壮性"四项原则构建，简称POUR原则，游戏UI设计者将这四项原则逐一映射到菜单、HUD、对话框等界面元素上。

游戏无障碍UI的需求在2010年代后期因主机平台的推动而获得广泛重视。微软于2020年推出的Xbox自适应控制器，以及索尼在《最后生还者2》（2020年）中实现的超过60种无障碍选项，共同将无障碍UI从"可选功能"升级为行业标配。据统计，全球约有15%的人口存在某种形式的残障，在游戏玩家群体中，忽视无障碍UI意味着放弃数亿潜在用户。

无障碍UI在游戏设计中的重要性不仅体现在包容性，也关乎商业价值与法律合规。欧盟《无障碍法案》于2025年6月正式生效，要求数字产品（包含游戏软件）满足无障碍标准，违规企业将面临罚款。因此，掌握游戏UI的WCAG应用方法，是现代游戏UI设计师的必备技能。

## 核心原理

### 可感知性：颜色对比度与文字可读性

WCAG 2.1对文字对比度制定了量化标准：普通文字（小于18pt）的对比度不得低于**4.5:1**，大文字（18pt及以上或粗体14pt以上）对比度不得低于**3:1**。游戏HUD中的血量数值、弹药计数、小地图标注均属普通文字范畴，设计时需用色彩对比分析工具（如Colour Contrast Analyser）验证每一组前景色与背景色的比值。许多游戏默认将白色文字叠加在半透明暗色HUD背景上，实测对比度往往仅达到3:1，未能满足标准。

游戏UI的可感知性还要求不能单独依赖颜色传递信息。例如，策略游戏中用"红色=敌方、蓝色=友方"的标记方案，对于红绿色盲玩家（男性中约8%存在此问题）完全失效。正确做法是在颜色基础上叠加形状、图案或文字标签作为第二信道，《星际争霸2》在2017年更新中引入了单位颜色自定义及图标辅助标记，正是此原则的典型落地。

### 可操作性：键盘/控制器导航与焦点管理

WCAG 2.1成功标准2.1.1要求所有功能均可通过键盘操作，这在游戏UI语境中转化为：菜单、背包、商店等所有界面必须支持纯手柄或键盘的完整导航，不能存在"陷阱焦点"（即焦点进入某组件后无法用键盘离开）。焦点顺序应遵循视觉逻辑顺序，例如RPG装备界面的焦点移动应从左上角的角色模型依次流向装备槽、属性面板、确认按钮。

按键响应时间也属于操作性范畴。WCAG 2.1标准2.5.1要求复杂手势操作需提供单一激活方式的替代方案，游戏中体现为：不能要求玩家必须快速连按或长按才能进行关键UI操作（如确认购买、保存进度），应提供切换模式允许单次按键触发。《神秘海域4》因加入了"单手模式"按键映射选项而荣获2020年BAFTA游戏无障碍奖项。

### 可理解性：一致性界面语言与错误预防

WCAG 2.1成功标准3.3.4要求对"重要交互"提供错误预防机制，在游戏UI中最典型的场景是"不可逆操作确认弹窗"。删除存档、出售稀有装备、放弃任务等操作必须呈现二次确认界面，且确认按钮与取消按钮在视觉层级和位置上应保持整个游戏的一致性——若游戏中"确认"按钮始终位于右侧，则不可在危险操作的弹窗中将"删除"按钮置于右侧，否则会造成习惯性误点击。

界面语言的一致性同样关键：相同功能的图标在所有界面中必须使用同一图形，例如"设置"齿轮图标不能在主菜单与暂停菜单中形态不同。《赛博朋克2077》发布初期因多个界面的图标风格不统一被无障碍审查员指出，1.5版本更新中进行了系统性修正。

### 健壮性：辅助技术兼容与字幕系统

WCAG健壮性原则要求UI内容能被辅助技术正确解析。PC端游戏应提供与屏幕阅读器（如NVDA、JAWS）兼容的UI文本输出，菜单项、按钮状态（选中/未选中）、进度数值均需附带语义化标签。主机平台则依赖系统级辅助功能，例如PS5的TextToSpeech API允许游戏将UI文字转交系统朗读。

字幕和说明文字属于无障碍UI的必要组成，WCAG 1.2.2要求所有预录制音频内容提供字幕。游戏字幕设计应包含：说话者姓名标注、环境音效说明（如"[枪声]""[警报声]"）、可调节的字体大小（建议最小值为32px屏幕字体）与背景遮罩（对比度不低于4.5:1）。《地平线：西之禁地》的字幕系统因同时支持颜色自定义、字号调节和说话者颜色区分，被IGDA基金会列为2022年字幕设计标杆。

## 实际应用

**HUD对比度验证流程**：在游戏的视觉开发阶段，UI美术每完成一版HUD配色方案后，应使用WebAIM对比度检查器输入十六进制颜色值进行4.5:1达标验证，同时在Photoshop中用"色盲模拟"滤镜分别检查Deuteranopia（绿色盲）和Protanopia（红色盲）视图，确认信息仍可读。

**可访问菜单导航测试**：在QA阶段设置专项测试场景，拔掉鼠标仅用键盘完整通关主菜单→设置→按键映射→返回主菜单的操作链，记录所有无法到达的焦点死角并归入缺陷列表。《蜘蛛侠：迈尔斯·莫拉莱斯》在发布前经过了14轮此类导航测试，确保手柄用户在每个UI界面的操作路径均不超过6步。

**字幕系统实施**：在字幕实施阶段，为每一条对话音频文件关联一个字幕文本ID，并在游戏本地化数据库中为字幕文本单独建立字符集，避免非拉丁语字符在字幕框渲染时出现乱码。字幕位置默认置于画面下方1/6区域，但应提供上移选项，防止字幕遮挡游戏下方的重要UI元素（如格斗游戏的体力条）。

## 常见误区

**误区一：WCAG标准是网页专用，游戏只需"参考"即可**
WCAG本身确实起源于网页领域，但XR Association（XRA）和IGDA无障碍特别兴趣小组均已发布将WCAG指标映射至游戏UI的具体指南文件（XR Accessibility User Requirements, 2020）。更重要的是，欧盟《无障碍法案》明确将游戏软件纳入监管范围，其技术标准直接引用EN 301 549，后者全面整合WCAG 2.1 AA级要求。因此，游戏中的WCAG合规已具法律效力，而非仅为设计建议。

**误区二：提供"无障碍模式"开关就已足够**
将所有无障碍功能集中在一个独立模式中，意味着默认游戏体验仍存在障碍，玩家必须主动发现并开启该模式才能正常游玩。正确做法是将核心无障碍特性（如充足的对比度、完整的键盘导航）内嵌为默认设计，将个人偏好类选项（字体大小、字幕颜色）开放为设置项，而非做成需要"开启"的特殊模式。

**误区三：对比度只需在PC显示器上验证**
游戏在电视机上的显示效果往往与显示器差异显著：电视机的伽马值通常为2.4，而PC显示器为2.2，同样的颜色对在电视上可能对比度下降约15%至20%。跨平台游戏的无障碍UI测试必须在目标平台的标准显示设备上进行实测，仅凭色彩数值计算无法保证实际可达标准。
