# 原型制作

## 核心内容

快速原型验证设计假设

## 实践方法

### 实操步骤
1. 明确设计目标和约束条件
2. 参考成功案例建立初始方案
3. 快速制作原型或纸面模拟
4. 组织测试并收集反馈
5. 基于数据迭代改进

### 工具推荐
- 电子表格(Excel/Google Sheets)用于数值模拟
- 白板/Miro用于流程梳理
- 纸面原型用于快速验证

## 常见误区

1. **过度设计**: 功能复杂不等于体验丰富，简洁的系统往往更优雅
2. **忽视玩家感受**: 数值正确不代表体验良好，需要结合测试验证
3. **照搬他人设计**: 每款游戏有独特的上下文，需要理解原理后适配

## 里程碑意义

本概念是游戏设计知识体系中的关键节点，掌握它将解锁更多高级设计概念的学习路径。建议深入理解后再继续。

## 学习建议

- 分析3-5款你熟悉的游戏中该概念的设计实现
- 尝试用一句话向非游戏从业者解释该概念
- 思考如果改变这个设计，整体体验会如何变化
