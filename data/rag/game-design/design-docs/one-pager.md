# 一页纸概念

## 概述

一页纸概念（One-Page Concept Document）是游戏设计流程中最早期的文档形式，要求设计者将整个游戏的核心想法压缩在单张A4纸或等同篇幅内完成表达。这份文档通常不超过500个英文单词或800个中文字，用来回答一个最基本的问题："这个游戏究竟是什么？"它的存在意义在于，在任何人花费时间和金钱深入开发之前，先以最低成本验证一个想法是否值得被讨论。

这一形式起源于好莱坞的电影剧本工业。编剧在正式提案前会撰写"logline"（一句话梗概）和"one-pager"（一页摘要），用极短篇幅吸引制片公司注意。20世纪90年代，Valve、EA等游戏公司将类似做法引入游戏开发管线，要求设计师在进入预制作阶段之前先提交一页纸概念文档，以快速筛选项目可行性。如今，这一形式已成为独立开发者向发行商投递项目时的标准第一步。

对于初学者来说，撰写一页纸概念最大的训练价值在于强迫设计者做出取舍——你无法在一页纸里描述所有功能，因此必须找到游戏真正的核心。这个过程暴露了设计者是否真正理解自己想做的东西。

## 核心原理

### 四大必要元素

一份合格的一页纸概念文档必须包含且仅需包含四个元素，缺一不可：

1. **游戏类型与平台**：明确说明这是什么类型的游戏（如"2D横版动作"），运行在哪个平台（PC/主机/移动端）。
2. **核心玩法循环**：用1~2句话描述玩家每隔30秒到3分钟会重复执行的基本动作，例如"玩家探索地牢、收集资源、击败BOSS后解锁新区域"。
3. **差异化卖点（USP）**：用一句话说明这款游戏与同类型作品的本质区别，必须具体到机制层面。
4. **目标受众与参考作品**：列出2~3款已有市场验证的竞品，并说明目标玩家群体。

### 电梯演讲公式

一页纸概念的核心陈述通常遵循一个被业界广泛使用的句式结构：

> "这是一款面向**[目标受众]**的**[游戏类型]**游戏，玩家将**[核心动词]**，其独特之处在于**[USP]**，类似于**[参考作品A]**与**[参考作品B]**的结合。"

例如："这是一款面向硬核策略玩家的回合制战棋游戏，玩家将指挥一支资源极度匮乏的小队完成不可能的任务，其独特之处在于每次任务失败都会永久改变地图地形，类似于《Into the Breach》与《Darkest Dungeon》的结合。"

这个公式逼迫设计者在动笔之前就对游戏有清晰的框架认知。

### 视觉布局规范

一页纸概念在视觉上也有具体要求：正文字体不小于10号，行间距不低于1.2倍，留有足够白边（每侧至少1.5厘米）。如果空间允许，可加入1张概念草图或参考截图，但图片占用面积不得超过整页面积的25%。这些规范不是审美要求，而是确保阅读者能在60秒内扫读完全文、提取关键信息——这正是发行商在早期评审阶段的实际阅读时间。

## 实际应用

**场景一：独立开发者向发行商投递项目**
开发者在提交完整游戏提案（Game Proposal，通常10~30页）之前，发行商几乎无一例外要求先看一页纸概念。Devolver Digital、Raw Fury等知名独立发行商的官方投递指南均明确列出"one-page concept"作为初筛材料。一页纸概念决定了提案是否会被进一步阅读。

**场景二：Game Jam原型开发**
在48小时或72小时制作的Game Jam（如Global Game Jam）中，团队成员在开始分工前往往先花30分钟共写一份一页纸概念，用来确认所有人对游戏方向的理解一致，避免程序员和美术师朝着不同方向开发。

**场景三：课堂或工作室内部评审**
在许多游戏设计专业的课程体系中（如南加州大学游戏设计项目），一页纸概念是学生提交的第一份正式作业，教授据此筛选哪些项目值得进入下一阶段的小组评审。

## 常见误区

**误区一：把"一页纸"当成摘要而非销售工具**
许多初学者认为一页纸概念是对游戏的客观描述，因此堆砌功能列表。实际上，这份文档的本质是说服读者"这个游戏值得花更多时间了解"，每一句话都应当服务于这个目的。功能列表没有说服力，情感化的差异点陈述才有。

**误区二：将目标玩家描述得过于宽泛**
"面向所有喜欢游戏的玩家"或"面向18~35岁男性"这类表述对评审者毫无价值。有效的目标受众描述应精确到玩家行为层面，例如"曾在《文明6》中花超过200小时但对其战斗系统感到单调的策略玩家"——这类描述让读者立刻理解游戏的市场定位和设计动机。

**误区三：混淆一页纸概念与游戏提案**
一页纸概念不包含制作计划、团队介绍、预算估算或详细关卡设计，这些内容属于游戏提案（Game Proposal）的范畴。在一页纸里加入这些信息会模糊核心信息，并传递出设计者尚未理清思路的信号。

## 知识关联

**前置知识：设计文档概述**
了解设计文档的整体分类（概念文档、提案文档、设计规范文档、技术文档）后，才能理解一页纸概念在整个文档链条中处于最上游的位置——它是所有后续文档的起点和依据。设计文档概述解释了"为什么要写文档"，而一页纸概念回答的是"从哪里开始写"。

**后续概念：游戏提案**
游戏提案（Game Proposal）是一页纸概念的直接扩展形式，通常在一页纸获得初步认可后撰写。游戏提案会将一页纸概念中的每一个要素展开为独立章节：核心玩法循环扩展为完整的机制描述，USP扩展为竞品分析矩阵，目标受众扩展为市场调研数据。因此，一页纸概念写得越清晰，后续扩展为游戏提案的结构越自然。如果一页纸概念本身存在逻辑矛盾，这些矛盾会在游戏提案阶段被放大并导致整个文档崩溃。
