# 测试方法论

## 核心内容

玩家测试的组织、观察与分析

## 实践方法

### 实操步骤
1. 明确设计目标和约束条件
2. 参考成功案例建立初始方案
3. 快速制作原型或纸面模拟
4. 组织测试并收集反馈
5. 基于数据迭代改进

### 工具推荐
- 电子表格(Excel/Google Sheets)用于数值模拟
- 白板/Miro用于流程梳理
- 纸面原型用于快速验证

## 常见误区

1. **过度设计**: 功能复杂不等于体验丰富，简洁的系统往往更优雅
2. **忽视玩家感受**: 数值正确不代表体验良好，需要结合测试验证
3. **照搬他人设计**: 每款游戏有独特的上下文，需要理解原理后适配

## 学习建议

- 分析3-5款你熟悉的游戏中该概念的设计实现
- 尝试用一句话向非游戏从业者解释该概念
- 思考如果改变这个设计，整体体验会如何变化
