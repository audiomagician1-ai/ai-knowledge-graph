# 制造系统

## 概述

制造系统（Crafting System）是游戏设计中让玩家将收集到的原材料按照特定配方组合，生产出新道具、装备或消耗品的机制。与单纯的拾取或购买不同，制造系统要求玩家主动参与物品的生产过程，核心由三个要素构成：**配方（Recipe）**、**原材料（Ingredient）**和**产出物（Output）**。

制造系统的设计历史可追溯至1985年的《Ultima IV》——该作首次引入了将草药组合为药剂的配方机制，玩家须记忆"BL+MR=治疗药"等字母代码。但真正将制造系统推广为主流机制的是2009年发布的《Minecraft》（Mojang），该游戏以3×3的合成格为界面，玩家须按照特定空间排列放置材料才能制造目标物品。在《Minecraft》发布后的五年内，Steam平台上带有"crafting"标签的游戏数量从不足50款增长至超过400款，制造系统成为生存类游戏的标配要素。

制造系统的核心价值在于将资源系统（Resource System）中积累的原材料转化为有意义的游戏进展。玩家在矿洞中挖掘铁矿石本身没有直接目标，但将铁矿石熔炼后制造成铁剑，就完成了一次"原始资源→中间品→最终产出"的价值链，驱动玩家持续参与采集循环。游戏研究者 Jesse Schell 在《游戏设计艺术》（《The Art of Game Design》，2008年，机械工业出版社中译版）中将这类机制称为"转化动词（transformation verbs）"，强调其驱动玩家能动性的关键作用。

## 核心原理

### 配方设计：信息的隐藏与揭示

配方是制造系统的规则集合，其设计核心在于决定**玩家何时知道哪些配方**。常见策略有三种：

- **全开放式**：所有配方默认可见（如《泰拉瑞亚》共有500+条配方全部陈列在合成界面），降低摸索成本，适合新手友好设计。
- **解锁式**：通过升级、阅读书籍或首次拾取某材料触发配方解锁——《星露谷物语》中，首次拾取铜矿石会自动解锁"炉子"配方，正是利用"触发式解锁"制造探索惊喜感。
- **实验式**：玩家须自行尝试材料组合，如《炼金术士：M》要求玩家用120+种材料反复实验，高自由度但可能引发挫败感，通常需配合外部攻略或游戏内提示系统缓解。

配方复杂度通常以**配方树深度（Recipe Tree Depth）**衡量。若制造A需要B，B又需要C，则深度为2。《深岩银河》（Deep Rock Galactic）的武器改造系统深度可达4层，而《动物森友会》的家具配方深度几乎全部为1（直接材料→成品），两者目标受众的操作负担截然不同。设计师在规划配方树时，深度超过3层即需要为玩家提供自动追踪所需材料的"购物清单"UI，否则认知负荷会显著降低完成意愿。

### 制造循环：收集→制造→消耗的闭环

制造系统的核心节奏由**制造循环（Crafting Loop）**决定，其时间构成可以用如下表达式建模：

$$T_{loop} = T_{gather} + T_{craft} + T_{consume}$$

其中 $T_{gather}$ 为材料收集时间，$T_{craft}$ 为制造等待/操作时间，$T_{consume}$ 为产出物被实际消耗使用的时间。

以《星露谷物语》为例：玩家平均需要 3 个游戏天采集足够材料（$T_{gather}=3$），1 天冶炼等待（$T_{craft}=1$），再用制造品应对矿洞挑战并耗尽耐久（$T_{consume}=1\sim2$），完整循环约 5\~6 游戏天。设计师须确保三段时间比例接近 **3:1:2**——若 $T_{gather}$ 远大于其余两项之和，玩家会感到"为了做而做"的空洞感；若 $T_{craft}$ 接近零（如即时合成），制造行为本身失去仪式感与期待感。《怪物猎人：世界》（2018，Capcom）将 $T_{gather}$ 与战斗难度深度绑定：特定部位破坏才能掉落稀有素材，使每次制造都附带清晰的挑战目标，大幅提升玩家的目的感。

### 材料稀有度分层与掉落率设计

制造系统通常将材料按稀有度分为 4\~5 个等级（白→绿→蓝→紫→橙），越稀有的材料制造的物品越强力。以《暗黑破坏神 3》（Diablo III，2012，暴雪娱乐）为例：橙色（传奇）材料来自特定精英怪，其掉落概率在普通难度约为 **2%**，在最高折磨难度（Torment XVI）约为 **15%**，通过难度系数放大掉落率，让玩家感受到挑战与回报的正相关。

设计师在规划材料掉落率时，常用如下估算公式确保"努力可见"：

$$N_{expected} = \frac{Q_{required}}{p_{drop}}$$

其中 $Q_{required}$ 为配方所需稀有材料数量，$p_{drop}$ 为单次获得概率。若 $N_{expected}$ 超过 50 次（即预期需要击败 50 个相同敌人），则应引入**保底机制（Pity System）**——《原神》的五星保底在 90 次抽取内必出，正是将此公式反向运用，将 $N_{expected}$ 限制在玩家心理耐受阈值以内。

## 关键设计模式与算法

在实现制造系统时，配方的数据结构直接影响系统的可扩展性。以下是一种常见的配方表示方式（以 Python 伪代码展示）：

```python
# 配方数据结构示例
class Recipe:
    def __init__(self, name, ingredients, output, output_quantity):
        self.name = name
        # ingredients: dict {材料ID: 所需数量}
        self.ingredients = ingredients
        self.output = output
        self.output_quantity = output_quantity

# 示例：《Minecraft》风格的木剑配方
wooden_sword = Recipe(
    name="木剑",
    ingredients={"oak_plank": 2, "stick": 1},
    output="wooden_sword",
    output_quantity=1
)

# 检查玩家背包是否满足配方需求
def can_craft(player_inventory: dict, recipe: Recipe) -> bool:
    for item_id, required_qty in recipe.ingredients.items():
        if player_inventory.get(item_id, 0) < required_qty:
            return False
    return True
```

上述结构中，`ingredients` 使用字典存储"材料ID→数量"的映射，支持**无序配方**（只验证种类和数量）。若要支持《Minecraft》的**空间配方**，则需将 `ingredients` 替换为 `3×3` 的二维矩阵，并额外实现旋转/镜像等价检测——这正是《Minecraft》早期版本中一个独立的 `ShapedRecipe` 类存在的原因。

## 实际应用

**《我的世界》的空间配方（2009，Mojang）**：玩家在 3×3 格中排列木棍和羽毛的特定图案才能制造箭矢，位置错误即无效。这种设计将"学习配方"本身变为游戏内容——一份弓的配方图需要木棍在第1、3、5格、弦在2、4、6格，玩家须将抽象图案转化为空间记忆，配方知识本身成为老玩家相对于新玩家的认知优势资产。

**《怪物猎人》系列的素材制造（1代起，2004，Capcom）**：每件武器/防具直接对应需要猎杀的怪物——例如《怪物猎人：崛起》中制造"雷狼龙剑斧"需要3块雷狼龙重甲片、2块雷狼龙锋爪，而这些部位素材的获取需要玩家专门攻击特定部位。制造系统将"我想要这件装备"精准转化为"我需要定点猎杀雷狼龙的爪子部位"，为每次出击提供具体的战术目标，是**目标驱动型制造系统**的行业典范。

**《星露谷物语》的季节性材料约束（2016，ConcernedApe）**：南瓜种子只在秋季出售，南瓜汤配方的材料只能在当年秋季（共28天游戏日）完成收集。若玩家错过，须等待整整4个现实赛季（约1.5小时真实游戏时间）后再试。这种**时间稀缺性设计**使制造系统与游戏的年历系统深度耦合，将单次制造行为升维为跨越多个时间周期的长期规划任务。

## 常见误区

**误区一：配方数量越多越好**。《No Man's Sky》初版（2016）包含超过200条制造配方，但玩家测试数据显示，75%的玩家在游戏前10小时内只使用了其中12条。配方膨胀会稀释每条配方的成就感，并造成UI信息过载。设计原则应是：**每增加一条配方，须为其设计至少一个"首次制造动机"**，即让玩家有明确理由在特定游戏节点首次解锁并使用它。

**误区二：制造等待时间等于深度感**。部分设计师将《Clash of Clans》式的真实时间等待（如8小时建造）引入单机制造系统，误认为等待本身能提升仪式感。实际上，仪式感来源于**制造过程中的玩家操作参与度**，而非被动等待时长。《炉石传说》的卡牌合成（花费100/400/800/1600粉尘对应四个稀有等级）是即时完成的，但因玩家需主动决策"拆哪张牌来换粉尘"，制造行为本身充满张力。

**误区三：忽视"反向制造"（Deconstruction）**。多数教程只讨论正向合成，但允许玩家将产出物分解回原材料（甚至有一定损耗比，如《暗黑破坏神 3》分解装备只返还约60%的材料价值）能显著降低玩家的错误决策恐惧，提高制造尝试频率，从而增加制造系统的整体活跃度。

## 知识关联

制造系统在游戏设计知识图谱中处于**资源系统**的下游节点：资源系统定义了材料的获取方式与稀缺程度，制造系统则规定了材料的消耗路径与价值转化规则。两者共同构成游戏经济循环的"生产端"——类比于经济学中"要素市场→生产函数→产品市场"的完整链条（参见 Eyal Roth & Doron Nir，《Game Economy Design》，2021，Springer）。

制造系统同时向上影响**进度系统（Progression System）**：若玩家只能通过制造获取顶级装备（如《怪物猎人》系列不提供强力装备的商店购买途径），则制造系统事实上成为角色成长的唯一通道，其材料获取难度曲线须与整体难度曲线精确对齐。
