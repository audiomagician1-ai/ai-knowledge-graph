# 循环分析方法

## 概述

循环分析方法是一套用于解构、评估并优化游戏核心循环的系统性工具集。它的核心操作是将玩家的游戏行为流分解为"触发→行动→反馈→奖励"四个节点，并对每个节点的效率和吸引力进行独立评分与整体审查。这套方法最早在2000年代初被职业游戏设计师用于后期调优（post-launch tuning），随着《游戏设计艺术》等教材的普及，逐渐成为开发前期设计审查（design review）的标准流程。

循环分析方法之所以不同于一般的玩法审查，在于它强调可量化的循环周期时长。一个设计良好的核心循环通常控制在30秒到3分钟之间，而循环分析方法的首要任务就是精确测量这个时长，并判断它是否与目标玩家群体的注意力窗口匹配。《部落冲突》的核心循环被测定在约90秒，而《黑暗之魂》的战斗循环则约为45秒，两者都经过了反复的循环分析调整。

## 核心原理

### 循环图绘制（Loop Mapping）

循环分析的第一步是绘制循环图，将游戏内所有玩家行为按时间顺序连接成闭合路径。具体操作是用有向图（Directed Graph）表示每个行为节点，节点之间的箭头代表因果关系，而非单纯的时间顺序。例如，在《英雄联盟》中，击杀小兵→获得金币→购买装备→提升输出→更容易击杀小兵，这条闭合路径就是可视化的核心循环。绘制时需注意识别"伪循环"——即看似闭合但实际缺少正向反馈节点的行为链。

### 摩擦力分析（Friction Analysis）

摩擦力（Friction）是循环分析方法中的专用术语，指阻碍玩家从一个节点顺利进入下一个节点的任何设计元素，包括操作难度、等待时间、信息不透明等。摩擦力分析使用一个0到10的评分量表：0分表示完全无摩擦（玩家无意识地自动完成），10分表示摩擦过大导致玩家放弃循环。理想核心循环的各节点摩擦力总和建议控制在12分以内。例如，如果"触发"节点摩擦力为4分（需要主动寻找目标），"行动"节点为5分（操作复杂），则"反馈"节点的摩擦力必须压低到3分以内，否则整个循环会因累积摩擦力过高而断裂。

### 奖励密度测量（Reward Density Measurement）

奖励密度是指单位时间内循环向玩家传递的有意义奖励数量，计算公式为：**RD = N_reward / T_cycle**，其中N_reward是一次完整循环中奖励事件的数量，T_cycle是循环周期时长（秒）。《糖果传奇》的奖励密度约为0.8次/秒，而《文明6》的战略循环奖励密度仅约0.03次/秒。奖励密度并非越高越好，而是需要与游戏类型和目标用户的期望匹配。休闲游戏的奖励密度通常比策略游戏高15到20倍，循环分析时必须基于同类型游戏进行对标。

### 元循环兼容性检验

在完成单层循环分析后，必须将核心循环放入其所属的元循环（Meta-loop）框架中进行兼容性检验。具体检验的问题是：核心循环每次执行是否会向元循环输送有意义的进度？如果核心循环重复100次后，元循环层面没有产生任何可见变化，则说明两者之间存在"进度断层"，需要添加累积机制（Accumulation Mechanic）加以连接。

## 实际应用

以《原神》的日常任务循环为例，应用循环分析方法的完整流程如下：首先绘制循环图，识别出"接取委托→前往地点→击败敌人/收集物品→完成委托→获得原石与经验"的五节点闭合路径。接着测量循环时长，实际游戏数据显示单次日常委托平均耗时约8分钟，四项委托合计约32分钟，属于中等周期循环。

摩擦力分析发现，"前往地点"节点存在较高摩擦力（需要手动开启地图、确认位置），得分约为5分；而"完成委托→获得奖励"节点因动画和音效反馈丰富，摩擦力仅为1分。分析结论建议在地图界面增加一键导航功能（米哈游在2.0版本中确实加入了类似优化），将"前往地点"节点摩擦力从5分降至2分，使循环总摩擦力符合推荐阈值。

## 常见误区

**误区一：将流失点等同于高摩擦力节点。**循环分析方法中，摩擦力高的节点不一定是玩家流失的位置；有时玩家在低摩擦力节点反而更容易流失，原因是过低的摩擦力导致奖励感不足（无意义的胜利）。分析时必须同时查看摩擦力数值和该节点的奖励密度，两者结合才能准确定位流失原因。

**误区二：只分析主循环，忽略旁路循环对主循环的干扰。**许多设计师用循环分析方法只聚焦在最明显的核心循环上，却忽略了那些与核心循环共用相同资源或时间的旁路循环（Side Loop）。例如，在《星露谷物语》中，社交系统的旁路循环会抢占玩家原本分配给农耕主循环的时间窗口，如果不在分析中同时建模两条循环的资源竞争关系，优化建议将产生矛盾。

**误区三：用同一套摩擦力标准评估不同类型的游戏。**休闲手游和硬核PC游戏的目标玩家对摩擦力的容忍度存在显著差异，前者的总摩擦力上限通常是8分，而后者可以接受高达20分的累积摩擦力（高摩擦力本身就是其核心乐趣来源，如《塔科夫》的道具管理系统）。将休闲游戏标准照搬到硬核游戏分析中，会错误地将其高摩擦力节点标记为"需要优化"。

## 知识关联

循环分析方法建立在元循环（Meta-loop）概念的基础上：只有先理解元循环的分层结构，才能在绘制循环图时准确区分哪些行为属于核心循环，哪些属于元循环层。如果将元循环的长期目标节点误归入核心循环，会导致循环周期测量数值虚高，从而产生错误的优化方向。

学习循环分析方法之后，自然进入循环指标（Loop Metrics）的学习。循环分析方法提供了分析动作和评估框架，而循环指标则将这些评估结果转化为可以持续追踪的数据字段，例如将摩擦力评分与实际的玩家行为日志（如节点停留时长、操作取消率）相对应，形成数据驱动的循环优化闭环。
