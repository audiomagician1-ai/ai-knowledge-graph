# 洗牌算法

## 概述

洗牌算法是将一个有序序列随机排列的算法，目标是使每种排列出现的概率完全相等。对于 n 个元素，共有 n! 种可能的排列，公平的洗牌算法要求每种排列被选中的概率恰好为 1/n!。在纸牌游戏、抽奖系统和随机关卡生成中，洗牌算法决定了玩家能否获得真正公平的随机体验。

Fisher-Yates 算法由 Ronald Fisher 和 Frank Yates 于1938年在统计学著作中首次提出，最初是一个手工计算的方法。1964年，Richard Durstenfeld 将其改造为计算机友好的现代版本，时间复杂度为 O(n)，即只需遍历数组一次即可完成洗牌。Donald Knuth 在《计算机程序设计艺术》中进一步推广了这一算法，因此该算法也常被称为 Knuth Shuffle。

在游戏设计领域，错误的洗牌实现会让玩家察觉出规律，破坏游戏公平性。2013年，在线扑克平台 Absolute Poker 就曾因使用缺陷洗牌算法导致部分牌型概率偏高，引发大规模玩家投诉。正确实现洗牌算法是随机性设计的基础工程技能。

## 核心原理

### Fisher-Yates 算法步骤

现代 Fisher-Yates 算法的伪代码如下：

```
for i from n-1 downto 1:
    j = random integer where 0 ≤ j ≤ i
    swap(array[i], array[j])
```

算法从数组末尾开始，每一步从当前位置及其之前的所有元素中随机选取一个，与当前位置交换。这样执行 n-1 次交换后，每种排列出现的概率严格等于 1/n!。以3张牌 [A, B, C] 为例：第一步从3张中随机选1张放到位置2，第二步从剩余2张中随机选1张放到位置1，最后1张自动落入位置0，共产生 3×2×1=6 种等概率排列。

### 常见错误实现：朴素洗牌法

许多开发者直觉上会写出"每次从全部 n 个元素中随机选取位置"的版本：

```
for i from 0 to n-1:
    j = random integer where 0 ≤ j ≤ n-1
    swap(array[i], array[j])
```

这种写法看似简单，实则产生 n^n 种操作序列，而不是 n! 种。当 n=3 时，该算法产生 27 种操作序列映射到 6 种排列，由于 27 不能被 6 整除，各排列的概率不均等。具体来说，[A,B,C] 保持原顺序的概率为 4/27，而某些排列只有 3/27 的概率出现，偏差约达 33%。

### 随机数生成器的选择

洗牌算法的公平性不仅依赖算法本身，还依赖底层随机数生成器（RNG）的质量。`Math.random()` 在大多数 JavaScript 引擎中使用 xorshift128+ 算法，其周期约为 2^128，对于52张扑克牌的 52!≈2^225 种排列而言远远不够覆盖所有可能性。这意味着某些洗牌结果在理论上永远不会出现。游戏服务器端洗牌应使用密码学安全的 CSPRNG（如 `/dev/urandom` 或 `crypto.getRandomValues()`），以确保每种排列都可达。

## 实际应用

**扑克游戏发牌**：标准52张扑克牌的洗牌直接使用 Fisher-Yates 算法，执行51次交换完成。德克萨斯扑克平台通常在每手牌开始前对整副牌执行一次完整洗牌，而不是每次发牌时临时随机抽取，因为前者能更精确地模拟物理洗牌过程。

**卡牌游戏抽牌堆**：《炉石传说》等数字卡牌游戏在构建抽牌堆时对30张卡牌执行 Fisher-Yates 洗牌，确保整局游戏的抽牌顺序在游戏开始时已完全确定且公平。每次"重新洗牌"效果触发时，算法对剩余牌库重新执行一次。

**随机音乐播放列表**：Spotify 在2014年发现用户抱怨"随机播放"不够随机——实际上是因为连续出现同一歌手导致的心理感知偏差。他们保留了 Fisher-Yates 正确实现，但在最终排列上增加了"歌手分散"后处理，使相邻歌曲的艺术家尽量不重复，以符合用户对"随机"的主观期望。

**关卡房间生成**：Roguelike 游戏中预设的关卡模板池（如20种房间模板）在每次游戏开始时用 Fisher-Yates 打乱顺序，再按顺序分配给玩家。这保证了同一游戏局内不会重复出现房间，同时每局的房间顺序完全随机。

## 常见误区

**误区一：多次执行洗牌更随机**。有开发者认为对同一副牌执行3次 Fisher-Yates 洗牌比执行1次"更随机"。实际上，单次完整的 Fisher-Yates 洗牌已经是数学上的均匀分布，再次洗牌并不会增加随机性，只会增加不必要的计算开销。多次洗牌的结果仍然是均匀分布，效果与单次完全相同。

**误区二：用排序配合随机数实现洗牌**。JavaScript 中常见错误写法 `array.sort(() => Math.random() - 0.5)` 看起来简洁，实际上由于比较排序算法（如 V8 引擎使用的 TimSort）的内部结构，不同元素被比较的次数不同，导致各排列的概率严重不均等。在52张牌的测试中，这种方法会使某些牌更倾向于出现在数组首部，偏差可达数倍。

**误区三：洗牌算法与随机数种子无关**。部分开发者认为只要算法正确，种子不重要。然而在可重现性需求（如竞技游戏回放、种子共享的 Roguelike）中，固定种子配合 Fisher-Yates 会产生完全相同的洗牌序列。这既是特性（允许分享关卡），也是安全风险——若种子可被预测，对手可以提前得知牌序。

## 知识关联

洗牌算法建立在概率设计的均匀分布概念之上：理解"每种排列概率为 1/n!"需要对排列组合和均匀分布有基本认识。Fisher-Yates 算法本质上是对"不放回抽样"过程的模拟，每一步从剩余元素中均匀抽取，这与概率设计中"无替换随机选择"的概念直接对应。

在游戏系统层面，洗牌算法是权重随机、保底机制等高级随机系统的基础实现手段。例如，带权重的随机抽卡池可以先将卡牌按权重展开为序列（高权重卡牌复制多份），再用洗牌算法打乱，从而在保证均匀性的同时实现权重控制。掌握 Fisher-Yates 的实现细节，能帮助设计者在需要"无重复随机序列"的所有场景中做出正确的技术选型。
