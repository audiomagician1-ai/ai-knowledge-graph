# 命中/闪避系统

## 概述

命中/闪避系统是游戏中用于判定攻击是否成功造成伤害的随机性框架，通过命中率与闪避率的对比运算决定每次攻击的结果为"命中"或"未命中"。这套系统在桌游时代已有成熟形态——《龙与地下城》1974年初版规则中的THAC0（To Hit Armor Class 0）就是命中判定的早期公式化表达，攻击者需投出一个超过目标防御值的骰子结果（通常是1d20）才能命中。例如，THAC0为15的战士攻击AC为3的敌人，需骰出 15 − 3 = 12 或以上的点数。

命中/闪避系统最独特的设计挑战在于玩家对"公平性"的主观感受，而非概率计算本身。当命中率为85%时，玩家在连续3次未命中后往往会感到"系统在作弊"，尽管这在统计上完全合理：连续3次未命中的概率为 $0.15^3 \approx 0.34\%$，在大量对局中必然频繁出现。这种感知与实际概率之间的断层，是命中/闪避系统区别于纯输出随机性（如伤害浮动）的最显著特征——未命中带来完全的0收益事件，而伤害浮动至少保证有伤害产生，玩家心理定锚点截然不同（Schell, 2008，《游戏设计艺术》）。

## 核心原理

### 命中概率的基础计算模型

最基础的命中判定公式为：

$$\text{命中率} = \text{Clamp}\left(\text{攻击者命中值} - \text{目标闪避值},\ R_{\min},\ R_{\max}\right)$$

其中 $R_{\min}$ 通常设为5%，$R_{\max}$ 通常设为95%，以避免绝对必中或必闪导致某类角色完全失效。《魔兽世界》经典版（2004年）中，玩家攻击同级目标的基础命中上限为99%，攻击高出自身3级的Boss时存在约16%的未命中惩罚（含6%闪避、4%招架、6%未命中），这个数字直接驱动了当时玩家对"命中上限"装备的极度追求，形成了完整的属性优化文化。

更复杂的实现采用**双重概率检定**：先以攻击者命中率掷第一次随机数，再以目标闪避率掷第二次，仅当两次检定均通过时才算命中。若攻击命中率为80%、目标闪避率为20%，双检定综合命中率为 $0.80 \times (1 - 0.20) = 64\%$，远低于直接相减的60%区间计算，需要设计师专门校准数值范围。

### 伪随机分布（PRD）对感知公平性的修正

玩家对低命中率的容忍度远低于等效的伤害浮动，因为"Miss"是一个视觉上清晰的失败事件，破坏操作节奏感。为修正这种感知偏差，《Dota 2》自2012年全面引入**伪随机分布（Pseudo-Random Distribution，PRD）**替代真随机数生成。其核心公式为：

$$P(n) = C \times n$$

其中 $n$ 为连续未触发次数，$C$ 为与期望概率 $p$ 对应的基础常数。$C$ 的值通过以下方程求解：

$$\sum_{k=1}^{\lceil 1/C \rceil} C \cdot k \cdot \prod_{j=1}^{k-1}(1 - C \cdot j) = p$$

以25%的闪避触发概率为例，对应的 $C \approx 0.0611$。这意味着第1次攻击触发概率仅为6.11%，第2次为12.22%，第3次为18.33%，第7次约为42.77%，触发后重置为0。实际触发频率维持在25%，但消除了连续7次以上未触发的长尾概率——真随机下连续7次未触发概率约为17%，PRD下该概率接近于0。

### 命中率的数值设计安全区间

设计师通常将稳定战斗场景的基础命中率控制在**70%至90%**之间。具体边界的含义如下：

- **低于60%**：玩家开始产生"系统在惩罚我"的挫败感，攻击行为的操作反馈与结果脱钩；
- **60%至70%**：适合高风险、高回报的专用角色（如《最终幻想5》的忍者双持配置命中率约65%，以换取双倍攻击次数）；
- **70%至90%**：主流设计甜区，战略选择（如地形、方向、状态异常）可在此区间内产生5%至15%的有意义差值；
- **高于95%**：闪避机制几乎丧失实际意义，数值闪避率退化为纯属性膨胀指标。

《最终幻想战略版》（1997年，Square）的命中率公式将地形、朝向、高度差、角色等级全部纳入计算：正面平地攻击命中率约为80%，从背后攻击可达95%以上，从高处攻击额外加成10%。这5%至15%的差值使得战略定位对每场战斗产生实质影响，而非停留于理论层面。

## 关键公式与算法实现

以下为PRD系统的简化代码实现，展示如何在游戏逻辑中替换真随机命中判定：

```python
class PRDChecker:
    """
    伪随机分布命中检定器
    基于 Dota 2 PRD 算法，消除连续未命中的长尾概率
    """
    # 预计算常见概率对应的 C 值（通过数值解方程得出）
    C_TABLE = {
        0.10: 0.0196, 0.15: 0.0303, 0.20: 0.0424,
        0.25: 0.0611, 0.30: 0.0750, 0.50: 0.1534
    }

    def __init__(self, base_prob: float):
        self.base_prob = base_prob
        self.C = self._get_C(base_prob)
        self.consecutive_misses = 0  # 连续未命中计数器

    def _get_C(self, p: float) -> float:
        # 查表或插值，生产环境应使用数值求解
        keys = sorted(self.C_TABLE.keys())
        for i, k in enumerate(keys):
            if abs(k - p) < 0.001:
                return self.C_TABLE[k]
        # 线性插值兜底
        return p * 0.25  # 简化近似

    def check_hit(self) -> bool:
        current_prob = min(self.C * (self.consecutive_misses + 1), 1.0)
        import random
        hit = random.random() < current_prob
        if hit:
            self.consecutive_misses = 0
        else:
            self.consecutive_misses += 1
        return hit

# 使用示例：25% 闪避率的PRD检定
checker = PRDChecker(0.25)
results = [checker.check_hit() for _ in range(10000)]
print(f"实际触发率: {sum(results)/len(results):.3f}")  # 预期输出约 0.250
```

此实现确保10000次检定后实际触发率稳定在目标概率±0.5%以内，同时最大连续未触发次数通常不超过 $\lceil 1/C \rceil = \lceil 1/0.0611 \rceil = 17$ 次，相较真随机理论上无上限的连续未触发，显著改善了玩家体验。

## 实际应用案例

**《火焰纹章》系列的隐藏命中修正**：该系列自GBA时代（《烈火之剑》，2003年）起长期存在一个刻意隐藏的"显示命中率偏差"机制。游戏对外显示的命中率与内部实际执行值不同：当显示命中率高于50%时，内部执行值会进一步向100%偏移，计算方式为取两次独立随机数的平均值再与显示命中率比较。一个显示为75%命中率的攻击，实际命中率约为 $1 - 0.25^2 = 93.75\%$（因为需要两次随机数均超过75%才算未命中）。这个设计在《苍炎之轨迹》（2005年）的开发文档中有明确记录，目的是大幅减少玩家在高胜算攻击时因未命中产生的愤怒情绪，系列沿用至《风花雪月》（2019年）。

例如，在一场显示命中率为80%的关键攻击中，真随机未命中概率为20%，而双随机数取均值后实际未命中概率仅为 $0.20^2 = 4\%$，玩家感知到的"命中失败"频率下降了约75%。

**《英雄联盟》移除数值闪避系统**：2012年S2赛季前，《英雄联盟》仍保留着继承自《DotA》的数值闪避属性（忍者足具提供12%闪避率，瞬息游侠的被动提供25%闪避率）。Riot在2012年1月的版本说明中指出，闪避机制造成了三类系统性问题：①网络延迟导致客户端与服务器命中判定不同步，产生"明明打中了却显示Miss"的体验；②闪避与暴击等其他随机系统叠加时概率行为难以预测；③反应时间窗口导致闪避型英雄在高延迟玩家群体中形成不公平优势。Riot随后完全移除了数值闪避，改为技能效果型规避机制（如忍者足甲的被动改为减少4%来自普通攻击的伤害），将闪避从随机性系统重新定义为确定性减伤系统。

## 常见误区

**误区一：提高命中率上限总能改善游戏体验**。将基础命中率提升至95%以上并不能解决感知不公平问题——玩家对"本该命中却未命中"的愤怒与期望值有关，而非绝对概率值。当玩家认为命中是默认结果时，哪怕1%的未命中也会触发强烈的挫败感。《魔兽世界》经典版中，许多玩家会为了把命中率从98%堆到99%而替换数件装备，尽管这1%的差值在300次攻击（约一场团本Boss战）中平均仅造成3次额外未命中。

**误区二：PRD适用于所有随机触发场景**。PRD设计的前提是"触发间隔应尽量均匀"，但某些机制恰恰需要真随机的长尾特性。例如，设计一个"极低概率掉落顶级装备"的稀有物品系统，若使用PRD则会让玩家可以精确预测"再过多少次必然掉落"，破坏稀有物品的神秘感与价值。《暗黑破坏神2》的顶级套装掉落采用真随机（低至1/100000的掉落率），刻意保留极端罕见性，玩家社区甚至将"某件装备一生只见过一次"视为荣誉叙事。

**误区三：闪避率设计应与命中率对称**。攻击者的命中率与防御者的闪避率在玩家心理上权重不同：命中率是"我能做到什么"（主动期望），闪避率是"对手能阻止我什么"（被动接受）。当闪避率高于20%时，攻击者往往比防御者更强烈地感受到系统不公平，即便双方承受的期望损失完全对称。这是数值设计师在平衡攻守双方数值时需要额外补偿的心理不对称性。

## 知识关联

**前置概念——输出随机性**：命中/闪避系统与伤害浮动（输出随机性）同属攻击结果的随机性设计，但两者对玩家心理的影响机制截然不同。伤害浮动在期望值附近连续分布，玩家的心理定锚是"平均伤害"；命中/闪避是离散的二元事件（命中/未命中），玩家的心理定锚是"应该命中"。两者混合使用时，设计师需特别注意：高伤害浮动配合中等命中率会产生"偶尔打出超高伤害，偶尔完全未命中"的双极体验，适合高风险角色的风格定位，但不适合作为游戏基础战斗节奏。
