# 伪随机分布

## 概述

伪随机分布（Pseudo-Random Distribution，简称 PRD）是一种用于游戏设计中的概率调节机制，其核心目标是防止真随机数生成器（True RNG）造成的"连续触发"或"长期不触发"现象。与纯粹的数学随机不同，PRD 的每次触发概率会随着连续未触发次数的增加而动态提升，一旦触发后概率立即重置为一个低于标称值的初始概率 P_base。这种机制首次被广泛研究是在《魔兽争霸 III》及其衍生游戏《Dota》系列的暴击机制分析中，玩家和开发者发现这套系统能有效压缩极端运气事件。

PRD 解决的是一个玩家体验层面的真实问题：假设某技能标称暴击率为 25%，在真随机系统下，玩家理论上可能连续 20 次不触发（概率约为 0.32%），也可能连续 5 次触发（概率约为 0.1%）。这两种极端情况在真实对局中虽然概率不高，但对于游戏平衡性和玩家情绪都会产生破坏性影响。PRD 通过牺牲短期随机性换取长期分布的收敛性，使触发间隔更加均匀，让玩家感受到更可预测的技能节奏。

## 核心原理

### 动态概率递增公式

PRD 的数学基础是一个逐次累加的条件概率系统。设 C 为 PRD 系数，第 n 次未触发后的下一次触发概率为：

**P(n) = C × n**

其中 n 从 1 开始递增，直至触发后归零。C 值是根据标称概率 p 反向求解的，满足以下期望值方程：

**p = Σ(n=1 到 MaxN) [C·n × Π(k=1 到 n-1)(1 - C·k)]**

以《Dota 2》中 25% 标称暴击率为例，对应的 C 值约为 0.0975（而非直接使用 0.25），这意味着第一次攻击触发概率仅有约 9.75%，但如果连续未触发，到第 7 次时概率已超过 50%，最多在第 11 次攻击必然触发。这套机制保证了期望触发间隔与标称概率一致，同时大幅削减了方差。

### 触发后的概率重置机制

当第 n 次成功触发后，计数器 n 归零，下一次触发概率重置为 C（而非原始标称概率 p）。这个设计细节至关重要：C < p 意味着刚触发后立即再次触发的概率显著低于玩家"直觉上"的期望值。对于 25% 标称暴击，触发后下一次暴击概率仅约 9.75%，这有效避免了"连暴"现象破坏游戏平衡。

### 方差压缩效果

PRD 相比真随机的最显著数学特征是标准差的降低。对于 25% 真随机概率，每次攻击的触发间隔标准差约为 3.46 次；而使用 PRD 后，触发间隔的标准差降至约 1.83 次，压缩幅度超过 47%。这种方差压缩在游戏实战中体现为：玩家使用暴击类技能时，输出曲线更加平滑，战斗结果更少受极端随机影响，策略决策的有效性更高。

## 实际应用

**《Dota 2》的暴击与逃脱机制**：Phantom Assassin（幻影刺客）的暴击技能 Coup de Grace 使用 PRD 系统，标称暴击率 15%/20%/25%，对应 C 值分别约为 0.053、0.073、0.0975。游戏内的"逃脱"（Blur）闪避率同样采用 PRD，确保高闪避英雄不会出现连续多次被命中或连续多次完全闪避的情况。

**《炉石传说》的卡牌抽取保护机制**：暴雪在多个版本中引入"天花板保护"机制，玩家连续若干局未获得传说卡时，下一张传说卡的出现概率会显著提升，本质上是 PRD 思路在抽卡系统中的变体应用，防止玩家连续 40 包未出传说的极端体验。

**手机游戏的保底系统**：《原神》的祈愿系统在软保底（第 74 抽后概率逐次提升）和硬保底（第 90 抽强制出五星）上结合了 PRD 逻辑与硬上限设计，让玩家明确知晓最坏情况下的资源消耗上限，从而做出更有效的游戏规划。

**RTS 游戏单位技能**：《星际争霸 II》中部分单位的特殊技能触发采用类似 PRD 的调节机制，使多单位编队的 DPS 输出更稳定，降低因随机波动导致的关键时刻战力失常概率。

## 常见误区

**误区一：PRD 会改变技能的长期期望伤害**

许多玩家认为 PRD 降低了暴击率的"实际期望"，因为初始概率 C 明显低于标称值 p。实际上，PRD 的 C 值是精确计算以保证期望触发次数与标称概率完全一致的。在足够长的攻击序列中（如 1000 次攻击），PRD 系统的暴击总次数与真随机 25% 概率的暴击总次数在统计上无显著差异。PRD 改变的是分布形状（方差），而非期望值（均值）。

**误区二：PRD 等同于"等间隔触发"**

有人误以为 PRD 会让技能每隔固定次数触发一次（如每4次一定有一次25%暴击），这会使系统完全可预测从而丧失随机性。真实的 PRD 仍然具有随机性——每次检定仍是概率事件，只是随着累积次数增加概率动态调整。玩家无法在对局中精确预测"下一次攻击必然暴击"，只知道随着时间推移暴击概率在上升。

**误区三：PRD 适合所有随机系统**

PRD 在"每次独立触发"类机制（如攻击暴击、技能闪避）中表现优秀，但不适合用于"开宝箱"类一次性事件或地图随机生成等场景，因为这些场景不存在连续触发累计的上下文环境，强行应用 PRD 反而会引入不必要的复杂性，甚至破坏概率设计意图。

## 知识关联

伪随机分布建立在**概率设计**的基础概念之上——理解期望值、方差、条件概率是分析 PRD C 值求解方程的前提，没有这些基础就无法解释为何 C ≠ p 仍能保证期望一致性。

PRD 直接导向**感知公平性**这一更上层的玩家心理研究课题。PRD 的工程价值不在于让系统在数学上"更公平"（期望值本已一致），而在于让玩家在主观体验层面感受到结果更受个人操作影响、运气成分更可控。感知公平性研究会进一步探讨为何人类对损失序列的负面感知强度远高于等概率收益序列，以及如何通过界面反馈（如显示累积计数）来强化 PRD 带来的心理安全感。
