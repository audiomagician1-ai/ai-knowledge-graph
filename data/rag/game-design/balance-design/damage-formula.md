# 伤害公式

## 概述

伤害公式（Damage Formula）是战斗系统中将攻击方属性、防御方属性与随机因素转化为最终扣血数值的数学模型，直接决定了游戏中每一次攻击的数值结果，是数值策划将"战斗手感"量化表达的核心工具。一套设计优良的伤害公式，要同时满足三个条件：属性收益可感知（玩家升级攻击力后伤害有明显变化）、无数值奇点（不存在某属性堆到极值后免疫伤害或造成无穷伤害的情况）、各属性之间存在合理的等价兑换关系（便于策划进行跨属性的平衡调整）。

伤害公式的历史可追溯至桌游时代。1974年由Gary Gygax与Dave Arneson设计的《龙与地下城》（D&D）首版采用了最简单的线性模型：`伤害 = 武器骰子结果 - 防御等级`，防御等级（Armor Class）越低代表防御越强，设计上存在明显的上下限问题。1987年，《最终幻想》（Square，FC版）引入了非线性防御减伤机制，使高防角色获得超线性的边际收益，为此后二十余年的JRPG伤害公式范式奠定了基础。2004年上线的《魔兽世界》则将护甲减伤公式精细化为双曲线型，并随资料片版本多次重新校准K值，成为网游伤害公式工程化管理的标杆案例（参见：Rampant Games设计分析文章，2009）。

## 核心原理

### 基础攻防计算结构：减法模型与乘法模型

最朴素的伤害公式为减法模型：

$$\text{最终伤害} = \text{基础攻击力} \times \text{技能系数} - \text{防御力}$$

这一模型的致命缺陷在于存在数值奇点：当防御力≥攻击力×技能系数时，伤害归零甚至为负值，高防角色对低攻击敌人实现完全免疫。《暗黑破坏神1》的近战伤害减免部分即因类似机制，导致满防角色对普通怪物近乎无敌，后续系列对此做出了根本性修正。

现代游戏主流方案为**乘法减伤模型**，以双曲线函数将防御力映射为减伤率：

$$\text{减伤率} = \frac{\text{防御力}}{\text{防御力} + K}$$

$$\text{最终伤害} = \text{基础攻击力} \times \text{技能系数} \times \left(1 - \frac{\text{防御力}}{\text{防御力} + K}\right)$$

其中 $K$ 为设计师自定义的平衡系数（常称"防御基准值"）。该公式的核心性质为：

- 当 防御力 $= K$ 时，减伤率恰好为 50%；
- 当 防御力 $= 3K$ 时，减伤率为 75%；
- 无论防御力趋向无穷大，减伤率永远小于 100%，从根本上消除"无敌防御"陷阱。

《魔兽世界》TBC资料片（2007年）的护甲公式中，K值对应等级70玩家约为10557；到《巫妖王之怒》资料片（2008年）角色等级上限提升至80后，K值被调整为约13882.5，保证同等级满甲角色的减伤率维持在约65%这一设计目标附近。

### 暴击系统的期望值融合

暴击机制将伤害公式从确定性函数扩展为期望值模型。设基础伤害为 $D$，暴击率为 $c$（0到1之间），暴击倍率为 $m$，则：

$$\text{期望伤害} = D \times (1 - c) + D \times m \times c = D \times \bigl(1 + c \times (m - 1)\bigr)$$

当暴击倍率固定为 $m = 2.0$（即造成200%伤害）时，公式化简为：

$$\text{期望伤害} = D \times (1 + c)$$

此时每1%暴击率提供恰好1%的期望伤害增益，与1%攻击力的增益完全等价。这正是大多数游戏将默认暴击倍率设为200%的根本原因——它使"1%暴击率 ≈ 1%攻击力"这一简洁等价关系成立，便于策划进行跨属性数值校准。若将暴击倍率调整为 $m = 2.5$（250%），则每1%暴击率等价于1.5%攻击力，整体权重上移，但等价关系依然线性清晰，仍属可控设计。

《暗黑破坏神3》在2012年发售时，部分职业暴击倍率可叠加至600%甚至更高，导致暴击率的权重远超其他属性，产生了严重的装备评估失衡问题，促使Blizzard在2.0版本（2014年）通过降低词缀上限对暴击倍率进行了大幅削减。

### 伤害浮动区间的设计边界

绝大多数RPG会在最终伤害上叠加一个均匀分布或正态分布的随机浮动系数。典型区间：

- `[0.85, 1.15]`：±15%浮动，《魔兽世界》早期版本采用；
- `[0.95, 1.05]`：±5%浮动，适用于竞技场等对公平性要求极高的场景；
- `[1, N]`宽区间：《暗黑破坏神2》部分技能最低1点伤害，最高可达数百点，导致技能伤害体验极不稳定。

浮动区间的设计原则是：**让势均力敌的战斗存在结果不确定性，同时不允许随机性完全掩盖战力差距**。若浮动系数方差过大（如±50%），一个强度高30%的角色仍有较大概率在单次战斗中被弱势方击败，破坏玩家的成长反馈感知；若浮动系数方差过小（如±1%），战斗结果近乎完全可预测，博弈空间被压缩为零。

## 关键公式与算法

### 《精灵宝可梦》完整伤害公式

《精灵宝可梦》系列自第一世代（1996年，Game Freak）起便采用了一套相对完整的伤害公式，后续历代在此基础上叠加修正项。第五世代（黑·白，2010年）的完整公式为：

$$\text{伤害} = \left\lfloor \left( \frac{2 \times \text{等级}}{5} + 2 \right) \times \frac{\text{攻击}}{\text{防御}} \times \text{威力} \div 50 + 2 \right\rfloor \times \text{修正系数}$$

其中修正系数是多个乘法修正项的连乘：属性相性倍率（×0.5 / ×1 / ×2 / ×4）、同属性加成STAB（×1.5）、随机浮动（均匀分布在0.85至1.00之间，以步长1/256离散化）、暴击（×1.5，第六世代前为×2.0）等。这一设计将所有修正项统一为乘法结构，避免了加法叠加导致的线性膨胀问题，也使得单项修正的收益可以独立计算而互不干扰。

### 通用伤害公式的代码实现

```python
import random

def calculate_damage(
    base_atk: float,
    skill_coef: float,
    defense: float,
    k: float,
    crit_rate: float,
    crit_multiplier: float = 2.0,
    variance: tuple = (0.85, 1.15)
) -> float:
    """
    通用乘法减伤伤害公式
    参数:
        base_atk       : 攻击方基础攻击力
        skill_coef     : 技能系数（普攻=1.0，重击=2.5等）
        defense        : 防御方防御力
        k              : 防御基准值（设计师调参）
        crit_rate      : 暴击率 [0.0, 1.0]
        crit_multiplier: 暴击倍率，默认2.0（200%）
        variance       : 伤害浮动区间，默认±15%
    返回:
        最终伤害值（浮点数）
    """
    # 步骤1：基础伤害（含技能系数）
    raw_damage = base_atk * skill_coef

    # 步骤2：乘法减伤
    damage_reduction = defense / (defense + k)
    damage_after_defense = raw_damage * (1.0 - damage_reduction)

    # 步骤3：暴击判定（伯努利随机变量）
    is_crit = random.random() < crit_rate
    damage_after_crit = damage_after_defense * (crit_multiplier if is_crit else 1.0)

    # 步骤4：随机浮动（均匀分布）
    variance_factor = random.uniform(variance[0], variance[1])
    final_damage = damage_after_crit * variance_factor

    return round(final_damage, 2)

# 示例：攻击力1000，技能系数1.5，防御500，K=1000，暴击率30%
for _ in range(5):
    dmg = calculate_damage(1000, 1.5, 500, 1000, 0.3)
    print(f"本次伤害：{dmg}")
```

上述实现中，`k=1000`、`defense=500`时，减伤率 = 500/1500 ≈ 33.3%，基础伤害1500经减伤后约为1000，再乘以浮动系数，期望输出约为975～1185之间，符合设计预期。

## 实际应用

**案例1：手游平衡性中的K值校准**

某MMORPG手游设计初期将防御基准值K设为500，而角色满级防御力上限约为2000，此时满防角色的减伤率 = 2000/2500 = 80%，实际承受伤害仅为原始伤害的20%。这导致堆防角色在PVP中极难被击杀，策划随后将K值上调至2000，满防减伤率降至50%，有效拉平了防御属性的边际收益曲线，使进攻型属性重新进入玩家的构筑考量。K值调整是在不改动任何角色面板数值的前提下实现全局平衡修正的最高效手段之一。

**案例2：《英雄联盟》的护甲穿透分层设计**

《英雄联盟》（Riot Games，2009年上线）将穿透系统分为两层：**固定穿透**（直接从防御数值中减去固定量，如20点护甲穿透）与**百分比穿透**（将敌方护甲乘以某一比例后再计入公式，如45%护甲穿透）。计算顺序为先乘后减：先对防御值乘以(1-百分比穿透系数)，再减去固定穿透值，最后代入减伤公式。这一设计使固定穿透对低防目标（如辅助）高效，百分比穿透对高防目标（如坦克）高效，从机制层面驱动了不同物理AD英雄针对不同阵容选择不同出装路线，极大地提升了装备选择的博弈深度。

**案例3：回合制RPG的技能系数体系**

在回合制RPG中，技能系数（也称"倍率"）是区分技能强度的核心变量。以某手游为例：普攻系数设为100%，一般主动技能系数为180%～220%，爆发技/大招系数为350%～500%，辅助/控制技能为80%～120%（因带附加效果而伤害系数偏低）。若所有技能系数过于接近，玩家会缺乏使用高CD技能的主动意愿；若爆发技系数过高（如800%以上），则单次释放可能秒杀对手，战斗时间被压缩至无法体现操作差异的程度。合理的技能系数梯度是维持战斗节奏多样性的基础。
