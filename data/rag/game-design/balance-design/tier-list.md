# 梯队划分

## 概述

梯队划分（Tier List）是竞技游戏平衡性设计中用于量化角色、装备或策略相对强度的系统性评级方法。设计师通过收集胜率、选用率（Pick Rate）、禁用率（Ban Rate）等可量化指标，将游戏内容分配至从S到D（或T0到T4）的强度层级，从而直观呈现当前版本的力量格局。

梯队划分的概念在1990年代格斗游戏社区中逐步形成，最早由《街头霸王II》（1991年，Capcom开发）的玩家群体用于非正式讨论不同角色的对战优势。彼时玩家在BBS和游戏厅布告栏上以手写或打印的表格记录角色间的相克关系，逐渐演化出"上中下"三层的原始分级结构。进入电子竞技时代后，《英雄联盟》《守望先锋》《炉石传说》等游戏的开发商将玩家社区的梯队评估纳入官方平衡性调整的参考依据，使梯队划分从民间经验演变为系统化的设计工具。

梯队划分的核心价值在于将复杂的多变量强度关系压缩为可操作的设计信号：当某个角色稳定居于T0档（胜率高于55%且禁用率超过30%）时，它会扭曲整个Meta的策略多样性，迫使其他玩家围绕应对该角色构建阵容，最终降低游戏整体乐趣。正因如此，Riot Games、暴雪娱乐、Valve等主流竞技游戏公司均配置了专职的"平衡性分析师"岗位，以梯队数据作为每个更新周期的核心决策输入。值得注意的是，梯队划分本质上是一种信息压缩工具——它将成千上万场对局数据中蕴藏的复杂力量关系，浓缩成设计师和玩家都能快速解读的层级信号，其准确性与所依托的数据质量和方法论直接相关。

## 核心原理

### 分层标准与量化指标

梯队划分不依赖主观印象，而依托三项核心数据构建评级体系：**胜率（Win Rate）**衡量该角色或装备在对局中的实际胜利概率，通常以50%为基准线；**选用率（Pick/Usage Rate）**反映高水平玩家的主动选择偏好，高选用率表明玩家认可其强度；**禁用率（Ban Rate）**在有禁选机制的游戏中，代表对手主动规避该对象的意愿强度。

《英雄联盟》职业赛场采用的"联合强度分"公式如下：

$$S = P \times 0.4 + B \times 0.4 + (W - 0.5) \times 0.2$$

其中 $S$ 为联合强度分，$P$ 为选用率（0到1之间的小数），$B$ 为禁用率（0到1之间的小数），$W$ 为胜率（0到1之间的小数），$(W - 0.5)$ 表示胜率相对于基准线50%的偏差量。

这一公式体现了梯队划分的核心逻辑：一个角色即使胜率普通（如 $W = 0.52$），若禁用率高达 $B = 0.80$，代入公式后其综合强度分 $S = 0.4 \times P + 0.32 + 0.004$，依然可居于T0级，因为对手宁愿付出禁用资源也不愿与之对抗。该公式中禁用率权重与选用率并列，正是为了捕捉这种"被动压制力"——一个即使不被使用、但因存在本身就压制对手战术选择的强势角色。

例如，以2022年《英雄联盟》世界赛赛季中后期的泽拉斯（Xerath）为例：其全球胜率仅约51.3%，但因其超远程骚扰技能组合几乎不存在克制手段，禁用率在高端赛事中飙升至82%。代入公式（假设选用率约为0.15）：$S = 0.15 \times 0.4 + 0.82 \times 0.4 + (0.513 - 0.5) \times 0.2 = 0.06 + 0.328 + 0.0026 \approx 0.391$，轻松超过T0阈值（通常设定为0.35）。这个案例清晰展示了为何单纯依赖胜率会严重低估某类角色的真实强度。

### 环境依赖性与版本绑定

梯队不是角色的固有属性，而是**特定版本环境（Meta）下的相对排名**。同一个角色在不同版本的强度可跨越三个梯队：《炉石传说》的德鲁伊职业在2017年KFT（冰封王座的骑士）系列加入"腐蚀伤齿龙"后从T3跃升T0，仅因为一张牌改变了整个环境的循环速度与进攻节奏。设计师在划分梯队时必须明确标注当前版本号，避免将历史梯队数据误用于现版本决策。

垂直梯队（同级内部排名）同样重要：同为T1级的六个角色，其强度差距可能大于T1与T2之间的差距。部分设计团队引入T1+和T1-来细化层级，但过细的分层会增加维护成本，大多数商业游戏采用5到7档划分作为实践平衡点。此外，环境依赖性还体现在地图机制、道具系统迭代等维度——《Dota 2》的每次大型地图改版（如2021年7.30版本引入"中立道具刷新"机制）都可能使依赖控场站位的英雄整体下移一至两个梯队档位，而机动型英雄则相应上移，这与角色本身的数值毫无关联。

### 梯队压缩与多样性阈值

健康的梯队分布应呈现**正态分布**而非幂律分布。若T0档集中了超过20%的可用角色，说明平衡性设计存在系统性问题——这通常由基础数值通货膨胀（Power Creep）引起。《万智牌》首席设计师Mark Rosewater在2002年的官方设计文章中提出"15%法则"：理想情况下，最强牌组与最弱可用牌组的胜率差距不应超过15个百分点，超出即触发禁牌（Banned）或限制调整（Restricted）机制（Rosewater, 2002）。这一原则后来被多家游戏公司借鉴，成为衡量竞技游戏平衡健康度的非正式行业参考标准。

从实践数据来看，《英雄联盟》在角色池扩充至160个以上后（2022年），T0级角色的平均占比约为4.5%至7%，T1级约为15%至22%，T2至T3级合计约占50%至60%，T4级约占15%至20%。这一分布大致符合正态规律，说明大型商业游戏在长期运营中已逐渐摸索出维持梯队健康分布的工程化方法。

### 样本量与统计显著性要求

梯队数据的可靠性取决于样本量的充分性。一般而言，胜率计算需要至少1000场对局数据才能将标准误差控制在±1.5%以内；对于选用率极低（低于2%）的角色，即便样本量达到500场，其胜率的置信区间依然过宽，不足以支撑梯队定级决策。《英雄联盟》开发团队内部设定的最低有效样本阈值为单个补丁周期内5000场对局，低于此阈值的角色数据仅供参考，不直接触发削弱或增强操作（Lin et al., 2021）。这意味着新上线角色往往需要经历1到2个版本周期的数据积累，才能获得可靠的梯队定位。

统计显著性检验在梯队决策中同样不可或缺。当两个角色的联合强度分差异小于0.03时，通常需要进行假设检验（如双样本Z检验）确认差异是否具有统计意义，而非直接将其分入不同梯队。忽略统计显著性的梯队划分，本质上是在噪声数据上做决策，可能导致"误伤"平衡性良好的角色，或放过真正需要调整的问题角色。

## 关键公式与模型

### 双维度评级矩阵

单纯依赖联合强度分仍可能遮蔽"高上限低下限"型角色的设计问题。部分游戏公司引入**双维度评级矩阵**，在强度评分之外加入**技术依赖系数（Skill Dependency Factor，SDF）**，将角色分布在二维坐标系中：

$$\text{SDF} = \frac{W_{\text{top}\_1\%} - W_{\text{global}}}{W_{\text{global}} - W_{\text{bottom}\_10\%}}$$

其中 $W_{\text{top}\_1\%}$ 为顶尖玩家（前1%）的胜率，$W_{\text{global}}$ 为全服平均胜率，$W_{\text{bottom}\_10\%}$ 为后10%玩家的胜率。SDF值越高，说明该角色的强度越依赖玩家技术水平，梯队定级时需区分"大众梯队"与"高端梯队"两套体系，避免因全服平均数据掩盖高端环境中的强度失衡问题。

例如，《英雄联盟》的卡萨丁（Kassadin）全服胜率长期维持在47%至48%之间，属于大众梯队的T3级角色；然而在钻石段位以上，其胜率稳定在57%以上，SDF值高达约3.2，在高端梯队中属于T0级。若仅以全服胜率定级，将错误触发数值增强，进而在高端环境造成更严重的失衡。

SDF矩阵的实用价值还体现在指导"设计目标"的制定上：SDF值长期高于2.5的角色，通常意味着其设计存在"技能上限过高而下限过低"的结构性问题，设计师需决定是简化操作逻辑（降低SDF）还是接受其作为"高技术奖励型"角色存在于游戏中，同时为低技术玩家提供替代选项。这是梯队数据反哺设计决策的典型路径。

### 梯队迁移速率模型

除静态梯队评级外，部分数据分析团队还追踪角色的**梯队迁移速率（Tier Migration Rate，TMR）**：

$$\text{TMR} = \frac{\Delta S}{\Delta t}$$

其中 $\Delta S$ 为联合强度分在相邻两个版本间的变化量，$\Delta t$ 为版本时间间隔（通常以天计）。TMR绝对值过大（如超过0.05/天）往往意味着该角色受到了环境剧烈变化的影响，或开发商进行了过度的数值调整，两者都是平衡性管理质量下降的信号。

## 实际应用

### 《守望先锋》角色重设计决策

暴雪在2021年对《守望先锋》坦克英雄路霸（Roadhog）的调整，直接来源于梯队数据：路霸在高段位（钻石以上）的"钩住即秒杀"连招成功率超过78%，导致他在该分段的禁用率维持在45%以上长达三个赛季。梯队分析指出其T0地位不来自整体数值过强，而来自特定技能序列的高确定性击杀窗口，因此调整策略聚焦于缩短钩子判定范围（从6米缩短至5.5米），而非削减整体血量或伤害。这是"精准调整"而非"数值蛮力"的典型案例：梯队数据不仅告诉设计师该角色需要调整，还通过技能维度分析指出了问题根源所在。
