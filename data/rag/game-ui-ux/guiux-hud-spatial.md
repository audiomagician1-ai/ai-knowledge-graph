# 空间HUD

## 概述

空间HUD（Spatial HUD）是将UI元素直接放置在游戏三维世界坐标系中的界面设计范式。与固定在摄像机视口平面的屏幕空间HUD不同，空间HUD的UI元素具有真实的世界坐标位置，会随摄像机视角变化而产生透视缩放、遮挡关系和景深模糊效果。典型表现形式包括悬浮在角色头顶的血条与名牌、附着在物体表面的交互提示标签，以及显示在NPC对话框位置的浮动文字气泡。

空间HUD的技术实践可追溯至1997年《网络创世纪》（Ultima Online）中玩家角色头顶的名称显示设计，但真正将其系统化的是2004年《魔兽世界》对命名头顶单位框架（Nameplate System）的规范化实现——该游戏允许显示同屏数十个敌方生命值条，每条生命值UI通过Billboard技术始终朝向玩家摄像机。这一设计奠定了大型多人在线游戏空间HUD的基本规范。

空间HUD的核心价值在于它将信息锚定于信息来源本身。当玩家需要判断"哪个敌人血量最低"时，空间HUD允许视线在战场上直接扫描而无需查阅屏幕角落的目标框，信息与世界对象的空间关联显著降低了认知跳转成本。

## 核心原理

### 世界空间坐标绑定与Billboard渲染

空间HUD元素的位置由一个三维世界坐标点加偏移向量决定，公式为：

**UI世界坐标 = 绑定对象坐标 + 本地偏移向量（offset）**

偏移向量通常为角色模型包围盒高度的1.1至1.3倍，以防止名牌与角色模型头部重叠。Billboard渲染技术使UI面板的法线向量始终等于摄像机到UI中心的反方向向量，从而实现无论摄像机俯仰角度如何，血条始终"正面"朝向玩家。Unity引擎中通过`transform.LookAt(Camera.main.transform)`实现，Unreal Engine则通过将Widget Component的空间设置为`Screen`模式完成。

### 深度与遮挡处理

空间HUD面临的核心技术挑战是遮挡（Occlusion）问题：当敌人被墙壁遮挡时，其血条是否应当继续显示？业界存在三种主流策略：**完全遮挡**（血条随角色被遮挡消失，《黑暗之魂》系列采用此策略以强化空间认知）、**半透明透视显示**（血条在遮挡时降低至30%-50%不透明度，《守望先锋》的队友状态条采用此方式）以及**仅在一定距离内显示**（超过特定距离阈值后隐藏以降低视觉噪声，《怪物猎人：世界》将队友血条的显示距离限制在约30个游戏单位内）。

### 屏幕边缘指示器与视野外处理

当绑定对象离开摄像机视锥体（Frustum）时，纯粹的空间HUD元素会从屏幕上消失，导致玩家失去方向感。解决方案是视野外（Off-Screen）指示系统：将世界坐标向量投影至视口空间，当投影点落在屏幕外时，将UI图标钳制（Clamp）到屏幕边缘并添加方向箭头。《极限竞速：地平线》系列将赛道指示标记的屏幕边缘钳制范围设置为距屏幕边界15像素，箭头旋转角度通过`Mathf.Atan2`计算对象相对摄像机的方位角得出。

### 距离自适应缩放

空间HUD元素若不做缩放处理，近处对象的血条会覆盖半个屏幕而远处的血条则细如发丝。标准处理方式是将UI尺寸保持在一个最小值与最大值之间：**显示尺寸 = Clamp(基础尺寸 / 距离, 最小尺寸, 最大尺寸)**。《英雄联盟》将英雄血条的最小显示宽度设定为36像素、最大宽度设定为72像素，超出该区间的距离段血条以固定像素尺寸显示。

## 实际应用

在MMORPG战斗中，空间HUD的典型实现是**仇恨指示器**：当玩家被超过3个以上怪物攻击时，每个怪物头顶的仇恨图标（通常为红色感叹号）使坦克玩家无需查阅技能释放日志即可瞬间识别战场优先级。《最终幻想XIV》的副本设计中，BOSS技能的范围警示圆圈同样属于空间HUD——它们被放置在地面世界坐标位置，与地形共享深度关系，令玩家能直接判断"站在这个位置是否安全"。

在第一人称射击游戏中，队友的空间HUD名牌需要解决队友与敌人的辨识问题。《Apex英雄》的处理方案是队友名牌在屏幕内持续显示、颜色与玩家选择的角色颜色绑定，而敌方单位在未被扫描前不显示任何空间HUD信息——这一设计直接影响了游戏的信息不对等战术深度。

在开放世界游戏中，空间HUD的交互提示标签（通常为"E键交互"类提示）需要设置**激活半径**：《塞尔达传说：旷野之息》的可交互物件在玩家进入约5米范围内才显示提示图标，避免开放世界中满屏漂浮图标的视觉污染。

## 常见误区

**误区一：所有UI信息都适合空间化处理。** 血量、名称、交互提示等与特定世界对象强关联的信息适合空间HUD，但玩家自身的技能冷却时间、小地图等全局信息若改为空间HUD会造成信息丢失——玩家必须主动转动视角才能查阅，违反了HUD"持续可见"的基础原则。《赛博朋克2077》初版将部分任务追踪信息放置在世界空间的路标上，玩家测试反馈显示这导致大量"迷失方向"的体验报告，后续版本恢复了屏幕空间的任务指示箭头。

**误区二：空间HUD等同于VR界面。** VR界面设计要求UI面板与用户保持固定的头部锁定关系（Head-Locked）或放置在至少0.5米的舒适视距之外，否则会引发眼睛调节辐辏冲突。普通3D游戏的空间HUD不存在该物理约束，血条可以贴近角色头部仅0.1个游戏单位。将VR的最小舒适距离规则硬套到普通3D游戏会导致名牌偏移过远、与绑定对象失去直观关联。

**误区三：Billboard朝向必须完全朝向摄像机。** 完全Billboard（Spherical Billboard）会使血条在摄像机俯仰角极大时出现奇异旋转。大多数MMO游戏采用**柱状Billboard**（Cylindrical Billboard）：UI面板仅在水平轴上旋转朝向摄像机，垂直轴保持与世界坐标Y轴对齐，这样从高空俯视时血条依然以"正常"角度显示而不会产生透视变形。

## 知识关联

空间HUD以**非叙事性HUD**的基础分类为前提——只有明确了"UI服务于功能信息而非叙事沉浸"的设计边界，才能正确判断何时将血条放入世界空间（功能优先）、何时将其彻底隐藏（叙事优先）。空间HUD中的距离感知与深度层次处理，与**VR界面设计**中的立体视差原理共享底层的三维坐标投影数学，但应用约束条件不同。

空间HUD向**元叙事HUD**方向延伸时，进入了一个更复杂的设计领域：若游戏世界内的角色能够"感知"到漂浮在自己头顶的血条（如《心灵杀手2》中角色对游戏界面的自我指涉台词），空间HUD便从功能性信息展示工具转变为叙事装置，触及界面与叙事边界模糊的元叙事设计问题。
