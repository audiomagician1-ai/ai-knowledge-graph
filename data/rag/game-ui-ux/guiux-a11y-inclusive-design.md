# 包容性设计思维

## 概述

包容性设计思维（Inclusive Design Thinking）是一种在产品开发**最初阶段**便将多样化用户需求纳入核心考量的设计方法论，而非事后补救式地为特定群体"打补丁"。微软无障碍团队在2016年发布的《包容性设计工具包》中将其定义为："为一个人设计，延伸至众人"（Design for one, extend to many）。这一原则与传统无障碍设计的根本区别在于：它不把残障视为个体的"问题"，而是把设计的局限视为需要解决的障碍。

该方法论的现代形态由微软首席设计师Kat Holmes在2010年代系统整理并推广。她在2018年出版的《Mismatch》一书中指出，全球约有10亿人拥有某种形式的残障，但若将临时性障碍（如骨折）和情境性障碍（如强光下无法看清屏幕）也纳入统计，受益人群实际可达**全部用户的40%以上**。在游戏领域，包容性设计思维意味着：UI的字体大小、色彩对比度、按键重映射等不再是"辅助功能选项"，而是从第一张原型图起就被纳入设计决策。

对游戏开发团队而言，这一方法论具有直接的商业价值。《最后生还者 第二部》（2020年）因其超过60项无障碍功能而被无障碍游戏奖授予最高荣誉，其销量与口碑的双丰收证明：在设计初期投入包容性思考，边际成本远低于后期改造，且能显著扩大玩家基数。

## 核心原理

### 三类障碍模型：永久、临时与情境

包容性设计思维的核心分析框架是将用户障碍划分为三个类型，而非仅聚焦于永久性残障。**永久性障碍**指用户长期的感官或运动差异，如单手玩家或色盲玩家；**临时性障碍**指短期状态，如手臂骨折或眼睛手术恢复期；**情境性障碍**指环境带来的限制，如在嘈杂的地铁上无法听到游戏音效，或在阳光直射下屏幕反光导致无法分辨UI颜色。为单手玩家设计的一键连发功能，同样惠及手持饮料的休闲玩家；为听障玩家设计的字幕系统，同样服务于所有在图书馆静音游玩的用户。

### "解决排斥点"而非"添加功能"

传统设计流程先建立"标准用户"模型，再为其他群体添加特殊功能；包容性设计思维则要求团队**主动寻找并消除设计中的排斥点（exclusion points）**。在UI/UX实践中，排斥点识别工具包含三个步骤：首先，列出所有交互动作（如：快速点击、分辨颜色差异、阅读小字体文本）；其次，标注哪类用户会在该步骤遭遇障碍；最后，将解决方案设为**默认设置或高度可配置**，而非隐藏于深层菜单。例如，Xbox自适应手柄的开发始于对AbleGamers公益组织用户需求的深度访谈，而非先开发标准手柄再做改型。

### 多元视角的设计参与原则

包容性设计思维的第三项原则是"被排斥者应参与设计，而非仅作为测试对象"。这一原则要求将残障玩家、老年玩家或认知差异玩家纳入**整个设计周期**——从概念阶段的头脑风暴，到原型测试，再到上线后的迭代反馈。游戏无障碍咨询机构SpecialEffect长期与开发商合作，在Alpha阶段即引入运动障碍玩家测试游戏的UI响应区域大小与容错率。实证研究表明，早期引入5名多样性用户测试，能发现后期引入50名同质用户测试所无法发现的设计盲点。

### 可配置性的层级设计

包容性设计不等于用一套默认设置满足所有人，而是建立**有意义的可配置层级**。UI设计中常见的包容性配置层级分为三档：第一档是符合WCAG 2.1 AA标准的基础配置（最低色彩对比度4.5:1，最小可点击区域44×44像素）；第二档是用户可在无障碍设置菜单中调整的参数（字体大小、字幕颜色、按键布局）；第三档是高度个性化的辅助方案（自定义色彩配置文件、单轴开关控制支持）。这三档的存在确保游戏对"从不需要帮助"到"需要全面辅助"的所有用户均保持可用。

## 实际应用

**《赛博朋克2077》的UI改版案例**体现了包容性设计思维的滞后代价。该游戏2020年首发时，UI大量使用黄色细字体叠加复杂背景，色盲玩家和低视力玩家反映核心信息几乎无法辨识。开发商CD Projekt Red在后续补丁中不得不重新设计整套HUD的色彩对比逻辑，耗费了原本可在初期通过包容性设计规避的大量资源。

**《地平线：西之绝境》（2022年）**则提供了正向范例。其UI团队在预制作阶段即制定了"无障碍设计清单"，包含：所有UI文本在默认状态下达到WCAG AA对比度标准、任务关键信息不依赖纯色彩区分（同时使用形状和图案）、所有弹出提示窗口支持读屏模式。这些决策在早期原型阶段几乎没有额外成本，却使游戏荣获2022年度无障碍游戏年度大奖。

在移动游戏领域，包容性设计思维推动了"大字幕模式优先"（Large Text First）的UI排版流程：设计师先以最大字体尺寸布局界面，再缩减至默认尺寸，而非相反。这一流程确保放大字体后界面不会出现元素重叠或信息截断的问题。

## 常见误区

**误区一：包容性设计会稀释核心玩家体验。**
这一担忧混淆了"简化游戏"与"降低进入门槛"的概念。包容性设计不要求删除游戏难度或特色，而是确保所有玩家都能到达游戏体验的起点。《黑暗之魂》系列的难度核心从未被无障碍选项削弱——《艾尔登法环》新增的HUD缩放和色彩盲辅助，丝毫未改变游戏本身的挑战曲线，只是让更多人能看清界面信息。

**误区二：包容性设计等同于无障碍设置菜单的功能堆砌。**
无障碍设置菜单是包容性设计的**输出产物之一**，但包容性设计思维是塑造整个设计决策流程的方法论。如果基础UI的字体对比度在设计之初就不达标，再完整的无障碍菜单也无法完全补救视觉疲劳问题。包容性设计要求在第一稿线框图阶段，就把"谁会被这个设计排斥"作为必答问题。

**误区三：只需要关注视觉和听觉障碍。**
游戏UI的包容性设计还必须覆盖**认知无障碍**（cognitive accessibility），这一维度常被忽视。认知无障碍涉及阅读障碍（提供无衬线字体选项）、注意力差异（减少同屏闪烁元素数量，控制在每秒3次以下以符合WCAG光敏癫痫标准）和工作记忆限制（在UI中持续显示当前目标而非依赖玩家记忆）。这些需求的用户基数庞大，却在许多团队的无障碍讨论中几乎缺席。

## 知识关联

**与无障碍设置菜单的关系**：无障碍设置菜单是玩家接触包容性设计成果的直接界面，承载了包容性设计在可配置性层级上的第二档和第三档实现。然而，如果设计团队将包容性思考的时机推迟到"需要填充无障碍设置菜单时"，就已经错过了成本最低、影响最深远的介入窗口——即概念设计阶段。包容性设计思维是决定无障碍设置菜单**设什么、为什么设、默认值如何定**的上游逻辑。

**与游戏UI/UX整体的关系**：包容性设计思维改变了用户研究（User Research）的招募方式——要求受测者样本覆盖不同感官能力、年龄段和使用设备，而非默认招募"18-35岁标准玩家"。它也直接影响设计系统（Design System）的建立方式，推动团队将WCAG色彩对比度公式（相对亮度比 = (L1 + 0.05) / (L2 + 0.05)，其中L1为较亮色的相对亮度，L2为较暗色）纳入调色板的自动校验规则，而非依赖人工目视检查。
