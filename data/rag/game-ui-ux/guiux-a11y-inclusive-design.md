# 包容性设计思维

> **领域**: 游戏UI/UX > 无障碍设计 | **难度**: ⭐⭐ | **预计学习时间**: 21分钟

## 核心概念

从设计之初考虑多样性需求的设计方法论

## 无障碍维度

1. **视觉** — 色盲、低视力、全盲
2. **听觉** — 听力受损、全聋
3. **运动** — 上肢障碍、精细运动困难
4. **认知** — 注意力障碍、学习困难、阅读障碍

## 设计标准

- **WCAG 2.1 AA** — Web内容无障碍指南(也适用于游戏UI)
- **Xbox Accessibility Guidelines** — 微软游戏无障碍最佳实践
- **CVAA** — 21世纪通信和视频无障碍法案

## 基本要求清单

- [ ] 色盲模式(红绿色盲最常见, 8%男性)
- [ ] 可调字体大小(最小到200%)
- [ ] 字幕选项(大小/背景/说话人标识)
- [ ] 控制器完全重映射
- [ ] 自动瞄准/辅助瞄准选项
- [ ] 屏幕阅读器兼容(基础菜单导航)

## 案例标杆

- **The Last of Us Part II** — 60+无障碍选项, 行业标杆
- **Forza Horizon 5** — 盲人可游玩的赛车游戏
- **Celeste** — 辅助模式(无敌/无限冲刺)

## 关键术语

- **包容性设计思维**: 从设计之初考虑多样性需求的设计方法论

## 学习建议

1. 先理解基本概念和设计原则
2. 分析优秀游戏中的实际案例
3. 尝试在自己的项目中实践
4. 收集用户反馈并迭代优化
