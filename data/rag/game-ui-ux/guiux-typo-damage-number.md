# 伤害数字设计（Floating Combat Text）

## 概述

伤害数字（Floating Combat Text，简称FCT）是MMORPG、动作RPG和Roguelite等战斗类游戏中用于实时反馈战斗数值的浮动UI元素，以飞舞的数字形式向玩家呈现当前攻击造成的伤害量、治疗量、格挡值、魔法消耗或增益/减益触发事件。FCT的历史可追溯至暴雪《魔兽世界》于2004年11月正式上线时内置的战斗文本模块，该功能将此前仅存在于战斗日志文本框中的数值"搬到"了角色头顶，使玩家平均判断当前输出效率的反应时间从分析文字日志的2~3秒缩短至约0.2秒的直觉感知。此后，FCT成为战斗类游戏UI设计的行业标准范式，从《暗黑破坏神3》的对数动态字号到《原神》的元素反应多层数字，均建立在同一套视觉感知原理之上。

FCT的设计目标是在**不遮挡游戏画面关键信息**的前提下，向玩家传递精确且分层的数值反馈。一个设计合理的FCT系统需同时处理三类信息层级：普通伤害（低优先级）、暴击伤害（中优先级）、特殊机制触发伤害如背刺或弱点爆破（高优先级）。这三个层级在字号、颜色、动画幅度上必须具备清晰的视觉差异，否则玩家在同屏数十个数字并发的密集战斗中会完全丧失数值感知能力。

参考文献：Robin Hunicke 等人在 GDC 2004 论文《MDA: A Formal Approach to Game Design and Game Research》中指出，战斗数值反馈属于"Mechanics→Dynamics→Aesthetics"框架中的即时动态反馈层，其信息编码效率直接影响玩家的 Flow 状态维持。

## 核心原理

### 字号与层级对比系统

伤害数字的字号设计遵循**感知对比原则（Perceptual Contrast Principle）**：人眼在快速移动的视觉场景中，能稳定区分的最小尺寸差异阈值约为**40%**——从24px到26px的变化在400ms内的运动数字中几乎不可见，而24px到34px（约42%增幅）则能被可靠识别。因此业界通行的三级字号设定通常为：

- **普通伤害**：基础字号，例如 24px（或归一化单位 1.0×）
- **暴击伤害**：基础字号的 1.5×，即 36px
- **特殊机制触发**（弱点爆破、元素反应、连击奖励）：基础字号的 2.0×，即 48px

《暗黑破坏神3》的FCT系统采用动态字号策略，数字大小与实际伤害值之间呈**对数关系**，公式为：

$$
\text{displaySize} = \text{baseSize} \times \left(1 + k \cdot \log_{10}\left(\frac{D}{D_{\text{ref}}}\right)\right)
$$

其中 $D$ 为本次实际伤害值，$D_{\text{ref}}$ 为参考基准伤害（通常为该角色当前等级的平均单击伤害），$k$ 为缩放系数（《暗黑破坏神3》约取 0.3）。该公式保证百万级伤害数字的显示尺寸不会超过基础字号的 2 倍，避免数字撑满屏幕遮挡游戏画面。

### 颜色编码规范

颜色是FCT系统中传递伤害类型最高效的信息通道，其优势在于颜色的前注意属性（Pre-attentive Attribute）——大脑在无需主动聚焦的情况下即可在约 50ms 内区分颜色差异（参考 Colin Ware《Information Visualization: Perception for Design》第3版，2012）。业界通行的颜色约定如下：

| 数字类型 | 常用颜色 | 典型游戏实现 |
|---|---|---|
| 普通物理伤害 | 白色 / 浅黄色 | 《魔兽世界》白色，《原神》白色 |
| 暴击伤害 | 橙色 / 红色 | 《英雄联盟》橙红色 |
| 治疗量 | 绿色 | 几乎所有MMORPG通用 |
| 魔法 / 元素伤害 | 蓝色、紫色、红色（元素对应） | 《原神》按七种元素分色 |
| 持续伤害（DoT/毒素） | 绿色（毒）/ 暗紫色（诅咒） | 《暗黑破坏神4》 |
| 护盾吸收 / 格挡 | 灰色 / 金色 | 《魔兽世界》金色Miss/Dodge |

颜色选取**必须**考虑色盲可访问性：红绿色盲（Deuteranopia/Protanopia）影响约 8% 的男性用户，纯红/绿区分对其完全不可见。《最终幻想14》在 2021年5月的 5.5 版本辅助功能更新中专门引入了FCT色盲模式，将暴击颜色从橙红改为亮黄，并为治疗数字添加了形状符号（⊕）作为第二编码维度。

### 弹出轨迹与动画时序

FCT的运动轨迹直接决定信息可读性与战斗感的传递效果。标准的三段式浮动动画如下：

1. **出现阶段（0~100ms）**：数字从 scale=0.5 弹性放大至 scale=1.2，配合 opacity 从 0 升至 1，产生"砸下来"的冲击感。
2. **飘移阶段（100~600ms）**：沿 Y 轴向上匀速移动约 80~120 像素（以 1080p 分辨率为参考），此阶段字号和透明度保持稳定，是玩家实际读取数字的主要窗口。
3. **消隐阶段（600~800ms）**：opacity 从 1.0 线性淡出至 0，同时可配合轻微的继续上移（约 20px）避免数字突然消失的跳脱感。

暴击数字在出现阶段增加水平方向随机偏移（±20px）和更强烈的缩放弹跳（scale从 0.3 → 1.6 → 1.0），以模拟"重击砸入"的物理感。**整个动画周期控制在 800ms 以内**是业界共识，超过 1000ms 的FCT会在 AoE 群体战中产生严重视觉堆积，导致屏幕被过期数字填满。

下面是一段描述FCT动画时序的伪代码示例：

```javascript
// FCT 动画时序控制（以 800ms 总周期为例）
function playFCT(damageValue, isCrit, startPos) {
  const baseDuration = 800; // ms
  const floatOffset = isCrit ? 130 : 100; // px，暴击浮动更高
  const scaleFrom = isCrit ? 0.3 : 0.5;
  const scalePeak = isCrit ? 1.6 : 1.2;

  // 阶段一：出现弹性放大（0~100ms）
  tween(node.scale, scaleFrom, scalePeak, 60, easeOutBack);
  tween(node.scale, scalePeak, 1.0, 40, easeInBack);

  // 阶段二：飘移上升（100~600ms）
  tween(node.posY, startPos.y, startPos.y + floatOffset, 500, easeLinear);

  // 阶段三：消隐（600~800ms）
  tween(node.opacity, 1.0, 0.0, 200, easeInQuad);

  // 暴击附加水平随机偏移
  if (isCrit) {
    const jitter = (Math.random() - 0.5) * 40; // ±20px
    tween(node.posX, startPos.x, startPos.x + jitter, 100, easeOutQuad);
  }
}
```

## 暴击的多维视觉强化

暴击表现不仅依赖字号和颜色，还需要配合多个感官通道形成完整的反馈闭环：

- **描边（Stroke）加粗**：暴击数字描边宽度通常为普通数字的 2~3 倍（例如普通 1px → 暴击 2.5px），增强在复杂背景下的可读性。
- **投影（Drop Shadow）扩散**：暴击数字的投影模糊半径扩大至普通数字的 3 倍（例如 2px → 6px），形成"发光"感。
- **屏幕微震（Camera Shake）**：震幅约 2~4px，持续 50~80ms，仅在暴击超过当前平均伤害 150% 时触发，避免频繁震动引发疲劳（《黑暗之魂3》的暴击几乎不震屏，而《深岩银河》则在击杀弱点时有明显震动反馈）。
- **音效层级**：暴击音效的峰值音量比普通攻击高约 3~6dB，并附加短促的音调提升（pitch shift +2~3 semitones）以形成听觉强调。
- **粒子辅助**：《英雄联盟》的暴击数字周围会额外出现放射状粒子（约 8 个短线条），将视觉信息从字体层扩展至特效层。

对于"超暴击"或技能特效触发的极大伤害，部分游戏会临时将数字改为金色或彩虹渐变色，并将停留时间延长至 1.2~1.5 秒（正常暴击为 0.8 秒），让玩家充分感知高光时刻。

## 实际应用案例

**案例一：《原神》的多元素并发FCT策略**

《原神》的战斗中，同一帧可能同时出现火、水、雷、冰四种元素反应的伤害数字。其FCT系统采用**队列偏移策略**：每个新出现的数字在 X 轴上相对前一个随机偏移 15~30px，Y 轴起始位置也有 ±10px 的随机抖动，防止完全重叠，形成自然的阶梯式排布，同时以元素颜色（火=橙红、水=蓝、雷=紫、冰=青）作为快速识别的第一维度，以字号区分元素反应伤害（1.8× 基础）与直接伤害（1.0×）。

**案例二：《魔兽世界》的开放AddOn生态**

《魔兽世界》允许玩家通过 AddOn 系统（如 MikScrollingBattleText / MSBT、ScrollingCombatText / SCT）完全自定义FCT的颜色规则、字号、滚动方向、过滤阈值。例如，高端副本玩家会将低于 5000 点的普通伤害完全隐藏，仅显示暴击和特殊机制触发的数字，以此减少屏幕视觉噪声。这一开放设计验证了玩家对FCT个性化的强烈需求，也促使暴雪在《魔兽世界》8.0 版本将内置FCT选项扩展至 12 个可配置参数。

**案例三：手机游戏的缩减策略**

在 1280×720 的手机屏幕上，FCT的设计面临额外约束：可用显示面积仅为 PC 1080p 的约 44%，而触控操作时拇指会遮挡屏幕下方约 20% 区域。《明日方舟》的解决方案是将FCT的浮动方向改为斜向右上方（约 60° 角），避免与屏幕底部的操作按钮区域重叠，同时将动画总时长从 800ms 压缩至 600ms，降低密集战斗中的视觉堆积。《崩坏：星穹铁道》则采用了更激进的策略：默认关闭普通攻击的FCT显示，仅保留暴击、弱点击破和关键增益触发的数字，将屏幕视觉复杂度降低约 60%。
