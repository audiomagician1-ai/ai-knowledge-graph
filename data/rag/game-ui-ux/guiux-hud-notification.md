# 通知与提示系统

## 概述

通知与提示系统（Notification & Toast System）是游戏HUD中负责向玩家传递即时、非阻断性信息的机制，包括任务更新、成就解锁、状态变化、队伍事件等各类短暂消息的展示与管理。与全屏弹窗或对话框不同，通知设计的核心约束是"不打断玩家操作流"——信息必须在玩家视野边缘可见，但绝不能遮挡战斗核心区域或输入焦点。

该系统最早在RTS游戏中以"事件日志"形式出现（《星际争霸》1998年使用左下角滚动文字列表），后来《魔兽世界》（2004年）引入了浮动"飘字"（Floating Text）和屏幕边缘Toast提示，奠定了现代游戏通知系统的基本形态。移动游戏时代进一步强化了这一系统，因为手机屏幕更小，通知与游戏内容的空间竞争更加激烈。

通知系统的设计质量直接影响玩家的信息过载程度。研究表明，玩家在高强度战斗中每秒最多可以无意识处理2~3条视觉提示，超过这一阈值的通知会被完全忽略。因此，优先级队列和显示时机的控制不只是UI美化问题，而是影响玩家决策质量的核心机制。

## 核心原理

### 优先级队列（Priority Queue）

通知系统本质上是一个带优先级的消息队列，每条通知在入队时会被赋予一个优先级数值（通常为1~5级或枚举类型）。优先级定义示例：

- **P1（紧急）**：队友死亡、自身受到致命威胁、目标点被占领
- **P2（重要）**：技能冷却完毕、任务阶段更新
- **P3（普通）**：经验值获得、物品拾取
- **P4（装饰）**：成就进度、每日签到奖励

当同一时间存在多条待显示通知时，系统按优先级降序出队。同优先级的通知通常采用**FIFO（先进先出）**原则，但部分系统会对同类型通知进行**合并（Coalescing）**：例如连续5条"获得金币"通知会被合并为"获得金币 ×5"，减少屏幕噪声。

合并逻辑的关键参数是**合并窗口（Merge Window）**，通常设置为200~500毫秒——在此时间窗口内到达的同类通知会被打包。窗口过短导致通知堆积，过长则让玩家感觉反馈延迟。

### 展示时机与持续时长

每条通知的**生命周期**由三段时间构成：

```
[进入动画] + [驻留时间] + [退出动画]
```

驻留时间根据信息密度动态计算，经典公式为：

> **T_display = T_base + N_chars × T_per_char**

其中 T_base 通常为1.5秒，T_per_char 约为50~80毫秒/字符。一条12个字符的普通通知驻留约2.1~2.5秒，而一条30字符的任务说明则驻留约3.0~3.9秒。

对于P1级紧急通知，系统通常会**中断（Interrupt）**当前正在显示的低优先级通知，将其立即推入退出动画，腾出显示槽位。这一机制称为**通知抢占（Preemption）**。

### 消退动画与空间布局

通知的进入和消退动画不仅是视觉效果，更是功能性信息编码：

- **上滑消退**（最常见）：表示通知"归档完成"，适用于普通提示
- **向右消退**：在许多西方游戏中代表"确认/完成"，例如《火箭联盟》的成就解锁
- **闪烁后消失**：表示紧急程度高，适用于P1通知
- **缩放消失（Scale-to-0）**：用于轻量级飘字，不占据固定布局槽位

多条通知并存时，需要**垂直栈布局（Vertical Stack）**管理空间。新通知从底部插入，旧通知向上位移，这个位移动画的缓动函数通常采用 `ease-out`（快入慢出），持续约150~200毫秒，保证画面不因通知的突然位移而产生视觉跳跃。

## 实际应用

**《英雄联盟》的通知系统**将通知区域分为三个独立通道：屏幕左侧边栏（击杀播报，P1）、屏幕正上方（地图目标，P2）、屏幕右下角（物品/等级，P3）。三个通道完全独立队列、独立显示，互不阻塞，从而实现多条高优先级信息同时呈现而不相互竞争。

**《原神》的通知系统**采用了典型的单通道右侧垂直栈，最多同时显示4条通知，第5条必须等待最早的通知退场后才能进入。针对移动端，通知字号为32px（PC端为24px），触控目标尺寸满足48×48dp的最小要求，同时通知本身可以向右滑动提前关闭（手势消退）。

**《命运2》的浮动战斗通知**则展示了空间型通知的设计：伤害数字、暴击提示、状态效果直接叠加在3D世界坐标上，随摄像机运动发生视差偏移，持续约0.8秒后向上漂移并淡出。这类通知与HUD通知是两套独立系统，避免相互影响刷新频率。

## 常见误区

**误区一：通知越多反馈越丰富**
许多开发者误以为增加通知数量能让玩家感受到"系统在响应我"。实际上，当屏幕上同时有4条以上通知时，眼动追踪数据显示玩家的注视点会完全回避通知区域。正确做法是对P3~P4通知严格合并，保持通知区域80%时间处于空白状态，用"稀缺性"提升每条通知的感知价值。

**误区二：固定驻留时间适用于所有平台**
PC端2秒的驻留时间在移动端可能完全不够——玩家在移动端的单次游戏时长更短、操作干扰更多，通知被遮挡的概率也更高。《荒野乱斗》的通知驻留时间在移动端比PC端延长了约40%（从1.8秒调整至2.5秒），同时字号也相应增大。

**误区三：通知动画是纯粹的装饰**
消退方向、速度和缓动函数实际上在编码语义信息。如果任务完成和任务失败的通知使用相同的退出动画，玩家在外周视觉中无法区分两者，只有主动阅读文字才能确认结果。不同类型通知应使用可被外周视觉识别的差异化动画——例如绿色上滑代表成功，红色向左弹出代表失败或警告。

## 知识关联

通知系统以**队伍信息显示**为直接前置概念：队伍事件（队友死亡、状态变化、目标分配）是游戏内高频通知的主要来源之一，队伍信息面板决定了哪些信息已通过持久化UI展示，从而影响通知系统对同类信息的**去重逻辑**——已经在队伍面板中实时可见的数据（如队友血量）不应再重复以通知形式弹出，否则会造成冗余。

从数据结构角度，通知系统的优先级队列实现与算法课程中的**最大堆（Max-Heap）**结构直接对应，插入和弹出复杂度均为 O(log n)，适合实时游戏场景中每帧可能多次触发的通知入队操作。理解这一底层结构有助于开发者在优化通知系统性能时做出正确的数据结构选型决策。
