# 弹簧物理动效

## 概述

弹簧物理动效（Spring Physics Animation）是一种以弹簧阻尼系统（Spring-Damper System）为数学模型驱动UI元素运动的动画技术，其核心特征是位移会超越目标值（Overshoot）后产生衰减振荡，最终收敛至静止状态。与传统贝塞尔曲线缓动（Cubic Bezier Easing）本质不同的是：贝塞尔动画的持续时长由设计师预先指定，弹簧动效的持续时长由物理参数与初始速度实时计算得出，因此可以在动画中途无缝接收新的目标位置，避免突兀的运动切断——这一特性使其成为游戏UI中处理连续交互状态变化的首选方案。

弹簧动效在UI领域的工程化应用成熟于2013年至2014年间。苹果公司在iOS 7全系统视觉重设计中将弹簧动效列为系统级标准动效，`UIView` 新增了 `animateWithDuration:delay:usingSpringWithDamping:initialSpringVelocity:options:animations:completion:` API，其中 `dampingRatio` 参数直接对应物理中的阻尼比 $\zeta$。2014年，Facebook工程师Kimon Tsinteris和Brandon Walkin开源了Pop动画库，将同样的弹簧模型封装为 `POPSpringAnimation`，并定义了 `springBounciness`（对应欠阻尼程度，范围0–20）和 `springSpeed`（对应系统响应速度，范围0–20）两个直觉化参数，推动弹簧动效在Android生态和游戏引擎UI层中大规模普及（Walkin, 2014）。

在游戏UI具体实践中，弹簧动效的"有质量感"特征使界面元素表现出真实物理实体的行为特征。按下按钮时的压缩回弹、奖励宝箱盖子的弹开、背包道具拖拽后的归位——这些交互的视觉说服力正是来自弹簧方程所保证的运动连续性和超调量，而非人为调整的曲线形状。

## 核心原理

### 弹簧阻尼微分方程

弹簧物理动效的运动基础是二阶线性常微分方程，描述了质量–弹簧–阻尼器三元系统的动力学行为：

$$m\ddot{x} + c\dot{x} + kx = F_{ext}$$

在UI动效语境中，$F_{ext}$ 通常为零（自由振动），方程退化为：

$$m\ddot{x} + c\dot{x} + kx = 0$$

各参数含义如下：
- $m$（mass，质量）：元素的惯性大小，质量越大启动越慢但惯性越强
- $c$（damping coefficient，阻尼系数）：摩擦阻力，决定振荡衰减速率
- $k$（stiffness，刚度）：弹簧恢复力强度，刚度越高运动越"硬朗"短促
- $x$：当前位置与目标位置的偏差
- $\dot{x}$：速度，$\ddot{x}$：加速度

该方程的通解在欠阻尼条件下为：

$$x(t) = e^{-\zeta\omega_n t}\left(A\cos(\omega_d t) + B\sin(\omega_d t)\right)$$

其中 $\omega_n = \sqrt{k/m}$ 为固有角频率，$\omega_d = \omega_n\sqrt{1-\zeta^2}$ 为有阻尼角频率，常数 $A$、$B$ 由初始位置和初始速度决定（Strogatz, 2015）。

### 阻尼比与三种运动状态

系统的本质行为由**阻尼比** $\zeta$（zeta）唯一决定：

$$\zeta = \frac{c}{2\sqrt{mk}}$$

**欠阻尼（$\zeta < 1$）**：动效越过目标值后产生衰减振荡，$\zeta$ 值越低振荡次数越多、超调量越大。游戏UI中最常用的区间为 $\zeta \in [0.3, 0.7]$，此区间内单次明显超调后迅速收敛，兼顾弹跳感与稳定性。《原神》道具获得弹窗的弹入动效实测阻尼比约为 0.55。

**临界阻尼（$\zeta = 1$）**：系统以理论上最快速度抵达目标，恰好不产生超调。适合导航菜单展开、模态框出现等需要"干净利落"响应的场景，iOS系统默认的 `dampingRatio: 1.0` 即此状态。

**过阻尼（$\zeta > 1$）**：元素缓慢爬行趋近目标，运动沉滞。在游戏UI中基本不使用，因其会让界面元素感觉像在黏稠液体中移动，破坏操作即时性感知。

### 数值积分：Verlet方法在实时渲染中的应用

游戏引擎中弹簧动效通常不直接求解解析解，而是在每帧以数值积分推进状态。速度Verlet（Velocity Verlet）方法因其能量守恒性优于简单欧拉法，是Unity、Unreal等引擎弹簧模拟的标准实现：

$$x_{n+1} = x_n + v_n \Delta t + \frac{1}{2}a_n \Delta t^2$$

$$v_{n+1} = v_n + \frac{a_n + a_{n+1}}{2}\Delta t$$

其中加速度 $a_n = -(k \cdot x_n + c \cdot v_n)/m$。当帧率波动时，固定物理时间步（Fixed Timestep，如 Unity 默认 0.02s）配合插值可保证弹簧行为帧率无关，避免高帧率下弹簧"发散爆炸"的数值不稳定问题。

## 关键参数与公式

### 主流UI框架的参数体系对照

不同框架对同一物理方程采用了不同的参数暴露方式，理解其底层等价关系对跨平台复现动效至关重要：

| 框架 | 参数名称 | 物理对应 |
|------|----------|----------|
| iOS UIKit | `dampingRatio` (0–1), `initialVelocity` | $\zeta$，$\dot{x}(0)$ |
| React Spring | `tension`, `friction`, `mass` | $k$，$c$，$m$ |
| Facebook Pop | `springBounciness` (0–20), `springSpeed` (0–20) | $\zeta$ 的映射，$\omega_n$ 的映射 |
| Framer Motion | `stiffness`, `damping`, `mass` | $k$，$c$，$m$ |
| Unity (DOTween) | `Elastic` ease 或自定义 Spring Modifier | 近似弹簧，非精确物理 |

React Spring 的 `tension` 与 `friction` 和物理参数的换算关系为（Chen, 2019）：

$$k = \text{tension}, \quad c = \text{friction}, \quad m = \text{mass（默认为1）}$$

因此阻尼比为 $\zeta = \text{friction} / (2\sqrt{\text{tension} \times \text{mass}})$。

例如，React Spring 常用预设 `{ tension: 170, friction: 26 }` 对应 $\zeta = 26/(2\sqrt{170}) \approx 0.997$，接近临界阻尼，适合抽屉菜单的展开收起。而游戏UI常用的弹跳预设 `{ tension: 300, friction: 20 }` 对应 $\zeta = 20/(2\sqrt{300}) \approx 0.577$，产生单次明显超调。

### 初始速度的接力传递

弹簧动效相对贝塞尔动效最重要的工程优势之一，是支持以当前运动速度为初始速度无缝启动新动画。当用户快速滑动列表后松手，惯性速度 $v_0$ 可直接注入弹簧方程作为 `initialVelocity`，元素会以自然的减速弧线停靠在目标位置，而非从零速度重新启动产生的"跳接"感。iOS的 `UIScrollView` 回弹效果正是以此机制实现，刚度约为 $k=500$，阻尼比 $\zeta \approx 0.75$（通过逆向工程测量）。

### 结束阈值与动效截止

弹簧动效理论上永远不完全停止（指数衰减趋近零），实践中需定义**结束阈值**（Settling Threshold）。通常当 $|x| < \epsilon_x$（位移阈值，如1像素）且 $|\dot{x}| < \epsilon_v$（速度阈值，如像素/秒）同时满足时，系统认定动画结束并冻结在目标值。React Spring 默认 $\epsilon_x = 0.01$，$\epsilon_v = 0.01$。游戏UI中若大量弹簧动效同时运行，适当放大阈值（如0.5像素）可减少CPU/GPU的无效帧计算。

## 实际应用

### 游戏UI场景的参数选型

**按钮按压反馈**：玩家点击按钮时，按钮缩小至0.92倍后弹回1.0倍。推荐参数：$k=600$，$\zeta=0.4$，$m=1$。高刚度保证快速响应（总动效约150ms），低阻尼比产生单次过冲至约1.03倍后归位，触感锐利。

**道具卡片弹出**：从屏幕外飞入并在目标位置弹跳停止。推荐：$k=200$，$\zeta=0.55$，配合 `initialVelocity` 设为卡片飞入速度。单次超调量约12%，视觉重量感与反应速度均衡。

**胜利界面标题放大**：标题文字从0.5倍缩放弹跳至1.0倍。推荐：$k=150$，$\zeta=0.35$，产生2–3次明显振荡，强化庆祝情绪张力。《英雄联盟》击杀提示动效采用类似参数配置（阻尼比约0.38，实测振荡约2.5次）。

**背包格子拖拽归位**：道具放错位置后弹回原格。推荐：$k=400$，$\zeta=0.7$，$m=1$。高阻尼比减少过冲（约5%），强调"归位准确性"而非弹跳表演。

### 串联与并联弹簧动效

复杂UI元素（如带有子组件的弹窗）可以通过对多个属性施加弹簧动效并施加时间偏移（Stagger Delay）来构建层次感。例如弹窗容器以 $\zeta=0.6$ 弹入，内部图标延迟80ms以 $\zeta=0.35$ 弹入，内部文字再延迟60ms淡入——三层时序产生"容器先到位，内容随后安顿"的视觉叙事，比单一动效层次更丰富。Framer Motion 的 `staggerChildren` API 直接支持此模式。

### 案例：《Pokémon GO》道具获得动效

《Pokémon GO》道具获得弹窗经逐帧分析（60fps录屏测量）呈现以下特征：弹窗从0.3倍缩放至1.08倍（超调8%）后回落至1.0倍，总时长约420ms，期间无二次振荡。这对应约 $\zeta=0.65$，$k \approx 220$，$m=1$ 的参数组合。道具图标有独立弹簧动效，延迟120ms启动，超调量更大（约15%），$\zeta \approx 0.45$——形成弹窗稳定后图标"蹦出"的双层惊喜感。

## 常见误区

### 误区一：用弹簧动效替代所有缓动

弹簧动效并非万能。**信息展示类动效**（如加载进度条、文字渐显）不需要超调特性，使用弹簧反而会让进度条超出100%后回弹，造成数据误读。**状态切换类动效**（如开关Toggle）若加入振荡会让用户误判状态尚未稳定。弹簧动效的适用核心是：元素需要传递"物理重量"或"情感强度"的场景。
