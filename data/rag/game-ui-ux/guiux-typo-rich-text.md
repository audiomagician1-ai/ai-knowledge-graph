# 富文本系统

## 概述

富文本系统（Rich Text System）是游戏UI中允许单条字符串同时携带多种视觉属性的渲染机制。与纯文本不同，富文本通过解析嵌入字符串的标签指令，在同一文本块内实现颜色切换、图标内嵌、字体大小变化和超链接响应等效果，无需为每个差异化片段创建独立的UI控件。

富文本标签语法最早在网页的HTML体系中成熟，后被游戏引擎吸收并简化。Unity引擎在2013年的4.x版本中正式引入TextMesh Pro（TMP）前身的富文本支持，其标签格式沿用类HTML的尖括号语法，例如`<color=#FF0000>文字</color>`。虚幻引擎则在UMG系统中提供了`<RichTextBlock>`组件，通过DataTable驱动自定义内联装饰器。

在游戏场景中，富文本系统的实用价值体现在技能描述、对话字幕和装备词条三个高密度文字场景。一条典型的MMORPG技能描述需要同时高亮伤害数值（黄色）、冷却时间（蓝色）、关键词超链接（带下划线）和属性图标，若用独立Label拼接则布局成本极高，富文本将这一切压缩进一个字符串资源。

## 核心原理

### 标签解析管线

富文本渲染器在将字符串送入字体光栅化之前，先执行一次**标签扫描（Tag Scan）**阶段。解析器从左至右遍历字符串，遇到`<`符号时进入标签读取模式，识别标签名和属性值，然后将该标签区间从可见字符流中剥离，将其属性压入一个**样式状态栈（Style State Stack）**。遇到闭合标签时弹栈，恢复前一状态。这个栈机制保证了嵌套标签的正确作用域，例如：

```
<color=red>攻击力 <b>+120</b> 点</color>
```

在此例中，加粗状态在红色状态的作用域内，弹出`</b>`时仅恢复粗细，颜色状态保持红色直到`</color>`。

### 内联图标的图集映射

图标内嵌不是真正的图片插入，而是将特定Unicode私用区（Private Use Area，U+E000至U+F8FF）中的字符映射到一张**图标图集（Icon Atlas）**的对应UV区域。TextMesh Pro使用Sprite Asset实现此功能：`<sprite="ItemIcons" name="sword">`标签会在当前文字光标位置插入一个与行高等比的精灵四边形，其顶点UV指向sword图标在图集中的坐标。图标与文字共享同一个网格批次的前提是两者使用相同的着色器通道，否则会触发额外的Draw Call，每个不同Sprite Asset引入1次额外批次。

### 超链接的事件注册机制

超链接标签`<link="item_id_1024">`不改变字符的光栅化方式，而是在解析阶段将被标签包围的字符范围（起始字符索引至结束字符索引）登记进一个**链接信息列表（LinkInfo Array）**。运行时点击事件发生时，UI系统将屏幕坐标转换为文本局部坐标，遍历LinkInfo Array检测落点是否在某段字符的包围盒内，匹配后返回该链接携带的ID字符串（此处为`"item_id_1024"`），业务层再凭ID查询物品数据并弹出提示框。这一机制要求富文本控件必须开启**射线投射（Raycast Target）**选项，否则点击事件穿透。

### 颜色标签的十六进制与命名色

颜色值支持三种格式：6位十六进制`#RRGGBB`、8位含Alpha的`#RRGGBBAA`，以及引擎预定义的颜色关键字如`red`、`yellow`。需要注意Alpha通道默认为`FF`（完全不透明），若使用`#FF000080`则得到50%透明红色，这在半透明提示文字中常用。颜色标签可以与`<gradient>`标签组合，TMP的`<gradient>`接受一个Gradient Asset名称，在单词范围内插值渐变色，常见于Boss名称的金色渐变效果。

## 实际应用

**技能描述词条**：《暗黑破坏神4》的技能面板使用颜色区分数值与说明文字，关键词如"中毒"、"易伤"是可点击的富文本超链接，点击后展开该词条的定义浮窗。整个面板的技能描述存储为单一本地化字符串，本地化团队在翻译时保留标签结构，只替换自然语言内容。

**对话字幕的说话人高亮**：在JRPG对话系统中，说话人名字通常以独立颜色显示，例如`<color=#F5C842>{speaker_name}</color>：{dialogue_text}`，其中`{speaker_name}`在运行时由脚本填入，颜色标签硬编码在模板中。这种设计使策划无需接触代码即可调整颜色方案，只需修改字符串模板。

**装备词条的数值突出**：网游装备UI中，正面属性数值（如`+15%暴击率`中的数字部分）用绿色或金色标注，负面词条用红色。这类字符串在服务器下发时已包含完整标签，客户端富文本控件直接渲染，不需要客户端解析属性逻辑。

## 常见误区

**误区一：富文本标签会被计入字符串长度影响文本溢出计算**
实际上，成熟富文本系统在计算文本布局宽度时只统计**可见字符**的字形宽度，标签字符在扫描阶段已从字符流中剔除。如果遇到标签被计入宽度导致错误截断，通常是因为使用了不支持富文本的基础Text组件而非TMP组件，或者富文本解析开关（Enable Rich Text）未开启。

**误区二：内联图标会增大字体纹理Atlas**
图标使用的是独立的Sprite Atlas，与字体的SDF（Signed Distance Field）纹理图集完全分离。字体Atlas只存储字形的灰度SDF数据，图标Atlas存储RGBA彩色精灵。两者互不影响，扩充图标集不会触发字体Atlas重建，也不会影响字体的内存占用。

**误区三：嵌套颜色标签颜色会混合叠加**
颜色标签采用**替换语义**而非混合语义，内层`<color=blue>`完全覆盖外层`<color=red>`，并非将红色与蓝色混合。闭合内层标签后，颜色回到外层红色，这是栈式状态管理的直接结果。期望实现混合效果需要在美术侧预先计算出混合后的十六进制值并写入标签。

## 知识关联

**前置概念：文本溢出处理**
文本溢出处理确定了富文本控件的可用渲染区域。富文本系统在溢出截断时需要特别处理未闭合标签的情况——若截断点落在`<color>`标签内部，渲染器必须在截断处插入虚拟闭合标签，防止颜色状态泄漏到后续UI元素。掌握溢出处理中的字符索引计算方式，是理解富文本截断安全性的前提。

**后续概念：伤害数字设计**
伤害数字系统是富文本能力的专项延伸，它将颜色区分（暴击金色、普通白色、治疗绿色）和数字动画结合，通常不直接使用静态富文本标签，而是将富文本的颜色映射逻辑内化为程序驱动的顶点色注入，以获得帧级别的颜色动画控制，这超出了标签系统的静态表达能力。
