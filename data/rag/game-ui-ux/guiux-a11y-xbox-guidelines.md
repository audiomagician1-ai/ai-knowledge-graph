# Xbox无障碍指南

## 概述

Xbox无障碍指南（Xbox Gaming Accessibility Guidelines，简称XGAG）是由微软Xbox团队于2020年正式发布的一套针对游戏开发者的无障碍设计规范，涵盖22个独立功能类别，旨在确保残障玩家能够充分参与Xbox平台上的游戏体验。该指南由微软与全球辅助技术社区合作编制，并在Xbox的游戏认证流程中被引用为参考标准，尽管大多数条目目前属于"推荐"而非"强制"级别。

XGAG的诞生背景源于微软2018年发布的Xbox Adaptive Controller的市场教育需求——硬件层面的适配若缺乏软件层面的相应支持，辅助控制器的使用效果将大打折扣。因此，该指南从运动与操控、视觉、认知、听觉四大维度展开，与Xbox Adaptive Controller的输入逻辑保持一致，使硬件与游戏设计标准协同工作。

对游戏开发者而言，满足XGAG中的关键条目是通过Xbox Game Pass认证及进入Xbox商店的重要参考依据之一，微软也在Xbox成就系统中设置了"无障碍标签"功能，帮助玩家在商店页面筛选支持特定无障碍功能的游戏，这直接推动了开发者对指南的重视程度。

## 核心原理

### 22个功能类别的结构体系

XGAG将所有要求划分为22个功能类别，每个类别下设若干条具体指导项。这22个类别包括：字幕与字幕内容、音频描述、视觉对比度、可调字体大小、色盲模式、重映射控制按键、按键辅助（如连按转换为单次触发）、难度调整、游戏暂停功能、输入计时容忍度等。每个类别中的条目标注为"基础（Foundation）"、"进阶（Advanced）"或"完全（Full）"三个达成级别，让开发团队可以根据项目体量选择目标覆盖度。

### 控制可达性核心要求

XGAG对控制方案的要求尤为细化，其中规定游戏必须支持**全按键重映射（Full Button Remapping）**，即玩家可以将任意功能指令指派到任意物理按键上，包括Xbox Adaptive Controller外接的扩展输入端口。此外，指南要求游戏不得依赖同时按住三个或三个以上按键的组合操作作为核心游戏机制，因为多指同按对运动障碍玩家而言往往无法实现。指南还专门为"一键式操作（One-Switch Access）"设定了支持建议，即游戏可通过单一输入信号完成基本流程。

### 视觉无障碍的量化标准

XGAG在视觉部分引用了WCAG 2.1的对比度标准，要求关键UI元素的文字与背景之间的对比度至少达到**4.5:1**（普通文字）或**3:1**（大字体文字，18pt以上）。对于色盲支持，指南要求游戏提供至少涵盖红绿色盲（Deuteranopia/Protanopia）的颜色替代方案，并明确禁止单独依赖颜色传递关键游戏信息——例如，地图中的敌我区分不能只靠红色和绿色，必须同时提供形状或图标差异。字体大小方面，指南建议游戏内UI文字在标准1080p分辨率下不得小于**24像素**，且必须提供字体缩放选项，缩放范围至少覆盖80%至140%的基准大小。

### 认知无障碍与游戏节奏

XGAG的认知类别要求游戏提供可关闭的"导航辅助"或目标提示功能，并规定在教程或过场动画中不得出现无法跳过的强制等待超过**15秒**的输入要求。游戏难度方面，指南鼓励开发者提供至少三个可调难度档位，并允许玩家在游戏进行中途随时更改难度，而非仅限于开始新游戏时设置。

## 实际应用

《地平线：零之曙光》PC版和《微软飞行模拟》均以XGAG为参考，在选项菜单中实现了完整的按键重映射界面，并将无障碍选项从通用设置中独立出来，形成专属的"辅助功能"子菜单，这一设计本身即符合XGAG关于"无障碍选项可发现性"的要求。

《光环：无限》在2021年发布时集成了XGAG推荐的"导航辅助"功能，允许玩家开启自动瞄准辅助的强度分级控制，并提供了专属的色盲预设，将游戏中的敌方标记从红色改为高饱和度黄色，以兼容红绿色盲玩家的视觉感知。

在微软内部，Xbox商店的游戏详情页现已支持显示"无障碍功能"徽章，列出该游戏具体实现了XGAG中的哪些类别，例如标注"支持字幕"、"支持按键重映射"、"支持色盲模式"，使玩家购买前即可判断游戏的适配程度，这一机制在2022年随Xbox系统更新正式上线。

## 常见误区

**误区一：认为仅满足字幕要求即可声称"无障碍合规"**。XGAG明确指出，字幕仅属于22个类别中的一项，且字幕本身还细分为"对话字幕"、"音效字幕（Sound Captions）"和"语音角色标注"三个层次。仅添加对话字幕而缺少音效描述（如"[爆炸声]"、"[脚步声来自左方]"），在XGAG评估中仍属于部分达标，无法获得完整的听觉无障碍类别评分。

**误区二：将色盲模式与颜色滤镜等同**。部分开发者通过在整个画面上叠加一层颜色滤镜来实现"色盲支持"，但XGAG明确指出这种方案会同时改变UI、HUD和游戏世界的色彩，导致某些元素在滤镜后反而对比度下降。正确做法是在游戏资产和着色器层面直接为色盲玩家提供差异化的颜色绑定，或为UI单独提供无需滤镜的色彩预设，确保对比度在调整后仍符合4.5:1的最低要求。

**误区三：认为难度调整等同于降低游戏性**。XGAG的难度无障碍类别并不要求游戏删减内容，而是通过参数调节（如降低敌方HP、延长输入窗口时长、提供无敌帧辅助）让内容保持完整的同时降低操作门槛，这与游戏的核心叙事和挑战设计并不矛盾。

## 知识关联

学习Xbox无障碍指南之前，开发者需要具备无障碍测试工具的使用经验，例如能够使用色盲模拟工具（如Coblis或Adobe Color盲视模式）验证游戏中的颜色对比方案，以及能够运用Xbox Dev Kit中内置的辅助功能测试模式对按键重映射逻辑进行验证。这些工具经验使得开发者在阅读XGAG条目时能够将规范描述直接转化为可测试的验收标准。

在掌握XGAG的整体框架后，下一步是深入研究聋人与听障玩家的专项适配策略。XGAG的听觉类别提出了基础框架要求，但针对听障社区的具体设计方案——例如如何在游戏叙事中以视觉化方式替代音频线索、如何设计振动反馈作为声音信号的补充通道——需要进一步的专题研究，这些内容构成了XGAG听觉部分的延伸与落地实践。
