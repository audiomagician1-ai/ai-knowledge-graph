# Xbox无障碍指南

## 概述

Xbox游戏无障碍指南（Xbox Gaming Accessibility Guidelines，简称XGAG）由Microsoft于2017年首次发布，是专门针对Xbox平台（包括主机和Windows PC Game Pass游戏）制定的一套结构化无障碍设计规范。该指南涵盖运动控制、视觉、听觉、认知和语言五大残障类别，将具体要求分为"基础（Foundation）"、"推荐（Recommended）"和"理想（Ideal）"三个优先级层级，帮助开发团队按资源量力而为地推进无障碍适配工作。

XGAG并非强制性平台发布要求，而是Microsoft通过Xbox官方文档中心公开提供的设计建议集合，开发者自愿采纳。但随着2021年The Game Awards新增"最佳无障碍游戏"奖项，以及Microsoft将XGAG提及率纳入ID@Xbox开发者支持评估，该指南的行业影响力持续上升。指南当前版本（2023年修订）包含超过70条具体可操作条目，每条均附有"为什么这很重要"、"实现方式"和"验证方法"三个子说明。

理解XGAG的关键在于：它不只是一份检查清单，而是以Xbox控制器为中心硬件现实出发构建的，所有运动无障碍条目都基于标准Xbox手柄的按键物理布局（A/B/X/Y + 两根摇杆 + 两个扳机 + 两个肩键）来定义功能可重映射性的最低范围。

## 核心原理

### 运动控制无障碍条目

XGAG的运动类条目数量最多，共24条，基础级别中最关键的是条目MO-1：**所有游戏功能必须可通过单根摇杆或单侧按键完成**，即玩家即使仅能操作手柄左半侧或右半侧，也应能执行游戏的核心循环动作。MO-4要求任何"同时按下多个按键"的操作（如长按LT+LB组合技）必须提供可拆分的替代输入方式。

推荐级别的MO-11要求游戏支持"按键重映射"，且重映射范围不得少于手柄上12个可按压输入（不含摇杆倾斜轴）。XGAG特别说明，仅提供预设配置方案（如"战士模式"、"射手模式"）**不等同于**满足MO-11，因为预设方案无法涵盖所有肢体差异组合。

### 视觉无障碍条目

视觉类共18条，其中VO-2（基础级）规定：所有UI文字必须支持至少放大至150%而不产生重叠或裁切，这一数值参考了低视力玩家在2米观看距离下的实际需求研究。VO-8（推荐）要求游戏提供高对比度模式，且该模式中主要UI元素的文字与背景对比度不低于**4.5:1**（直接引用WCAG 2.1 AA标准中的正文对比度阈值）。

XGAG视觉类的理想级条目VO-15要求游戏内重要视觉事件（如角色死亡、关键任务触发）配备"屏幕闪光强度控制"滑块，因为光敏性癫痫玩家即便关闭整体特效也可能因UI闪烁触发发作。该条目特别注明，游戏发布前应通过Harding光敏测试（Harding FPA标准）验证过场动画和UI动效。

### 听觉无障碍条目

听觉类16条，AU-1（基础级）要求所有传递关键游戏信息的音频提示必须同时有视觉替代反馈，例如"低血量警告声"必须同时配有屏幕边缘变红或图标闪烁等视觉形式。AU-6（推荐）规定对话字幕必须包含发言角色名称标识，且字幕背景不透明度可独立调节，不依赖全局UI透明度设置。

### 认知与语言条目

认知类12条，CL-3要求游戏提供"游戏速度调节"或"自动辅助瞄准强度"等降低即时反应压力的选项。CL-9（理想级）要求教程可在游戏任意时刻重新访问，且教程内容不与主线进度绑定——这直接针对"教程跳过后无法复习"这一认知障碍玩家的高频痛点设计。

## 实际应用

《光环：无限》（Halo Infinite，2021）是首批公开声明对照XGAG全面自评的Xbox第一方游戏，其按键重映射界面允许玩家独立重新分配13个输入，并提供实时预览冲突提示，直接对应MO-11的完整实现路径。《地平线5》（Forza Horizon 5）则引入了"辅助驾驶套件"选项，可将需要精准摇杆输入的漂移控制转化为单轴触发器操作，这一设计被XGAG官方案例库引用为MO-1的参考实现。

对于独立开发者，XGAG建议优先完成"基础"层级的24项条目，而完成所有基础条目大约需要额外100至200小时开发时间（Microsoft开发者博客2022年数据），其中按键重映射系统的初始搭建占约60%工时。Microsoft的Xbox开发者中心提供了可直接集成的GDK（Game Development Kit）无障碍API，包含预制的按键重映射UI模板，可将MO-11的实现成本降低约40%。

## 常见误区

**误区一：字幕存在即满足听觉条目要求。** XGAG的AU-6和AU-8明确区分了"对话字幕"与"音效字幕（Sound Captions）"两类需求，仅实现角色对话文字显示不能覆盖环境音效提示（如身后脚步声的方位字幕），这两类功能必须分别实现并提供独立开关。

**误区二：提供色盲滤镜等于完成视觉无障碍。** XGAG的VO-3明确指出：不应仅依赖颜色作为传递游戏状态的唯一信号（如红色=危险区域、绿色=安全区域），色盲滤镜是补救措施而非替代方案，正确做法是在设计阶段就为颜色信息添加形状、图标或纹理的第二维度编码。

**误区三：PC版游戏不需要遵守手柄中心的运动条目。** XGAG明确说明，在PC上发布的Game Pass游戏同样受MO系列条目约束，因为Xbox手柄是PC Game Pass玩家的主要输入设备之一，且键鼠输入的重映射需求由单独的KM系列条目覆盖，两者并行适用而非互相替代。

## 知识关联

在完成无障碍测试工具的学习后，开发者可将XGAG的每项条目转化为具体测试用例：例如对照MO-1使用Xbox自适应手柄（Xbox Adaptive Controller，XAC）进行单侧输入测试，或使用Xbox Game Bar内置的辅助功能诊断工具验证VO-2的文字缩放行为。XGAG的条目编号体系（MO/VO/AU/CL前缀+序号）与测试报告模板直接对应，便于建立可追踪的缺陷记录链。

在掌握XGAG整体框架后，听障适配（聋人/听障适配）方向会专门深化AU系列的设计实现，包括音效字幕的屏幕位置策略、多声道音频的视觉化方位映射，以及手语视频内嵌方案——这些内容在XGAG中仅列于理想级，但在专项无障碍设计实践中需要更系统的方法论支撑。
