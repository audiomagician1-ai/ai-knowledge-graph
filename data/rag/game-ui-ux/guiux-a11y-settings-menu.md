# 无障碍设置菜单

## 概述

无障碍设置菜单（Accessibility Settings Menu）是游戏中专门收录所有残障辅助选项的集中式界面，让玩家能够在一个统一的位置调整视觉、听觉、运动和认知四大类无障碍参数，而不必在多个系统子菜单中分散查找。这一设计理念最早由主机平台Xbox在2020年发布的《Xbox无障碍指南》中系统化推广，随后索尼和任天堂相继跟进，将集中式无障碍菜单纳入平台发行标准。

从历史发展看，早期游戏（2010年之前）几乎没有独立的无障碍菜单，相关选项零散埋藏在画面设置、音频设置或控制器设置内，色盲模式可能在"画面"下，字幕开关可能在"音频"下，导致有需求的玩家很难发现。《最后生还者 第二部》（2020年）是行业转折点，它提供了超过60项无障碍选项并集中于专属菜单，被视为集中式无障碍菜单的设计标杆。

无障碍设置菜单的重要性不仅在于功能齐全，更在于可发现性（Discoverability）——选项存在但无人知晓等于不存在。根据Microsoft 2021年的无障碍游戏调查，约有43%的玩家表示曾因找不到所需辅助选项而放弃游戏，集中式菜单能将选项发现率提升约3倍。

## 核心原理

### 四大分类架构

无障碍设置菜单的标准分类框架包含**视觉（Visual）、听觉（Audio）、运动（Motor）、认知（Cognitive）**四个一级分类。视觉类包含色盲滤镜（红绿色盲/蓝黄色盲）、UI缩放、高对比度模式；听觉类包含字幕样式、音频描述、声道平衡；运动类包含按键重映射、一键辅助、自动连击；认知类包含游戏速度调节、导航辅助、阅读辅助。这四类对应的是世界卫生组织ICF分类框架中的功能障碍维度，而非依照游戏内功能系统来划分，这是设计集中式菜单与普通设置菜单的本质区别。

### 首次运行配置（First-Run Accessibility Prompt）

首次运行配置是指玩家第一次启动游戏时，系统在进入主菜单之前主动弹出无障碍设置向导，而非等待玩家自行寻找。这一机制极为关键，因为运动障碍玩家可能根本无法操作默认控制方案来导航到设置菜单。《极限竞速：地平线5》（2021）实现了在加载画面完成前就触发无障碍向导，确保任何玩家在未做任何初始配置的情况下也不会陷入无法操作的困境。首次运行向导应包含不超过5个核心问题，避免在新玩家热情最高涨时带来配置疲劳。

### 实时预览与可逆性

无障碍设置菜单必须为每一项设置提供**实时预览**（Live Preview），即调整滑块或切换选项时，背景画面立即呈现效果，而不需要确认保存后才能看到变化。这对色盲滤镜尤为重要——色盲玩家无法在保存后返回画面验证效果是否正确再回头修改，实时预览消除了这一来回切换的障碍。此外，所有无障碍选项必须支持随时撤销，不能有"此选项需重启游戏生效"的设计，否则会对认知障碍玩家造成额外的记忆和操作负担。

### 键盘/控制器的完整可访问性

无障碍设置菜单本身的导航方式必须符合无障碍标准，这是一个容易被忽视的自引用问题：如果无障碍菜单本身需要精确鼠标操作或多点触控才能打开，运动障碍玩家将永远无法使用它。具体要求包括：所有交互元素的焦点顺序必须线性且可预测；下拉菜单必须支持键盘上下键遍历；滑块必须支持方向键逐步调整，步进值不超过总量的5%。

## 实际应用

**游戏内具体实现**：《神秘海域4》在2016年更新中为所有字幕增加了背景色、字体大小、颜色三项可调参数，这三项直接集中在字幕专属的二级子菜单下，每项调整效果即时反映在屏幕右下角的字幕预览框中，是早期集中式听觉无障碍配置的成功案例。

**跨平台一致性设计**：PC版游戏的无障碍菜单需要同时适配鼠标和手柄操作，因为部分PC玩家使用眼动仪或单开关控制器。Steam在2022年引入了"无障碍功能标签"系统，开发者需要在发布页面声明游戏的无障碍菜单支持哪些控制输入方式，这要求菜单UI本身必须通过Steam输入API进行全功能适配。

**设置持久化存储**：无障碍配置应存储在账号云端而非仅本地文件，这样玩家在不同设备（如从主机到掌机模式）切换时不需要重新配置。任天堂Switch的无障碍存档同步功能在2023年的固件14.0更新中才加入，此前Switch版游戏的无障碍设置丢失问题是听障和视障玩家投诉最多的问题之一。

## 常见误区

**误区一：将无障碍菜单藏在"高级设置"或"辅助功能"子菜单中**
许多开发者认为无障碍选项属于"小众需求"，将其折叠在多层菜单深处。但根据Xbox Game Studios内部数据，当无障碍菜单作为主菜单一级入口时，使用率比藏在三级子菜单时高出约470%。正确做法是将无障碍菜单与"游戏设置""图像设置"并列放置在主设置页面的顶层。

**误区二：认为首次运行提示会打扰"正常"玩家**
开发者有时担心无障碍向导会让不需要辅助功能的玩家感到困扰，因此设计为"跳过"按钮非常显眼而"配置"按钮不显眼。这种设计假设无障碍需求是少数，但美国疾控中心数据显示约26%的美国成年人有某种形式的残障，且相当比例的玩家在不同光线、疲劳程度或临时受伤状态下有阶段性辅助需求。首次运行提示应将"前往设置"和"跳过"两个按钮赋予同等视觉权重。

**误区三：复制粘贴操作系统无障碍功能即可**
部分开发者认为只需调用iOS/Android/Windows系统级无障碍API（如VoiceOver或Windows讲述人）即满足要求，无需在游戏内构建独立无障碍菜单。然而系统级无障碍工具无法控制游戏内的色彩渲染管线、游戏速度或按键映射逻辑，这些都需要游戏引擎层面的专属实现。系统API与游戏内无障碍菜单是互补关系，不能相互替代。

## 知识关联

无障碍设置菜单的构建以**聋人/听障适配**中的字幕技术规范为直接输入——字幕颜色、字体和背景的可配置项正是集中式听觉分类下的标准条目，设计师在实现字幕子菜单时需要将听障适配的具体参数转化为菜单控件（滑块、颜色选择器、下拉列表）。另一方面，无障碍设置菜单的设计经验是迈向**包容性设计思维**的具体基础：当设计师开始思考"菜单本身是否对运动障碍玩家可达"这一问题时，就从单项功能适配进入了系统性包容设计的思维模式——为边缘用户设计的导航逻辑、控件尺寸和焦点管理规范，最终也会改善所有玩家在嘈杂环境、小屏幕或注意力分散时的操作体验。
