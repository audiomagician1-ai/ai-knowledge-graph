# 主菜单设计

## 概述

主菜单是玩家启动游戏后看到的第一个交互界面，承载着"开始游戏""继续游戏""设置""退出"等核心功能入口。它不仅是功能导航层，更是游戏世界观与视觉风格的第一个展示窗口。玩家在主菜单停留的平均时间约为8至15秒，这段极短的时间内需要完成氛围传递和功能引导两项任务。

主菜单的设计规范随平台演进而发展。早期街机和FC时代的主菜单仅有静态文字列表（如1985年《超级马里奥兄弟》的"1 PLAYER / 2 PLAYERS"选择），受限于硬件只能提供最基本的选项排列。进入3D时代后，《最终幻想VII》（1997年）开始将动态CG背景引入主菜单，此后动态场景、环境音乐与菜单功能入口三者结合成为行业标准做法。

主菜单设计的质量直接影响玩家对整款游戏品质的第一印象。研究显示，视觉风格与游戏核心玩法一致的主菜单，能将玩家点击"开始游戏"的转化率提升约20%。对于独立游戏而言，主菜单往往是开发者最容易低估投入成本的界面，却是Steam商店页面截图中曝光率极高的画面之一。

## 核心原理

### 布局层级与视觉焦点

主菜单的布局遵循"一级菜单项不超过6个"的原则，超过6项时玩家需要额外的认知成本来扫描选项。按钮排列通常分为三种模式：**垂直列表**（适合PC端鼠标操作，如《巫师3》的左侧竖向菜单）、**水平排列**（适合主机手柄横向切换，如《战神》2018版）和**环形布局**（适合触屏或快捷操作场景）。

视觉焦点控制依赖**Z轴层次**设计：背景动态场景置于最深层，品牌Logo和主视觉角色位于中间层，功能按钮浮于前景层。这种三层结构确保玩家的视线路径为：Logo识别 → 氛围感知 → 功能点击，整体流程符合视觉引导的F型或中心型扫描规律。

### 氛围营造的四个元素

主菜单氛围营造依赖以下四个要素的协同：

1. **背景动态内容**：可以是游戏实机场景的循环片段（如《荒野大镖客：救赎2》的草原日落循环动画）、2D插画动效或粒子特效。循环时长建议在30至90秒之间，过短会产生明显割裂感，过长会增加内存占用。

2. **主题音乐**：主菜单BGM需要在20至30秒内完成情绪建立，通常采用游戏主旋律的变奏版本，音量级别控制在-18至-12 LUFS（响度单位），既能传递氛围又不至于在玩家操作时产生干扰。

3. **色彩基调**：主菜单色彩需与游戏核心美术风格一致。恐怖游戏常用低饱和冷色调配合高对比度文字，而JRPG类游戏倾向于高饱和暖色背景。《黑暗之魂》系列主菜单长期使用深灰与火焰橙的对比，强化了"绝望中存有希望"的世界观表达。

4. **动态反馈**：鼠标悬停或手柄选中时的按钮高亮、微动画（如文字缩放5%至10%）与音效反馈三者需同步触发，缺少任意一项都会降低操作的"手感确定性"。

### 功能入口的优先级排布

主菜单按钮的位置排列应遵循**使用频次优先**原则：

- 新玩家最常点击"新建游戏"或"开始游戏"，应置于视觉中心或列表第一项。
- 有存档的玩家最常点击"继续游戏"，因此有存档时需将"继续游戏"动态提升至第一位，覆盖"新建游戏"的默认位置——《黑暗之魂3》和《空洞骑士》均采用此动态置顶逻辑。
- "设置/选项"和"退出游戏"属于低频次操作，排列在列表末尾或设置为视觉权重更低的样式（如较小字号或灰色调）。

## 实际应用

**案例一：《只狼：影逝二度》的极简主菜单**
游戏主菜单仅有4个选项，背景为单一的黑色渐变配合武士剪影，主视觉Logo以毛笔字风格呈现。整个界面无任何动态背景动画，仅有轻微的烛火粒子效果。这种克制设计选择刻意传递"沉稳、孤独"的核心主题，与游戏高难度、单机体验的定位高度吻合。

**案例二：《英雄联盟》的动态主菜单**
作为多人在线游戏，《英雄联盟》的主菜单需要展示赛季主题、活动入口、好友状态等大量信息。其解决方案是采用模块化卡片布局，将活动Banner置于顶部横幅，核心导航（对局、商店、战队）采用底部固定Tab栏，右侧边栏展示好友列表。这种设计将功能密度扩展到12个以上入口，同时保持视觉秩序。

**案例三：移动游戏的主菜单适配**
《原神》移动端主菜单将所有功能入口压缩至屏幕底部1/3区域，顶部2/3保留给世界动态场景展示。针对竖屏与横屏自动切换时，按钮排列从单列变为双列，避免单手拇指操作的死角问题（拇指舒适操作区位于屏幕下方约40%的范围内）。

## 常见误区

**误区一：主菜单背景越华丽越好**
部分开发者将大量资源投入主菜单背景动画，结果导致游戏首次加载时间延长。实际上主菜单背景素材的总体积应控制在整个游戏资源包的1%至3%以内。《塞尔达传说：旷野之息》的主菜单背景仅为一张静态图加简单光晕动效，加载极快，且视觉表达力并不输给动态场景。

**误区二：把所有功能都放进主菜单**
将"成就系统""排行榜""DLC商店""更新日志"等二级功能直接暴露在主菜单层，会导致按钮数量超过8个，迫使玩家在开始游戏前花费过多时间做界面扫描。正确做法是将这些功能归入"额外内容"或"更多"的子菜单，保持主菜单只有4至6个高优先级入口。

**误区三：主菜单音乐不需要循环设计**
直接将BGM整轨播放而不做循环处理，会在曲目结束时产生一段突兀的静默，打破氛围沉浸感。专业做法是在音频编辑阶段设置无缝循环点（Loop Point），通常将循环点设置在小节线处，确保重复播放时节奏连续而无缝隙。

## 知识关联

**前置概念**：菜单系统概述提供了菜单层级划分和导航模型的基础认知，理解"主菜单是游戏菜单体系的顶层入口"这一定位有助于做出合理的功能归属决策——哪些选项属于主菜单层，哪些应下沉至子菜单。

**后续概念**：暂停菜单是主菜单之后需要掌握的下一个菜单类型。两者的关键区别在于触发场景不同：主菜单在游戏进程之外显示，有充足空间展示氛围与动态内容；暂停菜单在游戏进程中叠加显示，必须解决"不遮挡游戏画面信息"与"快速响应玩家操作"两个主菜单不需要面对的额外设计约束。理解主菜单中建立的按钮层级与视觉一致性规则，在暂停菜单设计中同样适用，但需根据游戏内叠加显示的特殊条件进行调整。
