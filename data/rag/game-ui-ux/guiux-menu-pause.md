# 暂停菜单

## 概述

暂停菜单（Pause Menu）是玩家在游戏进行中按下暂停键后弹出的临时覆盖层界面，它将游戏逻辑时间冻结，同时为玩家提供继续、存档、设置调整和退出等操作入口。与主菜单不同，暂停菜单的触发条件是"游戏已在运行中"，因此它必须承担"打断游戏流程"与"快速恢复游戏流程"两个相互矛盾的职责。

暂停菜单的概念随着1980年代家用主机的普及而成型。早期街机游戏没有真正的暂停功能，直到任天堂FC（红白机）时代，玩家才通过按下 **Start 键** 获得正式的暂停状态。《塞尔达传说》（1986年）开创性地将暂停菜单与道具管理界面合并，让暂停时间不仅用于中断，还用于策略规划。

暂停菜单的设计质量直接影响玩家的沉浸感恢复速度。研究表明，玩家打开暂停菜单的平均停留时间约为8至15秒，若操作层级超过两层，玩家的挫败感会显著上升。因此，暂停菜单的核心设计目标是：用最少的交互步骤完成最常用的操作，并以最快速度让玩家回到游戏。

## 核心原理

### 标准功能项构成

一个完整的暂停菜单通常包含以下固定功能项，排列顺序应遵循"最高频操作靠最近原则"：

1. **继续游戏（Resume / Continue）** — 必须是列表第一项或视觉焦点默认位置，因为这是超过70%的玩家打开暂停菜单后最终选择的操作。
2. **存档 / 读档（Save / Load）** — 仅在支持手动存档的游戏中出现，应显示最近的存档槽信息（如"自动存档 · 3分钟前"）以帮助玩家决策。
3. **设置（Settings）** — 暂停内嵌的设置通常只提供音量、字幕、亮度等高频调整项，而非完整设置界面的副本。
4. **退出到主菜单（Exit to Main Menu）** — 必须配合确认弹窗，防止误操作丢失进度，确认文案应具体说明后果，如"未保存的进度将丢失"。
5. **退出游戏（Quit Game）** — 部分平台（如主机）不暴露此选项，因为退出游戏由系统层管理。

### 快速操作与快捷入口

暂停菜单常集成若干**一键快捷操作**，跳过子菜单的层级直达功能：

- **快速存档按钮**：在暂停菜单主界面直接绑定一个按键（如PS手柄的△键），无需进入"存档"子菜单即可完成自动存档并提示"已存档 · 关卡3-2"。
- **地图/任务日志快捷标签**：部分RPG游戏（如《巫师3》）将暂停菜单设计成多标签页面，按L1/R1可在"道具""地图""任务""角色"间横向切换，将原本需要三次按键才能到达的内容压缩为一次侧移操作。
- **重新开始当前关卡（Restart）**：动作类和解谜类游戏的高频需求，《蔚蓝》（Celeste）将此项放在暂停菜单第二位，因为其玩家群体死亡频率极高，快速重试是核心体验。

### 返回游戏的流程设计

暂停菜单的**返回流程**是最常被忽视的设计细节，具体规则如下：

**按键对称原则**：触发暂停的键位（通常是Esc或Start）在暂停菜单中必须同样能关闭菜单，不能让玩家为了回到游戏而去鼠标点击"继续"按钮。这一规则被《黑暗之魂》系列严格执行：按下Start进入暂停，再次按Start立即恢复，无需移动光标。

**视觉过渡设计**：暂停菜单关闭时，推荐使用0.2至0.3秒的淡出动画，而非瞬间切回游戏画面。过短会造成视觉跳跃，过长（超过0.5秒）会打断玩家的心理准备节奏。

**背景模糊处理**：暂停菜单弹出时，游戏画面通常以模糊（Gaussian Blur，半径约8px）或降饱和度的方式处理，既明确告知玩家"当前处于暂停状态"，又保留场景信息供玩家参考（如敌人位置）。

## 实际应用

**《动物森友会》的无压力暂停设计**：该游戏没有传统暂停菜单，而是以"-"键打开"系统菜单"，功能仅有"存档并继续""存档并回标题""结束游戏"三项，因为游戏本身节奏缓慢，玩家无需在紧张状态下操作，故精简至极。

**《荒野大镖客：救赎2》的上下文敏感暂停菜单**：该游戏在单人模式与多人模式下呈现完全不同的暂停菜单。单人模式允许暂停并调出完整的地图与任务系统；而多人模式中按下暂停键只能进入个人设置，无法暂停世界时间，这是因为在线游戏的游戏时间不属于单一玩家控制。这一设计教训提示开发者：**暂停菜单必须区分单机与联机状态下的权限边界**。

**移动端暂停菜单的触控适配**：在手机游戏中，暂停按钮通常是屏幕角落的一个直径不小于44pt（iOS人机界面指南最小触控区域标准）的图标，暂停后弹出的菜单按钮高度至少为60pt，防止误操作关闭菜单误触"退出"。

## 常见误区

**误区一：暂停菜单是主菜单的缩小版**
许多初学者直接将主菜单的结构复制到暂停菜单中，加入"新游戏""章节选择""制作团队"等与当前游戏状态无关的选项。暂停菜单的所有功能项必须以"当前游戏会话"为上下文，"新游戏"意味着放弃当前进度，必须用"退出到主菜单"来间接实现，而非直接暴露。

**误区二：确认弹窗可以省略**
"退出到主菜单"和"退出游戏"两个操作如果不加确认弹窗，误触率在实机测试中通常高达12%至18%。确认弹窗的默认焦点应放在"取消"（即返回暂停菜单）而非"确认退出"，这一细节在《黑暗之魂》和《塞尔达传说：旷野之息》中均有体现。

**误区三：暂停时间不等于冻结所有系统**
部分开发者认为暂停意味着游戏引擎时间缩放（Time Scale）设为0即可。实际上，暂停菜单期间需要继续运行的系统包括：UI动画、背景音乐淡出、成就解锁监测、在线心跳包（保持联机连接）。在Unity中，暂停菜单的UI动画需要使用 `Time.unscaledDeltaTime` 驱动，否则当 `Time.timeScale = 0` 时动画会完全停止。

## 知识关联

**前置概念——主菜单设计**：主菜单确定了游戏的整体信息架构和视觉风格，暂停菜单作为游戏内的衍生菜单，其字体、色彩和按钮样式需与主菜单保持一致的设计语言。但两者的核心差异在于触发情境：主菜单在游戏外，玩家心理状态松弛；暂停菜单在游戏中，玩家可能处于紧张或专注状态，因此暂停菜单的可读性标准（字号不低于24sp、对比度不低于4.5:1）要比主菜单更严格。

**后续概念——设置界面设计**：暂停菜单通常内嵌一个精简版设置入口，而完整的设置界面设计则需要处理数十个参数项的分类、滑块精度、键位重映射等复杂交互问题。学习设置界面设计时，需重点理解暂停菜单中的设置入口应如何决定"哪些设置项值得在暂停层级暴露"——判断标准是该设置项是否会在一局游戏过程中被反复调整（如音量），而非一次性配置（如画质等级）。
