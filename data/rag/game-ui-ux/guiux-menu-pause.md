# 暂停菜单

## 概述

暂停菜单（Pause Menu）是玩家在游戏过程中通过按下特定按键临时中断游戏进程、访问功能选项的交互界面。与主菜单不同，暂停菜单并不终止游戏会话，而是将游戏逻辑冻结在当前帧状态，同时呈现一个覆盖层（Overlay）供玩家操作。在主机游戏中，这一触发键通常是"Start"或"Options"键；在PC游戏中则多为"Esc"键；移动端游戏则常在屏幕角落设置暂停按钮图标。

暂停菜单的设计可追溯到1980年代早期的街机及家用主机时代。《超级马里奥兄弟》（1985年，任天堂）将"Start键暂停游戏"确立为主机游戏的行业惯例，此后该交互模式被几乎所有游戏类型沿用。随着游戏内容复杂度提升，暂停菜单从最初仅显示"继续/退出"两项功能，逐步扩展为包含存档、设置、地图、任务追踪等多功能面板。

暂停菜单的意义在于为玩家提供一个"安全退出点"——不损失游戏进度的前提下处理现实事务，同时也是玩家访问设置、查看控制说明的第一入口。设计不当的暂停菜单（如层级过深、返回路径不明确）会直接打断游戏心流（Flow State），导致玩家挫败感上升。

## 核心原理

### 标准功能项结构

暂停菜单的功能项遵循"频率优先"原则排列：使用频率最高的"继续游戏（Resume）"通常排在第一位，且应为默认焦点选项，确保玩家按一次确认键即可快速返回游戏。典型的暂停菜单功能项层级如下：

1. **继续游戏**（Resume / Continue）
2. **存档/读档**（Save / Load）——仅在支持手动存档的游戏中显示
3. **设置**（Settings）——音频、画面、控制等子菜单入口
4. **查看地图/任务**（Map / Objectives）
5. **返回主菜单**（Return to Main Menu）
6. **退出游戏**（Quit Game）——桌面端特有

危险操作（如"退出游戏"）应与高频操作保持至少2个选项的视觉间距，并在触发时弹出二次确认对话框，防止误触。

### 快速操作（Quick Actions）设计

现代游戏常在暂停菜单中嵌入快速操作区，允许玩家无需进入子菜单即完成特定操作。例如《荒野大镖客：救赎2》（2018年）的暂停菜单直接在第一层级展示地图缩略图；《黑暗之魂》系列允许在暂停状态下直接装备物品和更换技能。快速操作区的设计原则是"零级深度"——所有操作在暂停菜单的当前页完成，无需跳转子页面。

快速操作的触发方式还包括手柄的快捷键映射（Shortcut Binding），例如长按暂停键直接进入存档界面，短按则进入标准暂停菜单，通过时长区分操作意图，将菜单层级从2次按键压缩为1次。

### 返回流程与状态恢复

暂停菜单的返回流程必须保证游戏状态的完整恢复，包括：
- **时间冻结**：游戏世界时钟停止，AI行为暂停，物理模拟停止更新
- **音频处理**：背景音乐通常保持低音量播放（而非完全静音），声效暂停
- **渲染状态**：当前帧画面作为背景模糊显示（Blur/Darken Overlay），深度值约为原图的40%~60%亮度

返回路径应支持**多出口一致性**：无论玩家是点击"继续游戏"、按下暂停键、还是按下手柄的B/Circle（返回键），均应执行相同的"关闭暂停菜单并恢复游戏"操作，不应出现按B键返回到上级菜单而非直接返回游戏的情况。

## 实际应用

**《塞尔达传说：旷野之息》（2017年）** 的暂停菜单将装备管理、材料查看、地图、任务、图鉴整合为横向Tab导航，玩家通过手柄左右肩键切换标签页，避免了垂直菜单的层级嵌套问题，整体操作深度不超过2层。

**《最后生还者》系列** 的暂停菜单采用极简设计：纯黑背景、白色文字、仅5个选项，刻意营造与游戏沉浸感一致的低干扰风格，并移除了所有动画过渡效果，以减少玩家在紧张场景暂停时产生的情绪断裂感。

**移动端游戏** 中，暂停菜单的触发逻辑与桌面端不同：当应用被切至后台时，游戏应自动触发暂停状态并弹出暂停菜单，这要求暂停逻辑与应用生命周期钩子（如iOS的`applicationWillResignActive`）绑定，而非仅响应玩家的主动操作。

## 常见误区

**误区一：将"退出游戏"设为暂停菜单的默认焦点选项**
部分开发者在布局时未设置初始焦点，导致手柄光标默认停留在列表最后一项（通常是"退出游戏"），玩家习惯性按确认键时误触退出，造成游戏进度损失。正确做法是强制将"继续游戏"设为暂停菜单打开时的默认焦点，并在代码层面固定该初始状态。

**误区二：暂停菜单中仍允许游戏世界事件继续推进**
在多人在线游戏中，由于游戏状态由服务器控制，玩家本地触发暂停并不能冻结游戏世界，这是合理的技术限制。但在单人游戏中，若因实现疏漏导致暂停期间AI仍在移动、伤害仍在结算，会严重破坏玩家对"暂停"语义的预期。单人游戏必须在暂停菜单激活时同步调用`Time.timeScale = 0`（Unity）或等效的引擎级时间冻结接口。

**误区三：暂停菜单的"设置"与主菜单的"设置"功能不一致**
部分游戏的暂停菜单设置仅包含音量调节，而主菜单设置包含完整的图形、键位等选项，导致玩家在游戏中途无法修改关键配置。标准做法是两处入口指向同一设置界面，或明确标注"完整设置请返回主菜单"，但后者会显著增加操作成本。

## 知识关联

**与主菜单设计的关联**：主菜单是游戏会话开始前的入口，其功能项（新游戏、载入存档、设置、退出）与暂停菜单的功能项存在约60%的重叠。设计时应保持两处界面的视觉风格、字体尺寸和交互逻辑一致，避免玩家在相似功能上产生不同的操作学习成本。主菜单中已建立的导航模式（如Tab切换、B键返回）应在暂停菜单中沿用。

**通向设置界面设计**：暂停菜单是玩家访问设置界面最频繁的入口路径，因此设置界面的设计必须考虑"从暂停菜单打开、修改后回到暂停菜单"的完整往返流程。设置界面的返回按钮需要区分"保存并返回"与"取消并返回"两种语义，且返回目标应为暂停菜单而非游戏主场景，这与从主菜单进入设置时"返回主菜单"的行为有所不同，是设置界面设计中需要专门处理的状态路由问题。
