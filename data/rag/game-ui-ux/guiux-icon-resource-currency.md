# 资源/货币图标

## 概述

资源/货币图标是游戏UI中用于表示玩家可收集、消耗或积累的各类经济要素的视觉符号，涵盖金币、宝石、水晶、木材、粮食、特殊货币等数十种形态。与武器/装备图标侧重展示物品形态不同，资源图标的核心任务是在极小尺寸下（通常16×16至32×32像素）让玩家在0.3秒内准确识别当前操作涉及的资源种类，并通过视觉语言传达该资源的稀缺程度与价值层级。

资源/货币图标的设计规范化始于1990年代早期的即时战略游戏。《魔兽争霸II》（1995年）首次系统性地建立了"金矿=黄色圆形、木材=棕色方形"的颜色-形状双维度编码体系，这一范式直接影响了此后二十余年的资源图标设计思路。到《英雄联盟》（2009年）时代，MOBA类游戏将资源货币进一步细分为金币、精粹、荣誉点等多套平行货币体系，要求每套体系在同屏共存时仍保持清晰区分，对差异化设计提出了更高要求。

资源图标的设计质量直接影响玩家的经济决策速度。研究数据显示，资源图标辨识度每提升10%，玩家在商店界面的平均停留时间缩短约8%，换算到竞技游戏中意味着更快的操作节奏和更低的认知负担。此外，资源图标往往出现在HUD的固定位置，每局游戏中可能被玩家扫视数百次，其视觉疲劳度控制同样是设计的重要指标。

## 核心原理

### 颜色-材质的价值层级编码

资源图标通过颜色和材质纹理共同构建一套价值阶梯，使玩家无需阅读文字即可感知资源稀缺性。普通资源（如铁矿、木材）通常采用低饱和度的土黄、棕灰色调，表面纹理粗糙；高级资源（如魔法水晶、传说材料）则采用高饱和度蓝紫色或金橙色，并配合内发光或镜面反射效果。这种"饱和度越高=价值越高"的视觉规律在《原神》的原石（深蓝发光）与摩拉金币（亮黄哑光）设计中体现得尤为典型——两者颜色温度差超过4000K，确保在小尺寸下不产生混淆。

### 形状语义与文化一致性

硬币类货币普遍采用圆形轮廓，因为跨文化研究表明圆形与"可流通、可交换"的货币认知存在深度绑定；而不可流通的特殊资源（如建筑材料）则多使用多边形或异形轮廓加以区分。具体数值规律是：货币图标的正圆率通常≥90%（即图标最终形态接近标准圆），而材料图标的轮廓凹凸变化点一般不少于3处，以制造"非标准化"的手工感。《部落冲突》中的金袋（圆形）与黑色精华（棱形瓶体）正是这一原则的典型实践。

### 多货币并存时的差异化策略

当游戏系统包含3种及以上资源时，仅靠颜色区分会因对比不足导致混淆，设计师必须引入"颜色+形状+尺寸感"三维矩阵。具体操作方法是：主货币（最高频使用）采用圆形+暖金色+强反光；次级货币采用与主货币色相差≥120°的颜色+不同几何形状；消耗型材料则通过添加独特"容器"（袋、瓶、晶体）来增加识别维度。《Clash Royale》的黄金、宝石、野卡三套货币严格遵循此矩阵，三者色相角分别约为45°（金）、150°（绿宝石）、270°（蓝紫），形成等间距色相分布。

### 动态状态的视觉反馈

资源图标不仅是静态符号，还需承载"充足/不足/即将耗尽"三种状态的视觉反馈。不足状态通常通过将图标整体降饱和度至原值的40%~60%并叠加灰色蒙版实现；耗尽状态则在此基础上追加轻微抖动动效（振幅2~4像素，频率约8Hz），在不影响布局的前提下传达紧迫感。《炉石传说》的费用水晶采用"满蓝→消耗变暗→变灰空洞"三阶段变化，让玩家不用数字便能感知剩余行动力。

## 实际应用

在《王者荣耀》的货币体系中，金币图标（圆形亮金）与点券图标（菱形+蓝紫渐变）被放置在大厅右上角并排显示，图标尺寸统一为40×40dp，两者色相差约180°，色彩对比度达到WCAG AA标准（对比比4.5:1以上），确保弱视玩家也能有效区分两种真实货币价值不同的购买渠道。

在《饥荒》（Don't Starve）中，所有资源图标均采用黑白线稿风格，通过轮廓形状而非颜色来区分200余种材料。这一设计决策源于游戏的画风约束，但反过来证明了轮廓形状是资源图标中最不可妥协的识别维度——即使去掉色彩，形状独特的资源图标仍能被正确识别。

移动端资源图标还需特别考虑"拇指遮挡区"问题。标准6英寸屏幕下，屏幕底部中央约120×80像素的区域会被右手拇指频繁遮挡，因此大多数移动端游戏（如《明日方舟》）将主货币图标放置在屏幕右上角，并将图标触控热区扩展至图标实际尺寸的150%，以支持快速点击查看详情。

## 常见误区

**误区一：用相似色调区分不同价值的货币**
许多初级设计师会给"金币"和"高级金币"使用深黄与亮黄两种相近颜色，误以为明度差异已足够区分。实际上，在游戏HUD长时间使用后，人眼对相邻色调的区分敏感度会下降约30%，仅靠色调变化无法在高压决策场景中稳定传递价值差异。正确做法是同时改变形状（如圆形→八边形）并引入材质差异（哑光→反光）。

**误区二：资源图标尺寸与装备图标统一**
资源图标频繁出现在数量显示旁边，需要与数字字符在视觉重量上保持平衡，其图标尺寸应比装备图标小20%~30%。若直接套用装备图标的64×64像素规格用于显示"×1500金币"，图标会在视觉上压制数字，导致玩家忽视数量信息，引发经济操作失误。

**误区三：认为动效越丰富、资源图标越吸引人**
部分设计师为稀有货币图标添加持续旋转、粒子喷射等复杂动效，试图强化"稀有感"。但资源图标存在于HUD的持久显示区，过度动效会在20分钟游戏后造成明显视觉疲劳，玩家反馈中"眼花"投诉率与图标动效帧率呈正相关。《暗黑破坏神3》早期版本的金币飞入动效因此被缩短至0.4秒以内，并在积累到阈值后改为数字跳变。

## 知识关联

在学习资源/货币图标之前，理解武器/装备图标的设计逻辑至关重要：装备图标建立了"图标=物品形态直观呈现"的基础认知，而资源图标则在此基础上加入了"抽象符号代表经济概念"的维度跳跃——金币并不需要展示真实硬币的所有细节，它需要的是一个能瞬间触发"可花费"认知的视觉锚点，这是两类图标最根本的设计目标差异。

资源/货币图标的设计经验直接为导航/功能图标的学习奠定基础。导航图标同样面临"极小尺寸下快速传递抽象概念"的挑战，资源图标中建立的颜色层级编码方法、三维差异化矩阵和动态状态反馈机制，可以直接迁移应用于功能按钮的激活/禁用状态设计和菜单层级视觉区分中。两者之间最核心的技术共享点是：在16×16像素极限尺寸下，形状轮廓的辨识度始终优先于细节雕琢。
