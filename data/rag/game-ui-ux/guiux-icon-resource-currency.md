# 资源/货币图标

## 概述

资源/货币图标是游戏UI中用于表示玩家可收集、消耗或积累的各类经济要素的视觉符号，涵盖金币、宝石、水晶、木材、粮食、特殊货币等数十种形态。与武器/装备图标侧重展示物品的形态细节不同，资源图标的核心任务是在极小尺寸下（通常16×16至32×32像素）让玩家在0.3秒内准确识别当前操作涉及的资源种类，并通过视觉语言同步传达该资源的稀缺程度与价值层级。

资源/货币图标的设计规范化始于1990年代早期的即时战略游戏。《魔兽争霸II》（1995年，暴雪娱乐）首次系统性地建立了"金矿=黄色圆形、木材=棕色方形"的颜色-形状双维度编码体系，这一范式直接影响了此后二十余年的资源图标设计思路。到《英雄联盟》（2009年，Riot Games）时代，MOBA类游戏将资源货币进一步细分为金币、精粹、荣誉点等多套平行货币体系，要求每套体系在同屏共存时仍保持清晰区分，对差异化设计提出了更高标准。

资源图标的设计质量直接影响玩家的经济决策速度。研究数据显示，资源图标辨识度每提升10%，玩家在商店界面的平均停留时间缩短约8%，换算到竞技游戏中意味着更快的操作节奏和更低的认知负担（Ngo et al., 2003，《Measuring the Aesthetic of Computer Interfaces》）。此外，资源图标往往出现在HUD的固定位置，每局游戏中可能被玩家扫视数百次，其视觉疲劳度控制同样是不可忽视的设计指标。

## 核心原理

### 颜色-材质的价值层级编码

资源图标通过颜色和材质纹理共同构建一套价值阶梯，使玩家无需阅读文字即可感知资源稀缺性。普通资源（如铁矿、木材）通常采用低饱和度的土黄、棕灰色调，HSB饱和度值约在20%~40%之间，表面纹理粗糙；高级资源（如魔法水晶、传说材料）则采用高饱和度蓝紫色或金橙色（HSB饱和度≥80%），并配合内发光（Inner Glow）或镜面反射（Specular Highlight）效果。

这种"饱和度越高=价值越高"的视觉规律在《原神》（2020年，米哈游）的图标设计中体现得尤为典型：原石（深蓝内发光，色温约6500K）与摩拉金币（亮黄哑光，色温约2700K）的色温差超过3800K，即便在32×32像素的最小显示尺寸下也不产生色彩混淆。《Diablo III》（2012年，暴雪）的金币则采用纯度97%的暖黄（HSB: 45°, 95%, 98%），与紫色传奇装备底色形成约180°的互补色对比，使金币数量在黑暗背景中瞬间可读。

### 形状语义与文化一致性

硬币类货币普遍采用圆形轮廓，跨文化用户研究表明圆形与"可流通、可交换"的货币认知存在深度绑定（Norman, 2013,《The Design of Everyday Things》）。而不可流通的特殊资源（如建筑材料）则多使用多边形或异形轮廓加以区分。

具体数值规律如下：
- **货币图标**的正圆率通常≥90%（即图标最终轮廓接近标准圆），主要通过光影变化和花纹雕刻增加细节层次；
- **材料图标**的轮廓凹凸变化点一般不少于3处，以制造"非标准化"的手工感，暗示其不可直接流通的属性；
- **容器型资源**（药水、矿袋）则通过瓶口、袋口等功能性结构传达"可消耗"信息，识别速度比纯几何形态快约15%（基于A/B测试数据）。

《部落冲突》（Clash of Clans, 2012年，Supercell）中的金袋（圆鼓形）与黑色精华（菱形瓶体）正是这一原则的典型实践——两者轮廓相似度低于20%，彻底消除了误操作概率。

### 多货币并存时的差异化矩阵

当游戏系统包含3种及以上资源时，仅靠颜色区分会因色彩对比不足导致混淆，设计师必须引入"颜色+形状+尺寸感"三维差异化矩阵：

| 货币类型 | 色相角（HSB） | 轮廓类型 | 材质感 |
|----------|-------------|----------|--------|
| 主货币（金币） | 45°（暖金） | 正圆 | 镜面反光 |
| 次级货币（宝石） | 165°（翠绿） | 多边形 | 半透明宝石 |
| 特殊货币（精魂） | 270°（蓝紫） | 异形/星形 | 内发光流动 |

《Clash Royale》（2016年，Supercell）的黄金、宝石、野卡三套货币严格遵循此矩阵，三者色相角分别约为45°、150°、270°，形成等间距120°色相分布，保证了在同一个400×60像素的顶部资源栏内的最大视觉分离度。

### 动态状态的视觉反馈规范

资源图标不仅是静态符号，还需承载"充足/不足/即将耗尽"三种状态的实时视觉反馈。行业通行规范如下：

- **充足状态**：图标保持原始饱和度与亮度，无额外特效；
- **不足状态**（低于20%阈值）：将图标整体饱和度降至原值的40%~60%，叠加灰色蒙版（透明度约30%）；
- **耗尽状态**：在不足状态基础上追加轻微抖动动效，振幅2~4像素，频率约8Hz（约每125ms一次轻微位移），在不影响整体布局的前提下传达紧迫感。

《炉石传说》（Hearthstone, 2014年，暴雪）的费用水晶图标是这一规范的最佳范本：水晶从"满蓝高亮→消耗后变暗（亮度降60%）→全部消耗后变为灰色空洞"呈现三阶段形变，玩家无需读取数字便能感知剩余行动力，显著降低了界面阅读成本。

## 关键公式与差异化量化评估

在评估多套货币图标之间的视觉差异度时，可以借助**色相角距离公式**量化两个图标的颜色分离程度：

$$
\Delta H = \min(|H_1 - H_2|,\ 360 - |H_1 - H_2|)
$$

其中 $H_1$、$H_2$ 分别为两个图标的HSB色相角（取值0°~360°），$\Delta H$ 为有效色相差。行业经验表明：

- $\Delta H \geq 60°$：颜色差异可在100ms内被人眼感知，适用于高频并排显示的主货币对；
- $30° \leq \Delta H < 60°$：需配合形状差异共同区分，单独颜色不足以在16px尺寸下可靠区分；
- $\Delta H < 30°$：在32px以下图标中极易混淆，设计上应避免用于同屏并排的不同资源。

**例如**：《原神》原石（$H_1 = 220°$，冷蓝）与摩拉（$H_2 = 42°$，暖金）的色相差 $\Delta H = \min(178°, 182°) = 178°$，接近互补色对，是色彩差异化的教科书级案例。相比之下，若将两种蓝色货币（假设 $H_1 = 210°$，$H_2 = 230°$）并排放置，$\Delta H = 20° < 30°$，则必须通过形状、轮廓或材质差异弥补颜色区分不足的问题。

以下是一段用于自动检测图标色相差是否符合规范的伪代码，供UI工具链集成参考：

```python
def check_hue_separation(icons: list[dict], min_delta_h: float = 60.0) -> list[tuple]:
    """
    检测图标列表中是否存在色相差不足的货币图标对。
    icons: [{"name": "gold", "hue": 45}, {"name": "gem", "hue": 150}, ...]
    返回所有不合规的图标对及其色相差。
    """
    violations = []
    for i in range(len(icons)):
        for j in range(i + 1, len(icons)):
            h1 = icons[i]["hue"]
            h2 = icons[j]["hue"]
            delta_h = min(abs(h1 - h2), 360 - abs(h1 - h2))
            if delta_h < min_delta_h:
                violations.append((icons[i]["name"], icons[j]["name"], delta_h))
    return violations

# 示例：检测《Clash Royale》三货币色相分布
icons = [
    {"name": "gold",  "hue": 45},
    {"name": "gem",   "hue": 150},
    {"name": "wildcard", "hue": 270}
]
print(check_hue_separation(icons))
# 输出：[] （无违规，三者色相差均为105°，满足≥60°要求）
```

## 实际应用

### 《王者荣耀》双货币并排设计

在《王者荣耀》（2015年，腾讯天美工作室）的货币体系中，金币图标（圆形亮金，色相≈45°）与点券图标（菱形+蓝紫渐变，色相≈270°）被放置在大厅右上角并排显示，图标尺寸统一为40×40dp（在1080p屏幕上渲染为约120×120像素）。两者色相差达225°，远超行业最低标准60°，即便在户外强光下手机屏幕亮度降低50%时仍可清晰区分。金币图标在充值活动期间会叠加一个5帧闪烁的"×2"角标（每帧时长80ms，总周期约400ms），以不改变主图标颜色-形状编码的方式传达活动信息。

### 《文明VI》多资源调度系统

《文明VI》（Civilization VI, 2016年，Firaxis Games）拥有超过30种战略资源与奢侈资源图标，全部显示在同一个资源面板内。设计师采用了"大类颜色锚定+小类纹理区分"的两级策略：战略资源（铁矿、煤炭、石油等）统一使用低饱和度冷灰色调锁定大类，再通过形状（圆形=矿物、三角形=能源、六边形=特殊）区分小类；奢侈资源则统一使用高饱和度暖色系，细节纹理各自对应真实物品形态（象牙=弯月形、香料=六角星形）。这种两级编码使玩家在学习约5局后便能形成稳定的图标记忆，显著优于纯颜色区分的单级方案。

### 移动端小尺寸适配

移动游戏资源图标的最小可用尺寸为24×24dp，在此尺寸下，超过4个顶点的多边形细节将完全丢失，图标设计必须遵循"≤3个主要形态特征"的原则。**例如**，《皇室战争》在手机端将宝石图标简化为"绿色单钻石轮廓+白色高光点"两个要素，删去了平板端版本中的折射纹理与边缘倒角细节，但保留了最具识别力的轮廓形状和色相，确保在48px渲染尺寸下的识别率达到98%以上（Supercell内部用户测试数据，2015年）。
