# 资源/货币图标

## 概述

资源/货币图标是游戏UI系统中用于表示金币、宝石、材料、能量等可消耗或可积累资产的视觉符号。与武器/装备图标不同，资源图标通常不代表唯一道具，而是代表一种**可叠加的数量型物品**，因此其设计语言强调"计量感"与"类型区分"，而非单件物品的精细细节。

资源图标设计的历史可追溯至1990年代早期的角色扮演游戏，《魔法门》（Might and Magic）系列率先使用圆形金黄色硬币作为货币符号，奠定了"圆形＝金币"的视觉公约。现代移动游戏（约2012年起）将资源种类扩展至软货币（软币，如金币）与硬货币（钻石/宝石）的双轨制，促使设计师必须在同一界面内清晰区分多种资源图标。

资源图标的差异化设计直接影响玩家的经济决策速度。眼动追踪研究显示，玩家识别熟悉资源图标的平均时间仅需**80-120毫秒**，而混淆相近图标会导致误操作率上升约35%。因此，资源图标的形状、色相、质感三要素必须在设计阶段被严格区分。

## 核心原理

### 1. 形状语言与资源类型绑定

不同资源类型应对应固定的几何原型，以触发玩家的快速视觉分类：

- **圆形/椭圆形**：金币、银币等标准货币。圆形象征流通性，《原神》中的摩拉图标即采用带有帝国徽章的金色圆盘。
- **多边形（5-8边）**：宝石、钻石等高价值稀有货币。锐角轮廓传递"稀缺感"，《部落冲突》的绿宝石使用了六边形切割面。
- **不规则有机形**：木材、矿石、草药等自然采集材料。轮廓的不规则性暗示"天然未加工"的属性。
- **符文/晶体形**：魔法能量、法力值等抽象资源，常采用几何对称的星形或菱形。

### 2. 色相分层与稀有度映射

资源图标的颜色体系需与游戏内的稀有度等级严格对应，常用配色方案遵循以下标准（以多数RPG游戏为基准）：

| 等级 | 色相范围 | 典型资源 |
|------|----------|----------|
| 普通 | 灰色/棕色（HSB饱和度≤30%） | 木材、石料 |
| 稀有 | 蓝色（H: 200-240°） | 魔法水晶 |
| 史诗 | 紫色（H: 270-300°） | 紫晶碎片 |
| 传说 | 金橙色（H: 30-50°）+ 发光特效 | 传说宝石 |

软货币（金币）使用暖黄色（H≈45°）而非橙色，与传说级道具区分；硬货币（钻石/宝石）使用冷色系（蓝色或绿色），视觉上强调其"高价值但不易获取"的心理暗示。

### 3. 质感与材质表现

资源图标的材质渲染必须与其游戏内价值感匹配。金属货币需使用**高光-阴影比约为3:1**的镜面反射效果，形成硬质光泽感；天然材料（木材、皮革）则使用漫反射为主（高光面积≤10%）的哑光处理，传递粗糙质感。

图标尺寸也决定材质细节的处理方式：在**32×32像素**的小尺寸状态下，金币图标应只保留圆形外轮廓和单一高光点，省略徽章细节；在**64×64像素**以上时才加入内部纹路。这种多分辨率适配原则可防止小尺寸下图标因细节过多而变成"泥点"。

### 4. 数量提示的图标延伸设计

资源图标不能脱离数字角标单独设计。图标右下角需预留**至少6×6像素**的安全区域用于显示数量（如"×99"），且图标整体视觉重心应偏向左上方，避免数字遮挡关键识别区域。《刀塔传奇》的资源图标因在早期设计中忽视了这一点，导致三位数数量被图标阴影覆盖，于2014年版本迭代中做了专项修正。

## 实际应用

**双货币界面布局（《皇室战争》案例）**
《皇室战争》在主界面顶部并排显示金币（黄色圆形图标）和宝石（绿色切割形图标），两者尺寸相同（约24dp），但通过色相对比（黄色vs绿色）和形状对比（圆形vs多边形）实现一眼区分。当玩家金币不足时，金币图标会触发红色脉冲动画而非宝石图标，依赖图标形状差异精准传递"是哪种资源不足"的信息。

**多材料合成界面（《明日方舟》案例）**
《明日方舟》的材料系统包含超过200种材料图标，采用**底板形状编码**：普通材料用圆角方形底板，精英材料用六边形底板，特殊材料用钻石形底板。即使不看颜色，玩家也能通过底板形状快速判断材料等级，这在色觉障碍玩家群体中尤为重要。

**能量/行动点资源（《杀手：狙击》案例）**
行动点类资源图标不代表实体物品，而代表抽象的"可执行次数"。《杀手：狙击》将子弹数量设计为逐渐缩小的圆点序列而非堆叠硬币，通过图标形态本身暗示"消耗后不可回收"的属性，区别于金币等可积累货币的设计逻辑。

## 常见误区

**误区一：所有货币图标都用金色表示"贵重"**
将金币和高级宝石都设计为金色，会导致玩家在高压决策时混淆软硬货币，引发误购。正确做法是只有低级别的日常货币（金币）使用金黄色，高价值的稀有货币必须使用**独立的冷色系色相**（如蓝宝石用H:220°，祖母绿用H:140°），通过色温对立而非明度变化来区分层级。

**误区二：材料图标越精细越好**
在材料合成界面，一个精细到包含反射光、内部纹理、阴影的32×32图标，反而比简化版图标识别速度慢约20%。过度渲染导致图标之间的视觉特征互相干扰。材料图标的区分依赖**轮廓形状和主色调**，而非纹理细节——细节是用于商店展示的大图，而非列表缩略图。

**误区三：资源图标可以复用装备图标的设计风格**
装备图标以展示外观为主，通常采用斜45°透视图和写实光影，适合大尺寸展示。资源图标则需要在成排列表中高速扫描，应采用**正面平视构图**和高对比度剪影，与装备图标形成风格隔离。两套图标若使用相同的透视角度和渲染风格，玩家会对"这是装备还是材料"产生混淆。

## 知识关联

在学习武器/装备图标之后，学习者已掌握"道具图标需传递用途与稀有度"的基础逻辑。资源/货币图标在此基础上引入了新的设计维度：**数量感、可消耗性与类型区分**。装备图标设计聚焦单件物品的唯一性展示，而资源图标设计聚焦同类型物品在不同情境下的快速识别——两者构成游戏图标系统中"唯一道具"与"计量资源"两类设计思路的对比。

掌握资源/货币图标的差异化设计原则后，下一步学习**导航/功能图标**时，将面临完全不同的设计挑战：功能图标不代表任何游戏内实体，而是代表操作指令（如"返回""设置""地图"），其设计逻辑从"物品可视化"转向"动作抽象化"，需要引入通用符号学（semiotics）和用户心智模型的知识框架。
