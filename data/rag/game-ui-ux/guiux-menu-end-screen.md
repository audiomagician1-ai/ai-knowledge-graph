# 结算/结束画面

## 概述

结算/结束画面（End-of-Round/Victory Screen）是指玩家完成一局游戏、一个关卡或一场对战后，系统自动弹出的综合性UI面板，用于展示本局成绩、经验值收益、奖励发放及荣誉称号等信息。它不同于排行榜界面的持久性静态展示，而是一个具有明确生命周期的"临时弹出层"——从关卡结束的瞬间触发，到玩家主动确认或倒计时结束后自动关闭。

这类界面最早可追溯至街机时代的"GAME OVER"画面，彼时仅显示最终得分和字母输入栏（用于填写排行榜缩写名）。随着网络游戏兴起，《魔兽世界》2004年上线时引入了副本掉落分配弹窗，《英雄联盟》2009年推出后将结算画面发展为包含KDA、伤害统计、金币收入、MVP评选的完整数据看板，使结算画面的信息密度和交互复杂度大幅提升。

结算画面的设计价值在于它是一局游戏中"反馈密度最高的单一时刻"：玩家在此处同时接收成败判定、个人表现量化、社交比较（组队时看到队友数据）、资源收益确认四类信息。若设计不当，玩家会产生"不知道自己得了什么"或"看不懂自己哪里做好了"的挫败感，直接影响下一局的开启意愿。

## 核心原理

### 信息层级与视觉权重分配

结算画面通常将内容分为三个层级：**首要信息**（胜负/通关状态，字号最大，占屏幕上方1/4区域）、**次要信息**（个人数据统计、MVP标识，居中主体区域）、**附属信息**（经验条进度、奖励物品图标，位于画面下方）。三层信息需在2秒内让玩家完成首要扫描，因此颜色对比度要求"胜利"文字与背景的对比度不低于WCAG 4.5:1标准。

《王者荣耀》的结算画面采用"金色粒子+镜头推进"动效来强化胜利的首要层，在0.5秒内完成从模糊到清晰的聚焦过渡，防止玩家注意力停留在次要统计数据上过久。失败状态则刻意使用低饱和度蓝灰色调，降低"失败"字样的视觉冲击，引导玩家目光更快移向"再来一局"按钮。

### 动态奖励动画的节奏设计

经验值增长动画和奖励解锁动画必须遵循严格的**播放时序**，通常分为三段：首先播放胜负特效（约1.5秒），随后触发统计数据的逐项飞入（每条数据间隔约0.3秒），最后触发经验条充能动画并在满条时产生"升级爆炸"特效。这三段若压缩到同时播放，玩家的多巴胺反馈会相互干扰，感知到的奖励价值反而下降。

《原神》的挑战结算采用"星级判定+宝箱开箱"双重奖励节奏：先播放3秒星级落下动画（给予"评分揭晓"的仪式感），再播放1.2秒宝箱开启动画（给予"实物获得"的满足感）。两段动画之间的0.4秒停顿是刻意设计的"呼吸间隔"，让玩家大脑完成前一段信息的处理后再接收下一段。

### MVP与荣誉称号系统的数据逻辑

MVP展示是多人对战结算画面的核心模块，需要基于后端实时计算的多维评分加权得出。以《英雄联盟》为例，MVP评定并非单纯依赖击杀数，而是综合击杀（25%权重）、助攻（15%）、伤害输出占比（20%）、承伤量（15%）、参团率（25%）五项指标的加权总分。UI层面需要在结算画面中为MVP玩家头像增加金色边框、特殊粒子效果，并在玩家昵称旁显示"MVP"徽章，以2.5倍于普通头像的尺寸呈现，形成自然的视觉锚点。

荣誉称号（如"最佳辅助""团灭发起者"）通常以小卡片形式在MVP展示完毕后以0.2秒间隔依次弹出，让每位参与玩家都能获得至少一条正向反馈，这一设计可降低非MVP玩家的挫败感，心理学上称为"参与感补偿机制"。

## 实际应用

**移动端关卡制游戏（如《愤怒的小鸟》系列）**：关卡结算画面在竖屏320×568像素基准下，三星评级图标占据画面中央40%面积，星星落下的物理模拟动画耗时约1.8秒，之后才显示"分享"和"下一关"按钮。按钮必须延迟出现，防止玩家因误触而跳过奖励动画，导致感知奖励价值降低。

**Battle Royale（大逃杀）游戏（如《PUBG》《Apex英雄》）**：结算画面需展示"存活排名/总玩家数"（如#3/100），同时提供本局击倒数、伤害总量、存活时长三项核心数据。《Apex英雄》的结算页面特别设计了"队友贡献对比条形图"，每名队员的伤害量用水平色条并排显示，让队友间的贡献差距一目了然，刺激下一局的竞争心理。

**角色扮演游戏经验条设计**：RPG结算画面中，经验值增长动画的时长应与增量成正比——获得500经验时动画播放0.8秒，获得5000经验时动画播放2.5秒，让玩家在视觉上"感受到"奖励数量的差异。若无论获得多少经验都播放等长动画，玩家会逐渐对经验数字失去敏感度。

## 常见误区

**误区一：把所有统计数据一次性全部显示**
部分开发者认为提供更多数据等于更高透明度，于是在结算画面塞入20项以上的战斗统计。实际上，人眼在一次视觉扫描中能有效处理的信息组块上限约为7±2个（Miller定律）。《守望先锋》早期版本曾因结算画面数据过密，在2.0版本中大幅削减至5项核心数据+1张英雄卡，玩家满意度随之上升。

**误区二：跳过动画的交互设计不当**
允许玩家跳过奖励动画是必要的（特别是多次重复刷图后），但"跳过"触发区域若覆盖整个屏幕，玩家在观看动画时的任何触屏都会意外跳过，造成"奖励被剥夺感"。正确做法是将跳过按钮限制在屏幕角落的明确区域（通常右下角，尺寸不超过44×44dp），并在首次进入时不显示跳过选项，强制播放完整动画以建立仪式感记忆。

**误区三：胜负画面使用相同的交互流程**
有些设计者为节省开发成本，胜利和失败结算使用完全相同的按钮布局和信息结构，仅替换标题文字和颜色。但失败状态下玩家处于负面情绪，"再来一局"按钮应比胜利状态下放大约20%、位置上移，减少视觉搜索时间；同时应移除或缩小社交分享按钮（玩家不愿分享失败结果），腾出空间给"战术建议"或"英雄推荐"等建设性内容。

## 知识关联

结算画面直接承接**排行榜界面**的设计逻辑：排行榜界面是玩家在游戏外通过全局历史数据进行自我定位的工具，而结算画面是玩家在单局游戏结束后立即获得的即时反馈面板。两者共享"排名可视化"和"数据对比"的设计语言，但结算画面强调时序性动画和情绪管理，排行榜界面则强调持久性信息检索和横向对比。

在技术实现层面，结算画面的数据来源涉及后端战斗统计服务、经验/等级系统接口、奖励发放服务三个独立模块，UI层需要处理"数据尚未返回时的Loading状态"和"奖励发放失败时的补偿提示"两类异常场景，这要求前端工程师在实现动画队列时必须设置超时保护（通常不超过8秒等待），超时后显示默认统计数据并标注"部分数据加载失败"。随着游戏体量增长，结算画面可进一步扩展接入赛季通行证进度、成就解锁弹出等更复杂的奖励系统，但每新增一类奖励模块，都需重新评估整体动画时序是否超过玩家的耐心阈值（通常为单局结算总时长不超过15秒）。
