# 菜单导航架构

## 概述

菜单导航架构是游戏UI系统中用于组织和连接多层级界面的结构设计方案，具体包含面包屑路径、标签页切换、层级跳转、历史回退等导航模式。其核心职责是让玩家在任意界面状态下都能准确感知"我在哪里"、"我能去哪里"以及"我如何回去"这三个空间定位问题。

菜单导航架构的系统化设计可追溯至2000年代初期PC游戏的多窗口设置界面，彼时《魔兽世界》（2004年）采用了垂直标签页（Character / Spellbook / Talents / PvP）将角色相关界面归类，奠定了现代游戏菜单分组导航的基础范式。移动游戏兴起后，屏幕空间限制迫使设计师进一步发展出底部标签栏、汉堡式折叠菜单和全屏堆叠层级等新型导航模式。

菜单导航架构的质量直接影响玩家的操作效率与认知负荷。研究表明，当玩家需要4次以上点击才能到达目标功能页时，放弃率会显著上升；而清晰的导航架构可将跨模块操作路径压缩至2-3步，这对于拥有背包、技能、任务、商店等十余个独立模块的中大型游戏尤为关键。

## 核心原理

### 层级深度控制：2-3层黄金法则

菜单导航架构遵循"最大层级深度不超过3层"的设计准则。第一层为主导航入口（如"装备"、"技能"、"设置"），第二层为功能分组（如"装备"下的"武器 / 防具 / 饰品"），第三层为具体操作界面（如单件装备的属性详情页）。一旦架构超过第三层，玩家的空间感知模型会崩溃——他们无法通过肌肉记忆记住返回路径，必须借助额外的视觉提示辅助定位。若内容量确实需要更深层级，设计师通常会通过将第三层改为水平滚动列表来"压平"深度，而非继续向下嵌套。

### 面包屑导航的空间锚定机制

面包屑（Breadcrumb）导航以线性文字路径展示当前位置，格式为 `主菜单 > 子菜单 > 当前页`。在游戏UI中，面包屑通常出现在界面顶部，字号约为正文的80%，采用低对比度色彩（如白底灰字）以避免与操作按钮竞争视觉焦点。面包屑的每一节点均可点击，点击任意祖先节点即跳回对应层级，这与单纯的"返回"按钮有本质区别——"返回"仅能逐层退出，而面包屑允许跨层跳跃。PC端RPG《路径放逐2》（2019年）的设置界面采用了三段式面包屑，允许玩家从"音频 > 环境音效 > 音量"直接跳回顶层"音频"，减少了重复点击次数。

### 标签页的并列分组逻辑

标签页（Tab Navigation）适用于同一层级下内容性质相互并列、且用户需要高频在组间切换的场景。标签页的核心设计约束是：**同一组内的标签数量应控制在3-7个**（来源：Miller's Law，人类短期记忆容量 7±2 个组块），超出范围时需进行二次分组或改用侧边栏列表。激活状态的标签页通常通过底部高亮线条、背景色填充或字重变化来区分，且激活标签页的内容区域需与非激活状态有明确的视觉边界。移动端游戏倾向于将标签页置于屏幕底部（Bottom Tab Bar），以适应拇指操作区；PC端则多用顶部横向标签页或左侧垂直标签页。

### 层级跳转与历史堆栈

层级跳转是指绕过常规导航路径、直接定位到深层界面的机制，常见触发方式包括：系统通知的直接跳转（如"您的邮件已送达"点击后直接进入邮箱详情）、快捷键呼出（如《暗黑破坏神4》中`I`键直接打开背包）、以及地图标记点击跳转至对应NPC界面。实现层级跳转时，后台必须维护一个导航历史堆栈（Navigation History Stack），记录玩家抵达当前界面的完整路径，确保"返回"按钮能按预期回溯，而非直接关闭界面。若历史堆栈设计不当，玩家会陷入"按返回无反应"或"连续点返回却绕回了主菜单"的导航陷阱。

## 实际应用

**手机RPG底部导航栏设计**：以《原神》为例，主界面底部并未采用常见的5图标底部标签栏，而是将频率最高的"背包"和"地图"设置为左右浮动按钮，减少了标签栏占用的屏幕高度，同时通过汉堡菜单折叠低频功能（邮件、公告、好友）。这一决策基于移动端1334×750像素分辨率下游戏画面优先的设计原则。

**PC策略游戏的侧边层级树**：《文明6》的城市管理界面使用左侧可折叠树状列表作为导航架构，玩家可在不离开主界面的情况下展开/收起各功能分支，避免了全屏切换带来的上下文丢失感。树状结构的缩进单位为16px，层级关系通过连接线和三角折叠图标传达。

**主机游戏的环形导航菜单**：《巫师3》采用径向菜单（Radial Menu）作为快捷导航架构，长按手柄`LB`键展开包含8个扇区的圆形选择器，结合模拟摇杆方向进行选择，将常用道具/技能的访问路径压缩至1步，彻底跳过层级导航。

## 常见误区

**误区一：将"返回"按钮等同于完整的导航系统**
部分开发者仅在每个界面放置"返回"按钮，认为这已足够支撑多层导航。但当玩家通过通知推送跳转至第三层界面时，按"返回"会回到第二层而非推送来源界面，与玩家预期不符。完整的导航架构需要区分"逐层返回"（Back）与"关闭并回到来源"（Close/Dismiss）两种逻辑，并在界面上用不同的图标（`←` vs `✕`）加以区分。

**误区二：标签页数量越多，功能覆盖越完整**
堆砌大量标签页会产生"标签过载"问题。当标签数超过7个时，玩家无法通过外围视觉扫描定位目标标签，必须逐个阅读文字，导致每次导航都产生额外认知成本。正确的应对方式是对标签进行功能聚合——将"外观"、"染色"、"时装"合并为"外观系统"单一入口，在二级界面内再做细分。

**误区三：面包屑导航在移动端可直接照搬PC设计**
PC端面包屑路径（如`设置 > 图像 > 后处理效果`）在1920px宽度下清晰可读，但在375px移动屏幕上会被截断或字号过小无法点击（建议最小触控区域44×44px）。移动端菜单导航架构通常需要将面包屑替换为单一"返回+标题"的顶部导航栏模式，仅展示当前页名称，用隐含的堆栈逻辑代替显式路径文字。

## 知识关联

菜单导航架构建立在**设置界面设计**的基础上——设置界面的多分类需求（图像、音频、控制、账号）是最早促使设计师系统化思考层级分组和标签切换的场景，学习者应理解设置界面中分组逻辑如何直接转化为导航架构的层级划分依据。

掌握菜单导航架构后，下一个学习重点是**背包/物品界面**。背包界面是游戏中导航复杂度最高的单一界面之一，通常包含"筛选标签页 + 网格列表 + 物品详情侧板"三级嵌套结构，其内部导航逻辑（如点击物品类别标签切换网格内容、选中物品展开详情面板）直接应用了面包屑定位、标签页分组和层级跳转的全套菜单导航架构原则。
