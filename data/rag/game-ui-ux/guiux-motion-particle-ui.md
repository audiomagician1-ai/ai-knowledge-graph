# UI粒子效果

## 概述

UI粒子效果是指在游戏界面中通过生成大量独立微小元素（粒子）并对其施加物理规则或曲线运动，从而产生视觉丰富反馈的动效技术。与骨骼动画或补间动画不同，粒子效果的视觉结果来自数十到数千个独立对象的集体行为，每个粒子拥有独立的生命周期、速度向量、透明度衰减和缩放曲线。常见形态包括按钮点击时向外爆散的光点尾迹、金币收集时沿贝塞尔曲线飞向计数器的硬币群、技能释放时扩散的符文碎片，以及签到奖励弹出时落下的彩色纸屑。

该技术在游戏UI领域的广泛应用可追溯到2010年前后移动游戏爆发期。《愤怒的小鸟》（2009年）率先将粒子爆炸反馈用于UI按钮，随后《Clash of Clans》（2012年）将金币粒子飞向资源栏的交互模式确立为移动策略游戏的事实标准。如今几乎所有主流手游的核心收集循环（core loop）都依赖粒子效果传递"获得感"。

UI粒子效果的价值不仅是装饰：它能将抽象的数值变化（如获得+500金币）转化为具身感知的视觉事件，将用户注意力从触发点精确引导至目标UI元素，从而强化操作→反馈→奖励的认知闭环。这一引导功能无法被静态图标或单纯的数字跳变替代。

## 核心原理

### 粒子参数系统：单粒子的五维生命周期

每个粒子的完整生命由五个参数轴控制：

- **生命时长（Lifetime）**：通常设置在 0.3s–1.2s 之间，金币飞溅类效果偏向 0.6s–0.8s，而按钮点击光晕类偏向 0.2s–0.4s。
- **初速度（Initial Velocity）**：以极坐标定义方向范围，例如向上方±30°扇形区域发射，初速度 800–1200 像素/秒。
- **透明度曲线（Alpha Curve）**：普遍采用"快入慢出"曲线，在生命周期前 20% 保持 Alpha=1，后 40% 线性衰减至 0，避免粒子突然消失的视觉切割感。
- **缩放曲线（Scale Curve）**：粒子在生命前期放大 1.0→1.3，后期收缩至 0，模拟爆炸膨胀再消散的物理感。
- **颜色渐变（Color Gradient）**：金币粒子常用黄→橙→透明的渐变路径，与资源图标颜色保持色彩一致性。

### 发射器类型与UI场景的对应关系

UI粒子效果主要使用三种发射器形态：

**点发射器（Point Emitter）** 适用于按钮点击反馈，粒子从触摸点爆散，在 0.05s 内一次性发射 15–30 个粒子，产生即时爆破感。

**路径发射器（Path Emitter）** 用于金币飞向目标栏效果。每枚金币粒子沿独立的二次贝塞尔曲线运动，控制点（Control Point）随机偏移±100–200像素，使多枚金币轨迹各异，避免整齐划一的机械感。飞行时延（Stagger Delay）设置为每枚间隔 30–60ms，形成"追随"的视觉节奏。

**区域发射器（Area Emitter）** 用于彩纸、星光背景等环境粒子，在整个面板区域持续低频发射（每秒 5–15 个粒子），生命周期较长（2s–4s），营造氛围而非传递即时反馈。

### 粒子数量与性能约束

移动端UI粒子效果的硬性约束是：**单帧活跃粒子总数不超过 200 个**（中端 Android 机型基准），超出此限制会导致 Draw Call 激增，帧率跌破 30fps。金币飞溅效果中，若一次性奖励触发 50 枚金币同时飞行，每枚金币若使用独立贴图则产生 50 次 Draw Call，需通过**粒子图集（Particle Atlas）** 合批为 1–2 次 Draw Call。Unity Particle System 的 GPU Instancing 选项和 Cocos Creator 的粒子合批功能均为解决该问题的标准方案。

## 实际应用

### 金币飞溅至资源栏

这是移动游戏中出现频率最高的UI粒子场景。标准实现流程：用户领取奖励 → 在奖励来源位置（如宝箱图标中心）生成 8–15 枚金币粒子 → 每枚粒子沿各自贝塞尔曲线飞向顶部资源栏的金币图标 → 粒子到达目标时触发资源数值跳变动画（数字+N弹出）。关键细节在于粒子**到达时刻**必须与数值变化**严格同步**，延迟超过 50ms 会破坏因果感知。

### 按钮点击粒子尾迹

在高频操作按钮（如"升级"、"抽卡"）上，点击瞬间在手指坐标发射放射状光点，粒子数量 20–40 个，颜色与按钮主色调匹配（蓝色按钮配白色+浅蓝粒子）。按住并拖动时若使用路径粒子尾迹，需注意将发射频率限制在 60Hz 以内，避免粒子密度过高遮挡按钮文字。

### 签到/成就解锁的礼花效果

全屏礼花效果常用于日历签到第7天或成就解锁弹窗。典型配置：4–6 种颜色的彩纸粒子从屏幕上方两侧角落发射，初速度向下偏移约 15°–30°，加入 ±200 度/秒的自转角速度使彩纸翻滚，并叠加 0.3–0.5 的重力系数使其自然下落。此类效果总粒子数控制在 80–120 个，生命时长 2–3s。

## 常见误区

### 误区一：粒子越多视觉效果越好

许多初学者认为增加粒子密度能提升"豪华感"，实际上超过 40 个同屏活跃粒子后，视觉上的"丰富感"边际收益急剧下降，而性能消耗线性增长。专业项目中金币飞溅通常只用 8–12 枚金币粒子，通过精心设计的贝塞尔轨迹差异和到达时延错开，营造远超数量的饱满感。粒子数量是最后考虑的参数，而非第一个调整的旋钮。

### 误区二：粒子效果可以独立于UI层级存在

粒子发射器的层级（Z-order）必须经过精确规划。金币飞溅粒子若渲染在弹窗遮罩层之下，会被半透明遮罩压暗，失去视觉冲击力；若置于所有UI之上，又会遮挡操作按钮导致误触。正确做法是为粒子效果单独划分 UI 渲染层，如在 Unity uGUI 中使用独立的 Canvas（Render Mode: Screen Space - Overlay，Sort Order 单独设定），并在粒子飞行期间禁用该区域的射线检测（Raycast Target = false）。

### 误区三：用循环粒子效果替代明确的状态指示

部分设计将持续循环的粒子效果（如按钮周围持续旋转的光点）用于表示"可点击"状态。这会产生两个问题：一是持续粒子消耗帧预算；二是粒子的视觉噪音会干扰用户扫视界面时对静态信息的读取。循环粒子效果的正确用途是**强调少数关键入口**（如每日任务红点+粒子组合），而非用于全局状态标识。一个界面中同时活跃的循环粒子发射器不应超过 2 个。

## 知识关联

UI粒子效果的触发与终止逻辑直接依赖**动画取消与中断**机制。当用户快速连续点击按钮时，前一次点击触发的粒子发射器必须能被安全中断——具体表现为已发射的存活粒子继续完成自身生命周期（不能强制清除，否则会产生粒子突然消失的跳变），但新的发射事件立即用新参数启动。这与骨骼动画的中断处理（需要混合过渡帧）不同，粒子系统的无状态特性使其天然支持重叠触发，每次点击独立启动一个发射脉冲即可。
