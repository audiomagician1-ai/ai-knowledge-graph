# 跨平台联机UI

## 概述

跨平台联机UI（Cross-Platform Multiplayer UI）是指在多人在线游戏中，当来自不同硬件平台（如PC、PlayStation、Xbox、Nintendo Switch、移动端）的玩家同处一个游戏会话时，界面系统需要统一展示每位玩家的平台身份标识、对应的输入设备图标以及操作提示的综合解决方案。其核心挑战在于：同一个"确认"操作，在PS5上是圆形或叉形按钮（因地区而异），在Xbox上是A键，在PC上是Enter键或鼠标左键，界面必须实时向每个人显示属于自己的正确提示，同时也要向队友展示对方的平台信息。

这一UI需求随着跨平台联机（Crossplay）功能的普及而变得紧迫。Epic Games于2019年率先在《堡垒之夜》中全面实装跨平台功能，随后索尼在2019年9月开放PlayStation网络的跨平台联机接口，使得开发者必须同时处理多达5种以上平台标识的显示问题。在此之前，游戏大厅只需展示单一平台图标；跨平台开放后，玩家列表中一行行来自不同平台的图标混排，成为标准UI需求。

这套方案的意义不仅是视觉装饰。当玩家在大厅中看到队友的平台图标（如Xbox圆圈标志或Switch手柄图标），可以预判对方的输入延迟特征和操作习惯差异；当系统显示"按 [A/X/空格] 跳跃"时，不同平台玩家各自看到正确按键，减少因操作提示错误导致的失误率。

## 核心原理

### 平台标识符体系与图标映射

跨平台联机UI的基础是一套平台枚举系统，通常定义为：`PLATFORM_PC`、`PLATFORM_PS4`、`PLATFORM_PS5`、`PLATFORM_XBOX_ONE`、`PLATFORM_XBOX_SERIES`、`PLATFORM_SWITCH`、`PLATFORM_IOS`、`PLATFORM_ANDROID`等枚举值。每个在线玩家的数据包中携带该枚举字段，UI层根据此字段从图标资源库中检索对应的SVG或PNG图标。

图标资源库需遵循各平台官方的品牌规范：PlayStation图标须使用索尼官方许可的PS标志（不得自行绘制），Xbox须使用微软提供的绿色圆圈X标志，Nintendo Switch须通过任天堂授权审核。许多中型工作室使用**Rewired**或**Unity's Input System**中附带的官方按键图标包，其中预置了约200+枚不同平台按键图标的矢量资源。

### 本地玩家与远程玩家的双重输入提示逻辑

跨平台联机UI需要区分两类显示场景，且逻辑完全不同：

**场景A（本地提示）**：当UI提示"当前玩家应按哪个键"，系统读取**本地设备的输入映射**，与远程玩家平台无关。例如PC玩家始终看到键盘图标，即使队友全为主机玩家。

**场景B（远程标识）**：当显示"队友X使用何种平台/输入设备"，系统读取**服务器同步的平台枚举字段**，仅做展示，不影响本地操作逻辑。

混淆这两个场景是导致跨平台UI实现错误的主要来源。两者的数据来源不同：本地提示来自`InputDevice.current`（Unity）或`GetMappedController()`（Unreal），远程标识来自网络状态同步层（如Photon、Mirror、或自研Network Manager）下发的玩家元数据。

### 平台图标的尺寸适配与深色/浅色主题

平台图标在不同背景色下的可读性差异显著：Xbox绿色图标在深色HUD中清晰，但叠加在亮色场景UI上时对比度不足。标准做法是为每套平台图标提供**两套色彩变体**（深色背景版/浅色背景版），并通过UI主题系统在运行时自动切换。图标最小可用尺寸通常为24×24像素（屏幕分辨率1080p下），低于此尺寸时平台标志的细节（如PS标志的字母轮廓）会因抗锯齿损失而失去辨识度。

在玩家列表（Lobby List）中，平台图标通常占据玩家名称左侧8~12像素宽的固定槽位，过宽会压缩玩家名称的显示空间，过窄则图标不可辨认。《使命召唤：现代战争2》（2022）的跨平台大厅采用了16×16的图标槽，被玩家社区反映过小，后续补丁调整至20×20。

## 实际应用

### 大厅玩家列表的平台标识显示

在组队大厅（Pre-Game Lobby）中，玩家列表每一行包含：平台图标 + 游戏ID + 延迟（ping值）。平台图标来自服务器端下发的`PlayerMetaData.platform`字段渲染，客户端本地无需知道远程玩家的具体设备型号，仅需对应枚举值到图标资源即可。《Rocket League》在2020年10月实装全平台跨联机后，其大厅UI就采用了此结构，且对账号系统从各平台独立账号统一迁移至Epic账号，平台标识保留显示以区分玩家来源平台。

### 游戏内HUD的动态输入提示

在游戏进行中，屏幕边缘或底部的操作提示条（如"[B/圆形/ESC] 取消互动"）需要根据当前活动输入设备动态更新。若PC玩家在游戏中途插入手柄，提示应在约0.5秒内从键盘图标切换为手柄图标。Unity Input System的`InputUser.onChange`事件和Unreal的`OnInputDeviceConnectionChange`委托均提供了此类回调，跨平台UI系统需监听这些事件并触发图标刷新。

### 跨平台语音频道与平台互通限制提示

部分平台存在语音互通限制（如Switch不支持与PC的原生语音），跨平台UI需在玩家列表中以灰色话筒图标+平台图标的组合，向用户明确标示哪些队友无法接收语音。这类"互通限制提示"是跨平台联机UI的延伸需求，需要在设计阶段提前规划图标状态枚举（可用/受限/不可用）。

## 常见误区

### 误区一：对所有玩家显示同一套输入提示

部分开发者错误地以"房主平台"为准，对整个大厅显示统一的输入图标（例如房主是PS5则所有人看PS5按键）。这会导致PC或Xbox玩家看到错误的操作指引。正确做法是每个客户端本地独立渲染输入提示，基于**本机当前活动输入设备**生成，与服务器端的平台元数据无关。

### 误区二：将平台图标硬编码为固定分辨率PNG

使用固定512×512的位图平台图标，在不同DPI屏幕（如4K显示器或Retina屏）缩放时会出现模糊失真。跨平台联机UI的平台图标应使用**矢量格式（SVG）**或**多分辨率图集（@1x/@2x/@3x）**，以保证在PC的4K分辨率和Switch掌机的720p屏幕上均保持清晰。

### 误区三：忽略同平台多账号的子平台区分

PS4与PS5玩家在同一跨平台房间中，若UI仅显示通用"PlayStation"图标，无法区分两者性能差异，在竞技场景中可能引发公平性争议。应区分`PS4`与`PS5`的独立枚举和图标（PS4使用蓝色PS图标，PS5使用白色PS5专属标识），《FIFA 22》在其跨平台匹配系统中就因初期未区分两代主机图标而受到玩家投诉。

## 知识关联

跨平台联机UI建立在**异形屏适配**的基础之上：异形屏适配解决了单平台设备上安全区（Safe Zone）和刘海区域的布局问题，为平台图标和输入提示确定了可靠的显示锚点。若安全区未正确配置，大厅玩家列表的平台图标可能被Switch的边框或iPhone的刘海遮挡。

在资源管理层面，跨平台联机UI与**UI图集（Sprite Atlas）**系统密切相关：所有平台图标（约8个平台 × 3种状态 × 2套主题 = 48枚图标）应打包进同一图集，避免运行时频繁切换图集引发Draw Call上升。在网络架构层面，平台元数据的同步依赖**玩家状态同步**模块，平台枚举字段应作为玩家进入房间时的首帧同步数据，确保大厅UI在玩家头像刷出的同一帧即可显示正确平台图标，而非等待第二次网络同步才补全。
