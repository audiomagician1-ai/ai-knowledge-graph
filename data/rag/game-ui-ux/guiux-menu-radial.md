# 环形/辐射菜单

## 概述

环形菜单（Radial Menu）是一种将选项呈放射状排列在圆形界面上的交互控件，玩家通过摇杆偏移方向或鼠标拖拽角度来激活对应选项，而非逐行点击。其核心设计逻辑基于菲茨定律（Fitts' Law）的"饼状菜单"变体——当所有选项到光标出发点的距离相等时，每个选项的命中难度理论上相同，这与传统线性列表中靠下选项更难点击的缺陷截然不同。

环形菜单最早可追溯至1988年Mark Tapia在学术论文中提出的"Pie Menu"概念，随后在1999年的《Never Winter Nights》工具集中获得商业游戏界的早期实践。真正让其普及的是2007年的《质量效应》（Mass Effect）系列，其对话轮盘与武器切换轮盘的双轮设计成为业界标杆，证明环形菜单在实时游戏节奏中能有效减少操作中断时间。

在游戏UI设计中，环形菜单的核心价值在于**控制器适配性**：PS/Xbox手柄的模拟摇杆天然输出一个角度值（0°–360°），与环形菜单的扇区划分形成一对一映射，无需玩家反复按方向键遍历列表。对于快节奏动作游戏，玩家在武器轮盘激活后通常有0.3–1.5秒的"时间减缓"窗口完成选择，这是该控件设计的标准配套机制。

## 核心原理

### 扇区划分与角度计算

环形菜单将360°圆周均分为N个扇区，每个扇区的中心角为 **θ = 360° / N**。以8选项武器轮盘为例，每扇区占45°，扇区边界分别落在22.5°、67.5°、112.5°等位置。判断玩家选择哪个选项的公式为：

> **选中索引 = floor((输入角度 + θ/2) / θ) mod N**

其中输入角度通常以12点钟方向（270°或-90°）为0点基准，顺时针递增。建议扇区数量控制在4–8个之间：少于4个时圆形空间浪费严重，超过8个时扇区过窄（<45°），手柄摇杆的机械误差会导致频繁误选。

### 死区（Dead Zone）设计

环形菜单必须在圆心附近设置**死区半径**，通常为圆形总半径的15%–25%。当摇杆偏移量或鼠标距离小于死区阈值时，系统不锁定任何选项，防止玩家轻触摇杆时误触发选择。死区半径过大会让玩家感到响应迟钝；死区半径过小则在手柄漂移（Stick Drift）情况下产生错误输入。Unity的Input System中可通过`Stick Deadzone`处理器直接配置此参数，推荐数值为0.125（归一化坐标系）。

### 选项数量与视觉反馈层级

环形菜单的可视化需要区分三种状态：**未悬停（Idle）、悬停高亮（Hover）、确认选中（Confirm）**。悬停扇区通常放大至原始尺寸的110%–125%并改变背景色，配合轻微的音效反馈（频率约800–1200Hz的短促提示音）。若选项不可用（如弹药耗尽的武器），应显示为灰色并保留在轮盘中，而非移除，因为动态扇区重排会破坏玩家形成的肌肉记忆——《暗黑破坏神3》的技能轮盘正是因为保持固定槽位布局而获得好评。

### 嵌套与分页策略

当选项超过8个时，推荐使用**嵌套环形菜单**而非简单增加扇区数。外层菜单显示类别（如武器/道具/技能），选定类别后内层菜单展开显示具体选项，形成两层放射结构。另一种方案是在环形菜单底部添加左右翻页箭头，但这会破坏纯角度操作的流畅性，仅适用于物品栏等管理界面，不适合战斗中的快速切换场景。

## 实际应用

**《侠盗猎车手V》武器轮盘**：GTA5的武器选择轮盘将武器分为8个类别扇区，激活时游戏进入子弹时间（时间流速降至25%）。玩家先用摇杆选定武器类别，松开LB/L1键即确认，整套操作平均耗时约0.8秒，远低于传统十字键循环切换的平均2.3秒。

**《无主之地2》技能快捷轮盘**：该游戏在PC端实现了鼠标驱动的4选项环形菜单，鼠标从菜单激活点拖拽超过40像素即开始锁定扇区，体现了PC端用距离代替角度作为激活阈值的设计变体。

**《守望先锋》通讯轮盘**：Blizzard将9条语音通讯短语分配到单个环形菜单中，通过将"我需要治疗"放置在最下方（270°方向，即手柄下拨最自然的方向）来提升高频指令的操作效率，这是利用扇区位置优先级服务游戏设计目标的典型案例。

## 常见误区

**误区一：选项越多越全面**  
部分设计者试图将所有动作塞入单个环形菜单，导致12个甚至16个扇区并存。当扇区角度小于22.5°时，手柄摇杆在该区域的有效像素分辨率不足（大多数手柄摇杆的实际角度精度约为±5°），玩家稳定选中目标扇区的成功率会大幅下降。黄金法则是**单层环形菜单不超过8个扇区**。

**误区二：在键鼠为主的PC界面直接照搬手柄设计**  
鼠标的输入特性与手柄摇杆不同——鼠标可以精确移动到屏幕任意位置，其操作天然适合传统下拉菜单。将环形菜单强行套用在PC纯键鼠游戏中，若未针对鼠标调整激活逻辑（如改为"从中心点拖拽后释放"而非"偏移角度持续检测"），会让玩家感到操作别扭。《地铁：离去》的PC版就因为照搬主机环形菜单逻辑而收到大量负面反馈。

**误区三：激活后立即暂停游戏**  
认为"必须暂停才能让玩家看清选项"是错误假设。《质量效应》和《GTA5》均采用时间减缓而非完全暂停，保留了战场紧张感，也防止多人游戏中产生无限暂停的公平性问题。完全暂停仅适用于单机游戏的装备管理场景，而非战斗中的快速选择场景。

## 知识关联

环形菜单与**弹窗和对话框**的核心区别在于交互模态：对话框要求玩家完全停止游戏操作并做出确认/取消的二元决策，而环形菜单设计目标是在不打断游戏流程的前提下完成多选一操作，两者适用的决策复杂度和时间压力完全不同。理解对话框的"模态覆盖"概念有助于设计者判断何时应升级为对话框（如不可逆操作的二次确认），何时应保持环形菜单的轻量交互。

进入**标签页系统**学习时，会发现两者在信息组织策略上有互补关系：标签页适合管理大量同类内容的横向切换（如角色属性的各个维度），而环形菜单适合少量异类快捷操作的即时触发。在复杂游戏界面设计中，装备管理界面可能同时包含标签页（分类展示物品）和嵌套环形菜单（快速装备/丢弃），两种控件各司其职，组合使用时需要明确激活条件的优先级以避免输入冲突。
