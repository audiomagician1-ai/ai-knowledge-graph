# 任务标记与导航

## 概述

任务标记与导航是HUD设计中专门用于引导玩家完成目标的视觉信息系统，涵盖任务目标指示器（Objective Marker）、路径导航（Pathfinding Indicator）、距离标识（Distance Label）和兴趣点标记（POI Marker）四类核心元素。这套系统的本质是将游戏世界坐标转化为玩家屏幕上的二维导航信息，从而消除玩家在三维空间中的方向迷失感。

任务标记系统的大规模普及始于2000年代初期。《光晕2》（2004）率先将任务标记固定为罗盘箭头形式，而《侠盗猎车手：圣安地列斯》（同年）则将小地图导航与世界内悬浮标记结合，确立了"双层导航"设计范式。2007年《刺客信条》引入了同步塔解锁与鹰眼视角标记系统，将兴趣点发现行为本身游戏化，改变了POI标记的呈现逻辑。

导航标记的设计质量直接影响游戏叙事沉浸感与操作流畅度之间的平衡。过度密集的标记会造成"导航依赖症"，削弱玩家的空间探索意愿；而标记缺失或信息不清则导致玩家频繁查阅菜单，打断游戏节奏。《荒野大镖客：救赎2》选择提供可关闭的导航系统，并用环境线索（日出方向、地貌特征）替代HUD标记，是任务标记设计哲学中最具代表性的极简化实验。

## 核心原理

### 世界坐标到屏幕坐标的投影计算

任务标记的技术基础是**Billboard投影**：将3D世界空间中目标点的坐标，通过摄像机的视图矩阵（View Matrix）和投影矩阵（Projection Matrix）变换为归一化设备坐标（NDC），再映射至屏幕像素位置。当目标点位于摄像机视锥体（Frustum）之外时，系统需要将标记"夹"（Clamp）至屏幕边缘并显示箭头，这一技术称为**屏幕边缘指示器（Screen-Edge Clamping）**。

屏幕边缘箭头的角度θ计算公式如下：

> **θ = atan2(目标NDC的Y分量, 目标NDC的X分量)**

该角度决定边缘箭头的朝向，配合一个固定偏移量（通常为距屏幕边缘32至48像素）确定箭头最终位置。《巫师3》的任务标记即采用此方案，并在箭头旁叠加距离数值。

### 距离标识的信息分级机制

距离标识并非简单显示米数，优秀的设计会对数值进行**分级离散化**处理。例如，《地平线：零之曙光》将距离分为四档显示逻辑：
- **0–20米**：不显示数字，标记放大并加入脉冲动画，提示玩家已抵近目标
- **20–100米**：精确显示米数（如"67m"）
- **100–1000米**：取整至10米（如"340m"）
- **1000米以上**：转换为千米单位（如"1.2km"）

这种分级处理减少了玩家眼睛在HUD与游戏画面之间的频繁跳转，同时避免数字不断跳变引起的视觉噪音。

### 兴趣点标记的优先级排队系统

当场景中同时存在多个POI标记时，重叠遮挡问题必须通过**标记优先级（Priority Queue）**和**布局防碰撞算法（Label Deconfliction）**解决。一般按以下优先级顺序排列：

1. 主线任务目标（Priority 1，始终可见）
2. 支线任务激活目标（Priority 2）
3. 可交互物件与NPC（Priority 3，近距离才显示）
4. 收集品与隐藏地点（Priority 4，需特定条件解锁）

《巫师3》的POI系统允许玩家在地图界面手动筛选各优先级的显示类别，将显隐控制权还给玩家，是标记管理的成熟设计方案。

### 路径导航的实现形式

路径导航在视觉呈现上主要分为三种形式，各有适用场景：

- **小地图箭头**：信息密度低，不干扰主视角，适合开放世界驾车场景（如《GTA5》）
- **世界内路径线**（World-Space Path Line）：直接在地面渲染引导路径，沉浸感强，计算开销高，代表作为《惑星探险》
- **动态罗盘**（Dynamic Compass）：屏幕顶部水平罗盘条，目标方向标记随摄像机朝向滚动，是FPS游戏的主流方案（《使命召唤》系列、《战地》系列均采用此设计）

## 实际应用

**《原神》的七天神像任务标记**采用了层叠式兴趣点设计：圆形任务标记在地图缩放时会根据当前缩放层级聚合（Cluster）为组图标，防止密集区域的标记堆叠混乱。玩家接近某组标记时，聚合图标自动展开为独立图标，这套"缩放自适应"逻辑至今是开放世界手游导航设计的标准参考。

**《死亡搁浅》**将任务标记与地形海拔整合：标记不仅显示水平距离，还叠加垂直高差指示（↑312m），帮助玩家判断路线难度，这在以地形攀爬为核心玩法的游戏中大幅降低了路线决策成本。

**《暗黑破坏神4》**的地下城内部使用固定指南针方向的小地图，配合橙色任务箭头，解决了迷宫环境下玩家在密集战斗中丢失方向感的问题，将标记颜色（橙）与敌人颜色（红）刻意区分，避免视觉混淆。

## 常见误区

**误区一：距离越精确越好**
许多初学者认为任务标记应实时显示精确米数，但持续跳动的数字（如"173m → 172m → 171m"）会造成玩家无意识地注视距离数字而非观察游戏画面。正确做法是采用前述分级离散化，或设置更新频率上限（如每0.5秒刷新一次而非每帧刷新）。

**误区二：所有POI标记应当始终可见**
将所有兴趣点标记设为全程可见会使探索感归零。《黑暗之魂》系列刻意不设置任何POI标记，其核心玩法正建立在玩家自行建立空间认知的设计哲学上。即便在需要导航标记的游戏里，也应区分"已发现"与"待发现"状态，避免提前剧透地图内容。

**误区三：屏幕边缘箭头指向目标的直线方向即可**
直线方向箭头在有障碍物的封闭空间（如室内、峡谷）中会严重误导玩家朝墙壁走去。实际开发中，屏幕边缘箭头应结合导航网格（NavMesh）的寻路结果，指向"通往目标的下一个导航节点"，而非目标的欧几里得直线方向。《最后生还者》系列的"倾听模式"标记即采用了路径节点方向而非直线方向。

## 知识关联

学习任务标记之前需掌握**弹药与资源显示**的布局原理——两者都涉及屏幕空间的优先级分配，弹药计数器通常固定于屏幕右下角，而任务标记则占用屏幕边缘与中央空间，两套系统必须在设计阶段协调避免相互遮挡。

完成本概念的学习后，下一步将进入**伤害方向指示**的设计逻辑。任务标记已建立了"屏幕边缘信息编码"的基本认知模型——用屏幕外围区域传递方向性信息；伤害方向指示沿用相同的屏幕边缘空间，但信息语义从"目标在哪里"转变为"威胁从哪里来"，两者在交互设计层面存在明确的视觉空间竞争关系，需要在HUD整体布局中统一规划其出现时机与透明度级别。
