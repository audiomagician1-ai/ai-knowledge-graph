# 工具提示设计

## 概述

工具提示（Tooltip）是游戏UI中一种当玩家悬停、长按或聚焦于某个可交互元素时，自动弹出的信息浮层。它的核心任务是在不中断游戏流程的前提下，按需提供补充说明，例如装备属性、技能效果或快捷键绑定。区别于全屏弹窗或模态对话框，工具提示通常持续时间短暂，玩家一旦移开焦点便自动消失，不需要额外的关闭操作。

工具提示的概念最早在桌面软件界面中成熟，1992年Windows 3.1工具栏引入了气泡式提示标签，随后被移植到游戏领域。《魔兽世界》（2004年）大幅拓展了游戏工具提示的内容密度，将装备对比、词缀说明、出售价格等多层信息整合进单一浮层，成为后续RPG游戏工具提示设计的参考范本。

在游戏中，工具提示承担着降低学习成本的关键职责。一个设计不当的工具提示——例如触发延迟超过500毫秒、文字超过80字仍不分层、或浮层遮挡当前操作目标——都会直接造成玩家挫败感和误操作率上升。正确的工具提示设计能让新玩家快速理解复杂机制，同时不干扰熟练玩家的操作节奏。

## 核心原理

### 触发方式与延迟控制

触发方式决定了工具提示何时出现，分为以下三类：

- **悬停触发（Hover）**：鼠标停留于元素上方，适用于PC端。标准延迟区间为300–500毫秒，低于300毫秒容易因误触而频繁弹出，高于500毫秒则让玩家感到迟钝。
- **长按触发（Long Press）**：手指按住元素0.6–1秒，是移动端的主流方案，避免与点击操作冲突。
- **焦点触发（Focus）**：手柄或键盘导航时，元素获得焦点即刻显示，延迟应控制在0毫秒以内，因为手柄玩家已有明确的主动意图。

消失触发同样重要：鼠标离开目标区域后，工具提示应在100–200毫秒内淡出，而非瞬间消失，以避免视觉跳变。

### 内容层级结构

工具提示的信息应遵循"标题—核心数据—扩展说明"三层结构：

1. **第一层（必读）**：元素名称 + 最关键的单一数值，例如"火球术 — 造成 350 点火焰伤害"。字号应比正文大2pt，高对比度显示。
2. **第二层（辅助）**：冷却时间、范围、消耗资源等次要属性，用较小字号或灰色文字区分。
3. **第三层（可选）**：词缀效果、与其他技能的联动说明、版本变更记录等深度内容，可折叠或通过按下特定键（如Shift）展开。

《暗黑破坏神4》的装备提示便采用了类似结构：主属性用白色加粗显示，词缀效果用金色文字，传奇词条用橙色，使玩家在0.5秒内完成视觉优先级排序。

### 显示位置策略

浮层位置需遵循"不遮挡操作目标"原则，通常采用优先级算法：

1. 优先显示在目标元素的**右侧**（西方阅读习惯）；
2. 若右侧空间不足（距屏幕右边缘小于浮层宽度），则翻转至左侧；
3. 若垂直方向空间不足，则上移或下移，确保浮层完整显示在安全区内；
4. 移动端还需额外避开虚拟键盘区域（屏幕下方约40%高度）。

计算公式示意：`tooltipX = clamp(targetX + offsetX, margin, screenWidth - tooltipWidth - margin)`，其中`margin`通常取8–12像素，保证浮层与屏幕边缘保持安全距离。

## 实际应用

**RPG装备对比**：在《原神》的角色强化界面，玩家悬停圣遗物时，工具提示会同步显示"当前装备"与"待装备"的属性差值，正向提升用绿色"▲+数值"标注，下降用红色"▼−数值"标注。这种对比式工具提示将两个独立信息块整合为一个决策工具，省去玩家手动记忆的步骤。

**策略游戏兵种说明**：《文明6》中，鼠标悬停于地图单位时，工具提示分两级展示：首层显示单位名称、战斗力数值和移动力；按住Alt键后展开第二层，列出对特定地形的战斗加成和促销条件。这种按需展开的机制避免了新玩家被数据淹没，同时满足了硬核玩家的信息需求。

**移动端技能描述**：《王者荣耀》对技能按钮采用长按0.8秒触发工具提示，同时屏蔽该次长按的技能释放，防止玩家在查看说明时意外操作。提示内容限制在120字以内，强制使用短句和数据替代定性描述（如"造成200%物理攻击伤害"而非"造成大量伤害"）。

## 常见误区

**误区一：工具提示越详细越好**
将所有相关信息堆入同一浮层会导致信息过载。研究表明，超过5行的工具提示完读率低于20%。正确做法是将深度内容保留给第三层（需主动触发），而非默认展示。《炉石传说》的牌面说明平均仅15–25字，证明精简描述同样能覆盖核心规则。

**误区二：触发延迟可以忽略不计**
部分开发者将延迟设为0毫秒以求"响应迅速"，实际效果是玩家在正常浏览界面时频繁触发无关提示，尤其在拥挤的背包界面中。0毫秒延迟在密集元素场景下会造成提示连续闪烁，干扰视线。正确方案是区分场景：稀疏元素界面可缩短至200毫秒，密集格子背包建议保持400毫秒以上。

**误区三：PC端工具提示方案可以直接移植到手机端**
悬停触发在触摸屏上不存在，若直接将PC端设计移植为"点击触发"，会与选择、使用等主要操作冲突。移动端必须为工具提示单独规划触发手势（长按）和退出逻辑（点击浮层外区域或提供明确关闭按钮），两套方案的交互逻辑根本不同。

## 知识关联

工具提示的**延迟控制逻辑**与进度条和加载指示共享同一套"感知等待时间"原则：两者都依赖100–500毫秒的感知阈值来区分"即时反馈"与"等待反馈"。进度条设计中学到的用户等待容忍度研究（Doherty阈值400毫秒），可直接用于校准工具提示的悬停延迟参数。

工具提示设计完成后，下一个交互反馈模块将聚焦**屏幕震动反馈**。工具提示属于"视觉信息层"，震动反馈属于"触觉感知层"，两者在游戏中常配合使用：当玩家长按触发装备提示的同时，轻微震动（通常为20–30毫秒短脉冲）能强化"已识别该物品"的确认感。理解工具提示如何在视觉层传递结构化信息，有助于判断哪些场景需要额外叠加触觉反馈，哪些场景中震动反而会打断玩家的阅读状态。
