# 聋人/听障适配

## 概述

聋人/听障适配是游戏无障碍设计中专门针对听觉障碍玩家的一套设计方案，核心任务是将游戏中所有依赖声音传达的信息转换为视觉、触觉或文字形式呈现。根据世界卫生组织2021年数据，全球约15亿人存在不同程度的听力损失，其中约4.3亿人的听力损失达到致残程度，这一群体在游戏领域长期面临信息获取障碍。

听障适配最早在主机游戏领域受到重视始于2000年代中期，《战争机器》（2006）因其在屏幕上将枪声、脚步声等关键音效以文字或图标形式标注而受到听障玩家社区广泛称赞，被视为该领域的早期里程碑。Xbox无障碍指南（Xbox Accessibility Guidelines，XAG）后来将听障适配拆解为多个子类要求，为开发者提供了系统性框架。

听障适配的重要性在于：游戏中大量关键信息（敌人方向、技能触发音效、剧情对话、环境危险警报）默认通过音频传递，若缺乏等效的非音频替代方案，听障玩家将在信息获取上处于系统性劣势，导致游戏体验被动降级而非主动选择。

## 核心原理

### 视觉提示替代音频提示

视觉替代音频的基本原则是**等效性**——视觉信息不应是音频信息的"摘要"，而必须传递相同的内容量和时效性。常见实现方式包括：

- **字幕系统（Subtitles vs. Closed Captions）**：需区分两种类型。字幕（Subtitles）仅转录对话台词；隐藏字幕（Closed Captions，CC）则额外包含音效描述，如`[脚步声接近]`、`[爆炸声，右侧]`。Xbox无障碍指南明确要求游戏提供CC级别的字幕，而非仅有对话字幕。
- **音频可视化指示器**：将定向音效（如枪声来源、敌人呼叫）转化为屏幕边缘的方向箭头或扇形高亮区域。《最后生还者2》（2020）实现了"音频描述气泡"系统，在屏幕上以同心圆扩散动画表示爆炸等事件的发生，同时附带方向标记。
- **视觉闪光替代音效触发**：当游戏音效代表技能冷却完毕、任务达成等事件时，配套加入屏幕边角的短促闪光或图标高亮动画，确保不依赖扬声器也能感知时机。

### 震动反馈（触觉替代）

控制器震动是听障适配的重要第三通道，利用触觉信息弥补声音缺失。XAG要求震动设计需满足以下标准：震动强度与频率应与对应的声音事件形成映射关系，而非单一强度的泛化震动。例如：
- 低频持续震动（左马达主导）→ 环境危险如地震、大型爆炸
- 高频短促震动（右马达主导）→ 受击反馈、子弹命中
- 双马达交替节奏震动 → 警报倒计时

PS5的DualSense控制器支持触觉反馈（Haptic Feedback）和自适应扳机（Adaptive Trigger），《Returnal》（2021）通过不同的扳机阻力表示武器状态，为听障玩家提供了不依赖音效的状态感知渠道。

### 方向指示系统

纯视觉的方向指示需解决"音源定位无法通过双眼判断"的根本问题。常用方案包括：

- **屏幕边缘方向指示弧**：在屏幕四周放置半透明弧形UI，危险来源方向的弧段会高亮并伴随图标。《使命召唤》系列自2010年前后引入"受击方向指示器"，以屏幕边缘红色扇区标示受弹方向。
- **小地图音源标注**：在小地图上以图标或闪烁点标注关键音源位置，如敌人叫喊、机关触发、道具掉落音。
- **镜头震动方向联动**：当爆炸或碰撞发生时，镜头轻微偏向事件方向，配合屏幕闪光，为听障玩家提供空间方位感知。

## 实际应用

**《最后生还者2》的综合听障模式**是目前业界最完整的实现案例。该游戏允许玩家独立开启"声音可视化"功能，所有环境音效（风声、水声、敌人移动）均转化为屏幕上的涟漪图标；对话字幕支持自定义字体大小（最大可调至屏幕高度的8%）、背景不透明度和颜色；同时内置"说话者颜色标识"，不同角色的字幕以不同颜色区分，解决多人对话归属混乱问题。

**《赛博朋克2077》**在2.0版本更新后新增了"声音提示视觉化"选项，将网络入侵成功音效、武器过热警告等30余种游戏状态音效对应的视觉指示器全部独立开关，允许听障玩家按需开启而不影响整体画面信息密度。

**《Forza地平线5》**为听障玩家提供赛车引擎声的视觉转速计强化版本——转速计在临近红线时会以颜色梯度（绿→黄→红）配合边框闪烁提示，无需依赖引擎音调变化判断换挡时机。

## 常见误区

**误区一：有字幕就足够了**
许多开发者认为添加对话字幕即完成了听障适配。然而对于动作游戏、恐怖游戏等高度依赖音效进行危险预警的类型，仅有对话字幕的听障玩家仍会在"听脚步音判断敌人位置"、"听警报音理解关卡机制"等核心玩法上处于严重劣势。隐藏字幕（CC）与完整的音源可视化系统缺一不可。

**误区二：震动反馈对聋人玩家无用**
部分开发者认为震动属于辅助体验而非信息渠道。事实上，大多数聋人玩家拥有正常的触觉感知，控制器震动是替代低频音效（爆炸、碰撞、心跳警示）最直接的手段。XAG专门将"意义明确的震动映射"列为听障适配的独立检查项，而非仅要求"提供震动选项"。

**误区三：听障适配会破坏游戏氛围**
恐怖游戏开发者常担心将恐怖音效可视化会削弱紧张感。《沉默岭》类游戏的实践表明，将噪音计（如《沉默岭》系列的收音机干扰指示器）转化为视觉UI反而可以创造独特的紧张感视觉语言，听障模式与非听障模式可共享同一套紧张感机制的不同呈现形式。

## 知识关联

聋人/听障适配建立在**Xbox无障碍指南（XAG）**所规定的无障碍分类框架之上，XAG将听觉障碍适配单独列为"Hearing"类别，提供了从字幕规范到震动设计的具体评分标准，是开发者实施本章所述技术的参照依据。

学习完听障适配后，下一步应深入**无障碍设置菜单**的设计。听障适配涉及的字幕开关、声音可视化强度、震动映射方案等均需通过无障碍设置菜单向玩家暴露，菜单的分类逻辑、选项命名规范和实时预览功能直接决定听障玩家能否独立发现并激活这些适配功能——一个隐藏在五级子菜单中的字幕开关，其实际价值与没有字幕功能几乎等同。
