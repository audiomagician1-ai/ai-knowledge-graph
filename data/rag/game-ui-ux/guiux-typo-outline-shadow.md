# 描边与阴影

## 概述

描边与阴影是游戏UI字体排版中用于提升文字可读性的两类视觉处理技术。描边（Stroke/Outline）指在字体轮廓外侧或内侧均匀附加一定像素宽度的色彩边缘；阴影（Drop Shadow）指在字体后方偏移位置渲染半透明或实色的字形投影。二者可以单独使用，也可以与半透明背板叠加组合，共同解决文字在复杂背景上失去对比度的问题。

这两种技术在早期像素游戏时代就已出现。任天堂FC（1983年）时代受限于硬件调色板数量，开发者用单像素白色描边使文字从深色背景中分离。进入现代3D游戏时代，描边粗细精确到0.5px级别，阴影方向和模糊半径也可实时计算，但其本质逻辑与早期像素描边并无本质区别——都是通过边界对比度让字形在任意背景上保持可辨识性。

在实际项目中，描边与阴影直接影响玩家在高动态场景（如爆炸特效、快速移动镜头）中读取血量、弹药等关键数值的速度。测量数据表明，添加1~2px白色描边可以将深色背景文字的可读性对比度从不达标的2.5:1提升至符合WCAG AA标准的4.5:1以上。

## 核心原理

### 描边粗细的选择规范

描边粗细通常以字号的百分比衡量。对于18px的游戏HUD字体，外描边宽度在1~2px时视觉干净；超过3px则会侵蚀字形内部笔画间距，导致细字体（如宋体细体）笔画粘连。描边颜色并不局限于黑白——深色描边可使高亮金色字体在白色背景上突出，浅色描边则用于深色调夜间场景UI。

内描边（Inner Stroke）与外描边（Outer Stroke）的渲染结果差异显著：外描边会等比向外扩展字形轮廓，实际显示尺寸比设计稿略大；内描边向字形内侧压缩，会削减笔画粗细，适合描边色与字体色接近的微妙质感处理。Unity中SDF（Signed Distance Field）字体渲染使用一个材质参数`OutlineWidth`（范围0~1，对应0~约4px），其描边边缘的平滑度由`OutlineSoftness`单独控制，不会与粗细参数耦合，这是传统位图字体描边无法做到的分离。

### 投影方向与偏移量

游戏UI阴影的方向约定通常遵循屏幕光源一致性原则：若UI整体光源在左上角，投影方向应为右下方（X偏移+2px，Y偏移+2px）。阴影偏移量过小（小于1px）在低分辨率屏幕上几乎不可见；偏移量过大（超过字号的25%）则产生"错位感"，使文字看起来像漂浮在背景上方。

阴影模糊半径（Blur Radius）决定投影边缘的扩散程度。硬阴影（模糊半径=0）常见于像素风格RPG，强调复古感；软阴影（模糊半径=2~4px）适合现代写实或科幻风格UI，柔化边缘使整体视觉更精致。CSS中`text-shadow`的语法为`text-shadow: Xpx Ypx Blur Color`，游戏引擎（如Godot的`Label`组件）也提供等价的`shadow_offset`与`shadow_color`属性，均遵循同一套参数模型。

### 半透明背板的搭配逻辑

半透明背板（Scrim/Overlay Panel）是描边和阴影之外的第三种可读性方案，也是三者组合时需要避免过度叠加的关键节点。背板通常使用`rgba(0,0,0,0.4)`至`rgba(0,0,0,0.6)`的黑色半透明矩形或圆角矩形，放置于文字层与背景层之间。其核心作用是局部降低背景亮度方差，让文字在统一的暗色底上显现，而非依赖描边逐字建立对比度。

当场景背景对比度极高（如白色雪地场景叠加深色岩石）时，单纯依靠2px描边仍可能失效，此时背板是更稳定的方案。但若背板与描边同时叠加，视觉重量会急剧增加，容易让HUD显得厚重压抑。实践标准是：**背板、描边、阴影三选二**——写实风格UI常用背板+软阴影，像素/卡通风格UI常用硬描边+硬阴影，二者任选后不再叠加第三种。

## 实际应用

**《英雄联盟》伤害数字**：红/黄色数字使用1.5px黑色外描边 + 右下方2px硬阴影，确保在任意地图色彩上保持可读，无背板。

**《原神》对话字幕**：白色正文文字下方铺设`rgba(0,0,0,0.5)`的渐变背板，无描边，投影模糊半径约2px，整体保持轻盈的视觉氛围。

**《塞尔达传说：荒野之息》任务提示**：黑色粗体文字配合白色2px外描边，没有投影，适配户外强光环境下的横向渐变背景条，描边与背景条共同保障可读性。

**《Among Us》名称标签**：2px白色外描边 + 无阴影 + 无背板，纯描边方案，在复杂多色背景中实现最低渲染开销的可读性保障，匹配其低多边形视觉风格。

## 常见误区

**误区一：描边越粗越清晰。** 对14px以下的小字号，3px以上外描边会填满字母之间的反白空间，例如"田""目"等汉字的内部笔画完全粘连，反而降低识别率。正确做法是小字号优先使用半透明背板或调整字体选择，而非一味加粗描边。

**误区二：阴影只有黑色一种选择。** 在暖色调夜间场景中，使用深棕色（`#3D1A00`）阴影比纯黑色更融合环境，同时依然提供足够的对比分离。冷色科幻UI中使用深蓝色阴影也是常见设计选择，纯黑阴影在这些场景中反而显得生硬突兀。

**误区三：阴影与描边可以替代可读性测试。** 描边和阴影是视觉补偿手段，不能替代对字号、字重的根本性规划。若基础字号低于16px且字重为Regular，即使叠加描边+阴影+背板，快速移动的动态场景中玩家仍无法在0.5秒内准确读出数值。描边与阴影只在字号和字重已经基本合理的前提下才能发挥稳定作用。

## 知识关联

**前置概念——风格化字体设计**：描边的颜色选择和粗细比例须与字体本身的笔画风格一致。像素字体（每笔画1~2px宽）只适合整像素描边；衬线风格化字体的衬线端点在外描边后容易产生锐角尖刺，需要在矢量层对锚点做圆角处理。不理解字体本身的笔画结构，就无法为描边宽度设定合理上限。

**后续概念——响应式文字**：描边和阴影的参数不能使用绝对像素值在多分辨率UI系统中固定写死。在响应式布局中，描边粗细应跟随字号按比例缩放（如始终保持字号的8%），阴影偏移量也需要绑定到UI缩放系数（UI Scale Factor）上同步变化。若不处理这层缩放关系，1080p下精心调校的1.5px描边在4K分辨率下会显示为不足1px的虚边，在320px宽手机屏幕下则变成压迫性的粗框。
