# 远程可用性测试

> **领域**: 游戏UI/UX > 可用性测试 | **难度**: ⭐⭐⭐ | **预计学习时间**: 29分钟

## 核心概念

线上远程测试的工具、流程和样本招募策略

## 测试方法

1. **定性方法** — 出声思维、观察法、深度访谈
2. **定量方法** — A/B测试、任务完成率、操作时间
3. **分析方法** — 热力图、漏斗分析、行为埋点

## 关键指标

- **任务完成率** — 成功完成指定操作的百分比
- **操作步骤数** — 完成任务所需的实际vs最优步骤
- **首次使用时间** — 新用户首次成功完成任务的时间
- **错误率** — 操作过程中的误操作频率

## 实践要点

- 测试应尽早开始(纸面原型阶段即可)
- 5-8个测试者即可发现80%的可用性问题
- 结合定性和定量方法获得全面洞察
- 迭代测试: 修复→重测→验证

## 工具推荐

- **Lookback.io** — 远程用户测试录制
- **Hotjar** — 热力图和行为分析
- **UserTesting** — 大规模用户测试平台

## 关键术语
