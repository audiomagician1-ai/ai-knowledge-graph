# 悬停状态设计

## 概述

悬停状态设计（Hover State Design）是指当玩家将鼠标指针移动到可交互UI元素上方但尚未点击时，该元素所呈现的视觉、听觉或信息变化的设计规范。这一状态存在于"静息态"与"按压态"之间，持续时间通常为玩家决策前的数百毫秒到数秒。游戏UI中悬停状态的核心职责是传达"此元素可以被点击"，同时提前展示点击后可能发生的内容或代价。

悬停状态的系统性设计规范最早在桌面端GUI中被确立。Windows 95的界面指南首次明确要求按钮需要在鼠标悬停时改变边框颜色，而游戏UI对其进行了大幅扩展——从《魔兽世界》2004年上线时引入的装备对比悬浮框，到现代游戏中带动态粒子效果的技能预览卡片，悬停状态承载的信息量已经远超简单的颜色变化。

在游戏UI中，悬停设计的重要性体现在它直接影响玩家的决策质量。RPG装备界面中，玩家在每次点击前通常需要阅读悬停弹出的属性对比信息；策略游戏中，悬停在地图单位上时显示的兵力预览决定了玩家能否做出有效部署。缺乏完善悬停设计的游戏界面会迫使玩家大量使用"试错式点击"，显著增加操作失误率。

## 核心原理

### 视觉变化的三种基本模式

悬停触发的视觉变化通常分为三种模式：**亮度提升**、**轮廓强调**和**尺寸缩放**。亮度提升是最常见的做法，将元素亮度提高10%–20%即可有效传达"激活感"，超过30%则会造成视觉刺激感过强。轮廓强调通过在元素外侧添加1–3像素的高亮描边实现，常见于《英雄联盟》的技能图标。尺寸缩放通常将元素放大至原尺寸的105%–115%，配合缓动动画（ease-out曲线，持续100–150毫秒）可以产生自然的"浮起"感。三种模式往往组合使用，但同时叠加三种效果容易导致界面噪声过高。

### 工具提示（Tooltip）的布局规则

工具提示是悬停状态中承载最大信息量的组件，其触发延迟通常设定在300–500毫秒之间。延迟过短（<200ms）会导致玩家快速移动鼠标时频繁触发无意义的弹出；延迟过长（>800ms）则会让玩家产生"界面没有响应"的错觉。工具提示的弹出位置应遵循**边界检测原则**：优先显示在目标元素右侧，当右侧空间不足时自动镜像至左侧，当上下空间受限时调整垂直方向，确保提示框始终完整显示在屏幕可视区域内。《暗黑破坏神4》的装备提示框就采用了这套动态定位算法，避免了提示内容被屏幕边缘截断的问题。

### 预览信息的分级展示

悬停状态下的信息展示应遵循**信息分级原则**，将内容按重要程度分为主要信息（Primary）和次要信息（Secondary）两层。以技能图标为例：主要信息包括技能名称、伤害数值、冷却时间；次要信息包括机制说明、数值成长公式（例如"伤害 = 基础值 × 攻击力系数 + 附加值"）。次要信息应使用更小的字号（通常比主要信息小2–4pt）或更低对比度的颜色呈现，使玩家能够快速扫视主要数据而不被繁杂的说明文字干扰。《最终幻想14》的技能说明框在这一点上做了明确的层次划分，熟练玩家只需看顶部数字，新玩家可以阅读全文。

### 装备对比悬停的特殊机制

在RPG游戏中，悬停于背包物品时自动与当前装备栏对应位置进行属性对比，是一项高度专用的悬停子功能。这一功能通常通过在工具提示中以**绿色数字**显示属性提升、**红色数字**显示属性下降来实现。《暗黑破坏神3》率先将装备对比框分为左右双栏并排显示，使玩家无需记忆当前装备数据即可完成对比决策，这一设计后来成为行业标准参考。

## 实际应用

在MOBA游戏的商店界面中，悬停于装备图标时需要同时展示该装备的完整合成路线图和属性加成。《英雄联盟》的商店悬停框在右侧显示合成树状图，每个子组件同样可以触发次级悬停，形成嵌套悬停（Nested Hover）结构，最深可达三层嵌套而不产生界面混乱，关键在于每层弹出框与上一层保持明确的空间偏移关系。

在策略游戏《文明6》中，悬停于地图格子上时会在右下角弹出该格的产出信息（食物、生产力、金币数值），同时在格子本身显示数字叠加层，实现了"原位反馈"与"独立面板"的双重悬停信息系统。这种设计允许玩家在不移开视线的情况下快速评估多个格子的差异。

射击游戏中的武器改装界面（如《命运2》）则将悬停用于"选前预览"：鼠标悬停于待选配件时，角色模型实时展示安装该配件后的外观变化，同时右侧面板更新属性曲线图。这要求悬停响应速度必须在100毫秒以内完成预览加载，否则玩家会感知到明显卡顿。

## 常见误区

**误区一：把悬停状态当作唯一的可交互提示**
部分设计者认为只要悬停时有视觉变化，就足以告知玩家该元素可交互。然而悬停本身只对鼠标用户有效——当游戏支持手柄输入时，悬停状态无法被触发，必须同时为聚焦状态（Focus State）设计等效的视觉变化。《黑暗之魂》系列在PC与主机双平台发布时就因为早期版本悬停与聚焦状态混用而受到批评。

**误区二：工具提示延迟越短越好**
直觉上延迟越短响应越灵敏，但将延迟降至50毫秒以下会导致玩家在快速浏览界面时被持续弹出的提示框干扰，尤其是在密集图标排列的背包或技能树界面中。300毫秒的延迟是经过用户测试验证的最低有效阈值，它对应了人眼判断"我想看这个元素"与"我只是路过"之间的自然停顿时长。

**误区三：悬停动画时长越华丽越好**
为了追求视觉品质，部分游戏将悬停过渡动画设置为超过300毫秒的复杂序列。当玩家需要在短时间内连续比较多个选项时（如快速浏览多件装备），每次悬停都等待漫长动画完成会严重拖慢操作节奏。悬停进入动画应控制在80–150毫秒，可以设计华丽的首次触发效果，但循环或持续动画需要保持克制，避免抢夺玩家对信息内容本身的注意力。

## 知识关联

悬停状态设计建立在**视觉确认反馈**的基础概念之上：视觉确认反馈确立了"用户操作必须得到即时可见响应"的原则，而悬停状态将这一原则应用于"意图阶段"——玩家尚未操作，仅凭鼠标位置即触发反馈。从视觉确认反馈中学到的对比度规范（至少3:1的前景背景对比比）同样适用于悬停状态下的文字叠加层设计。

学习完悬停状态设计后，下一个需要掌握的是**按压响应设计**。按压状态（Active State）与悬停状态共同构成了一个交互元素完整生命周期中的相邻阶段：悬停时元素"浮起"，按压时元素"下沉"——两者的视觉语言应形成一致且相反的方向感（例如悬停放大105%、按压缩小至98%），才能给玩家带来连贯的物理隐喻体验。悬停状态中建立的工具提示定位算法和信息分级原则，在按压后弹出的确认对话框设计中同样直接适用。
