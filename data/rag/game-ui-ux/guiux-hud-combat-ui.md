# 战斗UI系统

## 概述

战斗UI系统是专门服务于玩家战斗行为的实时信息显示层，核心职责是将技能栏、冷却计时、连击计数、资源条等动态数据以最低认知负荷传递给玩家。与静态的菜单UI不同，战斗UI必须在玩家注意力高度集中于游戏世界中心区域时，在屏幕边缘或角落完成信息传递，不得干扰瞄准或走位判断。

战斗UI系统作为独立设计分支的形成，可追溯至1990年代街机格斗游戏——《街头霸王II》（Capcom，1991年）将生命条、时间和角色名固定在屏幕顶部的双条式布局，确立了"战斗信息集中在屏幕四边、中央留空"的基本法则，这一布局原则沿用至今。2004年《魔兽世界》引入可自定义的动作条和冷却遮罩（Cooldown Sweep），使技能状态可视化成为战斗UI的标配模块。2009年《龙腾世纪：起源》进一步将战术暂停（Tactical Pause）与战斗HUD深度整合，证明战斗UI不必局限于实时反应场景，可以跨越不同战斗节奏设计。

战斗UI设计之所以在HUD体系中独立成域，是因为战斗状态下信息密度极高、更新频率极快——部分状态效果（Buff/Debuff）每帧刷新，主流游戏引擎（如Unity、Unreal Engine 5）中战斗UI组件的刷新频率通常与游戏帧率同步（60fps或120fps），而非菜单UI的按需刷新。设计者必须同时解决"信息是否可读"与"信息是否分散注意"这两个方向相反的需求。错误的战斗UI设计会直接导致玩家因无法判断冷却状态或误读资源数量而在游戏中死亡，后果具有即时且可量化的可测性，这在UI设计的其他分支中极为罕见。

参考文献：《游戏用户体验设计》（Celia Hodent，2017，中文版由电子工业出版社出版）第七章对战斗信息分层提出了系统性的认知负荷模型，是本文核心原理的重要理论依据。

## 核心原理

### 技能栏布局与格式塔感知

技能栏通常采用水平线性排列，固定在屏幕底部中央，原因是人眼水平扫视速度比垂直扫视快约30%（根据眼动仪实验数据，Rayner, 1998）。标准动作RPG（如《暗黑破坏神3》）将技能槽限定为6个以内，对应Miller（1956）提出的工作记忆上限"7±2项"。每个技能槽的尺寸在1080p分辨率下通常为64×64像素：图标尺寸足够辨认细节，同时不超过屏幕高度的6%。技能槽之间应保持至少4像素间距，防止玩家在快速余光扫视时发生槽位边界混淆。4K分辨率下该尺寸等比放大至128×128像素，但间距仍维持4～8像素，以保持视觉节奏一致性。

格式塔"接近律"在技能栏分组中尤为重要：《最终幻想XIV》将6个技能槽分为"主技能组（4槽）"和"辅助技能组（2槽）"，两组之间保持16像素的额外间距，使玩家不需思考即可区分主动技能与角色能力。

### 冷却时间可视化机制

冷却显示存在两种主流方案：**遮罩扫描式（Radial Sweep）**和**数字倒计时叠加式**。遮罩扫描式用顺时针旋转的半透明暗色扇形（透明度通常为60%黑色叠加）覆盖技能图标，剩余冷却比例通过扇形面积直观呈现，玩家无需阅读数字即可判断相对进度。数字叠加式在图标中央显示整数秒数（≥1秒显示整数，<1秒显示一位小数如"0.3"），适合冷却时间超过30秒的长CD技能。

《英雄联盟》（Riot Games，2009年）同时叠加两种方式：扇形提供相对感知，数字提供精确判断。这种双通道设计使玩家在低决策压力下依赖扇形进行粗粒度判断，在高压团战时快速读取数字做精确决策，两套信息不互相干扰。

### 连击计数的动态反馈设计

连击计数（Combo Counter）的可视化目标不仅是显示数字，更是通过动画强化玩家的击打节奏感。标准动画参数如下：数字出现时执行弹性缩放动画，从1.4倍缩回1.0倍，持续约0.15秒（9帧@60fps）；颜色随连击段数变化，典型配置为白色（1-9连击）→黄色（10-19连击）→橙色（20-49连击）→红色（50+连击）；字体尺寸以每满10连击递增2pt为基准。《鬼泣5》（Capcom，2019年）将连击评级字母（D/C/B/A/S/SS/SSS）附加在数字左侧，评级颜色独立变化，使玩家同时获得"量"（数字）和"质"（评级）两个维度的即时反馈。连击计数位置固定于屏幕左侧中央偏下区域，重置计时器为2.5秒无有效操作后归零，归零动画为数字向上飘散消失（约0.3秒）。

## 关键公式与算法

### 冷却进度计算

在UI层实现冷却遮罩时，扇形覆盖角度（以度为单位）通过以下公式计算：

$$
\theta = 360° \times \frac{t_{remaining}}{t_{total}}
$$

其中 $t_{remaining}$ 为当前剩余冷却时间（秒），$t_{total}$ 为该技能的总冷却时间（秒）。当 $\theta = 360°$ 时技能处于完全冷却状态（刚释放），$\theta = 0°$ 时冷却完毕可再次使用。

### 连击评分加权示例（伪代码）

以下为一种典型的连击评级计算逻辑，参考《鬼泣5》的评级分段机制：

```python
def get_combo_grade(combo_count: int, time_since_last_hit: float) -> str:
    """
    combo_count: 当前连击数
    time_since_last_hit: 距上次命中的秒数（超过2.5秒重置）
    返回: 连击等级字符串
    """
    if time_since_last_hit > 2.5:
        return "RESET"  # 触发归零动画

    thresholds = [
        (100, "SSS"),  # 深红色，字体scale 1.8x
        (60,  "SS"),   # 红色，字体scale 1.5x
        (40,  "S"),    # 橙色，字体scale 1.3x
        (25,  "A"),    # 黄色，字体scale 1.15x
        (15,  "B"),    # 白色，字体scale 1.05x
        (8,   "C"),    # 灰白，字体scale 1.0x
        (0,   "D"),    # 灰色，字体scale 1.0x
    ]
    for threshold, grade in thresholds:
        if combo_count >= threshold:
            return grade
```

此算法中，评级阈值（8/15/25/40/60/100）并非线性分布，而是指数递增，确保玩家在低连击阶段能快速升级以获得正反馈，在高连击阶段则需要持续高水平操作才能维持最高评级。

### HP危险状态脉冲频率

低血量警告的脉冲闪烁频率设计遵循视觉紧迫感与不适感之间的平衡：低于1Hz令人感到缓慢、不紧张；高于3Hz会引发视觉疲劳甚至光敏性不适。主流设计选择1.5Hz（每秒1.5次完整明暗循环），对应周期 $T = \frac{1}{1.5} \approx 0.667$ 秒，其中亮相持续0.2秒，暗相持续0.467秒，非对称设计使"危险"信号比"安全"信号更短促、更醒目。

## 实际应用

### 案例一：《黑暗之魂3》的资源条分层设计

《黑暗之魂3》（FromSoftware，2016年）将HP条（红色）、FP魔力条（蓝色）、Stamina精力条（绿色）垂直叠放在屏幕左上角，三条宽度均为屏幕宽度的25%（1080p下约480像素）。颜色选择遵循交通信号灯语义：红色=生命危险，蓝色=资源管理，绿色=行动能力。精力条（Stamina）的消耗与回复动画尤为关键——消耗时瞬间减少（<1帧），回复时以固定速率0.25格/帧增长（60fps下约15秒回满），这种非对称动画设计使玩家能精确感知"我还剩多少精力可以翻滚"，而无需读取数字。

### 案例二：《英雄联盟》技能栏快捷键绑定与视觉对应

《英雄联盟》的技能栏将Q/W/E/R四个按键与技能槽一一对应，槽位内永远显示对应的键位字母标注（字号约10pt，位于槽位左上角），确保玩家将肌肉记忆与视觉位置双重绑定。当技能进入冷却时，键位字母同步变为灰色，强化"此键当前不可用"的认知一致性。这一设计使新手玩家的技能误触率（按下冷却中的技能键）在前10局游戏内下降约40%（Riot Games UX Research，内部数据引用于GDC 2018演讲）。

### 案例三：移动端战斗UI的拇指区适配

移动端战斗UI需针对"拇指触控热区"重新规划布局。以6英寸屏幕为基准，用户左右拇指自然覆盖的区域约为屏幕底部左右各40%的弧形区域，中央20%为盲区。《原神》（miHoYo，2020年）将攻击键和跳跃键置于右侧拇指热区，将摇杆置于左侧热区，而将技能栏（4个技能槽，每槽直径约80dp）排列于右侧拇指热区的弧形内边缘，所有可交互元素的最小点击区域不低于48×48dp（符合Google Material Design触控规范），避免在高压战斗时发生误触。

## 常见误区

### 误区一：冷却数字精度过高导致认知过载

部分开发者将冷却倒计时精确到小数点后两位（如"12.47秒"），实际上玩家在战斗中对0.01秒级别的精度毫无决策价值。正确做法：≥10秒的冷却仅显示整数秒，1～9秒显示整数，<1秒显示一位小数（如"0.8"）。额外的数字位数只会增加阅读时间，《英雄联盟》的内部测试显示，去掉小数点后两位后，玩家的冷却读取速度平均提升22ms，在每秒决策密度极高的团战中积累效果显著。

### 误区二：连击计数位置占据中央视野

将连击计数放置于屏幕中央（如屏幕中间偏上）会直接遮挡玩家的目标观察区域。即便只是0.5秒的视线被吸引，在动作游戏中也足以导致一次被击中。连击计数应始终位于屏幕的四个角落之一，《鬼泣》系列25年来始终坚持将计数置于屏幕左上角，而非跟随敌人位置浮动显示。

### 误区三：资源条颜色与技能图标颜色系撞色

若技能图标普遍采用蓝色调（如冰系技能），而魔力条（MP）同样为蓝色，玩家在快速扫视时极易将MP读取与技能状态混淆。解决方案是为资源条添加独特的色相偏移：即便同为"蓝色系"，MP条可选择饱和度更高的纯蓝（Hue 210°），而技能图标
