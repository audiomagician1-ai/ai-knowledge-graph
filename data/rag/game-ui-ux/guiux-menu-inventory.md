# 背包/物品界面

## 概述

背包/物品界面（Inventory UI）是玩家在游戏中管理所有可携带物品的专用界面系统，通常以固定尺寸的网格布局呈现物品图标，并提供对物品进行筛选、排序、装备和丢弃等操作的交互入口。从1981年《Ultima III》引入图形化物品栏概念以来，背包界面经历了从线性列表到二维网格，再到多标签分类面板的演变轨迹。

早期RPG如《Diablo》（1996年）确立了至今仍被广泛沿用的"格子背包"范式：每个物品占据一个或多个格子，背包总容量由格子总数限定，玩家需要手动整理物品位置，也称"俄罗斯方块式背包"。这一设计以物品的物理尺寸模拟现实重量限制的直觉，使物品管理本身成为游戏玩法的一部分。

背包界面直接影响玩家对物品经济的感知：过于复杂的分类系统会让玩家迷失在菜单层级中，而过于简单的线性列表则无法快速定位特定物品。《原神》和《暗黑破坏神：不朽》等现代手游在触屏环境下对背包UI做出了大量针对性改造，说明该界面对不同输入设备的适配是核心设计挑战之一。

## 核心原理

### 物品网格与容量模型

物品网格的基本单位是"格子（Slot）"，每个格子存储一个物品引用。网格的视觉尺寸通常遵循**N×M**的矩阵结构，其中N为列数、M为行数，常见配置有5×8、6×10等。每个格子的像素尺寸需与物品图标的标准分辨率匹配，主机游戏普遍采用64×64像素或128×128像素的图标规格，手游因屏幕密度差异多用72×72至96×96像素。

部分游戏（如《Path of Exile》）允许物品占据多个格子，需要在格子数据结构中记录"占用坐标集合"，以避免物品重叠。判断放置合法性的算法需遍历目标物品所有格子坐标，确认每个坐标均为空位，时间复杂度为O(k)，k为物品占用格子数。

### 分类筛选与标签系统

分类筛选通过预定义的物品类型标签（Tag）实现，常见一级分类包括：武器（Weapon）、防具（Armor）、消耗品（Consumable）、材料（Material）、任务道具（Quest Item）。筛选时UI仅渲染匹配标签的物品图标，未匹配的格子被置灰或隐藏，视觉上缩短了玩家的扫描路径。

二级筛选可在一级分类下进一步按子类型过滤，如武器类下区分"单手剑""弓""法杖"等。《原神》还引入了基于角色绑定的快速筛选——"展示可装备当前角色的武器"，该功能通过对比武器的`weaponType`字段与角色的`allowedWeaponTypes`数组完成过滤。

### 排序逻辑与快速整理

背包排序支持多种键值：品质（稀有度数值降序）、获取时间（时间戳降序）、名称（字母/拼音升序）、堆叠数量（降序）。现代背包界面通常提供"一键整理"功能，其内部执行以品质为主键、类型为次键的多级排序，并将同类物品相邻放置，减少玩家手动拖拽次数。《怪物猎人：世界》的整理功能还会按猎人笔记中的使用频率自动将常用物品置顶。

### 装备对比与属性差值显示

装备对比（Item Comparison）是背包界面中降低决策成本的关键功能。当玩家将鼠标悬停于某件装备时，界面同步显示当前已装备的同部位装备属性，并以**绿色"▲+数值"**或**红色"▼-数值"**标注每条属性的差值，使玩家无需记忆当前装备数值即可判断是否应换装。公式如下：

> **差值 = 新物品属性值 − 当前装备属性值**
> 差值 > 0 显示绿色正差，差值 < 0 显示红色负差，差值 = 0 不显示或显示"="。

部分游戏（如《暗黑破坏神4》）支持三件装备同时对比，适用于玩家同时持有多件备选装备的场景。

### 快速操作与右键菜单

快速操作（Quick Action）通过右键弹出上下文菜单或长按手势触发，减少玩家在背包与其他界面之间的跳转次数。典型选项包括：使用/装备、丢弃、出售（需在商店NPC附近）、标记为收藏（防误删）、发送给仓库。快速出售功能通常要求玩家处于商店对话状态，否则该选项置灰，以防止误操作导致物品丢失。

## 实际应用

在《博德之门3》中，背包界面采用标签页+网格的混合方案：左侧为队伍成员切换栏，右侧主区域以6列网格陈列物品，底部提供按类型筛选的快捷标签，并支持在队员之间通过拖拽直接转移物品。其装备栏以人形剪影标注各部位插槽，点击部位即可高亮背包中该部位可用物品，将装备选择路径从"背包→拖拽→装备"缩短至"点击部位→点击目标物品"两步完成。

在《原神》手游端，背包界面通过左右滑动切换武器、圣遗物、材料等大类，每类下方有横向滚动的子类筛选栏。圣遗物子界面还内置"属性评分"标注，帮助玩家快速判断词条优劣，将本需玩家手动计算的评估步骤嵌入界面本身，这是背包UI与游戏核心数值系统深度整合的典型案例。

## 常见误区

**误区一：格子容量限制越大，玩家体验越好。** 实际上，背包格子数量过多（如超过200格）会导致物品扫描成本急剧上升，玩家反而需要更长时间定位目标物品。《暗黑破坏神：不朽》测试期间曾收到大量反馈，指出背包格子过多且缺乏快速定位功能，导致整理时间超过实际游玩时间，最终在正式版中加强了筛选与高亮功能，而非单纯扩大格子数量。

**误区二：一键整理功能应当完全自动决定物品位置。** 许多设计者忽视了玩家对自定义物品布局的记忆习惯——玩家常将特定物品摆放在固定格子以便快速取用。全自动整理会破坏这种记忆定式。《最终幻想XIV》的整理功能因此提供了"锁定格子（Lock Slot）"选项，允许玩家将常用物品固定在指定位置，整理时跳过已锁定格子。

**误区三：装备对比只需显示总体战斗力数值差即可。** 单一"战力"差值会掩盖属性权衡细节——例如新装备使攻击力+50但使防御力-30，简单加总后战力可能显示正增长，但对偏向坦克的玩家而言实为劣化。逐条属性的差值展示（如《暗黑破坏神4》的详细对比面板）才能支持玩家的真实决策需求，尤其在Build多样性较高的游戏中。

## 知识关联

背包界面建立在**菜单导航架构**所定义的层级跳转逻辑之上——玩家从主HUD打开背包属于一级跳转，背包内的子类标签切换属于同层导航，理解这一层级关系有助于在设计中合理分配返回/退出的触发位置，避免玩家迷失在多层嵌套的背包子菜单中。

背包界面中物品图标的视觉呈现直接引出**物品稀有度图标**的设计规范——稀有度颜色框（如灰/绿/蓝/紫/橙分别对应普通至传说品质）正是在背包网格中首次被玩家大量感知的，其色彩对比度和边框样式的设计决策需与背包网格的背景色统一考量。背包界面中展示的角色身上的已装备物品，则与**角色/属性界面**中的装备栏形成双向联动：在背包中换装后，角色界面的属性数值应即时更新，两个界面共享同一份装备状态数据源。
