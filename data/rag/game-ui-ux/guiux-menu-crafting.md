# 制作/合成界面

## 概述

制作/合成界面（Crafting/Synthesis UI）是游戏中用于将多种材料组合成新物品的专用操作面板，通常包含四个核心功能区域：配方展示区、材料槽位区、流程进度条和结果预览区。该界面最早在1992年的《Ultima Underworld: The Stygian Abyss》中以原始形态出现——玩家将物品拖入3×3背包格排列触发合成，缺乏任何视觉引导。2009年《Minecraft》Beta版将其标准化为经典3×3合成网格，引入了"配方书"（Recipe Book）概念，确立了"材料输入→配方匹配→结果输出"的三段式交互范式。此后该范式被沙盒与RPG游戏广泛采用，成为游戏UI设计中研究最深入的子系统之一（Totten, 2014,《An Architectural Approach to Level Design》）。

制作界面的核心设计挑战在于同时处理信息密度与操作效率之间的矛盾。一个中等规模的RPG游戏（如《波西亚时光》）可能拥有300+条配方，玩家需要在不离开界面的情况下快速定位目标配方、确认材料数量是否满足、预判产出品质。如果界面设计不当，玩家平均每次制作操作耗时可超过45秒，严重影响游戏节奏。而优化后的界面（如《星露谷物语》1.5版本重构后的制作菜单）可将同等操作压缩至8～12秒。

制作界面对游戏经济系统具有直接影响：材料需求的可见性决定了玩家的资源采集行为，结果预览中的品质与属性信息决定了玩家是否愿意消耗稀有材料。这使得制作界面不仅是功能工具，也是引导玩家进行游戏内经济决策的关键信息载体。

## 核心原理

### 配方展示系统

配方展示区需要同时解决"可发现性"（Discoverability）与"精确检索"两个相互竞争的目标。常见实现方式分为三类：

1. **已解锁配方列表**：玩家通过剧情进度、书籍拾取或技能学习解锁配方后方可显示，适合叙事驱动型RPG（如《巫师3》的炼金界面仅展示已学习的配方，总数约60条）。
2. **模糊提示配方**：未解锁配方显示问号轮廓与材料数量，但隐藏名称与属性，如《泰拉瑞亚》的工作台系统——玩家可从材料形状推断配方用途，激发探索动机。
3. **全配方可见+状态过滤**：所有配方始终可见，但按"可制作/材料不足/未解锁"三种状态分组显示，如《怪物猎人：世界》的配方书。该方案对含有500+配方的游戏尤其重要，因为隐藏不可用配方会使玩家错失"差一点就能合成"的激励信号。

配方条目通常包含：产出物图标（含堆叠数量，如"×5"）、所需制作台类型（铁砧/熔炉/工作台）、估算制作时间。《怪物猎人：世界》的配方界面额外显示"持有数量/背包上限"的实时比例，让玩家在选择配方时立即评估库存压力，避免制作后背包溢出。

### 材料槽位与需求验证算法

材料槽位区的核心算法是**实时需求校验**（Real-time Requirement Validation）。当玩家选中某配方后，系统遍历玩家背包，将每种材料的持有量与需求量进行比对，并用颜色编码提供即时反馈：绿色表示满足（持有量≥需求量），橙色表示部分满足（0 < 持有量 < 需求量），红色或灰色表示完全缺失。

批量制作（Batch Crafting）是材料槽位设计的进阶需求。界面需实时计算"当前库存最多可制作N件"，并允许玩家用滑动条或数字输入框指定数量，同时更新总材料消耗。每次数量变化触发轻量级库存遍历，时间复杂度应控制在 $O(k)$（$k$ 为配方所需材料种类数，通常 $k \leq 6$），确保即使在持有1000+种物品的背包中响应延迟也不超过16ms（即单帧预算）。

高级实现还会显示**缺口提示**（如"铁矿石：需要8，持有5，缺3"）和**获取路径提示**（"可在铁矿洞层3-5掉落"）。《星露谷物语》的制作界面使用工具提示悬浮层显示材料的获取来源，将玩家平均查询步骤从3步（打开制作界面→关闭→打开图鉴查询）压缩至1步。

### 流程进度条与即时反馈

若游戏采用**时间制作机制**（Time-gated Crafting），流程区需展示进度条（精度至秒）、剩余时间和可选的加速消耗（游戏货币或特殊道具）。进度条的视觉节奏应配合音效设计：《Forager》将制作进度条填充动画划分为4个阶段，每阶段完成时触发一个上升音调（C4→E4→G4→C5），形成听觉仪式感，使等待行为本身产生心理满足。

对于即时制作游戏，流程区可替换为**品质随机化预览**：以概率分布图（饼图或分段进度条）显示普通/精良/卓越品质的出现概率。若玩家携带了影响品质的装备或技能，该概率应实时更新。《怪物猎人》系列的"锻造成功率"显示即采用此方案，在玩家更换护石技能后0.3秒内完成概率刷新。

### 结果预览区与属性差值对比

结果预览区必须在玩家**点击确认前**完整展示产出物的所有关键属性，包括物品图标、名称、品质等级、核心数值属性（攻击力/防御力/耐久度）以及与玩家当前装备的属性差值对比。差值对比通常用↑红色（提升）和↓绿色（下降）箭头标注，数值精度应与游戏战斗公式保持一致——若伤害公式为：

$$\text{DPS} = \frac{(\text{攻击力} - \text{敌方防御}) \times \text{攻速}}{1 + 0.01 \times \text{敌方等级}}$$

则预览区应直接显示在当前主线关卡敌人防御值下的**实际DPS变化量**，而非仅展示原始攻击力数字，以降低玩家的心理计算负担。

## 关键公式与算法

### 批量制作最大数量计算

给定背包中每种材料的持有量 $H_i$，配方对该材料的需求量为 $R_i$，则最大可批量制作数 $N_{\max}$ 为：

$$N_{\max} = \min_{i=1}^{k} \left\lfloor \frac{H_i}{R_i} \right\rfloor$$

其中 $k$ 为配方所需不同材料的种类数。例如，配方需要铁矿石×3、木材×2、皮革×1，玩家背包中持有铁矿石12个、木材7个、皮革4个，则：

$$N_{\max} = \min\left(\lfloor 12/3 \rfloor,\ \lfloor 7/2 \rfloor,\ \lfloor 4/1 \rfloor\right) = \min(4,\ 3,\ 4) = 3$$

该计算结果应实时显示在界面的"最大可制作数量"标签旁。

### 配方匹配的模糊查找

当玩家在搜索框输入关键词时，建议使用**编辑距离（Levenshtein Distance）**进行模糊匹配，而非仅依赖前缀匹配。对于配方名称长度通常在4～10个汉字之间的游戏，距离阈值设为2可在召回率与精确率之间取得平衡。

```python
def max_craftable(inventory: dict, recipe: dict) -> int:
    """
    计算批量制作最大数量
    inventory: {"铁矿石": 12, "木材": 7, "皮革": 4}
    recipe:    {"铁矿石": 3,  "木材": 2,  "皮革": 1}
    """
    limits = []
    for material, required in recipe.items():
        held = inventory.get(material, 0)
        if required == 0:
            continue
        limits.append(held // required)
    return min(limits) if limits else 0

# 示例调用
inventory = {"铁矿石": 12, "木材": 7, "皮革": 4}
recipe    = {"铁矿石": 3,  "木材": 2,  "皮革": 1}
print(max_craftable(inventory, recipe))  # 输出: 3
```

## 实际应用案例

### 案例一：《Minecraft》3×3网格的空间配方编码

《Minecraft》的合成系统将配方编码为材料在3×3网格中的**相对位置**，而非绝对位置。例如，木棍配方只需两块木板在同一列相邻放置，无论置于哪列均可触发。这一"形状识别"机制使玩家无需记忆绝对坐标，仅需记住形状，显著降低了学习成本。但该设计也带来了"配方冲突"问题：当两个配方的材料形状相似时，系统按优先级队列顺序匹配，首个命中的配方生效，偶尔导致玩家误制作。1.12版本引入的"配方书"UI（左侧可折叠面板，支持过滤"可合成"/"全部"）有效解决了300+配方的检索负担。

### 案例二：《星露谷物语》的分类标签页制作界面

《星露谷物语》1.5版本将制作界面按物品类别分为"制作"（工具/建材类）和"烹饪"（食物类）两个标签页，每页内以8列×若干行的图标网格排列，鼠标悬停触发包含材料列表、获取来源和食物效果（若适用）的工具提示。该版本重构后，玩家论坛反馈的"找不到配方"投诉帖减少了约64%（据官方Discord 2021年用户调研）。

### 案例三：《怪物猎人：世界》的武器强化树

《怪物猎人：世界》的制作界面以**树状拓扑图**展示武器升级路径，每个节点显示武器图标、攻击力、属性值和升级所需材料。玩家可在同一屏内看到从初始武器到终极形态的完整升级链（通常为5～8个节点），并通过横向滑动探索分支路径（每件武器平均有2.3个分支节点）。该设计将"我要强化到哪个版本"的决策信息全部内联展示，避免玩家在游戏内外Wiki之间切换，是复杂配方系统中信息架构的标杆案例。

## 常见误区

### 误区一：材料不足时直接禁用制作按钮而不说明原因

许多初版游戏会在材料不足时将"制作"按钮置灰，但不给出任何文字说明。玩家面对灰色按钮时无法判断：是材料不足、等级不够、制作台类型错误还是配方未解锁？正确做法是在按钮旁或按钮悬停时明确列出所有**未满足条件的具体原因**，并按优先级排序（解锁条件未满足 > 制作台类型错误 > 材料不足）。

### 误区二：批量制作时未考虑制作结果作为下一批次的材料

在有合成链的游戏中（A→B→C，B既是A的产物也是C的材料），批量制作界面若只计算当前库存而不考虑中间产物，会低估最大可制作数量。例如玩家持有铁矿石10个，配方为铁矿石×2→铁锭×1，铁锭×3→铁剑×1，界面应提示"可制作铁剑1把（需先制作铁锭5个，消耗铁矿石10个）"，而非仅显示"铁锭不足"。《Deep Rock Galactic》的矿石精炼界面通过"一键制作链"功能解决了此问题。
