# 认知辅助设计

## 概述

认知辅助设计（Cognitive Assistance Design）是游戏UI/UX无障碍设计的专项分支，专门为存在认知障碍、学习困难或注意力缺陷的玩家提供交互支持。其核心目标是通过**简化UI模式、嵌入任务提醒和优化导航辅助**，将玩家在理解游戏规则、追踪任务进度和操作界面时承受的认知压力降至可控范围。

该设计领域服务的用户群体涵盖：阅读障碍（Dyslexia，全球发生率约10%）、注意力缺陷多动障碍（ADHD，全球成人发生率约2.5%）、轻度认知障碍（MCI）以及自闭症谱系障碍（ASD）。根据世界卫生组织2021年数据，全球约10亿人存在某种形式的残障，其中认知与神经发育差异群体在游戏玩家中的占比不可忽视——美国娱乐软件协会（ESA）2022年报告指出，约26%的玩家表示曾因界面设计问题而提前放弃一款游戏。

2018年，Xbox正式发布《游戏无障碍功能指南》（Xbox Accessibility Guidelines，XAG），其中第12至16条专项规定了认知辅助的最低设计标准，包括：任务进度持久化显示、多步骤操作的子任务分解、可调节的文字复杂度和字体替换功能。这一规范标志着认知辅助设计从开发者的自发实践演进为可量化、可审核的行业标准。

**思考问题**：当一款游戏同时面向ADHD玩家和ASD玩家进行认知辅助设计时，这两类用户群体的需求是否存在根本性矛盾？应如何在单一界面框架内协调？

## 核心原理

### 简化UI模式与工作记忆约束

认知辅助设计的视觉层级原则源自心理学家乔治·米勒（George A. Miller）1956年发表于《心理学评论》的经典论文"The Magical Number Seven, Plus or Minus Two"（Miller, 1956）。该研究确认人类工作记忆的容量上限约为 $7 \pm 2$ 个信息块（chunk）。认知辅助设计将此上限收紧至5个以内，原因是认知障碍玩家的工作记忆容量通常低于神经典型（neurotypical）均值——ADHD玩家的工作记忆容量实测中位数约为4至5个信息块。

据此，HUD（抬头显示）元素数量应硬性限制为≤5个，超出部分折叠至二级菜单。图标设计必须附带**持久性文字标签**（非悬停触发），因为认知障碍玩家的图形-语义映射能力弱于均值，无法可靠地将抽象图标与功能含义对应。《原神》PC端的HUD设计因同屏显示8个以上任务追踪图标而被多份无障碍评测报告列为认知负荷过高的典型案例。

字体选择是简化UI模式中最具量化依据的技术决策。专为阅读障碍设计的**OpenDyslexic字体**通过底部加重的字形设计，减少字母翻转混淆（b/d、p/q对），实验室测试显示可将阅读障碍受试者的单词辨识错误率降低约18%至25%（Wery & Diliberto, 2017）。《荒野大镖客：救赎2》（2018）、《对马岛之魂》（2020）和《赛博朋克2077》（1.6版本补丁后）均在无障碍设置中提供了字体替换选项。

### 任务提醒的触发逻辑设计

任务提醒的核心挑战是精确区分**干扰性提示**与**支持性提醒**。以固定时间间隔触发的弹窗提示属于干扰性设计，会打断玩家的当前心流状态；而基于玩家行为惰性触发的提醒属于支持性设计，在玩家已经迷失方向时介入。

行业标准的触发条件定义如下：当玩家在同一区域连续停留超过**90秒**且未产生任何目标性操作（移动距离 < 5米、未与任何可交互对象接触、未开启菜单）时，系统自动在屏幕边缘显示当前主线任务摘要，且该提示在玩家恢复操作后的3秒内自动淡出，不阻断视野。XAG第15.3条对此触发阈值有明确规范，并要求提示语言的阅读年龄（Reading Age）不超过8岁水平，即Flesch-Kincaid可读性分级≤3年级。

对ADHD玩家而言，**子任务的即时正向反馈**是维持任务坚持时长的关键机制。研究表明，每完成一个子任务后的即时双通道反馈（视觉高亮 + 音效确认）可使ADHD玩家在同一任务链上的平均坚持时长延长37至42分钟（Swing et al., 2010）。《巫师3：狂猎》的任务日志采用了"折叠式子任务链"设计，每个子任务完成后以划线+金色勾选框标注，是该原则的成熟实现案例。

### 导航辅助的多模态实现

导航辅助设计区别于通用地图设计，它要求将抽象的空间坐标翻译为**即时方向指引**，并支持多种感知通道。具体实现层次如下：

- **第一层：minimap标记**——目标标记尺寸应可调，最小不低于24×24像素（对应4K分辨率），且颜色不能仅依赖红/绿区分（色盲适配）；
- **第二层：屏幕边缘罗盘指向**——当目标不在当前视野内时，在屏幕对应方向边缘显示方向箭头；
- **第三层：环境内嵌导航线**——《量子破碎》（2016）率先在地面以发光光迹实现路径导引，《最后生还者 第二章》（2020）将其扩展为可设置亮度和颜色的"导航线"选项；
- **第四层：语音方向播报**——作为视觉导航的备份通道，适用于屏幕认知负荷已接近上限时。

对存在空间方向感障碍的ASD玩家，允许将导航标记从minimap**迁移至屏幕中央固定区域**（通常为屏幕底部中心位置），可消除玩家在游戏画面与minimap之间频繁切换视线所产生的视空间整合负担，这一设计已在《最后生还者 第二章》的"导航辅助"选项中实现。

## 关键指标与可读性公式

衡量游戏文本是否满足认知辅助标准，常用**Flesch-Kincaid年级公式**（Kincaid et al., 1975）对任务说明、NPC对话和教程文本进行量化评估：

$$
\text{FK Grade} = 0.39 \times \left(\frac{\text{总词数}}{\text{总句数}}\right) + 11.8 \times \left(\frac{\text{总音节数}}{\text{总词数}}\right) - 15.59
$$

认知辅助设计要求游戏内**所有关键任务说明**的FK年级分数≤4.0（即四年级阅读水平），**NPC主要对话**≤6.0。以下是一个计算任务文本FK年级的Python辅助工具片段：

```python
import textstat

def check_cognitive_accessibility(text: str) -> dict:
    """
    检测游戏文本是否符合认知辅助设计的可读性标准。
    FK Grade ≤ 4.0 为任务说明合格线，≤ 6.0 为对话合格线。
    """
    fk_grade = textstat.flesch_kincaid_grade(text)
    reading_ease = textstat.flesch_reading_ease(text)
    word_count = textstat.lexicon_count(text)
    
    return {
        "FK_grade": round(fk_grade, 2),
        "reading_ease": round(reading_ease, 2),  # 目标：≥ 70（易读）
        "word_count": word_count,
        "task_text_pass": fk_grade <= 4.0,
        "dialogue_pass": fk_grade <= 6.0
    }

# 示例：检测一段任务提示文本
sample_text = "前往山顶神庙，找到祭坛，使用圣火点燃三根蜡烛。"
result = check_cognitive_accessibility(sample_text)
print(result)
# 输出示例：{'FK_grade': 2.8, 'reading_ease': 82.1, 'word_count': 14, ...}
```

此外，UI元素的**认知负荷密度**（Cognitive Load Density，CLD）可定义为单屏同时活跃的信息块数量与工作记忆容量上限之比：

$$
\text{CLD} = \frac{N_{\text{active\_elements}}}{C_{\text{WM}}}
$$

其中 $N_{\text{active\_elements}}$ 为当前屏幕上需要玩家主动处理的信息单元数量，$C_{\text{WM}}$ 取值4（认知障碍玩家工作记忆下限参考值）。当 $\text{CLD} > 1.0$ 时，该界面对认知障碍玩家存在显著过载风险，需要进行UI精简。

## 实际应用

### 《最后生还者 第二章》的认知辅助全套实现

《最后生还者 第二章》（Naughty Dog，2020）被业界公认为游戏认知辅助设计的标杆案例，其无障碍选项多达60余项，其中认知辅助相关选项包括：

- **导航辅助**：可将路径导引从地图迁移至屏幕边缘实时箭头，方向精度至45°分辨率；
- **任务追踪持久化**：当前目标文字始终显示在屏幕左上角，不随地图开合消失；
- **高对比度字幕**：字幕背景不透明度可调（0%至100%），字号可在12pt至32pt间6档调节；
- **导航线颜色自定义**：支持8种预设颜色方案，覆盖全色盲（Achromatopsia）和部分色盲（红绿色盲、蓝黄色盲）；
- **简化输入模式**：将需要记忆多步骤操作序列的连招简化为单键长按，降低程序性记忆负担。

该游戏在2020年游戏开发者大会（GDC）无障碍专题中获得了来自残障玩家权益组织SpecialEffect和AbleGamers的联合最高评级。

### 教育游戏领域的认知辅助差异化设计

**例如**：针对7至12岁ADHD儿童设计的教育游戏《Math Ninja》在A/B测试中对比了两种任务提醒方案：方案A采用每5分钟固定弹窗提醒，方案B采用基于行为惰性的90秒触发提醒。结果显示方案B的任务完成率（78%）比方案A（54%）高出24个百分点，且玩家中途退出率从41%降至19%（Atkinson & Shiffrin模型指导下的干预研究，Pearce et al., 2019）。

## 常见误区

### 误区一：为所有玩家默认开启认知辅助等于降低游戏难度

认知辅助设计与游戏难度调节是**完全正交**的两个维度。任务提醒帮助玩家知道"去哪里"，但不影响到达目标后的战斗难度或谜题复杂度。《黑暗之魂3》即使不添加任何认知辅助功能，其战斗难度也不受影响；反之，为其增加持久化任务标记也不会削弱boss战的挑战性。将两者混淆的设计决策会导致认知辅助功能被错误地绑定在"简单模式"中，剥夺认知障碍玩家在正常难度下游玩的权利。

### 误区二：用图标密集的"信息图"替代文字说明可降低认知负荷

大量图标的密集排布并不能降低认知负荷，对于**图形-语义映射能力弱的认知障碍玩家**反而会增加负荷。仅在图标附带持久性文字标签、且图标数量≤5个的前提下，图标才有助于认知辅助。《命运2》早期版本的武器Perk图标界面（单屏显示15至20个无标签图标）是这
