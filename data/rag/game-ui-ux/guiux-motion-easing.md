# 缓动曲线

## 概述

缓动曲线（Easing Curve）是描述动画属性值随时间变化速率的数学函数，在游戏UI动效设计中决定了一个元素从起始状态到终止状态的运动"感觉"。最基础的缓动曲线是**线性函数**（Linear），即属性值随时间等速变化，但这种机械式的匀速运动在视觉上显得僵硬，因为自然界中几乎没有物体做完美的匀速运动。

缓动曲线的数学基础可追溯至1987年Robert Penner在Flash动画时代总结的缓动函数公式集（Robert Penner's Easing Functions），这套公式后来成为Web和游戏动效的通用标准。在CSS规范中，缓动曲线以**三次贝塞尔曲线**`cubic-bezier(x1, y1, x2, y2)`的形式表达，其中四个参数控制两个控制点的坐标，理论上可以描述无数种速度变化模式。

在游戏UI中，缓动曲线的选择直接影响玩家对界面"重量感"和"响应感"的主观判断。一个伤害数字弹出时使用ease-out与使用线性缓动，玩家对"打击力度"的感受会产生肉眼可辨的差异，这正是缓动曲线在游戏UI/UX中不可忽视的原因。

## 核心原理

### 三类基础缓动函数

最常用的三类缓动函数是**ease-in、ease-out、ease-in-out**，它们与物理运动有直接的类比关系：

- **Ease-in（缓入）**：动画开始时慢、结束时快，类比物体从静止开始加速。数学上对应函数斜率从小到大，如二次方ease-in公式为 `f(t) = t²`，其中 `t ∈ [0, 1]` 表示归一化时间。适用于元素**退场**（如界面关闭、元素消失），因为"快速离开"符合认知直觉。

- **Ease-out（缓出）**：动画开始时快、结束时慢，类比物体减速停止。公式为 `f(t) = 1 - (1-t)²`。适用于元素**入场**（如弹窗出现、奖励图标飞入），快速出现再平滑停止的感觉更自然有力。

- **Ease-in-out（缓入缓出）**：两端慢、中间快，完整模拟加速再减速的运动。CSS默认的`ease`关键字等价于`cubic-bezier(0.25, 0.1, 0.25, 1.0)`，常用于**场景切换**等需要平滑感的长距离位移动画。

### 弹性缓动与回弹缓动

**弹性缓动（Elastic Easing）**允许属性值超过目标值后再弹回，模拟弹簧的过冲效果。Penner公式中elastic ease-out的核心项包含 `sin` 函数：`f(t) = 2^(-10t) * sin((t - s/4) * 2π / s) + 1`，其中参数 `s` 控制弹性周期。游戏中商店解锁新道具、成就徽章出现时常用此效果强化"奖励感"。

**回弹缓动（Bounce Easing）**则模拟物体落地后多次反弹，最终停止。与Elastic不同，Bounce的值域始终在 [0, 1] 范围内（不超过终点），使用分段函数实现，通常有3-4次强度递减的回弹。游戏中血条归零或物品落入背包槽位时，Bounce能强化"实体碰撞"的物理感。

### 曲线的坐标系解读

缓动曲线在编辑器中以**横轴为时间（0→1）、纵轴为属性值进度（0→1）**的二维图表呈现。曲线斜率越陡，代表该时刻速度越快；斜率为零表示瞬间静止；斜率超过1（曲线超出纵轴范围）则代表属性值超过了目标值（即过冲）。Unity的Animation Curve编辑器和Unreal的Curve编辑器均采用此坐标约定，直接拖拽贝塞尔控制柄即可自定义缓动形状。

## 实际应用

**游戏内弹窗出现**：主界面弹出任务完成奖励窗口时，推荐对Scale属性使用ease-out（从0.8缩放至1.0），持续约**200-300ms**。Scale从小变大且末端减速，给人"弹出停稳"的实体感，而非线性缩放时的生硬机械感。

**伤害/得分数字飘字**：数字向上位移时使用ease-out，同时在约**600ms**处淡出消失。入场快（ease-out位移）确保玩家能立即看到数值，渐隐时可叠加ease-in的透明度变化让消失过程自然。

**按钮点击反馈**：按钮按下时Scale从1.0→0.92使用ease-in（约80ms），抬起时Scale从0.92→1.0使用弹性缓动elastic ease-out（约300ms，轻微过冲至1.05再回到1.0），这种"按下-弹起"的两段式缓动能显著提升按钮的物理存在感。

**技能CD转盘动画**：冷却完毕瞬间，建议使用**bounce ease-out**让图标抖动回弹，持续约**400ms**，3次递减回弹的总位移约为图标尺寸的8%，既能吸引注意力又不会干扰操作。

## 常见误区

**误区一：全部使用ease-in-out是"最安全"的选择**
ease-in-out两端慢中间快，用于退场动画时开头的"慢启动"会让界面显得迟钝，玩家会感觉UI"反应慢"。正确做法是退场动画优先选ease-in（快速开始离开），入场动画优先选ease-out（快速到位）。

**误区二：弹性缓动越夸张越有活力**
弹性缓动的过冲量超过目标值的**15%**后，在游戏UI中通常会让界面显得廉价或滑稽，而非充满活力。奖励类UI可接受轻微过冲（约5-10%），但导航类元素（如标签页切换）使用弹性缓动反而会破坏界面的稳定感，应坚持使用ease-out。

**误区三：缓动曲线只影响位移动画**
缓动曲线同样作用于**透明度、颜色、Scale、旋转角度**等任意可插值属性。一个常见错误是给淡出动画设置ease-out（开头快淡、末尾慢淡），导致元素"消失一半后还赖着不走"的视觉卡顿；透明度0→1的入场淡入应使用ease-out，1→0的淡出应使用ease-in。

## 知识关联

**前置概念衔接**：UI动效设计概述中介绍的"动画的时间与空间"属性（Duration、Delay、Property），是缓动曲线发挥作用的载体——缓动曲线描述的正是在给定Duration内属性值的变化节奏。动画12原则中的**慢入慢出（Slow In and Slow Out）**原则，本质上就是ease-in-out缓动的理论来源，理解该原则能帮助判断何时应打破ease-in-out的"舒适区"去使用纯ease-in或纯ease-out。

**后续概念延伸**：掌握缓动曲线后，进入微交互设计时将面临更复杂的多属性联动动画——例如一个点赞按钮同时需要对Scale（bounce ease-out）、颜色（linear）、粒子（ease-out）三条属性分别指定不同的缓动曲线，而非对整个动画使用单一缓动。此时对每条属性的缓动选择逻辑，正是基于本文所述的入场/退场/物理感原则的综合运用。
