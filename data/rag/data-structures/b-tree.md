# B树与B+树

## 核心概念

B树（B-Tree）是一种自平衡的多路搜索树，专为磁盘存储和数据库索引设计。与二叉搜索树不同，B树的每个节点可以有多个键和子节点，这减少了树的高度，从而减少磁盘I/O次数。

## B树的性质

对于阶为m的B树：
1. 每个节点最多有m个子节点
2. 每个内部节点至少有⌈m/2⌉个子节点
3. 根节点至少有2个子节点（除非是叶子）
4. 所有叶子在同一层
5. 有k个子节点的节点包含k-1个键

## B树示例（m=3，即2-3树）

```
        [10, 20]
       /    |    \
   [5, 7] [15] [25, 30]
```

## 基本操作

### 查找 O(log n)
从根节点开始，在每个节点内二分查找，然后递归到对应子节点。

### 插入
1. 像查找一样找到叶节点
2. 将键插入叶节点
3. 如果节点溢出（键数超过m-1）→ 分裂：中间键上提到父节点
4. 分裂可能向上传播

### 删除
1. 找到键所在节点
2. 如果在叶节点 → 直接删除
3. 如果在内部节点 → 用前驱/后继替换，转为叶节点删除
4. 如果节点下溢 → 从兄弟借或合并

## B+树的区别

| 特征 | B树 | B+树 |
|------|-----|------|
| 数据位置 | 所有节点 | 仅叶节点 |
| 叶子链接 | 无 | 叶子间有链表 |
| 范围查询 | 需要中序遍历 | 沿叶子链表遍历 |
| 非叶节点 | 存储键+数据指针 | 仅存储键(索引) |

B+树的优势：
- 内部节点更小，单次I/O读取更多键
- 范围查询高效（叶子链表）
- 所有查询路径长度相同

## 在数据库中的应用

几乎所有关系数据库的索引都使用B+树：
- **MySQL InnoDB**: 主键索引和二级索引都是B+树
- **PostgreSQL**: 默认索引类型为B树
- **SQLite**: 使用B树存储表和索引

## 与BST的关系

B树是二叉搜索树（BST）在多路方向上的泛化。BST每个节点1个键2个子节点，B树每个节点多个键多个子节点。B树解决了BST在大数据量下树高过大导致的I/O效率问题。
