# 二叉搜索树（BST）

## 概述

二叉搜索树（Binary Search Tree，BST）是一种满足特定排序性质的二叉树：对于树中任意节点 N，其左子树中所有节点的键值严格小于 N 的键值，其右子树中所有节点的键值严格大于 N 的键值。这一性质在整棵树中递归成立，使得 BST 天然维护了一种全序关系。

BST 的概念可追溯至 1960 年代，由 P. F. Windley、Andrew Donald Booth 等人独立提出，并在 1962 年前后正式形成理论体系。与普通二叉树不同，BST 不仅是一种存储结构，更是一种将数据组织为可检索形态的算法工具——它将二分查找的对数时间复杂度从静态数组扩展到了支持动态插入和删除的场景。

BST 的核心价值在于：在平衡状态下，查找、插入、删除的时间复杂度均为 O(log n)，其中 n 为节点数量。在 AI 工程中，BST 及其变体被广泛用于决策树路径剪枝、特征索引、近似最近邻搜索预处理等场景，是理解更高级树形索引结构的基础起点。

## 核心原理

### BST 的排序性质与中序遍历

BST 最具标志性的特性是：对其进行中序遍历（左-根-右）所得到的序列必然是严格升序排列。例如对包含键值 {5, 3, 7, 1, 4, 6, 8} 的 BST 执行中序遍历，输出为 1, 3, 4, 5, 6, 7, 8。这一性质使得 BST 可以以 O(n) 时间完成全局排序输出，也是 BST 被称为"搜索树"而非仅仅"有序树"的根本原因。

### 查找、插入与删除操作

**查找**：从根节点出发，将目标键值与当前节点比较。若目标值更小则向左子树递归，若更大则向右子树递归，直至找到或到达空节点。最坏情况下经过 h 步，h 为树的高度。

**插入**：按查找逻辑定位到空位后直接挂载新节点，不会改变现有节点位置。插入序列的顺序对树形结构影响极大：将有序序列 1, 2, 3, 4, 5 依次插入会退化为右斜链表，高度 h = n = 5，查找时间劣化为 O(n)。

**删除**：BST 删除分三种情况：
- 被删节点无子节点：直接移除。
- 被删节点有一个子节点：以子节点替换。
- 被删节点有两个子节点：需找其**中序后继**（右子树中最小节点）或**中序前驱**（左子树中最大节点）替换键值，再删除该后继/前驱节点。此步骤是 BST 删除逻辑中最易出错之处。

### 时间复杂度与树高的关系

BST 所有核心操作的时间复杂度均为 **O(h)**，其中 h 为树高。随机插入 n 个不同键值时，期望树高约为 **1.386 log₂ n**（即约 2ln n）。然而在最坏情况（输入已排序）下 h = n，退化为链表。这一不稳定性正是 AVL 树、红黑树等平衡 BST 出现的直接动机。BST 的空间复杂度始终为 O(n)，与树高无关。

## 实际应用

**数据库索引原型**：早期关系型数据库的内存索引使用 BST 的变体实现有序键值存储，支持范围查询（如 `WHERE age BETWEEN 20 AND 30`）。BST 的中序遍历特性使范围查询只需定位到下界节点后顺序遍历，无需扫描全表。

**AI 决策树的路径剪枝**：在构建 CART（分类回归树）时，特征阈值的选取过程需要对连续特征的候选分裂点进行有序遍历，将特征值以 BST 组织后可将候选阈值搜索从 O(n) 的线性扫描优化为对数时间定位。

**符号表实现**：编译器与解释器（包括 Python 的早期命名空间实现）使用 BST 作为符号表，将变量名作为键值存储，支持 O(log n) 的变量查找与作用域管理，比哈希表在有序输出场景（如调试时打印所有变量）更具优势。

## 常见误区

**误区一：BST 的查找总是 O(log n)**
许多初学者将 BST 的查找效率与二分查找的 O(log n) 画等号，但这仅在树平衡时成立。当插入有序数据（如按时间戳递增插入的日志记录）时，BST 退化为链表，查找退化为 O(n)。这与二分查找的静态数组不同——二分查找的 O(log n) 是无条件保证的，BST 并非如此。

**误区二：BST 的左子树所有节点 ≤ 根节点**
使用"小于等于"而非"严格小于"定义 BST 会导致重复键值处理逻辑混乱，且在删除时可能破坏排序性质。标准 BST 定义要求键值唯一且严格大于/小于，若需支持重复键值，需显式约定重复值统一放入左子树或右子树，并在整棵树中保持一致——两种约定不可混用。

**误区三：删除操作后 BST 性质仍自动保持**
有两个子节点的删除场景中，仅替换被删节点的键值而不删除中序后继节点，会导致树中出现重复键值，破坏 BST 的唯一性约束。正确做法是完成键值替换后，必须递归删除中序后继/前驱节点本身。

## 知识关联

**前置概念衔接**：BST 直接继承了**二叉树**的节点结构（左孩子、右孩子、父节点指针）和遍历算法，其核心搜索逻辑是**二分查找**在树形动态结构上的直接移植——二分查找每次将搜索范围折半，BST 的每次比较同样排除一个子树的所有节点。

**后续概念演进**：BST 的高度不稳定性直接催生了 **AVL 树**：AVL 树在 BST 基础上增加平衡因子（Balance Factor，值为左子树高度减右子树高度，必须保持在 {-1, 0, 1} 之间），通过旋转操作将最坏树高限制在 1.44 log₂ n，从而将所有操作稳定在 O(log n)。而面向磁盘存储的场景，BST 的二路分叉导致树高过大、磁盘 I/O 次数过多，这促使了每个节点存储多个键值的 **B 树与 B+ 树**的发展——B+ 树可视为将 BST 从二叉推广为 m 叉的多级索引结构，是现代数据库（如 MySQL InnoDB）的底层存储引擎核心。
