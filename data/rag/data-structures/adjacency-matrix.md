# 邻接矩阵

## 概述

邻接矩阵（Adjacency Matrix）是表示图结构的一种二维数组方式，用一个 $n \times n$ 的矩阵 $A$ 来表示含有 $n$ 个顶点的图，其中元素 $A[i][j]$ 的值表示顶点 $i$ 与顶点 $j$ 之间是否存在边（或边的权重）。对于无权图，$A[i][j] = 1$ 表示两顶点相邻，$A[i][j] = 0$ 表示不相邻；对于加权图，$A[i][j]$ 存储具体权值，无边处通常填 $0$ 或 $\infty$。

邻接矩阵的概念可追溯至线性代数中的关联矩阵研究。20世纪50年代，图论被 Dénes Kőnig（1950年去世前的遗著《有限图与无限图理论》）和 Claude Berge（1958年出版《图论及其应用》）系统化后，邻接矩阵逐渐成为计算机科学中图的标准表示法之一。1959年前后，随着计算机对稠密图（边数接近 $n^2$ 量级）分析需求的增加，邻接矩阵因其 $O(1)$ 时间查询两点是否相邻的特性被广泛采纳。Cormen 等人在2001年出版的《算法导论》（Introduction to Algorithms）中将邻接矩阵与邻接表并列为图的两种标准存储结构，并明确给出了各自适用的规模阈值（Cormen et al., 2009）。

邻接矩阵在AI工程中至关重要。图神经网络（GNN）中，邻接矩阵 $A$ 直接参与消息传递的矩阵运算。例如，GCN（图卷积网络）的核心传播公式为：

$$H^{(l+1)} = \sigma\!\left(\hat{A} H^{(l)} W^{(l)}\right)$$

其中 $\hat{A}$ 为经过归一化处理的邻接矩阵，$H^{(l)}$ 为第 $l$ 层节点特征矩阵，$W^{(l)}$ 为可学习权重矩阵，$\sigma$ 为激活函数（Kipf & Welling, 2017）。此外，知识图谱、社交网络分析、分子结构建模等任务都依赖邻接矩阵进行高效的线性代数加速计算。

## 核心原理

### 矩阵构建规则

对于有 $n$ 个顶点的图，邻接矩阵是一个 $n \times n$ 的二维数组，行索引和列索引分别对应图的顶点编号（通常从 $0$ 或 $1$ 开始）。

- **无向图**：若顶点 $i$ 和顶点 $j$ 之间有边，则 $A[i][j] = A[j][i] = 1$，矩阵必然是对称矩阵，即 $A = A^\top$。
- **有向图**：$A[i][j] = 1$ 仅表示存在从顶点 $i$ 指向顶点 $j$ 的有向边，反方向 $A[j][i]$ 不一定为 $1$，矩阵通常不对称。
- **自环处理**：若允许自环（顶点指向自身），则 $A[i][i]$ 可为 $1$；否则对角线全为 $0$。
- **加权图**：将"是否有边"替换为"边的权重"，无边处填 $\infty$ 或 $0$，具体取决于算法语义。

例如，以 $4$ 个顶点的无向图为例（边集为 $\{(0,1),(1,2),(2,3),(0,3)\}$），其邻接矩阵为：

```
     0  1  2  3
0  [ 0  1  0  1 ]
1  [ 1  0  1  0 ]
2  [ 0  1  0  1 ]
3  [ 1  0  1  0 ]
```

可以验证该矩阵关于主对角线对称，且对角线全为 $0$（无自环）。边的总数为矩阵中 $1$ 的个数除以 $2$，即 $8 / 2 = 4$ 条，与边集一致。

---

### 时间与空间复杂度

邻接矩阵的空间复杂度固定为 $O(n^2)$，无论图中实际有多少条边，都需要分配 $n^2$ 个存储单元。这使得它在稠密图（边数 $m \approx n^2$）场景下高效，但在稀疏图（$m \ll n^2$）场景下浪费大量空间。

| 操作 | 邻接矩阵 | 邻接表 |
|------|----------|--------|
| 查询 $(u,v)$ 是否相邻 | $O(1)$ | $O(\deg(u))$ |
| 遍历顶点 $u$ 的所有邻居 | $O(n)$ | $O(\deg(u))$ |
| 添加/删除边 | $O(1)$ | $O(1)$ 或 $O(\deg)$ |
| 空间占用 | $O(n^2)$ | $O(n + m)$ |

例如，当图的顶点数 $n = 10{,}000$ 时，邻接矩阵需要存储 $10^8$ 个元素，若每个元素以 32 位浮点数（4字节）存储，则需约 **400 MB** 内存。相比之下，若该图为稀疏图，平均度数仅为 $10$，则边数 $m \approx 50{,}000$，邻接表仅需约 **0.4 MB**，内存占用相差约 **1000 倍**。

判断使用邻接矩阵还是邻接表的经验阈值：图的边密度 $\rho = m / \binom{n}{2}$ 超过 $0.5$ 时优先选邻接矩阵，低于 $0.1$ 时选稀疏存储格式（如 CSR 或 COO）。

---

### 矩阵乘法与图的幂次关系

邻接矩阵最重要的数学性质之一：**$A^k$ 中的元素 $(A^k)[i][j]$ 等于从顶点 $i$ 到顶点 $j$ 恰好经过 $k$ 步的路径数目**。

这一性质直接来源于矩阵乘法定义：

$$A^2[i][j] = \sum_{k} A[i][k] \times A[k][j]$$

其含义是统计从 $i$ 出发，经过某中间节点 $k$，到达 $j$ 的所有长度为 $2$ 的路径总数。例如，对于上述 $4$ 个顶点的无向图，计算 $A^2[0][2]$：

$$A^2[0][2] = A[0][0] \cdot A[0][2] + A[0][1] \cdot A[1][2] + A[0][2] \cdot A[2][2] + A[0][3] \cdot A[3][2] = 0+1+0+1 = 2$$

这表示从顶点 $0$ 出发，恰好走 $2$ 步到达顶点 $2$ 的路径有 $2$ 条（路径 $0 \to 1 \to 2$ 和路径 $0 \to 3 \to 2$），与图的结构完全吻合。

这在图神经网络中具有深刻意义——GCN 的 $L$ 层叠加等价于聚合每个节点半径 $L$ 跳以内的邻居信息，本质上正是利用了邻接矩阵的幂次特性（Kipf & Welling, 2017）。

---

### 度矩阵与归一化邻接矩阵

在图神经网络中，原始邻接矩阵 $A$ 直接参与运算会导致数值不稳定（各行之和不等，且高度节点的梯度幅度远大于低度节点）。因此引入**度矩阵** $D$，其定义为对角矩阵：

$$D[i][i] = \sum_{j} A[i][j], \quad D[i][j] = 0 \;(i \neq j)$$

归一化邻接矩阵有两种常用形式：

- **行归一化**：$D^{-1}A$，每行之和为 $1$，等价于随机游走转移矩阵，适合消息传递中的均值聚合
- **对称归一化**：$D^{-1/2} A D^{-1/2}$，GCN 中使用的标准形式，保持矩阵对称性，谱性质更优

Kipf & Welling（2017）在 GCN 论文中还在归一化前对 $A$ 加上自连接矩阵 $I$，得到 $\tilde{A} = A + I$，然后计算：

$$\hat{A} = \tilde{D}^{-1/2} \tilde{A} \tilde{D}^{-1/2}$$

其中 $\tilde{D}$ 是 $\tilde{A}$ 对应的度矩阵，确保每个节点在聚合时也保留自身特征，从而避免过度平滑的初期退化问题。

## 实际应用

**图神经网络特征传播**：PyTorch Geometric（2019年由 Fey & Lenssen 开源，目前 GitHub 星标超过 21,000）等框架在处理小规模稠密图时，直接以邻接矩阵 $A$（形状为 $[n, n]$ 的张量）参与矩阵乘法实现消息传递。例如，一次矩阵乘法 $AX$（$X$ 为节点特征矩阵，形状 $[n, d]$）即完成所有节点的一阶邻居聚合，时间复杂度为 $O(n^2 d)$。当 $n = 1{,}000$、$d = 64$ 时，该操作约需 $6.4 \times 10^7$ 次浮点运算，在 GPU 上可在 **1 毫秒以内**完成。

**Floyd-Warshall 全对最短路**：经典算法（Cormen et al., 2009，第25章）直接以邻接矩阵（权重矩阵）作为输入，通过三重循环动态更新：

$$\text{dist}[i][j] = \min\!\left(\text{dist}[i][j],\ \text{dist}[i][k] + \text{dist}[k][j]\right)$$

时间复杂度 $O(n^3)$，空间复杂度 $O(n^2)$，是邻接矩阵在稠密图最短路问题中的标准用法。当 $n = 500$ 时，该算法约需 $1.25 \times 10^8$ 次操作，在现代CPU上运行时间约为 **0.5 秒**。

**分子性质预测**：在化学信息学中，分子被建模为图，原子为顶点、化学键为边。例如，葡萄糖分子（$C_6H_{12}O_6$）包含 $24$ 个重原子，其邻接矩阵为 $24 \times 24$ 的稀疏对称矩阵，结合原子特征矩阵（如原子序数、杂化状态等 $d = 78$ 维特征）输入 GNN，可用于预测分子毒性、溶解度、血脑屏障通透性等性质。DeepMind 的 AlphaFold2（2021年）在蛋白质结构预测中也使用了类似的残基接触矩阵（本质上是邻接矩阵的变体）来编码氨基酸的空间关系。
