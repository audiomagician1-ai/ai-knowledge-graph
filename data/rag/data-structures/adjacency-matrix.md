# 邻接矩阵

## 概述

邻接矩阵（Adjacency Matrix）是表示图结构的一种二维数组方法，用一个 n×n 的矩阵 A 来描述含 n 个顶点的图。矩阵中的元素 A[i][j] 表示顶点 i 与顶点 j 之间是否存在边（对于无权图取 0 或 1），或者边的权重（对于加权图取具体数值）。这种表示方式将图的拓扑关系完全编码进一个方阵，使得顶点间连通性的查询时间复杂度降至 O(1)。

邻接矩阵的概念伴随图论的数学形式化而生，可追溯至 Leonhard Euler 1736 年对柯尼斯堡七桥问题的研究。20 世纪中期随着计算机科学兴起，邻接矩阵成为图在计算机内存中存储的最早标准化方案之一。在 AI 工程领域，邻接矩阵直接支撑了图神经网络（GNN）中的消息传递机制，是 2017 年后深度图学习方法爆发式增长的数学基础。

邻接矩阵的独特价值在于其**代数可操作性**。矩阵乘法 A^k 的第 (i,j) 元素恰好等于从顶点 i 到顶点 j 恰好经过 k 步的路径数目，这一性质直接用于最短路径分析和图谱特征提取，是邻接表等其他图表示方式无法直接提供的能力。

## 核心原理

### 矩阵的构造规则

给定一个图 G = (V, E)，设顶点数 |V| = n，邻接矩阵 A 定义如下：

- **无向无权图**：若边 (i, j) ∈ E，则 A[i][j] = A[j][i] = 1，否则为 0。矩阵关于主对角线**对称**，主对角线通常全为 0（不考虑自环）。
- **有向无权图**：若有向边 (i→j) ∈ E，则 A[i][j] = 1，A[j][i] 不一定为 1。矩阵不再保证对称。
- **加权图**：A[i][j] = w(i,j)，即边的权重；若无边则通常赋值 0 或 ∞（依具体算法而定，Dijkstra 中常用 ∞）。

空间复杂度固定为 O(n²)，与边数 m 无关。这意味着对于仅有 n 条边的稀疏图，依然消耗与 n² 条边等量的存储空间。

### 度的计算与矩阵性质

对于无向图，顶点 i 的**度（degree）** 等于邻接矩阵第 i 行（或第 i 列）所有元素之和：

$$\deg(i) = \sum_{j=0}^{n-1} A[i][j]$$

对于有向图，**出度** = 第 i 行之和，**入度** = 第 i 列之和。

无向图的邻接矩阵是**实对称矩阵**，其特征值均为实数。这些特征值构成图的**谱（Spectrum）**，图谱理论由此而来。例如，完全图 K_n 的邻接矩阵特征值为 n-1（重数1）和 -1（重数 n-1），这一数值特性在图同构检测和聚类分析中被广泛利用。

### 幂次路径计数

邻接矩阵最重要的代数性质：**A^k[i][j] 等于从顶点 i 到顶点 j 恰好走 k 步的路径总数**。以下是推导逻辑：

- A¹[i][j]：直接相邻则为 1，即长度为 1 的路径数；
- A²[i][j] = Σ_k A[i][k]·A[k][j]：经由所有中间顶点 k 的长度为 2 的路径数；
- 以此类推，A^k 通过矩阵乘法递推得到。

利用矩阵快速幂，计算 A^k 的时间复杂度为 O(n³ log k)，此方法被用于计算图中三角形数量（对角线元素之和除以6等于三角形数，需计算 A³）。

## 实际应用

### 图神经网络（GNN）中的特征传播

在 Graph Convolutional Network（GCN，Kipf & Welling 2017）中，邻接矩阵 A 直接参与层间特征传播公式：

$$H^{(l+1)} = \sigma\left(\hat{D}^{-\frac{1}{2}} \hat{A} \hat{D}^{-\frac{1}{2}} H^{(l)} W^{(l)}\right)$$

其中 Â = A + I（加入自环），D̂ 为 Â 对应的度矩阵。这里 A 必须是邻接矩阵形式而非邻接表，因为整个传播过程本质上是矩阵乘法。在 PyTorch Geometric 等框架中，稀疏邻接矩阵以 COO 格式存储以节省内存，但逻辑结构仍是邻接矩阵。

### Floyd-Warshall 全源最短路径算法

Floyd-Warshall 算法直接以邻接矩阵作为输入，通过三重循环动态规划更新距离矩阵 D：

```
D[i][j] = min(D[i][j], D[i][k] + D[k][j])
```

时间复杂度 O(n³)，空间复杂度 O(n²)。该算法对邻接矩阵的依赖体现在：对任意两点 (i, j) 的直接距离访问必须是 O(1)，邻接表无法满足这一要求。

### 社交网络的连通性分析

在含 1000 个用户节点的小型社交网络分析任务中，邻接矩阵占用约 1000×1000×4 字节 ≈ 4MB 内存（int32 类型），可完整装入 L2/L3 缓存，使得批量矩阵运算效率极高，适合做好友推荐中的共同邻居计数（A²[i][j] 即共同邻居数量）。

## 常见误区

### 误区一：无权图的邻接矩阵只能用 0/1 表示

部分初学者认为邻接矩阵元素只能取 0 或 1。实际上，邻接矩阵可以存储任意数值权重。更重要的是，在某些算法（如谱聚类）中，矩阵元素可以是**边的相似度分数**（取值范围 0~1 的浮点数），此时矩阵称为**加权邻接矩阵**或**相似度矩阵**，其语义已超出"有边/无边"的二元范畴。

### 误区二：对称性等价于无向图

认为"邻接矩阵对称"等价于"图是无向图"是不完整的理解。有向图若每条边 (i→j) 都有对应的 (j→i)，其邻接矩阵同样对称，但该图仍属于有向图（存在方向信息）。判断图是否为无向图，不能仅依赖矩阵的对称性，还需结合建模语义。

### 误区三：稀疏图使用邻接矩阵效率更高

由于邻接矩阵查询 A[i][j] 是 O(1)，一些初学者误以为无论何种图都应优先使用邻接矩阵。实际上当边数 m << n²（稀疏图，如 m = O(n)）时，邻接矩阵浪费 O(n²) 空间且遍历所有邻居需要扫描整行（O(n) 时间），而邻接表遍历邻居仅需 O(deg(v)) 时间。通常以 m < n²/10 作为考虑切换到邻接表的经验阈值。

## 知识关联

**前置概念**：邻接矩阵建立在**图（Graph）**数据结构的基础概念之上，需要理解顶点集 V、边集 E、有向/无向、加权/无权这些图的基本定义，才能正确构造邻接矩阵并解读其元素含义。图的遍历算法（BFS/DFS）虽可用邻接矩阵实现，但邻接矩阵的理解不依赖于遍历算法。

**衍生工具**：从邻接矩阵出发，自然延伸至**拉普拉斯矩阵** L = D - A（其中 D 为度矩阵），它是谱图理论和图信号处理的核心工具。L 的最小非零特征值（代数连通度 λ₂）直接量化图的连通程度，广泛用于图聚类和半监督学习。此外，**归一化邻接矩阵** D^(-1)A 等变体在随机游走模型和 PageRank 算法中直接使用，其物理含义是状态转移概率矩阵。
