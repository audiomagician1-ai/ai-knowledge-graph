# 邻接矩阵

## 概述

邻接矩阵（Adjacency Matrix）是表示图结构的一种二维数组方式，用一个 n×n 的矩阵 A 来表示含有 n 个顶点的图，其中元素 A[i][j] 的值表示顶点 i 与顶点 j 之间是否存在边（或边的权重）。对于无权图，A[i][j] = 1 表示两顶点相邻，A[i][j] = 0 表示不相邻；对于加权图，A[i][j] 存储具体权值，无边处通常填 0 或 ∞。

邻接矩阵的概念可追溯至线性代数中的关联矩阵研究，20世纪50年代图论被系统化后逐渐成为计算机科学中图的标准表示法之一。1959年前后，随着计算机对稠密图（边数接近 n² 量级）的分析需求增加，邻接矩阵因其 O(1) 时间查询两点是否相邻的特性被广泛采纳。

邻接矩阵在AI工程中至关重要。图神经网络（GNN）中，邻接矩阵 A 直接参与消息传递的矩阵运算，如 GCN（图卷积网络）的核心公式 H^(l+1) = σ(ÂH^(l)W^(l)) 中，Â 即为经过归一化处理的邻接矩阵。此外，知识图谱、社交网络分析、分子结构建模等任务都依赖邻接矩阵进行高效的线性代数加速计算。

## 核心原理

### 矩阵构建规则

对于有 n 个顶点的图，邻接矩阵是一个 n×n 的二维数组，行索引和列索引分别对应图的顶点编号（通常从 0 或 1 开始）。

- **无向图**：若顶点 i 和顶点 j 之间有边，则 A[i][j] = A[j][i] = 1，矩阵必然是对称矩阵，即 A = Aᵀ。
- **有向图**：A[i][j] = 1 仅表示存在从顶点 i 指向顶点 j 的有向边，反方向 A[j][i] 不一定为 1，矩阵通常不对称。
- **自环处理**：若允许自环（顶点指向自身），则 A[i][i] 可为 1；否则对角线全为 0。
- **加权图**：将"是否有边"替换为"边的权重"，无边处填 ∞ 或 0，具体取决于算法语义。

以 4 个顶点的无向图为例（边集为 {(0,1),(1,2),(2,3),(0,3)}），邻接矩阵为：

```
     0  1  2  3
0  [ 0  1  0  1 ]
1  [ 1  0  1  0 ]
2  [ 0  1  0  1 ]
3  [ 1  0  1  0 ]
```

### 时间与空间复杂度

邻接矩阵的空间复杂度固定为 **O(n²)**，无论图中实际有多少条边，都需要分配 n² 个存储单元。这使得它在稠密图（边数 m ≈ n²）场景下高效，但在稀疏图（m << n²）场景下浪费大量空间。

| 操作 | 邻接矩阵 | 邻接表 |
|------|----------|--------|
| 查询 (u,v) 是否相邻 | O(1) | O(degree(u)) |
| 遍历顶点 u 的所有邻居 | O(n) | O(degree(u)) |
| 添加/删除边 | O(1) | O(1) 或 O(degree) |
| 空间占用 | O(n²) | O(n + m) |

当图的顶点数 n = 10,000 时，邻接矩阵需要存储 10⁸ 个元素，若每个元素占 4 字节，则需约 **400 MB** 内存，而一个稀疏图的邻接表可能仅需数 MB。

### 矩阵乘法与图的幂次关系

邻接矩阵最重要的数学性质之一：**A^k 中的元素 (A^k)[i][j] 等于从顶点 i 到顶点 j 恰好经过 k 步的路径数目**。

这一性质直接来源于矩阵乘法定义：A²[i][j] = Σₖ A[i][k] × A[k][j]，其含义是统计从 i 出发，经过某中间节点 k，到达 j 的所有长度为 2 的路径。这在图神经网络中具有深刻意义——GCN 的 L 层叠加等价于聚合每个节点半径 L 跳以内的邻居信息，本质上正是利用了邻接矩阵的幂次特性。

### 度矩阵与归一化邻接矩阵

在图神经网络中，原始邻接矩阵 A 直接参与运算会导致数值不稳定（各行之和不等）。因此引入度矩阵 D，其中 D[i][i] = Σⱼ A[i][j]（顶点 i 的度数），其余元素为 0。

归一化邻接矩阵有两种常用形式：
- **行归一化**：D⁻¹A，每行之和为 1，类似随机游走矩阵
- **对称归一化**：D^(-1/2) A D^(-1/2)，GCN 中使用的标准形式，保持矩阵对称性

GCN 论文（Kipf & Welling, 2017）还在归一化前对 A 加上自连接矩阵 I，得到 Ã = A + I，然后计算 Â = D̃^(-1/2) Ã D̃^(-1/2)，确保每个节点在聚合时也保留自身特征。

## 实际应用

**图神经网络特征传播**：PyTorch Geometric 等框架在处理小规模稠密图时，直接以邻接矩阵 A（形状为 [n, n] 的张量）参与矩阵乘法实现消息传递，一次矩阵乘法 AX（X 为节点特征矩阵，形状 [n, d]）即完成所有节点的一阶邻居聚合，时间复杂度为 O(n²d)。

**Floyd-Warshall 全对最短路**：经典算法直接以邻接矩阵（权重矩阵）作为输入，通过三重循环动态更新 dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])，时间复杂度 O(n³)，是邻接矩阵在稠密图最短路问题中的标准用法。

**分子性质预测**：在化学信息学中，分子被建模为图，原子为顶点、化学键为边。分子邻接矩阵通常为 n×n 稀疏对称矩阵（n 为原子数，通常小于 100），结合原子特征矩阵输入 GNN，用于预测分子毒性、溶解度等性质。

**网络连通性分析**：邻接矩阵的秩（rank）与图的连通分量数量相关；矩阵的特征值（谱分析）可揭示图的结构特性，如最小非零特征值（代数连通度/Fiedler值）衡量图的连通紧密程度，用于社区发现和图分割算法。

## 常见误区

**误区一：无向图邻接矩阵一定是01矩阵**
初学者常认为邻接矩阵元素只能是0或1。实际上，加权无向图的邻接矩阵存储边的权值，元素可以是任意实数。更进一步，多重图（两点间存在多条边）中 A[i][j] 可以存储边的条数，邻接矩阵退化为计数矩阵。GNN中的邻接矩阵经归一化后，元素通常是介于0和1之间的浮点数，而非整数。

**误区二：邻接矩阵适合所有规模的图**
许多人在实现图算法时不加区分地使用邻接矩阵。当图的顶点数 n 超过 10,000 时，邻接矩阵所需内存往往超过实际可用资源。真实世界的社交网络（如Twitter用户图，n > 10⁸）根本无法用邻接矩阵存储，必须使用邻接表或 COO/CSR 稀疏格式。判断使用邻接矩阵还是邻接表的经验阈值：图的边密度（m / n²）超过 0.5 时选邻接矩阵，低于 0.1 时选稀疏存储。

**误区三：A[i][j] = 0 一定表示无边**
在加权图中，若边的权重本身允许为 0（如某些距离或相似度场景），用 0 表示"无边"会引发歧义。正确做法是用 None、NaN 或 ∞ 表示无边，用实际权值（包括可能的 0）表示有边。Floyd-Warshall 算法初始化时就明确区分这两种情况：dist[i][i] = 0（自身到自身）而 dist[i][j] = ∞（无直接边）。
