# 邻接矩阵

## 概述

邻接矩阵（Adjacency Matrix）是表示图结构的一种二维数组方法，通过一个 n×n 的矩阵 A 来描述含有 n 个顶点的图中各顶点之间的连接关系。矩阵中第 i 行第 j 列的元素 A[i][j] 存储顶点 i 与顶点 j 之间的边信息：若二者直接相连，则记为 1（或对应权值）；若不相连，则记为 0。

邻接矩阵作为图的存储方式，最早随着图论在计算机科学中的系统化应用而出现，在 20 世纪 60 年代算法研究兴起时被广泛采纳。其直觉来源于线性代数中的矩阵运算——正是因为图的邻接矩阵可以参与矩阵乘法，才使得路径计数、传递闭包等问题获得了优雅的代数解法。

邻接矩阵之所以在 AI 工程的图处理任务中受到重视，是因为它支持 O(1) 的边存在性查询，并且能与 NumPy、PyTorch 等张量运算框架无缝结合，使图神经网络（GNN）中的消息传递操作可以直接用矩阵乘法实现。

## 核心原理

### 矩阵的构造规则

对于一个含 n 个顶点的图，邻接矩阵 A 是一个 n×n 的方阵，其元素定义如下：

- **无权图**：A[i][j] = 1 若边 (i, j) 存在，否则 A[i][j] = 0
- **带权图**：A[i][j] = w(i,j) 若边 (i, j) 存在（权重为 w），否则 A[i][j] = 0 或 ∞
- **无向图**：矩阵关于主对角线对称，即 A[i][j] = A[j][i]，对称矩阵存储时可压缩为上三角形式，节省约 50% 空间
- **有向图**：A[i][j] = 1 仅表示从顶点 i 到顶点 j 有有向边，矩阵通常不对称

对角线元素 A[i][i] 通常为 0，除非图中存在自环（self-loop）。

### 矩阵幂与路径计数

邻接矩阵最重要的代数性质：**A^k 的第 (i, j) 个元素恰好等于从顶点 i 到顶点 j 长度为 k 的路径总数**。

以 3 个顶点的完全图为例，A² 的计算揭示了所有长度为 2 的路径数量。若要判断图的连通性，可计算 S = A + A² + A³ + … + A^(n-1)，若 S[i][j] > 0 则 i 与 j 之间存在路径。这一性质直接支撑了传递闭包（Warshall 算法）和 Floyd-Warshall 全对最短路算法的矩阵化实现。

### 度数的直接计算

在无权无向图中，顶点 i 的**度数**等于邻接矩阵第 i 行（或第 i 列）所有元素的和：

```
degree(i) = Σ A[i][j]，j 从 0 到 n-1
```

在有向图中，第 i 行之和为**出度**（out-degree），第 i 列之和为**入度**（in-degree）。在图神经网络中，这一性质被用于构造度矩阵 D，进而计算归一化拉普拉斯矩阵 L = I - D^(-1/2) · A · D^(-1/2)，该公式是 GCN（Graph Convolutional Network）的核心算子。

### 空间与时间复杂度

邻接矩阵无论图的实际边数多少，始终占用 O(n²) 的存储空间。对于稀疏图（边数远小于 n²），这意味着大量空间被零元素浪费。而边的插入、删除和查询均为 O(1)，遍历所有邻居则需要 O(n) 时间——不论该顶点实际有多少条边。

## 实际应用

**图神经网络的特征传播**：在 PyTorch Geometric 等框架的底层实现中，若使用稠密邻接矩阵，消息传递等价于 H' = A · H · W，其中 H 是节点特征矩阵，W 是可学习权重矩阵，矩阵乘法可以在 GPU 上高度并行化。

**小规模路由网络分析**：交通或通信网络中顶点数在数百以内时，邻接矩阵可直接存储为二维数组，利用矩阵乘法在 O(n³) 时间内完成全对最短路径计算（Floyd-Warshall），代码实现极为简洁。

**关系数据库的图查询**：在知识图谱中，若实体总数不超过几千，使用邻接矩阵可以将"两跳关系查询"转化为矩阵平方运算，避免递归遍历带来的栈溢出风险。

**竞技游戏中的 AI 决策**：棋盘类游戏的状态图顶点数有限，使用邻接矩阵表示状态转移图，可以在预处理阶段用矩阵幂快速计算 k 步可达状态集合。

## 常见误区

**误区一：认为无向图的邻接矩阵一定浪费一半存储空间**。实践中，Python 的 NumPy 或 SciPy 库提供了稀疏矩阵格式（如 `scipy.sparse.csr_matrix`），可以只存储上三角元素；而 C/C++ 中也可只分配 n*(n+1)/2 大小的一维数组模拟上三角矩阵。邻接矩阵并不一定等同于完整的 n×n 内存分配。

**误区二：邻接矩阵适用于任意规模的图**。当图的顶点数 n = 10,000 时，邻接矩阵需要存储 10⁸ 个元素，若为浮点数则占用约 400 MB 内存；当 n = 100,000 时内存需求达到 40 GB，完全不可行。在 AI 工程中处理大规模知识图谱或社交网络时，必须改用邻接表（Adjacency List）或 CSR（Compressed Sparse Row）格式，而非邻接矩阵。

**误区三：邻接矩阵的 A[i][j] 与 A[j][i] 在有向图中含义相同**。这是最常见的编码错误之一。在有向图中，A[i][j] = 1 表示存在从 i 指向 j 的有向边，而 A[j][i] 表示从 j 指向 i 的边，二者完全独立。混淆方向会导致 GNN 中信息流向错误，或在最短路径算法中产生方向相反的结果。

## 知识关联

**与图（数据结构）的关系**：邻接矩阵是图这一抽象数据结构的具体存储实现之一。图定义了顶点和边的逻辑关系，而邻接矩阵将这种关系编码为可随机访问的二维数组。理解图的有向/无向、有权/无权等基本分类，是正确设置邻接矩阵元素的前提——例如无向图的对称性约束直接决定了矩阵的填写规则。

**与邻接表的对比**：邻接表（Adjacency List）是图的另一种主流存储方式，用链表或数组记录每个顶点的邻居列表，空间复杂度为 O(n + m)（m 为边数），适合稀疏图。邻接矩阵与邻接表的选择核心在于图的稠密程度：稠密图（m ≈ n²）用邻接矩阵更高效，稀疏图（m ≪ n²）用邻接表更节省内存。在 AI 工程中，图神经网络框架 PyTorch Geometric 默认采用 COO（Coordinate）稀疏格式的边列表，而非完整的邻接矩阵，正是出于对稀疏图内存效率的考量。

**与拉普拉斯矩阵的关联**：图拉普拉斯矩阵 L = D - A（D 为度矩阵）直接以邻接矩阵 A 为基础构造，是谱图卷积（Spectral Graph Convolution）和图信号处理的核心工具。掌握邻接矩阵的构造和性质，是理解 GCN、GraphSAGE 等图神经网络模型消息传递机制的必要前提。
