# 栈

## 概述

栈（Stack）是一种遵循**后进先出（LIFO, Last In First Out）**原则的线性数据结构。这意味着最后被压入栈的元素，将是第一个被弹出的元素——就像一摞盘子，只能从顶部放入或取出。栈的所有插入和删除操作都限制在同一端进行，这一端称为**栈顶（Top）**，另一端称为**栈底（Bottom）**。

栈的概念最早可追溯到1946年，由艾伦·图灵在描述子程序调用机制时隐式引入。1955年，Friedrich Ludwig Bauer和Klaus Samelson正式申请了"栈"作为计算机存储原理的专利，并将其应用于编译器设计。今天，栈已经是几乎所有编程语言运行时系统的底层支柱——每一次函数调用，都依赖调用栈（Call Stack）来保存返回地址和局部变量。

在AI工程的数据结构学习中，栈是实现**深度优先搜索（DFS）**、表达式求值、括号匹配和回溯算法的直接工具。理解栈的工作方式，能让你在实现神经网络计算图的反向传播、解析语法树等任务时，拥有清晰的思维模型。

## 核心原理

### 基本操作与时间复杂度

栈只支持三种核心操作，且每种操作的时间复杂度均为 **O(1)**：

- **Push（压栈）**：将元素添加到栈顶
- **Pop（弹栈）**：移除并返回栈顶元素
- **Peek / Top（查看栈顶）**：返回栈顶元素但不移除它

此外，`isEmpty()` 判断栈是否为空，`size()` 返回栈中元素个数。对空栈执行 Pop 操作会触发**栈下溢（Stack Underflow）**错误；当栈使用固定大小数组实现时，过度压栈则会触发**栈溢出（Stack Overflow）**错误——这正是著名问答网站名称的由来。

### 两种实现方式：数组 vs 链表

**基于数组的实现**：用一个整型变量 `top` 记录栈顶索引，初始值为 `-1`。Push 操作执行 `stack[++top] = value`，Pop 操作执行 `return stack[top--]`。这种方式内存连续，缓存命中率高，但容量固定。

```python
class ArrayStack:
    def __init__(self, capacity=100):
        self._data = [None] * capacity
        self._top = -1

    def push(self, val):
        self._top += 1
        self._data[self._top] = val

    def pop(self):
        val = self._data[self._top]
        self._top -= 1
        return val
```

**基于链表的实现**：每次 Push 在链表头部插入新节点，Pop 删除头部节点。容量动态增长，但每个节点有额外的指针开销（通常8字节）。Python 中可直接使用内置列表作为动态数组栈，`append()` 对应 Push，`pop()` 对应 Pop，两者均为均摊 O(1)。

### LIFO 原则的数学描述

若依次压入元素序列 $a_1, a_2, \ldots, a_n$，则弹出顺序必然是 $a_n, a_{n-1}, \ldots, a_1$。对于 $n$ 个不同元素，合法的出栈序列总数由**卡特兰数（Catalan Number）**给出：

$$C_n = \frac{1}{n+1}\binom{2n}{n}$$

例如，3个元素的合法出栈序列有 $C_3 = 5$ 种，而不是全部 $3! = 6$ 种排列。序列 `3,1,2` 就无法通过一个栈实现（压1后弹出3是不可能的）。

## 实际应用

### 括号合法性检验

这是栈最经典的入门应用。遍历字符串，遇到左括号 `(`、`[`、`{` 则压栈，遇到右括号时弹出栈顶并检查是否匹配。若字符串遍历结束后栈为空，则括号合法。这一逻辑直接应用于编译器的词法分析阶段，以及 AI 框架（如 TensorFlow）解析计算图定义文件时的语法检查。

### 函数调用栈与递归展开

每当程序调用函数时，运行时系统自动将**栈帧（Stack Frame）**压入调用栈，栈帧包含：返回地址、函数参数、局部变量。函数返回时，栈帧被弹出，控制权交回调用者。Python 默认的调用栈深度限制为 **1000 层**（可通过 `sys.setrecursionlimit()` 修改），超过则抛出 `RecursionError`。将递归算法改写为显式栈迭代，是规避此限制的标准技巧。

### 逆波兰表达式（后缀表达式）求值

表达式 `3 4 + 2 *` 对应中缀式 `(3 + 4) * 2`。求值规则：遇到数字压栈，遇到运算符则弹出两个操作数计算后将结果压回。最终栈中唯一剩余元素即为答案（本例结果为 `14`）。AI 框架的计算图在执行前向传播时，本质上也是在对一棵表达式树做后缀遍历。

## 常见误区

**误区1：认为栈只能用于"撤销操作"**
撤销（Undo）确实是栈的典型应用，但这仅是 LIFO 特性的一个场景。栈更本质的作用是**维护嵌套结构的上下文**，例如 DFS 遍历中维护待访问节点路径、编译器维护作用域链。混淆"撤销"与"栈"会让学生在遇到 DFS 或表达式解析时想不到用栈。

**误区2：认为 Python 的 `list` 不是真正的栈**
Python 的 `list` 配合 `append()` 和 `pop()` 完全符合栈的语义定义，且尾部操作的均摊时间复杂度为 O(1)。无需引入 `collections.deque` 来实现栈——`deque` 更适合队列（双端操作）。只有当你需要线程安全的栈时，才应使用 `queue.LifoQueue`。

**误区3：混淆"程序调用栈"与"手动实现的栈"**
调用栈是操作系统和语言运行时自动管理的内存区域，通常大小受限（Linux 默认 8MB）；手动实现的栈是堆内存上的数据结构，受限于可用内存总量。递归深度触发的 `RecursionError` 源于调用栈溢出，与你代码里自定义的栈对象无关。

## 知识关联

**前置知识——数组/列表**：栈最常见的实现直接复用数组的随机访问和尾部追加特性。理解数组的内存布局（连续存储、索引访问为 O(1)）能帮助你直觉化地理解为什么基于数组的栈 Push/Pop 如此高效。

**后续概念——深度优先搜索（DFS）**：DFS 的递归实现隐式依赖调用栈；迭代实现则需要手动维护一个节点栈。两者等价，但迭代版本不受递归深度限制，在处理深度可能达到数万的图时更为安全。

**后续概念——队列**：队列遵循 FIFO（先进先出）原则，与栈的 LIFO 正好相反。对比栈和队列在 BFS 与 DFS 中的不同角色，是理解两种搜索策略本质差异的最直接路径。

**后续概念——单调栈**：单调栈是在普通栈基础上增加了一个不变量（栈内元素单调递增或递减），专门用于解决"下一个更大元素"类问题，时间复杂度从暴力 O(n²) 降至 O(n)。掌握基础栈操作是理解单调栈维护过程的前提。
