# 树状数组

## 概述

树状数组（Binary Indexed Tree，BIT），又称 Fenwick Tree，由 Peter Fenwick 于 1994 年在论文《A New Data Structure for Cumulative Frequency Tables》中提出，专门用于高效解决动态前缀和查询与单点更新问题。其核心思想是利用二进制表示中的最低有效位（Lowest Set Bit，LSB）将数组下标分组，从而将朴素数组 O(n) 的前缀和查询降低到 O(log n)，同时保持 O(log n) 的更新复杂度。

树状数组在物理存储上仍是一个一维数组，大小与原数组相同，但每个位置 `c[i]` 存储的不是原数组的单个元素，而是原数组某个区间 `[i - lowbit(i) + 1, i]` 的区间和。例如，`c[6]`（二进制 110）的 `lowbit(6) = 2`，因此 `c[6]` 存储原数组 `a[5] + a[6]` 的和。这种非均匀的区间划分正是树状数组能以极简代码实现 O(log n) 操作的根本原因。

相比线段树，树状数组代码量约为其 1/3，常数因子更小，在竞赛和工程中用于前缀和类问题时往往是首选。但其功能天然受限：标准树状数组仅支持单点更新和前缀查询，而线段树可扩展至区间修改与区间查询的任意组合。

## 核心原理

### lowbit 运算

树状数组的一切操作都依赖 `lowbit(x)` 函数，其定义为：

```
lowbit(x) = x & (-x)
```

其原理源于补码：`-x` 将 `x` 的二进制位取反后加 1，与 `x` 按位与后恰好保留最低有效位。例如：
- `lowbit(6) = 110 & 010 = 010 = 2`
- `lowbit(8) = 1000 & 1000 = 1000 = 8`
- `lowbit(12) = 1100 & 0100 = 0100 = 4`

节点 `c[i]` 负责维护原数组中长度为 `lowbit(i)` 的区间 `[i - lowbit(i) + 1, i]`。下标越小的节点负责的区间越短，下标为 2 的幂次的节点负责最长的区间（长度等于自身下标）。

### 前缀和查询

查询前 `x` 项的前缀和，从位置 `x` 开始，每次将 `x` 减去 `lowbit(x)` 并累加 `c[x]`，直到 `x = 0`：

```python
def query(x):
    s = 0
    while x > 0:
        s += c[x]
        x -= lowbit(x)
    return s
```

以查询前 7 项为例：7（111）→ 6（110）→ 4（100）→ 0，依次累加 `c[7]`、`c[6]`、`c[4]`，共 3 步。由于每次操作去掉最低有效位，循环次数恰好等于 `x` 的二进制中 1 的个数，最多 log₂(n) 次。

### 单点更新

将位置 `i` 的值增加 `delta`，需要更新所有"覆盖"该位置的节点 `c[i]`。从 `i` 开始，每次将 `i` 加上 `lowbit(i)`，直到超出数组范围：

```python
def update(i, delta):
    while i <= n:
        c[i] += delta
        i += lowbit(i)
```

以更新位置 3（二进制 011）为例：3 → 4（100）→ 8（1000）→ …，依次更新 `c[3]`、`c[4]`、`c[8]`。每步操作将最低有效位"进位"，路径长度同样不超过 log₂(n) 步。

### 区间更新 + 区间查询（差分扩展）

标准树状数组只支持单点更新，但借助差分数组技巧可扩展为区间更新。设差分数组 `d[i] = a[i] - a[i-1]`，则 `a[x]` 的前缀和为：

```
sum(a[1..x]) = Σ_{i=1}^{x} Σ_{j=1}^{i} d[j]
             = Σ_{j=1}^{x} d[j] * (x - j + 1)
             = (x+1) * Σd[j] - Σ(j * d[j])
```

因此维护两个树状数组 `BIT1` 存 `d[i]`、`BIT2` 存 `i*d[i]`，即可在 O(log n) 内完成区间 `[l, r]` 加 `v` 的更新与区间前缀和的查询，而无需升级到线段树。

## 实际应用

**逆序对计数**：给定长度为 n 的数组，统计满足 `i < j` 且 `a[i] > a[j]` 的数对数量。离散化后，从右向左遍历，对每个元素查询比其小的元素已出现次数（`query(a[i]-1)`），然后 `update(a[i], 1)`。整个过程时间复杂度 O(n log n)，是归并排序法的等价替代，代码更简洁。

**动态前缀和在推荐系统中的应用**：在 AI 工程的在线学习（Online Learning）中，特征权重需要频繁单点更新，同时需要快速获取某特征区间的权重累积值以计算归一化因子。树状数组的 O(log n) 双操作使其在特征维度达到百万级时依然满足实时性要求，而全量重算前缀和需要 O(n) 时间无法接受。

**二维树状数组**：扩展至二维后，`c[i][j]` 负责矩形区域 `[i-lowbit(i)+1, i] × [j-lowbit(j)+1, j]` 的矩形和，单点更新和矩形前缀和查询均为 O(log² n)，常用于二维图像特征的累积直方图查询。

## 常见误区

**误区一：树状数组下标从 0 开始**。树状数组要求下标从 1 开始，因为 `lowbit(0) = 0`，会导致 `update` 和 `query` 陷入无限循环或死循环。将输入数据统一加 1 偏移是最简单的修复方式，忽略这一点是初学者最高频的 bug 来源。

**误区二：树状数组可以像线段树一样直接支持区间最值查询**。树状数组的更新路径具有方向性——`update` 沿 `i += lowbit(i)` 向高位传播，这导致区间最小值/最大值无法通过类似前缀和分解的方式正确合并。对区间最值需求必须使用线段树或稀疏表（Sparse Table），强行用树状数组实现会得到错误结果。

**误区三：初始化树状数组必须逐一调用 update**。逐一调用 `update` 初始化的时间复杂度为 O(n log n)。更高效的方式是 O(n) 线性建树：对每个位置 `i`，令 `c[i] += a[i]`，然后令 `j = i + lowbit(i)`，若 `j <= n` 则 `c[j] += c[i]`，该方法将建树时间减半，在 n = 10⁷ 规模时差异明显。

## 知识关联

**与前缀和的关系**：静态前缀和数组 `pre[i]` 以 O(n) 预处理、O(1) 查询为代价，完全不支持更新。树状数组是其动态化版本，以 O(log n) 查询和 O(log n) 更新为代价换取可修改性，是"静态 → 动态"这一结构演化路径中的标准中间站。

**与线段树的关系**：线段树每个节点显式存储区间信息，支持懒惰传播（Lazy Propagation），可处理区间修改 + 区间查询；树状数组通过差分技巧也能模拟部分功能，但无法支持懒惰传播本身。当问题仅需前缀查询时，优先使用树状数组；当需要任意区间查询或非加法类统计（如最值、最大公约数）时，才需升级到线段树。

**与二叉树的关系**：树状数组的"树形"结构是隐式的——其父子关系由 `lowbit` 运算定义，不需要指针或显式的左右孩子存储。位置 `i` 的父节点在 update 路径上是 `i + lowbit(i)`，在 query 路径上是 `i - lowbit(i)`，两条路径共同构成隐式二叉树的上行遍历，这也是其空间效率优于显式树结构的原因。
