# 树状数组（Binary Indexed Tree）

## 概述

树状数组（Binary Indexed Tree，简称 BIT），又称 Fenwick 树，由 Peter M. Fenwick 于 1994 年在论文《A New Data Structure for Cumulative Frequency Tables》中正式提出。其核心设计目标是：在 O(log n) 时间内完成单点更新与前缀和查询，同时将空间复杂度压缩至 O(n)，相比线段树节省约一半内存。

树状数组的名称来源于其内部结构——底层是一个普通数组 `c[]`，但每个下标 `i` 存储的不是单个元素值，而是从 `i - lowbit(i) + 1` 到 `i` 这一段区间的元素累加和。这种"每个节点覆盖不同长度区间"的安排使得它在形态上类似一棵隐式的二叉树，同时无需显式维护左右子节点指针，结构极为紧凑。

在 AI 工程中，树状数组频繁出现于在线学习系统的累积奖励统计、特征频次前缀查询、以及海量日志流的滑动窗口求和等场景。相比暴力前缀和数组，树状数组支持动态更新，避免了每次修改后 O(n) 重建的代价，因此在需要持续写入与实时查询并存的工程场景中具有不可替代的价值。

## 核心原理

### lowbit 操作——树状数组的基石

树状数组的一切计算都依赖 `lowbit(x)` 函数，其定义为：

```
lowbit(x) = x & (-x)
```

其中 `-x` 在二进制补码下等价于对 `x` 按位取反后加 1。例如，`x = 6`（二进制 `110`），则 `-x = 010`，`lowbit(6) = 6 & (-6) = 2`。`lowbit(x)` 的结果恰好等于 `x` 的二进制表示中最低位的 1 所对应的值，即 `x` 的最低有效位（LSB）。正是这个值决定了数组中每个位置所管辖区间的长度：`c[i]` 覆盖的区间长度恰好等于 `lowbit(i)`。

| 下标 i | 二进制 | lowbit(i) | 管辖区间       |
|--------|--------|-----------|----------------|
| 1      | 001    | 1         | [1, 1]         |
| 2      | 010    | 2         | [1, 2]         |
| 4      | 100    | 4         | [1, 4]         |
| 6      | 110    | 2         | [5, 6]         |
| 8      | 1000   | 8         | [1, 8]         |

### 前缀和查询——沿路径向左跳跃

查询 `[1, x]` 的前缀和时，算法从 `x` 出发，每次将 `x` 减去 `lowbit(x)`，累加 `c[x]`，直到 `x = 0` 为止：

```python
def query(x):
    s = 0
    while x > 0:
        s += c[x]
        x -= lowbit(x)
    return s
```

以查询 `x = 7`（二进制 `111`）为例：依次访问 `c[7]`（区间 [7,7]）、`c[6]`（区间 [5,6]）、`c[4]`（区间 [1,4]），三步累加得到 `[1,7]` 的前缀和。查询路径长度恰好等于 `x` 二进制表示中 1 的个数，最多 ⌊log₂n⌋ 步，即时间复杂度严格为 O(log n)。

### 单点更新——沿路径向右跳跃

更新位置 `x` 的值（加上增量 `delta`）时，需要更新所有覆盖了位置 `x` 的 `c[i]`。从 `x` 出发，每次将 `x` 加上 `lowbit(x)`，直到超出数组范围：

```python
def update(x, delta):
    while x <= n:
        c[x] += delta
        x += lowbit(x)
```

以更新 `x = 3` 为例：依次更新 `c[3]`（lowbit=1，跳至 4）、`c[4]`（lowbit=4，跳至 8）、`c[8]`……每次都跳向"父节点"，保证所有覆盖该位置的节点均被同步修改。

### 区间更新与区间查询——差分技巧的结合

纯粹的树状数组只支持单点更新+前缀查询。若需同时支持**区间更新**（将 `[l, r]` 每个元素加 `v`）和**区间求和查询**，需维护两个树状数组 `B1[]` 和 `B2[]`，基于差分数组推导出以下恒等式：

```
前缀和 S(x) = Σ a[i] (i=1..x)
            = B1 树状数组前缀和 × (x+1) - B2 树状数组前缀和
```

具体地，对差分数组 `d[i] = a[i] - a[i-1]` 进行推导：

```
S(x) = (x+1) × Σd[i] - Σ(i × d[i])，i=1..x
```

因此令 `B1[i] = d[i]`，`B2[i] = i × d[i]`，分别维护两棵树状数组，即可在 O(log n) 内完成区间更新和区间求和，代码量远少于线段树的懒标记实现。

## 实际应用

**逆序对计数**：给定长度为 n 的排列，统计逆序对数量。将元素从左到右依次插入树状数组，每插入元素 v 时，查询 `[v+1, n]` 的前缀和即为以 v 为右端点的逆序对数。整体时间复杂度 O(n log n)，在竞赛和数据预处理中被广泛使用。

**滑动窗口中位数**：维护两棵树状数组分别记录当前窗口内各值的出现频次，通过二分查找第 k 大元素（利用前缀和单调性），可在 O(log² n) 内求出滑动窗口中位数。这是 AI 特征工程中在线统计分位数的常见方案。

**强化学习中的优先经验回放（PER）**：在 Prioritized Experience Replay 中，每个样本具有优先级 `p_i`，需要按照概率 `p_i / Σp_i` 采样。使用树状数组维护优先级前缀和，可在 O(log n) 内完成优先级更新和按概率采样，替代朴素的 O(n) 线性扫描，是 DeepMind 2015 年 PER 论文中的工程实现细节。

## 常见误区

**误区一：树状数组的下标必须从 0 开始。** 实际上树状数组的下标**必须从 1 开始**。因为 `lowbit(0) = 0`，会导致 `update` 循环中 `x += lowbit(x)` 永远不增加，造成死循环。若原数组从 0 开始，必须整体偏移 +1 后再存入树状数组。

**误区二：树状数组可以替代线段树的所有功能。** 树状数组无法直接支持区间最大值/最小值查询（RMQ），因为最大值不具备"差分可逆"的代数性质——`max(a, b)` 无法通过减法还原单个元素。RMQ 必须使用线段树或稀疏表（Sparse Table）。树状数组仅适用于满足**群（Group）**代数结构的运算，即需要存在逆元（如加法的相反数）。

**误区三：树状数组的区间查询公式是 `query(r) - query(l)`。** 正确公式是 `query(r) - query(l - 1)`，即查询 `[l, r]` 区间和应减去 `[1, l-1]` 的前缀和。误写成 `query(l)` 会多减一个 `a[l]`，是初学者最高频的 off-by-one 错误，在代码调试中极难发现。

## 知识关联

**前置概念关联**：树状数组的前缀和查询直接继承了**前缀和**数组的思想，但前缀和数组是静态的（O(1) 查询，O(n) 更新），而树状数组通过牺牲少量查询常数换来了 O(log n) 的动态更新能力。学习者应先彻底理解 `sum[i] = sum[i-1] + a[i]` 的静态前缀和原理，再迁移到动态场景。**二叉树**的层级跳跃思想同样体现在 lowbit 的上下跳跃路径中——update 向上找父节点，query 向下拆分区间，与二叉树的祖先路径遍历本质相同。

**与线段树的对比**：**线段树**能处理树状数组不支持的 RMQ 问题，但常数因子约为树状数组的 3-4 倍，代码量约多 3 倍。在仅需前缀和/区间和的场景下，应优先选择树状数组。两者在前缀和查询的渐近复杂度上均为 O(log n)，但树状数组的缓存局部性（cache locality）更好，因其访问路径为数组的连续跳跃而非树结构的随机指针跳转，在 n = 10⁶ 规模下实测速度快约 2 倍。
