# 线段树

## 概述

线段树（Segment Tree）是一种基于完全二叉树结构的区间数据结构，专门用于高效处理区间查询和区间更新问题。其核心思想是将长度为 $n$ 的数组递归地拆分为子区间，每个节点存储对应区间的聚合信息（如区间和、区间最大值等），从而将朴素的 $O(n)$ 区间操作优化至 $O(\log n)$。

线段树的概念最早在计算几何领域被正式研究，1977年前后出现在处理线段交叉检测的算法文献中，后逐渐被引入竞技编程和工程领域。在 AI 工程的数据预处理、特征滑窗统计、在线学习样本管理等场景中，线段树因其严格的时间复杂度保证而被广泛采用。

对于长度 $n = 10^5$ 的数组，若需执行 $10^5$ 次区间求和查询并夹杂着区间批量修改，朴素数组的总时间复杂度为 $O(n^2) = O(10^{10})$，而线段树可将总复杂度压缩至 $O(n \log n) \approx O(1.7 \times 10^6)$，这一差距在实时推荐系统的特征统计场景中具有实际意义。

## 核心原理

### 树的构建与节点存储

线段树用一维数组 `tree[]` 存储，节点编号遵循堆式编码：根节点为 `tree[1]`，节点 $i$ 的左子节点为 `tree[2i]`，右子节点为 `tree[2i+1]`。为安全容纳所有节点，数组大小通常开为原数组长度的 **4倍**（严格上界为 $4n$）。

构建过程使用分治递归，代表区间 $[l, r]$ 的节点取中点 $mid = \lfloor (l+r)/2 \rfloor$，左子节点覆盖 $[l, mid]$，右子节点覆盖 $[mid+1, r]$。对于区间求和线段树，父节点值满足：

$$\text{tree}[i] = \text{tree}[2i] + \text{tree}[2i+1]$$

建树的时间复杂度为 $O(n)$，空间复杂度为 $O(n)$。

### 区间查询

查询区间 $[ql, qr]$ 时，从根节点出发递归判断当前节点区间 $[l, r]$ 与查询区间的关系：
- **完全包含**（$ql \leq l$ 且 $r \leq qr$）：直接返回当前节点的聚合值，无需继续递归；
- **完全不相交**（$ql > r$ 或 $qr < l$）：返回单位元（求和返回0，求最大值返回 $-\infty$）；
- **部分重叠**：递归查询左右子节点并合并结果。

最坏情况下查询路径涉及 $O(\log n)$ 个节点，单次查询时间复杂度为 $O(\log n)$。实验表明，对于 $n = 10^6$ 的数据，单次查询实际比较次数约为 $4 \log_2 n \approx 80$ 次。

### 懒标记（Lazy Propagation）优化

若不使用懒标记，对区间 $[ql, qr]$ 中每个元素执行加法更新，需递归修改所有叶子节点，时间复杂度退化为 $O(n)$。懒标记通过**延迟下传**解决这一问题。

每个节点额外维护一个 `lazy[]` 数组，含义为"该节点区间内所有元素待施加的增量"。区间更新时，若当前节点区间被查询区间完全覆盖，仅更新当前节点的值和 `lazy` 标记，**不立即递归到子节点**。只有当后续查询或更新需要访问子节点时，才通过 `pushdown` 操作将标记下推：

```
pushdown(node, l, r):
    mid = (l + r) / 2
    tree[2*node] += lazy[node] * (mid - l + 1)
    tree[2*node+1] += lazy[node] * (r - mid)
    lazy[2*node] += lazy[node]
    lazy[2*node+1] += lazy[node]
    lazy[node] = 0
```

引入懒标记后，区间更新的时间复杂度从 $O(n)$ 降至 $O(\log n)$，使得 $m$ 次混合操作总复杂度为 $O((n + m) \log n)$。

## 实际应用

**在线学习中的滑动窗口统计**：在流式机器学习系统（如实时 CTR 预估）中，需要维护最近 $k$ 步的梯度平方和用于 AdaGrad 更新。线段树可在 $O(\log k)$ 时间内完成单点更新和区间求和，优于重新遍历历史梯度的 $O(k)$ 方案。

**区间最大子数组问题**：线段树节点可同时存储区间总和、前缀最大值、后缀最大值和区间最大子数组和四个字段，合并时按固定公式计算，实现 $O(\log n)$ 的动态最大子数组查询，应用于强化学习中优先级经验回放（Prioritized Experience Replay）的优先级区间管理，SumTree 即为此结构的变体。

**数据库索引范围聚合**：部分列式数据库的聚合查询优化层使用线段树维护分段统计信息，支持 COUNT、SUM、MIN、MAX 的 $O(\log n)$ 区间查询，而无需全表扫描。

## 常见误区

**误区一：数组大小开 2 倍足够**  
由于线段树用堆式编号存储，节点数量为完全二叉树的节点总数而非原数组长度。对于 $n$ 个叶子节点，树高为 $\lceil \log_2 n \rceil$，最坏情况下节点总数可达 $2^{\lceil \log_2 n \rceil + 1} - 1$。当 $n = 5$ 时，该值为 $2^4 - 1 = 15$，而 $2n = 10$ 不够用。因此必须开 $4n$ 大小的数组。

**误区二：懒标记可以不初始化直接使用**  
`lazy[]` 数组必须初始化为单位元（加法更新初始化为 0，乘法更新初始化为 1）。若忘记初始化，C++ 中局部数组的 `lazy` 值为随机内存垃圾，会导致第一次 `pushdown` 时将错误增量累加到子节点，产生难以复现的数值错误。

**误区三：线段树与树状数组（BIT）功能等价**  
树状数组（Binary Indexed Tree）仅支持**单点更新 + 前缀查询**的特定模式，且无法直接支持区间最大值查询。线段树通过懒标记支持真正的**区间更新 + 任意区间查询**，以及任意可结合的聚合函数（如 GCD、异或）。两者的适用场景不同，不能简单互替。

## 知识关联

**与二叉树的关系**：线段树是完全二叉树的直接应用，其堆式存储、递归建树和后序聚合均依赖二叉树的遍历逻辑。若不熟悉二叉树的父子节点编号关系（$2i$ 和 $2i+1$），则难以正确实现线段树的节点访问。

**与分治法的关系**：线段树的建树过程本质上是分治法的具象化——将 $[l, r]$ 分为 $[l, mid]$ 和 $[mid+1, r]$ 两个子问题，递归求解后合并，正是标准分治框架。懒标记的下推逻辑也体现了分治"分解-处理-合并"的思路。

**通向树状数组**：学习线段树后，树状数组（Fenwick Tree，1994年 Peter Fenwick 提出）是其简化特例——牺牲区间更新的通用性，换取常数因子更小的代码和空间。理解线段树的区间分解逻辑后，更容易理解树状数组为何仅用 `lowbit` 操作即可完成前缀求和：两者本质上都在对区间进行二进制分层管理。
