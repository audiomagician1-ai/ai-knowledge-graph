# 并查集

## 概述

并查集（Union-Find 或 Disjoint Set Union，简称 DSU）是一种专门用于处理**动态连通性问题**的数据结构，维护若干个互不相交的集合，并支持两种核心操作：将两个集合合并（Union）和查询某个元素所属集合（Find）。它由 Bernard Galler 和 Michael Fischer 于 1964 年首次提出，后经 Robert Tarjan 在 1975 年通过路径压缩与按秩合并的分析，证明其均摊时间复杂度接近 O(α(n))，其中 α 是增长极其缓慢的 Ackermann 函数的反函数——对于任意实际应用规模，α(n) ≤ 5，因此几乎等同于常数时间。

并查集的底层存储依赖一个整数数组 `parent[]`，其中 `parent[i]` 存储节点 i 的父节点编号。初始状态下每个节点的父节点指向自身，即 `parent[i] = i`，表示每个元素各自构成一个独立的单元素集合。这种以"森林"表示若干不相交集合的设计，使得集合的合并与查询都可以通过树根节点的统一标识来实现，避免了显式维护集合成员列表的高昂代价。

在 AI 工程的图算法领域，并查集是判断图中节点连通性、检测环路、以及构建最小生成树的基础工具。Kruskal 算法正是依赖并查集在 O(E log E) 时间内完成最小生成树的构建，其中 E 是边数。

## 核心原理

### Find 操作与路径压缩

`Find(x)` 的目标是找到节点 x 所在树的根节点（即集合的代表元）。朴素实现是沿 `parent` 指针不断上溯直到 `parent[i] == i`，最坏情况下时间复杂度退化为 O(n)（当树退化为链时）。

**路径压缩（Path Compression）** 优化：在 Find 操作返回时，将路径上所有节点的 `parent` 直接指向根节点：

```python
def find(x):
    if parent[x] != x:
        parent[x] = find(parent[x])  # 递归压缩路径
    return parent[x]
```

路径压缩后，树的高度趋于 2（根节点加叶节点），后续 Find 操作几乎为 O(1)。

### Union 操作与按秩合并

`Union(x, y)` 将节点 x 和 y 所在的两棵树合并。朴素做法是直接将一棵树的根挂到另一棵树的根下，可能导致树高不断增加。

**按秩合并（Union by Rank）** 规则：维护额外数组 `rank[]`，初始均为 0，始终将较矮的树挂到较高的树的根下，仅当两棵树秩相等时才将秩加 1：

```python
def union(x, y):
    rx, ry = find(x), find(y)
    if rx == ry:
        return  # 已在同一集合，无需合并
    if rank[rx] < rank[ry]:
        rx, ry = ry, rx
    parent[ry] = rx
    if rank[rx] == rank[ry]:
        rank[rx] += 1
```

单独使用按秩合并时，树高上限为 ⌊log₂n⌋，Find 操作为 O(log n)。两者结合后达到接近 O(α(n)) 的均摊复杂度。

### 加权并查集（带权值的扩展）

标准并查集仅追踪"是否同集合"，加权并查集还在 `parent` 指针上附加**相对权值** `weight[x]`，表示 x 到其父节点的某种关系量（如差值或比值）。路径压缩时需要同步沿途累积权值，常用于解决差分约束中的带权连通性问题，例如判断节点间距离是否满足特定约束。

## 实际应用

**检测无向图中的环**：在逐条添加边 (u, v) 时，若 `Find(u) == Find(v)`，说明 u 和 v 已在同一连通分量中，再加此边必然成环。这一判断在 Kruskal 算法构建最小生成树时是核心步骤——每次选取权重最小的边后，用并查集检测是否会构成环，若不成环则执行 Union，否则跳过该边。

**社交网络中的连通分量统计**：给定 n 个用户和 m 条好友关系，使用并查集初始化 n 个集合，遍历每条好友关系执行 Union，最终统计 `parent[i] == i` 的节点数即为独立社交圈的数量。这一过程时间复杂度为 O(m · α(n))，远优于 BFS/DFS 的 O(n + m) 在需要**动态增量添加关系**的场景下的重复计算开销。

**LeetCode 经典题型**：第 547 题"省份数量"、第 684 题"冗余连接"、第 990 题"等式方程的可满足性"均以并查集为标准解法，其中第 990 题综合使用 Union 处理 `==` 关系、Find 验证 `!=` 关系是否矛盾，体现了并查集在约束传播中的应用。

**像素连通域分析（计算机视觉）**：在图像分割任务中，对二值图像的每个像素与其相邻像素执行并查集合并，最终每个连通域对应一个唯一的根节点，整个过程仅需一次扫描即可完成连通域标记，是传统两遍扫描算法的高效替代方案。

## 常见误区

**误区一：路径压缩会破坏按秩合并的秩信息**
路径压缩将节点直接挂到根节点后，`rank[]` 数组中的值与实际树高不再严格对应——这是正确且预期的行为。`rank` 此时退化为"秩的上界"而非精确树高，但这并不影响正确性：合并决策依然选择秩值较大的树作为父树，仍能防止树高过度增长，均摊复杂度分析依然成立。若坚持维护精确树高，需改用"按大小合并"（Union by Size），用 `size[]` 记录子树节点总数。

**误区二：并查集支持拆分（Split）操作**
并查集的设计假设集合合并是**不可逆的单向操作**。标准并查集不支持将已合并的集合重新拆分，因为路径压缩后的 `parent` 指针已丢失原始的父子关系信息。若需支持撤销合并，必须使用"可撤销并查集"（不启用路径压缩，仅使用按秩合并，通过栈记录每次 Union 的操作历史来实现 Undo），其单次 Find 操作退化为 O(log n)。

**误区三：并查集可直接枚举某集合的所有成员**
查询根节点是 O(α(n)) 的，但若需要列出集合内所有元素，标准并查集需要遍历全部 n 个节点执行 Find 并筛选，代价为 O(n · α(n))。若频繁需要枚举集合成员，应在并查集基础上额外维护一个哈希表或链表，将根节点映射到其所有成员列表，从而以 O(1) 额外空间维护代价换取 O(k) 的成员枚举（k 为集合大小）。

## 知识关联

并查集的输入通常来自**图（Graph）**的边集合——图中节点对应并查集的元素，图的连通关系通过 Union 操作建立。理解图的邻接表表示和连通分量概念，是正确初始化和解读并查集结果的前提。

掌握并查集后，**Kruskal 最小生成树**算法得以直接应用：Kruskal 算法将所有边按权重升序排序后依次处理，每条边的环检测和连通分量合并都委托给并查集完成，整体时间瓶颈在于排序的 O(E log E)，并查集部分仅贡献 O(E · α(V))。此外，并查集也是**最小生成树**理论中验证生成树唯一性的工具——若图中存在权重相同的边导致多棵可能的最小生成树，并查集的 Find 操作可用于在不同边选择策略间快速切换连通性状态来枚举所有候选树。
