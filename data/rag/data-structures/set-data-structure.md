# 集合

## 概述

集合（Set）是一种存储**不重复元素**的无序数据结构。与列表不同，集合中每个元素只能出现一次，且元素之间不存在固定的排列顺序。在Python中，集合使用花括号 `{}` 或 `set()` 函数创建，例如 `{1, 2, 3}` 或 `set([1, 2, 2, 3])` 会自动去重得到 `{1, 2, 3}`。

集合的数学根源来自19世纪末德国数学家格奥尔格·康托尔（Georg Cantor）于1874年创立的集合论。计算机科学中的集合数据结构正是对数学集合概念的程序化实现，支持数学集合的并集、交集、差集等基本运算。在AI工程和数据处理领域，集合常用于快速去重、成员判断以及特征工程中的类别处理。

集合的最大价值在于其**O(1)平均时间复杂度**的成员查找操作。当需要判断某个元素是否存在于一组数据中时，集合比列表的O(n)线性扫描快得多。在处理百万级别的用户ID去重、词汇表构建或图算法中的已访问节点标记时，集合的性能优势尤为突出。

## 核心原理

### 哈希表底层实现

Python中的集合基于**哈希表（Hash Table）**实现，本质上是一个只存储键、不存储值的字典。每个元素插入时，Python调用该元素的 `__hash__()` 方法计算哈希值，再通过 `hash(x) % table_size` 确定存储槽位。由于哈希表的随机访问特性，查找、插入、删除的**平均时间复杂度均为O(1)**，最坏情况（哈希冲突严重时）退化为O(n)。

正因为依赖哈希，**集合的元素必须是可哈希的（hashable）**。整数、字符串、元组可以放入集合；列表、字典、另一个普通集合则不能，因为它们是可变对象，无法计算稳定的哈希值。Python提供了 `frozenset` 作为不可变集合，`frozenset` 本身是可哈希的，可以作为集合的元素或字典的键。

### 四大集合运算

集合支持四种标准的数学集合运算，在Python中均有运算符和方法两种写法：

| 运算 | 运算符 | 方法 | 说明 |
|------|--------|------|------|
| 并集 | `A \| B` | `A.union(B)` | A和B中所有不重复元素 |
| 交集 | `A & B` | `A.intersection(B)` | A和B共同拥有的元素 |
| 差集 | `A - B` | `A.difference(B)` | 在A中但不在B中的元素 |
| 对称差集 | `A ^ B` | `A.symmetric_difference(B)` | 只属于A或只属于B的元素 |

例如在推荐系统中，`已购买商品集合 - 已推荐商品集合` 可快速得出"尚未推荐的已购买商品"，这种差集操作在O(min(len(A), len(B)))时间内完成。

### 元素唯一性的自动维护

集合在插入时自动执行去重逻辑：向集合添加已存在的元素不会报错，也不会产生副本，集合大小保持不变。`{1, 1, 2, 2, 3}` 在Python解释器中会立即被规范化为 `{1, 2, 3}`。这一特性使集合成为**去重的最惯用工具**：`list(set(my_list))` 是将列表去重的标准Python写法，时间复杂度为O(n)，远优于双重循环的O(n²)方案。

## 实际应用

**自然语言处理中的词汇表构建**：构建语料库词汇表时，直接将所有分词结果加入集合，即可自动获得无重复的词汇集合。例如处理100万条评论，使用集合收集词汇后调用 `len(vocab_set)` 即得词汇量，无需额外去重步骤。

**图的BFS/DFS中的已访问节点标记**：广度优先搜索中，使用集合 `visited = set()` 标记已访问节点，每次扩展邻居前执行 `if node not in visited` 判断。集合的O(1)查找使得整体图遍历保持O(V+E)的时间复杂度；若改用列表，每次查找为O(V)，总复杂度退化为O(V²+E)。

**数据清洗中的类别特征去重**：在特征工程中，`set(df['category'].tolist())` 可快速获取某列的所有唯一类别，用于构建one-hot编码的类别列表或检验训练集与测试集的类别分布是否一致（通过差集运算 `train_categories - test_categories` 发现测试集中未见过的新类别）。

**反爬虫与URL去重**：网络爬虫中，已爬取URL集合用集合存储，每次抓取新链接前先做成员检测，避免重复请求，节省带宽与计算资源。

## 常见误区

**误区一：认为集合有固定的迭代顺序**。集合是无序的，`for x in my_set` 的迭代顺序由哈希值决定，不保证与插入顺序一致，也不保证在不同Python版本或不同运行中顺序相同。如果需要有序且唯一的序列，应使用 `sorted(my_set)` 转换，或考虑使用Python 3.7+中保留插入顺序的字典键来模拟有序集合。

**误区二：混淆空集合的创建方式**。`{}` 在Python中创建的是**空字典**，而非空集合。创建空集合必须使用 `set()`，这是初学者极易犯错的地方。向误以为是集合的空字典执行 `.add()` 方法会立即抛出 `AttributeError`。

**误区三：认为集合操作总是快于列表**。集合的O(1)查找优势需要在**频繁成员检测**场景下才能体现。如果只需遍历所有元素一次，列表的内存开销更小（集合的哈希表需要预留约1/3的空槽以保证性能）。当元素数量极少（如少于10个）时，列表的线性扫描由于缓存局部性反而可能比集合查找更快。

## 知识关联

集合与**字典/映射**共享相同的哈希表底层实现，可以将集合理解为"只有键没有值的字典"。掌握字典的哈希冲突处理机制（开放寻址法或链地址法）后，集合的性能特征自然可以类推。集合的可哈希元素限制与字典键的可哈希要求完全相同，两者在这一约束上是统一的。

在AI工程实践中，集合常作为**图数据结构**中邻接表的基础组件——用集合而非列表存储邻居节点，可将"判断两节点是否相邻"从O(degree)降至O(1)，这对于大规模知识图谱查询尤为重要。同时，集合运算（交集大小/并集大小）是计算**Jaccard相似度**的直接工具：`Jaccard(A,B) = |A∩B| / |A∪B|`，该公式广泛用于文档相似度计算和协同过滤推荐中的用户/物品相似度度量。
