# AVL树

## 概述

AVL树是由苏联数学家Georgy Adelson-Velsky和Evgenii Landis于1962年发明的第一种自平衡二叉搜索树，其名称正取自两位发明者姓氏首字母。与普通二叉搜索树（BST）在最坏情况下退化为链表（时间复杂度O(n)）不同，AVL树通过维护严格的高度平衡条件，将查找、插入和删除操作的时间复杂度稳定保证在O(log n)。

AVL树的核心约束是**平衡因子（Balance Factor, BF）**：任意节点的左子树高度与右子树高度之差的绝对值不超过1，即 |BF| ≤ 1。一旦某次插入或删除操作导致任意节点的 |BF| = 2，AVL树必须立即通过旋转操作恢复平衡。这一严格约束使得AVL树的高度始终维持在 1.44 × log₂(n) 以内，是BST平均高度的常数倍。

AVL树在需要频繁查询且插入/删除相对较少的场景中表现优异，例如Linux内核早期版本的内存管理和部分数据库索引结构。理解AVL树的旋转机制是后续学习红黑树等更复杂平衡树的必要基础。

## 核心原理

### 平衡因子的计算与维护

每个AVL树节点除存储键值外，还必须记录其**高度（height）**。节点的平衡因子定义为：

```
BF(node) = height(left_child) - height(right_child)
```

空节点的高度约定为 -1，叶节点高度为 0。每次插入或删除后，需要从操作节点向上回溯，逐级更新祖先节点的高度，并检查BF是否越界。一棵含 n 个节点的AVL树，其最大高度由Fibonacci数列控制：高度为 h 的AVL树最少含 F(h+3) - 1 个节点，其中 F 是Fibonacci序列。

### 四种旋转类型

当 |BF| = 2 时，AVL树通过四种旋转之一恢复平衡，旋转类型取决于失衡节点及其子树的形态：

**LL旋转（右单旋）**：失衡节点的左子节点BF ≥ 0（左-左型失衡）。以失衡节点为轴执行右旋，左子节点上升为新父节点，原父节点变为其右子节点。

**RR旋转（左单旋）**：失衡节点的右子节点BF ≤ 0（右-右型失衡）。以失衡节点为轴执行左旋，右子节点上升为新父节点。

**LR旋转（先左后右双旋）**：失衡节点的左子节点BF < 0（左-右型失衡）。先对左子节点执行左旋（转为LL型），再对失衡节点执行右旋。

**RL旋转（先右后左双旋）**：失衡节点的右子节点BF > 0（右-左型失衡）。先对右子节点执行右旋（转为RR型），再对失衡节点执行左旋。

四种情况涵盖了所有失衡场景，旋转操作本身的时间复杂度为 O(1)，最多需要执行 O(log n) 次旋转。

### 插入与删除的旋转次数差异

AVL树的**插入操作**从失衡点向上最多只需执行**1次单旋或双旋**即可全树恢复平衡，因为单次插入只能使路径上的高度增加1，一次旋转足以消除失衡。而**删除操作**则可能触发 O(log n) 次旋转，因为删除可能导致沿祖先路径的多个节点相继失衡。这一非对称性是AVL树与红黑树在性能权衡上的关键区别。

## 实际应用

**内存分配器中的有序集合**：glibc早期版本的内存池使用AVL树管理空闲块，利用其严格平衡性保证 O(log n) 的空闲块查找速度，适合内存分配请求多于释放的场景。

**数据库中的内存索引**：当数据集完全加载在内存中且查询操作远多于写操作时（读写比约为9:1以上），AVL树比红黑树更有优势。由于AVL树高度更低（比红黑树矮约25%），每次查询经历的比较次数更少。

**编译器符号表**：编译器在语义分析阶段需要频繁查找变量名，而符号插入主要集中在解析阶段。这种"集中插入、持续查询"的访问模式恰好契合AVL树的优势。

**Python的`sortedcontainers`库设计参考**：虽然该库实际使用分段列表，但其设计文档明确对比了AVL树方案，说明AVL树是实现有序集合时的标准参考基准。

## 常见误区

**误区一：认为旋转后需要重新扫描全树**

很多初学者认为旋转后需要重新检查所有节点的平衡因子。实际上，AVL树的旋转具有**局部性**：单旋或双旋后，子树的高度恢复到插入/删除前的值，不会影响祖先节点的BF，因此只需从失衡点向上回溯检查即可，插入后最多1次旋转就能终止回溯。

**误区二：混淆LR型与RL型的旋转顺序**

LR型双旋的第一步是对**左子节点**左旋，而非对失衡节点操作。初学者常因手势混淆而写出错误实现。正确记忆方式：LR型中"L"代表失衡在左侧，"R"代表左子节点的右孩子引起失衡，第一步旋转的对象是左子节点，方向是向L的反方向（即向左旋）。

**误区三：认为AVL树在所有场景都优于红黑树**

AVL树高度严格，查询效率略优，但其删除操作最坏需要 O(log n) 次旋转，而红黑树的插入和删除旋转次数均被常数上界（最多3次）。在写操作密集的场景（如STL的`std::map`采用红黑树），AVL树的旋转开销反而更大。

## 知识关联

**与二叉搜索树的关系**：AVL树完全继承BST的节点结构和中序遍历有序性，所有BST查找算法在AVL树上直接适用。AVL树在BST基础上额外维护height字段，并在每次修改后触发旋转恢复平衡。若BST插入序列已有序（如1,2,3,...,n），BST退化为高度n的链表，而AVL树始终保持高度不超过1.44log₂n。

**通往红黑树的桥梁**：红黑树同样通过旋转维护平衡，但用5条颜色规则代替严格的BF约束，换取更少的旋转次数（插入≤2次，删除≤3次）。学习AVL树的四种旋转是理解红黑树双旋操作的直接前提，红黑树的LR/RL案例与AVL树完全同构。

**通往B树的抽象**：B树将AVL树的"每节点存1个键、2个子节点"推广为"每节点存m-1个键、m个子节点"，用降低树高的方式减少磁盘I/O。AVL树的平衡思想（通过结构调整维持高度上界）在B树中以"节点分裂与合并"的形式延续。
