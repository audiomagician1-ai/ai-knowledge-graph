# AVL树

## 概述

AVL树是由苏联数学家Georgy Adelson-Velsky和Evgenii Landis于1962年发明的第一种自平衡二叉搜索树，其名称正是取自两位发明者姓氏的首字母。它在普通二叉搜索树（BST）的基础上增加了严格的高度平衡约束：对于树中任意节点，其左子树与右子树的高度差（称为平衡因子）的绝对值不得超过1。

这一约束保证了AVL树的高度始终保持在 $O(\log n)$ 范围内，从而使查找、插入、删除操作的最坏时间复杂度均为 $O(\log n)$。相比之下，普通BST在最坏情况（如顺序插入）下会退化为链表，时间复杂度降至 $O(n)$。AVL树是数据库索引、内存有序集合等对查询性能要求极高的场景中的重要选择。

## 核心原理

### 平衡因子与高度计算

AVL树为每个节点维护一个**平衡因子（Balance Factor，BF）**，定义为：

$$BF(node) = Height(左子树) - Height(右子树)$$

其中空树的高度定义为 -1，单节点的高度为 0。合法的AVL树中每个节点的BF只能为 -1、0 或 +1。当插入或删除操作导致某节点的 $|BF| = 2$ 时，必须通过旋转操作恢复平衡。

### 四种旋转操作

AVL树通过四种旋转来修复失衡，所有旋转均在 $O(1)$ 时间内完成：

**LL旋转（右旋）**：失衡节点的左子节点的左子树过高（BF = +2，左孩子BF = +1）。对失衡节点执行单次右旋，将左孩子提升为子树根。

**RR旋转（左旋）**：失衡节点的右子节点的右子树过高（BF = -2，右孩子BF = -1）。对失衡节点执行单次左旋。

**LR旋转（先左旋后右旋）**：失衡节点的左子节点的右子树过高（BF = +2，左孩子BF = -1）。先对左孩子执行左旋，再对失衡节点执行右旋，共两步。

**RL旋转（先右旋后左旋）**：失衡节点的右子节点的左子树过高（BF = -2，右孩子BF = +1）。先对右孩子执行右旋，再对失衡节点执行左旋，共两步。

### 插入与删除后的回溯修复

**插入**时，新节点始终作为叶节点插入，之后从插入位置向根节点回溯，逐层更新每个祖先节点的平衡因子。一旦发现某祖先节点的 $|BF| = 2$，执行对应的旋转操作后，该子树高度恢复到插入前水平，**无需继续向上回溯**，最多只需1次旋转（LL/RR）或2次旋转（LR/RL）。

**删除**时情况更复杂：删除节点后同样需要向上回溯，但与插入不同，**可能需要沿路径进行多次旋转**，最坏情况下旋转次数为 $O(\log n)$。这是AVL树删除操作的一个重要特性，也是其相比红黑树的主要实现代价之一。

### 高度的精确上界

对于含有 $n$ 个节点的AVL树，其高度 $h$ 满足：

$$h \leq 1.44 \log_2(n+2) - 0.328$$

这一公式来源于AVL树最坏情况结构——Fibonacci树（又称AVL最小节点树）。高度为 $h$ 的AVL树最少包含 $N(h) = N(h-1) + N(h-2) + 1$ 个节点，其中 $N(0)=1, N(1)=2$，该递推关系与Fibonacci数列紧密相关。

## 实际应用

**C++标准库的有序容器**：`std::map` 和 `std::set` 在多数编译器实现中使用红黑树，但部分嵌入式或实时系统的有序集合实现选择AVL树，原因是AVL树的查询性能更严格（树高更低），适合读多写少的场景。

**数据库内存索引**：在内存数据库（如SQLite的内存模式、早期版本的PostgreSQL内部排序结构）中，AVL树用于维护有序的行指针集合，支持范围查询和精确查找各 $O(\log n)$ 完成。

**AI工程中的特征字典**：在自然语言处理的词表管理或强化学习的状态空间索引场景中，若词汇量在百万级别且插入频率低于查询频率，AVL树相比哈希表的优势在于能高效支持前缀范围查询，且无哈希碰撞导致的性能抖动。

## 常见误区

**误区1：认为AVL树的旋转只发生在插入节点的父节点处**。实际上，旋转发生在**从插入点向根回溯过程中第一个出现 $|BF|=2$ 的祖先节点**，该节点可能距离新节点多层，取决于路径上的平衡因子分布。

**误区2：混淆插入和删除的旋转次数上限**。插入后最多需要1次（双步视为1次修复动作），修复完成后子树高度不变，因此不再向上传播失衡。而删除后可能导致树高降低，失衡沿路径向上传播，需要 $O(\log n)$ 次旋转，这与插入的行为截然不同。

**误区3：平衡因子为0意味着节点是叶节点**。平衡因子为0仅表示左右子树等高，一个内部节点（拥有两个非空子树）同样可以有BF=0，例如一个节点左右各有一棵高度为3的子树。

## 知识关联

**前置知识（二叉搜索树BST）**：AVL树的节点插入和查找逻辑完全继承自BST，按左小右大的顺序导航。理解BST在顺序插入时退化为链表（高度 $O(n)$）的问题，正是AVL树引入高度约束的动机所在。

**后续概念（红黑树）**：红黑树同样是自平衡BST，但其平衡条件比AVL树宽松——通过"任意路径上黑节点数相同"保证树高在 $O(2\log n)$ 以内，换取了删除操作旋转次数最多为3次的优势，这使得红黑树在写入频繁的场景（如Linux内核调度器、Java的TreeMap）中性能优于AVL树。

**后续概念（B树与B+树）**：AVL树是每个节点存一个键值的二路平衡树，而B树将其推广到多路情形，每个节点可存 $t-1$ 到 $2t-1$ 个键（$t$ 为最小度数），专门针对磁盘I/O的块读取特性优化，是数据库磁盘索引的基础，与AVL树形成内存索引与磁盘索引的鲜明对比。
