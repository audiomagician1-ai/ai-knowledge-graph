# LOD阴影优化

## 概述

LOD阴影优化是一种专门针对实时阴影渲染性能瓶颈的技术策略：当物体距离摄像机超过一定阈值时，改用比可见渲染更低精度的网格来参与阴影投射计算。其核心依据是阴影投射不需要与可见几何体完全一致——投影到地面或墙壁上的阴影轮廓，在远距离下已经模糊到肉眼几乎无法区分简化误差的程度。

这项技术最早在游戏引擎开发中被系统化应用，Unity 的 `Shadow LOD` 参数与 Unreal Engine 的 `Shadow Distance` + `LOD for Shadow` 配置在2010年代的3A游戏管线中逐渐成为标准工作流的一部分。其重要性在于：阴影Pass本身通常需要对场景中所有投影物体进行一次额外的几何绘制调用（Draw Call），而使用高精度LOD0网格参与这次绘制在远处毫无视觉收益，只会消耗大量GPU顶点处理预算。

对于一个含有10,000个三角面的树木LOD0网格，若将其阴影投射切换为仅含200个三角面的LOD2网格，该物体阴影Pass的顶点处理量可降低约98%，这种优化在植被密集场景中效果尤为显著。

## 核心原理

### 阴影网格与渲染网格解耦

阴影优化的关键在于将"阴影投射使用的几何体"与"摄像机可见渲染使用的几何体"分离成两条独立管线。在技术实现上，引擎在执行阴影贴图（Shadow Map）生成Pass时，可以单独为每个物体指定一个专用的LOD级别，而不必与当前可见LOD保持一致。例如，一棵树的可见渲染使用LOD0（10,000三角面），但其阴影贴图绘制使用LOD2（200三角面），这两套网格各自服务于不同的渲染目标，互不干扰。

### 距离阈值与LOD偏移量

LOD阴影优化通常通过"阴影LOD偏移值（Shadow LOD Bias）"来控制，而非另设一套独立距离表。偏移值为N意味着阴影投射始终比可见渲染多使用N个LOD级别的简化网格。以Unreal Engine为例，将 `Shadow LOD Bias` 设置为1时，若摄像机当前渲染物体为LOD0，则该物体的阴影投射将使用LOD1网格；若已渲染LOD1，则阴影使用LOD2。这种偏移机制避免了为阴影单独配置距离参数，减少了维护成本。

计算阴影LOD实际使用级别的公式为：

```
ShadowLODLevel = clamp(VisibleLODLevel + ShadowLODBias, 0, MaxLODLevel)
```

其中 `VisibleLODLevel` 是当前摄像机距离对应的可见LOD级别，`ShadowLODBias` 为美术或引擎配置的偏移量，`clamp` 确保结果不超出网格实际拥有的LOD级别上限。

### 阴影贴图分辨率与几何精度的匹配关系

当阴影贴图分辨率为1024×1024时，1个像素对应的世界空间面积会随投影距离增大而显著放大。在50米距离处，1个阴影像素可能覆盖约0.25平方米的地面；在200米处，同一像素可能覆盖超过4平方米。此时即使用LOD0的精确轮廓投影，最终贴图精度也无法呈现几何细节，因此在200米处使用LOD3的粗略轮廓投射阴影，视觉损失几乎为零，而顶点处理量的节省却是实质性的。这种"阴影贴图像素覆盖面积"与"合理阴影LOD精度"之间的匹配关系，是选择偏移值的核心依据。

## 实际应用

**开放世界植被场景**：在《地平线：零之曙光》等开放世界游戏中，草地和树木数量极其庞大，阴影Pass是GPU瓶颈的主要来源之一。通常的配置是：0-20米范围使用LOD0投影，20-80米切换到LOD1投影，80米以外禁用自身阴影或仅保留一个极简的Capsule（胶囊体）阴影替代。

**Capsule阴影作为最终简化阶段**：在LOD链的末端，可以用一个椭圆胶囊体形状的"近似阴影"完全替代网格投影。Unity HDRP和Unreal Engine均支持对角色和树木启用Capsule Shadow，该方案将阴影投射几何体从数千三角面压缩到仅需2个参数（半径和高度）描述的隐式体，适用于80米以上的角色阴影。

**LOD阴影偏移值的典型配置经验**：对于静态建筑类网格，`Shadow LOD Bias = 1` 通常足够；对于植被类网格（剪影复杂），`Shadow LOD Bias = 2` 是更常见的选择；对于角色类（动态骨骼变形明显），偏移值一般保持为0或1，以避免动画状态下阴影轮廓失真过于明显。

## 常见误区

**误区一：阴影LOD与可见LOD必须同步切换**
部分美术在搭建LOD链时习惯性地让阴影网格与可见网格保持相同级别，认为"不一致会穿帮"。实际上，由于阴影贴图分辨率的物理限制，远处的阴影轮廓本身就已模糊，即便阴影网格比可见网格粗糙2个LOD级别，玩家也几乎察觉不到差异，而顶点成本的差异可达数十倍。

**误区二：关闭远处物体阴影比使用低LOD阴影更优**
直接将远距离物体的阴影投射完全关闭确实能节省性能，但会导致地面上出现明显的"阴影突然消失"的视觉跳变（Shadow Popping）。使用LOD2或LOD3的简化网格保持投影，可以在性能与视觉连贯性之间取得平衡，特别是在阴影距离过渡边界附近，简化阴影的存在能有效掩盖切换时的突变感。

**误区三：LOD阴影优化只对静态网格有效**
骨骼动画角色同样能从LOD阴影优化中获益。Unreal Engine支持为SkeletalMesh单独设置 `Shadow LOD Bias`，使角色在远处用低精度蒙皮网格参与阴影计算，这对多人同屏场景（如100人大逃杀游戏）的GPU顶点着色器负载有明显改善效果。

## 知识关联

**前置概念—LOD切换策略**：LOD阴影优化建立在LOD切换策略已经为资产配置好多级LOD网格链的基础上。若资产只有LOD0一个级别，则无法应用阴影偏移，美术需要在制作阶段专门为阴影投射需求额外制作低精度LOD层级。

**与阴影贴图（Shadow Map）技术的关系**：LOD阴影优化的有效性直接受阴影贴图分辨率影响——分辨率越低，更激进的LOD阴影偏移值越难被察觉，反之亦然。在制作质量规格书（Spec）时，阴影贴图分辨率与阴影LOD偏移值应作为配套参数一起确定。

**与剔除（Culling）策略的协同**：LOD阴影优化通常与阴影距离剔除（Shadow Distance Culling）配合使用——在偏移值用尽LOD级别之后，由距离剔除接管，完全停止超远距离物体的阴影投射，两者形成阴影渲染开销从高到低的完整梯度控制链。
