# 面数标准

## 概述

面数标准（Polygon Count Standard，也称三角面数预算标准）是技术美术制定的一套针对游戏或实时渲染项目中各类资产所允许使用的最大三角面（Triangle）数量的规范文件。三角面数直接影响GPU的顶点着色器（Vertex Shader）调用次数，面数越高，每帧渲染的顶点处理开销越大，因此面数标准是控制项目运行性能的第一道关卡。

面数标准的概念随着实时3D游戏的发展而逐步成型。1996年前后早期主机游戏时代，角色模型通常只有200–500个三角面；到了PS2/Xbox时代（2000–2005年），主角模型普遍在1500–5000个三角面之间；进入当代次时代（Current-Gen）流水线，主角模型可达到50,000–150,000个三角面，而这背后正是一套严格的面数标准在约束每个岗位的美术人员。

面数标准之所以重要，在于它是团队协作的契约文件：模型师据此建模，技术美术据此做性能预算，引擎团队据此规划DrawCall与LOD切换距离。没有面数标准，同一个项目中不同美术人员交付的资产面数可能相差十倍以上，导致场景优化工作量不可控。

## 核心原理

### 面数预算的分层结构

面数标准通常按资产类型分层制定，典型的分层结构如下：

| 资产类型 | 移动端（Mobile） | PC/主机（Console/PC） |
|---|---|---|
| 主角角色（Hero） | 3,000–8,000 tri | 40,000–80,000 tri |
| NPC（次要角色） | 1,500–3,000 tri | 15,000–30,000 tri |
| 主要道具（Weapon） | 500–1,500 tri | 3,000–8,000 tri |
| 场景大型建筑 | 2,000–5,000 tri | 10,000–30,000 tri |
| 场景小物件（Prop） | 100–500 tri | 500–2,000 tri |

上述数值并非固定不变，而是由目标平台的GPU性能、同屏角色数量和目标帧率（如30fps或60fps）共同决定。制定者需要先确定目标帧率对应的帧时间预算（Frame Budget），例如60fps对应16.6ms，再结合GPU厂商的理论顶点处理吞吐量反推可接受的总三角面数上限，最后按场景中各类资产的比例分配面数配额。

### 三角面数与顶点数的关系

面数标准中标注的单位通常是**三角面数（Tris/Triangles）**，而非四边面数（Quads/Polygons）。引擎在渲染前会把所有多边形拆分成三角形，因此1个四边面 = 2个三角面。一个3000四边面的模型导入引擎后实际会产生约6000个三角面，若不了解这一换算关系，美术人员会系统性地超出面数标准的约束。此外，由于UV缝合边（UV Seam）和硬边（Hard Edge/Smoothing Group）会导致顶点数膨胀，同样面数的模型在引擎中实际顶点数可能是DCC软件（如Maya、Blender）显示顶点数的1.5–3倍。

### LOD（多细节层次）与面数标准的配合

面数标准通常为同一资产规定多个LOD级别的面数上限，而非只规定LOD0（最高精度）。常见做法是：LOD0面数为N，LOD1约为N×50%，LOD2约为N×25%，LOD3约为N×10%。以次时代项目主角为例，LOD0可达60,000 tri，LOD1为30,000 tri，LOD2为15,000 tri，LOD3为6,000 tri，LOD4为1,500 tri（用于远景和Shadow Depth Pass）。LOD切换距离通常由摄像机距离或屏幕占比（Screen Size Ratio）触发，Unity和Unreal Engine均以屏幕高度占比百分比作为LOD切换阈值，这也是面数标准文档需要一并注明的参数。

## 实际应用

**移动端MMORPG场景搭建**：某移动端项目要求同屏稳定出现20个玩家角色，目标60fps，中端Android机型GPU顶点处理能力约为每帧1亿顶点。预留40%给场景后，角色总顶点预算约6000万。20个角色平均每个可用300万顶点，换算为三角面约为300万/3 ≈ 100万 tri，显然不现实，实际还需计入蒙皮骨骼顶点放大系数（约1.5倍）和阴影Pass（额外一倍渲染开销），最终主角面数被限制在3500 tri以内，这就是面数标准"逆向推算"的典型流程。

**次时代主机游戏角色审查**：项目面数标准规定近战武器LOD0上限为5,000 tri，某美术提交的剑模型为12,000 tri，其中大量面数集中于剑身的圆角倒角细节。技术美术审查时会用Unreal Engine的三角面统计工具（Triangle Count Visualization Mode）一键标红超标资产，要求美术人员将剑身圆角从8段减至2段，通过法线贴图（Normal Map）补偿视觉细节，最终将面数压缩至4,800 tri通过审查。

## 常见误区

**误区一：面数越低越好**
部分初级模型师误以为应尽量减少面数，实际上过低的面数会导致轮廓锯齿（Silhouette Jaggies）明显，蒙皮动画时出现严重的关节塌陷（Joint Collapse），这时反而需要在关节处补面。面数标准给出的是**上限（Maximum）**，而非强制要求使用所有配额。模型师应在上限内尽量还原外形的光滑轮廓。

**误区二：面数标准适用于所有视角和相机距离**
面数标准文档中规定的面数是针对特定摄像机距离或屏幕占比场景下的LOD0面数。若美术人员将远景小物件按LOD0面数标准建模，不为其制作LOD，则该物件在远处仍会消耗完整的高面数，造成性能浪费。面数标准必须与LOD策略一同执行。

**误区三：同一资产在所有平台的面数标准相同**
跨平台项目中，PC版和移动版往往需要两套独立的面数标准文档，甚至同一平台不同画质档位（低/中/高）也对应不同的面数上限。将PC版面数标准的数值直接套用到移动端，是导致移动端项目帧率不达标的常见原因之一。

## 知识关联

**前置知识**：学习面数标准需要先了解"美术标准概述"，掌握美术标准文档的整体框架与制定流程，才能理解面数标准作为其中定量性能约束文档的定位与作用范围。

**延伸至审查检查清单**：面数标准制定完成后，其核心数值会被直接提取为"审查检查清单（Review Checklist）"中的量化验收条目，例如"LOD0是否超过规定的X tri上限"、"是否提供了至少3级LOD"等，形成可操作的逐项核查机制。面数标准本身是数值规范，审查检查清单则是将其转化为可执行审查动作的操作手册。
