# 网格简化

## 概述

网格简化（Mesh Reduction/Mesh Decimation）是一种通过算法自动减少三角面数量，同时尽量保留原始几何形状视觉特征的技术。其核心目标是在给定的三角面预算限制下，以最小的视觉损失换取渲染性能提升。与手动建模减面不同，网格简化工具可在数秒内将一个拥有数十万面的高精度模型压缩至目标面数，成为现代游戏和实时渲染流水线中不可或缺的效率工具。

该技术的理论基础来自1997年Michael Garland和Paul Heckbert发表的论文《Surface Simplification Using Quadric Error Metrics》，其中提出的**二次误差度量（Quadric Error Metrics，QEM）**算法至今仍是Simplygon、虚幻引擎（Unreal Engine）内置Mesh Reduction等主流工具的核心算法。QEM通过计算每次合并顶点操作引入的几何误差，优先执行误差最小的边折叠操作，从而使减面过程在数学上最优化。

在现代游戏开发中，网格简化直接服务于LOD（Level of Detail）系统的生成。一个角色模型的LOD0可能拥有15,000个三角面，而经过网格简化后的LOD3只需保留约600个三角面，渲染开销降低约96%。合理运用网格简化工具，可以将原本需要数天手工制作LOD的工作量压缩至数分钟。

## 核心原理

### 二次误差度量（QEM）算法

QEM算法将每个顶点表示为一组平面方程的集合，每次执行**边折叠（Edge Collapse）**操作时，计算将边的两个端点合并为一个新顶点所引入的几何误差。误差值由公式 **Q(v) = vᵀAv** 计算，其中 `v` 是新顶点的齐次坐标向量，`A` 是由该顶点周围所有面的平面方程叠加得到的4×4对称矩阵。算法维护一个优先队列，每次从中选取误差值最小的边进行折叠，重复此过程直至达到目标面数。这一机制确保了平坦区域（误差小）被优先简化，而边缘、尖角等视觉关键区域（误差大）被保留。

### Simplygon的权重控制机制

Simplygon在QEM基础上引入了**几何重要性权重（Geometry Importance Weight）**和**材质通道权重（Material Channel Weight）**。通过在材质通道权重中提高法线贴图（Normal Map）权重（默认为1.0，建议高频细节区域调至2.0～4.0），可以指示算法在法线变化剧烈的区域（如面部、手指关节）保留更多面数。Simplygon还支持**顶点锁定（Vertex Lock）**功能，可将武器插槽、骨骼绑定点等关键顶点标记为不可合并，防止减面破坏骨骼蒙皮权重。

### 虚幻引擎内置Mesh Reduction

UE的静态网格编辑器和Skeletal Mesh Reduction工具基于Simplygon或内置的Quadric Mesh Reduction（QMR）后端运行。在LOD设置中，**屏幕尺寸阈值（Screen Size Threshold）**决定何时切换LOD等级，而**减面百分比（Percent Triangles）**和**最大偏差（Max Deviation）**是两种互斥的质量控制模式：前者直接指定目标面数比例（如LOD1 = 50%，LOD2 = 25%），后者则以世界单位中允许的最大几何偏移量作为终止条件，更适合对形变容忍度明确的硬表面模型。UE5中还引入了**Nanite**作为虚拟化几何体方案，但传统网格简化在移动端和不支持Nanite的平台上仍是主要手段。

## 实际应用

**角色模型的LOD链生成**：一个标准第三人称角色通常需要4个LOD等级。以10,000面的LOD0为基准，LOD1目标设为5,000面（50%），LOD2设为1,500面（15%），LOD3设为500面（5%）。在Simplygon中处理时，需单独为头部区域绘制**重要性图（Importance Map）**，赋予脸部权重值0.8～1.0，身体其余部分设为0.3，以确保面部识别度在远距离仍可接受。

**硬表面道具优化**：对于枪械、车辆等硬表面模型，QEM算法天然在平直面上表现良好，但在45°斜面的倒角（Chamfer）处易产生面扭曲。实践中常将最大偏差模式的阈值设为模型最小倒角宽度的20%（如倒角为0.5cm，则阈值设0.1cm），既能大幅减面又不破坏轮廓线。

**地形与建筑物件**：建筑外墙等大面积低曲率区域，使用50%面数目标往往能实现近乎无损的简化结果。而复杂镂空结构（如铁艺栏杆）需在减面前合并网格并烘焙遮罩到Opacity贴图，否则算法会错误保留大量构成孔洞轮廓的细碎三角面。

## 常见误区

**误区一：减面比例越保守，质量越好**
许多初学者倾向于将每级LOD设置为仅减少10%～20%的面数，认为这样更安全。实际上，QEM算法在减少80%面数时的效果通常远优于人们的直觉预期，尤其是对于低曲率区域。过于保守的LOD链会导致各级之间面数差距过小，无法有效降低GPU顶点处理压力，失去LOD系统的意义。建议先用减面50%的结果与原模型做A/B对比再决定实际目标。

**误区二：网格简化可以替代法线贴图烘焙**
部分流程会直接对高模运行网格简化生成游戏用模型，跳过烘焙步骤。然而QEM算法仅保留几何形状，无法将高频表面细节（如螺丝纹、布料织纹）转换为法线贴图信息。正确流程应是先从高模烘焙法线贴图到低模，再对低模执行网格简化生成LOD链，两者解决的是不同层级的细节问题。

**误区三：骨骼网格直接套用静态网格的减面参数**
骨骼网格（Skeletal Mesh）在执行网格简化时，必须同时考虑蒙皮权重的保真度。若不启用**骨骼权重保留（Bone Vertex Weight Preservation）**选项，算法在折叠关节区域的边时会将不同骨骼影响的顶点合并，导致动画播放时出现"橡皮筋"形变瑕疵。这是骨骼网格减面区别于静态网格的关键操作点。

## 知识关联

网格简化的应用以**三角面预算**为前提——只有明确了每个物件在不同LOD等级下的面数上限，才能为减面工具设置有意义的目标参数。三角面预算给出"要减到多少面"的约束，网格简化工具负责"如何高质量地减到该数字"。

掌握网格简化后，下一个核心概念是**代理网格（Proxy Mesh）**。代理网格是网格简化思路的延伸：当模型简化比例极高（如减至原始面数的1%以下）时，QEM算法无法保持任何有意义的几何形状，此时需要使用代理网格技术——将场景中多个物件烘焙合并为一个几何形状完全重建、贴图合并的单一低模，适用于远景建筑群和密集植被等场景。
