# 网格简化

## 概述

网格简化（Mesh Reduction）是指通过算法自动删除三角面、合并顶点，在保持模型视觉轮廓基本不变的前提下降低多边形数量的技术。其核心目标是用尽可能少的三角面传达尽可能多的形状信息，从而将运行时的顶点着色器调用次数和光栅化负担压缩到预设的三角面预算以内。

工业级网格简化算法的理论基础奠定于1997年——Garland与Heckbert在SIGGRAPH发表的论文《Surface Simplification Using Quadric Error Metrics》提出了**二次误差度量（QEM）**方法，至今仍是Simplygon、UE内置Mesh Reduction等工具的核心数学框架。该算法为每个顶点维护一个4×4的误差矩阵Q，通过最小化顶点合并时的几何误差来决定每次折叠（Edge Collapse）操作的优先顺序。

在实际生产中，一个高精度角色模型可能包含80,000至200,000个三角面，而移动端的单角色预算往往只有3,000至8,000个三角面。手动拓扑重建每一个LOD层级既耗时又难以保持风格一致性，网格简化工具使美术师能够在数秒内生成多套LOD，并通过参数调节在减面率与视觉质量之间取得平衡。

## 核心原理

### 二次误差度量（QEM）与边折叠

QEM算法将每次减面操作表述为**边折叠（Edge Collapse）**：将一条边的两个端点 v₁、v₂ 合并为新顶点 v̄，并删除与该边共享的三角面。新顶点位置由以下公式最小化误差得出：

$$\bar{v} = \arg\min_{v}\ v^T (Q_1 + Q_2)\ v$$

其中 Q₁、Q₂ 分别是两端点的二次误差矩阵，v 为齐次坐标（4维列向量）。算法维护一个最小堆，每次选取误差增量最小的边执行折叠，直到达到目标面数。这一机制保证了平坦区域（误差矩阵行列式接近零）会优先被简化，而高曲率棱角区域则被优先保留。

### Simplygon的权重通道与锁定顶点

Simplygon在标准QEM之上引入了若干工程化扩展。**材质边界权重（Material Border Weight）**可以将不同材质交界处的边折叠代价人为提高100倍以上，防止UV接缝在减面后产生跳变。**骨骼权重（Bone Weight）**保护则确保蒙皮网格在减面后仍能被正确驱动——若两顶点的骨骼影响列表不同，其合并代价会显著提升。此外，美术师可以将关键轮廓顶点标记为**锁定（Locked）**，使其完全不参与折叠，这对于武器配件的安装孔等结构性细节至关重要。

### UE引擎内置的Mesh Reduction参数

在Unreal Engine中，Static Mesh编辑器的LOD设置面板提供了以下关键参数：

- **Percent Triangles（三角面百分比）**：直接指定目标面数占原始面数的比例，例如LOD1设为50%、LOD2设为25%、LOD3设为10%是常见的四级LOD策略。
- **Max Deviation（最大偏差）**：以世界单位表示允许的最大几何偏移量，替代百分比目标，让引擎自动决定减面幅度。
- **Silhouette/Texture/Shading重要性**：分别控制轮廓保持、UV拉伸容忍和法线连续性三个维度的权重，取值为Off/Lowest/Low/Normal/High/Highest六档。
- **Welding Threshold**：合并距离小于该阈值（默认0.1 cm）的顶点，在减面前预处理裂缝。

## 实际应用

### 植被与场景道具的批量LOD生成

游戏场景中数量最多的通常是树木、草丛、石块等道具，它们可能出现数千个实例。以一棵树为例：原始模型包含约15,000个三角面，通过Simplygon的脚本接口批量生成LOD0（100%）、LOD1（40%）、LOD2（15%）、LOD3（Billboard替换）四级，整个批处理流程可在构建管线中自动执行，无需美术师逐一操作。Simplygon提供命令行工具`simplygon.exe`及Python/C# SDK，方便接入CI/CD流程。

### 角色装备的分层简化策略

对于带有法线贴图烘焙信息的角色装备，直接使用激进的减面率（如5%）会导致低多边形网格与高精度法线贴图之间产生明显的轮廓失配。常见做法是：LOD0保留100%面数，LOD1减至60%并重新烘焙法线，LOD2减至25%再次烘焙，确保每一级LOD的网格轮廓与其配套法线贴图相匹配。Simplygon的**Remeshing + Baking**流程可以将这两步合并为单次操作。

### 移动端资产的硬性面数控制

在移动平台（如iOS/Android）上，由于GPU的顶点处理吞吐量有限，通常要求屏幕占比30%以上的角色LOD0不超过5,000个三角面。使用UE的`Max Deviation`模式设置为2 cm，可以让引擎在2 cm误差容忍内尽可能多地减面，适合快速测试各资产在紧束约束下的外观底线。

## 常见误区

### 误区一：减面百分比越高，视觉质量损失一定越大

许多初学者认为减至原始面数的20%必然比减至50%效果更差。实际情况取决于原始网格的拓扑质量。一个面数虚高、内部充斥大量平坦细分面的低信息密度网格，减至20%后轮廓变化可能微乎其微；而一个拓扑精简、每个三角面都承载曲率信息的高质量网格，减至50%就可能出现明显的轮廓锯齿。因此应当以**视觉误差**（如Max Deviation）作为质量衡量标准，而非单纯看百分比数字。

### 误区二：自动减面可以完全替代手动LOD建模

自动减面在处理**非均匀曲率网格**（如人脸）时，算法无法理解"眼眶轮廓比脸颊平面更重要"这类语义信息。QEM只计算几何误差，不识别功能性区域。对于近景主角面部，手动制作LOD1（约5,000面）仍是质量上限更高的选择，自动减面更适合中远景或道具类资产。

### 误区三：减面后UV无需重新检查

边折叠操作会合并UV坐标，当两个顶点的UV位置相差较大时，合并后产生的UV拉伸会导致贴图在该三角面上产生明显的方向扭曲。必须在减面完成后使用**Texel Density检查工具**或UE的UV Overlay视图验证每级LOD的UV分布，必要时调高`Material Border Weight`或手动修复拉伸面。

## 知识关联

网格简化的前置概念是**三角面预算**：只有明确了每个平台、每种资产类型的面数上限（如主机平台主角角色LOD0上限80,000面），才能知道需要将原始模型简化到何种程度，以及需要生成几级LOD。没有预算约束，减面参数的设置将缺乏方向。

网格简化之后的进阶方向是**代理网格（Proxy Mesh）**。当模型距离摄像机非常远（例如超过100米），即使最低LOD的三角面数量仍显得过多，此时需要通过Simplygon的Remeshing功能或UE的Proxy Geometry Tool将整个建筑群、植被丛等多个独立网格**合并烘焙**成一个极低面数的代理网格，同时生成合批的Impostor贴图。代理网格与网格简化的关键区别在于：网格简化保持原始拓扑的连通性，而代理网格允许彻底重构拓扑，适用于更远的渲染距离和更激进的面数压缩需求。
