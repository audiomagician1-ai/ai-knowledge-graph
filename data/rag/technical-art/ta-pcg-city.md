# 程序化城市

## 概述

程序化城市（Procedural City Generation）是通过算法自动生成完整城市环境的技术体系，涵盖街道网格划分、地块分配、建筑立面批量生成、交通路网规划以及城市语义分区等多个层级。与单栋程序化建筑相比，程序化城市必须处理严格的层级依赖链：街道多边形决定地块边界形状，地块形状约束建筑底面轮廓，建筑密度反馈影响交通节点权重，交通热力图又可反向修正街道优先级。这种自顶向下、局部反馈的生成链条，使城市生成成为程序化内容生成（PCG）领域中系统耦合度最高的分支。

该技术的奠基性文献是Parish与Müller于2001年在SIGGRAPH发表的论文《Procedural Modeling of Cities》，该论文首次将扩展L-System用于道路网络生成，提出以"全局目标（Global Goals）+ 局部约束（Local Constraints）"双重机制驱动街道代理扩张的框架。这篇论文直接影响了此后几乎所有城市生成工具的架构，包括Esri CityEngine（2008年商业发布）和Houdini 19.0引入的GameDev城市工具集。在影视和游戏工业中，《蜘蛛侠：英雄远征》（2019）使用程序化辅助流程将纽约街景资产量压缩了约60%的人工建模工时，《看门狗：军团》（2020）的伦敦城市原型在程序化框架下完成了超过6,800栋建筑立面的批量迭代。

## 核心原理

### 街道网格生成

街道网络生成的主流方法分为两大类：**基于代理的L-System扩展**和**张量场积分法**。

Parish-Müller的原始框架中，道路代理以迭代步长（默认单步延伸约30–50米虚拟单位）向外扩张，每步通过查询人口密度图（Population Density Map，通常以灰度贴图编码，白色代表高密度）和海拔高度图来决定道路是否延伸、分叉或终止。道路扩展优先级评分如下：

$$P_{road} = \alpha \cdot D_{pop} + \beta \cdot F_{terrain} + \gamma \cdot A_{highway}$$

其中 $D_{pop}$ 为目标方向的人口密度归一化值（0–1），$F_{terrain}$ 为地形平坦度（坡度低于5°时为1，超过15°降至0），$A_{highway}$ 为与最近主干道的对齐奖励（夹角小于10°时激活）。通过调节 $\alpha$、$\beta$、$\gamma$ 三组权重，可以生成典型的曼哈顿正交网格（$\alpha$ 主导）、巴黎放射状路网（$\gamma$ 主导）或中世纪欧洲有机蔓延路网（$\beta$ 主导，坡地抑制效果显著）。

张量场方法由Chen等人于2008年在SIGGRAPH Asia提出（Chen, Esch, Wonka et al., 2008），将主次干道方向编码为二阶对称张量场，通过对张量场进行数值积分生成连续流线，再在流线上采样道路节点。该方法相比L-System更易生成曲线路网并自然贴合河流、山丘等地形要素，是Houdini GameDev Street Generator工具的底层算法之一。

### 地块分割（Lot Subdivision）

街道多边形围合区域通过递归分割算法拆分为可建设单元。最常用的是**OBB分割法（Oriented Bounding Box Subdivision）**：

1. 计算地块最小包围矩形（OBB），取其长轴作为切割方向；
2. 沿垂直于长轴的方向投影，按设定的地块宽度步长（通常10–20米）切割；
3. 递归执行直到子地块面积低于最小建设阈值（住宅区通常为200–400 m²，商业区可放宽至800 m²）；
4. 对面积不足阈值的残余多边形合并至相邻地块或标注为绿化保留区。

分割过程需处理三类特殊情形：①不规则多边形地块（切割线必须修正为与街道边平行，否则建筑底面出现非正交边导致后续规则文法失效）；②坡地地块（需对顶点高度双线性插值，确保建筑底面为水平面）；③受保护地块（公园、广场、历史遗址等，直接标记跳过分割流程）。Müller等人在CityEngine的配套论文《Procedural Modeling of Buildings》（2006, SIGGRAPH）中详细描述了这一流程的完整实现。

以下为Houdini VEX中简化的地块递归分割逻辑示意：

```vex
// 伪代码：OBB递归地块分割
function splitLot(polygon lot, float minArea) {
    if (area(lot) < minArea) {
        markAsBuildable(lot);
        return;
    }
    vector2 axis = longestAxis(OBB(lot));
    float splitPos = randomInRange(0.35, 0.65); // 避免过于均等
    polygon[2] halves = splitAlongAxis(lot, axis, splitPos);
    foreach (polygon half : halves) {
        splitLot(half, minArea);
    }
}
```

### 建筑风格分配与立面生成

程序化城市中的建筑通过**城市语义分区（Zoning）**批量绑定风格规则集，而非逐栋手工指定。典型的分区类型包括：住宅低密度区（R-Low）、住宅高密度区（R-High）、商业中心区（CBD）、轻工业区（Industrial）、历史保护区（Heritage）。每类分区绑定一套独立的Shape Grammar规则集，规则集中以硬性参数范围取代随机值：

- **CBD区**：层高3.8–4.5米，窗墙比0.55–0.75，立面材质强制选用玻璃幕墙贴图集；
- **住宅区（R-Low）**：层高2.8–3.2米，窗墙比0.25–0.40，坡屋顶坡度15°–35°；
- **历史区**：层高3.0–3.6米，窗墙比0.30–0.50，强制引用砖石纹理库，楼层间必须插入腰线细节节点。

Esri CityEngine使用的CGA语言（Computer Generated Architecture）是目前工业界最成熟的建筑批量生成描述语言，其规则文件可在城市尺度上（数千栋）并行执行，且每栋建筑的随机种子独立，保证视觉多样性不重复。

## 关键算法：街道与地块的完整生成流程

以Parish-Müller框架为基准，程序化城市的完整生成管线可分为以下五个有序步骤：

**Step 1 — 输入图层准备**：加载人口密度图（resolution建议256×256以上）、高程图、水体遮罩、分区语义图（RGB通道分别编码分区类型）。

**Step 2 — 主干道生成**：运行高优先级道路代理，步长50米，分叉角度限定在±15°以内，生成城市骨架路网（通常仅数十条主干道）。

**Step 3 — 次级道路填充**：在主干道围合的超级街区（Superblock）内，以20米步长运行次级代理，生成垂直或平行于主干道的街道网格，最终产生面积0.5–2公顷的街区多边形。

**Step 4 — 递归地块分割**：对每个街区多边形执行OBB分割，最终地块数量在大型城市场景中可达数万块。

**Step 5 — 建筑规则执行**：依据地块中心坐标查询分区图，绑定对应Shape Grammar规则集，逐地块并行生成建筑几何体与UV。

整条管线在Houdini中的CPU执行时间，对于1km²城市范围（约4,000块地块）通常在30–90秒内完成，使用PDG（Parallel Dependency Graph）并行加速可缩短至10–20秒。

## 实际应用

**游戏开发——城市原型快速迭代**：育碧在《看门狗：军团》开发初期使用程序化城市框架生成伦敦多个候选区域布局，设计师只需修改分区语义图和密度参数即可在数分钟内得到新版城市骨架，相比传统手工白盒阶段节省了估计2–3个月的概念迭代时间。

**影视VFX——背景城市填充**：在需要航拍视角的影视镜头中，远景城市往往只需程序化几何体+低精度纹理即可满足需求。索尼影视在《蜘蛛侠：英雄远征》中将程序化生成的街区用于视距300米以外的背景填充，仅在镜头焦点区域保留手工精模，整体背景建筑资产量减少了约60%的人工工时。

**城市规划可视化**：Esri CityEngine被多个城市规划机构用于建设方案快速可视化，例如苏黎世市政府曾使用CityEngine将GIS地块数据直接驱动三维建筑生成，在数小时内完成全市三维底图构建，该案例收录于Esri官方技术白皮书（Esri, 2013）。

**例如**：在Houdini中生成一个1km²的现代商业城市，典型参数配置为：主干道间距约200米，次级道路间距约60米，CBD区地块宽度10–15米，建筑高度40–120米随机，窗墙比0.6。运行完整管线后得到约3,200块地块、约2,800栋建筑，总面数约800万三角面，适合LOD 0级别的实时渲染。

## 常见误区

**误区一：街道网格生成后直接连接端点即可**。实际上，道路代理扩展结束后必须执行拓扑清理（Topology Cleanup）：合并距离小于3米的悬空端点（Dead End Snap），修复T形交叉口的道路中断，删除面积小于20 m²的退化多边形地块。跳过这一步会导致后续OBB分割产生大量无效地块，建筑规则文法报错率可高达30%以上。

**误区二：所有地块均使用统一最小面积阈值**。不同分区的合理地块面积差异极大：历史街区的联排住宅地块可小至60 m²，而工业厂房地块通常需要2,000 m²以上。使用统一阈值会导致商业区出现大量过于细碎的地块（建筑底面宽度不足5米），或工业区地块过大无法分割为独立建筑单元。

**误区三：建筑高度与分区密度直接线性正相关**。实际城市规划中，容积率（FAR, Floor Area Ratio）才是控制建筑总量的核心指标，而非单纯楼层高度。CBD区FAR通常为8–15，住宅区为1.5–3.0。程序化城市应将FAR作为约束输入，由算法反算允许的建筑高度上限，而非直接随机化楼层数。

**误区四：程序化城市可以完全替代手工美术**。程序化管线生成的是具备合理结构的城市骨架，但标志性建筑（地标）、特殊地形处理、叙事相关场景（游戏任务区域）仍需手工制作并以实例化方式注入城市场景，两者分工明确，不存在完全替代关系。

## 知识关联

**前置概念——程序化建筑**：单栋建筑的Shape Grammar规则集是程序化城市中立面批量生成的最小执行单元，掌握CGA语言或Houdini建筑规则节点是理解城市生成管线Step 5的必要前提。具体而言，理解Scope（作用域）、Split（分割）、Comp（组件提取）三个核心操作才能读懂城市级规则文件。

**相关领域——L-System与图形文法**：Parish-Müller框架的道路生成部分本质上是参数化L-System的图形版本，理解L-System的字符串重写规则有助于定制非标准路网拓扑（如地下城隧道网络或科幻城市的垂直立交结构）。

**扩展工具链**：Houdini GameDev Tools（免费插件）提供了开箱即用的街道生成（Street Generator SOP）和建筑填充（Building Generator SOP）节点，配合PDG可实现城市场景的批量变体导出，是目前游戏行业最主流的程
