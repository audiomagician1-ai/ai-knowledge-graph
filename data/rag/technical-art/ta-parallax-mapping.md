# 视差贴图

## 概述

视差贴图（Parallax Occlusion Mapping，简称 POM）是一种在平坦多边形表面上模拟真实深度凹凸感的渲染技术。它通过在片元着色器中对纹理坐标进行偏移，让观察者感受到表面存在实际高度差，而无需真正增加几何体的顶点数量。相比法线贴图只改变光照计算方向，视差贴图还会让贴图的采样位置随视角变化而位移，产生视差效果——这正是"Parallax"一词的由来。

视差贴图技术的演进可追溯到2001年 Kaneko 等人提出的基础视差映射（Simple Parallax Mapping），随后2005年 Brawley 和 Tatarchuk 在 ATI 的研究中完整提出了 Parallax Occlusion Mapping，引入光线步进（Ray Marching）方法解决了早期版本无法处理自遮挡的缺陷。这一技术在第七世代主机（Xbox 360、PS3 时代）被大量商业游戏采用，至今仍是实时渲染中性价比最高的深度模拟方案之一。

POM 的重要性在于它填补了法线贴图与真实位移贴图之间的空白地带。法线贴图无法产生视差偏移，而真实几何位移（Displacement Mapping）需要曲面细分支持，性能代价昂贵。POM 仅依赖一张高度图（Height Map）和片元着色器计算，即可在几乎不增加几何复杂度的前提下，让石砖、砖墙、皮革纹路等材质呈现出令人信服的立体深度。

## 核心原理

### 高度图与纹理坐标偏移

视差贴图的基础数据来源是一张灰度高度图，白色（值为1.0）代表表面最高点，黑色（值为0.0）代表最深凹陷。着色器首先读取当前像素位置的高度值 `h`，然后根据视线向量 `V`（在切线空间下）计算出纹理坐标的偏移量：

```
offset = (V.xy / V.z) * h * heightScale
UV_new = UV_original - offset
```

其中 `heightScale` 是控制深度强度的缩放系数，通常取 0.02 到 0.1 之间的值。`V.z` 除法确保视线越倾斜偏移越大，这正是现实中视差效果的物理依据。这种最基础的版本称为 Simple Parallax Mapping，计算仅需一次额外纹理采样。

### 陡峭视差映射与二分法求交

Simple Parallax Mapping 在表面高度变化剧烈时会出现明显的滑动失真。陡峭视差映射（Steep Parallax Mapping）改为沿视线方向进行多步离散采样，将深度区间等分为 N 层（通常 8 到 32 层），逐层检测视线与高度场的交点：

每一步将当前深度增加 `1.0/N`，采样对应 UV 处的高度图值；当检测到高度图值首次超过当前追踪深度时，即认为视线在此处与表面相交。层数越多，精度越高，但每层都是一次独立的纹理采样，性能代价随之线性增加。

完整的 POM 在陡峭映射找到粗略交点后，再用**二分法（Binary Search）**或**线性插值**在相邻两层之间精确求交，通常额外执行 4 到 8 次迭代，将最终误差控制在亚像素级别。这个两阶段策略让 POM 在 16 步主循环+6 步二分法的总计约22次采样内，达到媲美数百层暴力步进的精度。

### 自遮挡阴影（POM Shadows）

POM 的一个独特功能是可以计算高度场内部的自遮挡软阴影，称为 POM Soft Shadows。在确定最终着色 UV 后，沿光线方向（而非视线方向）再执行一次光线步进，统计有多少步中高度图高于光线追踪高度，以此计算遮挡比例：

```
shadow_factor = 1.0 - (occluded_steps / total_steps) * shadowStrength
```

这条阴影是纯粹在着色器内部"虚构"的——几何体完全是平面，但渲染结果中砖缝深处会出现合理的接触阴影，极大增强了立体感。

## 实际应用

**石砖与瓦片墙壁**：这是 POM 最经典的使用场景。砖缝的深度凹陷（`heightScale` 约 0.05）配合自遮挡阴影，让大面积重复墙壁材质即便在近距离镜头下仍显真实，而实际多边形可能仅有两个三角面。《巫师3》《Control》等写实游戏的环境材质中均大量使用了此类技术。

**皮革与织物纹路**：皮革表面的细小凸起和编织布料的交叉纹理需要微小的高度差（`heightScale` 0.01~0.02）来呈现材质质感。此时 POM 的层数可以降低到 8 层以节省性能，因为高度变化平缓不易产生步进失真。

**视角限制与淡出处理**：POM 在掠射角（视线与表面几乎平行）时会产生严重的纹理拉伸错误，因此实践中通常在 `V.z < 0.2` 时通过混合因子将效果淡出至普通法线贴图，避免瑕疵暴露。Unreal Engine 的 POM 材质节点内置了这一淡出逻辑，对应参数为 `Min Steps` 和 `Max Steps` 随 `V.z` 动态变化。

**地形细节层**：部分引擎将 POM 用于地形的近景细节叠加层，在摄像机 0~5 米范围内启用 POM 增强石头与泥土的近景质感，超出距离后自动退化为法线贴图，实现性能与视觉的动态平衡。

## 常见误区

**误区一：视差贴图等同于位移贴图**
位移贴图（Displacement Mapping）真实移动顶点位置，产生的几何体剪影会发生变化，物体边缘可以看到实际的凸起轮廓。POM 完全是着色器内的光学欺骗，物体轮廓依然是原始平面，在侧面看会穿帮。因此 POM 只适合用于中远景或不会被从侧面观察的表面，不能替代需要真实几何变形的场景。

**误区二：层数越多效果越好，应该直接用64层**
层数提升确实改善精度，但对于平缓的高度场（如皮革纹路），8层与64层视觉差异微乎其微，而片元着色器的采样次数是直接影响GPU带宽和ALU占用的关键指标。在移动端GPU上，POM 的每一次额外采样代价尤为突出。标准做法是根据 `V.z`（视线倾斜程度）动态计算层数：`numLayers = mix(maxLayers, minLayers, abs(V.z))`，使正对表面时用最少层数，掠射角时用最多层数。

**误区三：POM 可以正确处理遮挡剔除和投影阴影**
POM 仅改变像素着色结果，对深度缓冲（Depth Buffer）没有影响（除非显式使用 POM Depth Reconstruction 技术写回深度）。这意味着 POM 模拟出的深度凹陷不会影响屏幕空间环境光遮蔽（SSAO）的计算，也不会在其他物体上投射正确的阴影。这是 POM 相对于真实位移贴图最根本的局限性。

## 知识关联

**前置知识——PBR材质基础**：POM 通常与 PBR 材质工作流配合使用。高度图所偏移的最终 UV 会同时被用于采样 BaseColor、Roughness、Metalness、Normal 等所有 PBR 贴图通道，因此理解 PBR 贴图通道的作用是正确配置 POM 材质的前提。错误地仅对部分通道应用偏移后的 UV 会导致各通道之间的视差不一致，产生"层分离"的视觉错误。

**横向关联——法线贴图**：法线贴图在切线空间中修改光照计算法线，POM 则在切线空间中计算视线方向和光线步进——两者共用同一套切线空间（TBN矩阵）。POM 材质必然同时包含一张法线贴图，法线贴图负责微表面光照细节，POM 负责宏观视差偏移，两者相辅相成共同构建完整的表面深度感。

**技术演进方向**：POM 之后更进一步的技术是**视差遮挡映射与真实深度重建（POM + Depth Reprojection）**，通过将计算所得的虚拟深度写回深度缓冲，使 SSAO、屏幕空间反射（SSR）等后处理效果能够正确感知 POM 制造的虚假几何，进一步消弥 POM 的穿帮场景。
