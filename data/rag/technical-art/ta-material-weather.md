# 天气效果材质

## 概述

天气效果材质（Weather Effect Material）是一类专门用于模拟雨水、积雪、湿润、结冰等动态气候条件对物体表面视觉属性影响的着色器系统。它的核心挑战在于：真实世界中一场大雨会同时改变地面的反射率、粗糙度、颜色饱和度和法线细节，这些变化彼此耦合，不能独立处理。天气效果材质的职责就是通过参数驱动的方式，在运行时动态重建这种多通道联动的物理变化。

这类技术在电子游戏中大约从2013年前后的次世代主机时代开始成熟，《The Last of Us》和《Killzone Shadow Fall》中可见早期完整实现。其核心推动力来自基于物理渲染（PBR）管线的普及——PBR明确将材质拆分为基础色、金属度、粗糙度、法线等独立通道，使雨雪对每个通道的独立修改变得可量化、可控制。在此之前，旧式Phong着色器缺乏对这些通道的独立访问能力，天气模拟往往只能靠替换整张贴图来粗暴实现。

天气效果材质的重要性体现在世界沉浸感的即时性上：玩家视野内往往有数百个不同材质的物体，如果雨天时只有地面变湿而木箱、石墙、角色服装保持干燥状态，场景的可信度会立刻崩塌。因此一套统一的天气材质驱动系统比单个物体的精细着色更具工程价值。

## 核心原理

### 湿润效果的物理参数映射

湿润状态改变材质的机制在物理上非常明确：水膜填充了表面微观凹槽，使微表面变得更平滑，同时水的折射率（约1.33）高于空气，导致菲涅耳反射峰值上升。在着色器中，这对应两个直接修改：

- **粗糙度降低**：干燥混凝土的粗糙度约为0.85，完全浸湿后可降至0.15～0.25，差值幅度约0.6
- **基础色变暗**：水膜的多次内反射使光线能量在到达相机前被额外散射，深色材质尤为明显，亮度可降低约15%～30%

实现时通常引入一个标量参数 `WetnessAmount`（范围0.0～1.0），用lerp函数分别插值干湿两组粗糙度和基础色：

```
FinalRoughness = lerp(DryRoughness, WetRoughness, WetnessAmount)
FinalAlbedo    = lerp(DryAlbedo, DryAlbedo * 0.75, WetnessAmount)
```

金属表面的湿润响应与非金属不同——金属几乎不吸水，因此可通过 `Metalness` 通道遮蔽 `WetnessAmount` 的影响，避免铁板在雨中出现不合理的哑光效果。

### 积雪遮蔽与世界空间投影

积雪只沉积在朝上的水平或近水平表面，这一行为必须通过顶点/像素法线的世界空间Y分量来判断。常见公式为：

```
SnowMask = saturate((WorldNormal.y - SnowAngleThreshold) / (1.0 - SnowAngleThreshold))
```

其中 `SnowAngleThreshold` 通常设为0.5（对应约60度倾斜角以上不积雪）。`SnowMask` 随后与全局积雪深度参数 `SnowCoverage` 相乘，再通过lerp混入积雪的高反射率白色基础色（典型值约0.85）和接近0的粗糙度（约0.1，模拟压实的细粉末）。

积雪材质还需要添加细节噪波来打破表面均匀感。常用方案是用Voronoi噪波调制积雪法线，模拟雪粒的微观颗粒感，噪波强度随 `SnowCoverage` 减小时降低，因为薄雪层更平滑。

### 雨滴动态涟漪法线

静态湿润仅改变粗糙度和反射率，雨天的完整表现还需要雨滴打击水面产生的涟漪动画。涟漪法线动画通常通过在一张法线贴图上以不同速度UV滚动多层同心波纹来近似：两层同心圆法线贴图以时间偏移量 `Time * 0.4` 和 `Time * 0.7` 分别滚动，叠加后产生随机感。

更高质量的实现（如《荒野大镖客2》中记录的技术）使用Flipbook动画，预烘焙16～32帧的波纹扩散序列，在材质中通过帧索引采样，能精确还原圆形波峰扩散并在边缘消散的过程。涟漪法线的影响区域同样受 `WetnessAmount` 控制，干燥表面不显示任何涟漪。

## 实际应用

### 统一的全局天气参数系统

在大型开放世界游戏中，天气效果材质通常不由每个物体独立控制，而是从一个全局材质参数集合（Material Parameter Collection，Unreal Engine中简称MPC）读取统一数值。场景中的 `WeatherSystem` 蓝图负责根据当前天气状态（晴天/阴天/小雨/暴雨）插值修改MPC中的 `GlobalWetness`、`SnowDepth`、`IceAmount` 等参数，所有订阅了该MPC的材质实例同步响应变化，无需逐一驱动每个物体。

### 材质分层的积雪混合

角色服装的积雪效果需要与其基础布料材质进行物理合理的混合。以UE5材质函数为例，积雪层作为一个独立的材质函数节点插入基础角色材质图层之上，仅当 `SnowMask * SnowCoverage > 0.05` 时才有效混合，避免在完全无雪时产生额外性能开销。布料的纤维法线细节在积雪层厚度超过0.6时被完全覆盖，因为厚雪层会物理遮盖织物纹理。

### 水坑的镜面反射优化

地面水坑的高镜面反射需要特殊处理：直接使用SSR（屏幕空间反射）成本高昂，常见优化是在水坑区域切换为平面反射（Planar Reflection），并通过 `WetnessAmount` 参数在0.7以上时才激活该渲染路径，低湿度时使用IBL（Image-Based Lighting）近似替代。

## 常见误区

### 误区一：对所有材质统一应用相同的湿润响应

金属、织物、石材、皮肤对水的物理响应截然不同。金属表面几乎不改变颜色但镜面反射增强；织物吸水后颜色变深且粗糙度不降反升（因为纤维吸水膨胀）；皮肤湿润后次表面散射增强但高光变细。将同一套粗糙度降低曲线应用到所有材质类型会导致皮革变成塑料、毛巾变成铜板的错误视觉。正确做法是为不同材质类型定义独立的湿润响应曲线，并通过材质ID遮罩选择性应用。

### 误区二：积雪遮罩仅依赖法线方向，忽视几何遮蔽

使用世界空间法线Y分量判断积雪沉积是必要但不充分的条件。凹入几何体内部的朝上表面（如椅子坐垫与靠背的夹角底部）在现实中因几何遮蔽不会积雪，但法线方向完全朝上，单纯基于法线的计算会在此处错误显示积雪。正确实现需要结合AO（环境光遮蔽）贴图对积雪遮罩进行压制：`FinalSnowMask = SnowMask * pow(AO, 2.0)`，使遮蔽区域积雪密度显著降低。

### 误区三：天气参数突变而非渐进过渡

将全局天气参数从0直接跳变到1会使场景中所有物体同帧变湿，产生明显的画面跳变感。现实中雨水打湿地面需要数秒到数十秒的渐进过程，且不同材质的吸水速度也不同（沥青比混凝土变湿更快，孔隙较多的砂岩比瓷砖慢）。实现时应为每类材质定义独立的湿润速率 `WetnessRate`，全局参数通过 `DeltaTime * WetnessRate` 插值逐帧逼近目标值，而非直接赋值。

## 知识关联

天气效果材质直接建立在**材质混合技术**的基础之上：积雪层与基础材质的叠加是材质分层混合的典型应用，`WetnessAmount` 驱动的lerp插值是参数化材质混合的直接延伸。如果不理解如何通过遮罩控制不同材质通道的权重，积雪的世界空间投影和湿润的多通道联动将无法正确实现。
