# 混合基础

## 概述

动画混合（Animation Blending）是指在同一时间内对两个或多个动画片段按指定权重进行插值计算，从而输出一个新的、融合了各动画特征的中间姿态。最简单的情形是两个动画之间的线性权重插值：若动画 A 的权重为 `w`，动画 B 的权重为 `1 - w`，则最终每块骨骼的旋转和位移由这两个权重加权平均得出。当 `w = 1` 时完全播放动画 A，当 `w = 0` 时完全播放动画 B，当 `w = 0.5` 时两个动画各贡献一半姿态。

混合基础的思想最早在迪士尼1980年代的传统动画流水线中以"中间帧"手绘方式体现，进入计算机动画时代后，Alias PowerAnimator（Maya 的前身）在1990年代率先将骨骼权重混合写入工具链。现代游戏引擎（如 Unreal Engine 和 Unity）将其标准化为动画图（Animation Graph）中的基础节点，使每位动画师都能在不修改源动画的前提下通过调整单一浮点参数实时控制角色姿态。

理解混合基础的意义在于：几乎所有高级动画混合技术——包括 BlendSpace、叠加动画、分层混合——都以两动画插值这一最简单形式为数学出发点。掌握权重如何在骨骼层级传递，才能正确预判更复杂系统的运行结果。

## 核心原理

### 权重插值公式

两个动画片段 A 和 B 在某一时刻的混合姿态，由以下公式计算每根骨骼的变换：

```
Pose_out = Lerp(Pose_A, Pose_B, α)
         = Pose_A × (1 - α) + Pose_B × α
```

其中 `α ∈ [0, 1]` 称为**混合权重**，Pose 代表骨骼的位置（Position）、旋转（Rotation）和缩放（Scale）三元组。旋转通常以四元数（Quaternion）表示，因此插值函数应使用球面线性插值 **Slerp** 而非普通 Lerp，以避免万向节死锁并保证旋转路径最短：

```
Rotation_out = Slerp(Rot_A, Rot_B, α)
```

这是混合基础与纯数值插值最重要的区别：位置可以用线性插值，旋转必须用 Slerp。

### 归一化约束

两个动画混合时，所有参与动画的权重之和必须等于 1，即 `w_A + w_B = 1`。违反此约束会导致角色骨骼"缩水"或"膨胀"——例如两段动画权重均为 1 时，关节位移会翻倍。引擎内部通常在混合节点出口处自动执行归一化，但在手动编写混合逻辑时必须主动保证这一约束。

### 时间轴同步

混合两个动画时，若二者播放进度（Normalized Time）不对齐，中间帧会出现明显的"鬼影"抖动。解决方案是**同步播放**：以某一动画为主时钟，按比例拉伸另一动画的播放速率，使两者在关键姿态帧（如落脚帧）上对齐。Unreal Engine 中称此机制为 **Sync Group**，Unity 中称为 **Avatar Mask + Motion Time**。对于步行（约 1.0 m/s）和跑步（约 4.0 m/s）之间的混合，时间轴不同步会导致两脚落地节奏错乱，是初学者最常遭遇的视觉问题。

## 实际应用

**步行到慢跑的速度过渡**是最直观的混合基础示例。角色移动速度从 0 线性增加到 5 m/s，将速度映射为混合权重 `α = speed / 5`，以步行动画为 A、慢跑动画为 B。当 speed = 2.5 m/s 时，`α = 0.5`，角色呈现步幅和频率均居中的过渡姿态。

**受伤状态叠加**同样使用混合基础：用一个固定权重（如 `α = 0.4`）将"右腿受伤跛行"动画叠加到正常步行动画上，无需美术重新制作所有运动状态的受伤版本，节省了约 60% 的动画制作量（根据育碧在 GDC 2014 分享的流程数据）。

**武器切换过渡**中，在角色收枪（Holster）动画播放期间，权重 `α` 从 0 快速插值到 1，完成从持枪姿态到奔跑姿态的 0.2 秒平滑过渡，避免了硬切换产生的姿态跳变。

## 常见误区

**误区一：认为混合权重可以超出 [0, 1] 范围**
部分初学者尝试将 `α` 设为 1.5 以"强化"某个动画效果。实际上超出范围后骨骼会沿插值方向外推，导致肢体穿插模型甚至飞出屏幕。若需要放大某个动画的影响，正确做法是使用叠加动画（Additive Animation），而非超范围的混合权重。

**误区二：对旋转使用线性插值（Lerp）而非 Slerp**
用 Lerp 插值四元数时，当两个旋转相差约 180° 时，插值中间会产生旋转缩小（四元数模长趋向 0）的问题，表现为骨骼在过渡中途突然"萎缩"。Slerp 保持四元数单位长度不变，才能正确表现旋转过渡。

**误区三：忽略动画片段长度差异对混合结果的影响**
两个时长不同的动画（如 A 为 30 帧，B 为 45 帧）若不做时间同步直接混合，即使权重计算完全正确，也会因两者所处"动画阶段"错位而产生不自然的抖动。混合基础的权重插值只解决姿态问题，时间对齐需要额外处理。

## 知识关联

混合基础建立在**状态转换**的基础上：状态机定义了"何时切换"，混合基础解决了"如何平滑过渡"。状态转换提供触发条件和目标状态，而混合基础用 `α` 参数将两个状态的动画在骨骼层面无缝连接。

掌握两动画插值后，自然延伸至 **1D BlendSpace**：1D BlendSpace 本质上是将多个动画沿一条轴排列，运行时在相邻两个动画之间自动应用混合基础的插值逻辑，权重由输入参数与各动画"锚点"的距离决定。此外，**交叉淡入淡出**（Cross-Fade）是混合基础在时间维度的应用——过渡期间 `α` 随时间从 0 变化到 1；**混合曲线**则允许 `α` 不再随时间线性变化，而是遵循缓入缓出等非线性曲线，使过渡更具弹性和生命感。
