# 奔跑循环

## 概述

奔跑循环（Run Cycle）是角色动画中描述持续跑步动作的循环序列，其最核心的生物力学特征是**空中阶段（Airborne Phase）的存在**——即在完整步伐周期内，双脚同时离开地面的瞬间。这一特征从根本上区别了跑步与行走：行走时至少有一只脚始终接触地面，而跑步中每个步伐都包含一个双脚腾空的短暂区间，腾空时间通常占整个步伐周期的30%～40%。这不仅是视觉上的差异，更是驱动整个动画节奏与姿态设计的核心物理依据。

奔跑循环作为独立技术课题的科学记录，可追溯至19世纪末埃德沃德·迈布里奇（Eadweard Muybridge）1887年出版的《动物运动》（*Animal Locomotion*，University of Pennsylvania出版）。他在加州帕洛阿尔托使用24架相机、以1/2000秒曝光时间连续拍摄人类跑步，首次以科学方式证明了腾空阶段的存在，推翻了当时学院派画家对跑步姿态的错误描绘——即认为快速跑步时总有一只脚踩地。迪士尼动画师随后在20世纪30至40年代将这套分析系统化，弗兰克·托马斯（Frank Thomas）与奥利·约翰斯顿（Ollie Johnston）在其1981年著作《生命的幻觉》（*The Illusion of Life: Disney Animation*）中将奔跑循环的姿态分析写入迪士尼十二原则体系，形成现代商业动画的技术规范。

在游戏和影视生产中，奔跑循环是使用频率最高的角色动画资产之一。一个标准的AAA游戏角色往往拥有不止一个奔跑循环，包括普通跑（Run）、冲刺跑（Sprint）、受伤跑（Injured Run）、持枪跑（Strafe Run）等变体，每种都需要独立制作并通过混合树（Blend Tree）在引擎中切换。

## 核心原理

### 步伐周期与帧数分配

标准奔跑循环的完整周期（从右脚接触地面到下一次右脚接触地面）在传统24fps动画中通常为**8帧到12帧**，对应约0.33～0.5秒，而行走循环通常为16帧至24帧（约0.67～1.0秒）。游戏动画中，常见的奔跑循环帧率设定为**30fps下12帧一循环**（约0.4秒/步），冲刺动作甚至可压缩到8帧（约0.27秒/步）。这个更快的节奏要求动画师在更少的帧数内传达更大幅度的姿态变化，每一帧的信息密度远高于行走。

一个标准奔跑循环通常拆分为以下四个关键姿势（Pose），以12帧循环为例进行帧分配：

| 关键帧 | 帧位置（12帧循环） | 描述 |
|---|---|---|
| 接触帧（Contact） | 第1帧、第7帧 | 前脚跟触地，对侧手臂最前 |
| 俯冲帧（Down） | 第2帧、第8帧 | 重心最低，膝盖弯曲吸收冲击 |
| 腾空帧（Airborne） | 第4帧、第10帧 | 双脚离地，身体重心最高 |
| 蹬地帧（Up） | 第3帧、第9帧 | 后腿蹬离地面，推动身体前进 |

注意两侧动作相差半个循环（6帧），整个循环在左右脚之间保持严格的180°相位差。

### 腾空阶段的动画处理

腾空阶段是奔跑循环区别于行走循环的技术难点。在腾空帧中，角色的重心轨迹应呈现**抛物线弧形**而非直线，即使在2D逐帧动画中也需要在垂直方向上有轻微的上升或下落趋势。忽略这一弧形会导致角色看起来像是在平滑滑行，而非真实奔跑。腾空帧中，两腿通常呈现"剪刀形"交叉的极端拉伸（Stretch）姿态，后腿伸直延伸至臀部后方，前腿膝盖高抬至腰部高度，以强调速度感与力量感。

在3D动画中，腾空帧的处理还涉及**根骨骼（Root Bone）的垂直位移曲线**。跑步时根骨骼在Y轴（垂直轴）上的振幅通常比行走时大40%～60%，且曲线的波峰出现在腾空帧而非接触帧，这与行走循环的曲线规律相反——行走时重心最低点在俯冲帧，最高点在单脚支撑的过渡帧。正确理解这一差异是从行走循环过渡到奔跑循环制作时最易出错的环节之一。

### 身体各部位的幅度对比

与行走循环相比，奔跑循环中全身各部位的运动幅度均显著增大，以下数值参考真实人体运动捕捉数据（参见《计算机动画：算法与技术》Rick Parent, 2012, Morgan Kaufmann出版社）：

- **躯干前倾角度**：行走时约5°～10°，普通奔跑时增加到15°～25°，冲刺时可达30°以上；
- **手臂摆动**：行走时肘部弯曲约90°且摆动幅度较小（约30°），奔跑时手臂弯曲维持约90°但前后摆动幅度增大至约60°～80°，拳头可从腰部高度摆至下颌高度；
- **对侧扭转（Counter-rotation）**：跑步时骨盆与肩膀的反向旋转幅度明显大于走路，用以平衡角动量，在3D动画中骨盆横向旋转角度可达15°～20°，而行走时仅约8°～12°；
- **头部与颈部**：奔跑中头部应有轻微的上下点动（约3°～5°），但需保持基本水平，过度的头部抖动会破坏摄像机跟踪时的稳定感；
- **步幅长度**：普通成年角色行走步幅约为身高的45%，奔跑步幅可达身高的70%～80%，冲刺可超过90%。

## 关键公式与节奏计算

奔跑循环的节奏直接影响角色移动速度与视觉信息的一致性。若角色脚部滑步（Foot Sliding），根本原因是**每帧移动距离**与**脚步动画周期**不匹配。可用以下公式校验：

$$
V_{character} = \frac{S_{stride}}{T_{cycle}}
$$

其中：
- $V_{character}$ 为角色在世界空间中的移动速度（单位/秒）
- $S_{stride}$ 为一个完整步伐周期内角色前进的距离（单位）
- $T_{cycle}$ 为一个完整循环的时长（秒）

**案例计算**：若游戏中角色奔跑速度设定为 $V = 600\text{ cm/s}$，动画循环为30fps下12帧（$T_{cycle} = 12/30 = 0.4\text{ s}$），则动画中步幅应设计为：

$$
S_{stride} = V \times T_{cycle} = 600 \times 0.4 = 240\text{ cm}
$$

即动画师在制作时，角色从第1帧到第12帧（一个完整循环结束）的根骨骼水平位移必须恰好为240 cm，否则引擎中原地播放动画与角色实际移动速度不匹配，产生脚步滑动。

在Unreal Engine 5中，可通过Motion Matching系统自动匹配速度，但在传统循环动画工作流中，该公式是消除滑步问题的基本校验工具。下方为一段伪代码示例，用于在引擎运行时检测滑步误差：

```python
# 检测奔跑循环是否产生脚步滑动
def check_foot_sliding(anim_stride_cm, char_speed_cms, cycle_duration_s):
    expected_stride = char_speed_cms * cycle_duration_s
    error = abs(anim_stride_cm - expected_stride)
    tolerance = 5.0  # 允许5cm误差范围
    if error > tolerance:
        print(f"警告：滑步误差 {error:.1f} cm，请调整动画步幅或角色速度")
    else:
        print(f"通过：步幅误差 {error:.1f} cm，在容忍范围内")

# 示例调用
check_foot_sliding(anim_stride_cm=240, char_speed_cms=600, cycle_duration_s=0.4)
# 输出：通过：步幅误差 0.0 cm，在容忍范围内
```

## 实际应用

### 游戏角色奔跑循环制作（3D工作流）

在Unreal Engine 5或Unity中制作奔跑循环时，动画师通常遵循以下步骤：

1. **确定参数**：与技术团队确认角色移动速度（如600 cm/s）、循环帧数（如30fps/12帧），代入公式计算根骨骼水平位移量（240 cm）；
2. **布置极端姿势**：先在第1帧和第7帧制作左右接触帧，确保两侧动作完全对称（可翻转复制，但需手动调整细节避免机械感）；
3. **填充腾空帧**：在第4帧和第10帧制作剪刀腿的腾空姿势，此时根骨骼Y轴应处于整个循环的最高点；
4. **添加次级动作**：为头发、披风、腰带等辅助骨骼添加跟随动画，运动延迟通常设定为2～4帧；
5. **烘焙与导出**：在Maya或Blender中烘焙动画曲线，以FBX格式导出，确保动画首尾帧可无缝衔接。

### 2D逐帧奔跑循环（传统动画工作流）

传统2D动画（如Adobe Animate或TVPaint制作）中，8帧奔跑循环是最常见的轻量方案，适用于手机游戏或独立游戏。动画师Preston Blair在其著作《动画》（*Animation*, Walter Foster出版社, 1994）中给出了经典的8帧奔跑循环范式：每两帧为一个动作阶段，帧1-2为接触与俯冲，帧3-4为腾空，帧5-6为对侧接触与俯冲，帧7-8为对侧腾空，如此形成完整的左右步伐循环。

在2D奔跑循环中，**挤压与拉伸（Squash & Stretch）**的应用比行走循环更为极端：腾空帧中角色整体高度可比静止站立时拉伸8%～12%，而俯冲帧（重心最低）时可压缩5%～8%，这种夸张变形是传统卡通奔跑循环活泼感的来源。

## 常见误区

### 误区一：将行走循环的帧数直接减半来制作奔跑循环

行走循环并不能通过简单"加速播放"变成奔跑循环。两者的根本区别在于：行走循环缺少腾空帧，根骨骼垂直轨迹的波峰和波谷位置完全不同，手臂弯曲角度与摆动节奏也存在质的差异。直接将24帧行走循环压缩到12帧播放，会得到一个既无腾空阶段、又缺乏躯干前倾的"快走"，而非奔跑。

### 误区二：腾空帧中两脚呈水平对称分布

真实跑步的腾空瞬间，两腿并非完全对称展开。由于蹬地腿需要向后延伸以完成推进，而摆动腿膝盖高抬，两腿在矢状面上呈不对称的剪刀形：前腿膝盖高度通常比后腿高出15°～25°。忽视这一不对称性会导致腾空帧显得僵硬或漂浮。

### 误区三：忽略手部与手腕的动画
