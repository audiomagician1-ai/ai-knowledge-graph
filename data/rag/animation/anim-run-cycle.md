# 奔跑循环

## 概述

奔跑循环（Run Cycle）是角色动画中描述持续跑步动作的循环序列，其最核心的生物力学特征是**空中阶段（Airborne Phase）的存在**——即在完整步伐周期内，双脚同时离开地面的瞬间。这一特征从根本上区别了跑步与行走：行走时至少有一只脚始终接触地面，而跑步中每个步伐都包含一个双脚腾空的短暂区间。这不仅是视觉上的差异，更是驱动整个动画节奏与姿态设计的核心物理依据。

奔跑循环作为独立技术课题的研究，可追溯至19世纪末埃德沃德·迈布里奇（Eadweard Muybridge）1887年出版的《动物运动》（*Animal Locomotion*）。他使用多架相机连续拍摄人类跑步，首次以科学方式记录了腾空阶段的存在，推翻了当时画家对跑步姿态的错误描绘。迪士尼动画师随后在20世纪30至40年代将这套分析系统化，形成了现代商业动画奔跑循环的技术规范。

在游戏和影视生产中，奔跑循环是使用频率最高的角色动画资产之一。一个标准的游戏角色往往拥有不止一个奔跑循环，包括普通跑、冲刺跑、受伤跑等变体，每种都需要独立制作。因此，深度掌握奔跑循环的构造规律，直接影响动画师的生产效率与最终品质。

## 核心原理

### 步伐周期与帧数分配

标准奔跑循环的完整周期（从右脚接触地面到下一次右脚接触地面）在传统24fps动画中通常为**8帧到12帧**，而行走循环通常为16帧至24帧。游戏动画中，常见的奔跑循环帧率设定为**30fps下12帧一循环**（约0.4秒/步），冲刺动作甚至可压缩到8帧。这个更快的节奏要求动画师在更少的帧数内传达更大幅度的姿态变化，每一帧的信息密度远高于行走。

一个标准奔跑循环通常拆分为以下四个关键姿势（Pose）：
1. **接触帧（Contact）**：前脚跟触地，身体最低；
2. **俯冲帧（Down）**：重心最低，膝盖弯曲吸收冲击；
3. **腾空帧（Airborne/Pass）**：双脚离地，身体最高；
4. **蹬地帧（Up）**：后腿蹬离地面，推动身体前进。

### 腾空阶段的动画处理

腾空阶段是奔跑循环区别于行走循环的技术难点。在这一帧中，角色的重心轨迹应呈现**抛物线弧形**而非直线，即使在2D逐帧动画中也需要在垂直方向上有轻微的上升或下落趋势。忽略这一弧形会导致角色看起来像是在平滑滑行，而非真实跑步。腾空帧中，两腿通常呈现"剪刀形"交叉的极端拉伸（Stretch）姿态，以强调速度感。

在3D动画中，腾空帧的处理还涉及**根骨骼（Root Bone）的垂直位移曲线**。跑步时根骨骼在Y轴上的振幅通常比行走时大40%～60%，且曲线的峰值出现在腾空帧而非接触帧，这与行走循环的曲线规律相反。

### 身体各部位的幅度变化

与行走循环相比，奔跑循环中全身各部位的运动幅度均显著增大：

- **躯干前倾角度**：行走时约5°～10°，奔跑时增加到15°～25°，冲刺时可达30°以上；
- **手臂摆动**：行走时肘部弯曲约90°且摆动幅度较小，奔跑时手臂弯曲维持约90°但前后摆动幅度增大，拳头可从腰部高度摆至下颌高度；
- **对侧扭转（Counter-rotation）**：跑步时骨盆与肩膀的反向旋转幅度明显大于走路，用以平衡角动量，在3D动画中这一扭转角度可达15°～20°；
- **头部与颈部**：奔跑中头部应有轻微的上下点动，但需保持基本水平，避免过度抖动干扰视线追踪感。

## 实际应用

**游戏角色奔跑循环制作**：在Unreal Engine或Unity的角色运动系统中，奔跑循环通常与行走循环通过**混合树（Blend Tree）**进行速度插值过渡。这要求奔跑循环的步伐节拍必须与行走循环的步伐节拍保持节奏兼容性，否则过渡时会出现脚部滑步（Foot Sliding）。实践中，动画师会将奔跑循环导出时的移动速度（通常设为约5～6米/秒）精确匹配到根骨骼位移数据，确保原地循环（In-place Loop）与实际移动速度同步。

**卡通风格的奔跑夸张处理**：在风格化动画（如日本动画或2D平台游戏）中，奔跑循环常用"糊帧（Smear Frame）"技法——在腾空帧前后插入一帧拉伸变形的模糊姿态，模拟运动模糊效果。《猫和老鼠》和《忍者神龟》等经典动画大量使用这一技术，使8帧的低帧率跑步看起来更流畅有力。

**写实影视动作捕捉后期修正**：动作捕捉数据中的奔跑循环常因演员在有限摄影棚内跑步而产生速度不稳定问题。动画师需要在MotionBuilder中手动调整骨盆的垂直曲线，并确保每步的腾空帧高度一致，同时清理因地面反作用力不均匀导致的膝盖抖动噪点。

## 常见误区

**误区一：奔跑只是"加速的行走"**
初学者常误以为只需将行走循环的播放速度提高即可得到跑步效果。但跑步和走路是两种完全不同的步态结构。行走的重心轨迹呈倒摆运动（支撑腿为轴向前旋转），而跑步是弹簧-质点模型（Spring-Mass Model）——腿部如同弹簧储存和释放弹性势能。因此，直接加速行走循环会产生腿部姿态错误、缺少腾空阶段、重心弧线方向相反等多种问题。

**误区二：腾空帧中两腿应当平行伸展**
部分学生在制作腾空帧时，将双腿设计为向前和向后各伸展一次的对称姿态，但实际跑步的腾空帧中，后腿的膝盖通常是**高度弯曲并向上收起的**（即"折叠腿"），而非完全向后伸直。这种折叠动作是为了缩短摆动腿的惯性力矩，加速腿部向前的摆动速度。在奔跑循环中忽略这一折叠会使角色看起来步伐沉重而缺乏弹性。

**误区三：跑步时头部应随身体大幅上下运动**
由于根骨骼在Y轴上有较大的垂直位移，初学者常让头部跟随身体整体做大幅上下运动，导致角色看起来像是在跳跃而非奔跑。真实跑步时，颈部和头部的肌肉会主动抵消身体的上下振动，头部的垂直位移远小于骨盆的垂直位移。在动画中，头部的上下振幅通常应控制在骨盆振幅的**30%～50%以内**。

## 知识关联

**前置知识——行走循环**：掌握行走循环是学习奔跑循环的必要基础，特别是对接触帧、下沉帧、经过帧、上升帧四个关键姿势的理解，以及手臂与腿的对侧摆动规律。奔跑循环在这套框架上做了结构性修改：移除了双脚支撑阶段，将经过帧替换为腾空帧，并在全身幅度上进行了系统性放大。

**后续概念——过渡动画**：完成奔跑循环的制作后，下一个技术挑战是制作行走到奔跑（Walk-to-Run）以及奔跑到静止（Run-to-Stop）的过渡动画。这类过渡无法简单地通过融合两段循环来完成，因为步伐节拍和重心弧线规律均不同，需要专门制作2帧至8帧的过渡缓冲姿态，并处理脚步同步问题，是对奔跑循环原理理解深度的直接检验。
