# 手部IK

## 概述

手部IK（Hand Inverse Kinematics）是将逆向运动学应用于角色手部，使手掌和手指能够自动贴合环境表面、抓握物体或攀附墙壁的动画技术。与腿部IK以地面接触为主要目标不同，手部IK需要同时处理手腕旋转朝向和手指弯曲角度两个层级，因此其效果链条更长、约束条件更复杂。

手部IK技术在2000年代早期随着动作捕捉游戏的普及而成熟，《刺客信条》系列（2007年起）将攀爬手部IK推向了主流视野，促使各大引擎将手部IK支持内置到动画系统中。Unreal Engine的`AnimGraph`在4.16版本加入了专用的`Hand IK Retargeting`节点，Unity也在Animation Rigging 1.0包中提供了`TwoBoneIK`与`Multi-Aim Constraint`组合方案专供手部绑定使用。

手部IK的实用价值体现在减少大量手工K帧工作量：一个角色在攀爬网格时，若完全依赖烘焙动画，遇到不同形状的岩壁需为每种形状单独制作动画；而手部IK可让同一段攀爬循环动画在运行时自动适配任意凹凸表面，极大降低内容制作成本。

## 核心原理

### IK目标（IK Target）与射线检测

手部IK的效果触发通常依赖射线检测（Raycast）或球形检测（SphereCast），从角色手腕位置向前方或目标方向发射射线，命中表面后将命中点和命中法线作为IK目标（IK Target，也称Effector）的位置和旋转输入。命中法线决定手掌法线方向，例如扶墙时手掌面应垂直于墙面，计算公式为：

```
手腕目标旋转 = LookRotation(-hitNormal, upVector)
```

其中 `hitNormal` 为表面外法线，`upVector` 为世界向上方向或手臂弯曲参考方向。这步旋转计算若缺失，手掌会以默认朝向插入墙面，产生穿模。

### 双骨骼IK求解与极向量

手部IK在上臂（UpperArm）—前臂（Forearm）—手腕（Wrist）三节骨骼上应用双骨骼IK（Two-Bone IK）求解器，这正是本知识点依赖双骨骼IK作为前置的原因。极向量（Pole Vector）控制肘部弯曲朝向，通常设置为肘部当前世界坐标沿肩膀到手腕连线的垂直偏移点，以防止手臂翻转（Flipping）。极向量偏移距离一般设置为上臂长度的0.5到1.0倍，具体数值需根据角色比例调整。

### 手指IK与程序化弯曲

手腕就位后，手指需要单独处理。常见方案是为每根手指的末端骨骼做额外的短距离射线检测（检测距离通常为3—5厘米物理单位），命中后对手指三节骨骼（掌骨、近节、中节/远节）应用旋转插值，使指尖贴合表面。这种方法称为程序化手指弯曲（Procedural Finger Curl），公式为：

```
fingerRotation = Lerp(openPose, closedPose, curlAmount)
```

`curlAmount` 由指尖到目标表面的归一化距离（0为完全张开，1为完全握拢）驱动。抓握球形物体时，`curlAmount` 按各手指与物体中心的最近距离分别独立计算，使四根手指呈现自然的非均匀弯曲。

### IK权重与过渡混合

手部IK不会始终全权重激活，否则角色在伸手未及目标时手臂会被强行拉到奇异位置。实际实现中，IK权重（IK Weight）在角色接近目标时从0线性或用Ease-In曲线插值到1，通常在距目标0.4米处开始过渡，在0.15米处达到全权重。权重同时控制手腕的位置权重和旋转权重，二者可独立设置，例如扶墙时旋转权重为1（强制贴合法线），位置权重为0.9（允许轻微偏移以避免穿模）。

## 实际应用

**攀爬抓握**：在岩壁攀爬系统中，左右手分别维护独立的IK目标点。每步攀爬移动前，系统预计算下一个落手点，在角色肢体前摆阶段将IK权重从1降至0（手离开岩面），前摆完成后重新做射线检测，将新命中点设为IK目标并将权重从0升回1（手抓住新岩点）。《最后生还者》系列的攀爬系统即采用这种双手独立IK权重时间差方案，使左右手交替移动看起来更自然。

**扶墙/推门**：角色行走经过墙壁时，胸骨到手腕方向做SphereCast，半径约0.12米，命中墙面后激活手部IK并对肩膀添加一个轻微的后拉旋转（约8—15度），模拟手臂被墙面支撑时肩胛骨的真实受力姿态。

**拾取物品**：物品预设一个或多个抓握锚点（Grip Anchor），锚点存储手腕应到达的局部坐标和旋转。激活IK时，将锚点的世界变换赋值给IK目标，手指根据物品类型调取预设的`curlAmount`数据（如圆柱形握法 `curlAmount ≈ 0.75`，捏取握法 `curlAmount ≈ 0.4`）。

## 常见误区

**误区一：认为手部IK只需设置手腕位置即无需处理旋转**。这是最常见的错误。手腕位置正确但旋转未对齐法线时，手背可能插入墙体或手掌与表面形成反向夹角，视觉上比不用IK更差。正确做法是位置和旋转必须同步处理，缺一不可。

**误区二：对所有手指使用相同的curlAmount**。真实抓握时，小指通常比食指弯曲程度更大（约多15—20度），拇指弯曲轴也与其他四指不同。统一curlAmount会让抓握看起来像"机械爪"，需为每根手指设置独立权重偏移或分别做射线检测。

**误区三：忽略IK与FK动画的混合层级顺序**。如果在动画图中将手部IK节点放在FK混合之前执行，FK层的手腕旋转会覆盖IK结果，导致手部IK完全失效。正确的节点顺序是：基础FK动画 → 身体层IK（躯干偏移）→ 手部IK → 手指程序化弯曲，手部IK必须在最后几个阶段处理。

## 知识关联

手部IK直接建立在**双骨骼IK**的肩-肘-腕三节求解基础之上，如果不熟悉极向量约束的计算方式，手部IK中肘部翻转问题将无法排查和修正。

掌握手部IK后，**武器持握IK**是自然的下一步：武器握把相当于一种特殊的固定抓握锚点，但增加了双手协调约束（左手辅助握持点必须跟随武器变换矩阵），以及换弹/瞄准状态切换时IK权重的分层管理。**载具乘坐IK**则将手部IK扩展到方向盘、操纵杆等动态运动物体上，IK目标本身在世界空间中持续运动，需要额外的延迟跟随（Lag Follow）防止手腕抖动，是手部IK在动态父节点场景下的进阶应用。
