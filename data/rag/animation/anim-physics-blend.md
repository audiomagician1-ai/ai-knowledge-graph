# 物理-动画混合

## 概述

物理-动画混合（Physics-Animation Blending）是一种在预定义关键帧动画与实时物理模拟之间按比例插值的技术，其核心操作是对骨骼姿态（Pose）进行加权平均：最终骨骼变换 = α × 物理姿态 + (1-α) × 动画姿态，其中混合权重 α ∈ [0, 1]。α=0 时角色完全由动画剪辑驱动，α=1 时关节完全交由刚体物理引擎（如 PhysX 或 Havok）接管。

这项技术最早在2005年前后随 Havok Animation 中间件的商业普及而进入主流游戏引擎。Rockstar 在《侠盗猎车手IV》（2008年）中将其用于主角 Niko 的碰撞反应，使角色在被撞击时不再切换至固定死亡动画，而是在行走循环与布娃娃状态之间连续过渡，成为业界广泛引用的早期成功案例。

物理-动画混合的意义在于解决了两种驱动模式的"切换鸿沟"问题。纯关键帧动画无法响应运行时未预知的环境碰撞，而纯布娃娃系统又缺乏意图性运动（角色看起来完全"死"了）。通过调节 α，开发者可以让角色在受到枪伤时先展现受击动画，再根据伤害程度渐进地将控制权交给物理引擎，整个过程对玩家而言连贯自然。

## 核心原理

### 姿态混合的数学基础

骨骼变换由平移（Translation）、旋转（Rotation）和缩放（Scale）三部分构成，通称 TRS。物理-动画混合在这三个分量上分别执行插值：平移与缩放使用线性插值（LERP），旋转使用球面线性插值（SLERP）以避免万向节死锁。对于一具有 N 根骨骼的角色，每帧需要完成 N 次 SLERP 运算，典型人形角色约 60–80 根骨骼，因此实现时通常以 SIMD 指令批量处理四元数插值以控制 CPU 开销。

### 分层权重与骨骼蒙版

物理-动画混合并非对全身骨骼使用同一个 α 值。实践中将骨架划分为若干**混合层（Blend Layer）**，每层持有独立的权重曲线。例如角色的下半身（髋骨至脚踝）的 α 保持为 0（动画主导），以保证脚步与地面对齐；而上半身（脊柱至锁骨）的 α 随碰撞冲量动态升高。这种逐骨骼的权重分配称为**骨骼蒙版（Bone Mask）**，在 Unreal Engine 的 `PhysicsBlendWeight` 系统和 Unity 的 `Animator.SetBoneLocalRotation` 接口中均有直接体现。

### 混合权重的驱动策略

α 的变化本身需要一套控制逻辑，常见有三种策略：

1. **冲量阈值触发**：当施加在骨骼上的物理冲量（单位：N·s）超过预设阈值时，α 按固定速率（如每帧 +0.05）递增至目标值，之后以更慢速率（如每帧 -0.02）衰减回 0，形成"受击弹性"效果。
2. **生命值映射**：将角色血量百分比线性映射到 α，使角色在濒死状态下步态愈发不稳，物理特征逐渐凸显。《最后生还者》系列正是利用此策略制造残血视觉反馈。
3. **动画状态机触发**：在动画状态机（Animation State Machine）的过渡条件中直接嵌入 α 关键帧，将混合权重作为动画剪辑的一个额外曲线通道输出，由动画师在 DCC 工具中精确编辑。

### 物理姿态的来源与约束

物理引擎每帧输出的布娃娃骨骼位置受关节约束（Joint Constraint）限制，通常为铰链约束（Hinge）或锥形约束（Cone Constraint）。在执行混合之前，引擎需将物理世界坐标系中的刚体位姿转换回骨骼局部空间，这一步称为**物理姿态回读（Physics Pose Read-back）**。如果跳过此步骤直接混合，将导致物理骨骼与渲染网格产生视觉撕裂。Unreal Engine 的 `SyncComponentToRBPhysics` 函数即专门处理这一坐标系对齐问题。

## 实际应用

**受击反应系统**：第三人称射击游戏中，子弹命中角色肩部时，该骨骼的 α 在 0.1 秒内从 0 上升至 0.8，产生肩膀被打飞的物理感，随后在 0.3 秒内衰减回 0，角色继续播放原有移动动画。整个过程不触发任何动画状态切换，延迟感极低。

**死亡过渡**：角色死亡时，若直接切换至布娃娃会产生明显的"姿态跳变"。标准做法是在最后一帧动画姿态与当前物理姿态之间以 0.2–0.5 秒的时窗执行混合过渡（α 从 0 线性增至 1），消除视觉突变。《黑暗之魂》系列的死亡动画即采用此模式。

**载具乘员**：赛车游戏中驾驶员在车辆剧烈颠簸时，躯干上半身的 α 随加速度数值浮动（当垂直加速度超过 2g 时 α 提升），令驾驶员头部和手臂产生真实晃动，而双手仍保持锚定在方向盘动画上（对应骨骼的 α 恒为 0）。

## 常见误区

**误区一：混合权重越高效果越真实**

许多初学者认为将 α 直接设为 1 可以获得最佳物理效果，但实际上全物理状态下的角色会因为缺乏运动意图而产生"烂泥感"。正确做法是根据具体场景设定 α 的上限，通常受击状态下上限为 0.6–0.8 而非 1.0，从而保留动画姿态对关节的引导力。

**误区二：对所有骨骼使用统一权重**

忽略骨骼蒙版、对全身使用同一 α 值，会导致下半身骨骼也受到物理拉扯，进而破坏脚步 IK（Inverse Kinematics）对地面的吸附效果。正确实现必须区分**物理响应骨骼**（上半身、头部）与**动画锚定骨骼**（足部、手部接触点）分别管理权重。

**误区三：混合与布娃娃切换是同一件事**

布娃娃系统（Ragdoll）是将角色骨骼完全交由物理引擎接管（α 固定为 1）的特殊状态，而物理-动画混合是 α 在 (0,1) 区间内动态变化的连续过程。两者实现路径不同：布娃娃通常需要禁用动画图（Animation Graph）并直接驱动刚体，而混合系统需要动画图与物理引擎每帧并行运算并在输出端合并。

## 知识关联

**前置概念：布娃娃系统**
布娃娃系统建立了角色骨架与刚体链（Rigid Body Chain）之间的映射关系，并定义了每根骨骼对应的关节约束。物理-动画混合直接复用这套刚体结构，将其从"全接管"模式改造为"部分权重参与"模式。没有布娃娃系统中关节约束参数的正确配置，混合时物理骨骼的旋转范围将不符合人体解剖限制，产生穿模或关节过伸。

**后续概念：Active Ragdoll**
Active Ragdoll 在物理-动画混合的基础上为刚体关节添加 PD 控制器（Proportional-Derivative Controller），使物理关节主动向动画目标姿态施加扭矩，而非被动接受混合插值。可以理解为：物理-动画混合在**输出空间**合并两种姿态，而 Active Ragdoll 在**控制空间**让物理关节追踪动画姿态，二者解决同一问题但从不同层次介入。

**后续概念：载具物理动画**
载具物理动画将本文介绍的骨骼权重分层策略扩展到悬挂系统与车轮骨骼，车轮转向角和悬挂压缩量由物理引擎输出（α=1），而车体姿态可部分混合预烘焙的颠簸动画以增强视觉层次感。
