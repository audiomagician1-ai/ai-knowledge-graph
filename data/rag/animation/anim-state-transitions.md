# 状态转换

## 概述

状态转换（State Transition）是动画状态机中连接两个动画状态的有向箭头，定义了角色从一个动作切换到另一个动作的方式与时机。例如，从"待机"状态转换到"奔跑"状态，或从"跳跃"状态转换到"落地"状态，每一条转换路径都携带三项核心数据：触发条件、过渡持续时间、以及混合曲线。

状态转换的概念源自有限状态机（FSM）理论，但在游戏动画领域，Unity于2012年在Mecanim系统中将其可视化，使设计师可以直接在Animator窗口拖拽箭头来创建转换关系，并在Inspector面板中配置各项参数。这种可视化转换设计极大降低了动画逻辑的实现门槛。

状态转换的质量直接影响玩家对角色动作的感知流畅度。一条配置不当的转换——例如过渡时间设为0、切入点在动画中段——会造成动作"跳切"，破坏游戏的沉浸感。反之，合理的转换设置可以让角色在各种动作间无缝衔接，即使底层动画片段本身并没有专门制作过渡帧。

## 核心原理

### 转换条件（Transition Condition）

转换条件决定状态机"何时"执行这条转换。Unity Animator支持四种条件变量类型：**Bool**（布尔值，如`isRunning == true`）、**Float**（浮点数，如`speed > 0.5`）、**Int**（整数，如`weaponIndex == 2`）、**Trigger**（一次性触发器，读取后自动重置为false）。

一条转换可以设置多个条件，所有条件必须**同时满足**（逻辑AND）才会触发转换。如果需要OR逻辑，则需要创建多条独立的转换箭头分别指向目标状态。Trigger类型特别适合"攻击"、"受击"这类瞬时事件，使用Float/Bool则适合"速度大于某阈值时切换步态"这类持续性状态判断。

此外，**Has Exit Time**选项允许转换在不依赖任何参数条件的情况下，在当前动画播放到指定归一化时间点（Exit Time，取值0.0到1.0，如0.75表示播放至75%时）时自动触发，常用于攻击动画结束后自动回到待机状态。

### 过渡时间（Transition Duration）

过渡时间是两个动画状态发生交叉混合的总时长，在Unity中有两种度量方式：
- **归一化时间**：值为0.25表示以**目标动画**总时长的25%作为过渡窗口
- **秒数（Fixed Duration勾选后）**：直接指定绝对时长，如0.15秒

过渡时间为0时，系统在同一帧内直接切换，没有任何混合，适用于"死亡"等需要立即响应的状态。过渡时间过长（如超过0.5秒）则会导致动作显得迟钝，玩家输入与角色反应之间出现明显延迟感。对于大多数移动类转换，0.1到0.25秒是经验上较为合适的范围。

**Transition Offset**参数指定目标动画的起始播放位置（归一化值），例如设为0.3表示目标动画从30%处开始播放，而非从头播放，用于跳过目标动画开头多余的预备帧。

### 交叉淡入淡出（Cross Fade）

交叉淡入淡出描述过渡期间两个状态动画的权重混合方式。在过渡时间内，源状态动画权重从1.0线性（或按曲线）衰减至0.0，目标状态动画权重从0.0增加至1.0，两者权重之和始终为1.0，确保骨骼姿态不出现"缩水"现象。

Unity的Blend Curve允许将线性混合替换为自定义曲线，例如使用**EaseIn/EaseOut**曲线让过渡在起始和结束阶段更平滑，减少机械感。这对于"行走→奔跑"这类速度连续变化的转换尤其有效。代码层面也可以通过`Animator.CrossFadeInFixedTime("StateName", 0.2f)`直接触发一条指定时长的交叉淡入淡出，绕过参数条件检查。

## 实际应用

**角色移动系统**：在第三人称跑跳角色中，从"Idle"到"Run"的转换通常设置条件`speed > 0.1`、过渡时间0.15秒、Has Exit Time关闭，保证玩家按下方向键时角色立即响应，而不是等待当前帧播完。

**攻击连段**：格斗游戏中，"Attack1"转换到"Attack2"常使用Trigger条件，配合Exit Time设为0.6，意味着玩家必须在攻击动画播放超过60%之后按键才能衔接下一段，实现"连招窗口"机制，而不是随时都能打断。

**受击打断**：角色被攻击命中时需要立即播放受击动画，通常对"受击"Trigger设置转换，并将过渡时间设为0，Has Exit Time关闭，从而无论当前处于何种状态都能即时响应，保证战斗手感。

**落地缓冲**：从"Jump_Fall"转换到"Land"动画时，可将Transition Offset设为0.05，跳过落地动画最初几帧的脚部接触预备姿势，让落地瞬间视觉上更为干脆。

## 常见误区

**误区一：认为过渡时间越长动画就越流畅。** 过渡时间过长会造成两个语义上矛盾的动画同时呈现，例如"死亡"动画和"待机"动画混合0.5秒，会出现角色倒下的同时还在做呼吸动作的视觉错误。流畅度取决于源动画与目标动画在过渡点的姿态相似程度，而非过渡时间本身。

**误区二：混淆Trigger与Bool的使用场景。** 使用Bool代替Trigger处理一次性事件时，如果在触发后忘记将Bool重置为false，状态机会在每帧都满足条件并反复尝试触发转换，导致动画逻辑陷入死循环或意外状态跳转。Trigger在被状态机消费一次后自动清除，天然防止此类问题。

**误区三：忽略Interruption Source（中断来源）设置。** Unity状态转换默认不允许被新的转换打断，若不勾选"Current State"或"Next State"中断选项，玩家在过渡期间的输入会被完全忽略。对于需要即时响应的游戏（如动作游戏），必须为关键转换开启中断，否则会出现"输入吞噬"的手感问题。

## 知识关联

学习状态转换之前，需要掌握**状态机基础**中关于状态节点、Entry/Exit节点以及Animator Parameters的概念，这些是配置转换条件的前提。理解"当前状态"和"目标状态"的区别，才能正确设置过渡时间的参照基准。

掌握状态转换之后，可以进一步学习**转换规则**，它在状态转换的基础上引入优先级排序、Any State节点的特殊转换行为，以及Sub-State Machine之间的跨层转换逻辑。**移动状态机**会大量运用转换条件中的Float参数与速度阈值，是状态转换在角色控制器中的典型应用场景。**状态通知（State Notify）**则与Has Exit Time配合，在动画的特定帧触发游戏事件，需要对转换时序有精确理解。**混合基础**进一步延伸了交叉淡入淡出的概念，介绍1D/2D混合树如何在多个动画片段间进行权重分配，是状态转换混合机制的深化版本。
