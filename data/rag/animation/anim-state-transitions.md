# 状态转换

## 概述

状态转换（State Transition）是动画状态机中连接两个动画状态的有向链接，定义了角色从一个动画（如"待机"）切换到另一个动画（如"奔跑"）时的完整过渡行为。它不仅规定了**何时**触发切换，还精确控制两段动画如何在时间轴上交叠融合。

状态转换的概念源自有限状态机（Finite State Machine，FSM）理论，最早由数学家George Mealy（1955年，Mealy机）和Edward Moore（1956年，Moore机）在20世纪50年代分别形式化定义，两者的核心区别在于输出是否依赖当前输入——Mealy机的输出由当前状态和输入共同决定，而Moore机的输出仅由当前状态决定。游戏引擎将这一计算机科学概念引入动画系统，Unity在2012年随Mecanim系统的发布将状态转换可视化，使美术师无需编写代码即可通过Animator窗口拖拽连线、配置复杂的动画切换逻辑。虚幻引擎（Unreal Engine）的Animation Blueprint同样采用了相同的有向图范式，在Result节点中整合了过渡规则（Transition Rule）配置，并于UE5中新增了基于姿势相似度的自动过渡时长估算（Automatic Transition）功能。

理解状态转换对动画质量至关重要：错误配置的过渡时间会导致角色动作"跳帧"或"停顿"，而合理设置交叉淡入淡出则能让动作流畅自然，直接影响玩家的操控手感。对于一款角色动作游戏，其状态机可能包含数十乃至数百条转换链接，例如《黑暗之魂3》的主角状态机据社区逆向分析估计包含超过200条转换规则，每一条链接的参数配置都会对最终游戏表现产生可见影响。

## 核心原理

### 转换条件（Transition Conditions）

转换条件是触发状态切换的逻辑门控，由**参数类型**和**比较运算**组成。常用参数类型包括：

- **Bool**：`isGrounded == true` 触发落地动画
- **Float**：`speed > 0.5` 触发从走路到跑步
- **Int**：`weaponType == 2` 切换持枪动作
- **Trigger**：一次性触发信号，如`jumpTrigger`，消耗后自动重置

多个条件之间默认为**逻辑与（AND）**关系——即同一转换上的所有条件必须同时满足才会触发。若需要"或（OR）"逻辑，需要创建两条独立的转换线分别指向目标状态。

未勾选"Has Exit Time"时，转换完全依赖条件驱动，响应更及时；勾选后，动画必须播放到指定的退出时间比例（Exit Time，取值0到1）才允许发生转换，适用于攻击动画必须打完一套的场景。值得注意的是，当 Exit Time 设为大于1的值（如1.85）时，Unity会将其解释为动画循环1次后再播放至85%处触发，这一特性常被用于需要完整循环一次才允许退出的连招动画。

### 过渡时间（Transition Duration）

过渡时间定义了两个状态交叠融合所持续的帧长度或时间比例。在Unity Animator中，该值默认为**0.25秒（Normalized模式下为0.25个时间单位）**，以目标状态的时长为基准进行归一化。

过渡时间的选择遵循以下经验规律：

- **急促动作**（翻滚、格挡）：过渡时间设为0.05～0.1秒，保证动作切换的即时感
- **普通移动**（走→跑）：0.15～0.25秒，视觉上不产生突兀感
- **慢速过渡**（战斗→休息的呼吸动作）：0.3～0.5秒，营造放松的渐入感

过渡时间**不能超过源状态或目标状态中较短者的时长**，否则交叠区域会产生不可预期的权重分配错误。此外，过渡时间的计量模式分为两种：Fixed（固定秒数，与动画速度无关）和Normalized（归一化比例，随动画播放速率等比缩放），在动画速度会随游戏逻辑动态调整的场景中应优先选用Fixed模式以保证视觉稳定性。

### 交叉淡入淡出（Cross-Fade）

交叉淡入淡出是状态转换的实际混合机制：在过渡时间内，源状态动画的权重从1.0线性（或曲线）衰减到0.0，目标状态动画的权重同步从0.0升至1.0，两者之和在任意时刻保持为1.0。这一机制在骨骼层面对每一块骨骼的变换（位置、旋转、缩放）分别执行插值，因此当源动画与目标动画的关键帧姿势差异过大时，中间过渡帧会出现"穿插"或"拉伸"的视觉瑕疵。

过渡混合曲线（Blend Curve）控制权重变化的速率形状：

- **Linear（线性）**：权重均匀过渡，适合大多数地面移动
- **Ease In/Out（缓入缓出）**：开始和结束阶段变化慢、中间快，适合较长的情感动作
- **Custom（自定义）**：在Unity中可手动编辑曲线，针对特定动画做精细调整

`Animator.CrossFade(stateName, normalizedTransitionDuration)` 是通过代码强制触发交叉淡入淡出的API，第二个参数即归一化过渡时长，传入0则为硬切换（无混合）。`CrossFadeInFixedTime(stateName, fixedTransitionDuration)` 则对应Fixed模式，适合需要精确控制过渡秒数的场景。

### 转换中断（Transition Interruption）

Unity Mecanim还提供了"Interruption Source"选项，控制当前进行中的转换能否被新的转换打断。选项包括：

- **None**：过渡期间不可中断，必须等当前转换完成
- **Current State**：源状态上的其他转换可以中断当前过渡
- **Next State**：目标状态上的转换可以从过渡中途介入
- **Current State then Next State / Next State then Current State**：指定检查优先顺序

正确配置中断源对格斗游戏的"取消机制"（Cancel）至关重要：允许玩家在攻击前摇阶段立即响应翻滚输入，就需要将翻滚转换设置为可中断当前攻击转换。

## 关键公式与数学模型

### 线性权重插值公式

在标准线性交叉淡入淡出过程中，任意时刻 $t$ 处两个动画状态的权重由以下公式描述：

$$
w_{src}(t) = 1 - \frac{t}{T}, \quad w_{dst}(t) = \frac{t}{T}
$$

其中 $t$ 为过渡已经进行的时间（秒），$T$ 为总过渡时长（Transition Duration，单位秒），$w_{src}$ 为源状态权重，$w_{dst}$ 为目标状态权重，且满足归一化约束 $w_{src}(t) + w_{dst}(t) = 1$，$t \in [0, T]$。

当使用缓入缓出（Ease In/Out）曲线时，权重变化改为平滑步函数（Smoothstep）：

$$
w_{dst}(t) = 3\left(\frac{t}{T}\right)^2 - 2\left(\frac{t}{T}\right)^3
$$

该公式在 $t=0$ 和 $t=T$ 处的一阶导数均为0，意味着过渡在起止点速度为零，视觉上呈现出更加柔和的渐变效果，适用于情感动作或慢速过渡场景。

### 归一化时间与 Exit Time

在Unity Animator中，Exit Time以**归一化时间**（Normalized Time）表示，即动画当前播放进度占整个片段时长的比例：

$$
t_{normalized} = \frac{t_{current}}{t_{clip}}
$$

例如，一个时长为1.2秒的攻击动画，若将 Exit Time 设为0.85，则系统在播放至 $1.2 \times 0.85 = 1.02$ 秒时才允许状态转换触发。

### 有限状态机的形式化定义

动画状态转换系统的理论基础来自形式化有限状态机，其五元组定义为：

$$
M = (Q,\ \Sigma,\ \delta,\ q_0,\ F)
$$

其中 $Q$ 为有限状态集合（所有动画状态节点），$\Sigma$ 为输入字母表（所有参数与条件组合），$\delta: Q \times \Sigma \rightarrow Q$ 为状态转换函数（即"当前状态 × 触发条件 → 目标状态"的映射），$q_0 \in Q$ 为初始状态（通常是"待机"），$F \subseteq Q$ 为终止状态集合（在游戏动画中一般为"死亡"等不可逆状态）。

## 实际应用

**案例一：角色移动状态机的转换配置**

在第三人称游戏中，"待机→行走"转换通常设置条件 `Speed > 0.1`，过渡时间0.2秒，不使用Exit Time；反向"行走→待机"转换条件为 `Speed < 0.1`，过渡时间0.15秒。两条转换方向相反但过渡时长略有差异，原因是起步动作需要缓冲，而停步动作希望更快稳定。

例如，在《原神》类型的开放世界游戏中，角色的移动状态机通常包含：待机、慢走、快跑、冲刺四个主要状态，以及其间共8条双向转换，每条转换均针对该速度区间的姿势差异单独调整过渡时长，而非采用统一默认值。开发团队还会对"冲刺→急停"这一特殊路径额外叠加一个"急停缓冲（Deceleration）"中间状态，使动画在物理上更合理地表现惯性。

**案例二：攻击动画的不可打断转换**

格斗游戏中，"普通攻击"状态通常勾选"Has Exit Time"并将Exit Time设为0.85，意味着动画必须播放到85%才允许进入下一状态，防止玩家快速点击时动画被截断而出现手部穿模。同时，连击窗口（Combo Window）通常设在Exit Time 0.7～0.9之间，即在动画尚未完全结束前开放下一段攻击的输入检测，这要求开发者同时维护两条从"攻击"出发的转换：一条通向"连击攻击2"（在窗口期触发），一条通向"待机/收招"（超出窗口期后触发）。

**案例三：代码动态控制**

```csharp
// 过渡时间0.1秒的快速切换，适合受击反应
animator.CrossFade("Hit_React", 0.1f, 0, 0f);

// Fixed模式切换，过渡时间固定为0.15秒，不受动画速率影响
animator.CrossFadeInFixedTime("Hit_React", 0.15f, 0, 0f);
```

第三个参数 `layerIndex` 指定层级，第四个参数 `normalizedTimeOffset` 设为0确保目标动画从头播放。与之对比，直接调用 `animator.Play("Hit_React")` 为硬切换，无任何混合过渡，适合需要完全消除前一动画残影的瞬间反应场景（如角色死亡）。

**案例四：多层（Multi-Layer）状态机中的转换隔离**

在上半身独立控制射击动画、下半身处理移动动画的双层状态机中，两个层级的状态转换互相独立，各自拥有独立的 Exit Time 和 Transition Duration 配置。上半身层的"瞄准→换弹"转换可以设置0.05秒极短过渡以响应玩家输入，同时下半身层的"跑步→蹲步"转换维持0.25秒的平滑过渡，两者并行不干扰。这一架构被《使命召唤》系列等射击游戏广泛采用，可在保证移动动画流畅的同时确保武器操作动画的即时反馈感。
