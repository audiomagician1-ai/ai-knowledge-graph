# 状态转换

## 概述

状态转换（State Transition）是动画状态机中连接两个动画状态的有向链接，定义了角色从一个动画（如"待机"）切换到另一个动画（如"奔跑"）时的完整过渡行为。它不仅规定了**何时**触发切换，还精确控制两段动画如何在时间轴上交叠融合。

状态转换的概念源自有限状态机（Finite State Machine，FSM）理论，最早由数学家George Mealy（1955年，Mealy机）和Edward Moore（1956年，Moore机）在20世纪50年代分别形式化定义，两者的核心区别在于输出是否依赖当前输入。游戏引擎将这一计算机科学概念引入动画系统，Unity在2012年随Mecanim系统的发布将状态转换可视化，使美术师无需编写代码即可通过Animator窗口拖拽连线、配置复杂的动画切换逻辑。虚幻引擎（Unreal Engine）的Animation Blueprint同样采用了相同的有向图范式，在Result节点中整合了过渡规则（Transition Rule）配置。

理解状态转换对动画质量至关重要：错误配置的过渡时间会导致角色动作"跳帧"或"停顿"，而合理设置交叉淡入淡出则能让动作流畅自然，直接影响玩家的操控手感。对于一款角色动作游戏，其状态机可能包含数十乃至数百条转换链接，每一条链接的参数配置都会对最终游戏表现产生可见影响。

## 核心原理

### 转换条件（Transition Conditions）

转换条件是触发状态切换的逻辑门控，由**参数类型**和**比较运算**组成。常用参数类型包括：

- **Bool**：`isGrounded == true` 触发落地动画
- **Float**：`speed > 0.5` 触发从走路到跑步
- **Int**：`weaponType == 2` 切换持枪动作
- **Trigger**：一次性触发信号，如`jumpTrigger`，消耗后自动重置

多个条件之间默认为**逻辑与（AND）**关系——即同一转换上的所有条件必须同时满足才会触发。若需要"或（OR）"逻辑，需要创建两条独立的转换线分别指向目标状态。

未勾选"Has Exit Time"时，转换完全依赖条件驱动，响应更及时；勾选后，动画必须播放到指定的退出时间比例（Exit Time，取值0到1）才允许发生转换，适用于攻击动画必须打完一套的场景。

### 过渡时间（Transition Duration）

过渡时间定义了两个状态交叠融合所持续的帧长度或时间比例。在Unity Animator中，该值默认为**0.25秒（Normalized模式下为0.25个时间单位）**，以目标状态的时长为基准进行归一化。

过渡时间的选择遵循以下经验规律：

- **急促动作**（翻滚、格挡）：过渡时间设为0.05～0.1秒，保证动作切换的即时感
- **普通移动**（走→跑）：0.15～0.25秒，视觉上不产生突兀感
- **慢速过渡**（战斗→休息的呼吸动作）：0.3～0.5秒，营造放松的渐入感

过渡时间**不能超过源状态或目标状态中较短者的时长**，否则交叠区域会产生不可预期的权重分配错误。

### 交叉淡入淡出（Cross-Fade）

交叉淡入淡出是状态转换的实际混合机制：在过渡时间内，源状态动画的权重从1.0线性（或曲线）衰减到0.0，目标状态动画的权重同步从0.0升至1.0，两者之和在任意时刻保持为1.0。

过渡混合曲线（Blend Curve）控制权重变化的速率形状：

- **Linear（线性）**：权重均匀过渡，适合大多数地面移动
- **Ease In/Out（缓入缓出）**：开始和结束阶段变化慢、中间快，适合较长的情感动作
- **Custom（自定义）**：在Unity中可手动编辑曲线，针对特定动画做精细调整

`Animator.CrossFade(stateName, normalizedTransitionDuration)` 是通过代码强制触发交叉淡入淡出的API，第二个参数即归一化过渡时长，传入0则为硬切换（无混合）。

## 关键公式与数学模型

### 线性权重插值公式

在标准线性交叉淡入淡出过程中，任意时刻 $t$ 处两个动画状态的权重由以下公式描述：

$$
w_{src}(t) = 1 - \frac{t}{T}, \quad w_{dst}(t) = \frac{t}{T}
$$

其中 $t$ 为过渡已经进行的时间（秒），$T$ 为总过渡时长（Transition Duration，单位秒），$w_{src}$ 为源状态权重，$w_{dst}$ 为目标状态权重，且满足约束 $w_{src}(t) + w_{dst}(t) = 1$，$t \in [0, T]$。

当使用缓入缓出（Ease In/Out）曲线时，权重变化改为平滑步函数（Smoothstep）：

$$
w_{dst}(t) = 3\left(\frac{t}{T}\right)^2 - 2\left(\frac{t}{T}\right)^3
$$

该公式在 $t=0$ 和 $t=T$ 处的一阶导数均为0，意味着过渡在起止点速度为零，视觉上呈现出更加柔和的渐变效果，适用于情感动作或慢速过渡场景。

### 归一化时间与 Exit Time

在Unity Animator中，Exit Time以**归一化时间**（Normalized Time）表示，即动画当前播放进度占整个片段时长的比例：

$$
t_{normalized} = \frac{t_{current}}{t_{clip}}
$$

例如，一个时长为1.2秒的攻击动画，若将 Exit Time 设为0.85，则系统在播放至 $1.2 \times 0.85 = 1.02$ 秒时才允许状态转换触发。

## 实际应用

**案例一：角色移动状态机的转换配置**

在第三人称游戏中，"待机→行走"转换通常设置条件 `Speed > 0.1`，过渡时间0.2秒，不使用Exit Time；反向"行走→待机"转换条件为 `Speed < 0.1`，过渡时间0.15秒。两条转换方向相反但过渡时长略有差异，原因是起步动作需要缓冲，而停步动作希望更快稳定。

例如，在《原神》类型的开放世界游戏中，角色的移动状态机通常包含：待机、慢走、快跑、冲刺四个主要状态，以及其间共8条双向转换，每条转换均针对该速度区间的姿势差异单独调整过渡时长，而非采用统一默认值。

**案例二：攻击动画的不可打断转换**

格斗游戏中，"普通攻击"状态通常勾选"Has Exit Time"并将Exit Time设为0.85，意味着动画必须播放到85%才允许进入下一状态，防止玩家快速点击时动画被截断而出现手部穿模。

**案例三：代码动态控制**

```csharp
// 过渡时间0.1秒的快速切换，适合受击反应
animator.CrossFade("Hit_React", 0.1f, 0, 0f);
```

第三个参数 `layerIndex` 指定层级，第四个参数 `normalizedTimeOffset` 设为0确保目标动画从头播放。与之对比，直接调用 `animator.Play("Hit_React")` 为硬切换，无任何混合过渡，适合需要完全消除前一动画残影的瞬间反应场景（如角色死亡）。

**案例四：多层（Multi-Layer）状态机中的转换隔离**

在上半身独立控制射击动画、下半身处理移动动画的双层状态机中，两个层级的状态转换互相独立，各自拥有独立的 Exit Time 和 Transition Duration 配置。上半身层的"瞄准→换弹"转换可以设置0.05秒极短过渡以响应玩家输入，同时下半身层的"跑步→蹲步"转换维持0.25秒的平滑过渡，两者并行不干扰。

## 常见误区

**误区一：过渡时间越长动画越流畅**

很多初学者认为加大过渡时间能改善动画质量，但过渡时间超过0.3秒的移动切换会让角色看起来"飘"或"迟钝"。真正影响流畅度的是**源动画和目标动画在过渡点的姿势相似性**，姿势差异小的两个动画即使用0.05秒过渡也可以很流畅。动画师应优先保证关键帧姿势的匹配，而非依赖加长过渡时间来掩盖姿势跳变。

**误区二：Trigger参数等同于Bool参数的快速开关**

Trigger在被Animator消耗后自动重置为未激活状态，但若当前转换条件不满足（如被其他更高优先级转换拦截），Trigger会在**下一帧**仍然保持激活并继续尝试触发。这可能导致"意外的延迟跳转"，在需要精确一次性触发时应配合 `ResetTrigger()` 主动清除。

**误区三：Can Transition To Self可以忽略**

"Can Transition To Self"选项允许状态转换到自身，若条件始终满足（如 `Speed > 0` 而角色一直在移动），且同时勾选了Has Exit Time，会导致动画在播放完毕后**立即从头重播**，造成动画循环被强制打断的视觉闪烁。正确做法是对循环动画关闭"Can Transition To Self"，让Animator的内置循环机制自然驱动。

**误区四：忽视转换的优先级顺序**

当同一源状态上存在多条转换且条件同时满足时，Unity Animator严格按照Inspector面板中的**从上到下排列顺序**决定优先触发哪条转换。例如，若"攻击→死亡"排在"攻击→收招"之前，且死亡条件满足，则永远不会执行收招转换。初学者常因随意调整顺序而引发难以排查的逻辑错误。

## 知识关联

**前置概念——状态机基础**：状态转换依赖状态机中已定义的各个状态节点，理解状态（State）和层（Layer）的概念是配置转换的前提。没有状态节点，转换就没有起点和终点。有限状态机的形式化定义为五元组 $M = (Q, \Sigma, \delta, q_0, F)$，其中 $\delta: Q \times \Sigma \rightarrow Q$ 即为状态转换函数，这在动画系统中映射为"当前状态 × 触发条件 → 目标状态"。

**下游概念——转换规则**：当多条转换同时满足条件时，Animator按照转换在Inspector列表中的**排列顺序**决定优先级，这一机制构成了转换规则的核心内容，是状态转换配置的进阶方向。

**并行概念——混合基础**：状态转换的Cross-Fade机制与混合树（Blend Tree）的权重混合共用同一套底层插值系统，但混合树是在单个状态内部混合多个动画片段，而状态转换负责**跨状态**的时间轴交叠。

**扩展方向——状态通知**：可在状态的动画片段上添加Animation Event，在过渡开始或结束的特定帧触发回调，实现"转换到攻击状态的第3帧时播放挥剑音效"等精确的事件绑定。

**工具层对比**：在Unreal Engine 5的Animation Blueprint中，与Unity状态转换对应的概念称为"Transition Rule"，支持以蓝图节点图的形式描述复杂条件逻辑，且原生支持基于骨骼姿势相似度的自动过渡时长估算功能（Automatic Transition），这是Unity Animator目前尚不具备的特性。
