# 距离匹配（Distance Matching）

## 概述

距离匹配（Distance Matching）是 BlendSpace 系统中的一项动画技术，其核心目标是让动画片段中角色的位移量与游戏引擎中角色实际移动的距离保持精确一致。具体而言，它通过在动画时间轴上寻找"累计位移恰好等于当前游戏位移"的那一帧，而非简单地线性播放动画，从而消除脚步滑步（Foot Sliding）现象。

该技术由 Epic Games 在 Unreal Engine 5 的 Motion Matching 工作流中被广泛推广，核心节点为 `DistanceMatchToTarget`，并在 UE5.0（2022年发布）的 Lyra 示例项目中作为标准角色移动方案首次完整公开。在此之前，开发者通常依赖根运动（Root Motion）或手动调整播放速率来解决滑步问题，但这两种方式在变速运动下均存在明显局限。

距离匹配之所以关键，在于玩家操控的角色往往以不规则速度移动：从静止加速、遇障减速、或在不平地形上速度抖动。传统按时间驱动的动画播放无法应对这类速度变化，而距离匹配通过位移这一物理量直接驱动动画进度，彻底解耦了动画时间与游戏时间之间的绑定关系。

## 核心原理

### 位移曲线（Distance Curve）的预计算

每段需要参与距离匹配的动画资产（如启动动画 `Start_Fwd`、停步动画 `Stop_Fwd`）必须预先烘焙一条**距离曲线**。该曲线记录的是：从当前帧到动画结束帧（或从动画开始帧到当前帧），角色根骨骼在前向轴上的累计位移量（单位：厘米）。

例如，一段 0.5 秒的停步动画，其距离曲线在第 0 帧的值可能为 **-80cm**（负号表示角色还需再移动 80cm 才能完全停止），在最后一帧值为 0。Unreal Engine 使用 `AnimationModifier` 插件中的 `AddDistanceCurve` 工具自动生成该曲线，曲线名称通常命名为 `Distance`。

### 目标距离的查询逻辑

距离匹配的运行时逻辑可以表述为：

> **寻找动画中满足 `DistanceCurve(frame) == TargetDistance` 的帧，并跳转播放到该帧。**

`TargetDistance` 由游戏逻辑提供，通常有两种计算方式：
- **停步场景**：`TargetDistance = 预测停止位置 - 当前角色位置`（即制动距离）
- **启动场景**：`TargetDistance = 角色当前位置 - 启动触发位置`（已走过的距离）

查询本身是对距离曲线做一次**数值反查（Inverse Lookup）**，时间复杂度为 O(n)，其中 n 为动画帧数，在实践中由于曲线单调，可用二分查找优化至 O(log n)。

### 与播放速率缩放（Rate Scale）的本质差异

播放速率缩放通过公式 `RateScale = ActualSpeed / AnimationSpeed` 拉伸或压缩动画时间，本质上改变了动画播放速度。距离匹配则完全不调整播放速度，而是**直接定位动画帧**——每一帧的姿态都来自原始动画数据，不存在插值失真。这意味着当角色突然从 600cm/s 减速到 0 时，距离匹配可以正确播放停步动画的减速阶段姿态，而速率缩放方案在此情况下会产生"快进滑行"的视觉错误。

## 实际应用

### 停步动画中的刹车距离预测

在 Unreal Engine 的 Lyra 项目中，停步距离匹配的典型实现步骤如下：
1. 在 Tick 每帧调用 `PredictStopLocation`，基于当前速度与摩擦系数（通常为 `GroundFriction = 8.0`）计算制动距离。
2. 将制动距离作为 `TargetDistance` 传入 `DistanceMatchToTarget` 节点。
3. 节点在停步动画的距离曲线上反查对应帧，驱动动画状态机跳转到该帧。

这使得角色无论在何种速度下触发停步，脚部姿态始终与实际地面接触点对齐，消除滑步。

### 启动动画中的已行进距离跟踪

启动场景中，角色从静止开始移动，距离匹配持续追踪"自启动触发以来角色已移动多少厘米"，并将该值喂给距离曲线做正向查询，确保启步抬脚的时机与角色实际离地时刻完全同步。对于一段 0.4 秒、位移约 **120cm** 的启步动画，若玩家在角色走了 60cm 时突然停止，距离匹配会立刻切换到停步逻辑，不会让角色继续播放剩余的启步姿态。

## 常见误区

### 误区一：距离匹配等同于根运动

根运动（Root Motion）将动画中根骨骼的位移直接驱动游戏角色移动，动画数据决定运动距离；距离匹配则方向相反——游戏逻辑决定角色位置，动画系统通过距离曲线反查来**跟随游戏位置**。两者驱动方向完全相反：根运动是"动画驱动物理"，距离匹配是"物理驱动动画帧选择"。在支持布娃娃碰撞或网络同步的项目中，通常禁用根运动而使用距离匹配，正是因为游戏逻辑需要保持对角色位置的完全控制权。

### 误区二：距离曲线必须严格单调

虽然大多数直线移动动画的距离曲线是单调的，但转向动画或侧移动画在三维投影后可能出现非单调段，此时反查会返回多个候选帧。正确做法是只在动画片段的**有效匹配区间**内启用距离匹配（通过设置曲线的有效范围），超出区间后切换为普通时间驱动播放，否则会导致动画在某帧附近来回跳跃。

### 误区三：距离匹配可以完全替代所有过渡混合

距离匹配解决的是**单段动画内的位移对齐**问题，但动画状态之间的过渡（如从跑步混合到停步）仍需 BlendSpace 的混合权重或 Inertialization（惯性化混合）来平滑过渡姿态。若完全依赖距离匹配而省略混合过渡，在触发停步的瞬间会出现明显的姿态跳帧。

## 知识关联

### 与根运动的关系

根运动是距离匹配的重要前驱知识——理解根骨骼如何携带位移数据，才能理解距离曲线是从根骨骼轨迹中提取并预计算而来的。距离匹配可以看作是将根运动数据"反向利用"的技术：不用根运动驱动角色，而是用根运动数据生成的距离曲线来匹配游戏角色的实际位移。

### 在 Motion Matching 工作流中的位置

距离匹配是 Unreal Engine Motion Matching 系统的重要预处理步骤，通常在 Pose Search（姿态搜索）之前用于缩小候选动画范围。掌握距离匹配后，开发者可以进一步学习完整的 Motion Matching 流程，包括 `PoseSearchDatabase` 的构建、轨迹预测（Trajectory Prediction）以及姿态代价函数的设计，这些技术共同构成 UE5 中新一代程序化角色动画的核心工作流。
