# 逆向动力学（IK）

## 概述

逆向动力学（Inverse Kinematics，简称 IK）是一种通过指定骨骼链末端位置来反向计算各关节旋转角度的技术。与正向动力学（FK）从根节点逐级旋转关节不同，IK 让动画师只需拖动"手"或"脚"等末端执行器（End Effector）到目标位置，系统自动求解中间所有关节的角度。这种"结果驱动"的工作方式使脚部着地、手部抓握等需要精确定位的动画任务效率大幅提升。

IK 的数学基础可追溯到机器人学领域，1960年代工业机械臂的控制问题推动了相关算法的成熟。三维动画软件在1990年代将其引入，Maya、3ds Max 等工具将 IK 求解器（IK Solver）内置为标准功能。如今，游戏引擎（如 Unity 的 Animation Rigging 包、Unreal Engine 的 Full Body IK）也将运行时 IK 作为角色控制的核心工具，使角色双脚能在凹凸不平的地面上实时适应。

IK 的重要性在于它将"空间约束"的思维方式引入动画制作。当角色的脚必须踩在某块石头上时，使用 FK 需要反复调整大腿、小腿、踝关节三个旋转值；使用 IK 则只需将末端控制器吸附到石头表面，三个关节角度由求解器在毫秒内完成计算。

## 核心原理

### 末端执行器与目标位置

IK 系统由三个关键元素组成：**关节链（Joint Chain）**、**末端执行器（End Effector）**和**IK 目标（IK Target / Goal）**。关节链是一段有父子层级关系的骨骼序列，例如"髋关节 → 膝关节 → 踝关节"。末端执行器是链条最末端的点，IK 目标是动画师在世界空间中指定的期望位置。IK 求解器的任务就是计算使末端执行器与 IK 目标重合所需的各关节旋转值。

### 求解方法：解析解与迭代解

**解析解（Analytic Solution）**适用于关节数量少（通常为 2 个关节，即"双骨骼 IK"）的情况。以人类腿部的两段骨骼为例，已知大腿长度 $l_1$、小腿长度 $l_2$、以及目标点距髋关节的距离 $d$，可用余弦定理直接求出膝关节弯曲角度 $\theta$：

$$\cos\theta = \frac{l_1^2 + l_2^2 - d^2}{2 \cdot l_1 \cdot l_2}$$

此公式给出唯一（或两个对称）解，计算速度极快，Maya 的 **ikRPsolver**（Rotate Plane IK）正是基于此原理实现的。

**迭代解（Iterative Solution）**用于三个以上关节或需要多目标约束的复杂情形。常见算法包括：
- **FABRIK**（Forward And Backward Reaching Inverse Kinematics，2011年提出）：通过反复前向/后向伸展骨骼链来逼近目标，收敛速度快且实现简单，被 Unity Animation Rigging 广泛采用。
- **雅可比矩阵法（Jacobian-based IK）**：将关节速度与末端速度的关系用雅可比矩阵 $J$ 表示，通过 $\Delta\theta = J^+ \cdot \Delta e$（其中 $J^+$ 为伪逆矩阵）迭代求解，适合多末端执行器联动场景。

### 自由度与多解问题

人类肘关节只有 1 个自由度（屈/伸），但 IK 在数学上往往存在**无穷多解**。以二维平面上的两段骨骼为例，末端到达目标点时，膝盖可以向上弯也可以向下弯——这是两个有效解。三维空间中解的数量更多。**极向量（Pole Vector）**控制器正是为了解决这一歧义性而设计的：它在空间中指定一个方向，约束膝关节或肘关节朝向该方向弯曲，从而在所有数学合法解中选出动画师期望的那一个。

### IK 链的伸展限制

当 IK 目标距离根关节的距离 $d > l_1 + l_2$ 时，骨骼链无法到达目标，称为**超出可达范围（Over-extension）**。此时不同软件的处理策略不同：Maya 默认会将骨骼完全伸直，朝目标方向拉伸但停在最大伸展状态；Unreal Engine 的 Two Bone IK 节点则提供 `Allow Stretching` 选项，允许骨骼按比例拉伸以强制到达目标，常用于卡通风格角色。

## 实际应用

**脚部着地（Foot Planting）** 是 IK 最经典的应用场景。在游戏角色控制器中，每帧对地面进行射线检测（Raycast），将左右脚的 IK 目标分别吸附到射线碰撞点，踝关节 IK 目标高度 = 地面高度 + 脚踝偏移量。这样角色在斜坡或台阶上行走时，脚底不会穿入地面或悬浮在半空中。Unity 的 Humanoid Rig 内置了 `Foot IK` 选项，勾选后由引擎自动执行此射线检测逻辑。

**手部抓握（Hand Grabbing）** 场景中，IK 目标绑定到被抓取物体的抓握点（Grip Point）上。当角色的手接近梯子横档时，手部 IK 目标吸附到横档位置，上臂和前臂自动调整以完成抓握姿势，无需动画师手工制作每个抓握位置的独立动画。《最后生还者》系列大量使用此技术实现角色攀爬时的手部精确定位。

**椅子坐姿适配** 是另一个典型案例：角色坐下时，IK 将双脚约束到地面，避免因椅子高度不同导致脚悬空或嵌入地面的穿帮问题。

## 常见误区

**误区一：IK 和 FK 必须二选一**
实际上，现代绑定通常将两者混合使用（IK/FK 混合权重）。角色行走时脚部使用 IK 锁定到地面，同一时间上身挥手动作使用 FK 制作弧线感。Maya 的绑定中常见 `IK_FK_blend` 属性，值为 0 时纯 FK，值为 1 时纯 IK，中间值则线性混合两套控制结果。

**误区二：IK 目标移动等于骨骼根关节移动**
IK 目标（End Effector Goal）在世界空间中是独立的控制器，与骨骼链的根节点相互独立。当角色整体位移时，若不将 IK 目标设置为跟随根节点（即 IK 目标父级不是角色根骨骼），脚部 IK 目标会留在原地，导致腿部被"撕扯"到不合理角度——这是初学者绑定角色时极为常见的错误。

**误区三：IK 求解总能得到精确解**
迭代型 IK 求解器（如 FABRIK）设有最大迭代次数限制（通常为 10～30 次），当目标在每帧快速移动时，求解结果可能存在误差累积，末端执行器无法完全到达目标。此时需要增加迭代次数或降低目标移动速度的变化率。

## 知识关联

学习 IK 之前需要理解 **IK/FK 概述**中的关节链父子层级概念，以及骨骼的局部坐标系与世界坐标系转换——IK 求解的输入（目标位置）在世界空间，而输出（关节旋转）在局部空间，理解这一坐标空间转换是读懂求解器代码的前提。

掌握 IK 基本原理后，可以进入 **双骨骼 IK（Two Bone IK）** 的专项学习，深入了解余弦定理求解过程以及超出可达范围时的边界处理细节。**极向量（Pole Vector）** 是与双骨骼 IK 同步使用的约束工具，专门解决本文中提到的膝盖/肘部弯曲方向歧义问题。**Look At** 控制器则是 IK 思想在旋转维度上的延伸——它通过指定眼睛或头部的"注视目标"位置，反向求解头颈关节的旋转角度，与脚部 IK 的位置约束思路如出一辙。
