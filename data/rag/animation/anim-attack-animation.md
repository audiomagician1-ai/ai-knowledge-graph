# 攻击动画

## 概述

攻击动画是格斗游戏、动作游戏中角色战斗行为的视觉表达，其结构严格遵循**预备（Anticipation）→挥击（Strike）→恢复（Recovery）**三阶段模型。与行走循环的匀速循环节奏不同，攻击动画的时间分配高度不均——预备阶段通常占总帧数的40%~50%，挥击阶段极短（往往仅2~4帧），恢复阶段约占30%~40%。这种极端的节奏压缩正是攻击动画"爆发感"的核心来源。

攻击动画的三段式结构源自迪士尼动画师Ollie Johnston与Frank Thomas于1981年在《The Illusion of Life: Disney Animation》（Disney Editions出版）中系统化的"预备动作（Anticipation）"原则，后被游戏动画师移植并强化为战斗场景的标准制作框架。在街机格斗游戏鼎盛时期（1990年代），《街头霸王II》技术组将每段攻击动画的挥击窗口精确控制在2~3帧（约33~50毫秒），这一标准至今仍是格斗游戏判定框设计的重要参考。

掌握攻击动画的制作，意味着动画师需要同时处理**运动曲线的急变**与**角色重心的稳定性**两个相互矛盾的需求——挥击瞬间身体需要极速加速，但脚部支撑点不能飘移，否则角色会显得"失重"。

## 核心原理

### 预备阶段：蓄力与方向预告

预备动作的物理本质是**反向运动（Countermovement）**：角色在发力前必须先向攻击方向的反方向移动，通过肌肉拉伸储存弹性势能。以右勾拳为例，右肩需向右后方旋转至少15°~25°，同时重心下沉约为角色总身高的8%~12%。如果省略这一阶段直接进入挥击，观众在视觉上会感受到"力量凭空产生"，攻击缺乏重量感。

在关键帧设置上，预备阶段的运动曲线应呈**缓入（Ease In）**趋势——角色从静止逐渐加速至预备极限姿势。动画师常在预备末尾设置一个2~3帧的"cushion pose（缓冲关键帧）"，让观看者的眼睛有足够时间读取蓄力姿势，同时强化随后挥击的视觉对比冲击。根据《The Animator's Survival Kit》（Richard Williams，2001，Faber & Faber出版）的建议，预备动作的幅度应与随后攻击动作的力度成正比——轻拳预备幅度约5°~10°肩部旋转，重击则需要20°~35°甚至包含整个上半身的扭转。

### 挥击阶段：极速加速与模糊处理

挥击阶段是三段中时间最短但信息量最密集的部分。从蓄力极限姿势到最大伸展姿势（Full Extension Pose），帧数控制在**2~6帧**之间（以24fps标准计算，即约83~250毫秒）。动画师需要在这几帧内表现武器或肢体的运动轨迹，常用技术包括：

- **动态模糊（Motion Blur）**：在2D动画中通过手绘残影帧（Smear Frame）实现，即在挥击中间帧绘制被拉伸变形的手臂形状，单帧中手掌可被拉长至正常长度的2~3倍，这一技术最早由《猫和老鼠》（Tom and Jerry）动画师Ken Harris在1940年代系统使用
- **挥击弧线（Arc）**：拳头或武器的运动路径必须是弧线而非直线，弧线曲率半径建议不低于角色手臂长度的1.5倍，过小会显得僵硬，过大会削弱攻击的直接穿透感
- **挤压与拉伸（Squash & Stretch）**：拳头在接触帧前一帧可被压缩为正常体积的70%~80%，接触帧本身则可扩张至120%~130%，强化碰撞质感

挥击末尾的**接触帧（Contact Frame）**是攻击动画最关键的单帧，武器与目标接触瞬间角色全身的姿势需传达最大受力状态：肩膀前倾超过垂直轴10°~15°、后腿完全蹬直、手腕锁定在0°弯曲状态。

### 恢复阶段：惯性消化与破绽窗口

恢复阶段模拟攻击后身体动量的消散过程，其运动曲线呈**缓出（Ease Out）**特征。恢复动作的设计直接影响游戏的战斗策略深度——恢复帧数越长，该攻击的"破绽窗口（Vulnerability Window）"越大，攻守博弈价值越高。《街头霸王V》中豪鬼的升龙拳恢复帧长达42帧（约700毫秒），这正是该技能"高风险高收益"战略价值的动画层面根源。

恢复姿势通常分两步：首先是**过冲姿势（Follow-Through Pose）**，即肢体因惯性超过理想静止位置5°~15°；然后经过4~8帧的**缓冲过渡**才回归中立站姿。直接从接触帧跳回站立静止会产生"橡皮球弹回"的视觉效果，完全破坏物理重量感。

## 关键公式与时间分配

攻击动画的三阶段帧数分配遵循以下经验公式，设总帧数为 $N$：

$$
N_{预备} = \lfloor 0.45N \rfloor, \quad N_{挥击} = \max(2,\ \lfloor 0.12N \rfloor), \quad N_{恢复} = N - N_{预备} - N_{挥击}
$$

以一段总计20帧（24fps下约833毫秒）的标准轻拳为例：

- 预备：$\lfloor 0.45 \times 20 \rfloor = 9$ 帧（约375毫秒）
- 挥击：$\max(2,\ \lfloor 0.12 \times 20 \rfloor) = \max(2, 2) = 2$ 帧（约83毫秒）
- 恢复：$20 - 9 - 2 = 9$ 帧（约375毫秒）

对于重拳（总计32帧），挥击阶段扩展至4帧，但比例仍保持在12.5%以内，维持爆发感。

以下为使用Python模拟攻击动画三段式帧分配的示例代码：

```python
import math

def attack_animation_frames(total_frames: int, fps: int = 24) -> dict:
    """
    计算攻击动画三阶段帧数分配
    total_frames: 动画总帧数
    fps: 帧率（默认24fps）
    """
    anticipation = math.floor(0.45 * total_frames)
    strike = max(2, math.floor(0.12 * total_frames))
    recovery = total_frames - anticipation - strike

    def to_ms(frames):
        return round(frames / fps * 1000)

    return {
        "预备帧数": anticipation,
        "预备时长(ms)": to_ms(anticipation),
        "挥击帧数": strike,
        "挥击时长(ms)": to_ms(strike),
        "恢复帧数": recovery,
        "恢复时长(ms)": to_ms(recovery),
        "挥击占比": f"{strike/total_frames*100:.1f}%"
    }

# 轻拳（20帧）vs 重拳（32帧）对比
print(attack_animation_frames(20))
print(attack_animation_frames(32))
```

输出结果中，20帧轻拳的挥击占比为10.0%，32帧重拳的挥击占比为12.5%，直观体现了重攻击在"相对蓄力时间"上的更高投入。

## 实际应用

**2D格斗游戏单拳击动画帧表（以24fps为基准）：**

| 阶段 | 帧范围 | 帧数 | 关键帧内容 | 运动曲线 |
|------|--------|------|------------|----------|
| 预备 | 1~9 | 9帧 | 右肩后旋20°，重心下沉10% | Ease In |
| 缓冲 | 8~9 | 2帧 | 预备极限姿势停顿 | 平坦（Flat） |
| 挥击 | 10~11 | 2帧 | Smear Frame + 接触帧 | 线性（Linear） |
| 过冲 | 12~13 | 2帧 | 手臂超出目标5°~8° | Ease Out |
| 恢复 | 14~20 | 7帧 | 逐步回归中立站姿 | Ease Out |

**案例：《空洞骑士》（Hollow Knight，2017，Team Cherry）主角的竖劈攻击**

该攻击动画全长18帧（60fps下约300毫秒），预备阶段8帧（剑尖向上扬起45°），挥击阶段仅2帧（下劈弧度覆盖180°），恢复阶段8帧。Team Cherry动画师在挥击2帧中使用了夸张的剑身拉伸（Smear）——剑从正上方拉伸成弯月形，覆盖了理论运动弧线上所有中间位置，使得即便在60fps下快进逐帧查看也能清晰感知攻击方向。

**游戏判定框与动画帧的对齐原则**

攻击动画的挥击阶段必须与游戏逻辑层的**判定框激活帧（Active Hitbox Frames）**精确对齐。判定框通常在挥击第1帧激活，在接触帧后1~2帧关闭。若判定框比动画视觉早激活超过2帧，玩家会感受到"打空气"的不适；若晚激活，则出现视觉已击中但伤害未触发的"穿模"感。

## 常见误区

**误区一：预备幅度过小导致攻击"轻飘飘"**

初学者常为了让攻击显得"快速"而压缩预备阶段至1~3帧，结果反而因缺乏对比，挥击阶段的速度变化在视觉上无法被感知。正确做法是保持预备阶段不低于6帧（24fps），用充分的蓄力时间让玩家建立"即将爆发"的期待感。

**误区二：所有帧使用匀速插值（Linear Interpolation）**

在三维动画软件（如Maya、Blender）中，若直接对所有关键帧使用线性插值，挥击阶段看起来像机器人动作。预备阶段必须配置Ease In曲线（切线权重约0.3），恢复阶段配置Ease Out曲线（切线权重约0.3），只有挥击阶段本身才使用Linear，才能制造出"蓄→爆→泄"的节奏层次。

**误区三：Smear Frame画成固定方向的残影**

Smear Frame不是简单地在运动方向上复制手臂——它应该是一个**有意变形的单帧姿势**，反映的是该帧时间窗口内物体通过的空间路径。若攻击弧线是曲线，Smear Frame的手掌拉伸方向也应随之弯曲，而非保持直线拉伸。

**误区四：恢复阶段直接接循环待机动画**

从接触帧直接切换到待机循环动画的第1帧，会产生明显的"跳帧"感。正确做法是在恢复阶段末尾专门制作2~4帧的**过渡帧（Transition Pose）**，使恢复末尾姿势与待机循环的某一中间帧姿势尽量接近，再接入循环。

## 知识关联

**与行走循环的节奏差异**

行走循环的速度曲线在整个循环中变化幅度不超过±20%（重心上下起伏带来的速度波动），属于**周期性匀变速运动**。攻击动画则是**脉冲式运动**——挥击帧的角速度峰值可达预备阶段平均速度的8~15倍，这种量级的速度对比在行走循环中完全不会出现。因此，从行走循环过渡到攻击动画的学习，核心挑战在于建立"可以允许极度不均匀节奏"的制作思维。
