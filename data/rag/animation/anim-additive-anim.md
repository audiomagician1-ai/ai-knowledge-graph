# 叠加动画

## 概述

叠加动画（Additive Animation）是一种将增量变换叠加到现有基础动画之上的技术，其本质是将骨骼的旋转、位移值以**差值**形式相加，而非直接替换原始姿势。与普通混合动画不同，叠加动画的每一帧存储的不是绝对骨骼变换，而是相对于某个参考姿势（Reference Pose）的**偏差量**。这使得同一套叠加动画剪辑可以无缝附加到跑步、跳跃、蹲伏等完全不同的基础动作上，而不会破坏基础动作的运动逻辑。

叠加动画的概念随着游戏引擎骨骼动画系统的成熟而普及，Valve在《半衰期2》（2004年）的开发过程中将其系统化应用于角色持枪动画叠加，此后Unreal Engine和Unity相继将其纳入标准动画工具链。这项技术解决了一个核心矛盾：角色状态组合数量呈指数级增长（受伤×持枪×奔跑 = 独立动画剪辑），而叠加动画让美术人员只需制作少量差异动画，便可与任意数量的基础动作自由组合，将动画资源量压缩至原来的十分之一以下。

## 核心原理

### 叠加差值的计算方式

叠加动画的计算公式为：

```
最终姿势 = 基础姿势 + (叠加动画姿势 − 参考姿势)
```

其中**参考姿势**是录制叠加动画时使用的静止绑定姿势（T-Pose 或特定站立姿势）。例如，制作"受伤倾斜"叠加动画时，美术师在参考姿势（直立T-Pose）基础上，将脊柱骨骼向右旋转15°并弯曲上半身，存储这个相对偏差。当游戏运行时，无论角色当前处于奔跑还是蹲伏，该15°偏移量都会被累加到对应骨骼的当前旋转上，使角色始终保持受伤的佝偻姿态，同时保留完整的腿部移动动画。

### 参考姿势的选择与影响

参考姿势的选取直接决定叠加效果的质量。若参考姿势与基础动画的平均姿势差异过大，叠加后的关节角度可能超出骨骼约束范围，导致手肘、膝盖等部位出现穿模或极端扭曲。Unreal Engine 5中，叠加动画类型分为三种：`LocalSpace`（骨骼局部空间累加，适合脊柱弯曲）、`MeshSpace`（网格空间累加，适合瞄准偏转）和`RotationOffsetMeshSpace`（旋转偏移网格空间）。选择错误的叠加类型会导致角色在转向时武器指向出现明显漂移。

### 权重控制与淡入淡出

叠加动画支持0.0到1.0的权重值调节，公式扩展为：

```
最终姿势 = 基础姿势 + 权重 × (叠加动画姿势 − 参考姿势)
```

权重为0时完全不应用叠加效果，权重为1时完整叠加，权重为0.5时叠加量减半。这个机制被用于实现渐进式状态变化：角色血量从100%下降时，受伤叠加动画的权重从0线性增加至1，制造出逐渐弯腰喘气的视觉效果。疲劳系统通常将权重绑定到耐力值，冲刺耐力耗尽时叠加动画权重在约0.3秒内从0过渡到1。

## 实际应用

**受伤状态反馈**：第一人称射击游戏中，角色被击中时触发一段2到4帧的躯干震动叠加动画，该动画叠加在任意移动状态之上，给予玩家即时的受击反馈，同时不中断角色当前的移动逻辑。《荒野大镖客：救赎2》使用多通道叠加，将腿部受伤（跛行程度）、躯干受伤（弯腰幅度）和手臂受伤（持枪晃动）作为三个独立叠加层同时驱动，精细还原不同受伤部位的行为差异。

**持武器系统**：不同武器（手枪、步枪、双手剑）对应不同的手臂持握叠加动画。美术只需为每种武器制作一套持握姿势叠加动画，便可叠加到数十套角色基础动作上，避免为每个武器×每个动作的组合单独制作动画。Unity的Animator Controller中，专门的Override Layer（覆盖层）与Additive Layer（叠加层）区分即源于此需求，叠加层默认权重设置为1并勾选`Additive`混合模式。

**瞄准偏转（Aim Offset）**：这是叠加动画最经典的应用场景。制作一组覆盖上下各60°、左右各90°的脊柱与颈部旋转叠加动画，根据鼠标/摇杆输入实时插值，叠加在奔跑、蹲伏等基础动画上，实现角色身体朝向与视线方向的自由解耦。Unreal Engine将此封装为`AimOffset`资产类型，内部即为网格空间的叠加动画二维混合。

## 常见误区

**误区一：叠加动画等同于普通动画混合**。普通混合（Blend）在两个姿势之间做线性插值，权重0.5意味着取两个动画各50%的绝对旋转值，因此两个动画必须兼容（如同为站立状态）。叠加动画的权重0.5意味着应用50%的**偏差增量**，基础动画保持完整，因此可以叠加到任何基础状态而不产生姿势冲突。混淆两者会导致跑步时触发"受伤动画"后角色滑步——因为受伤动画被当作绝对姿势处理，覆盖了腿部移动数据。

**误区二：参考姿势任意选取无影响**。错误选用参考姿势（如将T-Pose改为蹲伏姿势）会使叠加动画在角色处于非蹲伏状态时产生错误的累积偏差。若基础动画本身就含有15°的脊柱弯曲（如步行动画），而参考姿势是T-Pose（0°），那么叠加动画的差值将包含这额外的15°，叠加后脊柱旋转量会超出预期，出现夸张的弯腰幅度。

**误区三：叠加权重可以无限叠加多层**。每叠加一层都会在骨骼旋转上累积偏差，多个叠加动画同时应用时必须注意总偏差量是否超出骨骼IK约束上限。以脊柱为例，若受伤叠加贡献20°弯曲，疲劳叠加再贡献25°弯曲，两者同时权重为1时脊柱将弯曲45°，可能与地面穿插或触发反向运动学的异常求解。

## 知识关联

叠加动画依赖**根运动**提供可靠的基础位移数据：根运动确保角色的位移由动画本身驱动，叠加动画只修改骨骼旋转偏差而不干扰根骨骼的位移轨迹，二者分工明确——若基础动画未启用根运动而使用代码控制位移，叠加动画在物理边界处理上会出现视觉与碰撞不同步的问题。

叠加动画的单层应用是引入**动画层系统**的直接前提。当项目需要对不同骨骼组（上半身/下半身）分别应用叠加效果，且每层需要独立的状态机管理时，单一叠加动画节点已无法满足需求，必须过渡到完整的多层动画架构。动画层系统本质上是将多个叠加动画节点组织为带优先级、遮罩（Mask）和混合模式的层级结构，叠加动画的差值计算逻辑在每一层中独立运行。
