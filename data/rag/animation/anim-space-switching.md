# 空间切换

## 概述

空间切换（Space Switching）是骨骼绑定中控制IK目标或约束目标参照哪个坐标系运动的技术。当角色的手部IK控制器需要在"跟随身体移动"和"固定在世界空间某点"之间切换时，例如角色抓住一根固定的栏杆，手部控制器必须从"局部空间跟随父级骨骼"切换到"世界空间锁定位置"，而不产生任何位置跳跃，这就是空间切换要解决的核心问题。

该技术在2000年代随着游戏引擎和三维动画软件（如Maya、3ds Max）的成熟而逐步标准化。早期动画师通过手动K帧来模拟空间切换效果，但随着约束系统（Constraint System）的发展，绑定师开始用程序化方法实现无缝切换。现代实现中，空间切换通常依赖于"约束权重插值"或"驱动关键帧（Driven Key）"配合一个枚举型属性（ENUM attribute）来驱动多个父级空间之间的混合。

空间切换的核心价值在于它解决了骨骼层级结构的固有矛盾：IK目标天然依附于某个父级节点，但动画需求却要求它在多个参照系之间自由切换。没有空间切换，动画师每次切换参照系都必须手动补偿偏移量，极大增加了工作量，也容易引入误差。

## 核心原理

### 三类空间的定义与区别

在Maya绑定体系中，空间切换涉及三种主要坐标系：

- **世界空间（World Space）**：以场景原点为参照，控制器坐标不受任何骨骼运动影响。适用于手持固定物体（握住门把手、站在地面推墙）的场景。
- **局部空间（Local Space / Parent Space）**：控制器跟随其父级骨骼运动，相当于绑定在父节点坐标系内。适用于手臂自然放在身体两侧随身体移动的情况。
- **自定义空间（Custom Space）**：以场景中任意指定节点为参照，例如以"头部骨骼"为父级，实现手部跟随头部的效果（如托腮动作）。

### 约束权重混合机制

技术实现上，空间切换依赖**父级约束（Parent Constraint）的权重插值**。设控制器对两个父级节点 A（世界空间代理节点）和 B（局部空间根节点）分别有权重 `wA` 和 `wB`，则切换条件为：

```
wA + wB = 1
```

当 `wA = 1, wB = 0` 时锁定世界空间；当 `wA = 0, wB = 1` 时跟随父级空间。切换时，动画师必须在切换帧的前后各插入一帧权重关键帧（通常间隔1帧），保证过渡点的权重值精确为0或1，避免约束权重处于中间值导致"漂移"。

### 无缝切换的补偿计算

空间切换最关键的技术难点是**切换瞬间位置不跳跃**。Maya的 `Match Transform` 功能或自定义Python脚本会在切换前计算控制器在目标空间中的等效变换值，然后同时设置控制器的平移/旋转关键帧和约束权重关键帧。设切换发生在第 N 帧：

- 第 N-1 帧：在旧空间记录控制器世界矩阵 M_world
- 第 N 帧：将 M_world 转换到新空间的局部矩阵，设置控制器位置，并将权重切换到新空间

这个矩阵转换公式为：`M_local_new = M_world × inverse(M_parent_new)`，其中 `M_parent_new` 是新父级节点在第 N 帧的世界矩阵。

### ENUM属性驱动

在实际绑定中，绑定师在控制器上添加一个名为"Space"的ENUM属性，其枚举值对应不同空间（如 0=World, 1=Body, 2=Head）。通过驱动关键帧将该ENUM值映射到各个父级约束的权重，动画师只需拨动一个属性值即可完成空间切换，无需直接操作约束权重。

## 实际应用

**角色拾取物体**：角色从桌上拿起一个杯子。杯子控制器初始处于世界空间（放在桌上不随身体移动），当手接触杯子的帧，切换到手部骨骼的局部空间，杯子随手部运动。切换帧通常选在接触最近的那一帧，此时视觉上即使有微小偏差也不易察觉。

**攀爬绑定**：角色双手抓住攀爬架时，左右手的IK目标分别切换到世界空间。身体控制器向上移动时，双手保持在攀爬架的固定点，产生真实的撑体感。若不切换空间，手部会随盆骨控制器上移而脱离抓握点。

**头部跟随手势**：演讲角色的手靠近脸部时，手部控制器切换到头部空间，实现"手跟着头一起转动"的效果，避免头部转向时手部静止在世界空间中显得突兀。

**Blender中的实现**：在Blender的Copy Location约束中，Target Space 和 Owner Space 的组合选项直接对应空间切换概念，绑定师通过Influence属性（范围0.0~1.0）实现两个空间之间的混合。

## 常见误区

**误区一：在切换帧上只设置权重关键帧而不设置位置关键帧。**
很多初学者以为切换约束权重就够了，实际上如果切换前后控制器的局部坐标值没有同步更新，切换瞬间控制器会跳到新父级坐标系下的旧局部坐标所对应的世界位置，产生明显跳跃。正确做法是在同一帧同时设置补偿后的位置/旋转关键帧与权重关键帧。

**误区二：将空间切换的ENUM权重设置为中间值（如0.5/0.5）以获得"混合"效果。**
父级约束在两个空间权重相等时，控制器会被拉向两个父级位置的算术平均点，这在大多数情况下产生物理上不真实的结果（手会悬浮在身体和世界位置的中间）。空间切换的权重应在单帧内从0跳变到1，不应用来做渐变动画。

**误区三：忽略旋转补偿只做位置补偿。**
空间切换同时影响平移和旋转。若父级节点存在旋转差异（如头部空间相对世界空间有30度仰角），仅补偿位置会导致控制器朝向在切换帧突变。必须同时将 `M_world` 中的旋转分量也转换到新父级空间并设置关键帧。

## 知识关联

空间切换建立在**绑定约束**的基础之上——父级约束、点约束（Point Constraint）和方向约束（Orient Constraint）是实现空间切换的三种基本工具，理解约束权重的累加规则（多个约束权重之和为1时产生加权平均）是学习空间切换的前提。

空间切换与**IK/FK切换**在实现机制上高度相似，两者都使用ENUM属性驱动约束权重，都需要在切换帧设置补偿变换值。不同之处在于IK/FK切换操作的是解算模式，而空间切换操作的是参照坐标系。在复杂的游戏角色绑定中，这两类切换常常叠加使用，例如在FK模式下的手臂同时需要支持世界/局部空间切换，要求绑定师对约束求值顺序（Constraint Evaluation Order）有清晰认识，避免循环依赖（Cycle Dependency）导致解算错误。
