# 运算符

## 概述

运算符（Operator）是编程语言中用于对一个或多个操作数执行特定计算或逻辑操作的符号。在Python中，运算符分为七大类：算术运算符、比较运算符、赋值运算符、逻辑运算符、位运算符、成员运算符和身份运算符。运算符本质上是编译器或解释器内置的函数简写形式，例如 `+` 对应整数加法时，Python底层调用的是 `int.__add__()` 方法。

运算符的概念最早可追溯至1950年代，随着FORTRAN语言（1957年发布）的出现，`+`、`-`、`*`、`/` 这四个算术运算符被正式引入高级编程语言。Python语言（1991年首发布）在此基础上扩展了 `**`（幂运算）、`//`（整除）、`%`（取模）等运算符，极大丰富了表达能力。

在AI工程的编程实践中，运算符直接影响模型训练数据的预处理逻辑。例如，利用 `%` 运算符可以快速划分训练集与验证集（`index % 5 == 0` 表示每5条取1条作为验证集），利用比较运算符可以过滤异常值，利用逻辑运算符可以组合多个筛选条件。掌握运算符是编写任何数据处理脚本的基础前提。

## 核心原理

### 算术运算符与运算优先级

Python的七个算术运算符按优先级从高到低排列为：`**`（幂）> `+x/-x`（一元正负）> `*`、`/`、`//`、`%` > `+`、`-`（加减）。这一优先级规则意味着 `2 + 3 * 4` 的结果是 `14` 而非 `20`。其中 `//` 执行**地板除法**（向负无穷取整），例如 `-7 // 2` 的结果是 `-4` 而非 `-3`，这与C语言的整除行为不同，是Python特有的设计。`%` 取模运算符的结果符号始终与**除数**相同，例如 `-7 % 3 = 2`（因为 `-7 = (-3)*3 + 2`）。

### 比较运算符与返回值类型

Python的六个比较运算符 `==`、`!=`、`>`、`<`、`>=`、`<=` 均返回布尔值 `True` 或 `False`。值得注意的是，Python支持**链式比较**语法：`0 < x < 10` 在Python中合法且等价于 `0 < x and x < 10`，而在Java或C语言中这种写法会产生错误结果。在AI数据清洗场景中，`df['age'] >= 0` 会返回一个布尔型Series，可直接用于Pandas的行过滤操作。

### 逻辑运算符的短路特性

Python的三个逻辑运算符 `and`、`or`、`not` 具有**短路求值**（Short-circuit Evaluation）特性。`and` 运算符在左侧为假时直接返回左侧值，不再求值右侧；`or` 运算符在左侧为真时直接返回左侧值。具体规则是：`A and B` 若 `A` 为假则返回 `A` 本身，否则返回 `B`；`A or B` 若 `A` 为真则返回 `A` 本身，否则返回 `B`。因此 `0 and func()` 不会调用 `func()`，这在避免空指针或除零错误时非常有用。逻辑运算符的优先级为 `not` > `and` > `or`。

### 赋值运算符与增量赋值

Python提供了 `+=`、`-=`、`*=`、`/=`、`//=`、`%=`、`**=` 七个增量赋值运算符。`x += 1` 对于不可变类型（如 `int`）等价于 `x = x + 1`，会创建新对象；但对于可变类型（如 `list`），`lst += [1]` 等价于 `lst.extend([1])`，原地修改对象，与 `lst = lst + [1]` 创建新列表的行为不同。此差异在AI训练循环中累积损失值（`total_loss += batch_loss`）时不会引发问题，但在处理列表参数时可能产生意外的副作用。

## 实际应用

**数据归一化预处理**：在准备神经网络输入数据时，最小-最大归一化公式为 `x_norm = (x - x_min) / (x_max - x_min)`，同时使用了 `-` 和 `/` 两个算术运算符。若 `x_max == x_min`（所有值相同），直接运算会触发除零错误，因此需要用比较运算符提前判断：`x_norm = 0 if x_max == x_min else (x - x_min) / (x_max - x_min)`。

**批次索引切片**：在mini-batch梯度下降中，利用 `//` 和 `%` 运算符计算当前批次：`batch_id = sample_index // batch_size`，`position_in_batch = sample_index % batch_size`。当 `batch_size=32` 时，第100个样本属于第3批次（`100 // 32 = 3`），在批内位置为第4个（`100 % 32 = 4`）。

**条件过滤组合**：在过滤训练数据时，`is_valid = (label >= 0) and (label < num_classes) and (not is_corrupted)` 使用比较运算符与逻辑运算符组合，一次性排除标签越界和损坏样本。

## 常见误区

**误区一：混淆 `=` 与 `==`**
`=` 是赋值运算符，将右侧值绑定到左侧变量；`==` 是比较运算符，返回布尔值。在条件判断中错误写成 `if x = 5:` 会导致Python直接报 `SyntaxError`（Python3中已禁止此写法），但在某些旧版语言（如C）中这是合法但危险的写法，会将5赋给x后判断非零为真。初学者应养成在比较时只用 `==` 的习惯。

**误区二：整除 `//` 对负数的行为误判**
许多学习者认为 `//` 就是"去掉小数部分"，但正确理解是**向负无穷方向取整**。`7 // 2 = 3`（正确），`-7 // 2 = -4`（很多人误以为是 `-3`）。这在计算数据集分片时若存在负数索引会产生意料外的结果，应特别注意。

**误区三：`and`/`or` 返回的不一定是布尔值**
`1 and 2` 返回 `2`（而非 `True`），`0 or 'default'` 返回 `'default'`（而非 `True`）。这种行为虽然可以用于编写简洁代码（如 `name = user_input or 'Anonymous'`），但在AI代码中若将逻辑运算结果直接作为数值参与计算，可能导致类型错误或数值错误。

## 知识关联

**前置知识衔接**：运算符直接作用于**变量与数据类型**中学习的各种类型值。不同数据类型对同一运算符的行为不同：`+` 对 `int` 执行加法（`3 + 4 = 7`），对 `str` 执行拼接（`'a' + 'b' = 'ab'`），对 `list` 执行合并（`[1] + [2] = [1, 2]`）。这种**运算符重载**现象源于Python面向对象机制中各类型实现了不同的 `__add__` 方法。

**后续知识铺垫**：**条件判断（if/else）** 的判断表达式几乎全部依赖比较运算符和逻辑运算符的组合。`if score >= 60 and score < 80:` 这一典型条件语句中，`>=`、`<` 产生两个布尔值，`and` 将它们合并为最终判断依据。若运算符的优先级和短路特性不熟练，就无法正确构造 `if/else` 的复合条件，会导致逻辑错误难以调试。
