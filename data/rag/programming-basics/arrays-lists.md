# 数组/列表

## 概述

数组（Array）和列表（List）是存储**同一变量名下多个元素的有序集合**，通过整数索引访问每个元素。最基础的形式是静态数组——在内存中占用一块连续的地址空间，长度在创建时固定；Python 的 `list` 则是动态数组，可以随时增删元素，底层自动扩容。理解二者的区别是 AI 工程中高效处理数据的第一步。

数组的概念最早随汇编语言和 Fortran（1957年）确立，Fortran 的数组语法 `DIMENSION A(10)` 直接影响了后续几乎所有编程语言的设计。Python 在 1991 年引入内置 `list` 类型，将动态扩容封装在语言内部，让开发者无需手动管理内存。NumPy 于 2006 年发布，提供了专为数值计算优化的 `ndarray`，成为 AI 工程中张量运算的基础。

在 AI 工程中，训练数据集、模型权重、批量预测结果全部以数组或多维数组的形式在内存与 GPU 之间传输。掌握数组的索引、切片、形变操作，是读懂任何深度学习代码的前提条件。

## 核心原理

### 索引与内存布局

数组的第一个元素索引为 **0**，而非 1，这直接来自内存寻址公式：

```
元素地址 = 基地址 + 索引 × 单个元素字节数
```

对于一个存储 `float32`（4 字节）的数组，索引 3 的元素地址 = 基地址 + 3 × 4 = 基地址 + 12。正因如此，随机访问任意索引的时间复杂度为 **O(1)**，与数组长度无关。

Python `list` 存储的不是数据本身，而是指向各对象的**指针**（64位系统上每个指针占 8 字节），因此可以混合存储不同类型，但会额外消耗内存。NumPy `ndarray` 则直接存储原始数值，访问速度比 `list` 快 10–100 倍。

### 切片操作（Slicing）

Python 数组/列表支持 `a[start:stop:step]` 语法，三个参数均可省略。`a[1:4]` 返回索引 1、2、3 的元素（**不含** stop=4），`a[::2]` 每隔一个取一个，`a[::-1]` 反转整个列表。

NumPy 的切片返回**视图（view）**而非副本，这意味着修改切片会同时修改原数组。例如：

```python
import numpy as np
a = np.array([10, 20, 30, 40])
b = a[1:3]   # b 是视图，不是新数组
b[0] = 99
print(a)     # [10, 99, 30, 40]  ← 原数组被修改！
```

Python 原生 `list` 的切片则返回**浅拷贝**，行为与此不同——这是工程中极易混淆的细节。

### 动态扩容机制

Python `list` 底层采用**预分配策略**：每次容量不足时，新容量约为旧容量的 **1.125 倍**（CPython 实现中具体为 `new_size = old_size + (old_size >> 3) + 6`）。这使得 `append()` 操作的**均摊时间复杂度为 O(1)**，但在极端情况下单次 `append` 可能触发 O(n) 的内存复制。

相比之下，在列表**头部**插入元素（`list.insert(0, x)`）需要将所有现有元素向后移动一位，时间复杂度为 **O(n)**，数据量大时性能极差——这是选用链表的重要动机。

### 多维数组与形状

AI 中的图像、文本嵌入等数据都以多维数组表示。NumPy 用 `shape` 属性描述各维度大小，例如一批 32 张 224×224 的 RGB 图像，形状为 `(32, 224, 224, 3)`，共包含 32 × 224 × 224 × 3 = **4,816,896** 个 `float32` 数值，占用约 **18.4 MB** 内存。`reshape()`、`transpose()` 等操作在不移动数据的前提下改变形状，底层只修改元数据。

## 实际应用

**批量推理数据组织**：将 64 条文本的嵌入向量堆叠成形状 `(64, 768)` 的 NumPy 数组，一次性传入模型，比循环逐条预测快约 30–50 倍，原因是矩阵乘法可以利用 GPU 的并行计算。

**数据预处理流水线**：在图像分类任务中，通常用 Python `list` 收集文件路径，再用 NumPy 数组批量加载像素值。`list` 负责灵活的增删（如过滤损坏文件），`ndarray` 负责高效的数值运算——二者分工明确。

**滑动窗口特征提取**：时序预测任务中，对长度为 N 的序列用步长 1、窗口大小 W 做切片，生成形状 `(N-W+1, W)` 的二维数组，每行是一个训练样本。用 NumPy 的 `stride_tricks.as_strided` 实现此操作无需复制数据，内存效率极高。

## 常见误区

**误区一：Python `list` 等同于数组**
`list` 存储的是对象指针，元素可以是任意类型，内存不连续（相对于数值本身）。真正意义上的数值数组是 `array.array` 或 `numpy.ndarray`。在 AI 工程中直接用 `list` 做矩阵运算，不仅慢，而且语义模糊，应始终优先使用 NumPy。

**误区二：切片不会消耗额外内存**
Python `list` 切片创建浅拷贝，新列表的指针数组是全新分配的，对大列表多次切片会显著占用内存。NumPy 的切片虽然是视图（不复制数据），但 `np.copy()` 或某些运算（如 `np.sort()`）会隐式创建副本，需要通过 `np.shares_memory(a, b)` 主动验证。

**误区三：负索引越界**
Python 中 `a[-1]` 合法，等价于 `a[len(a)-1]`，但 `a[-len(a)-1]` 会抛出 `IndexError`。NumPy 的负索引规则相同，但在多维数组中，每个维度独立计算，`a[-1, -1]` 指最后一行最后一列，初学者常误以为是某种"倒序遍历"。

## 知识关联

**前置知识**：变量与数据类型确定了每个数组元素存储的具体类型（`int32`、`float64`、`str` 等），数据类型直接决定数组的内存占用和运算精度。

**后续概念**：字典/映射在数组的键值对场景中补充了数组索引必须为整数的局限；数组内部原理将深入解释本文提到的连续内存布局和扩容算法；链表通过指针链接解决了数组头部插入 O(n) 的性能问题；栈和队列则是对数组端点操作的抽象封装，分别限制只访问尾部（`list.append/pop`）或首尾（`collections.deque`）。
