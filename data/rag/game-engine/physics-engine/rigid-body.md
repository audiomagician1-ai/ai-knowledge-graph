# 刚体动力学

## 概述

刚体动力学研究在力和扭矩作用下，不发生形变的理想固体的运动规律。在游戏物理引擎中，"刚体"（Rigid Body）指形状和内部质量分布完全固定的物体——无论受到多大外力，物体上任意两点之间的距离保持不变。这一理想化假设使运动方程可以精确求解，是实时物理模拟的核心数学基础。

刚体动力学的理论根基来自牛顿第二定律和欧拉于1765年提出的刚体转动方程（Euler's equations of rigid body rotation）。现代游戏引擎如PhysX、Bullet和Havok均将刚体作为最基础的物理仿真对象，每帧以固定时间步长（通常为1/60秒或1/120秒）对所有刚体的线速度、角速度、位置和朝向进行数值积分更新。

刚体动力学之所以在游戏引擎中不可或缺，是因为它以最小的计算代价实现了令人信服的物理效果：一个场景中可以同时存在数百个活跃刚体，每个刚体只需存储质量、惯性张量、线动量和角动量等少量状态变量，即可完整描述其物理行为。

## 核心原理

### 线性运动方程

刚体的线性运动由牛顿第二定律直接给出：

**F = ma**

其中 **F** 是作用在刚体质心上的合外力（牛顿），**m** 是刚体总质量（千克），**a** 是质心的线加速度（米/秒²）。在引擎中，通常以线动量 **p = mv** 的形式存储状态，每帧通过 **p(t+Δt) = p(t) + F·Δt** 更新动量，再除以质量得到速度。重力加速度 g = 9.8 m/s² 作为常驻力在每帧施加给所有未被冻结的刚体。

### 转动运动方程与惯性张量

刚体的转动由欧拉方程描述：

**τ = I·α**

其中 **τ** 是合扭矩（牛顿·米），**I** 是惯性张量（3×3对称矩阵，单位为千克·米²），**α** 是角加速度（弧度/秒²）。惯性张量 **I** 是刚体动力学区别于质点动力学的关键——它描述了物体质量相对于旋转轴的空间分布。对于均质长方体（尺寸为 a×b×c，质量为 m），沿主轴的转动惯量分别为：

- Iₓₓ = m(b² + c²)/12
- Iyy = m(a² + c²)/12  
- Izz = m(a² + b²)/12

在世界坐标系中，惯性张量会随物体旋转而变化：**I_world = R · I_local · Rᵀ**，其中 R 为旋转矩阵。引擎通常只存储局部坐标系中的对角化惯性张量（即主惯性矩），每帧通过旋转矩阵变换到世界空间。

### 角动量与角速度的数值积分

引擎以角动量 **L = I·ω** 作为积分状态变量（而非直接存储角速度），原因是角动量在无外力矩时守恒，数值上更稳定。每帧更新：

1. **L(t+Δt) = L(t) + τ·Δt**
2. **ω = I_world⁻¹ · L**
3. 用 ω 更新四元数表示的朝向：**q(t+Δt) = q(t) + 0.5·[0,ω]·q(t)·Δt**，结果归一化

使用四元数（Quaternion）而非欧拉角存储旋转，避免了万向节锁（Gimbal Lock）问题，这是PhysX、Bullet等引擎的标准实现方式。

### 扭矩的产生

扭矩由偏心力产生：当力 **F** 作用在距质心位移为 **r** 的点时，产生扭矩 **τ = r × F**（叉积）。例如，向右上方推动一个静止箱子的右下角，会同时产生向右的线加速度和顺时针的角加速度，这正是"施力点偏离质心"导致旋转的数学本质。

## 实际应用

**爆炸冲击波模拟**：当爆炸发生时，引擎对周围每个刚体施加一个从爆炸中心指向外的冲量（Impulse = F·Δt）。由于力的作用点往往不在各刚体质心，每个物体不仅会被抛飞（线速度变化）还会翻滚（角速度变化），产生自然的混乱感。

**车门被撞开**：车门绕铰链轴旋转，铰链约束将车门的自由度限制到一个旋转轴上。碰撞力经叉积计算得到绕铰链轴的扭矩，引擎通过约束求解器将多余的5个自由度的力分量消除，仅保留转动效果。

**堆叠物体稳定性**：在《半条命2》等游戏中，用物理堆叠的木箱依赖惯性张量的差异来区分行为：宽扁箱子（Izz 较大）比细高箱子更难被推倒，因为绕竖轴旋转需要克服更大的转动惯量。

**布娃娃系统（Ragdoll）**：人体骨骼的每块骨头对应一个刚体，相互之间以铰链或球窝约束连接。死亡动画切换至布娃娃模式时，各骨骼刚体的初始线速度和角速度从动画蒙皮速度中采样，之后完全由刚体动力学方程驱动。

## 常见误区

**误区一：扭矩等于力**。初学者常将力（Force）和扭矩（Torque）混用，认为"给物体施加一个向右的力"就会让它向右移动而不旋转。实际上只有力恰好通过质心时才是纯平移，偏心力必然同时产生扭矩。在引擎API中，`AddForce()` 和 `AddForceAtPosition()` 是两个不同函数，后者内部自动计算并叠加扭矩。

**误区二：惯性张量是标量**。许多人将转动惯量理解为一个数字，认为"质量越大越难转"。实际上惯性张量是3×3矩阵，同一物体绕不同轴旋转的阻力完全不同——一根细长棍绕长轴旋转（Iₓₓ 极小）几乎无阻力，绕短轴旋转（Iyy 极大）则非常困难。忽略这一点会导致物理行为严重失真。

**误区三：每帧直接积分角速度就足够**。直接对角速度进行欧拉积分（ω += α·Δt，然后更新角度）在旋转较快时会导致能量漂移和旋转不稳定。正确做法是以角动量为积分量，且旋转四元数在每帧更新后必须重新归一化（`q = normalize(q)`），否则累积误差会使物体形状"膨胀"或仿真发散。

## 知识关联

**前置知识——物理引擎概述**为刚体动力学提供了仿真循环（Simulation Loop）、时间步长管理和宽相/窄相碰撞流水线的框架背景。理解刚体动力学要求掌握物理引擎如何组织每帧的力积累、积分和状态更新这一完整流程。

**后续概念——碰撞检测**需要在刚体运动之后判断两个物体是否重叠：碰撞发生时产生接触法线和穿透深度，这些数据会转化为冲量（线冲量 + 角冲量）反馈回刚体的线动量和角动量，形成完整的碰撞响应闭环。

**后续概念——载具物理**在刚体动力学基础上添加了轮胎摩擦模型、悬挂弹簧约束和差速驱动力，车身本身就是一个刚体，四个车轮通过约束与车身刚体耦合。

**后续概念——流体模拟**中的浮力计算需要知道刚体的体积和质心位置，流体对刚体施加的阻力和升力最终也通过合力/合扭矩的接口输入刚体运动方程，两个模块通过力/扭矩接口解耦。
