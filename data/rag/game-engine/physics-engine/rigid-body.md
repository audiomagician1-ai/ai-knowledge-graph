# 刚体动力学

## 概述

刚体动力学研究的是在外力和扭矩作用下，不发生形变的理想刚性物体如何运动。所谓"刚体"，指物体上任意两点之间的距离在运动过程中保持不变——这与软体或布料模拟有本质区别。游戏引擎中的大多数可交互物体（木箱、车辆车身、武器）都被建模为刚体，因为刚体假设在保证视觉真实感的同时，将每帧的计算量控制在可接受范围内。

刚体动力学的数学基础由欧拉在18世纪中叶系统整理：他于1765年发表的《刚体运动理论》将牛顿第二定律推广到旋转运动，引入了惯性张量的概念。现代游戏物理引擎（如PhysX、Bullet、Havok）直接继承了欧拉方程的离散化形式，以每秒60至120步的固定时间步长推进仿真。

刚体动力学在游戏引擎中的重要性体现在：它是碰撞响应、载具悬挂、布娃娃系统的共同底层。当玩家踢飞一个油桶时，背后至少涉及线速度积分、角速度积分和冲量分配三个刚体动力学步骤，缺少任何一步都会导致物体运动失真。

## 核心原理

### 线性运动：牛顿第二定律的离散化

刚体的质心线性运动由以下方程描述：

**F = m · a**，展开为积分形式：

- 速度更新：**v(t+Δt) = v(t) + (F/m) · Δt**
- 位置更新：**x(t+Δt) = x(t) + v(t+Δt) · Δt**

其中 F 是所有外力（重力、碰撞冲量、弹簧力）的矢量合力，m 是标量质量，Δt 是时间步长（典型值为 1/60 秒）。游戏引擎通常存储质量的倒数 **1/m**，从而将除法转为乘法，在同时模拟数百个刚体时可节省约15%的计算时间。静态物体（地面、墙壁）则将 1/m 设为0，使其对任意大小的力都不产生加速度。

### 旋转运动：欧拉方程与惯性张量

旋转运动的核心方程是欧拉刚体方程：

**τ = I · α + ω × (I · ω)**

其中：
- **τ**（tau）= 扭矩矢量，单位 N·m
- **I** = 3×3 惯性张量，描述物体质量相对于三条旋转轴的分布
- **α** = 角加速度矢量
- **ω** = 当前角速度矢量
- **ω × (I · ω)** = 科里奥利项，在物体坐标系下旋转时产生，对陀螺进动等效果不可忽略

惯性张量 I 对均匀实心长方体（尺寸为 2a × 2b × 2c，质量 m）的对角元素为：
- **Ixx = m(b²+c²)/3**
- **Iyy = m(a²+c²)/3**
- **Izz = m(a²+b²)/3**

游戏引擎在世界坐标系下需要将局部惯性张量通过旋转矩阵 R 转换：**I_world = R · I_local · Rᵀ**，每步都需要重新计算这一3×3矩阵乘法。

### 四元数姿态积分

刚体的旋转姿态不能用欧拉角直接积分，否则会产生万向锁问题。主流引擎用单位四元数 **q** 表示姿态，角速度更新公式为：

**q(t+Δt) = q(t) + 0.5 · Δt · [0, ω] · q(t)**

更新后必须对 q 做归一化（**q = q / |q|**），防止浮点误差累积导致四元数偏离单位球面。Bullet 引擎在每步积分后强制执行此归一化，每隔约1000步会产生约 10⁻⁶ 量级的漂移，不归一化则漂移呈指数增长。

### 阻尼与休眠机制

真实物理仿真在无阻尼情况下能量不会耗散，导致物体永久抖动。引擎对每个刚体施加线性阻尼系数 **kd_linear**（典型值0.01–0.1）和角阻尼系数 **kd_angular**（典型值0.05–0.2），每步乘以 **(1 - kd · Δt)** 来模拟空气阻力和内部摩擦。当刚体的线速度低于 0.01 m/s 且角速度低于 0.01 rad/s 并持续 20 帧以上，引擎通常将其标记为"休眠"状态，完全跳过该刚体的动力学更新，这是处理静止场景中大量物体的关键性能优化。

## 实际应用

**爆炸冲量分配**：射击游戏中手雷爆炸时，引擎对爆炸半径内每个刚体施加冲量 **J = F · Δt**，同时计算冲量作用点相对于质心的偏移矢量 **r**，产生扭矩冲量 **L = r × J**，使被炸飞的木箱既向外飞出又翻滚旋转。Unity 的 `AddExplosionForce` API 正是封装了这一线冲量加角冲量的双重施加逻辑。

**布娃娃系统**：角色死亡时，骨骼中的每根骨骼被替换为一个刚体，相邻骨骼之间用约束关节（铰链或球窝）连接。每根骨骼独立运行刚体动力学方程，关节约束通过施加修正冲量来满足，形成完整的布娃娃链式响应。Havok 在此结构下可以在主机平台上实时模拟约60根骨骼的布娃娃。

**投掷物预测轨迹**：在多人射击游戏中，客户端需要预测手雷落点。由于手雷被建模为刚体（且飞行中旋转），服务器和客户端都运行同一套刚体积分方程，只要初始状态相同、时间步长一致（通常固定为16.67ms），双方的预测轨迹误差可控制在毫米级。

## 常见误区

**误区1：质量越大，物体就越难被推动，仅此而已**
实际上质量只影响线性加速度，而转动惯量（惯性张量的迹）决定旋转的难易程度。一根细长金属棒和一个同质量的实心球，在相同扭矩下的角加速度相差可达数倍。如果只调大质量而不相应调整惯性张量的非对角分量，会得到"像陀螺一样旋转的铅球"这种视觉上完全错误的结果。

**误区2：可以用欧拉角直接对旋转状态进行每帧积分**
欧拉角（偏航/俯仰/滚转）在某些组合下会发生万向锁（Gimbal Lock），导致旋转自由度突然丢失。在高速旋转物体（如子弹、陀螺）上，这种锁死会在单帧内产生180°以上的姿态跳变，表现为物体突然"翻转"。正确做法是全程使用四元数或旋转矩阵存储姿态，仅在输出给编辑器或动画系统时再转换为欧拉角。

**误区3：时间步长越小，物理仿真越稳定**
这在基于约束求解的引擎中并不总成立。某些求解器（如顺序脉冲法 Sequential Impulse）在时间步长过小时会导致约束修正量累加振荡，反而使关节连接的刚体链产生抖动。PhysX 官方文档建议时间步长不小于 1/120 秒；低于此值时需同时降低约束求解的修正系数。

## 知识关联

刚体动力学建立在物理引擎概述中介绍的"宽相/窄相碰撞检测—动力学求解—约束满足"管线之上：本文描述的力和扭矩积分发生在该管线的动力学求解阶段，是碰撞检测拿到接触点之后、约束求解器计算修正冲量之前的核心步骤。

掌握刚体动力学中的惯性张量和冲量方程后，可直接进入**碰撞检测**的学习，理解为何碰撞响应需要计算"有效质量"（Effective Mass）——这正是惯性张量在接触法线方向上的投影。**载具物理**在刚体基础上增加了车轮悬挂弹簧和轮胎侧偏力模型，其车身本身即为一个标准刚体，只是受力来源更复杂。而**流体模拟**中的浮力计算，本质上也是对浸入流体的刚体施加与排开液体体积成比例的向上合力，刚体动力学方程同样适用。
