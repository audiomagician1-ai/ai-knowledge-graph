# 物理调试可视化

## 概述

物理调试可视化（Physics Debug Visualization）是游戏引擎中将物理仿真的内部数据结构以几何图形的方式叠加渲染在游戏画面上的技术手段。它让开发者能够直接看到碰撞体的形状与位置、射线投射的路径、关节约束的锚点，以及刚体的速度向量等仅存在于物理世界中的隐形数据，从而快速定位碰撞穿透、错误触发区域或关节抖动等问题。

这一功能最早在专业物理中间件 Havok（2000 年商业化）和 PhysX（前身 NovodeX，2002 年）的 SDK 中作为调试渲染接口出现。早期需要开发者自行实现渲染回调，将物理引擎暴露的几何数据提交给图形管线。现代引擎如 Unity（通过 `Physics.DrawRay`、Gizmos 系统）和 Unreal Engine（通过控制台命令 `show Collision`）已将其封装为一键开启的编辑器功能。

物理调试可视化之所以不可替代，是因为物理计算使用的碰撞形状（Collision Shape）往往与渲染网格完全分离。一个角色的视觉模型可能是精细的 15000 面网格，而物理引擎实际使用的是简化的胶囊体（Capsule），两者若发生偏移，仅靠游戏画面无法察觉，必须开启调试可视化才能看出差距。

## 核心原理

### 碰撞体的几何绘制

物理引擎维护一个与渲染场景平行的"物理世界"，其中每个刚体（Rigid Body）挂载一个或多个碰撞体（Collider/Shape）。调试可视化系统在每帧结束物理步进（通常固定步长为 1/60 秒或 1/50 秒）后，从物理世界提取这些形状的变换矩阵和几何参数，调用图形 API 的线框（Wireframe）绘制接口进行叠加渲染。

不同碰撞体类型有不同的绘制规则：
- **球体（Sphere）**：绘制三个正交大圆（XY、YZ、XZ 平面各一个），用参数方程 `x = r·cos(θ), y = r·sin(θ)` 近似，通常以 32 段折线逼近。
- **胶囊体（Capsule）**：绘制两个半球轮廓线加两条侧面切线，半径 r 和高度 h 直接取自物理数据。
- **凸包（Convex Hull）**：遍历所有凸包面的边，绘制边框。
- **三角网格（Triangle Mesh）**：仅用于静态物体，绘制每条三角边，面数多时性能开销显著。

颜色编码是碰撞体可视化的重要约定：Unity 中处于休眠状态（Sleeping）的刚体碰撞体呈灰色，激活状态呈绿色，Trigger 触发器呈青色，这让开发者一眼就能判断刚体的唤醒状态。

### 射线与形状投射的可视化

射线投射（Raycast）和形状投射（Shapecast/Sweep）是物理引擎最常用的查询操作，但其执行过程完全不可见。调试可视化通过在查询调用点记录起点、方向和最大距离，将整条射线以彩色线段绘制出来。

`Physics.DrawRay(origin, direction * maxDistance, Color.red, duration)` 是 Unity 中最常用的射线可视化写法，`duration` 参数控制线段在场景视图中的留存帧数，设为 0 表示仅当帧可见。当射线命中目标时，通常在命中点（`RaycastHit.point`）额外绘制一个小球或叉形标记，并显示法线方向向量，帮助判断命中面的朝向是否符合预期。

胶囊体扫射（CapsuleCast）的可视化更复杂，需要在起点和终点各绘制一个胶囊体轮廓，并在两者之间绘制连接的侧面线，形成"扫过的体积"示意图。这对于角色控制器的地面检测和障碍物扫描调试尤为关键。

### 约束与关节的可视化

关节（Joint）约束定义了两个刚体之间允许的相对运动自由度。可视化系统将约束表示为：
- **锚点（Anchor Point）**：在世界坐标或局部坐标中以小球或轴心标记绘制。
- **自由度轴向**：以带箭头的线段表示可平移或可旋转的方向，Unreal 中铰链关节（Hinge Joint）的旋转轴呈黄色弧形显示。
- **限制范围**：角度限制（Angular Limit）以扇形区域绘制，绿色区域表示允许运动范围，红色表示已触碰限制边界。

PhysX 的关节调试显示还会用橙色标出当前关节的违规量（Constraint Violation），即关节被外力破坏当前约束状态的程度，数值超过求解器容差（通常默认 0.01 米）时会高亮提示。

## 实际应用

**角色无法正确站在平台上**：开启碰撞体可视化后，若发现角色胶囊体底部比视觉模型脚底高出 0.2 米，说明碰撞体偏移参数（Center Offset）设置错误，角色实际是"悬浮"在平台上的视觉假象。

**子弹穿墙问题（Tunneling）**：使用射线可视化配合 `Physics.DrawRay` 在子弹每帧移动前后绘制路径。若调试画面中射线跨越了墙体却没有命中标记，说明子弹速度过高（超过 CCD 连续碰撞检测阈值），需要在刚体上启用 `Collision Detection: Continuous Dynamic` 模式。

**布娃娃关节翻转抖动**：在 Unreal 中打开 `show Constraints` 命令，可以看到关节限制扇形区域中哪些关节已碰触红色边界并在边界处反复弹跳，据此调整 `Angular Damping`（角阻尼）参数或放宽限制角度。

**触发区域范围不符合预期**：场景中不可见的 Trigger 碰撞体在开启可视化前根本无法判断其实际边界，开启后青色线框立即暴露出触发器体积是否与策划设计的区域匹配。

## 常见误区

**误区一：调试可视化显示的形状就是引擎实际使用的碰撞形状**
实际上，可视化是对物理数据的**近似绘制**，并非完全精确。例如球体的 32 段折线近似会在极近距离下看出棱角，但物理碰撞计算使用的是精确球体方程。更重要的是，若引擎存在 Bug 或版本差异，可视化渲染与实际碰撞检测所使用的变换可能存在一帧的错位（尤其是插值渲染开启时），不能将可视化结果等同于物理计算的精确状态。

**误区二：物理调试可视化对运行时性能没有影响，可以常开**
物理调试绘制会在每帧向 CPU 端提交大量线段绘制调用，场景中若有数百个刚体，三角网格碰撞体的边框绘制可能产生数万条线段，导致帧率明显下降。Unity 官方文档指出，在移动平台上大量 Gizmos 绘制可导致 GPU 负载增加 30% 以上。这一功能应仅在编辑器或专用调试构建（Debug Build）中启用，发布版本必须通过预编译宏（如 `#if UNITY_EDITOR`）或条件编译将其剔除。

**误区三：看到碰撞体重叠就一定是穿透 Bug**
物理引擎的求解器在每个时间步中会允许碰撞体之间存在微小的重叠（称为 Penetration Depth，PhysX 默认允许 0.005 米的穿透量），依靠这个重叠深度来计算分离冲量。调试可视化中两个碰撞体轮廓线略有交叉是正常的求解器行为，只有当重叠量明显超过容差阈值（肉眼可见明显穿入）时才需要排查。

## 知识关联

物理调试可视化直接依赖对**物理引擎概述**中核心概念的理解——你必须先知道碰撞体、刚体、触发器和约束各自的定义，才能看懂可视化界面中不同颜色、形状所代表的含义。例如，不了解触发器（Trigger）与碰撞体（Collider）的本质区别，就无法理解为什么两者在可视化中用不同颜色区分（Unity 中绿色 vs 青色）。

从调试可视化中获得的直觉会帮助开发者更好地理解**碰撞检测算法**（如 GJK、SAT）、**连续碰撞检测（CCD）** 的必要性，以及**约束求解器**（如 PGS 迭代求解）的收敛行为——这些都是物理引擎更深层次的主题，而物理调试可视化是通往这些主题的感性认知入口。
