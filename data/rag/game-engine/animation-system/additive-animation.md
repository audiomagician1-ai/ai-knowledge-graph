# 叠加动画

## 概述

叠加动画（Additive Animation）是一种将**差量姿态（Delta Pose）叠加到基础动画上**的技术，其核心运算不是直接播放完整的骨骼变换，而是计算某一姿态相对于参考姿态（Reference Pose）的偏差值，再将这个偏差加到当前运行中的任意基础动画上。这与混合动画（Blend）的加权平均不同——叠加动画的数学本质是向量加法，而非插值。

该技术最早在游戏行业中由Naughty Dog等工作室在PS2时代开始探索，并在《神秘海域》系列制作期间形成较为成熟的管线。虚幻引擎在UE3中正式将Additive动画纳入动画蓝图系统，提供了`Apply Additive`节点；Unity的Animator Controller则通过将AnimationClip的`Wrap Mode`与Layer叠加权重结合来实现类似效果。叠加动画的意义在于它允许制作者用极少量的额外动画资产为角色叠加出数以百计的组合姿态，大幅降低动画内存与制作成本。

## 核心原理

### 差量姿态的计算方式

叠加动画的基础是从原始动画中提取差量。给定参考姿态 $R$ 和目标姿态 $T$，差量姿态 $D$ 的计算方式为：

- **位置（平移）**：$D_{pos} = T_{pos} - R_{pos}$
- **旋转（四元数）**：$D_{rot} = T_{rot} \times R_{rot}^{-1}$
- **缩放**：$D_{scale} = T_{scale} / R_{scale}$

旋转的差量之所以使用四元数乘法而非减法，是因为旋转空间不是线性空间，直接相减无物理意义。最终叠加时，将差量旋转左乘或右乘到基础动画的骨骼旋转上：$Final_{rot} = D_{rot} \times Base_{rot}$（世界空间叠加）或 $Final_{rot} = Base_{rot} \times D_{rot}$（局部空间叠加），两种顺序会产生不同的视觉效果。

### 参考姿态的选取策略

参考姿态的选取直接决定差量动画的"语义"。虚幻引擎提供三种参考姿态模式：
1. **Reference Pose（T-Pose/A-Pose）**：以绑定姿势为参考，差量表达的是相对于绑定姿势的偏移，适用于面部表情叠加、受伤倾斜等全局偏移效果。
2. **Animation Frame 0**：以目标动画第0帧为参考，常用于将一个走路循环"叠加"到另一个循环上。
3. **Mesh Reference Pose**：与蒙皮网格的默认姿势对齐，确保叠加结果在蒙皮计算中不会产生漂移。

参考姿态选错是导致叠加动画出现骨骼"爆炸"或角色扭曲的首要原因。

### 叠加权重与混合树的协同

叠加动画节点接受一个0到1的权重参数（Alpha），当Alpha=0时基础动画完全不受影响，当Alpha=1时差量姿态被完整叠加。权重可以由动画蓝图中的浮点变量驱动，例如用角色的`AimPitch`值（范围-90°到90°）归一化后驱动瞄准叠加动画的Alpha，从而实现上半身随鼠标俯仰的效果。在混合树（Blend Tree）下游连接`Apply Additive`节点，可以先让混合树处理移动状态（走/跑/蹲），再统一叠加受伤姿态，两套逻辑完全解耦。

## 实际应用

**瞄准偏移（Aim Offset）**是叠加动画最典型的应用场景。制作者录制角色在水平0°、垂直0°时的基础持枪动画作为参考帧，再录制向上45°、向下45°、左45°、右45°等若干关键方向的持枪姿态，提取差量后放入二维混合空间，由鼠标输入的Pitch/Yaw值驱动采样，叠加到任何运动状态上，使枪口始终跟随准星。

**程序化受伤反应**是另一常见用途。当角色被击中时，播放一段持续0.3秒的身体倾斜叠加动画，Alpha从1快速衰减到0，不打断当前的跑步或攀爬动画，视觉上呈现"被子弹推动"的感觉，且无需为每个运动状态分别制作受伤变体。

**呼吸动画**也适合以叠加方式实现：录制角色静止时的胸腔起伏动画，以T-Pose为参考提取差量，然后以极低的Alpha值（0.1~0.3）持续叠加到所有动画状态上，增加角色的生命感，而不影响根骨骼的位移计算。

## 常见误区

**误区一：叠加动画等同于层混合（Layer Blending）**。层混合通过骨骼遮罩让上半身播放一套动画、下半身播放另一套，两者均是"完整姿态"的组合；而叠加动画操作的是差量，叠加层本身不能独立显示为有意义的姿势——将一个差量动画单独播放，看到的会是相对于参考姿势的奇异扭曲。混淆两者会导致在应该用层混合时误用叠加，产生双倍旋转的骨骼扭曲。

**误区二：叠加动画可以无限叠加而不累积误差**。连续施加多个叠加层时，旋转误差会随层数指数累积，尤其是在局部空间叠加中，末端骨骼（如手指、脚趾）的偏差会比根部大一个数量级。虚幻引擎官方建议单角色的叠加层不超过3~4层，超过时应考虑将多个叠加动画合并为一个或改用IK后处理。

**误区三：参考姿态在运行时可以随意更换**。差量是在离线烘焙（Cook）阶段相对于固定参考姿态计算并存储的，运行时改变参考姿态设置不会重新计算差量，只会使叠加结果语义错误。如需动态切换参考基准，必须在动画资产层面预先制作多份差量版本。

## 知识关联

叠加动画以**混合树**作为直接前置知识：混合树输出的混合姿态（Blended Pose）是叠加动画最常见的基础层输入，理解混合树的权重归一化机制有助于判断叠加动画的插入位置。叠加层的Alpha驱动逻辑与混合树的浮点参数驱动逻辑高度相似，但意义截然不同——混合树的权重影响姿态选择，叠加层的Alpha控制差量强度。

在动画系统的完整管线中，叠加动画通常位于混合树之后、IK（逆向运动学）之前。IK会在叠加完成后再做末端骨骼的精确定位修正，如将叠加了瞄准偏移后的手腕位置输入双骨骼IK，确保枪口方向与射线检测结果严格对齐，而不是仅依赖叠加动画的近似结果。
