# 叠加动画

## 概述

叠加动画（Additive Animation）是一种将"差量姿势"（Delta Pose）叠加到基础动画之上的技术。其核心不是直接播放一个完整骨骼姿势，而是计算目标动画与参考姿势之间的**差值**，再将这个差值增量叠加到任意底层动画的输出结果上。这意味着同一个叠加动画剪辑可以在角色奔跑、行走或站立等不同状态下通用，而不会破坏底层动画的运动特征。

该技术在2000年代早期随着主机游戏对动画层次化需求的增长而被广泛采用。早期实现见于《光环2》（2004年）等作品，开发者需要让角色在执行各种移动动作的同时叠加瞄准、受击或疲劳等上半身姿势，若用传统混合方式则需要为每种组合单独制作动画，成本极高。叠加技术将这一问题从 M×N 的组合爆炸压缩为 M+N 的线性增长。

叠加动画的意义在于它实现了动画内容的**正交分离**：移动逻辑与表情/状态逻辑可以由不同动画师独立制作，最终在运行时动态合成，大幅降低了AAA级游戏的动画制作成本。

## 核心原理

### 差量姿势的计算方式

叠加动画的数学基础是差量姿势的提取公式。对于骨骼的局部变换，平移分量的差量计算为：

**ΔT = T_target − T_reference**

旋转分量（四元数）的差量计算为：

**ΔQ = Q_reference⁻¹ × Q_target**

其中 `Q_reference` 是参考帧的旋转四元数，`Q_target` 是目标动画中对应帧的旋转四元数，`×` 表示四元数乘法。叠加应用时，将 ΔQ 左乘或右乘到基础姿势的旋转上（具体顺序取决于引擎约定）。Unreal Engine 4/5 中使用的是 `Q_base × ΔQ` 的顺序，而 Unity 的 Humanoid 骨骼叠加则在肌肉空间中执行类似操作。

### 参考姿势的选择对结果的影响

参考姿势（Reference Pose）的选择直接决定叠加动画的视觉效果。常见选择有三种：

1. **绑定姿势（Bind Pose / T-Pose / A-Pose）**：提取的差量包含完整骨骼偏移，叠加强度（Blend Weight）为 1.0 时效果最为显著，适合大幅度的受击反应或夸张表情。
2. **动画第一帧**：将动画剪辑的第0帧作为参考，提取相对于该初始状态的增量变化，适合循环扭动、呼吸起伏等细微叠加。
3. **指定静止姿势**：由美术师手工指定一个与角色最常见状态相近的姿势作为参考，此时在该状态下叠加强度为 1.0 时最为自然。

Unreal Engine 在动画剪辑的 **Additive Anim Type** 设置中明确提供了 `Mesh Space`（网格空间）和 `Local Space`（局部空间）两种叠加空间选项，局部空间叠加更常用，网格空间叠加则用于需要保持世界方向的特殊骨骼（如眼球）。

### 叠加权重与多层叠加

叠加动画支持浮点权重 α ∈ [0, 1]，实际应用的变换为：

**Q_result = Q_base × Slerp(Q_identity, ΔQ, α)**

当 α = 0 时叠加量为零，基础动画不受影响；α = 1 时完全应用差量。多个叠加层可以串联，每层独立持有自己的 ΔQ 和 α，按顺序依次叠加到上一层的输出结果上。这在实现"上半身瞄准 + 头部注视 + 面部表情"三层独立叠加时非常典型，三者互不干扰地叠加在同一套移动动画之上。

## 实际应用

### 上半身瞄准覆盖

第三人称射击游戏中，角色需要在奔跑、蹲伏、匍匐等多种姿态下自由改变上半身朝向以瞄准目标。制作一套从俯视 −90° 到仰视 +90° 的瞄准叠加动画集（通常为9帧或17帧关键帧），以标准站立瞄准正前方姿势为参考姿势提取差量。运行时根据俯仰角在这些差量帧之间插值，并将结果叠加到下半身的移动动画上。《战地》系列和 Unreal 的 ALS（Advanced Locomotion System）插件均采用此模式。

### 受击反馈与命中反应

当子弹击中角色躯干某个部位时，触发对应的受击叠加动画（Hit Reaction Additive），该剪辑通常只有 8~12 帧，表现为骨骼在受力方向上的短暂偏移与弹回。由于是叠加方式，角色无需中断当前移动动画，受击细节与移动逻辑完全独立。

### 程序化呼吸与待机晃动

将角色静止时的微小呼吸起伏制作为叠加剪辑（以 A-Pose 为参考），在所有状态下以较低权重（α ≈ 0.3~0.5）持续叠加，可以让角色在任何动作状态下都保持生命感，而不必为每条基础动画单独嵌入呼吸节奏。

## 常见误区

### 误区一：叠加动画可以叠加任意参考姿势提取的剪辑

实际上，叠加剪辑与基础动画必须使用**相兼容的参考姿势**。若叠加剪辑以 T-Pose 为参考提取差量，但基础动画的姿势与 T-Pose 偏差极大（如角色深蹲），叠加后骨骼偏移量会被放大到不可接受的程度，产生骨骼穿插或扭曲。正确做法是选择与基础动画状态相近的姿势作为参考，或在引擎层面将叠加空间切换为 Mesh Space 以消除父骨骼累积旋转的影响。

### 误区二：叠加权重超过 1.0 会产生"更强"的效果

部分开发者误以为 α > 1 可以放大叠加效果，但四元数 Slerp 在超出 [0, 1] 范围时会产生外插（Extrapolation），旋转方向会越过目标值继续偏转，极易导致骨骼姿势失控翻转。如需放大效果，应在动画制作阶段直接调整差量幅度，而非依赖权重超出范围。

### 误区三：叠加动画与混合树中的权重混合等价

混合树的权重混合（Blend）是在两个完整姿势之间做加权平均，结果是一个位于两姿势"中间"的新姿势。叠加动画是将差量**累加**到任意姿势上，结果可以超出原始动画库中任何单一姿势的范围。两者数学性质不同：混合是凸组合，叠加是仿射偏移，不能相互替代。

## 知识关联

叠加动画以**混合树**的输出结果作为基础姿势的输入来源——混合树负责处理移动状态机和速度插值等逻辑，生成完整的基础骨骼姿势，叠加层随后在此基础上施加增量。两者在动画图（Animation Graph）中通常表现为串联节点：混合树节点的输出端直接连接叠加节点（Apply Additive）的 Base 输入端，而 Additive 输入端接收另一条叠加剪辑采样流。

在 Unreal Engine 的 AnimGraph 中，`Apply Additive` 节点明确区分了 `Base` 和 `Additive` 两个输入引脚，权重由 `Alpha` 引脚控制，精确对应上述数学模型。Unity 的 Animator 则通过将 Layer 的 Blending 模式设置为 `Additive`（相对于默认的 `Override`）来实现等价功能，该层中的动画状态机输出自动被解释为差量叠加而非姿势覆盖。
