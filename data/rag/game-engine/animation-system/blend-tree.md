# 混合树

## 概述

混合树（Blend Tree）是动画系统中用于将多个动画片段按照参数权重进行线性或多维插值混合的节点结构。与单纯的动画状态机不同，混合树不负责状态切换逻辑，而专门解决"同一状态内部，动画如何随参数连续变化"的问题。例如，角色的行走速度从 0 变化到 600 cm/s 时，混合树可以让动画在"站立"、"慢走"、"快走"、"奔跑"四个片段之间平滑过渡，而不产生硬切换。

混合树的概念由 Gready Heid 等人在 GDC 2004 的演讲中系统化阐述，随后被 Unity 的 Mecanim 系统（2012 年随 Unity 4.0 引入）和 Unreal Engine 的 AnimGraph 广泛采用。在 UE 中称为 **BlendSpace**，在 Unity 中直接称为 **Blend Tree**，两者的底层算法高度相似。混合树的重要性在于：它用极少的动画资产（4–8 个片段）覆盖了一个连续的参数空间，避免了为每种中间状态单独制作动画的巨大成本。

## 核心原理

### 1D 混合树（Linear BlendSpace）

1D 混合树沿单一参数轴排列若干动画片段，每个片段在轴上占据一个"锚点值"。设参数当前值为 *t*，左右相邻锚点分别为 *t₀* 和 *t₁*，则混合权重为：

> **w₁ = (t − t₀) / (t₁ − t₀)，w₀ = 1 − w₁**

引擎在每帧按此权重对两段动画的骨骼变换（位置、旋转、缩放）做线性插值（LERP），旋转通常使用四元数球面线性插值（SLERP）。1D 混合树最典型的用途是速度驱动的移动动画：将 0、150、300、600 四个速度值对应的动画片段排列在轴上，Speed 参数直接驱动插值。

### 2D 混合树（2D BlendSpace）

2D 混合树在一个平面坐标系内散布动画片段，使用两个参数（如水平速度 VelocityX 和垂直速度 VelocityY）确定当前位置，然后用**重心坐标插值（Barycentric Interpolation）**计算周围最近三角形三顶点的权重。UE 的 2D BlendSpace 默认使用 Delaunay 三角剖分自动生成这些三角形。典型应用是八方向移动：前、后、左、右、左前、右前、左后、右后共 8 个方向的行走动画分布在圆形空间中，角色的移动方向向量直接映射到该平面。

2D 混合树还分为**标准 2D BlendSpace** 和 **2D BlendSpace（方向型）**两种子类型。方向型专为极坐标设计，横轴表示方向角度（−180°～180°），纵轴表示速度大小，适合赛车或飞行类游戏中转向与速度的组合控制。

### 分层混合树（Layered Blend Per Bone）

分层混合树不按参数插值，而是按**骨骼层级**将两个动画混合到不同身体部位。配置时需要指定一个"起始骨骼"（如 `spine_01`），该骨骼及其所有子骨骼（上半身）使用第二层动画，其余骨骼（下半身）保留第一层动画。UE 中的节点名为 **Layered blend per bone**，每个层还有一个 0.0～1.0 的 Blend Weight 参数，控制混合强度。这一结构常用于"下半身跑步 + 上半身射击/攀爬"的组合。

混合树内部每帧的计算复杂度为 O(n·b)，其中 n 是参与混合的动画片段数，b 是骨骼数量。对于有 200 根骨骼的角色，同时混合超过 6 个片段会产生明显的 CPU 开销，因此引擎通常只激活参数点附近权重非零的 2–4 个片段参与计算。

## 实际应用

**第三人称射击游戏的移动系统**：用一个 2D BlendSpace 管理 9 个方向（含原地待机）的移动动画。横轴为 `AimYaw`（−180～180），纵轴为 `Speed`（0～600），配合瞄准偏移节点，角色在任意方向、任意速度下移动时都能获得流畅的动画表现。

**载具驾驶员身体倾斜**：摩托车游戏中，使用 1D BlendSpace，以转向角度（−45°～45°）为参数轴，将"直立驾驶"和"左/右倾斜"三个片段混合，完全通过物理输入数据驱动，无需手动调制任何混合权重。

**受伤状态叠加**：利用 Layered Blend Per Bone，将正常跑步动画（全身）与左臂受伤的单手持枪动画（`clavicle_l` 骨骼以下）分层混合，Blend Weight 由受伤布尔值控制（0.0 正常 / 1.0 受伤），切换过渡时间设为 0.3 秒以避免突变。

## 常见误区

**误区一：混合树可以替代状态机**。混合树只负责连续参数空间内的插值，无法处理"从攻击到受击"这类需要打断条件的离散状态切换。正确做法是在动画蓝图的状态机中为每个状态内部创建混合树，两者分工明确：状态机管理状态转换，混合树管理单状态内的连续变化。

**误区二：2D BlendSpace 的采样点可以任意放置**。Delaunay 三角剖分对采样点的分布极为敏感。若多个点位于同一直线，三角剖分会退化，导致某些区域的权重计算错误甚至出现动画"跳跃"。UE 官方建议采样点在参数空间中均匀分布，且避免三点共线的情况。

**误区三：分层混合中混合权重为 0.5 即"各占一半"**。Layered Blend Per Bone 的权重 0.5 并非线性混合上下身各一半，而是对**第二层动画**在目标骨骼上的影响强度进行缩放。权重 1.0 表示完全使用第二层的旋转值，0.5 表示在第一层和第二层之间插值，效果类似于半透明叠加而非面积分割。

## 知识关联

混合树建立在**动画蓝图**的 AnimGraph 框架之上，必须作为节点插入 AnimGraph 才能生效；驱动混合树的参数（如 Speed、Direction）由动画蓝图的事件图（Event Graph）在每帧从角色运动组件中读取并更新。理解动画蓝图的"更新 → 求值"双阶段机制，是正确配置混合树参数更新频率的前提。

混合树是**叠加动画（Additive Animation）**的前序概念。叠加动画在混合树的输出姿势基础上额外叠加一个差量姿势（Additive Pose），用于实现呼吸起伏、受击震动等细微动作。两者的区别在于：混合树输出的是完整的参考姿势，叠加动画输出的是相对于参考姿势的**偏差量**，相加后才得到最终姿势。掌握混合树的权重计算方式，有助于理解叠加动画中为何需要单独的 Additive Blend Space 类型而非普通 BlendSpace。
