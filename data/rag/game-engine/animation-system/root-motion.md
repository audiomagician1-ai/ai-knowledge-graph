# 根运动（Root Motion）

## 概述

根运动是一种动画播放技术，它允许角色在动画片段中记录的根骨骼（Root Bone）位移和旋转数据，直接驱动角色的游戏世界坐标发生变化。与传统的"原地动画"不同，开启根运动后，角色不再停留在固定位置播放动作，而是由动画师在制作动画时赋予角色的真实位移来推动角色在场景中移动。

这一技术最早在3D游戏动画管线成熟的2000年代中期开始被主流引擎广泛支持。在Unity引擎中，根运动通过`Animator.applyRootMotion`属性控制，并在`OnAnimatorMove()`回调中允许开发者拦截和修改根运动数据；在Unreal Engine中，对应选项为动画序列属性面板中的"Enable Root Motion"复选框，引擎会自动将根骨骼轨道的运动从动画空间转换到世界空间。

根运动的价值在于保证动画表现与实际位移的高度一致性。如果一段跑步动画中脚步步距为1.2米/秒，那么角色在场景中的移动速度也精确地等于1.2米/秒，不会产生"脚底打滑"（foot sliding）的视觉错误。这对追求真实感的动作游戏、格斗游戏和第三人称冒险游戏尤为重要。

## 核心原理

### 根骨骼的特殊地位

骨骼动画层级中，根骨骼（通常命名为`root`、`hips`或`pelvis`）是整个骨架的最顶层节点，所有其他骨骼都是它的直接或间接子节点。根运动技术专门提取根骨骼在每一帧之间的**增量变换（Delta Transform）**：

- **位移增量**：`ΔPosition = Position(frame_n) - Position(frame_n-1)`  
- **旋转增量**：`ΔRotation = Rotation(frame_n) * Inverse(Rotation(frame_n-1))`

引擎在每个更新帧将这两个增量应用到角色的`Transform`或`CharacterController`上，从而实现位移驱动。其余非根骨骼的动画数据仍然以相对于根骨骼的本地空间运算，保持肢体动作不变。

### 动画片段中的根运动烘焙

动画师在DCC工具（如Maya、MotionBuilder）中完成动画后，需要确保根骨骼包含真实的世界位移轨道，而非被约束固定在原点。导入Unity时，模型导入设置（Model Import Settings）中有四种根运动处理选项：**Original**（直接使用根骨骼原始轨道）、**Center of Mass**（使用质心计算位移）、**Feet**（以脚底为基准校正Y轴高度）等。不同选项会影响动画片段中存储的根运动曲线数据，直接决定最终位移的准确度。

### 循环动画与根运动偏移修正

对于循环播放的动画片段（如奔跑循环），动画片段的首帧根骨骼位置和末帧根骨骼位置往往存在误差，导致循环衔接时出现跳跃。Unity提供了**Loop Pose**和**Bake Into Pose**选项：勾选"Bake Into Pose"后，Y轴（高度）的根运动会被"烘焙"回本地骨骼动画，不再驱动世界坐标，适用于不希望垂直方向产生偏移的走路动画；水平面的XZ位移则保持为根运动输出。这种分轴控制让开发者可以精确选择哪些自由度由根运动驱动。

## 实际应用

**格斗游戏中的攻击位移控制**  
格斗游戏中，角色的突刺攻击或冲拳动作往往伴随身体前移。如果用代码手动设置速度，很难与动画师设计的动作时机完全吻合。启用根运动后，动画师在前冲帧中直接赋予根骨骼0.8米的位移，角色的碰撞体和攻击判定会在正确时刻随动画同步前移，攻击判定框与角色脚步完全对齐。

**运动捕捉数据的直接应用**  
动捕数据天然包含演员在真实空间中的位移，根运动技术可以将这些精确的运动路径直接还原到游戏中。例如，一段4.5秒的动捕跑步转向动画，演员实际移动了3.2米并旋转了90°，开启根运动后游戏角色将精确复现这一弧线轨迹，无需额外编写转向插值代码。

**爬梯与攀爬动画**  
攀爬梯子时，角色需要在垂直方向随动画逐帧上升。根运动的Y轴位移输出可直接驱动`CharacterController`的纵向移动，相比每帧计算当前攀爬进度更为稳健，且与动画帧率无关。

## 常见误区

**误区一：认为根运动与物理系统完全兼容**  
根运动直接操控角色的`Transform`位置，绕过了物理引擎的碰撞响应计算。在Unity中，若角色使用`Rigidbody`且根运动直接写入位置，会导致刚体碰撞检测失效。正确做法是在`OnAnimatorMove()`中读取`animator.deltaPosition`，再通过`Rigidbody.MovePosition()`将增量传递给物理系统，确保碰撞仍由物理引擎处理。

**误区二：根运动可以取代所有移动逻辑**  
根运动只能复现动画师预先制作好的位移，它无法响应运行时输入的速度变化。玩家推摇杆50%的力度时，游戏不能简单地把行走根运动乘以0.5来得到慢速行走，因为这会导致动画播放速度也减半，脚步动作变慢但步距不变，仍然打滑。此类变速移动通常需要结合动画混合树（Blend Tree）制作多段不同速度的动画，并各自烘焙根运动数据。

**误区三：开启根运动后Navmesh导航自动适配**  
在使用Unity的`NavMeshAgent`时，代理组件自身有一套移动控制逻辑。若同时启用根运动，两套位移会发生冲突，角色通常会抖动或偏移路径。正确方案是在`OnAnimatorMove()`中读取根运动增量，禁用`NavMeshAgent.updatePosition`，手动将增量位置传递给导航代理，保持导航系统的路径数据同步。

## 知识关联

根运动建立在**动画片段**的基础上：根运动数据本质上是动画片段导入阶段从根骨骼轨道提取的特殊曲线，没有完整的动画片段，根运动无从谈起。理解动画片段中关键帧数据的存储方式，有助于判断为什么某些动画的根运动提取结果不符合预期。

向上拓展方面，根运动与**动画状态机（Animator Controller）**深度绑定：不同状态切换时，根运动的增量在过渡期间会按照过渡权重混合，保证状态切换时位移平滑过渡而非突变。此外，**IK（反向动力学）**常配合根运动使用——根运动负责整体位移，IK负责修正脚部与地面的接触点，两者共同消除角色与地面之间的穿插和悬空问题。
