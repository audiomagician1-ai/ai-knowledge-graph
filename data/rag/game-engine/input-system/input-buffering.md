# 输入缓冲

## 概述

输入缓冲（Input Buffer）是格斗游戏和动作游戏中一种将玩家按键操作暂时存储的机制，允许在当前动作尚未结束时预先登记下一个输入指令，并在满足执行条件的瞬间自动触发。这一机制的核心目的是消除帧时序对操作成功率的苛刻要求——没有输入缓冲时，玩家必须在极短的有效窗口内精准按键，往往只有1~2帧的容错空间；引入缓冲后，这个容错窗口可以扩展到6帧、8帧乃至更长。

输入缓冲的设计概念在街机格斗游戏时代已有雏形，《街头霸王II》（1991年）的必杀技系统便包含了约8帧的方向+按键输入缓冲，使玩家可以在升龙拳指令的最后一帧之前提前完成一部分方向输入并保留在内存中。现代动作游戏中，《黑魂》系列（2011年起）的前摇缓冲、《鬼泣5》的连招缓冲都是该机制的典型实现。

输入缓冲对游戏手感的影响是双向的：缓冲窗口过短导致操作"很难打出"，缓冲窗口过长则导致玩家"误触发"了不想要的动作。因此如何调校缓冲时长是动作游戏设计的重要参数决策。

## 核心原理

### 缓冲队列结构

输入缓冲本质上是一个有容量上限的先进先出队列（FIFO Queue），通常在游戏逻辑层以帧为单位维护。每一帧更新时，系统将当前帧的输入事件（来自输入触发器的按下/持续/抬起状态）写入队列尾部，同时弹出超出缓冲窗口时效的旧条目。以60fps游戏为例，若设定缓冲窗口为10帧，队列最多保存最近10帧的输入快照；超过10帧未被消耗的输入将自动失效并出队。

缓冲条目通常存储以下字段：
- **输入类型**（如攻击键、跳跃键、方向杆角度）
- **写入帧编号**（用于计算存活时长）
- **已消耗标志位**（防止同一次按键触发多个动作）

### 缓冲消费时机

缓冲队列中的输入并非立刻生效，而是在**可取消窗口**（Cancel Window）开启时被系统轮询消费。以格斗游戏为例，一次普通攻击动画被划分为：前摇帧（Startup）、活跃帧（Active）、后摇帧（Recovery）三段，其中后摇帧的某个子区间会标记为"可被必杀技取消"的窗口。进入该窗口的那一帧，系统扫描缓冲队列，若发现匹配的必杀技指令序列则立即消费并执行。

《铁拳7》中，多数普通攻击的可取消窗口约在第20~35帧之间开启，而必杀技指令的缓冲有效期设置为14帧，这意味着玩家可以在前摇期间提前14帧输入完整指令，只要落在可取消窗口内就能精准衔接。

### 指令序列匹配与时效性

对于复合指令（如波动拳的 ↓↘→ + 拳），输入缓冲需要记录方向输入序列，并在一定帧数内匹配完整序列。《街头霸王V》中，该序列的总输入时限约为60帧（1秒），即序列中第一个方向输入与最后一个按键输入的间隔不得超过60帧，否则序列作废。系统每帧对队列进行正则式风格的模式匹配，一旦检测到完整序列且处于可执行状态，立即触发对应技能。

部分游戏还区分**硬缓冲**与**软缓冲**：
- **硬缓冲**：输入被强制存储，即使当前角色处于受击硬直中，缓冲依然有效，恢复后自动执行。
- **软缓冲**：仅在角色处于可行动状态的前N帧内接受预输入，受击状态下缓冲清空。

《街头霸王6》的驱动冲刺取消系统使用软缓冲，防止玩家在被压制时误触发反击动作。

## 实际应用

**格斗游戏连招衔接**：在《街头霸王V》中实现"立中拳 → 波动拳"连携时，玩家不必等到立中拳动画完全结束再输入波动拳指令，而是可以在立中拳攻击判定帧（约第7帧）出来的同时就开始输入 ↓↘→ 方向序列，只需在可取消窗口（约第12~20帧）之前完成拳键输入即可，操作窗口从约2帧扩展至8帧。

**动作RPG的动作衔接**：《艾尔登法环》中的攻击连段同样依赖输入缓冲，玩家在第一刀攻击动画的后半段按下攻击键，游戏将该输入存入约12帧的缓冲队列，待当前攻击动画进入可取消点时立即读取并接续第二刀，形成连段手感。若关闭该机制（某些速通Mod会测试此效果），则需要精确到单帧的时机才能打出连段。

**平台跳跃游戏的土狼时间配合**：部分平台游戏将输入缓冲与土狼时间（Coyote Time）联动——玩家在走出平台边缘后约6帧内按下跳跃键，尽管角色已经处于下落状态，跳跃输入因缓冲机制仍会被消费并执行跳跃，避免玩家感到"明明按了跳跃却没跳"的挫败感。

## 常见误区

**误区一：缓冲窗口越长越好**

更长的缓冲窗口并不总是提升体验。在竞技格斗游戏中，过长的缓冲会导致"指令污染"——玩家在防守时残留的方向输入被系统识别为攻击指令的一部分，错误触发技能。《街头霸王IV》因缓冲窗口较长（约15帧）曾被职业玩家批评容易误出升龙拳导致破防，后续《街头霸王V》将同类缓冲收紧至10帧以下以提高防守操作精度。

**误区二：输入缓冲等同于输入延迟**

输入缓冲存储的是**玩家超前预输入**的指令，并不增加输入到画面响应的时间。真正的输入延迟来自渲染管线和显示器的信号传输，与缓冲机制无关。一个有8帧预输入缓冲的游戏，其即时动作（如格挡）依然可以做到0帧额外延迟，只有被放入队列等待消费的预输入才会经历"延迟"，而这恰恰是玩家主动选择的操作策略。

**误区三：缓冲机制对所有动作均匀生效**

大多数游戏对不同动作类型设置了差异化的缓冲时长。以《黑暗之魂3》为例，滚动闪避的预输入缓冲约为8帧，而饮用瓶子的输入缓冲仅为约4帧，攻击动作的缓冲约12帧。这种差异化设计使高风险/高价值动作（如闪避）更宽容，而误触成本高的动作（如消耗品）更严格。

## 知识关联

输入缓冲建立在**输入触发器**的基础上：触发器负责将原始硬件信号转换为有语义的"按下事件"，而缓冲系统以这些事件为原子单位进行队列管理和模式匹配。没有触发器层的去抖动与状态分类，缓冲队列将充满噪声数据，导致误判。

在架构层面，输入缓冲通常位于输入系统与动画/状态机之间，作为数据暂存层解耦两者的帧率差异。若游戏采用可变帧率渲染但逻辑帧固定（如逻辑帧20fps、渲染帧60fps），缓冲系统须以逻辑帧为单位计时，否则缓冲时长会随帧率波动产生不一致的手感。进一步延伸，连招识别系统、取消系统（Cancel System）和动作优先级系统均以输入缓冲提供的预输入数据为消费来源，是构建完整格斗游戏操作逻辑的必要前置模块。
