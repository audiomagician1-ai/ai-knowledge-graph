# 视锥剔除

## 概述

视锥剔除（Frustum Culling）是一种在渲染前丢弃摄像机视野范围之外物体的优化技术。其核心逻辑是：对场景中的每个物体进行"是否在摄像机可视区域内"的测试，若完全位于视锥体之外，则直接跳过该物体的渲染提交，不向GPU发送任何Draw Call。这一技术在3D游戏引擎中属于CPU端最基础的渲染优化手段。

视锥剔除的理论基础来源于计算机图形学中的视锥体（View Frustum）模型。1984年前后，随着实时3D渲染硬件的发展，视锥剔除作为管线优化手段被系统性引入。现代引擎（如Unity、Unreal Engine 4/5）均默认启用视锥剔除，并在其场景管理模块中将其作为每帧必须执行的剔除阶段。

在实际项目中，一个拥有数千个静态物体的开放世界场景，摄像机朝向某一方向时，视野外的物体可能占总数量的60%到80%。通过视锥剔除，这些物体的顶点着色、片元着色计算均被完全省略，帧渲染时间可显著缩短。

## 核心原理

### 视锥体的数学定义

摄像机的视锥体由六个裁剪平面构成：近裁剪面（Near Plane）、远裁剪面（Far Plane）、左裁剪面、右裁剪面、上裁剪面、下裁剪面。每个平面用一个四元组 `(A, B, C, D)` 表示，满足平面方程：

```
Ax + By + Cz + D = 0
```

其中 `(A, B, C)` 为平面法向量（朝向视锥体内侧），D 为平面偏移量。点 `P` 到平面的有符号距离为：

```
d = A·Px + B·Py + C·Pz + D
```

若 `d < 0`，则点位于该平面的外侧（视锥体之外）。

视锥体的六个平面可以直接从投影视图矩阵（ViewProjection Matrix）中提取，这一方法由 Gil Gribb 和 Klaus Hartmann 于2001年在论文 *Fast Extraction of Viewing Frustum Planes from the WorldView-Projection Matrix* 中详细阐述，现已成为工业标准做法，避免了逐帧手动计算六个平面的繁琐步骤。

### 包围体与平面测试

逐顶点测试开销过高，实际引擎使用包围体（Bounding Volume）代替物体精确几何形状进行测试。最常用的是轴对齐包围盒（AABB，Axis-Aligned Bounding Box）和包围球（Bounding Sphere）。

**包围球与平面测试**：对于球心 `C`、半径 `r` 的包围球，只需计算球心到六个平面的有符号距离 `d`：
- 若任意一个平面的 `d < -r`，球完全在视锥体外，剔除。
- 若所有平面的 `d > r`，球完全在视锥体内，接受。
- 其他情况为相交，需进一步处理（通常保留）。

包围球测试仅需6次点积运算，是最快的包围体测试方式。

**AABB与平面测试**：AABB测试精度高于包围球，做法是对每个平面，选取AABB的"正顶点"（P-Vertex，沿平面法向量方向最远的角点）和"负顶点"（N-Vertex）进行测试。若负顶点在平面外侧，则AABB完全在视锥体外。AABB测试每个平面需要3次比较来选取正/负顶点，总开销约为包围球的3倍，但误剔除率（False Negative）更低。

### 批量剔除与空间分区结合

单纯遍历场景中所有物体逐一测试的复杂度为 O(n)，当场景物体数量超过10万时，即便每个测试仅需数十个CPU指令，累积开销也会对帧时间产生影响。因此，视锥剔除通常与空间分区结构结合使用：

- **四叉树/八叉树**：对树节点的包围体先做视锥测试，若节点完全在视锥体外，其所有子节点均被跳过，不再遍历。最优情况下，可将测试数量从 O(n) 降低到 O(log n) 级别。
- **BVH（层次包围体）**：Unreal Engine 使用基于BVH的剔除，在植被密集场景中，单帧剔除测试次数可从数十万次压缩到数千次。
- **PVS（Potential Visibility Set）预计算**：室内场景中，可预先烘焙每个区域的可见集合，视锥剔除在PVS结果的子集上执行，进一步减少测试范围。

## 实际应用

**Unity引擎的视锥剔除**：Unity在`Camera`组件的渲染循环中，每帧自动对场景中所有具有`Renderer`组件的物体执行视锥剔除。开发者可通过`GeometryUtility.CalculateFrustumPlanes(camera)`获取当前帧的六个裁剪平面，再调用`GeometryUtility.TestPlanesAABB(planes, bounds)`手动测试自定义包围体，用于非渲染物体（如AI感知区域、物理触发器）的剔除逻辑。

**开放世界地形渲染**：在地形块（Terrain Chunk）管理中，地图被划分为若干固定大小的方块（例如32×32米）。每帧视锥剔除只允许与视锥体相交的地形块进入渲染队列，摄像机水平视角为90°时，典型情况下仅需渲染全部地形块的约25%。

**粒子系统剔除**：粒子发射器本身使用包围球进行视锥测试，但粒子发射器的包围球半径需要根据最大粒子寿命和速度动态估算，以避免粒子在即将进入视野时突然弹出（Pop-in）。

## 常见误区

**误区一：视锥剔除会减少GPU负载**。视锥剔除在CPU上执行，其直接效果是减少提交给GPU的Draw Call数量以及传输的顶点数据量，从而减轻GPU顶点着色阶段的压力。但片元着色瓶颈（如全屏后处理）不受视锥剔除影响——被剔除的物体不参与渲染，GPU帧率提升来自Draw Call和顶点处理的减少，而非片元处理的减少。

**误区二：包围体越紧密，视锥剔除越快**。包围体的紧密程度影响误剔除率，但测试速度取决于包围体的类型，而非其大小。一个紧密贴合物体的凸多面体包围体测试开销远大于同一物体的AABB，即使它会产生更少的错误保留。实际引擎选择AABB或包围球，正是在测试速度与剔除精度之间取得平衡。

**误区三：视锥剔除可以替代遮挡剔除**。视锥剔除只能剔除摄像机视野范围之外的物体，对视野内被其他物体完全遮挡的物体（如建筑物背面的街道）无能为力。遮挡剔除（Occlusion Culling）专门处理这类情况，两者共同构成完整的可见性剔除流程。

## 知识关联

视锥剔除依赖空间分区结构（如八叉树、BVH）提供物体的空间组织，使得批量剔除时可通过节点包围体快速跳过大片区域，而非线性遍历所有物体。没有空间分区，视锥剔除在大场景下的时间复杂度会退化为 O(n)，失去实用价值。

掌握视锥剔除后，下一步自然延伸至可见性系统（Visibility System）的完整设计。可见性系统在视锥剔除的基础上增加遮挡剔除（Occlusion Culling）、细节层次剔除（LOD Culling）和门户剔除（Portal Culling）等机制，形成多层次的渲染可见性判断管线。视锥剔除是可见性系统中执行开销最低、覆盖效果最广的第一道过滤器，其输出结果（通过视锥测试的物体集合）直接作为后续遮挡剔除算法的输入。
