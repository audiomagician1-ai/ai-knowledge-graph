# 程序化放置

## 概述

程序化放置（Procedural Placement）是游戏引擎场景管理中利用算法和规则自动将游戏对象分布于三维空间的技术，属于程序化内容生成（PCG，Procedural Content Generation）的子领域。与手动摆放场景元素不同，程序化放置通过定义放置规则、密度参数和约束条件，让系统自主决定每个对象的位置、朝向和缩放，从而在数千平方公里的开放世界地图上生成数百万棵树木、岩石或建筑而无需美术人员逐一摆放。

该技术在2000年代初随着开放世界游戏的兴起而进入主流。《极限竞速：地平线》系列使用程序化植被放置系统自动根据地形坡度、海拔和湿度图填充植物；《无人深空》（2016）将程序化放置扩展为星球级别的地貌对象生成。虚幻引擎5的PCG Framework（2023年正式发布）将程序化放置作为一等功能内置进编辑器，允许开发者在运行时和编辑时同时执行放置图（Placement Graph）。

程序化放置的核心价值在于：一位场景设计师通过编写约50条规则，可以产生手工摆放需要20人·月才能完成的内容密度，同时保证每次生成结果都符合设计意图的变化范围，而非机械重复。

## 核心原理

### 密度控制与密度图（Density Map）

密度控制是程序化放置最基础的调控机制。系统将目标区域划分为网格单元，每个单元关联一个密度值 `ρ ∈ [0, 1]`，该值决定该区域内对象数量与最大允许数量之比。密度值通常从灰度纹理（密度图）中采样：白色像素（值=1.0）表示最大密度，黑色像素（值=0.0）表示不放置任何对象。

期望放置数量公式为：

```
N = ρ × A × k
```

其中 `ρ` 为密度值，`A` 为单元格面积（单位：平方米），`k` 为密度系数（棵/m²）。例如，若 `ρ=0.6`，`A=100m²`，`k=0.05棵/m²`，则该单元期望生成3棵树。实际放置数量通过泊松分布或简单取整确定，以避免过于规律的视觉效果。

### 规则驱动的约束过滤

程序化放置并非随机撒点，而是通过多层规则过滤候选位置。常见规则包括：

- **坡度过滤**：查询地形法线，若法线与垂直轴夹角超过阈值（如35°），则拒绝该位置放置大型树木
- **海拔分带**：将高度范围映射到植被类型，例如海拔0–200m放置阔叶林，200–500m放置针叶林，500m以上放置灌木
- **距离排斥**：使用泊松圆盘采样（Poisson Disk Sampling）保证相邻同类对象间距不小于最小间距 `r_min`，防止对象穿插
- **遮蔽检测**：对候选点发射射线，若命中已有对象则降低放置概率

这些规则组成一条过滤链，候选点必须通过全部规则才能最终落地。虚幻引擎PCG Framework将每条规则实现为图节点，允许美术人员以可视化方式拼接规则链。

### 泊松圆盘采样与空间分区的结合

泊松圆盘采样是程序化放置中最常用的候选点生成算法，Bridson在2007年发表的算法将时间复杂度降至 O(n)。该算法维护一个活跃列表和一个背景网格（单元格大小为 `r_min / √2`），通过快速邻近查询判断新候选点是否距离所有已接受点超过 `r_min`。

背景网格本质上是空间分区结构——通常是均匀网格或四叉树。在大规模地形（如 `16km × 16km`）上，直接遍历所有已放置对象来检查最小距离代价极高，而利用四叉树将邻域查询从 O(n) 降至 O(log n)。这说明程序化放置的运行效率直接依赖底层空间分区的质量。

## 实际应用

**植被填充工作流**：在《荒野大镖客：救赎2》的世界构建中，Rockstar报告使用程序化放置生成了超过1亿棵树木和植被对象，整个流程通过地形湿度图、海拔图和坡度图驱动，7种植被群落各有独立的密度图层，设计师仅需修改图层权重即可调整整片地区的植被风格。

**建筑碎片散布**：废弃城市场景中，程序化放置根据道路网格和建筑轮廓多边形生成碎石、破碎玻璃和废弃车辆。系统首先使用道路图遮罩排除道路区域，然后在建筑物轮廓向外偏移1.5m的缓冲区内提高碎片密度，模拟建筑物倒塌后碎片向外抛洒的物理效果。

**运行时动态放置**：部分引擎实现支持玩家移动时在摄像机前方实时执行放置逻辑，以 `500m × 500m` 的滑动窗口跟随玩家。超出窗口的对象被卸载，新进入窗口的区域重新触发放置计算，实现无缝的无限大世界效果。

## 常见误区

**误区一：认为纯随机采样等同于程序化放置**。简单的随机撒点（uniform random sampling）会产生明显的聚集和空洞，违反视觉上对自然分布的期待。泊松圆盘采样才是"看起来自然"分布的正确工具，它的本质是将随机性约束在最小间距之内，而非完全随机。

**误区二：认为程序化放置一定不需要美术干预**。实际工作流中，密度图和规则参数需要美术或关卡设计师进行迭代调整。程序化放置减少的是重复性的手工操作，而规则的设定本身依然是创作行为。许多商业项目会在程序化生成后叠加手动调整层（override layer），允许设计师在特定位置强制添加或删除对象。

**误区三：将密度系数 k 设置为固定值适用所有对象类型**。不同尺寸对象的 `r_min` 差异极大：地面碎石的 `r_min` 可能仅为0.1m，而巨型岩石的 `r_min` 可能达到8m。若统一使用小 `r_min` 生成大型对象，会导致对象穿插；若统一使用大 `r_min` 生成小型对象，会使地面显得异常空旷。正确做法是为每类对象单独定义密度参数集。

## 知识关联

**与空间分区的关系**：程序化放置在候选点邻近检测阶段必须调用空间分区结构。泊松圆盘采样内部的背景网格就是均匀网格空间分区的直接应用；在更大规模场景中则替换为四叉树或八叉树。空间分区的粒度直接影响程序化放置的执行速度——背景网格单元过大会导致误判，过小则内存占用过高，通常取 `r_min / √2` 作为最优单元尺寸。

**与LOD系统的联动**：程序化放置生成的大量对象若不加管理会严重影响渲染性能。实践中，放置系统输出的对象列表会直接输入分层细节（LOD）管理器，根据摄像机距离选择不同精度的网格。在虚幻引擎中，Hierarchical Instanced Static Mesh（HISM）组件正是为程序化植被放置产生的大量同类实例专门设计，将数千个相同网格合并为少量Draw Call。
