# 语音录制

## 概述

语音录制是游戏音频对白制作的起点，指在专业录音环境中捕获配音演员声音表演的完整技术流程。与影视配音录制不同，游戏语音录制通常需要为同一角色录制数百甚至数千条独立台词，每条台词都可能在不同游戏情境下单独触发播放，因此每句话的首尾处理、呼吸位置和情绪一致性都有独立而严格的要求。

这一实践的技术规范在2000年代随着《光晕》（Halo, 2001）和《质量效应》（Mass Effect, 2007）等叙事驱动型游戏的兴起而快速标准化。当时开发商发现，沿用影视录音习惯——允许句间连续呼吸音、依赖后期剪辑拼接——会导致游戏运行时的对白触发出现明显的拼接感与不一致性，促使行业形成专门针对游戏交互特性的录音规范。

游戏语音录制直接决定了后续所有处理环节的质量上限。原始录音中若存在本底噪声过高（通常要求信噪比不低于60dB）、演员嘴部杂音未处理、或动态范围压缩过度等问题，对白编辑和音频引擎阶段均难以完全弥补。行业标准参考文献《Game Audio Implementation》（Barnabas, 2016, Focal Press）对此有详细阐述，将录音质量列为整个对白制作管线的"单点故障"风险最高的环节。

## 核心原理

### 录音棚声学环境要求

游戏语音录制在混响时间（RT60）控制在 **80毫秒以内**的干声棚（Dry Vocal Booth）中进行，而非电影配音常用的稍带空间感的录音室。RT60的计算公式来源于赛宾方程（Sabine Equation）：

$$RT60 = \frac{0.161 \times V}{A}$$

其中 $V$ 为房间体积（立方米），$A$ 为总吸声量（sabins，即各表面面积与吸声系数之积的总和）。对于一个体积约为 $15\,\text{m}^3$ 的标准干声棚，$A$ 需达到约 $30\,\text{sabins}$ 才能将 RT60 压低至80ms以内。极短的RT60确保录制的声音是"干净的干声"，方便音频引擎在运行时根据游戏场景动态叠加混响（如洞穴回声、室内反射）。若原始录音本身带有超过100ms的房间感，便无法在引擎中被置于开放旷野等场景而不显得突兀。

本底噪声（Background Noise Floor）应控制在 **-65dBFS以下**，通风系统是最常见的噪声污染源，专业棚通常在录音期间关闭空调并依赖隔热密封结构维持温度，每隔约45分钟重新启动通风系统通气。

### 麦克风放置与信号链

游戏语音录制的标准麦克风摆位为距演员嘴部 **15至20厘米**，配合防喷罩（Pop Filter）置于麦克风与演员之间约5厘米处，以消除爆破音（B、P等辅音产生的气流冲击峰值可瞬间超过+6dBFS造成削波）。信号链标准配置如下：

```
大振膜电容话筒（如 Neumann U87 或 AKG C414）
    → 高质量前置放大器（如 Universal Audio 710）
    → 模数转换器（如 Apogee Symphony）
    → DAW录音轨（48kHz / 24bit）
```

采样率设置为 **48kHz / 24bit**，这是游戏音频行业标准格式，区别于音乐制作常用的44.1kHz——游戏引擎（如Unreal Engine、Unity）的音频系统原生以48kHz运行，混用44.1kHz文件会触发引擎内部实时重采样，引入微小的音质损耗和CPU开销。

增益设置上，演员正常对话时峰值电平应落在 **-12dBFS至-6dBFS** 之间，为喊叫或极端情绪表演留出至少6dB的动态余量，避免削波（Clipping）。一旦发生数字削波，该Take通常直接废弃，因为被截断的波形顶部在数字域中无法被还原（不同于模拟磁带的软饱和，数字削波产生的谐波失真刺耳且明显）。

### 配音会话结构与台词管理

专业游戏语音录制以"会话脚本"（Session Script）为核心文件，该文档列出所有台词的唯一ID（如 `PC_COMBAT_HURT_01` 到 `PC_COMBAT_HURT_24`）、台词文本、情绪标注（如"愤怒，中等强度，非喊叫"）和时长预估（通常以0.5秒为精度单位）。录音导演（Voice Director）根据这份脚本逐条引导演员，对白ID在DAW中的标记名称须与脚本完全一致，以确保后续对白编辑阶段的文件可追溯性。

每条台词录制完成后，录音师立即在DAW（通常为 **Pro Tools** 或 **Reaper**）中标记Take编号与状态（OK / NG / 备用），确保每个台词ID至少有一条确认可用的Take，以及通常2至3条备选Take，为对白编辑提供选择空间。每日录音结束后，所有原始文件以 `[项目代号]_[角色ID]_[日期]_RAW` 的命名规范归档备份至至少两个物理位置。

## 关键技术参数与录制规范

游戏语音录制涉及若干需要精确控制的数值参数，以下是行业通行的技术规范汇总：

| 参数项 | 标准值 | 说明 |
|---|---|---|
| 采样率 | 48kHz | 匹配游戏引擎原生采样率 |
| 位深 | 24bit | 保留约144dB动态范围用于后期处理 |
| 录制峰值电平 | -12dBFS 至 -6dBFS | 正常对话；喊叫时不超过 -3dBFS |
| 本底噪声 | ≤ -65dBFS | 空调关闭状态下测量 |
| RT60（混响时间） | ≤ 80ms | 干声棚标准 |
| 麦克风距离 | 15–20cm | 防止过强近讲效应（Proximity Effect） |
| 每日有效录音时长 | ≤ 4小时 | 超出后演员音色漂移风险显著上升 |
| 每条台词备选Take数 | 2–3条 | 为对白编辑提供选择余地 |

**近讲效应（Proximity Effect）**是大振膜电容话筒在近距离拾音时低频响应增强的物理现象：麦克风距离每缩短一半，低频提升约3至6dB（视话筒指向性而定）。游戏对白录制中若麦克风距离小于10cm，男声低频会明显"虚胖"，导致在耳机和移动设备扬声器上播放时显得混浊，因此15至20cm的距离是对近讲效应和嘴部细节捕捉之间的工程妥协结果。

## 实际应用

### 大型RPG游戏的批量录制流程

以一款中等规模RPG为例，主角可能需要录制 **1500至3000条**对白（包括战斗喊叫、环境反应、剧情对话三类）。录音团队通常按情绪类型分组安排录制顺序：

1. **剧情对话**（情绪平稳，音量适中）——优先录制，此时演员嗓音最佳
2. **环境反应台词**（如"发现可疑物体"、"角色受伤"等短句）——中段录制
3. **战斗喊叫与极端情绪台词**——最后录制，因高强度发声对嗓音损耗最大

每个录制日的实际有效工作时长约为4小时，超过后演员嗓音疲劳会导致音色明显漂移（基频F0可偏移10–20Hz），破坏角色声音在不同场次台词间的一致性。《The Complete Guide to Game Audio》（Collins, 2013, Focal Press）记录了多家AAA工作室在超时录制后因音色不一致而被迫召回演员重录的案例，平均额外成本约占原录制预算的15%。

### 战斗语音的特殊处理要求

战斗用的短促语音（通常0.3至1.5秒）有三项区别于剧情对话的特殊要求：

- **起始无呼吸音**：台词开头不能有吸气声，因为这类台词会在战斗中被高频次随机触发，若每次触发前均有吸气声，玩家会在密集战斗中听到明显的"吸气-喊叫-吸气-喊叫"节奏，极度出戏。录音导演需在每条战斗台词录制前提示演员"不要预备吸气，直接开口"。
- **结尾无拖尾**：台词结尾0.1秒内应恢复至近静音（<-60dBFS），避免音频引擎淡出处理时产生明显的截断感。
- **组内音量一致性**：同一类型战斗语音（如"受击呻吟"组的24条台词）的峰值电平差异不应超过±2dBFS，确保随机播放时音量不会忽大忽小。

### 远程录制（Remote Recording）的兴起

2020年新冠疫情后，远程语音录制（演员在家录制、导演远程监听指导）在游戏行业迅速普及。典型技术方案为：演员端使用便携式隔音罩（如 Kaotica Eyeball）配合USB接口电容话筒，通过 **Source Connect** 或 **Cleanfeed** 软件传输低延迟高质量音频流，导演端在Studio内实时监听并指导。远程录制的主要质量风险是演员家庭环境的本底噪声（空调、冰箱压缩机、交通噪声）普遍高于专业录音棚5至15dB，因此远程录制的原始文件在进入对白编辑前通常需要额外一道 **降噪预处理**（如使用 iZotope RX 的 Dialogue Isolation 模块）。

## 常见误区

### 误区一：游戏录音可以参照影视标准执行

影视配音中，一场连续对话戏通常一镜录完，后期通过剪辑拼接自然连贯。游戏录制中，同一对话场景的不同台词分支会在数周内分批录制，且每条台词在引擎中独立触发。若录音师允许演员在每条台词前后保留自然呼吸音（影视中完全正常），这些呼吸音在游戏中密集触发时会产生明显的"噪声堆积"。正确做法是要求每条台词头部静音长度不超过 **100ms**，尾部不超过 **200ms**，超出部分在录制结束后立即由录音师手动截除。

### 误区二：24bit录制等同于需要24bit交付

原始录制以24bit进行是为了在增益调整和后期处理中保留足够的动态精度（24bit约对应144dB动态范围，而16bit仅约96dB）。但最终交付给音频引擎的游戏对白文件通常被编码为 **16bit PCM** 或 **Vorbis/ADPCM等有损格式**，以节省游戏包体大小和运行时内存占用。在 Wwise 或 FMOD 等中间件中，原始24bit WAV会在导入时被自动转码为目标平台格式。混淆"录制位深"与"交付位深"是初入行的录音师常见的概念错误。

### 误区三：信噪比够高就不需要注意麦克风方向性
