# 声像摆位

## 概述

声像摆位（Panning）是将单声道或立体声音频信号分配到多声道输出系统中特定扬声器位置的算法过程。在游戏音频中，声像摆位负责将虚拟声源的三维坐标转换为各扬声器的增益系数组合，使听者感知到声音从某个特定方向传来。其核心任务是解决"如何将一个点声源的能量分配给有限数量的物理扬声器"这一问题。

这一概念的现代数学基础确立于1997年Ville Pulkki提出的VBAP（Vector Base Amplitude Panning，基于向量的振幅摆位）算法。在此之前，游戏中通行的是简单的线性左右摆位或基于距离衰减的音量缩放，无法处理三维空间的高度信息。VBAP的发表标志着声像摆位从经验性音量调节进化为基于扬声器几何关系的系统性算法。

声像摆位直接决定了玩家在游戏中对敌人位置、危险方向和环境层次的感知精度。对于电竞类游戏（如FPS），声像摆位算法的精度误差若超过10°，玩家就会出现明显的方位判断失误，因此该算法的选型对游戏体验有可量化的实战影响。

## 核心原理

### 1. 等功率摆位与线性摆位

最基础的立体声摆位分为两种数学规则。**线性摆位**（Linear Panning）对左右声道的增益分配公式为：
- 左声道增益 = (1 - pan) / 2
- 右声道增益 = (1 + pan) / 2
其中 pan ∈ [-1, 1]。

**等功率摆位**（Equal Power Panning）使用三角函数保持总声场能量恒定：
- 左声道增益 = cos(θ/2)
- 右声道增益 = sin(θ/2)
其中 θ = (pan + 1) × π/4，θ ∈ [0°, 90°]。

线性摆位在声像移动至中心点时会出现约3dB的能量下陷（"hole in the middle"现象），而等功率摆位通过三角恒等式 cos²θ + sin²θ = 1 保证了总功率守恒，是游戏引擎中更常用的选择。

### 2. VBAP三维扬声器摆位算法

VBAP将声源方向向量 **p** 分解为由两个或三个相邻扬声器方向向量 **l₁, l₂, l₃** 构成的线性组合：

**p** = g₁**l₁** + g₂**l₂** + g₃**l₃**

其中增益向量 **g** = [g₁, g₂, g₃] 通过矩阵求逆计算：**g** = **p** × L⁻¹，L 为扬声器方向向量矩阵。最终增益归一化使能量守恒：g_norm = g / √(g₁² + g₂² + g₃²)。

VBAP的关键约束是仅激活包围声源方向的最近扬声器三角形（2D系统为两个扬声器），其余扬声器增益为零。这一稀疏激活特性使其在Dolby Atmos 7.1.4或游戏常见的5.1系统上都能高效运行。

### 3. 基于距离的声像宽度收缩

在三维游戏场景中，纯粹的方向摆位还需叠加距离感知模型。当声源距听者超过一定阈值（Unity和Wwise默认通常为1米内的近场范围），声像宽度（Spread）参数会强制拓宽扬声器增益分布。具体实现是在VBAP增益向量上施加展开系数 s ∈ [0, 1]：

g_spread = (1-s) × g_vbap + s × g_uniform

其中 g_uniform 表示能量均匀分配到所有扬声器的状态。声源越近，s 越大，营造"包围感"。

### 4. 双耳摆位（Binaural Panning）

耳机用户无法依赖物理扬声器间距产生方位感，必须通过头部相关传递函数（HRTF）进行虚拟化摆位。HRTF将声源方向 (仰角 θ, 方位角 φ) 映射为左右耳的滤波器冲激响应，滤波后的信号包含了耳廓反射和耳间时延（ITD）等真实空间线索。Wwise、FMOD均内置了基于MIT KEMAR假人测量数据集的HRTF摆位处理器。

## 实际应用

**Wwise中的3D摆位配置**：在Sound Property Editor中，将Positioning Mode设为"3D"并选择"VBAP"算法后，需在Attenuation曲线中单独设定Spread随距离的变化，以及Spatialization混合比例（2D/3D混合权重），这使开发者可以为背景音乐保留全景声场同时精确定位近战音效。

**FPS游戏的水平精度要求**：Valve在《CS:GO》的音频系统迭代中记录了将等功率立体声摆位升级为5.1 VBAP后，玩家对脚步声方位判断的平均误差从约25°降低至约8°，直接影响了职业赛事中根据声音判断敌人位置的战术可靠性。

**虚幻引擎5的MetaSound**：MetaSound中的"Spatialization"节点内部使用等功率摆位作为回退方案，当检测到系统输出为2声道时自动降级，避免VBAP在双声道场景下计算错误扬声器三角形的问题。

## 常见误区

**误区1：声像摆位等于音量大小调节**
初学者常混淆摆位（Panning）和音量衰减（Attenuation）。摆位调整的是不同扬声器之间的增益比例，总能量基本守恒（等功率条件下精确守恒）；衰减则随距离降低信号总幅度。一个远处的声源应当是"增益低但摆位准确"，而非"将音量调小来模拟方向感"。

**误区2：VBAP可以精确再现高度方向**
VBAP的高度感知效果远弱于水平面。由于人耳对仰角的感知依赖耳廓形状产生的频谱线索，而非声压级差，VBAP在仰角分辨上的物理极限约为±10°误差，远大于水平面的±3°精度。若项目需要精确高度定位（如飞机从头顶飞过），必须结合HRTF双耳渲染而非单依赖VBAP。

**误区3：等功率摆位在所有场景下优于线性摆位**
等功率摆位在多轨混音叠加时会导致总线能量超载：若同一位置有4个音效同时等功率摆位到中心，叠加后的中心能量会远高于线性方案。在游戏中大量同类音效并发的场景（如枪声密集战斗）中，需要在总线上配合Limiter或使用Voice Limiting机制，否则等功率摆位会加剧削波失真。

## 知识关联

声像摆位的输入依赖**听者位置**模块提供的相对坐标转换结果——只有在将世界坐标系中的声源位置减去听者位置并转换到听者局部坐标系之后，摆位算法才能计算正确的方位角和仰角。若听者朝向矩阵更新延迟超过一帧（约16ms@60fps），摆位结果会产生可感知的"滞后漂移"，因此听者位置更新必须在声像摆位计算的同一音频帧内完成。

掌握声像摆位的数学细节后，可以进入**空间音频插件**的学习。商业插件（如Steam Audio、Microsoft Spatializer、Oculus Audio SDK）本质上是在VBAP或HRTF的基础上叠加了早期反射、混响尾音和遮挡模型，其参数调节逻辑直接建立在对基础摆位增益分配的理解之上。
