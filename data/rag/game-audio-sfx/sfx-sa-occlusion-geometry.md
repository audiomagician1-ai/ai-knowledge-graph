# 几何遮挡

## 概述

几何遮挡（Geometric Occlusion）是空间音频系统中一种通过场景几何体（如墙壁、柱子、地形）对声音传播路径进行遮断计算的技术。其核心思路是：当声源与听者之间存在不透明几何体时，系统通过射线检测（Ray Casting）确定声音是否被遮挡，并根据遮挡程度调整声音的音量、滤波频率和空间定位。

几何遮挡的计算方法在1990年代末随着3D游戏引擎的普及逐步被引入音频中间件。Miles Sound System 和后来的 Wwise、FMOD 均相继在 2000 年代初期实现了基于场景几何的遮挡接口。早期实现仅用单条射线做二值判断（完全遮挡/完全畅通），现代实现则采用多射线或球形投射来得到连续的遮挡因子（Occlusion Factor，范围 0.0～1.0）。

几何遮挡之所以在游戏音效设计中受到重视，是因为它直接影响玩家对声源位置的感知真实性。如果枪声透过混凝土墙壁毫无衰减地传入，玩家会立刻察觉音频与视觉场景的脱节；而正确的遮挡计算可以将高频分量衰减 6～12 dB，并将低频留存，模拟出真实的"闷声穿墙"效果。

## 核心原理

### 射线检测与遮挡因子

几何遮挡最基础的实现方式是从听者位置（Listener Position）向声源位置（Source Position）投射一条射线，检测该射线是否与场景碰撞体（Collision Mesh）相交。若相交则返回遮挡因子 = 1.0（完全遮挡），否则为 0.0。实际工程中通常不止投射一条射线，而是使用 **多射线球形采样**（Sphere Sampling）：在声源周围球面均匀采样 N 条射线（常见取值 N = 8 或 16），统计被遮挡射线数量，计算连续遮挡因子：

$$
O = \frac{N_{blocked}}{N_{total}}
$$

其中 $O$ 为遮挡因子（0～1），$N_{blocked}$ 为被几何体阻断的射线数，$N_{total}$ 为总采样射线数。$O$ 值越大，遮挡越严重，声音的高频衰减越明显。

### 基于遮挡因子的音频参数映射

遮挡因子 $O$ 计算完毕后，需要映射到可听的音频参数上。常见映射策略如下：

- **低通滤波截止频率**：遮挡因子为 0 时，截止频率设为 22 kHz（全通），遮挡因子为 1 时，截止频率降至 400～800 Hz，大量削减高频，模拟墙体对高频声波的吸收。
- **音量衰减**：在 Wwise 中，遮挡（Occlusion）参数通常附加约 −6 dB 到 −18 dB 的音量偏移，具体数值由设计师在 RTPC（实时参数控制）曲线中手动调节。
- **混响比例**：遮挡因子高时，Direct Path 信号减弱，但 Reverb Send 可以相对增大，使声音更像从远处漫射而来。

### 静态几何与动态几何的差异处理

游戏场景中既有静态几何体（Static Mesh，如建筑墙壁）也有动态几何体（Dynamic Mesh，如可移动的门、移动平台）。静态几何遮挡可以在场景烘焙阶段预计算遮挡信息，存储为遮挡查询表（Occlusion Lookup Table），运行时直接查表，开销极低。动态几何体则必须在运行时每帧（或每隔若干帧）重新执行射线检测；为了控制 CPU 开销，引擎通常只对距听者 **50 米以内** 且声音优先级高于阈值的声源执行逐帧遮挡更新，远处声源使用上一帧缓存值。

## 实际应用

**第一人称射击游戏（FPS）**：玩家躲在掩体后方时，对方枪声应通过几何遮挡产生明显的高频衰减。以 Unreal Engine 5 的 MetaSounds 为例，开发者可在 Geometry Trace Channel 中配置专属的"Audio Occlusion"碰撞通道，使只有音频相关的射线检测在该通道上执行，避免与物理碰撞通道混用导致的误检。

**开放世界地形遮挡**：山丘和岩石会对远处声源（如营地篝火声、战场爆炸声）造成部分遮挡。由于地形网格面数极高，此时通常不直接对地形 Mesh 做射线检测，而是用**简化碰撞体（Simplified Collision Hull）** 替代，将射线检测多边形数量从数十万缩减至数百，保证音频线程帧时间控制在 0.5 ms 以内。

**FMOD 中的实际配置**：FMOD Studio 提供内置的 Geometry API，允许开发者将场景中的多边形面组注册为音频遮挡几何体，并为每个面赋予 `direct` 和 `reverb` 两个遮挡系数（取值 0.0～1.0），分别控制直达声和混响声的衰减比例。例如，混凝土墙通常配置 `direct = 0.85, reverb = 0.30`，木门配置 `direct = 0.40, reverb = 0.10`。

## 常见误区

**误区一：遮挡等同于阻隔（Obstruction）**

几何遮挡特指声源与听者之间的直线路径被几何体完全或部分截断。而阻隔（Obstruction）在 Wwise 的语义中特指声源和听者 **不在同一房间** 但声音绕过障碍物传播的情况，两者计算逻辑和参数映射均不同。混淆两者会导致在同一房间内的遮挡场景（如躲在桌子后方）被错误地应用了跨房间阻隔的滤波参数，产生过度的高频截止。

**误区二：射线数量越多越好**

增加采样射线数量确实可以提升遮挡因子的精度，但代价是线性增长的 CPU 开销。在一个同时存在 50 个活跃声源的场景中，若每个声源使用 32 条射线，每帧执行一次，总射线检测量为 1600 次/帧，在复杂场景中可能消耗超过 2 ms 的 CPU 时间，严重影响音频线程性能。工程实践中，8 条射线已能满足大多数场景需求，精度提升与计算成本需要通过性能分析工具（如 Wwise Profiler）来做取舍。

**误区三：几何遮挡可以完全替代 Portal 声学系统**

几何遮挡仅处理直达声路径的遮断，无法模拟声音绕过开口（如门缝、窗户）衍射传播进入相邻房间的声学行为。在复杂室内场景中，仅靠几何遮挡会出现"关上门声音骤然消失"而非"声音从门缝处衍射变小"的感知失真，这正是需要引入 Portal 声学系统的根本原因。

## 知识关联

几何遮挡建立在**声音传播**的物理基础之上——具体而言，它处理的是声波直线传播（Direct Path）受到阻断的情况，而非衍射或多路径反射。学习几何遮挡前，需要掌握声速（343 m/s @ 20°C）、声波频率与穿透能力的关系（低频波长长、绕射能力强；高频波长短、易被吸收），这些物理规律直接决定了遮挡因子到低通滤波参数的映射逻辑。

在**遮挡与阻隔**的基础上，几何遮挡将抽象的"遮挡程度"与具体的三维场景网格绑定，使遮挡计算从手动设置区域触发器演进为自动化的几何查询。掌握几何遮挡之后，自然会遇到其局限性：纯几何遮挡无法处理声音通过门、窗等开口在房间间传播的路径，因此下一步需要学习 **Portal 声学系统**，后者正是在几何遮挡的射线检测框架上，叠加了开口（Portal）作为声音传输节点的拓扑路由逻辑，实现真正的室内声学模拟。
