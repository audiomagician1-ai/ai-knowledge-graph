# 粒度处理

## 概述

粒度处理（Granular Processing）是一种将音频信号切割成极短时间片段（称为"粒子"或grain）并对这些片段进行重新排列、叠加和变换的数字信号处理技术。每个grain的时长通常在10毫秒至100毫秒之间，多个grain以每秒数十至数百个的密度同时播放，形成连续的音频输出。该技术的理论基础来自物理学家丹尼斯·加博尔（Dennis Gabor）在1947年提出的"声音量子"概念，他认为任何声音都可以用时频平面上的基本粒子来表示。

粒度处理在游戏音效领域的独特价值在于它能够在不改变音调的前提下拉伸时间长度（称为粒度时间拉伸），也能在不改变时长的前提下移调——这与变调处理的传统方式形成互补。在游戏引擎实时音频管线中，粒度处理被用于创造环境声音的有机变化、角色呼吸的随机化扩展，以及爆炸余波的超长拖尾效果。

与简单的循环播放或交叉淡化不同，粒度处理能够打破音频素材固有的节奏结构，使一段3秒的爆炸声延伸为30秒的持续轰鸣，且听感上不存在明显的人工重复感。这种能力对游戏开发者尤其宝贵，因为它大幅减少了需要录制或存储的原始素材量。

## 核心原理

### 粒子参数体系

粒度处理的行为由一组相互独立的参数控制，每个参数对应grain生命周期的不同阶段。**粒子大小（Grain Size）**决定每个grain截取的时长，通常设置范围为1ms到500ms；较小的grain（10ms以下）会产生音调感，因为人耳开始将密集重复的片段感知为周期性振动。**粒子密度（Grain Density）**即每秒播放的grain数量，密度低于20个/秒时会产生明显的颗粒质感，超过100个/秒时声音趋于平滑。**散布（Scatter）**参数控制grain读取起始位置的随机偏移量，散布值为0时输出原始信号，散布值增大后声音逐渐失去时间信息变为音色质感。

每个grain通常被包络窗口函数整形，最常用的是汉宁窗（Hanning Window）：`w(n) = 0.5 × (1 - cos(2πn/N))`，其中N为grain的采样长度，n为当前采样位置。汉宁窗能消除grain首尾的点击噪声，使相邻grain叠加时过渡平滑。

### 粒度时间拉伸算法

粒度时间拉伸的核心操作是调整grain的**读取速率**与**播放速率**的比值。将一段音频拉伸为原来的4倍时长时，读取指针以原速率的25%前进，但每个grain依然以100%速率播放，通过增加grain的叠加层数来填满时间空隙。这与相位声码器（Phase Vocoder）时间拉伸的根本区别在于：粒度方法不需要进行FFT频域变换，因此不存在"相位模糊"导致的金属声，但在极端拉伸比例（超过10倍）时会产生特有的"云雾状"音质，这种音质在游戏中常被有意用于魔法或梦境音效。

**拉伸比率公式**：`Stretch Ratio = Output Duration / Input Duration = Grain Playback Rate / Pointer Advance Rate`

当Stretch Ratio = 1时输出原始时长；当Stretch Ratio > 1时为拉伸；Stretch Ratio < 1时为压缩。游戏音效中常用的拉伸比率范围为0.5x至8x，超出此范围artifact通常无法接受。

### 粒子合成与音色塑造

粒度合成（Granular Synthesis）进一步允许从多个不同音频源提取grain并混合，创造出原始素材中不存在的音色。在游戏音效设计中，典型做法是从金属撞击声中截取grain、从弦乐持续音中截取grain，设置两个grain流的混合比为6:4，再施加±30ms的随机散布，可以合成出接近"活体金属生物"的独特音效。游戏引擎如Audiokinetic Wwise的SoundSeed Grain插件支持最多4个grain源的实时混合，每个源可独立设置pitch偏移（以半音为单位，范围±24半音）和envelope参数。

## 实际应用

**环境音延伸**：录制一段8秒的森林风声素材，通过粒度时间拉伸以3.0x的比率扩展为24秒的无缝循环，grain size设置为80ms，scatter设置为40%。与直接循环8秒素材相比，玩家在游玩超过30分钟后几乎无法察觉重复。《地平线：零之曙光》的开放世界环境音使用了类似的粒度拉伸管线。

**爆炸余震**：将爆炸冲击波的尾部片段（约第2秒至第4秒）提取后以粒度处理重新合成，grain密度从60个/秒渐变为15个/秒，模拟爆炸后混响逐渐离散为独立回声颗粒的物理过程。这种方式比单纯使用混响尾音更具层次感。

**怪物声音变体**：在游戏中实时使用粒度处理对同一段怪物嚎叫进行随机化，每次播放时随机设置grain scatter在10%到35%之间、grain size在40ms到120ms之间，可以从单一录音素材产生听感差异明显的多个变体，节省约70%的音频存储空间。

## 常见误区

**误区一：粒度时间拉伸与变速播放等价**。降低采样率播放会同时改变时长和音调，而粒度时间拉伸的定义特征正是二者解耦——改变时长不影响基频。初学者在Wwise或FMOD中若错误使用pitch shift来"模拟"时间拉伸，会导致低沉化或尖锐化的音调偏移，通常体现为每10%的时间拉伸量对应约1.7个半音的非预期音调变化。

**误区二：grain size越小音质越好**。当grain size低于人耳时间分辨率阈值（约5ms）时，密集的grain边界会产生可闻的低频调制噪声，频率等于grain density。例如grain size为5ms、density为100个/秒时，会在100Hz附近产生明显的嗡嗡声。优化grain size需要与density参数配合调整，而非单独最小化。

**误区三：粒度处理对所有音频素材效果一致**。实际上，瞬态丰富的素材（鼓击、枪声）在大scatter值下会丢失所有时间信息变为音色噪声，而持续性素材（弦乐、环境音）则能承受更大范围的参数变化。在游戏音效实践中，对percussive素材使用粒度处理时scatter通常不应超过grain size的20%，否则攻击感完全消失。

## 知识关联

粒度处理建立在**声音合成基础**中对采样与重采样原理的理解上，grain的提取本质上是短时加窗操作，直接沿用了加窗函数（如汉宁窗、汉明窗）的概念。与**变调处理**的关系体现在互补性上：变调处理通常在频域（相位声码器）或时域（PSOLA算法）改变基频，而粒度处理在时间维度上重组音频片段，两者结合可实现时间与音调的完全独立控制。

学习粒度处理为后续的**侧链处理**奠定了动态控制的思维框架：理解grain density随时间动态变化后，可以进一步理解如何用外部信号（侧链信号）驱动grain参数实时变化，例如用角色心跳信号控制环境音的grain scatter值，随心跳加速而增加音频的颗粒化程度，强化紧张感的听觉传达。
