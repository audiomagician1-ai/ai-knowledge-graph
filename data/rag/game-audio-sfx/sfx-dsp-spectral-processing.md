# 频谱处理

## 概述

频谱处理（Spectral Processing）是指将音频信号从时域转换至频域，直接操作各频率分量的幅度与相位，再反变换回时域的数字信号处理技术。其理论基础是离散傅里叶变换（DFT），实际计算中几乎总使用快速傅里叶变换（FFT）算法，将一段音频帧分解为若干复数频率箱（bin），每个 bin 对应一段频率范围，宽度等于采样率除以 FFT 帧长（bin宽度 = Fs / N）。

频谱处理的发展与 1965 年 Cooley 和 Tukey 发表的 FFT 算法密切相关。该算法将 N 点 DFT 的计算量从 O(N²) 降至 O(N log₂N)，使实时频谱操作首次变得可行。到 1970 年代，Bob Moorer 等研究者在 CCRMA 将频谱方法引入音乐声音合成；进入 1990 年代，基于相位声码器框架的频谱处理成为游戏音效引擎的实用工具，可在不改变音高的前提下拉伸时间，或在不改变时长的前提下移调。

对于游戏音效设计，频谱处理具有不可替代的地位：它允许设计师对爆炸声的低频轰鸣与高频碎片声分别塑形，将人声通过谱乘法嫁接金属共鸣，或以频谱减法去除录音底噪，这些操作在时域中无法精确实现，因为时域均衡器无法在不引入相位旋转的情况下做到接近矩形的窗口频率响应。

## 核心原理

### STFT 分析-合成框架

短时傅里叶变换（STFT）是频谱处理的标准工作框架。其流程分为三步：分帧加窗→FFT→频域操作→IFFT→重叠相加（OLA）合成。每帧长度 N 常见取值为 1024、2048 或 4096 采样点，帧移（hop size）H 通常设为 N/4 或 N/8，以保证 75%～87.5% 的帧间重叠。加窗函数选择直接影响频率分辨率与时间分辨率的权衡：Hann 窗旁瓣抑制约 −31 dB，是游戏音效处理中最常用的折中方案；Blackman-Harris 窗旁瓣可低至 −92 dB，适合高精度频谱减法，但主瓣更宽。

第 n 帧的 STFT 定义为：

**X(n, k) = Σ x(m) · w(m − nH) · e^(−j2πkm/N)**

其中 x(m) 是原始时域信号，w 为窗函数，k 为频率箱序号，n 为帧序号。每个复数箱 X(n,k) 携带幅度 |X(n,k)| 与相位 ∠X(n,k) 两部分信息，频谱处理的全部魔法都在于如何在频域修改这两个量。

### 幅度谱操作与相位处理

幅度谱操作相对直观：将某段频率范围内所有 bin 的幅度乘以一个增益因子，等效于极精确的频率选择性滤波；将两个信号的幅度谱相乘（谱卷积）可实现卷积混响或声音嫁接；将一段背景噪声的平均幅度谱从目标信号幅度谱中减去，即为经典的谱减降噪算法（Boll, 1979）。

相位谱往往被忽视，但对音质至关重要。相位声码器（Phase Vocoder）在时间拉伸时，必须追踪每个 bin 的相位演化，按照拉伸比例 α 重新计算合成帧的相位增量：

**φ_synth(n, k) = φ_synth(n−1, k) + α · Δφ_true(n, k)**

其中 Δφ_true 是从分析帧提取的"真实"相位差（去除期望相位进展后的余量）。若直接复制分析相位而不做此修正，会产生特征性的"相位音叉"（phase smearing）人工噪音，在游戏中表现为金属性颤鸣。

### 频谱重塑与谱包络操作

频谱重塑（Spectral Reshaping）特指在保持信号音高轮廓或时间结构不变的前提下，修改其频谱包络（Spectral Envelope）。谱包络可通过倒谱（Cepstrum）方法或低阶 LPC（线性预测编码，通常 12−24 阶用于语音）估计，将其从细节激励频谱中分离。游戏音效中常见的"怪物化"处理正是将人声的激励频谱保留，用合成共鸣峰（formant）结构替换原始谱包络，再将两者重新相乘，产生非人类共鸣腔的音色。

频谱重塑还包括谱变换（Spectral Warping）：将频率轴按双线性变换 z → (z − a)/(1 − az*) 重映射，参数 a 控制弯曲量，使低频分辨率提升、高频压缩，可模拟人耳的 Bark 尺度感知。

## 实际应用

**游戏爆炸音效分层处理**：将单次爆炸录音通过 STFT（N=2048，Hann窗）分解后，单独放大 60−200 Hz 的低频 bin 幅度 +12 dB，同时对 3 kHz 以上的碎裂瞬态成分做移调（谱拉伸 k → 1.2k），分别重合成，再与干信号混合，可在不重录的情况下产生物理上更大规模的爆炸感。

**实时降噪预处理**：在人声 VO 录制完成后，取前 0.5 秒纯噪声段估计噪声幅度谱 N̂(k)，对正片信号每帧执行：Ŝ(n,k) = max(|X(n,k)| − β·N̂(k), 0)，其中超减因子 β 通常取 1.5−2.0。β 过小底噪残留，β 过大引入"音乐噪声"（Musical Noise）伪音，需根据 SNR 动态调整。

**科幻武器共鸣嫁接**：将激光枪录音的激励频谱与金属板共鸣的谱包络相乘，实现无法通过时域处理完成的音色移植，整个操作在 Reaper 的 ReaFIR 或 iZotope RX 的频谱编辑器中可手动完成。

## 常见误区

**误区一：FFT 帧长越大频率分辨率越高越好**。增大 N 确实提升频率分辨率（bin宽度 = Fs/N 变小），但同时延长了时间帧（帧时长 = N/Fs），导致时间分辨率下降。对于 48000 Hz 采样率、N=4096 的配置，单帧时长约 85 ms，任何短于此时长的瞬态在频域会产生严重的时间扩散，爆炸起始端的冲击感会被严重模糊。游戏音效中瞬态丰富的素材通常选择 N=512 或 N=1024。

**误区二：频谱减法可以完全去除噪声而不伤害信号**。谱减法假设噪声与信号不相干，幅度谱加法近似成立，但实际上两者在相同频率 bin 中的相位关系导致幅度相消不完全，残余的随机相位起伏正是"音乐噪声"的来源。引入维纳滤波（Wiener Filter）估计：H(k) = SNR(k)/(1+SNR(k)) 可缓解此问题，但无法彻底消除。

**误区三：复制幅度谱即可复制音色**。相位谱对音色的空间感和清晰度同样有决定性作用。两个幅度谱完全相同但相位随机化的信号，听起来会呈现出完全不同的纹理——前者清晰，后者如噪声涂抹。这是频谱处理中最容易被初学者忽略的相位重要性问题。

## 知识关联

频谱处理直接建立在声码器（Vocoder）的分析-合成思想之上：声码器以滤波器组近似频域分析，而频谱处理以 FFT 精确实现同一目标，bin 精度远高于声码器的通道数量（通常 16−32 个），因此可以完成声码器无法完成的细粒度谱包络操作。学习相位声码器框架后，可直接迁移至频谱处理的合成端。

频谱处理是瞬态塑形（Transient Shaping）的频域视角入口：瞬态的宽带能量突变对应频域中所有 bin 同时出现的幅度峰值，通过 STFT 时频图可定位瞬态起始帧并选择性放大或抑制，这是理解基于频谱的瞬态检测算法（如 HFC 高频内容检测）的必要前提。两者配合使用，可实现对游戏音效素材同时控制音色与动态形状的精细处理流程。
