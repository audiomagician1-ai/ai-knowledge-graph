# 问题解决

## 概述

问题解决（Problem Solving）是指个体在面临初始状态与目标状态之间存在障碍时，通过一系列认知操作寻找解决路径的心理过程。认知心理学将问题解决定义为：在**问题空间**（problem space）中进行搜索，该空间由初始状态、目标状态、操作算子以及中间状态共同构成。这一定义由纽厄尔（Allen Newell）与西蒙（Herbert Simon）于1972年在著作《人类问题解决》（*Human Problem Solving*, Prentice-Hall）中正式提出，至今仍是认知心理学最具影响力的理论框架之一。

问题解决研究的起源可追溯至格式塔心理学派。科勒（Wolfgang Köhler）于1917年通过对黑猩猩的研究发现了"顿悟"现象：黑猩猩苏尔坦在无法直接够到香蕉时，突然将两根竹竿接在一起完成目标，表现出跨越"问题空间"的跳跃式理解，而非逐步试误。这一发现挑战了当时桑代克（Edward Thorndike）盛行的联结主义试误学习解释。

问题解决的研究价值体现在极具体的领域差异上：国际象棋大师能在5秒内识别约50,000种棋局模式（Chase & Simon, 1973），而新手则必须逐步推演每一步落子可能。这种专家与新手之间的巨大表现差距，正是由算法策略与启发式捷径的选择、问题表征质量以及长期记忆中图式（schema）丰富程度的差异所共同决定的。

## 核心原理

### 算法策略：穷举搜索的保障与代价

**算法（Algorithm）** 是一种穷尽式的系统搜索程序，保证只要完整执行就能找到正确答案。以字谜破解为例，对5个字母的所有可能排列进行逐一检验，共需检验：

$$
5! = 5 \times 4 \times 3 \times 2 \times 1 = 120 \text{ 种排列}
$$

算法策略在计算资源充足时可靠有效，但随字母数量增加，计算量呈阶乘级增长——10个字母的字谜需检验 $10! = 3,628,800$ 种排列，人脑根本无法在合理时间内穷举。这正是人类大脑在日常问题解决中几乎不依赖纯算法的根本原因。

**计算机科学视角下的算法搜索**，可以用深度优先搜索（DFS）来模拟人类逐步推演行为：

```python
def dfs_solve(state, goal, operators, visited=None):
    if visited is None:
        visited = set()
    if state == goal:
        return [state]
    visited.add(state)
    for op in operators:
        next_state = op(state)
        if next_state not in visited:
            result = dfs_solve(next_state, goal, operators, visited)
            if result:
                return [state] + result
    return None
```

上述代码中，`state` 对应问题空间中的当前状态，`goal` 为目标状态，`operators` 为可用操作算子集合——这与纽厄尔和西蒙1972年对问题空间的形式化定义完全吻合。

### 启发式策略：高效但有代价的捷径

**启发式（Heuristic）** 是经验性的简化策略，不保证正确，但能将搜索范围缩减至可操作范围。认知心理学识别出若干核心启发式方法：

**手段-目的分析（Means-End Analysis, MEA）**：持续比较当前状态与目标状态的差距，每次选择能最大程度缩短该差距的操作。纽厄尔与西蒙用"通用问题解决器"（General Problem Solver, GPS）程序模拟了此过程，GPS于1957年首次运行，能够解决逻辑证明、汉诺塔等结构化问题。在汉诺塔实验中，被试面对3个圆盘时平均需7步完成，与MEA预测的最优解完全一致；面对5个圆盘时需31步最优解，被试平均用时约42步，偏差来自局部次优选择。

**逆向推理（Working Backward）**：从目标状态向初始状态倒推，适用于目标比起点更为清晰的问题，如数学几何证明。例如要证明"$\angle A = \angle B$"，可先问"什么条件能推出两角相等"，再逆向寻找前提条件。

**类比推理（Analogical Reasoning）**：将新问题映射到结构相似的已解决问题上。吉克（Mary Gick）与霍利奥克（Keith Holyoak）于1980年的实验表明，若预先读过"将军用分散兵力攻克要塞"的故事，被试解决"用低强度射线摧毁肿瘤"问题的成功率从10%提升至75%——前提是被试能主动识别两个问题在结构上的对应关系。

启发式的系统性代价是**认知偏差**。代表性启发（Representativeness Heuristic）令人忽略基础概率：医生在诊断中过度依赖症状匹配，忽视发病率数据，导致低概率疾病被过度诊断（Kahneman & Tversky, 1974）。

### 功能固着：对常规功能的认知锁定

**功能固着（Functional Fixedness）** 是指个体只能看到物体的惯常用途，无法将其重新概念化以解决新问题的认知障碍。邓克尔（Karl Duncker）于1945年发表的论文《论问题解决》（"On Problem-Solving", *Psychological Monographs*, 58(5)）中设计了经典的**蜡烛问题**：给被试图钉盒一个、蜡烛一支、火柴一盒，要求将蜡烛固定在墙上使其燃烧时不滴蜡。

解决方案是将图钉从盒中倒出，用图钉将空盒钉在墙上，再将蜡烛立在盒盖上——然而大多数被试无法想到将"装图钉的容器"重新定义为"托盘"。当实验者将图钉直接放在桌上（图钉盒为空盒）时，成功率显著提高，因为此时盒子不再承担"装图钉"的功能角色，功能固着强度降低。

功能固着在跨文化研究中表现出系统性差异：Birch与Rabinowitz（1951）的实验表明，近期使用过开关的被试更难将其用作钟摆重物，证明**近期功能使用经验**会直接强化功能固着程度。接触工具种类有限的文化群体（如部分非工业化社会成员）功能固着程度更低，表明这一认知模式是通过后天经验习得的，而非先天认知局限。

打破功能固着的训练方法包括：**属性列举法**——将物体分解为材质（金属、木质）、形状（圆形、柱状）、重量、导电性等物理属性，强迫大脑脱离对整体功能标签的依赖；**随机词汇联想**——将随机物体与当前问题强制关联，激活远距离语义连接。

### 顿悟与问题重表征

**顿悟（Insight）** 指问题解决过程中突然产生的理解飞跃，通常伴随显著的"啊哈！"体验（Aha Moment）。荣·比曼（Mark Jung-Beeman）等人于2004年在《公共科学图书馆·生物学》（*PLOS Biology*）发表的神经影像研究发现：顿悟发生前约300毫秒，右侧前颞叶（right anterior temporal lobe, rATL）出现约40Hz的γ波突然增强，提示远距离语义联系被激活——这与左脑负责近距离语义联系的模式形成鲜明对比（Jung-Beeman et al., 2004）。

顿悟的核心机制是**问题重表征（Problem Re-representation）**：问题解决者放弃原有的错误心理模型，重新组织问题要素之间的关系。经典示例是**九点问题**：在3×3排列的9个点上，要求用4条不间断的直线连通所有点。大多数人因为隐性假设"直线不能超出9个点的边界范围"而陷入僵局——这一约束条件从未在题目中出现，完全是解题者自行施加的。一旦重表征为"直线可以延伸到边界之外"，解答立刻变得显而易见。

顿悟与渐进式问题解决的关键区别在于**不可预测的进度曲线**：渐进式策略呈线性进步，而顿悟前往往出现"进度停滞"甚至主观感受上的"越来越难"，随后突然跳跃到解答——Metcalfe与Wiebe（1987）用"感觉温度计"（Feeling of Warmth）实验证明了这一点：顿悟型问题（如字谜）的主观接近感在解答前几秒才急剧上升，而非线性渐进型问题（如代数）则全程平稳上升。

## 关键公式与模型

问题空间的状态数量决定了搜索难度。对于具有 $b$ 个分支因子（每步可选操作数）、深度为 $d$ 的问题树，穷举搜索的计算复杂度为：

$$
O(b^d)
$$

例如，国际象棋的平均分支因子 $b \approx 35$，平均博弈深度 $d \approx 80$，因此完整搜索空间约为 $35^{80} \approx 10^{123}$，远超宇宙中原子总数（约 $10^{80}$）。这一数字直接解释了为何国际象棋不能用纯算法求解，而必须依赖启发式剪枝（如$\alpha$-$\beta$剪枝）与专家图式识别。

西蒙对人类工作记忆容量的估算同样关键：工作记忆一次只能处理约 $7 \pm 2$ 个组块（Miller, 1956），但专家的"组块"比新手大得多——国际象棋大师识别的是完整的棋局模式（含多个棋子的关系结构），而新手的每个组块仅对应单个棋子位置。

## 实际应用

**临床心理学中的问题解决治疗（Problem-Solving Therapy, PST）**：由内祖（Thomas D'Zurilla）与戈德弗里德（Marvin Goldfried）于1971年提出，专门针对抑郁症患者因问题解决能力受损而陷入无助感的情况。PST包含5个标准化步骤：（1）问题定向（培养积极解题态度）；（2）问题定义（将模糊困境转化为具体可操作问题陈述）；（3）方案产生（头脑风暴，暂停评价）；（4）决策（基于效益-代价分析选择方案）；（5）实施与验证（执行并评估结果）。元分析显示，PST对重度抑郁症状的效应量达 $d = 0.54$（Cuijpers et al., 2007），效果与认知行为疗法相当。

**案例**：某被诊断为抑郁症的28岁女性，将"我的生活一团糟"重新定义为"我每周工作时长超过60小时，导致睡眠不足6小时，朋友联系频率下降至每月不足1次"——这一具体化过程本身就是问题重表征的直接应用，随后她系统生成了7种可能方案，包括与上司谈判工时、建立固定社交日历等，最终选择并实施了其中2种。

**人工智能与启发式搜索**：A*算法结合了实际路径代价与启发式估算函数 $f(n) = g(n) + h(n)$，其中 $g(n)$ 为从起点到当前节点的实际代价，$h(n)$ 为从当前节点到目标的启发式估算。当 $h(n)$ 永不高估实际代价时（称为"可采纳启发式"），A*算法保证找到最优路径——这在形式上与手段-目的分析的心理学模型高度对应。
