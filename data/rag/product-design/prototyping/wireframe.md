# 线框图

## 概述

线框图（Wireframe）是产品设计中以黑白灰色块、线条和占位符文字表达页面结构与布局的低保真设计工具。它刻意省略颜色、字体样式和真实图片，只用矩形框表示图片区域、横线表示文字内容、简单标注说明交互逻辑，让设计师和团队将注意力集中在信息架构与功能布局上，而不是视觉风格的取舍。

线框图的概念最早可追溯至1990年代软件工程领域的界面规格文档。当时设计师在纸上手绘"盒子和箭头"来向开发者传达屏幕布局，这种方式后来被称为"纸面原型"（Paper Prototype）。2000年代后，随着 Balsamiq（2008年由Peldi Guilizzoni发布）等专用线框工具的出现，数字线框图成为产品设计标准流程的一部分。Balsamiq的核心设计哲学是"刻意手绘感"——其默认字体和图形带有轻微不规整感，目的是在视觉层面强化"这只是草稿"的心理暗示，防止评审者在结构未定稿前就纠结于像素级细节。如今Figma的Frame+矩形工具组合也承担了相同的职责。

线框图之所以重要，在于它把"页面上有什么"和"页面长什么样"两个问题彻底分开。根据Nielsen Norman Group（Budiu, 2021）的研究，使用低保真线框图进行早期用户测试，发现布局问题的效率与高保真原型相当，但制作成本仅为后者的约10%—30%。一个电商商品详情页的线框图只需20分钟就能画完，却能立刻暴露出"加购按钮是否出现在用户首屏视野内"这类结构性问题，避免团队在高保真稿完成后才发现布局缺陷，节省数倍的返工成本。

## 核心原理

### 信息层级的视觉权重表达

线框图用元素的尺寸大小和位置关系来表达信息层级，而非颜色对比度。主标题对应的矩形框高度通常是正文占位线条的3倍左右，这种尺寸差异直接反映内容的重要程度。导航栏（Nav Bar）、英雄区（Hero Section）、内容区（Content Area）、页脚（Footer）这四个层次在线框图中必须有可识别的高度比例差异，否则线框图失去结构指导意义。

在移动端设计中，一个常见的高度比例参考为：

$$
H_{nav} : H_{hero} : H_{content} : H_{footer} \approx 1 : 3 : 8 : 1
$$

其中 $H$ 代表各区域的竖向高度比。这一比例并非绝对标准，但它反映了"内容区占主导、导航与页脚保持克制"的移动端布局原则。常见规范是用粗线框（2px边框）代表可交互组件（按钮、输入框），用细线框（1px边框）代表静态容器（卡片、图片占位符），通过线宽差异在无色彩的图纸上区分可操作与不可操作元素。

### 占位符的标准化符号体系

线框图有一套约定俗成的占位符语言，其标准化程度已接近行业默认规范：

- **图片区域**：矩形内加"×"对角线（源自印刷排版时代的"图片待定"标记）
- **视频区域**：矩形内加三角形播放图标
- **正文文字**：等宽水平线段，通常以8px高、间距4px排列模拟行文
- **图标**：边长16px或24px的小方框加文字标注（如"[搜索图标]"）
- **用户头像**：圆形框内加斜线或人形剪影

这套符号体系让产品经理、开发工程师、设计师看到同一张线框图时能形成统一理解，而无需猜测某个灰色方块代表什么内容。《用户体验要素》（Jesse James Garrett, 2002, New Riders出版社）将这种层次称为"框架层"（Frame Layer），明确指出线框图是连接信息架构（结构层）与视觉设计（表现层）的桥梁文档。

### 灰度等级与保真度控制

低保真线框图通常只使用三种色值：纯黑（`#000000`）用于边框与标注文字、纯白（`#FFFFFF`）用于背景、中间灰（`#CCCCCC` 或 `#999999`）用于填充占位区域。这种刻意限制颜色数量的做法，使评审者无法因为"这个按钮颜色不好看"而偏离对布局结构的讨论。

当某个区域需要特别标注为"此处为主操作按钮"时，允许引入单一语义色（常用 `#1677FF`，即Ant Design的品牌蓝），但全图只能出现一处，以保持低保真特性。在Figma中，可以通过以下方式强制执行三色规范：

```json
// Figma 本地样式配置示例（Local Styles）
{
  "colorStyles": [
    { "name": "WF/Black",   "hex": "#000000" },
    { "name": "WF/Gray",    "hex": "#CCCCCC" },
    { "name": "WF/White",   "hex": "#FFFFFF" },
    { "name": "WF/Accent",  "hex": "#1677FF", "usage": "仅主操作按钮，全图唯一" }
  ]
}
```

将上述颜色封装为Figma Local Color Styles并发布为团队组件库后，任何使用该库的设计师将只能从调色板中选取这四种颜色，从流程层面保证线框图的低保真一致性。

### 标注规范与交互说明

线框图需要配合文字标注说明元素的交互状态和业务规则，这是线框图区别于截图或草图的关键。标注一般放在画框右侧或底部，用数字编号与图中对应元素连线。典型标注内容包括：

1. **条件显示逻辑**：例如"用户未登录时，此区域显示登录引导横幅；登录后隐藏"
2. **文本截断规则**：例如"商品标题最多显示2行，超出部分以'…'省略"
3. **跳转目标**：例如"点击后跳转至订单详情页（见Frame #12）"
4. **数据边界**：例如"购物车最多显示99件商品，超出显示'99+'"

没有标注的线框图只是一张结构示意图，有标注的线框图才是可交付的设计规格文档。按照《About Face 4》（Alan Cooper等, 2014, Wiley出版社）的定义，带完整标注的线框图属于"交互规格说明书"（Interaction Specification）的最小完整单元。

## 关键工具与绘制流程

### Figma中的标准线框图工作流

在Figma中绘制移动端线框图的标准步骤如下：

1. 新建Frame，选择预设尺寸 **375×812**（iPhone 14标准逻辑分辨率）
2. 在Frame属性面板中关闭"Clip Content"以便标注内容可延伸至画框外
3. 引用团队线框图组件库，拖入预制的导航栏、卡片、按钮等灰度组件
4. 用Rectangle工具（快捷键 `R`）绘制图片占位符，填充 `#CCCCCC`，内部用Line工具画"×"对角线
5. 用Text工具（快捷键 `T`）添加标注，字号统一使用12px，颜色 `#000000`
6. 完成后导出为PDF或使用Figma Prototype功能添加页面跳转热区，交付评审

### 手绘线框图的应用场景

数字工具并非唯一选择。在需求探索的最早期（如用户访谈刚结束、概念尚未收敛时），手绘线框图的速度优势远超数字工具。研究显示，一名有经验的设计师用A4纸和马克笔，在5分钟内可以完成一个包含6个页面节点的低保真用户流程草图，而同等内容在Figma中需要30—45分钟。手绘线框图通常配合点阵纸（Dot Grid Paper）使用，以5mm间距的点阵作为对齐参考，保持元素排列的整洁度。

## 实际应用案例

**案例一：移动端App新功能评审**

产品经理在需求评审前，用Figma在375×812像素的Frame内绘制线框图，排列出"顶部导航栏（44px高）+ 内容列表（可滚动区域）+ 底部固定操作栏（83px高，含iPhone底部安全区34px）"的三段式布局。这张线框图在评审会上投屏后，开发工程师立即判断出列表区域需要虚拟列表（Virtual List）优化，iOS开发确认底部安全区已预留，避免了一轮返工。整个线框图绘制耗时22分钟，节省了约3小时的高保真稿修改成本。

**案例二：电商首页改版的A/B方案对比**

设计团队针对同一改版需求绘制了两版线框图：方案A将搜索栏置于顶部固定位置，方案B将搜索栏嵌入英雄Banner内部。两张线框图打印后直接贴在办公室走廊，邀请非设计部门的同事用贴纸投票。这种"走廊测试"（Hallway Testing）在线框图阶段完成，成本接近零，而同类测试若在高保真阶段进行，仅制作两版视觉稿就需要4—6个工时。

例如，方案A线框图中搜索栏占据顶部44px固定高度，用户无需滚动即可触达；方案B中搜索栏在英雄区（高度240px）内，用户下滑后搜索入口消失，需依赖底部Tab栏的放大镜图标。这一结构差异在灰度线框图阶段一目了然，与颜色、配图风格完全无关。

## 常见误区

**误区一：线框图应该"画得漂亮"**

部分初次接触线框图的设计师会用精细的阴影、渐变灰度和等比缩放的真实图标来美化线框图。这种做法适得其反：当线框图"太好看"时，评审者会开始讨论"这个图标风格不对""这里的圆角太大"，而不是讨论"这个按钮应不应该出现在这里"。低保真的粗糙感是线框图的功能特性，不是缺陷。

**误区二：线框图等于最终布局方案**

线框图是探索工具，不是承诺文档。一张线框图在一次评审后被完全推翻是正常现象——这正是线框图的价值所在：用最低成本验证假设。把线框图当作"设计师的交付成果"而强制锁定后续视觉设计方向，是对这一工具的误用。

**误区三：跳过线框图直接做高保真**

当项目周期压缩时，团队常常决定"直接做高保真省时间"。然而，在高保真阶段发现布局问题的修改成本，远高于在线框图阶段发现的同类问题。根据经验法则，同一个布局错误在线框图阶段修复耗时约15分钟，在高保真阶段修复耗时约2小时，在前端开发完成后修复耗时约半天。这一成本倍增规律在《About Face 4》中被称为"越晚发现越贵"（Cost of Change Curve）原则。

## 知识关联

**与原型设计概述的关系**：线框图是原型保真度光谱（Fidelity Spectrum）中的最低端，对应"结构原型"（Structural Prototype）阶段。理解保真度光谱有助于判断在项目的哪个阶段应该使用线框图，而非直接跳入高保真设计。

**与Figma基础的关系**：Figma的Frame、Auto Layout、Component功能是数字线框图的核心技术基础。掌握Auto Layout后，线框图中的列表、卡片组件可以实现内容自适应高度，使线框图在内容数量变化时保持结构正确性，而无需手动调整每个元素的位置。

**与高保真原型的衔接**：线框图完成评审并锁定布局结构后，设计师在同一Figma文件中基于线框图的Frame直接叠加视觉样式，逐步替换灰色占位符为真实颜色、字体和图片，
