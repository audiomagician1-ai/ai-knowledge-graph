# 深色模式设计

## 概述

深色模式设计（Dark Mode Design）是指将界面背景设置为深色调（通常为黑色或深灰色），并相应调整文字、图标、控件颜色的视觉设计方案。这一模式最早在代码编辑器和终端工具中流行，如2000年代的 Vim 和 Terminal 配色方案，后来随着 Apple 在 2018 年于 macOS Mojave 中正式推出系统级深色模式，以及 Google 在 Android 10（2019年）中加入全局深色主题，深色模式逐渐成为主流消费类产品的标配功能。

深色模式之所以重要，并非单纯因为视觉风格的流行，而是因为它对用户在特定使用场景下的生理舒适度产生直接影响。在 OLED 屏幕上，深色像素实际处于关闭状态，因此深色模式可以显著降低屏幕耗电量——根据 Google 在 Pixel 手机上的测试数据，将界面从白色背景切换为纯黑（#000000）背景，在最高亮度下可节省约 63% 的屏幕电量。

## 核心原理

### 色彩亮度层级系统

深色模式的背景不应使用纯黑 #000000，而应采用深灰层级构建视觉层次。Material Design 3 规范推荐最底层背景使用 #121212（明度约 7%），在此基础上通过叠加不同透明度的白色"海拔层"来区分组件层级：海拔 1dp 叠加 5% 白色、海拔 2dp 叠加 7% 白色、海拔 3dp 叠加 8% 白色，以此类推。这种方式替代了浅色模式中常用的投影（box-shadow）来表达层次感，因为深色背景上的投影几乎不可见。

### 对比度与可读性标准

文字在深色背景上的可读性遵循 WCAG 2.1（Web内容无障碍指南）的对比度要求：正文文字需达到 AA 级标准，即文字与背景的对比度比值不低于 4.5:1；大号标题（18pt 以上或粗体14pt）不低于 3:1。在 #121212 背景上，纯白文字 #FFFFFF 的对比度为 19.6:1，远超标准，但长时间阅读会产生光晕效应（Halation）。因此，Material Design 推荐使用 87% 透明度的白色（rgba(255,255,255,0.87)）作为主要文字色，既满足对比度要求，又减少光晕不适。

### 色彩饱和度与品牌色调整

直接将浅色模式的品牌色用于深色背景往往会导致两类问题：高饱和度颜色在深色背景上产生视觉振动感（Vibration），而低亮度颜色则完全消失在深色背景中。解决方案是将品牌色在 HSL 色彩空间中提高亮度（Lightness）值，同时适当降低饱和度（Saturation）。例如，一个在浅色模式下使用 HSL(220, 90%, 50%) 的蓝色按钮，在深色模式下应调整为 HSL(220, 70%, 70%) 左右。图像和插画同样需要处理：在深色背景上叠加深色遮罩层或降低图片亮度，避免亮色图片孤立地"浮"在界面中。

## 实际应用

**移动端夜间阅读场景**：以微信阅读为例，其深色模式不仅将背景切换为深色，还对书籍封面缩略图添加了轻微的暗化处理（约叠加20%黑色遮罩），并将页面纸张色从白色改为 #1A1A1A，避免了纯黑带来的阅读疲劳。

**数据可视化图表**：在深色背景上，折线图和柱状图的颜色需要专门处理。坐标轴和网格线应使用低透明度白色（约15%-20%白色），而非浅色模式中常用的浅灰色，确保在不干扰数据本身的前提下保持可见性。

**状态颜色（语义色）的处理**：错误状态红色、成功状态绿色、警告状态黄色在深色模式下均需提亮。例如，通常的错误红色 #F44336 在深色背景上应调整为 #EF9A9A（较浅的粉红），保持语义含义的同时满足对比度要求。

## 常见误区

**误区一：深色模式就是把颜色全部反转**。直接使用 CSS 的 `filter: invert(100%)` 或将所有颜色取反，会导致图片颜色失真（照片变成负片效果）、品牌色失去意义、部分图标出现意外的高亮。正确做法是为每一个色彩角色（背景色、表面色、主色、文字色等）分别定义深色模式下的对应值。

**误区二：深色模式一定比浅色模式更省电**。这一结论仅适用于 OLED 和 AMOLED 屏幕技术。在 LCD 屏幕上，背光灯始终处于开启状态，深色模式对电量消耗几乎没有影响。设计时不应以"节省电量"作为向所有用户强推深色模式的理由，而应将其作为可选项，让用户根据自身设备和偏好选择。

**误区三：深色模式对所有用户都更护眼**。在明亮环境下使用深色模式，屏幕与周围环境的亮度对比反而更大，可能导致更严重的眼部疲劳。而对于有白内障或某些视觉障碍的用户，深色模式中低对比度的辅助文字（如占位符文字）会比浅色模式更难辨认。

## 知识关联

深色模式设计建立在基础色彩理论（HSL色彩空间、对比度计算）和可访问性标准（WCAG 2.1）之上，掌握这两者才能做出符合标准的深色配色方案。下一步学习**明暗主题设计**（Light/Dark Theme Design），将从单一的深色模式方案扩展到如何系统性地构建一套同时支持浅色和深色的双主题设计系统，涉及设计 Token 体系的建立、CSS 自定义属性（Custom Properties）的运用，以及如何在设计工具（如 Figma 的 Variables 功能）中管理跨主题的色彩映射关系。
