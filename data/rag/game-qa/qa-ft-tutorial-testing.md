# 新手引导测试

## 概述

新手引导测试（Onboarding Tutorial Testing）是游戏质量保证中针对玩家首次体验流程的专项测试，验证游戏内置教程能否在不同操作路径下完整、正确地将玩家引导至正式游戏状态。其测试目标不是功能是否"存在"，而是引导序列的**流程完整性**、**跳过机制的可靠性**以及**异常中断后的恢复能力**。

新手引导系统最早在1990年代的主机游戏中以强制线性动画出现，随着手机游戏在2010年代爆发，交互式逐步引导（Step-by-Step Interactive Tutorial）成为主流标准。目前主流游戏引擎如Unity和Unreal均提供引导状态机（Tutorial State Machine）节点，测试人员需要直接干预这些状态节点来构造测试场景。

新手引导的缺陷成本极高。根据移动游戏行业数据，玩家在新手引导阶段（通常为前3分钟）的流失率可达40%至70%。一个卡住流程的Bug或跳过后触发的空指针异常，会在上线首日造成大规模用户流失，且流失玩家几乎不会回头，因此新手引导测试的优先级通常排在P0级缺陷验证范围内。

## 核心原理

### 教程流程完整性验证

完整性验证要求测试人员按游戏设计文档（GDD）中定义的引导步骤序列，逐一确认每个步骤的**触发条件、展示内容和完成判定**三要素均正确执行。例如，若第3步引导要求玩家点击屏幕右下角的攻击按钮，则需验证：高亮遮罩（Highlight Mask）覆盖范围是否正确、点击响应是否仅限该按钮区域、完成后是否向引导状态机发送`STEP_COMPLETE(id=3)`事件。

测试用例设计通常以**引导步骤序列图**为基准，覆盖所有分支节点。部分游戏拥有条件分支引导（如角色职业不同导致不同引导路径），测试人员需为每个分支路径建立独立的用例集合，确保每条路径都能到达"引导完成"的终止状态。

### 跳过机制测试

跳过机制（Skip/Fast-Forward）是新手引导中最容易引入缺陷的功能点。测试需覆盖三类跳过场景：

1. **入口阶段跳过**：在引导第1步尚未执行前触发跳过按钮，验证游戏是否直接进入默认初始状态，而非停留在半初始化的中间状态；
2. **中途跳过**：在第N步（N≥2）触发跳过，此时游戏需将前N-1步引导所涉及的资源、道具、解锁内容**批量补发**给玩家，跳过逻辑的核心Bug往往是批量补发逻辑遗漏了某些步骤奖励；
3. **跳过后的状态一致性**：跳过完成后，玩家账号的引导完成标志位（Tutorial Complete Flag）必须被正确写入服务器存储，避免重启游戏后引导重新触发。

常见测试检查点是跳过后立即检查背包道具数量、技能解锁状态和主界面UI是否与"完整通过引导"后的状态完全一致，二者差异即为缺陷。

### 异常中断恢复测试

异常中断是指在引导进行中发生强制打断事件，包括：**来电/系统弹窗（iOS/Android权限请求）**、**App切换至后台**、**网络断线重连**、**游戏崩溃重启**等场景。每类中断的恢复要求不同：

- **后台切换**：重新进入游戏后，引导应从中断前的**同一步骤**继续，而非重置到步骤1或跳过剩余步骤；
- **崩溃重启**：引导进度需依赖服务端或本地持久化存储（如`PlayerPrefs`或服务器Session）在崩溃前写入最近一次已完成的步骤ID，重启后从该步骤的**下一步**继续；
- **网络断线**：在需要服务端验证的引导步骤（如首次充值引导、社交引导）中，断线应触发本地缓存机制，重连后补同步，而非直接跳过当前步骤或永久卡住流程。

测试中断恢复的标准方法是：在目标步骤执行到50%时强制触发中断，记录当前步骤ID，重进游戏后验证恢复点的准确性误差不超过1个步骤。

## 实际应用

**案例1：某RPG手游引导跳过缺陷**
该游戏引导第5步会向玩家发放"新手武器（攻击+50）"，跳过逻辑代码仅循环补发步骤1至4的奖励，遗漏了步骤5。测试人员通过对比"完整通关引导"与"第1步跳过"两种路径的背包截图，发现武器缺失，定位为跳过逻辑中的循环边界错误（`i < 5` 应为 `i <= 5`）。

**案例2：iOS权限弹窗中断引导**
某战略游戏在引导第2步触发推送通知权限请求，权限弹窗弹出后玩家选择"不允许"，游戏误判为引导完成并清除了引导状态，导致后续步骤永久消失。测试人员使用iOS模拟器的权限注入工具在引导第2步前预设"拒绝"状态，复现并记录该缺陷，修复方案为将权限请求响应与引导步骤推进事件解耦。

**案例3：弱网环境引导卡死**
测试人员使用网络限速工具将带宽限制至50kbps，在需要服务端确认的"好友邀请引导步骤"中模拟超时，发现游戏进入无限等待状态且无超时回退逻辑，需添加30秒超时后自动降级为本地完成的容错机制。

## 常见误区

**误区1：认为完整走一遍引导即为充分测试**
实际上仅走"正常路径"只覆盖了约30%的引导测试场景。跳过、中断恢复、二次登录后的引导状态、多设备同账号登录时的引导同步，这些路径均需独立测试用例覆盖。跳过后的状态不一致Bug是最常见的引导类缺陷，却最容易被"走一遍"测试策略遗漏。

**误区2：跳过按钮不可见即代表跳过机制已禁用**
部分版本在某些引导步骤会隐藏跳过按钮，但底层跳过逻辑代码仍然存在。测试人员需使用调试菜单或发送直接的跳过指令（如ADB命令或Cheat Code）来验证隐藏状态下的跳过路径是否同样执行了完整的状态清理，防止开发阶段的调试跳过路径泄漏至正式版本。

**误区3：引导测试与经济系统测试完全独立**
新手引导通常包含道具发放、货币奖励和商城解锁等步骤，这些步骤与经济系统直接耦合。若跳过引导导致首次充值优惠标志位（First Purchase Flag）未正确置位，玩家将无法触发首充折扣，直接影响商业化数据。因此引导测试用例需包含对经济系统关键标志位的检查。

## 知识关联

**前置依赖：经济系统测试**
在经济系统测试阶段，测试人员已建立了道具发放验证、货币数值校验和标志位检查的测试方法。这套方法在新手引导测试中直接复用于验证跳过补发逻辑和首充标志位的正确性——引导中的每个奖励发放节点本质上是一次小型的经济系统事务，需用相同的校验思路审查。

**后续概念：负面测试**
新手引导测试主要验证预期路径（正常走完、正常跳过、正常恢复），而负面测试将进一步探索引导系统在**非预期输入**下的表现：例如快速连续点击跳过按钮（重复提交）、在引导动画播放期间强制调用其他游戏模块的API、或在引导完成标志位写入过程中强制杀进程。负面测试能发现引导系统在边界压力下的崩溃和数据损坏问题，是新手引导测试的进阶层次。
