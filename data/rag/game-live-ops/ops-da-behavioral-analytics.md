# 行为分析

## 概述

行为分析（Behavioral Analytics）在游戏运营数据分析中，专指对玩家在游戏内产生的离散动作序列进行结构化解读的方法论体系。与仅统计"多少玩家登录"的聚合指标不同，行为分析关注的是"玩家按什么顺序做了什么事"——即将玩家的操作记录（事件流）还原为可计算的行为模式。

行为分析方法论的规模化应用始于2010年前后，随着Facebook游戏（如《FarmVille》）将事件埋点标准化，Mixpanel等工具在2009-2012年间将漏斗分析和留存曲线推向游戏行业主流。现代移动游戏平均每DAU每天产生约300-800条行为事件，这些原始事件日志是行为分析的基础数据来源。

行为分析之所以在游戏Live-Ops中不可替代，原因在于玩家流失往往不是某一时刻的突变，而是行为模式劣化的累积结果。通过分析行为序列，运营团队可以在玩家流失发生前约3-7天识别出"预流失行为特征"（如会话时长连续缩短、主线任务完成率下降），从而触发定向干预措施。

## 核心原理

### 行为序列分析（Sequence Analysis）

行为序列分析将玩家的操作记录为有序的事件元组：`E = (user_id, event_name, timestamp, properties)`。分析的核心目标是找到高频出现的事件子序列，即"频繁序列模式"。

常用算法为 **PrefixSpan**（前缀投影序列挖掘），其时间复杂度优于早期的 AprioriAll 算法，能在百万级事件日志上在分钟内完成计算。实操中，游戏分析师通常设定时间窗口（如"同一会话内"或"24小时内"），并定义最小支持度阈值（如支持度 ≥ 0.5%，即至少0.5%的玩家发生此序列）来过滤噪声。

典型应用场景：发现"进入商店 → 查看皮肤 → 退出未购买"的序列在付费玩家中出现频率远低于免费玩家，提示商店UI存在转化障碍。

### 路径分析（Path Analysis）

路径分析将游戏内界面或关键节点建模为有向图的节点，玩家的浏览/操作路径为图中的有向边，边的权重为转移概率。其核心数据结构是 **转移矩阵** `P[i][j]`，表示玩家从状态 `i` 转移至状态 `j` 的条件概率：

```
P[i][j] = count(i → j) / count(i → *)
```

路径分析重点识别两类节点：**汇聚节点**（多数路径在此终止，常为流失点）和**分叉节点**（玩家在此做出不同选择，常为关键决策点）。例如，某RPG手游通过路径分析发现，新手引导第4步"角色换装"是最大汇聚节点，37%的新玩家在此页面退出，由此确认了新手引导的优化优先级。

与序列分析不同，路径分析是概率性的，不要求事件严格相邻，更适合分析跨会话的宏观导航行为。

### 会话分析（Session Analysis）

会话（Session）是行为分析的基本计量单位，定义为同一玩家在一段连续活跃时间内的所有事件集合。行业通用的会话切割规则为：**连续30分钟无事件则会话终止**（iOS/Android统计SDK默认值），但高强度游戏（如MOBA）有时缩短为5-10分钟。

会话分析的核心指标包括：
- **会话时长（Session Length）**：单次会话从首个事件到末个事件的时长，分布通常呈对数正态分布
- **会话深度（Session Depth）**：单次会话内完成的关键行为数量（如关卡数、对局数）
- **会话间隔（Session Interval）**：相邻两次会话之间的时间差，间隔分布的变化是预测次日留存的强预测变量

会话级聚合数据（而非原始事件）是构建流失预测模型（见"预测建模"）的主要特征来源，通常按`user_id × session_date`维度汇总为特征向量。

## 实际应用

**案例一：《部落冲突》类策略游戏的新手漏斗诊断**

某策略手游运营团队通过行为序列分析，识别出新用户在前3次会话中必须完成"建造-升级-战斗"闭环才能留存至Day7。在此之前，运营团队仅能看到Day7留存率为18%，无从定位原因。行为分析揭示：有62%的新用户在首次会话中从未触发"战斗"事件，调整新手引导后该比例降至29%，Day7留存提升至26%。

**案例二：PVP游戏的付费转化路径优化**

通过路径分析计算"首次观看广告 → 商店浏览 → 首次付费"的转化链路，发现玩家在"皮肤详情页"的平均停留时长为18秒，但仅有4.2%转化付费。对比付费用户路径，发现其在详情页后会额外访问"好友装扮"页面，据此在详情页增加"看好友同款"入口，付费转化率提升至6.8%。

## 常见误区

**误区一：把"事件计数"当"行为分析"**

统计"关卡完成次数"是指标聚合，不是行为分析。行为分析的最小有效单位是"序列"或"路径"，必须保留事件的顺序信息和上下文关联。许多团队将埋点数据仅用于计算DAU/MAU，浪费了90%以上的行为信息。

**误区二：忽视时间窗口定义对结论的影响**

同样的"付费前行为序列"分析，采用"付费前1小时"和"付费前7天"作为时间窗口，识别出的关键前置行为可能完全不同。前者捕捉即时触发因素，后者反映习惯养成过程。未明确时间窗口的行为分析结论无法复现，也无法用于后续的预测建模特征工程。

**误区三：将路径分析的转移概率等同于因果关系**

高转移概率 `P[商店 → 付费] = 0.15` 仅表示"进入商店的玩家中有15%发生付费"，不代表"进入商店导致付费"。忽视选择偏差（Survivorship Bias）会导致错误的运营决策——例如强制将所有玩家导入商店，实际上可能因打断游戏节奏而降低整体付费率。

## 知识关联

**与运营SQL基础的衔接**：行为序列分析在SQL层面依赖窗口函数（`LAG()`、`LEAD()`、`ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY timestamp)`）来重建事件顺序；路径分析的转移矩阵计算本质是自连接（Self-Join）查询。掌握运营SQL基础中的事件表结构和时间戳处理，是独立完成行为分析数据提取的前提。

**与舆情分析的互补关系**：舆情分析捕捉玩家"说了什么"（论坛抱怨、评论情感），行为分析记录玩家"做了什么"（实际操作序列）。两者结合可以验证或推翻假设：玩家在论坛抱怨"商店太难找"，行为路径数据若显示商店入口点击率正常，则问题可能在商店内部流程而非入口位置。

**通向预测建模的桥梁**：行为分析产出的会话级特征向量（会话时长均值、关键动作完成率、行为多样性指数等）直接作为流失预测、LTV预测等监督学习模型的输入特征。行为分析阶段的特征工程质量——尤其是时间窗口的选择和序列特征的抽象粒度——直接决定后续预测模型的上限性能。
