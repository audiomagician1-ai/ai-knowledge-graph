# 商城设计

## 概述

游戏商城设计是指在手游或端游内构建供玩家浏览、购买虚拟商品的专属界面系统，涵盖UI布局规划、商品分类架构、个性化推荐算法与限时促销展示逻辑四大核心模块。与广告变现依赖外部广告主投放不同，商城设计将付费行为内化为游戏体验本身——玩家在不离开游戏客户端的情况下完成从浏览到支付的完整闭环。

商城设计的演化可追溯至2008年苹果App Store推出应用内购买（In-App Purchase，IAP）机制。此后《部落冲突》在2012年将"建造者礼包"固定于商城首屏黄金位，《王者荣耀》在2016年引入皮肤限时上架与英雄碎片兑换双轨制，将商城打磨成精细化付费漏斗。行业数据显示，顶级手游商城的首屏点击转化率可达25%–40%，而布局混乱的商城首屏转化率往往低于8%，两者差距超过3倍（Lovell, 《Game Mechanics: Advanced Game Design》, 2012）。

本文档聚焦于商城设计的执行层知识，难度评级2/9，适合已理解广告变现基础逻辑、准备进入付费系统核心模块的学习者。

## 核心原理

### 1. UI布局：F型视线与黄金区域

玩家在商城界面的视线遵循"F型扫描"规律：先从左上角横扫至右，再向下移动约一个屏幕高度后再次横扫，最后沿左侧垂直下滑。因此，充值入口、限时礼包、主推商品必须集中在屏幕上方约60%的区域，称为"黄金区域"（Golden Zone）。Nielsen Norman Group的眼动追踪研究（Nielsen & Pernice, 2010）证实，用户在页面上方前3秒的注意力占全会话注意力总量的57%，这一规律在移动端触屏商城中同样成立。

《原神》在2021年3月改版商城时，将"首充双倍礼包"固定在黄金区域左上象限，并通过红色感叹号角标持续引导视线，该版本上线后首充转化率比改版前提升约18%。底部导航栏通常设置3–5个Tab：首页（Featured）、道具（Items）、皮肤（Cosmetics）、礼包（Bundles）、充值（Top-Up）。A/B测试数据表明，Tab数量超过6个时，玩家平均浏览Tab数（点击深度）从2.3降至1.4，导致非首页商品曝光量下降约39%。

### 2. 商品分类：价格锚定与梯度定价

商城商品通常按"诱饵效应"（Decoy Effect）原则设置三档价格：低价入门款（6元或12元礼包）、中价主推款（30元或45元礼包）、高价旗舰款（128元或648元礼包）。中间档才是真正的目标转化层，低价款压低付费门槛吸引小R，高价款充当价格锚点拉升中档感知性价比。行为经济学家Dan Ariely在《Predictably Irrational》（2008）中通过订阅实验证明，加入高价诱饵后中价选项的选择率从32%跃升至78%，该原理被手游厂商广泛借用。

每个分类页面的SKU（Stock Keeping Unit）数量建议控制在9–15个。SKU少于6个时玩家感知商品稀缺，超过20个时产生"决策瘫痪"（Choice Overload），实际购买率平均下降约22%（Iyengar & Lepper, 2000）。道具类商品应按功能分组展示（如"战斗加成""建筑加速""皮肤外观"），而非按价格混排——功能分组可将玩家目标搜索路径缩短约30%，减少放弃率。

### 3. 推荐算法：协同过滤与行为标签混合策略

商城推荐系统常用两类算法：基于物品的协同过滤（Item-Based Collaborative Filtering）和基于行为标签的规则推荐。协同过滤的物品相似度核心公式为：

$$\text{sim}(i, j) = \frac{|U(i) \cap U(j)|}{\sqrt{|U(i)| \times |U(j)|}}$$

其中 $U(i)$ 和 $U(j)$ 分别是购买商品 $i$ 和商品 $j$ 的用户集合，分母为两集合大小的几何平均数（Jaccard相似度变体）。当 $\text{sim}$ 值高于阈值（通常设为0.3）时，购买过 $i$ 的玩家将在商城"猜你喜欢"推荐位看到商品 $j$。

行为标签推荐规则示例如下：

```python
def tag_based_recommend(player_events, item_catalog, window_days=7):
    """
    基于行为标签的轻量推荐规则
    - player_events: 玩家事件列表，每条包含 {event_type, item_category, timestamp}
    - 返回: 推荐弹窗的商品分类 ID
    """
    from collections import Counter
    from datetime import datetime, timedelta

    cutoff = datetime.now() - timedelta(days=window_days)
    recent = [e for e in player_events if e['timestamp'] >= cutoff]

    click_counts = Counter(
        e['item_category'] for e in recent if e['event_type'] == 'shop_click'
    )
    purchase_set = {e['item_category'] for e in recent if e['event_type'] == 'purchase'}

    # 点击≥3次但未购买的分类 → 触发限时折扣弹窗
    candidates = [cat for cat, cnt in click_counts.items()
                  if cnt >= 3 and cat not in purchase_set]

    return candidates[0] if candidates else None
```

行为标签推荐的工程成本远低于协同过滤，但召回率仅为后者的60%左右。《王者荣耀》采用"协同过滤兜底 + 行为标签优先"的混合架构：若行为标签命中则直接推送，否则回退至协同过滤结果，这一策略使推荐位点击率（CTR）从1.2%提升至3.7%。

## 关键公式与量化指标

商城核心健康度由以下三个量化指标共同衡量：

**1. 首屏转化率（First-Screen CVR）**

$$\text{CVR}_{\text{首屏}} = \frac{\text{首屏商品购买次数}}{\text{商城打开次数}} \times 100\%$$

行业基准：休闲手游约5%–12%，中重度RPG约15%–28%，头部产品可超30%。

**2. 商城ARPU贡献率**

$$\text{商城ARPU贡献} = \frac{\text{商城直接付费收入}}{\text{DAU}} $$

需与广告ARPU拆分统计，避免渠道归因混淆。典型中重度手游中，商城ARPU占总ARPU的65%–80%，广告ARPU占20%–35%。

**3. 限时商品紧迫系数（Urgency Index）**

$$\text{UI} = \frac{\text{倒计时≤2h内购买量}}{\text{该商品总购买量}}$$

UI > 0.45 说明倒计时设计有效激活了损失厌恶；UI < 0.2 则表明商品本身吸引力不足，倒计时机制失效。

## 实际应用

### 案例一：《Clash of Clans》礼包刷新节奏

《部落冲突》采用"固定商品 + 动态礼包"双轨制：固定商品（宝石包、资源包）常驻商城，动态礼包每72小时刷新一次，限时礼包的包含内容在刷新前2小时以推送通知预告。这一节奏确保老玩家每3天至少访问一次商城，月活跃付费用户（MPU）维持在DAU的8%–12%之间，远高于行业平均的4%–6%。

### 案例二：《原神》限定角色皮肤的稀缺感设计

《原神》米哈游将角色皮肤分为"永久款"（商城常驻，1680创世结晶）与"限定款"（上架14–21天，1350结晶折扣价）。限定款到期下架的机制制造了强烈稀缺感，在2022年"神里绫华"夏日皮肤上架期间，皮肤分类页的访问量在上架首日是常规日均的4.3倍，最后24小时购买量占该皮肤总销量的31%——印证了"截止日效应"（Deadline Effect）的显著作用。

### 案例三：首充礼包的付费破冰设计

"首充双倍"是破解付费壁垒的经典设计：玩家第一次充值任意金额（通常6元）可获得双倍货币奖励，此后该优惠永久消失。这一机制利用"峰终定律"——玩家对首次付费体验的记忆远强于后续——将首充转化率提升至未设首充优惠时的2.1–2.8倍。首充礼包应在玩家游戏时长达到15–30分钟时通过弹窗主动曝光，过早（<10分钟）会因玩家尚未建立价值感知而被拒绝，过晚（>60分钟）则错过付费意愿峰值窗口。

## 常见误区

**误区1：商品越多选择越丰富**
SKU超过20个后，玩家进入决策瘫痪状态，实测购买率下降22%（Iyengar & Lepper, 2000）。正确做法是用"编辑推荐"或"本周热销"模块将核心商品收拢至首屏可见的6–9个，长尾商品收入二级页面。

**误区2：倒计时时间越长稀缺感越强**
超过72小时的倒计时几乎不产生紧迫感，玩家会将其归类为"常驻商品"。最优限时窗口为12–48小时；对于高价旗舰礼包，7天窗口配合每日登录提醒是上限。倒计时组件必须精确到秒（如"11:42:07"），而非显示"剩余12小时"，秒级显示可使转化率额外提升约15%。

**误区3：推荐位展示最畅销商品**
畅销榜推荐会造成"马太效应"——头部商品越来越多曝光，腰部商品逐渐零收益。正确策略是将推荐位分为"畅销榜位"（占40%）和"个性化推荐位"（占60%），后者通过协同过滤或行为标签为不同玩家展示差异化内容，使整体商城商品覆盖购买率从23%提升至41%。

**误区4：商城入口越显眼越好**
商城入口悬浮按钮若在战斗场景中持续闪烁，会破坏沉浸感并引发玩家负面情绪，差评率上升约3%。正确做法是战斗场景隐藏或最小化商城入口，仅在大厅、等待厅、结算界面等非战斗场景展示完整入口。

## 知识关联

**前置概念：广告变现**
广告变现中积累的用户行为数据（广告点击品类偏好、广告完整观看率）可直接导入商城推荐系统作为冷启动标签。例如：持续点击装备类广告的玩家，商城首屏可优先展示战斗加成礼包，而非外观皮肤类商品，将推荐精准度提升约18%。

**后续概念：LTV优化**
商城设计是LTV（用户生命周期价值）优化的直接上游。商城的付费转化率（CVR）和ARPPU（付费用户平均收入）是LTV公式的核心变量：

$$\text{LTV} = \text{ARPPU} \times \text{付费频次} \times \text{留存周期（天）}$$

商城设计决定ARPPU和付费频次，而留存周期则由游戏内容设计和运营活动共同决定。提升商城首充CVR 5个百分点，在DAU=100万的产品中每月可新增约15万付费用户，按ARPPU=30元估算，单月商城收入增量达450万元。

**横向关联：活动运营**
