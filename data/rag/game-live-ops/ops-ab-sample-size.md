# 样本量计算

## 概述

样本量计算（Sample Size Calculation）是A/B测试启动前的必要定量步骤，用于确定每个实验组最少需要多少用户才能可靠地检测出预设的效果差异。如果样本量不足，实验即便真实存在效果差异，也会因统计功效不足而"看不见"；样本量过大则浪费流量资源并拖延决策周期。

样本量计算公式的理论基础可追溯到1931年耐曼（Jerzy Neyman）与皮尔逊（Egon Pearson）建立的假设检验框架。在游戏直播运营场景中，样本量计算尤其关键，因为直播间流量往往呈现高峰低谷交替的特征，若只截取流量高峰期数据，容易因总量"达标"但用户行为偏差而导致结论失真。

样本量计算的核心意义在于：在"假设制定"阶段确定了基线指标和最小可检测效应量（MDE）之后，样本量公式将这些业务决策转化为具体的流量配额，使后续随机分组操作有据可依。

## 核心原理

### 三大输入参数的含义与典型取值

样本量计算依赖三个核心参数：

- **显著性水平 α（置信水平的补数）**：即"第一类错误"的容忍概率，通常设为 0.05，对应95%置信水平。在游戏直播运营中，如果一个错误决策会导致大规模礼物系统改版，建议将 α 收紧至 0.01。
- **统计功效 1−β**：即"检测到真实差异"的概率，通常设为 0.80（即允许20%的第二类错误），高风险决策建议提升至 0.90。
- **效应量（Effect Size）**：预期对照组与实验组之间的差异大小，直播场景中常以"礼物收入提升率"或"场均观看时长提升X秒"来定义。

### 双比例检验的核心公式

对于比例型指标（如点击率、付费转化率），每组所需的最小样本量 *n* 为：

$$n = \frac{(z_{\alpha/2} + z_{\beta})^2 \cdot [p_1(1-p_1) + p_2(1-p_2)]}{(p_1 - p_2)^2}$$

其中：
- $p_1$ = 对照组基线转化率（来自历史数据）
- $p_2$ = 实验组预期转化率（= $p_1$ × (1 + MDE)）
- $z_{\alpha/2}$ = 双尾检验的Z分位数，α=0.05时取1.96
- $z_{\beta}$ = 功效对应的Z分位数，功效=0.80时取0.842

以某直播平台礼物引导按钮为例：若基线点击率 $p_1$ = 0.08，期望检测到相对提升10%（即 $p_2$ = 0.088），代入公式（α=0.05，功效=0.80），每组约需 **14,700人**，两组合计约29,400人。

### 效应量对样本量的非线性影响

效应量与样本量成平方反比关系：MDE减半，所需样本量约扩大4倍。例如，将上述案例的MDE从10%降至5%（$p_2$ = 0.084），每组样本量将从14,700激增至约58,800人。这一非线性关系意味着：在直播间流量有限的运营场景下，将MDE设置得过于激进（追求检测1%~2%的微小变化）往往不现实，需要在"敏感度需求"与"流量成本"之间权衡。

### 均值型指标的样本量计算

对于"场均观看时长"等连续指标，需要引入标准差 σ，公式变为：

$$n = \frac{2\sigma^2(z_{\alpha/2} + z_{\beta})^2}{\delta^2}$$

其中 δ 为预期的绝对差值，σ 需从历史数据中估算。连续型指标的标准差通常远大于其均值，因此所需样本量往往比比例型指标更大。

## 实际应用

**直播间新版礼物特效A/B测试**：某平台希望检测新礼物特效是否能将礼物按钮点击率从5%提升至5.75%（MDE=15%），α=0.05，功效=0.80，计算结果约每组需12,000用户。该平台日活直播间访客为80,000人，若以50/50流量分配，约需 **3天** 即可完成数据采集，符合运营节奏。

**充值转化率测试**：若基线充值率为0.3%，MDE设为20%（目标 $p_2$=0.36%），α=0.05，功效=0.80，每组需约 **240,000用户**。对于日活仅数万级别的直播间，这意味着实验需持续数周，运营团队需评估是否值得等待或放宽MDE。

## 常见误区

**误区1：用"感觉差不多了"替代公式计算**
部分运营人员在积累了约1,000条数据后便开始看结果，但若MDE为10%、基线转化率为5%，1,000人每组的统计功效仅约为28%——即便新方案真的更好，有72%的概率会"错过"这个结论。样本量不足导致的假阴性（漏报真实效果）是直播运营中最常见的决策陷阱之一。

**误区2：将整体流量误当单组样本量**
公式中的 *n* 是**每组**的样本量。若计算结果为每组10,000人，两组合计需要20,000人，而非总流量10,000人均分。将总量与单组量混淆会使实际样本量缩水一半，统计功效大幅下降。

**误区3：基线数据取错时间窗口**
用大促期间（如春节、电竞世界赛）的高点击率作为 $p_1$ 基线，计算出来的MDE对应的绝对差值会比平日大，样本量看似"变少"，但实验若在平日流量条件下运行，实际效应量会远小于预期，导致检验失效。基线指标应取与实验运行时段匹配的历史同期数据。

## 知识关联

样本量计算的上游是**假设制定**阶段输出的基线指标值（$p_1$ 或 $\sigma$）和最小可检测效应量（MDE），这两个业务决策直接决定公式输入，因此假设制定的精确度直接影响样本量计算的实用性。

样本量计算完成后，下一步是**随机分组**：计算结果给出了每组的用户数量目标，随机分组机制需要确保在达到该数量之前实验不会提前终止，同时要保证对照组与实验组的用户特征（如设备类型、历史消费等级）在统计上平衡，避免选择偏差使样本量计算的理论保证失效。
