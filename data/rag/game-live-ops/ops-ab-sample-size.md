# 样本量计算

## 概述

样本量计算（Sample Size Calculation）是A/B测试启动前必须完成的关键规划步骤，其目的是确定每个测试组需要多少用户才能可靠地检测出预期中的效果差异。在游戏运营场景中，如果样本量不足，一个真实存在的活动效果可能完全无法被统计检验识别；而样本量过大，则意味着浪费时间和运营资源，还可能让差版本长时间影响更多玩家。

这一方法论在20世纪中叶由统计学家雅各布·科恩（Jacob Cohen）系统化，他于1988年出版的《行为科学统计功效分析》确立了效应量（Effect Size）与样本量关系的标准框架。在此之前，许多实验者凭经验拍脑袋决定测试规模，导致大量实验因功效不足而得出"无显著差异"的错误结论。游戏运营A/B测试直接继承了这套方法，并针对高流量、短周期的运营节奏进行了应用适配。

在游戏运营中，样本量计算的重要性体现在每一个测试窗口都有明确的时间成本：一个版本迭代周期通常只有1到2周，在开测前就必须知道"这段时间内的流量够不够用"，而不是测完之后再发现数据量太小、结论不可信。

## 核心原理

### 三个决定样本量的输入参数

样本量计算由三个统计参数共同决定，缺少任何一个都无法得出有效结果。

**显著性水平 α（置信水平）**：通常设定为 0.05，表示允许5%的概率犯第一类错误（即实际无差异但误判为有差异）。α越小，需要的样本量越大。若将α从0.05收紧到0.01，在其他条件不变的情况下，所需样本量约增加40%至60%。

**统计功效 1-β（Power）**：通常设定为0.80，表示当真实差异存在时，有80%的概率能正确检测出来。功效不足的测试是游戏运营中最常见的错误——功效为0.5意味着抛硬币，有一半机会漏掉真实效果。将功效从0.80提升至0.90，所需样本量约增加25%。

**效应量（Effect Size）**：指预期中对照组与实验组之间差异的实际大小。对于比例型指标（如次日留存率），使用Cohen's h公式：`h = 2 × arcsin(√p₁) - 2 × arcsin(√p₂)`，其中p₁和p₂分别是两组的预期比例。效应量越小，所需样本量越大，且这种影响是非线性的——效应量减半会导致样本量增加约4倍。

### 双尾与单尾检验的选择

双尾检验（Two-tailed test）检验"A和B是否有差异"，单尾检验（One-tailed test）检验"B是否比A更好"。游戏运营中，大多数场景应使用双尾检验，因为新版本也可能带来负面效果（例如新手引导改版可能降低留存）。使用单尾检验虽然可以将所需样本量减少约20%，但这必须基于充分的业务理由，而非为了少收集数据而凑合使用。

### 样本量计算公式（双比例Z检验）

对于留存率、付费率等比例型指标，标准公式为：

```
n = (Z_{α/2} + Z_β)² × [p₁(1-p₁) + p₂(1-p₂)] / (p₁ - p₂)²
```

其中：
- `Z_{α/2}`：显著性水平对应的Z值，α=0.05时为1.96
- `Z_β`：功效对应的Z值，功效=0.80时为0.84
- `p₁`：对照组基准比例（如当前次日留存率0.40）
- `p₂`：实验组预期比例（如改版后预期0.43）

以次日留存率从40%提升至43%为例（相对提升7.5%），代入公式计算每组约需要**3,840名用户**，两组合计约7,680人。若日活只有1,000，则至少需要测试约8天才能达到样本量要求。

### 最小可检测效应量（MDE）

在已知流量的情况下，也可以反向使用公式——给定样本量，计算当前流量能检测出的最小效应量（Minimum Detectable Effect, MDE）。若7天内每组只能收集500用户，则MDE约为5到7个百分点的绝对差异，小于这个差异的效果将无法被可靠检测。这有助于运营人员在测试设计阶段就判断"这个测试在现有流量下是否有意义"。

## 实际应用

**活动付费率测试**：某手游的氪金活动改版，对照组当前付费率为8%，运营团队预期新活动付费率能达到10%（绝对提升2个百分点）。代入双尾Z检验公式，α=0.05，功效=0.80，计算得每组需约1,570名活跃用户。若该游戏日活5,000，分配40%（2,000人）给测试，则需要约1.6天即可达到样本量要求。

**新手引导流程优化**：若测试目标是将7日留存从20%提升至23%，每组需要约2,600名新注册用户。如果每日新增用户仅300，每个测试组50%流量则每天收集150人，需要约17天完成一次测试，接近一个版本周期上限，需谨慎评估是否值得占用如此长的测试窗口。

**多组测试的样本量膨胀**：若同时测试A、B、C三个版本，每组仍需达到相同的每组样本量，但由于多重比较问题，需要使用Bonferroni校正将α从0.05调整为0.05/2=0.025（对比两个实验组），导致每组所需样本量约上升35%。

## 常见误区

**误区一：用总样本量代替每组样本量**

样本量计算得出的结果是**每个测试组**所需的用户数，而非两组合计。若计算结果为4,000，则对照组和实验组各需4,000人，共计8,000人。初学者常将4,000人一分为二，导致每组只有2,000人，统计功效大幅下降至约0.50，实验结论可靠性极低。

**误区二：用相对提升率而非绝对差异输入公式**

"提升10%"在业务语境中通常指相对提升（从40%变成44%），但样本量公式需要输入的是绝对差异（p₁-p₂=0.04）以及两个具体的比例值。将10%直接代入公式会导致样本量计算结果严重偏低，测试功效不足。

**误区三：样本量达到后立即停止测试**

样本量是最低门槛，而非停止信号。达到预计样本量后，应完整收完当前自然日的数据（避免截断天内行为），并确保样本覆盖了完整的时间周期（如涵盖工作日和周末），以消除时间偏差对结论的影响。

## 知识关联

样本量计算建立在**假设制定**的输出之上——只有明确了零假设（H₀）和备择假设（H₁），才能确定需要检测的效应量是多少。若假设制定阶段没有给出对照组的基准指标（如当前留存率为40%），以及预期改善幅度（如提升至43%），样本量计算就缺乏输入参数，无法执行。

样本量计算的结论直接决定了下一步**随机分组**的流量分配方案。如果计算得出每组需要5,000用户，而日活只有8,000，那么随机分组时需要将62.5%的流量导入测试，同时意味着对照组的基准数据收集也需要在同一时间窗口内完成，而不能复用历史数据。样本量不足就进行随机分组并启动测试，是游戏A/B测试中最常见的前期规划失误。
