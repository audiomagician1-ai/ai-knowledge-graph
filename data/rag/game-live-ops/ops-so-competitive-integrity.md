# 竞技公正性

## 概述

竞技公正性是赛季运营体系中针对反作弊、匹配公平与代练打击所建立的系统性保障机制，其核心目标是确保同一赛季内所有玩家在相同规则约束下竞争，使排位积分真实反映个人竞技水平。这一概念在2010年代中期随着《英雄联盟》《守望先锋》等竞技游戏赛季化运营模式的普及而被工业化，Riot Games率先将赛季内公正性指标纳入运营健康度报告（Riot Games Engineering Blog, 2016），使其从被动应对问题转变为主动运营目标。

竞技公正性直接影响赛季经济价值：当赛季末排行榜前1%的玩家中存在可量化的作弊污染率时，正常玩家对赛季奖励（皮肤、称号、积分）的感知价值会随污染率线性下降。腾讯2019年内部研究显示，《王者荣耀》某赛季中被检测出的脚本用户若超过活跃玩家总量的0.3%，赛季末新赛季预购转化率下降约7%，这直接将竞技公正性与商业ROI挂钩。理解竞技公正性的完整框架，可参考《网络游戏运营管理》（陈曙光，高等教育出版社，2021），其中第八章专门讨论了竞技环境治理的量化体系。

## 核心原理

### 反作弊机制的赛季周期绑定

赛季化运营中的反作弊策略与单次即时打击存在本质区别，强调**批量延迟封禁（Wave Ban）**策略。反作弊系统在赛季中持续收集作弊行为数据，但故意延迟封禁执行时机至赛季结算前72小时内集中处理。这一设计具有双重目的：一是避免作弊工具开发者获得即时反馈从而快速迭代绕过检测逻辑；二是确保封禁动作与赛季积分清零同步，使违规账号无法将当前赛季的非法成果携带进入下一赛季。

Wave Ban的封禁批次通常以14天为一个评分积累周期。每个被监控账号持续累积异常行为置信度评分，当评分达到系统阈值后进入"待执行队列"，在预定的赛季关闭节点统一执行。以Valve反作弊系统VAC（Valve Anti-Cheat）为例，其公开逻辑要求作弊行为置信度≥98%方可入队，这意味着系统宁可放过边缘用户，也不接受误封正常玩家的代价。

此外，部分游戏针对高段位赛区引入**赛季内专项扫描（Mid-Season Sweep）**，即在赛季进行至第10～12周时对钻石及以上段位所有活跃账号进行一次强制深度扫描，该扫描会将内存级作弊工具（如Aimbot驱动层注入）的特征与当前赛季注册设备驱动信息进行交叉比对，触发率高于日常扫描约3.4倍（据暴雪2020年《守望先锋》作弊治理报告）。

### 段位匹配公平性的数学模型

竞技公正性中的匹配公平使用**期望胜率均衡模型**，其核心指标是每场对局的队伍预测胜率偏差 $\Delta$。理想状态下 $\Delta = 0$，即双方预测胜率各为50%。实际运营中将可接受阈值设为 $|\Delta| \leq 4\%$，超出此范围的对局被标记为"失配对局"。赛季运营健康度报告通常要求赛季全周期内失配率低于8%。

预测胜率的计算依赖 Elo 评分系统的扩展变体，基础公式为：

$$
E_A = \frac{1}{1 + 10^{(R_B - R_A)/400}}
$$

其中 $R_A$ 和 $R_B$ 分别为两支队伍的综合 Elo 分，$E_A$ 为队伍 A 的期望胜率。当组队人数为5v5时，队伍综合分取成员个人分的加权平均，权重通常向近期20局胜率倾斜（权重系数约0.6）。失配率的赛季周期统计公式如下：

$$
\text{失配率} = \frac{\sum_{i=1}^{N} \mathbf{1}[|\Delta_i| > 4\%]}{N} \times 100\%
$$

其中 $N$ 为赛季内总对局数，$\mathbf{1}[\cdot]$ 为指示函数。

当赛季中期出现**段位通货膨胀**（即大量玩家积分快速上涨导致高段位池人数暴增）时，匹配系统需动态收窄匹配范围，将等待时间容忍上限从默认的90秒压缩至60秒，以避免匹配系统因等待超时而被迫接受 $|\Delta|>4\%$ 的对局。《英雄联盟》2023赛季S13期间曾因赛季初两周铂金段位人数超出预期47%，触发了此类动态收窄机制，最终将失配率从峰值11.2%压回至7.8%。

### 代练打击的行为特征识别

代练（Boosting）行为在技术层面表现为账号的设备指纹、IP地址、操作习惯与历史数据之间的**统计断层**。检测维度至少涵盖以下四类信号：

**①操作精度突变**：玩家在被代练期间的 APM（每分钟操作次数）与其历史均值偏差超过40%，且技能命中率在5局内连续提升超过25个百分点，触发异常标记。

**②地理位置跳变**：同一账号在24小时内出现跨大时区（跨度≥5个时区）的登录记录，例如同一账号在北京 IP 与美国西海岸 IP 之间交替出现，且两次登录间隔不足3小时，视为高风险信号。

**③胜率曲线异常**：连续20局胜率超过75%且 KDA（击杀/死亡/助攻比）同步提升，以二项分布检验其自然达成概率，通常阈值设为 $p < 0.001$，即该段位玩家自然达成此成绩的概率低于千分之一方可标记。

**④账号交易行为溯源**：部分运营团队会在主流代练中介平台（含暗网论坛）部署**诱饵账号（Honeypot Account）**，即人工控制的中段位账号主动接触代练服务，完成真实交易流程后对涉事账号实施精准溯源封禁。2021年 Riot Games 的一次专项行动中，单批次诱饵账号操作共定位并封禁了北美赛区1,247个代练账号，覆盖其中83%的钻石以上段位代练服务者。

## 关键检测算法与代码示例

以下为一个简化的代练行为异常检测逻辑，展示如何通过统计阈值对账号近期表现打分：

```python
import numpy as np
from scipy import stats

def boosting_risk_score(account_history: dict) -> float:
    """
    计算账号代练风险评分 (0~100)
    account_history 包含:
      - 'apm_history': 历史APM列表 (近90局)
      - 'recent_apm': 最近20局APM列表
      - 'recent_winrate': 最近20局胜率 (0~1)
      - 'recent_kda': 最近20局KDA均值
      - 'ip_timezone_gap': 24h内最大时区跨度 (小时数)
    """
    score = 0.0

    # 1. APM 突变检测 (权重30)
    hist_mean = np.mean(account_history['apm_history'])
    recent_mean = np.mean(account_history['recent_apm'])
    apm_deviation = abs(recent_mean - hist_mean) / (hist_mean + 1e-6)
    if apm_deviation > 0.40:
        score += 30 * min(apm_deviation / 0.8, 1.0)

    # 2. 胜率异常检测 (权重35) — 二项分布单侧检验
    n_games = 20
    win_count = int(account_history['recent_winrate'] * n_games)
    # 假设该段位基准胜率为0.50
    p_value = stats.binom_test(win_count, n_games, p=0.50, alternative='greater')
    if p_value < 0.001:
        score += 35

    # 3. IP时区跳变 (权重20)
    if account_history['ip_timezone_gap'] >= 5:
        score += 20

    # 4. KDA异常提升 (权重15)
    if account_history['recent_kda'] > 4.0:
        score += 15

    return min(score, 100.0)

# 示例账号数据
sample = {
    'apm_history': [142, 138, 145, 140, 137] * 18,
    'recent_apm':  [215, 220, 218, 222, 219] * 4,
    'recent_winrate': 0.80,
    'recent_kda': 5.2,
    'ip_timezone_gap': 7
}
print(f"代练风险评分: {boosting_risk_score(sample):.1f} / 100")
# 输出: 代练风险评分: 100.0 / 100
```

该算法中，风险评分≥70分的账号进入人工复核队列，≥90分的账号直接触发Wave Ban待执行队列，与赛季结算节点同步封禁。

## 实际应用

**《英雄联盟》LP保护机制**：每当赛季内发生影响大规模对局公正性的服务器故障时，Riot会向受影响玩家补偿"LP保护"（League Points Protection），即在特定时间窗口（通常为故障后6小时）内，该玩家的段位积分不因对局失败扣减。2022赛季因服务器问题共触发3次LP保护，每次触发条件为单次故障影响玩家数超过50,000人，累计保护对局超过290万局。

**《PUBG》赛季积分回滚实践**：某赛季结算后，运营团队发现全服前500名排行榜中存在约23%的作弊账号。处理方案并非简单删除账号，而是执行**排名修复窗口（Rank Repair Window）**：将受污染账号的积分从榜单剔除后，对因名次上移而新晋奖励门槛的合法玩家补发赛季专属皮肤与称号，整个流程在赛季结束后7个工作日内完成，并通过官网公告向玩家公示修复前后的名次变动明细。

**移动端实名制与设备锁联动**：国内移动竞技游戏（如《王者荣耀》《和平精英》）自2020年起依据国家新闻出版署实名注册要求，将账号与身份证号码绑定，并引入**设备指纹锁（Device Fingerprint Lock）**机制：当同一身份证绑定账号在30天内被3台不同设备登录时，系统自动冻结排位功能并要求人脸识别二次验证。该机制于2021年Q3上线后，《王者荣耀》代练相关投诉工单量下降了约41%（腾讯游戏安全团队2021年度报告）。

**例如**：某代练工作室同时操控12个目标账号，设备池仅有4台手机，每台手机日均切换账号3次。设备指纹锁在检测到单设备账号切换频率异常（单设备24小时内切换≥3个不同实名账号）后，对全部涉及账号同步触发冻结，工作室因此一次性损失所有在途订单。

## 常见误区

**误区一：即时封禁优于延迟封禁**。许多运营新人认为，发现作弊应立即封号以减少污染。但即时封禁会暴露检测逻辑的边界特征，作弊工具厂商可以通过"试探性账号"测试哪些行为触发封禁、哪些安全，从而快速迭代规避版本。Wave Ban的延迟机制使作弊工具开发者无法获得实时反馈，检测逻辑的有效生命周期可从平均2周延长至6～8周。

**误区二：Elo系统本身保证公平**。Elo 分只能描述历史胜率期望，无法区分"该账号当前由谁操作"。一个被代练30局的账号 Elo 分虽已膨胀，但匹配系统仍会基于该
