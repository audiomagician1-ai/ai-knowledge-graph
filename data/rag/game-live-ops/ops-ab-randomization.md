# 随机分组

## 概述

随机分组（Random Assignment）是A/B测试中将玩家样本分配到实验组和对照组的核心操作，其本质是通过引入随机性来切断玩家特征与分组结果之间的系统性关联。在游戏运营场景下，随机分组确保了新道具定价、关卡难度调整或付费弹窗设计等实验变量的效果，不会被"高消费玩家恰好集中在实验组"这类非随机因素所污染。

随机分组方法在20世纪20年代由统计学家Ronald Fisher在农业实验中率先系统化，当时用于将田地随机分配到不同施肥方案。游戏运营将该方法引入数字产品场景后，最大区别在于玩家行为具有高度时序依赖性——同一玩家在不同游戏阶段的消费意愿可能相差3至10倍，这使得"随机"本身的实现方式远比农业实验复杂。

随机分组的质量直接决定A/B测试结论是否可信。若分组存在系统偏差，即使统计显著性p值低于0.05，结论也可能是伪阳性。以一款MMORPG为例，若实验组中高等级玩家（50级以上）比例比对照组高出15%，那么即便实验组在付费率上表现更好，也无法确定这是新功能还是玩家基础质量差异造成的。

## 核心原理

### 完全随机分组（Simple Randomization）

完全随机分组为每位进入实验的玩家独立分配一个随机数，通常通过对玩家唯一ID取哈希值再对分组数取模来实现。例如，将玩家UID执行MD5哈希后转为整数，对100取模，结果0–49进入A组，50–99进入B组。这种方法实现简单，但在样本量较小（通常低于1000人）时，可能导致实验组和对照组在关键指标上的初始基线出现偶然性偏差。具体而言，在500人实验中，完全随机可能产生两组付费率基线相差2至3个百分点的情况，而真实效果量（Effect Size）往往也在这一量级，造成信噪比严重下降。

### 分层随机分组（Stratified Randomization）

分层随机分组首先按照预设的分层变量将玩家池划分成若干子组（层），再在每层内部独立执行完全随机分组。游戏运营中常用的分层变量包括：玩家账龄（新手期0–7天 / 成长期8–30天 / 成熟期30天以上）、历史付费等级（R0 / 小R / 中R / 大R）、设备平台（iOS / Android）。分层随机的数学保证在于，若某变量与结果指标的相关系数r ≥ 0.3，分层后可将该变量造成的组间方差降低约r²比例。例如，付费等级与7日ARPU的相关系数约为0.6，分层后该变量贡献的组间方差降幅可达36%，显著提升实验检测功效（Statistical Power）。

### 聚类随机分组（Cluster Randomization）

聚类随机分组将玩家按照天然存在的社会单元（公会、战队、好友圈）划分为若干"簇"，以簇为单位进行随机分配。在具有强社交功能的游戏（如战略SLG或MMO）中，若对同一公会内的玩家同时分配到A/B两组，他们之间的聊天、资源共享和攻略交流会产生"实验污染"（Spillover Effect），导致两组行为趋同，削弱检测能力。聚类随机的代价是需要更大的样本量：有效样本量公式为 $n_{eff} = n / [1 + (m-1)\rho]$，其中 $m$ 为簇内平均玩家数，$\rho$ 为组内相关系数（Intra-cluster Correlation Coefficient，ICC）。若一个公会平均30人，ICC为0.05，则有效样本量仅为原始样本量的约40%，因此聚类随机要求实验设计阶段就必须基于样本量计算结果预留足够的簇数量。

### 偏差控制方法

除分组策略选择外，还需主动检测和控制几类典型偏差：

- **选择偏差（Selection Bias）**：仅对某时段登录玩家分组会引入活跃度偏差，应在连续多日的登录队列中进行分组
- **存活者偏差（Survivorship Bias）**：实验开始后流失的玩家若被排除分析，会使两组留存率均虚高，应按"意向性分析"（Intention-to-Treat）原则保留所有分配者
- **新奇效应（Novelty Effect）**：新功能上线初期玩家行为异常活跃，通常需要排除前3至7天的数据，以洗去因"第一次见到"产生的非稳态行为

## 实际应用

**案例：手游限时礼包价格测试**

某二次元手游计划测试将某限时礼包定价从30元调整为45元对购买转化率的影响。实验设计者首先依据样本量计算确定需要各组5000名活跃付费玩家。随后按历史付费等级（小R / 中R / 大R）进行三层分层随机，每层内以哈希取模方式各自50/50分组。分层后验证两组的7日ARPU基线差异从未分层时的±8%压缩至±1.2%，满足实验启动标准（组间基线差异不超过相对5%）。游戏内好友聊天不涉及礼包价格信息（污染风险低），因此无需使用聚类随机。

**案例：SLG游戏新手引导改版**

某策略游戏改版新手引导，实验对象为新注册玩家。由于新用户来源渠道（买量渠道A vs 自然量渠道B）与3日留存率相关系数高达0.45，对渠道进行分层随机分组，避免两组买量玩家比例不均导致留存率结论失真。

## 常见误区

**误区一：认为样本量越大，完全随机分组一定无偏**

部分运营人员认为只要实验样本量达到数万人，完全随机就足以平衡所有变量，无需分层。实际上，当某一分层变量与结果指标高度相关（r > 0.5）时，即使10万人规模的完全随机实验，该变量的组间不平衡仍可能产生足以影响决策的噪声。分层随机的收益与样本量无关，只与目标变量和分层变量的相关程度有关。

**误区二：聚类随机中将"簇数量"误算为"样本量"**

在公会实验中，将200个公会各100人等同于"20000人样本"是典型错误。由于公会内玩家行为高度相关（ICC通常在0.03–0.15之间），有效样本量远低于人头数。若使用普通样本量公式计算出需要10000人，对应的簇随机实验可能实际需要300–500个公会才能达到同等检测功效。

**误区三：分组后不做基线验证（AA Test）**

随机分组完成后，应在实验正式开始前运行1至3天的AA测试，即对两组施加完全相同的条件，验证核心指标（如ARPU、次日留存率）的组间差异是否在统计噪声范围内（p > 0.2）。跳过此步骤直接开始AB实验，一旦结果显著则无法判断显著性来自实验变量还是分组偏差。

## 知识关联

随机分组的设计依赖于**样本量计算**的输出结果：最小样本量决定了各层需要多少玩家、需要多少个聚类簇。选择分层变量时，需从样本量计算阶段已识别的高相关协变量中挑选（通常选取与主指标相关系数最高的2至3个变量），避免过度分层导致每层内样本量不足。

完成随机分组后，实验数据将进入**统计显著性**分析阶段。分组方式直接影响显著性检验的选择：完全随机分组通常使用独立样本t检验或卡方检验；分层随机分组的结果需要使用协方差分析（ANCOVA）或分层Z检验，以充分利用分层结构提升检验效能；聚类随机分组则需使用混合效应模型（Mixed-effects Model）或广义估计方程（GEE），以正确处理簇内观测值非独立的问题，否则会低估标准误、产生过多假阳性结论。
