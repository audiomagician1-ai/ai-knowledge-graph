# 云游戏平台

## 概述

云游戏平台是指通过远程服务器运行游戏逻辑与渲染，将压缩后的视频流传输至终端设备的发行渠道。与传统平台不同，云游戏平台上的玩家本地设备仅负责解码视频流和发送输入指令，游戏实际运行在数据中心的GPU集群上。对于游戏发行商而言，接入云游戏平台意味着游戏必须在服务商托管的硬件环境中稳定运行，而非玩家自有设备。

Xbox Cloud Gaming（前身为Project xCloud，2019年正式测试）和NVIDIA GeForce Now是目前主要的商业云游戏平台。前者属于微软Xbox Game Pass生态的组成部分，后者采用"自带游戏库"模式——玩家需已在Steam、Epic等平台拥有游戏所有权才能在GeForce Now上串流。这两种商业模式决定了游戏接入云游戏平台的路径和收益分配方式截然不同。

理解云游戏平台接入规则对发行商的重要性在于：云游戏不仅是一种分发渠道，更直接影响游戏的DRM授权逻辑、内购支付流程，以及已有平台协议中的权利授予范围。若合同中未明确授予"流媒体传输权"（streaming rights），在技术上将游戏部署到云游戏服务器可能构成授权违规。

## 核心原理

### Xbox Cloud Gaming的接入机制

Xbox Cloud Gaming要求游戏必须已加入Xbox Game Pass订阅目录，才能自动获得云游戏串流资格。微软对接入Xbox Cloud的游戏统一采用Azure数据中心的Xbox Series X刀片服务器运行，每台刀片服务器通过虚拟化技术同时支持多个游戏实例。开发者无需额外适配代码即可接入，但必须保证游戏在Xbox Series X规格（12 TFLOPS GPU，1TB NVMe SSD）上通过微软的云游戏兼容性认证测试，测试项目包括控制器输入延迟容忍度、网络断线重连逻辑，以及触摸屏虚拟按键适配（面向移动端玩家）。

### GeForce Now的"白名单"授权模式

GeForce Now与Xbox Cloud最大的区别在于其不向发行商提供分成，而是要求发行商主动将游戏加入白名单（opt-in）。NVIDIA在2021年因未经授权将部分游戏加入服务导致多家发行商（包括Activision Blizzard、Bethesda）大规模退出事件后，改变了早期默认接入政策，转为强制要求发行商书面授权。游戏开发商接入GeForce Now无需修改游戏本体，但需与NVIDIA签署GeForce Now合作协议，协议核心条款规定发行商保留原有平台的100%销售收益，NVIDIA通过向玩家收取订阅费（优先级会员约9.99美元/月）而非抽取游戏销售分成盈利。

### 内购支付的平台叠加问题

云游戏平台上的内购支付涉及"平台叠加"问题：玩家通过GeForce Now串流Steam上的游戏时，内购仍由Steam的支付系统处理，Steam正常抽取30%分成（或Valve新政后的25%/20%阶梯税率），GeForce Now对此不额外抽取佣金。但在Xbox Cloud Gaming上，游戏内购通过Xbox支付系统结算，微软统一收取30%平台税，与Xbox实体版/数字版一致。这意味着发行商接入这两个平台的内购净收益计算方式需要分别建模。

### 延迟与游戏设计的合规要求

云游戏串流的输入延迟通常在40-80毫秒之间（优质网络环境下），部分云平台明确要求游戏提供延迟补偿机制或降低对精确输入时序的依赖。Xbox Cloud Gaming的技术文档规定，游戏中不应存在需要玩家在16毫秒（单帧）内完成精确操作的必要交互（即"单帧输入"机制），否则将在云游戏环境中造成无法通过的游戏内容，影响平台兼容性评级。

## 实际应用

**发行合同的流媒体权利条款**：2020年前签订的许多游戏发行合同缺乏明确的"流媒体传输权"授予条款，导致发行商希望将旧版游戏上架GeForce Now时面临法律障碍。现代合同模板通常在权利授予条款中单独列出"云游戏和流媒体服务权利"，并约定是否需要为每个云平台单独授权。

**触摸屏适配的额外开发成本**：Xbox Cloud Gaming面向移动端用户提供虚拟触摸屏控制器覆盖层，但复杂操作的游戏需要开发者主动设计触摸布局。微软提供了Touch Adaptation Kit（TAK）工具，允许发行商通过JSON格式的配置文件自定义触摸控件位置与行为，无需修改游戏核心代码。未完成TAK适配的游戏在移动端显示为"仅控制器"，影响用户体验评分。

**多区域数据中心的内容合规**：云游戏平台的服务器部署在特定地区，NVIDIA GeForce Now在欧洲、北美和亚太分别设有数据中心。发行商若游戏在某区域有内容合规限制（如德国的暴力内容审查），需在GeForce Now合作协议中标注区域限制，防止通过云游戏绕过地区封锁。

## 常见误区

**误区一：接入GeForce Now不需要任何授权许可**
部分开发者错误认为GeForce Now只是帮助玩家"运行他们已购买的游戏"，因此无需发行商授权。但法律层面，NVIDIA在其服务器上运行游戏属于复制和传播行为，构成对发行商版权的使用，必须获得书面授权。2021年事件已明确确立了这一法律立场。

**误区二：云游戏接入会导致主机版销量下滑**
实际数据显示，Xbox Game Pass上的游戏（含云游戏访问权）上线首月的PC/主机销售额平均下降约约38%（微软2022年内部研究数据），但订阅收益补偿使总收益持平或上升。云游戏的核心受众是无法购买主机的移动端用户，与传统主机玩家群体重叠度有限。

**误区三：云游戏平台的DLC和扩展包会自动同步**
在Xbox Cloud Gaming上，基础游戏通过Game Pass提供后，DLC和扩展包并不自动包含在云游戏访问权中，除非DLC也单独加入了Game Pass目录。发行商有时误以为基础游戏接入即代表全部内容接入，导致玩家在云端无法访问已购DLC内容的客诉问题。

## 知识关联

本概念以**内购规则**为前提：理解Xbox Cloud和GeForce Now上的内购支付路径，必须先掌握各原生平台（Xbox、Steam）的内购分成结构，才能准确计算云游戏场景下的叠加收益模型。

本概念延伸至**平台特性利用**：云游戏平台提供了部分原生平台没有的特性，如Xbox Cloud的跨设备存档接力（玩家在手机上继续主机会话）和GeForce Now的RTX光线追踪渲染能力（即使玩家本地设备不支持RTX）。掌握云游戏接入规则后，发行商可以围绕这些独特特性设计差异化的营销策略和功能展示方案。
